合约分析

这个包提供了对 合约 执行的分析支持。

契约是执行不变量和产生良好错误消息的一种很好的机制,但是它们引入了运行时检查,这可能会带来巨大的成本。 合同分析器的目标是确定这些成本在哪里,并提供信息来帮助控制它们。

使用此工具的最简单方法是使用 raco contract-profile 命令,该命令以文件名作为参数, 并在该文件的 main 子模块(如果存在)或模块本身 (如果没有 main 子模块)上运行契约分析器。 该工具的输出如下所示。

 (require contract-profile) package: contract-profile

除了使用 raco contract-profile 之外,还可以通过编程方式调用契约分析器。 这允许分析程序的特定部分,并控制输出。

syntax

(contract-profile option ... body ...)

 
option = #:module-graph-view-file module-graph-view-file
  | #:boundary-view-file boundary-view-file
  | #:boundary-view-key-file boundary-view-key-file
  | #:report-space-efficient? report-space-efficient?
在标准输出中生成与 body 合同检查相关的性能成本报告。

具体来说,它显示了 body 运行时间中用于检查合约的比例, 并将该时间按合约进行分解,然后在使用它的不同合约值之间分解每个合约的成本。

如果 report-space-efficient? 是非假的,节省空间的合同在报告中特别标记。 当使用 raco contract-profile 时,可以使用 --report-space-efficient 标志来控制。

附加的可视化可按需使用,由指定其目标文件的关键字参数控制。参数 #f (默认值)禁用该可视化。

  • 模块图视图: 显示模块(节点)的图,以及运行 body 时它们之间的契约边界(边)。

    每个合约边界上的权值对应于检查在该边界上应用的合约所花费的时间。 用 Typed Racket 编写的模块显示为绿色,非类型化模块显示为红色。

    这些图形是使用 Graphviz 呈现的,并且只有在合同分析器可以定位到 Graphviz 安装时才可用。

    当使用 raco contract-profile 时,使用 --module-graph-view-file 标志进行控制。

  • 边界视图: 显示合同检查成本如何在合同功能之间分布的详细视图,并按合同边界进行分解。

    契约函数显示为矩形节点,根据检查其契约的成本着色。 边表示跨越契约边界并导致应用契约的函数调用。 这些边是从剖析信息中提取的,因此代表不完全信息。 因此,契约分析器有时无法确定契约函数的调用者。 跨边界调用收缩函数的非收缩函数显示为灰色椭球节点。 节点按模块组成集群。 每个节点报告其(与契约无关的)自身时间。 此外,契约函数节点列出了函数参与的契约边界,以及检查与每个边界相关的契约的成本。 由于篇幅的原因,完整的合约没有显示在图表上,而是用编号代替。 从数字到契约的映射在 boundary-view-key-file 中找到。

    这些图形是使用 Graphviz 呈现的,并且只有在合同分析器可以定位到 Graphviz 安装时才可用。

    当使用 raco contract-profile 时, 使用 --boundary-view-file 和 --boundary-view-key-file 标志进行控制。

Examples:
> (define/contract (sum* numbers)
    (-> (listof integer?) integer?)
    (for/fold ([total 0])
              ([n (in-list numbers)])
      (+ total n)))
> (contract-profile (sum* (range (expt 10 7))))

procedure

(contract-profile-thunk 
  thunk 
  [#:module-graph-view-file module-graph-view-file 
  #:boundary-view-file boundary-view-file 
  #:boundary-view-key-file boundary-view-key-file 
  #:report-space-efficient? report-space-efficient?]) 
 → any
  thunk : (-> any)
  module-graph-view-file : (or/c path-string #f) = #f
  boundary-view-file : (or/c path-string #f) = #f
  boundary-view-key-file : (or/c path-string #f) = #f
  report-space-efficient? : any/c = #f
类似于 contract-profile, 但作为一个函数,它需要一个 thunk 来作为参数。

Example:
> (contract-profile-thunk
    (lambda ()
      (sum* (range (expt 10 7)))))
