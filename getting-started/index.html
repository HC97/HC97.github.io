开始使用

要开始使用 Racket,请从网页上 下载 并安装它。 如果你是初学者或想使用图形环境来运行程序,请运行 DrRacket 可执行文件。 否则, racket 可执行文件将运行命令行的 "阅读-评价-打印-循环" (REPL) 。

在 Windows 中,你可以从 "开始" 菜单中的 Racket 条目启动 DrRacket。 在 Windows Vista 或更新版本中,你可以直接输入 DrRacket 。 你也可以从其文件夹中运行它,你可以在 Program Files → Racket → DrRacket 。

在 Mac OS 上,双击 DrRacket 图标。 它可能在你拖入 Applications 文件夹的 Racket 文件夹中。 如果你想使用命令行工具,Racket 的可执行文件在 Racket 文件夹的 "bin" 目录下 (如果你想设置 PATH 环境变量,你需要手动操作)。

在 Unix(包括 Linux)上,如果 drracket 可执行文件在你的路径中,可以直接从命令行中运行, 如果你在安装时选择了 Unix 风格的发行版,可能就是这种情况。 否则,导航到安装 Racket 发行版的目录, drracket 可执行文件将在 "bin" 子目录中。

如果你是编程新手,或者你有耐心读完一本教科书:

  • How to Design Programs, Second Edition 是最好的开始。

  • Continue: Web Applications in Racket 教程向你介绍了模块和构建网络应用。

  • Racket 指南 介绍了 Racket 语言的其余部分, 它比教科书中面向学习的语言要大得多。 由于你从教科书中学习了函数式编程,你就可以略过指南的第 1 和第 2 章。

如果你已经是一个程序员,而且你比较着急:

  • Quick: An Introduction to Racket with Pictures 让你尝到了 Racket 的甜头。

  • 更多: 用 Racket 进行系统编程 更深入、更快速地探讨了 Racket。 如果觉得太多,就跳到指南中去。

  • Racket 指南 从 Racket 基础知识的教程开始, 然后介绍了 Racket 语言的其他内容。

当然,你可以自由地混合和匹配上述两个轨迹,因为每个轨迹都有另一个轨迹所没有的信息。
