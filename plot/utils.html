7.1 格式化

procedure

(digits-for-range x-min    
  x-max    
  [base    
  extra-digits]) → exact-integer?
  x-min : real?
  x-max : real?
  base : (and/c exact-integer? (>=/c 2)) = 10
  extra-digits : exact-integer? = 3
给定一个范围,返回区分该范围内数字所需的小数位数。对于大范围,可能会返回负数。

Examples:
> (digits-for-range 0.01 0.02)

5

> (digits-for-range 0 100000)

-2

procedure

(real->plot-label x digits [scientific?]) → string?

  x : real?
  digits : exact-integer?
  scientific? : boolean? = #t
将一个实数转换为绘图标签。用于格式化轴刻度线标签、 point-label 和图例条目中的数字。

Examples:
> (let ([d  (digits-for-range 0.01 0.03)])
    (real->plot-label 0.02555555 d))

".02556"

> (real->plot-label 2352343 -2)

"2352300"

> (real->plot-label 1000000000.0 4)

"1×10⁹"

> (real->plot-label 1000000000.1234 4)

"(1×10⁹)+.1234"

procedure

(ivl->plot-label i [extra-digits]) → string?

  i : ivl?
  extra-digits : exact-integer? = 3
将一个区间转换为一个绘图标签。

如果 i = (ivl x-min x-max), 当两个端点都是 rational? 时, 使用的数字数是 (digits-for-range x-min x-max 10 extra-digits) 。 否则,它是未指定的,但可能仍是 15 。

Examples:
> (ivl->plot-label (ivl -10.52312 10.99232))

"[-10.52,10.99]"

> (ivl->plot-label (ivl -inf.0 pi))

"[-inf.0,3.141592653589793]"

procedure

(->plot-label a [digits]) → string?

  a : any/c
  digits : exact-integer? = 7
将一个 Racket 值转换为一个标签。 被 discrete-histogram 和 discrete-histogram3d 使用。

procedure

(real->string/trunc x e) → string?

  x : real?
  e : exact-integer?
和 real->decimal-string 一样,但是去掉任何尾部的零和任何尾部的小数点。

procedure

(real->decimal-string* x    
  min-digits    
  [max-digits]) → string?
  x : real?
  min-digits : exact-nonnegative-integer?
  max-digits : exact-nonnegative-integer? = min-digits
像 real->decimal-string 一样,但同时接受最大和最小的数字数。

Examples:
> (real->decimal-string* 1 5 10)

"1.00000"

> (real->decimal-string* 1.123456 5 10)

"1.123456"

> (real->decimal-string* 1.123456789123456 5 10)

"1.1234567891"

应用 (real->decimal-string* x min-digits) 得到的数值与 (real->decimal-string x min-digits) 相同。

procedure

(integer->superscript x) → string?

  x : exact-integer?
将一个整数转换为上标Unicode字符的字符串。

Example:
> (integer->superscript -1234567890)

"⁻¹²³⁴⁵⁶⁷⁸⁹⁰"

运行某些过时的 Windows XP 版本的系统对 4 和 4 以上的 Unicode 上标有困难。 因为 integer->superscript 被每个数字格式化函数用来格式化指数, 如果你有这样一个系统, Plot 显然不会正确地格式化所有带指数的数字(直到你更新它)。

7.2 取样

procedure

(linear-seq start    
  end    
  num    
  [#:start? start?    
  #:end? end?]) → (listof real?)
  start : real?
  end : real?
  num : exact-nonnegative-integer?
  start? : boolean? = #t
  end? : boolean? = #t
返回一个介于 start 和 end 之间的均匀间隔的实数列表。 如果 start? 是 #t, 这个列表就包括 start 。 如果 end? 是 #t, 列表中包括 end 。

这个函数在内部用于生成样本点。

Examples:
> (linear-seq 0 1 5)

'(0 1/4 1/2 3/4 1)

> (linear-seq 0 1 5 #:start? #f)

'(1/9 1/3 5/9 7/9 1)

> (linear-seq 0 1 5 #:end? #f)

'(0 2/9 4/9 2/3 8/9)

> (linear-seq 0 1 5 #:start? #f #:end? #f)

'(1/10 3/10 1/2 7/10 9/10)

> (define xs (linear-seq -1 1 11))
> (plot (lines (map vector xs (map sqr xs))))

procedure

(linear-seq* points    
  num    
  [#:start? start?    
  #:end? end?]) → (listof real?)
  points : (listof real?)
  num : exact-nonnegative-integer?
  start? : boolean? = #t
  end? : boolean? = #t
像 linear-seq, 但接受一个实数列表,而不是开始和结束。 #:start? 和 #:end? 关键字参数的作用与 linear-seq 相同。 这个函数不保证每个内部值都在返回的列表中。

Examples:
> (linear-seq* '(0 1 2) 5)

'(0 1/2 1 3/2 2)

> (linear-seq* '(0 1 2) 6)

'(0 2/5 4/5 6/5 8/5 2)

> (linear-seq* '(0 1 0) 5)

'(0 1/2 1 1/2 0)

procedure

(nonlinear-seq start    
  end    
  num    
  transform    
  [#:start? start?    
  #:end? end?]) → (listof real?)
  start : real?
  end : real?
  num : exact-nonnegative-integer?
  transform : axis-transform/c
  start? : boolean? = #t
  end? : boolean? = #t
生成一个小数的列表,如果使用 transform 进行变换,这些小数将是均匀间隔的。 这被用来为转换后的轴生成样本。

Examples:
> (linear-seq 1 10 4)

'(1 4 7 10)

> (nonlinear-seq 1 10 4 log-transform)

'(1.0 2.154434690031884 4.641588833612779 10.000000000000002)

> (parameterize ([plot-x-transform  log-transform])
    (plot (area-histogram sqr (nonlinear-seq 1 10 4 log-transform))))

procedure

(kde xs h [ws]) → 
(-> real? real?)
(or/c rational? #f)
(or/c rational? #f)
  xs : (listof real?)
  h : (>/c 0)
  ws : (or/c (listof (>=/c 0)) #f) = #f
给出可选择的加权样本和内核带宽,返回一个代表内核密度估计值的函数和边界,在边界之外密度估计值为零。 由 density 使用。

procedure

(silverman-bandwidth xs) → real?

  xs : (listof real?)
返回给定样本的 Silverman Bandwidth 估计器。 这是 violin 渲染器使用的默认带宽。 更多细节参见维基百科上的 kernel density estimation 一文。

Added in version 8.5 of package plot-lib.

7.3 图形颜色和样式

procedure

(color-seq c1 
  c2 
  num 
  [#:start? start? 
  #:end? end?]) 
 → (listof (list/c real? real? real?))
  c1 : color/c
  c2 : color/c
  num : exact-nonnegative-integer?
  start? : boolean? = #t
  end? : boolean? = #t
在颜色之间进行插值–红色、绿色和蓝色成分分别–使用 linear-seq 。 #:start? 和 #:end? 关键字参数的作用与 linear-seq 相同。

Example:
> (plot (contour-intervals (λ (x y) (+ x y)) -2 2 -2 2
                           #:levels 4 #:contour-styles '(transparent)
                           #:colors (color-seq "red" "blue" 5)))

procedure

(color-seq* colors 
  num 
  [#:start? start? 
  #:end? end?]) 
 → (listof (list/c real? real? real?))
  colors : (listof color/c)
  num : exact-nonnegative-integer?
  start? : boolean? = #t
  end? : boolean? = #t
使用 linear-seq* 在红、绿、蓝三色分量之间分别插值。 参数 #:start? 和 #:end? 的作用与 linear-seq 相同。

Example:
> (plot (contour-intervals (λ (x y) (+ x y)) -2 2 -2 2
                           #:levels 4 #:contour-styles '(transparent)
                           #:colors (color-seq* '(red white blue) 5)))

procedure

(->color c) → (list/c real? real? real?)

  c : color/c
将一个非整数的绘图颜色转换为 RGB 三联体。

符号被转换为字符串,字符串在 color-database<%> 中被查找。 列表是不变的, color% 对象直接转换。

Examples:
> (->color 'navy)

'(36 36 140)

> (->color "navy")

'(36 36 140)

> (->color '(36 36 140))

'(36 36 140)

> (->color (make-object color% 36 36 140))

'(36 36 140)

这个函数不把整数转换为 RGB 三联体,因为它没有办法知道这个颜色是用于笔还是用于刷。 使用 ->pen-color 和 ->brush-color 来转换整数。

procedure

(->pen-color c) → (list/c real? real? real?)

  c : plot-color/c
将 line 颜色转换为 RGB 三联体。 整数颜色在当前的 plot-pen-color-map 中查找,非整数颜色使用 ->color 转换。 当整数颜色大于颜色地图中的颜色数量时,它将被包围。

Examples:
> (equal? (->pen-color 0) (->pen-color 8))

#f

> (plot (contour-intervals
         (λ (x y) (+ x y)) -2 2 -2 2
         #:levels 7 #:contour-styles '(transparent)
         #:colors (map ->pen-color (build-list 8 values))))

上面的例子是使用内部颜色地图, plot-pen-color-map 设置为 #f 。

procedure

(->brush-color c) → (list/c real? real? real?)

  c : plot-color/c
将一个 fill 颜色转换为 RGB 三联体。 整数颜色在当前的 plot-brush-color-map 中查找,非整数颜色使用 ->color 转换。 当整数颜色大于颜色图谱中的颜色数量时,它将被包围起来。

Examples:
> (equal? (->brush-color 0) (->brush-color 8))

#f

> (plot (contour-intervals
         (λ (x y) (+ x y)) -2 2 -2 2
         #:levels 7 #:contour-styles '(transparent)
         #:colors (map ->brush-color (build-list 8 values))))

上面的例子是使用内部颜色地图,plot-brush-color-map 被设置为 #f 。 在这个例子中,在列表上 map ->brush-color 实际上是不必要的, 因为 contour-intervals 使用 ->brush-color 来转换填充颜色。

function-interval 函数通常使用填充色绘制区域,使用线色绘制线条。 这两种颜色的默认值都是 3 。 下面的例子颠覆了默认行为,即用 line 色 3 绘制区域, 用 fill 色 3 绘制线条:
> (plot (function-interval sin (λ (x) 0) -4 4
                           #:color (->pen-color 3)
                           #:line1-color (->brush-color 3)
                           #:line2-color (->brush-color 3)
                           #:line1-width 4 #:line2-width 4))

procedure

(->pen-style s) → symbol?

  s : plot-pen-style/c
将符号笔样式或数字转换为符号笔样式。 符号是不变的。整数笔的样式从 5 开始重复。

Examples:
> (eq? (->pen-style 0) (->pen-style 5))

#t

> (map ->pen-style '(0 1 2 3 4))

'(solid dot long-dash short-dash dot-dash)

procedure

(->brush-style s) → symbol?

  s : plot-brush-style/c
将一个符号画笔样式或一个数字转换为一个符号画笔样式。 符号没有变化。整数笔刷样式从 7 开始重复。

Examples:
> (eq? (->brush-style 0) (->brush-style 7))

#t

> (map ->brush-style '(0 1 2 3))

'(solid bdiagonal-hatch fdiagonal-hatch crossdiag-hatch)

> (map ->brush-style '(4 5 6))

'(horizontal-hatch vertical-hatch cross-hatch)

procedure

(color-map-names) → (listof symbol?)

返回可用于 plot-pen-color-map 和 plot-brush-color-map 的颜色地图名称列表。

Added in version 7.3 of package plot-lib.

procedure

(color-map-size name) → integer?

  name : symbol?
返回颜色地图 name 中的颜色数量。 如果 name 不是有效的颜色地图名称,该函数将发出错误信号。

Added in version 7.3 of package plot-lib.

procedure

(register-color-map name color-map) → void

  name : symbol?
  color-map : (vectorof (list byte? byte? byte?))
注册一个新的颜色地图 name ,颜色是一个 RGB 三联体的向量。 如果这个名字的颜色地图已经存在,它将被替换。

Added in version 7.3 of package plot-lib.

7.4.1 实数函数

procedure

(polar->cartesian θ r) → (vector/c real? real?)

  θ : real?
  r : real?
将二维极坐标转换为二维软轴坐标。

procedure

(3d-polar->3d-cartesian θ ρ r) → (vector/c real? real? real?)

  θ : real?
  ρ : real?
  r : real?
将三维极坐标转换为三维笛卡尔坐标。 参见 parametric3d 的使用实例。

procedure

(ceiling-log/base b x) → exact-integer?

  b : (and/c exact-integer? (>=/c 2))
  x : (>/c 0)
像 (ceiling (/ (log x) (log b))), 但是 ceiling-log/base 不容易受到浮点误差的影响。

Examples:
> (ceiling (/ (log 100) (log 10)))

2.0

> (ceiling-log/base 10 100)

2

> (ceiling (/ (log 1/1000) (log 10)))

-2.0

> (ceiling-log/base 10 1/1000)

-3

各种数字格式化的函数都使用这个。

procedure

(floor-log/base b x) → exact-integer?

  b : (and/c exact-integer? (>=/c 2))
  x : (>/c 0)
像 (floor (/ (log x) (log b))), 但是 floor-log/base 不容易出现浮点错误。

Examples:
> (floor (/ (log 100) (log 10)))

2.0

> (floor-log/base 10 100)

2

> (floor (/ (log 1000) (log 10)))

2.0

> (floor-log/base 10 1000)

3

这是对 order-of-magnitude 的一种概括。

procedure

(maybe-inexact->exact x) → (or/c rational? #f)

  x : (or/c rational? #f)
如果 x 是 #f ,返回 #f; 否则 (inexact->exact x) 。 使用这个方法可以将可能是 #f 的区间端点转换为精确数字。

procedure

(min* r ...) → real?

  r : real?

procedure

(max* r ...) → real?

  r : real?
计算一串数字的 min 或 max 。

Examples:
> (min* 3 5 4 2 1)

1

> (max* 3 5 4 2 1)

5

> (min*)

+inf.0

> (max*)

-inf.0

procedure

procedure

  v : (vectorof real?)
  c : real?

procedure

(v/ v c) → (vectorof real?)

  v : (vectorof real?)
  c : real?
相当于 vector-map 在向量上的算术运算,但专门用于 2 和 3 向量的运算,速度快得多。

Examples:
> (v+ #(1 2) #(3 4))

'#(4 6)

> (v- #(1 2) #(3 4))

'#(-2 -2)

> (vneg #(1 2))

'#(-1 -2)

> (v* #(1 2 3) 2)

'#(2 4 6)

> (v/ #(1 2 3) 2)

'#(1/2 1 3/2)

procedure

(v= v1 v2) → boolean?

  v1 : (vectorof real?)
  v2 : (vectorof real?)
像 equal? 专门针对数字向量,但使用 = 来比较元素。

Examples:
> (equal? #(1 2) #(1 2))

#t

> (equal? #(1 2) #(1.0 2.0))

#f

> (v= #(1 2) #(1.0 2.0))

#t

procedure

(vcross v1 v2) → (vector/c real? real? real?)

  v1 : (vector/c real? real? real?)
  v2 : (vector/c real? real? real?)
返回 v1 和 v2 的右手向量交叉积。

Examples:
> (vcross #(1 0 0) #(0 1 0))

'#(0 0 1)

> (vcross #(0 1 0) #(1 0 0))

'#(0 0 -1)

> (vcross #(0 0 1) #(0 0 1))

'#(0 0 0)

procedure

(vcross2 v1 v2) → real?

  v1 : (vector/c real? real?)
  v2 : (vector/c real? real?)
返回边长为 v1 和 v2 的二维平行四边形的带符号面积。 相当于 (vector-ref (vcross (vector-append v1 #(0)) (vector-append v2 #(0))) 2) 但速度更快。

Examples:
> (vcross2 #(1 0) #(0 1))

1

> (vcross2 #(0 1) #(1 0))

-1

procedure

(vdot v1 v2) → real?

  v1 : (vectorof real?)
  v2 : (vectorof real?)
返回 v1 和 v2 的点积。

procedure

(vmag^2 v) → real?

  v : (vectorof real?)
返回 v 的平方大小,等同于 (vdot v v) 。

procedure

(vmag v) → real?

  v : (vectorof real?)
返回 v 的大小,相当于 (sqrt (vmag^2 v)) 。

procedure

(vnormalize v) → (vectorof real?)

  v : (vectorof real?)
返回一个与 v 相同方向的法向量。 如果 v 是一个零向量,返回 v 。

Examples:
> (vnormalize #(1 1 0))

'#(0.7071067811865475 0.7071067811865475 0)

> (vnormalize #(1 1 1))

'#(0.5773502691896258 0.5773502691896258 0.5773502691896258)

> (vnormalize #(0 0 0.0))

'#(0 0 0.0)

procedure

(vcenter vs) → (vectorof real?)

  vs : (listof (vectorof real?))
返回包含 vs 的最小包围盒的中心。

Example:
> (vcenter '(#(1 1) #(2 2)))

'#(3/2 3/2)

procedure

7.4.3 区间和区间函数

struct

(struct ivl (min max)
    #:extra-constructor-name make-ivl)
  min : real?
  max : real?
代表一个封闭区间。

一个有两个实值端点的区间总是按顺序包含端点:
> (ivl 0 1)

(ivl 0 1)

> (ivl 1 0)

(ivl 0 1)

一个区间可以有无限的端点:
> (ivl -inf.0 0)

(ivl -inf.0 0)

> (ivl 0 +inf.0)

(ivl 0 +inf.0)

> (ivl -inf.0 +inf.0)

(ivl -inf.0 +inf.0)

返回矩形渲染器的函数,如 rectangles 和 discrete-histogram3d, 接受 ivl 的向量作为参数。 ivl 结构类型也由 plot 提供, 因此这类渲染器的用户不必要求 plot/utils 。

procedure

(rational-ivl? i) → boolean?

  i : any/c
如果 i 是一个区间,并且其每个端点都是 rational?,则返回 #t 。

Example:
> (map rational-ivl? (list (ivl -1 1) (ivl -inf.0 2) 'bob))

'(#t #f #f)

procedure

(bounds->intervals xs) → (listof ivl?)

  xs : (listof real?)
给出一个点的列表,返回每对点之间的区间。

用它来构建 rectangles 和 rectangles3d 的输入。

Example:
> (bounds->intervals (linear-seq 0 1 5))

(list (ivl 0 1/4) (ivl 1/4 1/2) (ivl 1/2 3/4) (ivl 3/4 1))

7.5 日期和时间

procedure

(datetime->real x) → real?

  x : (or/c plot-time? date? date*? sql-date? sql-time? sql-timestamp?)
在尊重时区偏移的情况下,将各种日期/时间表示法转换为 UTC 秒。

对于日期,返回的值是自 系统相关的 UTC 纪元以来的秒数 。 更多信息请参见 date-ticks 。

要使用从 SQL 数据库中提取的日期绘制时间序列, 只需将相关的轴刻度(可能是 plot-x-ticks) 设置为 date-ticks, 并在传递给 lines 之前使用 datetime->real 将日期转换为秒。 为了防止时区偏移影响绘图,首先将它们设置为 0 。

struct

(struct plot-time (second minute hour day)
    #:extra-constructor-name make-plot-time)
  second : (and/c (>=/c 0) (</c 60))
  minute : (integer-in 0 59)
  hour : (integer-in 0 23)
  day : exact-integer?
一个考虑到天数、负数时间(使用负数天数)和小数秒的时间表示。

Plot (特别是 time-ticks) 在内部使用 plot-time 来格式化时间, 但由于渲染器产生的函数只需要实值,用户代码不应该需要它。我们提供这个功能只是为了以防万一。

procedure

(plot-time->seconds t) → real?

  t : plot-time?

(seconds->plot-time s) → plot-time?

将 plot-time 转换为实数秒,反之亦然。

Examples:
> (define (plot-time+ t1 t2)
    (seconds->plot-time (+ (plot-time->seconds t1)

> (plot-time+ (plot-time 32 0 12 1)
              (plot-time 32 0 14 1))

(plot-time 4 1 2 3)

7.6 图形指标

interface

plot-metrics<%> : interface?

plot-metrics<%> 接口允许获取由 plot, plot-snip, plot-bitmap, plot/dc 以及它们的三维变体 plot3d, plot3d-snip, plot3d-bitmap 和 plot3d/dc 返回的图上的绘图区域位置。 所有由这些函数返回的绘图对象都实现了这个接口。
对于由 plot-pict 和 plot3d-pict, 创建的绘图, 有一组单独的函数提供同样的功能,例如,见 plot-pict-bounds 。

method

(send a-plot-metrics get-plot-bounds)

 → (vectorof (vector/c real? real?))
以最小值和最大值向量的形式返回绘图的边界,绘图中的每个轴都有一个边界。 对于 2D 绘图,该方法返回一个由两个元素组成的向量,用于 X 轴和 Y 轴, 而 3D 绘图返回一个由三个元素组成的向量,用于 X、Y 和 Z 轴。

返回的值是绘图坐标,要获得绘图表面的坐标(即图像坐标), 请对这些边界使用 plot->dc 。

交互式绘图的边界,像那些由 plot 和 plot-snip 产生的绘图, 可以随着用户放大和缩小绘图而改变, get-plot-bounds 总是返回绘图的当前边界, 但它们可能被用户的操作所废止。

method

(send a-plot-metrics plot->dc coordinates) → (vectorof real?)

  coordinates : (vectorof real?)
将 coordinates 从绘图坐标系转换为绘图坐标系(即图像坐标)。 对于二维绘图, coordinates 是一个包含两个值的向量,即绘图上的X和Y坐标, 而对于三维绘图,它是一个包含三个值的向量,即 X、Y 和 Z 坐标。

例如,这种方法可以用来确定图像上坐标 0, 0 的实际位置。 它还可以用来确定图像中绘图区域的位置,通过调用 get-plot-bounds 返回的绘图边界。

对于交互式绘图,坐标可能会随着用户放大或缩小绘图而改变。

method

(send a-plot-metrics dc->plot coordinates) → (vectorof real?)

  coordinates : (vectorof real?)
对于2D绘图,该方法返回与输入的 coordinates 相对应的 2D 绘图坐标,这些坐标在绘图上下文坐标系中。

对于三维绘图,该方法返回垂直于用户视图的平面上的三维位置。 与 plane-vector 返回的该平面的法向量一起,可以重建投影线。

这是与 plot->dc 相反的操作,同样适用于用户放大和缩小绘图的说法。

method

(send a-plot-metrics plane-vector) → (vectorof real?)

返回代表通过绘图原点的屏幕法线的单位向量。 对于 2D 绘图,它总是返回 #(0 0 1), 对于3D绘图,这个单位向量可用于从绘图背景坐标中重建绘图坐标。

对于交互式 3D 绘图,如果用户旋转绘图,返回值会发生变化。

Added in version 8.1 of package plot-lib.
