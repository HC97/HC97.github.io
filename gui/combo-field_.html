class

combo-field% : class?

  superclass: text-field%

combo-field% 对象是一个 text-field% 对象, 也类似于 choice% 对象,因为它在文本字段的右边有一个小的弹出按钮。 点击该按钮会弹出一个菜单,选择一个菜单项通常会将该项目复制到文本字段中。

如果 label 不是 #f, 它就被用作组合标签。 否则,组合不显示其标签。

如果 & 出现在 label 中,就会像 button% 一样被特别解析。

choices 指定了组合的弹出菜单的初始项目列表。 append 方法在菜单中添加一个新的项目,并有一个回调将被添加的项目安装到组合体的文本字段中。 get-menu 方法返回一个可以改变的菜单,以调整组合的菜单的内容和动作。

当用户改变组合中的文本或按下回车键时, callback 过程就会被调用 (而回车键不是由组合的框架或对话框处理的;见 on-traverse-char in top-level-window<%> )。 如果用户按了回车键,传递给回调的事件类型是 'text-field-enter, 否则是 'text-field 。

如果 init-value 不是 "", 文本项的最小宽度就会变成足以显示 init-value 的宽度。 否则,会选择一个内置的默认宽度。

如果 style 包括 'vertical-label, 那么 combo 会在控件上方创建标签; 如果 style 不包括 'vertical-label (可选择包括 'horizontal-label), 那么标签会在 combo 左侧创建。 如果 style 包含 'deleted, 那么 combo 就被创建为隐藏的,它不会影响其父辈的几何形状; combo 可以通过调用 parent 的 add-child 方法,在以后变成活动的。.

font 参数决定了控件的字体。 关于 enabled 参数的信息,请参见 window<%> 。 关于 horiz-margin 和 vert-margin 参数的信息,请参见 subarea<%> 。 关于 min-width, min-height, stretchable-width 和 stretchable-height 参数的信息,请参见 area<%> 。

method

(send a-combo-field append l) → void?

  l : label-string?
在组合的弹出菜单中添加一个新项目。 给定的标签用于项目的名称,项目的回调将标签安装到组合的文本区域。

method

(send a-combo-field get-menu) → (is-a?/c popup-menu%)

返回一个 popup-menu% ,当组合被点击时,该弹出菜单被有效地复制到组合的弹出菜单中。 只有菜单项的标签和回调被使用,启用状态、子菜单或分隔符被忽略。

method

(send a-combo-field on-popup event) → void?

  event : (is-a?/c control-event%)
Specification: 当用户点击组合的弹出按钮时被调用。 覆盖此方法以根据需要调整组合菜单的内容。

Default implementation: 什么都不做。
