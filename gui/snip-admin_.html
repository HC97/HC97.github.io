class

snip-admin% : class?

  superclass: object%

关于管理员的角色信息,请参见 管理员 。 snip-admin% 类从未被直接实例化。 大多数程序员甚至不会通过派生类来实例化它; 每个 text% 或 pasteboard% 对象都会创建自己的管理员。 然而,派生出这个类的一个新实例以在一个新的环境中显示剪报,可能是有用的。 另外,从一个拥有的剪贴画中调用一个现有的管理员的方法也可能是有用的。

要创建一个新的 snip-admin% 类,这里描述的所有方法都必须被重写。 它们都是由管理员的 snip 调用的。

因为一个 snip-admin% 对象通常拥有一个以上的 snip,许多方法需要一个 snip% 对象作为参数。

constructor

(new snip-admin%) → (is-a?/c snip-admin%)

创建一个(无用的)编辑器管理员。

method

(send a-snip-admin get-dc) → (or/c (is-a?/c dc<%>) #f)

获取一个适合于确定显示尺寸信息的绘图环境。 如果 snip 没有显示,则返回 #f 。

method

(send a-snip-admin get-editor)

 → (or/c (is-a?/c text%) (is-a?/c pasteboard%))
返回这个管理员所报告的编辑器(直接或间接)。

method

(send a-snip-admin get-view x y w h [snip]) → void?

  x : (or/c (box/c real?) #f)
  y : (or/c (box/c real?) #f)
  w : (or/c (box/c (and/c real? (not/c negative?))) #f)
  h : (or/c (box/c (and/c real? (not/c negative?))) #f)
  snip : (or/c (is-a?/c snip%) #f) = #f
Specification: 获取一个 snip 的可见区域的 location 和尺寸,以 snip 的坐标表示。 如果给定的 snip 不是由这个管理员管理的,则结果未定义。

如果 snip 不是 #f, 那么当前 snip 的可见区域被安装在 x, y, w 和 h 框中。 x 和 y 的值是相对于snip的左上角。 w 和 h 的值可能比剪贴画本身更大。

如果 snip 是 #f, 那么 snip 的顶层 display 的总可见区域将以编辑器坐标返回。 对 snip 使用 #f 类似于在 get-view in editor-admin% 中使用 #t 表示 full? 。

如果没有指定 snip,那么 snip 的编辑器的 location 和大小将被返回,而不是用编辑器坐标。

参见 get-view in editor-admin% 。

Default implementation: 用 0.0 填充所有方框。

method

(send a-snip-admin get-view-size w h) → void?

  w : (or/c (box/c (and/c real? (not/c negative?))) #f)
  h : (or/c (box/c (and/c real? (not/c negative?))) #f)
Specification: 获取管理员的 display 区域的可见尺寸。

如果 display 是一个编辑器的画布,也请参见 reflow-container 。

Default implementation: 用 0.0 填满所有的框。

method

(send a-snip-admin modified snip modified?) → void?

  snip : (is-a?/c snip%)
  modified? : any/c
Specification: 由 snip 调用,报告它的修改状态已经改变为修改或未修改。

Default implementation: 不做任何事情。

method

(send a-snip-admin needs-update snip    
  localx    
  localy    
  w    
  h) → void?
  snip : (is-a?/c snip%)
  localx : real?
  localy : real?
  w : (and/c real? (not/c negative?))
  h : (and/c real? (not/c negative?))
Specification: 由 snip 调用,请求 snip 的显示需要更新。 管理员决定何时实际更新 snip;snip 的 draw 方法最终被调用。

localx, localy, w 和 h 参数指定了一个需要更新的片段区域(在片段坐标中)。

如果给定的 snip 不是由这个管理员管理的,则不会发生更新。

Default implementation: 不做任何事情。

method

(send a-snip-admin popup-menu menu snip x y) → boolean?

  menu : (is-a?/c popup-menu%)
  snip : (is-a?/c snip%)
  x : real?
  y : real?
Specification: 在 display 中为该片段的编辑器打开一个弹出菜单。 如果弹出菜单成功,结果是 #t ,否则是 #f (与用户是否选择了弹出菜单中的项目无关)。

该菜单被放置在 snip 坐标的 x 和 y 处。

当菜单被弹出时,它的目标被设置为这个剪贴画编辑器的 display 中的顶级编辑器。 更多信息见 get-popup-target 。

Default implementation: 返回 #f 。

method

(send a-snip-admin recounted snip refresh?) → void?

  snip : (is-a?/c snip%)
  refresh? : any/c
Specification: 由一个狙击手调用,通知管理员指定的狙击手改变了它的 count 。 狙击手在改变其 count 后通常需要进行更新,但狙击手决定是否要立即进行更新。

如果 refresh? 不是 #f, 那么该狙击手就要求立即更新。 否则,最终也必须调用 needs-update 。

如果给定的 snip 不是由这个管理员管理的,该方法的调用将被忽略。

Default implementation: 什么都不做。

method

(send a-snip-admin release-snip snip) → boolean?

  snip : (is-a?/c snip%)
Specification: 请求释放指定的 snip。 如果这个管理员不是该狙击手的所有者,或者该狙击手不能被释放,那么将返回 #f 。 否则,将返回 #t ,并且该片段不再被拥有。

参见 release-snip in editor<%> 。

如果给定的 snip 不是由这个管理员管理的,那么结果是 #f 。

Default implementation: 返回 #f 。

method

(send a-snip-admin resized snip refresh?) → void?

  snip : (is-a?/c snip%)
  refresh? : any/c
Specification: 由一个 snip 调用,通知管理员指定的 snip 改变了它的显示尺寸。 一般来说,snip 在改变尺寸后需要进行更新,但是 snip 决定是否要立即进行更新。.

如果 refresh? 不是 #f, 那么 snip 就会要求立即更新, 就像调用 needs-update 一样。 否则, needs-update 最终也必须被调用。

如果给定的 snip 不是由这个管理员管理的,该方法的调用将被忽略。

Default implementation: 什么都不做。

method

(send a-snip-admin scroll-to snip    
  localx    
  localy    
  w    
  h    
  refresh?    
  [bias]) → boolean?
  snip : (is-a?/c snip%)
  localx : real?
  localy : real?
  w : (and/c real? (not/c negative?))
  h : (and/c real? (not/c negative?))
  refresh? : any/c
  bias : (or/c 'start 'end 'none) = 'none
Specification: 由 snip 调用,要求滚动,使给定区域可见。 一般来说,在滚动后需要更新 snip,但 snip 决定是否要立即更新。

localx, localy, w 和 h 参数指定了一个要通过滚动使之可见的 snip 区域(以 snip 坐标为单位)。

如果 refresh? 不是 #f, 那么编辑器就要求立即更新。

bias 参数是以下之一:

  • 'start — 如果范围不适合在可见区域内,显示左上角的区域

  • 'none — 没有特殊的滚动指令

  • 'end — 如果范围不适合在可见区域内,显示右下角区域

如果编辑器被滚动了,结果是 #t ,否则是 #f 。

如果给定的 snip 不是由这个管理员管理的,方法的调用会被忽略(结果是 #f) 。

Default implementation: 返回 #f 。

method

(send a-snip-admin set-caret-owner snip

Specification: 调用 thunk ,同时将光标改为观察光标: 调用 thunk,同时将当前事件空间中的所有窗口的光标改为观察光标。

method

(send a-snip-admin get-tabs [length    
  tab-width    
  in-units]) → (listof real?)
  length : (or/c (box/c exact-nonnegative-integer?) #f) = #f
  tab-width : (or/c (box/c real?) #f) = #f
  in-units : (or/c (box/c any/c) #f) = #f
Specification: Returns the current tab-position array as a list.

The length box is filled with the length of the tab array (and therefore the returned list), unless length is #f. The tab-width box is filled with the width used for tabs past the end of the tab array, unless tab-width is #f. The in-units box is filled with #t if the tabs are specified in canvas units or #f if they are specified in space-widths, unless in-units is #f.

Default implementation: 返回 null 。
