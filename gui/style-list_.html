class

style-list% : class?

  superclass: object%

一个 style-list% 对象包含一组 style<%> 对象,并维护它们之间的层次关系。 一个 style<%> 对象只能通过 style-list% 对象的方法来创建。 有一个全局的风格列表对象,即 the-style-list,但任何数量的独立列表都可以为单独的风格层次创建。 每个编辑器都创建它自己的私有风格列表。

更多信息见 风格 。

constructor

(new style-list%) → (is-a?/c style-list%)

根样式,命名为 "Basic", 被自动创建。

method

(send a-style-list basic-style) → (is-a?/c style<%>)

This method is final, so it cannot be overridden.
返回根样式。每个样式表都有自己的根样式。

关于 'GRacket:default-font-size preference 的信息, 也请参见 首选项 。

method

(send a-style-list convert style) → (is-a?/c style<%>)

  style : (is-a?/c style<%>)
转换 style,它可以来自另一个样式列表,到这个列表中的一个样式。 如果 style 已经在这个列表中,那么 style 将被返回。 如果 style 被命名,并且该名称的样式已经在这个列表中,那么将返回现有的命名样式。 否则,该样式将通过转换它的基本样式(如果 style 是一个连接样式,则转换样式), 然后在这个列表中创建一个新的样式。

method

(send a-style-list find-named-style name)

 → (or/c (is-a?/c style<%>) #f)
  name : string?
按名称查找一个样式。 如果找不到这样的样式,将返回 #f 。

method

(send a-style-list find-or-create-join-style base-style 
  shift-style) 
 → (is-a?/c style<%>)
  base-style : (is-a?/c style<%>)
  shift-style : (is-a?/c style<%>)
创建一个新的连接样式,或找到一个合适的现有样式。 返回的样式总是无名的。 更多信息请参见 风格 。

base-style 参数必须是这个样式列表中的一个样式。

method

(send a-style-list find-or-create-style base-style 
  delta) 
 → (is-a?/c style<%>)
  base-style : (is-a?/c style<%>)
  delta : (is-a?/c style-delta%)
创建一个新的派生样式,或者找到一个合适的现有样式。返回的样式总是无名的。 更多信息请参见 风格 。

base-style 参数必须是这个样式列表中的一个样式。 如果 base-style 不是一个连接样式,如果它没有名字, 如果它的 delta 可以用 delta (见 collapse in style-delta%), 那么折叠后的 delta 被用来代替 delta, base-style 的基本样式被用来代替 base-style; 这种折叠和基本样式的替换是递归进行的。

method

(send a-style-list forget-notification key) → void?

  key : any/c
参见 notify-on-change 。

key 参数是由 notify-on-change 返回的值。

method

(send a-style-list index-to-style i)

 → (or/c (is-a?/c style<%>) #f)
  i : exact-nonnegative-integer?
返回与给定索引相关的样式,如果索引错误,则返回 #f 。 参见 style-to-index 。

method

(send a-style-list new-named-style name 
  like-style) 
 → (is-a?/c style<%>)
  name : string?
  like-style : (is-a?/c style<%>)
创建一个新的命名样式,除非该名称已经被使用。

如果 name 已经被使用,那么 like-style 将被忽略,并返回与该名字相关的旧样式。 否则,将为 name 创建一个新的样式, 具有与 like-style 相同的特征(即相同的基本样式和相同的样式 delta 或 shift 样式)。

like-style 样式必须在这个样式列表中,否则,命名的样式将从基本样式派生出一个空的样式 delta。

method

(send a-style-list notify-on-change f) → any/c

  f : ((or/c (is-a?/c style<%>) #f) . -> . any)
将一个回调 f 附加到样式列表中。 每当一个样式被修改,回调 f 就会被调用。

通常情况下,一个样式的变化会触发其他几个派生样式的变化; 为了允许客户端在一个批次中处理所有的变化, #f 被传递给 f ,作为一组样式被处理后的变化样式。

notify-on-change 的返回值是一个不透明的键, 可以与 forget-notification 一起使用。

回调 f 替换任何它是 equal? 的回调,这有助于避免在重复注册的情况下出现多余的通知。 回调 f 只被弱保留(在 make-weak-box 的意义上), 但只要 f 所冒充的任何值是可达的,它就会被保留; 例如,如果 f 代表一个应用了契约的函数,那么只要原始(契约前)函数是可达的, f 就会被保留用于通知。 只要 notify-on-change 产生的不透明键是可及的,回调 f 也会被保留。

method

(send a-style-list begin-style-change-sequence) → void?

用 begin-style-change-sequence 和 end-style-change-sequence 对包含在 style-list% 中的样式进行括号修改, 以避免在样式修改过程中的额外工作。

对 begin-style-change-sequence 和 end-style-change-sequence 的调用可以任意嵌套; 在最后一次调用 end-style-change-sequence 之前, 对样式的改变不会传播到使用这个 style-list% 的 editor<%> ,多余的调用在这一点上被跳过。

Added in version 1.5 of package snip-lib.

method

(send a-style-list end-style-change-sequence) → void?

调用以匹配对 begin-style-change-sequence 的调用。

Added in version 1.5 of package snip-lib.

method

(send a-style-list number) → exact-nonnegative-integer?

返回列表中的样式数量。

method

(send a-style-list replace-named-style name 
  like-style) 
 → (is-a?/c style<%>)
  name : string?
  like-style : (is-a?/c style<%>)
与 new-named-style 类似,但如果名字已经被映射到一个样式,现有的映射将被替换。

method

(send a-style-list style-to-index style)

 → (or/c exact-nonnegative-integer? #f)
  style : (is-a?/c style<%>)
返回一个特定样式的索引。 一个样式的基本样式(以及转移样式,如果它是一个连接样式)的索引被保证低于该样式自己的索引。 (因此,根样式的索引总是 0 )。 每当一个新的样式被添加到列表中,或者另一个样式的基本样式或移位样式被改变,一个样式的索引就会改变。

如果给定的样式不在这个列表中,将返回 #f 。
