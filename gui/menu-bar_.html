class

menu-bar% : class?

  superclass: object%

  extends: menu-item-container<%>
一个 menu-bar% 对象是为一个特定的 frame% 对象创建的。 一个框架最多只能有一个菜单栏;当为一个已经有菜单栏的框架创建一个新的菜单栏时, an exn:fail:contract exception is raised 。

constructor

(new menu-bar% 
    [parent parent] 
    [[demand-callback demand-callback]]) 
 → (is-a?/c menu-bar%)
  parent : (or/c (is-a?/c frame%) 'root)
  demand-callback : ((is-a?/c menu-bar%) . -> . any)
   = (lambda (m) (void))
在指定的框架中创建一个菜单栏。 该菜单栏最初是空的。 如果提供 'root 作为 parent 节点,那么菜单栏只有在没有其他框架显示时才会被激活。 只有当 current-eventspace-has-menu-root? 返回 #t 时, 并且只有在之前没有创建过这样的菜单栏时,才允许使用 'root parent 父级, an exn:fail:contract exception is raised 。

demand-callback 程序是由默认的 on-demand 方法与对象本身一起调用的。

method

(send a-menu-bar enable enable?) → void?

  enable? : any/c
启用或禁用菜单栏(即其所有的菜单)。 每个菜单的 is-enabled? 方法仅在该菜单被特别禁用时返回 #f(除了菜单栏)。

method

(send a-menu-bar get-frame) → (or/c (is-a?/c frame%) 'root)

返回菜单栏的框架,如果菜单栏在没有显示其他框架的情况下被显示,则返回 'root 。

method

(send a-menu-bar is-enabled?) → boolean?

如果菜单栏被启用,返回 #t ,否则返回 #f 。
