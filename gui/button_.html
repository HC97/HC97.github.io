class

button% : class?

  superclass: object%

  extends: control<%>
每当一个按钮被用户点击时,该按钮的回调过程就会被调用。 当每个按钮被创建时,一个回调过程被作为初始化参数提供。

创建一个带有字符串标签、位图标签或两者的按钮。 If label 是 a bitmap, and 如果 the 位图有一个掩码(见 get-loaded-mask in bitmap%), 与该位图大小相同,那么该掩码将用于标签。当一个位图被用作标签时,对其进行修改会对显示的标签产生不明确的影响。 如果 label 是一个列表, 那么按钮就有一个位图和字符串标签,符号 'left, 'top, 'right 或 'bottom 指定图像相对于按钮上文本的位置。

如果 & 出现在 label 中(当 label 包括一个字符串时),它将被特别解析; 在 Windows 和 Unix 中, & 后面的字符在显示的控件中用下划线表示一个键盘助记符。 (在 Mac OS 上,不显示助记符的下划线。)下划线的助记符必须是一个字母或一个数字。 当控件的顶层窗口包含键盘焦点时,用户可以通过输入助记符有效地点击按钮。 如果键盘焦点目前在一个处理正常字母数字输入的控件中,用户还必须按住 Meta 或 Alt 键。 在为控件显示之前, & 本身会从 label 中删除; label 中的 && 会被转换为 &(没有助记符下划线)。 在 Mac OS 上,在显示标签之前,括号内的助记符会被删除(连同周围的任何空格),因为括号内的助记符通常用于非罗马语言。 最后,出于历史原因,在所有平台上,Tab 字符之后的任何文本都会被删除。 所有这些规则都与菜单项的标签处理一致(见 set-label)。 助记符键盘事件由 on-traverse-char 处理(但在Mac OS上不是)。

每当用户点击按钮时, callback 过程就会被调用(事件类型为 'button) 。

如果 style 包括 'border, 按钮会被画上一个特殊的边框, 向用户表明它是默认的动作按钮(见 on-traverse-char)。 如果 style 包括 'multi-line,那么按钮会以一种可以垂直伸展的方式绘制,并适应文本标签中的多行; 目前,这种样式只在 Mac OS 上有区别, 当 label 是一个包含 #\newline 或 #\return 的字符串时,它会被自动选择。 如果 style 包含 'deleted, 那么 button 就被创建为隐藏的,它不会影响其父辈的几何形状; button 可以通过调用 parent 的 add-child 方法,在以后变成活动的。

font 参数决定了控件的字体。 关于 enabled 参数的信息,请参见 window<%> 。 关于 horiz-margin 和 vert-margin 参数的信息,请参见 subarea<%> 。 关于 min-width, min-height, stretchable-width 和 stretchable-height 参数的信息,请参见 area<%> 。

Changed in version 1.47 of package gui-lib: 增加了 'multi-line 样式, 并使其在 label 包含 #\return 时被选中。

method

(send a-button set-label label) → void?

  label : 
(or/c label-string?
      (is-a?/c bitmap%))
Overrides set-label in window<%>.
当 label 是一个字符串时,与 set-label in window<%> 相同。

否则,设置位图按钮的位图标签。 由于 label 是 bitmap, 如果 位图有一个掩码(见 get-loaded-mask in bitmap%), 与该位图大小相同,那么该掩码将用于标签。当一个位图被用作标签时,对其进行修改会对显示的标签产生不明确的影响。 位图标签仅在以下情况下被安装，如果该控件最初是用位图标签创建的。

如果按钮同时有一个字符串和一个位图标签,那么可以用 set-label 来设置其中之一。
