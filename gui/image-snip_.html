class

image-snip% : class?

  superclass: snip%

image-snip% 是一个可以显示位图图像(通常从文件加载)的 snip。 当图像文件找不到时,会画出一个包含 "X" 的方框。

constructor

(make-object image-snip% [file 
  kind 
  relative-path? 
  inline? 
  backing-scale]) 
 → (is-a?/c image-snip%)
  file : (or/c path-string? input-port? #f) = #f
  kind : 
(or/c 'unknown 'unknown/mask 'unknown/alpha
      'gif 'gif/mask 'gif/alpha
      'jpeg 'png 'png/mask 'png/alpha
      'xbm 'xpm 'bmp 'pict)
 = 'unknown
  relative-path? : any/c = #f
  inline? : any/c = #t
  backing-scale : (>/c 0.0) = 1.0
(make-object image-snip% bitmap [mask]) → (is-a?/c image-snip%)
  bitmap : (is-a?/c bitmap%)
  mask : (or/c (is-a?/c bitmap%) #f) = #f
创建一个图像剪辑,如果指定了图像 file , 则加载图像文件(也见 load-file), 或使用给定的 bitmap 。

Changed in version 1.1 of package snip-lib: 增加了 backing-scale 参数。

method

(send an-image-snip load-file file    
  [kind    
  relative-path?    
  inline?    
  backing-scale]) → void?
  file : (or/c path-string? input-port? #f)
  kind : 
(or/c 'unknown 'unknown/mask 'unknown/alpha
      'gif 'gif/mask 'gif/alpha
      'jpeg 'png 'png/mask 'png/alpha
      'xbm 'xpm 'bmp 'pict)
 = 'unknown
  relative-path? : any/c = #f
  inline? : any/c = #t
  backing-scale : (>/c 0.0) = 1.0
通过将 file 和 kind 传给 load-file in bitmap% 来加载文件。 如果之前用 set-bitmap 指定了一个位图,该位图(和掩码)将不再被使用。 如果 file 是 #f, 那么当前图像被清除。

当指定了 'unknown/mask, 'gif/mask 或 'png/mask , 并且加载的位图对象包括一个掩码(见 get-loaded-mask), 该掩码将用于绘制位图(见 draw-bitmap)。 然而,推荐使用 'unknown/alpha, 'gif/alpha 或 'png/alpha 的变体。

如果 relative-path? 不是 #f ,并且 file 是一个相对路径, 那么将使用拥有编辑器的文件名的路径来读取该文件。 如果图片没有被内联,它将被保存为一个相对路径名。

如果 inline? 不是 #f, 当图像被保存或复制时, 图像数据将被直接保存到文件或剪贴板中(如果有的话,保留位图的掩码)。 源文件名和种类已不再重要。

Changed in version 1.1 of package snip-lib: 增加了 backing-scale 参数。

method

(send an-image-snip other-equal-to? snip    
  equal?) → boolean?
  snip : (is-a?/c snip%)
  equal? : (any/c any/c . -> . boolean?)
如果 an-image-snip 和 snip 都有位图且位图相同,返回 #t 。 如果两者都有一个与主位图相同尺寸的遮罩位图,那么遮罩必须是相同的(或者如果只有一个遮罩,它必须对应于一个实体遮罩)。

给定的 equal? 函数(用于递归比较)不被使用。

method

(send an-image-snip resize w h) → boolean?

  w : (and/c real? (not/c negative?))
  h : (and/c real? (not/c negative?))
Overrides resize in snip%.
位图将被裁剪以适应给定的尺寸。

method

(send an-image-snip set-bitmap bm [mask]) → void?

  bm : (is-a?/c bitmap%)
  mask : (or/c (is-a?/c bitmap%) #f) = #f
Sets the bitmap that is displayed by the snip.

在绘制位图时使用一个可选的 mask (见 draw-bitmap), 但直接提供掩码是不可取的。 如果没有提供遮罩,但位图的 get-loaded-mask 方法产生了一个相同尺寸的位图,它将被用作遮罩; 此外,当 snip 被保存到文件或被复制时,这样的遮罩也会被保存(而直接提供的遮罩不会被保存)。 然而,通常情况下, bm 应该有一个 alpha 通道,而不是一个单独的遮罩位图。
