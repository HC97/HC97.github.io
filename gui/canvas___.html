interface

canvas<%> : interface?

  implements: subwindow<%>
画布是一个子窗口,可以在上面绘制图形和文字。 画布也可以接收鼠标和键盘事件。

canvas<%> 接口是由两个类实现的:

  • canvas% — 一个用于任意绘图和事件处理的画布;和

  • editor-canvas% — 一个用于显示 editor<%> 对象的画布。

要在一个画布上作画,需要通过 get-dc 获得其设备上下文。 有两种更新画布的基本方法:

  • 绘制通常发生在画布的 on-paint 回调期间。 canvas% 类支持从默认的 on-paint 方法中调用一个 paint-callback 初始化参数。

    当窗口系统确定必须更新画布时,画布的 on-paint 方法会作为一个事件被自动调用, 例如当画布第一次被显示或被调整大小时。 使用 refresh 方法来明确地触发窗口系统的 on-paint 调用。 (在调用 on-paint 之前的多个刷新请求被联合为一个 on-paint 调用)。

    在窗口系统调用 on-paint 之前, 它可能会擦除画布的背景(见 erase), 这取决于画布的风格(例如,由 canvas% 的 style 初始化参数决定)。 即使画布的样式禁止明确清除画布,画布也可能因为窗口移动和调整大小的操作而被窗口系统擦除。 对于透明画布来说,"擦除" 意味着画布的父窗口会显示出来。

  • 绘画也可以在窗口系统的 on-paint 调用之外的任何时候发生, 包括从画布的事件空间的 handler thread 之外的线程。 在系统的 on-paint 回调之外的绘制是瞬时的, 因为视窗活动可以擦除画布,但只要不需要视窗刷新,绘制就是持久的。

    直接调用 on-paint 方法与在来自视窗系统的 on-paint 回调之外绘制是一样的。 对于 canvas% 来说,使用 refresh-now 来强制对 canvas 的内容进行即时更新, 这与用 refresh 排队更新的情况类似。

在画布的绘图上下文中绘图实际上是渲染到一个屏幕外的缓冲区。 该缓冲区会自动异步刷新到屏幕上,明确地通过 flush 方法刷新, 或者明确地通过 flush-display 刷新,除非该画布的刷新被禁用。 suspend-flush 方法会暂停画布的刷新, 直到匹配的 resume-flush 调用; 对 suspend-flush 和 resume-flush 的调用可以是嵌套的, 在这种情况下,刷新会暂停,直到最外层的 suspend-flush 被 resume-flush 平衡。 来自窗口系统的 on-paint 调用被隐含地包裹在 suspend-flush 和 resume-flush 调用中, 正如通过 refresh-now 对 paint 过程的调用一样。

在透明画布的情况下,线条和文字的平滑可能取决于作为画布背景的窗口。 例如,平滑可能会根据目标背景是白色还是灰色而对像素进行不同的着色。 然而,只有在画布的屏外缓冲区中记录了相对较少的绘图命令时,才支持对背景敏感的平滑。

method

(send a-canvas accept-tab-focus) → boolean?

(send a-canvas accept-tab-focus on?) → void?
  on? : any/c
获取 或设置画布是否启用了 Tab 焦点 (假设画布不是用 canvas% 的 'no-focus 样式创建的)。 当 tab 焦点被启用时,当用户用 Tab 和方向键在框架或对话框的控件中导航时,画布可以接收键盘焦点。 默认情况下,标签焦点是禁用的。

当 canvas% 对象启用 tab-focus 时, Tab、arrow、Enter 和 Escape 的键盘事件会被框架的默认 on-traverse-char 方法消耗。 (此外,对话框的默认方法会消耗 Escape 键事件。)否则, on-traverse-char 允许键盘事件被传播到 canvas。

对于 editor-canvas% 对象,Tab、箭头、Enter 和 Escape 键盘事件的处理由 allow-tab-exit 方法决定。

method

(send a-canvas get-canvas-background)

 → (or/c (is-a?/c color%) #f)
返回当前用于在调用 on-paint 之前 "擦除" 画布内容的颜色。 也请参见 set-canvas-background .

如果画布是用 'transparent 创建的,其结果是 #f , 否则它总是一个 color% 对象。

method

(send a-canvas get-scaled-client-size)

 → 
dimension-integer? dimension-integer?
返回画布的绘图区域的尺寸,以未缩放的像素为单位–也就是说,没有缩放(见 屏幕分辨率和文本缩放) , 这是对画布尺寸和内容隐含的应用。

例如,当 Mac OS 上的画布驻留在 Retina 显示器上时,它的背景比例为 2, 因此,从 get-scaled-client-size 得到的结果将是 get-client-size 的两倍。 如果同一个画布的框架被拖到非 Retina 屏幕上,它的背景比例可以变为 1, 在这种情况下, get-scaled-client-size 和 get-client-size 会产生相同的值。 画布的背景比例是否可以改变取决于平台。

get-scaled-client-size 报告的尺寸可能与 canvas% 实例中使用 'gl 样式的 OpenGL 绘图的视口尺寸一致。 然而,在 Mac OS 上,视口将与缩放的尺寸相匹配,除非画布是用 gl-config% 规范创建的, 并通过 set-hires-mode 调整为高分辨率模式。 请参阅 get-gl-client-size in canvas% 。

Added in version 1.13 of package gui-lib.

method

(send a-canvas min-client-height) → dimension-integer?

(send a-canvas min-client-height h) → void?
  h : dimension-integer?
获取或设置画布的最小高度,用于几何管理,基于客户的尺寸而不是全尺寸。 客户端高度是通过 min-height in area<%> 获得或改变的,适当地添加或减去边框和滚动条的尺寸。

当最小高度小于画布的 graphical minimum height 时,会被忽略。更多信息见 几何图形管理 。

method

(send a-canvas min-client-width) → dimension-integer?

(send a-canvas min-client-width w) → void?
  w : dimension-integer?
获取或设置画布的最小宽度,用于几何管理,基于画布的客户端尺寸而不是全尺寸。 客户端宽度是通过 min-width in area<%> 获得或改变的,适当地添加或减去边框和滚动条的尺寸。

当最小宽度小于画布的 graphical minimum width 时,会被忽略。 更多信息见 几何图形管理 。

method

(send a-canvas on-char ch) → void?

  ch : (is-a?/c key-event%)
Specification: 规范: 当画布收到一个键盘事件时被调用。参见 鼠标和键盘事件 。

Default implementation: 不做任何事情。

method

(send a-canvas on-event event) → void?

  event : (is-a?/c mouse-event%)
Specification: 当画布收到鼠标事件时被调用: 当画布收到一个鼠标事件时被调用。 参见 鼠标和键盘事件, 特别注意到某些鼠标事件会被丢弃。

Default implementation: 什么都不做。

method

(send a-canvas on-paint) → void?

Specification: 当画布被暴露或调整大小时被调用,以便画布中的图像可以被重新绘制。

当 on-paint 被调用以响应系统曝光事件,并且只有部分画布被新曝光时, on-paint 执行的任何绘图操作都被剪切到新曝光的区域; 然而,get-clipping-region 报告的剪切区域不会改变。

Default implementation: 什么都不做。

method

(send a-canvas on-tab-in) → void?

Specification: 当键盘焦点通过键盘导航事件进入画布时被调用。 在焦点改变时,也会像往常一样调用 on-focus 方法。 当键盘焦点由于导航事件而离开画布时,只调用 on-focus 。

也请参见 accept-tab-focus 和 on-traverse-char in top-level-window<%> 。

Default implementation: 什么都不做。

method

(send a-canvas resume-flush) → void?

参见 canvas<%> 以了解关于画布刷新的信息。

method

(send a-canvas set-canvas-background color) → void?

  color : (is-a?/c color%)
设置用于在调用 on-paint 之前 “erase” 画布内容的颜色。 (这个颜色通常与画布在低层次上相关联,因此,即使画布的完全刷新被其他活动延迟,也会使用这个颜色)。

如果画布是用 'transparent 创建的, an exn:fail:contract exception is raised 。

method

(send a-canvas set-resize-corner on?) → void?

  on? : any/c
在 Mac OS 上,当只有一个滚动条可见时,启用或禁用画布右下角的尺寸调整标签的空间。 这个方法在 Windows 或 Unix 上没有效果,而且当两个或没有滚动条都可见时,它也没有效果。 默认情况下,调整角是禁用的,但是当用 'resize-corner 样式创建画布时,它可以被启用。

method

(send a-canvas suspend-flush) → void?

请参阅 canvas<%> 了解关于画布刷新的信息。

请注意,在某些平台上,暂停画布的刷新会阻碍同一框架内其他窗口的刷新。
