9 WXME 解码

 (require wxme) package: wxme-lib
wxme 库提供了读取 WXME editor<%> 格式文件的工具 (见 文件格式) ,不需要 racket/gui 库。

procedure

(is-wxme-stream? in) → boolean?

  in : input-port?
从 in 中窥视,如果它以神奇的字节开始,则返回 #t , 表明是 WXME 格式的流(见 文件格式),否则返回 #f 。

procedure

(wxme-port->text-port in [close?]) → input-port?

  in : input-port?
  close? : any/c = #t
接收一个流以 WXME 格式数据开始的输入端口,并返回一个产生 WXME 内容的文本形式的输入端口, 就像在 DrRacket 中打开一个 WXME 文件并保存为文本的结果。

与 wxme-port->port 不同的是,这个函数可能会对 snips 进行随意的处理,从而使有效的程序失效。 例如,如果 wxme 流 in 包含一个位图图像,那么可能没有一个合理的纯文本版本, 因此 wxme-port->port 可能会把一个有效的 Racket 程序变成语法错误的文本, 尽管如此,其结果可能对人类读者或仅在无 GUI 环境下运行的近似程序处理工具仍然有用。

如果 close? 为真,那么关闭结果端口就会关闭原来的端口。

关于可以读取的非文本内容的种类,见 Snip 类映射 。

procedure

(wxme-port->port in [close? snip-filter]) → input-port?

  in : input-port?
  close? : any/c = #t
  snip-filter : (any/c . -> . any/c) = (lambda (x) x)
接收一个输入端口,其数据流以 WXME 格式的数据开始,并返回一个输入端口, 该端口产生的文本内容被转换为字节,非文本内容则是 "特殊" 值(见read-char-or-special) 。

这些由新的输入端口产生的特殊值与将文件读入 editor<%> 对象产生的特殊值不同。 特殊值通常是 object% 的简单扩展,而不是 snip% 的实例。 关于可以读取的非文本内容的种类,见 Snip 类映射 。

如果 close? 为真,那么关闭结果端口就会关闭原始端口。

snip-filter 程序被应用于为流产生的任何特殊值,其结果被用作替代的特殊值。

如果一个特殊值(可能是由过滤器程序产生的)是一个实现 readable<%> 接口的对象, 那么该对象的 read-special 方法被调用以产生特殊值。

procedure

(extract-used-classes in) → 
(listof string?) (listof string?)
  in : input-port?
返回两个值:一个由给定流使用的 snip 类名称的列表,和一个由流使用的数据类名称的列表。 如果流不是 WXME 流,结果是两个空列表。 给定的流不会被关闭,只有 WXME 流的数据(如果有的话)被消耗。

procedure

(register-lib-mapping! str mod-path) → void?

  str : string?
  mod-path : (cons/c 'lib (listof string?))
将一个 snip-class 名称映射到一个提供 reader% 实现的引号模块路径上。 模块路径必须具有 '(lib string ...) 的形式, 其中每个 string 只包含字母数字的 ASCII 字符 ., _, - 和空格。

procedure

(string->lib-path str gui?)

 → 
(or/c (cons/c 'lib (listof string?))
      #f)
  str : string?
  gui? : any/c
当 gui? 为 true 时,返回 str 的带引号的模块路径,用于 editor<%> 模式; 当 gui? 为 #f 时,返回 wxme 模式。 对于后者,使用内置映射和通过 register-lib-mapping! 注册的映射。 如果 str 不能被解析为库路径,并且没有可用的映射(因为类是内置的或者不知道),结果是 #f 。

parameter

(unknown-extensions-skip-enabled) → boolean?

(unknown-extensions-skip-enabled skip?) → void?
  skip? : any/c
一个参数。 当设置为#f(默认值)时,当在 WXME 流中遇到一个不被识别的 snip 类时,会引发一个异常。 当设置为真值时,未被识别的狙击类的实例将从转换后的流中省略。

parameter

(broken-wxme-big-endian?) → boolean?

(broken-wxme-big-endian? big?) → void?
  big? : any/c
一个参数。 一些老的和短命的 WXME 格式取决于保存文件的机器的 endian 顺序。 设置这个参数可以在读取文件时选择使用的 endian 顺序;默认是当前平台的 endian 顺序。

procedure

(wxme-read in) → any/c

  in : input-port?
与 read 类似,但对于以 WXME 格式数据开始的流。 如果 WXME 数据中有多个 S 表达式,它们都会被读取并与 'begin 组合。

如果 racket/gui/base 是可用的(由 gui-available? 决定), 那么会使用 open-input-text-editor 。 否则,使用 wxme-port->port 。

procedure

(wxme-read-syntax source-v in) → (or/c syntax? eof-object?)

  source-v : any/c
  in : input-port?
与 read-syntax 类似,但是对于 WXME 格式的输入流。 如果 WXME 数据中有多个 S 表达式,它们将被全部读取并与 'begin 组合。

如果 racket/gui/base 是可用的(由 gui-available? 决定), 那么会使用 open-input-text-editor 。 否则,会使用 wxme-port->port 。

interface

snip-reader<%> : interface?

一个由阅读器实现的接口,用于读取 WXME 流中特定种类的数据。 该接口有两个方法: read-header 和 read-snip 。

method

(send a-snip-reader read-header version    
  stream) → any
  version : exact-nonnegative-integer?
  stream : (is-a?/c stream<%>)
每个 WXME 流最多被调用一次,以初始化数据类型的流的特定信息。这个方法通常不做任何事情。

method

(send a-snip-reader read-snip text-only?    
  version    
  stream) → 
(if text-only?
    bytes?
    any/c)
  text-only? : boolean?
  version : exact-nonnegative-integer?
  stream : (is-a?/c stream<%>)
当在流中遇到一个数据类型的实例时被调用。 这个方法读取数据并返回字节,作为解码后的流的一部分,或者任何其他类型的值,作为解码后的流的 "特殊" 值返回。 结果值可以选择是一个实现 readable<%> 的对象。

当调用 wxme-port->text-port 时, text-only? 参数是 #f , 当调用 wxme-port->port 时是 #t 。

interface

readable<%> : interface?

一个由狙击手返回的值实现的接口。 唯一的方法是 read-special 。

method

(send a-readable read-special source    
  line    
  column    
  position) → any/c
  source : any/c
  line : (or/c exact-nonnegative-integer? #f)
  column : (or/c exact-nonnegative-integer? #f)
  position : (or/c exact-nonnegative-integer? #f)
与 read-special 类似,但用于非图形模式。 当一个值实现了这个接口,它的 read-special 方法会被调用, 并带有源位置信息,以便从 WXME 解码端口获得 "特殊" 结果。

interface

stream<%> : interface?

代表一个 WXME 输入流,供 snip-reader<%> 实例使用。

method

(send a-stream read-integer what)

 → 
(or/c (integer-in (- (expt 2 31)) (expt 2 31))
      (and/c real? inexact?))
  what : any/c
和 get-exact 一样,返回一个精确的整数或一个不精确的实数。

what 字段描述了正在读取的内容,以备错误信息,以防流中没有继续输入整数。

method

(send a-stream read-fixed-integer what) → exact-integer?

  what : any/c
读取一个在流中有固定大小的精确整数,类似于 get-fixed 。

参数 what 与 read-integer 相同。

method

(send a-stream read-inexact what) → (and/c real? inexact?)

  what : any/c
读取一个不精确的实数,类似于 get-inexact 。

what 参数与 read-integer 相同。

method

(send a-stream read-raw-bytes what) → bytes?

  what : any/c
读取原始字节,类似于 get-unterminated-bytes 。

what 参数与 read-integer 相同。

method

(send a-stream read-bytes what) → bytes?

  what : any/c
读取原始字节,类似于 get-bytes 。

what 参数与 read-integer 相同。

method

(send a-stream read-editor what) → input-port?

  what : any/c
读取一个嵌套的编辑器,产生一个新的输入端口来提取该编辑器的内容。

what 参数与 read-integer 相同。

procedure

(read-snip-from-port name who stream) → bytes?

  name : string?
  who : any/c
  stream : (is-a?/c stream<%>)
给定 name, 预计是一个 snipclass 的名称,使用该 snipclass 从给定流中的当前点读取。 返回处理过的字节,和 read-snip 方法很相似。

9.1 Snip 类映射

当图形数据被编入 WXME 格式时,它与一个 snip-class 名称相关联,以便在加载时与一个实现相匹配。 参见 剪辑类 。

理想情况下,狙击类的名字产生为

(format "~s" (list '(lib string ...)
                   '(lib string ...)))

其中 format 列表中的每个元素都是一个带引号的模块路径(见 module-path?)。 string 必须只包含字母-数字 ASCII 字符,加上 ., _, - 和空格,而且不能是 "." 或 ".." 。

在这种情况下,第一个引用的模块路径用于在图形模式下加载 WXME 文件; 相应的模块必须提供实现 snip-class% 类的 snip-class 对象。 第二个引号的模块路径被 wxme 库用来转换没有图形支持的 WXME 流; 相应的模块必须提供一个实现 snip-reader<%> 接口的 reader 对象。 当然, snip-class% 实例和 snip-reader<%> 实例预计会解析相同的格式, 但会产生适合不同语境(即,是否有图形)的不同结果。

如果一个 snip-class 的名字被生成为

(format "~s" '(lib string ...))

则图形模式使用唯一的模块路径,而 wxme 需要一个兼容性映射。 用 register-lib-mapping! 安装一个。

如果一个狙击类的名字没有上述两种格式,那么只有当一个狙击类的名字被注册, 或者它是一个内置类的名字时,图形模式才能使用这些数据: "wxtext", "wxtab", "wximage" 或 "wxmedia" (用于嵌套编辑器)。 如果 wxme 库不是内置类之一, 它需要用 register-lib-mapping! 安装一个兼容性映射。

为 wxme 库自动安装了几个兼容性映射。 它们对应于不同版本的 DrRacket 所支持的流行图形元素,包括注释框、分数、XML 框、Racket 框、文本框, 以及由 htdp/image 教学包 (或者更普遍的,由 mrlib/cache-image-snip)生成的图像,和测试案例框。

对于一个由 wxme-port->port 创建的端口, 嵌套的编辑器由 wxme/editor 库提供的 editor% 类的实例来表示。 这个类提供了一个方法, get-content-port, 它返回编辑器内容的端口。 图像被表示为由 wxme/image 库提供的 image% 类的实例。

注释框被表示为一个类的实例,该类扩展了 editor% 以实现 readable<%>; 见 wxme/comment 。 read 形式产生一个特殊的注释(由 make-special-comment 创建), 所以当 read 被用来读取流时,注释框就会消失;special-comment 的内容就是 readable 实例。 XML、Racket 和文本框同样产生 editor% 和 readable<%> 的实例,以通常的方式展开; 见 wxme/xml, wxme/scheme 和 wxme/text 。 htdp/image 教学包中的图像被打包成 wxme/cache-image 库中 cache-image% 的实例。 测试盒被打包成来自 wxme/test-case 库的 test-case% 实例。
