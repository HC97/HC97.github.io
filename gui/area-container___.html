interface

area-container<%> : interface?

  implements: area<%>
area-container<%> 是一个容器 area<%> 。

所有 area-container<%> 类都接受以下命名的实例化参数:

  • border — 默认为 0; 传递给 border

  • spacing — 默认为 0; 传递给 spacing

  • alignment — 默认为特定的类, 例如 vertical-panel% 的 '(center top) ; 列表元素被传递给 set-alignment

method

(send an-area-container add-child child) → void?

  child : (is-a?/c subwindow<%>)
将给定的子窗口添加到未被删除的子窗口集合中。 也请参见 change-children 。

method

(send an-area-container after-new-child child) → void?

  child : (is-a?/c subarea<%>)
Specification: 在创建了一个以该区域为容器的新的被包含区域后,该方法被调用。 新的子窗口被作为参数提供给该方法。

Default implementation: 不做任何事情。

method

(send an-area-container begin-container-sequence) → void?

暂停容器顶层窗口的几何图形管理,直到调用 end-container-sequence 。 begin-container-sequence 和 end-container-sequence 方法用于包围一组容器的修改,以便只计算一次产生的几何图形。 容器序列还可以通过 change-children 延迟显示和隐藏动作, 以及通过 show 延迟屏幕上的显示部分,直到序列完成。 序列的开始和结束命令可以任意深入地嵌套。

(send an-area-container border) → spacing-integer?

(send an-area-container border margin) → void?
  margin : spacing-integer?
获取或设置容器的边距,单位是像素。 在计算子区域的位置和大小之前,这个边距被用作面板客户区域的嵌入。

method

(send an-area-container change-children filter) → void?

  filter : 
((listof (is-a?/c subarea<%>))
 . -> . (listof (is-a?/c subarea<%>)))
接受一个过滤过程,并改变容器的非删除子项列表。 过滤过程接收一个子区域的列表并返回一个新的子区域列表。 新的列表必须由作为这个区域的子区域创建的子区域组成(也就是说, change-children 不能被用来改变子区域的父区域)。

在非删除子区的集合被改变后,容器计算新删除的和新不删除的子区的集合。 新删除的窗口被隐藏。新的非删除的窗口被显示出来。

因为非窗口区域不能被隐藏,所以非窗口区域不能被删除。 如果过滤程序删除了非窗口子区域,就会产生一个异常,并且不改变非删除子区域的集合。

method

(send an-area-container container-flow-modified) → void?

当一个被覆盖的流定义方法(如 place-children 的结果发生变化时,调用这个方法。 该调用通知几何管理器,容器的子节点的位置需要重新计算。

reflow-container 只在几何管理器认为自上次计算后放置位置发生变化时才重新计算子代位置。

method

(send an-area-container reflow-container) → void?

当一个容器窗口没有显示时,对容器的子集的改变不一定会触发对容器的大小及其子集的大小和位置的立即重新计算。 相反,重新计算被延迟到容器被显示出来,这就避免了一系列变化之间的冗余计算。 reflow-container 方法强迫立即重新计算容器及其子代的大小和位置。

在调用 reflow-container 方法后, get-size, get-client-size, get-width, get-height, get-x 和 get-y 会立即报告管理器应用于容器及其子对象的尺寸和位置,即使容器被隐藏。 容器的实现可以在任何时候调用 get-size 等函数来获取窗口的当前状态(因为这些函数不会触发几何管理)。

另请参见 container-flow-modified 。

method

(send an-area-container set-alignment horiz-align    
  vert-align) → void?
  horiz-align : (symbols 'right 'center 'left)
  vert-align : (symbols 'bottom 'center 'top)
设置容器的对齐规范,这决定了当容器有剩余空间时(当一个孩子在某个特定维度上不能伸展时),它如何定位其孩子。

当容器的水平对齐方式为 'left 时,子代在容器中是左对齐的,空白被插入到右边。 当容器的水平对齐方式为 'center 时,每个子节点在容器中都是水平居中的。 当容器的水平对齐方式为 'right 时,剩余的空白将被插入到左边。

同样地,容器的垂直对齐方式可以是 'top, 'center 或 'bottom 。

(send an-area-container spacing spacing) → void?
