class

snip% : class?

  superclass: object%

  extends: equal<%>
一个直接的 snip% 的实例是无趣的。 有用的 snips 是通过实例化派生的子类来定义的,但 snip% 类定义了基本功能。 参见 实现新的剪贴画 以获得更多关于派生一个新的 snip 类的信息。

constructor

(new snip%) → (is-a?/c snip%)

创建一个长度为1的、具有 the-style-list 中的 "Basic" 风格的普通 snip。

method

(send a-snip draw dc    
  x    
  y    
  left    
  top    
  right    
  bottom    
  dx    
  dy    
  draw-caret) → void?
  dc : (is-a?/c dc<%>)
  x : real?
  y : real?
  left : real?
  top : real?
  right : real?
  bottom : real?
  dx : real?
  dy : real?
  draw-caret : 
(or/c 'no-caret 'show-inactive-caret 'show-caret
      (cons/c exact-nonnegative-integer?
              exact-nonnegative-integer?))
Specification: 调用(由编辑器)将剪贴画绘制到给定的绘图环境中, 剪贴画的左上角在DC坐标的 location (x, y) 。

参数 left, top, right 和 bottom 定义了一个剪裁区域(DC 坐标),snip 可以用它来优化绘图,但它也可以忽略这些参数。

dx 和 dy 参数提供了可以从 x 和 y 中减去的数字, 以获得snip在编辑器坐标中的 location (与DC 坐标相反,后者用于绘图)。

关于 draw-caret 的信息,见 粗体字的所有权 。 当 draw-caret 是一对时,避免为所选区域绘制背景, 如果 (get-highlight-text-color) 返回一个颜色(而不是 #f), 使用该颜色绘制所选文本和其他所选前景元素。

在这个方法被调用之前,字体、文本颜色和笔的颜色将在绘图上下文中被设置为 Snip 的样式。 (对于 get-extent 或 partial-offset,绘图上下文 不 是这样配置的)。 draw 方法不能对绘图上下文的状态做任何其他假设,除了剪裁区域已经被设置为合适的东西。 在 draw 返回之前,它必须恢复它所改变的任何绘图上下文设置。

参见 on-paint in editor<%> 。

当这个方法被调用时,snip 的编辑器通常被内部锁定,以进行写入和回流(参见 内部编辑器锁), 并且它通常处于刷新状态(参见 编辑器和线程)。

Default implementation: 什么都不画。

method

(send a-snip get-extent dc    
  x    
  y    
  [w    
  h    
  descent    
  space    
  lspace    
  rspace]) → void?
  dc : (is-a?/c dc<%>)
  x : real?
  y : real?
  w : (or/c (box/c (and/c real? (not/c negative?))) #f) = #f
  h : (or/c (box/c (and/c real? (not/c negative?))) #f) = #f
  descent : (or/c (box/c (and/c real? (not/c negative?))) #f)
   = #f
  space : (or/c (box/c (and/c real? (not/c negative?))) #f) = #f
  lspace : (or/c (box/c (and/c real? (not/c negative?))) #f)
   = #f
  rspace : (or/c (box/c (and/c real? (not/c negative?))) #f)
   = #f
Specification: 计算剪贴的宽度、高度、下降(低于基线的高度)、空间(顶部 "填充 "空间的高度)和水平空间(左右两边 "填充" 空间的宽度)。 这些值通过填充 w, h, descent, space, lspace 和 rspace 框返回。

这个方法是由 snip 的管理员调用的;通常不会被其他人直接调用。 要获得一个剪裁的范围,可以在编辑器中使用 get-snip-location in editor<%> 。

为了查找字体大小,提供了一个绘图上下文,但不应该发生绘图。 get-extent 和 partial-offset 方法不能对绘图上下文的状态做任何假设, 除了它被正确缩放。 特别是,在调用该方法之前,snip的样式的字体不会在绘图上下文中自动设置。 (许多剪贴画缓存了它们的尺寸信息,所以自动设置字体会很浪费。) 如果 get-extent 或 partial-offset 改变了绘图上下文的设置, 它必须在返回之前恢复它们。 然而,这些方法不应该需要改变绘图上下文;只有字体设置可以影响设备上下文的测量结果, get-text-extent in dc<%> 接受一个 font% 的参数,用于覆盖该设备上下文的当前字体。

狙击手的左边和上面的 location 是以编辑器坐标中的 x 和 y 来提供的,以防狙击手的大小取决于其位置; x 和 y 参数通常被忽略。 在文本编辑器中, y 坐标是 line’s 的顶部位置; 在知道其高度之前,狙击点的实际顶部 location 可能是不确定的。

如果一个狙击手为了将来的回复而缓存了结果的大小,那么当调用 size-cache-invalid 时, 它应该使其缓存的大小失效(特别是当狙击手的大小依赖于任何设备上下文属性时)。

如果一个狙击手在收到对 get-extent 的调用后, 在收到对 size-cache-invalid 的调用前,其尺寸发生了变化, 那么狙击手必须将尺寸变化通知其管理员,以便管理员可以重新计算其派生的尺寸信息。 通过调用其 resized 方法来通知管理员尺寸的变化。

当这个方法被调用时,snip 的编辑器通常会被内部锁定,以便进行写入和回流(参见 内部编辑器锁)。

Default implementation: 用 0.0 填充所有方框。

method

(send a-snip get-flags) → (listof symbol?)

返回定义 snip 行为的标志,是以下符号的列表:

  • 'is-text — 这是一个从 string-snip% 衍生出来的文本剪贴,不要设置这个标志

  • 'can-append — 这个剪贴画可以和另一个相同类型的剪贴画合并。

  • 'invisible — 一个 invisible 的剪贴,用户看不到,比如换行。

  • 'hard-newline — 一个换行必须跟在剪贴的后面

  • 'newline — 目前换行跟在狙击点后面;只有拥有该标志的编辑才应该设置这个标志

  • 'handles-events — 这个剪贴画在拥有键盘焦点时可以处理键盘和鼠标事件

  • 'handles-all-mouse-events — 这个剪贴画可以处理触及剪贴画或紧随触及剪贴画的事件的鼠标事件, 即使该剪贴画没有键盘焦点(也见 on-goodbye-event)

  • 'handles-between-events — 这个剪贴画处理在剪贴画中的项目之间的鼠标事件 (而不是默认将鼠标点击视为设置位置或其他发生在 text% 或 pasteboard% 层面的事件处理)

  • 'width-depends-on-x — 这个片段的显示宽度取决于该片段在编辑器中的 x-location ;例如: tab

  • 'height-depends-on-y — 这个剪贴画的显示高度取决于剪贴画在编辑器中的 y-location

  • 'width-depends-on-y — 这个剪贴画的显示宽度取决于剪贴画在编辑器中的 y-location

  • 'height-depends-on-x — 这个剪贴画的显示高度取决于剪贴画在编辑器中的 x-location

  • 'uses-editor-path — 这个狙击手使用它的编辑器的路径名,当名称改变时应该被通知; 通知是作为对 set-admin 的一个多余的调用

method

(send a-snip get-text offset num [flattened?]) → string?

  offset : exact-nonnegative-integer?
  num : exact-nonnegative-integer?
  flattened? : any/c = #f
Specification: 返回该片段的文本,从片段中的 position offset 开始, 一直到总长度为 num 的 item 。 如果 offset 大于 snip 的 count ,那么将返回 "" 。 如果 num 大于 snip 的 count 减去偏移量,那么将返回从偏移量到 snip 结尾的文本。

如果 flattened? 不是 #f, 那么就会返回扁平化的文本。 关于扁平化与非扁平化文本的讨论,请参见 扁平化的文本 。

Default implementation: 返回 "" 。

method

(send a-snip merge-with prev) → (or/c (is-a?/c snip%) #f)

  prev : (is-a?/c snip%)
Specification: 将 a-snip 与 prev 合并,如果不能合并则返回 #f ,否则返回一个新的合并的 snip。 只有当两个片段都来自同一类别并且都有 'can-append 标志时,该方法才会被调用。

如果返回的 snip 没有预期的 count ,它的 count 将被强行修改。 如果返回的 snip 已经被另一个管理员所拥有,则会创建一个代理 snip。

当此方法被调用时,snip 的编辑器通常被内部锁定以备读取(参见 内部编辑器锁)。

Default implementation: 返回 #f 。

method

(send a-snip next) → (or/c (is-a?/c snip%) #f)

返回拥有该片段的编辑器中的下一个片段,如果这是最后一个片段,则返回 #f 。

在文本编辑器中,下一个片段是位于这个片段(最后一个) position 之后的 position 的片段。 在粘贴板中,下一个狙击点是紧跟在这个狙击点后面的那个。(关于粘贴板中的狙击点顺序,请参见 (参见 编辑器结构和术语 ,了解关于粘贴板中的剪裁顺序的信息。)。

(send a-snip release-from-owner) → boolean?

Specification: 要求狙击手尝试从其所有者那里释放自己。 如果该片段没有被拥有或释放成功,则返回 #t 。 否则,返回 #f ,该狙击手仍然拥有。参见 is-owned? 。

(send a-snip set-admin admin) → void?

设置 snip 的管理员。 只有管理员才能调用这个方法。

默认方法设置一个 snip 的内部状态以记录其管理员。 如果狙击手已经被一个管理员所拥有,并且该管理员没有祝福这个过渡,它将不会修改这个状态。 如果在 text% 或 pasteboard% 的实例对该方法的敏感调用中, 一个狙击手的管理员状态没有按预期修改,内部状态可能会被强行修改(如果新的管理员是 #f) 或者创建一个代理狙击手(如果该狙击手预期会收到一个新的管理员)。

method

(send a-snip split position first second) → void?

  position : exact-nonnegative-integer?
  first : (box/c (is-a?/c snip%))
  second : (box/c (is-a?/c snip%))
Specification: 将剪贴画分成两个剪贴画。 当一个片段有一个以上的 item ,并且有东西插入两个 item 之间时,这个方法被调用。

参数是一个相对 position 的整数和两个方框。 position 整数指定了有多少个 item 应该被赋予新的第一个片段;其余的被赋予新的第二个片段。 两个盒子必须用两个新的剪子来填充。(旧的剪子不再使用,所以它可以作为一个新的剪子被回收)。

如果返回的狙杀者没有预期的 count ,他们的 count 将被强行修改。 如果任何一个返回的 snip 已经被另一个管理员所拥有,则会创建一个代理 snip。

当这个方法被调用时,snip 的编辑器通常被内部锁定以备读取(参见 内部编辑器锁)。

Default implementation: 创建一个新的带有 position 元素的 snip% 实例, 并修改 a-snip ,使其计数按 position 递减。 嵌套的snip被安装到 first ，a-snip 被安装到 second 。
