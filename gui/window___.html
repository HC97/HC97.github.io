interface

window<%> : interface?

  implements: area<%>
window<%> 对象是一个具有图形表示的 area<%> ,可以对事件做出反应。

所有的 window<%> 类都接受以下命名的实例化参数:

  • enabled — 默认为 #t; 如果 #f,则传递给 enable

method

(send a-window accept-drop-files) → boolean?

(send a-window accept-drop-files accept-files?) → void?
  accept-files? : any/c
启用 或禁用窗口的拖放功能,或获得启用状态。拖放最初是禁用的。 参见 on-drop-file 。

method

(send a-window client->screen x y)

 → 
position-integer? position-integer?
  x : position-integer?
  y : position-integer?
将本地窗口坐标 转换 为屏幕坐标。

在 Mac OS 上,屏幕坐标从菜单栏左上方的 (0, 0) 开始。 相反,在 move in top-level-window<%> 中的移动认为 (0, 0) 是在菜单栏下面。 参见 get-display-left-top-inset 。

method

(send a-window enable enable?) → void?

  enable? : any/c
启用或禁用一个窗口,使输入事件被忽略。 (输入事件包括鼠标事件、键盘事件和关闭框的点击,但不包括焦点或更新事件。) 当一个窗口被禁用时,其子窗口的输入事件也会被忽略。

The enable state of a window 可以被 enabling a parent window 改变,这种改变不经过这个方法;使用 on-superwindow-enable 来监控 enable state 的变化。

如果 enable? 为真,窗口被启用,否则被禁用。

method

(send a-window focus) → void?

如果该窗口接受了键盘焦点,那么相对于它的顶层窗口, 移动 键盘焦点到该窗口。 如果焦点在窗口的顶层窗口中,或者窗口的顶层窗口是可见的和浮动的(即用 'float 样式创建的), 那么焦点会立即移动到这个窗口。 否则,焦点不会被立即移动,但当窗口的顶层窗口获得键盘焦点时,焦点会被委托给这个窗口。

参见 on-focus 。

注意,在Unix上,当用户选择一个菜单项时,键盘焦点可以移动到菜单栏上。

The current keyboard focus window 可以被 the user 改变,这种改变不经过这个方法;使用 on-focus 来监控 focus 的变化。

method

(send a-window get-client-handle) → cpointer?

返回当前平台的 GUI 工具箱的窗口 “inside” 的一个手柄。 指针所代表的值取决于平台:

  • Windows: HWND

  • Mac OS: NSView

  • Unix: GtkWidget

参见 get-handle 。

method

(send a-window get-client-size)

 → 
dimension-integer? dimension-integer?
获取窗口的内部尺寸,单位是像素。 对于一个容器来说,内部尺寸是可用于放置子窗口的尺寸(包括边框空白)。对于一个画布,这是可见的绘图区域。

客户端尺寸以两个值返回:宽度和高度(单位:像素)。

也请参见 reflow-container 。

method

(send a-window get-cursor) → (or/c (is-a?/c cursor%) #f)

返回窗口的光标,如果这个窗口的光标默认为父窗口的光标,则返回 #f 。 更多信息请参见 set-cursor 。

method

(send a-window get-handle) → cpointer?

返回当前平台的 GUI 工具箱的窗口 “outside” 的句柄。 指针所代表的值取决于平台:

  • Windows: HWND

  • Mac OS: NSWindow 用于 top-level-window<%> 对象, NSView 用于其他窗口

  • Unix: GtkWidget

参见 get-client-handle 。

method

(send a-window get-height) → dimension-integer?

返回窗口的总高度(单位:像素)。

也请看 reflow-container 。

method

(send a-window get-label)

 → 
(or/c label-string?
      (is-a?/c bitmap%)
      (or/c 'app 'caution 'stop)
      (list/c (is-a?/c bitmap%)
              label-string?
              (or/c 'left 'top 'right 'bottom))
      #f)
获取一个窗口的标签,如果有的话。控制窗口通常以某种方式显示其标签。 框架和对话框将其标签显示为窗口标题。 面板不显示它们的标签,但标签可用于识别目的。 消息、按钮和复选框可以有位图标签(只有当它们被创建为位图标签时),但所有其他窗口都有字符串标签。 此外,消息标签可以是一个图标符号 'app, 'caution 或 'stop, 而按钮可以有位图标签和字符串标签(以及位图的位置)。

一个标签字符串可以包含 &, 作为 Windows 和 Unix 上控件的键盘导航注释。 安培号不是控件显示的标签的一部分;相反,安培号在显示的标签中被删除(在所有平台上), 安培号前面的任何字符都有下划线(Windows 和 Unix),表示该字符是控件的助记符。 双连字符被转换为单连字符(不显示下划线)。 另请参见 on-traverse-char 。

如果该窗口没有标签,则返回 #f 。

method

(send a-window get-plain-label) → (or/c string? #f)

与 get-label 类似,除了:

  • 如果标签中包含 (&c) 的任何字符 c, 那么该序列和周围的任何空白将被删除。

  • 如果标签中包含任何字符 c 的 &c ,则删除 & 。

  • 如果标签包含一个 tab 字符,那么 tab 字符和所有后续字符都会被删除。

参见 button% 对标签的处理。

如果窗口没有标签,或者窗口的标签不是一个字符串,则返回 #f 。

method

(send a-window is-enabled?) → boolean?

表示该窗口当前是否启用。 如果该窗口在其祖先启用时被启用,则结果为 #t ; 如果该窗口在其祖先启用时仍被禁用,则结果为 #f 。 (也就是说,这个方法的结果只受调用 a-window 的 enable 的影响, 不受父窗口的 enable 状态的影响)。

method

(send a-window is-shown?) → boolean?

表明该窗口当前是否显示。 如果该窗口在其祖先显示时被显示,则结果为 #t ; 如果该窗口在其祖先显示时仍被隐藏,则结果为 #f 。 (也就是说,这个方法的结果只受 a-window 的 show 调用的影响,不受父窗口的可见性影响)。

method

(send a-window on-drop-file pathname) → void?

  pathname : path?
当用户将一个文件拖到窗口上时被 调用 。 (在 Unix 上,通过 XDND 协议支持拖放。) 必须首先用 accept-drop-files 为窗口启用拖放功能。

在Mac OS上,当应用程序运行时,用户双击一个应用程序处理的文件或将文件拖到应用程序的图标上, 主线程的应用程序文件处理程序被调用(见 application-file-handler)。 如果最近激活的框架启用了拖放功能,默认处理程序会调用该框架的 on-drop-file 方法, 这与该框架的事件空间无关(但该方法会在该框架的事件空间的处理程序线程中调用)。 当应用程序未运行时,文件名将作为命令行参数提供。

method

注意,在 Unix 上,当用户选择一个菜单项时,键盘焦点可以移动到菜单栏上。

Default implementation: 不做任何事情。

Specification: 当窗口被调整大小时被调用。 窗口的新尺寸(像素)被提供给该方法。 这个尺寸值是针对整个窗口的,而不仅仅是客户区。

Default implementation: 不做任何事情。

method

(send a-window on-subwindow-char receiver    
  event) → boolean?
  receiver : (is-a?/c window<%>)
  event : (is-a?/c key-event%)
Specification: 当这个窗口或一个子窗口收到一个键盘事件时被调用。 首先调用接收方顶级窗口的 on-subwindow-char 方法 (见 get-top-level-window); 如果返回值是 #f, 那么调用接收方路径中下一个子窗口的 on-subwindow-char 方法, 以此类推。 最后,如果接收方的 on-subwindow-char 方法返回 #f, 事件就会被传递给接收方的正常按键处理机制。

event 参数是为 receiver 窗口产生的事件。

on-subwindow-event 的原子性限制也适用于 on-subwindow-char 。 也就是说,一个不够合作的 on-subwindow-char 方法可以有效地禁用一个控件对关键事件的处理, 即使它返回 #f

注意:默认的 on-subwindow-char in frame% 和 on-subwindow-char in dialog% 方法会消耗某些用于在窗口中导航的键盘事件(例如,方向键、Enter)。 因为顶层窗口首先得到处理键盘事件的机会,所以有些事件永远不会到达 "接收者" 子窗口,除非默认的框架或对话框方法被重写。

Default implementation: 返回 #f 。

method

(send a-window on-subwindow-event receiver    
  event) → boolean?
  receiver : (is-a?/c window<%>)
  event : (is-a?/c mouse-event%)
Specification: 当这个窗口或一个子窗口收到一个鼠标事件时被调用。 首先调用接收者顶层窗口的 on-subwindow-event 方法 (见 get-top-level-window); 如果返回值为 #f, 则调用接收者路径上的下一个子窗口的 on-subwindow-event 方法, 以此类推。 最后,如果接收器的 on-subwindow-event 方法返回 #f, 事件将被传递给接收器的正常鼠标处理机制。

event 参数是为 receiver 窗口产生的事件。

如果 on-subwindow-event 方法链没有原子地完成(即不需要其他线程运行), 或者完成得不够快,那么相应的事件可能不会被传递给目标控件,比如一个按钮。 换句话说,一个不够合作的 on-subwindow-event 方法可以有效地禁用一个控件对鼠标事件的处理, 即使它返回 #f 。

Default implementation: 返回 #f 。

method

(send a-window on-subwindow-focus receiver    
  on?) → void?
  receiver : (is-a?/c window<%>)
  on? : boolean?
Specification: 当这个窗口或一个子窗口收到或失去键盘焦点时被调用。 这个方法在 receiver 的 on-focus 方法之后被调用。 首先调用接收器顶层窗口的 on-subwindow-focus 方法(见 get-top-level-window), 然后调用接收器路径中下一个子窗口的 on-subwindow-focus 方法,依次类推。

Default implementation: 不做任何事情。

method

(send a-window on-superwindow-activate active?) → void?

  active? : any/c

Added in version 1.54 of package gui-lib.

method

(send a-window on-superwindow-enable enabled?) → void?

  enabled? : any/c
Specification: 当一个窗口的启用状态发生变化时,通过事件队列调用该窗口的 enable 方法, 或者通过启用/禁用该窗口的一个祖先。 该方法的参数表明该窗口现在是否被启用。

这个方法在最初创建窗口时不被调用;只有在改变了窗口的初始启用状态后才会被调用。 此外,如果一个启用通知事件被排在窗口的队列中,而它在事件被派发之前恢复了它的启用状态,那么该派发就会被取消。

如果一个窗口的祖先的启用状态在该窗口被删除时发生了变化(例如,因为它是用 delete-child 删除的),那么就不会为被删除的窗口排定启用事件。 但如果该窗口后来被重新激活,进入与该窗口被删除时的状态不同的启用状态,那么就会立即排一个启用事件。

Default implementation: 什么都不做。

method

(send a-window on-superwindow-show shown?) → void?

  shown? : any/c
Specification: 当一个窗口的可见性发生变化时,通过事件队列调用该方法,可以通过调用该窗口的 show , 通过显示/隐藏该窗口的一个祖先,或者通过激活或停用该窗口或其在容器中的祖先(例如,通过 delete-child)。 该方法的参数表示该窗口现在是否可见。

这个方法在窗口最初创建时不被调用;只有在窗口的初始可见性发生变化后才会被调用。 此外,如果一个显示通知事件被排在窗口的队列中,而它在事件被派发前恢复了可见性,那么该派发将被取消。

Default implementation: 什么都不做。

method

(send a-window popup-menu menu x y) → void?

method

(send a-window wheel-event-mode)

 → (or/c 'one 'integer 'fraction)
(send a-window wheel-event-mode mode) → void?
  mode : (or/c 'one 'integer 'fraction)
获取或设置窗口中鼠标滚轮事件的模式。 滚轮事件表示为 key-event% 实例, 其中 get-key-code in key-event% 返回 'wheel-up, 'wheel-down, 'wheel-right 或 'wheel-left 。 一个窗口的滚轮事件模式决定了对底层平台报告的可变滚轮大小事件的处理。 具体来说,滚轮事件模式决定了系统为该窗口生成的事件在 get-wheel-steps in key-event% 的可能值:

  • 'one — 轮子事件总是被报告为单步,其中窗口累积增量直到达到整步,并且它为多步累积产生单独的事件。

  • 'integer — 转轮事件总是报告为整数大小的步数,其中小数步数被累积并根据需要保留,以达到整数增量。

  • 'fraction — 转轮事件在收到底层平台的数据后立即报告为正的实数值。

默认的滚轮事件模式是 'one, 但 editor-canvas% 将滚轮事件模式初始化为 'integer 。

Added in version 1.43 of package gui-lib.
