class

text% : class?

  superclass: object%

  extends: editor<%>
text% 对象是一个标准的文本编辑器。 文本编辑器是通过 editor-canvas% 对象或其他 display 方式在屏幕上显示的。

constructor

(new text%    
    [[line-spacing line-spacing]    
    [tab-stops tab-stops]    
    [auto-wrap auto-wrap]]) → (is-a?/c text%)
  line-spacing : (and/c real? (not/c negative?)) = 1.0
  tab-stops : (listof real?) = null
  auto-wrap : any/c = #f
line-spacing 参数设置了编辑器显示时每行之间插入的额外空间量(以 DC 单位)。 这个间距包含在每行的报告高度中。

参见 set-tabs 以了解关于 tabstops 的信息。

如果 auto-wrap 为 true,那么就通过 auto-wrap 启用自动包装功能。

为新的编辑器创建一个新的 keymap% 对象。 参见 get-keymap 和 set-keymap 。

为新的编辑器创建一个新的 style-list% 对象。 参见 get-style-list 和 set-style-list 。

(send a-text change-style delta    
  [start    
  end

  [start    
  end    
  counts-as-mod?]) → void?
  style : (or/c (is-a?/c style<%>) #f)

在编辑器中,通过应用一个样式 delta 或安装一个特定的样式来改变一个区域的样式。 如果 start 是 'start ， end 是 'end, 那么当前选择的 item 会被改变。 否则,如果 end 是 'end, 那么样式将从 start 到选择的结束处被改变。 如果 counts-as-mod? 是 #f, 那么 set-modified 在应用样式改变后不会被调用。

当 style 被提供时: 编辑器的样式列表必须包含 style, 否则样式不会被改变。参见 convert in style-list% 。

(send a-text find-line y [on-it]) → exact-nonnegative-integer?

  map : (or/c (is-a?/c editor-wordbreak-map%) #f)
设置标准断字函数所使用的断字图。 参见 editor-wordbreak-map% 获取更多信息。

如果 map 是 #f, 那么就使用标准 (the-editor-wordbreak-map) 。

method

(send a-text split-snip pos) → void?

  pos : exact-nonnegative-integer?
给定一个 position ,分割包括该 position 的剪贴(如果有的话),使该 position 处于两个剪贴之间。 狙击手可以拒绝分割,尽管没有一个内置的狙击手类会拒绝。

当编辑器为回流而被内部锁定时,分割一个片段是不允许的(也见 内部编辑器锁)。

method

(send a-text write-to-file stream [start end]) → boolean?

  stream : (is-a?/c editor-stream-out%)
  start : exact-nonnegative-integer? = 0
  end : (or/c exact-nonnegative-integer? 'eof) = 'eof
Extends write-to-file in editor<%>.
如果 start 是 0, end 是 'eof 负数,那么整个内容将被写入流中。 如果 end 是 'eof, 那么内容将从 start 写到编辑器的末端。 否则,将写入给定范围内的内容。
