class

keymap% : class?

  superclass: object%

keymap% 对象被 editor<%> 对象使用,以可扩展的方式将键盘和鼠标序列映射到任意的函数。 键图也可以在没有编辑器的情况下使用。一个 keymap% 对象包含

  • 从函数名到事件处理程序的映射;以及

  • 从按键和鼠标序列到函数名的映射。

一个按键图中的处理程序被一个 key-event% 对象或一个 mouse-event% 对象所调用。 它还被赋予另一个值,这个值取决于使用该键盘地图的上下文 (或者更具体地说, handle-key-event 或 handle-mouse-event 的参数)。 对于与 editor<%> 对象相关的键盘地图,额外的参数通常是接收键盘或鼠标事件的 editor<%> 对象。

constructor

(new keymap%) → (is-a?/c keymap%)

创建一个空的键盘地图。

method

(send a-keymap add-function name func) → void?

  name : string?
  func : (any/c (is-a?/c event%) . -> . any)
命名一个处理事件的新函数,在响应 handle-key-event, handle-mouse-event 或 call-function 时调用。 该过程的返回值被忽略。

如果已经有一个函数映射到这个名字,它将被替换成给定的函数。

当函数被调用时,它获得传递给 handle-key-event, handle-mouse-event 或 call-function 的参数。 对于与一个编辑器相关联的键盘地图,通常是目标编辑器。

method

(send a-keymap is-function-added? fname) → boolean?

  fname : string?
如果 fname 通过 keymap% 被添加到这个键盘地图上, 返回 #t ,否则返回 #f 。

这个方法不检查链式键图,看该函数是否被添加到其中一个键图。

method

(send a-keymap break-sequence) → void?

如果键盘图处于按键序列的中间,则清除其状态。 例如,用户可能点击了 escape,然后换到了另一个窗口; 如果 escape 是键盘序列的一部分,则需要清除键盘图的状态,因为用户不打算完成这个序列。

可以用 set-break-sequence-callback 安装一个 break 回调函数。

method

(send a-keymap call-function name    
  in    
  event    
  [try-chain?]) → boolean?
  name : string?
  in : any/c
  event : (is-a?/c event%)
  try-chain? : any/c = #f
直接调用一个命名的事件处理程序。 如果找不到该函数或者找到的处理程序不想处理该事件,则返回 #f 。 否则,返回值是事件处理程序的布尔型返回值。

in 和 event 参数将被传递给 keymap 处理程序,如果找到的话。

如果 try-chain? 不是 #f, 那么与这个 keymaps 连锁的 keymaps 会被搜索到函数名。 如果没有找到该函数,并且 try-chain? 是 #f; 也会产生一个异常,但异常处理程序不能逃脱(见 连续性和事件调度)。

method

(send a-keymap chain-to-keymap next    
  prefix?) → void?
  next : (is-a?/c keymap%)
  prefix? : any/c
将 next 从 a-keymap 上链。 next 将被用来处理 a-keymap 没有处理的事件。

如果 prefix? 是一个真值,那么 next 将优先于已经与 a-keymap 连锁的其他键位, 如果两个键位都映射了相同的键序。 当一个链式键图映射的键是另一个键的前缀时,那么较短的键序列总是被使用,与 prefix? 无关。

多个键图可以使用 chain-to-keymap 从一个键图上链起来。 当键图从一个主键图上链下来时,主键图没有处理的事件会传递给链上的键图,直到某个链上的键图处理这些事件。 键图可以在一个任意的非循环图中被链在一起。

键图链是有用的,因为多事件序列在链式组中得到正确处理。 如果没有链,一个事件序列可以在一个键图中产生状态,当一个键图中的回调被调用时,这个状态必须被重置。 这种状态可以通过 break-sequence 手动清除, 尽管调用 break-sequence 方法也会调用 set-break-sequence-callback 安装的处理程序。

method

(send a-keymap get-double-click-interval)

 → (integer-in 0 1000000)
返回一个双击的点击之间可以间隔的最大毫秒数。

默认的时间间隔是以特定平台的方式确定的,但它可以通过 'GRacket:doubleClickTime preference 偏好在全局范围内被重写; 见 首选项 。

method

(send a-keymap map-function keyname fname) → void?

  keyname : string?
  fname : string?
使用 keyname 中的字符串编码序列将一个输入状态序列映射到一个函数名称。 keyname 的格式是一个以分号分隔的输入状态序列;每个状态是由一个修改器标识符序列和一个键标识符组成的。

修改器的标识符是

  • s: — 所有平台: Shift

  • c: — 所有平台: Control

  • a: — Mac OS: Option

  • m: — Windows: Alt; Unix: Meta; Mac OS: Command, 当 map-command-as-meta-key 产生 #t 时

  • d: — Mac OS: Command

  • l: — 所有平台: Caps Lock

  • g: — Windows: Control 加 Alt 为 AltGr; 参见 get-control+meta-is-altgr in key-event%

  • ?: — 所有平台: 允许与 Shift、AltGr/Option 和/或 Caps Lock 的相反使用所产生的字符相匹配,如果有的话; 见 get-other-shift-key-code in key-event%

如果一个特定的修改器在状态字符串中没有提到,那么无论该修改器是否被按下,它都会匹配状态。 修饰符前面的 ~ 使得字符串只匹配相应修饰符未被按下的状态。 如果状态字符串以 : 开头, 那么该字符串仅在 Shift、Control、Option、Alt、Meta 和 Command 等字符串中未提及的修改器未被按下的情况下匹配状态。

一个键的标识符可以是键盘上的一个字符(例如 a, 2, ?) 或特殊名称如下:

对于一个特殊的关键词,大写字母并不重要。 然而,大写字母对于单字母的关键词来说是很重要的。 此外,单字母的 ASCII 键名会被特别处理: A 和 s:a 都被当作 s:A 处理。 然而,当在 Windows 中包含 c: 而没有 m:,或者在 Mac OS 中包含 d: 时, ASCII 字母不会被 s: 上调,因为 Shift 键的上调行为被 Control 取消了, 没有 Alt(在Windows)或 Command(在 Mac OS)。

一个状态可以匹配映射在一个键盘地图(或键盘地图链)中的多个状态字符串; 当一个状态匹配多个状态字符串时,会根据具体程度对这些字符串进行排序,从而选择一个映射。 提到更多按下的修改器的状态字符串比其他状态字符串排名更高, 如果两个字符串提到相同数量的按下的修改器,提到更多未按下的修改器的字符串排名更高。 最后,一个包含 ?: 的状态字符串,只有在相反地使用 Shift、AltGr/Option 和/或 Caps Lock 的情况下才能匹配, 其排名低于所有不依赖 ?: 的匹配,而一个需要相反地使用 Shift 和 AltGr/Option 的字符串排名更低。 在多个匹配字符串具有相同等级的情况下,将任意选择一个匹配。

例如:

  • "space" — 只要按下空格键就能匹配,不管修改键的状态如何。

  • "~c:space" — 只要按了空格键而没有按 Control 键就匹配。

  • "a" — 只要输入 a 就匹配,不管修改键的状态如何(除 Shift 外)。

  • ":a" — 只在输入 a 且未按下修饰键时匹配。

  • "~c:a" — 只要输入 a 并且没有按下 Shift 键和 Control 键,就可以匹配。

  • "c:m:~g:x" — 只要在输入 x 时按下 Control 键和 Alt 键(Windows)或 Meta 键(Unix), 只要 Control-Alt 组合在 Windows 上不是由 AltGr 组成,就可以匹配。

  • ":esc;:c:c" — 匹配按 Escape 键(无修饰符)后按 Control-C(无 Control 以外的修饰符)。

  • "?:d:+" — 匹配当 Command 被按下时产生 + 的键, 即使产生 + 通常需要按 Shift。

对 map-function 的调用将映射一个特定的按键序列作为前缀和完整的序列, 会引发一个异常,但异常处理程序不能逃脱(见 连续性和事件调度)。

在输入状态被映射到名字之前,一个函数的名字不需要被映射到处理程序;处理程序在调用的时候就被名字调度了。 映射到函数名的事件处理程序可以改变而不影响从输入状态到函数名的映射。

Changed in version 1.2 of package gui-lib: 增加了 g: 和 ~g: 支持。

method

(send a-keymap set-grab-key-function f) → void?

  f : 
((or/c string? false?)
 (is-a?/c keymap%)
 any/c
 (is-a?/c key-event%)
 . -> . any)
安装一个回调程序,在键盘图与函数名称匹配输入或匹配输入失败后被调用。 一次只能安装一个键盘抓取函数。 当键盘映射与带有抓取回调的键盘映射链在一起时,回调会在链式键盘映射的匹配中被调用(当链式键盘映射没有自己的抓取回调)。

如果抓取回调对匹配或不匹配的回调返回一个真值,事件被认为是被处理了。 如果回调对一个匹配的回调返回一个真值,那么匹配的键图函数就不会被键图调用。

回调程序 f 将被调用为:

(f str keymap editor event)

str 参数是匹配回调的函数名称,或者 #f 是不匹配的回调。 keymap 参数是匹配的 keymap(可能是与安装回调的 keymap 连锁的 keymap)或安装回调的 keymap。 editor 和 event 参数与传递给匹配的键盘地图函数的参数相同。

抓取键回调函数用 remove-grab-key-function 来取消安装。
