2 开始使用 Typed Racket

回顾一下 快速入门 中的类型化模块:

#lang typed/racket
(struct pt ([x : Real] [y : Real]))
 
(: distance (-> pt pt Real))
(define (distance p1 p2)
  (sqrt (+ (sqr (- (pt-x p2) (pt-x p1)))
           (sqr (- (pt-y p2) (pt-y p1))))))

让我们依次考虑这个程序的每个元素。

#lang typed/racket

这指定了该模块是用 typed/racket 语言编写的, 它是 racket 语言的类型化版本。 也提供了其他语言的类型化版本; 例如,typed/racket/base 语言对应于 racket/base 。

(struct pt ([x : Real] [y : Real]))

Typed Racket 提供了核心 Racket 表单的修改版本,允许类型注释。 以前的 Typed Racket 版本提供了带有 : 后缀的形式, 但现在为了向后兼容,这些形式只作为遗留的形式被包括在内。

这定义了一个新的结构,名为 pt, 有两个字段, x 和 y 。 两个字段都被指定为 Real 类型,对应于 real numbers 。 这个 struct 形式对应于 racket 的非类型对应物– 当把程序从 racket 移植到 typed/racket 时, 只需在现有的字段声明中添加类型注释。

(: distance (-> pt pt Real))

这里声明 distance 的类型是 (-> pt pt Real) 。

类型 (-> pt pt Real) 是一个函数类型,也就是说,是一个过程的类型。 输入类型,或者说域,是两个 pt 类型的参数,指的是 pt 结构的一个实例。 -> 表示这是一个函数类型。 范围类型,或者说输出类型,是函数类型中的最后一个元素,在这里是 Real 。

如果你熟悉 contracts, 函数类型的符号与函数契约组合器相似。

(define (distance p1 p2)
  (sqrt (+ (sqr (- (pt-x p2) (pt-x p1)))
           (sqr (- (pt-y p2) (pt-y p1))))))

这个定义与非类型化版本的代码没有变化。 Typed Racket 的目标是允许几乎所有的定义在不改变的情况下进行类型检查。 类型检查器在假设 p1 和 p2 具有 pt 类型的情况下, 验证函数的主体是否具有 Real 类型,这些类型来自先前的类型声明。 由于主体确实有这种类型,程序被接受。

在 Typed Racket REPL 中,调用 distance 将像往常一样显示结果,并且还将打印出结果的类型:

> (distance (pt 0 0) (pt 3.1415 2.7172))

- : Real

4.153576541969583

仅仅评估函数名就会打印出函数值和它的类型,这对于发现 Typed Racket 赋予 Racket 函数的类型很有用。 另外, :print-type 命令将只打印类型:

> distance

- : (-> pt pt Real)

#<procedure:distance>

> string-length

- : (-> String Index)

#<procedure:string-length>

> (:print-type string-ref)

(-> String Integer Char)

2.1 数据类型和统一体

许多数据结构涉及多个变体。 在 Typed Racket 中,我们使用 union types 来表示这些,写成 (U t1 t2 ...) 。

#lang typed/racket
(define-type Tree (U leaf node))
(struct leaf ([val : Number]))
(struct node ([left : Tree] [right : Tree]))
 
(: tree-height (-> Tree Integer))
(define (tree-height t)
  (cond [(leaf? t) 1]
        [else (max (+ 1 (tree-height (node-left t)))
                   (+ 1 (tree-height (node-right t))))]))
 
(: tree-sum (-> Tree Number))
(define (tree-sum t)
  (cond [(leaf? t) (leaf-val t)]
        [else (+ (tree-sum (node-left t))
                 (tree-sum (node-right t)))]))

在这个模块中,我们定义了两个新的数据类型: leaf 和 node 。 我们还定义了类型名称 Tree 为 (U node leaf), 它表示一棵二进制的数字树。 从本质上讲,我们是说 tree-height 函数接受一个 Tree, 它是一个 node 或一个 leaf,并产生一个数字。

为了计算关于树的有趣的事实,我们必须把它们拆开,并获取其内容。 但是由于像 node-left 这样的访问器需要一个 node 作为输入, 而不是一个 Tree, 我们必须确定我们被传递的是哪种输入。

为了这个目的,我们使用每个定义的结构所附带的谓词。 例如, leaf? 谓词将 leaf 与所有其他类型的 Racket 值区分开来。 因此,在 tree-sum 的 cond 子句的第一个分支中, 我们知道 t 是一个 leaf, 因此我们可以用 leaf-val 函数得到它的值。

在这两个函数的 else 子句中,我们知道 t 不是 leaf, 由于 t 的类型是 Tree ,通过排除法我们可以确定 t 一定是一个 node 。 因此,我们可以使用诸如 node-left 和 node-right 这样的访问器,将 t 作为输入。

Typed Racket 使用来自谓词检查的信息对 cond 子句的主体进行类型检查的过程被称为 occurrence typing , 在 出现类型 中有详细描述。

2.2 类型错误

当 Typed Racket 检测到模块中的类型错误时,它会在运行程序之前引发一个错误。

Example:
> (add1 "not a number")

eval:9:0: Type Checker: type mismatch

  expected: Number

  given: String

  in: "not a number"
