5.14 Garbage Collection Callbacks

 (require ffi/unsafe/collect-callback) package: base
ffi/unsafe/collect-callback 库提供了注册受限回调的函数,这些回调在垃圾收集之前和之后运行。

Added in version 7.0.0.9 of package base.

procedure

(unsafe-add-collect-callbacks pre post) → any/c

  pre : (vectorof vector?)
  post : (vectorof vector?)
注册外国函数的描述,以便在垃圾收集之前和之后调用。 外来函数不得分配垃圾收集的内存,而且它们的调用方式是不分配的, 这就是为什么 pre_desc 和 post_desc 是函数描述而不是 thunks。

一个描述是一个向量的向量,其中每个内向量都描述了一个单一的调用,并且调用是依次进行的。 每个调用向量以一个符号开始,表示要调用的外国函数的协议。以下协议被支持: 支持的协议集显然是任意的、异想天开的,足以让 DrRacket 显示一个垃圾收集图标。

  • 'int->void 对应于 void (*)(int) 。

  • 'ptr_ptr_ptr->void 对应于 void (*)(void*, void*, void*) 。

  • 'ptr_ptr->save 对应于 void* (*)(void*, void*, void*), 但是结果被记录为当前的 “save” 值。当前的 “save” 值以 NULL 开始。

  • 'save!_ptr->void 对应于 void (*)(void*, void*), 但是只有当当前的 “save” 值不是一个 NULL 指针, 并且传递该指针作为函数的第一个参数(这样只有一个额外的参数是来自描述向量的)。

  • 'ptr_ptr_ptr_int->void 对应于 void (*)(void*, void*, void*, int) 。

  • 'ptr_ptr_float->void 对应于 void (*)(void*, void*, float) 。

  • 'ptr_ptr_double->void 对应于 void (*)(void*, void*, double) 。

  • 'ptr_ptr_ptr_int_int_int_int_int_int_int_int_int->void 对应于 void (*)(void*, void*, void*, int, int, int, int, int, int, int, int, int) 。

  • 'osapi_ptr_int->void 对应于 void (*)(void*, int), 但在 Windows 下使用 stdcall 的调用习惯。

  • 'osapi_ptr_ptr->void 对应于 void (*)(void*, void*), 但在 Windows 上使用 stdcall 调用约定。

  • 'osapi_ptr_int_int_int_int_ptr_int_int_long->void 对应于 void (*)(void*, int, int, int, int, void*, int, int, long), 但是在 Windows 上使用 stdcall 调用约定。

在协议符号之后,向量应该包含一个指向外国函数的指针,然后是函数的每个参数的一个元素。 指针值的表示与 ffi/unsafe 定义的 _pointer 表示一样。

其结果是一个用于 unsafe-remove-collect-callbacks 的键。 如果键变得不可访问,那么回调将被自动删除(但要注意,前回调已经执行,后回调没有执行)。

procedure

(unsafe-remove-collect-callbacks key) → void?

  key : any/c
解除先前通过调用 unsafe-add-collect-callbacks 注册的前和后收集回调,该调用返回 v 。
