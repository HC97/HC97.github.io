5.9 Speculatively Atomic Execution

 (require ffi/unsafe/try-atomic) package: base
ffi/unsafe/try-atomic 库支持原子执行, 如果执行时间过长或某些外部事件导致尝试被放弃,可以暂停并以非原子模式恢复。

procedure

(call-as-nonatomic-retry-point thunk) → any

  thunk : (-> any)
以原子模式调用 thunk (参见 start-atomic 和 end-atomic) 同时允许 thunk 使用 try-atomic 。 任何用 try-atomic 开始的不完整的计算都会在 thunk 返回后以非原子方式运行。 thunk 的结果被用作 call-as-nonatomic-retry-point 的结果。

procedure

(try-atomic thunk    
  default-val    
  [#:should-give-up? give-up-proc    
  #:keep-in-order? keep-in-order?]) → any
  thunk : (-> any)
  default-val : any/c
  give-up-proc : (-> any/c) = run-200-milliseconds
  keep-in-order? : any/c = #t
在调用 call-as-nonatomic-retry-point 的动态范围内, 尝试以现有的原子模式运行 thunk 。 give-up-proc 过程被定期调用,以确定是否应该放弃原子模式; 默认的 give-up-proc 在 200 毫秒后返回 true。 如果原子模式被放弃,计算将被暂停,并返回 default-val 。 计算稍后会被包围的 call-as-nonatomic-retry-point 调用恢复。

如果 keep-in-order? 为真,那么如果 try-atomic 在先前的计算被暂停后被调用, 而同一个 call-as-nonatomic-retry-point 调用被暂停, 那么 thunk 会被 call-as-nonatomic-retry-point 立即排队等待完成,并返回 default-val 。

give-up-proc 回调只在原子模式嵌套级别(见 start-atomic, start-breakable-atomic 和 call-as-atomic) 与调用 try-atomic 时相同的情况下被轮询。

如果 thunk 使用 (default-continuation-prompt-tag) 中止了当前的 continuation, 那么该中止将被中止,并由周围的 call-as-nonatomic-retry-point 点恢复。 使用任何其他提示标签或续集逃往 thunk 的上下文都会被阻断(使用 dynamic-wind) , 并且仅仅从 thunk 返回 (void) 。
