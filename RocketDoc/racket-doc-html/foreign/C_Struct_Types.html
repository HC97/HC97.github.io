3.7 C Struct Types

procedure

(make-cstruct-type types    
  [abi    
  alignment    
  malloc-mode]) → ctype?
  types : (non-empty-listof ctype?)
  abi : (or/c #f 'default 'stdcall 'sysv) = #f
  alignment : (or/c #f 1 2 4 8 16) = #f
  malloc-mode : 
(one-of/c 'raw 'atomic 'nonatomic 'tagged
           'atomic-interior 'interior
           'stubborn 'uncollectable 'eternal)
   = 'atomic
原始类型构造函数用于创建新的 C 结构类型。 这些类型实际上是新的原始类型;它们没有相关的转换函数。 用于结构的相应 Racket 对象是指针,但当使用这些类型时,使用的是指针 refers to 的值,而不是指针本身。 这个值基本上是由一些字节组成的,根据给定的 types 列表列表可以知道。

如果 alignment 是 #f, 那么 types 中每个类型的自然对齐方式被用于其在结构类型中的对齐。 否则, alignment 将用于所有结构类型成员。

当类型的一个实例被分配来表示函数调用的结果时,会使用 malloc-mode 参数。 这种分配模式 不 用于 callback 的参数, 因为在这种情况下会使用在 C 堆栈上分配的临时空间(可能是通过调用惯例)。

Changed in version 7.3.0.8 of package base: 增加了 malloc-mode 参数。

procedure

(_list-struct [#:alignment alignment    
  #:malloc-mode malloc-mode]    
  type ...+) → ctype?
  alignment : (or/c #f 1 2 4 8 16) = #f
  malloc-mode : 
(one-of/c 'raw 'atomic 'nonatomic 'tagged
          'atomic-interior 'interior
          'stubborn 'uncollectable 'eternal)
   = 'atomic
  type : ctype?
使用 make-cstruct-type 函数构建一个结构体类型,并将其包裹在一个类型中, 该类型将结构体作为其组成部分的列表进行调取。 注意,结构体的空间必须使用带有 malloc-mode 的 malloc 来分配; _list-struct 类型的转换器会立即从分配的空间中分配并使用一个列表,所以效率很低。 使用下面的 define-cstruct 可以获得更有效的方法。

Changed in version 6.0.0.6 of package base: 增加了 #:malloc-mode 。

syntax

(define-cstruct id/sup ([field-id type-expr field-option ...] ...)
  property ...)
 
id/sup = _id
  | (_id _super-id)
     
field-option = #:offset offset-expr
     
property = #:alignment alignment-expr
  | #:malloc-mode malloc-mode-expr
  | #:property prop-expr val-expr
  | #:no-equal
  | #:define-unsafe
 
  offset-expr : exact-integer?
  alignment-expr : (or/c #f 1 2 4 8 16)
  malloc-mode-expr : 
(one-of/c 'raw 'atomic 'nonatomic 'tagged
          'atomic-interior 'interior
          'stubborn 'uncollectable 'eternal)
  prop-expr : struct-type-property?
定义了一个新的 C 结构类型,但与 _list-struct 不同的是, 产生的类型是以二进制形式处理 C 结构,而不是将它们编入 Racket 值。 语法类似于 define-struct,为原始结构值(是指针对象)提供访问函数; _id 必须以 _ 开头,最多可以为一个字段提供一个 #:offset , 最多可以提供一个 #:alignment 或 #:malloc-mode 。 如果没有提供 _super-id ,那么必须至少指定一个字段。

由此产生的绑定关系如下:

  • _id : 这个结构的新 C 类型。

  • _id-pointer: 当使用指向此结构值的指针时应使用的指针类型。

  • _id-pointer/null: 与 _id-pointer 类似,但允许 NULL 指针(在 Racket 中用 #f 表示)。

  • id?: 新类型的谓词。

  • id-tag: 与实例一起使用的标签对象。 标签对象可以是 id 的符号形式, 也可以是包含 id 符号和其他符号的符号列表,如 super-id 符号。

  • make-id : 一个构造函数,它期望每个字段有一个参数。

  • id-field-id : 每个 field-id 的访问函数; 如果字段有一个 C 结构类型,那么访问函数的结果是一个指向包围结构中的字段的指针,而不是字段的副本。

  • set-id-field-id! : 一个针对每个 field-id 的突变器函数。

  • id-field-id-offset : 如果 #:define-unsafe 存在,每个 field-id 的绝对偏移量,以字节为单位。

  • unsafe-id-field-id : 如果 #:define-unsafe 存在的话,每个 field-id 的一个不安全访问函数。

  • unsafe-set-id-field-id! : 如果存在 #:define-unsafe ,则为每个 field-id 提供一个不安全的变体函数。

  • id: 与 struct-out 或 match 兼容的结构类型信息 (但不是 struct 或 define-struct); 目前,只有在没有指定 super-id 时,该信息才是正确的。

  • id->list, list->id : 将结构体转换为字段值列表的函数,反之亦然。

  • id->list*, list*->id : 与 id->list, list->id 类似,但属于结构体的字段被递归地解压为列表或从列表打包。

  • struct:cpointer:id: 只有在指定 #:property 时才会出现–对应于反映属性的包装器的结构类型(见下文)。

  • make-wrap-id: 仅当指定了 #:property 时–一个接收 cpointer 并返回持有 cpointer 的包装器结构的函数。

新类型的对象实际上是 C 指针,其类型标签是 id 的符号形式或包含 id 的符号形式的列表。 由于结构体被实现为指针,它们可以被用于外国函数的 _pointer 输入:它们的地址将被使用。 为了使之更安全一些,相应的 cpointer 类型被定义为 _id-pointer 。 当期望使用指针时,不应该使用 _id 类型,因为它将导致结构被复制而不是使用指针值,导致内存损坏。

结构中的字段偏移量通常是自动计算的,但是字段的偏移量可以用 #:offset 来指定。 为一个字段指定 #:offset 会影响到所有剩余字段的默认偏移量的计算。

新类型的实例通常不是 Racket 结构实例。 然而,如果至少指定了一个 #:property 修改器, 那么结构创建和来自 _id 变体的胁迫将非 NULL 的 C 指针表示包在一个具有指定属性的 Racket 结构中。 封装的 Racket 结构也有一个 prop:cpointer 属性,因此封装的 C 指针可以和未封装的 C 指针一样被处理。 如果提供了一个 super-id ,并且它对应于具有包装器结构类型的 C 结构类型, 那么包装器结构类型就是 super-id 的包装器结构类型的一个子类型。 如果指定了 #:property 修改器,没有指定 #:no-equal , 并且如果没有指定任何 #:property 的 prop:equal+hash , 那么 prop:equal+hash 属性将自动实现包装器结构类型使用 ptr-equal? 。

如果第一个字段本身是一个 C 结构类型,那么除了新的标签之外,它的标签也将被使用。 这个特性支持对象继承的常见情况,即子结构是通过拥有一个作为其超级结构的第一个字段而形成的。 子结构的实例可以被认为是超级结构的实例,因为它们共享相同的初始布局。 使用初始C结构字段的标签意味着在 Racket 中可以实现相同的行为; 例如,超级结构的访问器和突变器可以用于新的子结构。请看下面的例子。

提供 super-id 是使用一个名为 super-id 的初始字段并使用 _super-id 作为其类型的简写。 因此,新结构除了使用自己的标签外,还将使用 _super-id 的标签, 这意味着 _id 的实例可以作为 _super-id 的实例使用。 除了语法上的糖,使用这种语法时,构造函数也是不同的:构造函数不再期待第一个参数是 _super-id 的实例,而是期待 _super-id 的每个字段的参数, 此外还有新字段的参数。 构造函数的这一调整,再次与使用 define-struct 的超类型相类似。

结构使用 malloc 分配, malloc-mode-expr 的结果,默认为 'atomic 。 (这种分配模式不适用于 callback 的参数;另见 define-cstruct-type 。) 默认的 'atomic 分配意味着垃圾收集器忽略了结构的内容; 因此,结构字段只能容纳非指针值、指向 GC 控制之外的内存的指针, 以及指向不可移动的 GC 管理值的其他可达指针(比如用 malloc 和 'internal 或 'internal-atomic 分配的指针)。

作为一个例子,考虑下面的C代码:

  typedef struct { int x; char y; } A;

  typedef struct { A a; int z; } B;

  

  A* makeA() {

    A *p = malloc(sizeof(A));

    p->x = 1;

    p->y = 2;

    return p;

  }

  

  B* makeB() {

    B *p = malloc(sizeof(B));

    p->a.x = 1;

    p->a.y = 2;

    p->z   = 3;

    return p;

  }

  

  char gety(A* a) {

    return a->y;

  }

使用简单的 _list-struct, 你可能期望这段代码能够工作:

(define makeB
  (get-ffi-obj 'makeB "foo.so"
    (_fun -> (_list-struct (_list-struct _int _byte) _int))))
(makeB) ; should return '((1 2) 3)

这里的问题是, makeB 返回一个指向结构的指针,而不是结构本身。 下面的代码如期工作:

(define makeB
  (get-ffi-obj 'makeB "foo.so" (_fun -> _pointer)))
(ptr-ref (makeB) (_list-struct (_list-struct _int _byte) _int))

如上所述, _list-struct 应该在效率不是问题的情况下使用。 我们继续使用 define-cstruct, 首先为 A 定义一个类型,这样就可以使用 makeA:

(define-cstruct _A ([x _int] [y _byte]))
(define makeA
  (get-ffi-obj 'makeA "foo.so"
    (_fun -> _A-pointer))) ; using _A is a memory-corrupting bug!
(define a (makeA))
(list a (A-x a) (A-y a))
; produces an A containing 1 and 2

使用 gety 也很简单:

(define gety
  (get-ffi-obj 'gety "foo.so"
    (_fun _A-pointer -> _byte)))
(gety a) ; produces 2

现在我们为 B 定义另一个 C 结构,并使用它来公开 makeB :

(define-cstruct _B ([a _A] [z _int]))
(define makeB
  (get-ffi-obj 'makeB "foo.so"
    (_fun -> _B-pointer)))
(define b (makeB))

我们可以使用天真的方法访问 b 的所有值:

(list (A-x (B-a b)) (A-y (B-a b)) (B-z b))

但这是低效的,因为它在每次访问时都会分配和复制一个 A 的实例。 检查标签 (cpointer-tag b) ,我们可以看到 A 的标签被包括在内, 所以我们可以简单地使用它的访问器和突变器,以及任何被定义为接受 A 指针的函数:

(list (A-x b) (A-y b) (B-z b))
(gety b)

在Racket中构建一个 B 实例需要分配一个临时的 A 结构:

(define b (make-B (make-A 1 2) 3))

为了使之更有效,我们改用另一种 define-cstruct 语法,它创建了一个构造函数,期望得到超级字段和新字段的参数:

(define-cstruct (_B _A) ([z _int]))
(define b (make-B 1 2 3))

Changed in version 6.1.1.8: 添加了 #:offset 字段的偏移量。
Changed in version 6.3.0.13: 添加了 #:define-unsafe 。

procedure

(compute-offsets types [alignment declare])

 → (listof exact-integer?)
  types : (listof ctype?)
  alignment : (or/c #f 1 2 4 8 16) = #f
  declare : (listof (or/c #f exact-integer?)) = '()
给出一个 C 结构类型的列表,返回这些类型的偏移量。

types 列表描述了一个 C 结构类型,与 make-cstruct-type 中的列表相同。

C 结构的对齐方式是用 alignment 设置的,行为与 make-cstruct-type 相同。

明确的位置可以用 declare 来设置。 如果提供,它是一个与 types 相同长度的列表。 在每个索引处,如果提供了一个数字,那么该类型就在该偏移处。否则,该类型是在偏移量之后的 alignment 字节。

Examples:
> (compute-offsets (list _int _bool _short))

'(0 4 8)

> (compute-offsets (list _int _bool _short) 1)

'(0 4 8)

> (compute-offsets (list _int _int _int) #f (list #f 5 #f))

'(0 5 12)

Added in version 6.10.1.2 of package base.
