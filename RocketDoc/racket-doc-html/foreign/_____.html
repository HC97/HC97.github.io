3.10 枚举和掩码

尽管下面的构造函数被描述为程序,但它们是作为语法实现的,这样错误信息就可以在语法上下文暗示的地方报告一个类型名称。

procedure

(_enum symbols [basetype #:unknown unknown]) → ctype?

  symbols : list?
  basetype : ctype? = _ufixint
  unknown : any/c = (lambda (x) (error ....))
接收一个符号列表并生成一个枚举类型。 枚举在给定的 symbols 列表中的一个符号和相应的整数之间进行映射,从 0 开始计算。

要调用一个以枚举为参数的外来函数,只需提供所需枚举的符号作为参数。

; example sdl call
(sdl-create-window "title" ... 'SDL_WINDOW_OPENGL)

列表中的 symbols 也可以通过在符号后面加上 '= 和一个精确的整数来设置符号的值。 例如,列表 '(x y = 10 z) 将 'x 映射到 0, 'y 映射到 10, 'z 映射到 11 。

basetype 参数指定了要使用的基本类型。

unknown 参数指定从外国转换未知整数的结果:它可以是一个应用于整数的单参数函数,也可以是一个要返回的值。 默认是抛出一个异常。

Examples:
; example from snappy-c.h
> (define _snappy_status
    (_enum '(ok = 0
             invalid_input
             buffer_too_small)))

注意,默认的基类是 _ufixint 。 这与 C 语言的枚举不同,C 语言可以在 _fixint 中使用任何值。 任何使用负值的 _enum 应该使用 _fixint 作为基类型。

Example:
> (define _negative_enum
    (_enum '(unkown = -1
             error = 0
             ok = 1)
           _fixint))

procedure

(_bitmask symbols [basetype]) → ctype?

  symbols : (or symbol? list?)
  basetype : ctype? = _uint
类似于 _enum, 但是产生的映射将一个符号列表翻译成一个数字,然后再翻译回来, 在单个符号的值上使用 bitwise-ior ,其中单个符号等同于一个只包含符号的列表。

换句话说,要调用一个使用比特掩码参数的外国函数,只需调用带有想要的标志列表的程序。

; example call from curl_global_init in curl.h
(curl-global-init '(CURL_GLOBAL_SSL CURL_GLOBAL_WIN32))

当一个符号没有给定的值时(通过 symbols 中符号后的 '= ), 它的值是比前一个符号的赋值大 2 的次幂(或第一个符号为 1 )。

默认的 basetype 是 _uint, 因为高位经常被用于标志。

Examples:
; example from curl.h
> (define _curl_global_flag
    (_bitmask `(CURL_GLOBAL_SSL = 1
                CURL_GLOBAL_WIN32 = 2
                CURL_GLOBAL_ALL = 3
                CURL_GLOBAL_NOTHING = 0
                CURL_GLOBAL_DEFAULT = 3
                CURL_GLOBAL_ACK_EINTR = 4)))
; example from XOrg
> (define _Modifiers
    (_bitmask '(ShiftMask = 1
                LockMask = 2
                ControlMask = 4
                Mod1Mask = 8
                Mod2Mask = 16
                Mod3Mask = 32
                Mod4Mask = 64
                Mod5Mask = 128
                Button1Mask = 256
                Button2Mask = 512
                Button3Mask = 1024
                Button4Mask = 2048
                Button5Mask = 4096
                Any = 32768)))
