5.3 安全的 C 向量

不安全的 cpointer-has-tag? 和 cpointer-push-tag! 操作管理标签以区分指针类型。

procedure

(_cpointer tag    
  [ptr-type    
  racket-to-c    
  c-to-racket]) → ctype?
  tag : any/c
  ptr-type : (or/c ctype? #f) = _pointer
  racket-to-c : (or/c (any/c . -> . any/c) #f) = values
  c-to-racket : (or/c (any/c . -> . any/c) #f) = values
(_cpointer/null tag    
  [ptr-type    
  racket-to-c    
  c-to-racket]) → ctype?
  tag : any/c
  ptr-type : (or/c ctype? #f) = _pointer
  racket-to-c : (or/c (any/c . -> . any/c) #f) = values
  c-to-racket : (or/c (any/c . -> . any/c) #f) = values
构建一个 C 语言的指针类型, _tag, 在转换到 Racket 时得到一个特定的标签,在去 C 语言时只接受这样的标签指针。 对于任何可选的参数, #f 被视为与参数的默认值相同。

ptr-type 被用作 _tag 的基本指针类型。 ptr-type 的值必须被表示为指针。

尽管任何值都可以被用作 tag, 但根据惯例,它是一个类型名的符号形式–没有前面的下划线。 例如,一个指针类型 _animal 通常会使用 'animal 作为标签。

指针标签用 cpointer-has-tag? 检查,用 cpointer-push-tag! 改变, 这意味着其他标签在现有的指针值上被保留。 具体来说,如果一个基础 ptr-type 被给定并且本身是由 _cpointer 产生的, 那么新类型将处理除了 ptr-type 的标签之外还有新标签的指针。 当标签是一对时,它的第一个值被用于打印,所以最近推送的标签(对应于继承类型)被显示。

一个要用作 _tag 值的 Racket 值首先被传递给 racket-to-c, 其结果必须是一个被标记为 tag 的指针。 类似地,一个要作为 _tag 值返回的 C 值最初被表示为带有 tag 标签的指针, 但随后它被传递给 c-to-racket 以获得 Racket 表示。 因此, _tag 值在C层次由一个指针表示, 但(与给定的 ptr-type 不同)它在 Racket 层次可以有任何表示, 如由 racket-to-c 和 c-to-racket 实现的。

_cpointer/null 函数与 _cpointer 类似, 除了它能容忍 NULL 指针进入 C 语言和返回。 注意, NULL 指针在 Racket 中表示为 #f ,所以它们没有被标记。

syntax

(define-cpointer-type _id)

(define-cpointer-type _id #:tag tag-expr)
(define-cpointer-type _id ptr-type-expr)
(define-cpointer-type _id ptr-type-expr #:tag tag-expr)
(define-cpointer-type _id ptr-type-expr
                      racket-to-c-expr c-to-racket-expr)
(define-cpointer-type _id ptr-type-expr
                      racket-to-c-expr c-to-racket-expr
                      #:tag tag-expr)
_cpointer 和 _cpointer/null 的宏版本,使用定义的名称作为标签符号, 同时定义一个谓词。 _id 必须以 _ 开头。

可选的表达式产生给 _cpointer 的可选参数。

除了将 _id 定义为由 _cpointer 产生的类型外, _id/null 还被绑定到由 _cpointer/null 类型产生的类型。 最后, id? 被定义为一个谓词, id-tag 被定义为一个访问器,用来获取一个标签。 如果提供,标签是 tag-expr, 否则是 id 的符号形式。

procedure

(cpointer-predicate-procedure? v) → boolean?

  v : any/c
如果 v 是由 define-cpointer-type 或 define-cstruct 生成的谓词过程, 返回 #t ,否则返回 #f 。

Added in version 6.6.0.1 of package base.

procedure

(cpointer-has-tag? cptr tag) → boolean?

  cptr : cpointer?
  tag : any/c
(cpointer-push-tag! cptr tag) → void?
  cptr : cpointer?
  tag : any/c
这两个函数将指针标签视为标签的列表。 正如 指针函数 所述,除了 Racket 代码使用它来区分指针外, 指针标签没有任何作用;这些函数将标签值视为标签列表,这使得构造可以被视为其他指针类型的指针类型成为可能, 主要用于通过 upcasts 实现继承(当一个结构包含一个超级结构作为其第一个元素时)。

cpointer-has-tag? 函数检查给定的 cptr 是否有 tag。 当一个指针的 tag 是 eq? 到 tag 或者是一个包含(在 memq 的意义上) tag 的列表时,它就有一个 tag 。

cpointer-push-tag! 函数将给定的 tag 值推送到 cptr 的标签上。 这个操作的主要属性是: (a) 推送任何标签将使以后对 cpointer-has-tag? 的调用在这个标签上成功, (b) 被推送的标签将在打印指针时被使用(直到有新的值被推送)。 从技术上讲,如果没有设置标签,推送标签将简单地设置它, 否则将其推送到一个现有的列表或现有的值上(被视为一个单元素列表)。
