5.18 文件安全-防护检查

 (require ffi/file) package: base

procedure

(security-guard-check-file who path perms) → void?

  who : symbol?
  path : path-string?
  perms : (listof (or/c 'read 'write 'execute 'delete 'exists))
检查 (current-security-guard) 是否允许以 perms 的权限访问 path 指定的文件。 参见 make-security-guard 获取更多关于 perms 的信息。

符号 who 应该是代表其进行安全检查的函数的名称;它被传递给安全卫士,在访问被拒绝的错误中使用。

procedure

(_file/guard perms [who]) → ctype?

  perms : (listof (or/c 'read 'write 'execute 'delete 'exists))
  who : symbol? = '_file/guard
像 _file 和 _path, 但从 Racket 到 C 的转换首先使用 path->complete-path 完成路径, 然后使用 cleanse-path 进行清理,然后检查当前的安全卫士是否允许在产生的完整路径上使用 perms 访问。 作为一个输出值,与 _path 。

value

_file/r : ctype?

value

_file/rw : ctype?

分别等同于 (_file/guard '(read) '_file/r) 和 (_file/guard '(read write) '_file/rw) 。

procedure

(security-guard-check-file-link who    
  path    
  dest) → void?
  who : symbol?
  path : path-string?
  dest : path-string?
检查 (current-security-guard) 是否允许将 path 作为一个链接 dest 来创建。 符号 who 与 security-guard-check-file 相同。

Added in version 6.9.0.5 of package base.

procedure

(security-guard-check-network who    
  host    
  port    
  mode) → void?
  who : symbol?
  host : (or/c string? #f)
  port : (or/c (integer-in 1 65535) #f)
  mode : (or/c 'client 'server)
检查 (current-security-guard) 是否允许在服务器或客户模式下的 host 和 port 的网络访问,如 mode 所指定。 符号 who 与 security-guard-check-file 相同。

Added in version 6.9.0.5 of package base.
