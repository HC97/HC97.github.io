5.6 Allocation and Finalization

 (require ffi/unsafe/alloc) package: base
ffi/unsafe/alloc 库提供了一些实用工具,以确保通过外来函数分配的值被可靠地去分配。

procedure

((allocator dealloc) alloc) → (or/c procedure? #f)

  dealloc : (any/c . -> . any)
  alloc : (or/c procedure? #f)
产生一个 allocator 过程,其行为类似于 alloc, 但是 allocator 的每个结果 v ,如果不是 #f, 就会被赋予一个终结器,对 v 调用 dealloc —除非该调用已经通过对 v 应用 deallocator (由 deallocator 产生)而被取消。 为 v 注册的任何现有的 dealloc 都被取消了。 如果且仅当 alloc 是 #f, ((allocator alloc) dealloc) 产生 #f 。

产生的 allocator 以 atomic mode 调用 alloc (见 call-as-atomic)。 来自 alloc 的结果是以原子模式接收和注册的,因此只要不引发异常,结果就会被可靠地去分配。

dealloc 过程将以原子模式被调用,它必须遵守与提供给 register-finalizer 的finalizer过程相同的约束。 dealloc 过程本身不需要专门是由 deallocator 产生的 deallocator 。 如果一个 deallocator 被明确地调用,它不需要和 dealloc 一样。

当一个非主 place 退出时, 在所有的 custodian-关闭动作之后, 对于每一个仍然通过 allocator 或 retainer 注册的 dealloc (来自 allocator 或 retainer),要去分配的值被视为立即无法到达。 在这一点上, dealloc 函数是按照其注册的相反顺序调用的。 注意,在一个 dealloc 函数的闭包中的引用并 不 妨碍为任何其他值运行一个 dealloc 函数。 如果去分配需要以与分配相反的顺序进行,请使用 retainer 插入一个新的去分配动作,它将提前运行。

Changed in version 7.0.0.4 of package base: 为 dealloc 增加了原子模式,并改变了非主要地方的退出方式, 以调用所有剩余的 dealloc 。
Changed in version 7.4.0.4: 当 alloc 为 #f 时产生 #f 。

procedure

((deallocator [get-arg]) dealloc) → procedure?

  get-arg : (list? . -> . any/c) = car
  dealloc : procedure?

procedure

((releaser [get-arg]) dealloc) → procedure?

  get-arg : (list? . -> . any/c) = car
  dealloc : procedure?
产生了一个行为类似 dealloc 的 deallocator 过程。 deallocator 以 atomic mode 调用 dealloc (参见 call-as-atomic), 对于它的一个参数, 它取消了最近由 allocator 或 retainer 注册的剩余 deallocator。

可选的 get-arg 过程决定 dealloc 的哪些参数对应于被释放的对象; get-arg 接收传递给 dealloc 的参数列表,所以默认的 car 选择第一个参数。 请注意, get-arg 只能选择 dealloc 的 by-position 参数之一, 尽管 deallocator 会要求并接受与 dealloc 相同的关键字参数,如果有的话。

releaser 过程是 deallocator 的一个同义词。

procedure

((retainer release [get-arg]) retain) → procedure?

  release : (any/c . -> . any)
  get-arg : (list? . -> . any/c) = car
  retain : procedure?
产生一个 retainer 过程,其行为类似于 retain 。 retainer 的行为与由 allocator 产生的 allocator 相同,除了

  • retainer 在注册 release 时不会取消任何现有的 release 或 dealloc 注册;并且

  • release 是为一个值 v 注册的,该值是 retainer 的一个参数, 而不是 allocator 的结果。

可选的 get-arg 过程决定 retainer 的哪些参数(也就是 retain 的哪些参数) 对应于被保留的对象 v; get-arg 接收一个传递给 retain 的参数列表,所以默认 car 选择第一个参数。 注意, get-arg 只能选择一个 by-position 参数来 retain, 尽管 retainer 将要求并接受与 retain 相同的关键字参数,如果有的话。

Changed in version 7.0.0.4 of package base: 为 release 增加了原子模式, 并改变了非主地方的退出,以调用所有剩余的 release 。
