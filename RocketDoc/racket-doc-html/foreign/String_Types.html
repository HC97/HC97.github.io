3.4 String Types

3.4.1 原始的字符串类型

参见 _bytes/nul-terminated 和 _bytes , 用于在字节字符串和 C 的 char* 类型之间进行转换。

value

_string/ucs-4 : ctype?

一个用于 UCS-4 格式的字符串的类型,包括一个 nul 结束符。 像往常一样,该类型将 #f 视为 NULL ,反之亦然。

对于 Racket 的 CS 实现,对外方的 Racket 字符串的转换是对 Racket 表示的复制,其中的复制由垃圾回收器管理。

对于 Racket 的 BC 实现,国外的 Racket 字符串的转换与 Racket 字符串表示法共享内存,因为 UCS-4 是这些变体的本地表示格式。 外方指针对应于 Racket 的 C API 中的　mzchar* 类型。

value

_string/utf-16 : ctype?

UTF-16 格式的 Unicode 字符串,包括一个 nul 终止符。 像往常一样,这些类型将 #f 视为 NULL ,反之亦然。

对外方来说,Racket 字符串的转换是 Racket 表示法的拷贝(重新编码),其中的拷贝由垃圾回收器管理。

value

_path : ctype?

简单的 char* 字符串是 nul 结尾的,对应于 Racket 的　path or string　。 像往常一样,该类型将 #f 视为 NULL ,反之亦然。

对于 Racket 的 BC 实现,对外方的 Racket 路径的转换与 Racket 路径表示法共享内存。 否则(对于 CS 实现或 Racket 字符串),对外方的转换会创建一个副本,由垃圾收集器管理。

请注意,通过　current-directory 改变当前目录并不会改变国外库函数所看到的操作系统级别的当前目录。 在将路径传递给国外函数之前,通常应该使用 path->complete-path (它使用 current-directory 参数)将其转换为绝对形式。

value

_symbol : ctype?

简单的 char* 字符串作为 Racket 符号(以 UTF-8 编码,以 nul 结尾),打算作为外国方面的只读。 使用这种类型的返回值被当作符号来处理。

对于 Racket 的 CS 实现来说,对外方的 Racket 符号的转换是对 Racket 表示的复制,其中的复制由垃圾收集器管理。

对于 Racket 的 BC 实现,对外方的 Racket 符号的转换与 Racket 符号表示法共享内存, 但指向符号分配内存的中间部分–因此字符串指针不得在垃圾回收中使用。

3.4.2 固定自动转换的字符串类型

value

_string/utf-8 : ctype?

_string/latin-1 : ctype?
_string/locale : ctype?
在 Racket 方面对应(字符)字符串的类型,在 C 方面对应　char* 字符串的类型。 两者之间的桥梁需要对字符串的内容进行转换。 像往常一样,这些类型将 #f 视为 NULL ,反之亦然。

value

_string*/utf-8 : ctype?

_string*/latin-1 : ctype?
_string*/locale : ctype?
类似于 _string/utf-8　等,但接受的数值范围更广: 允许使用 Racket 字节字符串,并按原样传递,Racket 路径使用 path->bytes　转换。

3.4.3 可变自动转换的字符串类型

在与国外代码交互时, _string/ucs-4 类型很少有用, 而使用 _bytes/nul-terminated 则有些不自然,因为它迫使 Racket 程序员使用字节字符串。 同时,使用　_string/utf-8　等,可能会过早地承诺将字符串作为字节的特定编码。 _string 类型支持 Racket 字符串和 char* 字符串之间的转换,使用参数确定的转换。

value

_string : ctype?

扩展到使用 default-_string-type 类型的参数。 当 _string 被评估时,会参考该参数的值,所以该参数应该在任何使用　_string　的接口定义前设置。

当你应该使用 _path　时,不要使用 _string 。 尽管 C 语言 API 通常将路径表示为字符串, 尽管默认的 _string (通过 default-_string-type) 甚至隐含地将 Racket 路径转换为字符串, 但使用 _path　可以确保将字符串作为路径的正确编码, 而这并不总是 UTF-8。 关于相对路径的注意事项,也请参见 _path 。

parameter

(default-_string-type) → ctype?

(default-_string-type type) → void?
  type : ctype?
一个参数,决定了 _string　的当前含义。 它最初被设置为 _string*/utf-8　。 如果你要改变它,请在定义接口 之前 进行。

3.4.4 其他字符串类型

value

_file : ctype?

和 _path　一样,但是当值从 Racket 到 C 时,会对给定值使用　cleanse-path 。 作为一个输出值,它和 _path　是一样的。

value

_bytes/eof : ctype?

类似于 _bytes 类型,除了国外的返回值 NULL 被翻译成 Racket eof 值。

value

_string/eof : ctype?

类似于 _string 类型,只是国外的返回值 NULL 被翻译成 Racket eof 值。
