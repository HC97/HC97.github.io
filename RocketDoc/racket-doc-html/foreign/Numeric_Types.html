3.2 Numeric Types

value

_int8 : ctype?

_sint8 : ctype?
_uint8 : ctype?
_int16 : ctype?
_sint16 : ctype?
_uint16 : ctype?
_int32 : ctype?
_sint32 : ctype?
_uint32 : ctype?
_int64 : ctype?
_sint64 : ctype?
_uint64 : ctype?
不同大小的基本整数类型。 s 或　u 前缀分别指定有符号或无符号整数;没有前缀的是有符号的。

value

_byte : ctype?

_sbyte : ctype?
_ubyte : ctype?
_sbyte 和 _ubyte 类型分别是 _sint8 和 _uint8　的别名。 _byte 类型和 _ubyte　一样,但是在 Racket 的负值上增加了 256, 可以作为 _sbyte　使用(也就是说,它把有符号的字节转换为无符号的字节)。

value

_wchar : ctype?

_wchar 类型是一个无符号整数类型的别名,比如 _uint16 或 _uint32, 对应于平台的　wchar_t 类型。

Added in version 7.0.0.3 of package base.

value

_word : ctype?

_sword : ctype?
_uword : ctype?
_sword 和 _uword 类型分别是 _sint16 和 _uint16　的别名。 _word 类型和 _uword　一样,但以与 _byte　相同的方式胁迫负值。

value

_short : ctype?

_sshort : ctype?
_ushort : ctype?
_int : ctype?
_sint : ctype?
_uint : ctype?
_long : ctype?
_slong : ctype?
_ulong : ctype?
_llong : ctype?
_sllong : ctype?
_ullong : ctype?
_intptr : ctype?
_sintptr : ctype?
_uintptr : ctype?
基本整数类型的别名。 _short 的别名对应于 _int16　。 _int 的别名对应于　_int32　。 _long 的别名对应于　_int32 或 _int64, 取决于平台的不同。 同样地, _intptr 别名对应于　_int32 或 _int64, 取决于平台。

value

_size : ctype?

_ssize : ctype?
_ptrdiff : ctype?
_intmax : ctype?
_uintmax : ctype?
更多基本整数类型的别名。 _size 和　_uintmax 类型是 _uintptr　的别名, 其余是 _intptr　的别名。

value

_fixnum : ctype?

_ufixnum : ctype?
对于速度重要且知道整数足够小的情况, _fixnum 和 _ufixnum 类型类似于 _intptr 和 _uintptr 但假设数量适合 Racket 的即时整数(即,不是 bignums)。

value

_fixint : ctype?

_ufixint : ctype?
类似于 _fixnum/_ufixnum, 但是基于　_int/_uint 而不是　_intptr/_uintptr, 并且来自 C 的 coercions 被检查为在范围内。

value

_float : ctype?

_double : ctype?
_double* : ctype?
_float 和 _double 类型代表相应的 C 类型。 单精度和双精度的 Racket 数字都可以通过 _float 和 _double　进行转换, 而 _float 和 _double　都将 C 语言的值强制转换成双精度的Racket数字。 类型 _double*　将任何 Racket 实数转换为 C 的 double　。

value

_longdouble : ctype?

在支持 long double 的平台上代表长双数类型,在这种情况下, Racket extflonums 可以转换为 long double　值。
