7 未导出的原始函数

ffi/unsafe 库的一部分是由 Racket 内置的 '#%foreign 模块实现的。 '#%foreign 模块并不打算直接使用,但是它输出了以下程序(除此之外)。

procedure

(ffi-obj objname lib) → ffi-obj?

  objname : bytes?
  lib : (or/c ffi-lib? path-string? #f)
从库中取出一个外来对象,返回一个可以作为 C 语言指针使用的值。 如果 lib 是一个路径或字符串,那么 ffi-lib 被用来创建一个库对象。

procedure

(ffi-obj? x) → boolean?

  x : any/c
(ffi-obj-lib obj) → ffi-lib?
  obj : ffi-obj?
(ffi-obj-name obj) → bytes?
  obj : ffi-obj?
由 ffi-obj 返回的对象的谓词,以及返回其相应的库对象和名称的访问函数。 这些值也可以作为 C 语言的指针对象使用。

procedure

(ctype-basetype type) → (or/c ctype? #f)

  type : ctype?
(ctype-scheme->c type) → procedure?
  type : ctype?
(ctype-c->scheme type) → procedure?
  type : ctype?
由 make-ctype 制作的 C 类型对象的组件的访问器。 ctype-basetype 选择器为原始类型返回一个命名类型的符号, 为 cstructs 返回一个 ctypes 列表,为用户定义的 ctypes 返回另一个 ctype。

procedure

(make-late-will-executor) → will-executor?

创建一个 “late” 遗嘱执行者,只有在没有正常的遗嘱执行者为 v 注册遗嘱的情况下, 才会为 v 准备遗嘱。 此外,对于 Racket 的 BC 实现,在晚期遗嘱执行者为 v 准备遗嘱之前, 对 v 的正常弱引用会被清除,但由 make-late-weak-box 和 make-late-weak-hasheq 创建的晚期弱引用不会被清除。 对于Racket的 CS 实现来说,只有当一个遗嘱不能从任何有迟到遗嘱的值中到达时,才会为 v 准备好; 如果一个值 v 可以从它本身到达(即通过 v 的任何字段,而不是直接的值本身), 一个为 v 准备的 "迟到" 遗嘱就不会准备好。

与正常的遗嘱执行人不同,如果一个迟到的遗嘱执行人变得不可访问,那么它有待决遗嘱的值就会保留在迟到的遗嘱执行人的位置上。

迟到的遗嘱执行器是为了在实现 register-finalizer 时使用。
