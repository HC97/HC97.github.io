5.5 Defining Bindings

 (require ffi/unsafe/define) package: base

syntax

(define-ffi-definer define-id ffi-lib-expr
  option ...)
 
option = #:provide provide-id
  | #:define core-define-id
  | #:default-make-fail default-make-fail-expr
  | #:make-c-id make-c-id
绑定 define-id 作为定义形式,从 ffi-lib-expr 产生的库中提取绑定物。 define-id 的语法是

(define-id id type-expr
  bind-option ...)
 
bind-option = #:c-id c-id
  | #:c-id (unquote c-id-expr)
  | #:wrap wrap-expr
  | #:make-fail make-fail-expr
  | #:fail fail-expr
  | #:variable

define-id 形式通过从 ffi-lib-expr 产生的库中提取外文名称为 c-id 的绑定来绑定 id ,其中 c-id 默认为 id 。 其他选项支持进一步的包装和配置:

  • 在提取的结果被绑定为 id 之前,它被传递给 wrap-expr 的结果, 后者默认为 values 。 诸如 (allocator delete) 或 (deallocator) 这样的表达式作为 wrap-expr 很有用。

  • #:make-fail 和 #:fail 选项是相互排斥的; 如果提供了 make-fail-expr , 它将应用于 'id 以获得 get-ffi-obj 的最后一个参数; 如果提供了 fail-expr ,它将直接作为 get-ffi-obj 的最后一个参数提供。 make-not-available 函数和 make-fail-expr 一样有用, 如果 c-id 在外国库中没有找到,那么在应用 id 的时候就会报告错误。

  • 如果 #:c-id 选项提供了一个标识符参数 c-id, 那么 c-id 就被用作外文名称。 如果参数的形式是 (unquote c-id-expr),那么外文名称就是计算 c-id-expr 的结果。 在这两种情况下, #:make-c-id 参数被忽略。

  • 如果 #:c-id 选项不存在,外文名称就基于 id 。 如果 define-id 是用 #:make-c-id 选项定义的, 那么它将使用 ffi identifier convention 约定来计算外文名,比如将连字符转换成下划线或骆驼大写。 ffi/unsafe/define/conventions 提供了几种约定。 如果 #:make-c-id 不存在,那么 id 就被用作外文名称。

  • 如果给出了 #:variable 关键字, 那么就用 make-c-parameter 代替 get-ffi-obj 来获取外域值。

如果 provide-id 被提供给了 define-ffi-definer, 那么 define-id 也会使用 provide-id 提供其绑定。 provide-protected 形式通常是 provide-id 的一个好选择。

如果 core-define-id 被提供给 define-ffi-definer, 那么 core-define-id 将在 define-id 的扩展中代替 define ,用于每个绑定。

如果 default-make-fail-expr 被提供给 define-ffi-definer, 它将作为 define-id 的默认 #:make-fail 值。

例如,

(define-ffi-definer define-gtk gtk-lib)

将 define-gtk 绑定为从 gtk-lib 中提取 FFI 绑定,因此 gtk_rc_parse 可以被绑定为

(define-gtk gtk_rc_parse (_fun _path -> _void))

如果没有找到 gtk_rc_parse , 那么 define-gtk 会立即报告一个错误。 如果 define-gtk 被定义为

(define-ffi-definer define-gtk gtk-lib
   #:default-make-fail make-not-available)

那么如果 gtk_rc_parse 在 gtk-lib 中找不到,那么只有在调用 gtk_rc_parse 时才会报告错误。

Changed in version 6.9.0.5 of package base: 增加了 #:make-c-id 参数。
Changed in version 8.4.0.5: 增加了 #:variable 选项。 增加了 #:c-id 参数的 unquote 变体。

procedure

(make-not-available name) → procedure?

  name : symbol?
返回一个可以接受任意数量参数的存储过程,包括关键字参数,并报告来自 name 的错误信息。 该函数用于与 #:make-fail 或 #:default-make-fail 在 define-ffi-definer 中使用。

Changed in version 8.3.0.5 of package base: 增加了对关键字参数的支持。

syntax

(provide-protected provide-spec ...)

相当于 (provide (protect-out provide-spec ...)) 。 在 define-ffi-definer 中, provide-protected 标识符对 #:provide 很有用。

5.5.1 FFI 标识符约定

 (require ffi/unsafe/define/conventions) package: base

本模块提供了几个 FFI identifier conventions , 供 define-ffi-definer 中的 #:make-c-id 使用。 一个 FFI identifier convention 是将一个标识符转换为另一个标识符的任何 syntax transformer 。

Added in version 6.9.0.5 of package base.

syntax

convention:hyphen->underscore

将标识符中的连字符转换为下划线的约定。 例如,标识符 gtk-rc-parse 将转换为 gkt_rc_parse 。

(define-ffi-definer define-gtk gtk-lib
  #:make-c-id convention:hyphen->underscore)
(define-gtk gtk-rc-parse (_fun _path -> _void))

syntax

convention:hyphen->camelcase

与 convention:hyphen->underscore 类似,但将标识符转换为骆驼大小写, 遵循 string-titlecase 函数。 例如,标识符 camelCaseVariable 将转换为 came-case-variable 。

(define-ffi-definer define-calib camel-lib
  #:make-c-id convention:hyphen->camelcase)
(define-calib camel-case-variable (_fun -> _void))
