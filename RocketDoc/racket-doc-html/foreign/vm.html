5.20 虚拟机基元

 (require ffi/unsafe/vm) package: base
ffi/unsafe/vm 库提供了对用于实现 Racket 的底层虚拟机中功能的访问。

Added in version 7.6.0.7 of package base.

procedure

(vm-primitive name) → any/c

  name : symbol?
访问运行中的 Racket 虚拟机级别的基元值,如果名称不是基元的 name,则返回 #f 。

虚拟机基元是可以在 linklet 主体中引用的基元。基元的具体集合取决于虚拟机。 racket/base 级别或甚至 '#%kernel 级别的许多 "基元" 都不是虚拟机级别的基元。 例如,如果 'eval 作为一个基元可用,它就不是来自 racket/base 的 eval 。

一般来说,基元是不安全的,只有在对 Racket 的实现有足够了解的情况下才能使用。下面是目前可用的虚拟机的一些提示:

  • (system-type 'vm) 是 'racket — 这个虚拟机中的基元大多与 racket/base 和 racket/unsafe/ops 等库中的基元相同。 因此,用 vm-primitive 访问虚拟机基元很少有用。

  • (system-type 'vm) 是 'chez-scheme — 这个虚拟机中的基元是 Chez Scheme 基元, 除了被 Racket 兼容层所替代。 'eval 基元是 Chez Scheme 的 eval 。

    小心直接调用在内部使用 Chez Scheme 参数或 dynamic-wind 的 Chez Scheme 基元。 请注意,尤其是 eval 就是这样一个原语。 问题是,Chez Scheme 的 dynamic-wind 不会自动与 Racket 的连续或线程合作。 要调用这类基元,请使用 call-with-system-wind 基元,它需要一个没有参数的过程, 在连接 Chez Scheme 的 dynamic-wind 和 Racket 的连续和线程的上下文中运行。举例来说、

    (define primitive-eval (vm-primitive 'eval))
    (define call-with-system-wind (vm-primitive 'call-with-system-wind))
    (define (vm-eval s)
      (call-with-system-wind
       (lambda ()
        (primitive-eval s))))

    是 Chez Scheme 上实现 vm-eval 的方式。

    符号、数字、布尔运算、对、向量、方框、字符串、字节字符串(即字节向量)和结构(即记录) 在 Racket 和 Chez Scheme 之间可以互换。 Chez Scheme 程序是一个 Racket 程序,但不是所有的 Racket 程序都是 Chez Scheme 程序。 要从 Chez Scheme 中调用 Racket 程序,请使用 #%app 形式, 该形式在 Chez Scheme 环境中托管 Racket 时被定义。

    请注意,你可以通过Chez Scheme的 $primitive 形式访问 Chez Scheme 的基元, 包括那些被 Racket 的基元所掩盖的基元。 例如, (vm-eval '($primitive call-with-current-continuation)) 访问 call-with-current-continuation 基元, 而不是 Racket 的替换(其中替换与 Racket 的连续和线程一起工作)。

procedure

(vm-eval s-expr) → any/c

  s-expr : any/c
使用最原始的评估器来评估 s-expr :

  • (system-type 'vm) 是 'racket — 使用 compile-linklet 和 instantiate-linklet 。

  • (system-type 'vm) 是 'chez-scheme — 使用 Chez Scheme 的 eval 。

参见 vm-primitive ,了解一些关于虚拟机基元如何与 Racket 交互的信息。
