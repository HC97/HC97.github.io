3.1 基本文档格式

 #lang scribble/base package: scribble-lib
scribble/base 语言提供了可以从 Racket 或 @ 表达式编写的代码中使用的函数和表单。 它本质上扩展了 racket/base, 除了使用 scribble/base 语言的模块中的顶级表单被当作文档内容(像 scribble/doclang)。
scribble/base 的名字也可以作为一个带 require 的库,在这种情况下, 它只提供本节定义的绑定,而且它也不设置阅读器,也不把默认的渲染格式设置为 Racket 手册格式。

这个库所提供的函数,如 title 和 italic, 可以从 Racket 中调用,如

(title #:tag "how-to"
       "How to Design " (italic "Great") " Programs")

它们也可以用 @ 符号来调用,如

  @title[#:tag "how-to"]{How to Design @italic{Great} Programs}

尽管这些程序主要是为了从 @ 模式中使用, 但在 Racket 模式中它们更容易被记录下来(部分原因是我们有 scribble/manual) 。

3.1.1 文档结构

procedure

(title [#:tag tag    
  #:tag-prefix tag-prefix    
  #:style style    
  #:version vers    
  #:date date]    
  pre-content ...+) → title-decl?
  tag : (or/c #f string? (listof string?)) = #f
  tag-prefix : (or/c #f string? module-path?) = #f
  style : (or/c style? #f string? symbol? (listof symbol?)) = #f
  vers : (or/c string? #f) = #f
  date : (or/c string? #f) = #f
  pre-content : pre-content?
产生一个 title-decl ,由 decode 或 decode-part 部分来接收。 decode pre-content (即用 decode-content 解析的内容)提供标题内容。 如果 tag 是 #f, 则会从内容中自动生成一个 tag 字符串。 标签字符串与符号 'part 结合,形成完整的标签。

style 参数可以是一个样式结构,也可以是以下之一: 一个对应于 "普通" 样式的 #f ,一个被用作 style name 的字符串, 一个被用作 style property 的符号,或一个被用作 style properties 的符号列表。 关于样式的信息,请参见 part 。 例如,一个 'toc 的样式会使子章节在多页 HTML 输出中被生成为独立的页面。

tag-prefix 参数被传播到生成的结构中(见 标签)。 如果 tag-prefix 是一个模块路径,则使用 module-path-prefix->string 将其转换为一个字符串。

vers 参数被传播到 title-decl 结构中。 使用 "" 作为 vers 来抑制输出中的版本渲染。

date 参数通过 document-date style property 传播到 title-decl 结构中。 使用 "" 作为 date 来抑制Latex输出中的日期渲染。

节的标题是由 decode-part 自动索引的。 对于索引键,前面的空白和前面的 "A"、"An" 或 "The"(后面有更多的空白)被删除。

procedure

(section [#:tag tag    
  #:tag-prefix tag-prefix    
  #:style style]    
  pre-content ...+) → part-start?
  tag : (or/c #f string? (listof string?)) = #f
  tag-prefix : (or/c #f string? module-path?) = #f
  style : (or/c style? #f string? symbol? (listof symbol?)) = #f
  pre-content : pre-content?
与 title 类似, 但会产生一个深度为 0 的 part-start ,由 decode 或 decode-part 拾取。

procedure

(subsection [#:tag tag    
  #:tag-prefix tag-prefix    
  #:style style]    
  pre-content ...+) → part-start?
  tag : (or/c #f string? (listof string?)) = #f
  tag-prefix : (or/c #f string? module-path?) = #f
  style : (or/c style? #f string? symbol? (listof symbol?)) = #f
  pre-content : pre-content?
与 section 类似, 但会产生一个深度为 1 的 part-start 。

procedure

(subsubsection [#:tag tag    
  #:tag-prefix tag-prefix    
  #:style style]    
  pre-content ...+) → part-start?
  tag : (or/c #f string? (listof string?)) = #f
  tag-prefix : (or/c #f string? module-path?) = #f
  style : (or/c style? #f string? symbol? (listof symbol?)) = #f
  pre-content : pre-content?
与 section 类似, 但产生一个深度为 2 的 part-start 。

procedure

(subsubsub*section [#:tag tag    
  #:tag-prefix tag-prefix    
  #:style style]    
  pre-content ...+) → paragraph?
  tag : (or/c #f string? (listof string?)) = #f
  tag-prefix : (or/c #f string? module-path?) = #f
  style : (or/c style? #f string? symbol? (listof symbol?)) = #f
  pre-content : pre-content?
与 section 类似, 但只是生成一个看起来像无编号的章节标题的段落(用于嵌套太深而无法包含在目录中的情况)。

syntax

(include-section module-path)

要求 module-path , 并返回其 doc 导出(不使任何导入对包围的上下文可见)。 由于这种形式扩展为 require, 它必须在模块或顶层上下文中使用。

procedure

(author auth ...) → block?

  auth : content?
生成一个样式名为 'author 的 paragraph ,以显示文档的作者, 其中每个作者都由 content 表示。 通常情况下,这个函数用在文档开头的 title 之后 。也请参见 author+email 。

例子:

@author{Alice P. Racketeer}

procedure

(author+email author-name    
  email    
  [#:obfuscate? obfuscate?]) → element?
  author-name : content?
  email : string?
  obfuscate? : any/c = #f
结合作者姓名和电子邮件地址。 如果 obfuscate? 为真,那么结果会稍微模糊电子邮件地址,以避免地址采集机器人。

请注意, author+email 不是 author 的替代品。 author+email 函数经常与 author 结合使用。

例子:

@author[(author+email "Bob T. Scribbler" "bob@racket-lang.org")]
3.1.2 区块

procedure

(para [#:style style] pre-content ...) → paragraph?

  style : (or/c style? string? symbol? #f) = #f
  pre-content : pre-content?
创建一个包含 decode 的 pre-content 的 paragraph (即用 decode-paragraph 解析的内容)。

style 参数可以是一个样式, #f 表示一个 "普通" 样式,一个用作 style name 的字符串, 或者一个用作 style name 的符号。 (注意, section 和 para 对符号作为 style 参数的处理是不同的)。

procedure

(nested [#:style style] pre-flow ...) → nested-flow?

  style : (or/c style? string? symbol? #f) = #f
  pre-flow : pre-flow?
创建一个 nested flow ,包含 decode 后的 pre-flow (即用 decode-flow 解析的)。

style 参数的处理与 para 相同。 'inset 和 'code-inset 样式使嵌套流与周围文本相比被嵌入,后者特别适用于嵌入代码。 默认样式由输出目的地指定(对于 HTML 输出,倾向于嵌入文本,对于 Latex 输出,则不嵌入)。

procedure

(centered pre-flow ...) → nested-flow?

  pre-flow : pre-flow?
产生一个内容居中的 nested flow 。

procedure

(margin-note pre-flow ... [#:left? left?]) → block?

  pre-flow : pre-flow?
  left? : any/c = #f
产生一个在边缘排版的 nested flow ,而不是内联。

如果 left? 为 true,那么注释将显示在通常显示的反面(对于 HTML 输出来说是左侧)。 小心与目录的输出相冲突。

procedure

(margin-note* pre-content ... [#:left? left?]) → element?

  pre-content : pre-content?
  left? : any/c = #f
产生一个在边缘排版的 element ,而不是内联。 与 margin-note 不同, margin-note* 可以在段落的中间使用;同时,它的内容被限制在页边的单一段落中。

procedure

(itemlist itm ... [#:style style]) → itemization?

  itm : items/c
  style : (or/c style? string? symbol? #f) = #f
构建一个给定项目序列的 itemization 。 典型的情况是每个 itm 都是由 item 构造的, 但 itm 可以是一个被强制到 item 的 block 。 最后, itm 可以是一个列表或 splice ,其元素被拼接(如果需要,可以递归)到 itemlist 序列中。

style 参数的处理与 para 相同。 'ordered 的样式对项目进行编号,而不是仅仅使用一个子弹。

value

items/c : flat-contract?

一个等同于以下递归规范的契约:

(or/c item? block? (listof items/c) (spliceof items/c))

procedure

(item pre-flow ...) → item?

  pre-flow : pre-flow?
创建一个项目,与 itemlist 一起使用。 decode 后的 pre-flow (即用 decode-flow 解析的)是项目内容。

procedure

(item? v) → boolean?

  v : any/c
如果 v 是 item 产生的项目,返回 #t ,否则返回 #f 。

procedure

(tabular cells    
  [#:style style    
  #:sep sep    
  #:column-properties column-properties    
  #:row-properties row-properties    
  #:cell-properties cell-properties    
  #:sep-properties sep-properties]) → table?
  cells : (listof (listof (or/c block? content? 'cont)))
  style : (or/c style? string? symbol? #f) = #f
  sep : (or/c block? content? #f) = #f
  column-properties : (listof any/c) = '()
  row-properties : (listof any/c) = '()
  cell-properties : (listof (listof any/c)) = '()
  sep-properties : (or/c list? #f) = #f
用给定的 cells 内容创建一个 table ,该表格以行的形式提供,每行有一个单元格的列表。 所有行的长度必须匹配。

使用 cells 中的 'cont 作为一个单元格,在行中继续前一个单元格的内容,否则将用于新单元格的空间。 'cont 不能作为行中的第一个单元格出现。

style 参数的处理与 para 相同。 识别的 style names 和 style properties 的列表见 table 。

默认的样式是在表列之间不留空格。 如果 sep 不是 #f, 它将作为一个新列插入表格的每一列之间; 新列的属性与前一列的属性相同,除非 sep-properties 提供了一个要使用的 style properties 列表。 当 sep 被放在 'cont 之前时,会插入一个 'cont 来代替。

column-properties, row-properties 和 cell-properties 参数为表格的列和单元格指定了 style properties ; 请参阅 table-columns 和 table-cells 关于公认属性的描述。 这些列表不包含可能为 sep 引入的列的条目,当非空时,它们会根据需要进行扩展,以匹配由 cells 决定的表格大小:

  • 如果 column-properties 的长度小于 cells 中每一行的长度, 列表的最后一项将被重复,以使列表足够长。

  • 如果 row-properties 的长度小于 cells 的长度,列表的最后一项将被复制,以使列表足够长。

  • 如果 cell-properties 的长度小于 cells 的行数,那么列表的最后一个元素会被复制,以使列表足够长。 cell-properties 中的每个列表被视为 column-properties 列表,并根据需要进行扩展,以匹配每行的列数。

column-properties 或 row-properties 的每个元素要么是一个 style property 值的列表, 要么是一个被包装成列表的非列表元素。 同样地,对于每个作为 cell-properties 元素的列表,该列表的非列表元素被包装成嵌套的列表。

如果 column-properties 是非空的, 那么它的属性列表被转换成一个 table-columns style property , 它被添加到 style 指定的样式中, 或者与现有的 table-columns style property 合并,以匹配 cells 的列形。 此外,如果 row-properties 或 cell-properties 不是空的, column-properties 的属性列表会与 row-properties 和 cell-properties 的属性列表合并。 如果 row-properties 或 cell-properties 不是空的, 合并后的列表被转换为一个 table-cells 的 style property , 该属性被添加到 style 指定的样式中, 或者与一个现有的 table-cells 的 style property 合并,以匹配 cells 的形状。

如果 column-properties 的样式列表既与 cell-properties 合并, 又转换为 table-columns ,那么 style 将包含一些多余的信息。 在这种情况下, column-attributes 属性将从 table-columns 中使用, 而其他属性将从合并到 table-cells 中使用。

Changed in version 1.1 of package scribble-lib: 增加了 #:column-properties, #:row-properties 和 #:cell-properties 参数。
Changed in version 1.12: 改变了在 'cont 前插入 sep 的方式。
Changed in version 1.28: 增加了 sep-properties ,并使前面的列的属性在没有指定的情况下被一致使用。

例子:
@tabular[#:sep @hspace[1]
         (list (list "soup" "gazpacho")
               (list "soup" "tonjiru"))]
 
@tabular[#:style 'boxed
         #:column-properties '(left right)
         #:row-properties '(bottom-border ())
         (list (list @bold{recipe}   @bold{vegetable})
               (list "caldo verde"   "kale")
               (list "kinpira gobō"  "burdock")
               (list "makizushi"     'cont))]

procedure

(verbatim [#:indent indent] elem ...+) → block?

  indent : exact-nonnegative-integer? = 0
  elem : content?
用打字机字体排版字符串 elem ,字符串 elem 中的换行符指定换行。 字符串 elem 中的连续空格被转换为 hspace ,以确保它们在输出中都被保留下来。 额外的空间(通过 hspace) 由 indent 指定,被添加到每一行的开头。 非字符串 elem 被视为单行内的内容。

字符串 elem 没有 用 decode-content 解码, 所以 (verbatim "---") 渲染成三个连字符而不是一个 em 破折号。 然而,请注意, reading @verbatim 会转换参数中的 @ 语法, 而且这种阅读会在运行时向 verbatim 传递参数之前发生。 为了禁止在 verbatim 参数中使用简单的 @ 符号, verbatim 通常使用 |{...}| 或类似的括号,像这样:

  @verbatim|{

    Use @bold{---} like this...

  }|

它显示为

  Use @bold{---} like this...

而

  @verbatim|{

    Use |@bold{---} like this...

  }|

显示为

  Use — like this...

即使使用 |{...}| 这样的括号,也要注意解析器会删除一致的前导空白; 更多信息见 替代的主体语法 。

也请参见 literal 。

3.1.3 文本样式和内容

procedure

(elem pre-content ... [#:style style]) → element?

  pre-content : pre-content?
  style : (or/c style? string? symbol? #f) = #f
将 decode 后的 pre-content 包裹成一个具有 style 风格的 element

procedure

(italic pre-content ...) → element?

  pre-content : pre-content?
Like elem, but with style 'italic.

procedure

(bold pre-content ...) → element?

  pre-content : pre-content?
Like elem, but with style 'bold.

procedure

(tt pre-content ...) → element?

  pre-content : pre-content?
与 elem 类似,但 'tt 样式用于 pre-content 参数中的直接字符串和符号。

要将 'tt 样式统一应用于所有 pre-content 参数, 请使用 (elem #:style 'tt pre-content ...), 代替。

procedure

(subscript pre-content ...) → element?

  pre-content : pre-content?
Like elem, but with style 'subscript.

procedure

(superscript pre-content ...) → element?

  pre-content : pre-content?
Like elem, but with style 'superscript.

procedure

(smaller pre-content ...) → element?

  pre-content : pre-content?
像 elem 一样,但有 'smaller 。 当使用 smaller 被嵌套时,文字会逐渐变小。

procedure

(larger pre-content ...) → element?

  pre-content : pre-content?
像 elem 一样,但样式为 'larger 。 当使用 larger 嵌套时,文本会逐渐变大。

procedure

(emph pre-content ...) → element?

  pre-content : pre-content?
像 elem 一样,但强调了。 通常情况下,斜体是用来强调的。 emph 的使用可以被嵌套;通常这会导致文本在斜体和正体之间交替出现。

procedure

(literal str ...+) → element?

  str : string?
产生一个完全包含 str 的元素,没有通过 decode-content 进行解码。

请注意, @ 对于一个 literal 调用在向 literal 传递参数之前会进行一些处理。 literal 形式可以用 |{...}| 或类似的括号来禁用 literal 参数中的 @ 符号,像这样:

  @literal|{@bold{---}}|

它显示为

  @bold{---}

参见 verbatim .

3.1.4 图像

procedure

(image path    
  [#:scale scale    
  #:suffixes suffixes    
  #:style style]    
  pre-content ...) → image-element?
  path : (or/c path-string? (cons/c 'collects (listof bytes?)))
  scale : real? = 1.0
  suffixes : (listof #rx"^[.]") = null
  style : (or/c style? string? symbol? #f) = #f
  pre-content : pre-content?
从给定的路径创建一个图像元素。 在不能显示图像的情况下, decode 后的 pre-content 可作为替代文本。

如果 path 是一个相对路径,它是相对于当前目录而言的,而当前目录是由 raco setup 为主文件的目录。 (然而,一般来说,使用 define-runtime-path 来表达相对路径更为可靠)。 path 参数可以是 path->main-collects-relative 的结果,而不是一个路径或字符串。

scale 参数将图像的比例设置为由 path 的内容决定的默认大小。 对于 HTML 输出,产生的 image-element 用 img 或 object (对于 SVG )标签渲染, scale 调整 width 和 height 属性;类名或 style 中的其他属性可以有效地覆盖该尺寸。

suffixes 中的字符串被过滤为给定的渲染器所支持的字符串,然后依次尝试可接受的后缀。 HTML 呈现器支持 ".png", ".gif" 和 ".svg", 而 Latex 呈现器支持 ".png", ".pdf" 和 ".ps" (但 ".ps" 仅在将 Latex 输出转换为 DVI 时有效, 而 ".png" 和 ".pdf" 仅在将 Latex 输出转换为 PDF 时有效)。

注意,当 suffixes 列表为非空时,那么 path 参数不应该有后缀。

Changed in version 1.3 of package scribble-lib: 增加了 #:style 参数。

3.1.5 间隔

procedure

(linebreak) → element?

产生一个强制换行的元素。

procedure

(nonbreaking pre-content ...) → element?

  pre-content : pre-content?
与 elem 类似,但在渲染内容时抑制了换行。

procedure

(hspace n) → element?

  n : exact-nonnegative-integer?
产生一个包含 n 个空格的元素,样式为 'hspace 。

value

~ : string?

一个包含非断裂空间字符的字符串,相当于 'nbsp 作为一个元素。

value

-~- : string?

一个包含非断裂连字符的字符串。

value

?- : string?

一个包含软连字符的字符串(即建议在哪里连字符,以便在渲染时跨行断开它)。

value

._ : element?

生成一个在句子中间结束缩写的句号,而不是一个结束句子的句号(因为后者可能排版时有额外的空格)。 在文件中使用 @._ ,而不是只用 . 来表示一个缩写结束的句号,它前面是小写字母,后面是空格。

参见 .__ 的例子。

value

.__ : element?

生成一个结束句子的句号(可以用额外的空格排版),而不是在句子中间结束缩写的句号。 在文件中使用 @.__ ,而不是只用 . 来表示句子结束时前面有一个大写字母。

下面的例子同时说明了 ._ 和 .__:

#lang scribble/base
My name is Mr@._ T@.__ I pity the fool who can't typeset punctuation.
3.1.6 链接

procedure

(hyperlink url    
  pre-content ...    
  [#:underline? underline?    
  #:style style]) → element?
  url : string?
  pre-content : pre-content?
  underline? : any/c = #t
  style : (or/c style? string? symbol? #f)
   = (if underline? #f "plainlink")
decode 后的 pre-content 被超链接到 url 。 如果没有提供 style ,那么 underline? 将决定链接的呈现方式。

procedure

(url dest) → element?

  dest : string?
生成一个字面的超链接的URL。

procedure

(secref tag    
  [#:doc module-path    
  #:tag-prefixes prefixes    
  #:underline? underline?    
  #:link-render-style ref-style]) → element?
  tag : string?
  module-path : (or/c module-path? #f) = #f
  prefixes : (or/c (listof string?) #f) = #f
  underline? : any/c = #t
  ref-style : (or/c link-render-style? #f) = #f
插入一个对章节标签 tag 的引用。

如果 #:doc module-path 模块路径被提供, 则该 tag 指的是一个由 module-path 决定的前缀的标签。 当 raco setup 渲染文档时,它会根据源模块自动添加一个标签前缀到文档中。 因此,举例来说,如果要引用 Racket 参考文献的某个部分, module-path 应该是 '(lib "scribblings/reference/reference.scrbl") 。

#:tag-prefixes prefixes 前缀参数同样支持选择一个由tag前缀路径决定的特定部分。 当提供了 #:doc 参数时, 那么 prefixes 应该追踪一条由标签前缀组成的子章节的路径来到达 tag 章节。 当没有提供 #:doc 时, prefixes 的路径是相对于任何包围的部分(即产生匹配的最年轻的祖先)。

对于结果的 link-element, 如果 ref-style 不是 #f, 那么它将作为一个 style property 被附加,并影响到链接的渲染。 另外,一个封闭的 part 可以有一个 link-render 样式,调整该 part 内所有链接的渲染样式。 参见 link-element,了解更多关于部分引用的渲染的信息。

如果 underline? 是 #f, 那么一个 style 会被附加到结果的 link-element 上,这样超链接就会在 HTML 中呈现,而没有下划线

在渲染成 HTML 的 Racket 文档中,点击章节标题通常会显示链接到该章节所需的 secref 调用。

Changed in version 1.25 of package scribble-lib: 增加了 #:link-render-style 参数。

procedure

(Secref tag    
  [#:doc module-path    
  #:tag-prefixes prefixes    
  #:underline? underline?    
  #:link-render-style ref-style]) → element?
  tag : string?
  module-path : (or/c module-path? #f) = #f
  prefixes : (or/c (listof string?) #f) = #f
  underline? : any/c = #t
  ref-style : (or/c link-render-style? #f) = #f
和 secref 一样,但如果引用的渲染形式以一个词开头(如 "section"),那么这个词将被大写。

Changed in version 1.25 of package scribble-lib: 增加了 #:link-render-style 参数。

procedure

(seclink tag    
  [#:doc module-path    
  #:tag-prefixes prefixes    
  #:underline? underline?    
  #:indirect? indirect?]    
  pre-content ...) → element?
  tag : string?
  module-path : (or/c module-path? #f) = #f
  prefixes : (or/c (listof string?) #f) = #f
  underline? : any/c = #t
  indirect? : any/c = #f
  pre-content : pre-content?
像 secref 一样,但链接标签是 decode 后的 pre-content ,而不是目标部分的名称。

除了 secref 的参数外, seclink 还支持一个 indirect? 参数。 当 indirect? 参数为真时,那么该节的超链接在 HTML 中的解析可能会被延迟; 参见 'indirect-link 中的 link-element 。

procedure

(other-doc module-path    
  [#:underline? underline?    
  #:indirect indirect]) → element?
  module-path : module-path?
  underline? : any/c = #t
  indirect : (or/c #f content?) = #f
像 secref 一样用于文档的隐含 "top" 标签。 使用这个函数来引用整个手册,而不是 secref, 以防将来手册的标题使用特殊的样式。

如果 indirect 不是 #f, 那么链接在 HTML 中的解析可以被延迟, 就像 seclink 中的 #:indirect? #t 。 indirect 内容的前缀是 "the",后缀是 "document",以生成链接的渲染文本。例如:

  @other-doc['(lib "parsack/parsack/parsack.scrbl")

             #:indirect "Parsec implementation in Racket"]

渲染为超链接的文本:

  the Parsec implementation in Racket documentation

(elemtag t pre-content ...) → element?

  t : (or/c taglet? generated-tag?)
  pre-content : pre-content?
标签 t 指的是 pre-content 的内容形式。

procedure

(elemref t

  [#:underline? underline?]) → element?

3.1.7 索引

procedure

(index words pre-content ...) → index-element?

  words : (or/c string? (listof string?))
  pre-content : pre-content?
给出一个纯文本字符串或一个层次结构的字符串列表,创建一个索引元素,如 '("strings" "plain") , 用于一个更一般的 "strings" 条目下面的一个 "plain" 条目。 作为索引键,字符串使用 clean-up-index-string 进行 "清理"。 字符串(未经清理)也作为文本呈现在索引中。 decode 后的 pre-content 是作为索引目标出现在内联的文本。

当索引条目应指向排版文件中的特定单词或短语(即 pre-content )时,使用 index 。 使用 section-index 来创建一个索引条目,指向一个章节,而不是该章节中的特定单词或短语。

procedure

(index* words word-contents pre-content ...) → index-element?

  words : (listof string?)
  word-contents : (listof list?)
  pre-content : pre-content?
像 index 一样,除了 words 必须是一个列表, 在索引中呈现的内容列表(与 words 平行)被作为 word-contents 提供。

procedure

(as-index pre-content ...) → index-element?

  pre-content : pre-content?
像 index 一样, 但要索引的词是通过在 decode 后的 pre-content 上应用 content->string 来确定的。

procedure

(section-index word ...) → part-index-decl?

  word : string?
创建一个 part-index-decl 通过 decode 与包围的部分相关联。 这些 word 既是键,也是索引中的键的渲染形式。

procedure

(index-section [#:tag tag]) → part?

  tag : (or/c #f string?) = "doc-index"
产生一个显示包围文件的索引的部分。 可选的 tag 参数被用作索引部分的标签。

3.1.8 目录表

procedure

(table-of-contents) → delayed-block?

返回一个延迟流元素,该元素扩展为包围部分的内容表。 然而,对于 Latex 输出来说,目录目前跨越了整个包围文档。 程序

procedure

(local-table-of-contents [#:style style]) → delayed-block?

  style : (or/c symbol? #f) = #f
返回一个延迟流元素,根据输出类型,该元素可能扩展为包围部分的内容表。 对于多页 HTML 输出,flow 元素是一个目录;对于 Latex 输出,flow 元素是空的。

style 参数的含义取决于输出类型,但是 'immediate-only 通常会创建一个只包含包围部分的直接子部分的目录。 也请参见 part 的 'quiet 样式(即在一个 part 结构中, 不作为 style 参数提供 local-table-of-contents), 它通常会抑制目录中的子部分条目。
