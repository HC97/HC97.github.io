4.7 Compatibility Libraries

4.7.1 兼容性结构与处理

 (require scribble/struct) package: scribble-lib
scribble/struct 兼容库主要是重新输出 scribble/core, 但使用了一些不同的名称(例如,用 blockquote 代替 nested-flow)。

下面的结构类型和函数被直接重新输出:

collect-info resolve-info tag? block?
delayed-block collected-info delayed-element
part-relative-element collect-info-parents
collect-element render-element generated-tag
tag-key content->string element->string
block-width element-width
info-key? part-collected-info collect-put!
resolve-get resolve-get/tentative resolve-get/ext?
resolve-search resolve-get-keys

下面的结构类型被重新导出,但是构造器和一些选择器被替换了,如下文所述:

part paragraph table itemization compound-paragraph
element toc-element target-element toc-target-element toc-target2-element
page-target-element redirect-target-element link-element
index-element

一些额外的兼容函数和结构类型也被导出。

procedure

(make-part tag-prefix    
  tags    
  title-content    
  style    
  to-collect    
  blocks    
  parts) → part?
  tag-prefix : (or/c false/c string?)
  tags : (listof tag?)
  title-content : (or/c false/c list?)
  style : any/c
  to-collect : list?
  blocks : (listof block?)
  parts : (listof part?)
For backward compatibility. Compared to the normal constructor for part, 解析 style 以将旧格式转换为当前格式。 另外,如果 title-content 是一个只有一个项目的列表,这个项目本身会被存储在生成的 part 中。

procedure

(part-flow p) → (listof block?)

  p : part?
For backward compatibility. part-blocks 的别名。

procedure

(part-title-content p) → list?

  p : part?
For backward compatibility. 像普通的选择器一样,但如果结果不是一个列表,它就被强制为一个列表。

procedure

(make-versioned-part tag-prefix    
  tags    
  title-content    
  style    
  to-collect    
  blocks    
  parts    
  version) → part?
  tag-prefix : (or/c false/c string?)
  tags : (listof tag?)
  title-content : (or/c false/c list?)
  style : any/c
  to-collect : list?
  blocks : (listof block?)
  parts : (listof part?)
  version : string?

procedure

(versioned-part? v) → boolean?

  v : any/c
For backward compatibility. 像 make-part 一样,但增加了一个使用给定 version 的 document-version style property 。 versioned-part? 谓词可以识别具有 document-version 属性的 part 。

procedure

(make-unnumbered-part tag-prefix    
  tags    
  title-content    
  style    
  to-collect    
  blocks    
  parts) → part?
  tag-prefix : (or/c false/c string?)
  tags : (listof tag?)
  title-content : (or/c false/c list?)
  style : any/c
  to-collect : list?
  blocks : (listof block?)
  parts : (listof part?)

procedure

(unnumbered-part? v) → boolean?

  v : any/c
For backward compatibility. 像 make-part 一样,但增加了 'unnumbered style property 。 unnumbered-part? 谓词识别具有 'unnumbered 属性的 part 。

procedure

(make-paragraph content) → paragraph?

  content : list?
For backward compatibility. Compared to the normal constructor for paragraph, 与段落的普通构造函数相比,省略了一个样式参数。 另外,如果 content 是一个包含单个项目的列表,该项目本身将被保存在生成的 paragraph 中。

procedure

(paragraph-content p) → list?

  p : paragraph?
For backward compatibility. 和普通的选择器一样,但如果结果不是一个列表,就会被强制为一个列表。

procedure

(make-styled-paragraph content style) → paragraph?

  content : list?
  style : any/c

procedure

(styled-paragraph? v) → boolean?

  v : any/c

procedure

(styled-paragraph-style p) → style?

  p : paragraph?
For backward compatibility. Compared to the normal constructor for paragraph, 解析 style ,将旧格式转换为当前格式。 styled-paragraph? 谓词和 styled-paragraph-style 访问器是 paragraph? 和 paragraph-style 的别名。

procedure

(make-omitable-paragraph content) → paragraph?

  content : list?

procedure

(omitable-paragraph? v) → boolean?

  v : any/c
For backward compatibility. 与 make-paragraph 类似,但增加了 'omitable style property 。 omitable-paragraph? 谓词检查是否有该属性的段落。

procedure

(make-table style blocksss) → table?

  style : any/c
  blocksss : (listof (listof (or/c (listof block?) (one-of/c 'cont))))
For backward compatibility. Compared to the normal constructor for table, 样式被转换,每个单元格有一个块的列表,而不是单个块。 如果任何这样的列表有多个块,它们会被合并成一个 nested-flow 。

procedure

(table-flowss table)

 → (listof (listof (or/c (listof block?) (one-of/c 'cont))))
  table : table?
For backward compatibility. 与 table-blockss 类似,但增加了一个列表包装器,与 make-table 一致。

procedure

(make-itemization blockss) → itemization?

  blockss : (listof (listof block?))
For backward compatibility. Compared to the normal constructor for itemization, 省略了一个样式参数。

procedure

(make-styled-itemization style blockss) → itemization?

  style : any/c
  blockss : (listof (listof block?))

procedure

(styled-itemization? v) → boolean?

  v : any/c

procedure

(styled-itemization-style i) → style?

  i : itemization?
For backward compatibility. Compared to the normal constructor for itemization, 解析 style 以将旧格式转换为当前格式。 styled-itemization? 谓词是 itemization? 的别名, 而 styled-itemization-style 是 itemization-style 的别名。

procedure

(make-blockquote style blocks) → nested-flow?

  style : any/c
  blocks : (listof block?)
For backward compatibility. 与 make-nested-flow 类似,但 style 被解析为当前格式。

procedure

(make-auxiliary-table style blocksss) → table?

  style : any/c
  blocksss : (listof (listof (or/c (listof block?) (one-of/c 'cont))))

procedure

(auxiliary-table? v) → boolean?

  v : any/c
For backward compatibility. 像 make-table ,但增加了 'aux style property 。 auxiliary-table? 谓词可以识别具有 'aux 属性的表。

procedure

(make-compound-paragraph style blocks) → compound-paragraph?

  style : any/c
  blocks : (listof block?)
For backward compatibility. Compared to the normal constructor for compound-paragraph, 解析 style ，将旧格式转换为当前格式。

procedure

(make-element style content) → element?

  style : any/c
  content : list?

procedure

(make-toc-element style content toc-content) → toc-element?

  content : list?
  toc-content : list?

(make-target-element style content tag) → target-element?

  content : list?
  tag : tag?

(make-toc-target-element style content tag) → toc-target-element?

  tag : tag?

(make-page-target-element style content tag)

(make-redirect-target-element style

(make-link-element style content tag) → link-element?

procedure

(make-index-element style    
  content    
  tag    
  plain-seq    
  entry-seq

  style : any/c
  content : list?
  tag : tag?
  plain-seq : (and/c pair? (listof string?))
  entry-seq : list?
  desc : any/c
For backward compatibility. 与普通的构造函数相比,解析 style 以将旧的格式转换为当前的格式。

procedure

(element? v) → boolean?

  v : any/c

procedure

(element-content e) → list?

  e : element?

procedure

(element-style e) → element-style?

  e : element?
For backward compatibility. 一个 content 列表被这些函数视为一个元素, 而且 element-content 的结果总是一个列表。

procedure

(make-aux-element style content) → element?

  style : any/c
  content : list?
For backward compatibility. 像 make-element 一样,但增加了 'aux style property 。

procedure

(make-hover-element style content text) → element?

  style : any/c
  content : list?
  text : string?
For backward compatibility. 像 make-element 一样, 但在元素的样式中增加了包含 text 的 hover-property 。

procedure

(make-script-element style    
  content    
  type    
  script) → element?
  style : any/c
  content : list?
  type : string?
  script : (or/c path-string? (listof string?))
For backward compatibility. 像 make-element 一样, 但在元素的样式中增加了包含 type 和 script 的 script-property 。

struct

(struct with-attributes (style assoc)
    #:extra-constructor-name make-with-attributes)
  style : any/c
  assoc : (listof (cons/c symbol? string?))
For backward compatibility. 用于一个 element 的样式,将一个基本的样式与任意的HTML属性相结合。 当 style 字段本身是 with-attributes 的一个实例时, 当它被使用时(例如,当渲染一个 element 或 paragraph 时), 它的内容会自动平铺到包围的 with-attributes 里。

struct

(struct target-url (addr style)
    #:extra-constructor-name make-target-url)
  addr : path-string?
  style : any/c
For backward compatibility. 作为一个 element 的样式使用。 这一层的 style 是超链接的样式。

struct

(struct image-file (path scale)
    #:extra-constructor-name make-image-file)
  path : 
(or/c path-string?
      (cons/c 'collects (listof bytes?)))
  scale : real?
For backward compatibility. 作为一个 element 的样式,用于内联一个图像。 path 字段可以是 path->main-collects-relative 的结果。

procedure

(element->string element) → string?

  element : content?
(element->string element renderer p info) → string?
  element : content?
  renderer : any/c
  p : part?
  info : resolve-info?
For backward compatibility. content->string 的一个别名。

4.7.2 兼容性基本函数

 (require scribble/basic) package: scribble-lib
scribble/basic 兼容库大多只是重新输出 scribble/base 。

procedure

(span-class style-name pre-content ...) → element?

  style-name : string?
  pre-content : any/c
For backward compatibility. 将 decode 后的 pre-content 包装成具有样式 style-name 的元素。

procedure

(itemize itm ... [#:style style]) → itemization?

  itm : (or/c whitespace? an-item?)
  style : (or/c style? string? symbol? #f) = #f
For backward compatibility. 像 itemlist 一样,但 itm 之间的空白字符串被忽略。
