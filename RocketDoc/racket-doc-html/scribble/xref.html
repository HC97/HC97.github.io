6.8 交叉引用实用程序

 (require scribble/xref) package: scribble-lib
scribble/xref 库提供了用于查询从文档构建中收集的交叉引用信息的工具。

procedure

(xref? v) → boolean?

  v : any/c
如果 v 是由 load-xref 创建的交叉引用记录,返回 #t ,否则返回 #f 。

procedure

(load-xref sources 
  [#:demand-source-for-use demand-source-for-use 
  #:demand-source demand-source 
  #:render% using-render% 
  #:root root-path 
  #:doc-id doc-id-str]) 
 → xref?
  sources : (listof (-> (or/c any/c (-> list?))))
  demand-source-for-use : (tag? symbol? -> (or/c (-> any/c) #f))
   = (lambda (tag use-id) (demand-source tag))
  demand-source : (tag? -> (or/c (-> any/c) #f))
   = (lambda (tag) #f)
  using-render% : (implementation?/c render<%>)
   = (render-mixin render%)
  root-path : (or/c path-string? false/c) = #f
  doc-id-str : (or/c path-string? false/c) = #f
给出一个函数列表 sources ,创建一个交叉引用记录。

让 source 是 sources 中的一个函数。 source 函数通常返回序列化的信息,即 info, 这些信息以前是通过 serialize-info in render<%> 获得的。 source 的结果可以选择是另一个函数,而这个函数又负责返回一个 info 的列表。 最后,每个 info 可以是序列化的信息,也可以是被忽略的 #f , 或者是由 make-data+root 或 make-data+root+doc-id 产生的值, 其中 data 部分被用作序列化的信息, root 部分覆盖 root-path 用于反序列化, doc-id 部分(如果有)覆盖 doc-id-string 用于识别源文档。

demand-source-for-use 函数可以有效地将一个新的源添加到 sources 中, 以响应对给定渲染中的给定标记的信息的搜索, 其中 use-id 对于特定的渲染请求、特定的转移(在 xref-transfer-info 的意义上) 或者所有交叉引用信息的直接查询(比如通过 xref-binding->definition-tag) 是唯一的。 demand-source-for-use 函数应该返回 #f , 以表示没有新的来源满足给定的 use-id 的标签。

默认的 demand-source-for-use 函数使用 demand-source, 它只是为了向后兼容而提供。 由于 demand-source 函数只接受一个标签, 它只适用于 load-xref 的结果只有一个使用环境的情况下,比如说一次渲染。

由于序列化信息的格式是特定于渲染类的,可选的 using-render% 参数接受相关的类。 它默认为 HTML 渲染,部分原因是 HTML 格式的信息可以被其他格式使用(包括 Latex/PDF 和文本)。

如果 root-path 不是 #f, 那么被序列化为相对于实例化提供的 root-path 的文件路径将被反序列化为相对于给定的 root-path, 但是任何 info 的 make-data+root 结果为 info 的 data 反序列化提供了一个替代路径。

如果 doc-id-str 不是 #f, 它将识别每个交叉引用条目是来自 doc-id-str 。 这个标识在渲染链接到交叉引用条目作为外部查询时使用; 参见 render-mixin 的 set-external-tag-path 方法。

使用 setup/xref 中的 load-collections-xref 来获取已安装文档的所有交叉引用信息。

Changed in version 1.1 of package scribble-lib: 增加了 #:doc-id 参数。
Changed in version 1.34: 增加了 #:demand-source-for-use 参数。

procedure

(xref-binding->definition-tag xref    
  binding    
  mode) → (or/c tag? false/c)
  xref : xref?
  binding : 
(or/c identifier?
      (list/c (or/c module-path?
                    module-path-index?)
              symbol?)
      (list/c module-path-index?
              symbol?
              module-path-index?
              symbol?
              (one-of/c 0 1)
              (or/c exact-integer? false/c)
              (or/c exact-integer? false/c))
      (list/c (or/c module-path?
                    module-path-index?)
              symbol?
              (one-of/c 0 1)
              (or/c exact-integer? false/c)
              (or/c exact-integer? false/c)))
  mode : (or/c exact-integer? false/c)
在 xref 中找到一个记录模块输出的标签。 绑定的方式有几种,如下所述;所有的可能性都是对输出模块和符号名称的编码。 该名称必须是从指定的模块中导出的。文档可以在指定的模块中找到, 或者,如果导出的名字是从其他模块中重新导出的,则在其他模块中找到(过渡性的)。

mode 参数指定了绑定的相关阶段级别。 binding 是以四种方式之一指定的:

  • 如果 binding 是一个标识符,那么 identifier-binding 与 mode 一起使用来确定绑定。

  • 如果 binding 是一个两元素的列表,那么第一个元素提供导出的模块,第二个元素提供导出的名称。 mode 参数被有效地忽略了。

  • 如果 binding 是一个七元素的列表, 那么它对应于使用 mode 的 identifier-binding 的结果。

  • 如果 binding 是一个五元素的列表,那么第一个元素与两元素列表的情况一样, 剩下的元素与七元素的情况的最后四个元素一样。

如果在 xref 中存在一个文档点,就会返回一个标签, 这个标签可以用 xref-tag->path+anchor 来使用,或者嵌入到通过 xref-render 渲染的文档中。 如果在 xref 中没有找到定义点,结果是 #f 。

procedure

(xref-tag->path+anchor xref 
  tag 
  [#:external-root-url root-url 
  #:render% using-render%]) 
 → 
(or/c false/c path?)
(or/c false/c string?)
  xref : xref?
  tag : tag?
  root-url : (or/c string? #f) = #f
  using-render% : (implementation?/c render<%>)
   = (render-mixin render%)
根据交叉引用 xref 返回一个由关键 tag 指定的路径和锚点字符串。 如果没有发现给定标签的映射,第一个结果是 #f 。 如果第一个结果是 #f ,则第二个结果是 #f , 如果标签指的是一个页面而不是一个页面中的特定点,则第二个结果也可能是 #f 。

如果提供了 root-url ,那么对主安装中的文档的引用会被重定向到给定的URL。

可选的 using-render% 参数与 load-xref 相同。

procedure

(xref-tag->index-entry xref tag) → (or/c false/c entry?)

  xref : xref?
  tag : tag?
提取一个 entry 结构,提供关于 tag 所引用的定义(任何定义)的额外信息。 这个函数可以与 xref-binding->definition-tag 组成, 以获得关于一个绑定的信息,例如输出绑定的库和它的原始名称。

procedure

(xref-render xref 
  doc 
  dest 
  [#:render% using-render% 
  #:refer-to-existing-files? use-existing?]) 
 → (or/c void? any/c)
  xref : xref?
  doc : part?
  dest : (or/c path-string? false/c)
  using-render% : (implemenation?/c render<%>)
   = (render-mixin render%)
  use-existing? : any/c = (not dest)
使用 xref 中的交叉引用信息将 doc 渲染到目的地 dest 。 例如, doc 可能是一个使用 xref 中描述的链接标签生成的搜索结果的文件。

如果 dest 是 #f, 则不写文件,结果是渲染的页面的X-表达式。 否则,文件 dest 被写入,结果是 #<void> 。

可选的 using-render% 参数和 load-xref 一样。 它决定了生成的输出的种类。

如果 use-existing? 为真,那么在渲染过程中引用的文件(例如图像文件)将从它们的现有位置被引用, 而不是复制到 dest 目录中。

procedure

(xref-transfer-info renderer ci xref) → void?

  renderer : (is-a?/c render<%>)
  ci : collect-info?
  xref : xref?
将交叉引用信息转移到 ci, 这是最初从 renderer 收集的信息。

procedure

(xref-index xref) → (listof entry?)

  xref : xref?
将索引信息 xref 转换为一个 entry 结构的列表。

struct

(struct entry (words content tag desc)
    #:extra-constructor-name make-entry)
  words : (and/c (listof string?) cons?)
  content : list?
  tag : tag?
  desc : any/c
代表 Scribble 文档索引中的一个条目。

words 列表对应于 index-element-plain-seq 。 content 列表对应于 index-element-entry-seq 。 desc 值对应于 index-element-desc 。 tag 是索引链接到主文件的目的地。
