4.3 Racket

 (require scribble/racket) package: scribble-lib
 (require scribble/scheme)
scribble/racket 库(或 scribble/scheme 用于向后兼容)提供了用于排版 Racket 代码的实用程序。 scribble/manual 表格提供了一个更高级的接口。

syntax

(define-code id typeset-expr)

(define-code id typeset-expr uncode-id)
(define-code id typeset-expr uncode-id d->s-expr)
(define-code id typeset-expr uncode-id d->s-expr stx-prop-expr)
将 id 绑定到一个类似于 racket 或 racketblock 的表单,用于排版代码。 由 define-code 生成的表单处理源位置信息, 默认通过 unquote 转义,保留绑定和属性信息,并支持 element transformers 。

提供的 typeset-expr 表达式应该产生一个执行实际排版的过程。 这个表达式通常是 to-element 或 to-paragraph 。 提供给 typeset-expr 的参数通常是一个语法对象,但更常见的是应用 d->s-expr 的结果。

可选的 uncode-id 指定了 id 识别的默认转义字面代码, uncode-id 的默认值是 unsyntax 。 id 形式的使用可以通过 #:escape 指定一个替代的转义, 就像在 racketblock 或 racket 中一样。

可选的 d->s-expr 应该产生一个过程,接受适合 datum->syntax 的三个参数: 一个语法对象或 #f, 一个任意值,以及一个源位置的向量。 其结果应该根据 typeset-expr 对代码进行排版的需要,记录尽可能多或尽可能少的参数信息。 通常情况下, d->s-expr 是 datum->syntax 。

stx-prop-expr 应该产生一个程序来记录 'paren-shape 属性, 当与 id 一起使用的源表达式有这样的属性时。 默认是 syntax-property 。

procedure

(to-paragraph v    
  [#:expr? expr?    
  #:escapes? escapes?    
  #:color? color?    
  #:wrap-elem wrap-elem]) → block?
  v : any/c
  expr? : any/c = #f
  escapes? : any/c = #t
  color? : any/c = #t
  wrap-elem : (element? . -> . element?) = (lambda (e) e)
对一个由语法对象表示的 S 表达式进行排版,语法对象中的源位置信息控制着生成的布局。 当源位置信息不可用时,使用默认间距(与 to-element 相同的单行风格)。

有 for-label 绑定的标识符会根据在其他地方(通过 defproc, defform 等)声明的定义进行排版和超链接。 除非 escapes? 是 #f, 否则标识符 code:line, code:comment, code:blank, code:hilite 和 code:quote 的处理与 racketblock 一样,以 _ 开头的标识符也是如此。

此外,给定的 v 可以包含 var-id, shaped-parens, just-context 或 literal-syntax 结构, 这些结构将被专门排版(详见每种结构类型),或者它可以包含直接用于输出的 element 结构。

如果 expr? 为真,那么 v 将以表达式风格呈现,很像 print , 参数 print-as-expression 设置为 #t 。 在这种情况下,标识符上的 for-label 绑定会被忽略,因为标识符在输出中都是引号。 通常情况下, expr? 被设置为 true,用于打印结果值。

如果 color? 是 #f, 那么输出将被排版而不着色。

wrap-elem 程序被应用于为结果块构建的每个元素。 当与 #f 的 color? 的颜色相结合时, 例如, wrap-elem 过程可以用来给一个元素提供一个样式。

procedure

((to-paragraph/prefix prefix1    
  prefix    
  suffix)    
  v    
  [#:expr? expr?    
  #:escapes? escapes?    
  #:color? color?    
  #:wrap-elem wrap-elem]) → block?
  prefix1 : any/c
  prefix : any/c
  suffix : any/c
  v : any/c
  expr? : any/c = #f
  escapes? : any/c = #t
  color? : any/c = #f
  wrap-elem : (element? . -> . element?) = (lambda (e) e)
像 to-paragraph 一样,但是 prefix1 被预置到第一行, prefix 被预置到任何后续的行,而 suffix 被添加到最后。 prefix1, prefix 和 suffix 参数与 content 一样使用, 只是如果 suffix 是一个元素列表,它将被添加到它自己的行的末尾。

procedure

(to-element v    
  [#:expr? expr?    
  #:escapes? escapes?    
  #:defn? defn?]) → element?
  v : any/c
  expr? : any/c = #f
  escapes? : any/c = #t
  defn? : any/c = #f
和 to-paragraph 一样,除了源位置信息大部分被忽略,因为结果是要被内联到一个段落中。 如果 defn? 为真,那么标识符就会被作为一个定义站点来处理。

procedure

(to-element/no-color v    
  [#:expr? expr?    
  #:escapes? escapes?]) → element?
  v : any/c
  expr? : any/c = #f
  escapes? : any/c = #t
与 to-element 类似,但 for-syntax 绑定被忽略,而且生成的文本是无色的。 这个变体通常用于对结果进行排版。

struct

(struct var-id (sym)
    #:extra-constructor-name make-var-id)
  sym : (or/c symbol? identifier?)
当 to-paragraph 和变体遇到 var-id 结构时,它被排成变量字体的 sym , 就像 racketvarfont—除非 var-id 出现在 quote 或 quasiquote 下, 在这种情况下, sym 被排成一个符号。

struct

(struct shaped-parens (val shape)
    #:extra-constructor-name make-shaped-parens)
  val : any/c
  shape : char?
当 to-paragraph 和变体遇到 shaped-parens 结构时, 它的类型设置类似于具有值为 shape 的 'paren-shape 属性的语法对象。

struct

(struct long-boolean (val)
    #:extra-constructor-name make-long-boolean)
  val : boolean?
当 to-paragraph 和变体遇到一个 long-boolean 结构时, 它被排版为 #true 或 #false ,而不是 #t 或 #f 。

struct

(struct just-context (val context)
    #:extra-constructor-name make-just-context)
  val : any/c
  context : syntax?
当 to-paragraph 和变体遇到一个 just-context 结构时, 它将使用 val 的源位置信息来排版,而只是 ctx 的词法上下文。

struct

(struct literal-syntax (stx)
    #:extra-constructor-name make-literal-syntax)
  stx : any/c
当 to-paragraph 和变体遇到一个 literal-syntax 结构时, 它被排版为 stx 的字符串形式。 这可以用来将一个语法对象的值按照默认打印机的方式排版。

procedure

(element-id-transformer? v) → boolean?

  v : any/c
for-syntax提供;如果 v 是由 make-element-id-transformer 创建的 element transformer ,则返回 #t ,否则返回 #f 。

procedure

(make-element-id-transformer proc) → element-id-transformer?

  proc : (syntax? . -> . syntax?)
提供 for-syntax ;创建一个 element transformer 。 当一个标识符有一个与 element transformer 绑定的变换器时,由 define-code (包括 racket 和 racketblock) 通过对标识符应用 proc 来排版。 其结果必须是一个表达式,其值通常是一个 element, 被传递给 to-paragraph 等函数。

procedure

(variable-id? v) → boolean?

  v : any/c
提供 for-syntax; 如果 v 是由 make-variable-id 创建的element transformer , 则返回 #t ,否则返回 #f 。

procedure

(make-variable-id sym) → variable-id?

  sym : (or/c symbol? identifier?)
提供 for-syntax; 像 make-element-id-transformer 一样, 产生 sym 排版为变量的转化器(像 racketvarfont)— 除非它出现在 quote 或 quasiquote 下,在这种情况下 sym 被排版为符号。
