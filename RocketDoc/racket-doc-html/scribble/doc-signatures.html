4.2.5 文档签名

syntax

(defsignature id (super-id ...) pre-flow ...)

定义了一个扩展了 super-id 签名的签名 id 。 任何在 decode 的 pre-flow 中定义的元素– 包括窗体、程序、结构类型、类、接口和混合器–都被定义为签名的成员而不是直接绑定。 这些定义可以通过 sigelem 而不是 racket 来引用。

decode 后的 pre-flow 在排版输出中嵌入到签名声明下,所以不能启动新的部分等。

syntax

(defsignature/splice id (super-id ...) pre-flow ...)

像 defsignature 一样,但 decode 的 pre-flow 不在签名声明下排版, 新的部分等可以在 pre-flow 中启动。

procedure

(signature-desc pre-flow ...) → any/c

  pre-flow : pre-flow?
产生一个不透明的值, defsignature 识别到排版形式中的过时。 这对描述签名整体的文本在签名声明之后出现很有用。

syntax

(sigelem sig-id id)

排版标识符 id ,并有超链接指向其定义,作为 sig-id 命名的签名的成员。
