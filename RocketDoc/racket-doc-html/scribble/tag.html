6.9 标签实用程序

 (require scribble/tag) package: scribble-lib
scribble/tag 库提供了构建交叉参考 tags 的实用程序。该库由 scribble/base 重新输出。

procedure

(make-section-tag name    
  [#:doc doc-mod-path    
  #:tag-prefixes tag-prefixes]) → tag?
  name : string?
  doc-mod-path : (or/c module-path? #f) = #f
  tag-prefixes : (or/c #f (listof string?)) = #f
形成一个引用章节的 tag ,该章节的 "标签" (例如,由 section 的 #:tag 参数提供)是 name 。 如果提供了 doc-mod-path , tag 就会从文档外部引用 doc-mod-path 实现的文档中的一个部分。 附加的标签前缀(通常用于中间部分)可以作为 tag-prefixes 提供。

procedure

(make-module-language-tag lang) → tag?

  lang : symbol?
形成一个 tag ,指向包含语言 lang 的 defmodulelang 的部分。

procedure

(taglet? v) → boolean?

  v : any/c
如果 v 是一个 taglet ,返回 #t ,否则返回 #f 。

taglet 是一个可以通过 list 与符号结合形成 tag 的值, 但它不是一个 generated-tag 。 因此, taglet 作为 tag 的一部分是有用的, 特别是作为可以获得前缀的标签的一部分(例如,从文档之外指代文档的某一部分)。

procedure

(doc-prefix mod-path taglet) → taglet?

  mod-path : (or/c #f module-path?)
  taglet : taglet?
(doc-prefix mod-path extra-prefixes taglet) → taglet?
  mod-path : (or/c #f module-path?)
  extra-prefixes : (or/c #f (listof taglet?))
  taglet : taglet?
假设 mod-path 不是 #f, 将在 mod-path 实现的文档内工作的交叉引用 tag 的一部分转换为在文档外工作的标签。 也就是说, mod-path 被转换为一个 taglet ,并作为前缀添加到一个现有的 taglet 中。

如果提供了 extra-prefixes , 那么它的内容会在 mod-path 的前缀被添加之前作为额外的前缀元素被添加。 extra-prefixes 的 #f 值等同于 '() 。

如果 mod-path 是 #f, 那么 taglet 返回时没有前缀(除了添加 extra-prefixes,如果提供的话)。

procedure

(module-path-prefix->string mod-path) → string?

  mod-path : module-path?
通过将模块路径解析为路径,并使用 path->main-collects-relative ,将模块路径转换为一个字符串。

procedure

(module-path-index->taglet mpi) → taglet?

  mpi : module-path-index?
将模块路径索引转换为 taglet—路径作为S表达式的规范化编码–通过 intern-taglet 进行内部处理。

taglet 的字符串形式被用作 tag 的前缀,以形成由 mpi 引用的模块所实现的文档的交叉引用。

procedure

(intern-taglet v) → any/c

  v : any/c
返回一个与 v equal? 的值, 多次调用 intern-taglet 的 equal? v 产生相同(即 eq?) 的值。

procedure

(definition-tag->class/interface-tag definition-tag)

 → class/interface-tag?
  definition-tag : definition-tag?
构造一个类似 definition-tag 的标签,除了它与该类的文档相匹配。 如果 definition-tag 没有记录一个类或接口,这个函数仍然返回类或接口文档会有的标记, 就像 definition-tag 记录了一个类或接口一样。

Added in version 1.11 of package scribble-lib.

procedure

(class/interface-tag->constructor-tag class/interface-tag)

 → constructor-tag?
  class/interface-tag : class/interface-tag?
构造一个类似 definition-tag 的标签,只是它与类的构造函数的文档相匹配。

Added in version 1.11 of package scribble-lib.

procedure

(get-class/interface-and-method method-tag) → 
symbol? symbol?
  method-tag : method-tag?
返回 method-tag 处 docs 所记录的方法的类名和方法名(分别)。

Added in version 1.11 of package scribble-lib.

procedure

(definition-tag? v) → boolean?

  v : any/c
识别定义标签。如果 (definition-tag? v) 是 #t, 那么 (tag? v) 也是。

Added in version 1.11 of package scribble-lib.

procedure

(class/interface-tag? v) → boolean?

  v : any/c
识别类或接口标签。 如果 (class/interface-tag? v) 是 #t, 那么 (tag? v) 也是。

Added in version 1.11 of package scribble-lib.

procedure

(method-tag? v) → boolean?

  v : any/c
识别方法标签。 如果 (method-tag? v) 是 #t, 那么 (tag? v) 也是。

Added in version 1.11 of package scribble-lib.

procedure

(constructor-tag? v) → boolean?

  v : any/c
识别类的构造函数标签。 如果 (constructor-tag? v) 是 #t, 那么 (tag? v) 也是。

Added in version 1.11 of package scribble-lib.
