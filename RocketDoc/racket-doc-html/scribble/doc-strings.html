4.2.6 各种字符串形式

procedure

(aux-elem pre-content ...) → element?

  pre-content : pre-content?
和 elem 一样,但增加了一个 'aux style property 。

procedure

(defterm pre-content ...) → element?

  pre-content : pre-content?
将 decode 的 pre-content 作为定义的术语进行排版(例如,用斜体)。 不过可以考虑使用 deftech 代替,这样 tech 的使用可以超链接到定义。

procedure

(onscreen pre-content ...) → element?

  pre-content : pre-content?
将 decode 的 pre-content 排版为一个出现在 GUI 中的字符串,例如一个按钮的名称。

procedure

(menuitem menu-name item-name) → element?

  menu-name : string?
  item-name : string?
对 GUI 的菜单和项目名称的给定组合进行排版。

procedure

(filepath pre-content ...) → element?

  pre-content : pre-content?
将 decode 的 pre-content 作为文件名进行排版(例如,用打字机字体和引号)。

procedure

(exec content ...) → element?

  content : content?
将 content 排版为命令行(例如,用打字机字体)。

procedure

(envvar pre-content ...) → element?

  pre-content : pre-content?
将给定的 decode 的 pre-content 作为环境变量进行排版(例如,用打字机字体)。

procedure

(Flag pre-content ...) → element?

  pre-content : pre-content?
将给定的 decode 的 pre-content 作为一个标志进行排版 (例如,用打字机字体,前面有一个 -) 。

procedure

(DFlag pre-content ...) → element?

  pre-content : pre-content?
将给定的 decode 的 pre-content 排成一个长的标志(例如,用打字机字体和两个领先的 -)。

procedure

(PFlag pre-content ...) → element?

  pre-content : pre-content?
将给定的 decode 的 pre-content 排成 + 标志 (例如,用打字机字体,前面有一个 +) 。

procedure

(DPFlag pre-content ...) → element?

  pre-content : pre-content?
将给定的 decode 的 pre-content 排成一个长的 + 标志 (例如,用打字机字体,有两个领先的 +)。
