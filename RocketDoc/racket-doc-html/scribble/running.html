7 运行 scribble

scribble 命令行工具(也可以用 raco scribble) 运行一个 Scribble 文档, 并将其渲染成特定格式。 用下列标志之一选择格式,其中输出名称 fn 默认为不带文件后缀的文件源名称:

  • --html — 单个HTML页面 "fn.html", 加上 CSS 源和需要的图像文件;如果没有指定格式,这个模式是默认的

  • --htmls — 在 "fn" 目录下的多个HTML页面(和相关文件), 以 “"fn/index.html” 开始。"

  • --html-tree ‹n› — 目录树中的 HTML 页面,最深为 ‹n› 层; 树的深度为 0 相当于使用 --html, 树的深度为 1 相当于使用 --htmls

  • --latex — LaTeX 源代码 "fn.tex", 加上任何运行 latex 或 pdflatex 所需的额外文件(如非标准类文件)。

  • --pdf — 通过 pdflatex 生成的 PDF "fn.pdf"

  • --xelatex — 通过 xelatex 生成的 PDF "fn.pdf"

  • --lualatex — 通过 lualatex 生成的 PDF "fn.pdf"

  • --dvipdf — 通过 latex, dvips 和 pstopdf 生成的 PDF "fn.pdf"

  • --latex-section ‹n› — LaTeX 源文件 "fn.tex" 加上额外的 ".tex" 文件,包含在包围文件的序言中, 其中包围文件必须使用 UTF-8 输入编码和 T1 字体编码; 用 1 表示 ‹n› ,使渲染的文件成为一个章节,用 2 表示一个小节,等等。

  • --text — 单个文件 "fn.txt" 中的纯文本,非 ASCII 内容编码为 UTF-8

  • --markdown — Markdown 文本, 在单个文件 "fn.md" 中,非 ASCII 编码的内容为 UTF-8。

使用 --dest-name 来指定默认名称以外的 fn ,但只有在提供单一源文件的情况下。 使用 --dest 标志来指定一个目标目录(对于任何数量的源文件)。 使用 --dest-base 为生成或复制到目的地的每个支持文件的名称添加前缀; 前缀可以包含一个目录路径,非目录的结尾元素被用作支持文件名称的前缀。 使用 --keep-at-dest-base 来避免用支持文件覆盖现有的文件 (但是当内容与其他方式写入的内容相匹配时,就会使用现有文件)。

在所有标志之后,提供一个或多个文件源,其中每个源声明了一个模块。 该模块应该有一个 doc submodule , 将 doc 作为一个 part 导出,或者它应该直接将 doc 作为一个 part 导出。 (首先尝试子模块,如果子模块可以单独加载,则不直接加载或评估主模块)。 使用 --doc-binding 来访问一个替代的导出名称以代替 doc 。

当多个文档同时被渲染时,一个文档中的交叉引用信息对其他文档是可见的。请参阅 处理交叉引用 ,以了解关于交叉引用单独构建的文档的信息。

Changed in version 1.4: 添加了 --dvipdf 。
Changed in version 1.18: 添加了 --doc-binding 。
Changed in version 1.19: 添加了 --xelatex 。
Changed in version 1.40: 添加了 --keep-at-dest-base , 并修正了 --dest-base ,使之能在 HTML 输出时正确工作。
Changed in version 1.45: 添加了 --lualatex 。

7.1 额外的和特定格式的文件

使用 --style 标志来指定一个特定格式的文件,以调整某些格式的输出样式文件。 对于 HTML (单页或多页)输出,该样式文件应该是一个 CSS 文件,在所有其他 CSS 文件之后应用,因此可能覆盖一些样式属性。 对于 Latex (或 PDF )输出,样式文件应该是一个 ".tex" 文件,可以重新定义 Latex 命令。 然而,当一个特定的 Scribble 函数需要特定的 CSS 或 Latex 支持时, 更好的选择是使用 css-addition 或 tex-addition 样式属性,以便自动包含支持; 更多信息见 扩展和配置 Scribble 输出 。

在极少数情况下,使用 --style 标志来指定一个特定格式的基本样式文件。 对于 HTML (单页或多页)输出,该样式文件应该是一个 CSS 文件, 以替代 "scribble" 集合中的 "scribble.css" 。 对于 Latex (或 PDF )输出,样式文件应该是一个 ".tex" 文件, 以代替 "scribble" 集合中的 "scribble.tex" 。 --style 标志很少有用,因为 "scribble.css" 或 "scribble.tex" 的内容被弱化了; 替换文件必须定义所有相同的样式,而且不同版本的 Racket 的样式集可能会改变。

使用 --prefix 来指定输出文件的另一个特定格式开始。 对于 HTML 输出,起始文件指定每个输出 HTML 文件的 DOCTYPE 声明, 作为 "scribble" 集合中 "scribble-prefix.html" 的替代。 对于 Latex (或 PDF )输出(但不是 Latex-section 输出), 起始文件指定了 \documentclass 声明和初始 \usepackage 声明, 作为 "scribble" 集合中的 "scribble-prefix.tex" 的替代。 也请参见 html-defaults, latex-defaults 和 扩展和配置 Scribble 输出 。

对于任何输出形式,使用 ++extra 标志将需要的文件添加到构建目标中, 例如在生成的输出中引用的图像文件,但没有通过 image (它自动复制文件)包含。

7.2 处理交叉引用

一个文档或一起呈现的文档中的交叉引用总是被解决的。 当交叉引用跨越单独呈现的文档时,交叉引用信息需要明确地被保存和加载。 交叉引用信息是特定格式的,但 HTML 格式的信息可以用于 Latex (或 PDF )或文本渲染。

Racket 的安装包括所有安装的文档的 HTML 格式的交叉引用信息。 每个文档的信息都在一个单独的文件中,所以加载所有相关文件会很繁琐。 +m 或 ++main-xref-in 标志可以加载所有已安装文档的交叉引用信息,所以

  scribble +m mine.scrbl

将 "mine.scrbl" 渲染成 "mine.html" ,并将其与Racket安装的文档进行交叉引用链接。 (必须安装 "racket-index" 包才能使用 +m/++main-xref-in 。)

++xref-in 标志通过调用指定模块的函数来加载交叉引用信息。 setup/xref 模块提供了 load-collections-xref 来加载所有已安装的文档的交叉参考信息, 而 +m 或 ++main-xref-in 只是 ++xref-in setup/xref load-collections-xref 的一个缩写。

HTML 输出的 --redirect-main 标志将本地安装的文档(不是用户范围的文档)的链接重定向到一个指定的 URL, 例如 http://docs.racket-lang.org/ 。 注意,文档链接有时会改变(尽管 Scribble 以相对稳定的方式生成 HTML 路径和锚), 所以 http://download.racket-lang.org/docs/version/html/ , 在用安装的 version 构建时可能更可靠。对于非 HTML 输出, --redirect-main 标志被忽略。

--redirect 标志和 --redirect-main 一样,只是它建立在给定的 URL 上, 表示一个比 HTML 路径和锚更稳定的交叉参考标签(例如,在一个函数的文档改变章节的情况下), 它可以为安装在用户范围内的文档生成重定向链接。 URL https://docs.racket-lang.org/local-redirect/index.html 可以为这些重定向工作。

对于不属于 Racket 安装的文档之间的交叉引用,使用 --info-out 来保存文档构建的信息, 使用 ++info-in 来加载之前保存的信息。 例如,如果 "c.scrbl" 引用了 "a.scrbl" 和 "b.scrbl" 中的信息,那么

  scribble --info-out a.sxref a.scrbl
  scribble --info-out b.sxref b.scrbl
  scribble ++info-in a.sxref ++info-in b.sxref c.scrbl

构建 "c.html" ,并在 "a.html" 和 "b.html" 中建立交叉引用链接。

当构建通常作为软件包的一部分安装的文件时, 通过文件系统路径引用文件可能会产生与通过 raco setup 运行文件不同的交叉引用链接。 区别在于解决相对路径导入的方式,特别是 for-label 的方式。 当从一个本身通过文件系统路径引用的模块开始时,相对路径引用算作文件系统路径引用, 或者当从一个本身使用基于集合的路径引用的模块开始时,它算作基于集合的模块路径。 使用 --lib/-l 标志可以用模块路径而不是文件系统路径来引用文件。

Changed in version 1.47: 增加了 --lib/-l 标志。

7.3 选择图像格式

使用 ++convert ‹fmt› 标志来选择 ‹fmt› 作为首选的图像格式, 以便在渲染包含可转换为不同图像格式的值的文档时使用。 ‹fmt› 参数可以是 pdf, ps, png, svg 或 gif, 但一个渲染器通常只支持这些格式中的一个子集。

多次使用 ++convert ‹fmt› 可以指定多种首选格式, 在命令行中较早的 ‹fmt› 优先于后来指定的 ‹fmt› 。

例如,要生成带有封装 PostScript 格式图像的 Latex 源 (这样结果就可以在 latex 而不是 pdflatex 中使用), 请将 --latex 与 ++convert ps 结合。 要生成 HTML 页面,并将图像转换为 SVG 格式而不是 PNG 格式, 请将 --html 与 ++convert svg 结合起来。

Changed in version 1.4: 增加了 ++convert 支持。

7.4 将命令行参数传递给文件

当 scribble 加载和渲染一个文档模块时, 默认情况下它会将 current-command-line-arguments 设置为一个空矢量。 使用 ++arg 标志(任意次数)将一个字符串添加到 current-command-line-arguments 。

例如、

  scribble ++arg --mode ++arg fast turtle.scrbl

导致 (current-command-line-arguments) 在 "turtle.scrbl" 加载和渲染时返回 '#("--mode" "fast") , 如果 "turtle.scrbl" 使用 current-command-line-arguments ,则可能影响它生成的内容。

Changed in version 1.1: 增加了 empty-vector 默认和 ++arg 标志。

7.5 附加选项

  • --quiet — 抑制输出文件和 undefined-tag 报告。

  • --make 或 -y — 在加载文档模块时启用自动生成和更新编译的 ".zo" 文件。 具体来说,在加载模块时, (make-compilation-manager-load/use-compiled-handler) 的结果被安装为 current-load/use-compiled 的值。

  • --doc-binding ‹id› — 渲染作为 ‹id› 提供的文档,而不是 doc

  • --errortrace — 启用 errortrace

Changed in version 1.38: 增加了 --errortrace 标志。
Changed in version 1.44: 增加了 --make/-y 标志。
