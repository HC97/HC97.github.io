2.6 Pict-Staging Helper

 (require slideshow/step) package: slideshow-lib
slideshow/step 库提供了将复杂的幻灯片分解成步骤的语法, 这些步骤比 slide 序列中的 'next 和 'alts 所能处理的要复杂。

syntax

(with-steps (id ...) body ...)

对每个 id 评估一次 body , 如果一个 id 的名字以 ~ 结尾且 condense? 为真,则跳过该 id。 每次迭代的最后一个 body 的结果被收集到一个列表中,这就是 with-steps 形式的结果。

在 body 中,几个关键词被非卫生地绑定(使用第一个 body 的词法上下文):

  • (only? id) — 在 id 步骤中 (即在对 id 的 body 进行评估期间)返回 #t ,否则返回 #f 。

  • (vonly id) — 在 id 步骤中返回身份函数,否则返回 ghost 。

  • (only id then-expr) 在 id 步骤中返回 then-expr 的结果, 否则是 values 。

  • (only id then-expr else-expr) 在 id 步骤中返回 then-expr 的结果, 否则返回 else-expr 的结果。

  • (before? id) — 返回 #t 在 id 步骤之前, #f 开始为 id 和之后。

  • (vbefore id), (before id then-expr) 或 (before id then-expr else-expr) — 类似于 vonly 和 only 。

  • (after? id) — 在 id 步骤之后返回 #t , 通过 id 步骤返回 #f 。

  • (vafter id), (after id then-expr) 或 (after id then-expr else-expr) — 类似于 vonly 和 only 。

  • (between? a-id b-id) — 返回 #t , 从 a-id 步骤开始到 b-id 步骤,否则返回 #f 。

  • (vbetween a-id b-id), (between a-id b-id then-expr) 或 (between a-id b-id then-expr else-expr) — 类似于 vonly 和 only 。

  • (between-excel? a-id b-id) — 返回 #t , 从 a-id 步骤开始到 b-id 步骤之前的步骤, #f 为 b-id 步骤和之后的步骤。

  • (vbetween-excl a-id b-id), (between-excl a-id b-id then-expr) 或 (between-excl a-id b-id then-expr else-expr) — 类似于 vonly 和 only 。

syntax

(with-steps~ (id ...) body ...)

与 with-steps 类似,但是当 condense? 为真时, 那么 expr 只对最后一个 id 进行评估 (与最后一个 id 的名字是否以 ~ 结尾无关)。
