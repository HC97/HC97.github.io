4 测试工具

4.1 检查文档的完整性

 (require rackunit/docs-complete) package: racket-index

procedure

(check-docs lib [#:skip skip]) → any

  lib : module-path?
  skip : 
(or/c regexp?
      symbol?
      (listof (or/c regexp? symbol?))
      (-> symbol? any)
      #f)
 = #f
检查由 lib 命名的模块路径(例如 'racket/list)是否已经记录了所有的输出, 如果没有,则打印一个错误信息到 (current-error-port) 。

如果 skip 是一个 regexp,那么与该 regexp 匹配的导出将被忽略。 如果它是一个符号,那么该出口将被忽略。 如果它是一个符号和 regexps 的列表,那么任何与任何符号或 regexps 匹配的输出都被忽略。 如果它是一个函数,那么该函数被视为一个谓词,并传递给该模块的每个出口。 如果 skip 是 #f, 则不跳过任何导出。

Changed in version 1.10 of package racket-index: 修改了 lib 以接受任何模块路径。

4.2 记录测试结果

 (require rackunit/log) package: testing-util-lib

Rackunit 提供了一个通用的库,用于跟踪测试结果并显示摘要信息。

procedure

(test-log! result) → void?

  result : any/c
将测试结果添加到运行日志中。如果 result 是假的,那么该测试被认为是失败的。

procedure

(test-log [#:display? display? 
  #:exit? exit?]) 
 → 
(cons/c exact-nonnegative-integer?
        exact-nonnegative-integer?)
  display? : any/c = #f
  exit? : any/c = #f
处理运行中的测试日志。 第一个整数是失败的测试,第二个整数是总测试数。 如果 display? 为真,则显示一条信息。 如果有失败,信息将打印在 (current-error-port) 上。 如果 exit? 为真,那么如果有失败,则调用 (exit 1)。

Changed in version 1.11 of package testing-util-lib: 允许 display? 和 exit? 参数的任何值,而不仅仅是布尔值。

parameter

(test-log-enabled?) → boolean?

(test-log-enabled? enabled?) → void?
  enabled? : any/c
 = #t
当设置为 #f 时, test-log! 是一个无用的选项。 这对于动态地禁用某些测试是很有用的,这些测试的失败是预期的,不应该被计入测试日志, 例如在测试一个自定义检查的失败行为时。

Added in version 1.1 of package testing-util-lib.
Changed in version 1.11: 允许参数的任何值,并将其胁迫为布尔值。
