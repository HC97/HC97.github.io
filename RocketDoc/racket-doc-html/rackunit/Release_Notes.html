6.2 版本 3

这个版本的 RackUnit 在很大程度上向后兼容第2版,但对底层模型有重大改变,因此需要增加主要版本号。 这些变化在 RackUnit 的理念 中得到了最好的解释。

这个版本有一些遗漏,希望能在以后的次要版本中得到纠正:

  • 没有图形用户界面,特别是没有与 DrRacket 集成。

  • test-suite 的语义不是我们想要的。 特别是,只有检查和测试用例的评估被测试套件延迟;其他表达式将在构建套件之前被评估。 这不会影响以第2版风格编写的测试。 特别是这并不影响包含其他测试套件的测试套件;它们继续以预期的方式工作。 然而,从普通检查到测试套件的增量开发测试的人可能会感到惊讶。 我希望很少有人会这样做,以至于在它被修复之前没有人注意到。
