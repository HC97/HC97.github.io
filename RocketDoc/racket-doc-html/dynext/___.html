3 文件名

 (require dynext/file) package: base

procedure

(append-zo-suffix s) → path?

  s : (or/c string? path?)
将 ".zo" 文件后缀添加到 s 中,返回一个路径。 现有的后缀(如果有的话)将被保留,并像 path-add-suffix 那样进行转换。

procedure

(append-object-suffix s) → path?

  s : path-string?
将平台标准的编译对象文件后缀加到 s 中,返回一个路径。

procedure

(append-c-suffix s) → path?

  s : path-string?
向 s 添加平台标准的 C 源文件后缀,返回一个路径。

procedure

(append-constant-pool-suffix s) → path?

  s : (or/c string? path?)
将常量库文件的后缀 ".kp" 加到 s 中,返回一个路径。

procedure

(append-extension-suffix s) → path?

  s : (or/c string? path?)
将平台标准的动态扩展文件后缀加到 s 中,返回一个路径。

procedure

(extract-base-filename/ss s [program]) → (or/c path? false/c)

  s : path-string?
  program : any/c = #f
将 Racket 文件后缀从 s中剥离,并返回一个剥离后的路径。 识别的后缀是第一次调用 extract-base-filename/ss 时由 (get-module-suffixes #:group 'libs) 报告的。

与下面的其他函数不同,当 program 不是 #f 时,那么任何后缀(包括没有后缀)都是允许的。 如果 s 不是 Racket 文件并且 program 是 #f,则返回 #f 。

procedure

(extract-base-filename/c s [program]) → (or/c path? false/c)

  s : path-string?
  program : any/c = #f
剥离 s 中的 Racket 文件后缀,并返回一个剥离后的路径。 如果 s 不是一个 Racket 文件名,并且 program 是一个符号,则会发出错误信号。 如果 s 不是一个 Racket 文件,并且 program 是 #f,则返回 #f 。

procedure

(extract-base-filename/kp s [program]) → (or/c path? false/c)

  s : path-string?
  program : any/c = #f
与 extract-base-filename/c 相同,但适用于恒定池文件。

procedure

(extract-base-filename/o s [program]) → (or/c path? false/c)

  s : path-string?
  program : any/c = #f
与 extract-base-filename/c 相同,但用于编译对象文件。

procedure

(extract-base-filename/ext s [program]) → (or/c path? false/c)

  s : path-string?
  program : any/c = #f
与 extract-base-filename/c 相同,但是对扩展文件而言。
