2 链接

 (require dynext/link) package: dynext-lib

procedure

(link-extension quiet?    
  input-files    
  output-file) → any
  quiet? : any/c
  input-files : (listof path-string?)
  output-file : path-string?
链接对象文件以创建一个扩展(通常是可以用 load-extension 加载的形式)。

quiet? 参数表示命令是否应该被回显到当前的输出端口。 input-files 参数是已编译对象文件名的列表, output-file 是目标扩展文件名。

2.1 链接参数

parameter

(current-extension-linker) → (or/c path-string? false/c)

(current-extension-linker linker) → void?
  linker : (or/c path-string? false/c)
一个决定用作链接器的可执行文件的参数。

默认设置是通过使用 PATH 环境变量搜索可执行文件, 或者使用 LD 或 MZSCHEME_DYNEXT_LINKER 环境变量(如果定义了的话,后者优先)。 在 Windows 上,它寻找 "cl.exe", 然后是 "ld.exe" (gcc), 然后是 "ilink32.exe" (Borland)。 在 Cygwin, Solaris, FreeBSD 2.x 或 HP/UX 上,它寻找 "ld" 。 在其他 Unix 变体上,它寻找 "cc" 。 #f 表示没有找到链接器。

parameter

(current-extension-linker-flags)

 → (listof (or/c path-string? (-> (listof string?))))
(current-extension-linker-flags flags) → void?
  flags : (listof (or/c path-string? (-> (listof string?))))
一个参数,决定提供给链接器的标志。 参见 expand-for-link-variant 。

在 Windows 中,对于 "cl.exe" 默认为 (list "/LD") , 对于 "ld.exe" 默认为 (list "--dll") , 而对于 "ilink32.exe" 默认为 (list "/Tpd" "/c") 。 在 Unix 上,不同平台的默认值有很大不同。 如果定义了 LDFLAGS 或 MZSCHEME_DYNEXT_LINKER_FLAGS (后者优先)环境变量, 那么它的值被解析为一个字符串列表,附加在默认值之前。

parameter

(current-make-link-input-strings)

 → (-> path-string? (listof string?))
(current-make-link-input-strings proc) → void?
  proc : (-> path-string? (listof string?))
一个处理链接器输入参数的参数;该参数值接受一个输入文件路径,并返回一个命令行的字符串列表。 默认值是 list ，

parameter

(current-make-link-output-strings)

 → (-> path-string? (listof string?))
(current-make-link-output-strings proc) → void?
  proc : (-> path-string? (listof string?))
一个处理链接器输出参数的参数;参数值需要一个输出文件路径,并为命令行返回一个字符串列表。

在 Windows 上,对于 "cl.exe",默认将 "file" 转换为 (list "/Fefile") , 对于 "ld.exe",默认转换为类似 (list "-e" "_dll_entry@12" "-o" "file") , 对于 "ilink32.exe" ,默认转换为复杂。 在 Unix 上,默认将 "file" 转换为 (list "-o" "file") 。

parameter

(current-standard-link-libraries)

 → (listof (or/c path-string? (-> (listof string?))))
(current-standard-link-libraries libs) → void?
  libs : (listof (or/c path-string? (-> (listof string?))))
一个决定提供给链接器的库的参数,此外还有其他输入。参见 expand-for-link-variant 。

对于大多数平台,默认是

(list (build-path (collection-path "mzscheme" "lib")
                  (system-library-subpath)
                  (mzdyn-thunk)))

其中 mzdyn-thunk 对于 'cgc 变量产生 (list "mzdyn.o") , 对于 '3m 变量产生 (list "mzdyn3m.o") 。 参见 current-use-mzdyn)。

parameter

(current-use-mzdyn) → boolean?

(current-use-mzdyn use-mzdyn?) → void?
  use-mzdyn? : boolean?
一个参数,决定了默认的标准链接库是否包括 "mzdyn" 库,它允许通过 load-extension 加载结果文件。 默认值为 #t 。

parameter

(link-variant) → (one-of/c 'normal 'cgc '3m)

(link-variant variant-symbol) → void?
  variant-symbol : (one-of/c 'normal 'cgc '3m)
表示链接目标的参数,其中 'normal 是 (system-type 'gc) 的结果的别名。
