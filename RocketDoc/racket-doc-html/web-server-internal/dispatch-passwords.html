2.11 密码保护

 (require web-server/dispatchers/dispatch-passwords)
  package: web-server-lib
web-server/dispatchers/dispatch-passwords 模型 定义了一个执行 HTTP Basic 认证过滤的调度器构造器。

value

denied?/c : contract?

相当于 (-> request? (or/c false/c string?)) 。 如果请求未被授权,则返回认证境界的字符串;如果请求 被 授权,则返回 #f 。

procedure

(make denied? 
  [#:authentication-responder authentication-responder]) 
 → dispatcher/c
  denied? : denied?/c
  authentication-responder : (url? header? . -> . response?)
   = (gen-authentication-responder "forbidden.html")
一个调度器,根据 denied? 来检查请求是否被拒绝。 如果是,那么就用一个请求证书的 header 来调用 authentication-responder 。 如果不是,则调用 next-dispatcher 。

value

authorized?/c : contract?

相当于 (-> string? (or/c false/c bytes?) (or/c false/c bytes?) (or/c false/c string?)) )。 输入是作为字符串的 URI 和作为字节的用户名和密码。 如果用户没有被授权,则返回认证境界的字符串,如果请求 被 授权,则返回 #f 。

procedure

(make-basic-denied?/path authorized?) → denied?/c

  authorized? : authorized?/c
从一个授权程序创建一个拒绝程序。

procedure

(password-file->authorized? password-file)

 → 
(-> void) authorized?/c
  password-file : path-string?
根据给定的密码文件创建一个授权过程。 第一个返回值是一个刷新授权过程所使用的密码缓存的过程。

password-file 被解析为:
(list ([domain : string?]
       [path : string?] ; This string is interpreted as a regex
       (list [user : symbol?]
             [pass : string?])
       ...)
      ...)
例如:

'(("secret stuff" "/secret(/.*)?" (bubba "bbq") (Billy "BoB")))
