2.9 记录

 (require web-server/dispatchers/dispatch-logresp)
  package: web-server-lib
web-server/dispatchers/dispatch-logresp 模型 定义了一个用于透明记录请求和响应的调度器构造函数。

value

format-reqresp/c : contract?

相当于 (or/c (-> request? string?) (-> request? response? string?)) 。

value

paren-format : format-reqresp/c

通过以下方式形成一个请求和一个响应:
(format
 "~s\n"
 (list 'from (request-client-ip req)
       'to (request-host-ip req)
       'for (url->string (request-uri req))
       'at (date->string
            (seconds->date (current-seconds)) #t)
       'code (response-code resp)))

value

extended-format : format-reqresp/c

通过以下方式形成一个请求和一个响应:
(format
 "~s\n"
 `((client-ip ,(request-client-ip req))
   (host-ip ,(request-host-ip req))
   (referer
    ,(let ([R (headers-assq*
               #"Referer"
               (request-headers/raw req))])
       (if R
           (header-value R)
           #f)))
   (uri ,(url->string (request-uri req)))
   (time ,(current-seconds))
   (code ,(response-code resp))))

value

apache-default-format : format-reqresp/c

像 Apache 的缺省值一样格式化一个请求和一个响应。 然而,Apache 的默认值包括返回给客户端的对象的大小信息,这个函数无法访问这些信息, 所以它将最后一个字段默认为 - 。

value

log-format/c : contract?

等同于 (symbols 'parenthesized-default 'extended 'apache-default) 。

procedure

(log-format->format id) → format-reqresp/c

  id : log-format/c
将 'parenthesized-default 映射到 paren-format, 'extended 映射到 extended-format, 'apache-default 映射到 apache-default-format 。

procedure

(make [#:format format    
  #:log-path log-path]    
  dispatcher) → dispatcher/c
  format : (or/c log-format/c format-reqresp/c) = paren-format
  log-path : (or/c path-string? output-port?) = "log"
  dispatcher : dispatcher/c
如果 dispatcher 被成功调度, 将请求和响应(不包括 body 信息)记录到 log-path, 它可以是一个文件路径或一个 output-port?, 使用 format 来格式化请求和响应(如果 format 只接受一个参数,则只记录请求)。 如果 format 是一个符号,那么将使用 log-format->format 来默示一个日志格式化器。

Added in version 1.12 of package web-server-lib.
