6 故障排除和提示

6.1 如何在 Racket Web 服务器上使用 Apache?

你可能想把 Apache 放在你的 Racket Web 服务器应用程序前面。 Apache 可以重写和代理私人(或公共)Racket Web 服务器的请求:

RewriteEngine on

RewriteRule ^(.*)$ http://localhost:8080/$1 [P,NE]

RewriteRule 的第一个参数是一个匹配模式。 第二个是如何重写 URL。括号内的部分包含指定重写类型的标志, 在本例中, P 标志指示 Apache 代理该请求。 (如果你不包括这个,Apache 将返回一个 HTTP 重定向响应, 客户端将向 localhost:8080 发出第二个请求,这在不同的机器上将无法工作)。 此外,需要 NE 标志来避免对 URL 的部分内容进行转义–如果没有这个标志, 一个 ; 被转义为 %3B ,这将破坏代理请求。

关于 RewriteRule 的更多细节,请参见 Apache 的文档。

6.2 服务器能否创建一个 PID 文件?

服务器没有这方面的选项,但你可以非常容易地添加它。 有两个技巧。

首先,如果你使用 UNIX 平台,在你的 shell 启动脚本中你可以使用

echo $$ > PID

exec run-web-server

使用 exec 将重复使用同一个进程,因此,PID 文件将是准确的。

第二,如果你想制作自己的 Racket 启动脚本,你可以写:

(require racket/os)
(with-output-to-file your-pid-file (lambda () (write (getpid))))
(start-server)

6.3 我如何设置服务器使用 HTTPS?

这需要一个 SSL 证书和私钥。 这是非常具体的平台,但我们将提供在 UNIX 上使用 OpenSSL 的细节:

  openssl genrsa -des3 -out private-key.pem 4096

这将产生一个新的私钥,但它会有一个口令。你可以通过以下方式删除它:

  openssl rsa -in private-key.pem -out private-key.pem
  chmod 400 private-key.pem

现在,我们生成一个自签名的证书:

  openssl req -new -x509 -nodes -sha1 -days 365 -key private-key.pem > server-cert.pem

(每个证书颁发机构对生成证书签名请求有不同的说明。)

现在我们可以通过以下方式启动服务器:

  plt-web-server --ssl

网络服务器将使用我们创建的 "private-key.pem" 和 "server-cert.pem" 在 443 端口启动(可以用 -p 选项覆盖)。
