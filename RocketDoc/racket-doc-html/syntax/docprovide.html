13 将文档附加到输出上

 (require syntax/docprovide) package: base

NOTE: This library is deprecated; use scribble/srcdoc, instead.

syntax

(provide-and-document doc-label-id doc-row ...)

 
doc-row = (section-string (name type-datum doc-string ...) ...)
  | (all-from prefix-id module-path doc-label-id)
  | (all-from-except prefix-id module-path doc-label-id id ...)
     
name = id
  | (local-name-id external-name-id)
一个导出名称和记录文档信息的表格。

doc-label-id 标识符被用作通过 lookup-documentation 访问文档的钥匙。 实际的文档被组织成 "行",每个行都有一个章节的标题。

一个 row 有以下形式之一:

  • (section-string (name type-datum doc-string ...) ...)

    创建一个标题为 section-string 的文档部分,并提供/记录每个 name 。 type-datum 是任意的,供调用 lookup-documentation 的客户端使用。 doc-string 也是任意的文档信息,通常由客户连接起来。

    一个 name 是一个标识符或一个重命名序列 (local-name-id extenal-name-id) 。

    多个具有相同章节名称的 row 将在文档输出中被合并。 节的最终顺序与每节第一次提到的顺序一致。

  • (all-from prefix-id module-path doc-label-id)

  • (all-from-except prefix-id module-path doc-label-id id ...)

    将指定模块的文档和规定合并到当前模块中; prefix-id 用于将导入到当前模块中的前缀(以便它们可以被重新导出)。 如果提供了 id ,指定的 id 不会被重新导出,其文档也不会被合并。

procedure

(lookup-documentation module-path-v    
  label-sym) → any
  module-path-v : module-path?
  label-sym : symbol?
返回指定模块和标签的文档。 module-path-v 参数是一个带引号的模块路径,就像 dynamic-require 的参数一样。 label-sym 标识了一组文档,使用符号作为 provide-and-document 中的标签标识符。
