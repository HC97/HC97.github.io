7 解析 for 主体

 (require syntax/for-body) package: base
syntax/for-body 模块为 for-like 句法形式提供了一个辅助函数,该函数在扩展到另一个 for-like 形式的同时包裹了该形式的主体, 并且该包裹器应该只在主体中最后一个 #:break 或 #:final 子句之后应用。

procedure

(split-for-body stx body-stxes) → syntax?

  stx : syntax?
  body-stxes : syntax?
body-stxes 参数必须具有 (pre-body ... post-body ...) 的形式, 它被改写为 ((pre-body ...) (post-body ...)) , 使得 (post-body ...) 尽可能大而不包含 #:break 或 #:final 子句。

stx 参数只用于报告语法错误。

使用 split-for-body ,而不是假设 for-like 表单中的最后一个表单可以被单独包装。 特别是,最后一个表单可能包含一些定义,这些定义需要与前面的表单在同一定义上下文中进行拼接,以创建相互递归的定义。
