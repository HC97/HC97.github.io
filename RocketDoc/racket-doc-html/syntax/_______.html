1.10 调试和检查工具

 (require syntax/parse/debug) package: base

以下是用于调试语法类的特殊形式。

syntax

(syntax-class-attributes syntax-class-id)

返回一个语法类的属性列表。 每个属性条目由属性的名称和省略号深度组成。

syntax

(syntax-class-arity syntax-class-id)

syntax

(syntax-class-keywords syntax-class-id)

分别返回语法类的算术和关键字。 与 procedure-arity 和 procedure-keywords 比较。

syntax

(syntax-class-parse syntax-class-id stx-expr arg ...)

 
  stx-expr : syntax?
在由 stx-expr 产生的语法对象上运行语法类的解析器(由 arg-expr 参数化)。 成功后,结果是一个向量列表,代表该语法类的属性绑定。 每个向量包含属性名称、深度和相关值。 失败时,结果是失败的一些内部表示。

syntax

(debug-parse stx-expr S-pattern ...+)

 
  stx-expr : syntax?
试图将 stx-expr 与 S-pattern 匹配。 如果匹配成功,返回符号 'success 。 否则,返回一个描述失败的 S 表达式。

失败的 S 表达式显示原始的失败集合(未排序)和具有最大进展的失败。 最大的失败根据其进展情况被分为等价类(进展是一个部分顺序); 也就是说,在一个等价类中的失败具有相同的进展,并且原则上将同一个词作为问题词来定位。 多个等价类只产生于匹配计算术语的 ~parse 模式(或等价的 #:with 条款) 或允许计算术语被定位的 ~fail (#:fail-when,等等)条款。

procedure

(debug-syntax-parse!) → void?

安装一个 syntax-parse 报告处理程序,当 syntax-parse 错误发生时, 将调试信息打印到当前错误端口。

Added in version 6.5.0.3 of package base.
