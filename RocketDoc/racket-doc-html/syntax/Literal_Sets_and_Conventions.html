1.7 Literal Sets and Conventions

有时,同一个字词会在许多不同的地方被识别。 最常见的例子是完全展开的程序的字元,它被用于许多分析和转换工具。 单独指定字词是很麻烦的,而且容易出错。 作为一种补救措施, syntax/parse 提供了 literal sets 。 字面集通过 define-literal-set 定义, 并通过 syntax-parse 的 #:literal-set 选项使用。

syntax

(define-literal-set id maybe-phase maybe-imports maybe-datum-literals
  (literal ...))
 
literal = literal-id
  | (pattern-id literal-id)
     
maybe-phase = 
  | #:for-template
  | #:for-syntax
  | #:for-label
  | #:phase phase-level
     
maybe-datum-literals = 
  | #:datum-literals (datum-literal ...)
     
maybe-imports = 
  | #:literal-sets (imported-litset-id ...)
将 id 定义为一个 literal set 。 每个 literal 可以有一个单独的 pattern-id 和 literal-id 。 pattern-id 决定了模式中的哪些标识符被当作字面符号。 literal-id 决定了字面意思与哪些标识符匹配。 如果 #:literal-sets 选项存在,则包括给定的 imported-litset-id 的内容。

Examples:
> (define-literal-set def-litset
    (define-values define-syntaxes))
> (syntax-parse #'(define-syntaxes (x) 12)
    #:literal-sets (def-litset)
    [(define-values (x:id ...) e:expr) 'v]
    [(define-syntaxes (x:id ...) e:expr) 's])

's

字面意义集中的字面意义总是指阶段性的 phase-level relative to the enclosing module 的绑定。 如果给出了 #:for-template 选项, phase-level 是 -1; #:for-syntax 意味着 1, 而 #:for-label 意味着 #f 。 如果没有给出 phase 关键字选项,那么 phase-level 就是 0 。

For example:

> (module common racket/base
    (define x 'something)
    (provide x))
> (module lits racket/base
    (require syntax/parse 'common)
    (define-literal-set common-lits (x))
    (provide common-lits))

在 literal set common-lits 中, literal x 总是能识别绑定在模块 'common 中定义的变量 x 的标识符。

下面的模块定义了一个等价的字词集,但是导入了 'common 模块 for-template :

> (module lits racket/base
    (require syntax/parse (for-template 'common))
    (define-literal-set common-lits #:for-template (x))
    (provide common-lits))

当一个字词集与 #:phase phase-expr 选项一起 使用 时, 字词的固定绑定会与输入字词在指定阶段的绑定进行比较。继续这个例子:

> (require syntax/parse 'lits (for-syntax 'common))
> (syntax-parse #'x #:literal-sets ([common-lits #:phase 1])
    [x 'yes]
    [_ 'no])

'yes

模式中出现的 x 匹配任何标识符, 这些标识符在第1阶段的绑定是来自模块 'common 的 x 。

syntax

(literal-set->predicate litset-id)

给出一个字词集的名称,产生一个谓词,识别字词集中的标识符。 该谓词需要一个必要参数,即标识符 id, 和一个可选参数, 即检查 id 绑定的阶段 phase ; phase 参数默认为 (syntax-local-phase-level) 。

Examples:
> (define kernel? (literal-set->predicate kernel-literals))
> (kernel? #'lambda)

#f

> (kernel? #'#%plain-lambda)

#t

> (kernel? #'define-values)

#t

> (kernel? #'define-values 4)

#f

syntax

(define-conventions name-id convention-rule ...)

 
convention-rule = (name-pattern syntax-class)
     
name-pattern = exact-id
  | name-rx
     
syntax-class = syntax-class-id
  | (syntax-class-id expr ...)
定义了为模式变量提供默认语法类的 conventions 。 一个没有明确语法类别的模式变量会被检查到每个 name-pattern 中,第一个匹配的决定了该模式的语法类别。 如果没有匹配的 name-pattern ,那么该模式变量就没有语法类。

Examples:
> (define-conventions xyz-as-ids
    [x id] [y id] [z id])
> (syntax-parse #'(a b c 1 2 3)
    #:conventions (xyz-as-ids)
    [(x ... n ...) (syntax->datum #'(x ...))])

'(a b c)

> (define-conventions xn-prefixes
    [#rx"^x" id]
    [#rx"^n" nat])
> (syntax-parse #'(a b c 1 2 3)
    #:conventions (xn-prefixes)
    [(x0 x ... n0 n ...)
     (syntax->datum #'(x0 (x ...) n0 (n ...)))])

'(a (b c) 1 (2 3))

通过 syntax-parse 和语法类定义的 #:local-conventions 关键字参数引入的本地约定, 可以指的是本地绑定:

Examples:
> (define-syntax-class (nat> bound)
    (pattern n:nat
             #:fail-unless (> (syntax-e #'n) bound)
                           (format "expected number > ~s" bound)))
> (define-syntax-class (natlist> bound)
    #:local-conventions ([N (nat> bound)])
    (pattern (N ...)))
> (define (parse-natlist> bound x)
    (syntax-parse x
      #:local-conventions ([NS (natlist> bound)])
      [NS 'ok]))
> (parse-natlist> 0 #'(1 2 3))

'ok

> (parse-natlist> 5 #'(8 6 4 2))

?: expected number > 5

  at: 4

  in: (8 6 4 2)

  parsing context:

   while parsing nat>

    term: 4

    location: eval:21:0

   while parsing natlist>

    term: (8 6 4 2)

    location: eval:21:0
