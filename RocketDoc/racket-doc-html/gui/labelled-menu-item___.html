interface

labelled-menu-item<%> : interface?

  implements: menu-item<%>
labelled-menu-item<%> 对象是一个带有字符串标签的 menu-item<%> (即除了分隔符以外的任何菜单项)。 更具体地说,它是 menu-item% (一个普通的菜单项)、 checkable-menu-item% (一个可检查的菜单项)或 menu% (一个子菜单)的一个实例。

method

(send a-labelled-menu-item enable enabled?) → void?

  enabled? : any/c
启用或停用该菜单项。如果该项目是一个子菜单(或菜单栏中的菜单),则整个菜单被禁用, 但每个子菜单项目的 is-enabled? 方法只有在该项目被特别禁用时才会返回 #f (除了子菜单之外)。

method

(send a-labelled-menu-item get-help-string)

 → (or/c label-string? #f)
返回菜单项的帮助字符串,如果该项没有帮助字符串,则返回 #f 。

当一个项目有 help 时,这个字符串可以用来向用户显示帮助信息。

method

(send a-labelled-menu-item get-label) → label-string?

返回项目的标签。

参见 set-label 和 get-plain-label 。

method

(send a-labelled-menu-item get-plain-label) → label-string?

和 get-label 一样, 除了标签中的 & 和 tab 字符会以和 set-label 一样的方式被剥离。

method

(send a-labelled-menu-item is-enabled?) → boolean?

如果菜单项被启用,返回 #t ,否则返回 #f 。

参见 enable 。

method

(send a-labelled-menu-item on-demand) → void?

Specification: 通常在用户点击包含该项目的菜单栏时(在用户看到任何菜单项之前),在包含该项目的弹出菜单弹出之前, 或者在检查包含该项目的菜单栏是否有快捷键绑定之前调用。 更多细节请参见 on-demand in menu-item-container<%> 。

一个 on-demand in menu-item-container<%> 方法可以被重写, 这样容器就不会调用其项目的 on-demand 方法。

Default implementation: 调用创建对象时提供的 demand-callback 程序。

method

(send a-labelled-menu-item set-help-string help) → void?

  help : (or/c label-string? #f)
设置菜单项的帮助字符串。 使用 #f 来删除项目的帮助字符串。

method

(send a-labelled-menu-item set-label label) → void?

  label : label-string?
设置菜单项的标签。如果该项目有一个快捷方式,该快捷方式不受影响。

如果标签包含 & ,并且窗口是一个控件,标签会被特别解析; 在 Windows 和 Unix 中, & 后面的字符在显示的菜单中会有下划线,以表示一个键盘记忆法。 在菜单栏中按下Alt键时,菜单名称中的下划线字符会导致该菜单被选中(通过 on-menu-char)。 当一个菜单拥有焦点时,记忆字符被用于导航而不需要Alt。 标签中的 && 被一个字面(非导航)的 & 取代。 在 Mac OS 上,标签中的 & 的解析方式与 Unix 和 Windows 相同,但不显示助记符下划线。 在 Mac OS 上,在显示标签之前,括号内的助记符会被删除(连同周围的任何空格),因为括号内的助记符通常用于非罗马语言。 最后,由于历史原因,如果一个标签包含一个制表符,那么在显示的菜单中,制表符和所有剩余的字符都会被隐藏。 所有这些规则都与 button% 和其他窗口中的标签处理一致。

在 get-label 返回的标签中总是保留一个 & , 但在 get-plain-label 返回的标签中从不保留。
