class

clipboard-client% : class?

  superclass: object%

clipboard-client% 对象允许程序接管剪贴板并为剪贴板数据的请求提供服务。 更多信息请参见 clipboard<%> 。

当一个 clipboard-client% 对象成为当前客户端时,它被关联到一个事件空间;更多信息请参见 set-clipboard-client 。

constructor

(new clipboard-client%) → (is-a?/c clipboard-client%)

创建一个不支持任何数据格式的剪贴板客户端。

method

(send a-clipboard-client add-type format) → void?

  format : string?
在剪贴板客户端支持的列表中添加一个新的数据格式名称。

format 字符串通常是四个大写字母。 (在 Mac OS 上,只有四个字符的 format 被使用)。 例如, "TEXT" 是 UTF-8 编码的字符串格式的名称。 新的格式名称可以用来沟通应用程序和平台特定的数据格式。

method

(send a-clipboard-client get-data format)

 → (or/c bytes? string? #f)
  format : string?
当一个进程请求剪贴板数据时被调用,而这个客户端是剪贴板的当前客户端。 请求的格式被传递给该方法,结果应该是一个与请求的格式相匹配的字节字符串,如果请求不能被满足,则为 #f 。

只有客户端列表中的数据格式名称才会被传递给这个方法;见 add-type 。

当这个方法被剪贴板调用时,当前的事件空间与客户端的事件空间相同。 如果在剪贴板请求的时候,当前的事件空间不是客户端的事件空间,那么当前线程保证是客户端事件空间的处理线程。

method

(send a-clipboard-client get-types) → (listof string?)

返回一个剪贴板客户端所支持的数据格式的名称列表。

method

(send a-clipboard-client on-replaced) → void?

当剪贴板客户端作为剪贴板所有者被解雇时被调用 (因为剪贴板被另一个客户端或外部应用程序占用)。
