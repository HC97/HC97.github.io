class

frame% : class?

  superclass: object%

  extends: top-level-window<%>
一个框架是一个顶级的容器窗口。 它有一个标题栏(显示框架的标签),一个可选的菜单栏,以及一个可选的状态线。

constructor

(new frame% 
    [label label] 
    [[parent parent] 
    [width width] 
    [height height] 
    [x x] 
    [y y] 
    [style style] 
    [enabled enabled] 
    [border border] 
    [spacing spacing] 
    [alignment alignment] 
    [min-width min-width] 
    [min-height min-height] 
    [stretchable-width stretchable-width] 
    [stretchable-height stretchable-height]]) 
 → (is-a?/c frame%)
  label : label-string?
  parent : (or/c (is-a?/c frame%) (is-a?/c dialog) #f) = #f
  width : (or/c dimension-integer? #f) = #f
  height : (or/c dimension-integer? #f) = #f
  x : (or/c position-integer? #f) = #f
  y : (or/c position-integer? #f) = #f
  style : 
(listof (or/c 'no-resize-border 'no-caption
              'no-system-menu 'hide-menu-bar
              'toolbar-button 'float 'metal
              'fullscreen-button 'fullscreen-aux))
   = null
  enabled : any/c = #t
  border : spacing-integer? = 0
  spacing : spacing-integer? = 0
  alignment : 
(list/c (or/c 'left 'center 'right)
        (or/c 'top 'center 'bottom))
   = '(center top)
  min-width : (or/c dimension-integer? #f) = #f
  min-height : (or/c dimension-integer? #f) = #f
  stretchable-width : any/c = #t
  stretchable-height : any/c = #t
label 字符串显示在框架的标题栏中。 如果框架的标签被改变(见 set-label),标题栏就会被更新。

parent 参数可以是 #f 或一个现有的框架或对话框。 在 Windows 上,如果 parent 不是 #f, 新的框架总是在其父级之上。 在 Windows 和 Unix 上(对于许多窗口管理器),当一个框架的父对象被图标化时,该框架也被图标化。

如果 parent 是 #f, 那么新框架的事件空间就是当前的事件空间, 由 current-eventspace 决定。 否则, parent 的事件空间就是新框架的事件空间。

如果 width 或 height 参数不是 #f, 它指定了一个框架的初始尺寸(像素),假设它大于最小尺寸,否则就使用最小尺寸。

如果 x 或 y 参数不是 #f, 它指定了一个框架的初始位置。 否则,就会自动选择一个位置(在创建框架和对话框时将其平铺)。

style 标志可以在某些平台上调整框架的外观:

  • 'no-resize-border — 省略窗口周围可调整大小的边框(Windows,Unix), 调整窗口大小的能力(Mac OS),或右下角的增长框(旧的 Mac OS)。

  • 'no-caption — 省略框架的标题栏(Windows、Mac OS、Unix)。

  • 'no-system-menu — 省略系统菜单(Windows)

  • 'toolbar-button — 在框架的标题栏上包括一个工具条按钮(Mac OS 10.6 和更早); 点击工具条按钮会触发对 on-toolbar-button-click 的调用

  • 'hide-menu-bar — 当框架处于活动状态时隐藏菜单栏和 dock(Mac OS), 或者要求窗口管理器使框架全屏(Unix)

  • 'float — 导致框架停留在所有其他非浮动窗口的前面(Windows、Mac OS、Unix); 在 Mac OS 上,浮动框架与活动的非浮动框架共享焦点; 当这个样式与 'no-caption 结合时,显示框架不会导致键盘焦点转移到窗口, 在 Unix 上,点击框架不会移动焦点;在 Windows 上,浮动框架没有任务栏按钮

  • 'metal — 忽略(以前支持 Mac OS)。

  • 'fullscreen-button — 在框架的标题栏上有一个按钮, 可以将框架置于全屏模式(Mac OS 10.7 及以后版本)

  • 'fullscreen-aux — 允许框架伴随另一个全屏模式的框架(Mac OS 10.7 及更高版本)。

即使框架不被显示,在创建时也会有一些通知事件被排在框架的后面。 因此,新框架的资源(如内存)不能被回收,直到一些事件被处理,或者框架的事件空间被关闭。

关于 enabled 参数的信息,请参见 window<%> 。 关于 border, spacing 和 alignment 参数的信息,请参见 area-container<%> 。 关于 min-width, min-height, stretchable-width 和 stretchable-height 参数的信息,请参见 area<%> 。

Changed in version 1.1 of package gui-lib: 为 style 添加了 'fullscreen-button 和 'fullscreen-aux 选项。
Changed in version 1.66: 允许将 dialog% 例作为 parent 对象

method

(send a-frame create-status-line) → void?

在框架的底部创建一个状态线。状态线的宽度是整个框架的宽度(调整大小时自动调整),高度和文字大小是特定平台的。

参见 set-status-text 。

method

(send a-frame fullscreen fullscreen?) → void?

  fullscreen? : any/c
将框架置于全屏模式或将框架恢复到非全屏模式。框架的显示状态不受影响。

A frame’s mode 可以被 the user 改变,而这种改变并不经过这个方法。 除了通过轮询 is-fullscreened?,程序无法检测到 a frame has been put in fullscreen mode 的时间。

在 Mac OS 上, frame% 必须以 'fullscreen-button 的样式创建, 全屏模式才能发挥作用,并且需要 Mac OS 10.7 或更高版本。

Added in version 1.9 of package gui-lib.
Changed in version 1.18: 改变了 #t 的 fullscreen , 在 Windows 和 Mac OS 上不意味着 show 。

method

(send a-frame get-menu-bar) → (or/c (is-a?/c menu-bar%) #f)

返回框架的菜单栏,如果没有为该框架创建菜单栏,则返回 #f 。

method

(send a-frame has-status-line?) → boolean?

如果框架的状态线已经被创建,返回 #t ,否则返回 #f 。 参见 create-status-line 。

method

(send a-frame iconize iconize?) → void?

  iconize? : any/c
图标化 (minimizes) 或去图标化(恢复)框架。去图标化会把框架带到前面。

A frame’s iconization 可以被 the user 改变,而这种改变并不经过这个方法。 除了通过轮询 is-iconized?,程序无法检测到 a frame has been iconized 的时间。

method

(send a-frame is-fullscreened?) → boolean?

如果框架处于全屏模式,返回 #t ,否则返回 #f 。

Added in version 6.0.0.6 of package gui-lib.

method

(send a-frame is-iconized?) → boolean?

如果框架被图标化(最小化),返回 #t ,否则返回 #f 。

method

(send a-frame is-maximized?) → boolean?

在 Windows 和 Mac OS 上,如果框架被最大化,则返回 #t ,否则返回 #f 。 在 Unix 上,结果总是 #f 。

method

(send a-frame maximize maximize?) → void?

  maximize? : any/c
Specification: 在 Windows 和 Mac OS 上,最大化或恢复框架;框架的显示状态不受影响。 在 Windows 上,一个图标化的框架不能被最大化或恢复。

A window’s maximization 可以被 the user 改变,这种改变不经过这个方法;使用 on-size 来监控 size 的变化。

Default implementation: 如果 maximize? 为 #f, 则窗口被恢复,否则被最大化。

method

(send a-frame modified) → boolean?

(send a-frame modified modified?) → void?
  modified? : any/c
获取或设置反映给用户的框架的修改状态。 在 Mac OS 上,修改状态反映为框架的关闭按钮上的一个点。 在 Windows 和 Unix 上,修改状态反映在框架显示的标题末尾的一个星号上。

method

(send a-frame on-menu-char event) → boolean?

  event : (is-a?/c key-event%)
如果框架有一个带有键盘快捷键的菜单栏, 并且如果按键事件包括 Control、Alt、Option、Meta、Command、Shift 或 Function 键, 那么 on-menu-char 会尝试将给定的事件与一个菜单项匹配。 如果找到了匹配,将返回 #t ,否则将返回 #f 。

当匹配到一个完整的快捷键组合时,菜单项的回调被调用(在 on-menu-char 返回之前)。

如果该事件不对应于一个完整的快捷键组合,如果它对应于菜单栏中的一个助记符 (即菜单标题中的下划线字母,通过在菜单的标签中加入一个安培符来安装),那么该事件还是可以被处理。 如果找到了记忆体匹配,键盘焦点就会被移到菜单栏上(选择有记忆体的菜单),并返回 #t 。
