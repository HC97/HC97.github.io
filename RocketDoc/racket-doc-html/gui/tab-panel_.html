class

tab-panel% : class?

  superclass: vertical-panel%

一个标签面板将其子窗口安排在一列,但也包括面板顶部的一列水平标签。 参见 panel% 。

当一个新的标签被选中时, tab-panel% 类并没有实现面板内容的虚拟互换。 相反,它只是调用了一个回调过程来表示用户改变了标签的选择。

创建一个标签面板,其中的 choices 列表指定了标签的标签。

choices 中的每个字符串都可以包含一个安培尔,(在未来)它可以创建一个点击相应标签的记忆法。 一个双连字符被转换为一个单连字符。

当用户改变标签选择时, callback 过程被调用(事件类型为 'tab-panel) 。

如果 style 列表包括 'no-border, 面板内容周围就不会有边界。 如果 style 列表包括 'can-reorder, 那么用户可以拖动标签来重新排序,在这种情况下, on-reorder 被调用; 如果 'no-border 也包含在 style 中,重新排序总是被启用的。 如果 style 列表中包括 'can-close, 那么用户可以点击一个标签的关闭图标,在这种情况下, on-close-request 被调用; 如果 'no-border 也包含在 style 中,关闭总是被启用。 如果 style 列表中包括 'flat-portable , 或者在加载 racket/gui 时定义了 PLT_FLAT_PORTABLE_TAB_PANEL 环境变量, 并且如果样式列表中还包括 'no-border, 那么标签控件将使用一个独立于平台的实现; 如果 'can-reorder 或 'can-close 被包括在 style 中, 'flat-portable 标志实际上总是包括在 Windows 上。 如果 style 列表包括 'new-button ,并且标签控件使用了独立于平台的实现, 那么一个新的标签按钮就会被添加到最后一个标签的右边,以允许插入新的标签。如果新标签按钮被点击,就会调用 on-new-request 。 如果 style 包含 'deleted, 那么 tab panel 就被创建为隐藏的,它不会影响其父辈的几何形状; tab panel 可以通过调用 parent 的 add-child 方法,在以后变成活动的。

font 参数决定了控件的字体。 关于 enabled 参数的信息,请参见 window<%> 。 关于 horiz-margin 和 vert-margin 参数的信息,请参见 subarea<%> 。 关于 min-width, min-height, stretchable-width 和 stretchable-height 参数的信息,请参见 area<%> 。

Changed in version 1.55 of package gui-lib: 增加了 'can-reorder 和 'can-close 样式。
Changed in version 1.56: 增加了 'flat-portable 样式,在 Windows 下支持重新排序和关闭。
Changed in version 1.62: 增加了 'new-button 样式。

method

(send a-tab-panel append choice) → void?

  choice : label-string?
在面板顶行标签的右端增加了一个标签。

标签字符串的 choice 可以包含 &, 这(在未来)可以为点击新标签创造一个记忆法。 一个 && 被转换为 & 。

  n : exact-nonnegative-integer?
删除一个现有的标签。如果 n 等于或大于面板上的标签数量, an exn:fail:contract exception is raised 。

  n : exact-nonnegative-integer?
按位置获取一个标签的标签。 标签的编号从 0 开始。 如果 n 等于或大于面板上标签的数量, an exn:fail:contract exception is raised 。

method

(send a-tab-panel get-number) → exact-nonnegative-integer?

返回面板上标签的数量。

method

(send a-tab-panel get-selection)

 → (or/c exact-nonnegative-integer? #f)
返回当前选择的标签的索引(从 0 开始计算)。如果面板上没有标签,结果是 #f 。

method

(send a-tab-panel on-reorder former-indices) → void?

  former-indices : (listof exact-nonnegative-integer?)
Refine this method with augment.
当用户通过拖动来重新排序标签时调用,在创建面板时, 通过包含 'can-reorder 样式(可能是 'no-border) 来启用。 对于每个新的标签位置, former-indices 列表报告标签在重新排序前的位置。

Added in version 1.55 of package gui-lib.

method

Added in version 1.55 of package gui-lib.
