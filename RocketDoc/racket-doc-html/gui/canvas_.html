class

canvas% : class?

  superclass: object%

  extends: canvas<%>
canvas% 对象是一个通用的窗口,用于绘图和处理事件。 关于在 canvas 上绘图的信息,见 canvas<%> 。

constructor

(new canvas% 
    [parent parent] 
    [[style style] 
    [paint-callback paint-callback] 
    [label label] 
    [gl-config gl-config] 
    [enabled enabled] 
    [vert-margin vert-margin] 
    [horiz-margin horiz-margin] 
    [min-width min-width] 
    [min-height min-height] 
    [stretchable-width stretchable-width] 
    [stretchable-height stretchable-height]]) 
 → (is-a?/c canvas%)
  parent : 
(or/c (is-a?/c frame%) (is-a?/c dialog%)
      (is-a?/c panel%) (is-a?/c pane%))
  style : 
(listof (or/c 'border 'control-border 'combo
              'vscroll 'hscroll 'resize-corner
              'gl 'no-autoclear 'transparent
              'no-focus 'deleted))
 = null
  paint-callback : ((is-a?/c canvas%) (is-a?/c dc<%>) . -> . any)
   = void
  label : (or/c label-string? #f) = #f
  gl-config : (or/c (is-a?/c gl-config%) #f) = #f
  enabled : any/c = #t
  vert-margin : spacing-integer? = 0
  horiz-margin : spacing-integer? = 0
  min-width : (or/c dimension-integer? #f) = #f
  min-height : (or/c dimension-integer? #f) = #f
  stretchable-width : any/c = #t
  stretchable-height : any/c = #t
style 参数表示以下一种或多种样式:

  • 'border — 给予画布一个薄的边界

  • 'control-border — 给予画布一个边界,就像一个 text-field% 的控件一样

  • 'combo — 给予画布一个组合按钮,就像一个 combo-field% 的控件; 这个样式旨在与 'control-border 一起使用, 而不是与 'hscroll 或 'vscroll 一起使用。

  • 'hscroll — 启用水平滚动(最初是可见的,但不活动)。

  • 'vscroll — 启用垂直滚动(最初可见,但不活动)。

  • 'resize-corner — 当只有一个滚动条可见时,在画布的右下方为调整大小的控件留下空间

  • 'gl — 创建一个用于 OpenGL 绘图的画布,而不是普通的 dc<%> 绘图; 在 get-dc 的结果上调用 get-gl-context 方法; 这种风格通常与 'no-autoclear 结合使用。

  • 'no-autoclear — 防止窗口系统自动擦除画布;关于画布刷新的信息请看 canvas<%> 。

  • 'transparent — 窗口系统通过让其父级显示出来来 “erased” 画布; 关于窗口刷新以及 'transparent 和屏幕外缓冲的交互信息, 请参见 canvas<%> ;如果该标志与 'no-autoclear 相结合,结果将无法确定。

  • 'no-focus — 防止画布在被点击或调用 focus 方法时接受键盘焦点。

  • 'deleted — 创建的画布最初是隐藏的,并且不影响 parent 的几何形状; 画布可以在以后通过调用 parent 的 add-child 方法使其激活。

'hscroll 和 'vscroll 样式创建了一个具有初始非活动滚动条的画布。 滚动条可以通过 init-manual-scrollbars 或 init-auto-scrollbars 激活, 它们也可以通过 show-scrollbars 隐藏并重新显示出来。

paint-callback 参数是由默认的 on-paint 方法调用的, 使用画布和 get-dc 返回的DC作为参数。

label 参数为 get-label 命名画布,但它不与画布一起显示。

gl-config 参数决定了这个画布的 OpenGL 上下文的属性,这是通过画布的绘图上下文获得的。 请参见 get-dc 和 get-gl-context in dc<%> 。

关于 enabled 参数的信息,请参见 window<%> 。 关于 horiz-margin 和 vert-margin 参数的信息,请参见 subarea<%> 。 关于 min-width, min-height, stretchable-width 和 stretchable-height 参数的信息,请参见 area<%> 。

method

(send a-canvas get-gl-client-size)

 → 
dimension-integer? dimension-integer?
对于一个具有 'gl 风格的 canvas% 实例,返回以 OpenGL 单位表示的 canvas 的绘图区域尺寸。

在没有 'gl 样式的画布中,或者在 Windows 和 Unix 中, 其结果与 get-scaled-client-size 相同。 在 Mac OS 上,如果创建时提供的 gl-config% 规范没有指定高分辨率模式, 则结果可能与 get-client-size 相同。

Added in version 1.16 of package gui-lib.

(send a-canvas get-view-start)

 → 
dimension-integer? dimension-integer?
如果水平和垂直滚动条被初始化为自动滚动条(见 init-auto-scrollbars), 则根据水平和垂直滚动条的当前值获取画布可见部分的起始位置。 与 get-client-size 相结合, 一个应用程序可以有效地只重绘画布的可见部分。这些值的单位是像素。

(send a-canvas get-virtual-size)

 → (value dimension-integer? dimension-integer?)
获取可滚动画布区域的设备单位大小(相对于客户端大小,即当前可见的画布区域)。 除非滚动条被初始化为自动的(见 get-client-size) , 否则这个尺寸与客户端的尺寸相同(由 init-auto-scrollbars 返回)。

method

(send a-canvas init-auto-scrollbars horiz-pixels    
  vert-pixels    
  h-value    
  v-value) → void?
  horiz-pixels : (or/c positive-dimension-integer? #f)
  vert-pixels : (or/c positive-dimension-integer? #f)
  h-value : (real-in 0.0 1.0)
  v-value : (real-in 0.0 1.0)
启用并初始化画布的自动滚动条。 水平或垂直滚动条只能在分别用 'hscroll 或 'vscroll 样式标志创建的画布中被激活。

有了自动滚动条,程序员指定了画布所需的虚拟尺寸,滚动条被自动处理,以允许用户在虚拟区域内滚动。 如果不需要的话,滚动条不会被自动隐藏;参见 show-scrollbars 。

鼠标事件的坐标(传递给 on-event) 不会被调整以考虑到滚动条的位置; 使用 get-view-start 方法来找到合适的偏移量。

参见 init-manual-scrollbars ,了解有关手动滚动条的信息。 水平和垂直滚动条总是要么都是手动的,要么都是自动的,但它们是独立启用的。 自动滚动条可以被重新初始化为手动,反之亦然。

如果 horiz-pixels 或 vert-pixels 是 #f, 则相应方向的滚动条不会被启用,画布在该方向的虚拟尺寸与它的客户端尺寸相同。

h-value 和 v-value 参数指定了滚动条的初始值,是滚动条范围的一个分数。 0.0 的值将滚动条初始化为其左/顶部,而 1.0 的值将滚动条初始化为其右/底部。

可以通过再次调用这个函数来调整虚拟尺寸。

参见 on-scroll 和 get-virtual-size 。

method

(send a-canvas init-manual-scrollbars h-length    
  v-length    
  h-page    
  v-page    
  h-value    
  v-value) → void?
  h-length : (or/c dimension-integer? #f)
  v-length : (or/c dimension-integer? #f)
  h-page : positive-dimension-integer?
  v-page : positive-dimension-integer?
  h-value : dimension-integer?
  v-value : dimension-integer?
启用并初始化画布的手动滚动条。 水平或垂直滚动条只能在分别用 'hscroll 或 'vscroll 样式标志创建的画布中被激活。

使用手动滚动条,程序员负责管理滚动条的所有细节,滚动条的状态对画布的虚拟大小没有影响。 相反,画布的虚拟尺寸与它的客户端尺寸相同。

关于自动滚动条的信息,请参见 init-auto-scrollbars 。 水平和垂直滚动条总是同时是手动或同时是自动的,但它们是独立启用的。 自动滚动条可以被重新初始化为手动,反之亦然。

h-length 和 v-length 参数指定了每个滚动条在滚动步骤中的长度(即每个滚动条的最大值)。 如果这两个参数是 #f, 则相应方向的滚动条被禁用。

h-page 和 v-page 参数设置了一个页面中的滚动条步数, 即在滚动条控件中按下数值指示器上方或下方时移动的数量。

h-value 和 v-value 参数指定了滚动条的初始值。

如果 h-value 大于 h-length 或 v-value 大于 v-length, an exn:fail:contract exception is raised。 (The page step may be (页面的步长可能大于滚动条的总尺寸)。

参见 on-scroll 和 get-virtual-size 。

method

(send a-canvas on-scroll event) → void?

  event : (is-a?/c scroll-event%)

method

(send a-canvas refresh-now [paint-proc    
  #:flush? flush?]) → void?
  paint-proc : ((is-a?/c dc<%>) . -> . any)
   = (lambda (dc) (send a-canvas on-paint))
  flush? : any/c = #t
用画布的绘图上下文调用 paint-proc 来立即更新画布(与 refresh 相反, refresh 只是排队等待更新请求,由窗口系统决定是否处理)。

在调用 paint-proc 之前,画布的刷新被禁用。 另外,画布会被擦除,除非画布具有 'no-autoclear 的风格。 在 paint-proc 返回后,冲洗被启用,如果 flush? 为真, 则立即调用 flush 。

method

(send a-canvas scroll h-value v-value) → void?

  h-value : (or/c (real-in 0.0 1.0) #f)
  v-value : (or/c (real-in 0.0 1.0) #f)
设置自动滚动条的值。(这个方法对手动滚动条没有影响)。

如果任何一个参数是 #f, 滚动条的值就不会在相应的方向上改变。

h-value 和 v-value 参数分别指定了滚动条移动的一个分数。 0.0 的值将滚动条设置在其左边/顶部,而 1.0 的值将滚动条设置在其右边/底部。 0.5 的值将滚动条设置在中间。 一般来说,如果画布的虚拟尺寸是 v, 它的客户端尺寸是 c, 并且 (> v c), 那么滚动到 p 会将视图的起点设置为 (floor (* p (- v c))) 。

参见 init-auto-scrollbars 和 get-view-start 。

(send a-canvas show-scrollbars show-horiz?    
  show-vert?) → void?

显示或隐藏由 show-horiz? 和 show-vert? 指示的滚动条。 如果 show-horiz? 为真,并且画布不是用 'hscroll 样式创建的, an exn:fail:contract exception is raised 。 同样,如果 show-vert? 为真,并且画布不是用 'vscroll 样式创建的, an exn:fail:contract exception is raised 。

method

(send a-canvas swap-gl-buffers) → void?

在 get-dc 返回的这个画布的 DC 的 get-gl-context 的结果上调用 swap-buffers 。

swap-buffers in gl-context<%> 方法会获得一个可重入的锁,因此在不同的线程或 OpenGL 上下文中嵌套调用 swap-gl-buffers 或 with-gl-context 会阻塞或死锁。

method

(send a-canvas with-gl-context thunk    
  [#:fail fail]) → any
  thunk : (-> any)
  fail : (-> any) = (lambda () (error ....))
将给定的 thunk 传给 get-gl-context 的结果的 call-as-current, 即 get-dc 返回的这个画布的DC。 如果 get-gl-context 返回 #f, 那么就会调用 fail 。 call-as-current in gl-context<%> 会获得一个可重入的锁, 因此在不同的线程或 OpenGL 上下文中嵌套调用 with-gl-context 或 swap-gl-buffers 会阻塞或死锁。
