class

editor-admin% : class?

  superclass: object%

关于管理员的角色信息,请参见 管理员 。 editor-admin% 类从未被直接实例化。 大多数程序员甚至不会通过派生类来实例化它; 每个 editor-canvas% 和 editor-snip% 对象都会创建自己的管理员。 然而,派生出这个类的一个新实例以在一个新的环境中显示编辑器可能是有用的。 此外,从一个拥有的编辑器中调用一个现有的管理员的方法也可能是有用的。

要创建一个新的 editor-admin% 类,这里描述的所有方法都必须被重写。 它们都是由管理员的编辑器调用的。

constructor

(new editor-admin%) → (is-a?/c editor-admin%)

创建一个(无用的)编辑器管理员。

method

(send an-editor-admin get-dc [x y]) → (or/c (is-a?/c dc<%>) #f)

  x : (or/c (box/c real?) #f) = #f
  y : (or/c (box/c real?) #f) = #f
Specification: 返回编辑器所显示的绘图环境,或当前正在绘制的环境。 当编辑器没有被嵌入时,返回的上下文始终是编辑器被显示的绘图内容。 如果编辑器没有被显示,则返回 #f 。

绘图环境的原点也会被返回,并被转换为编辑器的本地坐标。 对于一个嵌入的编辑器,返回的原点只在编辑器被绘制时,或者在它接收到鼠标或键盘事件时才可靠。

The x box is filled with the x-origin of the DC in editor coordinates, unless x is #f. The y box is filled with the y-origin of the DC in editor coordinates, unless y is #f.

See also editor-location-to-dc-location in editor<%> and dc-location-to-editor-location in editor<%>.

Default implementation: 用 0.0 填充所有方框,并返回 #f 。

method

(send an-editor-admin get-max-view x    
  y    
  w    
  h    
  [full?]) → void?
  x : (or/c (box/c real?) #f)
  y : (or/c (box/c real?) #f)
  w : (or/c (box/c (and/c real? (not/c negative?))) #f)
  h : (or/c (box/c (and/c real? (not/c negative?))) #f)
  full? : any/c = #f
Specification: 与 get-view 相同,除非编辑器在多个标准 display 中可见。 如果编辑器有多个 display ,会计算出一个区域,包括所有 display 中的可见区域。

参见 get-view 。

Default implementation: 用 0.0 填充所有方框。

method

(send an-editor-admin get-view x y w h [full?]) → void?

  x : (or/c (box/c real?) #f)
  y : (or/c (box/c real?) #f)
  w : (or/c (box/c (and/c real? (not/c negative?))) #f)
  h : (or/c (box/c (and/c real? (not/c negative?))) #f)
  full? : any/c = #f
Specification: 获取编辑器在其 display 中的可见区域(以编辑器坐标为单位), 或编辑器顶层 display 中的查看区域的整体大小(对于嵌入式编辑器)。

如果 display 是一个编辑器画布,也请看 reflow-container 。 编辑器画布内的查看区域不是画布的全部客户端区域, 因为编辑器画布在客户端区域内显示的编辑器周围安装了一个空白边界。

编辑器的可见区域的计算是基于顶层 display 的当前尺寸和滚动条值。 对于一个编辑器画布的 display, get-view 所报告的区域并不取决于画布是否被隐藏, 是否被其他窗口遮挡,或者是否被移到了屏幕的边缘。

The x box is filled with 以编辑器坐标表示的可见区域的左边缘, unless x is #f. The y box is filled with 编辑器坐标中可见区域的上边缘, unless y is #f. The w box is filled with 可见区域的宽度,可能大于编辑器本身的宽度, unless w is #f. The h box is filled with 可见区域的高度,可能大于编辑器本身的高度, unless h is #f.

如果一个编辑器是完全可见的,并且 full? 是 #f, 那么 x 和 y 都将被填充为 0 。

如果 full? 是一个真值,那么返回的区域就是编辑器的顶层 display 的视图区域。 只有当编辑器被嵌入到另一个编辑器中时,这个结果才是不同的; 在这种情况下, x 和 y 的值可能是没有意义的, 因为它们是在顶层 display 的直接编辑器的坐标系统中。

Default implementation: 用 0.0 填充所有方框。

method

(send an-editor-admin grab-caret [domain]) → void?

  domain : (or/c 'immediate 'display 'global) = 'global
Specification: 由编辑器调用以请求键盘焦点。 如果请求被批准,那么被管理的编辑器的 own-caret 方法将被调用。

参见 set-caret-owner 以获得关于 domain 的可能值的信息。

Default implementation: 什么都不做。

method

(send an-editor-admin modified modified?) → void?

  modified? : any/c
Specification: 由编辑器调用,报告它的修改状态已经改变为修改或未修改。

参见 set-modified in editor<%> 。

Default implementation: 什么都不做。

method

(send an-editor-admin needs-update localx    
  localy    
  w    
  h) → void?
  localx : real?
  localy : real?
  w : (and/c real? (not/c negative?))
  h : (and/c real? (not/c negative?))
Specification: 由编辑器调用,要求刷新: 由编辑器调用,要求刷新其显示的表示。 当管理员决定应该刷新显示的内容时,它会调用编辑器的 refresh 方法。

localx, localy, w 和 h 参数指定要更新的编辑器区域(在编辑器坐标中)。

Default implementation: 什么都不做。

method

(send an-editor-admin popup-menu menu x y) → boolean?

  menu : (is-a?/c popup-menu%)
  x : real?
  y : real?
Specification:
在指定的坐标(在这个窗口的坐标中)弹出给定的 popup-menu% 对象, 并在处理了未指定数量的事件后返回;当此方法返回时,菜单可能仍然被弹出。 如果从弹出的菜单中选择了一个菜单项,就会调用该菜单项的回调。 (菜单项的回调的事件空间是 administrator’s display 的事件空间)。
当菜单被弹出时,其目标被设置为 top-level editor in this administrator’s display 。 参见 get-popup-target 以了解更多信息。
The result is #t if the popup succeeds, #f otherwise (independent of whether the user selects an item in the popup menu).

菜单显示在编辑器坐标的 x 和 y 处。

Default implementation: 返回 #f 。

method

(send an-editor-admin refresh-delayed?) → boolean?

Specification: 如果当前在这个管理员的 display 上的更新被延迟了, 则返回 #t (通常是通过包围编辑器的 begin-edit-sequence in editor<%> )。

Default implementation: 返回 #f 。

method

(send an-editor-admin resized refresh?) → void?

  refresh? : any/c
Specification: 由编辑器调用,通知它的 display ,编辑器的尺寸或滚动计数已经改变,所以需要调整滚动条以反映新的尺寸。 编辑器一般需要在调整大小后进行更新,但是编辑器决定是否要立即进行更新。 如果 refresh? 不是 #f, 那么编辑器就会要求立即更新。

Default implementation: 什么都不做。

method

(send an-editor-admin scroll-to localx    
  localy    
  w    
  h    
  [refresh?    
  bias]) → boolean?
  localx : real?
  localy : real?
  w : (and/c real? (not/c negative?))
  h : (and/c real? (not/c negative?))
  refresh? : any/c = #t
  bias : (or/c 'start 'end 'none) = 'none
Specification: 由编辑器调用,要求滚动,以便给定的区域是可见的。 编辑器一般需要在滚动后进行更新,但是编辑器决定是否要立即进行更新。

localx, localy, w 和 h 参数指定编辑器的一个区域,以便通过滚动使其可见(以编辑器坐标为单位)。

如果 refresh? 不是 #f, 那么编辑器就会要求立即被更新。

bias 参数是以下之一:

  • 'start — 如果范围不适合在可见区域内,显示左上角的区域

  • 'none — 没有特殊的滚动指令

  • 'end — 如果范围不适合在可见区域内,显示右下角区域

如果 display 被滚动了,返回值是 #t ， 如果没有，则 #f (要么是因为请求的区域已经可见,要么是因为显示器的尺寸为零,要么是因为编辑器正在打印)。

如果一个编辑器有多个 displays ,那么如果任何一个显示器当前拥有键盘焦点,它就会被滚动。 否则,编辑器的 "主要所有者"(见 call-as-primary-owner) 将被滚动。

Default implementation: 返回 #f

method

(send an-editor-admin update-cursor) → void?

Specification: 为这个编辑器的 display 的光标排一个更新队列。 实际使用的光标将通过调用编辑器的 adjust-cursor 方法决定。

Default implementation: 什么都不做。
