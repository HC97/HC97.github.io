class

snip-class% : class?

  superclass: object%

有用的狙击类是通过实例化 snip-class% 的派生子类来定义的。 从 snip-class% 派生出来的类是一种狙击的 "元类"; 每个狙击都与 snip-class% 的实例有关,作为它的狙击类。 关于派生一个新的狙击类的更多信息,请参见 实现新的剪贴画 。

constructor

(new snip-class%) → (is-a?/c snip-class%)

创建一个(无用的)snip类。

method

(send a-snip-class get-classname) → string?

返回该类的名字,一个唯一指定该狙击类的字符串。 例如,标准的文本狙击类的名称是 "wxtext" 。 以 wx 开头的名字是保留的。

一个狙击类的名字通常应该是 "((lib ...) (lib ...))" 的形式,以便能够按需加载该类。 详见 剪辑类 。

method

(send a-snip-class get-version) → exact-integer?

返回这个狙击类的版本。 当试图加载一个包含相同类名但不同版本的 snip 的文件时,用户会被警告。

method

(send a-snip-class read f) → (or/c (is-a?/c snip%) #f)

  f : (is-a?/c editor-stream-in%)
Specification: 从一个给定的流中读取一个 snip,返回一个新创建的 snip 作为结果,如果有错误则返回 #f 。

Default implementation: 返回 #f 。

method

(send a-snip-class read-header f) → boolean?

  f : (is-a?/c editor-stream-in%)
Specification: 为读取头信息而调用: 调用该方法是为了读取头信息,这些信息可能对该类中读取的每个 snip 有用。 这个方法在每个编辑器的读取会话中只被调用一次,并且只在流中包含本类的头信息时调用。

如果发生读取错误,返回值是 #f ,否则就是其他。

参见 write-header 。

Default implementation: 返回 #t 。

method

(send a-snip-class reading-version stream) → exact-integer?

  stream : (is-a?/c editor-stream-in%)
返回当前从给定流中读取的该狙击类的版本号。

method

(send a-snip-class set-classname name) → void?

  name : string?
设置该类的名称。参见 get-classname 。

method

(send a-snip-class set-version v) → void?

  v : exact-integer?
设置该类的版本。参见 get-version 。

method

(send a-snip-class write-header stream) → boolean?

  stream : (is-a?/c editor-stream-out%)
Specification: 调用该方法是为了写入头信息,这些信息可能对为该类写的每个片段都有用。 这个方法在每个编辑器写会话中只被调用一次,而且只在编辑器中包含这个类的片段时调用。

当读回片段时, read-header 只有在 write-header 写了一些数据到流中时才会被调用。

如果发生写入错误,返回值是 #f ,否则就是其他。

Default implementation: 返回 #t 。
