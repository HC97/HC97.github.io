class

key-event% : class?

  superclass: event%

一个 key-event% 对象包含了关于一个按压或释放按键事件的信息。 按键事件主要由 on-subwindow-char in window<%> 和 on-char in canvas<%> 处理。

对于一个按键事件,一个虚拟按键代码由 get-key-code 提供。 对于一个按键释放事件, get-key-code 报告 'release, 而虚拟按键代码由 get-key-release-code 提供。

另见 鼠标和键盘事件 。

由 get-key-release-code 返回的释放键代码被初始化为 'press 。

Changed in version 1.2: 增加了 control+meta-is-altgr 。

method

(send a-key-event get-alt-down) → boolean?

如果 Option(Mac OS)键在事件中被按下,返回 #t 。 当 Alt 键在 Windows 中被按下时,它被报告为 Meta 键(见 get-meta-down) 。

method

(send a-key-event get-caps-down) → boolean?

如果事件中 Caps Lock 键处于开启状态,则返回 #t 。

method

(send a-key-event get-control-down) → boolean?

如果事件中 Control 键被按下,则返回 #t 。

在 Mac OS 上,如果按 Control 键与鼠标按键结合在一起,该事件被报告为右键点击, get-control-down 对该事件的报告为 #f 。

method

(send a-key-event get-control+meta-is-altgr) → boolean?

如果在 Windows 上 Control 加 Meta 事件应被视为 AltGr 事件,则返回 #t 。 默认情况下,如果 Control 键是左键,而 Alt 键(作为 Meta 键)是右键,在有右 Alt 键的键盘上这样键入, 或者由一个 AltGr 键产生,则适用 AltGr 处理。 也请参见 any-control+alt-is-altgr,它控制其他 Control 加 Alt 的组合是否被视为 AltGr。

Added in version 1.2 of package gui-lib.

特殊按键符号试图捕捉那些没有标准 ASCII 表示的有用按键。 有几个键的标准表示方式并不明显:

  • #\space — 空格键

  • #\return — Enter 或 Return 键(在所有平台上), 但不一定是靠近数字键盘的 Enter 键(被报告为 'numpad-enter Unix 和 Mac OS)

  • #\tab — tab 键

  • #\backspace — 退格键

  • #\rubout — 删除键

如果没有合适的特殊键符号或 ASCII 表示,则报告为 #\nul (NUL 字符)。

一个 'wheel-up, 'wheel-down, 'wheel-left 或 'wheel-right 的事件可能被发送到一个非键盘焦点的窗口, 因为一些平台是根据鼠标指针的位置而不是键盘焦点来生成滚轮事件的。

在 Windows 上,当按下 Control 键而不按 Alt 键时,ASCII 字符的键码会下移,大致上取消了 Shift 键的效果。 在 Mac OS 上,当按下 Control 或 Command 键时,键码的计算不受 Caps Lock 的影响; 在 Control 的情况下,如果通过 special-control-key 禁用 Control 键的特殊处理,则会正常使用 Caps Lock。 在 Unix 中,当按下 Control 键而不按下 Alt 键时,计算出的键码具有 Caps Lock 效果。

参见 get-other-shift-key-code 。

Changed in version 6.1.0.8 of package gui-lib: 将小键盘回车的报告改为文档中的 'numpad-enter , 而不是 #\u0003 。

method

(send a-key-event get-other-shift-key-code)

 → (or/c char? key-code-symbol? #f)
由于键盘的映射方式不同,有时在按键映射中,如果 Shift 键以不同的方式切换,程序会知道键盘对一个事件产生的结果,这是非常有用的。 get-other-shift-key-code 产生其他的映射, 如果替代的映射不可用,返回 #f ,否则返回与 get-key-code 相同的结果。

get-other-altgr-key-code 方法提供了与 Windows 和 Unix 中 AltGr 键(即 Alt 与 Control 结合)或 Mac OS 中 Option 键相同的信息。 get-other-shift-altgr-key-code 方法在 Shift 和 AltGr/Option 都与实际事件不同的情况下, 报告一个映射。

get-other-shift-key-code, get-other-altgr-key-code 和 get-other-shift-altgr-key-code 的结果都报告了 Caps Lock 关闭时的按键映射, 与实际事件中 Caps Lock 是否开启无关。 get-other-caps-key-code 方法报告了一个映射, 在这种情况下,Caps Lock 状态的处理与 get-key-code 的结果相反。 (Caps Lock 通常没有效果或与 Shift 有相同的效果, 所以涉及 Caps Lock 和其他修饰键的进一步组合通常不会产生进一步的替代方案。)

不是所有的事件都有备用映射。 在 Windows 上,当替代映射产生 ASCII 字母、ASCII 数字和 ASCII 符号时,会被报告。 在 Mac OS 和 Unix 上,通常有备用的映射。

method

(send a-key-event get-wheel-steps) → nonnegative-real?

返回 'wheel-up,'wheel-down, 'wheel-left 或 'wheel-right 事件所代表的轮子步数。 对于系统生成的事件,轮子事件的值总是正的,而其他事件的值总是 0.0 。 一个新创建的 key-event% 的初始值是 0.0 。

参见 wheel-event-mode in window<%> 。

Added in version 1.43 of package gui-lib.
