4.1 对话框

这些函数从用户那里获得输入和/或显示消息。

  filters

  style : (listof (or/c 'packages 'enter-packages 'common))
   = null
  filters : (listof (list/c string? string?)) = '(("Any" "*.*"))
  dialog-mixin : (make-mixin-contract path-dialog%) = (λ (x) x)
通过特定平台的标准(模式)对话框从用户那里获得一个文件路径名,如果指定了 parent 作为父窗口, 则使用 message 作为对话框顶部的信息,如果它不是 #f 。

如果用户取消对话框,结果是 #f ,否则是所选路径名。 返回的路径名可能存在也可能不存在,尽管对话框的风格是针对选择现有的文件。

如果 directory 不是 #f, 它将被用作文件选择器的起始目录 (否则起始目录将以特定平台的方式自动选择, 通常基于当前目录和用户在先前调用 get-file, put-file 等时的交互)。 如果 filename 不是 #f, 在适当的时候会被用作默认的文件名, 它 不 应该包含一个目录路径前缀。

在 Windows 下,如果 extension 不是 #f, 如果用户没有提供扩展名,返回的路径将使用该扩展名; extension 字符串不应包含句号。 在其他平台上,扩展名会被忽略。

style 列表中可以包含 'common,则使用与平台无关的版本的对话框,而不是本地对话框。 在 Mac OS 上,如果 style 列表包含 'packages, 则允许用户选择一个软件包目录,这是一个具有特殊后缀(例如".app")的目录,Finder 通常像文件一样显示。 如果列表中包含 'enter-packages, 则允许用户选择 package 目录中的一个文件。 如果列表中同时包含 'packages 和 'enter-packages, 则前者会被忽略。

在Windows和Unix中, filters 决定了一组过滤器,用户可以在对话框中选择。 filters 列表中的每个元素都包含两个字符串:一个是用户看到的过滤器的描述,另一个是与文件名匹配的过滤器模式。 模式字符串可以是一个简单的 "glob" 模式,也可以是由一个 ; 字符分隔的若干 glob 模式。 这些模式不是正则表达式,只能与 * 通配符一起使用。例如, "*.jp*g;*.png" 。 在 Unix 上, "*.*" 模式被隐含地替换为 "*" 。 在 Mac OS 上,后缀名从所有与固定后缀相匹配的 glob 中提取(例如, "foo" 和 "bar" 的两个后缀从 "*.foo;*.bar;*.baz*" 模式中提取),在任何过滤器中具有这些后缀的文件都可以被选择; "*.*" glob 使所有文件都可以被选择。

dialog-mixin 在为这个对话框创建一个类的实例之前被应用于 path-dialog% 。

参见 path-dialog% 以获得更丰富的接口。

通过特定平台的标准(模式)对话框从用户那里获得文件路径名,如果指定了 parent 作为父窗口, 则使用 message 作为对话框顶部的信息,如果不是 #f 。

如果用户取消对话框,结果是 #f ,否则是所选路径名。 返回的路径名可能存在,也可能不存在,尽管对话框的风格是指向创建一个新文件。

如果 directory 不是 #f, 它将被用作文件选择器的起始目录(否则起始目录将以特定平台的方式自动选择, 通常基于当前目录和用户在先前调用 get-file, put-file 等时的交互情况)。 如果 filename 不是 #f, 在适当的时候会被用作默认的文件名, 它 不 应该包含一个目录路径前缀。

在 Windows 上,如果 extension 不是 #f,如果用户没有提供扩展名,返回的路径将得到一个默认的扩展名。 如果相应的模式为 (string-append "*." an-extension) 形式, 则扩展名来自用户的 filters 选择,如果选择有多个模式,则使用第一个此类模式。 如果用户的选择有 "*.*" 模式,并且 extension 是空字符串,那么就不会添加默认的扩展名。 最后,如果 extension 是空字符串以外的任何字符串, 当用户的 filters 选择模式为 "*.*" 时, extension 被用作默认扩展名。 同时, filters 参数的格式和辅助作用与 get-file 相同。 特别是,如果 filters 中唯一的模式是 (string-append "*." extension), 那么结果路径名就保证有一个扩展名映射 extension 。

在 Mac OS 10.5 及以后的版本中,如果 extension 不是 #f 或 "", 如果用户没有提供扩展名,返回的路径将得到一个默认的扩展名。 如果 filters 包含 "*.*" 模式,那么用户可以提供系统认可的任何扩展; 否则,返回路径上的扩展将是 filters 中任何 (string-append "*." other-extension) 模式的 extension 或 other-extension 。 特别是,如果 filters 中唯一的模式是空的或者只包含 (string-append "*." extension), 那么结果路径名保证有一个 extension 映射的扩展。

在 10.5 之前的 Mac OS 版本上,只有当 extension 不是 #f, extension 不是 "", 并且 filters 只包含 (string-append "*." extension) 时, 返回的路径才会得到一个默认的扩展名。

在 Unix 上, extension 被忽略, filters 用于过滤可见的文件列表,如同 get-file 。

style 列表与 get-file 的处理方式相同。

也见 message-box/custom 。

在一个(模态)对话框中向用户显示一条信息,如果指定了父窗口,就使用 parent 。 对话框的标题是 title 。 message 字符串可以是任意长的,并且可以包含明确的换行符或回车符用于断行。

样式必须正好包括以下内容之一:

  • 'ok — 对话框只有一个 OK 按钮,并且总是返回 'ok 。

  • 'ok-cancel — 消息对话框有 Cancel 和 OK 按钮。 如果用户点击了 Cancel, 结果是 'cancel, 否则结果是 'ok 。

  • 'yes-no — 消息对话框有 Yes 和 No 按钮。 如果用户点击 Yes,结果是 'yes, 否则结果是 'no 。 注意:代替 Yes/No 对话框, 最佳实践的 GUI 设计是使用 message-box/custom ,并给按钮以有意义的标签, 这样用户就不必仔细阅读消息文本来做出选择。

此外, style 可以包含 'caution 来使对话框使用警告图标, 而不是应用程序(或通用的 "信息")图标, 'stop 来使对话框使用停止图标, 或者 'no-icon 来抑制图标。 如果 style 中包含多个 'caution, 'stop 和 'no-icon, 那么 'no-icon 优先,然后是 'stop 。

实现对话框的类提供了一个 get-message 方法,该方法不需要任何参数,并以字符串的形式返回消息的文本。 (该对话框可以通过 get-top-level-windows 函数访问)。

message-box 框函数可以在相关事件空间的处理程序线程以外的线程中调用 (即, parent 的事件空间,或者如果 parent 是 #f 的话,则是当前的事件空间), 在这种情况下,当前线程会阻塞,而对话框会在处理程序线程中运行。

在一个(模态)对话框中向用户显示一条信息,如果指定了 parent,则使用 parent 作为父窗口。 对话框的标题是 title 。 message 字符串可以是任意长的,并且可以包含明确的换行符或回车符用于断行。

该对话框最多包含三个按钮供用户点击。 这些按钮的标签是 button1-label, button2-label 和 button3-label, 其中标签的 #f 表示该按钮应被隐藏。

如果用户点击标签为 button1-label 的按钮,就会返回 1 , 以此类推, 2 和 3 也是如此。 如果用户以其他方式关闭对话框–这只有在 style 不包含 'disallow-close 时才允许, 那么结果就是 close-result 的值。 例如,用户通常可以通过输入 Escape 来关闭一个对话框。 通常, 2 是 close-result 的一个合适的值, 特别是当 Button 2 是一个 Cancel 按钮时。

如果 style 不包括 'number-order, 按钮的顺序是平台特定的,应该根据按钮的作用给它们分配标签:

  • 按钮 1 是正常动作,它通常是默认按钮。例如,如果对话框有一个 OK 按钮,就是这个按钮。 在 Windows 上,这个按钮在最左边;在 Unix 和 Mac OS 上,它在最右边。 (参见 system-position-ok-before-cancel?) 。 对于只包含一个按钮的对话框,使用这个按钮。

  • 按钮 2 紧挨着按钮1,它经常扮演 Cancel 的角色(即使默认动作是取消,如确认文件替换时)。

  • 按钮 3 往往与其他两个分开(在 Mac OS 上,它在对话框中是左对齐的)。 只在三按钮的对话框中使用这个按钮。

尽管有上述准则,在对话框中允许任何可见按钮的组合。

如果 style 包括 'number-order,那么按钮就会在对话框中从左到右显示, 所有按钮之间的间距相等,尽管在对话框中以特定平台的方式对齐(居中或右对齐)。 尽量少用 'number-order 。

style 列表必须包含 'default=1, 'default=2, 'default=3 和 'no-default 中的一个, 以确定哪个按钮(如果有的话)是默认的。 当用户输入 Return 时,默认按钮被 "点击"。 如果提供了 'default=n 但按钮 n 没有标签,那么它就等同于 'no-default 。

此外, style 可以包含 'caution, 'stop 或 'no-icon 来调整出现在对话框中的图标,对 message-box 也是如此。

如果 return-the-dialog? 是一个真值,那么就不显示对话框,而是从 message-box/custom 返回。 对话框会对这三个额外的消息(通过 send) 做出反应:

  • get-message 这个方法不接受任何参数,以字符串形式返回消息的文本。

  • set-message 这个方法接受一个字符串参数,并将对话框的消息改为给定参数。

  • show-and-return-results 此方法不接受任何参数,显示对话框。 它在对话框关闭后返回,如果 return-the-dialog? 是 #false 的话, message-box/custom 会返回这个结果。

该对话框也可以通过 get-top-level-windows 函数访问。

message-box/custom 函数可以在相关事件空间的处理程序线程以外的线程中被调用(即 parent 的事件空间,如果 parent 是 #f ,则为当前的事件空间), 在这种情况下,当前线程被阻塞,而对话框在处理程序线程中运行。

(yield) → boolean?

(yield v) → any/c
  v : (or/c 'wait evt?)
产生对事件调度的控制。详见 事件调度和事件空间 。

系统在调用 yield 时调用的处理程序本身可以调用 yield, 从而创建一个额外的嵌套的(但单线程的)事件处理层次。

参见 sleep/yield 。

如果没有提供参数, yield 会派发未指定数量的事件, 但只有当当前线程是当前事件空间的处理程序线程时(否则,没有影响)。 如果有事件被处理,结果是 #t ,否则是 #f 。

如果 v 是 'wait, 并且 yield 在事件空间的处理线程中被调用, 那么 yield 开始处理该事件空间中的事件,直到

  • 事件空间中没有顶层窗口是可见的;

  • 事件空间中没有定时器在运行;

  • 在事件空间中没有排队的回调;并且

  • 没有为带有 'root 的事件空间创建 menu-bar% (也就是说,创建 'root 菜单栏可以防止事件空间解除阻塞)。

当在一个非处理程序线程中调用时, yield 立即返回。在这两种情况下,结果都是 #t 。

因此,评估 (yield 'wait) 与 (yield (current-eventspace)) 类似, 只是它对当前线程是否是处理程序线程敏感,而不是对 current-eventspace 的参数值敏感。

如果 v 是 Racket 意义上的事件(不要与 GUI 事件混淆), yield 在 v 上的阻塞方式与 sync 相同, 只是它可以在 v 上多次启动 sync (但它最多只能在 v 上完成一次 sync )。 如果当前线程是当前事件空间的处理程序线程,事件会被派发,直到事件边界上的 v 同步成功。 对于其他线程,用 Racket 事件调用 yield 等同于调用 sync 。 在这两种情况下,结果都与 sync 相同; 但是,如果一个包装程序通过 handle-evt 与 v 相关联, 它就不会在 yield 的尾部位置被调用。

总是使用 (yield v) 而不是忙等待循环。

on 后面的背景可能是也可能不是 canvas 的通常内容,所以 on 应该是一个实体图像。 在绘制位图时,画布的比例和滚动位置都不被应用。 只有 on 在 w 和 h 像素内的部分被使用; 如果 on-x 和 on-y 被指定,它们指定了位图内的偏移量,用于绘制; 类似的 off-x 和 off-y 指定了 off 内的偏移量。

如果 canvas 变得不可见和不可访问,blit 会自动取消注册。可以为同一个 canvas 安装多个注册。

参见 unregister-collecting-blit 。
