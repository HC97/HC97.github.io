7.3 包路径和数据库

 (require pkg/path) package: base
pkg/path 库提供了用于处理软件包路径和已安装软件包数据库的实用程序。

struct

(struct pkg-info (orig-pkg checksum auto?)
    #:prefab)
  orig-pkg : 
(or/c (list/c 'catalog string?)
      (list/c 'catalog string? string?)
      (list/c 'url string?)
      (list/c 'git string?)
      (list/c 'file string?)
      (list/c 'dir string?)
      (list/c 'link string?)
      (list/c 'static-link string?)
      (list/c 'clone string? string?))
  checksum : (or/c #f string?)
  auto? : boolean?
一个用于报告安装包信息的结构类型。

orig-pkg 字段描述了所安装软件包的来源,其中 'catalog 指的是通过查阅软件包名称的目录而安装的软件包,以此类推。 双元素的 'catalog 表格记录了 Git 或 GitHub 软件包来源的 URL,当目录报告了这样的来源时, 该 URL 用于调整 'clone-form 安装的操作。

'git 形式用于以 git+https:// 或 git+http:// 开头的 URL, 或者在解析 URL 时指定了 'git-url 类型。 其他的 Git 引用(包括以 git:// 开头的)则使用 'url 。

Changed in version 6.1.1.5 of package base: 为 orig-pkg 增加了 'clone 和两级 'catalog 变体。
Changed in version 8.0.0.13: 添加了 'git 。

struct

(struct sc-pkg-info pkg-info ())

一个结构子类型,表示以单集形式安装的软件包。

struct

(struct pkg-info/alt pkg-info (dir-name))

  dir-name : string?

struct

(struct sc-pkg-info/alt sc-pkg-info (dir-name))

  dir-name : string?
结构子类型,当软件包的安装目录与软件包名称不一致,而是 dir-name 时使用。 目录名总是包括一个 + (这在软件包名称中是不允许的)。

procedure

(path->pkg path #:cache cache) → (or/c string? #f)

  path : path-string?
  cache : (or/c #f (and/c hash? (not/c immutable?)))
如果有的话,返回包含 path 的已安装软件包。

如果 cache 不是 #f, 那么它将被查阅并修改, 以便在调用 path->pkg 的过程中缓存已安装软件包的信息 (假设在收到相同 cache 的调用中,已安装软件包的集合不会改变)。

procedure

(path->pkg+subpath path #:cache cache)

 → 
(or/c string? #f) (or/c path? 'same #f)
  path : path-string?
  cache : (or/c #f (and/c hash? (not/c immutable?)))
和 path->pkg 一样,但返回第二个值,代表软件包目录中 path 的剩余部分。

procedure

(path->pkg+subpath+scope path 
  #:cache cache) 
 → 
(or/c string? #f)
(or/c path? 'same #f)
(or/c 'installation 'user (and/c path? complete-path?) #f)
  path : path-string?
  cache : (or/c #f (and/c hash? (not/c immutable?)))
和 path->pkg+subpath 一样,但返回代表软件包安装范围的第三个值。

procedure

(path->pkg+subpath+collect path 
  #:cache cache) 
 → 
(or/c string? #f)
(or/c path? 'same #f)
(or/c string? #f)
  path : path-string?
  cache : (or/c #f (and/c hash? (not/c immutable?)))
和 path->pkg+subpath 一样,但是如果软件包是一个单一的集合, 则返回集合名称的第三个值,否则返回 #f 。

procedure

(path->pkg+subpath+collect+scope path 
  #:cache cache) 
 → 
(or/c string? #f)
(or/c path? 'same #f)
(or/c string? #f)
(or/c 'installation 'user (and/c path? complete-path?) #f)
  path : path-string?
  cache : (or/c #f (and/c hash? (not/c immutable?)))
与 path->pkg+subpath+collect 类似,但返回软件包安装范围的第四个值。

procedure

(get-pkgs-dir scope [user-version]) → path?

  scope : 
(or/c 'installation 'user
       (and/c path? complete-path?))
  user-version : string? = (version)
返回持有指定范围内已安装软件包的目录的路径。 user-version 参数用于生成 'user 范围的结果。

procedure

(read-pkgs-db scope) → (hash/c string? pkg-info?)

  scope : 
(or/c 'installation 'user
       (and/c path? complete-path?))
返回一个代表当前在指定范围内安装的软件包的哈希表。

procedure

(read-pkg-file-hash path) → hash?

  path : path?
从 path 中读取一个哈希表,记录任何错误,如果 path 不存在或遇到错误,则返回一个空的哈希表。
