7.5 包目录数据库

 (require pkg/db) package: base
pkg/db 库提供了在本地数据库中存储和检索 package catalog 信息的工具。

pkg/db 提供的函数实际上并不管理软件包; 它们不会改变或查询任何 package scope 中已安装模块的本地数据库。 pkg/db 提供的功能只是反映了一个 package catalog 和单个包可能提供的信息的本地拷贝 (但不能保证与实际的 package catalog 或包同步)。

该数据库被实现为一个有自己锁的 SQLite 数据库,因此在访问数据库时不需要额外的锁,但要注意数据库的并发变化可能会破坏你的程序逻辑。

struct

(struct pkg (name catalog author source checksum desc)
    #:extra-constructor-name make-pkg
    #:transparent)
  name : string?
  catalog : string?
  author : string?
  source : string?
  checksum : string?
  desc : string?
代表数据库中的一个包的实现。 name (package name) 和 catalog (package catalog, 通常是一个 URL)字段总是非空字符串。否则,未知字段由空字符串表示。

parameter

(current-pkg-catalog-file) → path-string?

(current-pkg-catalog-file file) → void?
  file : path-string?
一个参数,决定了用于保存 SQLite 数据库的文件路径。 默认值是在用户的 add-on 目录下,由 (find-system-path 'addon-dir) 决定, 并在 get-installation-name 决定的子目录下。

procedure

(call-with-pkgs-transaction proc) → any

  proc : (-> any)
调用 proc ,以便将对其他 pkg/db 函数的多次调用归为一个数据库事务, 这就避免了使每个单独调用成为自己的事务的开销。

Added in version 6.1.1.5 of package base.

procedure

(get-catalogs) → (listof string?)

procedure

(set-catalogs! catalogs) → void?

  catalogs : (listof string?)
返回或设置数据库中所有 package catalog 的字符串列表。 (在数据库中,每个 package catalog 都有自己的识别号)。 列表中的索引顺序代表了一个搜索顺序。

set-catalogs! 函数会从数据库中删除任何其他 package catalogs 的信息。

procedure

(get-pkgs [#:name name #:catalog catalog]) → (listof pkg?)

  name : (or/c #f string?) = #f
  catalog : (or/c #f string?) = #f
获取一个软件包描述的列表。 如果 name 或 catalog 不是 #f (或者两者都不是 #f), 那么结果只包括匹配的包。

结果列表是按照 package catalog 的优先级排序的。

procedure

(set-pkgs! catalog 
  pkgs 
  #:clear-other-checksums? clear-other-checksums?) 
 → void?
  catalog : string?
  pkgs : (listof (or/c string? pkg?))
  clear-other-checksums? : #t
设置所有被 package catalog catalog 认可的软件包的列表。

关于 catalog 的任何其他软件包的信息将从数据库中删除。 如果为 pkgs 提供了一个字符串,它将被视为包名;如果数据库中已经记录了该包名的额外信息,那么额外信息将被保留。

如果 clear-other-checksums? 为 true,那么对于 pkgs 中的每个元素, 如果其校验和不是 "", 那么数据库中的任何特定于另一个校验和的信息(例如模块路径列表)都将从数据库中删除。

procedure

(set-pkg! name 
  catalog 
  author 
  source 
  checksum 
  desc 
  #:clear-other-checksums? clear-other-checksums?) 
 → void?
  name : string?
  catalog : string?
  author : string?
  source : string?
  checksum : string?
  desc : string?
  clear-other-checksums? : (not (equal? checksum ""))
设置 package catalog catalog 所识别的特定包 name 的信息。

如果 clear-other-checksums? 为真, 那么当信息(如模块路径列表)具体到 checksum 以外的校验时,就会从数据库中删除。

(get-pkg-tags name catalog) → (listof string?)

  name : string?
  catalog : string?

procedure

(set-pkg-tags! name catalog module-paths) → void?

  name : string?
  catalog : string?
  module-paths : (listof string?)
获取或设置由 package catalog catalog 识别的包 name 的标记列表。

procedure

(get-pkg-ring name catalog)

 → (or/c #f exact-nonnegative-integer?)
  name : string?
  catalog : string?

procedure

(set-pkg-ring! name catalog ring) → void?

  name : string?
  catalog : string?
  ring : (or/c #f exact-nonnegative-integer?)
获取或设置 name 所识别的包 package catalog catalog 的环号。

PLT 支持的软件包目录报告了一个策划的环号,以反映关于软件包偏好和冲突的建议, 其中 ring-0 和 ring-1 的软件包集合被期望没有冲突(即没有多重定义的模块、文件名等)。 raco pkg 工具并不关注软件包的环号,但目录的其他用途可能会参考环号。

Added in version 6.10.0.3 of package base.

procedure

(get-pkg-dependencies name catalog checksum) → (listof list?)

  name : string?
  catalog : string?
  checksum : string?

procedure

(set-pkg-dependencies! name    
  catalog    
  checksum    
  dependencies) → void?
  name : string?
  catalog : string?
  checksum : string?
  dependencies : (listof any/c)
获取或设置一个由 package catalog catalog 识别的软件包 name 的依赖关系列表,以及特定的 checksum 。

依赖列表必须具有 包的元数据 中描述的 deps "info.rkt" 字段的形状。 get-pkg-dependencies 的结果是规范化的:每个依赖关系用一个列表表示, 依赖关系中的版本总是在 '#:version 之前,如果同时包含版本和平台规范, '#:version 出现在 '#:platform 之前。

procedure

(get-pkg-modules name catalog checksum) → (listof module-path?)

  name : string?
  catalog : string?
  checksum : string?

procedure

(set-pkg-modules! name    
  catalog    
  checksum    
  module-paths) → void?
  name : string?
  catalog : string?
  checksum : string?
  module-paths : (listof module-path?)
获取或设置一个模块路径列表,该列表为 package catalog catalog 所识别的包 name 和特定的 checksum 所提供。 模块路径应该在 collapse-module-path 的意义上被规范化。

procedure

(get-module-pkgs module-path) → (listof pkg?)

  module-path : module-path?
报告一个实现给定 module-path 的包的列表,这些包应该在 collapse-module-path 的意义上被规范化。

procedure

(get-pkgs-without-modules [#:catalog catalog]) → (listof pkg?)

  catalog : (or/c #f string?) = #f
返回一个数据库中没有模块记录的软件包的列表(可选择约束为 catalog)。

每个返回的 pkg 都有它的 name, catalog 和 checksum ,但其他字段可能是 "" 。
