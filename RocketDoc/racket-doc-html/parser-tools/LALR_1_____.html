2 LALR(1) 解析器

 (require parser-tools/yacc) package: parser-tools-lib

syntax

(parser clause ...)

 
clause = 
(grammar (non-terminal-id
          ((grammar-id ...) maybe-prec expr)
          ...)
         ...)
  | (tokens group-id ...)
  | (start non-terminal-id ...)
  | (end token-id ...)
  | (error expr)
  | (precs (assoc token-id ...) ...)
  | (src-pos)
  | (suppress)
  | (expected-SR-conflicts num)
  | (expected-RR-conflicts num)
  | (debug filename)
  | (yacc-output filename)
     
maybe-prec = 
  | (prec token-id)
     
assoc = left
  | right
  | nonassoc
创建一个解析器。分句可以是任何顺序,只要没有重复的,并且所有非 OPTIONAL 的声明都存在:

  • (grammar (non-terminal-id
              ((grammar-id ...) maybe-prec expr)
              ...)
             ...)

    声明要解析的语法。每个 grammar-id 可以是 tokens 声明中命名的 group-id 中的 token-id , 也可以是 grammar 声明中声明的 non-terminal-id 。 可选的 prec 声明与 precs 声明一起工作。 expr 是一个 "语义动作",当发现输入与它相应的生产相匹配时,它就会被评估。

    每个动作都是 Racket 代码,其范围与它的解析器定义相同,只是变量 $1, ..., $i 被绑定,其中 i 是相应生产中的 grammar-id 的数量。 如果生产右边的第 k 个语法符号是一个非终端, $k 被绑定到其动作的结果; 如果它是一个终端, $k 被绑定到存储在标记中的值。 如果解析器中存在 src-pos 选项,那么变量 $1-start-pos, ..., $i-start-pos 和 $1-end-pos, ..., $i-end-pos 和也是可用的, 它们指的是对应 grammar-symbol 的开始和结束的位置结构。 定义为空符号的语法符号没有 $k 关联, 但有 $k-start-pos 和 $k-end-pos 。 同样, $n-start-pos 和 $n-end-pos 被绑定到最大的开始和结束位置, (即 $i-start-pos 和 $i-end-pos) 。

    一个 error production 可以通过提供一个形式为 (error α) 的生产来定义, 其中α是一串语法符号,可能为空。

    一个给定的非终端的所有生产必须与它分组。 也就是说,任何 non-terminal-id 都不能在解析器的左手边出现两次。

  • (tokens group-id ...)

    声明每个 group-id 中定义的所有令牌–如 define-tokens 或 define-empty-tokens—可以被 grammar 声明中的解析器使用。

  • (start non-terminal-id ...)

    声明语法的起始非端点列表。

  • (end token-id ...)

    指定一组令牌,任何有效的解析后都必须有其中的某个成员。 例如,对于解析整个文件的解析器将指定一个 EOF 标记,而对于单独解析整个行的解析器将指定一个换行标记。

  • (error expr)

    expr 应该求值为一个函数,每当解析器遇到错误时,该函数将被执行其副作用。

    如果 src-pos 声明存在,该函数应该接受 5 个参数:

    (lambda (tok-ok? tok-name tok-value start-pos end-pos)
      ....)

    否则,它应该接受3个参数:

    (lambda (tok-ok? tok-name tok-value)
      ....)

    第一个参数将是 #f ,如果且仅当错误是收到了一个无效的标记。 第二个和第三个参数将是检测到错误的标记的名称和值。 第四和第五个参数,如果存在的话,提供该标记的源位置。

    在这两种情况下,该函数被允许接受一个额外的关键字参数,名为 #:stack。 这个参数是解析自动机堆栈的代表。 例如,这可以用来生成上下文敏感的错误信息, 如 Clinton L. Jeffrey 的 Generating LR syntax error messages from examples, by Clinton L. Jeffrey 中所述。

  • (precs (assoc token-id ...) ...) OPTIONAL

    顺序声明,以解决 yacc/bison 中的 shift/reduce 和 reduce/reduce 冲突。 一个 assoc 必须是 left, right 或 nonassoc 中的一个。 有多个 shift/reduce 或 reduce/reduce 冲突的状态(或它们的一些组合)不会用优先级来解决。

  • (src-pos) OPTIONAL

    导致生成的解析器期望以 (make-position-token token start-pos end-pos) 的形式而不是简单的 token 输入。 当将解析器与用 lexer-src-pos 生成的词法器一起使用时,请包含这个选项。

  • (debug filename) OPTIONAL

    导致解析器生成器将LALR表写入名为 filename 的文件中(除非该文件存在), 其中 filename 是一个字面字符串。 此外,如果指定了一个调试文件,当运行中的生成器在某个输入文件上遇到解析错误时, 在用户指定的错误表达式返回后,完整的解析堆栈会被打印出来,以帮助调试该特定解析器的语法。 堆栈中的数字对应于 LALR 表文件中的状态数字。

  • (yacc-output filename) OPTIONAL

    导致解析器生成器以近似 yacc/bison 的语法写一个语法文件。 该文件可能不是一个有效的 yacc 文件,因为 Racket 语法可以使用 C 语言中无效的符号。

  • (suppress) OPTIONAL

    导致解析器生成器不报告 shift/reduce 或 reduce/reduce 冲突。

  • (expected-SR-conflicts n) OPTIONAL

    导致解析器生成器预期正好有 num shift/reduce 冲突,其中 num 必须是一个字面数字。 suppress 选项会覆盖这个选项。

  • (expected-RR-conflicts n) OPTIONAL

    导致解析器生成器预期正好有 num reduce/reduce 冲突, 其中 num 必须是一个字面数字。 suppress 选项会覆盖这个选项。

有一个 start 非终端的 parser 表达式的结果是一个函数, parse, 它需要一个参数。 这个参数必须是一个零参数的函数, gen, 每次调用它都会产生连续的输入标记。 如果需要, gen 可以返回符号而不是令牌, 解析器将把符号视为相应名称的令牌(以 #f 为值,所以通常只在空令牌的情况下返回符号)。 parse 函数返回由语义动作与解析树相关的值。 如果解析器遇到了错误,在调用所提供的错误函数之后, 它将尝试使用 error production 来继续解析。如果不能,它会引发 exn:fail:read 。

如果在 start 中提供了多个非终端, parser 表达式会产生一个解析函数的列表,每个非终端有一个相同的顺序。 每个解析函数就像一个只有一个 start 非终端的解析器表达式的结果。

每次编译 parser 的 Racket 代码 (例如,当一个包含 parser 形式的 ".rkt" 文件被加载时),解析器生成器就会被运行。 为了避免这种开销,将分析器放入一个模块,并将该模块编译为 ".zo" 字节码文件。

Examples:
; Use the lexer and tokens from Tokens
; and the sample input from (part "Creating_a_Lexer")
> (define the-parser
    (parser
      [start expr]
      [end EOF]
      [error void]
      [tokens basic-tokens punct-tokens]
      [grammar
       [expr [(LPAREN exprs RPAREN) $2]
             [(NUM) $1]
             [(ID) $1]]
       [exprs [() '()]
              [(expr exprs) (cons $1 $2)]]]))
> (define p (open-input-string sample-input))
> (the-parser (λ () (the-lexer/tokens p)))
