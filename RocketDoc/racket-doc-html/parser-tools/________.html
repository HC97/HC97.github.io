3 上下文自由分析器

 (require parser-tools/cfg-parser)
  package: parser-tools-lib
parser-tools/cfg-parser 库提供了一个分析器生成器,它是 parser-tools/yacc 的替代品。

syntax

(cfg-parser clause ...)

 
clause = 
(grammar (non-terminal-id
          ((grammar-id ...) maybe-prec expr)
          ...)
         ...)
  | (tokens group-id ...)
  | (start non-terminal-id ...)
  | (end token-id ...)
  | (error expr)
  | (src-pos)
创建一个类似于 parser 的解析器。 与 parser不同的是, cfg-parser 可以消费任意的和潜在的模糊的上下文自由文法。 它的接口是 parser-tools/yacc 的一个子集,但有以下区别:

  • (start non-terminal-id)

    与 parser 不同, cfg-parser 只允许一个 non-terminal-id 。

  • cfg-parser 形式不支持 parser 的 precs, suppress, expected-SR-conflicts, expected-RR-conflicts, debug 或 yacc-output 选项。
