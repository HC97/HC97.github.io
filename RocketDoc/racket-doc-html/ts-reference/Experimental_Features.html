15 Experimental Features

这些特性目前是试验性的,可能会发生变化。

syntax

(declare-refinement id)

声明 id 可用于 Refinement 类型。

syntax

(Refinement id)

包括用 id 谓词测试过的值,id 谓词必须是用 declare-refinement 指定的。 这些基于谓词的精炼不同于 Typed Racket 更通用的 Refine 形式。

syntax

(define-typed-struct/exec forms ...)

Defines an executable structure.

syntax

(define-new-subtype name (constructor t))

定义一个新的类型 name ,它是 t 的子类型。 constructor 被定义为一个函数,它接收一个 t 类型的值,并产生一个新类型 name 的值。 define-new-subtype 定义只允许在文件或模块的顶层使用。

这纯粹是类型级的区别,在运行时无法区分新类型和基本类型。 由 make-predicate 创建的谓词将无法正确区分它们,因此它们将对基础类型的谓词返回 true 的所有值返回 true。 这通常不是我们想要的,所以不应该在这些类型中使用 make-predicate 。

Examples:
> (module m typed/racket
    (provide Radians radians f)
    (define-new-subtype Radians (radians Real))
    (: f : [Radians -> Real])
    (define (f a)
      (sin a)))
> (require 'm)
> (radians 0)

- : Real [more precisely: Radians]

0

> (f (radians 0))

- : Real

0

15.1 逻辑精炼和线性整数推理

Racket 类型允许通过逻辑命题对类型进行 "细化" 或 "约束"。 这些命题可以提到某些程序术语,允许程序的类型依赖于术语的值。

syntax

(Refine [id : type] proposition)

 
proposition = Top
  | Bot
  | (: symbolic-object type)
  | (! symbolic-object type)
  | (and proposition ...)
  | (or proposition ...)
  | (when proposition proposition)
  | (unless proposition proposition)
  | (if proposition proposition proposition)
  | (linear-comp symbolic-object symbolic-object)
     
linear-comp = <
  | <=
  | =
  | >=
  | >
     
symbolic-object = exact-integer
  | symbolic-path
  | (+ symbolic-object ...)
  | (- symbolic-object ...)
  | (* exact-integer symbolic-object)
     
symbolic-path = id
  | (path-elem symbolic-path)
     
path-elem = car
  | cdr
  | vector-length
(Refine [v : t] p) 是用逻辑命题 p 对类型 t 的细化,或者换句话说, 它描述了逻辑命题 p 对其成立的类型 t 的任何值 v 。

Example:
> (ann 42 (Refine [n : Integer] (= n 42)))

- : Integer [more precisely: (Refine (x₀ : Integer) (= 42 x₀))]

42

注意:细化类型中的标识符在命题内部是有范围的,但在类型中没有。

(: o t) 作为一个命题,当符号对象 o 是 t 类型时成立。

syntax

(! sym-obj type)

这是 (: o t) 的对偶,当 o 不属于 t 类型时保持。

命题也可以描述线性不等式(例如,当 x 小于或等于 42 时, (<= x 42) 成立), 使用以下任何一种关系: <=, <, =, >=, > 。

以下逻辑组合器根据其子成分的成立情况而成立: and, or, if, not 。

(when p q) 等同于 (or (not p) (and p q)) 。

(unless p q) 等同于 (or p q) 。

除了推理有关类型的命题(即某物是否属于某种特定类型), Typed Racket 还配备了一个线性整数算术求解器,必要时可以证明线性约束。 要打开这个求解器(以及其他一些细化推理),你必须在指定程序的语言时加入 #:with-refinements 关键字:

#lang typed/racket #:with-refinements

打开这个语言选项后,对以下基元的类型检查将产生更具体的逻辑信息(当它们被应用于 2 或 3 个参数时): *, +, -, <, <=, =, >=, > 和 make-vector 。

这使得像下面这样的代码可以进行类型检查:

(if (< 5 4)
    (+ "Luke," "I am your father")
    "that's impossible!")

即在启用细化推理的情况下,Typed Racket 检测到比较保证会产生 #f , 因此类型检查器忽略了明显类型错误的 ’then’ 分支,因为它被保证为死代码。

15.2 依赖函数类型

类型化的 Racket 支持明确的依赖函数类型:

syntax

(-> ([id : opt-deps arg-type] ...)
    opt-pre
    range-type
    opt-props)
 
opt-deps = 
  | (id ...)
     
opt-pre = 
  | #:pre (id ...) prop
     
opt-props = 
  | opt-pos-prop opt-neg-prop opt-obj
     
opt-pos-prop = 
  | #:+ prop
     
opt-neg-prop = 
  | #:- prop
     
opt-obj = 
  | #:object obj

其语法类似于 Racket 的依赖合约语法(即 ->i) 。

每个函数参数都有一个名字,一个它所依赖的标识符的可选列表,一个参数类型。 一个参数的类型可以通过名字提及(即依赖)其他参数,如果这些参数出现在它的依赖列表中。依赖关系不能是循环的。

一个函数也可以有一个前提条件。 使用 #:pre 关键字引入先决条件,然后是它所依赖的参数列表和描述先决条件的命题。

函数的范围可以取决于它的任何参数。

支持的命题和符号对象(即 prop 和 obj) 语法与 Refine 语法中的 proposition 和 symbolic-object 语法相同。

例如,这里是 Racket 的 vector-ref 的依赖类型版本,它在类型检查时而不是运行时消除了 vector 边界错误:

> (require racket/unsafe/ops)
> (: safe-ref1 (All (A) (-> ([v : (Vectorof A)]
                             [n : (v) (Refine [i : Natural]
                                              (< i (vector-length v)))])
                            A)))
> (define (safe-ref1 v n) (unsafe-vector-ref v n))
> (safe-ref1 (vector "safe!") 0)

- : String

"safe!"

> (safe-ref1 (vector "not safe!") 1)

eval:10:0: Type Checker: Polymorphic function `safe-ref1'

could not be applied to arguments:

Argument x₀ (position 1):

  Expected: (Vectorof A)

  Given:    (Mutable-Vector String)

Argument y₀ (position 2):

  Expected: (Refine (z₀ : Nonnegative-Integer) (< z₀

(vector-length x₀)))

  Given:    (Refine (z₀ : One) (= 1 z₀))

  in: 1

这里是一个使用前置条件而不是细化类型的等价类型:

> (: safe-ref2 (All (A) (-> ([v : (Vectorof A)]
                             [n : Natural])
                            #:pre (v n) (< n (vector-length v))
                            A)))
> (define (safe-ref2 v n) (unsafe-vector-ref v n))
> (safe-ref2 (vector "safe!") 0)

- : String

"safe!"

> (safe-ref2 (vector "not safe!") 1)

eval:14:0: Type Checker: could not apply function;

 unable to prove

  precondition: (<= 2 (vector-length a))

  in: 1

使用先决条件可以提供更详细的类型检查错误信息,例如,它们可以指出参数类型正确,但先决条件无法证明。
