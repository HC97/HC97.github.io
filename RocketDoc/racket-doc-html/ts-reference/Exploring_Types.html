7 Exploring Types

除了打印 REPL 结果的类型摘要外,Typed Racket 还提供互动的实用程序来探索和查询类型。 以下绑定仅在 Typed Racket REPL 中可用。

syntax

(:type maybe-verbose t)

 
maybe-verbose = 
  | #:verbose
打印类型 t 。如果 t 是一个类型别名(例如 Number), 那么在打印时它将被扩展为其表示形式。 由 t 命名的类型中的任何其他类型别名将保持不被扩展。

如果提供了 #:verbose ,所有类型别名都会在打印的类型中展开。

Examples:
> (:type Number)

(U Exact-Number Imaginary Inexact-Complex Real)

[can expand further: Exact-Number Inexact-Complex Imaginary Real]

> (:type Real)

(U Negative-Real Nonnegative-Real)

[can expand further: Negative-Real Nonnegative-Real]

> (:type #:verbose Number)

(U 0

   1

   Byte-Larger-Than-One

   Exact-Complex

   Exact-Imaginary

   Float-Complex

   Float-Imaginary

   Float-Nan

   Float-Negative-Zero

   Float-Positive-Zero

   Negative-Fixnum

   Negative-Float-No-NaN

   Negative-Integer-Not-Fixnum

   Negative-Rational-Not-Integer

   Negative-Single-Flonum-No-Nan

   Positive-Fixnum-Not-Index

   Positive-Float-No-NaN

   Positive-Index-Not-Byte

   Positive-Integer-Not-Fixnum

   Positive-Rational-Not-Integer

   Positive-Single-Flonum-No-Nan

   Single-Flonum-Complex

   Single-Flonum-Imaginary

   Single-Flonum-Nan

   Single-Flonum-Negative-Zero

   Single-Flonum-Positive-Zero)

syntax

(:print-type e)

打印 e 的类型,它必须是一个表达式。 这将打印整个类型,有时它可能相当大。

Examples:
> (:print-type (+ 1 2))

Positive-Index

> (:print-type map)

(All (c a b ...)

  (case->

   (-> (-> a c) (Pairof a (Listof a)) (Pairof c (Listof c)))

   (-> (-> a b ... b c) (Listof a) (Listof b) ... b (Listof c))))

syntax

(:query-type/args f t ...)

给定一个函数 f 和参数类型 t, 显示 f 的结果类型。

Example:
> (:query-type/args + Integer Number)

(-> Integer Number Number)

syntax

(:query-type/result f t)

给定一个函数 f 和一个期望的返回类型 t, 显示为了返回一个类型 f 的值,应该给 t 的参数类型。

Examples:
> (:query-type/result + Integer)

(-> Integer * Integer)

> (:query-type/result + Float)

(case->

 (-> Flonum Flonum * Flonum)

 (-> Real Real Flonum Real * Flonum)

 (-> Real Flonum Real * Flonum)

 (-> Flonum Real Real * Flonum))

syntax

(:kind e)

打印良好类型的类型级表达式 e 的类型。 当 e 是一个类型时,它打印出 * 。 当 e 是一个类型构造函数时, -> 在打印结果中的开放括号后面表示 e 是生产性的, -o 表示其他。

Examples:
> (:kind Integer)

*

> (:kind Listof)

(-> * *)

> (:kind Pairof)

(-> * * *)

> (:kind U)

(-o * ... *)

Added in version 1.15 of package typed-racket-lib.
