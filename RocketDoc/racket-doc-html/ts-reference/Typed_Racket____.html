11 Typed Racket 的优化

查看 Typed Racket 中的优化 指南中关于如何充分利用优化器的提示。

Typed Racket 提供了一个类型驱动的优化器,它可以重写类型良好的程序,从而有可能使其更快。

Typed Racket 檚优化器是默认开启的。 如果你想停用它(例如,为了调试),你必须在指定程序的语言时加入 #:no-optimize 关键字:

#lang typed/racket #:no-optimize

如果环境变量 PLT_TR_NO_OPTIMIZE 被设置为(任何值),或者当前的代码检查器(见 Code Inspectors) 不够强大,无法访问 racket/unsafe/ops, 例如在沙盒中执行时 (见 Sandboxed Evaluation), 优化器也会被禁用。 这可以防止不受信任的代码通过利用类型系统中的错误来访问这些操作。

11.1 契约优化

类型化的 Racket 为其出口生成合约,以保护它们免受未类型化代码的影响。 默认情况下,这些契约并不检查类型化代码是否服从类型。 如果你想生成平等地检查双方的合约(用于分析,用于教学等),那么将环境变量 PLT_TR_NO_CONTRACT_OPTIMIZE 设置为任何值并重新编译。
