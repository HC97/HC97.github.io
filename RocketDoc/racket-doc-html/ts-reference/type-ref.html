1.1 基本类型

1.1.1 数字类型

这些类型代表了 Racket 的 numbers 层次结构。 下图显示了层次结构中各类型之间的关系。

带有实心边框的区域是数字层次结构的 layers ,对应于整数或有理数等数字集合。 包含在另一个层中的层是包含它们的层的子类型。 例如, Exact-Rational 是 Exact-Number 的一个子类型。

Real 层也被划分为正数和负数类型(用虚线表示)。 Integer 层又分为几个固定宽度的整数类型,在本节后面详细介绍。

type

Number

type

Complex

Number 和 Complex 是同义词。 这是最通用的数字类型,包括所有的 Racket 数字,包括精确的和不精确的,包括复数。

type

Integer

包括 Racket 的精确整数,对应于 exact-integer? 谓词。 这是最通用的类型,对索引和其他需要积分值的操作仍然有效。

type

Single-Flonum

包括 Racket 的单精度浮点数,对应于 single-flonum? 谓词。 该类型不包括双精度浮点数。

type

Inexact-Real

包括 Racket 的所有浮点数,包括单精度和双精度。

type

Exact-Rational

包括 Racket 的精确有理数,其中包括分数和精确整数。

type

Real

包括 Racket 的所有实数,包括精确有理数和所有浮点数。 这是最一般的类型,为其定义了比较(如 <) 。

Real-Zero

Natural 和 Exact-Nonnegative-Integer 是同义词。 整数和精确整数类型也是如此,浮点数和弗洛尼姆类型也是如此。 Zero 只包括整数 0 . Real-Zero 包括精确的 0 和所有浮点的 0。

这些类型在强制要求数值具有特定符号时非常有用。 然而,当类型检查器不能保证符号约束得到遵守时,使用它们的程序可能需要额外的动态检查。

除了按符号划分外,整数还被进一步细分为有范围约束的类型。 大多数有范围限制的类型之间的关系显示在这个图中:

Nonpositive-Fixnum

One 只包括整数 1 。 Byte 包括从 0 到 255 的数字。 Index 以 0 和最长的 Racket 向量的长度为界。 Fixnum 包括所有被 Racket 表示为机器整数的数字。 对于后两个系列,类型中包含的数值集是与架构相关的,但类型检查是与架构无关的。

这些类型对于强制执行数值的界限是很有用的,但是鉴于这些类型提供的封闭属性数量有限,可能需要动态检查来在运行时检查所需的界限。

Path-String

Path 和 String 类型的结合。 请注意,这与谓词 path-string? 识别的内容不完全一致。 例如,包含字符 #\nul 的字符串的类型是 Path-String , 但是 path-string? 对这些字符串返回 #f 。 关于 path-string? 接受哪些字符串的完整说明,请看它的文档。

1.2 单例类型

某些类型的数据被默认为单子类型。 特别是, booleans, symbols 和 keywords 的类型只由特定的布尔值、符号或关键字组成。 这些类型分别是 Boolean, Symbol 和 Keyword 的子类型。

Examples:
> #t

- : True

#t

> '#:foo

- : '#:foo

'#:foo

> 'bar

- : 'bar

'bar

1.3 基本类型构造函数和超类型

type constructor

(Pairof s t)

返回一个包含 s 为 car, t 为 cdr 的 pair 类型。

Examples:
> (cons 1 2)

- : (Pairof One Positive-Byte)

'(1 . 2)

> (cons 1 "one")

- : (Pairof One String)

'(1 . "one")

type constructor

(Listof t)

返回一个 t 的同质 list 的类型

type constructor

(List t ...)

对于提供给 List 类型构造函数的每个类型,按顺序返回一个有一个元素的列表类型。

type constructor

(List t ... trest ... bound)

返回列表的类型, 其中每个 t 都有一个元素,加上与 trest 对应的元素序列, 其中 bound 必须是表示与 ... 绑定的类型变量的标识符

type constructor

(List* t t1 ... s)

相当于 (Pairof t (List* t1 ... s))。 (List* s) 等同于 s 本身。

Examples:
> (list 'a 'b 'c)

- : (List 'a 'b 'c)

'(a b c)

> (plambda: (a ...) ([sym : Symbol] boxes : (Boxof a) ... a)
    (ann (cons sym boxes) (List Symbol (Boxof a) ... a)))

- : (All (a ...)

      (-> Symbol (Boxof a) ... a (Pairof Symbol (List (Boxof a) ... a))))

#<procedure>

> (map symbol->string (list 'a 'b 'c))

- : (Pairof String (Listof String))

'("a" "b" "c")

(MListof t)

type constructor

(MPairof t u)

type

MPairTop

是一个具有未知元素类型的 mutable pair 的类型,是所有可变对类型的超类型。 这种类型通常通过发生类型和 mpair? 的组合出现在程序中。

Example:

- : (-> Any MPairTop)

#<procedure>

type constructor

(HashTable k v)

type constructor

(Immutable-HashTable k v)

Example:

- : (Immutable-HashTable Symbol Integer)

'#hash((a . 1) (b . 2))

Added in version 1.8 of package typed-racket-lib.

type constructor

(Mutable-HashTable k v)

返回键类型为 k 、值类型为 v 的强持键的易变 hash table 的类型 (见 弱盒)。

Example:
> (make-hash '((a . 1) (b . 2)))

- : (Mutable-HashTable Symbol Integer)

'#hash((a . 1) (b . 2))

Added in version 1.8 of package typed-racket-lib.

type constructor

(Weak-HashTable k v)

返回一个可变的 hash table 的类型, 该哈希表以键类型 k 和值类型 v 的方式持有弱键。

Example:
> (make-weak-hash '((a . 1) (b . 2)))

- : (Weak-HashTable Symbol Integer)

'#hash((a . 1) (b . 2))

Added in version 1.8 of package typed-racket-lib.

type

Example:
> (lambda: ([x : Any]) (if (hash? x) x (error "not a hash table!")))

- : (-> Any HashTableTop)

#<procedure>

type

type

当提供 opt-proposition 时, 它指定了函数类型的 proposition (关于 Typed Racket 中命题的介绍,请参见 命题和谓词)。 对于几乎所有的用例来说,只有最简单的命题形式,即在 : 后面有一个单一的类型,是必要的:

该命题规定,当 (string? x) 对一个条件分支求值为真值时, 可以假定该分支中的变量 x 的类型为 String 。 同样地,如果表达式在一个分支中求值为 #f ,则该变量 不 具有 String 类型。

在某些情况下,不对称的类型信息在命题中是有用的。 例如, filter 函数的第一个参数只指定了一个正命题:

Example:
> filter

- : (All (a b)

      (case->

       (-> (-> a Any : #:+ b) (Listof a) (Listof b))

       (-> (-> a Any) (Listof a) (Listof a))))

#<procedure:filter>

使用 #:+ 表示当应用于变量的函数求值为真值时,可以假定该变量的给定类型。 然而,在结果为 #f 的分支中,类型检查器没有获得任何信息。

反之, #:- 指定一个函数为条件的假分支提供信息。

其他的命题情况很少需要,但为了完整起见,语法记录了它们。 它们对应于对命题的逻辑操作。

函数的类型也可以用紧接在 rng 类型之前的 infix -> 来指定。 第四至第六种形式与前三种情况相匹配,但采用了箭头的 infix 样式。

> (: add2 (Number -> Number))
> (define (add2 n) (+ n 2))

目前,由于不支持存在类型的显式打包操作,存在类型的结果只被用来注释 Struct-Property 的访问器

(Some (a ...) type : #:+ proposition) 为 rng 指定一个 existential type result, 其中类型变量 a ... 可以出现在 type 和 opt-proposition 中。 在检查函数的应用时,会自动对存在型结果进行解包。
