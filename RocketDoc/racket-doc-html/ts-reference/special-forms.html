2 特殊形式参考

Typed Racket 提供了各种高于 Racket 中的特殊形式。 它们用于用类型来注解变量,创建新的类型,以及注解表达式。

2.1 绑定窗体

loop, f, a 和 var 是名字, type 是类型, e 是表达式, body 是块。

syntax

(let maybe-tvars (binding ...) maybe-ret . body)

(let loop maybe-ret (binding ...) . body)
 
binding = [var e]
  | [var : type e]
     
maybe-tvars = 
  | #:forall (tvar ...)
  | #:∀ (tvar ...)
     
maybe-ret = 
  | : type0
本地绑定,如 let, 每个都有相关的类型。

在第一种形式中, maybe-ret 只能和 maybe-tvars 一起出现, 所以如果你只想指定返回类型,你应该把 maybe-tvars 设置为 #:forall () 。

Examples:
> (let ([x : Zero 0]) x)

- : Integer [more precisely: Zero]

0

> (let #:forall () ([x : Zero 0]) : Natural x)

- : Integer [more precisely: Zero]

0

> (let ([x : Zero 0]) : Natural x)

eval:4:0: :: bad syntax

  in: :

如果提供了多态类型的变量,它们将被绑定在变量绑定的类型表达式中。

Example:
> (let #:forall (A) ([x : A 0]) x)

- : Integer [more precisely: Zero]

0

在第二种形式中, type0 是 loop 结果的类型 (因此也是整个表达式的结果,以及 body 中的最终表达式)。 类型注释是可选的。

Examples:
> (: filter-even (-> (Listof Natural) (Listof Natural) (Listof Natural)))
> (define (filter-even lst accum)
    (if (null? lst)
        accum
        (let ([first : Natural (car lst)]
              [rest  : (Listof Natural) (cdr lst)])
          (if (even? first)
              (filter-even rest (cons first accum))
              (filter-even rest accum)))))
> (filter-even (list 1 2 3 4 5 6) null)

- : (Listof Nonnegative-Integer)

'(6 4 2)

Examples:
> (: filter-even-loop (-> (Listof Natural) (Listof Natural)))
> (define (filter-even-loop lst)
    (let loop : (Listof Natural)
         ([accum : (Listof Natural) null]
          [lst   : (Listof Natural) lst])
      (cond
        [(null? lst)       accum]
        [(even? (car lst)) (loop (cons (car lst) accum) (cdr lst))]
        [else              (loop accum (cdr lst))])))
> (filter-even-loop (list 1 2 3 4))

- : (Listof Nonnegative-Integer)

'(4 2)

syntax

(let* (binding ...) . body)

syntax

(let-values ([(var+type ...) e] ...) . body)

syntax

(letrec-values ([(var+type ...) e] ...) . body)

类型注释版本的 letrec, let*, let-values, letrec-values 和 let*-values 。 与 let 一样,类型注解是可选的。

2.2 匿名函数

syntax

(lambda maybe-tvars formals maybe-ret . body)

 
formals = (formal ...)
  | (formal ... . rst)
     
formal = var
  | [var default-expr]
  | [var : type]
  | [var : type default-expr]
  | keyword var
  | keyword [var : type]
  | keyword [var : type default-expr]
     
rst = var
  | [var : type *]
  | [var : type ooo bound]
     
maybe-tvars = 
  | #:forall (tvar ...)
  | #:∀ (tvar ...)
  | #:forall (tvar ... ooo)
  | #:∀ (tvar ... ooo)
     
maybe-ret = 
  | : type
构造一个匿名函数,就像 racket/base 中的 lambda 形式,但允许对形式参数进行类型注解。 如果没有类型注解,那么形式将具有 Any 类型。

Examples:
> (lambda ([x : String]) (string-append x "bar"))

- : (-> String String)

#<procedure>

> (lambda (x [y : Integer]) (add1 y))

- : (-> Any Integer Integer)

#<procedure>

> (lambda (x) x)

- : (-> Any Any)

#<procedure>

类型注释也可以被指定给关键字和可选参数:

Examples:
> (lambda ([x : String "foo"]) (string-append x "bar"))

- : (->* () (String) (String : (Top | Bot)))

#<procedure:eval:15:0>

> (lambda (#:x [x : String]) (string-append x "bar"))

- : (-> #:x String String)

#<procedure:eval:16:0>

> (lambda (x #:y [y : Integer 0]) (add1 y))

- : (-> Any [#:y Integer] Integer)

#<procedure:eval:17:0>

> (lambda ([x 'default]) x)

- : (->* () (Any) Any)

#<procedure:eval:18:0>

lambda 表达式也可以指定多态的类型变量,这些变量为 formal 中的类型表达式所绑定。

Examples:
> (lambda #:forall (A) ([x : A]) x)

- : (All (A) (-> A A))

#<procedure>

> (lambda #:∀ (A) ([x : A]) x)

- : (All (A) (-> A A))

#<procedure>

此外,可以选择用统一类型或使用多态类型为 rest argument 指定一个类型。 在前一种情况下,休息参数被赋予类型 (Listof type) , 其中 type 是所提供的类型注释。

Examples:
> (lambda (x . rst) rst)

- : (-> Any Any * (Listof Any))

#<procedure>

> (lambda (x    rst : Integer *)  rst)

- : (-> Any Integer * (Listof Integer))

#<procedure>

> (lambda #:forall (A ...) (x    rst : A ... A)  rst)

- : (All (A ...) (-> Any A ... A (List A ... A)))

#<procedure>

syntax

(λ maybe-tvars formals maybe-ret . body)

使用 lambda 的相同形式的别名。

syntax

(case-lambda maybe-tvars [formals body] ...)

一个有多个算式的函数。 除了不允许使用关键字和可选参数外,这些 formals 与 lambda 形式所接受的形式完全相同。

如果提供了多态的类型变量,那么这些变量将被绑定在 formals 的类型表达式中。

注意,每个 formals 必须有不同的 arity。

Example:
> (define add-map
    (case-lambda
     [([lst : (Listof Integer)])
      (map add1 lst)]
     [([lst1 : (Listof Integer)]
       [lst2 : (Listof Integer)])
      (map + lst1 lst2)]))

要了解如何为 add-map 声明一个类型,请看 case-> 类型构造器。

2.4 定义

syntax

(define maybe-tvars v maybe-ann e)

(define maybe-tvars header maybe-ann . body)
 
header = (function-name . formals)
  | (header . formals)
     
formals = (formal ...)
  | (formal ... . rst)
     
formal = var
  | [var default-expr]
  | [var : type]
  | [var : type default-expr]
  | keyword var
  | keyword [var : type]
  | keyword [var : type default-expr]
     
rst = var
  | [var : type *]
  | [var : type ooo bound]
     
maybe-tvars = 
  | #:forall (tvar ...)
  | #:∀ (tvar ...)
  | #:forall (tvar ... ooo)
  | #:∀ (tvar ... ooo)
     
maybe-ann = 
  | : type
就像 racket/base 中的 define ,但允许对变量进行可选的类型注释。

第一种形式是为表达式 e 的求值结果定义一个变量 v 。 这个变量可以有一个可选的类型注解。

Examples:
> (define foo "foo")
> (define bar : Integer 10)

如果提供了多态类型的变量,那么它们将被绑定在类型注释中使用。

Example:
> (define #:forall (A) mt-seq : (Sequenceof A) empty-sequence)

第二种形式允许在任何变量上定义带有可选的类型注解的函数。 如果提供了返回类型注解,那么它被用来检查函数的结果。

像 lambda 一样,支持可选参数和关键字参数。

Examples:
> (define (add [first : Integer]
               [second  : Integer]) : Integer
    (+ first second))
> (define #:forall (A)
          (poly-app [func : (A A -> A)]
                    [first : A]
                    [second  : A]) : A
    (func first second))

函数定义的形式也允许带有相应类型注解的咖喱函数参数。

Examples:
> (define ((addx [x : Number]) [y : Number]) (+ x y))
> (define add2 (addx 2))
> (add2 5)

- : Number

7

注意,与 racket/base 的 define 不同, define 不会将带有关键字参数的函数与这些函数的静态信息绑定。

2.5 结构定义

syntax

(struct maybe-type-vars name-spec ([f : t] ...) options ...)

 
maybe-type-vars = 
  | (v ...)
     
name-spec = name-id
  | name-id parent
     
options = #:transparent
  | #:mutable
  | #:prefab
  | #:constructor-name constructor-id
  | #:extra-constructor-name constructor-id
  | #:property property-id property-expr
  | #:type-name type-id
定义一个名称为 name-id 的 structure ,其中字段 f 的类型为 t, 类似于 racket/base 中 struct 的行为。

Examples:
> (struct camelia-sinensis ([age : Integer]))
> (struct camelia-sinensis-assamica camelia-sinensis ())

如果没有指定 type-id , name-id 将被用于与声明的结构实例相关的类型名称。 否则, type-id 将被用于类型名,在这种情况下使用 name-id 将导致类型错误。

Examples:
> (struct apple () #:type-name BigApple)
> (ann (apple) BigApple)

- : BigApple

#<apple>

> (ann (apple) apple)

eval:45:0: Type Checker: parse error in type;

 type name `apple' is unbound

  in: apple

type-id 也可以作为 name-id 的别名使用,也就是说, 它将是一个转化器绑定,封装了与 name-id 相同的结构信息。

Examples:
> (struct avocado ([amount : Integer]) #:type-name Avocado)
> (struct hass-avocado Avocado ())
> (struct-copy Avocado (avocado 0) [amount 42])

- : Avocado

#<avocado>

当 parent 出现时,该结构是 parent 的一个子结构。

当 maybe-type-vars 出现时,该结构在类型变量 v 中是多态的。 如果 parent 也是一个多态结构,那么必须至少有与父类类型相同数量的类型变量, 并且父类类型被实例化,其类型变量的前缀与它所需要的数量相符。

Examples:
> (struct (X Y) 2-tuple ([first : X] [second : Y]))
> (struct (X Y Z) 3-tuple 2-tuple ([third :  Z]))

提供的选项与来自 racket/base 的 struct 形式的含义相同（ #:type-name 除外,如上所述)。

一个 prefab 结构类型声明将把给定的 name-id 或 type-id 绑定到一个 Prefab 类型。 与 racket/base 中的 struct 形式不同,非 prefab 结构类型不能扩展一个 prefab 结构类型。

Examples:
> (struct a-prefab ([x : String]) #:prefab)
> (:type a-prefab)

(Prefab a-prefab String)

> (struct not-allowed a-prefab ())

eval:53:0: Type Checker: Error in macro expansion -- parent

type not a valid structure name: a-prefab

  in: ()

Changed in version 1.4 of package typed-racket-lib: 增加了 #:type-name 选项。

2.6 类型名称

syntax

(define-type name t maybe-omit-def)

(define-type (name v ...) t maybe-omit-def)
 
maybe-omit-def = #:omit-define-syntaxes
  | 
第一种形式是将 name 定义为 type,其含义与 t 相同。 第二种形式将 name 定义为一个类型构造函数,其参数为 v ... ，主体为 t 。 如果没有声明参数,定义的类型构造函数就等同于 (define-type name t maybe-omit-def) 。 类型名可以指代定义在相同或包围范围内的其他类型。

Examples:
> (define-type IntStr (U Integer String))
> (define-type (ListofPairs A) (Listof (Pair A A)))

如果指定了 #:omit-define-syntaxes ,就不会创建 name 的定义。 在这种情况下,一些其他的 name 的定义是必要的。

如果类型定义的主体指的是它自己,那么这个类型定义就是递归的。 递归也可能相互发生,如果一个类型引用到其他类型的链上,最终又引用到自己。

Examples:
> (define-type BT (U Number (Pair BT BT)))
> (let ()
    (define-type (Even A) (U Null (Pairof A (Odd A))))
    (define-type (Odd A) (Pairof A (Even A)))
    (: even-lst (Even Integer))
    (define even-lst '(1 2))
    even-lst)

- : (Even Integer)

'(1 2)

然而,递归引用只有在传递给生产性类型构造函数时才被允许:

Examples:
> (define-type Foo Foo)

eval:58:0: Type Checker: Error in macro expansion -- parse

error in type;

 not in a productive position

  variable: Foo

  in: Foo

> (define-type Bar (U Bar False))

eval:59:0: Type Checker: Error in macro expansion -- parse

error in type;

 not in a productive position

  variable: Bar

  in: False

> (define-type Bar (U (Listof Bar) False))

2.7 自动生成谓词

syntax

(make-predicate t)

评估为类型 t 的谓词,其类型为 (Any -> Boolean : t) 。 t 不能包含函数类型,也不能包含可能引用可变数据的类型,如 (Vectorof Integer) 。

syntax

(define-predicate name t)

相当于 (define name (make-predicate t)) 。

2.8 类型注释和实例化

syntax

(: v t)

(: v : t)
这声明了 v 具有类型 t 。 v 的定义必须出现在这个声明之后。 这可以在任何可以使用定义形式的地方使用。

Examples:
> (: var1 Integer)
> (: var2 String)

第二种形式允许类型注解为函数类型省略一级括号。

Examples:
> (: var3 : -> Integer)
> (: var4 : String -> Integer)

syntax

(provide: [v t] ...)

这宣告了vv 具有 t 的类型,同时也提供了所有的 v 。

syntax

#{v : t}

这宣告了变量 v 具有类型 t 。 这只对 v 的绑定出现是合法的。

如果在当前的 readtable 中已经存在一个关于 #\{ 的调度宏,那么这个语法将被禁用。

syntax

(ann e t)

确保 e 具有类型 t, 或者某个子类型。 整个表达式的类型为 t 。 这只在表达式上下文中是合法的。

syntax

#{e :: t}

(ann e t) 的一个读者缩写。

如果在当前 readtable 中已经存在 #\{ 的调度宏,那么这个语法将被禁用。

syntax

(cast e t)

整个表达式的类型是 t, 而 e 可以有任何类型。 整个表达式的值是由 e 返回的值,由一个确保其类型为 t 的契约来保护。只有在表达式中才是合法的。

Examples:
> (cast 3 Integer)

- : Integer

3

> (cast 3 String)

broke its own contract

  promised: string?

  produced: 3

  in: string?

  contract from: cast

  blaming: cast

   (assuming the contract is correct)

  at: eval:66:0

> (cast (lambda ([x : Any]) x) (String -> String))

- : (-> String String)

#<procedure:val>

> ((cast (lambda ([x : Any]) x) (String -> String)) "hello")

- : String

"hello"

该值实际上由两个契约来保护。 第二个契约检查新的类型,但第一个契约是为了执行旧的类型,以保护值的高阶使用。

Examples:
> ((cast (lambda ([s : String]) s) (Any -> Any)) "hello")

- : Any

"hello"

> ((cast (lambda ([s : String]) s) (Any -> Any)) 5)

contract violation

  expected: string?

  given: 5

  in: the 1st argument of

      (-> string? any)

  contract from: typed-world

  blaming: cast

   (assuming the contract is correct)

  at: eval:70:0

cast 将把值 e 包裹在一个契约中,这将影响读取和更新该值的运行时性能。 当 e 是一个复杂的数据类型时,这是有必要的,比如一个哈希表。 然而,当值的类型可以用一个简单的谓词来检查时,可以考虑用 assert 代替。

syntax

(inst e t ...)

(inst e t ... t ooo bound)
用类型 t ... 或用聚点类型 t ... t ooo bound 实例化 e 的类型。 e 须有一个多态类型,可以应用于所提供的类型变量的数量。 然而,对于非聚点函数,可以提供较少的参数,并且省略的类型默认为 Any 。 inst 只在表达式上下文中合法。

Examples:
> (foldl (inst cons Integer Integer) null (list 1 2 3 4))

- : (Listof Integer)

'(4 3 2 1)

> (: my-cons (All (A B) (-> A B (Pairof A B))))
> (define my-cons cons)
> (: foldl-list : (All (α) (Listof α) -> (Listof α)))
> (define (foldl-list lst)
    (foldl (inst my-cons α (Listof α)) null lst))
> (foldl-list (list "1" "2" "3" "4"))

- : (Listof String)

'("4" "3" "2" "1")

> (: foldr-list : (All (α) (Listof α) -> Any))
> (define (foldr-list lst)
    (foldr (inst my-cons α) null lst))
> (foldr-list (list "1" "2" "3" "4"))

- : Any

'("1" "2" "3" "4")

> (: my-values : (All (A B ...) (A B ... -> (values A B ... B))))
> (define (my-values arg . args)
    (apply (inst values A B ... B) arg args))

syntax

(row-inst e row)

用 row 实例化 e 的 row-polymorphic 类型。这只在表达式上下文中是合法的。

Examples:
> (: id (All (r #:row)
          (-> (Class #:row-var r) (Class #:row-var r))))
> (define (id cls) cls)
> ((row-inst id (Row (field [x Integer])))
   (class object% (super-new) (field [x : Integer 0])))

- : (Class (field (x Integer)))

#<class:eval:84:0>

syntax

#{e @ t ...}

(inst e t ...) 的读者缩略语。

syntax

#{e @ t ... t ooo bound}

读者对 (inst e t ... t ooo bound) 的缩写。

2.9 需求

这里, m 是一个模块规格, pred 是命名一个谓词的标识符, maybe-renamed 是一个可以选择重命名的标识符。

syntax

(require/typed m rt-clause ...)

 
rt-clause = [maybe-renamed t]
  | 
[#:struct maybe-tvars name-id ([f : t] ...)
     struct-option ...]
  | 
[#:struct maybe-tvars (name-id parent) ([f : t] ...)
     struct-option ...]
  | [#:opaque t pred]
  | [#:signature name ([id : t] ...)]
     
maybe-renamed = id
  | (orig-id new-id)
     
maybe-tvars = 
  | (type-variable ...)
     
struct-option = #:constructor-name constructor-id
  | #:extra-constructor-name constructor-id
  | #:type-name type-id
这种形式要求模块 m 的标识符,给它们指定类型。

第一种情况要求 maybe-renamed, 赋予其类型 t 。

第二和第三种情况 需要名称为 name-id 的结构,并创建一个名称为 type-id 的新类型, 如果没有提供 type-id , 则为 name-id ,有字段 f ...,其中每个字段的类型为 t 。 第三种情况允许指定一个 parent 结构类型。 父结构类型必须是 Typed Racket 已知的结构类型,无论是内置的还是通过 require/typed 。 该结构谓词具有适当的 Typed Racket 过滤类型,因此它可以作为 Typed Racket 中 if 表达式的谓词使用。

Examples:
> (module UNTYPED racket/base
    (define n 100)
  
    (struct IntTree
      (elem left right))
  
    (provide n (struct-out IntTree)))
> (module TYPED typed/racket
    (require/typed 'UNTYPED
                   [n Natural]
                   [#:struct IntTree
                     ([elem  : Integer]
                      [left  : IntTree]
                      [right : IntTree])]))

第四种情况 是使用函数 pred 作为 predicate, 定义了一个新的 opaque type t 。 (模块 m 必须提供 pred ， pred 必须具有 (Any -> Boolean) 类型)。 类型 t 被定义为正是那些 pred 返回 #t 的值。 不透明的类型在使用前必须是词法上的要求。

Examples:
> (require/typed racket/base
                 [#:opaque Evt evt?]
                 [alarm-evt (Real -> Evt)]
                 [sync (Evt -> Any)])
> evt?

- : (-> Any Boolean : Evt)

#<procedure:evt?>

> (sync (alarm-evt (+ 100 (current-inexact-milliseconds))))

- : Any

#<alarm-evt>

#:signature 关键字 在签名环境中注册了所需的签名。 关于在 Typed Racket 中使用签名的更多信息,请参见 typed/racket/unit 的文档。

在所有情况下,标识符都是用 contracts 来保护的,这些契约强制执行指定的类型。 如果这个契约失败,模块 m 就会受到责备。

一些类型,特别是像 number? 这样的谓词的类型,不能转换为契约, 当以 require/typed 形式使用时,会引发静态错误。 下面是一个在 require/typed 中使用 case-> 的例子。

(require/typed racket/base
               [file-or-directory-modify-seconds
                (case->
                  [String -> Exact-Nonnegative-Integer]
                  [String (Option Exact-Nonnegative-Integer)
                          ->
                          (U Exact-Nonnegative-Integer Void)]
                  [String (Option Exact-Nonnegative-Integer) (-> Any)
                          ->
                          Any])])

file-or-directory-modify-seconds 有一些参数是可选的,所以我们需要使用 case-> 。

Changed in version 1.4 of package typed-racket-lib: 添加了 #:type-name 选项。
Changed in version 1.6: 增加了 struct 类型变量的语法,只在不安全的 require 中有效。
Changed in version 1.12: 为省略的 inst args 添加了默认类型 Any 。

syntax

(require/typed/provide m rt-clause ...)

与 require/typed 类似,但也提供了导入的标识符。 在模块顶层之外使用会引发错误。

Examples:
> (module evts typed/racket
    (require/typed/provide racket/base
                           [#:opaque Evt evt?]
                           [alarm-evt (Real -> Evt)]
                           [sync (Evt -> Any)]))
> (require 'evts)
> (sync (alarm-evt (+ 100 (current-inexact-milliseconds))))

- : Any

#<alarm-evt>

2.11 特殊结构类型属性

value

prop:procedure : struct-type-property?

与其他许多结构类型属性不同, prop:procedure 没有为其属性值预定义类型。 当一个结构被分配给 prop:procedure 时,其构造器的返回类型是结构类型和属性值指定的函数类型的交集。

Examples:
> (struct animal ([a : Number] [b : (-> Number Number)])
    #:property prop:procedure
    (struct-field-index b))
> (animal 2 add1)

- : (∩ (-> Number Number) animal)

#<procedure:add1>

> (struct plant ([a : Number])
    #:property prop:procedure
    (lambda ([me : plant] [a1 : String]) : Number
      (+ (plant-a me) (string-length a1))))
> (plant 31)

- : (∩ (-> String Number) plant)

#<procedure:plant>

换句话说,不允许有一个指向函数的变量。

与 Racket 不同,在 Typed Racket 中只允许以下类型的表达式之一:非负的字面意义, (struct-index-field field-name), 或 lambda 表达式。 请注意,在最后一种情况下,如果没有提供代码域的类型注释,类型检查器将使用 Any 作为返回类型。

与其它结构类型属性类似,当一个结构的基结构为 prop:procedure 指定一个值时, 如果该结构没有指定自己的值,那么它将继承该值。

Examples:
> (struct cat animal ([c : Number]))
> (cat 2 add1 42)

- : (∩ (-> Number Number) cat)

#<procedure:add1>

> (struct a-cat cat ())
> (a-cat 2 add1 42)

- : (∩ (-> Number Number) a-cat)

#<procedure:add1>

程序性结构的函数类型不强制执行子类型关系。 一个子结构可以为 prop:procedure 指定一个不同的字段索引或一个具有不同于其基结构的 arity 和/或类型的程序。

Examples:
> (struct b-cat cat ([d : (-> Number String)])
     #:property prop:procedure (struct-field-index d))
> (b-cat 2 add1 42 number->string)

- : (∩ (-> Number String) b-cat)

#<procedure:number->string>
