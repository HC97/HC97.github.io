3 Typed Racket 提供的库

typed/racket 语言对应于 racket 语言–也就是说, 任何由 racket 提供的标识符,如 modulo ,在 typed/racket 中默认都是可用的。

#lang typed/racket
(modulo 12 2)

typed/racket/base 语言对应于 racket/base 语言。

一些库在 typed 集合中有对应的库,它们提供了与非 typed 版本相同的输出。 这样的库包括 srfi/14,net/url, 和许多其他的库。

#lang typed/racket
(require typed/srfi/14)
(char-set= (string->char-set "hello")
           (string->char-set "olleh"))

其他库可以通过 require/typed 与 Typed Racket 一起使用。

#lang typed/racket
(require/typed version/check
               [check-version (-> (U Symbol (Listof Any)))])
(check-version)

3.1 将非类型化模块移植到 Typed Racket 中

要改造一个未包含在 Typed Racket 中的 Racket 库,需要采取以下步骤:

  • 确定库所操作的数据,以及它在 Typed Racket 中的表示方式。

  • 在 Typed Racket 中指定这些数据,使用 require/typed 和 #:opaque 和/或 #:struct 。

  • 使用数据类型来导入库中的各种函数和常量。

  • 从新的适配器模块中提供所有相关的标识符。

例如,下面的模块适应了未定型的 racket/bool 库:

#lang typed/racket
(require/typed racket/bool
               [true Boolean]
               [false Boolean]
               [symbol=? (Symbol Symbol -> Boolean)]
               [boolean=? (Boolean Boolean -> Boolean)]
               [false? (Any -> Boolean)])
(provide true false symbol=? boolean=? false?)

在 typed 集合中可以得到更多实质性的例子。
