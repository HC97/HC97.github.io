6.16 严格性

parameter

(array-strictness) → Boolean

(array-strictness strictness) → void?
  strictness : Boolean
决定 math/array 函数是否返回严格的数组。 默认值是 #t 。

关于非严格数组的讨论,请参见 非限制性数组 。

procedure

(array-strict? arr) → Boolean

  arr : (Array A)
当 arr 是 strict 的时候,返回 #t 。

Examples:
> (define arr
    (parameterize ([array-strictness #f])
      (array+ (array 10) (array #[0 1 2 3]))))
> (array-strict? arr)

- : Boolean

#f

> (array-strict! arr)
> (array-strict? arr)

- : Boolean

#t

procedure

(array-strict! arr) → Void

  arr : (Array A)
导致 arr 计算并存储其所有的元素。 此后, arr 通过从存储空间中获取元素来计算其元素。

如果 arr 已经是严格的, (array-strict! arr) 不做任何事情。

syntax

(array-strict arr)

 
  arr : (Array A)
是 array-strict! 的表达形式,通常更方便。 首先评估 (array-strict! arr), 然后返回 arr 。

这是一个宏,所以 Typed Racket 会准确地保留 arr 的类型。 如果它是一个函数, (array-strict arr) 的类型将总是 (Array A), 即使 arr 是 (Array A) 的一个子类型,比如 (Mutable-Array A) 。

procedure

(array-default-strict! arr) → Void

  arr : (Array A)

syntax

(array-default-strict arr)

 
  arr : (Array A)
像 array-strict! 和 array-strict 一样, 但是当 array-strictness 为 #f 时,什么都不做。

对库函数的返回值应用其中之一,以确保用户默认得到严格的数组。 详见 非限制性数组 。

procedure

(build-simple-array ds proc) → (Array A)

  ds : In-Indexes
  proc : (Indexes -> A)
和 build-array 一样,但是返回一个没有存储空间的数组,不管 array-strictness 的值是多少,都被认为是严格的。 当 array-strict! 或 array-strict 被应用于数组时, 这样的数组将 不 会缓存其元素。

使用 build-simple-array 来创建代表其索引的简单函数的数组。 例如,基本的数组构造函数,如 make-array ,是用这个函数或其不安全的对应函数来定义的。

对这个函数要小心。 虽然它创建的数组总是很节省内存, 但如果用它来定义那些元素查找永远都很昂贵的数组,就很容易毁掉你的程序的性能。 在错误的情况下,使用它而不是 build-array 可以将一个线性算法变成指数算法!

一般来说,在以下情况下使用 build-simple-array

  • 计算一个元素永远不会比计算一个行主索引,然后应用 vector-ref 更昂贵。 一个例子是 index-array, 它只计算行主索引。

  • 计算一个元素是独立于任何其他数组的元素的。 在这种情况下,不可能组成一些无界数的可能昂贵的数组 procedures 。

  • 你可以证明每个元素在程序的整个生命周期中最多被计算一次。 这是真的,例如,当结果只被发送到一个对其进行复制的函数, 如 array-lazy 或 array->mutable-array 。

最后一种情况的例子见 array-lazy 。

procedure

(array-lazy arr) → (Array A)

  arr : (Array A)
返回一个不可变的、 nonstrict 意义上的数组,其元素与 arr 相同,但是元素的计算被缓存。

也许使用 array-lazy 的最自然的方式是所谓的 "动态编程",或者对一个刚好有矩形域的函数进行记忆。 例如,这可以在线性时间内计算出前 10 个斐波那契数:
> (define: fibs : (Array Natural)
    (array-lazy
     (build-simple-array
      #(10) (λ: ([js : Indexes])
              (define j (vector-ref js 0))
              (cond [(j . < . 2)  j]
                    [else  (+ (array-ref fibs (vector (- j 1)))
                              (array-ref fibs (vector (- j 2))))])))))
> fibs

因为 build-simple-array 从不存储它的元素,它的过程参数可能是指它返回的数组。 用 array-lazy 包装它的结果,使得每个 array-ref 的时间不超过线性时间; 此外,当 fibs 的元素按顺序计算时,每个数组的时间是恒定的。 如果没有 array-lazy, 计算 fibs 的元素将需要指数级的时间。

打印一个懒惰数组会计算并缓存其所有的元素, 就像对其应用 array-strict! 或 array-strict 一样。

除了由 build-simple-array 返回的数组, 对一个 strict 数组应用 array-lazy 是没有用的。 使用懒惰副本而不是原始副本只会降低性能。

虽然看起来 array-lazy 应该只是在 arr 是严格的时候返回 arr , 但这违反了 array-lazy 返回不可变的数组的不变性。比如说:
> (: array-maybe-lazy (All (A) ((Array A) -> (Array A))))
> (define (array-maybe-lazy arr)
    (if (array-strict? arr) arr (array-lazy arr)))
> (define arr (mutable-array #[0 1 2 3]))
> (define brr (array-maybe-lazy arr))
> (array-set! arr #(0) -1000)
> brr
