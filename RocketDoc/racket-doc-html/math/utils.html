10.1 平行化

parameter

(max-math-threads) → Positive-Integer

(max-math-threads num) → void?
  num : Positive-Integer
并行的 math 函数将使用的最大线程数。 默认值是 (max 1 (processor-count)) 。

10.2 离散傅里叶变换约定

parameter

(dft-convention) → (List Real Real)

(dft-convention lst) → void?
  lst : (List Real Real)
一个控制离散傅里叶变换比例约定的参数,例如由 array-fft 执行的变换。 默认值是 '(1 -1), 代表信号处理中使用的惯例。

一般来说,如果 lst 是 (list a b) ， n 是转换后的数组轴或矢量的长度,那么

  • 每个和的比例是 (expt n (/ (- a 1) 2)) 。

  • 和中的每个指数的参数都按 b 的比例缩放。

方便的是,具有约定俗成的 (list (- a) (- b)) 的傅里叶变换是具有约定俗成的 (list a b) 的傅里叶变换的逆运算。

参见 Mathematica documentation on Fourier, 这个伟大的想法就是从那里偷来的。

procedure

(dft-inverse-convention) → (List Real Real)

返回用于逆傅里叶变换的约定,给定当前的约定。

10.3 Floating-Point 兼容测试

procedure

(test-floating-point n) → (Listof (List Any Any))

  n : Natural
对系统的 IEEE 754(浮点)兼容性进行全面测试,并报告意外的不准确和错误。

在每个测试中,一个函数被应用于一些精心挑选的值,以及 n 个额外的随机值。 其相应的 bigfloat 函数被应用于相同的值,并对答案进行比较。 每个测试都会返回一个失败的列表,这些列表会被附加并返回。

失败列表中的每个失败都是格式化的

(list (list name args ...) reason)

其中 name 是一个函数的名称,如 'fl+, args ... 是它所应用的参数, reason 是失败的原因。

如果 reason 是一个 flonum,说明失败是由于不准确。例如、

(list (list 'fl+ 4.5 2.3) 0.76)

表示 (fl+ 4.5 2.3) 的结果偏离了 0.76 ulps 。

报告意外不准确的阈值取决于测试的函数。 例如,所有由 racket/flonum 输出的算术和无理数函数, 必须有不超过 0.5 的误差,才能符合要求。

另外两个可能的失败原因是
(list 'different-zero 0.0 -0.0)
(list 'different-zero -0.0 0.0)
第一个零是函数返回的答案,第二个零是预期的答案。

其他可能的失败原因有以下形式

(list 'not-fl2? x y)

意味着结果 (values x y) 不是一个有效的 flonum 扩展。 这样的原因只对名称以 fl2 开头或包含 /error 的函数的失败给出。 这些函数目前还没有文档,但被用来实现许多 math/flonum, math/special-functions 和 math/distributions 函数。

对操作和返回 flonum 扩展的函数的测试是最严格的测试,要求硬件算术完全符合 IEEE 754 标准。 它们在看似无害的不符合规定的行为上可靠地失败,例如用 80 位精度计算中间结果。

parameter

(print-fp-test-progress?) → Boolean

(print-fp-test-progress? print?) → void?
  print? : Boolean
当 (print-fp-test-progress?) 是 #t 时,浮点测试会在运行过程中打印并刷新其进度表。 默认值是 #t 。
