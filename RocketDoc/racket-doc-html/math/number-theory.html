4.1 共轭和模块化算术

Wikipedia: Divisor

procedure

(divides? m n) → Boolean

  m : Integer
  n : Integer
如果 m 除以 n,返回 #t ,否则返回 #f 。

从形式上看,当存在一个唯一的整数 k ,使得 (* m k) = n 时, 一个整数 m 就能除以一个整数 n 。

Examples:
> (divides? 2 9)

#f

> (divides? 2 8)

#t

注意, 0 不能除以任何东西:
> (divides? 0 5)

#f

> (divides? 0 0)

#f

实际上,如果 (divides? m n) 是 #t, 那么 (/ n m) 返回一个整数, 不会引发 exn:fail:contract:divide-by-zero 。

Wikipedia: Bezout’s Identity

procedure

(bezout a b c ...) → (Listof Integer)

  a : Integer
  b : Integer
  c : Integer
给定整数 a b c ... 返回一个整数列表 (list u v w ...) , 使得 (gcd a b c ...) = (+ (* a u) (* b v) (* c w) ...) 。

Examples:
> (bezout 6 15)

'(-2 1)

> (+ (* -2 6) (* 1 15))

3

> (gcd 6 15)

3

Wikipedia: Coprime

procedure

(coprime? a b ...) → Boolean

  a : Integer
  b : Integer
如果整数 a b ... 是共素数,则返回 #t 。 从形式上看,如果一组整数的最大公除数是 1,则被认为是共素数(也称为相对素数)。

Example:
> (coprime? 2 6 15)

#t

Wikipedia: Pairwise Coprime

procedure

(pairwise-coprime? a b ...) → Boolean

  a : Integer
  b : Integer
如果整数 a b ... 是 成对 的共素数,即每一对整数都是共素数, 则返回 #t 。

数字 2,6 和 15 是共素数,但不是 成对 共素数,因为 6 和 15 共享因子 3:
> (pairwise-coprime? 2 6 15)

#f

Wikipedia: Chinese Remainder Theorem

procedure

(solve-chinese as ns) → Natural

  as : (Listof Integer)
  ns : (Listof Integer)
给出一个长度为 k 的整数列表 as 和一个长度为 k 的共素数列表 ns, (solve-chinese as ns) 返回最小自然数 x ,它是方程的解

x = a1 (mod n1)
 ...
x = ak (mod nk)

解决方案 x 小于 (* n1 ... nk) 。

模数 ns 必须全部为正数。

什么是最小的数字 x ,当除以 3 时留下 2 的剩余,当除以 5 时留下 3 的剩余,当除以 7 时留下 2 的剩余?
> (solve-chinese '(2 3 2) '(3 5 7))

23

Wikipedia: Quadratic Residue

procedure

(quadratic-residue? a n) → Boolean

  a : Integer
  n : Integer
如果 a 是模数为 n 的二次方残差,则返回 #t ,否则返回 #f 。 模数 n 必须是正数, a 必须是非负数。

从形式上看,如果存在一个数字 x ,使得 (* x x) = a (mod n), 那么 a 就是模数为 n 的二次残差。 换句话说,当 a 是对 n 的完全平方时, (quadratic-residue? a n) 是 #t 。

Examples:
> (quadratic-residue? 0 4)

#f

> (quadratic-residue? 1 4)

#t

> (quadratic-residue? 2 4)

#f

> (quadratic-residue? 3 4)

#f

Wikipedia: Legendre Symbol

procedure

(quadratic-character a p) → (U -1 0 1)

  a : Integer
  p : Integer
返回对素数 p 的二次方字符的值。 也就是说,对于一个非零的 a ,当 a 是一个二次残差时,会返回数字 1 , 而当 a 是一个非残差时,会返回 -1 1。 如果 a 为零,则返回 0 。

如果 a 是负数或者 p 不是正数, 则 quadratic-character 会产生一个错误。 如果 p 不是质数, (quadratic-character a p) 是不确定的。

这个函数也被称为 Legendre symbol 。

> (quadratic-character 0 5)

0

> (quadratic-character 1 5)

1

> (quadratic-character 2 5)

-1

> (quadratic-character 3 5)

-1

Wikipedia: Jacobi Symbol

procedure

(jacobi-symbol a n) → (U -1 0 1)

  a : Nonnegative-Integer
  n : Positive-Integer
计算任何非负整数 a 和任何正奇数 n 的雅可比符号。

如果 n 不是奇数正整数, (jacobi-symbol a n) 会抛出一个异常。

> (jacobi-symbol 1 1)

1

> (jacobi-symbol 8 11)

-1

> (jacobi-symbol 39 27)

0

> (jacobi-symbol 22 59)

1

> (jacobi-symbol 32 8)

jacobi: contract violation

  expected: odd?

  given: 8

  argument position: 2nd

  other arguments...:

   32

Wikipedia: Multiplicative Inverse

procedure

(modular-inverse a n) → Natural

  a : Integer
  n : Integer
如果 a 和 n 互素,则返回 a modulo n 的倒数,否则会引发错误。 模数 n 必须是正数, a 必须是非零。

形式上,如果 a 和 n 是共素数, b = (modular-inverse a n) 是小于 n 的唯一自然数, 使得 (* a b) = 1 (mod n) 。

> (modular-inverse 2 5)

3

> (modulo (* 2 3) 5)

1

procedure

(modular-expt a b n) → Natural

  a : Integer
  b : Integer
  n : Integer
计算 (modulo (expt a b) n), 但效率更高。 模数 n 必须是正数。

Examples:
> (modulo (expt -6 523) 19)

13

> (modular-expt -6 523 19)

13

> (modular-expt 9 158235208 19)

4

> (modular-expt 2 -1 11)

6

> ; don't try this at home!
  (modulo (expt 9 158235208) 19)

4

4.1.1 参数化模块算术

Wikipedia: Modular Arithmetic

math/number-theory 库支持以当前模数为参数的模块化算术。例如,代码

(with-modulus n
  ((modexpt a b) . mod= . c))

对应于数学语句 ab = c (mod n).

当前的模数被存储在一个参数中,由于性能的原因,只能 with-modulus来设置。 (基本的模块化运算符对参数的读取进行了缓存,这一限制保证了缓存的值是最新的)。

syntax

(with-modulus n body ...)

 
  n : Integer
在 body 的动态范围内改变当前的模数。 表达式 n 必须评估为一个正整数。

默认情况下,当前模数是 1, 这意味着每个没有引发错误的模块化算术表达式都返回 0 。

procedure

(current-modulus) → Positive-Integer

返回当前的模数。

Examples:
> (current-modulus)

1

> (with-modulus 5 (current-modulus))

5

procedure

(mod x) → Natural

  x : Exact-Rational
将一个有理数 x 转换为一个小于当前模数的自然数。

如果 x 是一个整数,这等同于 (modulo x n) 。 如果 x 是一个分数,通过将其分子乘以分母的模块逆数来生成一个整数输入。

Examples:
> (with-modulus 7 (mod (* 218 7)))

0

> (with-modulus 7 (mod 3/2))

5

> (with-modulus 7 (mod/ 3 2))

5

> (with-modulus 7 (mod 3/7))

modular-inverse: expected argument that is coprime to

modulus 7; given 7

procedure

(mod+ a ...) → Natural

  a : Integer

procedure

(mod* a ...) → Natural

  a : Integer
分别等同于 (modulo (+ a ...) (current-modulus)) 和 (modulo (* a ...) (current-modulus)), 但生成的中间值较小。

procedure

(modsqr a) → Natural

  a : Integer

procedure

(modexpt a b) → Natural

  a : Integer
  b : Integer
分别等同于 (mod* a a) and (modular-expt a b (current-modulus)) 。

procedure

(mod- a b ...) → Natural

  a : Integer
  b : Integer
相当于 (modulo (- a b ...) (current-modulus)), 但产生的中间值较小。 注意 (mod- a) = (mod (- a)) 。

procedure

(mod/ a b ...) → Natural

  a : Integer
  b : Integer
用 (* b ...) 除以 a ,方法是将 a 乘以 (* b ...) 的乘法逆数。 单参数变体返回 a 的模块逆。

注意 (mod/ a b ...) 不 等同于 (modulo (/ a b ...) (current-modulus)); 见 mod= 的演示。

procedure

(mod= a b ...) → Boolean

  a : Integer
  b : Integer

procedure

(mod< a b ...) → Boolean

  a : Integer
  b : Integer

procedure

(mod<= a b ...) → Boolean

  a : Integer
  b : Integer

procedure

(mod> a b ...) → Boolean

  a : Integer
  b : Integer

procedure

(mod>= a b ...) → Boolean

  a : Integer
  b : Integer
每一个都等同于 (op (mod a) (mod b) ...), 其中 op 是相应的数字比较函数。 此外,当给出一个参数时,不等式测试总是返回 #t 。

假设我们想知道为什么 17/4=8(mod 15),但 51/12(mod 15) 是未定义的,尽管通常 51/12=17/4。在代码中、
> (with-modulus 15 (mod/ 17 4))

8

> (/ 51 12)

17/4

> (with-modulus 15 (mod/ 51 12))

modular-inverse: expected argument that is coprime to

modulus 15; given 12

我们可以尝试用蛮力进行除法:在 15 的模上, 找到所有 (mod* a 4) 为 17 的数字 a , 然后找到所有(mod* a 12) 为 51 的数字 b 。
> (with-modulus 15
    (for/list ([a  (in-range 15)]
               #:when (mod= (mod* a 4) 17))
      a))

'(8)

> (with-modulus 15
    (for/list ([b  (in-range 15)]
               #:when (mod= (mod* b 12) 51))
      b))

'(3 8 13)

所以问题不在于 b 不存在,而在于 b 不是 唯一的 。

4.2 素数

Wikipedia: Prime Number

procedure

(prime? z) → Boolean

  z : Integer
如果 z 是素数,返回 #t ,否则返回 #f 。

从形式上看,当 z 的唯一正除数是 1 和 (abs z) 时,整数 z 是素数。

低于 20 的正素数是:
> (filter prime? (range 1 21))

'(2 3 5 7 11 13 17 19)

相应的负素数是:
> (filter prime? (range 1 -21 -1))

'(-2 -3 -5 -7 -11 -13 -17 -19)

procedure

(odd-prime? z) → Boolean

  z : Integer
如果 z 是一个奇数素数,返回 #t ,否则返回 #f 。

> (odd-prime? 2)

#f

> (odd-prime? 3)

#t

procedure

(nth-prime n) → Natural

  n : Integer

3

> (nth-prime 2)

5

procedure

(random-prime n) → Natural

函数 random-prime 挑选低于 n 的随机数,直到找到一个素数。

> (random-prime 10)

> (random-prime 10)

(next-primes z n) → (Listof Integer)

(prev-primes z n) → (Listof Integer)

Wikipedia: Integer Factorization

procedure

(factorize n) → (Listof (List Natural Natural))

  n : Natural
返回一个自然数 n 的因式分解。 因式分解包括一个相应的素数和指数的列表。 素数将以升序排列。

600 = 2^3 * 3^1 * 5^2 的质因式分解:
> (factorize 600)

'((2 3) (3 1) (5 2))

procedure

(defactorize f) → Natural

  f : (Listof (List Natural Natural))
返回自然数,其因式分解是由 f 给出的。 因式分解 f 的表示方法如 factorize 中所述。

> (defactorize '((2 3) (3 1) (5 2)))

600

procedure

(divisors z) → (Listof Natural)

  z : Integer
返回一个整数 z 的所有正除数的列表。 被除数以升序出现。

> (divisors 120)

'(1 2 3 4 5 6 8 10 12 15 20 24 30 40 60 120)

> (divisors -120)

'(1 2 3 4 5 6 8 10 12 15 20 24 30 40 60 120)

procedure

(prime-divisors z) → (Listof Natural)

  z : Natural
返回一个整数 z 的所有正素数除数的列表。 被除数以升序出现。

> (prime-divisors 120)

'(2 3 5)

procedure

(prime-exponents z) → (Listof Natural)

  z : Natural
返回整数 z 的因式分解中的指数列表。

> (define z (* 2 2 2 3 5 5))
> (prime-divisors z)

'(2 3 5)

> (prime-exponents z)

'(3 1 2)

  m : Natural

4.4 幂

procedure

(max-dividing-power a b) → Natural

  a : Integer
  b : Integer
返回以 a 为基数的幂的最大指数 n ,该指数除以 b 。

也就是说, (expt a n) 除以 b ,但 (expt a (+ n 1)) 并不除以 b 。

> (max-dividing-power 3 (expt 3 4))

4

> (max-dividing-power 3 5)

0

procedure

> (perfect-power (expt 3 4))

'(3 4)

> (perfect-power (+ (expt 3 4) 1))

#f

(as-power m) →

4.5 乘法和算术函数

本节中的函数都是 multiplicative (Von Mangoldt 函数除外)。 在数论中,一个乘法函数是一个函数 f , 使得 (f (* a b)) = (* (f a) (f b)) 对于所有共质自然数 a 和 b 。

Wikipedia: Euler’s Totient

procedure

(totient n) → Natural

  n : Natural
返回从 1 到 n 中与 n 共质的整数的数量。

这个函数被称为 Eulers totient 或 phi 函数。

> (totient 9)

6

> (length (filter (curry coprime? 9) (range 10)))

6

Wikipedia: Moebius Function

procedure

(moebius-mu n) → (U -1 0 1)

  n : Natural
返回:

  • 如果 n 是偶数素数的无平方积,则返回 1

  • 如果 n 是奇数素数的无平方积,则返回 -1

  • 如果 n 有一个多质数因子,则返回 0

4.11 组 Zn 和原始根

Wikipedia: The Group Zn

数字 0, 1, ..., n-1 与 n 的加法和乘法是一个叫做 Zn 的环。

Zn 中关于乘以 n 的单位群被称为 Un 。

Un 中一个元素 x 的阶是最小的 k>0 , 使得 x^k=1 mod n 。

群 Un 中的一个生成器被称为 primitive root modulo n 。 请注意,当且仅当 order(g)=totient(n) 时, g 是一个原始根。 一个有生成器的群被称为 cyclic 。

procedure

(unit-group n) → (Listof Positive-Integer)

  n : Integer
返回 Un 的所有元素的列表,即单位群的模数 n 。 模数 n 必须是正数。
> (unit-group 5)

'(1 2 3 4)

> (unit-group 6)

'(1 5)

procedure

(unit-group-order x n) → Positive-Integer

  x : Integer
  n : Integer
返回 x 在群 Un 中的顺序;两个参数都必须是正数。 如果 x 和 n 不是共素数, (unit-group-order x n) 会引发一个错误。
> (unit-group-order 2 5)

4

> (unit-group-order 2 6)

unit-group-order: expected coprime arguments; given 2 and 6

procedure

(unit-group-orders n) → (Listof Positive-Integer)

  n : Integer
返回一个列表 (list (unit-group-order x0 n) (unit-group-order x1 n) ...) , 其中 x0, x1, ... 是 Un 的元素。 模数 n 必须是正数。
> (unit-group-orders 5)

'(1 4 4 2)

> (map (curryr unit-group-order 5) (unit-group 5))

'(1 4 4 2)

procedure

(primitive-root? x n) → Boolean

  x : Integer
  n : Integer
如果 Un 中的元素 x 是模数为 n 的原始根, 返回 #t ,否则返回 #f 。 如果 x 不是 Un 的成员,则发出错误信号。两个参数都必须是正数。
> (primitive-root? 1 5)

#f

> (primitive-root? 2 5)

#t

> (primitive-root? 5 5)

primitive-root?: expected coprime arguments; given 5 and 5

procedure

(exists-primitive-root? n) → Boolean

  n : Integer
如果组 Un 有一个原始根(即它是循环的),则返回 #t ,否则返回 #f 。 换句话说,如果 n 是 1, 2, 4, p^e, 2*p^e 中的一个, 其中 p 是奇数,则返回 #t ,否则返回 #f 。 模数 n 必须是正数。
> (exists-primitive-root? 5)

#t

> (exists-primitive-root? 6)

#t

> (exists-primitive-root? 12)

#f

procedure

(primitive-root n) → (Union Natural #f)

  n : Integer
如果存在 Un 的原始根,则返回一个原始根,否则将返回 #f 。 模数 n 必须是正数。
> (primitive-root 5)

2

> (primitive-root 6)

5

procedure

(primitive-roots n) → (Listof Natural)

  n : Integer
返回 Un 的所有原始根的列表。 模数 n 必须是正数。
> (primitive-roots 3)

'(2)

> (primitive-roots 5)

'(2 3)

> (primitive-roots 6)

'(5)
