7.1 简介

从 math/matrix 中的函数的角度来看, matrix 是一个有两个轴和至少一个条目的 Array , 或者一个数组,对于这个数组, matrix? 返回 #t 。

从技术上讲,矩阵的条目可以是任何类型,一些完全多态的矩阵函数, 如 matrix-row 和 matrix-map ,可以操作任何类型的矩阵。 其他函数,如 matrix+, 要求其矩阵参数包含数字值。

7.1.1 函数类型

math/matrix 函数的文档中,当函数要求参数是 matrix 或确保返回值是矩阵时, 就会使用 Matrix 类型,这是 Array 的同义词。

大多数实现矩阵算法的函数被记录为接受 (Matrix Number) 值。 这包括 (Matrix Real), 它是一个子类型。 这些函数中的大多数都有一个比文档中更精确的类型。 例如, matrix-conjugate 的类型是
(case-> ((Matrix Flonum)        -> (Matrix Flonum))
        ((Matrix Real)          -> (Matrix Real))
        ((Matrix Float-Complex) -> (Matrix Float-Complex))
        ((Matrix Number)        -> (Matrix Number)))
但在文件中被描述为具有 ((Matrix Number) -> (Matrix Number)) 的类型。

精确的函数类型使得 Typed Racket 可以证明更多关于 math/matrix 客户端程序的事实。 特别是,它通常很容易证明,对实数矩阵的操作会返回实数矩阵:
> (matrix-conjugate (matrix [[1 2 3] [4 5 6]]))

- : (Array Real)

(array #[#[1 2 3] #[4 5 6]])

以及对不精确矩阵的操作会返回不精确的矩阵:
> (matrix-conjugate (matrix [[1.0+2.0i 2.0+3.0i 3.0+4.0i]
                             [4.0+5.0i 5.0+6.0i 6.0+7.0i]]))

- : (Array Float-Complex)

(array #[#[1.0-2.0i 2.0-3.0i 3.0-4.0i] #[4.0-5.0i 5.0-6.0i 6.0-7.0i]])

7.1.2 失败的参数

在许多矩阵操作中,比如反转,失败很容易在计算过程中检测出来,但提前检测的代价和操作本身一样高。 在这些情况下,实现这些操作的函数接受一个可选的 failure thunk, 或者一个零参数函数,在失败时返回操作结果。

例如, matrix-inverse 逆的(简化)类型是
(All (F) (case-> ((Matrix Number)        -> (Matrix Number))
                 ((Matrix Number) (-> F) -> (U F (Matrix Number)))))
因此,如果给了一个失败的 thunk,调用站点就需要明确检查 F 类型的返回值。

默认的失败提示符通常会引发一个错误,并且具有 (-> Nothing) 的类型。 对于这样的失败推理, (U F (Matrix Number)) 等同于 (Matrix Number), 因为 Nothing 是每个类型的一部分。 (在 Racket 中,任何表达式都可能引发错误)。 因此,在这种情况下,没有必要对 F 类型的值进行明确的测试 (当然,它们可以用 with-handlers 或类似的方式来捕获)。

7.1.3 广播

与数组操作不同,当给定不同轴长的矩阵时,顺时针矩阵操作 不会 broadcast 其参数:
> (matrix+ (identity-matrix 2) (matrix [[10]]))

matrix-map: matrices must have the same shape; given (array

#[#[1 0] #[0 1]]) (array #[#[10]])

如果你需要广播,请使用数组操作:
> (array+ (identity-matrix 2) (matrix [[10]]))

7.1.4 严格性

由 math/matrix 导出的函数根据 array-strictness 参数的值返回 strict 或 nonstrict 的数组。 详见 非限制性数组 。
