6.6 类型、谓词和访问器

(Settable-Array A)

其元素可以被突变的数组的父类型。 像 array-set! 和 array-slice-set! 这样的函数接受这种类型的参数。 子类型的例子有 Mutable-Array, FlArray 和 FCArray 。

这个类型是 invariant, 也就是说,如果 A 和 B 是不同的类型, 即使 A 是 B 的一个子类型, (Settable-Array A) 也 不是 (Settable-Array B) 的一个子类型:
> (define arr (mutable-array #[1 2 3 4 5]))
> arr

(Mutable-Array A)

可变数组的类型。它的类型参数是数组元素的类型。

这种类型的数组将其元素存储在一个 (Vectorof A) 中:
> (define arr (mutable-array #[#[1 2] #[3 4]]))
> (vector-set! (mutable-array-data arr) 0 -10)
> arr

- : #(struct:Mutable-Array

      (Indexes

       Index

       (Boxof Boolean)

       (-> Void)

       (-> Indexes Integer)

       (-> Indexes Integer Void)

       (Vectorof Integer))

      #<syntax:/home/ice/racket-8.9-2023-07-14/share/pkgs/math-lib/math/private/array/typed-mutable-array.rkt:14:13 prop:custom-write>)

syntax

Indexes

math/array 函数 produced 的数组形状和数组索引的类型。 定义为 (Vectorof Index) 。

Example:
> (array-shape (array #[#[#[0]]]))

- : Indexes

'#(1 1 1)

syntax

In-Indexes

math/array 函数 accepted 的数组形状和数组索引的类型。 定义为 (U Indexes (Vectorof Integer)) 。

Examples:
> (define ds #(3 2))
> ds

- : (Immutable-Vector Positive-Byte Positive-Byte)

'#(3 2)

> (make-array ds (void))

- : #(struct:Array

      (Indexes Index (Boxof Boolean) (-> Void) (-> Indexes Void))

      #<syntax:/home/ice/racket-8.9-2023-07-14/share/pkgs/math-lib/math/private/array/typed-array-struct.rkt:56:13 prop:equal+hash>

(array #[#[#<void> #<void>] #[#<void> #<void>] #[#<void> #<void>]])

这使得接受索引的函数更容易使用, 因为说服 Typed Racket 相信一个向量包含 Integer 元素比相信一个向量包含 Index 元素更容易。

In-Indexes 不被定义为 (Vectorof Integer) , 因为像 Vector 和 Vectorof 这样的可变容器类型是不变的。 特别是, (Vectorof Index) 不是 (Vectorof Integer) 的一个子类型:
> (define js ((inst vector Index) 3 4 5))
> js

Array, Settable-Array 和 Mutable-Array 类型的谓词。

因为 Settable-Array 和它的子类型是不变的, settable-array? 和它的子类型的谓词在发生类型化中一般没有用。 例如,如果我们知道我们有一个 Array ,但是如果它恰好是一个 Mutable-Array, 我们想对它进行不同的处理,那么我们基本上就没有什么好运气了:
> (: maybe-array-data (All (A) ((Array A) -> (U #f (Vectorof A)))))
> (define (maybe-array-data arr)
    (cond [(mutable-array? arr)  (mutable-array-data arr)]
          [else  #f]))

eval:106:0: Type Checker: Polymorphic function

`mutable-array-data' could not be applied to arguments:

Argument 1:

  Expected: (Mutable-Array A)

  Given:    (Struct Mutable-Array)

  in: #f

一般来说,带有 Struct 过滤器的谓词不会让条件分支访问结构的访问器。 因为 Settable-Array 和它的后代是不变的,所以它们的谓词有 Struct 过滤器:

> settable-array?

> mutable-array?

#<procedure:Mutable-Array?>

procedure

(array-shape arr) → Indexes

  arr : (Array A)
返回 arr 的 shape, 一个包含 arr 轴的长度的索引向量。

Examples:
> (array-shape (array 0))

- : Indexes

'#()

> (array-shape (array #[0 1]))

- : Indexes

'#(2)

> (array-shape (array #[#[0 1]]))

- : Indexes

'#(1 2)

> (array-shape (array #[]))

- : Indexes

'#(0)

procedure

(array-size arr) → Index

  arr : (Array A)
返回 arr 中的元素数,是其轴长的乘积。

Examples:
> (array-size (array 0))

- : Integer [more precisely: Index]

1

> (array-size (array #[0 1]))

- : Integer [more precisely: Index]

2

> (array-size (array #[#[0 1]]))

- : Integer [more precisely: Index]

2

> (array-size (array #[]))

- : Integer [more precisely: Index]

0

procedure

(array-dims arr) → Index

  arr : (Array A)
返回 arr 的尺寸数。 相当于 (vector-length (array-shape arr)) 。
