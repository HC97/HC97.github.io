8 统计函数

Neil Toronto <ntoronto@racket-lang.org>

 (require math/statistics) package: math-lib

这个模块导出了计算 statistics 的函数,意思是样本集合的摘要值,以及管理加权或未加权样本序列的函数。

大多数计算统计数据的函数都接受与样本值一一对应的非负整数序列。 这些被用作权重;相当于计数、伪计数或非标准化的概率。 虽然这使得加权样本的工作变得很容易,但它在偏差校正中引入了一些微妙的问题。 特别是,默认情况下,中心矩必须在没有偏倚校正的情况下进行计算。 讨论见 预期值 。

8.1 预期值

本节中记录的计算更高中心矩的函数,如 variance,stddev 和 skewness, 可以选择对它们的估计值进行偏差修正。 例如,当 variance 被赋予参数 #:bias #t 时, 它将结果乘以 (/ n (- n 1)), 其中 n 是样本的数量。

然而,对于加权样本,"偏差校正 "的含义变得不那么清楚。 通常情况下,权重代表计数,所以当时刻估计函数收到 #:bias #t 时, 它们会将其解释为 “对 n 使用 ws 之和”。 在下面的例子中,样本 4 首先被计数了两次,然后被赋予权重 2; 因此,在这两种情况下 n = 5 :
> (variance '(1 2 3 4 4) #:bias #t)

- : Real [more precisely: Nonnegative-Real]

17/10

> (variance '(1 2 3 4) '(1 1 1 2) #:bias #t)

- : Real [more precisely: Nonnegative-Real]

17/10

然而,样本权重往往不代表计数。 对于这些情况, #:bias 关键词后面可以跟上一个实值的伪计数,这个伪计数被用于 n:
> (variance '(1 2 3 4) '(1/2 1/2 1/2 1) #:bias 5)

- : Real [more precisely: Nonnegative-Real]

17/10

因为在没有用户指导的情况下,加权样本的偏差校正的大小是无法知道的,在所有情况下,偏差参数默认为 #f 。

8.2 运行预期值

statistics 对象允许在 O(1) 空间内计算样本序列的最小、最大、计数、平均值、方差、偏度和过剩峰度。

要使用它,从 empty-statistics 开始, 然后使用 update-statistics 来获得一个具有更新值的新的统计对象。 使用 statistics-min, statistics-mean 和类似的函数来获得当前的估计值。

Example:
> (let* ([s  empty-statistics]
         [s  (update-statistics s 1)]
         [s  (update-statistics s 2)]
         [s  (update-statistics s 3)]
         [s  (update-statistics s 4 2)])
    (values (statistics-mean s)
            (statistics-stddev s #:bias #t)))

- : (values Flonum Flonum) [more precisely: (Values Flonum Nonnegative-Flonum)]

2.8

1.3038404810405297

struct

(struct sample-bin (min max values weights))

  min : B
  max : B
  values : (Listof A)
  weights : (U #f (Listof Nonnegative-Real))
代表一个 bin, 或一个区间内的一组样本的总顺序。 值和边界有不同的类型,以允许 bin-samples/key 根据其值的函数来分组元素。

procedure

(bin-samples bounds lte? xs ws) → (Listof (sample-bin A A))

  bounds : (Sequenceof A)
  lte? : (A A -> Any)
  xs : (Sequenceof A)
  ws : (U #f (Sequenceof Real))
类似于 (sort xs lte?), 但另外将样本分组为 bins。 在对 xs 进行分选之前,对 bins 的 bounds 进行排序。

如果 n = (length bounds), 那么 bin-samples 至少返回 返回 (- n 1) 个bins,每一对相邻的(排序的) bounds 有一个。 如果 xs 中的一些值小于最小的边界,它们会被分组到前面的一个bin中。 如果有些值大于最大的边界,它们就被分组到最后的一个仓中。

Examples:
> (bin-samples '() <= '(0 1 2 3 4 5 6))

- : (Listof

     #(struct:sample-bin

       (Byte Byte (Listof Byte) (U (Listof Nonnegative-Real) False))))

(list (sample-bin 0 6 '(0 1 2 3 4 5 6) #f))

> (bin-samples '(3) <= '(0 1 2 3 4 5 6))

- : (Listof

     #(struct:sample-bin

       (Byte Byte (Listof Byte) (U (Listof Nonnegative-Real) False))))

(list (sample-bin 0 3 '(0 1 2 3) #f) (sample-bin 3 6 '(4 5 6) #f))

> (bin-samples '(2 4) <= '(0 1 2 3 4 5 6))

- : (Listof

     #(struct:sample-bin

       (Byte Byte (Listof Byte) (U (Listof Nonnegative-Real) False))))

(list

 (sample-bin 0 2 '(0 1 2) #f)

 (sample-bin 2 4 '(3 4) #f)

 (sample-bin 4 6 '(5 6) #f))

> (bin-samples '(2 4) <=
               '(0 1 2 3 4 5 6)
               '(10 20 30 40 50 60 70))

- : (Listof

     #(struct:sample-bin

       (Byte Byte (Listof Byte) (U (Listof Nonnegative-Real) False))))

(list

 (sample-bin 0 2 '(0 1 2) '(10 20 30))

 (sample-bin 2 4 '(3 4) '(40 50))

 (sample-bin 4 6 '(5 6) '(60 70)))

如果 lte? 是小于或等于的关系, 则这些仓代表半开的区间 (min, max] (可能除了第一个,它可能是封闭的)。 如果 lte? 是小于等于的关系, 那么仓代表半开的区间 [min, max) (除了可能是最后一个,可能是封闭的)。 在任何一种情况下,应用于 bounds 和 xs 的排序都是稳定的。

因为在概率测量中使用的区间通常在左边是开放的,所以倾向于对 lte? 使用小于或等于的关系。

如果 ws 是 #f, bin-samples 返回具有 #f 权重的 bin。

8.5 阶次统计

procedure

(sort-samples lt? xs) → (Listof A)

  lt? : (A A -> Any)

(sort-samples lt? xs ws)
 → (Values (Listof A) (Listof Nonnegative-Real))

> (sort-samples < '(5 2 3 1))

- : (Listof Positive-Byte)

'(1 2 3 5)

> (sort-samples < '(5 2 3 1) '(50 20 30 10))

- : (values (Listof Positive-Byte) (Listof Nonnegative-Real))

'(1 2 3 5)

'(10 20 30 50)

> (sort-samples < '(5 2 3 1) #f)

- : (values (Listof Positive-Byte) (Listof Nonnegative-Real))

'(1 2 3 5)

'(1 1 1 1)

因为 sort-samples 是根据 sort 定义的, 只有当 lt? 是严格的小于关系时,才能保证排序是稳定的。

(median lt? xs [ws]) → A

  lt? : (A A -> Any)
  xs : (Sequenceof A)
  ws : (U #f (Sequenceof Real)) = #f
相当于 (quantile 1/2 lt? xs ws) 。

procedure

(quantile p lt? xs [ws]) → A

  p : Real
  lt? : (A A -> Any)
  xs : (Sequenceof A)
  ws : (U #f (Sequenceof Real)) = #f
计算样本 xs 所代表的经验 cdf 的倒数,这些样本可以选择用 ws 加权。

Examples:
> (quantile 0 < '(1 3 5))

- : Integer [more precisely: Positive-Byte]

1

> (quantile 0.5 < '(1 2 3 4))

- : Integer [more precisely: Positive-Byte]

2

> (quantile 0.5 < '(1 2 3 4) '(0.25 0.2 0.2 0.35))

- : Integer [more precisely: Positive-Byte]

3

如果 p = 0, quantile 返回排序关系 lt? 下 xs 中最小的元素。 如果 p = 1, 它将返回最大的元素。

对于加权样本, quantile 将 xs 和 ws 放在一起排序(使用 sort-samples), 然后找到其累积权重的比例大于或等于 p 的最小 x 。

对于非加权样本, quantile 使用 quickselect 算法来找到如果 xs 被排序后会处于索引 (ceiling (- (* p n) 1)) 的元素, 其中 n 是 xs 的长度。

本节中的函数支持蒙特卡洛模拟;例如,量化从样本中估计的统计数据的不确定性。

如果 xs 是随机变量样本,那么

(define θ (mean xs ws))

(mc-variance xs ws #:bias bias)
(mc-stddev xs ws #:bias bias)

估计从 1000 个样本计算的平均值的标准差的两种不同方法是
> (mc-stddev (sample (normal-dist 0 1) 1000))

- : Real [more precisely: Nonnegative-Real]

0.031083422191045082

> (stddev (for/list : (Listof Real) ([_  (in-range 100)])
            (mean (sample (normal-dist 0 1) 1000))))

- : Real [more precisely: Nonnegative-Real]

0.030039267400410245

在这种情况下,使用 mc-stddev 要快 100 倍,而在大多数统计应用中,将一个抽样过程复制 100 次是不可行的。
