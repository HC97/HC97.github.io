1 常数和基本函数

Neil Toronto <ntoronto@racket-lang.org>

 (require math/base) package: math-lib

为了方便起见, math/base 重新输出了 racket/math, 同时也提供了下面的数值文档。

一般来说, math/base 提供的函数是 elementary 函数, 或那些可以用有限数量的算术运算、对数、指数、三角函数和常数来定义的函数。 对于其他函数,见 math/special-functions 和 math/distributions 。

1.1 常数

如果你需要比下面的 flonums 更精确的近似值,请参见, 例如, phi.bf 和 bigfloat->rational 。

value

phi.0 : Positive-Flonum

φ 的近似值,即 golden ratio 。
> phi.0

1.618033988749895

value

euler.0 : Positive-Flonum

e 近似值,或 Euler’s number 。
> euler.0

2.718281828459045

> (exp 1)

2.7182818284590455

value

gamma.0 : Positive-Flonum

γ 的近似值,即 Euler-Mascheroni constant 常数。
> gamma.0

0.5772156649015329

value

catalan.0 : Positive-Flonum

G 的近似值,即 Catalan’s constant 。
> catalan.0

0.915965594177219

1.2 函数

procedure

(float-complex? v) → Boolean

  v : Any
当 v 是 Float-Complex 类型时返回 #t 。类似于 flonum? 。

procedure

(number->float-complex x) → Float-Complex

  x : Number
返回一个具有 flonum 实部和 flonum 虚部的新复数。 类似于 real->double-flonum 。

procedure

(power-of-two? x) → Boolean

  x : Real
当 x 是 2 的整数次方时,返回 #t 。

Examples:
> (power-of-two? 1.0)

#t

> (power-of-two? 1/2)

#t

> (power-of-two? (flnext 2.0))

#f

procedure

(asinh z) → Number

  z : Number

procedure

(acosh z) → Number

  z : Number

procedure

(atanh z) → Number

  z : Number
sinh, cosh 和 tanh 的逆运算, 在 racket/math 中定义 (并由 math/base 重新输出)。

procedure

(sum xs) → Real

  xs : (Listof Real)
就像 (apply + xs), 但在添加不精确的数字时只产生一次舍入错误。 (事实上, xs 中的不精确数字是用 flsum 单独求和的)。

1.3 随机数的产生

procedure

(random-natural k) → Natural

  k : Integer
返回一个小于 k 的随机自然数,它必须是正数。 当 k 可能大于 4294967087 时, 使用 (random-natural k) 而不是 (random k) 。

procedure

(random-integer a b) → Integer

  a : Integer
  b : Integer
返回一个随机整数 n ,使得 (<= a n) 和 (< n b) 。

procedure

(random-bits num) → Natural

  num : Integer
返回一个小于 (expt 2 num) 的随机自然数; num 必须为非负数。 对于2的幂,这比使用 random-natural 要快, 因为 random-bits 是用随机比特实现的,使用的是有偏见的拒绝采样。

作为一个使用的例子, bfrandom 返回的数字的有效值是由 (random-bits (bf-precision)) 选择的。

1.4 测量误差

procedure

(absolute-error x r) → Real

  x : Real
  r : Real
通常使用精确的有理数计算 (abs (- x r)) ,但也特别处理非有理数,如 +inf.0 。

Examples:
> (absolute-error 1/2 1/2)

0

> (absolute-error 0.14285714285714285 1/7)

7.93016446160826e-18

> (absolute-error +inf.0 +inf.0)

0.0

> (absolute-error +inf.0 +nan.0)

+inf.0

> (absolute-error 1e-20 0.0)

1e-20

> (absolute-error (- 1.0 (fl 4999999/5000000)) 1/5000000)

5.751132903242251e-18

procedure

(relative-error x r) → Real

  x : Real
  r : Real
衡量相对于 r 的大小,近似值 x 与正确值 r 的接近程度。

这个函数通常使用精确的有理数计算 (abs (/ (- x r) r)) , 但也可以处理非有理数,如 +inf.0 ,以及 r = 0 。

Examples:
> (relative-error 1/2 1/2)

0

> (relative-error 0.14285714285714285 1/7)

5.551115123125783e-17

> (relative-error +inf.0 +inf.0)

0.0

> (relative-error +inf.0 +nan.0)

+inf.0

> (relative-error 1e-20 0.0)

+inf.0

> (relative-error (- 1.0 (fl 4999999/5000000)) 1/5000000)

2.8755664516211255e-11

在最后两个例子中,相对误差很高,因为结果接近零。 (用 absolute-error 比较相同的例子)。 因为 flonum 在零附近特别密集,这使得相对误差比绝对误差更能衡量 flonum 近似的误差。 一个更好的是 ulps 的误差;见 flulp-error 。
