1 运行 Web Servlets

运行 Web servlet 的方法有很多。

1.1 即时 Servlets

 #lang web-server/insta package: web-server-doc

让 Servlet 在 Web 服务器中运行的最快方法是使用 DrRacket 中的 "Insta" 语言。 在 DrRacket 中输入以下内容:

#lang web-server/insta
 
(define (start req)
  (response/xexpr
   `(html (head (title "Hello world!"))
          (body (p "Hey out there!")))))

并按下 Run 。一个 Web 浏览器将打开,显示你的新 Servlet。 这个 servlet 只能从你的本地机器上访问。

在幕后,DrRacket 使用 serve/servlet 来启动一个新的服务器, 该服务器使用你的 start 函数作为 servlet。 你会得到整个 web-server/servlet 的 API。

下面的 API 被提供用来定制服务器实例:

procedure

(no-web-browser) → void

调用它将指示 DrRacket 在你按下 Run 时 不 启动 Web 浏览器。

procedure

(static-files-path path) → void

  path : path-string?
这将指示 Web 服务器从 path 中提供静态文件,如样式表和图片。

如果你想对特定参数进行更多控制,请继续阅读 web-server/servlet-env 。

1.2 简单的单一 Servlet 服务器

 (require web-server/servlet-env) package: web-server-lib

Web Server 提供了一种快速配置和启动 servlet 的方法, 比 web-server/insta 提供的定制能力更强。 这是由 web-server/servlet-env 模块提供的。

1.2.1 示例

下面是一个简单的使用例子:
#lang racket
(require web-server/servlet
         web-server/servlet-env)
 
(define (start req)
  (response/xexpr
   `(html (head (title "Hello world!"))
          (body (p "Hey out there!")))))
 
(serve/servlet start)

与 web-server/insta 语言不同, start 不是一个特殊的标识符,所以我们完全可以把这个例子写成:
#lang racket
(require web-server/servlet
         web-server/servlet-env)
 
(define (my-app req)
  (response/xexpr
   `(html (head (title "Hello world!"))
          (body (p "Hey out there!")))))
 
(serve/servlet my-app)

让我们看看 serve/servlet 允许的一些定制。

假设你想把端口改成其他的东西,把最后一行改成:
(serve/servlet my-app
               #:port 8080)

假设你想接受来自外部机器的连接:
(serve/servlet my-app
               #:listen-ip #f)

默认情况下,你的 servlet 的 URL 是 "http://localhost:8000/servlets/standalone.rkt", 假设你想把它改成 "http://localhost:8000/hello.rkt":
(serve/servlet my-app
               #:servlet-path "/hello.rkt")

假设你想让它捕捉顶级的请求:
(serve/servlet my-app
               #:servlet-regexp #rx"")
或者,也许只是一些漂亮的顶级名称:
(serve/servlet my-app
               #:servlet-path "/main")

假设你想使用在你的桌面 ("/Users/jay/Desktop/") 上找到的一个样式表 ("style.css") :
(serve/servlet my-app
               #:extra-files-paths
               (list
                (build-path "/Users/jay/Desktop")))
这些文件是在 #:server-root-path "htdocs" 目录中的文件 之外 提供的。 你可以传递任何数量的额外路径。

如果你想在 Web 应用程序的启动脚本中使用 serve/servlet , 并且不希望打开浏览器或打印 DrRacket 的横幅,那么你可以写:
(serve/servlet my-app
               #:command-line? #t)

1.2.1.1 无状态的 Servlets

假设你想为提供 start 的无状态 Web servlet "servlet.rkt" 启动一个服务器:
#lang racket
(require "servlet.rkt"
         web-server/servlet-env)
 
(serve/servlet start #:stateless? #t)

你也可以把对 serve/servlet 的调用直接放在 web-server 模块中:
#lang web-server
(require web-server/servlet-env)
 
(define (start req)
  (start
   (send/suspend
    (lambda (k-url)
      (response/xexpr
       `(html (body (a ([href ,k-url]) "Hello world!"))))))))
 
(serve/servlet start #:stateless? #t)
像往常一样,你甚至不需要保存该文件。

这将设置并启动一个相当默认的服务器实例。

start 作为一个 servlet 被加载,并对符合 servlet-regexp 的请求做出响应。 执行 servlet 的当前目录是 servlet-current-directory 。

如果 launch-browser? 为 true, 那么 Web 浏览器将被打开到 "http://localhost:<port><servlet-path>" 。 如果提供了 servlet-regexp ， servlet-path , 就没有其他用途。

如果 quit? 为真,那么 URL "/quit" 就会结束服务器。

如果 stateless? 为真,那么 servlet 将作为一个无状态的

#lang web-server

模块, stuffer 被用作 stuffer 。

serve/servlet 是比 serve/launch/wait, dispatch/servlet 和一些标准 调度器 更简单的接口。 一些选项,如 port 和 safety-limits , 被透明地传递给 serve/launch/wait 。 一些高级定制需要直接使用 web-server 的这些底层部分。 然而,许多简单的定制则不需要,本节的其余部分将对此进行描述。

服务器在 listen-ip 和端口 port 上监听。 如果 listen-ip 是 #f, 那么服务器会接受所有监听机器地址的连接。 否则,服务器只接受与给定字符串相关的接口的连接。 例如,提供 "127.0.0.1" (默认值)作为 listen-ip , 会创建一个只接受来自本地机器的 "127.0.0.1" (回环接口)连接的服务器。

如果 ssl-cert 和 ssl-key 不是假的,那么服务器就以 HTTPS 模式运行, ssl-cert 和 ssl-key 作为证书和私钥。

该 servlet 是以 manager 作为其延续管理器加载的。 (默认的管理器将内存量限制在 64MB,并处理内存压力,在 make-threshold-LRU-manager 文档中讨论过)。

服务器文件根植于 server-root-path （默认是 web-server 集合中的 "default-web-root" 目录)。 除了 server-root-path 下的 "htdocs" 目录外, 还可以用 extra-files-paths 提供文件路径。 这些路径首先被检查,按照它们在列表中出现的顺序。

其他的 servlet 会从 servlets-root 提供服务。 由 servlet-namespace 指定的模块在 servlets-root 中找到的 servlets 和当前命名空间之间共享(因此也包括 start 程序。)

如果一个文件找不到, file-not-found-responder 被用来生成一个错误响应。 如果一个 servlet 加载失败,就会使用 responders-servlet-loading 。 如果一个 servlet 在其操作过程中出错,则使用 responders-servlet 。

如果 banner? 为真,那么就会打印一个信息性的横幅。 你可能想在从命令行运行时使用它,在这种情况下, command-line? 选项控制类似的选项。

MIME 类型是在 mime-types-path 中查找的。 默认情况下,使用 server-root-path 中的 "mime.types" 文件, 但如果该文件不存在,则使用 Web 服务器中的文件。 当然,如果给出了一个路径,那么它将覆盖这一行为。

如果给出了 log-file ,那么它就会使用 log-format 作为格式来记录请求。 允许的格式是那些 log-format->format 所允许的。 如果 log-format 是一个函数,它被直接用来呈现日志条目。 如果 log-file 是一个文件路径,给定的文件将在不同的线程中打开并写入。 如果 log-file 是一个 output-port?, 日志将直接写入该端口。

如果 connection-close? 是 #t, 那么每个连接在一次请求后就会被关闭。 否则,客户端会根据其使用的 HTTP 版本来决定。

Changed in version 1.3 of package web-server-lib: 增加了对提供 log-file 作为输出端口的支持。
Changed in version 1.6: 增加了与 serve/launch/wait 一样的 safety-limits 参数: 见 compatability note 。

1.3 命令行工具

Web Server 提供了一个命令行工具:

  plt-web-server [-f <file-name> -p <port> -a <ip-address> --ssl]

可选的 file-name 参数指定了 configuration-table S 表达式的路径(语法文档见 configuration-table->sexpr 。) 如果没有提供这个参数,就会使用服务器提供的默认配置。 可选的 port 和 ip-address 数覆盖了 configuration-table 的相应部分。 如果提供了 ssl 选项,那么服务器将使用 HTTPS, 在当前目录中使用 "server-cert.pem" 和 "private-key.pem" ,443 为默认端口。 (关于 SSL 实现的细节,请参见 openssl 模块)。

configuration-table 被交给 configuration-table->web-config@ , 用来构建一个 web-config^ 单元,并与 web-server@ 单元相连接。 由此产生的单元被调用,并且服务器一直运行到进程被杀死。
