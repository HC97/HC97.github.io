7 模板: 视图的分离

 (require web-server/templates) package: web-server-lib

Web Server 提供了一个强大的 Web 模板系统,用于分离 Web 应用程序的表现逻辑, 并使非程序员能够为基于 Racket 的 Web 应用程序作出贡献。

虽然这里的所有例子都是生成 HTML, 但模板语言和它所基于的 Text Generation 可以用来生成任何基于文本的格式: C、SQL、表单邮件、报告等。

7.1 静态

假设我们有一个文件 "static.html" 的内容:

  <html>

   <head><title>Fastest Templates in the West!</title></head>

   <body>

    <h1>Bang!</h1>

    <h2>Bang!</h2>

   </body>

  </html>

如果我们在代码中写下以下内容:

(include-template "static.html")

那么 "static.html" 的内容将 在编译时 被读取并编译成一个 racket 程序, 以字符串的形式返回 "static.html" 的内容:

"<html>\n  <head><title>Fastest Templates in the West!</title></head>\n  <body>\n    <h1>Bang!</h1>\n    <h2>Bang!</h2>\n  </body>\n</html>"

7.2 动态

include-template 给了模板对包含程序的 完整词法上下文 的访问权。 这个上下文可以通过 @ 语法 语法来访问。例如,如果 "simple.html" 包含:

  <html>

   <head><title>Fastest @thing in the West!</title></head>

   <body>

    <h1>Bang!</h1>

    <h2>Bang!</h2>

   </body>

  </html>

那么
(let ([thing "Templates"])
  (include-template "simple.html"))
评价为与静态例子相同的内容。

对于如何填充模板的词法上下文没有任何限制。 例如,你可以通过在一个函数中包装模板的包含物来建立模板抽象:
(define (fast-template thing)
  (include-template "simple.html"))
 
(fast-template "Templates")
(fast-template "Noodles")
评估为两个具有可预测内容的字符串:

  <html>

   <head><title>Fastest Templates in the West!</title></head>

   <body>

    <h1>Bang!</h1>

    <h2>Bang!</h2>

   </body>

  </html>

和

  <html>

   <head><title>Fastest Noodles in the West!</title></head>

   <body>

    <h1>Bang!</h1>

    <h2>Bang!</h2>

   </body>

  </html>

此外,对模板使用的 Racket 没有任何限制:它们可以使用宏、结构、延续标记、线程等。 然而,最终返回的 Racket 值必须是 Text Generation 可打印的。 例如,考虑以下对 fast-template 的不同调用的标题行的输出:

  • (fast-template 'Templates)

      ...<title>Fastest Templates in the West!</title>...

  • (fast-template 42)

      ...<title>Fastest 42 in the West!</title>...

  • (fast-template (list "Noo" "dles"))

      ...<title>Fastest Noodles in the West!</title>...

  • (fast-template (lambda () "Thunks"))

      ...<title>Fastest Thunks in the West!</title>...

  • (fast-template (delay "Laziness"))

      ...<title>Fastest Laziness in the West!</title>...

  • (fast-template (fast-template "Embedding"))

      ...<title>Fastest ...<title>Fastest Embedding in the West!</title>... in the West!</title>...

7.3 缺点

7.3.1 @ Syntax: @ 字符,标识符和空格

要在模板输出中获得 @ 字符,你必须转义, 因为 @ 是 @ 语法 的转义字符。例如,要获得:

  <head><title>Fastest @s in the West!</title></head>

你必须写

  <head><title>Fastest @"@"s in the West!</title></head>

作为你的模板:字面的 @ 必须用 @"@" 代替。 (注意,双引号基本上是一个 Racket 表达式,它也可以用于较长的字符串)。

@ 语法 会读取 Racket 标识符,所以它不会将标识符终止在标点符号或 XML 角括号上。所以、

  <head><title>Fastest @thing in the @place!</title></head>

会抱怨标识符 place!</title></head> 是未定义的。 你可能想颠覆这一点,在标识符后面加一个空格:

  <head><title>Fastest @thing in the @place !</title></head>

这将消除错误,但生成的 HTML 将不像你想要的那样。有时这并不重要,但有时却很重要。 最安全的做法是用 | 来明确限定标识符:

  <head><title>Fastest @thing in the @|place|!</title></head>

如果你打算经常使用模板,你应该熟悉 @ 语法 的细节。

7.3.2 迭代

由于模板被编译成 Racket 程序,所以只有其结果会被打印出来。例如,假设我们有了这个模板:

  <table>

   @for[([c clients])]{

    <tr><td>@(car c), @(cdr c)</td></tr>

   }

  </table>

如果这被包含在一个词法上下文中,并将 clients 绑定到

(list (cons "Young" "Brigham") (cons "Smith" "Joseph"))

那么该模板将被打印为:

  <table>

  </table>

因为 for 并不返回主体的值。 假设我们将模板改为使用 for/list (将它们合并为一个列表):

  <table>

   @for/list[([c clients])]{

    <tr><td>@(car c), @(cdr c)</td></tr>

   }

  </table>

现在的结果是:

  <table>

  </td></tr>

   </td></tr>

  </table>

因为结果中只包括 for/list 主体的最终表达。 我们可以通过在主体中使用 list 来捕获所有的子表达式:

  <table>

   @for/list[([c clients])]{

    @list{

     <tr><td>@(car c), @(cdr c)</td></tr>

    }

   }

  </table>

现在的结果是:

  <table>

   <tr><td>Young, Brigham</td></tr>

   <tr><td>Smith, Joseph</td></tr>

  </table>

模板库提供了一种语法形式来为你处理这个问题,叫做 in:

  <table>

   @in[c clients]{

    <tr><td>@(car c), @(cdr c)</td></tr>

   }

  </table>

请注意它也避免了第二行中荒谬的标点符号。

7.3.3 转移

感谢Michael W.对这一节的贡献。

因为模板对很多东西都有用(脚本、CSS、HTML等),Web 服务器不认为模板是用于类似 XML 的内容。 因此当模板被展开时,默认情况下不会进行 XML 转义。 谨防 cross-site scripting 漏洞! 例如,假设一个 servlet 提供以下模板,其中 some-variable 是一个由客户提供的输入字符串:

  <html>

   <head><title>Fastest Templates in the West!</title></head>

   <body>

    @some-variable

   </body>

  </html>

如果该 servlet 包含如下内容:

(let ([some-variable (get-input-from-user)])
 (include-template "static.htm"))

没有什么可以阻止攻击者输入 <script type="text/javascript">...</script> 来使模板扩展成:

  <html>

   <head><title>Fastest Templates in the West!</title></head>

   <body>

    <script type="text/javascript">...</script>

   </body>

  </html>

现在,服务器将把攻击者的代码发送给数百万的无辜用户。 为了防止在提供 HTML 时发生这种情况,可以使用 xml 模块中的 xexpr->string 函数。

这可以在servlet中完成:

(require xml)
 
(let ([some-variable (xexpr->string (get-input-from-user))])
 (include-template "static.htm"))

或者,让模板负责自己的转义:

  <html>

   <head><title>Fastest Templates in the West!</title></head>

   <body>

    @(xexpr->string some-variable)

   </body>

  </html>

改进后的版本显示为:

  <html>

   <head><title>Fastest Templates in the West!</title></head>

   <body>

     &lt;script type=\"text/javascript\"&gt;...&lt;/script&gt;

   </body>

  </html>

在编写模板时,一定要记得转义用户提供的输入。

7.4 HTTP 响应

从模板生成 HTTP 响应的最快方法是使用 response? 结构:
(response/output
 (λ (op) (display (include-template "static.html") op)))

最后,如果你想在一个更大的 X-expression 中包含模板的内容:

`(html ,(include-template "static.html"))

将导致字面字符串被包含(和实体转义)。 如果你实际上希望模板不被转义,那么使用 include-template/xml 来断言内容是有效的 XML。

`(html ,(include-template/xml "static.html"))

7.5 API 细节

syntax

(include-template path-spec)

(include-template #:command-char command-char path-spec)
在 path-spec 处使用 @ 语法 语法在包围的词法上下文中编译模板。 path-spec 与 include 使用的格式相同。 使用 command-char 关键字来定制转义字符。

例子:
(include-template "static.html")
(include-template #:command-char #\$ "dollar-static.html")

syntax

(include-template/xml path-spec)

(include-template/xml #:command-char command-char path-spec)
像 include/template 一样,但扩展到 cdata 结构。

Added in version 1.3 of package web-server-lib.

syntax

(in x xs e ...)

展开到
(for/list ([x xs])
  (begin/text e ...))

模板例子:

  @in[c clients]{

   <tr><td>@(car c), @(cdr c)</td></tr>

  }

Racket 例子:

(in c clients "<tr><td>" (car c) ", " (cdr c) "</td></tr>")

7.6 转换实例

Al Church 用 Racket 维护一个博客已经有好几年了,他想转换为 web-server/templates 。

他使用的数据结构被定义为:
(define-struct post (title body))
 
(define posts
  (list
   (make-post
    "(Y Y) Works: The Why of Y"
    "Why is Y, that is the question.")
   (make-post
    "Church and the States"
    "As you may know, I grew up in DC, not technically a state.")))
实际上,Al Church 对这些帖子进行了编码,但出于解释的原因,我们将使用结构。

他把他的代码分为演示函数和逻辑函数。 我们先看一下演示函数。

第一个表现函数定义了所有页面的共同布局。
(define (template section body)
  (response/xexpr
   `(html
     (head (title "Al's Church: " ,section))
     (body
      (h1 "Al's Church: " ,section)
      (div ([id "main"])
           ,@body)))))

这里需要注意的一点是 body 参数上的 unquote-splicing 。 这表明 body 是 X-expression 的列表。 如果他不小心只用了 unquote ,那么在将返回值转换为 HTTP 响应时就会出现错误。

(define (blog-posted title body k-url)
  `((h2 ,title)
    (p ,body)
    (h1 (a ([href ,k-url]) "Continue"))))

下面是一个简单的 body 的例子,它使用一个 X-expression 的列表来显示新发布的博客条目,然后继续重新显示主页面。 我们来看看一个更复杂的主体:

(define (blog-posts k-url)
  (append
   (apply append
          (for/list ([p posts])
            `((h2 ,(post-title p))
              (p ,(post-body p)))))
   `((h1 "New Post")
     (form ([action ,k-url])
           (input ([name "title"]))
           (input ([name "body"]))
           (input ([type "submit"]))))))

这个函数显示了一些 X-expression 所需的常见模式。 首先, append 用于组合不同的 X-expression 列表。 第二, apply append 用于折叠和组合 for/list 的结果, 其中每个迭代的结果都是一个 X-expression 列表。 我们将看到,这些模式在模板中是不必要的。 当 Al 试图在他的博客的所有页面上添加 CSS 样式和一些来自 Google Analytics 的 JavaScript 时,另一个恼人的模式出现了。 他把 template 的功能改为:

(define (template section body)
  (response/xexpr
   `(html
     (head
      (title "Al's Church: " ,section)
      (style ([type "text/css"])
             "body {margin: 0px; padding: 10px;}"
             "#main {background: #dddddd;}"))
     (body
      (script
       ([type "text/javascript"])
       ,(make-cdata
         #f #f
         "var gaJsHost = ((\"https:\" =="
         "document.location.protocol)"
         "? \"https://ssl.\" : \"http://www.\");"
         "document.write(unescape(\"%3Cscript src='\" + gaJsHost"
         "+ \"google-analytics.com/ga.js' "
         "type='text/javascript'%3E%3C/script%3E\"));"))
      (script
       ([type "text/javascript"])
       ,(make-cdata
         #f #f
         "var pageTracker = _gat._getTracker(\"UA-YYYYYYY-Y\");"
         "pageTracker._trackPageview();"))
      (h1 "Al's Church: " ,section)
      (div ([id "main"])
           ,@body)))))

通过使用这里的字符串,其中一些问题就会消失,正如 读取字符串 的文档中所描述的。

我们注意到的第一件事是,将 CSS 编码为字符串是相当原始的。 用字符串编码 JavaScript 甚至更糟糕,原因有二:首先,我们更有可能需要手动转义字符,如 "; 其次,我们需要使用 CDATA 对象,因为大多数 JavaScript 代码使用了 "需要" 在 XML 中转义的字符, 如 &,但如果这些字符被实体编码,大多数浏览器会失败。这些都是有了模板就会消失的问题。

在转向模板之前,让我们看看逻辑功能:
(define (extract-post req)
  (define binds
    (request-bindings req))
  (define title
    (extract-binding/single 'title binds))
  (define body
    (extract-binding/single 'body binds))
  (set! posts
        (list* (make-post title body)
               posts))
  (send/suspend
   (lambda (k-url)
     (template "Posted" (blog-posted title body k-url))))
  (display-posts))
 
(define (display-posts)
  (extract-post
   (send/suspend
    (lambda (k-url)
      (template "Posts" (blog-posts k-url))))))
 
(define (start req)
  (display-posts))

要使用模板,我们只需要改变 template, blog-posted 和 blog-posts:

(define (template section body)
  (response/full
   200 #"Okay"
   (current-seconds) TEXT/HTML-MIME-TYPE
   empty
   (list (string->bytes/utf-8 (include-template "blog.html")))))
 
(define (blog-posted title body k-url)
  (include-template "blog-posted.html"))
 
(define (blog-posts k-url)
  (include-template "blog-posts.html"))

下面给出了每一个模板:

"blog.html":

  <html>

   <head>

    <title>Al's Church: @|section|</title>

    <style type="text/css">

     body {

      margin: 0px;

      padding: 10px;

     }

  

     #main {

      background: #dddddd;

     }

    </style>

   </head>

   <body>

    <script type="text/javascript">

     var gaJsHost = (("https:" == document.location.protocol) ?

       "https://ssl." : "http://www.");

     document.write(unescape("%3Cscript src='" + gaJsHost +

       "google-analytics.com/ga.js'

        type='text/javascript'%3E%3C/script%3E"));

    </script>

    <script type="text/javascript">

     var pageTracker = _gat._getTracker("UA-YYYYYYY-Y");

     pageTracker._trackPageview();

    </script>

  

    <h1>Al's Church: @|section|</h1>

    <div id="main">

      @body

    </div>

   </body>

  </html>

请注意,这部分的表现形式要简单得多,因为 CSS 和 JavaScript 可以逐字包含,而不需要借助于任何特殊的转义 –escaping 模式。 同样,由于 body 被表示为一个字符串,所以不需要记住是否需要拼接。

"blog-posted.html":

  <h2>@|title|</h2>

  <p>@|body|</p>

  

  <h1><a href="@|k-url|">Continue</a></h1>

"blog-posts.html":

  @in[p posts]{

   <h2>@(post-title p)</h2>

   <p>@(post-body p)</p>

  }

  

  <h1>New Post</h1>

  <form action="@|k-url|">

   <input name="title" />

   <input name="body" />

   <input type="submit" />

  </form>

将这个模板与原来的演示功能相比较:不需要担心管理列表的嵌套方式:默认值 just work 。
