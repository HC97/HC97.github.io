23.1 更多的 Rackets

“Racket” 与其说是一种语言,不如说是一种关于编程语言的想法。 宏可以扩展基础语言(如 宏 中所述), 而替代分析器可以从头开始构建一种全新的语言(如 创造语言 中所述)。

启动 Racket 模块的 #lang 行声明了该模块的基础语言。 我们所说的 "Racket",通常是指 #lang 后面跟着基础语言 racket 或 racket/base (其中 racket 是一个扩展)。 Racket 发行版提供了额外的语言,包括以下几种:

  • typed/racket — 和 racket 一样,但是是静态类型的;见 theTyped Racket Guidedocumentation 。

  • lazy — 和 racket/base 一样,但是在需要表达式的值之前避免对其进行评估; 见 the Lazy Racket documentation 。

  • frtime — 以一种更激进的方式改变评估,以支持反应式编程; 见 the FrTime documentation 。

  • scribble/base — 一种语言,看起来更像 Latex 而不是 Racket,用于编写文档; 见 Scribble: Racket 文档工具

这些语言中的每一种都是通过在 #lang 后面的语言名称开始模块使用的。 例如,本文档的源代码以 #lang scribble/base 开始。

此外,Racket 用户可以定义自己的语言,正如在 创造语言 中讨论的那样。 通常情况下,语言名通过添加 /lang/reader 的模块路径映射到其实现; 例如,语言名 scribble/base 被扩展为 scribble/base/lang/reader, 它是实现表面语法分析器的模块。 一些语言名称充当语言加载器; 例如, #lang planet planet-path 通过 PLaneT 下载、安装和使用一种语言。
