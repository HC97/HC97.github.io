1 使用 OpenGL

sgl/gl 库提供了对 C 语言风格的 OpenGL API 的直接访问, 而 sgl 库则提供了一个更像 Racket 的接口。 sgl/gl 库为 OpenGL 1.5 和 GLU 1.3 中的每个 #defined 的常量和大多数函数提供了一个绑定。 这些函数与 C 语言的对应函数进行了类似的检查;它们检查参数的类型,但不检查数组参数的长度。 sgl 库为 sgl/gl 库中的许多函数提供了包装器, 以呈现一个更适合 Racket 的界面,包括遵循Racket惯例的函数名称,以及检查、符号枚举参数和数组长度的检查。

关于安全的警告: OpenGL 编程本身是不安全的,即使只使用 sgl 库也是如此。 尽管 sgl 会检查每个函数调用的参数,但违反系统的高层假设–檚 OpenGL 库会导致它崩溃, 使整个 Racket 系统崩溃。 例如,在一个 glBegin 中发送大量的顶点,至少会导致一些 GL 实现的崩溃。

一些例子可以在 Racket 安装中的 "sgl" 集合的 "examples" 目录下找到。 对于 "alpha.rkt", 试着选择 "icons" 集合中 Racket 分配的 "sk.jpg" 图片; 如果旋转的立方体是空白的,你可能需要按几次 t 键。
