4.19 程序

procedure

(procedure? v) → boolean?

  v : any/c
如果 v 是一个过程,则返回 #t , 否则返回 #f 。

procedure

(apply proc v ... lst #:<kw> kw-arg ...) → any

  proc : procedure?
  v : any/c
  lst : list?
  kw-arg : any/c

apply 函数 in Racket 指南 introduces apply.

使用 (list* v ... lst) 的内容作为(by-position)参数应用 proc 。 #:<kw> kw-arg 序列也作为关键字参数提供给 proc , 其中 #:<kw> 代表任何关键字。

给定的 proc 必须接受与 v 的数量加上 lst 的长度一样多的参数,它必须接受提供的关键字参数, 并且它不能要求任何其他关键字参数;否则,将引发 exn:fail:contract exception is raised 。 给定的 proc 在 apply 调用的尾部位置被调用。

Examples:
> (apply + '(1 2 3))

6

> (apply + 1 2 '(3))

6

> (apply + '())

0

> (apply sort (list (list '(2) '(1)) <) #:key car)

'((1) (2))

procedure

(compose proc ...) → procedure?

  proc : procedure?

procedure

(compose1 proc ...) → procedure?

  proc : procedure?
返回一个组合给定函数的过程,先应用最后一个 proc ,最后应用第一个 proc 。 compose 函数允许给定的函数消耗和产生任意数量的值,只要每个函数产生的值与前一个函数消耗的值相同, 而 compose1 则将内部值传递限制为一个值。 在这两种情况下,最后一个函数的输入 arity 和第一个函数的输出 arity 都是不受限制的, 它们成为结果组合的相应 arity(包括输入端的关键字参数)。

当没有给出 proc 参数时,结果是 values 。 当正好有一个参数被给出时,就会返回。

Examples:
> ((compose1 - sqrt) 10)

-3.1622776601683795

> ((compose1 sqrt -) 10)

0.0+3.1622776601683795i

> ((compose list split-path) (bytes->path #"/a" 'unix))

'(#<path:/> #<path:a> #f)

注意,在许多情况下, compose1 是首选。 例如,当一个函数被扩展为返回两个值,而前面的函数有一个具有不同语义的可选输入时, 将 compose 与两个库函数一起使用可能会导致问题。 此外, compose1 可能会创建更快的组合。

procedure

(procedure-rename proc name [realm]) → procedure?

  proc : procedure?
  name : symbol?
  realm : symbol? = 'racket
返回一个与 proc 类似的过程, 除了由 object-name 返回的名称(以及为调试而打印的名称)是 name ， 其 realm (可能用于调整错误信息)是 realm 。

如果产生的过程被应用于错误的参数数,给定的 name 和 realm 将用于打印和调整错误信息。 此外,如果 proc struct, make-struct-field-accessor 或 make-struct-field-mutator 产生的 accessor 或 mutator , 当其(第一个)参数的类型错误时,产生的过程也会使用 name 。 然而,更典型的情况是, name 并不用于报告错误,因为过程名称通常被硬连接到一个内部检查中。

Changed in version 8.4.0.2 of package base: 增加了 realm 参数。

procedure

(procedure-realm proc) → symbol?

  proc : procedure?
报告一个过程的 realm ,这可能取决于创建过程的模块, 过程代码编译时的 current-compile-realm 值, 或者通过 procedure-rename 等函数明确分配的境界。

Added in version 8.4.0.2 of package base.

procedure

(procedure->method proc) → procedure?

  proc : procedure?
返回一个与 proc 类似的过程,但当应用于错误的参数数量时,产生的错误会隐藏第一个参数, 就像该过程在编译时使用了 'method-arity-error 语法属性一样。

4.19.1 关键词和构架

procedure

(keyword-apply proc    
  kw-lst    
  kw-val-lst    
  v ...    
  lst    
  #:<kw> kw-arg ...) → any
  proc : procedure?
  kw-lst : (listof keyword?)
  kw-val-lst : list?
  v : any/c
  lst : list?
  kw-arg : any/c

apply 函数 in Racket 指南 introduces keyword-apply.

和 apply 一样,但是 kw-lst 和 kw-val-lst 除了提供 v 和 lst 的 by-position 参数外, 还提供了 #:<kw> kw-arg 序列中直接提供的关键词参数,其中 #:<kw> 代表任何关键词。

给定的 kw-lst 必须使用 keyword<? 进行排序。 没有关键字可以在 kw-lst 中出现两次, 或者同时出现在 kw-list 中并作为 #:<kw> 出现, 否则将引发 exn:fail:contract exception is raised 。 给定的 kw-val-lst 必须与 kw-lst 具有相同的长度,否则,会引发 exn:fail:contract exception is raised 。 给定的 proc 必须接受 kw-lst 中的所有关键字和 #:<kw> , 它不能要求任何其它的关键字, 而且它必须接受与通过 v 和 lst 提供的参数相同数量的副位置参数; 否则,将引发 exn:fail:contract exception is raised 。

Examples:
(define (f x #:y y #:z [z 10])
  (list x y z))

 

> (keyword-apply f '(#:y) '(2) '(1))

'(1 2 10)

> (keyword-apply f '(#:y #:z) '(2 3) '(1))

'(1 2 3)

> (keyword-apply f #:z 7 '(#:y) '(2) '(1))

'(1 2 7)

procedure

(procedure-arity proc) → normalized-arity?

  proc : procedure?
返回有关 proc 所接受的副位置参数数量的信息。 参见 procedure-arity?, normalized-arity? 和 procedure-arity-mask 。

procedure

(procedure-arity? v) → boolean?

  v : any/c
一个有效的 a 是以下之一:

  • 一个精确的非负整数,这意味着该过程只接受 a 参数。

  • 一个 arity-at-least 实例, 这意味着该过程接受 (arity-at-least-value a) 或更多的参数。

  • 一个包含整数和 arity-at-least 实例的列表, 这意味着该过程接受任何数量的参数,这些参数可以与 a 中的一个元素相匹配。

在 normalized-arity? 的意义上, procedure-arity 的结果总是规范化的。

Examples:
> (procedure-arity cons)

2

> (procedure-arity list)

(arity-at-least 0)

> (arity-at-least? (procedure-arity list))

#t

> (arity-at-least-value (procedure-arity list))

0

> (arity-at-least-value (procedure-arity (lambda (x . y) x)))

1

> (procedure-arity (case-lambda [(x) 0] [(x y) 1]))

'(1 2)

procedure

(procedure-arity-mask proc) → exact-integer?

  proc : procedure?
返回与 procedure-arity 相同的信息,但编码方式不同。 arity 被编码为一个精确的整数 mask ,如果 proc 接受 n 个参数, (bitwise-bit-set? mask n) 返回 true。

掩码编码的 arity 通常更容易测试和操作,而且 procedure-arity-mask 有时比 procedure-arity 更快,同时总是至少一样快。

Added in version 7.0.0.11 of package base.

procedure

(procedure-arity-includes? proc k [kws-ok?]) → boolean?

  proc : procedure?
  k : exact-nonnegative-integer?
  kws-ok? : any/c = #f
如果该过程可以接受 k 个旁位参数,则返回 #t ,否则返回 #f 。 如果 kws-ok? 是 #f , 只有当 proc没有必要的关键字参数时,其结果才是 #t 。

Examples:
> (procedure-arity-includes? cons 2)

#t

> (procedure-arity-includes? display 3)

#f

> (procedure-arity-includes? (lambda (x #:y y) x) 1)

#f

> (procedure-arity-includes? (lambda (x #:y y) x) 1 #t)

#t

procedure

(procedure-reduce-arity proc    
  arity    
  [name    
  realm]) → procedure?
  proc : procedure?
  arity : procedure-arity?
  name : (or/c symbol? #f) = #f
  realm : symbol? = 'racket
返回一个与 proc 相同的过程(包括由 object-name 返回的相同的名称), 但该过程只接受与 arity 一致的参数。 特别是,当 procedure-arity 应用于生成的过程时, 它返回一个与 arity 的规范化形式 equal? 的值。

如果 arity 规范允许不在 (procedure-arity proc) 中的参数, 就会引发 exn:fail:contract exception is raised 。 如果 proc 接受关键字参数,要么关键字参数必须都是可选的(并且它们不被 arity-reduced 过程接受), 要么 arity 必须是空列表(这使得一个过程不能被调用);否则,将引发 exn:fail:contract exception is raised 。

如果 name 不是 #f ,那么结果过程的 object-name 产生 name, 而结果过程的 procedure-realm 产生 realm 。 否则,结果过程的 object-name 和 procedure-realm 产生的结果与 proc 。

Examples:
> (define my+ (procedure-reduce-arity + 2))
> (my+ 1 2)

3

> (my+ 1 2 3)

+: arity mismatch;

 the expected number of arguments does not match the given

number

  expected: 2

  given: 3

> (define also-my+ (procedure-reduce-arity + 2 'also-my+))
> (also-my+ 1 2 3)

also-my+: arity mismatch;

 the expected number of arguments does not match the given

number

  expected: 2

  given: 3

Changed in version 7.0.0.11 of package base: 增加了可选的 name 参数。
Changed in version 8.4.0.2: 增加了 realm 参数。

procedure

(procedure-keywords proc) → 
(listof keyword?)
(or/c (listof keyword?) #f)
  proc : procedure?
返回关于一个过程所要求和接受的关键字参数的信息。 第一个结果是应用 proc 时需要的不同关键词的列表(按 keyword<? 排序)。 第二个结果是一个可接受的关键字列表(按 keyword<? 排序),或者用 #f 来表示接受任何关键字。 当第二个结果是一个列表时,第一个列表中的每个元素也在第二个列表中。

Examples:
> (procedure-keywords +)

'()

'()

> (procedure-keywords (lambda (#:tag t #:mode m) t))

'(#:mode #:tag)

'(#:mode #:tag)

> (procedure-keywords (lambda (#:tag t #:mode [m #f]) t))

'(#:tag)

'(#:mode #:tag)

procedure

(procedure-result-arity proc) → (or/c #f procedure-arity?)

  proc : procedure?
返回过程 proc 的结果数,如果不知道结果数,则返回 #f , 这可能是由于 procedure-result-arity 的实现有缺陷, 或者是由于 proc 的行为不够简单。

Examples:
> (procedure-result-arity car)

1

> (procedure-result-arity values)

#f

> (procedure-result-arity
   (λ (x)
     (apply
      values
      (let loop ()
        (cond
          [(zero? (random 10)) '()]
          [else (cons 1 (loop))])))))

#f

Added in version 6.4.0.3 of package base.

procedure

(make-keyword-procedure proc [plain-proc]) → procedure?

  proc : (((listof keyword?) list?) () #:rest list? . ->* . any)
  plain-proc : procedure?
   = (lambda args (apply proc null null args))
返回一个接受所有关键字参数的过程(不要求任何关键字参数)。

当 make-keyword-procedure 返回的过程被调用并带有关键字参数时,就会调用 proc ; 第一个参数是按 keyword<? 排序的不同关键字的列表, 第二个参数是包含每个关键字值的并行列表,其余参数是 by-position 参数。

当由 make-keyword-procedure 返回的过程被调用时, 没有关键字参数,那么就会调用 plain-proc –可能比通过 proc 调度更有效。 通常情况下, plain-proc 的行为应该与以空列表作为前两个参数调用 proc 的行为相同, 但这种对应关系并没有被强制执行。

在新过程中, procedure-arity 和 object-name 的结果与 plain-proc 的结果相同。 另请参见 procedure-reduce-keyword-arity 和 procedure-rename 。

Examples:
(define show
  (make-keyword-procedure (lambda (kws kw-args . rest)
                            (list kws kw-args rest))))

 

> (show 1)

'(() () (1))

> (show #:init 0 1 2 3 #:extra 4)

'((#:extra #:init) (4 0) (1 2 3))

 

(define show2
  (make-keyword-procedure (lambda (kws kw-args . rest)
                            (list kws kw-args rest))
                          (lambda args
                            (list->vector args))))

 

> (show2 1)

'#(1)

> (show2 #:init 0 1 2 3 #:extra 4)

'((#:extra #:init) (4 0) (1 2 3))

procedure

(procedure-reduce-keyword-arity proc    
  arity    
  required-kws    
  allowed-kws    
  [name    
  realm]) → procedure?
  proc : procedure?
  arity : procedure-arity?
  required-kws : (listof keyword?)
  allowed-kws : 
(or/c (listof keyword?)
      #f)
  name : (or/c symbol? #f) = #f
  realm : symbol? = 'racket
和 procedure-reduce-arity 一样,但是根据 required-kws 和 allowed-kws, 来限制关键字参数,这两个参数必须使用 keyword<? 进行排序,并且不包含重复的内容。 如果 allowed-kws 是 #f, 那么产生的过程仍然接受任何关键字, 否则 required-kws 中的关键字必须是 allowed-kws 中的关键字的一个子集。 原始的 proc 要求的关键字不能超过 required-kws 中列出的关键字, 而且它必须至少允许 allowed-kws 中的关键字(如果 allowed-kws 是 #f , 它必须允许所有的关键字)。

(define orig-show
  (make-keyword-procedure (lambda (kws kw-args . rest)
                            (list kws kw-args rest))))
(define show (procedure-reduce-keyword-arity
              orig-show 3 '(#:init) '(#:extra #:init)))

 

> (show #:init 0 1 2 3 #:extra 4)

'((#:extra #:init) (4 0) (1 2 3))

> (show 1)

arity mismatch;

 the expected number of arguments does not match the given

number

  expected: 3 plus an argument with keyword #:init plus an

optional argument with keyword #:extra

  given: 1

  arguments...:

   1

> (show #:init 0 1 2 3 #:extra 4 #:more 7)

application: procedure does not expect an argument with

given keyword

  procedure: #<procedure>

  given keyword: #:more

  arguments...:

   1

   2

   3

   #:extra 4

   #:init 0

   #:more 7

struct

(struct arity-at-least (value)
    #:extra-constructor-name make-arity-at-least)
  value : exact-nonnegative-integer?
用于 procedure-arity 结果的结构类型。 参见 procedure-arity? 。

value

prop:procedure : struct-type-property?

一个 structure type property ,用于识别其实例可以作为程序应用的结构类型。 特别是,当 procedure? 被应用于实例时,其结果将是 #t, 当实例被用于应用表达式的函数位置时,将从实例中提取一个过程并用于完成过程调用。

如果 prop:procedure 属性值是一个精确的非负整数,它指定了结构中应该包含一个过程的字段。 这个整数必须介于 0 (包括)和结构类型中的非自动字段的数量之间(不包括,不计算超类型字段)。 指定的字段也必须被指定为不可变的,这样在结构的实例被创建后,其过程就不能被改变。 (否则,实例的算术和名称可能会改变,而这种突变通常是程序所不允许的)。 当实例被用作应用程序表达式中的过程时,实例中指定字段的值被用来完成过程调用。 (这个过程可以是另一个充当过程的结构;过程字段的不变性不允许在过程图中出现循环, 所以过程调用最终会以一个非结构的过程继续进行)。 该过程接收来自应用表达式的所有参数。 该过程的名称(见 object-name), arity (见 procedure-arity)和关键字协议(见 procedure-keywords) 也用于结构的名称、arity 和关键字协议。 如果指定字段中的值不是一个过程,那么这个实例的行为就像 (case-lambda) (即一个不接受任何数量参数的过程) 。 另请参见 procedure-extract-target.

为 make-struct-type 提供一个整数的 proc-spec 参数, 就等于既为 prop:procedure 属性提供了值, 又将字段指定为不可变的(这样,属性绑定或不可变的指定就是多余的,不被允许)。

Examples:
> (struct annotated-proc (base note)
    #:property prop:procedure
               (struct-field-index base))
> (define plus1 (annotated-proc
                  (lambda (x) (+ x 1))
                  "adds 1 to its argument"))
> (procedure? plus1)

#t

> (annotated-proc? plus1)

#t

> (plus1 10)

11

> (annotated-proc-note plus1)

"adds 1 to its argument"

当 prop:procedure 值是一个过程时,它应该接受至少一个非关键字的参数。 当应用表达式中使用结构的一个实例时,属性值过程被调用,该实例是第一个参数。 属性值过程的其余参数是来自应用表达式的参数(包括关键字参数)。 因此,如果应用程序表达式提供了五个非关键字参数,那么属性值过程就会被调用,有六个非关键字参数。 实例的名称(见 object-name)和它的关键字协议(见 procedure-keywords) 不受属性-值过程的影响, 但是实例的arity是通过从属性-值过程的每个可能的非关键字参数数中减去 1 来确定的。 如果属性-值过程不能接受至少一个参数,那么该实例的行为就像 (case-lambda) 。

向 make-struct-type 提供一个过程 proc-spec 参数与提供 prop:procedure 属性的值是一样的(这样就不允许特定的属性绑定)。

Examples:
> (struct fish (weight color)
    #:mutable
    #:property
    prop:procedure
    (lambda (f n)
      (let ([w (fish-weight f)])
        (set-fish-weight! f (+ n w)))))
> (define wanda (fish 12 'red))
> (fish? wanda)

#t

> (procedure? wanda)

#t

> (fish-weight wanda)

12

> (for-each wanda '(1 2 3))
> (fish-weight wanda)

18

如果为 prop:procedure 属性提供的值不是一个精确的非负整数或一个过程, 就会引发 exn:fail:contract exception is raised 。

procedure

(procedure-struct-type? type) → boolean?

  type : struct-type?
如果 type 所代表的结构类型的实例是过程(根据 procedure?), 返回 #t ,否则返回 #f 。

procedure

(procedure-extract-target proc) → (or/c #f procedure?)

  proc : procedure?
如果 proc 是具有 prop:procedure 属性的结构类型的实例, 并且属性值表示结构的一个字段,如果字段值是一个过程, 那么 procedure-extract-target 返回字段值。 否则,结果是 #f.

当 prop:procedure 属性值是一个过程时, procedure-extract-target 不会 返回该过程。 这样的过程与通过结构字段访问的过程不同,因为它需要消耗一个额外的参数,这个参数总是作为过程应用的结构。 保持存储过程的私有性可以确保它总是以合适的第一个参数被调用。

value

prop:arity-string : struct-type-property?

一个 structure type property , 当具有 prop:procedure 属性的结构类型被应用于错误的参数数量时,用于报告 arity-mismatch 错误。 prop:arity-string 属性的值必须是一个过程,该过程接收一个参数,也就是被错误应用的结构,并返回一个字符串。 结果字符串被用在 "期望" 一词之后,在错误信息中,它的后面是实际参数的数量。

当 prop:arity-string 属性没有与过程结构类型相关联时, Arity-mismatch报告自动使用 procedure-extract-target 。

Examples:
> (struct evens (proc)
    #:property prop:procedure (struct-field-index proc)
    #:property prop:arity-string
    (lambda (p)
      "an even number of arguments"))
> (define pairs
    (evens
     (case-lambda
      [() null]
      [(a b . more)
       (cons (cons a b)
             (apply pairs more))])))
> (pairs 1 2 3 4)

'((1 . 2) (3 . 4))

> (pairs 5)

arity mismatch;

 the expected number of arguments does not match the given

number

  expected: an even number of arguments

  given: 1

  arguments...:

   5

value

prop:checked-procedure : struct-type-property?

一个 structure type property ,与 checked-procedure-check-and-extract 一起使用, 这是一个钩子,允许编译器提高关键字参数的性能。 该属性只能附加到一个没有超类型并且至少有两个字段的 structure type 。

procedure

(checked-procedure-check-and-extract type    
  v    
  proc    
  v1    
  v2) → any/c
  type : struct-type?
  v : any/c
  proc : (any/c any/c any/c . -> . any/c)
  v1 : any/c
  v2 : any/c
如果 v 是 type 的一个实例,则从 v 中提取一个值, 这个实例必须有 prop:checked-procedure 属性。 如果 v 是这样一个实例,那么 v 的第一个字段被提取出来并应用于 v1 和 v2; 如果结果是一个真值,那么结果就是 v 的第二个字段的值。

如果 v 不是一个 type 的实例, 或者如果 v 的第一个字段应用于 v1 和 v2 产生 #f, 那么 proc 就会应用于 v, v1 和 v2 , 其结果由 checked-procedure-check-and-extract 返回。

(procedure-specialize proc) → procedure?

返回 proc 或其等价物,但向运行时系统提供一个提示,即它应该花费额外的时间和内存来专门化 proc 的实现。

目前,当 proc 是 lambda 或 case-lambda 形式的值, 引用 lambda 或 case-lambda 之外的变量时,以及 proc 之前未被应用时,就会使用该提示。

4.19.2 对基元的反思

primitive procedure 是一种可以用低级语言实现的内置过程。 并非所有 racket/base 的过程都是基元,但很多都是。 区分基元和其他过程可能对其他低级代码有用。

procedure

(primitive? v) → boolean?

  v : any/c
如果 v 是一个原始过程,返回 #t ,否则返回 #f 。

procedure

(primitive-closure? v) → boolean

  v : any/c
如果 v 在内部被实现为一个原始封闭,而不是一个简单的原始过程,则返回 #t ,否则返回 #f 。

procedure

(primitive-result-arity prim) → procedure-arity?

  prim : primitive?
返回原始过程 prim 的结果的 arity(而不是由 procedure-arity 返回的过程的输入 arity)。 对于大多数基元,这个过程返回 1, 因为大多数基元在应用时都返回一个值。

procedure

(curry proc) → procedure?

  proc : procedure?
(curry proc v ...+) → any
  proc : procedure?
  v : any/c
(curry proc) 的结果是一个过程,它是 proc 的 curried 版本。 当第一次应用所产生的过程时,除非根据 (procedure-arity proc) 给出它可以接受的最大参数数, 否则其结果是一个可以接受额外参数的过程。

Examples:
> ((curry list) 1 2)

#<procedure:curried:list>

> ((curry cons) 1)

#<procedure:curried:cons>

> ((curry cons) 1 2)

'(1 . 2)

在第一次应用 (curry proc) 的结果后,每一次进一步的应用都会累积参数, 直到根据 (procedure-arity proc) 累积到可接受的参数数,这时会调用原来的 proc 。

Examples:
> (((curry list) 1 2) 3)

'(1 2 3)

> (((curry list) 1) 3)

'(1 3)

> ((((curry foldl) +) 0) '(1 2 3))

6

> (define foo (curry (lambda (x y z) (list x y z))))
> (foo 1 2 3)

'(1 2 3)

> (((((foo) 1) 2)) 3)

'(1 2 3)

一个函数调用 (curry proc v ...) 等同于 ((curry proc) v ...) 。 换句话说, curry 本身就是被 curried 。

Examples:
> (map ((curry +) 10) '(1 2 3))

'(11 12 13)

> (map (curry + 10) '(1 2 3))

'(11 12 13)

> (map (compose (curry * 2) (curry + 10)) '(1 2 3))

'(22 24 26)

curry 函数也支持带有关键字参数的函数:关键字参数将以与位置参数相同的方式被累积, 直到根据 (procedure-keywords proc) 提供所有需要的关键字参数。

Examples:
(define (f #:a a #:b b #:c c)
  (list a b c))

 

> ((((curry f) #:a 1) #:b 2) #:c 3)

'(1 2 3)

> ((((curry f) #:b 1) #:c 2) #:a 3)

'(3 1 2)

> ((curry f #:a 1 #:c 2) #:b 3)

'(1 3 2)

Changed in version 7.0.0.7 of package base: 增加了对关键字参数的支持。

(normalized-arity? arity) → boolean?

  arity : any/c
一个规范化的 arity 有以下形式之一:

  • 空列表;

  • 一个精确的非负整数;

  • 一个 arity-at-least 实例;

  • 一个由两个或更多严格增加的精确非负整数组成的列表;或

  • 一个或多个严格递增的精确非负整数的列表,后面有一个 arity-at-least 实例,其值比前面的整数至少大2。

每一个归一化的 arity 都是一个有效的程序 arity,并且满足 procedure-arity? 。 任何两个归一化的 arity 值如果是 arity=? ,也必须是 equal? 。

#f

> (normalized-arity? (list 0 (arity-at-least 2)))

#t

> (normalized-arity? (list (arity-at-least 2) 0))

#f

> (normalized-arity? (list 0 2 (arity-at-least 3)))

#f
