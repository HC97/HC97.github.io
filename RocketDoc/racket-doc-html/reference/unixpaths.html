15.1.3 Unix and Mac OS 路径

Unix and Mac OS 的路径中, / 分隔了路径中的元素, . 作为一个路径元素总是指前面的路径所指示的目录, .. 作为一个路径元素总是指前面的路径所指示的目录的父目录。 路径中领先的 ~ 不被特别处理, 但是 expand-user-path 可以用来将领先的 ~ 元素转换为用户特定的目录。 在路径中没有其他字符或字节具有特殊意义。多个相邻的 /等同于一个 / (即,它们作为一个单一的路径分隔符)。

路径根总是 / 。 以 / 开头的路径是一个绝对的、完整的路径,而以任何其他字符开头的路径是一个相对路径。

任何以 / 结尾的路径名在语法上都是指一个目录, 任何最后一个元素为 . 或 .. 的路径也是如此。

A Unix and Mac OS 的路径是通过用一个 / 替换多个相邻的 / 来 cleanse 。

对于 (bytes->path-element bstr), bstr 必须不包含任何 /, 否则会引发 exn:fail:contract exception is raised 。 (path-element->bytes path) 或 (path-element->string path) 的结果总是与 (path->bytes path) 和 (path->string path) 的结果相同。 然而,由于其他平台并非如此,在转换单个路径元素时,应该使用 path-element->bytes 和 path-element->string 。

在 Mac OS 上,Finder 别名是零长度的文件。

15.1.3.1 Unix 的路径表示法

在 Unix and Mac OS 上,路径本身是一个字节字符串。 为了向用户展示和进行其他基于字符串的操作,路径被转换为/从字符串,使用当前地区的编码, 用 ? (编码)或 #\uFFFD (解码)来代替错误。 请注意,编码可能无法将所有可能的路径作为不同的字符串。
