6.12 代理人

 (require racket/surrogate) package: base
The bindings documented in this section are provided by the racket/surrogate library, not racket/base or racket.

racket/surrogate 库为建立 proxy design pattern 的实例提供了一个抽象的概念。 该模式由两个对象组成,一个 host 和一个 surrogate 对象。 主机对象将方法调用委托给它的代理对象。 每个主机都有一个动态分配的代理对象,所以一个对象仅仅通过改变代理对象就可以完全改变其行为。

syntax

(surrogate use-wrapper-proc method-spec ...)

 
use-wrapper-proc = #:use-wrapper-proc
  | 
     
method-spec = (augment default-expr method-id arg-spec ...)
  | (override method-id arg-spec ...)
     
arg-spec = (id ...)
  | id
surrogate 形式产生四个值:一个宿主 mixin (一个接受并返回一个类的过程), 一个宿主 interface ,一个代用 class ,以及一个代用 interface 。

如果 #:use-wrapper-proc 没有出现,宿主混元就会在其参数中添加一个单独的私有字段。 它还添加了 get-surrogate 和 set-surrogate 方法来获取和设置该字段的值。 set-surrogate 方法接受由 surrogate 形式或 #f 返回的类的实例, 它用参数更新字段;然后, set-surrogate 对字段的前一个值调用 on-disable-surrogate , 对字段的新值调用 on-enable-surrogate 。 get-surrogate 方法返回该字段的当前值。

如果 #:use-wrapper-proc 确实出现了, 那么宿主混合器就会添加第二个私有字段及其 get-surrogate-wrapper-proc 和 set-surrogate-wrapper-proc 方法。 这个额外的字段持有一个包装过程,其契约是 (-> (-> any) (-> any) any) ,因此该过程被调用时有两个 thunk。 第一个thunk是一个回退,调用原始对象的方法,跳过代理程序。 第二个thunk调用代理程序。 默认的封装过程是
(λ (fallback-thunk surrogate-thunk)
  (surrogate-thunk))
也就是说,它只是推迟了在代理程序上被调用的方法。 请注意,包装程序可以通过改变参数值等方式调整对代理程序的动态调用范围。 在调用代理程序的 on-disable-surrogate 和 on-enable-surrogate 方法时,也会调用包装程序。

host mixin 对于 surrogate 形式的每个 method-id 都有一个覆盖的方法(甚至是用 augment 指定的方法)。 这些方法中的每一个都被定义为 case-lambda ,每个 arg-spec 都有一个臂。 每个臂都有作为 arg-spec 中的参数的变量。 每个方法的主体都会测试私有代理字段。 如果字段的值是 #f ,该方法只是返回调用超级或内部方法的结果。 如果字段的值不是 #f ,则调用字段中对象的相应方法。 这个方法接收的参数与原始方法相同,另外还有两个额外的参数。 额外的参数出现在参数列表的开头。第一个是原始对象。 第二个是调用超级或内部方法(即传递给mixin或扩展的类的方法,或覆盖类中的方法)的过程,以及该过程收到的参数。

例如,这个surrogate的host-mixin:

(surrogate (override m (x y z)))

将覆盖 m 方法,并像这样调用代理程序:
(define/override (m x y z)
  (if surrogate
      (send surrogate m
            this
            (λ (x y z) (super m x y z))
            x y z)
      (super m x y z)))
其中, surrogate 被绑定到最近传递给host mixin的 set-surrogate 方法的值上。

宿主接口有 set-surrogate, get-surrogate 的名字,以及每个 method-id 的原始形式。

代用类在 surrogate 形式中的每个 method-id 都有一个公共方法。 这些方法被 mixin 构建的类所调用。 每个方法都有一个相应的方法签名,如上一段所述。 每个方法只是将其参数传递给它所接收的超级过程。

在上面的例子中,这就是代用类中的 m 方法:
(define/public (m original-object original-super x y z)
  (original-super x y z))

如果你从代理类派生出一个类,不要同时调用 super 参数和代理类本身的超级方法。 只调用其中一个,因为默认的方法都是调用 super 参数的。

最后,接口包含了 surrogate 的参数中指定的所有名字,加上 on-enable-surrogate 和 on-disable-surrogate 。 由 surrogate 返回的类实现了这个接口。
