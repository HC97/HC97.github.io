3.8 Procedure Expressions: lambda and case-lambda

lambda 函数（过程） in Racket 指南 introduces procedure expressions.

syntax

(lambda kw-formals body ...+)

syntax

(λ kw-formals body ...+)

 
kw-formals = (arg ...)
  | (arg ...+ . rest-id)
  | rest-id
     
arg = id
  | [id default-expr]
  | keyword id
  | keyword [id default-expr]
生成一个过程。 kw-formals 决定了过程所接受的参数数量和关键字参数。

只考虑到第一个 arg 的情况,一个简单的 kw-formals 有以下三种形式之一:

(id ...)

该过程接受的非关键字参数值的数量与 id 的数量相同。 每个 id 按位置与一个参数值相关联。

(id ...+ . rest-id)

该过程接受任何数量的非关键字参数, 大于或等于 id 的数量。 当程序被应用时, id 按位置与参数值相关联, 所有剩余的参数被放入一个与 rest-id.

rest-id

该过程接受任何数量的非关键字参数。 所有的参数都被放在一个与 rest-id 相关的列表中。

更一般地说,一个 arg 可以包括一个关键字和/或默认值。 因此,上面的前两种情况被更完整地规定为:

(arg ...)

每个 arg 都有以下四种形式:

id

在程序所接受的非关键字参数的最小和最大数量上都增加一个。 id 通过位置与实际参数相关联。

[id default-expr]

在程序所接受的非关键字参数的最大数量上增加一个。 id 通过位置与一个实际的参数相关联,如果没有提供这样的参数, default-expr 会被评估以产生一个与 id 相关的值。 任何带有 default-expr 式的 arg 都不能出现在没有 default-expr 和没有 keyword 的 id 之前。

keyword id

该程序需要一个基于关键字的参数, 使用 keyword 。 该 id 与一个基于关键字的实际参数相关,使用 keyword。

keyword [id default-expr]

该过程接受一个基于关键字的参数,使用 keyword。 如果在应用程序中提供, id 使用 keyword 与基于关键字的实际参数相关联; 否则,将计算 default-expr 以获得与 id 关联的值。

keyword arg 在 kw-formals 中的位置并不重要, 但每个指定的 keyword 必须是不同的。

(arg ...+ . rest-id)

和前面的情况一样, 但是程序接受任何数量的非关键字参数,超过其最小参数数量。 当提供的参数多于 arg 中的 keyword 参数时, 额外的参数将被放入一个与 rest-id 相关的列表中。

换句话说,带有缺省值表达式的参数绑定被评估,类似于 let*。

kw-formals 标识符被绑定在 body 中。 当程序被应用时,为每个标识符创建一个新的 location, 并且用相关的参数值填充该位置。 locations 的创建和填充是按顺序进行的, default-expr 会根据需要被评估以填充位置。

如果任何标识符出现在 body 中,而不是 kw-formals 中的标识符之一, 那么它所指的位置与它出现在 lambda 表达式中的位置一样。 (换句话说,变量引用是有词法范围的)。

当多个标识符出现在 kw-formals 中时, 根据 bound-identifier=? ,它们必须是不同的。

如果由 lambda 产生的过程被应用于比它所接受的更少或更多的 by-position 或 by-keyword 参数, 被应用于它不接受的 by-keyword 参数,或者没有必要的 by-keyword 参数, 那么将引发 exn:fail:contract exception is raised 异常。

最后一个 body 表达式相对于过程主体来说处于尾部位置。

Examples:
> ((lambda (x) x) 10)

10

> ((lambda (x y) (list y x)) 1 2)

'(2 1)

> ((lambda (x [y 5]) (list y x)) 1 2)

'(2 1)

> (let ([f (lambda (x #:arg y) (list y x))])
   (list (f 1 #:arg 2)
         (f #:arg 2 1)))

'((2 1) (2 1))

在编译 lambda 或 case-lambda 表达式时, Racket 会寻找附加在表达式上的 'method-arity-error 属性 (参见 语法对象属性)。 如果它的值为真,并且该过程的任何情况都不接受零参数, 那么该过程就会被标记为 exn:fail:contract:arity 异常, 涉及该过程的第一个参数将被隐藏,如果有的话。 (当存储过程实现了一个方法时,隐藏第一个参数是非常有用的,因为第一个参数在原始源中是隐含的)。 这个属性只影响 exn:fail:contract:arity 异常的格式, 而不是 procedure-arity 的结果。

当一个接受关键字的程序以某些方式绑定到一个标识符上,并且该标识符被用于申请表的函数位置时, 那么申请表可能会以这样的方式被扩展,即原来的绑定被掩盖为申请表的目标。 为了帮助暴露函数应用和函数声明之间的联系, 在函数应用的扩展中的标识符被标记为 syntax property, 可以通过 syntax-procedure-alias-property 访问,如果它实际上是原始标识符的别名。 如果扩展中的标识符除了参数被转换为扁平化的形式外,与原始标识符一样, 则被标记为 syntax property , 可通过 syntax-procedure-converted-arguments-property 访问。 关键字参数、必要的副位置参数、副位置可选参数和其他参数–所有这些都是必要的副位置参数; 关键字参数按关键字名称排序,每个可选关键字参数后面都有一个布尔值来表示是否提供了一个值, #f 用来表示一个没有提供值的可选关键字参数; 可选副位置参数包括每个没有提供的参数的 #f , 然后可选参数值的序列后面有一个平行的布尔值序列来表示每个可选参数值是否被提供。

syntax

(case-lambda [formals body ...+] ...)

 
formals = (id ...)
  | (id ...+ . rest-id)
  | rest-id
生成一个过程。每个 [formals body ...+] 子句都类似于一个 lambda 过程; 应用 case-lambda 生成的过程与应用对应于其中一个子句的过程是一样的, 即第一个接受给定参数数的过程。 如果没有对应的过程接受给定的参数数,就会引发 exn:fail:contract exception is raised 异常。

请注意, case-lambda 子句只支持 formals , 而不是 lambda 的更一般的 kw-formals 。 也就是说, case-lambda 并不直接支持关键字和可选参数。

Example:
> (let ([f (case-lambda
            [() 10]
            [(x) x]
            [(x y) (list y x)]
            [r r])])
    (list (f)
          (f 1)
          (f 1 2)
          (f 1 2 3)))

'(10 1 (2 1) (1 2 3))

syntax

(#%plain-lambda formals body ...+)

像 lambda ,但不支持关键字或可选参数。
