4.3.3 Flonums

 (require racket/flonum) package: base

racket/flonum 库提供了像 fl+ 这样只消耗和产生 flonums 的操作。 当持续使用时,flonum 特定的操作可以提供更好的性能,并且它们和通用操作如 + 一样安全。

See also Fixnum 和 Flonum 优化 in Racket 指南.

4.3.3.1 Flonum 算术

(fl+ a ...) → flonum?

procedure

(fl- a b ...) → flonum?

  a : flonum?

(fl* a ...) → flonum?

  a : flonum?

procedure

(fl/ a b ...) → flonum?

  a : flonum?
  b : flonum?

procedure

(flabs a) → flonum?

  a : flonum?
像 +, -, *, / 和 abs 一样, 但被限制为消耗 flonums 。 其结果总是一个 flonum 。

Changed in version 7.0.0.13 of package base: 允许 fl+ 和 fl* 有零个或多个参数, fl- 和 fl/ 有一个或多个参数。

(fl= a b ...) → boolean?

  a : flonum?
  b : flonum?

procedure

(fl< a b ...) → boolean?

  a : flonum?
  b : flonum?

procedure

(fl> a b ...) → boolean?

  a : flonum?
  b : flonum?

procedure

(fl<= a b ...) → boolean?

  a : flonum?
  b : flonum?

procedure

(fl>= a b ...) → boolean?

  a : flonum?
  b : flonum?

procedure

(flmin a b ...) → flonum?

  a : flonum?
  b : flonum?

procedure

(flmax a b ...) → flonum?

  a : flonum?
  b : flonum?
就像 =, <, >, <=, >=, min 和 max 一样,但被限制为消耗 flonums 。

Changed in version 7.0.0.13 of package base: Allow one argument, in addition to allowing two or more.

procedure

(flround a) → flonum?

  a : flonum?

procedure

(flfloor a) → flonum?

  a : flonum?

procedure

(flceiling a) → flonum?

  a : flonum?

procedure

(fltruncate a) → flonum?

  a : flonum?
像 round, floor, ceiling 和 truncate 一样,但被限制在消耗 flonums 。

procedure

(flsingle a) → flonum?

  a : flonum?
返回一个类似于 a 的值,但是有可能丢弃精度和范围, 这样结果就可以表示为单精度的 IEEE 浮点数(即使不支持 single-flonums )。

在 fl+, fl-, fl*, fl/ 和 flsqrt 的参数和结果上使用 flsingle– 即对可表示为单精度的值进行双精度操作,然后将结果舍入为单精度 –总是与执行相应的单精度操作相同 [Roux14]。 (对于其他操作,IEEE 浮点规范并没有做出足够的保证,因此无法对与 flsingle 的交互作用做出更多说明)。

Added in version 7.8.0.7 of package base.

procedure

(flsin a) → flonum?

  a : flonum?

procedure

(flcos a) → flonum?

  a : flonum?

procedure

(fltan a) → flonum?

  a : flonum?

procedure

(flasin a) → flonum?

  a : flonum?

procedure

(flacos a) → flonum?

  a : flonum?

procedure

(flatan a) → flonum?

  a : flonum?

procedure

(fllog a) → flonum?

  a : flonum?

procedure

(flexp a) → flonum?

  a : flonum?

procedure

(flsqrt a) → flonum?

  a : flonum?
与 sin, cos, tan, asin, acos, atan, log, exp 和 sqrt 类似,但受限于消耗和产生 flonums 。 当给 flasin 或 flacos 一个超出 -1.0 到 1.0 范围的数字, 或者给 fllog 或 flsqrt 一个负数时,结果是 +nan.0 。

procedure

(flexpt a b) → flonum?

  a : flonum?
  b : flonum?
与 expt 类似,但被约束为消耗和产生 flonums 。

由于结果的约束,在以下情况下与 expt 相比,结果有所不同: 这些特殊情况对应于 C99 [C99] 中的 pow 。

  • (flexpt -1.0 +inf.0) — 1.0

  • (flexpt a +inf.0) 其中 a 为负数 — (expt (abs a) +inf.0)

  • (flexpt a -inf.0) 其中 a 是负数 — (expt (abs a) -inf.0)

  • (expt -inf.0 b) 其中 b 是一个非整数:

    • b 是负数 — 0.0

    • b 是正数 — +inf.0

  • (flexpt a b) 其中 a 是负数, b 不是整数 — +nan.0

procedure

(->fl a) → flonum?

  a : exact-integer?
像 exact->inexact ,但限制在消耗精确的整数,所以结果总是一个 flonum 。

procedure

(fl->exact-integer a) → exact-integer?

  a : flonum?
与 inexact->exact 类似,但被限制为消耗一个 integer flonum, 所以结果总是一个精确的整数。

procedure

(make-flrectangular a b)

 → 
(and/c complex?
       (lambda (c) (flonum? (real-part c)))
       (lambda (c) (flonum? (imag-part c))))
  a : flonum?
  b : flonum?

procedure

(flreal-part a) → flonum?

  a : 
(and/c complex?
       (lambda (c) (flonum? (real-part c)))
       (lambda (c) (flonum? (imag-part c))))

procedure

(flimag-part a) → flonum?

  a : 
(and/c complex?
       (lambda (c) (flonum? (real-part c)))
       (lambda (c) (flonum? (imag-part c))))
与 make-rectangular, real-part 和 imag-part 类似,但复数的两个部分必须是不精确的。

procedure

(flrandom rand-gen) → (and flonum? (>/c 0) (</c 1))

  rand-gen : pseudo-random-generator?
相当于 (random rand-gen) 。

4.3.3.2 Flonum 向量

flvector 就像一个 vector ,但它只保存不确切的实数。 这种表示方法可以更紧凑,而且对 flvector 的不安全操作(参见 racket/unsafe/ops) 比对非精确实数的 vectors 的不安全操作执行得更有效率。

由 ffi/vector 提供的 f64vector 可以存储与 flvector 相同的值, 但有额外的间接性,这使得 f64vectors 在与外国库合作时更加方便。 没有间接因素使得不安全的 flvector 访问更有效率。

如果两个 flvectors 具有相同的长度, 并且 flvectors 的相应槽中的值 equal? ,则它们是 equal? 。

打印的 flvector 以 #fl( 开始,可以选择在 #fl 和 ( 之间加上一个数字。 See 读向量 for information on reading flvectors and 打印矢量 for information on printing flvectors.

procedure

(flvector? v) → boolean?

  v : any/c
如果 v 是一个 flvector,则返回 #t ,否则返回 #f 。

procedure

(flvector x ...) → flvector?

  x : flonum?
创建一个包含给定非精确实数的 flvector 。

Example:
> (flvector 2.0 3.0 4.0 5.0)

(flvector 2.0 3.0 4.0 5.0)

procedure

(make-flvector size [x]) → flvector?

  size : exact-nonnegative-integer?
  x : flonum? = 0.0
创建一个具有 size 元素的 flvector , 其中 flvector 中的每个槽都被填充为 x 。

Example:
> (make-flvector 4 3.0)

(flvector 3.0 3.0 3.0 3.0)

procedure

(flvector-length vec) → exact-nonnegative-integer?

  vec : flvector?
返回 vec 的长度(即 flvector 中的槽数)。

procedure

(flvector-ref vec pos) → flonum?

  vec : flvector?
  pos : exact-nonnegative-integer?
返回 vec 的槽 pos 中的不确切的实数。 第一个槽位是 0, 最后一个槽位是比 (flvector-length vec) 少一个的。

procedure

(flvector-set! vec pos x) → flonum?

  vec : flvector?
  pos : exact-nonnegative-integer?
  x : flonum?
设置 vec 的槽 pos 中的不精确实数。 第一个槽位是 0, 最后一个槽位是比 (flvector-length vec) 少一个的位置。

procedure

(flvector-copy vec [start end]) → flvector?

  vec : flvector?
  start : exact-nonnegative-integer? = 0
  end : exact-nonnegative-integer? = (vector-length v)
创建一个大小为 (- end start) 的新 flvector , 其中包含 vec 的所有元素,从 start (包括)到 end (不包括)。

procedure

(in-flvector vec [start stop step]) → sequence?

  vec : flvector?
  start : exact-nonnegative-integer? = 0
  stop : (or/c exact-integer? #f) = #f
  step : (and/c exact-integer? (not/c zero?)) = 1
当没有提供可选参数时,返回一个等同于 vec 的序列。

可选参数 start, stop 和 step 与 in-vector 相同。

当一个 in-flvector 应用程序直接出现在 for 子句中时, 可以为 flvector 迭代提供更好的性能。

syntax

(for/flvector maybe-length (for-clause ...) body ...)

syntax

(for*/flvector maybe-length (for-clause ...) body ...)

 
maybe-length = 
  | #:length length-expr
  | #:length length-expr #:fill fill-expr
 
  length-expr : exact-nonnegative-integer?
  fill-expr : flonum?
像 for/vector 或 for*/vector 一样, 但对 flvector 而言。 默认的 fill-expr 产生 0.0 。

procedure

(shared-flvector x ...) → flvector?

  x : flonum?
创建一个包含给定非精确实数的 flvector 。 为了在 places 之间进行通信, 新的 flvector 被分配到 shared memory space 中。

Example:
> (shared-flvector 2.0 3.0 4.0 5.0)

(flvector 2.0 3.0 4.0 5.0)

procedure

(make-shared-flvector size [x]) → flvector?

  size : exact-nonnegative-integer?
  x : flonum? = 0.0
创建一个具有 size 元素的 flvector , 其中 flvector 中的每个槽都被填充为 x 。 对于 places 之间的通信,新的 flvector 被分配到 shared memory space 。

Example:
> (make-shared-flvector 4 3.0)

(flvector 3.0 3.0 3.0 3.0)
