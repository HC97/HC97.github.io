7.2 调用单元

syntax

(invoke-unit unit-expr)

(invoke-unit unit-expr (import tagged-sig-spec ...))
调用由 unit-expr 产生的单元。对于单元的每个导入, invoke-unit 表达式必须在 import 子句中包含一个 tagged-sig-spec ;关于 tagged-sig-spec 的语法见 unit 。 如果单元没有导入, import 子句可以被省略。

当没有提供 tagged-sig-spec 时, unit-expr 必须产生一个不期望导入的单元。 为了调用这个单元,所有的绑定首先被初始化为 #<undefined> 值。 接下来,单元的主体定义和表达式被依次评估;在定义的情况下,评估设置相应变量的值。 最后,单元中最后一个表达式的结果是 invoke-unit 表达式的结果。

每个提供的 tagged-sig-spec 从周围的上下文中获取绑定并把它们变成被调用单元的导入。 单元不需要为每个提供的 tagged-sig-spec 声明导入, 但必须为单元的每个声明的导入提供一个 tagged-sig-spec 。 对于每个提供的 tagged-sig-spec 中的每个变量标识符, 标识符在周围环境中的绑定值被用于被调用单元的相应导入。

syntax

(define-values/invoke-unit unit-expr
  (import tagged-sig-spec ...)
   (export tagged-sig-spec ...)
  maybe-results-clause)
 
maybe-results-clause = 
  | (values result-id ...)
  | (values result-id ... . rest-results-id)
像 invoke-unit 一样,但单元的导出值被复制到新的绑定中。

由 unit-expr 产生的单元与 invoke-unit 一样被链接和调用。 此外, export 条款被当作一种导入本地定义上下文的方式。 也就是说,对于每一个在使用 export 子句的 tagged-sig-spec 作为导入的单元中可用的绑定,将为 define-values/invoke-unit 形式的上下文生成一个定义。

如果没有提供 maybe-results-clause ,单元体可以返回任意数量的值,所有这些值都被忽略。 否则,从单元主体返回的值将按顺序绑定到给定的 result-id 上。 如果没有提供 rest-results-id ,单元主体必须返回与 result-id 相同数量的值, 但是如果提供了,单元主体可以返回任意多的值,并且 rest-results-id 被绑定到一个包含额外结果的列表。

Changed in version 8.8.0.7 of package base: 增加 maybe-results-clause 。
