16.2 外显子

ephemeron [Hayes97] 是 weak box 的一个泛化(见 弱盒) 。 一个 ephemeron 不是只包含一个值,而是包含两个值:一个被认为是 ephemeron 的值,另一个是 ephemeron 的键。 就像弱盒中的值一样,附子中的值可以被 #f 替换, 但是当 key 不再可及时(除了可能通过弱引用),而不是值不再可及时。

只要一个 ephemeron 的值被保留,该引用就被认为是一个非弱引用。 然而,通过值对键的引用被特殊对待,因为该引用不一定算作键的可及性。 一个 weak box 可以被看作是 ephemeron 的一个特殊化,其中键和值是相同的。

一个特别常见的 ephemerons 的用法是将其与弱散列表(见 Hash Tables) 结合起来, 以产生一种映射,其中内存管理器可以回收键-值对,即使值是指键;见 make-ephemeron-hash 。 一个相关的用途是保留对一个值的引用,只要它是一个 impersonator 的任何值是可以到达的; 见 impersonator-ephemeron 。

更确切地说、

  • 当自动内存管理程序可以证明,无论是 ephemeron 还是 key 都只能通过弱引用来达到, 那么 ephemeron 中的值就会被 #f 所取代(参见 弱盒);以及

  • 在一个子代子中的值的可及性不计入子代子的键的可及性(无论是同一子代子还是其他子代子), 除非相同的值可以通过非弱引用到达,或者除非该值的子代子的键可以通过非弱引用到达 (关于弱引用的信息请参见 弱盒 ) 。

procedure

(make-ephemeron key v) → ephemeron?

  key : any/c
  v : any/c
返回一个新的 ephemeron ,其键为 key ,其值最初为 v 。

procedure

(ephemeron-value ephemeron [gced-v retain-v]) → any/c

  ephemeron : ephemeron?
  gced-v : any/c = #f
  retain-v : any/c = #f
返回包含在 ephemeron 中的值。 如果垃圾收集器已经证明 ephemeron 的key只是弱引用,那么结果是 gced-v (默认为 #f)。

retain-v 参数被保留为可达,直到 ephemeron 的值被提取。 例如,当 ephemeron 是通过一个弱的,基于 eq? 的映射从 key 获得的, 并且 ephemeron 是以 key 为键创建的; 在这种情况下,提供 key 作为 retain-v 可以确保 ephemeron 保留它的值足够长的时间来提取它, 即使 key 在其他方面是不可达的。

Changed in version 7.1.0.10 of package base: 增加了 retain-v 参数。

procedure

(ephemeron? v) → boolean?

  v : any/c
如果 v 是一个 ephemeron,返回 #t ,否则返回 #f 。
