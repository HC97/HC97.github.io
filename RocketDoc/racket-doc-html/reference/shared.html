3.11 构建图: shared

 (require racket/shared) package: base
The bindings documented in this section are provided by the racket/shared and racket libraries, but not racket/base.

syntax

(shared ([id expr] ...) body ...+)

根据 expr 将 id 与共享结构绑定, 然后对 body 进行评估,返回最后一个表达式的结果。

shared 形式与 letrec 相似, 只是 expr 的特殊形式被识别(在部分宏扩展之后), 以构建图结构的数据,而相应的 letrec 反而会产生一个使用前初始化的错误。

每个 expr (部分扩展后)都与以下共享 shared-expr 语法相匹配, 其中生产中的早期变体优先于后期变体。:

  shared-expr = shell-expr
  | plain-expr
     
  shell-expr = (cons in-immutable-expr in-immutable-expr)
  | (list in-immutable-expr ...)
  | (list* in-immutable-expr ...)
  | (append early-expr ... in-immutable-expr)
  | (vector-immutable in-immutable-expr ...)
  | (box-immutable in-immutable-expr)
  | (mcons patchable-expr patchable-expr)
  | (vector patchable-expr ...)
  | (box patchable-expr)
  | (prefix:make-id patchable-expr ...)
     
  in-immutable-expr = shell-id
  | shell-expr
  | early-expr
     
  shell-id = id
     
  patchable-expr = expr
     
  early-expr = expr
     
  plain-expr = expr

上面的 prefix:make-id 标识符匹配三种引用。 第一种是任何名字中间有 make- 的绑定, 并且 prefix:id 有一个 transformer 绑定到结构信息,有一整套的 mutator 绑定; 参见 结构类型转化器绑定 。 第二种是一个标识符,它本身有一个与结构信息绑定的 transformer 。 第三种是一个标识符,它有一个 'constructor-for syntax property 属性,其值是一个与结构信息有 transformer 绑定的标识符。 同时,一个 shell-id 必须是 shared 形式绑定到 shell-expr 的 id 之一。

当 shared 形式的 expr 被解析为 shared-expr 时 (考虑到解析优先级的变体顺序), 通过 early-expr 解析的子表达式将在 shared 形式被评估时被首先评估。 在这些表达式中,它们按照在 shared 表单中出现的顺序被评估。 然而,任何对由 shared 绑定的 id 的引用都会产生一个使用前初始化的错误, 即使 id 的绑定出现在 shared 形式中相应的 early-expr 之前。

shell-id 和 shell-expr (不包括 patchable-expr 和 early-expr 子表达)接下来会被有效地评估:

  • 一个 shell-id 引用产生的值与对应的 id 在 body 中产生的值相同, 假设 id 从未被 set! 突变过。 这种对 shell-id 引用的特殊处理是 shared 支持创建循环数据的一种方式, 包括不可变的循环数据。

  • 形式为 (mcons patchable-expr patchable-expr), (vector patchable-expr ...), (box patchable-expr) 或 (prefix:make-id patchable-expr ...) 的 shell-expr 产生一个可变值, 其内容位置被初始化为 undefined 。 每一个内容位置在随后评估相应的 patchable-expr 表达式后被 patched (即,更新)。

接下来, plain-expr 被评估为 letrec, 其中对一个 id 的引用如果在 id 绑定的右侧之前被评估, 就会引发 exn:fail:contract:variable 。

最后, patchable-expr 被评估, 它们的值取代 shell-expr 结果中的 undefined 。 在这一点上,所有的 id 都被绑定了,所以 patchable-expr 可以创建数据循环(但只能是通过变异创建的循环)。

Examples:
> (shared ([a (cons 1 a)])
    a)

#0='(1 . #0#)

> (shared ([a (cons 1 b)]
           [b (cons 2 a)])
    a)

#0='(1 2 . #0#)

> (shared ([a (cons 1 b)]
           [b 7])
    a)

'(1 . 7)

> (shared ([a a]) ; no indirection...
    a)

a: undefined;

 cannot use before initialization

> (shared ([a (cons 1 b)] ; b is early...
           [b a])
    a)

a: undefined;

 cannot use before initialization

> (shared ([a (mcons 1 b)] ; b is patchable...
           [b a])
    a)

#0=(mcons 1 #0#)

> (shared ([a (vector b b b)]
           [b (box 1)])
    (set-box! b 5)
    a)

'#(#&5 #&5 #&5)

> (shared ([a (box b)]
           [b (vector (unbox a)   ; unbox after a is patched
                      (unbox c))] ; unbox before c is patched
           [c (box b)])
    b)

#0='#(#0# #<undefined>)
