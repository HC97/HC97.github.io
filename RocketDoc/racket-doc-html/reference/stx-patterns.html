12.1 基于模式的语法匹配

syntax

(syntax-case stx-expr (literal-id ...)
  clause ...)
 
clause = [pattern result-expr]
  | [pattern fender-expr result-expr]
     
pattern = np-pattern
  | (pattern ...)
  | (pattern ...+ . np-pattern)
  | (pattern ... pattern ellipsis pattern ... . np-pattern)
     
np-pattern = _
  | id
  | #(pattern ...)
  | #(pattern ... pattern ellipsis pattern ...)
  | #&pattern
  | #s(key-datum pattern ...)
  | #s(key-datum pattern ... pattern ellipsis pattern ...)
  | (ellipsis stat-pattern)
  | const
     
stat-pattern = id
  | (stat-pattern ...)
  | (stat-pattern ...+ . stat-pattern)
  | #(stat-pattern ...)
  | #&stat-pattern
  | #s(key-datum stat-pattern ...)
  | const
     
ellipsis = ...
找到第一个与 stx-expr 产生的语法对象相匹配的 pattern ,并且相应的 fender-expr (如果有的话)产生一个真值;结果来自相应的 result-expr , 它处于 syntax-case 形式的尾部位置。 如果没有 clause 匹配,那么就会引发 exn:fail:syntax exception is raised ;这个异常是通过调用 raise-syntax-error 产生的,参数是 #f 作为 "name", 一个带有通用错误信息的字符串,以及 stx-expr 的结果。

一个语法对象与一个 pattern 相匹配,如下所示:

_

一个 _ 模式(即一个与 _ 具有相同约束力的标识符,并且不在 literal-id 中)可以匹配任何语法对象。

id

当一个 id 没有与 ... 或 _ 绑定, 并且没有与任何 literal-id 相同的绑定时,它可以匹配任何语法对象。 对于相应的 fender-expr (如果有的话)和 result-expr 来说, 这个 id 被进一步绑定为 pattern variable 。 模式-变量绑定是一个转化器绑定; 模式变量只能通过 syntax 这样的形式被引用。 绑定的值是与模式匹配的语法对象,其 depth marker 为 0 。

在 stat-pattern 中, ... 不被特别对待。 它要么匹配一个 literal-id ,要么被绑定为一个模式变量。

一个与 id 有相同绑定的 literal-id 有相同绑定的id可以匹配一个语法对象, 该对象是一个有相同绑定的 free-identifier=? 的标识符。 匹配不会引入任何 pattern variables 。

(pattern ...)

(pattern ...) 模式匹配一个语法对象,其基准形式(即没有词法信息)是一个列表, 其元素数量与模式中的子 pattern 一样多, 并且每个对应于列表中某个元素的语法对象都匹配相应的子 pattern 。

任何由子 pattern 绑定的 pattern variables 都被完整的模式所绑定;这些绑定必须都是不同的。

(pattern ...+ . np-pattern)

就像之前的那种模式,但是匹配的语法对象不一定是列表; 对于最后的 np-pattern 之前的 n 个子 pattern , 语法对象的数据必须是一个对,这样 n-1 个 cdr 就会产生对。 最后的 np-pattern 与第 n 个 cdr 对应的语法对象相匹配 (或者使用最近的包围语法对象的词法上下文和源位置对数据进行 datum->syntax 强制处理)。

(pattern ... pattern ellipsis pattern ...)

类似于 (pattern ...) 类型的模式, 但是在相对于其他子 pattern 的相应位置上匹配子 pattern 后面跟着 ellipsis 的任何数量(零或更多)元素的语法对象。

对于每个由子 pattern 后面的 ellipsis 绑定的模式变量, 更大的模式将相同的模式变量绑定到一个值的列表上,每个与子 pattern 匹配的语法对象的元素都有一个, 并有一个递增的 depth marker 。 (子pattern 本身可能包含 ellipsis, 导致模式变量绑定到语法对象的列表上， depth marker 为 2,以此类推。)

所有带有 ellipsis 的模式形式只在 ellipsis 不在 literal-id 中时适用。

(pattern ... pattern ellipsis pattern ... . np-pattern)

就像之前的那种模式,但是有一个最终的 np-pattern ,因为 (pattern ...+ . np-pattern)。 最后的 np-pattern 永远不会匹配数据为一对的语法对象。

#(pattern ...)

类似于 (pattern ...) 模式, 但是匹配一个矢量语法对象,其元素与相应的子 pattern 相匹配。

#(pattern ... pattern ellipsis pattern ...)

类似于 (pattern ... pattern ellipsis pattern ...) 模式, 但匹配一个矢量语法对象,其元素与相应的子 pattern 匹配。

#&pattern

匹配一个内容与 pattern 相匹配的方框语法对象。

#s(key-datum pattern ...)

类似于 (pattern ...) 模式,但匹配一个 prefab 结构语法对象, 其字段与相应的子 pattern 相匹配。 key-datum 必须对应于 make-prefab-struct 的有效第一参数。

#s(key-datum pattern ... pattern ellipsis pattern ...)

类似于 (pattern ... pattern ellipsis pattern ...) 模式, 但是匹配一个 prefab 结构语法对象,其元素与相应的子 pattern 相匹配。

(ellipsis stat-pattern)

匹配与 stat-pattern 相同的内容,它就像一个 pattern , 但具有绑定 ... 的标识符与其他 id 一样被处理。

const

const 是指不匹配前面一种形式的任何数据; 当一个语法对象的数据与 quote const equal? 时, 它就匹配了一个 const 模式。

如果 stx-expr 产生一个非 syntax object , 那么其结果将使用 datum->syntax 和词法上下文以及 stx-expr 的源位置转换为一个语法对象。

如果 stx-expr 产生的语法对象是有 tainted , 那么任何由 pattern 绑定的语法对象都是有 tainted 。

Examples:
> (require (for-syntax racket/base))
> (define-syntax (swap stx)
    (syntax-case stx ()
      [(_ a b) #'(let ([t a])
                   (set! a b)
                   (set! b t))]))
> (let ([x 5] [y 10])
    (swap x y)
    (list x y))

'(10 5)

> (syntax-case #'(ops 1 2 3 => +) (=>)
    [(_ x ... => op) #'(op x ...)])

#<syntax:eval:675:0 (+ 1 2 3)>

> (syntax-case #'(let ([x 5] [y 9] [z 12])
                   (+ x y z))
               (let)
    [(let ([var expr] ...) body ...)
     (list #'(var ...)
           #'(expr ...))])

'(#<syntax:eval:676:0 (x y z)> #<syntax:eval:676:0 (5 9 12)>)

syntax

(syntax-case* stx-expr (literal-id ...) id-compare-expr
  clause ...)
和 syntax-case 一样,但是 id-compare-expr 必须产生一个接受两个参数的过程。 在 pattern 中的 literal-id 匹配一个标识符, 当给定要匹配的标识符(作为第一个参数)和 pattern 中的标识符(作为第二个参数)时,该过程返回 true。

换句话说, syntax-case 就像 syntax-case* , 有一个 id-compare-expr ,产生 free-identifier=? 。

syntax

(with-syntax ([pattern stx-expr] ...)
  body ...+)
类似于 syntax-case ,它将一个 pattern 与一个语法对象匹配。 与 syntax-case 不同的是,所有 pattern 都被匹配, 每个模式都与相应的 stx-expr 的结果相匹配, 所有匹配的模式变量(必须是不同的)都与一个 body 序列绑定。 with-syntax 形式的结果是最后一个 body 的结果, 相对于 with-syntax 形式而言,它处于尾部位置。

如果任何 pattern 不能匹配相应的 stx-expr, 就会引发 exn:fail:syntax exception is raised 。

with-syntax 形式大致相当于下面的 syntax-case 形式:

(syntax-case (list stx-expr ...) ()
  [(pattern ...) (let () body ...+)])

然而,如果任何一个单独的 stx-expr 产生一个非 syntax object , 那么它将被转换为一个使用 datum->syntax 和词法上下文以及单独的 stx-expr 的源位置。

Examples:
> (define-syntax (hello stx)
    (syntax-case stx ()
      [(_ name place)
       (with-syntax ([print-name #'(printf "~a\n" 'name)]
                     [print-place #'(printf "~a\n" 'place)])
         #'(begin
             (define (name times)
               (printf "Hello\n")
               (for ([i (in-range 0 times)])
                    print-name))
             (define (place times)
               (printf "From\n")
               (for ([i (in-range 0 times)])
                    print-place))))]))
> (hello jon utah)
> (jon 2)

> (define-syntax (math stx)
    (define (make+1 expression)
      (with-syntax ([e expression])
        #'(+ e 1)))

      [(_ numbers ...)

                      (map make+1
                           (syntax->list #'(numbers ...)))])

syntax

(syntax template)

 
template = id
  | (head-template ...)
  | (head-template ...+ . template)
  | #(head-template ...)
  | #&template
  | #s(key-datum head-template ...)
  | (~? template template)
  | (ellipsis stat-template)
  | const
     
head-template = template
  | head-template ellipsis ...+
  | (~@ . template)
  | (~? head-template head-template)
  | (~? head-template)
     
stat-template = like template, but without ..., ~?, and ~@
     
ellipsis = ...
根据 template 构建一个语法对象,其中可以包括由 syntax-case 或 with-syntax 绑定的 pattern variables 。

一个 template 产生一个单一的语法对象。 一个 head-template 产生一个由零个或多个语法对象组成的序列。 stat-template 和 template 一样, 除了 ..., ~? 和 ~@ 被解释为常数而不是模板形式。

一个 template 产生的语法对象如下:

id

如果 id 被绑定为 pattern variable , 那么作为模板的 id 会产生 pattern variable 的匹配结果。 除非 id 是一个子 template ,被 ellipsis 复制到一个更大的 template 中, 否则 pattern variable 的值必须是一个 depth marker 为 0 的语法对象(而不是一个匹配列表)。

更一般地说,如果 pattern variable 的值有一个深度标记 n, 那么它只能出现在一个至少被 n 个 ellipsis 复制的模板内。 在这种情况下,模板将被复制足够的次数,以便至少使用一次每个匹配结果。

如果 id 没有被绑定为模式变量,那么 id 作为模板产生 (quote-syntax id) 。

(head-template ...)

产生一个语法对象,其基准是一个列表,列表中的元素对应于由 head-template 产生的语法对象。

(head-template ... . template)

和前面的形式一样,但结果不一定是一个列表; 相反,空列表在结果的语法对象基准中的位置被 template 产生的语法对象所占据。

#(head-template ...)

与 (head-template ...) 形式类似,但产生的语法对象的基准是一个向量而不是一个列表。

#&template

产生一个语法对象,其基准点是一个盒子,里面装着由 template 产生的语法对象。

#s(key-datum head-template ...)

与 (head-template ...) 形式类似,但产生的语法对象的基准是一个 prefab 结构,而不是一个列表。 key-datum 必须对应于 make-prefab-struct 的有效的第一个参数。

(~? template1 template2)

如果 template1 没有 "缺失值" 的模式变量,产生 template1 的结果; 否则,产生 template2 的结果。

由 syntax-case 约束的模式变量永远不会有缺失值, 但由 syntax-parse 约束的模式变量(例如, ~or 或 ~optional 模式)可以有缺失值。

Examples:
> (syntax-parse #'(m 1 2 3)
    [(_ (~optional (~seq #:op op:expr)) arg:expr ...)
     #'((~? op +) arg ...)])

#<syntax:eval:3:0 (+ 1 2 3)>

> (syntax-parse #'(m #:op max 1 2 3)
    [(_ (~optional (~seq #:op op:expr)) arg:expr ...)
     #'((~? op +) arg ...)])

#<syntax:eval:4:0 (max 1 2 3)>

(ellipsis stat-template)

产生与 stat-template 相同的结果,它就像一个 template, 但 ..., ~? 和 ~@ 被当作一个 id (没有模式绑定)。

const

一个 const 模板是任何不符合前面情况的形式,它产生的结果是 (quote-syntax const) 。

一个 head-template 产生一个语法对象序列;该序列被 "内联" 到包围 template 的结果中。 head-template 的结果定义如下:

template

产生一个语法对象,根据上面 template 的规则。

head-template ellipsis ...+

通过将 head-template “map” 到其模式变量的值上,生成一序列的语法对象。 迭代的次数取决于子模板中引用的 pattern variables 的值。

更确切地说: outer 成为 inner ,后面加一个省略号。 如果一个 pattern variable 出现的深度等于它的 depth marker, 它就是 outer 的 iteration pattern variable 。 必须至少有一个;否则就会出现错误。 如果有多个迭代变量,那么它们的值都必须是相同长度的列表。 outer 的结果是通过在 iteration pattern variable 值上 map inner 模板并在 inner 内将其有效 depth markers 减少 1 而产生的。 outer 的结果是通过附加 inner 的结果形成的。

因此,如果一个 pattern variable 出现的深度大于它的 depth marker , 它就会被用作最里面的椭圆的 iteration pattern variable ,而不是最外面的。 一个 pattern variable 不能出现在一个小于其 depth marker 的深度,否则会产生一个错误。

syntax

(~@ . template)

产生由 template 产生的语法列表中的元素序列。 如果 template 没有产生一个合适的语法列表,就会引发一个异常。

Examples:
> (with-syntax ([(key ...) #'('a 'b 'c)]
                [(val ...) #'(1 2 3)])
    #'(hash (~@ key val) ...))

#<syntax:eval:2:0 (hash (quote a) 1 (quote b) 2 (quote c) 3)>

> (with-syntax ([xs #'(2 3 4)])
    #'(list 1 (~@ . xs) 5))

#<syntax:eval:3:0 (list 1 2 3 4 5)>

syntax

(~? head-template1 head-template2)

如果其模式变量没有 "缺失值",则产生 head-template1 的结果; 否则产生 head-template2 的结果。

(~? head-template)

如果 head-template 的模式变量中没有 "缺失值",则产生 head-template 的结果; 否则不产生任何结果。

相当于 (~? head-template (~@)) 。

(syntax template) 的形式通常被缩写为 #'template; 参见 读取引号 。 如果 template 不包含模式变量, 那么 #'template 就等同于 (quote-syntax template) 。

Changed in version 6.90.0.25 of package base: 增加 ~@ 和 ~? 。

syntax

(quasisyntax template)

和 syntax 一样, 但 (unsyntax expr) 和 (unsyntax-splicing expr) 转义到 template 内的表达式。

expr 必须产生一个语法对象(或语法列表),以取代准绳模板内的 unsyntax 或 unsyntax-splicing 形式,就像 quasiquote 内的 like unquote 和 unquote-splicing 一样, 只是哈希表值位置不是 quasisyntax 的转义位置。 (如果转义的表达式没有生成语法对象,它将被转换为与 with-syntax 的右侧相同的方式)。 嵌套 quasisyntax 引入准引层的方式与嵌套的 quasiquote 相同。

同样类似于 quasiquote, 读者将 #` 转换为 quasisyntax, #, 转换为 unsyntax, #,@ 转换为 unsyntax-splicing 。 参见 读取引号 。

syntax

(unsyntax expr)

作为表达式形式不合法。 unsyntax 形式只适用于 quasisyntax 模板。

syntax

(unsyntax-splicing expr)

作为表达式形式不合法。 unsyntax-splicing 形式只适用于 quasisyntax 模板。

syntax

(syntax/loc loc-expr template)

 
  loc-expr : 
(or/c #f srcloc? syntax?
      (list/c any/c
              (or/c exact-positive-integer? #f)
              (or/c exact-nonnegative-integer? #f)
              (or/c exact-positive-integer? #f)
              (or/c exact-nonnegative-integer? #f))
      (vector/c any/c
                (or/c exact-positive-integer? #f)
                (or/c exact-nonnegative-integer? #f)
                (or/c exact-positive-integer? #f)
                (or/c exact-nonnegative-integer? #f)))
与 syntax 类似,只是立即产生的语法对象从 loc-expr 的结果中获取其源位置信息。

只有直接结果的源位置–"最外层" 的语法对象–被调整。 如果 loc-expr 的源码和位置都 是#f,则源码位置 不 会被调整。 只有当产生的语法对象来自模板本身,而不是来自语法模式变量的值时,才会调整源位置。 例如,如果 x 是一个语法模式变量, 那么 (syntax/loc loc-expr x) 就不会使用 loc-expr 的位置。

Changed in version 6.90.0.25 of package base: 以前,如果 template 只是一个模式变量, syntax/loc 不会对 loc-expr 执行契约。
Changed in version 8.2.0.6: 允许 loc-expr 是 datum->syntax 接受的任何源位置值。

(quasisyntax/loc loc-expr template)

(or/c #f srcloc? syntax?
      (list/c any/c
              (or/c exact-positive-integer? #f)
              (or/c exact-nonnegative-integer? #f)
              (or/c exact-positive-integer? #f)
              (or/c exact-nonnegative-integer? #f))
      (vector/c any/c
                (or/c exact-positive-integer? #f)

像 quasisyntax 一样,但像 syntax/loc 一样有源位置分配。

Changed in version 8.2.0.6 of package base: 允许 loc-expr 成为 datum->syntax 接受的任何源位置值。

syntax

(quote-syntax/prune id)

像 quote-syntax 一样, 但是 id 的词法上下文通过 identifier-prune-lexical-context 进行修剪, 只包括 id 的符号名称和 '#%top 的绑定。 当标识符的词法信息不会被转移到其他语法对象时, 使用这种形式来引用标识符(除了可能被转移到 '#%top 的顶级绑定)。

syntax

(syntax-rules (literal-id ...)
  [(id . pattern) template] ...)
相当于

(lambda (stx)
  (syntax-case stx (literal-id ...)
    [(generated-id . pattern) (syntax-protect #'template)]  ...))

其中每个 generated-id 在相应的 template 中没有绑定标识符。

syntax

(syntax-id-rules (literal-id ...)
  [pattern template] ...)
相当于

(make-set!-transformer
 (lambda (stx)
   (syntax-case stx (literal-id ...)
     [pattern (syntax-protect #'template)]  ...)))

syntax

(define-syntax-rule (id . pattern) template)

相当于

(define-syntax id
  (syntax-rules ()
   [(id . pattern) template]))

但语法错误可能是用 pattern 来表述的。

syntax

...

... 转化器绑定禁止将 ... 作为表达式使用。 这个绑定只在语法模式和模板(或其他不相关的表达形式,像 -> 一样特别对待它)中有用,它表示模式或模板的重复。 参见 syntax-case 和 syntax 。

syntax

_

_ 转化器绑定禁止将 _ 作为表达式使用。 这种绑定只在语法模式中有用,它表示匹配任何语法对象的模式。 参见 syntax-case 。

syntax

~?

syntax

~@

~? 和 ~@ 变换器绑定禁止将这些形式作为表达式使用。 这些绑定只在语法模板中有用。参见 syntax 。

Added in version 6.90.0.25 of package base.

procedure

(syntax-pattern-variable? v) → boolean?

  v : any/c
如果 v 是一个值,作为转化器的绑定值,使绑定的变量成为 syntax 和其他形式的模式变量, 则返回 #t 。 要检查一个标识符是否是模式变量,可以使用 syntax-local-value 来获取标识符的转化器值, 然后用 syntax-pattern-variable? 测试该值。

syntax-pattern-variable? 程序是由 racket/base 提供的 for-syntax 。
