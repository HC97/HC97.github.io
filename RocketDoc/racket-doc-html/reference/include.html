12.11 文件包含

 (require racket/include) package: base
The bindings documented in this section are provided by the racket/include and racket libraries, but not racket/base.

syntax

(include path-spec)

 
path-spec = string
  | (file string)
  | (lib string ...+)
内嵌文件中的语法,由 path-spec 指定,以代替 include 表达式。

path-spec 类似于 require 的 mod-path 形式的一个子集, 但它指定了一个文件,其内容不需要是一个模块。 也就是说, string 指的是使用独立于平台的相对路径的文件, (file string) 指的是使用特定平台符号的文件, (lib string ...) 指的是一个集合中的文件。

如果 path-spec 指定了一个相对路径,那么如果该源是一个完整的路径字符串, 那么该路径将相对于 include 表达式的源进行解析。 如果源文件不是一个完整的路径字符串,那么如果 path-spec 不是 #f, 它将被解析为相对于 (current-load-relative-directory) , 否则就是相对于 (current-directory) 。

被包含的语法被赋予 include 表达式的词法上下文,而被包含的语法的源位置是指它的实际源。

syntax

(include-at/relative-to context source path-spec)

与 include 类似,除了 context 的词法上下文被用于包含的语法, 并且相对于 source 的source解析一个 path-spec 。 否则,展开将丢弃 context 和 source 元素。

syntax

(include/reader path-spec reader-expr)

和 include 一样,除了由表达式 reader-expr 产生的程序被用来读取包含的文件, 而不是 read-syntax 。

reader-expr 在扩展时在 transformer environment 中被评估。 由于它作为 read-syntax 的替代,表达式的值应该是一个过程, 它消耗两个输入–代表源的字符串和一个输入端口–产生一个语法对象或 eof 。 这个过程将被反复调用,直到产生 eof 。

该过程返回的语法对象应该有源位置信息,但通常没有词法上下文;语法对象中的任何词法上下文都将被忽略。

syntax

(include-at/relative-to/reader context source path-spec reader-expr)

结合 include-at/relative-to 和 include/reader 。
