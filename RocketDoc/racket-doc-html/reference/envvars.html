15.7 环境变量

一个 environment variable set 集封装了一个从字节串到字节串的部分映射。 Racket 进程的初始 environment variable set 与操作系统的环境变量相连: 对环境变量集的访问或改变会读取或改变 Racket 进程的操作系统环境变量。

由于 Windows 的环境变量是不区分大小写的,所以 Windows 上的 environment variable set 的关键字节串是大小写折叠的。 更确切地说,关键字节字符串被强制为 UTF-8 编码的字符,通过 string-locale-downcase 转换为小写。

当前的 environment variable set ,由 current-environment-variables 参数决定, 在 subprocess 创建时传播到 subprocess 中。

procedure

(environment-variables? v) → boolean?

  v : any/c
如果 v 是一个 environment variable set ,返回 #t ,否则返回 #f 。

parameter

(current-environment-variables) → environment-variables?

(current-environment-variables env) → void?
  env : environment-variables?
一个确定 environment variable set 的 parameter , 它被传播到 subprocess 中,并被用作 getenv 和 putenv 的默认集。

procedure

(bytes-environment-variable-name? v) → boolean?

  v : any/c
如果 v 是一个字节字符串,并且对环境变量名有效,则返回 #t 。 一个环境变量名必须不包含值为 0 或 61 的字节,其中 61 是 (char->integer #\=) 。 在 Windows 上,一个环境变量名也必须有一个非零的长度。

procedure

(make-environment-variables name val ... ...)

 → environment-variables?
  name : bytes-environment-variable-name?
  val : bytes-no-nuls?
创建一个新的 environment variable set ,用给定的 name 到 val 的映射进行初始化。

procedure

(environment-variables-ref env name)

 → (or/c #f (and/c bytes-no-nuls? immutable?))
  env : environment-variables?
  name : bytes-environment-variable-name?
返回 env 中 name 的映射,如果 name 没有映射则返回 #f 。

通常情况下, name 应该是一个使用当前 locale 默认编码的字符串的字节字符串编码。 在 Windows 中, name 被强制为 UTF-8 编码,并且大小写正常化。

procedure

(environment-variables-set! env    
  name    
  maybe-bstr    
  [fail]) → any
  env : environment-variables?
  name : bytes-environment-variable-name?
  maybe-bstr : (or/c bytes-no-nuls? #f)
  fail : (-> any) = 
(lambda ()
  (raise (make-exn:fail ....)))
将 env 中 name 的映射改为 maybe-bstr 。 如果 maybe-bstr 是 #f , 并且 env 是Racket进程的初始 environment variable set , 那么 name 的操作系统环境变量映射将被移除。

通常情况下, name 和 maybe-bstr 应该是使用当前 locale 的默认编码的字符串的字节串编码。 在 Windows 中,如果 env 是 Racket 进程的初始 environment variable set , name 将被强制为 UTF-8 编码并进行大小写规范化处理, 而 maybe-bstr 则被强制为 UTF-8 编码。

成功时, environment-variables-set! 的结果是 #<void> 。 如果 env 是 Racket 进程的初始 environment variable set , 那么试图调整操作系统的环境变量映射可能会因为某些原因而失败,在这种情况下, fail 会在相对于 environment-variables-set! 的尾部位置被调用。 默认的 fail 会引发一个异常。

procedure

(environment-variables-names env)

 → (listof (and/c bytes-environment-variable-name? immutable?))
  env : environment-variables?
返回一个对应于由 env 映射的名字的字节字符串的列表。

procedure

(environment-variables-copy env) → environment-variables?

  env : environment-variables?
返回一个 environment variable set ,它的初始化与 env 的映射相同。

procedure

(getenv name) → (or/c string-no-nuls? #f)

  name : string-environment-variable-name?

procedure

(putenv name value) → boolean?

  name : string-environment-variable-name?
  value : string-no-nuls?
对 environment-variables-ref 和 environment-variables-set! 的方便封装, 在字符串和字节字符串之间转换,使用当前 locale 的默认编码(使用 #\? 作为编码错误的替换字符), 并且总是使用当前 current-environment-variables 中的 environment variable set 。 putenv 函数成功时返回 #t ,失败时返回 #f 。

procedure

(string-environment-variable-name? v) → boolean?

  v : any/c
如果 v 是一个字符串,并且根据 bytes-environment-variable-name? , 其使用当前 locale 的编码对环境变量名称有效,则返回 #t 。
