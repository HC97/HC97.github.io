4.3.2 通用数字运算

大多数 Racket 的数字操作都是针对任何类型的数字。

4.3.2.1 Arithmetic

procedure

(+ z ...) → number?

  z : number?
返回 z 的总和,从左到右成对地添加。 如果没有提供参数,结果是 0 。

Examples:
> (+ 1 2)

3

> (+ 1.0 2+3i 5)

8.0+3.0i

> (+)

0

procedure

(- z) → number?

  z : number?
(- z w ...+) → number?
  z : number?
  w : number?
当没有提供 w 时,返回 (- 0 z) 。 否则,返回从 z 中减去 w 的结果,从左到右成对进行。

Examples:
> (- 5 3.0)

2.0

> (- 1)

-1

> (- 2+7i 1 3)

-2+7i

procedure

(* z ...) → number?

  z : number?
返回 z 的乘积,从左到右成对相乘。 如果没有提供参数,结果是 1 。 任何数字与精确的 0 相乘都会产生精确的 0 。

Examples:
> (* 2 3)

6

> (* 8.0 9)

72.0

> (* 1+2i 3+4i)

-5+10i

procedure

(/ z) → number?

  z : number?
(/ z w ...+) → number?
  z : number?
  w : number?
当没有提供 w 时,返回 (/ 1 z) 。 否则,返回 z 除以 w 的结果,从左到右依次进行。

如果 z 是精确 0 ,并且没有 w 是精确 0 , 那么结果就是精确 0 。 如果有任何 w 是精确的 0, 就会引发 exn:fail:contract:divide-by-zero exception is raised 。

Examples:
> (/ 3 4)

3/4

> (/ 81 3 3)

9

> (/ 10.0)

0.1

> (/ 1+2i 3+4i)

11/25+2/25i

procedure

(quotient n m) → integer?

  n : integer?
  m : integer?
返回 (truncate (/ n m)) 。

Examples:
> (quotient 10 3)

3

> (quotient -10.0 3)

-3.0

> (quotient +inf.0 3)

quotient: contract violation

  expected: integer?

  given: +inf.0

procedure

(remainder n m) → integer?

  n : integer?
  m : integer?
返回与 n 相同符号的 q ,以便

  • (abs q) 在 0 (包括)和 (abs m) (包括)之间,并且

  • (+ q (* m (quotient n m))) 等于 n 。

如果 m 正好是 0 ,就会引发 exn:fail:contract:divide-by-zero exception is raised。

Examples:
> (remainder 10 3)

1

> (remainder -10.0 3)

-1.0

> (remainder 10.0 -3)

1.0

> (remainder -10 -3)

-1

> (remainder +inf.0 3)

remainder: contract violation

  expected: integer?

  given: +inf.0

procedure

(quotient/remainder n m) → 
integer? integer?
  n : integer?
  m : integer?
返回 (values (quotient n m) (remainder n m)), 但组合的计算可能比单独调用 quotient 和 remainder 更有效。

Example:
> (quotient/remainder 10 3)

3

1

procedure

(modulo n m) → integer?

  n : integer?
  m : integer?
返回与 m 相同符号的 q ,其中

  • (abs q) 在 0 (包括)和 (abs m) (不包括)之间,并且

  • q 和 (- n (* m (quotient n m))) 之间的差是 m 的倍数。

如果 m 正好是 0, 就会引发 exn:fail:contract:divide-by-zero exception is raised。

Examples:
> (modulo 10 3)

1

> (modulo -10.0 3)

2.0

> (modulo 10.0 -3)

-2.0

> (modulo -10 -3)

-1

> (modulo +inf.0 3)

modulo: contract violation

  expected: integer?

  given: +inf.0

procedure

(add1 z) → number?

  z : number?
返回 (+ z 1) 。

procedure

(sub1 z) → number?

  z : number?
返回 (- z 1) 。

procedure

(abs x) → number?

  x : real?
返回 x 的绝对值。

Examples:
> (abs 1.0)

1.0

> (abs -1)

1

procedure

(max x ...+) → real?

  x : real?
返回 x 中最大的一个,如果任何一个 x 是 +nan.0 , 则返回 +nan.0 。 如果任何一个 x 是不确切的,结果将被强制为不确切的。 参见 argmax。

Examples:
> (max 1 3 2)

3

> (max 1 3 2.0)

3.0

procedure

(min x ...+) → real?

  x : real?
返回 x 中最小的一个,如果任何一个 x 是 +nan.0 , 则返回 +nan.0 。 如果任何一个 x 是不精确的,结果将被强制为不精确的。 参见 argmin 。

Examples:
> (min 1 3 2)

1

> (min 1 3 2.0)

1.0

procedure

(gcd n ...) → rational?

  n : rational?
返回 n 的 greatest common divisor (一个非负数); 对于非整数 n ,结果是分子的 gcd 除以分母的 lcm 。 如果没有提供参数,结果是 0 ;如果所有参数都是0,结果是0。

Examples:
> (gcd 10)

10

> (gcd 12 81.0)

3.0

> (gcd 1/2 1/3)

1/6

procedure

(lcm n ...) → rational?

  n : rational?
返回 n 的 least common multiple 一个非负数); 非整数 n ,结果是乘积的绝对值除以 gcd 。 如果没有提供参数,结果是 1 。 如果任何参数是 0,结果是 0;此外,如果任何参数是精确的 0, 结果是精确的 0。

Examples:
> (lcm 10)

10

> (lcm 3 4.0)

12.0

> (lcm 1/2 2/3)

2

procedure

(round x) → (or/c integer? +inf.0 -inf.0 +nan.0)

  x : real?
返回最接近 x 的整数,解决并列问题,有利于偶数, 但 +inf.0, -inf.0 和 +nan.0 都是四舍五入。

Examples:
> (round 17/4)

4

> (round -17/4)

-4

> (round 2.5)

2.0

> (round -2.5)

-2.0

> (round +inf.0)

+inf.0

procedure

(floor x) → (or/c integer? +inf.0 -inf.0 +nan.0)

  x : real?
返回不超过 x 的最大整数, 但 +inf.0, -inf.0 和 +nan.0 的底限为自己。

Examples:
> (floor 17/4)

4

> (floor -17/4)

-5

> (floor 2.5)

2.0

> (floor -2.5)

-3.0

> (floor +inf.0)

+inf.0

procedure

(ceiling x) → (or/c integer? +inf.0 -inf.0 +nan.0)

  x : real?
返回至少与 x 一样大的最小的整数, 但是 +inf.0, -inf.0 和 +nan.0 的上限为自己。

Examples:
> (ceiling 17/4)

5

> (ceiling -17/4)

-4

> (ceiling 2.5)

3.0

> (ceiling -2.5)

-2.0

> (ceiling +inf.0)

+inf.0

procedure

(truncate x) → (or/c integer? +inf.0 -inf.0 +nan.0)

  x : real?
返回离 0 最远的整数,即距离 0 不比 x ,但 +inf.0, -inf.0, and +nan.0 截断到自己。

Examples:
> (truncate 17/4)

4

> (truncate -17/4)

-4

> (truncate 2.5)

2.0

> (truncate -2.5)

-2.0

> (truncate +inf.0)

+inf.0

procedure

(numerator q) → integer?

  q : rational?
将 q 胁迫为一个精确的数字,找到以最简单的分数形式表示的数字的分子,并将这个数字胁迫为 q 的精确性。

Examples:
> (numerator 5)

5

> (numerator 17/4)

17

> (numerator 2.3)

2589569785738035.0

procedure

(denominator q) → (and/c integer? positive?)

  q : rational?
将 q 胁迫为一个精确的数字,找到以最简单的分数形式表示的数字的分母, 并返回这个被胁迫为 q 的精确性的数字。

Examples:
> (denominator 5)

1

> (denominator 17/4)

4

> (denominator 2.3)

1125899906842624.0

procedure

(rationalize x tolerance) → real?

  x : real?
  tolerance : real?
在 x 的 (abs tolerance) 范围内的实数中,返回对应于精确数的 denominator 最小的那个。 如果多个整数在 x 的 tolerance 内,则使用最接近 0 的那个。

Examples:
> (rationalize 1/4 1/10)

1/3

> (rationalize -1/4 1/10)

-1/3

> (rationalize 1/4 1/4)

0

> (rationalize 11/40 1/4)

1/2

4.3.2.2 数字比较

procedure

(= z w ...) → boolean?

  z : number?
  w : number?
如果所有的参数在数字上都相等, 则返回 #t ,否则返回 #f 。 当一个不确切的数字的确切协数是一个确切的数字时,这个不确切的数字在数值上等于一个确切的数字。 另外, 0.0 和 -0.0 在数值上是相等的,但是 +nan.0 在数值上不等于它自己。

Examples:
> (= 1 1.0)

#t

> (= 1 2)

#f

> (= 2+3i 2+3i 2+3i)

#t

> (= 1)

#t

4.3.2.3 平方和开方

procedure

(sqrt z) → number?

  z : number?
返回 z 的主 square root 。 如果 z 是精确的,并且 z 的平方根是有理的,则结果是精确的。 参见 integer-sqrt 。

Examples:
> (sqrt 4/9)

2/3

> (sqrt 2)

1.4142135623730951

> (sqrt -1)

0+1i

procedure

(integer-sqrt n) → complex?

  n : integer?
返回正 n 的 (floor (sqrt n)) 。 如果 n 是精确的,结果就是精确的。 对于负数 n, 结果是 (* (integer-sqrt (- n)) 0+1i) 。

Examples:
> (integer-sqrt 4.0)

2.0

> (integer-sqrt 5)

2

> (integer-sqrt -4.0)

0.0+2.0i

> (integer-sqrt -4)

0+2i

procedure

(integer-sqrt/remainder n) → 
complex? integer?
  n : integer?
返回 (integer-sqrt n) 和 (- n (expt (integer-sqrt n) 2)) 。

Examples:
> (integer-sqrt/remainder 4.0)

2.0

0.0

> (integer-sqrt/remainder 5)

2

1

procedure

(expt z w) → number?

  z : number?
  w : number?
返回将 z 提高到 w 的幂的结果。

如果 w 是精确的 0, 结果是精确的 1 。 如果 w 是 0.0 或 -0.0 , 并且 z 是一个非精确 1 或 0 的 real number , 结果是 1.0 (即使 z 是 +nan.0)。

如果 z 是精确的 1, 结果就是精确的 1 。 如果 z 是 1.0 并且 w 是一个 real number, 结果是 1.0 (即使 w 是 +nan.0)。

如果 z 是精确的 0, 结果如下:

  • w 是精确的 0 — 结果是 1

  • w 是 0.0 或 -0.0 — 结果是 1.0

  • w 的实数部分为负数–引发 exn:fail:contract:divide-by-zero exception is raised

  • w 是非实数,实数部分为非正数–引发 exn:fail:contract:divide-by-zero exception is raised

  • w 是 +nan.0 — 结果是 +nan.0

  • 否则–结果为 0

如果 w 是精确的 1/2, 结果与 (sqrt z) 相同,可以是精确的。 其他分数幂不会以这种方式被特别处理:

Examples:
> (expt 9 1/2)

3

> (expt 9 0.5)

3.0

> (expt 16 1/4)

2.0

> (expt 16 0.25)

2.0

当 w 是一个 real number 时的进一步特殊情况: 这些特殊情况与 C99 [C99] 中的 pow 相对应, 除了 z 为负数和 w 为非整数时。

  • (expt 0.0 w):

    • w 为负数 - 结果为 +inf.0

    • w 是正数–结果是 0.0

  • (expt -0.0 w):

    • w 为负数:

      • w 是奇数 - 结果是 -inf.0

      • w 为有理数–结果为 +inf.0

    • w 是正数:

      • w 是一个奇数整数–结果是 -0.0

      • w 否则有理数–结果为 0.0

  • (expt z -inf.0) 为正 z:

    • z 小于 1.0 — 结果为 +inf.0

    • z 大于 1.0 — 结果为 0.0

  • (expt z +inf.0) 为正 z:

    • z 小于 1.0 — 结果为 0.0

    • z 大于 1.0 — 结果为 +inf.0

  • (expt -inf.0 w) 对于整数 w:

    • w 是负数:

      • w 是奇数 - 结果是 -0.0

      • w 是偶数 - 结果是 0.0

    • w 是正数:

      • w 是奇数 - 结果是 -inf.0

      • w 是偶数 - 结果是 +inf.0

  • (expt +inf.0 w):

    • w 是负数 - 结果是 0.0

    • w 是正数 - 结果是 +inf.0

Examples:
> (expt 2 3)

8

> (expt 4 0.5)

2.0

> (expt +inf.0 0)

1

procedure

(exp z) → number?

  z : number?
返回欧拉数升至 z 的幂。 其结果通常是不精确的,但当 z 是精确的 0 时,它是精确的 1 。 参见 expt 。

Examples:
> (exp 1)

2.7182818284590455

> (exp 2+3i)

-7.315110094901103+1.0427436562359045i

> (exp 0)

1

procedure

(log z [b]) → number?

  z : number?
  b : number? = (exp 1)
返回 z 的自然对数。 结果通常是不精确的,但是当 z 是精确的 1 时,它是精确的 0 。 当 z 是精确的 0 时,会引发 exn:fail:contract:divide-by-zero exception is raised。

如果提供了 b ,它可以作为一个替代基数。 它等同于 (/ (log z) (log b)), 但有可能运行得更快。 如果 b 是精确的 1,会引发 exn:fail:contract:divide-by-zero exception is raised。

当准确性很重要时,可以考虑使用 math/flonum 中的 fllogb 代替。

Examples:
> (log (exp 1))

1.0

> (log 2+3i)

1.2824746787307684+0.982793723247329i

> (log 1)

0

> (log 100 10)

2.0

> (log 8 2)

3.0

> (log 5 5)

1.0

Changed in version 6.9.0.1 of package base: Added second argument for arbitrary bases.

  z : number?

-0.9999999999964793

> (cos 1.0+5.0i)

  z : number?

> (tan 1.0+5.0i)

8.256719834229597e-5+1.0000377833796008i

procedure

procedure

(bitwise-bit-field n start end) → exact-integer?

  n : exact-integer?
  start : exact-nonnegative-integer?
  end : 
(and/c exact-nonnegative-integer?
       (>=/c start))
从 n 中提取位置 start 和 (- end 1) (包括)之间的位, 并将它们下移到数字的最小有效部分。

这个操作等同于计算

(bitwise-and (sub1 (arithmetic-shift 1 (- end start)))
             (arithmetic-shift n (- start)))

下面的每一对例子都使用了相同的数字,以二进制和整数形式显示结果。

Examples:
> (format "~b" (bitwise-bit-field (string->number "1101" 2) 1 1))

"0"

> (bitwise-bit-field 13 1 1)

0

> (format "~b" (bitwise-bit-field (string->number "1101" 2) 1 3))

"10"

> (bitwise-bit-field 13 1 3)

2

> (format "~b" (bitwise-bit-field (string->number "1101" 2) 1 4))

"110"

> (bitwise-bit-field 13 1 4)

6

(arithmetic-shift n m) → exact-integer?

> (arithmetic-shift 255 -3)

4.3.2.7 随机数

procedure

(random k [rand-gen]) → exact-nonnegative-integer?

  k : (integer-in 1 4294967087)
  rand-gen : pseudo-random-generator?
   = (current-pseudo-random-generator)
(random min max [rand-gen]) → exact-integer?
  min : exact-integer?
  max : (integer-in (+ 1 min) (+ 4294967087 min))
  rand-gen : pseudo-random-generator?
   = (current-pseudo-random-generator)
(random [rand-gen]) → (and/c real? inexact? (>/c 0) (</c 1))
  rand-gen : pseudo-random-generator?
   = (current-pseudo-random-generator)
当用一个整数参数 k 调用时,返回一个 0 到 k-1 范围内的随机精确整数。

当用两个整数参数 min 和 max 调用时, 返回 min 到 max-1 范围内的一个随机的精确整数。

当以 0 个参数调用时,返回一个 0 到 1 之间的随机非精确数字,不包括在内。

在每种情况下,数字都是由给定的伪随机数生成器提供的(默认为当前的,由 current-pseudo-random-generator 产生)。 生成器保持一个内部状态来生成数字。 随机数生成器使用 L’Ecuyer 的 [L'Ecuyer02] ,其状态空间实际上是 192 位。

如果担心安全问题,请使用 crypto-random-bytes 而不是 random 。

racket/math 库提供了 additional functions for random number generation ,而不受 4294967087 的限制。

Changed in version 6.4 of package base: 增加了对范围的支持。

procedure

(random-seed k) → void?

  k : (integer-in 0 (sub1 (expt 2 31)))
用 k 对当前的伪随机数发生器进行播种。 给一个生成器播种可以确定地设置它的内部状态;也就是说,用一个特定的数字给生成器播种, 可以迫使它产生一串跨运行和跨平台的伪随机数。

random-seed 函数在某些情况下是很方便的, 但是请注意,伪随机数发生器的状态空间要比 k 的允许值空间大很多。 使用 vector->pseudo-random-generator! 可以将一个伪随机数发生器设置为其任何可能的状态。

procedure

(real->decimal-string n [decimal-digits]) → string?

将实数 x 转换为长度为 size-n 的字节串中的IEEE表示, 该字节串必须为 4 或 8 。 如果 big-endian? 为真,那么数字中最有意义的 8 位将被编码在所产生的字节串的第一个字节中, 否则最没有意义的位将被编码在第一个字符中,以此类推。

dest-bstr 参数必须是一个长度为 size-n 的可变字节字符串。 n 的编码从字节 start 开始被写入 dest-bstr ， dest-bstr 作为结果被返回。

如果提供的 dest-bstr 少于 start 和 size-n 字节, 将引发 exn:fail:contract exception is raised 。

procedure

(system-big-endian?) → boolean?

如果运行 Racket 的机器对数字的本地编码是大编码,则返回 #t ; 如果本地编码是小编码,则返回 #f 。
