3.23 块: block

 (require racket/block) package: base
The bindings documented in this section are provided by the racket/block library, not racket/base or racket.

syntax

(block defn-or-expr ...)

支持表达式和相互递归定义的混合,就像在 module 主体中一样。 与 internal-definition context 不同, 最后一个 defn-or-expr 不需要是一个表达式。

如果 block 形式是一个表达式, 那么它的结果就是最后一个 defn-or-expr 的结果, 否则就是 #<void> 。 如果没有提供 defn-or-expr (在扁平化 begin 形式之后), 结果是 #<void>。

如果最后的 defn-or-expr 是一个表达式,则在尾部位置执行。

Examples:
> (define (f x)
    (block
      (define y (add1 x))
      (displayln y)
      (define z (* 2 y))
      (+ 3 z)))
> (f 12)
