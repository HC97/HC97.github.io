13.1.4 计算位置、行和列

默认情况下,Racket 跟踪端口中的 position , 即从任何端口读取或写入的字节数(与读/写位置无关,读/写位置是通过 file-position 访问或改变的)。 然而,Racket 可以选择以字符(UTF-8 解码后)而不是字节来跟踪位置, 并且可以跟踪 line locations 和 column locations; 这种可选的跟踪必须通过 port-count-lines! 或 port-count-lines-enabled 参数为端口特别启用。 一个端口的位置、行和列的位置由 read-syntax 使用。 位置和行的位置是从 1 开始编号的;列的位置是从 0 开始编号的。

在计算行数时,Racket 将换行符、回车符和回车-换行符的组合视为行的结束符和单个位置(在所有平台)。 每个制表符都会将列数推进到下一个 8 的倍数之前。 当 128 到 253 范围内的字节序列构成一个字符的 UTF-8 编码时,位置/列会为每个字节递增一次, 当发现一个完整的编码序列时,再适当递减。关于端口的 UTF-8 解码的更多信息, 还请参见 端口 。

任何一个端口的位置都是已知的,只要它的值可以表示为一个 fixnum (对于现实的应用来说,例如语法错误报告,这已经是足够的跟踪了)。 如果一个端口的位置超过了最大的 fixnum 的值,那么这个端口的位置就变成了未知的,并且行和列的跟踪被禁用。 只有在启用行和列计数时,回车-换行组合才会被当作单个字符的位置。

Custom ports 可以定义自己的计数函数,这些函数不受上述规则的约束, 只是只有在用 port-count-lines! 特别启用跟踪时才会调用这些计数函数。

procedure

(port-count-lines! port) → void?

  port : port?
开启一个 port 的 line location 和 column location 计数。 计数功能可以在任何时候开启,不过通常是在从端口读取或写入任何数据之前开启。 在开启行计数时, port-next-location 通常会开始报告自启用行计数以来读取的字符数(多一个), 而不是自端口打开以来读取的字节数(多一个),作为其最后的结果。

当一个端口被创建时,如果 port-count-lines-enabled 参数的值为 true,那么该端口的行计数就会自动启用。 在启用行数统计后,不能再禁用端口的行数统计。

procedure

(port-counts-lines? port) → boolean?

  port : port?
如果 port 的 line location 和 column location 计数已被启用, 则返回 #t ,否则返回 #f 。

procedure

(port-next-location port)

 → 
(or/c exact-positive-integer? #f)
(or/c exact-nonnegative-integer? #f)
(or/c exact-positive-integer? #f)
  port : port?
返回三个值:一个整数或 #f 表示下一个读/写项目的行号, 一个整数或 #f 表示下一个项目的列,以及一个整数或 #f 表示下一个项目的位置。 下一个列和位置通常会随着从端口读出或写入的字节而增加,但如果为 port 启用了行/字符计数, 在读出或写入一个结束UTF-8编码序列的字节后,列和位置结果会减少。

如果一个端口没有启用行计数,那么前两个结果是 #f, 最后一个结果是比到目前为止读到的字节数多一个。 在启用行计数时,前两个结果通常变成非 #f, 最后一个结果开始报告字符而不是字节,通常从启用行计数时开始。

即使启用了行计数,如果端口无法跟踪行、列或位置,它也可能返回 #f 值。

procedure

(set-port-next-location! port    
  line    
  column    
  position) → void?
  port : port?
  line : (or/c exact-positive-integer? #f)
  column : (or/c exact-nonnegative-integer? #f)
  position : (or/c exact-positive-integer? #f)
设置 port 的下一个行、列和位置。 如果 port 没有启用计行功能, 或者 port 是定义了自己的计行功能的 custom port , 则 set-port-next-location! 没有作用。

parameter

(port-count-lines-enabled) → boolean?

(port-count-lines-enabled on?) → void?
  on? : any/c
一个决定是否为新创建的 port 自动启用行计数的 parameter 。 默认值是 #f 。
