12.8 Syntax Taints

代码检查员和语法污染 in Racket 指南 introduces syntax taints.

一个 tainted 的标识符会被宏扩展器拒绝作为绑定或表达式使用。 如果一个语法对象 stx 是 tainted, 那么 (syntax-e stx) 的结果中的任何语法对象都是 tainted, 并且以 stx 为第一参数的 datum->syntax 会产生一个 tainted 的语法对象。 (syntax-property stx key) 的结果中的任何语法对象也是有污点的, 如果它位于 datum->syntax 的转换所能到达的值中的某个位置。污点不能被移除。

当一个语法对象被宏扩展器包含在一个异常中时, 或者当它被像 expand 这样的函数使用一个不是原始代码检查器的 code inspector 产生时,它就被污损了。 函数 syntax-taint 也会返回一个有污点的语法对象。

以前的 Racket 版本包括一个 arming 和 disarming 语法的概念,以触发污点或避免污点。 现在已经不支持这种间接性了,操作 syntax-arm, syntax-disarm, syntax-rearm 和 syntax-protect 对其参数没有影响。 与此类似,语法属性(见 语法对象属性) 'taint-mode 和 'certify-mode 以前是用来控制语法武装的, 现在已经不再被宏扩展器特别识别。

procedure

(syntax-tainted? stx) → boolean?

  stx : syntax?
如果 stx 是 tainted,返回 #t ,否则返回 #f 。

procedure

(syntax-arm stx [inspector use-mode?]) → syntax?

  stx : syntax?
  inspector : (or/c inspector? #f) = #f
  use-mode? : any/c = #f
返回 stx 。

Changed in version 8.2.0.4 of package base: 改为只返回 stx ,而不是返回 "武装" 语法。

procedure

(syntax-protect stx) → syntax?

  stx : syntax?
返回 stx 。

Changed in version 8.2.0.4 of package base: 改为只返回 stx ,而不是返回 "武装" 语法。

procedure

(syntax-disarm stx inspector) → syntax?

  stx : syntax?
  inspector : (or/c inspector? #f)
返回 stx 。

Changed in version 8.2.0.4 of package base: 改为只返回 stx ,而不是潜在的 "解除武装" 语法。

procedure

(syntax-rearm stx from-stx [use-mode?]) → syntax?

  stx : syntax?
  from-stx : syntax?
  use-mode? : any/c = #f
返回 stx 。

Changed in version 8.2.0.4 of package base: 改为只返回 stx 而不是潜在的 "布防" 语法。

procedure

(syntax-taint stx) → syntax?

  stx : syntax?
返回 stx 的污点版本,如果它已经被 tainted ,那么它就是 stx 。
