13.8 Printer Extension

value

gen:custom-write : any/c

一个 generic interface (见 通用接口) , 它提供了一个方法,即默认打印机用来 display, write 或 print 结构类型的实例的 write-proc 。

A write-proc 方法需要三个参数:要打印的结构、目标端口和一个参数, 其中 #t 表示 write模式, #f 表示 display 模式, 或 0 或 1 表示 print 模式的当前 quoting depth 。 该程序应该使用 write, display, print, fprintf, write-special 等方式将值打印到给定的端口。

port write handler, port display handler 和 print handler 都是为给定的自定义写程序的端口特别配置的。 通过 display, write 或 print 向端口打印,会递归地打印一个带有共享注释的值。 要避免递归打印(即,打印时不考虑与当前正在打印的值共享),可以打印到一个字符串或管道, 然后使用 write-string 或 write-special 将结果传送到目标端口。 要向自定义写程序所给的端口以外的端口递归打印,需要将给定端口的写处理程序、显示处理程序和打印处理程序复制到其他端口。

给予 write-proc 的端口不一定是实际的目标端口。 特别是,为了检测循环、共享和引用模式(在 print 的情况下), 打印机调用一个自定义写程序的端口,记录递归打印的信息,而不保留任何其他输出。 这个信息收集阶段需要与后来打印的对象(在 eq? 意义上)相同,这样记录的信息就可以与打印值相关联。

递归打印操作可能会触发对 write-proc 的调用的逃逸。 例如,在 pretty-printing 过程中,打印可能会逃逸,因为暂定的打印尝试会溢出行, 或者在打印被限制在有限宽度的错误输出时,它可能逃逸。

下面是一个 tuple 类型的定义示例,包括一个 write-proc 过程, 该过程在 write 和 print 模式下使用角括号打印元组的列表内容, 在 display 模式下不使用角括号。

Examples:
(define (tuple-print tuple port mode)
  (when mode (write-string "<" port))
  (let ([l (tuple-ref tuple)]
        [recur (case mode
                 [(#t) write]
                 [(#f) display]
                 [else (lambda (p port) (print p port mode))])])
    (unless (zero? (vector-length l))
      (recur (vector-ref l 0) port)
      (for-each (lambda (e)
                  (write-string ", " port)
                  (recur e port))
                (cdr (vector->list l)))))
  (when mode (write-string ">" port)))

 

(struct tuple (ref)
        #:methods gen:custom-write
        [(define write-proc tuple-print)])

 

> (display (tuple #(1 2 "a")))

1, 2, a

> (print (tuple #(1 2 "a")))

<1, 2, "a">

> (let ([t (tuple (vector 1 2 "a"))])
    (vector-set! (tuple-ref t) 0 t)
    (write t))

#0=<#0#, 2, "a">

make-constructor-style-printer 函数可以帮助实现一个 write-proc, 如本例:

Examples:
(require racket/struct)

 

(struct point (x y)
  #:methods gen:custom-write
  [(define write-proc
     (make-constructor-style-printer
      (lambda (obj) 'point)
      (lambda (obj) (list (point-x obj) (point-y obj)))))])

 

> (print (point 1 2))

(point 1 2)

> (write (point 1 2))

#<point: 1 2>

Changed in version 8.7.0.5 of package base: 增加了一个检查,所以现在省略 write-proc 是一个语法错误。

value

prop:custom-write : struct-type-property?

一个 structure type property (见 结构类型属性) 提供了一个与 gen:custom-write 的 write-proc 相对应的过程。 我们不鼓励使用 prop:custom-write 属性; 请使用 gen:custom-write generic interface 。

procedure

(custom-write? v) → boolean?

  v : any/c
如果 v 具有 prop:custom-write 属性,返回 #t ,否则返回 #f 。

procedure

(custom-write-accessor v)

 → (custom-write? output-port? (or/c #t #f 0 1) . -> . any)
  v : custom-write?
返回与 v 相关的自定义写程序。

value

prop:custom-print-quotable : struct-type-property?

value

custom-print-quotable? : struct-type-property?

value

custom-print-quotable-accessor : struct-type-property?

一个属性和相关的谓词和访问器。 属性值是 'self, 'never, 'maybe 或 'always 中的一个。 当一个结构除了 prop:custom-write 属性值之外还有这个属性时, 那么这个属性值会影响 print 模式下的打印;见 打印机 。 当一个值没有 prop:custom-print-quotable 时,它相当于拥有 'self 属性值, 这既适用于自带引号的表单,也适用于不可读的打印表单。
