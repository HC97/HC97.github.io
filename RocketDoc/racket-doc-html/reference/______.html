12.9 扩展顶层形式

procedure

(expand top-level-form [insp]) → syntax?

  top-level-form : any/c
  insp : inspector? = (current-code-inspector)
扩展 top-level-form 中的所有非基本语法,并为扩展后的形式返回一个语法对象, 该对象只包含核心形式,与 完全展开的程序 所指定的语法相匹配。

在 top-level-form 被扩展之前, 它的词法上下文被 namespace-syntax-introduce 所丰富,就像 eval 一样。 使用 syntax->datum 将返回的语法对象转换为可打印的数据。

如果 insp 不是原始 code inspector (即 Racket 启动时 (current-code-inspector) 的值), 那么结果的语法对象就有 tainted 。

下面是一个在模块上使用 expand 的例子:

(parameterize ([current-namespace (make-base-namespace)])
 (expand
  (datum->syntax
   #f
   '(module foo scheme
      (define a 3)
      (+ a 4)))))

下面是一个在非顶层表单上使用 expand 的例子:

(define-namespace-anchor anchor)
(parameterize ([current-namespace
                (namespace-anchor->namespace anchor)])
 (expand
  (datum->syntax
   #f
   '(delay (+ 1 2)))))

Changed in version 8.2.0.4 of package base: 增加了 insp 参数和污点。

procedure

(expand-syntax stx [insp]) → syntax?

  stx : syntax?
  insp : inspector? = (current-code-inspector)
和 (expand stx insp) 一样, 除了参数必须是一个 syntax object,而且在扩展前不对其词法上下文进行充实。

Changed in version 8.2.0.4 of package base: 增加了 insp 参数和污点。

procedure

(expand-once top-level-form [insp]) → syntax?

  top-level-form : any/c
  insp : inspector? = (current-code-inspector)
部分扩展 top-level-form ,并为部分扩展的表达式返回一个语法对象。 由于扩展机制的限制,一些上下文信息可能会丢失。 特别是,在结果上调用 expand-once 可能会产生一个与通过 expand 进行扩展不同的结果。

在 top-level-form 被扩展之前,它的词法上下文被 namespace-syntax-introduce 充实, 就像 eval 一样。

insp 参数决定了结果是否有 tainted,与 expand 的情况相同。

Changed in version 8.2.0.4 of package base: 增加了 insp 参数和污点。

procedure

(expand-syntax-once stx [insp]) → syntax?

  stx : syntax?
  insp : inspector? = (current-code-inspector)
和 (expand-once stx) 一样,除了参数必须是一个 syntax object , 而且在扩展前不对其词法上下文进行充实。

Changed in version 8.2.0.4 of package base: 增加了 insp 参数和污点。

procedure

(expand-to-top-form top-level-form [insp]) → syntax?

  top-level-form : any/c
  insp : inspector? = (current-code-inspector)
部分扩展 top-level-form 以显示最外层的句法形式。 这种部分扩展主要用于检测 begin 的顶层使用。 与 expand-once 的结果不同,用 expand 将 expand-to-top-form 的结果展开, 产生的结果与在原始语法上使用 expand 相同。

在 stx-or-sexpr 被扩展之前,它的词汇上下文被 namespace-syntax-introduce 所丰富, 就像 eval 一样。

insp 参数决定了结果是否有 tainted ,与 expand 的情况相同。

Changed in version 8.2.0.4 of package base: 增加了 insp 参数和污点。

procedure

(expand-syntax-to-top-form stx [insp]) → syntax?

  stx : syntax?
  insp : inspector? = (current-code-inspector)
和 (expand-to-top-form stx) 一样,除了参数必须是一个 syntax object, 并且在扩展前不对其词法上下文进行充实。

Changed in version 8.2.0.4 of package base: 添加了 insp 参数和tainting。

12.9.1 扩展模块的信息

扩展 module 声明的信息被存储在附加在语法对象上的一组 syntax properties 中(见 语法对象属性) :

  • 'module-body-context — 一个语法对象, 其 lexical information 与模块内部相对应,所以它包括扩展的 outside-edge scope 和 inside-edge scope; 也就是说,这个语法对象模拟了一个存在于原始模块体中的标识符, 并且不能被任何宏操作,所以它的词法信息包括模块的导入和定义的绑定关系。

    Added in version 6.4.0.1 of package base.

  • 'module-body-inside-context — 一个语法对象, 其 lexical information 对应于一个开始没有词法上下文的标识符,并被移入宏中, 因此它只包括 expulsions 的 inside-edge scope 。

    Added in version 6.4.0.1 of package base.

  • 'module-body-context-simple? — 一个 boolean, 其中 #t 表示模块主体的绑定(记录在 'module-body-inside-context 属性值的 lexical information 中)可以直接从直接导入模块的模块中重建, 包括导入的 for-syntax、 for-meta 和 for-template。

    Added in version 6.4.0.1 of package base.

Changed in version 7.0 of package base: 删除 'module-variable-provides, 'module-syntax-provides, 'module-indirect-provides 和 'module-indirect-for-meta-provides 属性。
