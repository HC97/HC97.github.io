5.2 创建结构类型

procedure

(make-struct-type name 
  super-type 
  init-field-cnt 
  auto-field-cnt 
  [auto-v 
  props 
  inspector 
  proc-spec 
  immutables 
  guard 
  constructor-name]) 
 → 
struct-type?
struct-constructor-procedure?
struct-predicate-procedure?
struct-accessor-procedure?
struct-mutator-procedure?
  name : symbol?
  super-type : (or/c struct-type? #f)
  init-field-cnt : exact-nonnegative-integer?
  auto-field-cnt : exact-nonnegative-integer?
  auto-v : any/c = #f
  props : 
(listof (cons/c struct-type-property?
                any/c))
 = null
  inspector : (or/c inspector? #f 'prefab) = (current-inspector)
  proc-spec : 
(or/c procedure?
      exact-nonnegative-integer?
      #f)
 = #f
  immutables : (listof exact-nonnegative-integer?) = null
  guard : (or/c procedure? #f) = #f
  constructor-name : (or/c symbol? #f) = #f
创建一个新的结构类型,除非 inspector 是 'prefab, 在这种情况下, make-struct-type 会访问一个 prefab 结构类型。 name 参数被用作类型名称。 如果 super-type 不是 #f, 生成的类型是相应结构类型的子类型。

产生的结构类型有 init-field-cnt+auto-field-cnt 字段 (除了来自 super-type 的任何字段之外),但是只有 init-field-cnt 构造函数参数(除了来自 super-type 的任何构造函数参数之外)。 其余的字段用 auto-v 进行初始化。 总的字段数(包括 super-type 的字段)必须不超过 32768。

props 参数是一个成对的列表,其中每对的 car 是一个结构类型的属性描述符, cdr 是一个任意的值。 一个属性可以在 props 中指定多次(包括由直接包含在 props 中的属性自动添加的属性), 只有当相关的值是 eq? 时,才会引发 exn:fail:contract exception is raised 。 关于属性的更多信息,请参见 结构类型属性 。 当 inspector 是 'prefab 时,那么 props 必须为 null 。

inspector 参数通常控制对结构类型及其实例的反射性信息的访问; 更多信息请参见 结构检查器 。 如果 inspector 是 'prefab, 那么产生的 prefab 结构类型及其实例总是透明的。 如果 inspector 是 #f, 那么结构类型的实例是透明的。

如果 proc-spec 是一个整数或过程,那么结构类型的实例将作为过程。 更多信息请参见 prop:procedure 。 为 proc-spec 提供一个非 #f 的值与在 props 结尾处将该值与 prop:procedure 配对是一样的, 另外当 proc-spec 是一个整数时,在 immutables 中包括 proc-spec 。

immutables 参数提供了一个字段位置的列表。 列表中的每个元素必须是唯一的,否则会引发 exn:fail:contract exception is raised 。 每个元素还必须在 0 (包括)到 init-field-cnt (不包括)的范围内, 否则会 exn:fail:contract exception is raised 。

guard 参数要么是一个有 n+1 个参数的过程,要么是 #f, 其中 n 是新结构类型构造函数的参数数 (即 init-field-cnt 加上 super-type 所隐含的构造函数参数,如果有的话)。 如果 guard 是一个过程,那么每当构造一个类型的实例或创建一个子类型的实例时,该过程就会被调用。 guard 的参数是为结构的前 n 个字段提供的值, 然后是被实例化的结构类型的名称(即 name, 除非是被实例化的子类型)。 guard 的结果必须是 n 个值,这些值将成为该结构的字段的实际值。 guard 可以引发一个异常,以防止创建一个具有给定字段值的结构。 如果一个结构的子类型有自己的防护,那么子类型的防护将首先被应用, 由子类型的防护程序产生的前 n 个值将成为 guard 的前 n 个参数。 当 inspector 是 'prefab 时,那么 guard 必须是 #f 。

如果 constructor-name 不是 #f, 它将被用作由 object-name 返回的生成的 constructor 过程的名称, 或者以打印出来的constructor值的形式使用。

make-struct-type 的结果是五个值:

  • 一个 structure type descriptor,

  • 一个 constructor 程序,

  • 一个 predicate 程序,

  • 一个 accessor 程序,它消耗一个结构和一个在 0 (包括)和 init-field-cnt+auto-field-cnt (不包括)之间的字段索引,以及

  • 一个 mutator 程序,它消耗一个结构、一个字段索引和一个字段值。

Examples:
(define-values (struct:a make-a a? a-ref a-set!)
  (make-struct-type 'a #f 2 1 'uninitialized))
(define an-a (make-a 'x 'y))

 

> (a-ref an-a 1)

'y

> (a-ref an-a 2)

'uninitialized

> (define a-first (make-struct-field-accessor a-ref 0))
> (a-first an-a)

'x

 

(define-values (struct:b make-b b? b-ref b-set!)
  (make-struct-type 'b struct:a 1 2 'b-uninitialized))
(define a-b (make-b 'x 'y 'z))

 

> (a-ref a-b 1)

'y

> (a-ref a-b 2)

'uninitialized

> (b-ref a-b 0)

'z

> (b-ref a-b 1)

'b-uninitialized

> (b-ref a-b 2)

'b-uninitialized

 

(define-values (struct:c make-c c? c-ref c-set!)
  (make-struct-type
   'c struct:b 0 0 #f null (make-inspector) #f null
   ; guard checks for a number, and makes it inexact
   (lambda (a1 a2 b1 name)
     (unless (number? a2)
       (error (string->symbol (format "make-~a" name))
              "second field must be a number"))
     (values a1 (exact->inexact a2) b1))))

 

> (make-c 'x 'y 'z)

make-c: second field must be a number

> (define a-c (make-c 'x 2 'z))
> (a-ref a-c 1)

2.0

 

(define p1 #s(p a b c))
(define-values (struct:p make-p p? p-ref p-set!)
  (make-struct-type 'p #f 3 0 #f null 'prefab #f '(0 1 2)))

 

> (p? p1)

#t

> (p-ref p1 0)

'a

> (make-p 'x 'y 'z)

'#s(p x y z)

procedure

(make-struct-field-accessor accessor-proc    
  field-pos    
  [field/proc-name    
  arg-contract-str    
  realm]) → procedure?
  accessor-proc : struct-accessor-procedure?
  field-pos : exact-nonnegative-integer?
  field/proc-name : (or/c symbol? #f)
   = (symbol->string (format "field~a" field-pos))
  arg-contract-str : (or/c string? symbol? #f) = #f
  realm : symbol? = 'racket
返回一个字段访问器,相当于 (lambda (s) (accessor-proc s field-pos)) 。 accessor-proc 必须是一个由 make-struct-type 返回的 accessor 。

field/proc-name 参数决定了产生的过程的名称,用于错误报告和调试。 如果 field/proc-name 是一个符号,并且 arg-contract-str 不是 #f, 那么 field/proc-name 将作为过程的名称。 如果 field/proc-name 是一个符号,并且 arg-contract-str 是 #f, 那么 field/proc-name 将与 accessor-proc 的结构类型的名称结合起来,形成过程的名称。 如果 field/proc-name 是 #f, 那么就使用 'accessor 作为过程名称。

arg-contract-str 参数决定了当 accessor 过程应用于一个不是 accessor-proc 结构类型实例的值时, 如何报告错误。 如果它是一个字符串或符号,那么该字符串或符号的文本被用作错误报告的契约。 否则,契约文本由 accessor-proc 的结构类型的名称合成。

realm参数也用于错误报告。 它指定了一个 realm ,错误信息调整器可以使用这个境界来决定如何调整一个错误信息。 realm 参数也决定了访问器过程的 procedure-realm 的结果。

关于例子,请参见 make-struct-type 。

Changed in version 8.4.0.2 of package base: 增加了 arg-contract-str 和 realm 参数。

procedure

(make-struct-field-mutator mutator-proc    
  field-pos    
  [field/proc-name    
  arg-contract-str    
  realm]) → procedure?
  mutator-proc : struct-mutator-procedure?
  field-pos : exact-nonnegative-integer?
  field/proc-name : (or/c symbol? #f)
   = (symbol->string (format "field~a" field-pos))
  arg-contract-str : (or/c string? symbol? #f) = #f
  realm : symbol? = 'racket
返回一个与 (lambda (s v) (mutator-proc s field-pos v)) 等价的字段突变器。 mutator-proc 必须是一个由 make-struct-type 返回的 mutator 。

field-name, arg-contract-str 和 realm 参数用于错误和调试目的,类似于 make-struct-field-accessor 的相同参数。

关于例子,见 make-struct-type 。

Changed in version 8.4.0.2 of package base: 增加了 arg-contract-str 和 realm 参数。

value

prop:sealed : struct-type-property?

一个 structure type property 可以将结构类型声明为 sealed 。 与该属性相关的值被忽略;该属性本身的存在使结构类型被密封。

一个 sealed 结构类型不能作为另一个结构类型的超类型使用。 将一个结构类型声明为 sealed 通常只是一种性能提示, 因为检查一个密封结构类型的实例可能比检查一个可能有子类型的结构类型的实例要快一些。

Added in version 8.0.0.7 of package base.
