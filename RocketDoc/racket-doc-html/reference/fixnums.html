4.3.4 Fixnums

 (require racket/fixnum) package: base

racket/fixnum 库提供了像 fx+ 这样的操作,只消耗和产生 fixnums 。 这个库中的操作是不安全操作的安全版本,如 unsafe-fx+ 。 这些安全操作通常不会比使用通用基元(如 + 快。

racket/fixnum 库的预期用途是在代码中用

(require (filtered-in
          (λ (name)
            (and (regexp-match #rx"^unsafe-fx" name)
                 (regexp-replace #rx"unsafe-" name "")))
          racket/unsafe/ops))

代替 racket/fixnum 的 require 。 来放弃不安全版本的库。 另外,当遇到使用不安全 fixnum 操作的代码崩溃时, 可以使用 racket/fixnum 库来帮助调试问题。

4.3.4.1 Fixnum 算术

procedure

(fx+ a ...) → fixnum?

  a : fixnum?

procedure

(fx- a b ...) → fixnum?

  a : fixnum?
  b : fixnum?

procedure

(fx* a ...) → fixnum?

  a : fixnum?

procedure

(fxquotient a b) → fixnum?

  a : fixnum?
  b : fixnum?

procedure

(fxremainder a b) → fixnum?

  a : fixnum?
  b : fixnum?

procedure

(fxmodulo a b) → fixnum?

  a : fixnum?
  b : fixnum?

procedure

(fxabs a) → fixnum?

  a : fixnum?
unsafe-fx+, unsafe-fx-, unsafe-fx*, unsafe-fxquotient, unsafe-fxremainder, unsafe-fxmodulo 和 unsafe-fxabs 的安全版本。 如果算术结果不是一个固定值,则会引发 exn:fail:contract:non-fixnum-result exception is raised 。

Changed in version 7.0.0.13 of package base: 允许 fx+ 和 fx* 有零个或多个参数, fx- 有一个或多个参数。

procedure

(fxand a ...) → fixnum?

  a : fixnum?

procedure

(fxior a ...) → fixnum?

  a : fixnum?

procedure

(fxxor a ...) → fixnum?

  a : fixnum?

procedure

(fxnot a) → fixnum?

  a : fixnum?

procedure

(fxlshift a b) → fixnum?

  a : fixnum?
  b : fixnum?

procedure

(fxrshift a b) → fixnum?

  a : fixnum?
  b : fixnum?
与 bitwise-and, bitwise-ior, bitwise-xor, bitwise-not 和 arithmetic-shift 一样,但限制于消耗 fixnums; 结果总是 fixnum 。unsafe-fxlshift 和 unsafe-fxrshift 操作对应于 arithmetic-shift ,但需要非负参数; unsafe-fxlshift 是正(即左)移位, unsafe-fxrshift 是负(即右)移位,其中要移位的位数必须不超过用于表示一个 fixnum 的位数。 如果算术结果不是一个 fixnum,则会引发 exn:fail:contract:non-fixnum-result exception is raised 。

Changed in version 7.0.0.13 of package base: 允许 fxand, fxior 和 fxxor 有任意数量的参数。

procedure

(fxpopcount a) → fixnum?

  a : (and/c fixnum? (not/c negative?))

procedure

(fxpopcount32 a) → fixnum?

  a : (and/c fixnum? (integer-in 0 #xFFFFFFFF))

procedure

(fxpopcount16 a) → fixnum?

  a : (and/c fixnum? (integer-in 0 #xFFFF))
计算 a 的二补表示中的位数。根据平台的不同,当已知结果不超过 32 或 16 时, fxpopcount32 和 fxpopcount16 操作会更快。

Added in version 8.5.0.7 of package base.

procedure

(fx+/wraparound a b) → fixnum?

  a : fixnum?
  b : fixnum?

procedure

(fx-/wraparound a b) → fixnum?

  a : fixnum?
  b : fixnum?

procedure

(fx*/wraparound a b) → fixnum?

  a : fixnum?
  b : fixnum?

procedure

(fxlshift/wraparound a b) → fixnum?

  a : fixnum?
  b : fixnum?
与 fx+, fx-, fx* 和 fxlshift 类似, 但对于任何允许的参数(即对于任何 fixnum 参数,除了第二个 fxlshift/wraparound 参数必须在 0 和 fixnum 的位数之间,包括在内)都会产生一个 fixnum 结果。 结果是通过简单地丢弃不适合在 fixnum 表示中的位而产生的。 如果保留的位中最高的位被设置,结果就是负的,例如,如果该值是由两个正的 fixnum 相加产生的,也是如此。

Added in version 7.9.0.6 of package base.

(fx= a b ...) → boolean?

  b : fixnum?

procedure

(fx< a b ...) → boolean?

  a : fixnum?
  b : fixnum?

procedure

(fx> a b ...) → boolean?

  a : fixnum?
  b : fixnum?

procedure

(fx<= a b ...) → boolean?

  a : fixnum?
  b : fixnum?

procedure

(fx>= a b ...) → boolean?

  a : fixnum?
  b : fixnum?

procedure

(fxmin a b ...) → fixnum?

  a : fixnum?
  b : fixnum?

procedure

(fxmax a b ...) → fixnum?

  a : fixnum?
  b : fixnum?
就像 =, <, >, <=, >=, min 和 max, 但被限制在消耗 fixnums 。

Changed in version 7.0.0.13 of package base: Allow one argument, in addition to allowing two or more.

procedure

(fx->fl a) → flonum?

  a : fixnum?

procedure

(fl->fx fl) → fixnum?

  fl : flonum?
在 fixnums 和 flonums 之间进行转换, 在将 flonum 转换为 fixnum 的情况下进行截断。

fx->fl 函数与 exact->inexact 或 ->fl 约束到一个 fixnum 参数相同。

fl->fx 函数与 truncate 后的 inexact->exact 或 fl->exact-integer 约束返回一个 fixnum 是一样的。 如果被截断的 flonum 不适合于一个 fixnum,就会引发 exn:fail:contract exception is raised 。

Changed in version 7.7.0.8 of package base: 将 fl->fx 改为 truncate。

procedure

(fixnum-for-every-system? v) → boolean?

  v : any/c
如果 v 是一个 fixnum ,并且在每个 Racket 实现中都用 fixnum 表示, 则返回 #t ,否则返回 #f 。

Added in version 7.3.0.11 of package base.

4.3.4.2 Fixnum 向量

fxvector 就像一个 vector ,但它只保存 fixnums 。 与 vector 相比, fxvector 的唯一优点是可以用 shared-fxvector 等函数创建一个共享版本。

如果两个 fxvectors 具有相同的长度, 并且 fxvectors 的相应槽中的值是 equal? ,那么它们就是 equal? 。

一个打印的 fxvector 以 #fx( 开始, 可以选择在 #fx 和 ( 之间加上一个数字。 See 读向量 for information on reading fxvectors and 打印矢量 for information on printing fxvectors.

procedure

(fxvector? v) → boolean?

  v : any/c
如果 v 是一个 fxvector ,返回 #t ,否则返回 #f 。

procedure

(fxvector x ...) → fxvector?

  x : fixnum?
创建一个包含给定 fixnums 的 fxvector 。

Example:
> (fxvector 2 3 4 5)

(fxvector 2 3 4 5)

procedure

(make-fxvector size [x]) → fxvector?

  size : exact-nonnegative-integer?
  x : fixnum? = 0
创建一个具有 size 元素的 fxvector , 其中 fxvector 中的每个槽都被填充为 x 。

Example:
> (make-fxvector 4 3)

(fxvector 3 3 3 3)

procedure

(fxvector-length vec) → exact-nonnegative-integer?

  vec : fxvector?
返回 vec 的长度(即 fxvector中的槽数)。

procedure

(fxvector-ref vec pos) → fixnum?

  vec : fxvector?
  pos : exact-nonnegative-integer?
返回 vec 的槽 pos 中的 fixnum 。 第一个槽位是 0 ,最后一个槽位是比 (fxvector-length vec) 少一个的位置。

procedure

(fxvector-set! vec pos x) → fixnum?

  vec : fxvector?
  pos : exact-nonnegative-integer?
  x : fixnum?
设置 vec 的槽 pos 中的 fixnum 。 第一个槽是位置 0 ,最后一个槽是小于 (fxvector-length vec) 的一个。

procedure

(fxvector-copy vec [start end]) → fxvector?

  vec : fxvector?
  start : exact-nonnegative-integer? = 0
  end : exact-nonnegative-integer? = (vector-length v)
创建一个新的 fxvector ,大小为 (- end start) , 包含 vec 的所有元素,从 start (包括)到 end (不包括)。

procedure

(in-fxvector vec [start stop step]) → sequence?

  vec : fxvector?
  start : exact-nonnegative-integer? = 0
  stop : (or/c exact-integer? #f) = #f
  step : (and/c exact-integer? (not/c zero?)) = 1
当没有提供可选参数时,返回一个等同于 vec 的序列。

可选参数 start, stop 和 step 与 in-vector 相同。

in-fxvector 应用程序直接出现在 for 子句中时, 可以为 fxvector 迭代提供更好的性能。

syntax

(for/fxvector maybe-length (for-clause ...) body ...)

syntax

(for*/fxvector maybe-length (for-clause ...) body ...)

 
maybe-length = 
  | #:length length-expr
  | #:length length-expr #:fill fill-expr
 
  length-expr : exact-nonnegative-integer?
  fill-expr : fixnum?
类似于 for/vector 或 for*/vector ,但适用于 fxvector 。 默认的 fill-expr 产生 0 。

procedure

(shared-fxvector x ...) → fxvector?

  x : fixnum?
创建一个包含给定 fixnums 的 fxvector。 对于 places 之间的通信,新的 fxvector 被分配在 shared memory space 中。

Example:
> (shared-fxvector 2 3 4 5)

(fxvector 2 3 4 5)

procedure

(make-shared-fxvector size [x]) → fxvector?

  size : exact-nonnegative-integer?
  x : fixnum? = 0
创建一个 size 为元素的 fxvector , 其中 fxvector 中的每个槽都被填充为 x 。 为了在 places 之间进行通信,新的 fxvector 被分配到 shared memory space 中。

Example:
> (make-shared-fxvector 4 3)

(fxvector 3 3 3 3)

4.3.4.3 Fixnum 范围

procedure

(most-positive-fixnum) → fixnum?

procedure

(most-negative-fixnum) → fixnum?

返回幅度最大的正数和负数的 fixnums. (most-positive-fixnum) 和 (most-negative-fixnum) 的值取决于平台和虚拟机,但是所有的 fixnum 都在 (most-negative-fixnum) 到 (most-positive-fixnum) 的范围内,并且这个范围内的所有精确整数都是 fixnum。

Added in version 8.1.0.7 of package base.
