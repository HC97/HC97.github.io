5.3 结构类型属性

通用接口在结构类型属性之上提供了一个高级 API。

structure type property 允许将每个类型的信息与结构类型相关联(而不是与结构值相关联的每个实例信息)。 属性值通过 make-struct-type 过程(参见 创建结构类型) 或通过 struct 的 #:property 选项与结构类型关联。 子类型继承其父类型的属性值,子类型可以用一个新的值覆盖继承的属性值。

procedure

(make-struct-type-property name 
  [guard 
  supers 
  can-impersonate? 
  accessor-name 
  contract-str 
  realm]) 
 → 
struct-type-property?
(any/c . -> . boolean?)
procedure?
  name : symbol?
  guard : (or/c procedure? #f 'can-impersonate) = #f
  supers : 
(listof (cons/c struct-type-property?
                (any/c . -> . any/c)))
 = null
  can-impersonate? : any/c = #f
  accessor-name : (or/c symbol? #f) = #f
  contract-str : (or/c string? symbol? #f) = #f
  realm : symbol? = 'racket
创建一个新的结构类型属性并返回三个值:

  • structure type property descriptor,用于 make-struct-type 和 struct;

  • property predicate 过程,它接受一个任意值, 如果该值是一个描述符或具有属性值的结构类型实例,则返回 #t ,否则返回 #f ;

  • property accessor 过程,返回与给定的描述符或其实例之一的结构类型相关联的值; 如果结构类型没有该属性的值,或者提供了任何其他类型的值, 则会 exn:fail:contract exception is raised ,除非向过程提供第二个参数 failure-result 。 在这种情况下,如果 failure-result 是一个过程,则调用它(通过尾部调用), 不带参数以产生属性访问器过程的结果;否则, failure-result 本身作为结果返回。

如果可选 guard 是作为过程提供的, 则在将该属性附加到新的结构类型之前由 make-struct-type 调用。 守卫必须接受两个参数:一个是提供给 guard 的属性值,另一个是包含新结构类型信息的列表。 如果 make-struct-type 跳过当前检查器控件检查, 则该列表包含 struct-type-info 将为新结构类型返回的值。

调用 guard 的结果与目标结构类型中的属性关联,而不是提供给 make-struct-type 的值。 要拒绝属性关联(例如,因为提供给 make-struct-type 的值不适合该属性), guard 可以引发异常。 这种异常阻止 make-struct-type 返回结构类型描述符。

如果 guard 是 'can-impersonate, 那么该属性的访问器可以通过 impersonate-struct 重定向。 这个选项与提供 #t 作为 can-impersonate? 参数相同,是为了向后兼容。

可选的 supers 参数是一个属性列表,当新创建的属性被关联到某个结构类型时,这些属性会被自动关联到该结构类型。 supers 中的每个属性都与一个过程配对,该过程接收为新属性提供的值(在它被 guard 处理后),并为关联的属性返回一个值(然后被发送到该属性的guard(如果有的话))。

可选的 can-impersonate? 参数决定了结构类型属性是否可以通过 impersonate-struct 重定向。 如果该参数是 #f, 那么重定向是不允许的。 否则,该属性访问器可以通过结构冒充者重定向。

可选的 accessor-name 参数为返回的 accessor 函数提供了一个名称(在 object-name 的意义上), 以使用该名称。 如果 accessor-name 是 #f , 则通过在 name 的末尾添加 -accessor 来创建一个名字。

可选的 contract-str 参数提供了一个合同, 当返回的访问器被应用于一个不是属性实例的值时(并且 failure-result 参数没有被提供给访问器), 该合同将被包含在错误信息中。 如果 contract-str 是 #f, 则通过在 name 的末尾添加 ? 来创建一个契约。

可选的 realm 参数提供了一个 realm (在 procedure-realm 的意义上),与返回的访问器相关联。

Examples:
> (define-values (prop:p p? p-ref) (make-struct-type-property 'p))
> (define-values (struct:a make-a a? a-ref a-set!)
    (make-struct-type 'a #f 2 1 'uninitialized
                      (list (cons prop:p 8))))
> (p? struct:a)

#t

> (p? 13)

#f

> (define an-a (make-a 'x 'y))
> (p? an-a)

#t

> (p-ref an-a)

8

> (define-values (struct:b make-b b? b-ref b-set!)
    (make-struct-type 'b #f 0 0 #f))
> (p? struct:b)

#f

> (define-values (prop:q q? q-ref) (make-struct-type-property
                                    'q (lambda (v si) (add1 v))
                                    (list (cons prop:p sqrt))))
> (define-values (struct:c make-c c? c-ref c-set!)
    (make-struct-type 'c #f 0 0 'uninit
                      (list (cons prop:q 8))))
> (q-ref struct:c)

9

> (p-ref struct:c)

3

Changed in version 7.0 of package base: 在提供给保护过程的信息中, Racket 的 CS 实现跳过了检查器检查是否暴露祖先结构类型(如果有的话)。
Changed in version 8.4.0.2: 增加了 accessor-name, contract-str 和 realm 参数。
Changed in version 8.5.0.2: 更改了 Racket 的 BC 实现,跳过检查器检查, 与 CS 实现相同,为提供给保护过程的祖先信息。

procedure

(struct-type-property? v) → boolean?

  v : any/c
如果 v 是 structure type property descriptor 值则返回 #t , 否则返回, #f 。

procedure

(struct-type-property-accessor-procedure? v) → boolean?

  v : any/c
如果 v 是由 make-struct-type-property 生成的访问器过程, 则返回 #t ,否则返回 #f 。

procedure

(struct-type-property-predicate-procedure? v    
  [prop]) → boolean?
  v : any/c
  prop : (or/c struct-type-property? #f) = #f
如果 v 是由 make-struct-type-property 生成的谓词过程, 并且 prop 是 #f 或由同样调用 make-struct-type-property 生成, 则返回 #t ,否则返回 #f 。

Added in version 7.5.0.11 of package base.
