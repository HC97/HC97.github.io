8.10 Collapsible Contracts

 (require racket/contract/collapsible) package: base

Added in version 7.1.0.10 of package base.

Collapsible contracts 是合约系统中的一项优化,旨在避免合约包装器在高阶值上的特殊病态堆积。 vectorof, vector/c 和 -> 合约组合器支持向量合约的折叠, 支持返回单个值的函数合约的折叠。

直观地说,一个可折叠的合约是一个树状结构。 tree nodes 代表高阶契约(例如 ->)， tree leaves 代表平坦契约的序列。 两棵树可以通过 merge 程序折叠成一棵树,该程序将不必要的平面合约从树叶中移除。

关于可折叠合约的动机和设计的更多信息,见 [Feltey18] 。 关于理论基础,见 [Greenberg15] 。

警告: 本节描述的功能是实验性的,可能不足以实现新的可折叠合约。 实现新的可折叠合约需要使用不安全的陪护者和冒名顶替者,而这只支持向量和过程值。 这篇文档的存在主要是为了方便将来对 racket/contract/collapsible 库的维护。 结束警告

procedure

(get/build-collapsible-late-neg-projection c)

 → (-> blame? (values (-> any/c any/c any/c) collapsible-contract?))
  c : contract?
返回 c 的 collapsible-late-neg 投影。

如果 c 没有 collapsible-late-neg 投影, 那么这个函数使用它的原始投影并构造一个叶子作为它的可折叠表示。

value

collapsible-contract-continuation-mark-key

 : continuation-mark-key?
在可折叠合约检查过程中出现的延续标记所使用的键。 如果当前合约是可折叠的,这些标记的值是 #t 。

syntax

(with-collapsible-contract-continuation-mark body ...)

插入一个延续标记,告知合约分析器当前的合约是可折叠的。

value

prop:collapsible-contract : struct-type-property?

实现此属性的结构可以作为可折叠合约使用。与此属性相关的值应该通过调用 build-collapsible-contract-property 来构建。

procedure

(collapsible-contract? v) → boolean?

  v : any/c
一个识别具有 prop:collapsible-contract 属性的结构的谓词。

procedure

(merge new-cc new-neg old-cc old-neg) → collapsible-contract?

  new-cc : collapsible-contract?
  new-neg : any/c
  old-cc : collapsible-contract?
  old-neg : any/c
将两个可折叠合约合并为一个可折叠合约。 new-neg 和 old-neg 参数应该是指责方,类似于传递给 late neg projection 的参数。

procedure

(collapsible-guard cc val neg-party) → any/c

  cc : collapsible-contract?
  val : any/c
  neg-party : any/c
与 late neg projection 类似,这个函数用可折叠契约 cc 来保护值 val 。

procedure

(collapsible-contract-property? v) → boolean?

  v : any/c
这个谓词表明一个值可以被用作 prop:collapsible-contract 的属性。

procedure

(build-collapsible-contract-property 
  [#:try-merge try-merge 
  #:collapsible-guard collapsible-guard]) 
 → collapsible-contract-property?
  try-merge : 
(or/c #f
      (-> collapsible-contract?
          any/c
          collapsible-contract?
          any/c
          (or/c #f collapsible-contract?)))
 = #f
  collapsible-guard : (-> collapsible-contract? any/c any/c any/c)
   = 
(λ (cc v neg)
  (error
   "internal error: contract does not support `collapsible-guard`" cc))
从一个合并函数和一个卫兵构建一个 collapsible contract property 。 try-merge 参数与 merge 类似, 但可能会返回 #f 而不是可折叠合约,并且可以专门用于特定的可折叠合约。 collapsible-guard 参数应该是专门针对正在实现的特定可折叠合约的。

struct

(struct collapsible-ho/c (latest-blame missing-party latest-ctc))

  latest-blame : blame?
  missing-party : any/c
  latest-ctc : contract?
一个用于高阶值的可折叠合约的通用父结构。 latest-blame 字段持有最近附加的合约的责备对象。 同样地,缺失方字段保存了传递给合同的最新的 missing-party 。 latest-contract 字段存储了附加在该值上的最新的合同。

struct

(struct collapsible-leaf/c (proj-list
    contract-list
    blame-list
    missing-party-list))
  proj-list : (listof (-> any/c any/c any/c))
  contract-list : (listof contract?)
  blame-list : (listof blame?)
  missing-party-list : (listof any/c)
一个代表可折叠合约的叶子节点的结构。 proj-list 字段持有部分应用的 late neg projections 的列表。 contract-list, blame-list 和 missing-party-list 字段分别持有合同、责备对象和责备失踪方的列表。

value

impersonator-prop:collapsible : impersonator-property?

procedure

(has-impersonator-prop:collapsible? v) → boolean?

  v : any/c

procedure

(get-impersonator-prop:collapsible v) → collapsible-property?

  v : any/c
一个冒名顶替者属性(及其访问器),它应该被附加到用可折叠合约保护的陪护或冒名顶替的值上。

struct

(struct collapsible-property (c-c neg-party ref))

  c-c : collapsible-contract?
  neg-party : any/c
  ref : (or/c #f impersonator?)
属性的父结构,这些属性应该被附加到用可折叠合约保护的值的陪护者或冒充者。 c-c 字段存储现在或将来附加到该值的可折叠合约。 neg-party 字段存储了传递给价值上的合同的最新缺失的责备方。 ref 字段是可变的,它存储了对这个属性所附着的陪护者或冒名者的引用。 这对于确定一个未知的陪护者是否在被可折叠合约保护后被附加到一个值上是必要的。

struct

(struct collapsible-count-property collapsible-property (count
    prev))
  count : natural-number/c
  prev : (or/c collapsible-count-property? any/c)
这个属性在值完全进入可折叠模式之前与 impersonator-prop:collapsible 属性相关联。 这些属性在 count 字段中跟踪一个值的合约数量,并在 prev 字段中持有对前一个 count property 的引用,或没有合约的原始值。 这允许合约系统遍历附加合约链,并将它们合并到一个单一的可折叠合约中,以保护原始值。

struct

(struct collapsible-wrapper-property collapsible-property
  (checking-wrapper)
  checking-wrapper : impersonator?
当一个值被一个可折叠的合约所保护时,这个属性被使用。 checking-wrapper 字段持有一个陪护者或冒名顶替者, 该陪护者或冒名顶替者会被派往存储在该属性中的可折叠合约,以执行任何必要的合约检查。 当该值收到另一个合约并发生合并时,检查包装器将保持不变,即使附加到该值的特定可折叠合约可能发生变化。
