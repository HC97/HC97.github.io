12.6 带有拼接体的本地绑定

 (require racket/splicing) package: base
The bindings documented in this section are provided by the racket/splicing library, not racket/base or racket.

splicing-parameterize

像 let (不是 named let), letrec, let-values, letrec-values, let-syntax, letrec-syntax, let-syntaxes, letrec-syntaxes, letrec-syntaxes+values, local 和 parameterize, 除了在定义上下文中,主体形式被拼接到包围的定义上下文中(与 begin 的方式相同)。

Examples:
> (splicing-let-syntax ([one (lambda (stx) #'1)])
    (define o one))
> o

1

> one

one: undefined;

 cannot reference an identifier before its definition

  in module: top-level

当拼接的绑定形式出现在 top-level context 或 module context 中时,其局部绑定的处理方式与定义类似。 特别是,语法绑定在每次 visit 模块时都会被评估,而不是像 let-syntax 等那样在编译时只评估一次。

Example:
> (splicing-letrec ([x bad]
                    [bad 1])
    x)

bad.1: undefined;

 cannot reference an identifier before its definition

  in module: top-level

如果一个拼接形式中的定义旨在成为拼接体的本地定义,那么该标识符的 'definition-intended-as-local syntax property 应该为真值。 例如,当 splicing-let 扩展到定义序列时,它本身会将该属性添加到本地绑定的标识符上, 因此,在拼接表单中嵌套 splicing-let 会按预期工作(没有任何模糊的绑定)。

Changed in version 6.12.0.2 of package base: 增加 splicing-parameterize 。

syntax

splicing-syntax-parameterize

和 syntax-parameterize 一样,除了在定义上下文中, 主体形式被拼接到包围的定义上下文中(和 begin 的方式一样)。 在定义上下文中, splicing-syntax-parameterize 的主体可以是空的。

请注意, require transformers 和 provide transformers 不受语法参数化的影响。 尽管所有 require 和 provide 的使用都将被拼接到包围的上下文中, 但派生的导入或导出规范将被展开,就像它们没有在 splicing-syntax-parameterize 中一样。

此外,用 module* 定义的 submodules , 如果用 #f 代替 module path , 会 受到 语法参数化的影响, 但其他子模块(那些用 module 或 module* 定义的 module path) 则 不 受影响。

Examples:
> (define-syntax-parameter place (lambda (stx) #'"Kansas"))
> (define-syntax-rule (where) `(at ,(place)))
> (where)

'(at "Kansas")

> (splicing-syntax-parameterize ([place (lambda (stx) #'"Oz")])
    (define here (where)))
> here

'(at "Oz")

Changed in version 6.11.0.1 of package base: 修改后,对指定 #f 代替 module path 的 module* 子模块进行语法参数化。
