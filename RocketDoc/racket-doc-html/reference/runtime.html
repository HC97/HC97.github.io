15.8 环境和运行时间信息

procedure

(system-type [mode])

 → (or/c symbol? string? bytes? exact-positive-integer? vector? #f)
  mode : 
(or/c 'os 'os* 'arch 'word 'vm 'gc 'link 'machine 'target-machine
      'so-suffix 'so-mode 'fs-change 'cross)
   = 'os
返回有关运行中的 Racket 的操作系统、构建模式或机器的信息。 (安装工具应该使用 cross-system-type, 以支持交叉安装)。

在 'os 模式下,可能的符号结果是:

  • 'unix

  • 'windows

  • 'macosx

在引入 'os* 和 'arch 模式之前, (system-library-subpath #f) 可以在一定程度上间接地获得这些信息。

在 'os* 模式下,结果类似 'os 模式,但是细化到一个特定的操作系统, 比如 'linux 或 'freebsd, 而不是一般的 'unix 分类。

在 'arch 模式下,结果是一个代表架构的符号。 可能的结果包括 'x86_64, 'i386,'aarch64, 'arm (32位) 和 'ppc (32位)。

在 'word 模式下,结果是 32 或 64 , 表示 Racket 是作为一个 32 位程序还是 64 位程序运行。

关于 'vm 和 'gc 模式结果的更多信息, 请参见 Racket 虚拟机的实现 。

在 'vm 模式下,可能的符号结果是(另见 实施方式):

  • 'racket

  • 'chez-scheme

在 'gc 模式下,可能的符号结果是(也见 实施方式):

  • 'cgc — 当 (system-type 'vm) 是 'racket 时

  • '3m — 当 (system-type 'vm) 是 'racket 时

  • 'cs — 当 (system-type 'vm) 是 'chez-scheme 时

在 'link 模式下,可能的符号结果是:

  • 'static (Unix)

  • 'shared (Unix)

  • 'dll (Windows)

  • 'framework (Mac OS)

未来的 Racket 端口可能会扩展 'os, 'os*, 'arch, 'vm, 'gc 和 'link 结果的列表。

在 'machine 模式下,结果是一个字符串,它以特定平台的格式包含了关于当前机器的进一步细节。

在 'target-machine 模式下,结果是一个运行中的 Racket 本地字节码格式的符号, 如果除了与机器无关的格式外没有本地格式,则是 #f 。 如果结果是一个符号,那么 compile-target-machine? 在应用于该符号时返回 #t ; 另见 current-compile-target-machine 。

在 'so-suffix 模式下,那么结果是一个字节字符串,代表当前平台上用于共享对象的文件扩展名。 字节串以句号开头,所以它适合作为 path-replace-suffix 的第二个参数。

在 'so-mode 模式下,如果外国库默认以 "local" 模式打开(如在大多数平台上), 那么结果就是 'local ;如果外国库应该以 "global" 模式打开,那么就是 'global 。

在 'fs-change 模式下,结果是一个由四个元素组成的不可改变的向量。 每个元素要么是 #f ,要么是一个符号,其中一个符号表示存在一个属性, #f 表示没有一个属性。 可能的符号,依次是:

  • 'supported — filesystem-change-evt 可以产生一个 filesystem change event ,以监视文件系统的变化; 如果这个符号在向量中不在第一位,其他向量元素都是 #f

  • 'scalable — filesystem change event 所消耗的资源实际上只受可用内存的限制, 而不是文件描述符的限制;在 Mac OS 和 BSD 的 Unix 变体上,这个属性是 #f

  • 'low-latency — filesystem change event 的创建和检查几乎是瞬间完成的; 这个属性在Linux上是 #f

  • 'file-level — filesystem change event 可以跟踪文件级别的变化, 而不是文件的目录;这个属性在 Windows 上是 #f。

在 'cross 模式下,结果报告是否选择了跨平台的构建模式 (通过 racket 的 -C 或 --cross 参数;见 命令行) 。 可能的符号是:

  • 'infer — 根据 (system-type) 和 (cross-system-type) 是否报告相同的符号推断出跨平台模式

  • 'force — 使用跨平台模式,即使当前和目标系统类型相同,因为当前和目标可执行文件可能不同

Changed in version 6.8.0.2 of package base: 添加了 'vm 模式。
Changed in version 6.9.0.1: 添加了 'cross 模式。
Changed in version 7.1.0.6: 添加了 'target-machine 模式。
Changed in version 7.9.0.6: 添加了 'os* 和 'arch 模式。

procedure

(system-language+country) → string?

返回一个字符串以识别当前用户的语言和国家。

在 Unix 和 Mac OS 上,这个字符串是五个字符:两个小写 ASCII 字母代表语言,一个下划线,两个大写 ASCII 字母代表国家。 在 Windows 上,字符串可以任意长,但语言和国家是英文的(所有 ASCII 字母或空格),用下划线分开。

在 Unix 上,结果是通过检查 LC_ALL, LC_TYPE 和 LANG 环境变量来确定的, 依次为(如果环境变量的值以两个小写 ASCII 字母、下划线和两个大写 ASCII 字母开头,后面没有任何东西或句号,则使用该结果)。 在 Windows 和 Mac OS 上,结果由系统调用决定。

procedure

(system-library-subpath [mode]) → path?

  mode : (or/c 'cgc '3m 'cs #f) = (system-type 'gc)
返回一个相对目录路径。 这个字符串可以用来建立系统特定文件的路径。 例如,当 Racket 在 Sparc 架构的 Solaris 上运行时,子路径以 "sparc-solaris" 开始,而 i386 架构的 Windows 的子路径以 "win32\\i386" 开始。

可选的 mode 参数指定了相关的垃圾收集变量,它是 (system-type 'gc): 'cgc, '3m 或 'cs 。 它也可以是 #f, 在这种情况下,结果与垃圾收集的变体无关。

安装工具应该使用 cross-system-library-subpath,以支持交叉安装。

Changed in version 7.0 of package base: 增加了 'cs 模式。

procedure

(version) → (and/c string? immutable?)

返回一个不可变的字符串,表示当前执行的 Racket 版本。

procedure

(banner) → (and/c string? immutable?)

返回一个不可变的字符串,表示 Racket 启动时的标语文本(或嵌入程序的标语文本,如 GRacket)。 旗帜字符串以换行结束。

parameter

(current-command-line-arguments)

 → (vectorof (and/c string? immutable?))
(current-command-line-arguments argv) → void?
  argv : (vectorof string?)
一个 parameter ,在 Racket 启动时用命令行参数进行初始化(不包括任何被视为系统标志的命令行参数)。

在 Unix 和 Mac OS 上,命令行参数是以 byte strings 的形式提供给 Racket 进程的。 参数被转换为 strings , 使用 bytes->string/locale 和 #\uFFFD 作为编码错误的字符。

parameter

(current-thread-initial-stack-size) → exact-positive-integer?

(current-thread-initial-stack-size size) → void?
  size : exact-positive-integer?
一个 parameter ,它提供了关于为新创建的线程的局部变量保留多少空间的提示。 计算所使用的实际空间会受到 JIT 编译的影响,但它与平台无关。

procedure

(vector-set-performance-stats! results [thd]) → void?

  results : 
(and/c vector?
       (not/c immutable?))
  thd : (or/c thread? #f) = #f
设置 results 中的元素以报告当前的性能统计。 如果 thd 不是 #f, 就会报告一组特定的线程特定的统计数据, 否则就会报告一组不同的全局(在当前 place) 统计数据。

对于全局统计,最多可以在向量中设置 12 个元素,从头开始。 如果 results 有 n 个元素,其中 n < 12, 那么 n 个元素被设置为前 n 个性能统计值。 报告的统计值如下,按照它们在 results 中的设置顺序:

  • 0: 与 current-process-milliseconds 返回的值相同。

  • 1: 与 current-milliseconds 返回的值相同。

  • 2: 与 current-gc-milliseconds 返回的值相同。

  • 3: 自启动以来,在当前 place 内执行的垃圾收集的数量。

  • 4: 自启动以来执行的线程上下文切换的数量。

  • 5: 自启动以来处理的内部堆栈溢出的数量。

  • 6: 当前安排执行的线程数量(即正在运行的线程,没有暂停,也没有因同步而未安排)。

  • 7: 自启动以来从编译代码中读取的语法对象的数量。

  • 8: 所进行的哈希表搜索的数量。 当这个计数器达到 fixnum 的最大值时,它会溢出到最负的 fixnum 。

  • 9: 为完成哈希搜索而搜索的额外哈希槽的数量(使用双重哈希)。 当这个计数器达到一个 fixnum 的最大值时,它会溢出到最负的 fixnum 。

  • 10: 分配给机器代码的字节数,不由 current-memory-use 报告。

  • 11: 在垃圾回收之前分配的字节数的峰值。

对于特定的线程统计,向量中最多设置 4 个元素:

  • 0: #t 如果线程正在运行, #f 否则(与 thread-running? 的结果相同)。

  • 1: #t 如果线程已经终止, #f 否则(与 thread-dead? 结果相同)。

  • 2: #t 如果该线程目前在一个可同步的事件上被阻塞(或者在一些毫秒的时间内处于睡眠状态), #f 否则。

  • 3: 当前用于线程继续运行的字节数。

Changed in version 6.1.1.8 of package base: 为全局统计增加了向量位置 11 。
