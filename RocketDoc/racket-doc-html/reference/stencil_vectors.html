4.13 Stencil Vectors

stencil vector 就像一个 vector , 但它有一个相关的掩码 fixnum ,掩码中设置的位数决定了向量的长度。 模板向量对于实现一些数据结构非常有用 [Torosyan21], 例如哈希数组映射的三角形(HAMT)。

从概念上讲,一个模版向量的掩码表示全尺寸模版向量的哪些虚拟元素存在, 但是掩码位对通过 stencil-vector-ref 和 stencil-vector-set! 的访问或变异没有影响。 例如,这样一个模版向量有一个掩码 25 ,也可以写成 #b11001; 从低位到高位读,该掩码代表存在于第一、第四和第五虚拟槽的值。 如果这个模版向量的元素是 'a, 'b 和 'c, 那么 'a 在虚拟槽 0,以索引 0 访问,'b 在虚拟槽3, 以索引 1 访问,而 'c 在虚拟槽 4,以索引 2 访问。

掩码中位的相对顺序 与 stencil-vector-update 的功能更新操作有关。 要移除的元素用移除掩码指定,要添加的元素通过添加掩码相对于其余元素排序。 例如,从掩码为 #b11001 、元素为 'a, 'b 和 'c 的钢网向量开始, 使用添加掩码 #b100100 添加新元素 'd 和 'e , 产生一个掩码为 #b111101 的钢网向量,其元素依次为 'a, 'b, 'd, 'c 和 'e。

模板向量的最大尺寸在 64 位平台上是 58 个元素,在 32 位平台上是 26 个元素。 这种有限的大小使得内部表示紧凑,并确保更新操作相对简单。 模板向量是可变的,尽管它们主要是为了在没有变异的情况下用来实现一个持久的数据结构。

如果两个模版向量有相同的掩码,并且模版向量的相应槽中的值是 equal? , 那么它们就是 equal? 。

一个打印的向量以 #<stencil ...> 开始,这种打印的形式不能被 read 解析。 s-exp->fasl 和 serialize 函数不支持模版向量, 部分原因是 64 位平台上的模版向量可能无法在32位平台上表示。 其目的是,模版向量被用作数据类型实现的内存表示。

Added in version 8.5.0.7 of package base.

procedure

(stencil-vector? v) → boolean?

  v : any/c
如果 v 是一个 stencil vector ,返回 #t ,否则返回 #f 。

Examples:
> (stencil-vector #b10010 'a 'b)

#<stencil 18: a b>

> (stencil-vector #b111 'a 'b 'c)

#<stencil 7: a b c>

Added in version 8.5.0.7 of package base.

procedure

(stencil-vector-mask-width) → exact-nonnegative-integer?

返回当前平台上模版向量中允许的最大元素数。 在 64 位平台上结果是 58 ,在 32 位平台上是 26 。

procedure

(stencil-vector mask v ...) → stencil-vector?

  mask : (integer-in 0 (sub1 (expt 2 (stencil-vector-mask-width))))
  v : any/c
返回一个由元素 v 组成的 mask 向量。 所提供的 v 的数量必须与 mask 的二补表示法中设置的位数一致。

Added in version 8.5.0.7 of package base.

procedure

(stencil-vector-mask vec)

 → (integer-in 0 (sub1 (expt 2 (stencil-vector-mask-width))))
  vec : stencil-vector?
返回 vec 的掩码。 注意,模版向量的掩码是在创建时确定的,以后不能改变。

Example:
> (stencil-vector-mask (stencil-vector #b10010 'a 'b))

18

Added in version 8.5.0.7 of package base.

procedure

(stencil-vector-length vec)

 → (integer-in 0 (sub1 (stencil-vector-mask-width)))
  vec : stencil-vector?
返回 vec 的长度(即矢量中的槽数)。 其结果与 (fxpopcount (stencil-vector-mask vec)) 相同。

Example:
> (stencil-vector-length (stencil-vector #b10010 'a 'b))

2

Added in version 8.5.0.7 of package base.

procedure

(stencil-vector-ref vec pos) → any/c

  vec : stencil-vector?
  pos : exact-nonnegative-integer?
返回 vec 的槽 pos 中的元素。 第一个槽是位置 0, 最后一个槽是小于 (stencil-vector-length vec) 的一个。

Examples:
> (stencil-vector-ref (stencil-vector #b10010 'a 'b) 1)

'b

> (stencil-vector-ref (stencil-vector #b111 'a 'b 'c) 1)

'b

Added in version 8.5.0.7 of package base.

procedure

(stencil-vector-set! vec pos v) → avoid?

  vec : stencil-vector?
  pos : exact-nonnegative-integer?
  v : any/c
更新 vec 的槽 pos ,使其包含 v 。

Examples:
> (define st-vec (stencil-vector #b101 'a 'b))
> st-vec

#<stencil 5: a b>

> (stencil-vector-set! st-vec 1 'c)
> st-vec

#<stencil 5: a c>

Added in version 8.5.0.7 of package base.

procedure

(stencil-vector-update vec    
  remove-mask    
  add-mask    
  v ...) → stencil-vector?
  vec : stencil-vector?
  remove-mask : (integer-in 0 (sub1 (expt 2 (stencil-vector-mask-width))))
  add-mask : (integer-in 0 (sub1 (expt 2 (stencil-vector-mask-width))))
  v : any/c
返回一个与 vec 相似的模版向量,但是删除了与 remove-mask 对应的元素, 并在与现有(未删除)元素相对的位置添加了给定的 v ,由 add-mask 决定。

Examples:
> (define st-vec (stencil-vector #b101 'a 'b))
> (stencil-vector-update st-vec #b0 #b10 'c)

#<stencil 7: a c b>

> (stencil-vector-update st-vec #b0 #b1000 'c)

#<stencil 13: a b c>

> st-vec ; unchanged by updates

#<stencil 5: a b>

> (stencil-vector-update st-vec #b1 #b1 'c)

#<stencil 5: c b>

> (stencil-vector-update st-vec #b100 #b100 'c)

#<stencil 5: a c>

> (stencil-vector-update st-vec #b100 #b0)

#<stencil 1: a>

Added in version 8.5.0.7 of package base.
