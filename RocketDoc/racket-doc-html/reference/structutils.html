5.6 结构实用程序

procedure

(struct->vector v [opaque-v]) → vector?

  v : any/c
  opaque-v : any/c = '...
创建一个代表 v 的向量。 结果向量的第一个槽包含一个符号,其打印的名称具有 struct:id 形式。 剩下的每个槽都包含 v 中某个字段的值,如果该字段可以通过当前检查器访问的话, 或者 opaque-v 代表一个不可访问的字段。 对于连续的不可访问的字段,在向量中使用一个 opaque-v 值。 (因此,如果有一个以上的字段不可访问,则向量的大小与 struct 的大小不一致)。

procedure

(struct? v) → any

  v : any/c
如果 struct-info 揭示了任何具有当前检查器的 v 的结构类型, 则返回 #t ,否则返回 #f 。

通常情况下,当 (struct? v) 为真时,那么 (struct->vector v) 至少暴露了一个字段值。 然而, v 的唯一可见类型也有可能贡献零字段。

procedure

(struct-type? v) → boolean?

  v : any/c
如果 v 是一个结构类型描述符的值, 则返回 #t ,否则返回 #f 。

procedure

(struct-constructor-procedure? v) → boolean?

  v : any/c
如果 v 是由 struct 或 make-struct-type 生成的构造程序,则返回 #t ,否则返回 #f 。

procedure

(struct-predicate-procedure? v) → boolean?

  v : any/c
如果 v 是由 struct 或 make-struct-type 生成的谓词过程,则返回 #t ,否则返回 #f 。

procedure

(struct-accessor-procedure? v) → boolean?

  v : any/c
如果 v 是由 struct, make-struct-type 或 make-struct-field-accessor 生成的存取程序,则返回 #t ,否则返回 #f 。

procedure

(struct-mutator-procedure? v) → boolean?

  v : any/c
如果 v 是由 struct, make-struct-type 或 make-struct-field-mutator 生成的mutator过程,则返回 #t ,否则返回 #f 。

procedure

(prefab-struct-key v) → (or/c #f symbol? list?)

  v : any/c
如果 v 不是 prefab 结构类型的实例,则返回 #f 。 否则,结果是可以用 make-prefab-struct 来创建结构类型实例的短键。

Examples:
> (prefab-struct-key #s(cat "Garfield"))

'cat

> (struct cat (name) #:prefab)
> (struct cute-cat cat (shipping-dest) #:prefab)
> (cute-cat "Nermel" "Abu Dhabi")

'#s((cute-cat cat 1) "Nermel" "Abu Dhabi")

> (prefab-struct-key (cute-cat "Nermel" "Abu Dhabi"))

'(cute-cat cat 1)

procedure

(make-prefab-struct key v ...) → struct?

  key : prefab-key?
  v : any/c
创建一个 prefab 结构类型的实例,使用 v 作为字段值。 key 和 v 的数量决定了 prefab 结构类型。

一个 key 根据一个包含以下项目的列表来确定结构类型:

  • 一个结构类型名称的符号。

  • 一个精确的非负整数,代表结构类型中非自动字段的数量,不包括来自超类型的字段(如果有)。

  • 一个包含两项内容的列表,其中第一项是一个精确的非负整数, 代表结构类型中不来自超类型的自动字段的数量(如果有的话),第二项内容是一个任意值,是自动字段的值。

  • 一个精确的非负整数的向量,表示结构类型中可变的非自动字段, 从 0 开始计算,不包括来自超类型的字段(如果有)。

  • 如果该结构类型没有超类型,则没有其他内容。 否则,列表的其余部分是超类型的关键。

一个空的向量和一个以 0 开头的自动字段列表可以被省略。 此外,第一个整数(表示非自动字段的数量)可以省略,因为它可以从提供的 v 的数量中推断出来。 最后,可以用一个符号代替只包含一个符号的列表(在结构类型没有超类型、没有自动字段、没有可变字段的情况下)。

总字段数不得超过 32768。 如果 key 表示的字段数与提供的 v 的数量不一致,将引发 exn:fail:contract exception is raised 。

Examples:
> (make-prefab-struct 'clown "Binky" "pie")

'#s(clown "Binky" "pie")

> (make-prefab-struct '(clown 2) "Binky" "pie")

'#s(clown "Binky" "pie")

> (make-prefab-struct '(clown 2 (0 #f) #()) "Binky" "pie")

'#s(clown "Binky" "pie")

> (make-prefab-struct '(clown 1 (1 #f) #()) "Binky" "pie")

'#s((clown (1 #f)) "Binky" "pie")

> (make-prefab-struct '(clown 1 (1 #f) #(0)) "Binky" "pie")

'#s((clown (1 #f) #(0)) "Binky" "pie")

procedure

(prefab-struct-type-key+field-count type)

 → (or/c #f (cons/c prefab-key? (integer-in 0 32768)))
  type : struct-type?
如果 structure type descriptor type 代表一个 prefab 结构类型, 则返回一对包含 prefab 键和字段计数的数据,否则返回 #f 。

Added in version 8.5.0.8 of package base.

procedure

(prefab-key->struct-type key field-count) → struct-type?

  key : prefab-key?
  field-count : (integer-in 0 32768)
返回由 key 和 field-count 组合指定的 prefab 结构类型的 structure type descriptor 。

如果 key 指示的字段数与 field-count 不一致,将引发 exn:fail:contract exception is raised 。

procedure

(prefab-key? v) → boolean?

  v : any/c
如果 v 可以成为一个 prefab 结构类型的 key,返回 #t ,否则返回 #f 。

参见 make-prefab-struct 关于有效键型的描述。

5.6.1 额外的结构实用程序

 (require racket/struct) package: base
The bindings documented in this section are provided by the racket/struct library, not racket/base or racket.

procedure

(make-constructor-style-printer get-constructor 
  get-contents) 
 → (-> any/c output-port? (or/c #t #f 0 1) void?)
  get-constructor : (-> any/c (or/c symbol? string?))
  get-contents : (-> any/c sequence?)
产生一个适合作为 gen:custom-write 或 prop:custom-write 的值的函数。 该函数以 "构造函数风格" 打印值。 当值被 print 为表达式时,它被显示为构造函数(由 get-constructor 返回)对内容(由 get-contents 返回)的一个应用。 当给 write 时,它被显示为一个不可读的值,构造函数和内容用冒号隔开。

Examples:
> (struct point (x y)
    #:methods gen:custom-write
    [(define write-proc
       (make-constructor-style-printer
        (lambda (obj) 'point)
        (lambda (obj) (list (point-x obj) (point-y obj)))))])
> (print (point 1 2))

(point 1 2)

> (write (point 1 2))

#<point: 1 2>

该函数也与 pretty-print 合作:

> (parameterize ((pretty-print-columns 10))
    (pretty-print (point 3000000 4000000)))

(point

 3000000

 4000000)

> (parameterize ((pretty-print-columns 10))
    (pretty-write (point 3000000 4000000)))

#<point:

 3000000

 4000000>

请注意,打印机使用一个单独的属性 prop:custom-print-quotable 来确定一个结构实例是否可引用。 如果是,打印机可以在某些情况下以 write 模式打印它,比如在列表中。例如:
> (print (list (point 1 2) (point 3 4)))

'(#<point: 1 2> #<point: 3 4>)

使用 #:property prop:custom-print-quotable 'never 来防止结构实例被视为可引用。比如说:
> (struct point2 (x y)
    #:property prop:custom-print-quotable 'never
    #:methods gen:custom-write
    [(define write-proc
       (make-constructor-style-printer
        (lambda (obj) 'point)
        (lambda (obj) (list (point2-x obj) (point2-y obj)))))])
> (print (list (point2 1 2) (point2 3 4)))

(list (point 1 2) (point 3 4))

关键词参数可以用 unquoted-printing-string 来模拟:

; Private implementation
> (struct kwpoint-impl (x y)
    #:methods gen:custom-write
    [(define write-proc
       (make-constructor-style-printer
        (lambda (obj) 'kwpoint)
        (lambda (obj)
          (list (unquoted-printing-string "#:x")
                (kwpoint-impl-x obj)
                (unquoted-printing-string "#:y")
                (kwpoint-impl-y obj)))))])
; Public ``constructor''
> (define (kwpoint #:x x #:y y)
    (kwpoint-impl x y))
; Example use
> (print (kwpoint #:x 1 #:y 2))

(kwpoint #:x 1 #:y 2)

> (write (kwpoint #:x 3 #:y 4))

#<kwpoint: #:x 3 #:y 4>

Added in version 6.3 of package base.

procedure

(struct->list v [#:on-opaque on-opaque]) → (or/c list? #f)

  v : any/c
  on-opaque : (or/c 'error 'return-false 'skip) = 'error
返回一个包含结构实例 v 的字段的列表。 与 struct->vector 不同的是,结构名称本身并不包括在内。

如果 v 的任何字段无法通过当前检查器访问, 那么 struct->list 的行为将由 on-opaque 决定。 如果 on-opaque 是 'error (默认值),就会产生一个错误。 如果它是 'return-false, struct->list 会返回 #f 。 如果是 'skip,则从列表中省略不可访问的字段。

Examples:
> (struct open (u v) #:transparent)
> (struct->list (open 'a 'b))

'(a b)

> (struct->list #s(pre 1 2 3))

'(1 2 3)

> (struct secret open (x y))
> (struct->list (secret 0 1 17 22))

struct->list: expected argument of type <non-opaque struct>;

given: (secret 0 1 ...)

> (struct->list (secret 0 1 17 22) #:on-opaque 'return-false)

#f

> (struct->list (secret 0 1 17 22) #:on-opaque 'skip)

'(0 1)

> (struct->list 'not-a-struct #:on-opaque 'return-false)

#f

> (struct->list 'not-a-struct #:on-opaque 'skip)

'()

Added in version 6.3 of package base.
