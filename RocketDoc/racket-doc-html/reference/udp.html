15.3.2 UDP

 (require racket/udp) package: base
The bindings documented in this section are provided by the racket/udp and racket libraries, but not racket/base.

关于 UDP 的一般信息,见 W.Richard Stevens 的 TCP/IP Illustrated,Volume 1 。

procedure

(udp-open-socket [family-hostname    
  family-port-no]) → udp?
  family-hostname : (or/c string? #f) = #f
  family-port-no : (or/c port-number? #f) = #f
创建并返回一个 UDP socket ,用于发送和接收数据报(允许广播)。 最初,该套接字没有绑定或连接到任何地址或端口。

如果 family-hostname 或 family-port-no 不是 #f, 那么套接字的协议族就由这些参数决定。 该套接字 没有 绑定到主机名或端口号。 例如,参数可能是将通过套接字发送消息的主机名和端口,这可以确保套接字的协议族与目的地一致。 另外,参数可能与将来调用 udp-bind! 时的参数相同,这样可以确保套接字的协议族与绑定的协议族一致。 如果 family-hostname 和 family-port-no 都不是 #f, 那么该套接字的协议族就是 IPv4。

procedure

(udp-bind! udp-socket    
  hostname-string    
  port-no    
  [reuse?]) → void?
  udp-socket : udp?
  hostname-string : (or/c string? #f)
  port-no : listen-port-number?
  reuse? : any/c = #f
将一个未绑定的 udp-socket 绑定到本地端口号 port-no 。 如果 port-no 为0,则 udp-socket 被绑定到一个短暂的端口, 这可以通过调用 udp-addresses 来确定。

如果 hostname-string 是 #f, 那么套接字接受与所有监听机器的IP地址在 port-no 的连接。 否则,套接字只接受与给定名称相关的IP地址的连接。 例如,提供 "127.0.0.1" 作为 hostname-string 通常会创建一个只接受来自本地机器的 "127.0.0.1" 连接的监听器。

一个套接字在被绑定到一个本地地址和端口之前不能接收数据报。 如果一个套接字在被用于发送程序 udp-send, udp-send-to 等之前没有被绑定, 那么发送程序会将套接字绑定到一个随机的本地端口。 同样,如果 udp-send-evt 或 udp-send-to-evt 中的一个事件被选择用于同步(见 事件), 那么该套接字就被绑定了;如果该事件没有被选择,该套接字可能成为也可能不成为绑定。

被绑定的套接字的绑定不能被改变,但有一个例外:在一些系统上,如果套接字在发送时被自动绑定, 如果套接字通过 udp-connect! 被断开,并且如果套接字后来在发送时再次被使用, 那么后来的发送可能会改变套接字的自动绑定。

如果 udp-socket 已经被绑定或关闭,就会引发 exn:fail:network exception is raised 。

如果 reuse? 参数为真,那么 udp-bind! 将在绑定前设置 SO_REUSEADDR 套接字选项, 允许在使用 UDP 组播时在一台机器上的许多进程之间共享对 UDP 端口的访问。

procedure

(udp-connect! udp-socket    
  hostname-string    
  port-no) → void?
  udp-socket : udp?
  hostname-string : (or/c string? #f)
  port-no : (or/c port-number? #f)
如果 hostname-string 是一个字符串, port-no 是一个精确的整数, 将套接字连接到指定的远程地址和端口。

如果 hostname-string 是 #f, 那么 port-no 也必须是 #f, 并且端口被断开(如果已经连接)。 如果 hostname-string 或 port-no 中的一个是 #f , 而另一个不是,就会引发 exn:fail:contract exception is raised 。

连接的套接字可以用于 udp-send (而不是 udp-send-to), 它只接受来自连接的地址和端口的数据报。 一个套接字不需要被连接来接收数据报。 一个套接字可以被连接、重新连接和断开任意次数。

如果 udp-socket 被关闭,会引发 exn:fail:network exception is raised 。

procedure

(udp-send-to udp-socket    
  hostname    
  port-no    
  bstr    
  [start-pos    
  end-pos]) → void?
  udp-socket : udp?
  hostname : string?
  port-no : port-number?
  bstr : bytes?
  start-pos : exact-nonnegative-integer? = 0
  end-pos : exact-nonnegative-integer? = (bytes-length bstr)
将 (subbytes bytes start-pos end-pos) 作为数据报从未连接的 udp-socket 发送到端口为 port-no 的远程机器 hostname-address 的socket。 udp-socket 不需要被绑定或连接;如果它没有被绑定, udp-send-to 将它绑定到一个随机的本地端口。 如果套接字的出站数据报队列太满,无法支持发送, udp-send-to 就会阻塞,直到数据报可以被排队。

如果 start-pos 大于 bstr 的长度, 或者 end-pos 小于 start-pos 或大于 bstr 的长度, 就会引发 exn:fail:contract exception is raised .

如果 udp-socket 被关闭或连接,则引发 exn:fail:network exception is raised 。

procedure

(udp-send udp-socket bstr [start-pos end-pos]) → void?

  udp-socket : udp?
  bstr : bytes?
  start-pos : exact-nonnegative-integer? = 0
  end-pos : exact-nonnegative-integer? = (bytes-length bstr)
和 udp-send-to 一样,除了 udp-socket 必须是连接的,而且数据报会被发送到连接目标。 如果 udp-socket 被关闭或未连接,会引发 exn:fail:network exception is raised 。

procedure

(udp-send-to* udp-socket    
  hostname    
  port-no    
  bstr    
  [start-pos    
  end-pos]) → boolean?
  udp-socket : udp?
  hostname : string?
  port-no : port-number?
  bstr : bytes?
  start-pos : exact-nonnegative-integer? = 0
  end-pos : exact-nonnegative-integer? = (bytes-length bstr)
与 udp-send-to 类似,但从不阻塞;如果套接字的出站队列太满,无法支持发送,则返回 #f f, 否则数据报被排队,结果是 #t 。

procedure

(udp-send* udp-socket bstr [start-pos end-pos]) → boolean?

  udp-socket : udp?
  bstr : bytes?
  start-pos : exact-nonnegative-integer? = 0
  end-pos : exact-nonnegative-integer? = (bytes-length bstr)
和 udp-send 一样,除了(和 udp-send-to 一样)它从不阻塞并返回 #f 或 #t 。

procedure

(udp-send-to/enable-break udp-socket    
  hostname

  [start-pos    
  end-pos]) → void?
  udp-socket : udp?

  bstr : bytes?

像 udp-send-to 一样,但是在试图发送数据报的时候会启用 breaking(见 中断) 。 如果在调用 udp-send-to/enable-break 时禁用 break,那么要么发送数据报, 要么引发 exn:break ,但不能同时发送。

procedure

(udp-receive! udp-socket    
  bstr    
  [start-pos    
  end-pos]) → 
exact-nonnegative-integer?
string?
port-number?
  udp-socket : udp?
  bstr : (and/c bytes? (not immutable?))
  start-pos : exact-nonnegative-integer? = 0
  end-pos : exact-nonnegative-integer? = (bytes-length bstr)
接受 udp-socket 的下一个传入数据报的 end-pos-start-pos 字节到 bstr ,从 bstr 中的 start-pos 位置开始写数据报字节。 udp-socket 必须被绑定到一个本地地址和端口(但不需要连接)。 如果没有立即收到的数据报, udp-receive! 就会阻塞,直到有数据报可用。

将返回三个值:收到的字节数(在 0 和 end-pos-start-pos 之间), 表示数据报源地址的主机名字符串,和表示数据报源端口的整数。 如果收到的数据报长于 end-pos-start-pos 字节,剩余的部分被丢弃。

如果 start-pos 大于 bstr 的长度, 或者 end-pos 小于 start-pos 或大于 bstr 的长度, 会产生 exn:fail:contract exception is raised 。

procedure

(udp-set-receive-buffer-size! udp-socket    
  size) → void?
  udp-socket : udp?
  size : exact-positive-integer?
设置 udp-socket 的接收缓冲区大小 (SO_RCVBUF) 。 使用较大的缓冲区可以最大限度地减少由于连接的缓慢轮询而可能发生的数据包丢失,包括在主要的垃圾收集期间。

如果 size 超过系统允许的最大值,则会引发 exn:fail:network exception is raised 。

Added in version 7.1.0.11 of package base.

procedure

(udp-close udp-socket) → void?

  udp-socket : udp?
关闭 udp-socket, 丢弃未接收的数据报。 如果套接字已经被关闭,则会引发 exn:fail:network exception is raised 。

(udp? v) → boolean?

(udp-connected? udp-socket) → boolean?

  udp-socket : udp?
如果 udp-socket 被连接到一个远程地址和端口,则返回 #t ,否则返回 #f 。

procedure

(udp-send-ready-evt udp-socket) → evt?

  udp-socket : udp?
返回一个 synchronizable event (见 事件) , 当 udp-socket 上的 udp-send-to 会阻塞时,该事件处于阻塞状态。 synchronization result 就是事件本身。

procedure

(udp-receive-ready-evt udp-socket) → evt?

  udp-socket : udp?
返回一个 synchronizable event (见 事件), 当 udp-socket 上的 udp-receive! 会被阻塞时,该事件处于阻塞状态。 synchronization result 是事件本身。

procedure

(udp-send-to-evt udp-socket    
  hostname    
  port-no    
  bstr    
  [start-pos    
  end-pos]) → evt?
  udp-socket : udp?
  hostname : string?
  port-no : port-number?
  bstr : bytes?
  start-pos : exact-nonnegative-integer? = 0
  end-pos : exact-nonnegative-integer? = (bytes-length bstr)
返回一个 synchronizable event 。 当 udp-socket 的 udp-send-to 会阻塞时, 该事件处于阻塞状态。 否则,如果事件在同步中被选择,数据将被发送, 因为 (udp-send-to udp-socket hostname-address port-no bstr start-pos end-pos), 并且同步结果是 #<void> 。(如果不选择该事件,则不发送字节)。

procedure

(udp-send-evt udp-socket    
  bstr    
  [start-pos    
  end-pos]) → evt?
  udp-socket : udp?
  bstr : bytes?
  start-pos : exact-nonnegative-integer? = 0
  end-pos : exact-nonnegative-integer? = (bytes-length bstr)
返回一个 synchronizable event 。 当 udp-socket 上的 udp-send 不会阻塞时,该事件为 ready for synchronization 。 否则,如果事件在同步中被选择,数据将被发送,因为 (udp-send-to udp-socket bstr start-pos end-pos), synchronization result 是 #<void> 。 (如果没有选择事件,则不发送字节)。 如果 udp-socket 被关闭或未连接,在同步尝试中会引发 exn:fail:network exception is raised 。

procedure

(udp-receive!-evt udp-socket    
  bstr    
  [start-pos    
  end-pos]) → evt?
  udp-socket : udp?
  bstr : (and/c bytes? (not immutable?))
  start-pos : exact-nonnegative-integer? = 0
  end-pos : exact-nonnegative-integer? = (bytes-length bstr)
返回一个 synchronizable event 。 当 udp-socket 上的 udp-receive 不会阻塞时, 该事件为 ready for synchronization 。 否则,如果该事件在同步中被选中,数据将被接收到 bstr 中, 因为 (udp-receive! udp-socket bytes start-pos end-pos), synchronization result 是三个值的列表,对应于 udp-receive! 的三个结果。 (如果不选择该事件,则不会收到任何字节,而且 bstr 的内容也不会被修改)。

procedure

(udp-addresses udp-port [port-numbers?])

 → 
(or/c (values string? string?)
      (values string? listen-port-number?
              string? listen-port-number?))
  udp-port : udp?
  port-numbers? : any/c = #f
当 port-numbers? 为 #f (默认)时,返回两个字符串。 第一个字符串是给定的 UDP socket 连接所查看的本地机器的互联网地址。 (对于大多数机器来说,答案与当前机器的唯一互联网地址相对应,但是当一台机器提供多个地址时,结果是特定的连接。) 第二个字符串是连接的另一端的互联网地址。

如果 port-numbers? 为 true,那么将返回四个结果:一个字符串表示本地机器的地址, 一个 1 到 65535 之间的精确整数表示本地机器的端口号, 如果套接字未被绑定则为 0 ;一个字符串表示远程机器的地址,一个 1 到 65535 之间的精确整数表示远程机器的端口号,如果套接字未被连接则为 0 。

如果给定的端口已被关闭,将引发 exn:fail:network exception is raised 。

  multicast-addr

添加或删除 udp-socket 到一个指定的组播组。

multicast-addr 参数必须是一个有效的 IPv4 组播IP地址;例如, "224.0.0.251" 是 mDNS 协议的适当地址。 hostname 参数选择套接字用来接收(而不是发送)组播数据报的接口; 如果 hostname 是 #f 或 "0.0.0.0", 内核会自动选择一个接口。

离开一个组需要与加入该组时相同的 multicast-addr 和 hostname 参数。
