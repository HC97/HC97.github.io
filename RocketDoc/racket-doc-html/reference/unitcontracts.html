7.9 单元契约

syntax

(unit/c
  (import sig-spec-block ...)
  (export sig-spec-block ...)
  init-depends-decl
  optional-body-ctc)
 
sig-spec-block = (tagged-sig-spec [id contract] ...)
  | tagged-sig-spec
     
init-depends-decl = 
  | (init-depend tagged-sig-id ...)
     
optional-body-ctc = 
  | contract
  | (values contract ...)
一个 unit contract 包裹着一个单元,并检查其导入和导出的标识符,以确保它们与适当的合约相匹配。 这允许程序员为单个单元值添加合约检查,而无需为导入和导出的签名添加合约。

单位价值必须导入单位合同中所列的导入签名的子集,并导出单位合同中所列的导出签名的超集。 此外,单元值必须声明初始化依赖性,这些依赖性是单元合约中指定的那些依赖性的一个子集。 任何没有为给定签名列出的标识符都是单独的。 在一个给定的 contract 表达式中使用的变量首先是指任何列出的签名中的其他变量, 然后是指 unit/c 表达式的上下文。 如果指定了一个主体契约,那么调用单元值的结果将被包裹在给定的契约中,否则值将按原样返回。

Changed in version 8.8.0.7 of package base: 修改了 sig-spec-block 以允许任意的 tagged-sig-spec ,而不是只允许 tagged-sig-id 。 使得来自 all 签名的绑定在每个 contract 表达式的范围内可见,而不是只允许来自同一签名的绑定。 此外,签名绑定的契约在 contract 表达式中被强制执行。

syntax

(define-unit/contract unit-id
  (import sig-spec-block ...)
  (export sig-spec-block ...)
  init-depends-decl
  optional-body-ctc
  unit-body-expr-or-defn
  ...)
 
sig-spec-block = (tagged-sig-spec [id contract] ...)
  | tagged-sig-spec
     
optional-body-ctc = 
  | #:invoke/contract contract
  | #:invoke/contract (values contract ...)
define-unit/contract 形式定义了一个与链接推理兼容的单元,其进口和出口都是用单元合同来约定的。 单位名称用于合同的正责。

Changed in version 8.8.0.7 of package base: 使得来自 all 签名的绑定在每个 contract 表达式的范围内可见, 而不是只有来自同一签名的绑定。 此外,签名绑定的契约在 contract 表达式中被强制执行。
