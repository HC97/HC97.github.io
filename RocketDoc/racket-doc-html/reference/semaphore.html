11.2.3 Semaphores

一个 semaphore 有一个内部计数器;当这个计数器为 0 时, semaphore 可以阻止一个线程的执行(通过 semaphore-wait), 直到另一个线程增加这个计数器(使用 semaphore-post)。 一个 semaphore 的内部计数器的最大值与平台有关,但总是至少为 10000 。

一个 semaphore 的计数器是以单线程的方式更新的,因此 semaphores 可以被用于可靠的同步。 信号灯的等待是 fair: 如果一个线程在一个信号灯上被阻塞,并且信号灯的内部值无限次为非零, 那么这个线程最终会被解除阻塞。

除了与 semaphore 特定的程序一起使用外, semaphore 还可以作为一个 synchronizable event 来使用(参见 事件)。 当 semaphore-wait 不会阻塞时,一个 semaphore 是 ready for synchronization 。 在同步化时,信号灯的计数器被递减, the synchronization result of a semaphore is the semaphore itself 。

procedure

(semaphore? v) → boolean?

  v : any/c
如果 v 是一个 semaphore,则返回 #t ,否则返回 #f 。

procedure

(make-semaphore [init]) → semaphore?

  init : exact-nonnegative-integer? = 0
创建并返回一个新的 semaphore,其计数器最初设置为 init 。 如果 init 大于一个 semaphore 的最大内部计数器值,则会产生 exn:fail exception is raised 。

procedure

(semaphore-post sema) → void?

  sema : semaphore?
增加信号灯的内部计数器并返回 #<void> 。 如果 semaphore 的内部计数器已经达到其最大值,则会产生 exn:fail exception is raised 。

procedure

(semaphore-wait sema) → void?

  sema : semaphore?
阻断,直到 semaphore sema 的内部计数器为非零。 当计数器为非零时,它被递减, semaphore-wait 返回 #<void> 。

procedure

(semaphore-try-wait? sema) → boolean?

  sema : semaphore?
与 semaphore-wait 类似, 但是 semaphore-try-wait? 不会阻塞执行。 如果 sema 的内部计数器为 0, semaphore-try-wait? 立即返回 #f 而不递减计数器。 如果 sema 的计数器为正数,则会被递减并返回 #t 。

procedure

(semaphore-wait/enable-break sema) → void?

  sema : semaphore?
像 semaphore-wait 一样, 但是在等待 sema 的时候启用break(见 中断)。 如果在调用 semaphore-wait/enable-break 时禁用 break,那么要么 semaphore 的计数器被递减, 要么 exn:break 被引发,但不会同时发生。

procedure

(semaphore-peek-evt sema) → semaphore-peek-evt?

  sema : semaphore?
创建并返回一个新的 synchronizable event (例如,与 sync 一起使用),当 sema 准备好时, ready for synchronization ,但是同步事件并不递减 sema 的内部计数。 The synchronization result of a semaphore-peek event is the semaphore-peek event itself.

procedure

(semaphore-peek-evt? v) → boolean?

  v : any/c
如果 v 是由 semaphore-peek-evt 产生的 semaphore 包装器, 则返回 #t ,否则返回 #f .

procedure

(call-with-semaphore sema    
  proc    
  [try-fail-thunk]    
  arg ...) → any
  sema : semaphore?
  proc : procedure?
  try-fail-thunk : (or/c (-> any) #f) = #f
  arg : any/c
使用 semaphore-wait 在 sema 上等待,用所有的 arg 调用 proc , 然后发布到 sema 。 continuation barrier 阻止了进入或离开 proc 的完全延续跳转(见 提示、限定的延续和障碍物), 但允许转义跳转,并且 sema 在转义时被发布。 如果提供了 try-fail-thunk ,并且不是 #f , 那么就在 sema 上调用 semaphore-try-wait? , 而不是 semaphore-wait ,如果等待失败就调用 try-fail-thunk 。

procedure

(call-with-semaphore/enable-break sema    
  proc    
  [try-fail-thunk]    
  arg ...) → any
  sema : semaphore?
  proc : procedure?
  try-fail-thunk : (or/c (-> any) #f) = #f
  arg : any/c
与 call-with-semaphore 类似, 除了 semaphore-wait/enable-break 在非尝试模式下与 sema 一起使用。 当提供 try-fail-thunk 而不是 #f 时, 那么在 sema 上使用 semaphore-try-wait? 时,会启用 break。
