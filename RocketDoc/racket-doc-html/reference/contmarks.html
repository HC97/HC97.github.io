10.5 Continuation Marks

参见 延续框架和标记 和 提示、限定的延续和障碍物 ,了解有关延续标记的一般信息。

一个键 k 和一个扩展了 C0 的续集 C 的续集标记列表定义如下:

  • 如果 C 是一个空的延续,那么标记列表为 null 。

  • 如果 C 的第一帧包含 k 的标记 m , 那么 C 的标记列表是 (cons m lst), 其中 lst 是 C0 中 k 的标记列表。

  • 如果 C 的第一帧不包含以 k 为键的标记, 那么 C 的标记列表就是 C0 的标记列表。

with-continuation-mark 形式在当前延续的第一帧上安装一个标记(见 延续标记: with-continuation-mark)。 像 current-continuation-marks 这样的程序允许检查标记。

每当 Racket 为一个原始异常创建一个异常记录时, 它就会用(current-continuation-marks) 的值填充 continuation-marks 字段, 从而提供一个异常发生时的 continuation 标记的快照。

当调用由 call-with-current-continuation 或 call-with-composable-continuation 返回的 continuation 过程时, 它将恢复捕获的 continuation,并将 continuation 框架中的标记恢复到调用 call-with-current-continuation 或 call-with-composable-continuation 时存在的标记。

procedure

(continuation-marks cont [prompt-tag]) → continuation-mark-set?

  cont : (or/c continuation? thread? #f)
  prompt-tag : continuation-prompt-tag?
   = (default-continuation-prompt-tag)
返回一个不透明的值,其中包含 continuation cont (如果是线程,则为 cont 的当前 continuation)中所有键的 continuation 标记集, 直到由 prompt-tag 标记的提示。 如果 cont 是 #f ,产生的延续标记集是空的。 如果 cont 是一个转义续集(见 提示、限定的延续和障碍物), 那么当前续集必须扩展 cont ,否则会引发 exn:fail:contract exception is raised 。 如果 cont 不是相对于 prompt-tag 捕获的, 并且不包括对 prompt-tag 的提示,则会引发 exn:fail:contract exception is raised 。 如果 cont 是一个死线程,结果是一个空的延续标记集。

procedure

(current-continuation-marks [prompt-tag])

 → continuation-mark-set?
  prompt-tag : continuation-prompt-tag?
   = (default-continuation-prompt-tag)
返回一个不透明的值,其中包含当前续篇中所有键的续篇标记集,直到 prompt-tag 。 换句话说,它产生的值与

(call-with-current-continuation
  (lambda (k)
    (continuation-marks k prompt-tag))
  prompt-tag)

procedure

(continuation-mark-set->list mark-set    
  key-v    
  [prompt-tag]) → list?
  mark-set : (or/c continuation-mark-set? #f)
  key-v : any/c
  prompt-tag : continuation-prompt-tag?
   = (default-continuation-prompt-tag)
返回一个新创建的包含 mark-set 中 key-v 的标记的列表, mark-set 是由 current-continuation-marks 或 #f 作为 (current-continuation-marks prompt-tag) 的缩写返回的标记集。 结果列表被截断在第一个点上,如果有的话,在这个点上,延续帧最初被一个用 prompt-tag 标记的提示符分开。 产生结果所需的时间与 mark-set 所反映的延续帧的大小成正比。

Changed in version 8.0.0.1 of package base: 改为允许 mark-set 为 #f 。

procedure

(continuation-mark-set->list* mark-set    
  key-list    
  [none-v    
  prompt-tag]) → (listof vector?)
  mark-set : (or/c continuation-mark-set? #f)
  key-list : (listof any/c)
  none-v : any/c = #f
  prompt-tag : continuation-prompt-tag?
   = (default-continuation-prompt-tag)
返回一个新创建的列表,其中包含 key-list 中的键的 mark-set 中的标记向量,直到 prompt-tag, 其中 mark-set 的 #f 值相当于 (current-continuation-marks prompt-tag)。 结果列表中每个向量的长度与 key-list 的长度相同,一个特定向量位置上的值就是 key-list 中相应键的值。 只有当标记为 mark-set 中的同一延续帧时,多个键的值才会出现在一个向量中。 none-v 参数用于向量元素,表示没有值。 产生结果所需的时间与 mark-set 反映的续集的大小乘以 key-list 的长度成正比。

Changed in version 8.0.0.1 of package base: 改为允许 mark-set 为 #f 。

procedure

(continuation-mark-set->iterator mark-set 
  key-list 
  [none-v 
  prompt-tag]) 
 → (-> (values (or/c vector? #f) procedure?))
  mark-set : (or/c continuation-mark-set? #f)
  key-list : (listof any/c)
  none-v : any/c = #f
  prompt-tag : continuation-prompt-tag?
   = (default-continuation-prompt-tag)
像 continuation-mark-set->list*一样,但不是返回一个值的列表, 而是返回一个功能性的迭代器,其形式是一个过程,返回会列表的一个元素和一个新的迭代器函数用于会列表的其余部分。 当没有更多的元素可用时,迭代过程会返回 #f 而不是一个向量; 在这种情况下,返回的迭代过程与调用的迭代过程一样,不会产生更多的值。 每一步所需的时间与 key-list 的长度乘以 mark-set 所反映的在 key-list 中有 key 的帧之间的延续段的大小成正比。

Added in version 7.5.0.7 of package base.
Changed in version 8.0.0.1: 改为允许 mark-set 为 #f 。

procedure

(continuation-mark-set-first mark-set    
  key-v    
  [none-v    
  prompt-tag]) → any
  mark-set : (or/c continuation-mark-set? #f)
  key-v : any/c
  none-v : any/c = #f
  prompt-tag : continuation-prompt-tag?
   = (default-continuation-prompt-tag)
返回由 (continuation-mark-set->list (or mark-set (current-continuation-marks prompt-tag)) key-v prompt-tag) 返回的列表的第一个元素, 如果结果是空列表,则返回 none-v 。

这个结果是在(摊销后的)恒定时间内产生的。 通常情况下,使用 continuation-mark-set-first 计算这个结果比使用 continuation-mark-set->list 或者使用 continuation-mark-set->iterator 并只迭代一次更快。

尽管 #f 和 (current-continuation-marks prompt-tag) 对于 mark-set 来说是等价的, 但提供 #f 作为 mark-set 可以实现捷径,使其更加快速。

procedure

(call-with-immediate-continuation-mark key-v    
  proc    
  [default-v]) → any
  key-v : any/c
  proc : (any/c . -> . any)
  default-v : any/c = #f
用当前延续的第一帧中与 key-v 相关的值来调用 proc (即,如果调用 call-with-immediate-continuation-mark 被替换为 with-continuation-mark 形式,使用 key-v 作为关键表达式,那么这个值将被替换)。 如果第一帧中不存在这样的值,则将 default-v 传递给 proc 。 相对于 call-with-immediate-continuation-mark 的调用, proc 在尾部位置被调用。

这个函数可以用 with-continuation-mark, current-continuation-marks 和 continuation-mark-set->list* 的组合来实现,如下图所示, 但 call-with-immediate-continuation-mark 的实现更有效率;它只检查当前续集的第一帧。

; Equivalent, but inefficient:
(define (call-with-immediate-continuation-mark key-v proc [default-v #f])
  (define private-key (gensym))
  (with-continuation-mark
   private-key #t
   (let ([vecs (continuation-mark-set->list* (current-continuation-marks)
                                             (list key-v private-key)
                                             default-v)])
     (proc (vector-ref (car vecs) 0)))))

procedure

(make-continuation-mark-key) → continuation-mark-key?

(make-continuation-mark-key sym) → continuation-mark-key?
  sym : symbol?
创建一个与任何其他值(包括 make-continuation-mark-key 的先前和未来结果) 不 equal? 的继续标记键。 继续标记键可以作为 with-continuation-mark 或访问程序 (如 continuation-mark-set-first 的键参数。 与其他用作标记键的值不同,标记键可以被陪衬或冒充。

如果提供了可选的 sym 参数,则会在打印延续标记时使用。

procedure

(continuation-mark-key? v) → boolean?

  v : any/c
如果 v 是由 make-continuation-mark-key 创建的标记键, 返回 #t ,否则返回 #f 。

procedure

(continuation-mark-set? v) → boolean?

  v : any/c
如果 v 是一个由 continuation-marks 或 current-continuation-marks 创建的标记集,则返回 #t ,否则返回 #f 。

procedure

(continuation-mark-set->context mark-set    
  [realms?]) → list?
  mark-set : continuation-mark-set?
  realms? : any/c = #f
返回一个代表 mark-set 的延续的近似 “stack trace” 的列表。 如果 realms? 是 #f, 则该列表包含成对的内容, 其中每一对的 car 包含 #f 或者过程名称的符号, 每一对的 cdr 包含 #f 或者过程源位置的 srcloc 值(见 计算位置、行和列); car 和 cdr 永远不会同时是 #f 。 如果 realms? 为真,列表中包含3个元素的向量,其中前两个元素就像一对的值,第三个元素是一个境界符号。

从概念上讲,堆栈跟踪列表是 continuation-mark-set->list 与 mark-set 和 Racket 的程序调用标记私钥的结果。 然而,实现可能是不同的,结果可能只是近似于正确的答案。 因此,虽然结果可能包含对人类有用的关于表达式上下文的提示,但对于程序性使用来说,它不够可靠。

对于 exn:fail:user 以外的异常,堆栈跟踪会从异常中提取并由默认的错误显示处理程序(见 error-display-handler) 显示(见 引发异常 的 raise-user-error )。

Examples:
> (define (extract-current-continuation-marks key)
    (continuation-mark-set->list
     (current-continuation-marks)
     key))
> (with-continuation-mark 'key 'mark
    (extract-current-continuation-marks 'key))

'(mark)

> (with-continuation-mark 'key1 'mark1
    (with-continuation-mark 'key2 'mark2
      (list
       (extract-current-continuation-marks 'key1)
       (extract-current-continuation-marks 'key2))))

'((mark1) (mark2))

> (with-continuation-mark 'key 'mark1
    (with-continuation-mark 'key 'mark2 ; replaces previous mark
      (extract-current-continuation-marks 'key)))

'(mark2)

> (with-continuation-mark 'key 'mark1
    (list ; continuation extended to evaluate the argument
     (with-continuation-mark 'key 'mark2
        (extract-current-continuation-marks 'key))))

'((mark2 mark1))

> (let loop ([n 1000])
    (if (zero? n)
        (extract-current-continuation-marks 'key)
        (with-continuation-mark 'key n
          (loop (sub1 n)))))

'(1)
