1.4 打印机

Racket 打印机支持三种模式:

  • write 模式打印核心数据类型,其方式是在输出上使用 read , 产生一个与打印值 equal? 的值;

  • display模式将核心数据类型打印成 "终端用户" 风格,而不是 "程序员" 风格; 例如,一个字符串 display 为其内容的字符,没有周围的 " 或转义。

  • print 模式–默认情况下, 当 print-as-expression 为 #t 时– 打印大多数数据类型的方式是将输出作为表达式进行评估,产生一个与打印值 equal? 的值; 当 print-as-expression 被设置为 #f 时, print 模式就像 write 模式。

在 print 模式下,当 print-as-expression 为 #t (默认值)时, 一个值的打印深度为 quoting depth ,即 0 (无引号)或 1 (引号)。 初始引用深度被 print 接受为一个可选参数,一些复合数据类型的打印会调整组件值的打印深度。 例如,当打印一个列表时,引用深度为 0 ,并且其所有元素都是 quotable, 该列表被打印为 ' 前缀,并且该列表的元素被打印为引用深度 1。

当 print-graph 参数设置为 #t 时,那么打印机首先扫描一个对象以检测周期。 扫描会遍历对、可变对、向量、盒子(当 print-box 为 #t 时)、 哈希表(当 print-hash-table 为 #t ，且key被强持有时)、 由 struct->vector 暴露的结构字段(当 print-struct 为 #t 时), 以及当结构类型具有 prop:custom-write 属性时由打印暴露的结构字段。 如果 print-graph 是 #t, 那么这些信息将用于打印通过图定义和引用的共享(参见 读数图结构)。 如果在初始扫描中检测到一个循环,那么 print-graph 实际上被自动设置为 #t 。

除了显示 byte strings 之外,打印是以 Unicode 字符来定义的; 关于字符流如何写入端口的底层字节流的信息,请参见 端口 。

1.4.1 打印符号

含有空格或特殊字符的 Symbols 使用转义 \ 和引号 | write 。 当 read-case-sensitive 参数设置为 #f 时, 含有大写字母的符号也使用转义 \ 和引号 |。 此外,当符号与数字常数或划线的 . (当 read-accept-dot 为 #t)时)的打印方式相同时, 会用| 或前面的 \ 引号。

当 read-accept-bar-quote 为 #t 时, 当开头的一个 | 和结尾的一个 | 足以正确打印符号时,将使用 | 。 否则, \ 总是用于转义特殊字符,而不是用 | 引用它们。

当 read-accept-bar-quote 为 #f 时, 那么 | 被视为特殊字符。以下是永远的特殊字符:

   ( ) [ ] { } " , ' ` ; \

此外,当 # 出现在符号的开头,并且后面没有 % 时,它是一个特殊字符。

符号的 display 不需要转义或引用特殊字符。 也就是说,一个符号的显示形式与应用于该符号的 symbol->string 的显示形式相同。

符号的 print 方式与它们的 write 方式相同, 除非 print-as-expression 被设置为 #t (默认值), 并且当前的 quoting depth 为 0 。 在这种情况下,符号的 print 形式会以 ' 为前缀。 对于打印封闭数据类型的目的,符号是 quotable 。

1.4.2 打印数字

一个 number 在 write, display 和 print 模式下的打印方式是一样的。 为了打印封闭的数据类型,一个数字是 quotable。

一个不是 real number 的 complex number 总是打印成 ‹m›+‹n›i 或 ‹m›-‹n›i, 其中 ‹m› 和 ‹n› (对于非负的虚部)或 -‹n› (对于负的虚部)分别是其实部和虚部的打印形式。

一个精确的 0 打印为 0 。 一个正的、精确的 integer ,打印为不以 0 开头的数字序列。 一个正的、精确的、实数的、非整数的数字打印为 ‹m›/‹n›, 其中 ‹m› 和 ‹n› 是数字的分子和分母的打印形式(由 numerator 和 denominator 决定)。 负的 exact number 在打印时,在数字的精确负值的打印形式上加一个 - 的前缀。 当把一个数字打印成十六进制时(例如,通过 number->string), 数字 a 到 f 都以小写打印。 一个 #e 或小数点标记,如 #d 不会 给数字加前缀。

一个双精度的 inexact number (即 flonum) ,如果是 rational number ， 打印时要么有一个 . 小数点,要么有一个 e 指数标记和非零指数,要么两者都有。 选择这种形式是为了保持输出的简短性, 同时限制了将打印出来的形式读回去产生一个 equal? 的数字。 #i 不 作为数字的前缀,而且 # 永远不会被用来代替数字的位置。 + 不作为正数的前缀,但如果有 e , 则在指数前打印 + 或 - 。 正无穷大打印为 +inf.0,负无穷大打印为 -inf.0, 而非数字则打印为 +nan.0。

作为 rational number 的单精度 inexact number 的打印方式与双精度数相同, 但总是带有指数,用 f 代替 e 来表示该数的精度; 如果该数在其他情况下没有指数,则 0 (没有 +) 被打印为指数部分。 单精度正无穷大打印为 +inf.f, 负无穷大打印为 -inf.f, 而非数字打印为 +nan.f。

1.4.3 打印 Extflonums

在 write, display 和 print 模式下, extflonum 的打印方式相同。 对于打印封闭数据类型的目的,extflonum是 quotable。

一个 extflonum 的打印方式与单精度非精确数(参见 打印数字)相同, 但总是带有 t 或 T 的指数标记或作为 +inf.t, -inf.t 或 +nan.t 的后缀。 当支持 extflonum 操作时,打印总是使用小写的 t; 当不支持 extflonum 操作时,extflonum 的打印与它的阅读器(参见 阅读器)源相同, 因为读取是产生 extflonum 的唯一方法。

1.4.4 打印布尔值

根据 print-boolean-long-form 的值， boolean 常数 #t 在所有模式 (display, write 和 print) 下都打印为 #true 或 #t ，常数 #f 打印为 #false 或 #f。 为了打印封闭的数据类型,一个符号是 quotable。

1.4.5  打印对子和列表

在 write 和 display 模式下, 一个空的 list 打印为 ()。 一个 pair 通常以 ( 开始打印,后面是 car 的打印形式。 其余的打印形式取决于 cdr:

  • 如果 cdr 是一个对或空列表,那么对的打印形式与 cdr 的打印形式一起完成, 只是省略了 cdr 的打印形式中前面的 ( 。

  • 否则,对的打印形式将以一个空格， ., 另一个空格、 cdr 的打印形式和一个 ) 继续。

如果 print-reader-abbreviations 被设置为 #t, 那么在 write 模式下的对子打印将被调整,如果对子开始是一个两元素的列表, 其第一个元素是 'quote, 'quasiquote, 'unquote, 'unquote-splicing, 'syntax, 'quasisyntax, 'unsyntax 或 'unsyntax-splicing。 在这种情况下,该对句子将与相应的读者语法一起打印:分别是 ', `, ,, ,@, #', #`, #, 或 #,@。 在读者语法之后,列表的第二个元素被打印出来。 当列表是一个包围列表的尾部时,尾部会在包围列表中的 . 之后被打印出来 (在这之后,阅读器的缩写会起作用),而不是将尾部作为包围列表的两个元素。 如果阅读器语法 , 或 #, 后面有一个符号, 打印时有一个领先的 @, 那么打印机会在 @ 前面增加一个额外的空格。

在 write 模式和 display 模式下,对的打印形式都是一样的, 只是对的 car 和 cdr 的打印形式随模式不同而不同。 如果 print-as-expression 为 #f 或引用深度为 1 , 则 print 形式也是相同的。

对于 print 模式, 当 print-as-expression 为 #t , 且 quoting depth 为 0 时, 则空列表打印为 '()。 对于 car 和 cdr 是 quotable 的一对, 该对以 write 模式打印,但有一个 ' 前缀;该对的内容以 quoting depth 1 打印。 否则,当 car 或 cdr 不 quotable 时, 对以 cons (当 cdr 不是一个对时)), list (当对是一个列表时) 或 list* (否则)在开头的 ( 后打印, 任何本来要打印的 . 被抑制,并且对的内容以 quoting depth 0 打印。 在所有情况下,当 print 模式的 print-as-expression 为 #t 时, print-reader-abbreviations 的值被忽略, 读者缩写总是用于以 quoting depth 1 打印的列表。

默认情况下,可变对(如用 mcons 创建的)与 write 和 display 的对打印相同,只是用 { 和 } 代替了 ( 和 )。 请注意,阅读器在输入时对 {...} 和 (...) 的处理是等同的,在两种情况下都会创建不可变的对。 在 print-as-expression 为 #f 或 quoting depth 为 1 的 print 模式下, 可变对也使用 { 和 }。 在 print 模式下,如果 print-as-expression 为 #t， quoting depth 为 0, 则可变对打印为 (mcons , 在 quoting depth 为 0 时打印的 mcar 和 mcdr ,用空格隔开,还有一个结束符 )。

如果 print-pair-curly-braces 参数设置为 #t, 不使用 print-as-expression 为 #t 的 print 模式和 quoting depth 为 0 的情况下, 对使用 { 和 } 进行打印。 如果 print-mpair-curly-braces 参数设置为 #f, 那么在该模式下,可变对使用 ( 和 ) 打印。

为了打印封闭的数据类型,一个空的列表总是 quotable, 当一个对的 car 和 cdr 是 quotable 的时候, 它就是 quotable ,而一个可变的列表是永远不 quotable。

Changed in version 6.9.0.6 of package base: 在打印 , 或 #, 时增加了一个空格, 后面的符号打印时有一个前导的 @。

1.4.6 打印字符串

所有 strings 都 display 为其字面字符序列。

字符串的 write 或 print 形式以 " 开始, 以另一个 " 结束。 在 " 之间,每个字符都被表示。 每个图形或空白字符(根据 char-graphic? 和 char-blank?) 被表示为其本身,但有两个例外: " 被打印为 \", 而 \ 被打印为 \\。 一个非图形的、非空白的字符,如果是以图形字符开始的字素序列的一部分,也表示为其本身。 每个其他的非图形、非空白字符都使用 读取字符串 中描述的转义序列进行打印, 如果可能的话,使用 \a, \b, \t, \n, \v, \f, \r 或 \e ，否则使用带有四个十六进制数字的 \u 或带有八个十六进制数字的 \U (只有在字符值不适合四位数时才使用后者)。

所有的字节字符串都 display 为其字面的字节序列; 这个字节序列可能不是有效的 UTF-8 编码,所以它可能不对应于一个字符序列。

字节串的 write 或 print 形式以 #" 开始, 以 " 结束。 在 " 之间,如果字节在 0 到 127 之间,并且字符是图形或空白(根据 char-graphic? 和 char-blank?),每个字节都使用相应的ASCII解码来写入。 否则,如果可能的话,使用 \a, \b, \t, \n, \v, \f, \r 或 \e 来书写字节,否则使用 \ 后面跟着 1 到 3 个八进制数字(只要是必要的数字)。

为了打印包围的数据类型,一个字符串或一个字节字符串是 quotable。

1.4.7 打印矢量

在 display 模式下, vector 的打印形式是 # ,后面是应用于向量的 vector->list 的打印形式。 在 write 模式下,打印形式相同, 只是当 print-vector-length 参数为 #t 时, 在 # 后面打印一个十进制的整数,而重复的最后一个元素只打印一次。

除非 print-as-expression 被设置为 #t， 并且当前的 quoting depth 为 0 , 否则矢量的 print 形式与它们的 write 相同。 在这种情况下,如果向量的所有元素都是 quotable, 那么向量的 print 形式会以 ' 为前缀, 其元素以 quoting depth 1 打印。 如果它的元素不都是 quotable, 那么向量的 print 形式是 (vector , quoting depth 为 0 的元素,以及一个结束语 ) 。 当一个向量的所有元素都是 quotable 的时候,它就是 quotable。

在 write 或 display 模式下, flvector 的打印方式与 vector 相同, 但前缀为 #fl 而不是 # 。 同样, fxvector 的打印也是以 #fx 为前缀,而不是 #。 print-vector-length 参数对 flvector 和 fxvector 打印的影响与 vector 打印相同。 在 print 模式下, flvectors 和 fxvectors 是不 quotable ， 它们分别使用 (flvector  或 (fxvector  前缀, 像 vector 一样在 quoting depth 0 处打印。

1.4.8 打印结构

当 print-struct 参数被设置为 #t 时, structures 的打印方式取决于结构类型的细节,该结构是一个实例:

  • 如果结构类型是 prefab 结构类型,那么它在 write 或 display 模式下使用 #s( ,后面是 prefab 结构类型键, 然后是结构中每个字段的打印形式,再后面是 ) 进行打印。

    在 print 模式下, 当 print-as-expression 被设置为 #t 且当前 quoting depth 为 0 时,如果该结构的内容都是 quotable, 那么该结构的 print 形式将以 ' 为前缀, 其内容以 quoting depth 1 打印。 如果它的任何内容都是不可引用的,那么该结构类型的打印结果与非 prefab 结构类型相同。

    当一个 prefab 结构类型的实例的所有内容都是 quotable , 它就是 quotable。

  • 如果该结构有一个 prop:custom-write 的属性值, 那么相关的过程将被用来打印该结构,除非 print-unreadable 参数被设置为 #f。

    对于 print 模式, 如果结构类型的有 prop:custom-print-quotable 属性的值为 'always， 那么具有 prop:custom-write 属性的结构类型的实例被视为 quotable 。 如果它的属性值是 'maybe ,那么如果该结构的内容是 quotable , 那么它就被视为 quotable, 其中内容是由该结构的 prop:custom-write 程序递归打印的值决定的。 最后,如果该结构的属性值是 'self ,那么它将被视为 quotable。

    在 print 模式下,当 print-as-expression 为 #t 时, 结构的 prop:custom-write 过程会以 0 或 1 作为 quoting depth 被调用,通常取决于结构的 prop:custom-print-quotable 属性值。 如果该属性值为 'always, 则 quoting depth 通常为 1。 如果该属性值为 'maybe, 则如果该结构是 quotable, 则 quoting depth 为 1 ,否则通常为 0 。 如果属性值是 'self, 那么引用深度可能是 0 或 1; 如果该结构没有作为 quotable 值的一部分被打印出来,那么它通常是 0 , 即使该结构被视为 quotable。 最后,如果属性值是 'never ,那么 quoting depth 通常为 0 。 如果结构在打印时有一个明确的引号深度为 1 , 那么 quoting depth 可以不同于其正常值。

  • 如果该结构的类型是透明的, 或者任何祖先是透明的(即实例上的 struct? 产生 #t), 那么该结构在 display 模式、 write 模式或 print 模式中, 当 print-as-expression 被设置为 #f 或 quoting depth 为 0 时,打印为由 struct->vector 产生的向量。

    在 print 模式下, print-as-expression 为 #t ， quoting depth 为 0, 在 write 模式下,结构内容被打印为 ( , 后面是结构的类型名称(由 object-name) 决定); 其余元素被 print 为 quoting depth 为 0 , 并由空格分隔,最后是结束语 )。

    不是 prefab 结构类型的透明结构类型是 quotable。

  • 对于任何其他结构类型,该结构会打印成不可读的值; 更多信息请参见 打印不可读的值 。

如果 print-struct 参数设置为 #f, 那么所有没有 prop:custom-write 属性的结构都会打印为不可读的值 (参见 打印不可读的值) ,并算作 quotable。

1.4.9 打印哈希表

当 print-hash-table 参数设置为 #t 时, 在 write 和 display 模式下, 只要 hash table 强烈保留键值,就会分别以 #hash(, #hasheqv( 或 #hasheq( 表示使用 equal?, eqv? 或 eq? 键比较的表开始打印。 在前缀之后,每个键值映射显示为 (, 键的打印形式,一个空格 ., 一个空格,相应值的打印形式,以及 ), 如果键值对不是最后被打印的,则有一个额外的空格。 在所有键值对之后,打印的形式以 ).

在 print 模式下,当 print-as-expression 为 #f 或 quoting depth 为 1 时,打印的形式与 write 相同。 否则,如果哈希表的键和值都是 quotable, 则该表以 ' 前缀打印, 并且该表的键和值以 quoting depth 1 print。 如果某些键或值是 quotable, 则哈希表打印为 (hash , (hasheqv  或 (hasheq  后面是交替 print 的键和值, quoting depth 为 1 ,用空格隔开,最后是结束语 ) 。 当一个哈希表的所有键和值都是 quotable 的时候,它就是 quotable。

当 print-hash-table 参数被设置为 #f 或者哈希表弱保留其键值时, 哈希表会被打印为 #<hash> 并被算作 quotable。

1.4.10 打印 Boxes

当 print-box 参数设置为 #t 时, 当 print-as-expression 为 #f 或 quoting depth 为 1 时， 在 write, display 或 print 模式下, 一个 box 会以 #& 的形式打印,后面是其内容的打印形式。

在 print 模式下,当 print-as-expression 为 #t , 且 quoting depth 为 0 时,当一个盒子的内容可 quotable 时, 打印时带有 ' 前缀,其值在 quoting depth 1 处打印, 否则,盒子打印为 (box  后面是 quoting depth 为 0 的内容和一个结束 ) 当一个盒子的内容是 quotable ,它就是 quotable。

当 print-box 参数被设置为 #f 时, 一个盒子会被打印成 #<box> ,并算作 quotable。

1.4.11 打印字符

具有 阅读字符 中描述的特殊名称的 Characters 使用相同的名称 write 和 print 。 (有些字符有多个名字; #\newline 和 #\nul 的名字被用来代替 #\linefeed 和 #\null。) 其他图形字符(根据 char-graphic?) write 成 #\ 后面的单个字符,所有其他字符都用 #\u 符号写成四位数或 #\U 符号写成八位数(只有在字符值不适合用四位数时才使用后者)。

所有的字符都直接 display 为自己(即单个字符)。

对于打印包围数据类型的目的,一个字符是 quotable。

1.4.12 打印关键词

Keywords 的 write, print 和 display 与符号 (参见 打印符号) 相同,只是在前面加上 #: (在 print 模式中添加的任何前缀 ' 之后), 对于初始的 # 或当打印的形式与数字或限定的 . 相匹配时,没有特殊处理(因为 #: 区分了关键词)。

为了打印封闭的数据类型,关键字是 quotable。

1.4.13 打印正则表达式

Regexp values write, display 和 print 以 #px (对于基于 pregexp 的正则表达式)或 #rx (对于基于 regexp 的正则表达式)开始, 后面是正则表达式的源字符串或字节字符串的 write 形式。

为了打印封闭的数据类型,一个 regexp 的值是 quotable。

1.4.14 打印路径

Paths 以 #<path:....> 的形式 write 和 print 。 路径 display 与 path->string 产生的字符串相同。 在打印封闭的数据类型时,路径算作是 quotable。

虽然路径可以用 path->string 转换为字符串, 也可以用 path->bytes 转换为字节字符串, 但对于打印路径和读回路径来说,两者显然都不是正确的选择。 如果路径值要在不同的平台之间移动,那么字符串可能是正确的选择, 尽管在将路径转换为字符串时可能会丢失信息。 对于要在同一平台上重新读取的路径,字节串可能是正确的选择, 因为它以一种不可移植的方式保存了信息。 路径不以可读的方式打印,这样程序员就不会被误导,以为这两种选择都是合适的。

1.4.15 打印不可读的值

对于任何没有其他打印规范的值, 假设 print-unreadable 参数被设置为 #t, 则输出形式为 #<‹something›>, 其中 ‹something› 是针对值的类型的,有时是针对值本身。 如果 print-unreadable 被设置为 #f, 那么试图打印一个不可读的值会引发 exn:fail。

对于打印封闭数据类型的目的来说,一个打印出来的不可读的值还是算作 quotable。

1.4.16 打印编译后的代码

编译后的代码由使用 #~ 的 compile 打印产生。 用 #~ 打印的编译代码本质上是 Racket 的汇编代码, 当 read-accept-compiled 参数设置为 #t 时, 读取这样的表格会产生一个编译的表格。

如果当前的代码检查器(参见 current-code-inspector)不是原始的代码检查器, 那么从 #~ 解析出来的编译代码将被标记为不可运行; 在试图评估或重新优化不可运行的字节码时,会引发 exn:fail exception is raised 异常。 否则,从 #~ 中解析出来的编译代码可能包含对模块中未输出或受保护的绑定的引用。 从概念上讲,字节码中的引用与当前的代码检查器相关联,只有当该检查器控制相关模块的调用时, 代码才会执行(参见 Code Inspectors)—但无论如何, 原始代码检查器控制所有其他检查器。

一个编译形式的对象可能包含由 gensym 或 string->uninterned-symbol 创建的 uninterned 符号(参见 符号) 。 当通过 #~ 读取编译后的对象时, 原始形式中的每个未terned符号被映射到一个新的未 terned 符号, 其中单个符号的多个实例被一致地映射到同一个新符号。 原始符号和新符号具有相同的打印表示。 Unreadable symbols ,通常是在扩展和编译过程中间接产生的, 通过 #~ 保存和恢复一致。

当 gensym 或 string->uninterned-symbol 被用来构建顶层或模块绑定的标识符时, uninterned 符号的动态性质及其在 #~ 内的本地化可能会导致问题(取决于标识符及其引用是如何编译的)。 为了避免问题,用 generate-temporaries 或者将 make-syntax-introducer 的结果应用于现有的标识符来生成不同的标识符; 这些函数导致顶层和模块变量具有 unreadable symbol 名称, 并且只要扩展是确定的,这些名称就是确定的。

当一个编译形式的对象有字符串和字节字符串字面时,当编译对象 interned 时, 它们被使用 datum-intern-literal 进行内部处理。 然而,数字和其他 read-syntax 会内化的值, 在编译后的对象中作为引号字头读回时不会内化。

编译后的表格可能包含路径字词。 尽管路径通常不是以可以回读的方式打印出来的, 但路径字词可以作为编译代码的一部分被写入和读取。 current-write-relative-directory 参数用于在写入时将路径转换成相对路径, 然后 current-load-relative-directory 参数(回落到 current-directory) 用于在读取时将任何相对路径转换回来。

对于语法对象源中的路径, 如果没有设置 current-write-relative-directory 参数, 或者该路径不是相对于 current-write-relative-directory 参数的值, 那么该路径将被强制为一个只保留部分路径的字符串 (努力使其与构建时文件系统的联系减少,这可能不同于运行时的文件系统)。

最后,如果结构的源字段是某个系统的路径、 一个字符串、一个字节串、一个符号或 #f, 则编译后的表单可能包含 srcloc 结构。 对于一个路径值(与当前平台的惯例相匹配), 如果不能根据 current-write-relative-directory 将其记录为相对路径, 那么它将被转换为一个最多包含两个路径元素的字符串; 如果路径包含两个以上的元素, 那么这个字符串包含 .../、次要的元素、 / 和最后的元素。 对 srcloc 值的约束和源字段的转换的意图是保留一些源信息, 但不暴露或记录一个在不同文件系统或平台上没有意义的路径。

了在 Racket 的 BC 实现中进行内部测试, 当 PLT_VALIDATE_LOAD 环境变量被设置时, 阅读器会对从 #~ 解析的字节码运行一个验证器。 验证器可能会捕捉到错误的编译或字节码文件损坏。 验证器可以懒洋洋地运行,比如只在过程被调用时才检查该过程。

Changed in version 6.90.0.21 of package base: 调整了改变代码检查器对解析字节码的影响, 使阅读器将加载的代码标记为一般不可运行, 而不是在读取时拒绝对不安全操作的引用。
Changed in version 7.0: 允许在编译的代码中嵌入一些 srcloc 值。
