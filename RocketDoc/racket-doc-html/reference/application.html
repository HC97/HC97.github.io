3.7 程序应用 和 #%app

函数调用 in Racket 指南 introduces procedure applications.

syntax

(proc-expr arg ...)

应用一个程序,当 proc-expr 不是一个具有转化器绑定的标识符时 (参见 扩展 )。

更确切地说,扩展器将这种形式转换为 (#%app proc-expr arg ...), 给 #%app 提供与原始形式相关的词法上下文 (即结合 proc-expr 及其参数的一对)。 通常情况下,该对的词法上下文表示接下来要描述的过程-应用 #%app 。 另见 扩展步骤。

Examples:
> (+ 1 2)

3

> ((lambda (x #:arg y) (list y x)) #:arg 2 1)

'(2 1)

syntax

(#%app proc-expr arg ...)

应用一个程序。每个 arg 是以下之一:

arg-expr

结果值是一个非关键字的参数。

keyword arg-expr

结果值是一个使用 keyword 的关键字参数。 应用程序中的每个 keyword 必须是不同的。

proc-expr 和 arg-expr 是按顺序评估的,从左到右。 如果 proc-expr 的结果是一个接受与非 keyword arg-expr 一样多的参数的过程,如果它接受应用程序中所有 keyword 的参数,并且如果所有需要的基于关键字的参数在应用程序中的 keyword 中得到体现,那么该过程将以 arg-expr 的值调用。 否则,就会引发 exn:fail:contract exception is raised 异常。

过程调用的延续性与应用表达式的延续性相同,所以过程的结果就是应用表达式的结果。

基于 keyword 的参数的相对顺序只对 arg-expr 的评估顺序有影响; 参数是根据 keyword 与应用程序中的参数变量相关联的,而不是它们的位置。 与此相反,其他 arg-expr 值是根据它们在应用程序中的顺序与变量相关联的。

关于扩展器如何引入 #%app 标识符的信息,请参见 扩展步骤 。

Examples:
> (#%app + 1 2)

3

> (#%app (lambda (x #:arg y) (list y x)) #:arg 2 1)

'(2 1)

> (#%app cons)

cons: arity mismatch;

 the expected number of arguments does not match the given

number

  expected: 2

  given: 0

syntax

(#%plain-app proc-expr arg-expr ...)

(#%plain-app)
与 #%app 类似,但不支持关键字参数。 作为一种特殊情况, (#%plain-app) 产生 '()。
