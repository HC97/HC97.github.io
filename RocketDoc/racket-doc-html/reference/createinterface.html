6.1 创建接口

类和对象 in Racket 指南 introduces classes, objects, and interfaces.

syntax

(interface (super-interface-expr ...) name-clause ...)

 
name-clause = id
  | (id contract-expr)
生成一个接口。 这些 id 必须是相互独立的。

每一个 super-interface-expr 在 interface 表达式被评估时(按顺序)被评估。 每个 super-interface-expr 的结果必须是一个接口值,否则会引发 exn:fail:object exception is raised 。 由 super-interface-expr 返回的接口是新接口的超接口,它们都由新接口扩展。 任何实现了新接口的类也实现了所有的超接口。

interface 表达式的结果是一个包含所有指定 id 和所有超级接口标识符的接口。 超级接口中重复的标识符名称将被忽略, 但如果一个超级接口包含 interface 表达式中的一个 id , 则会引发 exn:fail:object exception is raised 。 给定的 id 可以与相应的 contract-expr 配对。

如果没有提供 super-interface-expr ,那么结果接口的派生需求是微不足道的: 实现该接口的任何类都必须从 object% 派生。 否则,结果接口的实现需求是其超接口中最特定的需求。 如果超级接口指定不一致的派生要求,则引发 exn:fail:object exception is raised 。

Examples:
(define file-interface<%>
  (interface () open close read-byte write-byte))
(define directory-interface<%>
  (interface (file-interface<%>)
    [file-list (->m (listof (is-a?/c file-interface<%>)))]
    parent-directory))

syntax

(interface* (super-interface-expr ...)
            ([property-expr val-expr] ...)
  name-clause ...)
 
name-clause = id
  | (id contract-expr)
像 interface 类似, 但也将 property-expr 生成的结构类型属性与相应的 val-expr 关联到接口。

当生成的接口(或从接口派生的子接口)由类通过 class* 表单显式实现时, 每个属性都将其值附加到由类实例实例化的结构类型。 具体来说,该属性附加到一个直接字段为零的结构类型, 该结构类型被扩展为类实例的内部结构类型(因此,如果有的话,结构类型属性的守卫无法访问任何关于字段的信息)。

Example:
(define i<%> (interface* () ([prop:custom-write
                              (lambda (obj port mode) (void))])
               method1 method2 method3))
