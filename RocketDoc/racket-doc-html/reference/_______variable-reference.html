3.6 位置 : #%variable-reference

syntax

(#%variable-reference id)

(#%variable-reference (#%top . id))
(#%variable-reference)
产生一个不透明的 variable reference , 代表 id 的 location ,它必须被绑定为一个变量。 如果没有提供 id ,产生的值指的是在包围的上下文中定义的 "匿名" 变量 (即在包围的模块中,如果表单不在一个模块中,则在顶层)。

当 (#%top . id) 被使用时,那么这个变量引用指的是与 (#%top . id) 相同的变量。 需要注意的是,如果 id 是本地绑定的,或者在一个模块内, 如果 id 是作为一个转化器绑定的, (#%top . id) 是不允许的。

variable reference 可以与 variable-reference->empty-namespace, variable-reference->resolved-module-path 可以与 variable-reference->namespace 一起使用,但是像 define-namespace-anchor 和 namespace-anchor->namespace 这样的设施将其包裹起来,以提供一个更清晰的接口。 variable reference 对低级别的扩展也很有用, 参见 Inside: Racket C API。

Changed in version 8.2.0.7 of package base: 改变了 #%top 的处理方式, 使之与 #%top 本身一致。
