4.7 符号

符号（Symbol） in Racket 指南 introduces symbols.

symbol 就像一个不可变的字符串,但是符号通常是 interned, 所以两个字符内容相同的符号通常是 eq? 。 所有由默认阅读器(参见 阅读符号) 生的符号都是 interned 。

两个程序 string->uninterned-symbol 和 gensym 生成 uninterned 符号,即与任何其他符号都不 eq?, eqv? 或 equal? 的符号,尽管它们可能与其他符号的打印结果相同。

过程 string->unreadable-symbol 返回一个 unreadable symbol ,它是部分内含的。 默认的阅读器(参见 阅读符号) 从不产生一个不可读的符号, 但是两次调用 string->unreadable-symbol 的 equal? 字符串会产生 eq? 结果。 一个不可读的符号可以和一个内含的或未内含的符号一样打印。 不可读符号在扩展和编译中很有用,可以避免与源码中出现的符号发生碰撞; 它们通常不会直接生成,但它们可以出现在像 identifier-binding 这样的函数结果中。

内部的和不可读的符号只被内部符号表弱化地持有。 这个弱点永远不会影响 eq?, eqv? 或 equal? 测试的结果, 但是当一个符号被放入一个弱盒(参见 弱盒), 被用作弱 hash table 的键(参见 Hash Tables), 或被用作 ephemeron 键(参见 外显子) 时,它可能会消失。

See 阅读符号 for information on reading symbols and 打印符号 for information on printing symbols.

procedure

(symbol? v) → boolean?

  v : any/c
如果 v 是一个符号,返回 #t ,否则返回 #f 。

Examples:
> (symbol? 'Apple)

#t

> (symbol? 10)

#f

procedure

(symbol-interned? sym) → boolean?

  sym : symbol?
如果 sym 是 interned , 则返回 #t ,否则返回 #f 。

Examples:
> (symbol-interned? 'Apple)

#t

> (symbol-interned? (gensym))

#f

> (symbol-interned? (string->unreadable-symbol "Apple"))

#f

procedure

(symbol-unreadable? sym) → boolean?

  sym : symbol?
如果 sym 是一个 unreadable symbol , 返回 #t ,否则返回 #f 。

Examples:
> (symbol-unreadable? 'Apple)

#f

> (symbol-unreadable? (gensym))

#f

> (symbol-unreadable? (string->unreadable-symbol "Apple"))

#t

procedure

(symbol->string sym) → string?

  sym : symbol?
返回一个新分配的可变字符串,其字符与 sym 中的相同。

也请参见 racket/symbol 中的 symbol->immutable-string 。

Example:
> (symbol->string 'Apple)

"Apple"

procedure

(string->symbol str) → symbol?

  str : string?
返回一个 interned 的符号, 其字符与 str 中的相同。

Examples:
> (string->symbol "Apple")

'Apple

> (string->symbol "1")

'|1|

procedure

(string->uninterned-symbol str) → symbol?

  str : string?
与 (string->symbol str) 类似,但产生的符号是一个新的 uninterned 符号。 用同一个 str 调用 string->uninterned-symbol 两次会返回两个不同的符号。

Examples:
> (string->uninterned-symbol "Apple")

'Apple

> (eq? 'a (string->uninterned-symbol "a"))

#f

> (eq? (string->uninterned-symbol "a")
       (string->uninterned-symbol "a"))

#f

procedure

(string->unreadable-symbol str) → symbol?

  str : string?
与 (string->symbol str) 类似,但产生的符号是一个新的 unreadable symbol 。 用相等的 str 调用 string->unreadable-symbol 两次,返回相同的符号, 但是 read 永远不会产生符号。

Examples:
> (string->unreadable-symbol "Apple")

'Apple

> (eq? 'a (string->unreadable-symbol "a"))

#f

> (eq? (string->unreadable-symbol "a")
       (string->unreadable-symbol "a"))

#t

procedure

(gensym [base]) → symbol?

  base : (or/c string? symbol?) = "g"
返回一个新的 uninterned 符号, 并有一个自动生成的名称。 可选的 base 参数是一个前缀符号或字符串。

Example:
> (gensym "apple")

'apple7280988

procedure

(symbol<? a-sym b-sym ...) → boolean?

  a-sym : symbol?
  b-sym : symbol?
如果参数被排序,则返回 #t ,其中每对符号的比较与使用 symbol->string 与 string->bytes/utf-8 和 bytes<? 。

Changed in version 7.0.0.13 of package base: Allow one argument, in addition to allowing two or more.

4.7.1 额外的符号函数

 (require racket/symbol) package: base
The bindings documented in this section are provided by the racket/symbol library, not racket/base or racket.

Added in version 7.6 of package base.

procedure

(symbol->immutable-string sym) → (and/c string? immutable?)

  sym : symbol?
像 symbol->string, 但结果是一个不可变的字符串,不一定是新分配的。

Examples:
> (symbol->immutable-string 'Apple)

"Apple"

> (immutable? (symbol->immutable-string 'Apple))

#t

Added in version 7.6 of package base.
