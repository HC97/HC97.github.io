3.18 迭代和理解: for, for/list, ...

迭代和推导 in Racket 指南 introduces iterations and comprehensions.

for 的迭代形式是基于 SRFI-42 [SRFI-42]。

3.18.1 迭代和理解的形式

syntax

(for (for-clause ...) body-or-break ... body)

 
for-clause = [id seq-expr]
  | [(id ...) seq-expr]
  | #:when guard-expr
  | #:unless guard-expr
  | #:do [do-body ...]
  | break-clause
  | #:splice (splicing-id . form)
     
break-clause = #:break guard-expr
  | #:final guard-expr
     
body-or-break = body
  | break-clause
 
  seq-expr : sequence?
迭代地评估 body 。 for-clause 引入绑定,其范围包括 body , 并决定 body 被评估的次数。 无论是 for-clause 还是 body 中的 break-clause 都会停止进一步的迭代。

在简单的情况下,每个 for-clause 都有其前两种形式之一, 其中 [id seq-expr] 是 [(id) seq-expr] 的简写。 在这种简单的情况下, seq-expr 从左到右被评估, 每个都必须产生一个序列值(参见 Sequences)。

for 形式通过从每个序列中抽取一个元素进行迭代; 如果任何一个序列是空的,那么迭代停止, #<void> 是 for 表达式的结果。 否则就为每个 id 创建一个位置来保存每个元素的值; 由 seq-expr 产生的序列必须为每个迭代返回与对应 id 相同数量的值。

然后 id 被绑定在 body 中, body 被评估,其结果被忽略。 迭代继续进行,每个序列中的下一个元素和每个 id 的新位置。

一个有零个 for-clause 的 for 形式等同于一个单一的 for-clause , 它将一个未引用的 id 绑定到一个包含单一元素的序列上。 根据 bound-identifier=? ,所有的 id 必须是不同的。

如果任何 for-clause 的形式是 #:when guard-expr, 那么只有前面的子句(不包含 #:when, #:unless 或 #:do) 才能像上面那样确定迭代,并且 body 被有效地包装成

(when guard-expr
  (for (for-clause ...) body ...+))

使用剩余的 for-clause 。 形式为 #:unless guard-expr 的 for-clause 对应于相同的转换, 用 unless 来代替 when 。 形式为 #:do [do-body ...] 的 for-clause 类似于创建嵌套,对应于

(let ()
  do-body ...
  (for (for-clause ...) body ...+))

其中 do-body 形式可以引入在其余 for-clause 中可见的定义。

一个 #:break guard-expr 子句类似于 #:unless guard-expr 子句, 但是当 #:break 避免了对 body 的计算时, 它也有效地结束了 for 形式中的所有序列。 一个 #:final guard-expr 子句类似于 #:break guard-expr, 但是它不是立即结束序列并跳过 body , 而是允许从每个后面的序列中最多增加一个元素, 并最多对下面的 body 进行一次评估。 在 body 中,除了停止迭代和阻止后面的 body 评估外, #:break guard-expr 或 #:final guard-expr 子句开始一个新的内部定义环境。

一个 #:splice (splicing-id . form) 子句被通过扩展 (splicing-id . form) 产生的一系列形式所取代, 其中 splicing-id 是使用 define-splicing-for-clause-syntax 绑定的。 该扩展的绑定上下文包括来自 #:splice 形式和 #:when, #:unless, #:do, #:break 或 #:final 形式前面的任何子句的绑定。 一个 #:splice 扩展的结果可以包括更多的 #:splice 形式来进一步交错子句绑定和扩展。 对 #:splice 子句的支持不是为了直接在 source for 表单中使用, 而是为了构建扩展为 for 的新表单。

在 list 和 stream 序列的情况下, for 形式本身并不保持每个元素的可及性。 如果一个由 seq-expr 产生的列表或流在其他方面无法到达, 并且如果 for 体不能再引用一个列表元素的 id , 那么该元素将被 garbage collection 。 make-do-sequence 序列构造函数支持额外的序列, 这些序列以这种方式表现得像列表和流。

如果一个 seq-expr 是一个带引号的字面列表、向量、精确的整数、 字符串、字节字符串、不可变的哈希,或者扩展到这样的字面, 那么它可以被视为使用了一个序列转换器,比如 in-list , 除非 seq-expr 的 'for:no-implicit-optimization 语法属性为真值;在大多数情况下,这可以提高性能。

Examples:
> (for ([i '(1 2 3)]
        [j "abc"]
        #:when (odd? i)
        [k #(#t #f)])
    (display (list i j k)))

(1 a #t)(1 a #f)(3 c #t)(3 c #f)

> (for ([i '(1 2 3)]
        #:do [(define neg-i (* i -1))]
        [j (list neg-i 0 i)])
    (display (list j)))

(-1)(0)(1)(-2)(0)(2)(-3)(0)(3)

> (for ([(i j) #hash(("a" . 1) ("b" . 20))])
    (display (list i j)))

(a 1)(b 20)

> (for ([i '(1 2 3)]
        [j "abc"]
        #:break (not (odd? i))
        [k #(#t #f)])
    (display (list i j k)))

(1 a #t)(1 a #f)

> (for ([i '(1 2 3)]
        [j "abc"]
        #:final (not (odd? i))
        [k #(#t #f)])
    (display (list i j k)))

(1 a #t)(1 a #f)(2 b #t)

> (for ([i '(1 2 3)]
        [j "abc"]
        [k #(#t #f)])
    #:break (not (or (odd? i) k))
    (display (list i j k)))

(1 a #t)

> (for ()
    (display "here"))

here

> (for ([i '()])
    (error "doesn't get here"))

Changed in version 6.7.0.4 of package base: 增加了对可选第二个结果的支持。
Changed in version 7.8.0.11: 增加了对隐式优化的支持。
Changed in version 8.4.0.2: 增加 #:do。
Changed in version 8.4.0.3: 增加 #:splice。

syntax

(for/list (for-clause ...) body-or-break ... body)

像 for 一样进行迭代,但 body 中的最后一个表达式必须产生一个单一的值, for/list 表达式的结果是一个按顺序排列的结果列表。 当由于 #:when 或 #:unless 子句而跳过一个 body 的评估时, 结果列表不包括相应的元素。

Examples:
> (for/list ([i '(1 2 3)]
             [j "abc"]
             #:when (odd? i)
             [k #(#t #f)])
    (list i j k))

'((1 #\a #t) (1 #\a #f) (3 #\c #t) (3 #\c #f))

> (for/list ([i '(1 2 3)]
             [j "abc"]
             #:break (not (odd? i))
             [k #(#t #f)])
    (list i j k))

'((1 #\a #t) (1 #\a #f))

> (for/list () 'any)

'(any)

> (for/list ([i '()])
    (error "doesn't get here"))

'()

syntax

(for/vector maybe-length (for-clause ...) body-or-break ... body)

 
maybe-length = 
  | #:length length-expr
  | #:length length-expr #:fill fill-expr
 
  length-expr : exact-nonnegative-integer?
像 for/list 一样进行迭代,但结果被累积成一个向量而不是一个列表。

如果指定了可选的 #:length 子句, length-expr 的结果将决定结果向量的长度。 在这种情况下,可以更有效地进行迭代, 当向量满了或者已经进行了所要求的迭代次数时(以先到者为准),迭代就会终止。 如果 length-expr 指定的长度长于迭代次数, 那么向量的剩余槽将被初始化为 fill-expr 的值, 该值默认为 0 (即 make-vector 的默认参数)。

Examples:
> (for/vector ([i '(1 2 3)]) (number->string i))

'#("1" "2" "3")

> (for/vector #:length 2 ([i '(1 2 3)]) (number->string i))

'#("1" "2")

> (for/vector #:length 4 ([i '(1 2 3)]) (number->string i))

'#("1" "2" "3" 0)

> (for/vector #:length 4 #:fill "?" ([i '(1 2 3)]) (number->string i))

'#("1" "2" "3" "?")

for/vector 形式可以分配一个向量, 并在 body 的每次迭代后对其进行变异, 这意味着在 body 期间捕获一个延续并多次应用它可能会变异一个共享向量。

syntax

(for/hash (for-clause ...) body-or-break ... body)

syntax

(for/hasheq (for-clause ...) body-or-break ... body)

syntax

(for/hasheqv (for-clause ...) body-or-break ... body)

syntax

(for/hashalw (for-clause ...) body-or-break ... body)

像 for/list 一样,但结果是一个不可变的 hash table; for/hash 用 equal? 来区分键创建一个表, for/hasheq 用 eq? 产生一个表, for/hasheqv 用 eqv? 产生一个表, for/hashalw 用 equal-always? 产生一个表。 body 中的最后一个表达式必须返回两个值: 一个键和一个值来扩展迭代所积累的哈希表。

Example:
> (for/hash ([i '(1 2 3)])
    (values i (number->string i)))

'#hash((1 . "1") (2 . "2") (3 . "3"))

Changed in version 8.5.0.3 of package base: 增加了 for/hashalw 形式。

syntax

(for/and (for-clause ...) body-or-break ... body)

像 for 一样进行迭代, 但是当 body 的最后一个表达式产生 #f 时,迭代终止, for/and 表达式的结果是 #f 。 如果 body 从未被评估过, 那么 for/and 表达式的结果是 #t 。 否则,结果是 body 最后一次评估的(单个)结果。

Examples:
> (for/and ([i '(1 2 3 "x")])
    (i . < . 3))

#f

> (for/and ([i '(1 2 3 4)])
    i)

4

> (for/and ([i '(1 2 3 4)])
    #:break (= i 3)
    i)

2

> (for/and ([i '()])
    (error "doesn't get here"))

#t

syntax

(for/or (for-clause ...) body-or-break ... body)

像 for 一样进行迭代, 但是当 body 的最后一个表达式产生一个 #f 以外的值时, 那么迭代终止, for/or 表达式的结果是相同的(单个)值。 如果 body 从未被评估过, 那么 for/or 达式的结果就是 #f 。否则,结果是 #f。

Examples:
> (for/or ([i '(1 2 3 "x")])
    (i . < . 3))

#t

> (for/or ([i '(1 2 3 4)])
    i)

1

> (for/or ([i '()])
    (error "doesn't get here"))

#f

syntax

(for/sum (for-clause ...) body-or-break ... body)

像 for 一样进行迭代, 但最后一个 body 的每个结果都被累积成一个带有 + 的结果。

Example:
> (for/sum ([i '(1 2 3 4)]) i)

10

syntax

(for/product (for-clause ...) body-or-break ... body)

像 for 一样进行迭代, 但最后一个 body 的每个结果都被累积成一个带 * 的结果。

Example:
> (for/product ([i '(1 2 3 4)]) i)

24

syntax

(for/lists (id ... maybe-result)
           (for-clause ...)
  body-or-break ... body)
 
maybe-result = 
  | #:result result-expr
与 for/list 类似, 但是最后一个 body 表达式应该产生与给定 id 一样多的值。 这些 id 被绑定到迄今为止在 for-clause 和 body 中积累的反转列表上。

如果提供了一个 result-expr ,当迭代结束时,它将被用于 for/fold; 否则,结果是所提供的 id 一样多的列表。

id 绑定的范围与 for/fold 中累加器标识符的范围相同。 突变一个 id 会影响累积的列表,以产生非列表的方式突变它, 会导致每个 id 的最终 reverse 失败。

Examples:
> (for/lists (l1 l2 l3)
             ([i '(1 2 3)]
              [j "abc"]
              #:when (odd? i)
              [k #(#t #f)])
    (values i j k))

'(1 1 3 3)

'(#\a #\a #\c #\c)

'(#t #f #t #f)

> (for/lists (acc)
             ([x '(tvp tofu seitan tvp tofu)]
              #:unless (member x acc))
    x)

'(tvp tofu seitan)

> (for/lists (firsts seconds #:result (list firsts seconds))
             ([pr '((1 . 2) (3 . 4) (5 . 6))])
    (values (car pr) (cdr pr)))

'((1 3 5) (2 4 6))

Changed in version 7.1.0.2 of package base: 增加了 #:result 形式。

syntax

(for/first (for-clause ...) body-or-break ... body)

像 for 一样进行迭代,但是在 body 第一次被评估后, 迭代就终止了, for/first 的结果是 body 的(单一)结果。 如果 body 从未被评估过,那么 for/first 表达式的结果就是 #f。

Examples:
> (for/first ([i '(1 2 3 "x")]
              #:when (even? i))
     (number->string i))

"2"

> (for/first ([i '()])
    (error "doesn't get here"))

#f

syntax

(for/last (for-clause ...) body-or-break ... body)

像 for 一样进行迭代, 但是 for/last 的结果是 body 最后一次评估的(单个)结果。 如果 body 从未被评估过, 那么 for/last 表达式的结果就是 #f。

Examples:
> (for/last ([i '(1 2 3 4 5)]
              #:when (even? i))
     (number->string i))

"4"

> (for/last ([i '()])
    (error "doesn't get here"))

#f

syntax

(for/fold ([accum-id init-expr] ... maybe-result) (for-clause ...)
  body-or-break ... body)
 
maybe-result = 
  | #:result result-expr
像 for 一样进行迭代。 在迭代开始前, init-expr 被评估以产生初始累加器值。 在每次迭代开始时,为每个 accum-id 生成一个位置, 并将相应的当前累加器值放入该位置。 body 中的最后一个表达式必须产生与 accum-id 同样多的值, 这些值成为当前的累加器值。 当迭代结束时,如果提供了一个 result-expr , 那么 for/fold 的结果就是评估 result-expr 的结果(范围内有 accum-id 并绑定到它们的最终值), 否则 for/fold 表达式的结果就是累积器的值。

Examples:
> (for/fold ([sum 0]
             [rev-roots null])
            ([i '(1 2 3 4)])
    (values (+ sum i) (cons (sqrt i) rev-roots)))

10

'(2 1.7320508075688772 1.4142135623730951 1)

> (for/fold ([acc '()]
             [seen (hash)]
             #:result (reverse acc))
            ([x (in-list '(0 1 1 2 3 4 4 4))])
    (cond
      [(hash-ref seen x #f)
       (values acc seen)]
      [else (values (cons x acc)
                    (hash-set seen x #t))]))

'(0 1 2 3 4)

accum-id 和 init-expr 绑定和评估顺序并不完全遵循文本上的, 相对于 for-clause 的从左到右顺序。 相反, for-clause 中决定最外层迭代的序列表达式首先被评估, 然后 init-expr 被评估, accum-id 被绑定,最后最外层迭代的标识符被绑定。 一个后果是,在最外层初始化的 for-clause 中, accum-id 没有被绑定。 同时,当一个 accum-id 被用作最外层迭代的 for-clause 绑定时, for-clause 绑定会在循环体中影射 accum-id 绑定(这是你在语法上期望的)。 在每个嵌套的迭代中,每个 accum-id 都有一个新的变量(在一个新的位置), 这些迭代是由后面的 for-clause 组创建的(例如,在 #:when 或 #:unless 之后)。

Changed in version 6.11.0.1 of package base: 增加 #:result 形式。

syntax

(for/foldr ([accum-id init-expr] ... accum-option ...)
           (for-clause ...)
  body-or-break ... body)
 
accum-option = #:result result-expr
  | #:delay
  | #:delay-as delayed-id
  | #:delay-with delayer-id
类似于 for/fold ,但类似于 foldr 而不是 foldl: 给定的序列仍然以相同的顺序迭代,但循环体是以相反的顺序进行评估。 for/foldr 表达式的评估使用的空间与它执行的迭代次数成正比, 并且由给定序列产生的所有元素都被保留, 直到循环体的反向评估开始(假设该元素事实上是在循环体中被引用)。

Examples:
> (define (in-printing seq)
    (sequence-map (lambda (v) (println v) v) seq))
> (for/foldr ([acc '()])
             ([v (in-printing (in-range 1 4))])
    (println v)
    (cons v acc))

1

2

3

3

2

1

'(1 2 3)

此外,与 for/fold 不同的是, accum-id 不被绑定在出现在 break-clause 之前的 guard-expr 或 body-or-break 形式中。

虽然上述限制使得 for/foldr 在一般情况下不如 for/fold 有用, 但是 for/foldr 提供了额外的能力, 可以通过 #:delay、 #:delay-as 和 #:delay-with 选项进行懒散的迭代, 这可以减轻 for/foldr 的许多缺点。 如果至少指定了一个这样的选项,循环体就可以明确地控制迭代的时间: 默认情况下,每个 accum-id 都被绑定到一个 promise 上, 当强制执行时,产生 accum-id 的当前值。

在这种模式下,迭代不会继续,直到一个这样的承诺被强制执行,这将触发任何必要的额外迭代以产生一个值。 如果循环体在其 accum-id 中是懒惰的–也就是说, 它返回一个值而不强制任何一个–那么循环(或其任何一个迭代)将在迭代完全结束之前产生一个值。 如果对至少一个这样的承诺的引用被保留下来, 那么强制它将从暂停的地方恢复迭代,即使控制已经离开了循环体的动态范围。

Examples:
> (for/foldr ([acc '()] #:delay)
             ([v (in-range 1 4)])
    (printf "--> ~v\n" v)
    (begin0
      (cons v (force acc))
      (printf "<-- ~v\n" v)))

--> 1

--> 2

--> 3

<-- 3

<-- 2

<-- 1

'(1 2 3)

> (define resume
    (for/foldr ([acc '()] #:delay)
               ([v (in-range 1 5)])
      (printf "--> ~v\n" v)
      (begin0
        (cond
          [(= v 1) (force acc)]
          [(= v 2) acc]
          [else    (cons v (force acc))])
        (printf "<-- ~v\n" v))))

--> 1

--> 2

<-- 2

<-- 1

> (force resume)

--> 3

--> 4

<-- 4

<-- 3

'(3 4)

这种对迭代顺序的额外控制使得 for/foldr 既可以消耗也可以构建无限序列, 只要它至少有时在累加器中是懒惰的。

也请参见 for/stream ,它是一种更方便(尽管不那么灵活)的懒惰地转换无限序列的方法。 (在内部, for/stream 是根据 for/foldr来定义的。)

Examples:
> (define squares (for/foldr ([s empty-stream] #:delay)
                             ([n (in-naturals)])
                    (stream-cons (* n n) (force s))))
> (stream->list (stream-take squares 10))

'(0 1 4 9 16 25 36 49 64 81)

由 #:delay 选项引入的暂停通常不影响循环的最终返回值, 但是如果 #:delay 和 #:result 相结合, accum-id 将在 result-expr 的范围内被延迟, 就像它们在循环主体中被延迟一样。 如果需要的话,这可以用来在整个循环的评估过程中引入一个额外的暂停层。

Examples:
> (define evaluated-yet? #f)
> (for/foldr ([acc (set! evaluated-yet? #t)] #:delay) ()
    (force acc))
> evaluated-yet?

#t

> (define evaluated-yet? #f)
> (define start
    (for/foldr ([acc (set! evaluated-yet? #t)] #:delay #:result acc) ()
      (force acc)))
> evaluated-yet?

#f

> (force start)
> evaluated-yet?

#t

如果提供了 #:delay-as 选项, 那么 delayed-id 将被绑定到一个额外的承诺上, 该承诺将一次性返回所有 accum-id 的值。 当提供了多个 accum-id 时,强制执行这个承诺会比强制执行与 accum-id 单独绑定的承诺更有效率。

如果提供了 #:delay-with 选项, 给定的 delayer-id 将被用于暂停嵌套迭代(而不是默认的 delay)。 一个形状 (delayer-id recur-expr) 被构造出来并放置在表达式位置, 其中 recur-expr 是一个表达式,当被评估时,将执行下一次迭代并返回其结果(或结果)。 delayer-id 的合理选择包括 lazy, delay/sync, delay/thread, 或任何其他来自 racket/promise 的承诺构造器, 以及来自 racket/function 的 thunk 。 然而,要注意像 thunk 或 delay/name 这样的选择可能会多次评估它们的子表达式, 这可能会导致有状态的序列出现无意义的结果,因为状态将在 recur-expr 的所有评估中共享。

如果给出了多个 accum-id ,提供了 #:delay-with 选项, 并且 delayer-id 没有绑定到 delay, lazy, delay/strict, delay/sync, delay/thread 或 delay/idle 中,那么 accum-id 根本不会被绑定, 即使是在循环体中。 相反,必须指定 #:delay-as 选项, 通过 delayed-id 访问累加器的值。

Added in version 7.3.0.3 of package base.

syntax

(define-sequence-syntax id
  expr-transform-expr
  clause-transform-expr)
 
  expr-transform-expr : 
(or/c (-> identifier?)
      (syntax? . -> . syntax?))
  clause-transform-expr : (syntax? . -> . syntax?)
将 id 定义为语法。 当用于在 for-clause 的 for (或其变体之一)中生成一个序列时, (id . rest) 形式被特别处理。 在这种情况下,会调用 clause-transform-expr 的过程结果来转换该子句。

当 id 被用在任何其他表达式的位置时, 将使用 expr-transform-expr 的结果。 如果它是一个零参数的过程,那么结果必须是一个标识符 other-id , 并且任何对 id 的使用都被转换为对 other-id 的使用。 否则, expr-transform-expr 必须产生一个(有一个参数的)过程, 作为宏转换器使用。

当使用 clause-transform-expr 变换器时, 它被赋予一个 for-clause 作为参数, 其中 clause 的形式被规范化了,所以左边是一个标识符的括号序列。 右手边的形式是 (id . rest) 。 结果可以是 #f, 表示这些形式不应该被特别处理 (也许是因为绑定的标识符的数量与 (id . rest) 的形式不一致), 或者是一个新的 for-clause 来替换给定的那个。 新的子句可能使用 :do-in 。 为了保护 clause-transform-expr 结果中的标识符, 使用 for-clause-syntax-protect 而不是 syntax-protect。

Examples:
> (define (check-nat n)
    (unless (exact-nonnegative-integer? n)
      (raise-argument-error 'in-digits "exact-nonnegative-integer?" n)))
> (define-sequence-syntax in-digits
    (lambda () #'in-digits/proc)
    (lambda (stx)
      (syntax-case stx ()
        [[(d) (_ nat)]
         #'[(d)
            (:do-in
              ([(n) nat])
              (check-nat n)
              ([i n])
              (not (zero? i))
              ([(j d) (quotient/remainder i 10)])
              #t
              #t
              [j])]]
        [_ #f])))
> (define (in-digits/proc n)
    (for/list ([d (in-digits n)]) d))
> (for/list ([d (in-digits 1138)]) d)

'(8 3 1 1)

> (map in-digits (list 137 216))

'((7 3 1) (6 1 2))

syntax

(:do-in ([(outer-id ...) outer-expr] ...)
        outer-check
        ([loop-id loop-expr] ...)
        pos-guard
        ([(inner-id ...) inner-expr] ...)
        pre-guard
        post-guard
        (loop-arg ...))
一个只能在 for 的 for-clause (或其变体之一)中作为 seq-expr 使用的形式。

在 for 中, :do-in 形式的片段被拼接到迭代中,基本上如下所示:

(let-values ([(outer-id ...) outer-expr] ...)
  outer-check
  (let loop ([loop-id loop-expr] ...)
    (if pos-guard
        (let-values ([(inner-id ...) inner-expr] ...)
          (if pre-guard
              (let body-bindings
                   (if post-guard
                       (loop loop-arg ...)
                       done-expr))
              done-expr))
         done-expr)))

其中 body-bindings 和 done-expr 是来自 :do-in 的使用环境。 for 子句所绑定的标识符通常是 ([(inner-id ...) inner-expr] ...) 部分的一部分。

注意 body-bindings 和 done-expr 可以包含任意的表达式, 可能包括对 outer-id 或 inner-id 标识符的 set! , 如果它们在原始 for 形式中是可见的, 所以要小心在 post-guard 和 loop-arg 中依赖这样的标识符。

实际的 loop 绑定和调用有额外的循环参数,以支持与 :do-in 形式的并行迭代,其他的部分也同样伴随着来自并行迭代的部分。

关于 :do-in 的例子,参见 define-sequence-syntax。

3.18.3 Do 循环

syntax

(do ([id init-expr step-expr-maybe] ...)
    (stop?-expr finish-expr ...)
  expr ...)
 
step-expr-maybe = 
  | step-expr
迭代地评估 expr ,只要 stop?-expr 返回 #f。

为了初始化循环, init-expr 被依次评估并与相应的 id 绑定。 除 init-expr 外,表单中的所有表达式都绑定了 id 。

在 id 被绑定之后, stop?-expr 被评估。 如果它产生 #f ,每个 expr 都会被评估其副作用。 然后, id 被有效地更新为 step-expr 的值, 其中 id 的默认 step-expr 只是 id; 更确切地说,迭代继续进行, id 的新位置被初始化为相应 step-expr 的值。

当 stop?-expr 产生一个真值时,然后依次评估 finish-expr , 最后一个 expr 在尾部位置被评估,产生 do 表单的整体值。 如果没有提供 finish-expr ， do 形式的值是 #<void>。
