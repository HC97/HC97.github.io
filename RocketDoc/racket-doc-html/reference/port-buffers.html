13.1.3 端口缓冲区和位置

一些端口–特别是那些从文件中读取和写入文件的端口–是内部缓冲的:

  • 一个输入端口通常是默认的块缓冲,这意味着在任何一次读取时,缓冲区都被填充了立即可用的字节,以加快未来的读取速度。 因此,如果在对文件的两次读取之间修改了文件,第二次读取会产生陈旧的数据。 调用 file-position 来设置一个输入端口的文件位置会冲刷其缓冲区。

  • 默认情况下,输出端口通常是块缓冲区,尽管终端输出端口是行缓冲区,而初始错误输出端口是无缓冲区的。 当缓冲区满了(在块模式下)、写下换行时(在行模式下)、通过 close-output-port 关闭端口时、 或者通过 flush-output 这样的过程明确要求刷新时,输出缓冲区就会被填充一连串要提交的字节。

如果一个端口支持缓冲,它的缓冲模式可以通过 file-stream-buffer-mode 来改变(即使该端口不是一个 file-stream port)。

对于一个输入端口,偷看总是将偷看的字节放入该端口的缓冲区,即使该端口的缓冲区模式是 'none; 此外,在某些平台上,测试端口的输入(通过 char-ready? 或 sync)可以通过偷看来实现。 如果一个输入端口的缓冲区模式是 'none, 那么对于 read-bytes-avail!*, read-bytes-avail!, peek-bytes-avail!* 或 peek-bytes-avail! 来说,最多读取一个字节; 如果端口中有任何字节被缓冲(例如,为了满足先前的 peek),这些程序可以访问多个被缓冲的字节,但不会进一步读取字节。

此外,当初始的当前输出和错误端口是终端端口时(见 terminal-port?) ,以及在初始的标准输入端口上执行 read, read-line, read-bytes, read-string 等操作时,会自动刷新。 (更准确地说,是由默认的端口读取处理程序来代替 read, 见 port-read-handler 。)

procedure

(flush-output [out]) → void?

  out : output-port? = (current-output-port)
强制 将给定输出端口中的所有缓冲数据进行物理写入。 只有 file-stream ports、TCP 端口和自定义端口 (见 Custom Ports)使用缓冲区; 当在没有缓冲区的端口上调用时, flush-output 没有效果。

如果冲刷 file-stream port 或 TCP port 在写入时遇到错误,那么该端口中所有缓冲的字节都会被丢弃。 因此,进一步尝试刷新或关闭该端口将不会失败。

Changed in version 7.4.0.10 of package base: 一致地,在出错时丢弃缓冲字节,包括在 TCP 输出端口。

procedure

(file-stream-buffer-mode port) → (or/c 'none 'line 'block #f)

  port : port?
(file-stream-buffer-mode port mode) → void?
  port : port?
  mode : (or/c 'none 'line 'block)
获取或设置 port 的缓冲模式,如果可能的话。 File-stream ports 支持设置缓冲区模式,TCP端口(见 网络) 支持设置和获取缓冲区模式, 而自定义端口(见 Custom Ports) 可能支持获取和设置缓冲区模式。

如果提供了 mode ,它必须是 'none, 'line (仅输出)或 'block 中的一个,并且端口的缓冲会被相应地设置。 如果端口不支持设置模式,将引发 exn:fail exception is raised 。

如果没有提供 mode ,将返回当前的模式,如果不能确定模式,则返回 #f 。 如果 port 是一个输入端口,并且 mode 是 'line, 则会引发 exn:fail:contract exception is raised 。

procedure

(file-position port) → exact-nonnegative-integer?

  port : port?
(file-position port pos) → void?
  port : port?
  pos : (or/c exact-nonnegative-integer? eof-object?)
返回或设置 port 的当前读/写位置。

在没有位置的情况下调用 file-stream port 或 string port 以外的端口的 file-position， 如果位置已知,则返回从该端口读取的字节数(见 计算位置、行和列), 否则会引发 exn:fail:filesystem exception is raised 。

对于 file-stream ports 和 string ports, 位置设置变体将读/写位置设置为相对于文件或(字节)字符串的开头的 pos (如果 pos 是数字), 或者相对于文件或(字节)字符串的当前末端的位置(如果 pos 是 eof )。 在位置设置模式下, file-position 对于除 file-stream ports 和 string ports 之外的其他端口类型会引发 exn:fail:contract 。 此外,并不是所有的 file-stream ports 都支持设置位置; 如果在这样的 file-stream port 上用一个位置参数调用 file-position , 会引发 exn:fail:filesystem exception is raised 。

当 file-position 设置的位置 pos 超过了输出文件或 (字节) 字符串的当前大小时, 文件/字符串会被放大到 pos 的大小,并且新的区域被填充为 0 字节;在文件的情况下。 在文件输出端口的情况下,文件可能不会被扩大,直到有更多的数据被写入文件; 在这种情况下,要注意在 Unix 和 Mac OS 上以 'append 打开的文件, 在每次写入 前 会将文件指针重置到文件的末端,这就破坏了通过 file-position 的文件扩大。 如果 pos 超出了输入文件或(字节)字符串的末尾,那么此后的读取就会返回 eof ,而不会改变端口的位置。

当改变一个输出端口的文件位置时,如果该端口的缓冲区不是空的,就会首先刷新。 类似地,为一个输入端口设置位置会清空该端口的缓冲区(即使新的位置与旧的位置相同)。 然而,尽管由 open-input-output-file 产生的输入和输出端口共享文件位置, 通过一个端口设置位置并不会刷新另一个端口的缓冲区。

procedure

(file-position* port) → (or/c exact-nonnegative-integer? #f)

  port : port?
与单个参数的 file-position 类似,但如果不知道位置则返回 #f 。

procedure

(file-truncate port size) → void?

  port : (and/c output-port? file-stream-port?)
  size : exact-nonnegative-integer?
将 port 写入的文件的大小设置为 size, 假设该端口与一个可以设置大小的文件相关。

新的文件大小可以大于或小于其当前大小,但这个函数名称中的 "truncate" 反映了它通常用于减小文件的大小, 因为向文件写入或使用 file-position 可以扩展文件的大小。
