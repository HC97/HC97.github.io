8.3 参数化契约

 (require racket/contract/parametric) package: base

使用参数化契约最方便的方法是使用 contract-out 的 #:exists 关键字。 racket/contract/parametric 提供了一些更多的、通用的参数化合约。

syntax

(parametric->/c (x ...) c)

为参数化的多态函数创建一个契约。 每个函数都由 c 保护,其中每个 x 都被绑定在 c 中, 并指向一个多态类型,该类型在每次应用该函数时都被实例化。

在每次应用函数时, parametric->/c 契约为每个 x 构造一个新的不透明的包装器; 流入多态函数的值(即由相对于 parametric->/c 处于负数位置的某个 x 保护的值) 被包裹在相应的不透明的包装器中。 流出多态函数的值(即被相对于 parametric->/c 而言处于正位置的某个 x 所保护的值) 被检查是否有合适的包装器。 如果它们有,就会被解开;如果没有,就会发出违反契约的信号。

Examples:
> (define swap-ctc (parametric->/c [A B] (-> A B (values B A))))
> (define/contract (good-swap a b)
    swap-ctc
    (values b a))
> (good-swap 1 2)

procedure

(new-∀/c [name]) → contract?

  name : (or/c symbol? #f) = #f
构建一个新的通用契约。

万能合约在负数位置(例如,函数输入)时接受所有的值,并将它们包裹在一个不透明的结构中,隐藏精确的值。 在正数位置(比如函数返回),万能合约只接受之前在负数位置接受的值(通过检查包装器)。

名称用于在错误信息中识别合约,并默认为基于 new-∀/c 的词法上下文的名称。

例如,这个合同:
(let ([a (new-∀/c 'a)])
  (-> a a))
描述了身份函数(或一个非终结函数)。 也就是说, a 的第一次使用出现在一个负数的位置,因此该函数的输入被包裹在一个不透明的结构中。 然后,当函数返回时,会检查以确定结果是否被包裹,因为第二个 a 出现在一个正数位置。

new-∀/c 合约构造函数与 new-∃/c 是对应的。

procedure

(new-∃/c [name]) → contract?

  name : (or/c symbol? #f) = #f
构建一个新的存在性契约。

当处于正数位置时(例如,函数返回),存在性契约接受所有的值,并将它们包裹在一个不透明的结构中,隐藏精确的值。 在负数位置(比如函数输入),它们只接受之前在正数位置接受的值(通过检查包装器)。

名称用于在错误信息中识别合约,并默认为基于 new-∀/c 的词法上下文的名称。

例如,这个合同:
(let ([a (new-∃/c 'a)])
  (-> (-> a a)
      any/c))
描述了一个接受身份函数(或非终结函数)并返回一个任意值的函数。 也就是说,第一次使用的 a 出现在正数位置,因此该函数的输入被不透明的结构所包裹。 然后,当函数返回时,要检查其结果是否被包裹,因为第二个 a 出现在负数位置。

new-∃/c 构造函数与 new-∀/c 是对等的。
