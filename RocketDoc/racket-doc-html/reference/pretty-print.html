13.6 漂亮的打印

 (require racket/pretty) package: base
The bindings documented in this section are provided by the racket/pretty and racket libraries, but not racket/base.

procedure

(pretty-print v    
  [port    
  quote-depth    
  #:newline? newline?]) → void?
  v : any/c
  port : output-port? = (current-output-port)
  quote-depth : (or/c 0 1) = 0
  newline? : boolean? = #t
漂亮地打印值 v ,使用与默认 print 模式相同的打印形式,但插入换行符和空白, 以避免超过 (pretty-print-columns) 的行, 由 (pretty-print-current-style-table) 控制。 除非换行参数为 false 或 pretty-print-columns 参数被设置为 'infinity, 否则打印出来的表格默认以 newline? 结束。 如果 port 启用了计行功能 (参见 计算位置、行和列), 则打印时对打印开始时的列很敏感–既用于确定初始换行,也用于缩进后续行。

除了本节定义的参数外, pretty-print 还符合 print-graph, print-struct, print-hash-table, print-vector-length, print-box 和 print-as-expression 参数。

pretty-print 检测具有 prop:custom-write 属性的结构并调用相应的 custom-write 程序。 自定义写程序可以检查参数 pretty-printing ,以便与 pretty-printer 合作。 递归打印到端口会自动使用 pretty 打印,但是如果该结构有多个递归打印的子表达式, 自定义写程序可能需要更多的合作来插入显式换行。 使用 port-next-location 来确定当前的输出列, 使用 pretty-print-columns 来确定目标打印宽度, 使用 pretty-print-newline 来插入换行符(以便 pretty-print-print-line 参数中的函数可以被适当地调用)。 使用 make-tentative-pretty-print-output-port 获得暂定递归打印的端口(例如,检查输出的长度)。

如果省略 newline? 参数或提供 true, pretty-print-print-line 回调将以 false 作为第一个参数被调用,以打印出打印值后的最后一个换行。 如果提供的是 false,则在打印值之后不调用 pretty-print-print-line 回调。

Changed in version 6.6.0.3 of package base: 增加了 newline? 参数。

  newline? : boolean? = #t
与 pretty-print 相同,但 v 的打印方式是 write ,而不是 print 。

Changed in version 6.6.0.3 of package base: 增加了 newline? 参数。

procedure

(pretty-display v [port #:newline? newline?]) → void?

  v : any/c
  port : output-port? = (current-output-port)
  newline? : boolean? = #t
与 pretty-print 相同,但 v 的打印方式是 display 而不是 print 。

Changed in version 6.6.0.3 of package base: 增加了 newline? 参数。

procedure

(pretty-format v [columns #:mode mode]) → string?

  v : any/c
  columns : exact-nonnegative-integer? = (pretty-print-columns)
  mode : (or/c 'print 'write 'display) = 'print
和 pretty-print 一样,只是它返回一个包含 pretty-print 值的字符串,而不是将输出发送到一个端口。

可选的参数 columns 参数用于参数化 pretty-print-columns 。

关键字参数 mode 控制打印是否像 pretty-print (默认)、 pretty-write 或 pretty-display 那样进行。

Changed in version 6.3 of package base: 增加了一个 mode 参数。

procedure

(pretty-print-handler v) → void?

  v : any/c
如果 v 不是 #<void> ,则 Pretty-prints v ; 如果 v 是 #<void> ,则不打印。 将此程序传递给 current-print , 以将 pretty 打印机安装到 read-eval-print-loop 运行的 REPL 中。

13.6.1 基本的漂亮打印选项

parameter

(pretty-print-columns)

 → (or/c exact-positive-integer? 'infinity)
(pretty-print-columns width) → void?
  width : (or/c exact-positive-integer? 'infinity)
一个 parameter ,决定漂亮打印的默认宽度。

如果显示宽度为 'infinity, 那么 pretty-print 的输出就不会被分成几行,而且输出的结尾也不会添加换行。

parameter

(pretty-print-depth) → (or/c exact-nonnegative-integer? #f)

(pretty-print-depth depth) → void?
  depth : (or/c exact-nonnegative-integer? #f)
控制递归漂亮打印的默认深度的参数。 打印到 depth 意味着比 depth 嵌套更深的元素被替换为"..."; 特别是,深度为 0 表示只打印简单的值。 深度为 #f (默认)允许打印到任意深度。

parameter

(pretty-print-exact-as-decimal) → boolean?

(pretty-print-exact-as-decimal as-decimal?) → void?
  as-decimal? : any/c
一个 parameter ,决定如何打印精确的非整数。 如果该参数的值是 #t, 那么一个精确的非整数就会被打印成小数而不是分数。初始值是 #f 。

parameter

(pretty-print-.-symbol-without-bars) → boolean?

(pretty-print-.-symbol-without-bars on?) → void?
  on? : any/c
一个 parameter ,控制打印名称仅为句号的符号的打印。 如果设置为真值,那么这样的符号将被打印为只有句号。 如果设置为假值,则打印为句号,周围有竖条。

parameter

(pretty-print-show-inexactness) → boolean?

(pretty-print-show-inexactness show?) → void?
  show? : any/c
一个 parameter ,决定如何打印不精确的数字。 如果该参数的值是 #t, 那么不精确的数字总是以前面的 #i 来打印。 初始值是 #f 。

13.6.2 每个符号的特殊打印

parameter

(pretty-print-abbreviate-read-macros) → boolean?

(pretty-print-abbreviate-read-macros abbrev?) → void?
  abbrev? : any/c
一个 parameter ,用于控制 quote, unquote, unquote-splicing 等是否用 ', ,, ,@ 等进行缩写。 默认情况下,缩略语被启用。

参见 pretty-print-remap-stylable 。

procedure

(pretty-print-style-table? v) → boolean?

  v : any/c
如果 v 是一个用于 pretty-print-current-style-table 的样式表, 返回 #t ,否则返回 #f 。

parameter

(pretty-print-current-style-table) → pretty-print-style-table?

(pretty-print-current-style-table style-table) → void?
  style-table : pretty-print-style-table?
一个 parameter ,持有一个样式映射表。 参见 pretty-print-extend-style-table 。

procedure

(pretty-print-extend-style-table style-table 
  symbol-list 
  like-symbol-list) 
 → pretty-print-style-table?
  style-table : pretty-print-style-table?
  symbol-list : (listof symbol?)
  like-symbol-list : (listof symbol?)
通过扩展现有的 style-table 来创建一个新的样式表,这样,原表中 like-symbol-list 的每个符号的样式映射就被用于新表中 symbol-list 的相应符号。 symbol-list 和 like-symbol-list 列表必须有相同的长度。 style-table 参数可以是 #f, 在这种情况下,会使用原表中的默认映射(见下文)。

一个符号的样式映射控制了在打印以该符号开始的列表时插入空白的方式。 在没有任何映射的情况下,当一个列表被分成多行时,列表的每个元素都被打印在自己的行上,每个元素都有相同的缩进。

默认的样式映射包括以下符号的映射,因此输出遵循流行的代码格式化规则:

'lambda 'λ 'case-lambda
'define 'define-macro 'define-syntax
'let 'letrec 'let*
'let-syntax 'letrec-syntax
'let-values 'letrec-values 'let*-values
'let-syntaxes 'letrec-syntaxes
'begin 'begin0 'do
'if 'set! 'set!-values
'unless 'when
'cond 'case 'and 'or
'module
'syntax-rules 'syntax-case 'letrec-syntaxes+values
'import 'export 'link
'require 'require-for-syntax 'require-for-template 'provide
'public 'private 'override 'rename 'inherit 'field 'init
'shared 'send 'class 'instantiate 'make-object

parameter

(pretty-print-remap-stylable)

 → (any/c . -> . (or/c symbol? #f))
(pretty-print-remap-stylable proc) → void?
  proc : (any/c . -> . (or/c symbol? #f))
一个 parameter ,用于控制样式的重映射和读者速记符号的确定。

这个过程是对作为序列中第一个元素出现的每个子表达式的调用。 如果它返回一个符号,就会使用样式表,就像该符号位于序列的头部一样。 如果它返回 #f,样式表将被正常处理。 同样地,当决定是否缩写读者宏时,会参考这个参数。

13.6.3 线性输出钩

procedure

(pretty-print-newline port width) → void?

  port : output-port?
  width : exact-nonnegative-integer?
调用与 pretty-print-print-line 参数相关的程序,以向 port 打印换行, 如果 port 是被重定向到原始输出端口进行打印的输出端口,否则将向 port 打印一个普通的换行。 width 参数应该是目标列的宽度,通常从 pretty-print-columns 获得。

parameter

(pretty-print-print-line)

 → 
((or/c exact-nonnegative-integer? #f)
 output-port?
 exact-nonnegative-integer?
 (or/c exact-nonnegative-integer? 'infinity)
 . -> .
 exact-nonnegative-integer?)
(pretty-print-print-line proc) → void?
  proc : 
((or/c exact-nonnegative-integer? #f)
 output-port?
 exact-nonnegative-integer?
 (or/c exact-nonnegative-integer? 'infinity)
 . -> .
 exact-nonnegative-integer?)
一个 parameter ,决定了打印 pretty-printed 值的行间换行分隔符的程序。 调用该程序时有四个参数:新行号、输出端口、旧行的长度和目标列的数量。 proc 的返回值是它在新行开头打印的额外字符数。

proc 过程在打印任何字符之前被调用,行号为 0 ,旧行长度为 0 。 每当 pretty-printer 开始一个新行时,就会调用 proc , 并给出新行的编号(其中第一个新行的编号为 1) 和刚刚完成的行的长度。 proc 的目的地列参数始终是目的地打印区域的总宽度, 如果 pretty-printed 值不分成行,则为 'infinity 。

如果 #:newline? 参数被省略或提供了一个真值, 那么在打印完数值的最后一个字符后也会调用 proc ， #f 作为行号,并提供最后一行的长度。

默认的 proc 过程,只要行号不是 0 ,列数不是 'infinity, 就会打印一个换行符,总是返回 0 。 自定义 proc 过程可用于在每行漂亮的输出前打印额外的文本; 每行前打印的字符数应由 proc 返回,以便正确选择下一个换行。

提供给 proc 的目标端口通常不是提供给 pretty-print 或 pretty-display 的端口(或当前的输出端口), 但对该端口的输出最终会被重定向到提供给 pretty-print 或 pretty-display 的端口。

13.6.4 值输出钩

parameter

(pretty-print-size-hook)

 → 
(any/c boolean? output-port?
 . -> .
 (or/c #f exact-nonnegative-integer?))
(pretty-print-size-hook proc) → void?
  proc : 
(any/c boolean? output-port?
 . -> .
 (or/c #f exact-nonnegative-integer?))
一个决定 pretty-printing 的尺寸钩的 parameter 。

尺寸钩子被应用于每个要打印的值。 如果钩子返回 #f, 那么打印将由 pretty-printer 内部处理。 否则,该值应该是一个整数,指定打印值的字符长度; 打印钩子将被调用以实际打印该值(见 pretty-print-print-hook) 。

调整大小的钩子接收三个参数。 第一个参数是要打印的值。第二个参数是一个布尔值: #t 表示像 display 一样打印, #f 表示像 write 一样打印。 第三个参数是目标端口;该端口是提供给 pretty-print 或 pretty-display 的端口 (或当前的输出端口)。 在 pretty-printing 过程中,大小钩可以多次应用于一个单一的值。

parameter

(pretty-print-print-hook)

 → (any/c boolean? output-port? . -> . void?)
(pretty-print-print-hook proc) → void?
  proc : (any/c boolean? output-port? . -> . void?)
一个 parameter ,用于确定 pret-printing 的打印钩。 当尺寸钩子(见 pretty-print-size-hook) 为一个值返回一个整数尺寸时, 打印钩子程序将应用于该值进行打印。

打印钩子接收三个参数。第一个参数是要打印的值。 第二个参数是一个布尔值: #t 表示像 display 一样打印, #f 表示像 write 一样打印。 第三个参数是目标端口; 这个端口通常不是提供给 pretty-print 或 pretty-display 的端口(或当前的输出端口), 但这个端口的输出最终会被重定向到提供给 pretty-print 或 pretty-display 的端口。

parameter

(pretty-print-pre-print-hook)

 → (any/c output-port? . -> . void)
(pretty-print-pre-print-hook proc) → void?
  proc : (any/c output-port? . -> . void)
一个 parameter ,它决定了一个钩子程序在对象被打印之前被调用。 该钩子接收两个参数:对象和输出端口。 端口是提供给 pretty-print 或 pretty-display 的端口(或当前的输出端口)。

parameter

(pretty-print-post-print-hook)

 → (any/c output-port? . -> . void)
(pretty-print-post-print-hook proc) → void?
  proc : (any/c output-port? . -> . void)
一个 parameter ,它决定了一个钩子程序在对象被打印后被调用。 该钩子接收两个参数:对象和输出端口。 该端口是提供给 pretty-print 或 pretty-display 的端口(或当前的输出端口)。

13.6.5 额外的自定义输出支持

parameter

(pretty-printing) → boolean?

(pretty-printing on?) → void?
  on? : any/c
当 pretty 打印机调用自定义写程序(见 prop:custom-write)时, 一个 parameter 被设置为 #t ,以便在支持换行的模式下输出。 当 pretty printer 调用自定义写程序只是为了检测周期或试图在单行上打印时,它将此参数设置为 #f 。

procedure

(make-tentative-pretty-print-output-port out 
  width 
  overflow-thunk) 
 → output-port?
  out : output-port?
  width : exact-nonnegative-integer?
  overflow-thunk : (-> any)
产生一个适合递归漂亮打印的输出端口,而不实际产生输出。 当适当的输出取决于递归打印的大小时,使用这样的端口来试探性地打印。 在打印之后,使用 file-position 来确定暂定输出的大小。

out 参数应该是一个 pretty-printing 端口,比如当 pretty-printing 设置为 "true "时, 提供给自定义写程序的端口,或者其他暂定输出端口。 width 参数应该是一个目标列的宽度,通常从 pretty-print-columns 中获得, 可能要减去,以便为终止符留出空间。 如果超过 width 的项目被打印到端口,或者通过 pretty-print-newline 打印到端口, 就会调用 overflow-thunk 过程;它可以通过一个 continuation 作为快捷方式逃离递归打印, 但 overflow-thunk 也可以返回,在这种情况下,以后每次有额外的输出被写入端口时都会调用它。

在试探性打印之后,要么用 tentative-pretty-print-port-transfer 接受结果, 要么用 tentative-pretty-print-port-cancel 拒绝结果。 如果不能正确接受或取消,就会干扰图形结构的打印、对钩子程序的调用等。 明确取消暂定打印,即使 overflow-thunk 从递归打印中逃脱。

procedure

(tentative-pretty-print-port-transfer tentative-out    
  orig-out) → void?
  tentative-out : output-port?
  orig-out : output-port?
导致写到 tentative-out 的数据被转移,就像写到 orig-out 一样。 tentative-out 参数应该是由 make-tentative-pretty-print-output-port 产生的端口, 而 orig-out 应该是一个 pretty-printing 端口(提供给自定义写程序)或另一个 tentative 输出端口。

procedure

(tentative-pretty-print-port-cancel tentative-out) → void?

  tentative-out : output-port?
取消由 make-tentative-pretty-print-output-port 产生的 tentative-out 的内容。 取消的主要作用是撤销图形引用的定义,这样, 将来打印图形引用对象时就包括定义的 #‹n›= 。
