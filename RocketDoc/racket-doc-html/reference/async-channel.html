11.2.4 缓冲的异步通道

 (require racket/async-channel) package: base
The bindings documented in this section are provided by the racket/async-channel library, not racket/base or racket.

11.2.4.1 创建和使用异步通道

参见 线程邮箱。

一个 asynchronous channel 就像一个 channel , 但是它缓冲了数值,这样发送操作就不会在接收操作上等待。

除了与异步通道特有的程序一起使用外, 一个异步通道还可以作为一个 synchronizable event使用(见 事件)。 当 async-channel-get 不会阻塞时,一个异步通道是 ready for synchronization ; 异步通道的 synchronization result 与 async-channel-get 的结果相同。

procedure

(async-channel? v) → boolean?

  v : any/c
如果 v 是一个异步通道,返回 #t ,否则返回 #f 。

procedure

(make-async-channel [limit]) → async-channel?

  limit : (or/c exact-positive-integer? #f) = #f
返回一个异步通道,其缓冲区限制为 limit 项。 当通道是空的时候,获取操作会被阻止,而当通道已经有 limit 项的时候,投放操作会被阻止。 如果 limit 是 #f, 通道的缓冲区没有限制(所以put永远不会阻塞)。

procedure

(async-channel-get ach) → any/c

  ach : async-channel?
阻塞,直到 ach 中至少有一个值可用,然后返回被放入 async-channel 的第一个值。

procedure

(async-channel-try-get ach) → any/c

  ach : async-channel?
如果 ach 中至少有一个值是立即可用的,则返回被放入 ach 中的第一个值。 如果 async-channel 是空的,结果是 #f 。

procedure

(async-channel-put ach v) → void?

  ach : async-channel?
  v : any/c
将 v 放入 ach 中,如果 ach 的缓冲区已满,则阻塞,直到有可用空间。

procedure

(async-channel-put-evt ach v) → evt?

  ach : async-channel?
  v : any/c
当 (async-channel-put ach v) 会返回一个值时(即,当通道持有的值已经少于其限制时), 返回一个 synchronizable event , 该事件是 ready for synchronization ; the synchronization result of a asynchronous channel-put event is the asynchronous channel-put event itself。

Examples:
(define (server input-channel output-channel)
  (thread (lambda ()
            (define (get)
              (async-channel-get input-channel))
            (define (put x)
              (async-channel-put output-channel x))
            (define (do-large-computation)
              (sqrt 9))
            (let loop ([data (get)])
              (case data
                [(quit) (void)]
                [(add) (begin
                         (put (+ 1 (get)))
                         (loop (get)))]
                [(long) (begin
                          (put (do-large-computation))
                          (loop (get)))])))))
(define to-server (make-async-channel))
(define from-server (make-async-channel))

 

> (server to-server from-server)

#<thread>

> (async-channel? to-server)

#t

> (printf "Adding 1 to 4\n")

Adding 1 to 4

> (async-channel-put to-server 'add)
> (async-channel-put to-server 4)
> (printf "Result is ~a\n" (async-channel-get from-server))

Result is 5

> (printf "Ask server to do a long computation\n")

Ask server to do a long computation

> (async-channel-put to-server 'long)
> (printf "I can do other stuff\n")

I can do other stuff

> (printf "Ok, computation from server is ~a\n"
          (async-channel-get from-server))

Ok, computation from server is 3

> (async-channel-put to-server 'quit)

11.2.4.2 异步通道上的契约和冒名顶替者

procedure

(async-channel/c c) → contract?

  c : contract?
返回一个能识别异步通道的契约。 放入通道或从通道获取的值必须与 c 匹配。

如果 c 参数是一个平坦的合同或一个伴侣合同,那么结果将是一个伴侣合同。 否则,结果将是一个冒充者合同。

当一个 async-channel/c 合约被应用于一个异步通道时,其结果不是 eq? 到输入。 根据合同的类型,结果将是输入的 chaperone 或 impersonator 。

procedure

(impersonate-async-channel channel 
  get-proc 
  put-proc 
  prop 
  prop-val ... 
  ...) 
 → (and/c async-channel? impersonator?)
  channel : async-channel?
  get-proc : (any/c . -> . any/c)
  put-proc : (any/c . -> . any/c)
  prop : impersonator-property?
  prop-val : any
返回 channel 的impersonator, 它可以重定向 async-channel-get 和 async-channel-put 操作。

get-proc 必须接受 async-channel-get 在 channel 上产生的值; 它必须产生一个替换值,这是在impersonator上get操作的结果。

put-proc 必须接受传递给在 channel 上调用的 async-channel-put 的值; 它必须产生一个替换值,即传递给在原始通道上调用的put过程的值。

get-proc 和 put-proc 过程在所有从通道获取或放入值的操作中被调用, 而不仅仅是 async-channel-get 和 async-channel-put 。

prop 和 prop-val (impersonate-async-channel 的参数数量必须是奇数) 的对子增加了 impersonator properties 属性或覆盖了 channel 的 impersonator property 值。

procedure

(chaperone-async-channel channel 
  get-proc 
  put-proc 
  prop 
  prop-val ... 
  ...) 
 → (and/c async-channel? chaperone?)
  channel : async-channel?
  get-proc : (any/c . -> . any/c)
  put-proc : (any/c . -> . any/c)
  prop : impersonator-property?
  prop-val : any
与 impersonate-async-channel 类似,但 get-proc 过程必须产生相同的值或原始值的陪衬, 而 put-proc 必须产生相同的值或原始值的陪衬。
