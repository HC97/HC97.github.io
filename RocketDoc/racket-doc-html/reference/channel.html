11.2.2 Channels

一个 channel 既能同步一对线程,又能从一个线程向另一个线程传递一个值。 通道是同步的;发送方和接收方都必须阻塞,直到(原子)交易完成。 多个发送方和接收方可以同时访问一个通道,但每个事务都会选择一个发送方和接收方。

通道同步是 fair: 如果一个线程在一个通道上被阻塞,而通道的交易机会无限地出现,那么这个线程最终会参与到一个交易中。

除了与通道特定的程序一起使用外,一个通道还可以作为一个 synchronizable event (参见 事件)。 当 channel-get 不会阻塞时,一个通道是 ready for synchronization ; 该通道的 synchronization result 与 channel-get 的结果相同。

对于缓冲的异步通道,见 缓冲的异步通道 。

procedure

(channel? v) → boolean?

  v : any/c
如果 v 是一个 channel,返回 #t ,否则返回 #f 。

procedure

(make-channel) → channel?

创建并返回一个新的通道。 该通道可以与 channel-get 、 channel-try-get 一起使用, 或者作为一个 synchronizable event (参见 事件),通过该通道接收一个值。 该通道可以与 channel-put 一起使用,或者通过 channel-put-evt 的结果,通过该通道发送一个值。

procedure

(channel-get ch) → any

  ch : channel?
阻断直到发送者准备好通过 ch 提供一个值。 其结果是发送的值。

procedure

(channel-try-get ch) → any

  ch : channel?
如果发送方立即准备好,则从 ch 接收并返回一个值,否则返回 #f 。

procedure

(channel-put ch v) → void?

  ch : channel?
  v : any/c
阻断,直到一个接收者准备好通过 ch 接受值 v 。

procedure

(channel-put-evt ch v) → channel-put-evt?

  ch : channel?
  v : any/c
返回一个新的 synchronizable event ,与 sync 一起使用。 当 (channel-put ch v) 不会阻塞时,该事件是 ready for synchronization , 并且该事件的 synchronization result 是该事件本身。

procedure

(channel-put-evt? v) → boolean?

  v : any/c
如果 v 是由 channel-put-evt 产生的 channel-put 事件, 返回 #t ,否则返回 #f 。
