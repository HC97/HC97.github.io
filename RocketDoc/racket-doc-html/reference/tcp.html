15.3.1 TCP

 (require racket/tcp) package: base
The bindings documented in this section are provided by the racket/tcp and racket libraries, but not racket/base.

关于 TCP 的一般信息,见 W.Richard Stevens 的 TCP/IP Illustrated, Volume 1 。

procedure

(tcp-listen port-no    
  [max-allow-wait    
  reuse?    
  hostname]) → tcp-listener?
  port-no : listen-port-number?
  max-allow-wait : exact-nonnegative-integer? = 4
  reuse? : any/c = #f
  hostname : (or/c string? #f) = #f
在本地机器上创建一个 "监听" 服务器,端口号由 port-no 指定。 如果 port-no 为 0,套接字将绑定到一个短暂的端口,这可以通过调用 tcp-addresses 来确定。 max-allow-wait 参数决定了可以等待接受的客户端连接的最大数量。 (当 max-allow-wait 客户端在等待接受时,不能有新的客户端连接。)

如果 reuse? 参数为真,那么即使端口处于 TIME_WAIT 状态, tcp-listen 也会创建一个监听器。 这种对 reuse? 的使用违背了 TCP 协议的某些保证;详情请见 Stevens 的书。 此外,在许多现代平台上,只有当监听器先前是以 reuse? 的真值创建的, reuse? 的真值才会覆盖 TIME_WAIT 。

如果 hostname 是 #f (默认值),那么监听器接受与所有监听机器地址的连接。 否则,监听器只接受与给定主机名相关的接口的连接。 例如,提供 "127.0.0.1" 作为 hostname 会创建一个只接受来自本地机器的 "127.0.0.1" (回环接口)连接的监听器。

(Racket 实现了一个具有多个套接字的监听器,如果有必要的话,以适应不同协议系列的多个地址。 在 Linux 上,如果 hostname 同时映射到 IPv4 和 IPv6 地址,那么行为取决于是否支持 IPv6, 以及 IPv6 套接字是否可以配置为只监听 IPv6 连接:如果不支持 IPv6 或不能配置 IPv6 套接字,那么 IPv6 地址将被忽略; 否则,每个 IPv6 监听器只接受 IPv6 连接)。

tcp-listen 的返回值是一个 TCP listener 。 这个值可以在未来调用 tcp-accept, tcp-accept-ready? 和 tcp-close 时使用。 每个新的 TCP 监听器值都会被放入当前托管人的管理中(见 Custodians) 。

如果服务器不能被 tcp-listen 启动,就会引发 exn:fail:network exception is raised 。

TCP 监听器可以作为一个 synchronizable event (见 事件) 。 当 tcp-accept 不会阻塞时,一个 TCP 监听器就 ready for synchronization 了; the synchronization result of a TCP listener is the TCP listener itself 。

procedure

(tcp-connect hostname    
  port-no    
  [local-hostname    
  local-port-no]) → 
input-port? output-port?
  hostname : string?
  port-no : port-number?
  local-hostname : (or/c string? #f) = #f
  local-port-no : (or/c port-number? #f) = #f
试图以客户身份连接到一个监听服务器。 hostname 参数是服务器主机的互联网地址名, port-no 是服务器正在监听的端口号。

(如果 hostname 与多个地址相关联,它们将被逐一尝试,直到连接成功。名称 "localhost" 一般是指本地的机器)。

可选的 local-hostname 和 local-port-no 指定客户端的地址和端口。 如果两者都是 #f (默认值),客户端的地址和端口会被自动选择。 如果 local-hostname 不是 #f ,那么 local-port-no 必须是非 #f 。 如果 local-port-no 不是 #f ,并且 local-hostname 是 #f, 那么就会使用给定的端口,但是会自动选择地址。

tcp-connect 会返回两个值:一个输入端口和一个输出端口。 数据可以通过输入端口从服务器接收,通过输出端口发送到服务器。 如果服务器是一个 Racket 程序,它可以通过 tcp-accept 获得端口来与客户端通信。 这些端口被放入当前托管人的管理中(见 Custodians)。

最初,返回的输入端口是块缓冲的,而返回的输出端口是块缓冲的。 使用 file-stream-buffer-mode 改变缓冲模式。 当TCP输出端口为块缓冲时,该端口的 Nagle 算法被禁用,这与设置 TCP_NODELAY 套接字选项相对应。

两个返回的端口都必须被关闭,以终止 TCP 连接。 当两个端口仍然开放时,用 close-output-port 关闭输出端口会向服务器发送一个 TCP 关闭 (如果服务器通过一个端口读取连接,这将被视为文件结束)。 相反, tcp-abandon-port (见下文)关闭了输出端口,但在输入端口也被关闭之前不会发送 TCP 关闭。

注意,TCP 协议不支持一端愿意发送但不愿意读取的状态,也不包括连接的一端完全关闭时的自动消息。 相反,连接的另一端只有在发送数据时才会发现一端完全关闭; 特别是,在仍然开放的一端的一些写操作可能看起来是成功的,尽管写操作最终会产生一个错误。

如果 tcp-connect 不能建立连接,就会引发 exn:fail:network exception is raised 。

Changed in version 8.8.0.8 of package base: 更改了块缓冲,以暗示 TCP_NODELAY 。

procedure

(tcp-connect/enable-break hostname 
  port-no 
  [local-hostname] 
  local-port-no) 
 → 
input-port? output-port?
  hostname : string?
  port-no : port-number?
  local-hostname : (or/c string? #f) = #f
  local-port-no : (or/c port-number? #f)
像 tcp-connect 一样,但在尝试连接时启用中断(见 中断) 。 如果在调用 tcp-connect/enable-break 时禁用 break,那么要么返回端口, 要么引发 exn:break ,但不能同时进行。

procedure

(tcp-accept listener) → 
input-port? output-port?
  listener : tcp-listener?
接受与 listener 相关的服务器的客户连接。 如果在监听端口上没有客户连接在等待,对 tcp-accept 的调用将被阻止。 (参见 tcp-accept-ready? 。)

tcp-accept 会返回两个值:一个输入端口和一个输出端口。 数据可以通过输入端口从客户端接收,通过输出端口发送到客户端。 这些端口被置于当前托管人的管理之中(见 Custodians) 。

在缓冲和连接状态方面,这些端口与 tcp-connect 的端口作用相同。

如果一个连接不能被 tcp-accept 接受,或者监听器被关闭,就会引发 exn:fail:network exception is raised 。

procedure

(tcp-accept/enable-break listener) → 
input-port? output-port?
  listener : tcp-listener?
像 tcp-accept 一样,但在试图接受一个连接时,会启用断裂(见 中断) 。 如果在调用 tcp-accept/enable-break 时禁用 break,那么要么返回端口, 要么引发 exn:break ,但不能同时进行。

procedure

(tcp-accept-ready? listener) → boolean?

  listener : tcp-listener?
测试一个未被接受的客户端是否已经连接到与 listener 相关的服务器。 如果有客户端在等待,返回值为 #t ,否则为 #f 。 用 tcp-accept 过程接受一个客户端,该过程返回与客户端通信的端口,并将该客户端从未接受的客户端列表中移除。

如果监听器被关闭,就会引发 exn:fail:network exception is raised 。

procedure

(tcp-close listener) → void?

  listener : tcp-listener?
关闭与 listener 相关的服务器。 所有未被接受的客户端都会收到服务器发出的文件结束的消息;被接受的客户端的连接不受影响。

如果监听器已经被关闭,就会引发 exn:fail:network exception is raised 。

监听器的端口号可能不会立即对新的监听器可用(使用 tcp-listen 的默认 reuse? 参数)。 更多信息请参见 Stevens 对 TIME_WAIT 状态的解释。

procedure

(tcp-listener? v) → boolean?

  v : any/c
如果 v 是由 tcp-listen 创建的 TCP listener , 返回 #t ,否则返回 #f 。

procedure

(tcp-accept-evt listener) → evt?

  listener : tcp-listener?
当 listener 上的 tcp-accept 不会阻塞时, 返回一个 ready for synchronization (见 事件), 该事件是 synchronizable event 。 synchronization result 是一个包含两个项目的列表,对应于 tcp-accept 的两个结果。 (如果在 sync 中没有选择该事件,则不接受任何连接)。 在 tcp-accept-evt 被调用时,这些端口被放入作为当前监护人的管理中(见 Custodians) 。

procedure

(tcp-abandon-port tcp-port) → void?

  tcp-port : tcp-port?
与 close-output-port 与 close-input-port 一样(取决于 tcp-port 是输入端口还是输出端口),但如果 tcp-port 是输出端口, 其相关的输入端口尚未关闭,那么 TCP 连接的另一端就不会收到 TCP 关闭消息,直到输入端口也被关闭。

TCP 协议不包括连接上的 "不再读取" 状态,所以 tcp-abandon-port 等同于输入 TCP ports 上的 close-input-port 。

procedure

(tcp-addresses tcp-port [port-numbers?])

 → 
(or/c (values string? string?)
      (values string? port-number?
              string? listen-port-number?))
  tcp-port : (or/c tcp-port? tcp-listener? udp?)
  port-numbers? : any/c = #f
当 port-numbers? 为 #f (默认)时,返回两个字符串。 第一个字符串是由给定的 TCP port 的连接、TCP 监听器或 UDP 套接字查看的本地机器的互联网地址。 (当一台机器提供多个地址时,如果你算上环回设备,它通常会这样做,结果是特定于连接或特定于监听器的。) 如果给了一个监听器或 UDP 套接字,而它没有特定的主机,第一个字符串的结果是 "0.0.0.0" 。 第二个字符串是连接另一端的互联网地址,或者对于监听器或未连接的 UDP 套接字,总是 "0.0.0.0" 。

如果 port-numbers? 为true,那么将返回四个结果:本地机器地址的字符串, 本地机器端口号的 1 到 65535 之间的精确整数,远程机器地址的字符串, 远程机器端口号的 1 到 65535 之间的精确整数,或者监听器的 0 。

如果给定的端口、监听器或套接字已被关闭,将引发 exn:fail:network exception is raised 。

procedure

(tcp-port? v) → boolean?

  v : any/c
如果 v 是一个 TCP port—也就是 tcp-accept, tcp-connect, tcp-accept/enable-break 返回的端口,或者 tcp-connect/enable-break—#f , 否则返回 #t 。

value

port-number? : contract?

相当于 (integer-in 1 65535) 。

Added in version 6.3 of package base.

value

listen-port-number? : contract?

相当于 (integer-in 0 65535) 。

Added in version 6.3 of package base.
