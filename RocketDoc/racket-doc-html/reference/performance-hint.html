3.25 性能提示: begin-encourage-inline

 (require racket/performance-hint) package: base
The bindings documented in this section are provided by the racket/performance-hint library, not racket/base or racket.

syntax

(begin-encourage-inline form ...)

为每个 form 附加一个 'compiler-hint:cross-module-inline syntax property , 当 form 是一个函数定义时,这很有用。参见 define-values。

begin-encourage-inline 形式也由 (submod racket/performance-hint begin-encourage-inline) 模块提供, 它比 racket/performance-hint 的依赖性更少。

Changed in version 6.2 of package base: 增加 (submod racket/performance-hint begin-encourage-inline) 子模块。

syntax

(define-inline id expr)

(define-inline (head args) body ...+)
 
head = id
  | (head args)
     
args = arg ...
  | arg ... . rest-id
     
arg = arg-id
  | [arg-id default-expr]
  | keyword arg-id
  | keyword [arg-id default-expr]
和 define 一样,但确保定义在其调用位置被内联。 递归调用不被内联,以避免无限内联。 支持高阶使用,但也不内联。 错误的应用(通过提供错误的参数数量或错误的关键字参数)也不会被内联, 而是作为一个运行时错误留下。

define-inline 的形式可能会干扰 Racket 编译器自身的内联启发法, 因此只有在其他内联尝试(如 begin-encourage-inline) 失败时才可以使用。

Changed in version 8.1.0.5 of package base: 改为将错误应用作为运行时错误处理。
