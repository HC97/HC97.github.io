5 结构

程序员定义的数据类型 in Racket 指南 introduces structure types via struct.

structure type 是一个由若干 fields 组成的记录数据类型。 一个 structure ,一个结构类型的实例,是一个一级值,它包含了结构类型的每个字段的值。 一个结构实例是通过一个特定类型的 constructor 程序创建的, 它的字段值是通过特定类型的 accessor 和 mutator 程序来访问和改变。 此外,每个结构类型都有一个 predicate 过程,对结构类型的实例回答 #t , 对任何其他值回答 #f 。

一个结构类型的字段基本上是不命名的,尽管为了报告错误,支持命名。 构造函数过程为结构类型的每个字段取一个值,但结构类型的某些字段可以是 automatic fields ; 自动字段被初始化为一个与结构类型相关的常量,相应的参数在构造函数过程中被省略。 结构类型中的所有自动字段跟随非自动字段。

一个结构类型可以作为现有基础结构类型的 structure subtype 被创建。 一个结构子类型的实例总是可以被用作基结构类型的实例,但是该子类型有自己的谓词过程, 而且除了基类型的字段之外,它还可以有自己的字段。

一个结构子类型 "继承" 了它的基类型的字段。 如果基类型有 m 个字段,如果为新的结构子类型指定了 n 个字段,那么产生的结构类型就有 m+n 个字段。 自动字段的值在子类型中可以与它的基类型中不同。

如果原来的 m 个字段中的 m′ 是非自动的(其中 m′<m) , 而新的字段中的 n′ 是非自动的(其中 n′<n),那么必须向子类型的构造过程提供 m′+n′ 字段值。 一个子类型实例的前 m 个字段的值用原始基类型(或其超类型)的选择程序来访问, 最后 n 个字段用子类型特定的选择程序来访问。 子类型特定的 accessors 和前 m 个字段的 mutators 并不存在。

struct 形式和 make-struct-type 过程通常会创建一个新的结构类型, 但它们也可以访问全局共享的 prefab (即之前制造的)结构类型, 其实例可以被默认的阅读器(参见 阅读器) 和打印机(参见 打印机)解析和写入。 预制结构类型只能继承自其他预制结构类型, 而且它们不能有守护(参见 创建结构类型) 或属性(参见 结构类型属性) 。 对于名称、超类型、字段数、自动字段数、自动字段值(当至少有一个自动字段时)和字段突变性的每种组合, 都存在一个预制结构类型。

Serialization also provides information on reading and writing structures.

两个 结构值是 eqv? ,当且仅当它们是 eq? 。 如果两个结构值是 eq?,它们就是 equal? 。 默认情况下,如果两个结构值是同一结构类型的实例,没有字段是不透明的, 并且对结构应用 struct->vector 的结果是 equal? ,那么它们也是 equal? 。 (因此,结构的 equal? 测试可能取决于当前的检查器)。 结构类型可以通过 gen:equal+hash 或 gen:equal-mode+hash generic interface 覆盖默认的 equal? 定义。
