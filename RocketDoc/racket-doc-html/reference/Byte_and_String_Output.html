13.3 Byte and String Output

procedure

(write-char char [out]) → void?

  char : char?
  out : output-port? = (current-output-port)
写入一个字符到 out; 更准确地说,是将 char 的 UTF-8 编码的字节写到 out 。

procedure

(write-byte byte [out]) → void?

  byte : byte?
  out : output-port? = (current-output-port)
写一个字节到 out 。

procedure

(newline [out]) → void?

  out : output-port? = (current-output-port)
与 (write-char #\newline out) 相同。

procedure

(write-string str [out start-pos end-pos])

 → exact-nonnegative-integer?
  str : string?
  out : output-port? = (current-output-port)
  start-pos : exact-nonnegative-integer? = 0
  end-pos : exact-nonnegative-integer? = (string-length str)
从 str 中从索引 start-pos (包含)到 end-pos (不包含)将字符写入 out 。 和 substring 一样,如果 start-pos 或 end-pos 超出了 str 的范围, 将引发 exn:fail:contract exception is raised 。

结果是写入 out 的字符数,始终为 (- end-pos start-pos) 。

如果 str 是可变的, write-string 返回后的突变不会影响写入 out 的字符。 (这种不受突变影响的独立性并不是 write-string 的特殊属性,相反,输出函数通常都是这样。)

procedure

(write-bytes bstr [out start-pos end-pos])

 → exact-nonnegative-integer?
  bstr : bytes?
  out : output-port? = (current-output-port)
  start-pos : exact-nonnegative-integer? = 0
  end-pos : exact-nonnegative-integer? = (bytes-length bstr)
类似 write-string, 但写入字节而不是字符。

procedure

(write-bytes-avail bstr    
  [out    
  start-pos    
  end-pos]) → exact-nonnegative-integer?
  bstr : bytes?
  out : output-port? = (current-output-port)
  start-pos : exact-nonnegative-integer? = 0
  end-pos : exact-nonnegative-integer? = (bytes-length bstr)
类似于 write-bytes, 但在写入尽可能多的字节后返回,不阻塞,因为它可以立即刷新。 只有当没有字节可以立即刷新时,它才会阻塞。 结果是写入和刷新到 out 的字节数; 如果 start-pos 与 end-pos 相同,则结果可以为 0 (表示成功刷新任何缓冲数据), 否则结果在 1 到 (- end-pos start-pos) 之间,包括。

write-bytes-avail 过程从不丢弃字节; 如果 write-bytes-avail 成功地写入一些字节,然后遇到错误,它将抑制错误并返回写入的字节数。 (该错误将由未来的写入触发。)如果在写入任何字节之前遇到错误,则会引发异常。

procedure

(write-bytes-avail* bstr 
  [out 
  start-pos 
  end-pos]) 
 → (or/c exact-nonnegative-integer? #f)
  bstr : bytes?
  out : output-port? = (current-output-port)
  start-pos : exact-nonnegative-integer? = 0
  end-pos : exact-nonnegative-integer? = (bytes-length bstr)
类似 write-bytes-avail ,但从不阻塞,如果端口包含不能立即写入的缓冲数据,则返回 #f ; 如果端口的内部缓冲区(如果有)被刷新,但不能立即写入额外的字节,则返 0 。

procedure

(write-bytes-avail/enable-break bstr 
  [out 
  start-pos 
  end-pos]) 
 → exact-nonnegative-integer?
  bstr : bytes?
  out : output-port? = (current-output-port)
  start-pos : exact-nonnegative-integer? = 0
  end-pos : exact-nonnegative-integer? = (bytes-length bstr)
类似 write-bytes-avail, 只不过在写入过程中启用了中断。 该过程提供了关于写入和 exn:break 交互的保证: 如果在调用 write-bytes-avail/enable-break 时禁用了breaking, 并且如果调用的结果引发了 exn:break 异常,那么没有字节将被写入 out 。 参见 中断 。

procedure

(write-special v [out]) → boolean?

  v : any/c
  out : output-port? = (current-output-port)
如果端口支持特殊写,则直接将 v 写入 out ;如果端口不支持特殊写,则抛出 exn:fail:contract 。 结果始终为 #t,表示写入成功。

procedure

(write-special-avail* v [out]) → boolean?

  v : any/c
  out : output-port? = (current-output-port)
类似 write-special, 但没有阻塞。 如果 v 不能立即写入,则结果为 #f 而不写入 v, 否则结果为 #t 而写入 v 。

procedure

(write-bytes-avail-evt bstr    
  [out    
  start-pos    
  end-pos]) → evt?
  bstr : bytes?
  out : output-port? = (current-output-port)
  start-pos : exact-nonnegative-integer? = 0
  end-pos : exact-nonnegative-integer? = (bytes-length bstr)
类似于 write-bytes-avail, 但它不是立即写入字节,而是返回一个可同步的事件(参见 事件)。 out 必须支持原子写,如 port-writes-atomic? 所示。

对象上的同步从 bstr 开始写入,当字节写入(未缓冲)到端口时,事件就绪。 如果 start-pos 和 end-pos 相同,则当端口的内部缓冲区(如果有)被刷新时,同步结果为 0 , 否则结果为正的精确整数。 如果在同步中没有选择该事件,那么没有字节将被写入 out 。

procedure

(write-special-evt v [out]) → evt?

  v : any/c
  out : output-port? = (current-output-port)
类似于 write-special,但它不是立即写入特殊值,而是返回一个可同步的事件(参见 事件)。 out 必须支持原子写,如 port-writes-atomic? 所示。

对象上的同步开始写入特殊值,当该值被写入端口时(未缓冲),该事件就准备好了。 如果在同步中没有选择事件,则没有值被写入 out 。

procedure

(port-writes-atomic? out) → boolean?

  out : output-port?
如果 write-bytes-avail/enable-break 可以为 out 提供一个排他或保证(break 或 write,但不是两者), 并且如果端口可以与 write-bytes-avail-evt 这样的过程一起使用,则返回 #t 。 球拍的文件流端口,管道,字符串端口和 TCP 端口都支持原子写; 使用 make-output-port (参见 Custom Ports) 创建的端口可能支持原子写入。

procedure

(port-writes-special? out) → boolean?

  out : output-port?
如果 write-special 等过程可以向端口写入任意值,则返回 #t 。 球拍的文件流端口、管道端口、字符串端口和 TCP 端口都拒绝特殊值,但是使用 make-output-port (参见 Custom Ports)创建的端口可能支持这些特殊值。
