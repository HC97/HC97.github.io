14.6 Security Guards

procedure

(security-guard? v) → boolean?

  v : any/c
如果 v 是一个由 make-security-guard 创建的安全防护值, 则返回 #t ,否则返回 #f 。

security guard 提供了一组访问检查程序,当线程通过一个原始过程启动对文件、目录或网络连接的访问时,将被调用。 例如,当一个线程调用 open-input-file 时,线程当前的安全防护被查询,以检查该线程是否被允许对该文件进行读取访问。 如果允许访问,线程就会收到一个它可以无限期使用的端口,而不管安全防护的变化如何(尽管该端口的监护人可以关闭该端口; 见 Custodians) 。

一个线程的当前安全防护是由 current-security-guard 参数决定的。 每个安全卫士都有一个父级,每当子级的访问程序被调用时,就会调用父级的访问程序。 因此,一个线程不能通过安装一个新的安全卫士来任意增加自己的访问。 初始安全卫士不执行任何访问限制,除了主机平台执行的限制。

procedure

(make-security-guard parent    
  file-guard    
  network-guard    
  [link-guard]) → security-guard?
  parent : security-guard?
  file-guard : 
(symbol?
 (or/c path? #f)
 (listof symbol?)
 . -> . any)
  network-guard : 
(symbol?
 (or/c (and/c string? immutable?) #f)
 (or/c (integer-in 1 65535) #f)
 (or/c 'server 'client)
 . -> . any)
  link-guard : (or/c (symbol? path? path? . -> . any) #f) = #f
创建一个新的安全防护,作为 parent 的子级。

file-guard 程序必须接受三个参数:

  • 一个触发访问检查的原始过程的符号,这对引发拒绝访问的异常是有用的。

  • 一个路径(见 Paths) 或 #f 用于无路径查询,如 (current-directory), (filesystem-root-list) 和 (find-system-path symbol)。 提供给 file-guard 的路径在检查访问之前不会被扩展或其他方式规范化;例如,它可能是一个相对路径。

  • 一个包含一个或多个下列符号的列表:

    • 'read — 读取一个文件或目录

    • 'write — 修改或创建一个文件或目录

    • 'execute — 执行一个文件

    • 'delete — 删除一个文件或目录

    • 'exists — 确定一个文件或目录是否存在,或确定一个路径字符串是否格式良好

    在 file-guard 的最后一个参数中, 'exists 符号从不与其他符号结合,但任何其他组合都是可能的。 当 file-guard 的第二个参数是 #f 时,最后一个参数总是只包含 'exists 。

network-guard 过程必须接受四个参数:

  • 一个触发访问检查的原始操作的符号,这对于引发拒绝访问的异常很有用。

  • 一个不可变的字符串,代表客户端连接的目标主机名或监听服务器的接受主机名; #f 代表监听服务器或 UDP 套接字,接受主机所有地址的连接;或者 #f 代表一个未绑定的 UDP 套接字。

  • 一个在 1 和 65535 (包括)之间的精确整数,代表端口号, 或者 #f 代表一个未绑定的 UDP 套接字。 在客户端连接的情况下,端口号是服务器上的目标端口。 对于一个监听服务器,端口号是本地端口号。

  • 符号,可以是 'client 或 'server, 表明该检查是为了创建一个客户连接还是一个监听服务器。 打开一个未绑定的 UDP 套接字被认定为 'client 连接;明确绑定套接字被认定为 'server 动作。

link-guard 参数可以是 #f 或三个参数的过程:

  • 触发访问检查的原始过程的符号,这对于引发拒绝访问的异常很有用。

  • 一个完整的路径(见 Paths) ,代表要创建为链接的文件。

  • 代表链接内容的路径,它可能是相对于第二个参数的路径;这个路径在检查访问之前不会被扩展或以其他方式规范化。

如果 link-guard 是 #f, 那么将使用一个默认的程序,它总是引发 exn:fail 。

file-guard, network-guard 或 link-guard 的返回值被忽略。 要拒绝访问,该过程必须引发一个异常或以其他方式从原始调用的上下文中逃脱。 如果过程返回,则在相同的输入上调用父级的相应过程,如此循环,直到安全防护链。

file-guard, network-guard 和 link-guard 程序会在调用访问检查基元的线程中被调用。 可以启用也可以不启用断点(见 中断)。 进入或离开 file-guard 或 network-guard 调用的全续跳转被阻止 (见 提示、限定的延续和障碍物) 。

parameter

(current-security-guard) → security-guard?

(current-security-guard guard) → void?
  guard : security-guard?
一个 parameter ,决定了当前控制文件系统和网络访问的安全防护。
