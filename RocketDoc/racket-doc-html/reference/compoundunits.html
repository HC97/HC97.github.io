7.3 链接单元和创建复合单元

syntax

(compound-unit
  (import link-binding ...)
  (export tagged-link-id ...)
  (link linkage-decl ...))
 
link-binding = (link-id : tagged-sig-id)
     
tagged-link-id = (tag id link-id)
  | link-id
     
linkage-decl = ((link-binding ...) unit-expr tagged-link-id ...)
将几个单元链接成一个新的复合单元,而不立即调用任何被链接的单元。 link 子句中的 unit-expr 决定了在创建复合单元时要链接的单元。 unit-expr 在评估 compound-unit 形式时被评估。

import 子句决定了复合单元的导入。 在复合单元外部,这些导入的行为与普通单元一样;在复合单元内部,它们被传播到一些链接的单元。 export 子句决定了复合单元的出口。 同样,在复合单元外部,这些出口的处理方式与普通单元相同;在复合单元内部,它们从链接单元的出口中提取。 最后, link 子句中每个声明的左手和右手部分指定了复合单元的导入和导出是如何传播到链接单元的。

进口或出口的签名的单个元素在复合单元内是不可用的。 相反,进口和出口是在整个签名的层面上连接的。 每个具体的进口或出口(即某个签名的实例,可能有标签)被赋予一个 link-id 名称。 具体来说,一个 link-id 被 import 子句或 link 子句中的声明的左边部分所绑定。 被绑定的 link-id 在 link 子句中的声明的右边部分或 export 子句中被引用。

link 声明的左侧给出了由相应的 unit-expr 产生的单元的每个预期输出的名称。 实际的单元可以导出额外的签名,它可以导出特定签名的扩展,而不仅仅是指定的签名。 如果单元没有输出指定的签名之一(如果有的话,加上指定的标签), 那么在评估 compound-unit 形式时,会引发 exn:fail:contract exception is raised 。

link 声明的右侧指定了要提供给由相应的 unit-expr 产生的单元的进口。 实际的单元可以导入较少的签名,也可以导入由指定签名扩展的签名。 如果单元导入的签名(有特定的标签)不包括在所提供的导入中,当 compound-unit 形式被评估时, 会引发 exn:fail:contract exception is raised 。 每个作为导入提供的 link-id 必须被绑定在 import 子句或 link 子句中的某个声明中。

link 子句中的声明顺序决定了链接单元的调用顺序。 当复合单元被调用时,第一个 unit-expr 产生的单元首先被调用,然后是第二个,以此类推。 如果 link 子句中指定的顺序与实际单元的 init-depend 声明不一致, 那么当 compound-unit 形式被评估时,将引发 exn:fail:contract exception is raised 。
