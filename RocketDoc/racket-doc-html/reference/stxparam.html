12.5 语法参数

 (require racket/stxparam) package: base
The bindings documented in this section are provided by the racket/stxparam library, not racket/base or racket.

syntax

(define-syntax-parameter id expr)

将 id 作为语法绑定到一个 syntax parameter 上。 expr 是 transformer environment 中的一个表达式, 作为 syntax parameter 的默认值。 这个值通常由转化器使用 syntax-parameter-value 获得。

id 可以与 syntax-parameterize 或 syntax-parameter-value (在一个转化器中)一起使用。 如果 expr 产生一个只有一个参数的过程或者一个 make-set!-transformer 的结果, 那么 id 可以作为一个宏使用。 如果 expr 产生一个 make-rename-transformer 的结果, 那么 id 可以作为一个宏来使用,扩展到目标标识符的使用, 但是 id 的 syntax-local-value 不会产生目标的值。

Examples:
> (define-syntax-parameter current-class #f)
> (define-syntax-parameter yield (make-rename-transformer #'abort))
> (define-syntax-parameter define/public
    (λ (stx)
      (raise-syntax-error #f "use of a class keyword not in a class" stx)))
> (begin-for-syntax (displayln (syntax-parameter-value #'current-class)))

#f

> (yield 5)

5

syntax

(syntax-parameterize ([id expr] ...) body-expr ...+)

参见 splicing-syntax-parameterize 。

每个 id 必须使用 define-syntax-parameter 绑定到一个 syntax parameter 。 每个 expr 是 transformer environment 中的一个表达式。 在 body-expr 的扩展过程中,每个 expr 的值被绑定到相应的 id 上。

如果一个 expr 产生一个参数的过程或者一个 make-set!-transformer 的结果, 那么它的 id 可以在 body-expr 的扩展过程中作为一个宏使用。 如果 expr 产生一个 make-rename-transformer 的结果, 那么 id 可以作为一个宏,扩展到目标标识符的使用, 但是 id 的 syntax-local-value 不会产生目标的值。

Examples:
> (define-syntax-parameter abort (syntax-rules ()))
> (define-syntax forever
    (syntax-rules ()
      [(forever body ...)
       (call/cc (lambda (abort-k)
         (syntax-parameterize
             ([abort (syntax-rules () [(_) (abort-k)])])
           (let loop () body ... (loop)))))]))
> (define-syntax-parameter it (syntax-rules ()))
> (define-syntax aif
    (syntax-rules ()
      [(aif test then else)
       (let ([t test])
         (syntax-parameterize ([it (syntax-id-rules () [_ t])])
           (if t then else)))]))

syntax

(define-rename-transformer-parameter id expr)

将 id 作为语法绑定到必须与 make-rename-transformer 结果绑定的 syntax parameter 上, 与 define-syntax-parameter 不同的是, id 的 syntax-local-value 确实 会产生目标值, 包括在 syntax-parameterize 中。

Examples:
> (define-syntax (test stx)
   (syntax-case stx ()
     [(_ t)
      #`#,(syntax-local-value #'t)]))
> (define-syntax one 1)
> (define-syntax two 2)
> (define-syntax-parameter not-num
    (make-rename-transformer #'one))
> (test not-num)

#<procedure:syntax-parameter>

> (define-rename-transformer-parameter num
    (make-rename-transformer #'one))
> (test num)

1

> (syntax-parameterize ([num (make-rename-transformer #'two)])
    (test num))

2

Added in version 6.3.0.14 of package base.

12.5.1 语法参数检查

 (require racket/stxparam-exptime) package: base

procedure

(syntax-parameter-value id-stx) → any

  id-stx : syntax?
这个过程是为了在 transformer environment 中使用, 其中 id-stx 是一个在正常环境中与 syntax parameter 绑定的标识符。 其结果是 syntax parameter 的当前值,由 syntax-parameterize 形式调整。

这个绑定是由 racket/stxparam 提供的 for-syntax ,因为它通常是在转化器中使用。 它通常由 racket/stxparam-exptime 提供。

procedure

(make-parameter-rename-transformer id-stx) → any

  id-stx : syntax?
这个过程是为了在转化器中使用,其中 id-stx 是一个与 syntax parameter 绑定的标识符。 其结果是一个表现为 id-stx 的转化器, 但它不能与 syntax-parameterize 或 syntax-parameter-value 一起使用。

使用 make-parameter-rename-transformer 类似于定义了一个调用参数的过程。 这样的过程可以输出给其他人,以允许访问参数值,但不能改变参数值。 类似地, make-parameter-rename-transformer 允许将一个 syntax parameter 作为一个宏使用,但不能改变。

与 make-parameter-rename-transformer 的结果不同, make-rename-transformer 的结果不会被 syntax-local-value 特别处理。

这个绑定是由 racket/stxparam 提供的 for-syntax ,因为它通常被用在转换器中。 它通常由 racket/stxparam-exptime 提供。
