12.10 Serializing Syntax

procedure

(syntax-serialize 
  stx 
  #:preserve-property-keys preserve-property-keys 
  [#:provides-namespace provides-namespace 
  #:base-module-path-index base-module-path-index]) 
 → any/c
  stx : syntax?
  preserve-property-keys : (listof symbol?)
  provides-namespace : (or/c namespace? #f)
   = (current-namespace)
  base-module-path-index : (or/c module-path-index? #f) = #f
将 stx 转换为适合使用 s-exp->fasl 或 serialize 的序列化形式。 尽管 stx 可以用 (compile `(quote-syntax ,stx)) 进行序列化, 然后写入编译后的形式,但 syntax-serialize 提供了对序列化的更多控制。:

  • preserve-property-keys 列出了在序列化中应该保留其值的语法-属性键, 即使该属性值没有被添加到 syntax-property 中作为保留值(因此它在编译后会被丢弃)。 与要保存的属性相关的值必须是可序列化的,符合 syntax-property 对被保存的属性所要求的意义。

  • 参数 provides-namespace 限制了序列化的语法对象在多大程度上可以依赖 bulk bindings ,这是出口模块提供的共享绑定表。 如果 provides-namespace 是 #f , 那么完整的绑定信息将被记录在语法对象的序列化形式中,并且在反序列化时将不需要来自命名空间的批量绑定。 否则,批量绑定将只用于在 provides-namespace 中声明的模块 (即,反序列化时的命名空间将有与 provides-namespace 相同的模块声明); 注意,提供一个没有模块绑定的命名空间等同于提供 #f 。

  • base-module-path-index 参数指定了一个 module path index ， stx 中的绑定信息是相对的。 例如,如果一个语法对象来自模块主体中的 quote-syntax ， 那么 base-module-path-index 可以有效地成为包围模块的模块路径索引, 由模块中的 (variable-reference->module-path-index (#%variable-reference)) 产生。 在反序列化时,可以提供一个不同的模块路径索引来代替 base-module-path-index, 它将任何相对于序列化时模块身份的绑定转移到相对于反序列化时提供的模块身份。 如果 base-module-path-index 是 #f, 那么在反序列化时不支持移位,并且当时提供的任何 base-module-path-index 被忽略。

一个序列化的语法对象在其他方面与编译的代码相似:它是特定版本的, 反序列化将需要一个足够强大的 code inspector 。

Added in version 8.0.0.13 of package base.

procedure

(syntax-deserialize 
  v 
  [#:base-module-path-index base-module-path-index]) 
 → syntax?
  v : any/c
  base-module-path-index : (or/c module-path-index? #f) = #f
将 syntax-serialize 的结果转换为一个语法对象。 更多信息请参见 syntax-serialize 。

Added in version 8.0.0.13 of package base.
