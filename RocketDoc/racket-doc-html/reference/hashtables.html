4.15 Hash Tables

散列表（Hash Table） in Racket 指南 introduces hash tables.

一个 hash table (或简称 hash) 将其每个键映射到一个单一的值。 对于一个给定的哈希表,键通过 equal?, equal-always?, eqv? 或 eq? ,并且键被强保留、弱保留(参见 弱盒) ,或像 ephemerons 一样保留。 哈希表也可以是可变的或不可变的。 不可变的哈希表支持有效的恒定时间访问和更新,就像可变的哈希表一样; 不可变的操作的常数通常更大,但是不可变的哈希表的功能性质在某些算法中可以得到回报。 使用 immutable? 来检查一个哈希表是否是不可变的。

不可变的哈希表实际上提供了 O(log N) 的访问和更新。 由于 N 受到地址空间的限制,所以 log N 被限制在 30 或 62 以下(取决于平台), log N 可以被合理地视为一个常数。

对于基于 equal? 的散列, strings, pairs, lists, vectors, prefab 或透明 structures 等的内置散列函数所花费的时间与值的大小成正比。 复合数据结构(如列表或向量)的哈希代码取决于对容器的每一项进行哈希, 但这种递归哈希的深度是有限的(以避免循环数据的潜在问题)。 对于非list pair 来说, car 和 cdr 的散列都被视为更深的散列,但是 list 的 cdr 被视为与列表的散列深度相同。

一个哈希表可以作为一个双值序列使用(参见 Sequences) 。 散列表的键和值作为序列的元素(即,每个元素是一个键和它相关的值)。 如果在迭代过程中,映射被添加到哈希表中或从哈希表中移除,那么迭代步骤可能会以 exn:fail:contract 的方式失败,或者迭代会跳过或重复键和值。 参见 in-hash, in-hash-keys, in-hash-values 和 in-hash-pairs 。

两个哈希表不能 equal? ,除非它们具有相同的可变性,使用相同的键比较过程 (equal?, equal-always?, eqv? 或 eq?), 都是强、弱、或像 ephemerons 一样持有键。 空的不可变的哈希表是 eq? ,当它们是 equal? 。

Changed in version 7.2.0.9 of package base: 使得空的不可变的哈希表在 equal? 时变成 eq? 。

关于并发修改的注意事项: 一个可变的哈希表可以被多个线程用 hash-ref, hash-set! 和 hash-remove! 并发地操作,并且这些操作在需要时由表的特定信号保护。 然而,有几项注意事项需要注意:

  • 如果一个线程在应用 hash-ref, hash-ref-key, hash-set!, hash-remove!, hash-ref!, hash-update! 或 hash-clear! 到一个使用 equal?, equal-always? 或 eqv? 键比较的哈希表时被终止,所有对该哈希表的当前和未来操作都可能被无限期地阻塞。

  • hash-map, hash-for-each 和 hash-clear! 程序不使用表的信号来保护整个遍历(如果需要遍历,在 hash-clear! 的情况下)。 一个线程对哈希表的改变会影响到另一个线程在遍历同一个哈希表的过程中所看到的键和值。

  • hash-update! 和 hash-ref! 函数在其功能的 hash-ref 和 hash-set! 部分独立地使用表的信号,这意味着整体的更新不是 "原子的"。

  • 添加一个可变的哈希表作为一个键本身就很麻烦,理由是键正在被变异(见下面的注意事项), 但这也是对哈希表的一种并发使用:计算哈希表的哈希代码可能需要等待表的信号灯, 但信号灯已经被持有用于修改哈希表,所以哈希表的添加可以无限期地阻塞。

关于可变键的注意事项: 如果一个基于 equal? 的哈希表中的键被改变了 (例如,用 string-set! 修改了一个键字符串), 那么哈希表的插入和查找操作的行为就变得不可预测了。

字面或打印的哈希表以 #hash, #hashalw, #hasheqv 和 #hasheq 开始。 See 读哈希表 for information on reading hash tables and 打印哈希表 for information on printing hash tables.

procedure

(hash? v) → boolean?

  v : any/c
如果 v 是一个 hash table,返回 #t ,否则返回 #f 。

参见 immutable-hash? 和 mutable-hash? 。

Added in version 8.5.0.3 of package base.

(hasheqv key val ... ...)

 → (and/c hash? hash-eqv? immutable? hash-strong?)
  key : any/c

创建一个不可变的哈希表,每个给定的 key 映射到下面的 val; 每个 key 必须有一个 val ,所以 hash 的参数总数必须是偶数。

hash 过程创建了一个键与 equal? 的比较表, hashalw 过程创建了一个键与 equal-always? 的比较表, hasheq 过程创建了一个键与 eq? 的比较表, hasheqv 过程创建了一个键与 eqv? 的比较表。

key 到 val 的映射是按照它们在参数列表中出现的顺序添加到表中的, 所以如果 key 相等,后面的映射可以隐藏前面的映射。

(make-hashalw [assocs])

(make-hasheqv [assocs])

 → (and/c hash? hash-eqv? (not/c immutable?) hash-strong?)
  assocs : (listof pair?) = null

procedure

(make-hasheq [assocs])

 → (and/c hash? hash-eq? (not/c immutable?) hash-strong?)
  assocs : (listof pair?) = null
创建一个可变的哈希表,强行持有键。

make-hash 过程创建了一个用 equal? 比较键的表, make-hasheq 过程创建了一个用 eq? 比较键的表, make-hasheqv 过程创建了一个用 eqv? 比较键的表, make-hashalw 创建了一个用 equal-always? 比较键的表。

该表以 assocs 的内容进行初始化。 在 assocs 的每个元素中, car 是一个键, cdr 是相应的值。 映射是按照它们在 assocs 中出现的顺序添加到表中的,所以后来的映射可以隐藏早期的映射。

另请参见 make-custom-hash 。

请注意,弱哈希表中的值是正常保留的。 如果表中的一个值指向它的键,那么该表将保留该值,从而保留该键;即使该键变得不可访问,该映射也不会从表中删除。 为了避免这个问题,请使用由 make-ephemeron-hash, make-ephemeron-hashalw, make-ephemeron-hasheqv 或 make-ephemeron-hasheq 创建的 ephemeron 哈希表。 对于不指向键的值来说,使用 ephemeron 哈希表而不是弱哈希表有一定的额外成本, 但在有疑问的情况下,最好使用 ephemeron 哈希表。

procedure

(hash-ref hash key [failure-result]) → any

  hash : hash?
  key : any/c
  failure-result : failure-result/c
   = 
(lambda ()
  (raise (make-exn:fail:contract ....)))
返回 hash 中 key 的值。 如果没有找到 key 的值,那么 failure-result 就决定了结果:

  • 如果 failure-result 是一个过程,它被调用(通过尾部调用),没有参数,产生结果。

  • 否则, failure-result 被作为结果返回。

Examples:
> (hash-ref (hash) "hi")

hash-ref: no value found for key

  key: "hi"

> (hash-ref (hash) "hi" 5)

5

> (hash-ref (hash) "hi" (lambda () "flab"))

"flab"

> (hash-ref (hash "hi" "bye") "hi")

"bye"

> (hash-ref (hash "hi" "bye") "no")

hash-ref: no value found for key

  key: "no"

See also the caveats concerning concurrent modification and the caveat concerning mutable keys above.

procedure

(hash-ref-key hash key [failure-result]) → any

  hash : hash?
  key : any/c
  failure-result : failure-result/c
   = 
(lambda ()
  (raise (make-exn:fail:contract ....)))
根据 hash 的密钥比较功能,返回 hash 持有的与 key 相当的密钥。 如果没有找到钥匙,那么就像在 hash-ref 中一样,使用 failure-result 来确定结果。

如果 hash 不是一个 impersonator ,那么返回的密钥,假设它被找到, 将与 hash 实际保留的密钥 eq?-equivalent :

Examples:
> (define original-key "hello")
> (define key-copy (string-copy original-key))
> (equal? original-key key-copy)

#t

> (eq? original-key key-copy)

#f

> (define table (make-hash))
> (hash-set! table original-key 'value)
> (eq? (hash-ref-key table "hello") original-key)

#t

> (eq? (hash-ref-key table "hello") key-copy)

#f

如果一个易变的哈希被多次更新,使用的键不是 eq? ,但根据哈希的键比较程序是等价的,那么哈希会保留第一个键:

Examples:
> (define original-key "hello")
> (define key-copy (string-copy original-key))
> (define table (make-hash))
> (hash-set! table original-key 'one)
> (hash-set! table key-copy 'two)
> (eq? (hash-ref-key table "hello") original-key)

#t

> (eq? (hash-ref-key table "hello") key-copy)

#f

相反,一个不可变的散列保留了最近用于更新它的钥匙:

Examples:
> (define original-key "hello")
> (define key-copy (string-copy original-key))
> (define table0 (hash))
> (define table1 (hash-set table0 original-key 'one))
> (define table2 (hash-set table1 key-copy 'two))
> (eq? (hash-ref-key table2 "hello") original-key)

#f

> (eq? (hash-ref-key table2 "hello") key-copy)

#t

如果 hash 是一个 impersonator, 那么返回的密钥将按照 impersonate-hash 的文档中的描述来确定。

See also the caveats concerning concurrent modification and the caveat concerning mutable keys above.

Added in version 7.4.0.3 of package base.

procedure

(hash-update! hash    
  key    
  updater    
  [failure-result]) → void?
  hash : (and/c hash? (not/c immutable?))
  key : any/c
  updater : (any/c . -> . any/c)
  failure-result : failure-result/c
   = 
(lambda ()
  (raise (make-exn:fail:contract ....)))
通过对值应用 updater 来更新 hash 中 key 所映射的值。 由 updater 返回的值成为 key 的新映射,覆盖了 hash 中的原始值。

Examples:
(define h (make-hash))
(hash-set! h 'a 5)

 

> (hash-update! h 'a add1)
> h

'#hash((a . 6))

可选的 failure-result 参数在 key 已经不存在映射的情况下使用, 方式与 hash-ref 相同。

Examples:
(define h (make-hash))

 

> (hash-update! h 'b add1)

hash-update!: no value found for key: 'b

procedure

(hash-map hash proc [try-order?]) → (listof any/c)

  hash : hash?
  proc : (any/c any/c . -> . any/c)
  try-order? : any/c = #f
将过程 proc 以非指定的顺序应用于 hash 中的每个元素,将结果累积为一个列表。 每次调用过程 proc 时都有一个键和它的值,过程的各个结果会按顺序出现在结果列表中。

如果一个哈希表在 hash-map 或 hash-for-each 的遍历过程中被扩展了新的键(通过 proc 或其他线程),任意的键值对可以在遍历中被放弃或重复。 键值映射可以被删除或重新映射(由任何线程进行),而不会产生不利影响; 如果键值已经被看到,则这种改变不会影响遍历,否则,遍历会跳过被删除的键值或使用重新映射的键值。

See also the caveats concerning concurrent modification above.

如果 try-order? 为真,那么在某些情况下,传递给 proc 的键和值的顺序会被规范化, 包括每个键都是下列之一,并且有下列顺序(前面的子弹在后面):

  • booleans 排序为 #f 在 #t 之前;

  • characters sorted by char<?;

  • real numbers sorted by <;

  • symbols sorted with uninterned symbols before unreadable symbols before interned symbols, then sorted by symbol<?;

  • keywords sorted by keyword<?;

  • strings sorted by string<?;

  • byte strings sorted by bytes<?;

  • null;

  • #<void>; and

  • eof.

Changed in version 6.3 of package base: Added the try-order? argument.
Changed in version 7.1.0.7: Added guarantees for try-order?.

                 (lambda (k v) (values k (string-upcase v))))

> (define frozen-capital
    (hash-map/copy (make-hash '((a . "apple") (b . "banana")))

                   #:kind 'immutable))
> frozen-capital

'#hash((a . "APPLE") (b . "BANANA"))

> (immutable? frozen-capital)

procedure

(hash-iterate-first hash)

 → (or/c #f exact-nonnegative-integer?)
  hash : hash?
如果 hash 不包含任何元素,则返回 #f ,否则返回一个整数, 该整数是 hash 表中第一个元素的索引;"第一" 指的是一个未指定的表元素排序,并且索引值不一定是连续的整数。

4.15.1 额外的哈希表函数

 (require racket/hash) package: base
The bindings documented in this section are provided by the racket/hash library, not racket/base or racket.

procedure

(hash-union h0 
  h ... 
  [#:combine combine 
  #:combine/key combine/key]) 
 → (and/c hash? immutable?)
  h0 : (and/c hash? immutable?)
  h : hash?
  combine : (-> any/c any/c any/c)
   = (lambda _ (error 'hash-union ....))
  combine/key : (-> any/c any/c any/c any/c)
   = (lambda (k a b) (combine a b))
通过函数更新计算 h0 与每个哈希表 h 的联合,依次将每个 h 的每个元素加入 h0 。 对于每个键 k 和值 v ,如果从 k 到某个值 v0 的映射已经存在, 它将被替换为从 k 到 (combine/key k v0 v) 的映射。

Examples:
> (hash-union (make-immutable-hash '([1 . one]))
              (make-immutable-hash '([2 . two]))
              (make-immutable-hash '([3 . three])))

'#hash((1 . one) (2 . two) (3 . three))

> (hash-union (make-immutable-hash '([1    one uno]  [2    two dos]))
              (make-immutable-hash '([1    eins un]  [2    zwei deux]))
              #:combine/key (lambda (k v1 v2) (append v1 v2)))

'#hash((1 . (one uno eins un)) (2 . (two dos zwei deux)))

procedure

(hash-intersect h0 
  h ... 
  [#:combine combine 
  #:combine/key combine/key]) 
 → (and/c hash? immutable?)
  h0 : (and/c hash? immutable?)
  h : hash?
  combine : (-> any/c any/c any/c)
   = (lambda _ (error 'hash-intersect ...))
  combine/key : (-> any/c any/c any/c any/c)
   = (lambda (k a b) (combine a b))
构建哈希表,该表是 h0 与每个哈希表 h 的交集。 在得到的哈希表中,一个键 k 被映射到 k 在每个哈希表中被映射到的值的组合。 最终的值是通过应用 (combine/key k v vi) 或 (combine v vi) 逐步组合出现在每个哈希表中的值来计算的,其中 vi 是 k 在第 i 个哈希表 h 中被映射到的值， v 是前面步骤中值的累积。 第一个参数的比较谓词 (eq?, eqv?, equal-always?, equal?) 决定了结果的那个。

Examples:
> (hash-intersect (make-immutable-hash '((a . 1) (b . 2) (c . 3)))
                  (make-immutable-hash '((a . 4) (b . 5)))
                  #:combine +)

'#hash((a . 5) (b . 7))

> (hash-intersect (make-immutable-hash '((a . 1) (b . 2) (c . 3)))
                  (make-immutable-hash '((a . 4) (b . 5)))
                  #:combine/key
                  (lambda (k v1 v2) (if (eq? k 'a) (+ v1 v2) (- v1 v2))))

'#hash((a . 5) (b . -3))
