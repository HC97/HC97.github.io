6 Classes and Objects

类和对象 in Racket 指南 introduces classes and objects.

 (require racket/class) package: base
The bindings documented in this section are provided by the racket/class and racket libraries, but not racket/base.

一个 class 指定了

  • 一个字段的集合;

  • 一个方法的集合;

  • 字段的初始值表达式;以及

  • 与初始化参数绑定的初始化变量。

在类系统的背景下,一个 object 是一个字段绑定的集合,这些字段根据类的描述被实例化。

类系统允许程序使用继承、覆盖和增强的方式在现有类(superclass)的基础上定义一个新的类 (derived class) :

  • inheritance: 一个派生类的对象支持派生类的超类所声明的方法和实例化的字段, 以及派生类表达式中声明的方法和字段。

  • overriding: 在超类中声明的一些方法可以在派生类中被替换。 对超类中被覆盖的方法的引用使用派生类中的实现。

  • augmenting: 一些在超类中声明的方法可以在派生类中仅仅被扩展。 超类的方法特别委托给派生类中的增强方法。

一个 interface 是要由一个类来实现的方法名称的集合,与派生要求相结合。 一个类 implements 一个接口,当它

  • 为接口中的每个变量声明(或继承)一个公共方法;

  • 派生自该接口所要求的类,如果有的话;并且

  • 特别声明它打算实现该接口。

一个类可以实现任意数量的接口。 一个派生类会自动实现其超类所实现的任何接口。 每个类还实现一个与该类相关的隐式定义的接口。 隐式定义的接口包含了该类的所有公共方法名称,它要求所有其他接口的实现都是从该类派生出来的。

一个新的接口可以用额外的方法名 extend 一个或多个接口;每个实现了扩展接口的类也实现了原始接口。 原始接口的派生要求必须是一致的,扩展接口继承了原始接口中最具体的派生要求。

类、对象和接口都是价值。 然而,一个类或接口不是一个对象(即,没有 "元类" 或 "元接口")。
