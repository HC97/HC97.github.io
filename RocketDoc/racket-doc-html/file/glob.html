15 全球通用

 (require file/glob) package: base

file/glob 库实现了对 path-string? 值的 globbing 。 glob 是一个路径字符串, 它使用下列 wildcards 来匹配一组路径字符串:

  • 一个六分位 (*) 匹配文件或目录名称中的任何字符序列。

  • 两个六边形 (**) 匹配任何字符序列和任何数量的路径分隔符。

  • 一个问号 (?) 匹配文件或目录名称中的任何单个字符。

  • 方括号限定的字符组,例如 [abc], 匹配该组中的任何字符。 方括号在 globbs 中的含义与在正则表达式中的含义相同, 见 正则表达式语法 。

  • 如果 glob 以路径分隔符结尾(在任何 (system-type) 上为 / , 在 'windows 上为 \ ),那么它只匹配目录。

在 Windows 上,通配符不能被转义,因为 \ 是一个路径分隔符。

默认情况下,通配符不会匹配名称以句点开头的文件或目录(又称 "dotfiles")。 如果要覆盖,请将参数 glob-capture-dotfiles? 设置为一个非 #f 值, 或者使用 #:capture-dotfiles? 关键字提供一个类似的值。

value

glob/c : (or/c path-string? (sequence/c path-string?))

一个接受 glob 或 glob 序列的 flat contract 。

所有的 file/glob 函数都接受 glob/c 值。 这些函数也接受大括号 ({}) 作为描述多个 glob 的 meta-wildcard 。

大括号会在其他通配符 之前 被解释。

  • 以大括号为界,以逗号分隔的字符组,例如 {foo,bar}, 在 file/glob 模块开始匹配之前,扩展为多个 glob。 例如, glob/c 值 "{foo,bar}.rkt" 与 '("foo.rkt" "bar.rkt") 具有相同的含义。

procedure

(glob pattern 
  [#:capture-dotfiles? capture-dotfiles?]) 
 → (listof path-string?)
  pattern : glob/c
  capture-dotfiles? : boolean? = (glob-capture-dotfiles?)
构建一个当前文件系统中与 pattern 中的任何 glob 相匹配的所有路径的列表。 结果中的路径顺序是未指定的。

如果 pattern 包含通配符 **, 那么 glob 就会递归地搜索文件系统以找到匹配的路径。 例如, "/**.rkt" 将在 entire filesystem 中搜索后缀为 ".rkt" 的文件或目录(也就是 Racket 文件)。

示例:
> (glob "*.rkt")
;; Lists all Racket files in current directory
 
> (glob "*/*.rkt")
;; Lists all Racket files in all sub-directories of the current directory.
;; (Does not search sub-sub-directories, etc.)
 
> (glob (build-path (find-system-path 'home-dir) "**" "*.rkt"))
;; Recursively searches home directory for Racket files, lists all matches.
 
> (glob "??.rkt")
;; Lists all Racket files in current directory with 2-character names.
 
> (glob "[a-z0-9].rkt")
;; Lists all Racket files in current directory with single-character,
;; alphanumeric names.
 
> (glob '("foo-bar.rkt" "foo-baz.rkt" "qux-bar.rkt" "qux-baz.rkt"))
;; Filters the list to contain only files or directories that exist.
 
> (glob "{foo,qux}-{bar,baz}.rkt")
;; Same as above, returns at most 4 files.

procedure

(in-glob pattern 
  [#:capture-dotfiles? capture-dotfiles?]) 
 → (sequence/c path-string?)
  pattern : glob/c
  capture-dotfiles? : boolean? = (glob-capture-dotfiles?)
返回一个与 glob pattern 匹配的所有路径流,而不是急于建立一个列表。

procedure

(glob-match? pattern    
  path    
  [#:capture-dotfiles? capture-dotfiles?]) → boolean?
  pattern : glob/c
  path : path-string?
  capture-dotfiles? : boolean? = (glob-capture-dotfiles?)
类似于 regexp-match?; 如果 path 与 pattern 中的任何 glob 匹配,则返回 #true 。

(glob-match? pattern path) 与

(member path (glob pattern))

不 一样

因为 glob 只返回存在的文件/目录, 而 glob-match? 并不检查 path 是否存在。

这个操作访问了文件系统。

procedure

(glob-quote str) → string?

  str : string?
(glob-quote path) → path?
  path : path?
在给定的字符串或路径字符串中逃避所有 glob wildcards 和 glob meta-wildcards 。

Examples:
> (glob-quote "*.rkt")

"\\*.rkt"

> (glob-quote "[Ff]ile?{zip,tar.gz}")

"\\[Ff\\]ile\\?\\{zip\\,tar.gz\\}"

> (glob-quote "]")

"\\]"

parameter

(glob-capture-dotfiles?) → boolean?

(glob-capture-dotfiles? capture-dotfiles?) → void?
  capture-dotfiles? : boolean?
 = #f
确定通配符是否与以 #\. 字符开头的名称相匹配。 如果 #t, 通配符将匹配 dotfiles。 如果 #f, 则使用诸如 ".*" 这样的 glob 来明确匹配 dotfiles。
