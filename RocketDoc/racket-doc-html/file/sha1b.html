10 SHA1 信息摘要

 (require file/sha1) package: base

参见 openssl/sha1 的更快实现。

procedure

(sha1 in [start end]) → string?

  in : (or/c bytes? input-port?)
  start : exact-nonnegative-integer? = 0
  end : (or/c #f exact-nonnegative-integer?) = #f
返回一个 40 个字符的字符串,代表 in 中内容的 SHA-1 哈希值(十六进制符号)。 in, start 和 end 参数的处理与 racket/base 中的 sha1-bytes 相同。

sha1 函数将 bytes->hex-string 与 sha1-bytes 组合在一起。

Example:
> (sha1 (open-input-bytes #"abc"))

"a9993e364706816aba3e25717850c26c9cd0d89d"

Changed in version 7.0.0.5 of package base: 允许一个字节字符串作为 in , 并增加了 start 和 end 参数。

procedure

(sha1-bytes in [start end]) → bytes?

  in : (or/c bytes? input-port?)
  start : exact-nonnegative-integer? = 0
  end : (or/c #f exact-nonnegative-integer?) = #f
与 racket/base 中的 sha1-bytes 相同, 返回一个 20 字节的字符串,代表 in 中内容的 SHA-1 哈希值。

Example:
> (sha1-bytes (open-input-bytes #"abc"))

#"\251\231>6G\6\201j\272>%qxP\302l\234\320\330\235"

Changed in version 7.0.0.5 of package base: 允许一个与 in 一样的字节字符串, 并添加了 start 和 end 参数。

procedure

(bytes->hex-string bstr) → string?

  bstr : bytes?
将给定的字节字符串转换为字符串表示, 其中 bstr 中的每个字节在结果字符串中被转换为其两位数的十六进制表示。

Example:
> (bytes->hex-string #"turtles")

"747572746c6573"
