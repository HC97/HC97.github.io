1 可转换的: 数据转换协议

 (require file/convertible) package: base

file/convertible 库提供了一个协议,在不同格式的数据提供者和格式的消费者之间进行调解。 例如,一个实现了 prop:convertible 的数据类型可能可以 convertible 为 GIF 或 PDF 流, 在这种情况下,它将产生 'gif-bytes 或 'pdf-bytes 请求的数据。

任何符号都可以被用于转换请求,但下面这些应该被视为标准:

  • 'text — 一个字符串,表示人类可读的文本

  • 'gif-bytes — 一个包含 GIF 图像编码的字节字符串

  • 'png-bytes — 一个包含 PNG 图像编码的字节字符串

  • 'png-bytes+bounds — 一个包含一个字节串和四个数字的列表; 字节串包含一个 PNG 文件,四个数字是图像的尺寸信息: 宽度、高度、下限(包含在高度中)和额外的垂直顶部空间(包含在高度中),依次排列

  • 'png-bytes+bounds8 — 一个包含一个字节字符串和八个数字的列表; 就像 'png-bytes+bounds, 但是字节字符串中的图像编码可以在每个方向上进行填充(以允许绘制的区域超出其 "边界框"), 列表中额外的四个数字指定了添加到图像中的填充量:左、右、上和下

  • 'png@2x-bytes — 就像 'png-bytes 一样, 但是对于一个打算以 1/2 比例绘制的图像来说

  • 'png@2x-bytes+bounds — 就像 'png-bytes+bounds 一样,但用于以 1/2 比例绘制的图像, 其中结果列表中的数字已经被缩放了(例如,字节字符串编码的图像是结果列表中第一个数字宽度的两倍)。

  • 'png@2x-bytes+bounds8 — 像 'png-bytes+bounds8, 但是用于以 1/2 比例绘制的图像,并且结果列表中的数字已经被缩放。

  • 'svg-bytes — 一个包含 SVG 图像编码的字节字符串

  • 'svg-bytes+bounds — 像 'png-bytes+bounds,但用于 SVG 图像

  • 'svg-bytes+bounds8 — 像 'png-bytes+bounds8, 但适用于 SVG 图像。

  • 'ps-bytes — 一个包含 PostScript 文档的字节字符串

  • 'eps-bytes — 一个包含封装的 PostScript 文档的字节字符串

  • 'eps-bytes+bounds — 像 'png-bytes+bounds, 但是,对于封装的 PostScript 文档而言。

  • 'eps-bytes+bounds8 — 像 'png-bytes+bounds8, 但是,对于封装的 PostScript 文档来说,是一个封装的 PostScript 文档。

  • 'pdf-bytes — 一个包含 PDF 文档的字节字符串

  • 'pdf-bytes+bounds — 像 'png-bytes+bounds, 但是,是指一个 PDF 文档。

  • 'pdf-bytes+bounds8 — 像 'png-bytes+bounds8, 但是,对于一个 PDF 文档来说,是一个字节字符串。

(struct-type-property/c
 (->i ([v convertible?] [request symbol?] [default default/c])

一个属性,其值被 convert 调用。

该过程的 v 参数是结构, request 参数是请求转换的符号, default 参数是要返回的值(如果不支持转换,通常是 #f )。 该过程的结果取决于请求的转换,如上所述。

default/c 合同是由 new-∀/c 生成的。

procedure

(convertible? v) → boolean?

  v : any/c
如果 v 支持该转换协议,返回 #t ,否则返回 #f 。

请求从 v 中进行数据转换,其中 request 表示请求的数据类型, default 是转换器在不能产生 request 所示格式的数据时应该返回的值。

default/c 合约是由 new-∀/c 创建的, 它保证 convert 的结果是给定的默认参数(如果没有提供默认参数,则是 #f )。
