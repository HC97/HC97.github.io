4 zip 文件的创建

 (require file/zip) package: base
file/zip 库提供了创建 zip 档案文件的工具, 它与 Windows 和 Unix(包括 Mac OS)的解包都兼容。实际的压缩是由 deflate 实现的。

procedure

(zip zip-file    
  path ...    
  [#:timestamp timestamp    
  #:get-timestamp get-timestamp    
  #:utc-timestamps? utc-timestamps?    
  #:round-timestamps-down? round-timestamps-down?    
  #:path-prefix path-prefix    
  #:system-type sys-type]) → void?
  zip-file : path-string?
  path : path-string?
  timestamp : (or/c #f exact-integer?) = #f
  get-timestamp : (path? . -> . exact-integer?)
   = 
(if timestamp
    (lambda (p) timestamp)
    file-or-directory-modify-seconds)
  utc-timestamps? : any/c = #f
  round-timestamps-down? : any/c = #f
  path-prefix : (or/c #f path-string?) = #f
  sys-type : symbol? = (system-type)
创建 zip-file ,它持有所有 path 的完整内容。

给出的 path 都被认为是现有目录和文件的相对路径名(即相对于当前目录)。 如果一个嵌套的路径被作为 path 提供, 它的祖先目录也会被添加到产生的 zip 文件中,直到当前目录(使用 pathlist-closure) 。

文件像通常的 zip 文件一样被打包,包括 Windows和 Unix(包括 Mac OS)的权限位。 权限位由 file-or-directory-permissions 决定,它不保留所有者/组/其他权限之间的区别。 另外,符号链接总是被遵循。

get-timestamp 函数用来获取文件或目录的修改日期,以记录在存档中。 通常情况下, zip 存档记录的是本地时间的修改日期, 但如果 utc-timestamps? 为真,则记录的是 UTC 时间。 zip 档案中的时间戳只精确到两秒;默认情况下,时间是向未来四舍五入的(像 WinZip 或 PKZIP), 但如果 round-timestamps-down? 为真,时间会向过去四舍五入(像 Java)。

sys-type 参数决定了存档中记录的系统类型。

如果 path-prefix 不是 #f, 那么它就会在 zip 文件中写入每个路径的名称前缀, 并且为 path-prefix 的每个元素添加目录条目。

Changed in version 6.0.0.3 of package base: 增加了 #:get-timestamp 和 #:system-type 参数。
Changed in version 6.0.1.12: 增加了 #:path-prefix, #:utc-timestamps? 和 #:utc-timestamps-down? 参数。

procedure

(zip->output paths 
  [out 
  #:timestamp timestamp 
  #:get-timestamp get-timestamp 
  #:utc-timestamps? utc-timestamps? 
  #:round-timestamps-down? round-timestamps-down? 
  #:path-prefix path-prefix 
  #:system-type sys-type]) 
 → void?
  paths : (listof path-string?)
  out : output-port? = (current-output-port)
  timestamp : (or/c #f exact-integer?) = #f
  get-timestamp : (path? . -> . exact-integer?)
   = 
(if timestamp
    (lambda (p) timestamp)
    file-or-directory-modify-seconds)
  utc-timestamps? : any/c = #f
  round-timestamps-down? : any/c = #f
  path-prefix : (or/c #f path-string?) = #f
  sys-type : symbol? = (system-type)
对给定的每个 paths 进行压缩, 并将其打包成一个 zip “file” ,直接写到 out 。 与 zip 不同的是,指定的 paths 是在不关闭目录的情况下被包含的: 如果指定了一个目录,它的内容不会被自动添加,嵌套的目录被添加,没有父目录。

Changed in version 6.0.0.3 of package base: 增加了 #:get-timestamp 和 #:system-type 参数。
Changed in version 6.0.1.12: 增加了e #:path-prefix, #:utc-timestamps? 和 #:utc-timestamps-down? 参数。

parameter

(zip-verbose) → boolean?

(zip-verbose on?) → void?
  on? : any/c
一个在 zip 操作中控制输出的参数。 将这个参数设置为真值,会使 zip 显示到 (current-error-port) 当前正在压缩的文件名。
