5 zip 文件的提取

David Herman

 (require file/unzip) package: base
file/unzip 库提供了一个从 zip 档案中提取项目的功能。

procedure

(unzip in    
  [entry-reader    
  #:must-unzip? must-unzip?    
  #:preserve-attributes? preserve-attributes?    
  #:preserve-timestamps? preserve-timestamps?    
  #:utc-timestamps? utc-timestamps?]) → void?
  in : (or/c path-string? input-port?)
  entry-reader : 
(cond
  [preserve-attributes?
   (bytes? boolean? input-port? (and/c hash? immutable?)
           . -> . any)]
  [preserve-timestamps?
   (bytes? boolean? input-port? (or/c #f exact-integer?)
           . -> . (or/c #f (-> any)))]
  [else
   (bytes? boolean? input-port? . -> . any)])
   = (make-filesystem-entry-reader)
  must-unzip? : any/c = #t
  preserve-attributes? : any/c = #f
  preserve-timestamps? : any/c = #f
  utc-timestamps? : any/c = #f
从 in 中解压整个 zip 压缩文件。 如果 in 不是以 zip-archive 的神奇字节开始, 只有当 must-unzip? 为真时才会报告错误,否则结果是 (void) , 没有从 in 中消耗任何字节。 如果 in 是一个输入端口,并且 preserve-attributes? 是一个真值, 它必须支持通过 file-position 来设置位置。

对于归档文件中的每一个条目, entry-reader 过程被调用,有三个或四个参数: 代表条目名称的字节字符串,指示条目是否代表目录的布尔标志,包含条目膨胀内容的输入端口, 以及(如果 preserve-attributes?)哈希表或 (如果 preserve-timestamps? 而不 preserve-attributes?) #f 或时间戳。 默认的 entry-reader 会将条目解压到文件系统中; 调用 make-filesystem-entry-reader 来配置解压的各个方面,比如目标目录。

当 preserve-attributes? 为 true 时,传递给 entry-reader 的哈希表会提供额外的文件属性, entry-reader 必须为 post-action thunk 产生 #f 。 所有的 post-action 都按最后一次调用 entry-reader 后的顺序运行; 例如,在所有包含的文件被写入后,这些 cions 对于设置一个目录的权限非常有用。 属性在哈希表中使用以下键值进行映射,但其中一个键值可能不存在:

  • 'timestamp — 一个精确的整数,代表文件的时间戳

  • 'permissions — 一个确切的整数,代表文件或目录的权限

尽管 preserve-attributes? 和 preserve-timestamps? 为 entry-reader, 提供了额外的信息, 但解压条目和保存属性及时间戳的工作还是由 entry-reader 来完成。 由 make-filesystem-entry-reader 产生的阅读器, 除了 Windows 上的目录或已经存在的目录外,保留了它所给的任何信息, 并且它只在给了一个目录和一个时间戳和/或权限属性时才返回一个 post-action thunk 。

对于时间戳, zip 存档通常以本地时间记录修改日期, 但如果 utc-timestamps? 为真,那么存档中的时间就被解释为 UTC。

当 preserve-attributes? 为 #f 时,只要能找到文件条目,就会一次性读取 in 。 要注意的是,如果输入代表的归档文件有不被归档中的 "中心目录" 所引用的文件条目,那么无论如何都要解压相应的文件。

Changed in version 6.0.1.12: 增加了 #:utc-timestamps? 参数。

procedure

(make-filesystem-entry-reader [#:dest dest-path 
  #:strip-count strip-count 
  #:permissive? permissive? 
  #:exists exists]) 
 → 
((bytes? boolean? input-port?) ((or/c hash? #f exact-integer?))
 . ->* . (or/c void? #f (-> void?)))
  dest-path : (or/c path-string? #f) = #f
  strip-count : exact-nonnegative-integer? = 0
  permissive? : any/c = #f
  exists : 
(or/c 'skip 'error 'replace 'truncate
      'truncate/replace 'append 'update
      'can-update 'must-truncate)
 = 'error
创建一个可以与 unzip 或 unzip-entry 一起使用的 zip 条目阅读器, 其行为是将条目保存到本地文件系统。 在创建文件之前,如果有必要,总是会创建中间目录。 目录条目被创建为文件系统中的目录,其条目内容被忽略。

如果 dest-path 不是 #f, 存档中的每个路径都有前缀,以确定提取的条目的目标路径。

如果 strip-count 是正数, 那么 strip-count 路径元素将从存档的条目路径中移除(在用 dest-path 对路径进行前缀之前); 如果条目的路径包含 strip-count 元素,那么它就不会被提取。

除非 permissive? 为真,否则包含上层目录指示符的条目是不允许的, 而目标为绝对路径或包含上层目录指示符的链接条目也是不允许的。 绝对路径总是被禁止的。 一个不允许的路径会触发一个异常。

如果 exists 是 'skip ，并且某个条目的文件已经存在,那么该条目将被跳过。 否则, exists 将被传递给 open-output-file ,用于写入条目的膨胀内容。

当返回的过程被调用时,它将产生 (void) ,除非给它一个哈希表作为第四个参数。 当给定一个哈希表时,其结果是 #f 或 thunk。 在 Unix 和 Mac OS 上,当参数指向一个不存在的目录,并且提供了时间戳属性、权限属性或两者时,会返回一个 thunk。

Changed in version 6.3: 增加了 #:permissive? 参数。
Changed in version 8.7.0.9: 在结果函数中增加了对可选的属性哈希表参数的支持。

procedure

(read-zip-directory in) → zip-directory?

  in : (or/c path-string? input-port?)
读取一个 zip 文件的中心目录, 并生成一个代表 zip 文件内容的 zip directory 。 如果 in 是一个输入端口,它必须支持通过 file-position 进行位置设置。

这个过程执行有限的 I/O:它从 zip 文件中读取条目列表,但不对其任何内容进行膨胀。

procedure

(unzip-entry in 
  zipdir 
  entry 
  [entry-reader 
  #:preserve-attributes? preserve-attributes? 
  #:preserve-timestamps? preserve-timestamps? 
  #:utc-timestamps? utc-timestamps?]) 
 → (if preserve-attributes? void? (or/c #f (-> any)))
  in : (or/c path-string? input-port?)
  zipdir : zip-directory?
  entry : (or/c bytes? path-string?)
  entry-reader : 
(cond
  [preserve-attributes?
   (bytes? boolean? input-port? (and/c hash? immutable?)
           . -> . any)]
  [preserve-timestamps?
   (bytes? boolean? input-port? (or/c #f exact-integer?)
           . -> . any)]
  [else
   (bytes? boolean? input-port? . -> . any)])
   = (make-filesystem-entry-reader)
  preserve-attributes? : any/c = #f
  preserve-timestamps? : any/c = #f
  utc-timestamps? : any/c = #f
基于先前从 read-zip-directory 读取的 zip directory, zipdir, 从 zip 存档中解压一个条目。 如果 in 是一个输入端口,它必须支持通过 file-position 来设置位置。

entry 参数是一个字节字符串,其名称必须在压缩文件的中心目录中找到。 如果 entry 不是一个字节串,则用 path->zip-path 进行转换。

The entry-reader 参数用于以与 unzip 相同的方式读取 zip 条目的内容。 当 preserve-attributes? 为 true 时, unzip-entry 会返回 entry-reader 的结果, 它将是 #f 或 post-action 。 返回的 post-action 应该在从 in 提取完成后全部被调用。

如果 entry 不在 zipdir 中,会产生 exn:fail:unzip:no-such-entry 异常。
