3 gzip 解压缩

 (require file/gunzip) package: base
file/gunzip 库提供了以 gzip 格式解压归档文件的实用程序, 或者简单地使用 pkzip “inflate” 方法解压数据。

procedure

(gunzip file [output-name-filter]) → void?

  file : path-string?
  output-name-filter : (string? boolean? . -> . path-string?)
   = (lambda (file archive-supplied?) file)
提取使用 gzip 工具(或 gzip 函数)压缩的数据,将未压缩的数据直接写入一个文件中。 file 参数是包含压缩数据的文件的名称。 默认的输出文件名是存储在 file 中的压缩文件的原始名称。 如果有这个名字的文件存在,它将被覆盖。 如果源文件中没有存储原始名称,则使用 "unzipped" 作为默认输出文件名。

在创建目标文件之前, output-name-filter 程序被应用于两个参数–默认的目标文件名和一个布尔值, 如果这个名字是从 file 中读取的,则为 #t 。 文件的返回值被用作实际的目标文件名(将用 open-output-file 的 'truncate 标志打开)。

如果压缩后的数据被破坏了,就会引发 exn:fail 异常。

procedure

(gunzip-through-ports in out) → void?

  in : input-port?
  out : output-port?
读取使用 gzip 工具创建的压缩数据的端口 in ,将未压缩的数据写到端口 out 。

如果压缩的数据被证明是损坏的,就会产生 exn:fail 异常。 解压缩过程可能会窥视到比解压缩数据所需更多的 in ,但它不会消耗不需要的字节。

procedure

(inflate in out) → void?

  in : input-port?
  out : output-port?
从端口 in 读取 pkzip 格式的 “deflated” 数据, 并将未压缩的 (“inflated”) 数据写到端口 out 。 由 gzip 创建的文件中的数据使用这种格式(前面有一些头信息)。

如果压缩后的数据被破坏了,就会引发 exn:fail 异常。 inflate 进程可能会窥探到比解压数据所需更多的 in ,但它不会消耗不需要的字节。
