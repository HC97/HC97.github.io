3 Futures 跟踪

 (require future-visualizer/trace)
  package: future-visualizer-pict

futures trace 模块公开了关于使用 future 编写的并行程序执行的低级信息。

syntax

(trace-futures e ...)

procedure

(trace-futures-thunk thunk) → (listof indexed-future-event?)

  thunk : (-> any)
trace-futures 宏和 trace-futures-thunk 函数跟踪使用期货的程序的执行, 并以 indexed-future-event 结构列表的形式返回程序跟踪。

这个程序:

(require racket/future
         future-visualizer/trace)
 
(trace-futures
 (let ([f (future (lambda () ...))])
   ...
   (touch f)))

相当于:

(require racket/future
         future-visualizer/trace)
 
(start-future-tracing!)
(let ([f (future (lambda () ...))])
  ...
  (touch f))
(stop-future-tracing!)

procedure

(start-future-tracing!) → void?

procedure

(stop-future-tracing!) → void?

procedure

(timeline-events) → (listof indexed-future-event?)

start-future-tracing! 过程可以收集与未来相关的执行数据。 这个函数应该在执行程序员想要分析的代码之前立即调用。

stop-future-tracing! 过程必须用于指示程序员希望跟踪的代码的结束。 跟踪工作通过简单地使用日志接收器来记录所有与未来相关的日志事件; 这个过程记录一条特殊的消息,日志接收者知道它的意思是 ’stop recording’ 。

timeline-events 过程以 indexed-future-event 结构列表的形式返回程序跟踪。

struct

(struct indexed-future-event (index event)
    #:extra-constructor-name make-indexed-future-event)
  index : exact-nonnegative-integer?
  event : any
表示程序跟踪中的单个日志消息。 除了未来的事件,跟踪代码还记录垃圾收集事件; 因此 event 字段可能包含 future-event 或 gc-info prefab 结构 (参见 垃圾收集), 后者描述 GC 操作。 因为多个 future-event 结构可能包含相同的时间戳, 所以 index 字段按照它们在日志输出中记录的顺序对它们进行排序。

struct

(struct future-event (future-id
    proc-id
    action
    time-id
    prim-name
    user-data)
    #:extra-constructor-name make-future-event
    #:prefab)
  future-id : (or exact-nonnegative-integer? #f)
  proc-id : exact-nonnegative-integer?
  action : symbol?
  time-id : real?
  prim-name : (or symbol? #f)
  user-data : (or #f symbol? exact-nonnegative-integer?)
表示运行时系统记录的未来事件。 更多信息请参见 Future 性能日志 。
