1 执行时间

execution timeline 显示在分析器窗口的左上角,它显示了程序的历史以及与它的未来相关的所有事件, 操作系统级别的线程或 processes 沿着y轴组织,时间沿着x轴增加。 垃圾收集显示为半透明的栗色条,横跨时间轴的高度。 使用着色约定来区分不同类型的事件(参见 Future 性能日志 以获得这些事件类型的完整描述):

  • Blue dot: 'create

  • Green bar: 'start-work, 'start-0-work

  • Orange dot: 'sync

  • Red dot: 'block, 'touch

  • White dot: 'result, 'end-work

  • Green dot: 'touch-pause, 'touch-resume

  • Maroon bar: 'gc

将鼠标悬停在任何非 gc 事件上,通过紫色线将其连接到未来的事件序列。 此外,带有箭头的橙色虚线可以显示从一个未来到另一个未来执行的操作 (例如: 'create 或 'touch 动作)。 若要同时查看两个事件的详细信息,可以通过单击鼠标来添加所选内容。

时间线以 100 微秒的间隔显示垂直线。 请注意,虽然时间间隔是固定的,但是对于任何给定的时间范围,线之间的像素距离根据事件密度而变化,以防止重叠事件圈。

procedure

(timeline-pict events 
  [#:timeline-width timeline-width 
  #:x x 
  #:y y 
  #:width width 
  #:height height 
  #:selected-event-index selected-event-index]) 
 → pict?
  events : (listof indexed-future-event?)
  timeline-width : exact-nonnegative-integer? = 1000
  x : (or #f exact-nonnegative-integer?) = #f
  y : (or #f exact-nonnegative-integer?) = #f
  width : (or #f exact-nonnegative-integer?) = #f
  height : (or #f exact-nonnegative-integer?) = #f
  selected-event-index : (or #f exact-nonnegative-integer?) = #f
返回一个 pict ,显示 events 中跟踪的执行时间轴。 可选的 timeline-width 提供了时间线的最小宽度。 可选参数 x, y, width 和 height 可用于获取时间轴图像的特定区域(以像素为单位)。 如果指定了 selected-event-index 参数,则显示时间轴图像, 就好像用户将鼠标指针放在带有相应索引的 indexed-future-event 上一样。

Changed in version 1.1 of package future-visualizer: 增加了 #:timeline-width 参数。
