版本: Racket 版本检查

"version" 集合包含了几个与版本有关的片段,这些片段被 Racket 使用。 也请参见 racket/base 中的 version 。

1 已安装的补丁级别

 (require version/patchlevel) package: base

value

patchlevel : exact-nonnegative-integer?

表示当前已安装的补丁级别,通常为零,但可能被 DrRacket 的补丁更新。

2 检查可用版本

 (require version/check) package: base

procedure

(check-version) → (or/c symbol? list?)

检查 Racket 网站上当前可用的版本 (https://download.racket-lang.org),并返回一个值,表示当前的安装状态:

  • `ok — 你很好。

  • `(ok-but ,version) — 你有一个很好的稳定版本, 但要注意有一个较新的阿尔法版本可用,编号为 version 。

  • `(newer ,version) — 你有一个旧版本。请升级到 version 。

  • `(newer ,version ,alpha) — 你有一个旧的但稳定的版本,请升级到 version; 你也可以考虑较新的 alpha 版本,编号为 alpha 。

  • `(error ,message) — 发生了一个错误, message 是一个字符串,表示错误。

  • `(error ,message ,additional-info) — 发生了一个错误; message 是一个表示错误的字符串, additional-info 是一个包含系统错误的字符串。 additional-info 的内容总是有括号的, 所以 message 是一个简短的错误, (string-append message " " additional-info) 是一个冗长的错误。

注意,根据网络条件, check-version 可能会持续尝试很长时间(目前是 30 秒)才返回 '(error "timeout") 。 为了测试,当环境变量 PLT_CHECK_VERSION_SIMULATE_TIMEOUT 被设置时, check-version 将模拟这种网络条件, logging 一个 'warning 级别的消息,主题为 'version/check ,然后休眠直到超时。

3 版本实用程序

 (require version/utils) package: base
version/utils 库提供了一些处理版本字符串的便利工具。

procedure

(valid-version? v) → boolean?

  v : any/c
如果 v 是一个有效的 Racket 版本字符串,返回 #t ,否则返回 #f 。

一个有效的版本有以下形式之一:

  • ‹maj›.‹min›

  • ‹maj›.‹min›.‹sub›

  • ‹maj›.‹min›.‹sub›.‹rel›

受以下约束:

  • ‹maj›, ‹min›, ‹sub› 和 ‹rel› 都是自然数的典型十进制表示(即没有前导 0 的十进制数字,除非该数字正好是 0)

  • ‹rel› 不是 0

  • ‹sub› 不是 0 ，除非包括 ‹rel›

  • ‹min› 的数字不超过 2 位

  • ‹sub› 和 ‹rel› 的数字不超过 3 位

这些约束条件迫使版本号以一种规范的形式出现。 例如,一个可能的版本字符串 "4.3.0" 必须写成 "4.3", "4.3.1.0" 必须写成 "4.3.1", 而 "4" 必须写成 "4.0" 。

procedure

(version->list str)

 → (list/c integer? integer? integer? integer?)
  str : valid-version?
返回给定版本字符串所代表的四个数字的列表。

procedure

(version<? str1 str2) → boolean?

  str1 : valid-version?
  str2 : valid-version?
如果 str1 代表一个严格小于 str2 的版本 #t ,否则返回 #f 。

procedure

(version<=? str1 str2) → boolean?

  str1 : valid-version?
  str2 : valid-version?
如果 str1 代表一个小于或等于 str2 的版本,则返回 #t ,否则返回 #f 。

procedure

(alpha-version? str) → boolean?

  str : valid-version?
如果 str 所代表的版本是一个 alpha 版本,则返回 #t 。

如果 ‹min› 是 90 或更多, ‹sub› 是 900 或更多, 或 ‹rel› 是 900 或更多, 那么形式为 ‹maj›.‹min›, ‹maj›.‹min›.‹sub›, 或 ‹maj›.‹min›.‹sub›.‹rel› 的版本号就是一个 alpha 版本。

procedure

(version->integer str) → (or/c integer? #f)

  str : string?
将版本字符串转换为整数。 对于版本 "X.YY.ZZZ.WWW", 结果将是 XYYZZZWWW 。 这个函数也适用于传统的 Racket 版本,将 "XYY.ZZZ" 转换为 XYYZZZ000 。 因此,得到的整数可以用来方便地比较任何两个(有效的)版本字符串。 如果版本字符串是无效的,无论是普通版本字符串还是传统版本字符串,结果都是 #f.

请注意,这是唯一一个处理传统版本字符串的函数。
