3.3 键盘快捷键

大多数 按键只是在编辑器中插入一个字符,如 a, 3 或 ( 。 其他按键和按键组合作为键盘快捷键,可以移动闪烁的光标,删除一行,复制选择,等等。 键盘快捷键通常由使用 Control、Meta 或 Command 键的组合键触发。

许多按键绑定的动作也可以用菜单项来执行。

C-‹key› 是指按下 Control 键,按住它,然后按 ‹key› ,再把它们都放开。 例如: C-e (Control-E) 将闪烁的光标移到当前行的末端。

M-‹key› 与 C-‹key› 相同,但有Meta键。 根据你的键盘,Meta 可能被称为 "左"、"右",或者有一个钻石符号,但它通常在底排,紧挨着空格键。 M-‹key› 也可以作为一个双字符序列来执行:首先,敲击并释放 Escape 键,然后敲击 ‹key› 。 在 Mac OS 上,默认情况下,Meta 只能通过 Escape 键使用。 但偏好设置对话框(在 Editing 面板的 General Editing 子面板中)有一些复选框, 可以调整 Alt 键或 Command 键的处理方式,使之成为元。

DEL 是删除键。

在大多数键盘上,"<" 和 ">" 是移位的字符。 因此,要想得到 M->,你实际上必须输入 Meta-Shift->。 也就是说,同时按住 Meta 和 Shift 键,然后敲 ">"。

在 Windows 下(有时在 Unix 下),其中一些按键绑定实际上是标准的菜单项目。 这些按键绑定将根据菜单来进行操作,除非取消勾选 Enable keybindings in menus 的首选项。 例如,上面提到的 C-e 键绑定实际上是切换了交互窗口的可见性。

如果 你最熟悉 Emacs 风格的按键绑定 (特别是在 windows 或一些 linux 安装中,控制键默认为菜单快捷键), 你应该取消勾选 Enable keybindings in menus 偏好。 下面的许多键位绑定是受 Emacs 的启发。 关于如何有选择地将按键绑定到菜单项上的建议,请参见 定义自定义快捷方式 。

最后,键盘绑定的权威来源是 Edit 菜单的 Show Active Keybindings 菜单项。 DrRacket 中的按键绑定通常对拥有键盘焦点的窗口很敏感, 所以 Show Active Keybindings 打开的窗口内容取决于选择菜单时键盘焦点的位置。

3.3.1 左右移动

  • C-f : 向前移动一个字符

  • C-b : 向后移动一个字符

  • M-f : 向前移动一个字

  • M-b : 向后移动一个字

  • C-v : 向前移动一页

  • M-v : 向后移动一页

  • M-< : 移动到文件的开头

  • M-> : 移动到文件的末尾

  • C-a : 移动到行首(左)

  • C-e : 移动到行末(右)

  • C-n : 移动到下一行(向下)

  • C-p : 移动到上一行(向上)

  • M-C-f : 向前移动一个 S 表达式

  • M-C-b : 向后移动一个 S 表达式

  • M-C-u : 从一个 S 表达式中向上移动

  • M-C-d : 在一个嵌套的 S 表达式中向下移动

  • M-C-SPACE : 选择向前的 S 表达式

  • M-C-p : 向后匹配小括号

  • M-C-left : 向后移动到最近的编辑框中

  • A-C-left : 向后移动到最近的编辑框中

  • M-C-right : 向前移动到最近的编辑框中

  • A-C-right : 向前移动到最近的编辑框中

  • M-C-up : 从一个嵌入的编辑器中向上移动

  • A-C-up : 从一个嵌入的编辑器中向上移动

  • M-C-down : 下移到一个嵌入式编辑器中

  • A-C-down : 向下移动到一个嵌入式编辑器中

  • C-C C-Z : 将光标移至交互窗口

  • C-F6 : 在不同窗口之间移动光标(通常是交互和定义窗口,也包括搜索窗口和 DrRacket 的其他可编辑部分)。 另外,在 Show Active Keybindings 菜单的窗口中搜索 "shift-focus",可以找到更多具有这种功能的特定平台键位。

3.3.2 编辑操作

  • C-_ : undo

  • C-+ : redo

  • C-x u : undo

  • C-d : 向前删除一个字符

  • C-h : 向后删除一个字符

  • M-d : 向前删除一个字

  • M-DEL : 向后删除一个字

  • C-k : 向前删除到行尾

  • M-C-k : 向前删除一个 S 表达式

  • M-w : 复制选择到剪贴板

  • C-w : 删除选择到剪贴板(切)

  • C-y : 从剪贴板上粘贴(yank)

  • C-t : 转位字符

  • M-t : 移位单词

  • M-C-t : 转置表达式

  • M-C-m : 切换匹配括号的深绿色标记

  • M-C-k : 剪切完整的表达式

  • M-( : 在小括号中包裹选择

  • M-[ : 在方括号中包裹选择内容

  • M-{ : 在大括号中包住选择的内容

  • M-S-L : 在 (lambda () ...) 中包裹选择,并将插入点放在 lambda 的参数列表中

  • C-c C-o : 插入点之后的表达式被放在其包含的表达式的位置上

  • C-c C-e : 删除包含表达式的第一个和最后一个字符(通常是括号)

  • C-c C-l : 在插入点后面的表达式上包裹一个 let,并在该点上放一个 printf (对调试很有用)。

  • M-o : 切换 overwrite mode

  • C-x r a : 调整附近的 ASCII 艺术矩形(使用 +, - 或 |),以使用 Unicode 字符。

    例如,如果插入点在这个矩形的旁边:

    +-+

    | |

    +-+

    那么击键将把它变成这个:

    ╔═╗

    ║ ║

    ╚═╝

    同样,如果插入点附近的矩形有 Unicode 和 ASCII 的混合字符,它将全部转化为 Unicode 字符。

  • C-x r w : 加宽附近的 ASCII 艺术矩形。

    例如,如果插入点就在这个矩形的中间线的左边:

    ╔═╦══╗

    ║ ║ ║

    ╠═╬══╣

    ║ ║ ║

    ╚═╩══╝

    那么这个按键将把它变成这个矩形:

    ╔══╦══╗

    ║ ║ ║

    ╠══╬══╣

    ║ ║ ║

    ╚══╩══╝

  • C-x r v : 使附近的 ASCII 艺术矩形变高。

    例如,如果插入点正好在这个矩形的中线上方:

    ╔══╦══╗

    ║ ║ ║

    ╠══╬══╣

    ║ ║ ║

    ║ ║ ║

    ╚══╩══╝

    那么这个按键将把它变成这个:

    ╔══╦══╗

    ║ ║ ║

    ║ ║ ║

    ╠══╬══╣

    ║ ║ ║

    ║ ║ ║

    ╚══╩══╝

  • C-x r c : 将当前行的内容放在包围的 ASCII 艺术矩形的包围单元格内。

  • C-x r o : 切换 ASCII 艺术矩形的编辑模式。 当该模式被启用时,通常会破坏矩形的击键会使其放大。具体来说:

    • Return 和 Enter 为包围的矩形添加一条线,并将插入点放在包围的单元格的第一列。

    • 当处于覆盖模式时,如果一个键会覆盖单元格的一个墙,那么这个墙会首先被移过去以容纳新的键

    • 当不处于覆盖模式时,插入一个字符将始终扩大包含的单元格。

3.3.7 交互

interactions window 具有与 definitions window 相同的所有键盘快捷键,另外还有一些:

  • M-p : 将先前输入的表达式下拉到提示符处

  • M-n : 将表达式历史中当前表达式之后的表达式下拉到提示符下

  • M-h : 在一个单独的窗口中显示当前的表达式历史

另外,在交互窗口中,回车键被特别处理,特别是为了确定在提示符下输入的表达式是否准备好被评估。有三种情况:

  • 当输入 shift-return 时:表达式不被提交,而是开始一个新行。

  • 当输 入alt-return 或 control return 时:表达式被提交进行评估。

  • 当键入没有修改的返回时:咨询语言以确定前两个动作中哪个最合理。 通常情况下,会咨询语言的解析器,看表达式是否看起来完整。 如果是,表达式就被提交,如果不是,就开始一个新的行。 (参见 'drracket:submit-predicate 。)

3.3.9 定义自定义快捷方式

在 Edit 的 Keybindings 子菜单中, Add User-defined Keybindings... 菜单项可以选择一个包含 Racket 键盘绑定定义的文件。 该文件必须包含一个使用特殊键盘绑定语言的模块,即 framework/keybinding-lang 。 要做到这一点,请在文件的开头加上这一行:

#lang s-exp framework/keybinding-lang

framework/keybinding-lang 语言提供了所有来自 racket, racket/class 和 drracket/tool-lib 的绑定,只是它调整了 #%module-begin 以引入 keybinding 形式:

(keybinding string-expr proc-expr)

声明一个键绑定,其中 string-expr 必须为 map-function in keymap% 产生一个合适的第一参数, 而 proc-expr 必须为 add-function in keymap% 产生一个合适的第二参数。

例如,这里将组合键 "control-a" 重映射为"!"。

#lang s-exp framework/keybinding-lang
(keybinding "c:a" (λ (editor evt) (send editor insert "!")))

由于该文件包含普通的 Racket 代码, 你可以编写使用 DrRacket 的 Extension API 的按键绑定文件。 例如,下面的文件将 "control-t" 和 "control-=" 分别绑定为执行程序和打开新标签,因为它们在 5.2 版之前就被使用了。

#lang s-exp framework/keybinding-lang
 
(define modifiers
  (apply string-append
         (map (λ (p)
                (case p
                  [(ctl) "c:"] [(cmd) "d:"] [(alt meta) "~c:m:"]
                  [(shift) "s:"] [(option) "a:"]))
              (get-default-shortcut-prefix))))
 
(define-syntax-rule (frame-key key command)
  (keybinding
   (string-append modifiers key)
   (λ (ed evt)
     (when (is-a? ed text:basic<%>)
       (define fr (send ed get-top-level-window))
       ; note: fr could be #f
       (when fr (send fr command))))))
 
(frame-key "t" execute-callback)
(frame-key "=" create-new-tab)

另一个例子,这个文件重新绑定了 "control-w" 来删除插入点后面的单词,但它是通过设置一个新的键来实现的,是一个现有的键盘快捷键。 如果你在 Show Active Keybindings 对话框中看到一个键(在 Edit 菜单的 Keybindings 子菜单中),那么你可以用它的名字和你想要的新键位一起使用,像这样:

#lang s-exp framework/keybinding-lang
 
(define (rebind key command)
  (keybinding
   key
   (λ (ed evt)
     (send (send ed get-keymap) call-function
           command ed evt #t))))
 
(rebind "c:w" "backward-kill-word")

这个例子显示了如何将一个菜单项(根据其名称)与一个特定的键绑定。例子结尾处的调用将 "control-a" 与 Run 菜单项绑定。

#lang s-exp framework/keybinding-lang
 
(define (menu-bind key menu-item)
  (keybinding
   key
   (λ (ed evt)
     (define canvas (send ed get-canvas))
     (when canvas
       (define menu-bar (find-menu-bar canvas))
       (when menu-bar
         (define item (find-item menu-bar menu-item))
         (when item
           (define menu-evt
             (new control-event%
                  [event-type 'menu]
                  [time-stamp
                   (send evt get-time-stamp)]))
           (send item command menu-evt)))))))
 
(define/contract (find-menu-bar c)
  (-> (is-a?/c area<%>) (or/c #f (is-a?/c menu-bar%)))
  (let loop ([c c])
    (cond
      [(is-a? c frame%) (send c get-menu-bar)]
      [(is-a? c area<%>) (loop (send c get-parent))]
      [else #f])))
 
(define/contract (find-item menu-bar label)
  (-> (is-a?/c menu-bar%)
      string?
      (or/c (is-a?/c selectable-menu-item<%>) #f))
  (let loop ([o menu-bar])
    (cond
      [(is-a? o selectable-menu-item<%>)
       (and (equal? (send o get-plain-label) label)
            o)]
      [(is-a? o menu-item-container<%>)
       (for/or ([i (in-list (send o get-items))])
         (loop i))]
      [else #f])))

注意,DrRacket 不会在你做修改时自动重新加载键位绑定文件,所以你需要重新启动 DrRacket 才能看到文件的修改。

3.3.10 向 REPL 发送程序片段

熟悉 Emacs 和传统 Lisp/Scheme 风格的 "下级进程" 交互方式的用户通常要求在 DrRacket 中使用按键绑定, 以发送程序片段,在提示符下进行评估。 这种交互方式充满了困难,特别是对初学者来说,所以 DrRacket 默认不支持这种方式。 相反,点击 DrRacket 的 "运行" 按钮会从一个干净的地方开始,并发送定义窗口的全部内容,确保 REPL 中的状态与你阅读程序源代码所期望的一致。

基于多年使用 Emacs 模式的经验,一些作者认为这种交互模式也适合于有经验的程序员。事实上,他们通过巨大的努力来模仿 Emacs 中的这种行为。

也就是说,有些人可能希望使用这种递增的击键模式。 因此,本节的其余部分将通过一个键盘绑定文件的例子来说明如何为你的个人使用添加这种增量模式。 具体来说,该文件展示了如何增加向交互窗口零散地发送表达式的能力。 它还演示了如何将 DrRacket 的实现及其库中的一堆碎片拉到一起,实现按键。

#lang s-exp framework/keybinding-lang
 
(require drracket/tool-lib)
 
(module test racket/base)
 
(keybinding "c:c;c:e" (lambda (ed evt) (send-toplevel-form ed #f)))
(keybinding "c:c;c:r" (lambda (ed evt) (send-selection ed #f)))
(keybinding "c:c;~c:m:e" (lambda (ed evt) (send-toplevel-form ed #t)))
(keybinding "c:c;~c:m:r" (lambda (ed evt) (send-selection ed #t)))
 
(define/contract (send-toplevel-form defs shift-focus?)
  (-> any/c boolean? any)
  (when (is-a? defs drracket:unit:definitions-text<%>)
    (define sp (send defs get-start-position))
    (when (= sp (send defs get-end-position))
      (cond
        [(send defs find-up-sexp sp)
         ;; we are inside some top-level expression;
         ;; find the enclosing expression
         (let loop ([pos sp])
           (define next-up (send defs find-up-sexp pos)) 
           (cond
             [next-up (loop next-up)]
             [else
              (send-range-to-repl defs 
                                  pos
                                  (send defs get-forward-sexp pos)
                                  shift-focus?)]))]
        [else
         ;; we are at the top-level
         (define fw (send defs get-forward-sexp sp))
         (define bw (send defs get-backward-sexp sp))
         (cond
           [(and (not fw) (not bw)) 
            ;; no expressions in the file, give up
            (void)]
           [(not fw) 
            ;; no expression after the insertion point;
            ;; send the one before it
            (send-range-to-repl defs 
                                bw
                                (send defs get-forward-sexp bw)
                                shift-focus?)]
           [else 
            ;; send the expression after the insertion point
            (send-range-to-repl defs 
                                (send defs get-backward-sexp fw)
                                fw
                                shift-focus?)])]))))
              
(define/contract (send-selection defs shift-focus?)
  (-> any/c boolean? any)
  (when (is-a? defs drracket:unit:definitions-text<%>)
    (send-range-to-repl defs
                        (send defs get-start-position) 
                        (send defs get-end-position) 
                        shift-focus?)))
 
(define/contract (send-range-to-repl defs start end shift-focus?)
  (->i ([defs (is-a?/c drracket:unit:definitions-text<%>)]
        [start exact-positive-integer?]
        [end (start) (and/c exact-positive-integer? (>=/c start))]
        [shift-focus? boolean?])
       any)
  (unless (= start end) ;; don't send empty regions
    (define ints (send (send defs get-tab) get-ints))
    (define frame (send (send defs get-tab) get-frame))
    ;; copy the expression over to the interactions window
    (send defs move/copy-to-edit 
          ints start end
          (send ints last-position)
          #:try-to-move? #f)
    
    ;; erase any trailing whitespace
    (let loop ()
      (define last-pos (- (send ints last-position) 1))
      (when (last-pos . > . 0)
        (define last-char (send ints get-character last-pos))
        (when (char-whitespace? last-char)
          (send ints delete last-pos (+ last-pos 1))
          (loop))))
    
    ;; put back a single newline
    (send ints insert
          "\n"
          (send ints last-position)
          (send ints last-position))
    
    ;; make sure the interactions is visible 
    ;; and run the submitted expression
    (send frame ensure-rep-shown ints)
    (when shift-focus? (send (send ints get-canvas) focus))
    (send ints do-submission)))
