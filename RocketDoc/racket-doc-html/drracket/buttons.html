1.1 按钮

DrRacket 中一排按钮的左端有一个微型按钮,上面有 当前文件的名称。 点击该按钮可以打开一个菜单,显示文件的完整路径名。 选择其中一个菜单项会产生一个从相应目录开始的打开文件对话框。

在文件名按钮旁边有一个 (define ...) button , 用于弹出一个在定义窗口中定义的名称菜单。 从菜单中选择一个项目可以将插入点(闪烁的光标)移动到相应的定义。

每当定义窗口被修改时, Save button 就会出现。 点击该按钮可以将定义窗口的内容保存到一个文件中。 当前的文件名出现在 Save 按钮的左边,但如果该文件以前从未被保存过,就会出现一个文件选择的对话框。

Step button—仅出现在 How to Design Programs 教学语言 Beginning Student 到 Intermediate Student with Lambda 中–启动 Stepper, 它以一系列小步骤显示程序的评估过程。 每个评估步骤都会使用 DrRacket 的评估规则将程序中的一个表达式替换成一个等价的表达式。 例如,一个步骤可以用 3 替换 (+ 1 2) ,这些都是 DrRacket 用来评估程序的规则。 点击 Step 可以打开一个新窗口,其中包含定义窗口中的程序,还有几个新按钮:这些按钮可以将评估作为一系列步骤进行导航。

调试界面将在本手册后面的 图形化的调试界面 中进一步描述。

Debug button—在 How to Design Programs 的教学语言中 不 出现–启动一个更传统的步进式 debugger 。 它像 Run 按钮一样在定义窗口中运行程序,但同时也打开一个带有其他几个按钮的调试面板,提供对程序执行的控制。

点击 Check Syntax button 可以注释定义窗口中的程序文本。它添加了以下注释:

  • Syntactic Highlighting: 导入的变量和本地定义的变量会用颜色变化来突出显示。 文档标识符被超链接(通过右键点击)到文档页面。

  • Lexical Structure: 词法结构以箭头形式显示在程序文本上。 当鼠标光标经过一个变量时,DrRacket 会画出一个箭头,从绑定位置到该变量,或者从绑定位置到该变量的每个绑定位置。

    除了用蓝色箭头表示明确的引用外, DrRacket 还画箭头表示宏定义中的潜在引用。 潜在的箭头是用紫色画的,并用问号来表示不确定性,因为 DrRacket 无法预测这些标识符最终会被如何使用。 它们的作用可能取决于宏的参数和宏的使用环境。

    此外, 在变量上点击右键(或在 Mac OS 上点击 Control 键)可以激活一个弹出式菜单, 让你从绑定位置跳到绑定位置,反之亦然, α-rename 变量,或粘住箭头使其不消失。

  • Tail Calls:任何相对于其包围的上下文而言(语法上) 处于尾部位置的子表达式都被注释为从尾部表达式到其周围表达式的浅紫色箭头。

  • Require Annotations: 在 require 的参数上点击右键(或在 Mac OS 上点击 Control 键)会激活一个弹出菜单, 让你打开包含 require 模块的文件。

    将鼠标光标移到模块内的 require 表达式上,会显示所有被该 require 表达式使用的变量。 此外,如果该 require 表达式没有使用任何变量,它就会像一个未绑定的变量一样被着色。

    最后,将鼠标光标移到一个从模块中导入的变量上,会在框架底部的状态行中显示该变量所导入的模块。

当你编辑程序时,检查语法也会自动运行,DrRacket 窗口的最右下角会显示其状态。 红点意味着出了问题;将鼠标移到红点上,就能找出问题所在。

没有问题时,颜色表示程序的处理阶段:蓝色(扩展)、紫色(计算检查语法信息)、兰色(用检查语法信息更新编辑器)和绿色(完成)。

另外,在该区域点击右键(或点击控制键)会产生一个菜单,让你禁用(或重新启用)自动检查语法。

Run button 评估 definitions window 中的程序并重置 interactions window 。

Break button 会中断评估,如果 DrRacket 没有评估任何东西,则会发出提示音。 例如,在点击 Run 或在交互窗口中输入表达式后,点击 Break 取消评估。 点击 Break 按钮一次,尝试优雅地中断评估;点击按钮两次,立即终止评估。
