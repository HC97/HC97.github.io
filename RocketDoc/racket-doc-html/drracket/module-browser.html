1.9 模块浏览器

模块浏览器显示你的程序中所有文件的结构。 它可以通过 Show 菜单打开, 或通过 Racket 菜单中的 Module Browser 菜单项打开。 它也可以通过 raco dependencies-graph 从命令行中获得。

模块浏览器窗口包含每个模块的一个方块。 方块的颜色是根据模块中的代码行数来决定的。 如果一个模块有更多的代码行,它的颜色会更深。 如果一个模块是红色的,意味着 DrRacket 没有找到它的源文件。

此外,对于每个正常的导入,都会画一条蓝色的线,从模块到导入模块。 同样,对于每个 for-syntax、for-template 或 for-meta 的导入,也会画上紫色的线。 在最初的模块布局中,左边的模块导入右边的模块,但由于模块可以交互式地移动,这一属性可能不会被保留。

要打开与模块对应的文件,请双击该模块的方框。

模块浏览器也会显示每个模块加载的阶段;在 Names 弹出菜单中选择 Long, with phases 菜单项。 整数表示阶段,如果 #f 出现,意味着该模块是 for-label 加载的。

沿着底部的条形图可以帮助你在模块图中找到方向。 具体来说,如果你在这里输入一些东西,那么所有文件名与你输入的内容相符的模块在模块窗口中都会变成绿色。 该栏只在独立的模块浏览器窗口中可见(通过 Racket 菜单)。

Visible Packages 菜单控制哪个子集的文件是可见的。 开始时,所有与初始文件所在包相同的文件都会显示。
