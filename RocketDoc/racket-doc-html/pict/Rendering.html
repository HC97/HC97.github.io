14 Rendering

parameter

(dc-for-text-size) → (or/c #f (is-a?/c dc<%>))

(dc-for-text-size dc) → void?
  dc : (or/c #f (is-a?/c dc<%>))
一个用于确定用 text 创建的图片的 bounding box 的参数。

这个参数中安装的绘图上下文不必与最终的绘图上下文相同,但它应该以相同的方式测量文本。 在正常情况下,字体度量对所有绘图上下文都是一样的, 所以 dc-for-text-size 的默认值是一个 bitmap-dc% ,绘制成1乘1的位图。

parameter

(convert-bounds-padding)

 → (list/c (>=/c 0) (>=/c 0) (>=/c 0) (>=/c 0))
(convert-bounds-padding padding) → void?
  padding : (list/c (>=/c 0) (>=/c 0) (>=/c 0) (>=/c 0))
一个参数,它决定了在转换为 'png@2x-bytes+bounds8(见 file/convertible)这样的格式时, 在 pict 的每个边缘添加的填充量。 该参数的默认值是 '(3 3 3 3), 它在每条边上增加三个像素, 以适应在 pict 的 bounding box 之外的少量绘图。

Added in version 1.2 of package pict-lib.

procedure

(pict->bitmap pict    
  [smoothing    
  #:make-bitmap make-a-bitmap]) → (is-a?/c bitmap%)
  pict : pict-convertible?
  smoothing : (or/c 'unsmoothed 'smoothed 'aligned) = 'aligned
  make-a-bitmap : 
(exact-positive-integer? exact-positive-integer?
 . -> . (is-a?/c bitmap%))
   = make-bitmap
返回一个 bitmap% ， pict 画在它的左上角 (0, 0)。 将 pict 绘制到位图中使用了平滑模式 smoothing; 关于平滑模式的更多信息见 set-smoothing 。

make-a-bitmap 函数被用来创建位图,给它 pict 的尺寸,四舍五入到整数尺寸。 默认的函数, make-bitmap, 创建一个带有 alpha 通道的位图。 提供 make-screen-bitmap, 以创建一个与直接在屏幕上绘图一致的位图。

Changed in version 1.7 of package pict-lib: 增加了 #:make-bitmap 参数。

procedure

(pict->argb-pixels pict [smoothing]) → bytes?

  pict : pict-convertible?
  smoothing : (or/c 'unsmoothed 'smoothed 'aligned) = 'aligned
返回 pict->bitmap 所返回的与位图对应的像素的 bytes? 。 每个像素在结果中有四个字节:阿尔法、红色、绿色和蓝色成分。

Examples:
> (pict->argb-pixels
   (filled-rectangle 1 1))

#"\377\0\0\0"

> (pict->argb-pixels
   (colorize (filled-rectangle 1 1) "red"))

#"\377\377\0\0"

Added in version 1.1 of package pict-lib.

procedure

(argb-pixels->pict bytes width) → pict?

  bytes : bytes?
  width : exact-nonnegative-integer?
从 bytes 中构造一个宽度为 width 的 pict 。 结果中的每个像素都对应于 bytes 中的四个条目:阿尔法值,以及红、绿、蓝值。

Example:
> (let ([b (make-bytes (* 40 40 4) 255)])
    (for ([x (in-range (bytes-length b))])
      ; when in one of two vertical bands (10-20 & 30-40)
      (when (or (<= 10 (modulo (quotient x 4) 40) 20)
                (<= 30 (modulo (quotient x 4) 40) 40))
        ; change the red and green fields of the pixel
        (when (= 1 (modulo x 4)) (bytes-set! b x 0))
        (when (= 2 (modulo x 4)) (bytes-set! b x 150))))
    (argb-pixels->pict b 40))

Added in version 1.1 of package pict-lib.
