6 轴变换和刻度线

 (require plot) package: plot-gui-lib

6.1 轴变换

通过在不同的 plot-x-transform, plot-y-transform 和 plot-z-transform 值上设置参数, 可以对任何绘图的 x, y 和 z 轴进行独立转换。 例如,用对数变换来绘制 x 轴:
> (parameterize ([plot-x-transform  log-transform])
    (plot (function sin 1 100)))

然而,大多数 log-transform 的绘图使用的是与默认的、均匀间隔的刻度线不同的刻度线。 要在 x 轴上放置对数刻度,请设置 plot-x-ticks 参数:
> (parameterize ([plot-x-transform  log-transform]
                 [plot-x-ticks      (log-ticks)])
    (plot (function sin 1 100)))

参见 轴刻度 ,了解更多关于参数化图轴刻度的细节。

为了进行非线性采样,一个变换的 inverse 被应用于线性采样的点。 参见 make-axis-transform 和 nonlinear-seq 。 渲染器通过非线性采样与当前的变换合作。比如说、
> (parameterize ([plot-x-transform  log-transform])
    (plot3d (surface3d + 0.01 1 0.01 1)))

请注意,尽管 x 轴刻度线的间距并不均匀,但表面的采样是均匀的。

变换被应用于构成绘图的原始形状:
> (parameterize ([plot-x-transform  log-transform])
    (plot3d (surface3d + 0.01 1 0.01 1 #:samples 3)))

在这里, surface3d 返回的渲染器不需要弯曲它所绘制的多边形; plot3d 会自动地(通过递归细分)这样做。

(plot-x-transform) → axis-transform/c

(plot-x-transform transform) → void?
  transform : axis-transform/c
 = id-transform

parameter

(plot-y-transform) → axis-transform/c

(plot-y-transform transform) → void?
  transform : axis-transform/c
 = id-transform

parameter

(plot-z-transform) → axis-transform/c

(plot-z-transform transform) → void?
  transform : axis-transform/c
 = id-transform
独立的、每轴的、单调的、非线性的变换。 Plot 带有一些典型的(和一些非典型的)轴变换,紧接着在下面记录。

value

id-transform : axis-transform/c

特征轴变换,所有轴的默认变换。

value

log-transform : axis-transform/c

一个对数变换。使用它来生成具有对数尺度轴的图。 任何这样的轴都必须有正的界限。

轴变换和刻度线 部分的开头有一个工作实例。 一个超出边界的例子是
> (parameterize ([plot-x-transform  log-transform])
    (plot (function (λ (x) x) -1 1)))

log-transform: expects type <positive real> as 1st argument, given: -1; other arguments were: 1

参见 axis-transform-bound 和 axis-transform-append 以了解如何绕过轴变换的边界限制。

procedure

(stretch-transform a b scale) → axis-transform/c

  a : real?
  b : real?
  scale : (>/c 0)
返回一个拉伸有限区间的轴变换。

下面的例子使用 stretch-transform 来吸引人们对区间 [-1,1] 的注意, 以说明当 x 接近零时 sin(x)/x 的极限(证明 sin(x) 导数的一个关键部分):
> (parameterize ([plot-x-transform  (stretch-transform -1 1 20)]
                 [plot-x-ticks  (ticks-add (plot-x-ticks) '(-1 1))])
    (plot (list (y-axis -1 #:ticks? #f) (y-axis 1 #:ticks? #f)
                (function (λ (x) (/ (sin x) x)) -14 14
                          #:width 2 #:color 4 #:label "y = sin(x)/x")
                (point-label (vector 0 1) "y → 1 as x → 0"
                             #:anchor 'bottom-right))
          #:y-max 1.2))

procedure

(collapse-transform a b) → axis-transform/c

  a : real?
  b : real?
返回一个轴变换,将有限区间折叠到其中点。 例如,去掉 atan(x) 向 π/2 方向漫长而枯燥的渐近过程的一部分:
> (parameterize ([plot-x-transform  (collapse-transform 50 150)])
    (plot (function atan 10 200 #:label "y = atan(x)")
          #:legend-anchor 'center))

在这种情况下,在折叠区间的端点已经有了刻度。 如果没有,就有必要使用 ticks-add 来让观众准确地知道被折叠的区间。 (参见 stretch-transform 的例子)。

value

cbrt-transform : axis-transform/c

一个 "立方根" 变换,主要用于测试。 与对数变换不同,它是在整个实线上定义的,这使得它更适合于测试具有非线性变换轴的图的外观。

procedure

(hand-drawn-transform freq) → axis-transform/c

  freq : (>/c 0)
一个 extremely important 的测试案例, 它确保 Plot 可以使用任何单调的、可逆的函数作为轴的变换。 freq 参数控制变换的 "抖动" 程度。 在高值时,它使绘图看起来像花生动画片。

Examples:
> (parameterize ([plot-x-transform  (hand-drawn-transform 200)]
                 [plot-y-transform  (hand-drawn-transform 200)])
    (plot (function sqr -1 1)))

> (parameterize ([plot-x-transform  (hand-drawn-transform 50)]
                 [plot-y-transform  (hand-drawn-transform 50)]
                 [plot-z-transform  (hand-drawn-transform 50)])
    (plot3d (contour-intervals3d (λ (x y) (- (sqr x) (sqr y)))
                                 -1 1 -1 1 #:samples 9)))

value

axis-transform/c : contract?

 = (-> real? real? invertible-function? invertible-function?)
轴变换的合同。

构建新的轴变换的最简单的方法是使用轴变换组合器 axis-transform-append, axis-transform-bound 和 axis-transform-compose, 或者将 make-axis-transform 应用于 invertible-function 。

procedure

(axis-transform-append t1 t2 mid) → axis-transform/c

  t1 : axis-transform/c
  t2 : axis-transform/c
  mid : real?
返回一个轴变换,该轴变换将小于 mid 的值像 t1 一样进行变换,并将大于 mid 的值像 t2 一样进行变换。 (无论它是像 t1 还是 t2 一样变换 mid ,都是不重要的,因为变换后的 mid 无论如何都等于 mid 。)

Example:
> (parameterize ([plot-x-transform  (axis-transform-append
                                     (stretch-transform -2 -1 10)
                                     (stretch-transform 1 2 10)
                                     0)])
    (plot (function (λ (x) x) -3 3)))

procedure

(axis-transform-bound t a b) → axis-transform/c

  t : axis-transform/c
  a : real?
  b : real?
返回一个轴变换,在区间 [a,b] 内像 t一样变换数值,但在区间外像同一变换。 例如,将 log-transform 绑定到一个区间,在这个区间中它是明确定义的、
> (parameterize ([plot-x-transform  (axis-transform-bound
                                     log-transform 0.01 +inf.0)])
    (plot (function (λ (x) x) -4 8 #:label "y = x")))

procedure

(axis-transform-compose t1 t2) → axis-transform/c

  t1 : axis-transform/c
  t2 : axis-transform/c
组成两个轴变换。 例如,要折叠一个 log-transform 轴的一部分,可以尝试这样的方法
> (parameterize ([plot-x-transform  (axis-transform-compose
                                     log-transform
                                     (collapse-transform 2 4))])
    (plot (function (λ (x) x) 1 5)))

参数顺序很重要,但预测交换参数的影响可能很困难。 幸运的是,这些影响通常是轻微的。

procedure

(make-axis-transform fun) → axis-transform/c

  fun : invertible-function?
给出一个单调的 invertible-function, 返回一个轴变换。 单调性是必要的,但不能强制执行。 反函数被用来沿着变换后的轴均匀地取样(见 nonlinear-seq)。

Example:
> (parameterize ([plot-y-transform  (make-axis-transform
                                     (invertible-function sqrt sqr))])
    (plot (function (λ (x) x) 0 5)))

由 make-axis-transform (或上述任何组合器)创建的轴变换不会对轴的边界端点进行变换,误差不超过浮点。 比如说、
> (match-let ([(invertible-function f g)
               (apply-axis-transform log-transform 1 3)])
    (define xs '(1 2 3))
    (define new-xs (map f xs))
    (define old-xs (map g new-xs))
    (values new-xs old-xs))

'(1.0 2.261859507142915 3.0)

'(1.0 2.0 2.9999999999999996)

技术上来说, fun 不需要是真正的可反转的。 给定 fun = (invertible-function f g), 只要 f 是 g 的 left inverse 就够了; 也就是说,总是 (f (g x)) = x ,但不一定 (g (f x)) = x 。 如果 f 和 g 必须是彼此的严格倒数,就不可能有 collapse-transform 。

procedure

(apply-axis-transform t x-min x-max) → invertible-function?

  t : axis-transform/c
  x-min : real?
  x-max : real?
返回一个可逆函数,在给定的轴线范围内转换轴线点。 这个方便的函数在内部被用来在渲染前转换点,但为了完整起见,我们提供了这个函数。

6.2 轴刻度

每个绘图轴都有两组独立的刻度线: near 度线和 far 度线。

(plot-y-far-ticks) → ticks?

(plot-y-far-ticks ticks) → void?
  ticks : ticks?
 = (ticks-mimic plot-y-ticks)

parameter

(plot-z-ticks) → ticks?

(plot-z-ticks ticks) → void?
  ticks : ticks?
 = (linear-ticks)

parameter

(plot-z-far-ticks) → ticks?

(plot-z-far-ticks ticks) → void?
  ticks : ticks?
 = (ticks-mimic plot-z-ticks)

Example:
> (parameterize ([plot-x-label      "Near x axis"]
                 [plot-y-label      "Near y axis"]
                 [plot-z-label      "Near z axis"]
                 [plot-x-ticks      (date-ticks)]
                 [plot-y-ticks      (time-ticks)]
                 [plot-z-ticks      (fraction-ticks)]
                 [plot-x-far-label  "Far x axis"]
                 [plot-y-far-label  "Far y axis"]
                 [plot-z-far-label  "Far z axis"]
                 [plot-x-far-ticks  (linear-ticks)]
                 [plot-y-far-ticks  (currency-ticks)]
                 [plot-z-far-ticks  (log-ticks #:base 2)])
    (plot3d (lines3d '(#(1 1 1) #(40000000 4 4)) #:style 'transparent)
            #:angle 45 #:altitude 50
            #:title "Axis Names and Tick Locations"))

在任何 #:angle 下,远处的 x 和 y 刻度线都在绘图的后面, 而远处的 z 刻度线则在右边。 如果远处的刻度线与其对应的近处的刻度线相同,就会被画出来,但不加标注。 默认情况下,它们总是相同的。

Major ticks 比 minor ticks 长。 主要刻度线的标签总是被画出来,除非与附近的刻度线折叠在一起。小刻度线的标签永远不会被绘制。

由 contours 和 contour-intervals 产生的渲染器使用 plot-z-ticks 的值来放置和标记轮廓线。 例如,比较使用 contour-intervals 和 contour-intervals3d 渲染的同一函数的图:
> (parameterize ([plot-z-ticks  (currency-ticks)])
    (define (saddle x y) (- (sqr x) (sqr y)))
    (values
     (plot (contour-intervals saddle -1 1 -1 1 #:label "z")
           #:legend-anchor 'center)
     (plot3d (contour-intervals3d saddle -1 1 -1 1 #:label "z")
             #:legend-anchor 'center)))

procedure

(contour-ticks z-ticks    
  z-min    
  z-max    
  levels    
  intervals?) → (listof tick?)
  z-ticks : ticks?
  z-min : real?
  z-max : real?
  levels : (or/c 'auto exact-positive-integer? (listof real?))
  intervals? : boolean?
返回用于轮廓值的刻度线。 这是由 contours, contour-intervals, contours3d, contour-intervals3d 和 isosurfaces3d 返回的渲染器内部使用的,但为了完整性而提供。

当 levels 为 'auto 时, 返回的值与 z-ticks 返回的 ticks 值不 完全 对应:它们可能缺少端点值。比如说、
> (map pre-tick-value
       (filter pre-tick-major? (ticks-generate (plot-z-ticks) 0 1)))

'(0 1/5 2/5 3/5 4/5 1)

> (map pre-tick-value
       (contour-ticks (plot-z-ticks) 0 1 'auto #f))

'(1/5 2/5 3/5 4/5)

(struct ticks (layout format)
    #:extra-constructor-name make-ticks)

一个近轴或远轴的 ticks 由一个 layout 函数和一个 format 函数组成, 前者决定刻度线的数量和它们的放置位置,后者决定刻度线的标签。

(ticks-generate ticks min max) → (listof tick?)

  ticks : ticks?

  max : real?
生成 [min, max] 范围内的 ticks 值,其布局和格式由 tick 指定。

Example:
> (ticks-generate (plot-x-ticks) 1/3 2/3)

(list

 (tick 7/20 #f ".35")

 (tick 2/5 #t ".4")

 (tick 9/20 #f ".45")

 (tick 1/2 #t ".5")

 (tick 11/20 #f ".55")

 (tick 3/5 #t ".6")

parameter

(ticks-default-number) → exact-positive-integer?

(ticks-default-number number) → void?
  number : exact-positive-integer?
 = 4
大多数刻度线布局函数(以及它们相应的 ticks-constructing 函数) 都有一个 #:number 关键字参数,默认为 (ticks-default-number) 。 这个数字是什么意思,取决于刻度线布局函数。 大多数人用它来表示主要刻度的平均数。

它不太可能意味着精确的主要刻度线的数量。 如果不调整刻度线的数量,布局函数通常不能找到均匀间隔的刻度线,在格式化后会有简单的标签。 例如,下面的图显示了当要求的刻度数为 8 时,区间 [0,x] 的实际主要刻度数, 由 linear-ticks-layout 生成:
> (plot (function (λ (x)
                    (count pre-tick-major?
                           ((linear-ticks-layout #:number 8) 0 x)))
                  0.1 10)
        #:x-label "Interval [0,x]" #:y-label "Number of ticks")

6.2.1 线性刻度

procedure

(linear-ticks-layout [#:number number 
  #:base base 
  #:divisors divisors] 
  #:scientific? scientific?) 
 → ticks-layout/c
  number : exact-positive-integer? = (ticks-default-number)
  base : (and/c exact-integer? (>=/c 2)) = 10
  divisors : (listof exact-positive-integer?) = '(1 2 4 5)
  scientific? : #t

procedure

(linear-ticks-format #:scientific? scientific?)

 → ticks-format/c
  scientific? : #t

procedure

(linear-ticks [#:number number    
  #:base base    
  #:divisors divisors]) → ticks?
  number : exact-positive-integer? = (ticks-default-number)
  base : (and/c exact-integer? (>=/c 2)) = 10
  divisors : (listof exact-positive-integer?) = '(1 2 4 5)
布局函数、格式函数和组合 ticks 用于均匀间隔的刻度。

为了布置刻度, linear-ticks-layout 找到最接近坐标轴区间大小的 base 的幂, 选择一个简单的第一个刻度,然后使用 divisors 选择一个跳过长度, 在不超过 number 的情况下使刻度的数量最大化。 关于非默认参数的战略使用, 请参见 bit/byte-ticks, currency-ticks 和 fraction-ticks 。 默认参数对应于标准的 1-2-5-in-base-10 规则,几乎在所有的绘图刻度布局中使用。

为了格式化刻度线, linear-ticks-format 使用 real->plot-label 传递 scientific? 的值, 并使用 digits-for-range 来确定小数点扩展中的最大小数点。

Changed in version 1.1 of package plot-gui-lib: 为 linear-ticks-format 和 linear-ticks 添加了 #:scientific? 参数。

6.2.2 对数刻度

procedure

  base : (and/c exact-integer? (>=/c 2)) = 10

procedure

(log-ticks-format [#:base base]    
  #:scientific? scientific?) → ticks-format/c
  base : (and/c exact-integer? (>=/c 2)) = 10
  scientific? : #t

procedure

(log-ticks [#:number number    
  #:base base]    
  #:scientific? scientific?) → ticks?
  number : exact-positive-integer? = (ticks-default-number)
  base : (and/c exact-integer? (>=/c 2)) = 10

布局函数、格式函数和组合 ticks 用于指数间隔的主要刻度。 (中间的小刻度是均匀间隔的)。 对 log-transform 轴使用这些刻度, 因为当指数间隔的刻度位置被 log-transform 时,它们会变成均匀的间隔。

  t : ticks?
