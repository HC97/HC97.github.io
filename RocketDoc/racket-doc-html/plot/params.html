8.1 兼容性

parameter

(plot-deprecation-warnings?) → boolean?

(plot-deprecation-warnings? warnings) → void?
  warnings : boolean?
 = #f
当 #t 时,在第一次使用 mix, line, contour, shade, surface 或 plot 或 plot3d 的关键字参数时, 会向 current-error-port 打印一个弃用警告,这只是为了向后兼容。

(plot-new-window?) → boolean?

(plot-new-window? new-window?) → void?

当 #t 时, plot 和 plot3d 为每个绘图打开一个新窗口,而不是返回一个 image-snip% 。

命令行 Racket 的用户不能显示图像剪辑,应在使用

(plot-new-window? #t)

8.3 一般外观

parameter

(plot-aspect-ratio) → (or/c (and/c rational? positive?) #f)

(plot-aspect-ratio ratio) → void?
  ratio : (or/c (and/c rational? positive?) #f)
 = #f
控制绘图区的长宽比,独立于整个绘图的宽度和高度。

当长宽比为 #f 时,绘图区填满整个绘图区,只留下空间给轴标签和标题。

当长宽比为正数时,绘图区域将保持这个长宽比,可能会在绘图周围留下空白区域。

当需要保持长宽比以使绘图输出看起来正确时,这个功能很有用,例如在绘制一个圆时:

> (parameterize ([plot-aspect-ratio 1]
                 [plot-background "lightyellow"])
    (plot (polar (lambda (t) 1)) #:width 400 #:height 200))

Added in version 8.1 of package plot-gui-lib.

parameter

(plot-legend-layout)

 → (list/c (or/c 'columns 'rows) positive-integer? (or/c 'equal-size 'compact))
(plot-legend-layout layout) → void?
  layout : (list/c (or/c 'columns 'rows) positive-integer? (or/c 'equal-size 'compact))
 = '(columns 1 equal-size)
定义图例中各个条目的放置方式。 这是一个包含三个元素的列表:

  • 放置方向 ('columns 或 'rows)

  • 列或行的数量

  • 是否所有条目都有相同的尺寸 ('equal-size), 或者条目只占用最小尺寸 ('compact)

例如,值 '(columns 1 equal-size) 将把图例条目从上到下垂直放置,所有条目将有相同的高度。 值 '(rows 2 'compact) 将把图例条目水平放置在两行上–当图例放置在绘图的顶部或底部时, 这种类型的布局很有用。

Added in version 7.9 of package plot-gui-lib.
