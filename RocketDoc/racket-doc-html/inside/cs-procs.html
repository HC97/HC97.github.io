4 调用程序 (CS)

作为 Racket 的入口点,C 程序通常应该通过使用 racket_apply 来调用 Racket 过程, 在Racket 主位的初始线程中调用该过程。 Chez Scheme 的入口点,如 Scall0 和 Scall , 直接调用任何 Racket 线程之外的过程,这将无法正确使用 Racket 的设施,如线程、参数、延续或延续标记。

本节中的函数是作为 Racket 的入口,但不是作为 re-entry 点。 当 Racket 调用一个C函数,而该函数又反过来调用 Racket 时, 最好的方法是使用 FFI(见 Racket 外国接口) , 这样 C 函数的调用就会收到一个被包装成普通 C 函数回调的 Racket 回调。 这样,FFI 可以处理 Racket 和 C 之间的边界交叉细节。

ptr

 

racket_apply

(

ptr proc,

 

 

 

 

ptr arg_list)

将 Racket 过程 proc 应用于参数列表 arg_list 。 该过程在主 Racket 地方的原始 Racket 线程中被调用。 应用 proc 时不能引发异常或以其他方式逃避对 proc 的调用。

结果是一个结果值的列表,其中来自 proc 的一个结果会导致 racket_apply 返回一个长度为 1 的列表。

在调用 proc 的过程中,其他 Racket 线程可以运行。 在 proc 产生结果的时候,主 Racket 地方的所有 Racket 线程调度都被暂停。 不会发生垃圾回收,所以其他 Racket 地方可以阻塞等待垃圾回收。

将 Chez Scheme 程序 proc 应用于 0、1、2 或 3 个参数。 请注意,不是所有的 Racket 程序都是 Chez Scheme 程序。 (例如,具有 prop:procedure 的结构类型的实例不是 Chez Scheme 过程)。

该过程是在任何 Racket 线程之外调用的,在调用 proc 的过程中,其他 Racket 线程不会被安排。 可能会发生垃圾回收。

与 Scall0 类似,但这些函数被依次用来对任意数量的参数应用 Chez Scheme 过程。 首先, Sinitframe 被调用,参数的数量是多少。 然后,用 Sput_arg 安装每个参数,其中 i 参数表示 argumenrt 的位置, arg 是参数值。 最后,用程序和参数的数量(必须与提供给 Sinitframe 的数量一致) 调用 Scall 。
