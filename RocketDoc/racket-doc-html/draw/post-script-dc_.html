class

post-script-dc% : class?

  superclass: object%

  extends: dc<%>
post-script-dc% 对象是一个 PostScript 设备上下文,它可以在任何平台上写入 PostScript 文件。 也请参见 ps-setup% 和 pdf-dc% 。

Be sure to use the following methods to start/end drawing:

  • start-doc

  • start-page

  • end-page

  • end-doc

Attempts to use a drawing method outside of an active page raises an exception.

另请参见 printer-dc% 。

constructor

(new post-script-dc% 
    [[interactive interactive] 
    [parent parent] 
    [use-paper-bbox use-paper-bbox] 
    [as-eps as-eps] 
    [width width] 
    [height height] 
    [output output]]) 
 → (is-a?/c post-script-dc%)
  interactive : any/c = #t
  parent : (or/c (is-a?/c frame%) (is-a?/c dialog%) #f) = #f
  use-paper-bbox : any/c = #f
  as-eps : any/c = #t
  width : (or/c (and/c real? (not/c negative?)) #f) = #f
  height : (or/c (and/c real? (not/c negative?)) #f) = #f
  output : (or/c path-string? output-port? #f) = #f
如果 interactive 为真,用户会得到一个设置打印参数的对话框(见 get-ps-setup-from-user); 得到的配置被安装为当前配置)。 如果用户选择打印到一个文件(在 Windows 和 Mac OS 上是唯一的可能性),会给出另一个对话框来选择文件名。 如果用户在这两个对话框中都点击了取消,那么 ok? 就会返回 #f 。

如果 parent 不是 #f, 它就会被用作配置对话框的父窗口。

如果 interactive 是 #f, 那么就会使用 current-ps-setup 返回的设置。 如果 get-file 方法返回 #f 并且 output 为 #f, 那么文件对话框仍然会呈现给用户,在这种情况下,用户可以点击 Cancel , 这样 ok? 就会返回 #f 。

如果 use-paper-bbox 是 #f, 那么输出的 PostScript 边框是由 width 和 height 决定的(使用 ceiling 向上取整)。 如果 use-paper-bbox 不是 #f, 那么边界框就由当前的纸张尺寸决定(由 current-ps-setup 指定)。 当 width 或 height 为 #f 时,那么相应的尺寸由纸张尺寸决定, 即使 use-paper-bbox 为 #f 。

如果 as-eps 是 #f, 那么生成的 PostScript 不包括封装的 PostScript (EPS)标头,而是包括一个通用的PostScript标头。 只有当 as-eps 为 #f 时,才会使用 current-ps-setup 指定的边距和翻译因子。 如果 as-eps 为真,那么生成的 PostScript 包括一个标识为 EPS 的头。

当 output 不是 #f 时,文件模式的输出被写入 output 中。 如果 output 为 #f, 那么目的地将通过 current-ps-setup 或提示用户输入路径名来确定。 当 output 为端口时,数据由一个线程写入 port , 该线程由 post-script-dc% 实例创建; 如果该写入线程的监护人被关闭,调用 end-doc 可以恢复写入端口的线程, 并以 thread-resume 和 (current-thread) 作为第二个参数。

参见 ps-setup% 和 current-ps-setup 。 一个特定的 post-script-dc% 对象的设置在该对象被创建时被固定为当前配置中的值(当 interactive 为真时, 在用户交互式调整后)。
