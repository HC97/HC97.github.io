class

bitmap% : class?

  superclass: object%

一个 bitmap% 对象是一个基于像素的图像,可以是单色的,也可以是彩色的,还可以是带有阿尔法通道的彩色图像。 参见 make-bitmap, make-platform-bitmap, make-screen-bitmap (来自 racket/gui/base), make-bitmap in canvas% (来自 racket/gui/base), 和 可移植性和位图变体。

一个位图有一个 backing scale ,它是对应于位图的一个绘图单位的像素数, 当位图被用作绘图的目标时,或者当位图被绘制到另一个上下文中时。 例如,在 Mac OS 上,当主显示器处于 Retina 模式时, make-screen-bitmap 返回一个支持比例为 2.0 的位图。 在 Windows 和 Unix 上,屏幕位图的背景比例对应于全系统的文本比例 (见 屏幕分辨率和文本缩放) 。 一个单色的位图总是有一个 1.0 的背景比例。

一个位图可以通过 file/convertible 协议转换为 'png-bytes 。

(make-object bitmap% bits width height) → (is-a?/c bitmap%)
  bits : bytes?

  height : exact-positive-integer?
make-bitmap, make-monochrome-bitmap 和 read-bitmap 函数创建 bitmap% 例, 但它们也比直接用 bitmap% 的 make-object 更受欢迎,因为这些函数的重载较少,而且它们默认启用 alpha 通道。 另见 可移植性和位图变体.

当提供 width 和 height 时: 创建一个新的位图。 如果 monochrome? 为真,该位图是单色的; 如果 monochrome? 为 #f 且 alpha? 为真,该位图有一个 alpha 通道; 否则,该位图是彩色的,没有 alpha 通道。 backing-scale 参数设置位图的 backing scale, 如果 monochrome 为真,它必须是 1.0 。 位图的初始内容是 "空" 的:全部是白色,如果是有阿尔法通道的位图,则阿尔法为零。

当 in 被提供时: 从一个文件格式创建一个位图,其中 kind 指定了格式。 详见 load-file 。 backing-scale 参数设置位图的 backing scale , 这样位图的尺寸(由 get-width 和 get-height 报告)就是 in 中的位图尺寸的 ceiling 除以 backing-scale; 如果位图是单色的或带掩码的,支持比例必须是 1.0 。

当提供一个 bits 字节字符串时: 从一个位值数组创建一个单色位图, 其中 bits 中的每个字节指定 8 个位,并添加填充位,以使每个位图行开始于字符边界。 1 比特值表示黑色, 0 表示白色。 如果 width 乘以 height 大于 bits 长度的 8 倍, an exn:fail:contract exception is raised 。

关于 save-data-from-file? 的信息见 get-data-from-file 。

Changed in version 1.1 of package draw-lib: 增加了 backing-scale 的可选参数。
Changed in version 1.17: 增加了 save-data-from-file?

当 unscaled? 为 #f 时,产生与 get-argb-pixels in bitmap-dc% 相同的结果, 但位图不必被选入DC(即使位图被选入另一个 DC,作为按钮标签附加,等)。

如果位图的 backing scale 不是 1.0 ,并且 unscaled? 为 true, 那么结果就对应于位图的像素忽略了 backing scale 。 在这种情况下, x, y, width 和 height 实际上是像素,而不是绘图单位。

Changed in version 1.1 of package draw-lib: 增加了 #:unscaled? 可选参数。

获取位图的高度,单位是绘图单位(如果 backing scale 为1.0,则与像素相同)。

method

(send a-bitmap get-loaded-mask) → (or/c (is-a?/c bitmap%) #f)

返回一个与此位图一起存储的遮罩位图。

当用 'gif/mask 或 'unknown/mask 加载一个 GIF 文件, 并且该文件包含一个透明的 "颜色" 时,会生成一个掩码位图来识别透明像素。 遮罩位图是单色的,在加载的位图是透明的地方有白色像素,其他地方有黑色像素。

当用 'png/mask 或 'unknown/mask 加载一个 PNG 文件, 并且该文件包含一个遮罩或 alpha 通道时,会生成一个遮罩位图以识别遮罩或 alpha 通道。 如果文件包含一个只有极端值的遮罩或阿尔法通道,遮罩位图是单色的,否则是灰度的(代表阿尔法通道的倒置)。

当一个 XPM 文件以 'xpm/mask 或 'unknown/mask 加载时,会生成一个蒙版位图来指示哪些像素被设置。

当 'unknown/alpha 和类似的模式被用来加载一个位图时,透明度信息是由一个 alpha 通道来表示的,而不是由一个掩码位图。

与阿尔法通道不同,遮罩位图 不 会被绘图程序自动使用。 遮罩位图可以被提取并作为一个遮罩明确地提供(例如,作为 draw-bitmap 的第六个参数)。 如果遮罩的尺寸与保存的位图相同,那么在将位图保存为 'png 时, save-file 会使用遮罩位图。 当位图是一个控制标签时,掩码位图也会自动使用。

method

(send a-bitmap get-width) → exact-positive-integer?

获取位图的宽度,单位是绘图单位(与 backing scale 为 1.0 的像素相同)。

method

(send a-bitmap load-file 
  in 
  [kind 
  bg-color 
  complain-on-failure? 
  #:save-data-from-file? save-data-from-file?]) 
 → boolean?
  in : (or/c path-string? input-port?)
  kind : 
(or/c 'unknown 'unknown/mask 'unknown/alpha
      'gif 'gif/mask 'gif/alpha
      'jpeg 'jpeg/alpha
      'png 'png/mask 'png/alpha
      'xbm 'xbm/alpha 'xpm 'xpm/alpha
      'bmp 'bmp/alpha)
 = 'unknown
  bg-color : (or/c (is-a?/c color%) #f) = #f
  complain-on-failure? : any/c = #f
  save-data-from-file? : any/c = #f
从一个从 in 读取的文件格式中加载一个位图, 除非该位图是由 make-platform-bitmap, make-screen-bitmap 或 make-bitmap in canvas% 产生的(在这种情况下会引发 an exn:fail:contract exception is raised )。 如果该位图正被一个 bitmap-dc% 对象或一个控件使用,图像数据就不会被加载。 位图会改变其大小和深度以匹配加载的图像。 如果在读取文件格式时遇到错误,只有在 complain-on-failure? 为真时(这 不 是默认的)才会引发异常。

kind 参数指定了文件的格式:

  • 'unknown — 检查文件以确定其格式;创建一个没有 alpha 通道的单色或彩色位图

  • 'unknown/mask — 和 'unknown一样,但见 get-loaded-mask

  • 'unknown/alpha — 和 'unknown 一样,但是如果位图是彩色的, 它有一个alpha通道,图像文件的透明度记录在 alpha 通道中

  • 'gif — 载入一个 GIF 位图文件,创建一个彩色位图

  • 'gif/mask — 像 'gif, 但见 get-loaded-mask

  • 'gif/alpha — 像 'gif, 但有一个 alpha 通道

  • 'jpeg — 载入一个 JPEG 位图文件,创建一个彩色位图

  • 'jpeg/alpha — 像 'jpeg, 但有一个 alpha 通道

  • 'png — 载入一个 PNG 位图文件,创建一个彩色或单色位图

  • 'png/mask — 像 'png, 但见 get-loaded-mask

  • 'png/alpha — 像 'png, 但总是彩色的,并且有一个 alpha 通道

  • 'xbm — 载入一个X位图 (XBM) 文件;创建一个单色位图

  • 'xbm/alpha — 像 'xbm, 但创建一个带有 alpha 通道的彩色位图

  • 'xpm — 加载一个 XPM 位图文件,创建一个彩色位图

  • 'xpm/alpha — 像 'xpm, 但有一个 alpha 通道

  • 'bmp — 加载一个Windows位图(BMP)文件,创建一个彩色位图

  • 'bmp/alpha — 像 'bmp, 但有一个 alpha 通道

XBM 图像总是作为单色位图加载。 一个没有遮罩或 alpha 通道的 1 位灰度 PNG 也是作为单色位图加载的。 任何其他格式的图像总是作为彩色位图加载。

对于 PNG 和 BMP 的加载,如果 bg-color 不是 #f, 那么在加载图像时, 它会与文件的 alpha 通道或掩码(如果有的话)结合起来; 在这种情况下,不会生成单独的掩码位图,alpha通道会填充位图, 即使为格式指定了 'unknown/mask, 'png/mask 。 如果格式被指定为 'unknown 或 'png ,并且没有指定 bg-color, 则会查询 PNG 文件,以确定加载时使用的背景色,如果文件中没有指明背景色,则使用白色。

在 所有 PNG 加载模式中,如果文件提供了一个伽马值,就会应用伽马校正,否则就不应用伽马校正。 当前显示器的伽玛系数由 SCREEN_GAMMA 环境变量决定,如果该变量被定义的话。 如果偏好和环境变量都未定义,则使用特定平台的默认值。

创建位图后,只有当位图的 backing scale 为 1.0 时,才能使用 load-file 。

关于 save-data-from-file? 的信息见 get-data-from-file

Changed in version 1.17 of package draw-lib: 增加了 save-data-from-file?

method

(send a-bitmap save-file name    
  kind    
  [quality    
  #:unscaled? unscaled?]) → boolean?
  name : (or/c path-string? output-port?)
  kind : (or/c 'png 'jpeg 'xbm 'xpm 'bmp)
  quality : (integer-in 0 100) = 75
  unscaled? : any/c = #f
写一个位图到指定的文件或输出流。

kind 参数决定了被创建的文件的类型,其中之一:

  • 'png — 保存一个 PNG 文件

  • 'jpeg — 保存一个 JPEG 文件

  • 'xbm — 保存一个 X 位图 (XBM) 文件

  • 'xpm — 保存一个 XPM bitmap 文件

  • 'bmp — 保存一个 Windows 位图文件

quality 参数只用于保存为 'jpeg, 在这种情况下, 它指定了图像精度(高质量与 bitmap% 对象的内容更精确地匹配)和大小(低质量更小)之间的权衡。

当保存为 'png 时,如果 get-loaded-mask 返回一个与此相同大小的位图, 一个灰度版本将作为 alpha 通道包含在 PNG 文件中。

一个保存为 'png 的单色位图,如果没有遮罩位图, 会产生一个1位灰度的 PNG 文件(当用 load-file 读取时, 会创建一个单色 bitmap% 对象。)

如果位图的 backing scale 不是 1.0,那么在保存之前,除非 unscaled? 为 true, 否则它将被有效地转换为每个绘图单位的单一像素。

Changed in version 1.1 of package draw-lib: 增加了 #:unscaled? 可选参数。
