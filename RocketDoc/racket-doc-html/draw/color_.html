class

color% : class?

  superclass: object%

颜色是一个代表红-绿-蓝(RGB)三原色组合的对象,加上一个不透明的 "alpha"。 颜色的每个红、绿、蓝成分都是 0 到 255 范围内的精确整数,阿尔法值是 0 到 1 之间的实数,包括在内。 例如,(0,0,0,1.0)是纯黑色,(255,255,255,1.0)是纯白色,(255,0,0,1.0)是纯红色,而(255,0,0,0.5)是半透明的红色。

关于使用颜色名称获取颜色对象的信息,请参见 color-database<%> ,也请参见 make-color 。

constructor

(make-object color%) → (is-a?/c color%)

(make-object color% red green blue [alpha]) → (is-a?/c color%)
  red : byte?
  green : byte?
  blue : byte?
  alpha : (real-in 0 1) = 1.0
(make-object color% color-name-or-obj) → (is-a?/c color%)
  color-name-or-obj : (or/c string? (is-a?/c color%))
创建一个新的颜色。

如果向构造函数提供了三个或四个参数,就会用这些 RGB 和 alpha 值创建颜色。

如果提供了一个单一的 color% 对象,那么颜色将以与给定颜色相同的 RGB 和 alpha 值被创建。

如果提供了一个字符串,那么它将被传递给 color-database<%> 的 find-color 方法来寻找一个颜色(如果颜色不在 color-database<%> 的 get-names 方法的结果中, 则表示错误)。

如果没有提供参数,新的颜色就是黑色。

method

(send a-color red) → byte?

返回颜色的红色分量。

method

(send a-color green) → byte?

返回颜色的绿色分量。

method

(send a-color blue) → byte?

返回颜色的蓝色分量。

method

返回颜色的 alpha 分量(即不透明度)。

method

(send a-color set red green blue [alpha]) → void?

  red : byte?
  green : byte?
  blue : byte?
  alpha : (real-in 0 1) = 1.0
设置颜色的四个(红、绿、蓝、阿尔法)分量值。

method

(send a-color copy-from src) → (is-a?/c color%)

  src : (is-a?/c color%)
将另一个颜色对象的 RGB 值复制到这个对象中,并将这个对象作为结果返回。

method

(send a-color is-immutable?) → boolean?

如果颜色对象是不可改变的,返回 #t 。

参见 make-color 和 find-color in color-database<%> 。

method

(send a-color ok?) → #t

返回 #t ,表示颜色对象是有效的。

(历史上,结果可能是 #f, 但现在颜色对象总是有效的)。

6.1 平等性

我们可以使用 equal? 来比较 color% 的实例。 如果两个 color% 实例的红、绿、蓝和 alpha 值相等,它们就是相等的。 也就是说,一个可变的和一个不可变的 color% 实例是相等的,只要它们的值相等。
