class

dc-path% : class?

  superclass: object%

路径是一组由曲线定义的图形。 路径可以使用 dc<%> 对象的 draw-path 方法,将路径的曲线画成直线, 或者填充路径曲线所包围的区域,或者两者兼而有之。 路径也可以与 region% 对象的 set-path 方法一起使用,生成一个由路径的曲线限定的区域。

一条路径由零条或多条 closed sub-paths 组成,可能还有一条 open sub-path 。 一些 dc-path% 方法扩展开放子路径,一些 dc-path% 方法关闭开放子路径, 一些 dc-path% 方法添加封闭子路径。 这种绘制表述的方法是从 [Adobe99] 继承下来的。

当路径被画成一条线时,封闭的子路径被画成一个封闭的图形,类似于一个多边形。 开放式子路径的起点和终点是不相连的,类似于 draw-lines in dc<%> 绘制的线条。

当路径被填充或作为区域使用时,开放的子路径(如果有的话)被当作封闭的路径处理。 路径的内容是通过 'even-odd 或 'winding 确定的,这是在填充路径或用于生成区域时选择的。

路径不与任何特定的 dc<%> 对象相连,所以设置 dc<%> 的原点或比例不会影响路径操作。 相反, dc<%> 的原点和比例在绘制路径或用于设置区域时适用。

constructor

(new dc-path%) → (is-a?/c dc-path%)

创建一个不包含子路径(也不包含 open sub-path) 的新路径。

method

(send a-dc-path append path) → void?

  path : (is-a?/c dc-path%)
将 path 的子路径添加到 a-dc-path 中。 path 的 Closed sub-paths 作为 closed sub-paths 被添加到 a-dc-path 中。 如果两条路径都有一条 open sub-path ,那么这条路径的子路径将由给定路径的 open sub-path 扩展, 从这条路径的当前终点到给定路径的起点添加一条线。 如果只有一条路径有 open sub-path, 那么它就成为(或保持)这条路的 open sub-path 。

method

(send a-dc-path arc x    
  y    
  width    
  height    
  start-radians    
  end-radians    
  [counter-clockwise?]) → void?
  x : real?
  y : real?
  width : real?
  height : real?
  start-radians : real?
  end-radians : real?
  counter-clockwise? : any/c = #t
用一条对应于椭圆的一段的曲线来扩展或开始路径的 open sub-path 。 如果 width 和 height 都是非负数, 那么椭圆就是由左上角为 (x, y) 、 尺寸为 width 乘以 height 的矩形所限定的; 如果 width 为负数,那么矩形的右边缘为 x, 椭圆的宽度为 (abs width), 而负的 height 同样使 y 为椭圆的底边,高度为 (abs height) 。 对负的 width 和 height 的支持有助于避免最终目的地的对齐绘制的舍入问题, 因为 arc 将其输入减少为一连串的曲线。 相反, draw-arc in dc<%> 可以自动纠正舍入问题,因为绘图模式是立即知道的。 椭圆部分从角度 start-radians 开始 (0 是 3 点钟方向,半 π 是 12 点钟方向), 一直到角度 end-radians; 如果 counter-clockwise? 为真, 那么弧线从 start-radians 到 end-radians 逆时针运行,否则是顺时针运行。

如果路径没有 open sub-path, 则以弧的起点开始一个新的路径。 否则,弧线会延伸现有的子路径,现有的路径会用一条线连接到弧线的起始点。

method

(send a-dc-path close) → void?

关闭路径的 open sub-path 。 如果路径没有 open sub-path, an exn:fail:contract exception is raised 。

method

(send a-dc-path curve-to x1 y1 x2 y2 x3 y3) → void?

  x1 : real?
  y1 : real?
  x2 : real?
  y2 : real?
  x3 : real?
  y3 : real?
用贝塞尔曲线将路径的 open sub-path 延伸到给定的点 (x3,y3), 用 (x1, y1) 和 (x2,y2) 作为控制点。 如果该路径没有 open sub-path, an exn:fail:contract exception is raised 。

method

(send a-dc-path ellipse x y width height) → void?

  x : real?
  y : real?
  width : (and/c real? (not/c negative?))
  height : (and/c real? (not/c negative?))
关闭 open sub-path (如果有的话),并添加一个 closed sub-path , 该路径代表一个由矩形限定的椭圆,矩形的左上角是 (x, y) , 其尺寸是 width 乘以 height 。 (这个方便的方法是用 close 和 arc 实现的。)

method

(send a-dc-path get-bounding-box) → 
real? real? real? real?
返回一个包围着路径上的点的矩形。返回值是矩形的左边、上面、宽度和高度。

对于路径中的曲线,边界框包围了两个控制点以及起点和终点。 因此,边界框并不总是紧紧地包围着路径。

method

(send a-dc-path get-datum) → 
(listof (listof vector?))
(listof vector?)
返回一个以向量列表形式表示的路径。 第一个结果是一个列表,包含每个 closed sub-path 的列表,第二个结果是 open sub-path 的列表。 如果路径没有 open sub-path ,则第二个结果是空列表。

每个代表子路径的列表以两个数字的向量开始,代表路径的起点。 之后的每个元素要么是两个数字的向量,代表连接前一点和新一点的直线,要么是六个数字的向量,代表连接前一点和新一点的曲线; 在后一种情况下,向量中的第五和第六个数字代表曲线的结束点,第一和第二个数字代表曲线的第一个控制点, 第三和第四个数字代表曲线的第二个控制点。

Added in version 1.8 of package draw-lib.

method

(send a-dc-path line-to x y) → void?

  x : real?
  y : real?
用一条线将路径的 open sub-path 延伸到给定的点。 如果路径没有 open sub-path, an exn:fail:contract exception is raised 。

method

(send a-dc-path lines points    
  [xoffset    
  yoffset]) → void?
  points : 
(or/c (listof (is-a?/c point%))
      (listof (cons/c real? real?)))
  xoffset : real? = 0
  yoffset : real? = 0
用一连串的线扩展路径的 open sub-path 到指定的点。 一对点被视为一个点,这对点的 car 是 x 值, cdr 是 y 值。 如果该路径没有 open sub-path, an exn:fail:contract exception is raised 。 (这个方便的方法是以 line-to 的形式实现的)。

method

(send a-dc-path move-to x y) → void?

  x : real?
  y : real?
在关闭 open sub-path 后,如果有的话,以给定的初始点开始一个新的 open sub-path 。

method

(send a-dc-path open?) → boolean?

如果该路径有一个 open sub-path,返回 #t ,否则返回 #f 。

method

(send a-dc-path rectangle x y width height) → void?

  x : real?
  y : real?
  width : (and/c real? (not/c negative?))
  height : (and/c real? (not/c negative?))
关闭 open sub-path (如果有的话),并添加一个代表矩形的封闭路径, 这个矩形的左上角是 (x, y) ,其尺寸是 width 乘以 height 。 (这个方便的方法是用 close, move-to 和 line-to实现的。)

method

(send a-dc-path reset) → void?

移除路径的所有子路径。

method

(send a-dc-path reverse) → void?

颠倒所有子路径中所有点的顺序。 如果该路径有一个 open sub-path, 那么起点就变成了终点,对 open sub-path 的扩展就建立在这个新的终点上。 在确定 'winding 下的路径内容时,颠倒一个 closed sub-path 会影响它与其他子路径的结合。

method

(send a-dc-path rotate radians) → void?

  radians : real?
调整路径中的所有点(包括所有子路径),将它们围绕 (0, 0) 逆时针旋转 radians 。 未来添加到路径中的点不会被这个调用所旋转。

method

(send a-dc-path rounded-rectangle x    
  y    
  width    
  height    
  [radius]) → void?
  x : real?
  y : real?
  width : (and/c real? (not/c negative?))
  height : (and/c real? (not/c negative?))
  radius : real? = -0.25
关闭 open sub-path (如果有的话),并添加一个 closed sub-path , 代表一个左上角为 (x y) 的圆角矩形,其尺寸为 width 乘 height 。 (这个方便的方法是用 close, move-to、 arc 和 line-to 实现的)。

如果 radius 是正的,该值被用作圆角的半径。 如果 radius 是负的,绝对值被用作矩形的最小尺寸的 proportion 。

如果 radius 小于 -0.5 或超过 width 或 height 的一半, an exn:fail:contract exception is raised 。

method

(send a-dc-path scale x y) → void?

  x : real?
  y : real?
调整 路径中的所有点(包括所有子路径), 将每个 x 坐标乘以 x ,每个 y 坐标乘以 y 。 负数的缩放会将路径翻转到相应的轴上。以后增加的路径不会被这个调用所缩放。

method

(send a-dc-path text-outline font    
  str    
  x    
  y    
  [combine?]) → void?
  font : (is-a?/c font%)
  str : string?
  x : real?
  y : real?
  combine? : any/c = #f
关闭 open sub-path (如果有的话), 并使用 font 为轮廓 str 添加一个 closed sub-path 。 文本的左上方被定位在 x 和 y 处。 combine? 参数使字符间距和字符组合与 draw-text in dc<%> 一样。

method

(send a-dc-path transform m) → void?

  m : (vector/c real? real? real? real? real? real?)
通过应用 m 所代表的变换,调整路径中的所有点(包括所有子路径)。

参见 get-initial-matrix ,了解由向量 m 代表的矩阵的信息。

method

(send a-dc-path translate x y) → void?

  x : real?
  y : real?
调整路径中的所有点(包括所有子路径),然后将 x 向右移动, y 向下移动。 未来添加到路径中的点不会被这个调用所转换。
