1 概述

racket/draw 库提供了一个基于 PostScript 绘图模型的绘图 API。 它支持线条绘制、形状填充、位图复制、alpha 混合和仿射变换(即缩放、旋转和平移)。

参见 类和对象 , 了解 Racket 中的类和接口的介绍。

用 racket/draw 绘图需要一个 drawing context (DC), 它是 dc<%> 口的一个实例。 例如, post-script-dc% 类实现了一个 dc<%> 用于绘制 PostScript 文件, 而 bitmap-dc% 用于绘制位图。 当使用 racket/gui 库进行 GUI 时, canvas的 get-dc 方法返回一个 dc<%> 实例,用于绘制到 canvas 窗口。

关于在 GUI 窗口中绘图的介绍, 请参见 在画布上绘图 。

1.1 线条和简单形状

要在一个位图中绘图,首先用 make-bitmap 创建位图,然后创建一个 bitmap-dc% ,在新位图中绘图:

(define target (make-bitmap 30 30)) ; A 30x30 bitmap
(define dc (new bitmap-dc% [bitmap target]))

然后,使用 DC 上的 draw-line 等方法来绘制到位图中。例如,这个序列

(send dc draw-rectangle
      0 10   ; Top-left at (0, 10), 10 pixels down from top-left
      30 10) ; 30 pixels wide and 10 pixels high
(send dc draw-line
      0 0    ; Start at (0, 0), the top-left corner
      30 30) ; and draw to (30, 30), the bottom-right corner
(send dc draw-line
      0 30   ; Start at (0, 30), the bottom-left corner
      30 0)  ; and draw to (30, 0), the top-right corner

在一个较小的矩形上面画一个 "X" 到位图 target 中。 如果你用 (send target save-file "box.png" 'png) 把位图保存到一个文件中, "box.png" 包含 PNG 格式的图像

的 PNG 格式的图像。

像 draw-line 这样的画线操作使用 DC 当前的 pen 来画线。 笔有一个颜色、线宽和样式,其中笔的样式包括 'solid, 'long-dash 和 'transparent 。 像 draw-rectangle 这样的封闭形状的操作同时使用当前的笔和 DC 当前的 brush 。 笔刷有颜色和样式,其中笔刷样式包括 'solid, 'cross-hatch 和 'transparent 。

在DrRacket中,你可以使用 (require racket/gui) 和 (make-object image-snip% target) 在 DrRacket 交互窗口中查看位图, 而不是将 target 保存到文件中查看文件中的图像。

例如,在绘图操作前设置画笔和钢笔,在绿色矩形上画一个厚的、红色的 "X",边框是薄的、蓝色的:

(send dc set-brush "green" 'solid)
(send dc set-pen "blue" 1 'solid)
(send dc draw-rectangle 0 10 30 10)
(send dc set-pen "red" 3 'solid)
(send dc draw-line 0 0 30 30)
(send dc draw-line 0 30 30 0)

要画一个没有轮廓的填充形状,把笔设置为 'transparent 模式(有任何颜色和线宽)。 比如说、

(send dc set-pen "white" 1 'transparent)
(send dc set-brush "black" 'solid)
(send dc draw-ellipse 5 5 20 20)

默认情况下, bitmap-dc% 绘制的是实心像素,而不对形状的边界进行平滑处理。 要启用平滑,将平滑模式设置为 'smoothed 或 'aligned:

(send dc set-smoothing 'aligned)
(send dc set-brush "black" 'solid)
(send dc draw-ellipse 4 4 22 22) ; a little bigger

'aligned 模式和 'smoothed 模式之间的区别与位图中像素的相对粗糙的粒度有关。 从概念上讲,绘图坐标对应于像素之间的线,笔以线为中心。 在 'smoothed 模式下,在一条线上绘图会导致笔在线的两侧以一半的力度绘图,这对一支1像素的黑笔来说会产生以下结果:

1.2 笔、笔刷和颜色对象

DC 的 set-pen 和 set-brush 方法接受 pen% 和 brush% 对象, 它们将笔和刷子的设置组合在一起。

(require racket/math)
 
(define no-pen (new pen% [style 'transparent]))
(define no-brush (new brush% [style 'transparent]))
(define blue-brush (new brush% [color "blue"]))
(define yellow-brush (new brush% [color "yellow"]))
(define red-pen (new pen% [color "red"] [width 2]))
 
(define (draw-face dc)
  (send dc set-smoothing 'aligned)
 
  (send dc set-pen no-pen)
  (send dc set-brush blue-brush)
  (send dc draw-ellipse 25 25 100 100)
 
  (send dc set-brush yellow-brush)
  (send dc draw-rectangle 50 50 10 10)
  (send dc draw-rectangle 90 50 10 10)
 
  (send dc set-brush no-brush)
  (send dc set-pen red-pen)
  (send dc draw-arc 37 37 75 75 (* 5/4 pi) (* 7/4 pi)))
 
(define target (make-bitmap 150 150))
(define dc (new bitmap-dc% [bitmap target]))
 
(draw-face dc)

get-pen 和 get-brush 方法返回 DC 当前的笔和刷子,因此在为绘图而临时改变它们之后,可以恢复它们。

除了分组设置外, pen% 或 brush% 对象还包括直接使用 set-pen 或 set-brush 无法获得的额外设置。 例如,一支笔或画笔可以有一个 stipple, 这是一个位图,在绘图时用来代替纯色。 例如,如果 "water.png" 的图像为

那么它就可以用 read-bitmap 加载并安装为 blue-brush 的 stipple:

(send blue-brush set-stipple (read-bitmap "water.png"))
(send dc erase)
(draw-face dc)

沿着类似的思路, color% 对象让你通过它的红、绿、蓝成分来指定一个颜色,而不是一个内置的颜色名称。 由于 color% 的初始化方式是重载的, 使用 make-object 而不是 new 来实例化 color%, 或者使用 make-color 函数:

(define red-pen
  (new pen% [color (make-color 200 100 150)] [width 2]))
(send dc erase)
(draw-face dc)

1.3 变换

任何提供给绘图命令的坐标或长度都被 DC 当前的变换矩阵所变换。 变换矩阵可以缩放图像,以偏移量绘制图像,或者旋转所有绘图。 变换可以直接设置,也可以用 scale, translate 或 rotate 等方法对当前的变换进行进一步变换::

(send dc erase)
(send dc scale 0.5 0.5)
(draw-face dc)
(send dc rotate (/ pi 2))
(send dc translate 0 150)
(draw-face dc)
(send dc translate 0 -150)
(send dc rotate (/ pi 2))
(send dc translate 150 150)
(draw-face dc)
(send dc translate -150 -150)
(send dc rotate (/ pi 2))
(send dc translate 150 0)
(draw-face dc)

使用 get-transformation 方法获得 DC 的当前变换, 并使用 set-transformation 恢复已保存的变换(或任何仿射变换)。

1.4 绘制路径

像 draw-line 和 draw-rectangle 这样的绘图函数实际上是更通用的 draw-path 操作的便利函数。 draw-path 操作需要一个 path, 它描述了一组要用笔绘制的线段和曲线, 如果是封闭的线段和曲线集,则用当前画笔填充。

dc-path% 的一个实例持有一个路径。 从概念上讲,一个路径有一个当前笔的位置,这个位置可以通过 move-to, line-to 和 curve-to 等方法进行操作。 move-to 方法开始一个子路径, line-to 和 curve-to 则扩展它。 close 方法将笔从当前位置直线移动到起始位置,完成子路径并形成一个可以用画笔填充的封闭路径。 一个 dc-path% 对象可以有多个封闭的子路径和一个最终的开放路径,其中开放路径只能用笔来画。

比如说

(define zee (new dc-path%))
(send zee move-to 0 0)
(send zee line-to 30 0)
(send zee line-to 0 30)
(send zee line-to 30 30)

除了核心的 move-to, line-to,curve-to 和 close 方法外, dc-path% 还包括很多方便的方法,比如 ellipse ,用于在路径上添加一个封闭的椭圆子路径。

1.5 文本

使用 draw-text 方法绘制文本,该方法需要一个要绘制的字符串和绘制文本的左上角的位置:

(define text-target (make-bitmap 100 30))
(define dc (new bitmap-dc% [bitmap text-target]))
(send dc set-brush "white" 'transparent)
 
(send dc draw-rectangle 0 0 100 30)
(send dc draw-text "Hello, World!" 5 1)

用于绘制文本的字体是由 DC 的当前字体决定的。 字体由 font% 对象描述,并通过 set-font 安装。 绘制文本的颜色与笔或画笔分开,可以用 set-text-foreground 来设置。

(send dc erase)
(send dc set-font (make-font #:size 14 #:family 'roman
                             #:weight 'bold))
(send dc set-text-foreground "blue")
(send dc draw-rectangle 0 0 100 30)
(send dc draw-text "Hello, World!" 5 1)

要计算绘制的文本所使用的尺寸,可以使用 get-text-extent, 它返回四个值:总宽度、总高度、基线和总高度之差,以及一行中文本上方的额外空间(如果有)。 例如, get-text-extent 的结果可以用来将文本定位在一个盒子的中心:

(send dc erase)
(send dc draw-rectangle 0 0 100 30)
(define-values (w h d a) (send dc get-text-extent "Hello, World!"))
(send dc draw-text "Hello, World!" (/ (- 100 w) 2) (/ (- 30 h) 2))

1.6 阿尔法通道和阿尔法混合

当你创建或 erase 一个位图时,其内容是虚无的。"无" 并不等同于白色;它是指没有绘画。 例如,如果你把上一节中的 text-target 用 draw-bitmap 复制到另一个 DC 上, 那么黑色的矩形和蓝色的文字就被转移了,而背景则被单独留下:

(define new-target (make-bitmap 100 30))
(define dc (new bitmap-dc% [bitmap new-target]))
(send dc set-pen "black" 1 'transparent)
(send dc set-brush "pink" 'solid)
 
(send dc draw-rectangle 0 0 100 30)
(send dc draw-bitmap text-target 0 0)

关于位图的哪些像素被绘制(与 "无" 相对)的信息是位图的 alpha channel 。 不是所有的 DC 都保持一个 alpha 通道,但用 make-bitmap 创建的位图默认保持一个 alpha 通道。 用 read-bitmap 加载的位图通过位图的 alpha 通道在图像文件中保留透明度。

阿尔法通道并不是全有或全无。 例如,当边缘文字被 draw-text 抗锯齿化时,像素是部分透明的。 当这些像素被转移到另一个 DC 时,部分透明的像素会与目标像素混合,这个过程称为 alpha blending 。 此外, DC 有一个适用于所有绘图操作的阿尔法值;阿尔法值为 1.0 对应于实心绘图, 阿尔法值为 0.0 使绘图没有效果,而介于两者之间的值使绘图变成半透明。

例如,在调用 draw-bitmap 之前将 DC 的 alpha 值设置为 0.25 , 会使 "Hello, World!" 位图的蓝色和黑色在与目标图像混合时具有四分之一强度:

(send dc erase)
(send dc draw-rectangle 0 0 100 30)
(send dc set-alpha 0.25)
(send dc draw-bitmap text-target 0 0)

1.7 剪裁

除了控制绘图操作的不透明度外 DC 还有一个 clipping region ,将所有绘图限制在该区域内。 在最简单的情况下,剪裁区域对应于一个封闭的路径,但它也可以是两个路径的联合、相交、相减或相斥。

例如,一个剪裁区域可以被设置为三个圆,以剪裁一个矩形的绘制( 0.25 alpha 仍然有效):

(define r (new region%))
(let ([p (new dc-path%)])
  (send p ellipse 0 0 35 30)
  (send p ellipse 35 0 30 30)
  (send p ellipse 65 0 35 30)
  (send r set-path p))
(send dc set-clipping-region r)
(send dc set-brush "green" 'solid)
(send dc draw-rectangle 0 0 100 30)

剪辑区域可以被看作是路径填充或用僵点绘制的一种方便的替代方法。 反过来说,带硬点的绘图可以被看作是对剪裁重复调用 draw-bitmap 的一种方便的替代。

然而,将区域与具有梯度的 pen% 对象相结合,不仅仅是一种便利,因为它允许我们以其他方式无法绘制的组合来绘制形状。 为了说明这一点,这里有一些代码,它画出了文本及其下面的反射。

; First compute the size of the text we're going to draw,
; using a small bitmap that we never draw into.
(define bdc (new bitmap-dc% [bitmap (make-bitmap 1 1)]))
(define str "Racketeers, ho!")
(define the-font (make-font #:size 24 #:family 'swiss
                            #:weight 'bold))
(define-values (tw th)
  (let-values ([(tw th ta td)
                (send dc get-text-extent str the-font)])
    (values (inexact->exact (ceiling tw))
            (inexact->exact (ceiling th)))))
 
; Now we can create a correctly sized bitmap to
; actually draw into and enable smoothing.
(send bdc set-bitmap (make-bitmap tw (* th 2)))
(send bdc set-smoothing 'smoothed)
 
; next, build a path that contains the outline of the text
(define upper-path (new dc-path%))
(send upper-path text-outline the-font str 0 0)
 
; next, build a path that contains the mirror image
; outline of the text
(define lower-path (new dc-path%))
(send lower-path text-outline the-font str 0 0)
(send lower-path transform (vector 1 0 0 -1 0 0))
(send lower-path translate 0 (* 2 th))
 
; This helper accepts a path, sets the clipping region
; of bdc to be the path (but in region form), and then
; draws a big rectangle over the whole bitmap.
; The brush will be set differently before each call to
; draw-path, in order to draw the text and then to draw
; the shadow.
(define (draw-path path)
  (define rgn (new region%))
  (send rgn set-path path)
  (send bdc set-clipping-region rgn)
  (send bdc set-pen "white" 1 'transparent)
  (send bdc draw-rectangle 0 0 tw (* th 2))
  (send bdc set-clipping-region #f))
 
; Now we just draw the upper-path with a solid brush
(send bdc set-brush "black" 'solid)
(draw-path upper-path)
 
; To draw the shadow, we set up a brush that has a
; linear gradient over the portion of the bitmap
; where the shadow goes
(define stops
  (list (list 0 (make-color 0 0 0 0.4))
        (list 1 (make-color 0 0 0 0.0))))
(send bdc set-brush
      (new brush%
           [gradient
            (new linear-gradient%
                 [x0 0]
                 [y0 th]
                 [x1 0]
                 [y1 (* th 2)]
                 [stops stops])]))
(draw-path lower-path)

1.8 可移植性和位图变体

绘图效果在不同的平台、不同的实现 dc<%> 的类或不同种类的位图之间并不能完全移植。 特别是字体和文字,可以在不同的平台和 DC 的类型之间变化,但画线所触及的精确的像素集也可以变化。

不同种类的位图会产生不同的结果:

  • 在 make-bitmap (或从 bitmap% 实例化)产生的位图上绘图,在不同平台上的绘制方式是最一致的。

  • 对由 make-platform-bitmap 产生的位图进行绘制时,尽可能地使用特定平台的绘制操作。 然而,在 Windows 上,由 make-platform-bitmap 产生的位图没有 alpha 通道, 它比由 make-bitmap 产生的位图使用更多的限制性资源(由于全系统、每个进程的 GDI 限制)。

    作为一个特定平台差异的例子,Mac OS 上的文本默认是用亚像素抗锯齿平滑的, 而 make-bitmap 结果中的文本平滑只使用灰色。 线条或曲线的绘制可能会接触到与 make-bitmap 产生的位图不同的像素,而且位图的缩放也可能不同。

    在 Windows 下处理 GDI 限制的一个可能的方法是在 make-platform-bitmap 的结果中绘图, 然后将绘图的内容复制到 make-bitmap 的结果中。 这种方法保留了 make-platform-bitmap 的绘制结果,但它只在绘制过程中保留了受限资源。

  • 从 racket/gui/base 向由 make-screen-bitmap 产生的位图绘图, 使用了与向 canvas% 实例绘图相同的特定平台绘图操作。 由 make-screen-bitmap 产生的位图在 Windows 或 Mac OS 上使用与 make-platform-bitmap 相同的特定平台绘图,但可能是按比例的,在 Unix 上它可能是按比例的或对 X11 服务器敏感。

    在 Mac OS 上,当主屏幕处于 Retina 模式时(在创建位图时),位图也会被内部缩放,以便一个绘图单位使用两个像素。 同样,在 Windows 或 Unix 上,当主屏幕的文本比例在操作系统级别配置时 (见 屏幕分辨率和文本缩放), 位图也是内部缩放的,常见的配置是将一个绘图单位映射为 1.25, 1.5 或 2 像素。

    当在将图像传输到屏幕前作为屏外缓冲区绘制位图时,或者由于其他原因需要与屏幕绘制保持一致时, 请使用 make-screen-bitmap 。

  • 由 racket/gui/base 的 make-bitmap in canvas% 产生的位图就像 make-screen-bitmap 产生的位图一样,但在 Mac OS 上, 位图被优化为绘制到屏幕上(通过利用系统 API,反过来,可以利用图形硬件)。

    使用 make-bitmap in canvas% 可以达到与 make-screen-bitmap 类似的目的, 特别是当位图以后将被绘制到一个已知的目标画布上时。
