29 不安全库

racket/draw 库目前是用 Cairo 和 Pango 实现的。 get-handle in bitmap% 方法暴露了 bitmap% 对象的底层 Cairo 表面, 而 make-handle-brush 支持从现有的 Cairo 表面创建一个画笔。 然而,如果 racket/draw 库在未来以不同的方式实现,这些方法的手柄的表示方法可能会改变。

29.1 手柄画笔

 (require racket/draw/unsafe/brush) package: draw-lib

procedure

(make-handle-brush handle    
  width    
  height    
  transformation    
  [#:copy? copy?]) → (is-a?/c brush%)
  handle : cpointer?
  width : exact-nonnegative-integer?
  height : exact-nonnegative-integer?
  transformation : 
(or/c #f (vector/c (vector/c real? real? real?
                             real? real? real?)
                   real? real? real? real? real?))
  copy? : any/c = #t
创建一个刷子,给定的 handle (目前)是一个 cairo_surface_t 。 如果 copy? 是 true,那么这个表面就会被复制,这样它就可以在画笔创建后被释放或修改; 如果 copy? 是 #f, 那么只要画笔能被使用,这个表面就必须保持可用和不变的状态。

width 和 height 数指定曲面的边界,当曲面必须被复制时,即使 copy? 是 #f 。 例如,在绘制单色目标时,可能需要将曲面转换为点阵图。

给出的曲面的处理方式与 stipple bitmap 很相似:它是隐式重复的, 并且给定的 transformation (如果有的话)决定了曲面相对于目标绘图环境的对齐。

当画笔与 record-dc% 对象一起使用时,如果该对象的 get-recorded-datum 方法被调用, 那么该表面就会有效地转换为结果基准的 stigpple bitmap。

29.2 Cairo 库

 (require racket/draw/unsafe/cairo-lib) package: draw-lib

value

cairo-lib : (or/c ffi-lib? #f)

对 Cairo 库的引用,用于 get-ffi-obj 等函数,如果 Cairo 不可用,则为 #f 。
