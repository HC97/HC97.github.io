procedure

(get-face-list [kind 
  #:all-variants? all-variants?]) 
 → (listof string?)
  kind : (or/c 'mono 'all) = 'all
  all-variants? : any/c = #f
返回当前系统中可用的字体名称的列表。如果 kind 是 'mono, 那么列表中只包括已知的对应于单空格字体的字体。

如果 all-variants? 是 #f (默认值),那么结果是一个更标准的字体家族名称的列表, 这些名称与样式和重量选项相结合,得出一个面孔;如果 all-variants? 是 true, 那么结果包括该家族中每个可用面孔的一个字符串。

关于 family 的信息请参见 font% for information about.

procedure

(make-bitmap width    
  height    
  [alpha?    
  #:backing-scale backing-scale]) → (is-a?/c bitmap%)
  width : exact-positive-integer?
  height : exact-positive-integer?
  alpha? : any/c = #t
  backing-scale : (>/c 0.0) = 1.0
返回 (make-object bitmap% width height #f alpha? backing-scale), 但这个程序是首选,因为它以更有用的方式默认了 alpha? 。

参见 make-platform-bitmap 和 可移植性和位图变体 。

Changed in version 1.1 of package draw-lib: 增加了 #:backing-scale 可选参数。

procedure

(make-brush [#:color color    
  #:style style    
  #:stipple stipple    
  #:gradient gradient    
  #:transformation transformation    
  #:immutable? immutable?]) → (is-a?/c brush%)
  color : (or/c string? (is-a?/c color%)) = (make-color 0 0 0)
  style : 
(or/c 'transparent 'solid 'opaque
      'xor 'hilite 'panel
      'bdiagonal-hatch 'crossdiag-hatch
      'fdiagonal-hatch 'cross-hatch
      'horizontal-hatch 'vertical-hatch)
 = 'solid
  stipple : (or/c #f (is-a?/c bitmap%)) = #f
  gradient : 
(or/c #f
     (is-a?/c linear-gradient%)
     (is-a?/c radial-gradient%))
 = #f
  transformation : 
(or/c #f (vector/c (vector/c real? real? real?
                             real? real? real?)
                    real? real? real? real? real?))
   = #f
  immutable? : any/c = #t
创建一个 brush% 实例。 与使用 make-object 和 brush% 相比,这个过程提供了一个几乎等同的接口, 但它也支持创建不可变的画笔(默认情况下创建不可变的画笔)。

当 stipple 是 #f, gradient 是 #f, transformation 是 #f, immutable? 是 true, 并且 color 是 color% 对象或 the-color-database 中的一个字符串时, 结果笔刷是通过 the-brush-list 的 find-or-create-brush 创建的。

画笔转换与 get-transformation in dc<%> 的表示和含义相同。

procedure

(make-color red green blue [alpha]) → (is-a?/c color%)

  red : byte?
  green : byte?
  blue : byte?
  alpha : (real-in 0 1) = 1.0
创建一个 color% 实例。 与使用 color% 的 make-object 相比, 这个过程提供了一个几乎等同的接口,但它创建了一个不可变的 color% 对象。

要在一个颜色字符串的基础上创建一个不可变的颜色,请使用 find-color 或 the-color-database 。

procedure

(make-font [#:size size 
  #:face face 
  #:family family 
  #:style style 
  #:weight weight 
  #:underlined? underlined? 
  #:smoothing smoothing 
  #:size-in-pixels? size-in-pixels? 
  #:hinting hinting 
  #:feature-settings feature-settings 
  #:font-list font-list]) 
 → (is-a?/c font%)
  size : (real-in 0.0 1024.0) = 12
  face : (or/c string? #f) = #f
  family : 
(or/c 'default 'decorative 'roman 'script
      'swiss 'modern 'symbol 'system)
 = 'default
  style : (or/c 'normal 'italic 'slant) = 'normal
  weight : font-weight/c = 'normal
  underlined? : any/c = #f
  smoothing : 
(or/c 'default 'partly-smoothed
      'smoothed 'unsmoothed)
 = 'default
  size-in-pixels? : any/c = #f
  hinting : (or/c 'aligned 'unaligned) = 'aligned
  feature-settings : font-feature-settings/c = (hash)
  font-list : (or/c (is-a?/c font-list%) #f)
   = (current-font-list)
查找或创建一个 font% 的实例。

当 font-list 是 #f 时, make-font 使用带有 font% 的 make-object 创建一个新的字体。 否则,当 font-list 是一个 font-list% 的实例时, make-font 使用它的 find-or-create-font 方法找到或创建一个字体。 在这两种情况下, make-font 只是提供了一个更方便的接口。

Changed in version 1.4 of package draw-lib: 改变了 size ,允许非整数和零值。
Changed in version 1.14: 改变了 weight ,允许整数值和符号 'thin, 'ultralight, 'semilight, 'book, 'medium, 'semibold, 'ultrabold, 'heavy 和 'ultraheavy 。
Changed in version 1.19: 增加了可选的 feature-settings 和 font-list 参数。

procedure

(make-monochrome-bitmap width height [bits]) → (is-a?/c bitmap%)

  width : exact-positive-integer?
  height : exact-positive-integer?
  bits : (or/c bytes? #f) = #f
如果 bits 是 #f ,则返回 (make-object bitmap% width height #t) , 否则返回 (make-object bitmap% bits width height) 。 这个过程比在 bitmap% 上使用 make-object 更好,因为它的重载更少。

  #:style style    
  #:cap cap    
  #:join join    
  #:stipple stipple

  width : (real-in 0 255) = 0

(or/c 'transparent 'solid 'xor 'hilite

      'xor-dot 'xor-long-dash 'xor-short-dash

(make-platform-bitmap width 
  height 
  [#:backing-scale backing-scale]) 
 → (is-a?/c bitmap%)
  width : exact-positive-integer?
  height : exact-positive-integer?

创建的位图尽可能地使用平台特定的绘图操作,这与 Windows 和 Mac OS 上的 make-bitmap 结果不同。 参见 可移植性和位图变体 以了解更多信息。

procedure

(read-bitmap in 
  [kind 
  bg-color 
  complain-on-failure? 
  #:get-data-from-file? get-data-from-file? 
  #:backing-scale backing-scale 
  #:try-@2x? try-@2x?]) 
 → (is-a?/c bitmap%)
  in : (or path-string? input-port?)
  kind : 
(or/c 'unknown 'unknown/mask 'unknown/alpha
      'gif 'gif/mask 'gif/alpha
      'jpeg 'jpeg/alpha
      'png 'png/mask 'png/alpha
      'xbm 'xbm/alpha 'xpm 'xpm/alpha
      'bmp 'bmp/alpha)
   = 'unknown/alpha
  bg-color : (or/c (is-a?/c color%) #f) = #f
  complain-on-failure? : any/c = #t
  get-data-from-file? : any/c = #f
  backing-scale : (>/c 0.0) = 1.0
  try-@2x? : any/c = #f
返回 (make-object bitmap% in kind bg-color complain-on-failure? backing-scale), 但这个过程是首选,因为它以一种更有用的方式默认了 kind 和 complain-on-failure? 。

如果 try-@2x? 为真, in 是一个路径,并且 kind 不是 /mask 符号之一, 那么 read-bitmap 会检查是否存在一个与 in 匹配的文件, 但在文件名中加入了 "@2x" 如果有的话,在文件后缀之前)。 如果 "@2x" 路径存在,它将被用来代替 in, 并且 backing-scale 被乘以 2 。

如果 get-data-from-file? 不是 #f, 那么生成的位图的 get-data-from-file 方法将返回文件的字节数。

Changed in version 1.1 of package draw-lib: 增加了 #:backing-scale 和 #:try-@2x? 可选参数。
Changed in version 1.17: 增加了 #:get-data-from-file? 参数。
