class

record-dc% : class?

  superclass: object%

  extends: dc<%>
一个 record-dc% 对象记录了绘图操作,以便在另一个绘图上下文中重放。 记录的绘图操作可以通过 get-recorded-procedure 提取为一个过程, 或者通过 get-recorded-datum 提取为一个数据 (可以用 write 打印,用 read 重新创建)。

当绘制记录的动作时,目标绘图上下文的笔、笔刷、字体、文本、背景、文本背景和文本前景都不会影响记录的动作。 目标绘图上下文的变换、阿尔法和剪裁区域与记录的动作中的设置组成 (因此,例如,一个设置剪裁区域的记录的动作在记录的命令被重放时实际上与绘图上下文的剪裁区域相交)。 在重新编码的命令被重放之后,目标绘图上下文中的所有设置,比如它的剪裁区域或当前字体,都和重放之前一样。

constructor

(new record-dc%    
    [[width width]    
    [height height]]) → (is-a?/c record-dc%)
  width : (>=/c 0) = 640
  height : (>=/c 0) = 480
创建一个新的记录 DC。 可选的 width 和 height 参数决定了记录 DC 上的 get-size 的结果; width 和 height 参数不会剪切绘图。

method

(send a-record-dc get-recorded-datum) → any/c

将记录的图画提取为一个值,可以用 write 来打印,用 read 来重新读取。 使用 recorded-datum->procedure 将基准点转换为一个绘图过程。

method

(send a-record-dc get-recorded-procedure)

 → ((is-a?/c dc<%>) . -> . void?)
将记录的绘图提取为一个过程,该过程可以应用于另一个 DC,以便将绘图命令重放至给定的 DC。

get-recorded-procedure 方法可以比合成 get-recorded-datum 和 recorded-datum->procedure 更有效率。
