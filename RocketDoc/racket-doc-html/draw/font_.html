class

font% : class?

  superclass: object%

font 是一个决定文本外观的对象,主要是在向设备上下文绘制文本时。 一个字体由八个属性决定:

  • size — 文本的大小,可以是点(默认),也可以是逻辑绘图单位,取决于 "size-in-pixels" 属性(见下文)。

  • family — 一个独立于平台和设备的字体指定。 这些家族有:

    • 'default

    • 'decorative

    • 'roman

    • 'script

    • 'swiss

    • 'modern (固定宽度)

    • 'symbol (希腊字母和更多)

    • 'system (类似于绘制控制标签的字体,但见 normal-control-font)

    术语 "家族" 和 "面孔" 相对于其通常的含义来说是混用的。 在通常的术语中,一个 font% 的 "面 "实际上更像是一个字体家族,或者更普遍地作为一个面的描述字符串, 与其他 font% 的属性结合起来,得出一个面。 font% "家族 "是一种抽象的字体家族,它被映射到特定平台上的一个特定字体家族。

  • face — 一个字符串的字体名称,比如说 "Courier" 。 字体名称的格式和含义是针对平台和设备的。 如果一个字体的面名是 #f,那么该字体的外观就只取决于该字体家族。 如果提供了一个字体,但没有对该字体名称的映射(对于特定的平台或设备),那么该字体名称将被忽略,而使用字体家族。 参见 font-name-directory<%>，以了解脸部名称如何被映射用于绘制文本。

  • style — 字体的倾斜样式,其中之一:

    • 'normal

    • 'slant (又称 “oblique”)

    • 'italic

  • weight — 字体的重量,其中之一:

    • (integer-in 100 1000)

    • 'thin (相当于 100)

    • 'ultralight (相当于 200)

    • 'light (相当于 300)

    • 'semilight (相当于 350)

    • 'book (相当于 380)

    • 'normal (相当于 400)

    • 'medium (相当于 500)

    • 'semibold (相当于 600)

    • 'bold (相当于 700)

    • 'ultrabold (相当于 800)

    • 'heavy (相当于 900)

    • 'ultraheavy (相当于 1000)

    Changed in version 1.14 of package draw-lib: 改为允许整数值和符号 'thin, 'ultralight, 'semilight, 'book, 'medium, 'semibold, 'ultrabold, 'heavy 和 'ultraheavy 。

  • underline? — #t 表示下划线, #f 表示普通。

  • smoothing — 抗锯齿平滑的数量,其中之一:

    • 'default (平台特定,有时用户可配置)

    • 'partly-smoothed (灰色抗锯齿)

    • 'smoothed (子像素抗锯齿)

    • 'unsmoothed

  • size-in-pixels? — #t 如果字体的大小是以逻辑绘图单位(即未缩放的屏幕或位图绘图环境中的像素)为单位, #f 如果字体的大小是以 points,为单位, 在 Mac OS 上一个点等于1个像素,在Windows和Unix上一个点等于 (/ 96 72) 像素

  • hinting — 字体度量值是否应该被四舍五入为整数:

    • 'aligned (默认) — 四舍五入为整数,以提高基于像素的目标的字母间距的一致性, 但代价是使度量值无法扩展。

    • 'unaligned — 禁用四舍五入

  • feature settings — OpenType feature 设置的哈希值,用于启用或禁用 OpenType 字体的可选排版特征。 散列中的每个条目都将一个四字母的 OpenType 特征标签映射到它所需的值。 对于布尔型 OpenType 特征,值为 0 表示 "禁用",值为 1 表示 "启用"; 对于其他特征,值的含义是不同的(甚至可能取决于字体本身)。

    Added in version 1.19 of package draw-lib.

为了避免创建具有相同特征的多个字体,请使用全局的 font-list% 对象 the-font-list 。

另请参见 font-name-directory<%> 。

Changed in version 1.2 of package draw-lib: 在 Windows 上将 "点" 定义为 (/ 96 72) 像素,与屏幕分辨率无关。
Changed in version 1.14: 改变了 "weight",允许整数值和符号 'thin, 'ultralight, 'semilight, 'book, 'medium, 'semibold, 'ultrabold, 'heavy 和 'ultraheavy 。
Changed in version 1.19: 添加了 "特征设置" 属性来控制 OpenType 特征。

method

(send a-font get-face) → (or/c string? #f)

获取字体的面名,如果没有指定,则获取 #f 。

method

(send a-font get-family) → font-family/c

获取字体的族。关于字族的信息,请参见 font% 。

method

(send a-font get-feature-settings) → font-feature-settings/c

获取字体的 OpenType feature settings 。

method

(send a-font get-font-id) → exact-integer?

获取字体的 ID,用于 font-name-directory<%> 。 该 ID 仅由字体的面和族的规格决定。

method

(send a-font get-hinting) → font-hinting/c

获取字体的提示。关于提示的信息请参见 font% 。

method

(send a-font get-point-size) → (integer-in 1 1024)

获取字体的大小,四舍五入到最接近的非零整数。 尽管该方法的名字是这样的,但其结果是以逻辑单位或 points 为单位的, 这取决于 get-size-in-pixels 的结果。

请参阅 get-size ,而不是。 get-point-size 方法是为了向后兼容而提供的。

method

(send a-font get-size [in-pixels?]) → (real-in 0.0 1024.0)

  in-pixels? : any/c = (send a-font get-size-in-pixels)
获取字体的大小(大约是高度)。 如果 in-pixels? 是 #f, 那么尺寸就是以 points 为单位, 否则就是以逻辑单位为单位。

由于字体设计者在字体中包含的空间,字体往往会生成比标称尺寸略高的文本。

Added in version 1.4 of package draw-lib.
Changed in version 1.19: 增加了 in-pixels? 参数。

method

(send a-font get-size-in-pixels) → boolean?

如果 get-size 默认为以逻辑绘图单位报告尺寸,则返回 #t ; 如果默认为报告 points ,则返回 #f 。

method

(send a-font get-smoothing) → font-smoothing/c

获取字体的抗锯齿平滑模式。 关于平滑的信息请参见 font% 。

method

(send a-font get-style) → font-style/c

获取字体的斜面样式。 关于样式的信息,请参见 font% 。

method

(send a-font get-underlined) → boolean?

如果字体有下划线,返回 #t ,否则返回 #f 。

method

(send a-font get-weight) → font-weight/c

获取字体的重量。有关权重的信息,见 font% 。

method

(send a-font screen-glyph-exists? c    
  [for-label?]) → boolean?
  c : char?
  for-label? : any/c = #f
如果给定的字符在绘制屏幕或位图时有相应的字形,则返回 #t ,否则返回 #f 。

如果第二个参数为真,结果表明该字形是否可用于控制标签。 否则,它表示该字形是否可用于 dc<%> 绘图。

对于 dc<%> 绘制,由于在绘制或测量文本时自动进行字体替换,这个方法的结果并不取决于这个字体的属性(大小、面等)。 字体的属性只是为字形搜索提供一个提示。

另请参阅 glyph-exists? 。
