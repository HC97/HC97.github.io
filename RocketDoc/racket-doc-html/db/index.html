DB: 数据库连接

Ryan Culpepper <ryanc@racket-lang.org>

为函数式程序员提供的数据库接口。

 (require db) package: db-lib

这个库提供了一个连接几个数据库系统的高级接口。 目前支持以下数据库系统:

  • PostgreSQL 7.4 及以后的版本。 这个库实现了 PostgreSQL 的电线协议,所以不需要本地客户端库。

  • MySQL 5 及以后的版本。 这个库实现了 MySQL 的线程协议,所以不需要本地客户端库。

  • SQLite 第3版。 需要 SQLite 本地客户端库;见 SQLite 要求 。

  • Cassandra 2.1.0及以后的版本。 该库实现了 Cassandra 线协议(v3),所以不需要本地客户端。

  • ODBC. 需要一个 ODBC 驱动管理器和适当的 ODBC 驱动;见 ODBC 要求 。 以下数据库系统已知可以通过 ODBC 与该库协同工作(详见 ODBC 状态 ): DB2, Oracle 和 SQL Server 。

查询操作在精神上是功能性的:查询返回结果或引发异常,而不是把它们的状态存储到游标对象中,供以后浏览和检索。 查询参数和结果字段被自动翻译成适当的 Racket 值。 资源由垃圾收集器和托管人自动管理。 连接是内部同步的,所以多个线程可以同时使用一个连接。

鸣谢 感谢 Dave Gurnell, Noel Welsh, Mike Burns, and Doug Orleans 对 spgsql 的贡献,这个库只有 PostgreSQL 的前身。 对 SQLite 的支持部分是基于 Jay McCarthy 的 sqlite 包的代码。
