Quickscript, DrRacket 的脚本插件

Laurent Orseau <laurent.orseau@gmail.com>

1 简介

通过 Quickscript，用户可以使用小型 Racket 脚本轻松扩展 DrRacket，从而自动执行编辑器中的某些操作，同时无需重启 DrRacket。

创建新脚本非常简单，只需点击 Scripts | New script…。 每个脚本都会自动添加到 Scripts 菜单中，无需重启 DrRacket。 用户可以为脚本分配快捷键（通过菜单项）。 默认情况下，脚本将当前选中的文本作为输入，并输出替换文本。 此外，还可以直接访问 DrRacket GUI 的某些元素，如 DrRacket 的框架和定义或交互编辑器，用于高级脚本编写。

2 安装

Quickscript 随 DrRacket 自动安装，因此您无需进行任何操作。

2.1 安装脚本: Quickscript Extra

您可以单独使用 Quickscript，但是 Quickscript Extra 软件包中包含了大量有用的脚本以及一些供用户自定义的示例脚本。

安装时，在 DrRacket 菜单 File|Package Manager 中查找 quickscript-extra ，或者运行raco命令：
  raco pkg install quickscript-extra

然后点击 Scripts|Manage|Compile scripts。(无需重启 DrRacket）。

2.2 安装脚本: More scripts

更多脚本可以在 Racket wiki 找到–如果你认为自己的脚本对其他人有用，也可以添加到那里。

3 制作自己的脚本： 第一个简单的例子

点击 Scripts|Manage|New script… 菜单项，输入 Reverse 作为脚本名。 这将在用户的脚本目录下创建并打开文件 reverse.rkt。 同时， Scripts 菜单中会自动出现一个新项目。

在 DrRacket 中打开的 .rkt 文件中，修改 define-script 定义如下:

不要将你的脚本函数命名为 reverse, 这会影响 Racket 自己的脚本，使脚本挂起。

如果以后修改了 #:label 属性，需要在保存文件后点击 Scripts|Manage|Reload menu 重新加载）。

(define-script reverse-selection
  #:label "Reverse"
  (λ (selection)
    (list->string (reverse (string->list selection)))))
并保存文件。

然后在新标签页中输入一些文本，选中后点击 Scripts|Reverse, 就可以了！

4 进入更多细节

Quickscript 为 DrRacket 主窗口添加了 Scripts 菜单。 该菜单有几个项目，后面是脚本列表。

New script 项要求输入脚本名称，并在用户的脚本目录下创建相应的 .rkt 文件，然后在 DrRacket 中打开。

每个脚本都用 define-script 定义，定义脚本的作用之一是在 DrRacket 的脚本菜单中添加一个条目。 单个脚本文件可以包含多个 define-script 调用。

默认情况下，新脚本被简化为最简单的形式。 然而，脚本可以通过几个可选 properties 和 arguments 进行扩展。 当所有属性和参数都被使用时，脚本可以是这样的：
(define-script a-complete-script
  ; Properties:
  #:label "Full script"
  #:help-string "A complete script showing all properties and arguments"
  #:menu-path ("Submenu" "Subsubmenu")
  #:shortcut #\a
  #:shortcut-prefix (ctl shift)
  #:output-to selection
  #:persistent
  #:os-types (unix macosx windows)
  ; Procedure with its arguments:
  (λ (selection #:frame fr
                #:editor ed
                #:definitions defs
                #:interactions ints
                #:file f)
    "Hello world!"))

需要注意的是，属性参数是字面形式，而不是表达式形式，因此 不 能加引号。 下面我们首先详细介绍存储过程及其参数，然后介绍脚本的属性。

 (require quickscript) package: quickscript

syntax

(define-script name
  property ...
  proc)
 
property = #:label label-string
  | #:help-string string
  | #:menu-path (label-string ...)
  | #:shortcut char | symbol | #f
  | #:shortcut-prefix (shortcut-prefix ...)
  | #:persistent? #t | #f
  | #:output-to output-to
  | #:os-types (os-type ...)
     
shortcut-prefix = alt | cmd | meta | ctl | shift | option
     
output-to = selection | new-tab | message-box | clipboard | #f
     
os-type = macosx | unix | windows
     
proc = 
(λ (selection-id
    [#:editor editor-id]
    [#:definitions definitions-id]
    [#:interactions interactions-id]
    [#:frame frame-id]
    [#:file file-id])
  body-expr ...
  return-expr)
请参阅下面的小节以获得完整的描述。

再次注意属性的参数是字面形式而不是表达式. 这是因为两次读取脚本文件的目的不同。 第一次，Quickscript 读取脚本文件，提取在 DrRacket 中构建菜单项所需的最少信息。 在这一阶段不执行任何 Racket 操作，以便尽可能轻便快捷。 然后，当点击相应的菜单项时，Quickscript 第二次读取脚本文件，这次是实际读取和访问 Racket 模块，并调用相应的过程。 也就是说，脚本模块只在需要时才被实例化，以减少加载时间和内存占用。

4.1 脚本的过程

当点击 DrRacket 脚本菜单中的脚本标签时，相应的过程将被调用。 该过程至少需要一个 selection 参数，即当前编辑器中选中的字符串。 存储过程必须返回 #f 或 string? 。 如果返回 #f, 则不对当前编辑器进行任何更改，但如果返回字符串，则用返回值替换当前选择。

如果在存储过程中指定了上述关键字，脚本插件会检测到它们并传递相应的值，因此存储过程可以有多种形式：
(λ (selection) ....)
(λ (selection #:frame fr) ....)
(λ (selection #:file f) ....)
(λ (selection #:editor ed #:file f) ....)
....

下面是关键字参数的含义：

  • #:file : (or/c path? #f)

    定义窗口当前文件的路径，如果没有该文件（即未保存的编辑器），则为 #f 。

    Example:
    (define-script current-file-example
      #:label "Current file example"
      #:output-to message-box
      (λ (selection #:file f)
        (string-append "File: " (if f (path->string f) "no-file")
                       "\nSelection: " selection)))

    另请参见： file-name-from-path, filename-extension, path->string, split-path 。

  • #:definitions : text%

    当前定义窗口的 text% 编辑器。更多详情请参见 text% 。

  • #:interactions : text%

    当前交互窗口的 text% 编辑器。与 #:definitions 类似。

  • #:editor : text%

    text% 当前编辑器，定义或交互编辑器。与 #:definitions 类似。

  • #:frame : drracket:unit:frame<%>

    DrRacket 的框架。用于高级脚本编写。

    Example:
    (require racket/class)
    (define-script number-tabs
      #:label "Number of tabs"
      #:output-to message-box
      (λ (selection #:frame fr)
        (format "Number of tabs in DrRacket: ~a"
                (send fr get-tab-count))))

注意: 脚本过程可以有额外的可选参数（关键字或非关键字）和其余参数，但不能有额外的必选参数。例如
(define-script append-plop
  #:label "Append plop"
  (λ (selection [more ""] #:even-more [even-more ""])
    (string-append selection "_plop" more even-more)))
 
(define-script append-plop-plip
  #:label "Append plop plip ploop"
  (λ (selection)
    ; Call the first script's procedure:
    (append-plop selection "_plip" #:even-more "_ploop")))

4.2  脚本属性

属性只是数据, 不能包含表达式。

大多数属性 (#:label, #:shortcut, #:shortcut-prefix, #:help-string) 与 menu-item% 构造函数相同。 特别是，键盘快捷键可以分配给一个项目。

如果一个属性没有出现在字典中，它将使用默认值。

还有一些额外的属性：

  • #:menu-path : (listof string?) = () 这是一个子菜单列表，脚本的标签将被放置在脚本菜单下。

    注意不同文件中的不同脚本可以共享相同的子菜单。

  • #:output-to : (or/c selection new-tab message-box clipboard #f) = selection

    如果是 selection, 存储过程的输出将替换当前编辑器（定义或交互）中的选择，如果没有选择，则在光标处插入输出。 如果是 new-tab, 返回值将被写入一个新的标签页中。 如果是 message-box, 回值（如果是字符串）将显示在 message-box 中。 如果是 clipboard, 返回值（如果是字符串）将被复制到剪贴板。 如果为 #f, 则不使用返回值。

    如果该值被更改，请确保使用 Scripts|Manage|Reload menu 重新加载菜单。

  • #:persistent

    如果 没有 提供关键字 #:persistent ，脚本的每次调用都在一个新的命名空间中完成。

    但如果提供了 #:persistent ，只有在第一次调用时才会创建一个新的命名空间，随后的调用会重新使用相同的命名空间。 请注意，每个文件只保留一个命名空间，因此如果同一文件中的不同脚本被标记为持久化，它们将共享同一个命名空间（以及变量）。 同样需要注意的是，被标记为非持久化的脚本不会与同一文件中被标记为持久化的其他脚本共享相同的命名空间。

    请看下面的脚本：
    (define count 0)
     
    (define-script persistent-counter
      #:label "Persistent counter"
      #:persistent
      #:output-to message-box
      (λ (selection)
        (set! count (+ count 1))
        (number->string count)))

    如果脚本是持久脚本，那么每次通过菜单调用脚本时，计数器都会增加，而如果脚本不是持久脚本，计数器总是显示 1。

    注意: 可以通过点击 Scripts|Manage|Stop persistent scripts 菜单项来停止和重置持久脚本。 在上例中，这将重置计数器。 确保在编辑持久脚本后停止该脚本。 Scripts|Manage|Reload menu 和 Scripts|Manage|Compile scripts 也可以停止持久脚本。

    技术要点: 脚本的过程在 dynamic-require 脚本的命名空间 外 调用， 在 DrRacket 框架的命名空间内调用，以便访问该框架中的对象。

  • #:os-types (listof (one-of/c unix macosx windows))

    该关键字后必须有一个支持的 os 类型列表。 默认为所有类型，即 (unix macosx windows) 。

如果更改了这些属性，可能需要通过点击 Scripts|Manage|Reload menu 来重新加载脚本菜单。

5 脚本库

当用户创建一个新脚本时，该脚本会被放置到 (find-system-path 'pref-dir) 的一个子目录中。 通过 Scripts|Manage|Open script… 菜单项可以直接访问该文件夹。

可以通过 Scripts|Manage|Library 菜单项添加更多用于查找脚本的目录。 当一个目录被添加到库中时，它的所有 .rkt 文件（非递归）都会被视为脚本。 特定的文件可以从库中排除。

6 影子脚本

当从第三方软件包（如 quickscript-extra）安装脚本时，它会自带一组属性值。 这些值可能并不适合用户，用户可能希望重新定义其中的一些值，例如菜单路径或键盘快捷键。 对于用户来说，一个显而易见的选择是复制/粘贴整个脚本，但这样做将无法受益于原始脚本编写者所做的进一步错误修复和增强。

为了解决这个问题，用户可以制作一个 shadow script,在用户的目录中创建一个新的脚本，这个脚本有自己的属性集， 用户可以对其进行修改，但是这个脚本的程序与原始脚本的程序是绑定的。

要创建影子脚本，请在 Scripts|Manage|Library 中打开脚本库，导航到第三方脚本并点击 Shadow 。

7 更新 quickscript 包

要更新已经安装的 Quickscript，可以通过 DrRacket 的 File|Package Manager 菜单， 或者运行 raco pkg update quickscript 。

在此过程中，用户的脚本不会被修改。

8 发布自己的脚本

发布小脚本的 最简单 方法是将其发布为 gist 或发布在 PasteRack 上，并共享链接。用户可以复制/粘贴内容到新脚本中。 别忘了附上许可协议，如MIT/Apache 2。

发布脚本的 最佳 方式是创建一个包，用户只需安装该包即可。 假设你的脚本存储在 "scripts" 子目录下， 在包的根目录下包含一个文件（比如 "register.rkt") ，其中包含以下代码：

如果 "register.rkt" 文件不在根目录下，运行时路径需要相应修改。

#lang racket/base
(require (for-syntax racket/base
                     racket/runtime-path
                     (only-in quickscript/library
                              add-third-party-script-directory!)))
 
;; This file is going to be called during setup and will automatically
;; register the scripts subdirectory in quickscript's library.
(begin-for-syntax
  (define-runtime-path script-dir "scripts")
  (add-third-party-script-directory! script-dir))

可以通过 quickscript-extra 查看示例。

不要忘记在 Racket server 上注册你的软件包。
