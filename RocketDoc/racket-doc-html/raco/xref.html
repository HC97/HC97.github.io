6.15 已安装手册的交叉引用的 API

 (require setup/xref) package: racket-index

procedure

(load-collections-xref [on-load]) → xref?

  on-load : (-> any/c) = (lambda () (void))
创建并缓存或返回用 make-collections-xref 创建的缓存交叉参考记录。 只有当先前缓存的记录没有被返回时,才会调用 on-load 函数。

procedure

(make-collections-xref 
  [#:no-user? no-user? 
  #:no-main? no-main? 
  #:doc-db db-path 
  #:quiet-fail? quiet-fail? 
  #:register-shutdown! register-shutdown!]) 
 → xref?
  no-user? : any/c = #f
  no-main? : any/c = #f
  db-path : (or/c #f path?) = #f
  quiet-fail? : any/c = #f
  register-shutdown! : ((-> any) . -> . any) = void
与 load-xref 类似,但会自动找到所有已安装 raco setup 的手册的交叉参考文件。 产生的交叉参考记录利用交叉参考数据库 db-path 优势,当支持的时候,延迟加载交叉参考的细节直到需要。

如果 no-main? 或 no-user? 为 #t , 则交叉参考信息将被跳过,分别安装在主安装位置或用户特定位置。

如果 quiet-fail? 为真,那么在加载交叉引用信息时,错误会被抑制。

register-shutdown! 回调可以被调用来注册一个函数, 当不再需要 make-collections-xref 的结果时关闭数据库连接。 如果没有提供 register-shutdown! ,或者发送至 register-shutdown! 的函数从未被调用, 那么数据库连接将只通过一个 custodian 关闭。

procedure

(get-rendered-doc-directories no-user?    
  no-main?) → (listof path?)
  no-user? : any/c
  no-main? : any/c
返回所有已安装的集合的所有文档的目录列表,如果 no-main? 或 no-user? 为 #t , 则分别省略安装在主安装或用户特定位置的文档。

procedure

(get-current-doc-state) → doc-state?

记录每当文档被修改时被触及的文件的时间戳。

Added in version 1.2 of package racket-index.

procedure

(doc-state-changed? doc-state) → boolean?

  doc-state : doc-state?
当 doc-state 中的文件的时间戳发生变化(或出现新文件)时,返回 #t ,否则返回 #f 。

如果结果是 #t ,那么Racket 的这个安装中的文档已经改变,否则就没有改变。

Added in version 1.2 of package racket-index.

procedure

(doc-state? v) → boolean?

  v : any/c
一个谓词来识别 get-current-doc-state 的结果。

Added in version 1.2 of package racket-index.
