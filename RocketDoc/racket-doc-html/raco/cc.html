12.1 编译和链接 C 语言扩展

dynamic extension 是一个共享库(又称 DLL),它使用 C 语言 API 对 Racket 进行扩展。 一个扩展可以通过 load-extension 显式加载, 也可以通过 require 或 load/use-compiled 隐式加载,以代替源 file ,当扩展位于

(build-path "compiled" "native" (system-library-subpath)
            (path-add-suffix file (system-type 'so-suffix)))

相对于 file 的位置。

关于编写扩展的信息,请参见 Inside: Racket C API 。

raco ctool 是由 "cext-lib" 包提供的。

三种 raco ctool 模式有助于构建扩展:

  • --cc : 运行主机系统的 C 语言编译器,自动提供标记来定位 Racket 头文件,并将其编译为共享库。

  • --ld : 运行主机系统的 C 语言链接器,自动提供标记来定位和链接到 Racket 库,并生成一个共享库。

  • --xform : 将没有明确的 GC 合作钩子的 C 代码转化为与 Racket 的 3m 垃圾收集器合作; 见 Inside: Racket C API 中的 Overview (BC) 。

编译和链接建立在 dynext/compile 和 dynext/link 库上。 下列 raco ctool 标志对应于设置或访问这些库的参数: --tool, --compiler, --ccf, --ccf, --ccf-clear, --ccf-show, --linker, ++ldf, --ldf, --ldf-clear, --ldf-show, ++ldl, --ldl-show, ++cppf, ++cppf ++cppf-clear 和 --cppf-show 。

--3m 标志指定扩展将被加载到 Racket 的 3m 变体中。 --cgc 标志指定该扩展将与 CGC 一起使用。 默认情况取决于 raco: 如果 raco 本身在 3m 中运行,则为 --3m ; 如果 raco 本身在 CGC 中运行,则为 --cgc 。

12.1.1 3m 转换的 API

 (require compiler/xform) package: cext-lib

procedure

(xform quiet?    
  input-file    
  output-file    
  include-dirs    
  [#:keep-lines? keep-lines?]) → any/c
  quiet? : any/c
  input-file : path-string?
  output-file : path-string?
  include-dirs : (listof path-string?)
  keep-lines? : boolean? = #f
将没有明确的 GC 合作钩子的 C 代码转换为与 Racket 的 3m 垃圾收集器合作; 见 Inside: Racket C API 中的 Overview (BC) 。

参数与 compile-extension 相同;此外, keep-lines? 可以 #t 来生成 GCC 风格的注释,以连接生成的 C 代码和原始源码的位置。

由 xform 生成的文件可以通过 compile-extension 进行编译。
