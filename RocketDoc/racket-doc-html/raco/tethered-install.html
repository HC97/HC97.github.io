6.18 拴住的安装

Racket 的 tethered 安装是一个层(见 分层安装) , 它包括一个包装可执行文件,用于安装层中的每个可执行文件。 每个包装可执行文件都指向新层的 "config.rktd" (见 安装配置和搜索路径) , 无需使用 PLTCONFIGDIR 环境变量或 --config 标志。 换句话说,系留安装提供的可执行文件,如 racket, raco 和 drracket ,都是与该层绑定的。 因此,Tethering 有助于创建一个安装层,它以一种更自足的方式行事,但对底层的重复程度最小。

Tethering 在 user 层或 installation 层工作:

  • 一个带有系留功能的 user 层由一个新的目录 ‹addon-dir› 和一个 "‹addon-dir›/etc/config.rktd" 文件表示,该文件将 'addon-tethered-console-bin-dir 映射到 ‹tethered-bin-dir›, 'addon-tethered-gui-bin-dir 映射到 ‹tethered-gui-bin-dir›, 并且(可选择) 'addon-tethered-apps-dir 映射到 ‹tethered-apps-dir› 。用以下方法初始化系留层

      racket -A ‹addon-dir› -l- raco setup --avoid-main

  • 一个有系留的 installation 层就像一个没有系留的安装层 (见 分层安装), 但是该层的 "‹layer-dir›/etc/config.rktd" 文件将 'config-tethered-console-bin-dir 映射到 ‹tethered-bin-dir›, 'config-tethered-gui-bin-dir 映射到 ‹tethered-gui-bin-dir›, (可选择) 'config-tethered-apps-dir 映射到 ‹tethered-apps-dir› 。 'bin-dir 和 'gui-bin-dir 的配置可以指向相同的目录, 但可执行文件不会由 raco setup 在那里专门创建。用以下方法初始化系留层

      racket -G ‹layer-dir›/etc -l- raco setup

在任何一种情况下,初始化都会在 ‹tethered-bin-dir› 和 ‹tethered-gui-bin-dir› 目录下创建系留可执行文件, 在 ‹tethered-apps-dir› (如果指定的话)中写入 ".desktop" 文件(对于 Unix)。 此后,像 ‹tethered-bin-dir›/racket 和 ‹tethered-bin-dir›/raco 这样的系留可执行文件可以用来与系留层工作。
