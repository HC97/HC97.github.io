9 raco link: 库收藏链接

raco link 命令检查并修改了一个 collection links file , 以显示、添加或删除馆藏名称与文件系统目录之间的映射关系。

直接管理链接是有点不可取的。 取而代之的是使用软件包管理器(见 Racket 中的包管理), 它安装和管理链接(即,它建立在 raco link 之上)的方式更优雅地导致与他人共享集合。 尽管如此, raco link 还是可以直接使用的。

例如,命令

  raco link maze

为 "maze" 集合安装了一个特定用户和特定版本的链接, 将其映射到当前目录的 "maze" 子目录下。 提供多个目录路径以一次创建多个链接,特别是用命令壳通配符:

  raco link *

默认情况下,链接的集合名称与每个目录的名称相同,但可以用 --name 标志为单个目录单独设置集合名称。

要删除上面第一个例子所创建的链接,使用

  raco link --remove maze

或

  raco link -r maze

与添加链接模式一样,删除模式接受多个目录路径来删除多个链接,所有与任何目录相匹配的链接都会被删除。 如果 --name 与 --remove 一起使用,那么只有同时匹配集合名称和目录的链接才会被删除。

完整的命令行选项:

  • -l 或 --list — 如果提供了任何其他的命令行参数来修改链接表,该表将在修改后显示。 如果没有提供目录参数,并且没有指定 -u, --user, -i, --installation, -f 或 --file ,那么链接表将显示所有用户特定的和安装范围的 collection links files 。

  • -n ‹name› 或 --name ‹name› — 设置用于添加单个链接或删除匹配链接的集合名称。 默认情况下,添加的链接的集合名称是由目录名称派生的。 当同时使用 -r 或 --remove 标志时,只有集合名称与 ‹name› 匹配的链接被删除, 如果没有提供目录参数,所有与 ‹name› 匹配的链接都被删除。 这个标志与 -d 和 --root 是相互排斥的。

  • -d 或 --root — 将每个目录视为包含集合目录的集合根,而不是一个特定集合的目录。 当 -r 或 --remove 标志也被使用时,只有与目录相匹配的集合根链接被删除。 这个标志与 -n 或 --name 是相互排斥的。

  • -D 或 --static-root — 和 -d 或 --root 一样, 但是只要链接文件本身没有变化,每个目录都被假定为有一个恒定的子目录集(以提高集合搜索缓存的使用)。

  • -x ‹regexp› 或 --version-regexp ‹regexp› — 设置一个版本的 regexp,限制链接只能由 Racket 版本(由 version 报告)匹配 ‹regexp› 使用。 这个标志通常与 -u 或 --user 一起使用,用于不同版本但具有相同安装名称的安装。 当同时使用 -r 或 --remove 标志时,只有与 ‹regexp› 匹配的版本 regexp 的链接会被删除。

  • -r 或 --remove — 选择删除模式而不是添加模式。

  • -u 或 --user — 将链接的列举和移除限制在用户特定的 collection links file 中, 而不是在整个安装 collection links file 中。 这个标志与 -i, --installation, -f 和 --file 相互排斥。

  • -i 或 --installation — 读取和写入安装范围内的 collection links file 的链接, 而不是用户特定的 collection links file 。 这个标志与 -u, --user, -f 和 --file 相互排斥。

  • -f ‹file› 或 --file ‹file› — 读取和写入 ‹file› 中的链接, 而不是用户特定的 collection links file 。 这个标志与 -u, --user, -s, --shared, -i 和 --installation 相互排斥。

  • -v ‹vers› 或 --version ‹vers› — 选择 ‹vers› 作为相关的安装名称, 以便对用户特定的 collection links file 进行操作。

  • --repair — 当内容有误时,启用对现有文件内容的修复。在可能的情况下,文件会通过删除单个链接来修复。

9.1 收集链接的 API

 (require setup/link) package: base

procedure

(links dir    
  ...    
  [#:user? user?    
  #:user-version user-version    
  #:file file    
  #:name name    
  #:root? root?    
  #:static-root? static-root?    
  #:version-regexp version-regexp    
  #:error error-proc    
  #:remove? remove?    
  #:show? show?    
  #:repair? repair?    
  #:with-path? with-path?]) → list?
  dir : path?
  user? : any/c = #t
  user-version : string? = (get-installation-name)
  file : (or/c path-string? #f) = #f
  name : (or/c string? #f) = #f
  root? : any/c = #f
  static-root? : any/c = #f
  version-regexp : (or/c regexp? #f) = #f
  error-proc : (symbol? string? any/c ... . -> . any) = error
  remove? : any/c = #f
  show? : any/c = #f
  repair? : any/c = #f
  with-path? : any/c = #f
raco link 命令的一个函数版本,它总是在单个文件上工作–如果是一个路径字符串, 则是 file ,如果 user? 为真, 则是用户特定的 collection links file , 否则就是整个安装的 collection links file 。 如果 user? 为真,那么 user-version 决定了相关的安装名称(默认为当前安装的名称)。

static-root? 标志值被忽略,除非 root? 为真, remove? 为假, 在这种情况下,如果 static-root? 为真,每个给定的 dir 都被添加为静态根。

error-proc 参数被调用以引发对 raco link 命令来说是致命的异常。

如果 remove? 为真,结果是一个被从文件中删除的条目列表。 如果 remove? 是 #f ，但 root? 是真的,结果是一个集合根的路径列表。 如果 remove? 和 root? 都是 #f, 结果是一个按 file 映射的顶级集合的列表,并且适用于Racket的运行版本; 如果 with-path? 是 #f, 该列表是集合名称的字符串列表,如果 with-path? 是 true,则是集合名称的字符串和完整路径的对列表。
