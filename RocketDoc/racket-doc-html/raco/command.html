18 添加一个 raco 命令

raco 支持的命令集可以通过安装的包、 PLaneT 和其他集合来扩展。 一个命令是通过在一个集合的 "info.rkt" 库中定义 raco-commands 来添加的(见 "info.rkt" File Format), 然后 raco setup (如直接调用或作为包或 PLaneT 安装的一部分)必须索引 "info.rkt" 文件。

绑定到 raco-commands 的值必须是一个 command specifications 的列表, 其中每个规范是一个包含四个值的列表:

(list command-string
      implementation-module-path
      description-string
      prominence)

command-string 是命令的名称。 命令名称的任何明确的前缀都可以提供给 raco 来调用该命令。

implementation-module-path 通过一个模块路径(在 module-path? 的意义上)来命名实现。 模块被加载并通过 dynamic-require 调用以运行命令。 该模块可以通过 current-command-line-arguments 参数访问命令行参数, 该参数在加载命令模块之前被调整为只包括命令的参数。 current-command-name 参数也被设置为用于加载命令的命令名称。 当 raco help 被用在一个命令上时, 该命令会在 current-command-line-arguments 中以一个初始的 --help 参数启动。

description-string 是一个简短的字符串,用于描述响应 raco help 的命令。 描述不应该大写,也不应该以句号结尾。

prominence 值应该是一个实数或 #f 。 #f 值意味着该命令不应包括在 "常用命令" 的短列表中。 一个数字表示该命令的相对突出性; help 命令的值是 110, 可能没有任何命令应该更突出。 pack 工具,目前被列为常用命令中最不突出的,其数值为 10 。

作为一个例子, "compiler" 集合中的 "info.rkt" 可能含有

(define raco-commands
  '(("make" compiler/commands/make "compile source to bytecode" 100)
    ("decompile" compiler/commands/decompile "decompile bytecode" #f)))

这样, make 就被当作一个经常使用的命令,而 decompile 则作为一个不经常使用的命令。

18.1 命令参数解析

 (require raco/command-name) package: base
raco/command-name 库提供了一些函数来帮助 raco 命令向用户识别自己。

parameter

(current-command-name) → (or/c string? #f)

(current-command-name name) → void?
  name : (or/c string? #f)
目前正在通过 dynamic-require 加载的命令的名称,如果 raco 没有加载任何命令,则为 #f 。

命令实现可以使用这个参数来确定它是通过 raco 还是通过其他方式调用的。

procedure

(short-program+command-name) → string?

返回一个标识当前命令的字符串。 当 current-command-name 是一个字符串时,那么结果是 raco 可执行文件的短名称, 后面是一个空格和命令名称。 否则,它就是当前可执行文件的短名称,通过从 (find-system-path 'run-file) 的结果中剥离路径来确定。 在这两种情况下,在 Windows 系统中,可执行文件名的扩展名 ".exe" 被删除。

这个函数的结果适合与 command-line 一起使用。 例如, decompile 工具解析命令行参数时,可以用

(define source-files
  (command-line
   #:program (short-program+command-name)
   #:args source-or-bytecode-file
   source-or-bytecode-file))

这样, raco decompile --help 就会打印出

  usage: raco decompile [ <option> ... ] [<source-or-bytecode-file>] ...

  

  <option> is one of

  

    --help, -h

       Show this help

    --

       Do not treat any remaining argument as a switch (at this level)

  

   Multiple single-letter switches can be combined after

   one `-`. For example, `-h-` is the same as `-h --`.

procedure

(program+command-name) → string?

就像 short-program+command-name 一样,但是路径(如果有的话)不会从当前可执行文件的名称中剥离出来。

18.2 访问 raco 命令

 (require raco/all-tools) package: base
raco/all-tools 库收集了已安装包、 PLaneT 包和其他集合的 raco-commands 规范。

procedure

(all-tools)

 → (hash/c string? (list/c string? module-path? string? (or/c real? #f)))
返回一个以集合名称为键,以 command specifications 为值的哈希表。 例如,下面的程序调用了 raco make file.rkt:
(require raco/all-tools)
 
(define raco-make-spec (hash-ref (all-tools) "make"))
 
(parameterize ([current-command-line-arguments (vector "file.rkt")])
  (dynamic-require (second raco-make-spec) #f))
