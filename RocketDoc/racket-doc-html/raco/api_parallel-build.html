1.5 并行构建的 API

 (require setup/parallel-build) package: base
setup/parallel-build 库提供了 raco setup 和 raco make 的并行编译功能。

parallel-compile-files 和 parallel-compile 日志信息都是在 'setup/parallel-build 主题下的 'info 级别。 这些消息是 parallel-compile-event 预制结构的实例:

(struct parallel-compile-event (worker event) #:prefab)

worker 字段是创建该事件的 worker 的索引。 事件字段是 make-compilation-manager-load/use-compiled-handler 中记载的 compile-event 。

procedure

(parallel-compile-files list-of-files 
  [#:worker-count worker-count 
  #:use-places? use-places? 
  #:handler handler]) 
 → (or/c void? #f)
  list-of-files : (listof path-string?)
  worker-count : exact-positive-integer? = (processor-count)
  use-places? : any/c = #t
  handler : 
(->i ([worker-id exact-integer?]
      [handler-type symbol?]
      [path path-string?]
      [msg string?]
      [out string?]
      [err string?])
     void?)
 = void
raco make 使用 parallel-compile-files 工具函数来并行编译一个路径列表。 可选的 #:worker-count 参数指定了在并行编译过程中产生的编译工作者的数量。 如果 use-places? 为 true,编译工作者将被实现为 Racket places,否则编译工作者将被实现为独立的Racket进程。 回调 handler 被调用,符号 'done 作为每个成功编译的文件的 handler-type 参数, 'output 当成功编译产生 stdout/stderr 输出时, 'error 当发生编译错误时, 'fatal-error 当发生不可恢复的错误时。 其他参数为每个状态更新提供更多信息。 如果成功,返回值是 (void) ,如果有错误,返回值是 #f 。

(parallel-compile-files
  source-files
  #:worker-count 4
  #:handler
  (lambda (type work msg out err)
    (match type
      ['done (when (verbose) (printf " Made ~a\n" work))]
      ['output (printf " Output from: ~a\n~a~a" work out err)]
      [_ (printf " Error compiling ~a\n~a\n~a~a"
                 work
                 msg
                 out
                 err)])))

Changed in version 7.0.0.19 of package base: 增加了 #:use-places? 参数。

procedure

(parallel-compile worker-count    
  setup-fprintf    
  append-error    
  collects-tree    
  [#:use-places? use-places?]) → (void)
  worker-count : non-negative-integer?
  setup-fprintf : 
(->i ([stage string?] [format string?])
     ()
     #:rest (listof any/c) void)
  append-error : 
(->i ([cc cc?]
      [prefix string?]
      [exn (or/c exn? (cons/c string? string?) #f)]
      [out string?]
      [err string?]
      [message string?])
     void?)
  collects-tree : (listof any/c)
  use-places? : any/c = #t
parallel-compile 函数被 raco setup 用来并行编译集合。 worker-count 参数指定了在并行编译过程中产生的编译工人的数量。 use-places? 参数指定是否使用地方,否则就使用单独的进程。 setup-fprintf 和 append-error 函数传达中间的编译结果和错误。 collects-tree 参数是一个复合数据结构,包含 collects 目录的内存中树状表示。

当 append-error 的 exn 参数是一对字符串时,第一个字符串是错误信息的长形式, 第二个字符串是短形式(例如省略了评估上下文信息)。

Changed in version 6.1.1.8 of package base: 更改了 append-error ,允许一对错误字符串。
Changed in version 7.0.0.19: 增加了 #:use-places? 参数。
