1.9 编译为原始字节码

raco make 的 --no-deps 模式是一种改进的编译形式,因为它不跟踪导入的依赖关系。 然而,它确实支持在最初导入 scheme 的命名空间中的非模块源代码的编译。

在模块之外,顶层的 define-syntaxes,module, #%require, define-values-for-syntax 和 begin 表达式由 raco make --no-deps 特别处理: 表达式的编译时部分被评估,因为它可能会影响后面的表达式。

例如,当编译包含

(require racket/class)
(define f (class object% (super-new)))

来自 racket/class 库的 class 形式必须在编译时被绑定在编译命名空间中。 因此, require 表达式既被编译(出现在输出代码中)又被评估(用于进一步计算)。

许多定义形式扩展为 define-syntaxes 。 例如, define-signature 扩展到 define-syntaxes 。 在 --no-deps 模式下, raco make --no-deps 会在扩展后检测 define-syntaxes 和其他表达式, 所以顶级的 define-signature 表达式会影响后面的表达式的编译,正如程序员所期望的那样。

与此相反,源文件中的 load 或 eval 表达式在编译源文件时进行编译 (但 不进行求值 。 即使 load 表达式加载了语法或签名定义,这些也不会在文件编译时被加载。 影响读者的应用表达式也是如此,例如 (read-case-sensitive #t)。 raco make 的 -p 或 --prefix 标志会在编译命令行上指定的源文件之前取一个文件并加载它。

默认情况下,用于编译的命名空间是由 scheme 的 require 初始化的。 如果指定了 --no-prim 标志,命名空间就会改用 namespace-require/copy 来初始化, 它允许变异和重新定义所有的初始绑定(在变异的情况下,语法形式除外)。

一般来说,一个更好的解决方案是把所有要编译的代码放到一个模块中, 并在其默认模式下使用 raco make 。
