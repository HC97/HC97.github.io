6.7 安装 ".plt" 档案的API

setup/plt-single-installer 模块提供了一个用于安装单个 ".plt" 文件的功能。

6.7.1 非GUI安装程序

 (require setup/plt-single-installer) package: base

procedure

(run-single-installer 
  file 
  get-dir-proc 
  [#:show-beginning-of-file? show-beginning-of-file?]) 
 → void?
  file : path-string?
  get-dir-proc : (-> (or/c path-string? #f))
  show-beginning-of-file? : any/c = #f
创建一个单独的线程和命名空间,在该线程中以新的命名空间运行安装程序,并在该线程完成或死亡时返回。 它还创建了一个托管人(参见 Custodians) 来管理创建的线程, 为线程设置退出处理程序以关闭托管人,并在创建的线程终止或死亡时明确关闭托管人。

如果安装程序需要一个目标目录进行安装, get-dir-proc 过程被调用, #f 的结果意味着用户取消了安装。 通常情况下, get-dir-proc 是 current-directory 。

如果 show-beginning-of-file? 为真值,并且安装失败, 那么 run-single-installer 会打印文件的前 1000 个字符(试图帮助调试失败的原因)。

procedure

(install-planet-package file directory spec) → void?

  file : path-string?
  directory : path-string?
  spec : 
(list/c string? string?
        (listof string?)
        exact-nonnegative-integer?
        exact-nonnegative-integer?)
与 run-single-installer 类似,但运行安装程序, 将归档文件 file 安装到 directory 中,作为 spec 描述的 PLaneT 包。 用户专用的文档索引不会被重建, 所以 reindex-user-documentation 应该在安装完一组 PLaneT 包后运行。

procedure

(reindex-user-documentation) → void?

与 run-single-installer 类似,但只运行安装过程中重建用户专用文档起始页、搜索页和主索引的部分。

procedure

(clean-planet-package directory spec) → void?

  directory : path-string?
  spec : 
(list/c string? string?
        (listof string?)
        exact-nonnegative-integer?
        exact-nonnegative-integer?)
撤销 install-planet-package 的工作。用户专用的文档索引不会被重建, 所以 reindex-user-documentation应该在移除一组 PLaneT 软件包后运行。
