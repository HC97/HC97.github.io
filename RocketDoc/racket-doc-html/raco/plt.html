10 raco pack: 打包图书馆集合

raco pack 命令创建了一个文件和目录的存档。 以前,这种归档文件被直接用来向 Racket 用户分发库文件,但现在软件包管理器(见 Racket 中的包管理) 是首选的分发机制。

一个打包的档案通常有 ".plt" 的后缀。 raco pkg 命令可以识别 ".plt" 归档文件,将其作为一个包进行安装。 raco setup 命令(见 raco setup: 安装管理) 在使用 -A 标志时也支持 ".plt" 解压和安装, 但这种安装不能受益于 raco pkg 更通用的管理设施,而 raco unpack 命令(见 raco unpack: 解压图书馆收藏) 在本地解压压缩包,但不尝试安装。DrRacket 能识别 ".plt", 目前对此类归档文件的处理方式与 raco setup -A 相同。

一个归档文件包含以下内容:

  • 一组需要解压的文件和目录,以及表明这些文件和目录是相对于 Racket add-ons 目录(这是用户特定的)、 Racket 安装目录还是用户选择的目录进行解压的标志。

    归档的文件和目录是在命令行中提供给 raco pack 的,可以直接提供, 也可以在使用 --collect 标志时以集合名称的形式提供。

    --at-plt 标志表示文件和目录应该相对于用户的 add-ons 目录进行解包, 除非用户在解包时指定 Racket 安装目录。 --collection-plt 标志意味着 --at-plt。 --all-users 标志覆盖了 --at-plt, 它表明文件和目录应该始终相对于 Racket 安装目录进行解压。

  • 每个文件都有一个标志,表示在解压时是否覆盖现有的文件; 默认情况是保留旧的文件,但 --replace 标志可以替换存档中的所有文件。

  • 存档解压后要设置的集合列表(通过 raco setup); ++setup 标志会在存档的列表中添加一个集合名称, 但 --collection-plt 的每个集合都会自动添加。

  • 存档的名称,由解包界面报告给用户; --plt-name 标志设置存档的名称,但使用 --collect 时,会自动确定一个默认名称。

  • 一个所需集合的列表(有相关的版本号)和一个冲突集合的列表; raco pack 命令总是在所需列表中命名 "racket" 集合(使用集合的打包时间版本), raco pack 在冲突列表中命名每个打包的集合(这样一个集合就不会在同一集合的不同版本之上被解包), 当使用 --collect 时, raco pack 从集合的 "info.rkt" 文件中提取其他要求和冲突。

当不使用 --collect 时,为存档指定个别目录和文件。 每个文件和目录都必须用相对路径指定。 默认情况下,如果用 DrRacket 解压归档文件,会提示用户选择目标目录, 如果使用 raco setup 解压归档文件,文件和目录会相对于当前目录解压。 如果提供了 --at-plt 标志,文件和目录将相对于用户的 Racket add-ons 目录进行解包。 最后,如果提供了 --all-users 标志,文件和目录将被解压到 Racket 安装目录。

使用 --collect 标志来打包一个或多个集合;在所有平台上都可以用 / 作为路径分隔符来指定子集合。 在这种模式下, raco pack 会自动使用相对于 Racket 安装目录或 add-ons 目录的路径来归档文件, 并且在解包后会对这些集合进行设置。 此外, raco pack 会查阅每个集合的 "info.rkt" 文件,如下所述,以确定所需和冲突的集合。 最后, raco pack 查阅第一个集合的 "info.rkt" 文件,以获得存档的默认名称。 例如,下面的命令创建了一个 "sirmail.plt" 归档,用于分发 "sirmail" 集合:

  raco pack --collect sirmail.plt sirmail

当打包集合时, raco pack 检查每个集合的 "info.rkt" 文件的以下字段(见 "info.rkt" File Format):

  • requires — 形式为 (list (list coll vers) ...) 的列表, 其中每个 coll 是一个非空的相对路径字符串的列表,每个 vers 是一个(可能是空的)精确整数的列表。 指明的集合必须在解包时被安装,其版本序列与相应的 vers 中指定的版本序列尽可能地匹配。

    一个集合的版本由其 "info.rkt" 文件中的一个 version 字段表示,默认的版本是空列表。 版本序列概括了主要和次要版本号。 例如,当需要 '(), '(2), '(2 5), '(2 5 4) 或 '(2 5 4 7) 中的任何一个时,可以使用一个集合的版本 '(2 5 4 7) 。

  • conflicts — 形式为 (list coll ...) 的列表, 其中每个 coll 是一个非空的相对路径字符串的列表。 指明的集合在解包时必须 不 被安装。

例如, "sirmail" 集合中的 "info.rkt" 文件可能包含以下 info 声明:

#lang info
(define name "SirMail")
(define mred-launcher-libraries (list "sirmail.rkt"))
(define mred-launcher-names (list "SirMail"))
(define requires (list (list "mred")))

然后, "sirmail.plt" 文件(由上面的命令行例子创建)将包含 "SirMail" 这个名称。 当档案被解压时,解压程序将检查 "mred" 集合是否已经安装, 并且 "mred" 的版本与创建 "sirmail.plt" 时相同。

10.1 ".plt" 档案的格式

分发文件的扩展名 ".plt" 不是必须的,但 ".plt" 扩展名的惯例可以帮助用户识别分发文件的目的。

下面描述了分发文件的原始格式。 这种格式是未压缩的,对通信模式(文本与二进制)很敏感,所以发行格式是从原始格式中衍生出来的, 首先用 gzip 压缩文件,然后用 MIME base64 标准对 gzip 文件进行编码(它只依赖 A-Z, a-z, 0-9, +, / 和 = 等字符;当 base64 编码的文件被解码时,所有其他字符被忽略)。

原始格式是

  • PLT 是前三个字符。

  • 一个 S 表达式匹配

    (lambda (request failure)
      (case request
        [(name) name]
        [(unpacker) (quote mzscheme)]
        [(requires) (quote requires)]
        [(conflicts) (quote conflicts)]
        [(plt-relative?) plt-relative?]
        [(plt-home-relative?) plt-home-relative?]
        [(test-plt-dirs) test-dirs]
        [else (failure)]))

    其中 name, requires 等元变量代表 S 表达式,如下所示:

    • name — 一个描述档案内容的人可读字符串。这个名字只用于在解包时向用户打印信息。

    • requires — 一个在解压前需要安装的程序集的列表,其中有相关的版本;详情请参见 pack 。

    • conflicts — 一个在解压前 不 需要安装的集合的列表。

    • plt-relative? — 一个布尔值;如果是真的,那么存档的内容应该相对于 plt add-ons 目录解压。

    • plt-home-relative? — 一个布尔值;如果为真,并且如果 'plt-relative? 为真, 那么归档的内容应该相对于 Racket 的安装进行解包。

    • test-plt-dirs — #f 或 'paths , 其中 paths 是一个路径字符串的列表;在后一种情况下, 如果列表中的任何一个目录(相对于 Racket 安装)不能被用户写入,则 plt-home-relative? 的真值会被取消。

    S 表达式是用 read 从存档中提取的(结果 不 eval) 。

  • 一个匹配的 S 表达式

    (unit (import main-collects-parent-dir mzuntar)
          (export)
          (mzuntar void)
          (quote collections))

    其中 collections 是一个集合路径列表(每个集合路径是一个字符串列表); 一旦档案被解压, raco setup 将编译和设置指定的集合。

    S-表达式是用 read 从存档中提取出来的(结果 不 eval)。

存档继续进行 unpackables 。 Unpackables 被提取,直到找到文件的结尾(如 base64 编码的输入档案中的 = 所表示的)。

一个 unpackable 文件是以下的一种:

  • 符号 'dir 后面是一个列表 S 表达式。 build-path 程序将应用于列表,以获得目录的相对路径(相对路径与目标目录路径相结合,以获得完整路径)。

    'dir 符号和列表是用 read 从档案中提取的(而且结果 不 eval)。

  • 符号 'file, 一个列表,一个数字,一个星号,以及文件数据。列表指定了文件的相对路径,就像目录一样。 数字表示要解压的文件的大小,单位是字节。 星号表示文件数据的开始;接下来的 n 个字节被写入文件,其中n是文件的指定大小。

    符号、列表和数字都是用 read 的方式从档案中提取的(结果 不 eval)。 数字被读取后,输入的字符被丢弃,直到找到一个星号。 档案数据必须紧跟这个星号。

  • 符号 'file-replace 的处理方法与 'file 相同, 但如果文件已经存在于磁盘上,则存档中的文件将取代磁盘上的文件。

10.2 打包的API

 (require setup/pack) package: base
尽管 raco pack 命令可以用来创建大多数 ".plt" 文件, setup/pack 库提供了一个更通用的API来制作 ".plt" 档案。

procedure

(pack-collections-plt 
  dest 
  name 
  collections 
  [#:replace? replace? 
  #:at-plt-home? at-home? 
  #:test-plt-collects? test? 
  #:extra-setup-collections collection-list 
  #:file-filter filter-proc]) 
 → void?
  dest : path-string?
  name : string?
  collections : (listof (listof path-string?))
  replace? : boolean? = #f
  at-home? : boolean? = #f
  test? : boolean? = #t
  collection-list : (listof path-string?) = null
  filter-proc : (path-string? . -> . boolean?) = std-filter
创建由路径名 dest 指定的 ".plt" 文件, 使用该 name 作为报告给 raco setup 的名称作为存档的描述。

归档文件包含 collections 中列出的集合,它应该是一个集合路径的列表;每个集合路径又是一个相对路径字符串的列表。

如果 #:replace? 参数是 #f, 那么当任何一个集合已经存在时, 尝试解压归档将报告错误,否则解压归档将覆盖一个现有集合。

如果 #:at-plt-home? 参数是 #t, 那么归档的集合将被安装到Racket安装目录下, 而不是用户的目录下,如果主 "collects" 目录是用户可以写的。 如果 #:test-plt-collects? 参数是 #f (默认是 #t) , 并且 #:at-plt-home? 参数是 #t, 那么如果主 "collects" 目录不可写,则安装失败。

可选的 #:extra-setup-collections 参数是一个收集路径的列表, 它不包括在归档文件中,但在归档文件被解压时被设置。

可选的 #:file-filter 参数与 pack-plt 相同。

  encode? : boolean? = #t

  collection-list : (listof path-string?) = null
  plt-relative? : any/c = #f
  at-plt-home? : any/c = #f

  mod-and-version-list :

创建由路径名 dest 指定的 ".plt" 文件, 使用字符串 name 作为报告给 raco setup 的存档描述。 paths 参数必须是一个目录和文件的相对路径列表;这些文件和目录的内容将被打包到存档中。 可选的 as-paths 列表为 paths 的每个元素提供了要记录在归档文件中的路径 (因此,解压后的路径可能与打包后的路径不同)。

在调用 #:file-filter 过程时,会给出每个候选打包的相对路径。 如果它对某些路径返回 #f ,那么该文件或目录就会从存档中省略。 如果它返回 'file 或 'file-replace ,则该文件将以该模式打包,而不是默认模式。 默认是 std-filter 。

如果 #:encode? 参数是 #f, 那么输出的归档文件是原始形式的, 仍然必须进行 gzipped 和 mim-encoded (按照这个顺序)。默认值是 #t 。

#:file-mode 参数必须是 'file 或 'file-replace, 表示存档中的文件的默认模式。 默认值是 'file 。

#:unpack-unit 参数通常为 #f 。 否则,它必须是一个描述解包的 S 表达式;参见 ".plt" 档案的格式 ,了解更多关于该单元的信息。 如果 #:unpack-unit 参数是 #f, 就会生成一个合适的S表达式。

#:collections 参数是一个集合路径的列表,在档案解压后将被编译。 默认为 null 。

如果 #:plt-relative? 参数为真(默认为 #f), 那么归档文件和目录将相对于用户的 add-ons 目录或 Racket 安装目录进行解压, 这取决于 #:at-plt-home? 参数是否为真以及 #:test-plt-dirs 指定的目录是否可由用户写入。

如果 #:at-plt-home? 参数为真(默认为 #f), 那么 #:plt-relative? 必须为真, 存档会相对于 Racket 安装目录进行解压。 在这种情况下,以 "collects" 开头的相对路径会被映射到安装的主 "collects" 目录, 以此类推,用于下面的初始目录名称:

  • "collects"

  • "doc"

  • "lib"

  • "include"

如果 #:test-plt-dirs 是一个 list, 那么 #:at-plt-home? 必须是 #t 。 在这种情况下,当存档被解包时,如果 #:test-plt-dirs 列表中的任何一个相对目录被当前用户不可写, 那么存档毕竟是在用户的 add-ons 目录下解包的。

#:requires 参数的形状应该是 (list (list coll-path version) ...) , 其中每个 coll-path 是一个非空的相对路径字符串列表, 而每个 version 是一个(可能是空的)精确整数列表。 指明的集合必须在解包时被安装,其版本序列与对应的 version 中指定的版本序列尽可能地匹配。 一个集合的版本由其 "info.rkt" 文件中的 version 字段表示。

#:conflicts 参数应该具有 (list coll-path ...) 的形状, 其中每个 coll-path 是一个相对路径字符串的非空列表。 指明的集合在解包时必须 not 被安装。

procedure

(std-filter p) → boolean?

  p : path-string?
返回 #t ，除非 p 在剥离其目录路径并转换为字节字符串后,与以下正则表达式之一相匹配: ^[.]git, ^[.]svn$, ^CVS$, ^[.]cvsignore, ^compiled$, ^doc, ~$, ^#.*#$, ^[.]# 或 [.]plt$ 。

procedure

(mztar path    
  [#:as-path as-path]    
  output    
  filter    
  file-mode) → void?
  path : path-string?
  as-path : path-string? = path
