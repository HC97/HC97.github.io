运行带有各种选项的 raco setup :

  • file — 如果不是 #f, 将 file 安装为 ".plt" 存档。

  • collections — 如果不是 #f, 将设置限制在指定的集合(以及 pkgs 和 planet-specs, 如果有的话)

  • pkgs — 如果不是 #f, 则将设置限制在指定的软件包上(以及 collections 和 planet-specs, 如果有的话)

  • planet-spec — 如果不是 #f, 则将设置限制在指定的 PLaneT 包 (以及 collections 和 pkgs, 如果有的话)

  • make-user? — 如果 #f, 则禁用任何针对用户的设置动作

  • avoid-main? — 如果为真,则避免影响主安装的设置操作,而不是用户目录

  • make-docs? — 如果 #f, 则禁用任何针对文档的设置操作

  • make-doc-index? — 如果为真,则在 collections 之外建立文档索引集,假设文档已经建立

  • force-user-docs? — 如果为真,那么在构建文档时, 将创建一个用户特定的文档入口点,即使它的内容与安装时相同

  • check-pkg-deps? — 如果为真,那么即使 collections, pkgs 或 planet-specs 为非 #f ,也会启用包-依赖性检查

  • fix-pkg-deps? — 如果为真,则意味着 check-pkg-deps? , 并试图自动纠正发现的软件包依赖性问题

  • unused-pkg-deps? — 如果为真,则意味着 check-pkg-deps? ,并报告那些未使用的依赖关系

  • clean? — 如果为真,启用清洁模式,而不是设置模式

  • tidy? — 如果为真,即使 collections 或 planet-specs 为非 #f , 也能对文档和元数据索引进行全局整理

  • recompile-only? — 如果为真,不允许从源码编译,只允许时间戳调整和从独立于机器的形式重新编译

  • jobs — 如果不是 #f, 确定用于设置的最大并行任务数

  • fail-fast? — 如果为真,一旦发现错误就会中断当前线程

  • get-target-dir — 如果不是 #f, 则作为 current-target-directory-getter 的一个值。

如果 raco setup 完成时没有错误,结果是 #t ,否则是 #f 。

setup 不再使用 PLT_COMPILED_FILE_CHECK, 而是对 use-compiled-file-check 参数敏感。

Changed in version 6.1 of package base: 增加了 fail-fast? 参数。
Changed in version 6.1.1: 增加了 force-user-docs? 参数。
Changed in version 7.2.0.7: 增加了 check-pkg-deps?, fix-pkg-deps? 和 unused-pkg-deps? 参数。
Changed in version 7.2.0.8: 增加了 recompile-only? 参数。

6.6.1 raco setup 单元

 (require setup/setup-unit) package: compiler-lib

setup/setup-unit 库以单元形式提供了 raco setup 。 相关的 setup/option-sig 和 setup/option-unit 库提供了为 raco setup 的运行设置选项的接口。

例如,要解压一个 ".plt" 档案 "x.plt", 将 archives 参数设置为 (list "x.plt") , 并将 specific-collections 设为 null 。

链接选项和设置单元,以便你的选项设置代码在它们之间被初始化,例如:

(compound-unit
  ...
  (link ...
    [((OPTIONS : setup-option^)) setup:option@]
    [() my-init-options@ OPTIONS]
    [() setup@ OPTIONS ...])
  ...)

value

setup@ : unit?

导入

  • setup-option^

  • compiler^

  • compiler:option^

  • launcher^

  • dynext:file^

并不输出任何东西。调用 setup@ 开始设置过程。

6.6.4 设置启动模块

 (require setup) package: base
setup 库实现了 raco setup, 包括在它自己的实现需要编译时引导 raco setup 的部分。

当通过 racket 运行 setup 时, 提供 -N raco 以确保命令行参数的解析方式与 raco setup 相同,而不是传统的命令行模式。
