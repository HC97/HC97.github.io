1.2 字节码文件

一个文件 "‹name›.‹ext›" 被编译成字节码, 相对于该文件被保存为 "compiled/‹name›_‹ext›.zo" 。 因此,当 "‹name›.‹ext›" 需要作为一个模块时, 通常会自动使用该字节码文件,因为底层的 load/use-compiled 操作会检测到这样一个字节码文件。

例如,在一个包含以下文件的目录中:

  • "a.rkt":

    #lang racket
    (require "b.rkt" "c.rkt")
    (+ b c)

  • "b.rkt":

    #lang racket
    (provide b)
    (define b 1)

  • "c.rkt":

    #lang racket
    (provide c)
    (define c 1)

那么

  raco make a.rkt

触发创建 "compiled/a_rkt.zo", "compiled/b_rkt.zo" 和 "compiled/c_rkt.zo" 。随后

  racket a.rkt

从生成的 ".zo" 文件中加载字节码,只注意 ".rkt" 源, 以确认每个 ".zo" 文件有一个较晚的时间戳 (除非 PLT_COMPILED_FILE_CHECK 环境变量被设置为 exists, 在这种情况下,编译的文件被使用,没有时间戳检查)。

与此相反、

  raco make b.rkt c.rkt

将只创建 "compiled/b_rkt.zo" 和 "compiled/c_rkt.zo" , 因为 "b.rkt" 或 "c.rkt" 都没有导入 "a.rkt" 。
