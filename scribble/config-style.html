6.11.2 配置输出

库所使用的样式的实现在某种程度上取决于可单独配置的参数,配置也可以通过替换样式实现来实现。 Latex 输出在前一种方式中更具有可配置性,因为一个文档类决定了一组页面布局和字体属性,这些属性被其他命令所使用。 样式替换的配置方式对应于重新定义 Latex 宏或覆盖 CSS 类属性。 当 raco setup 构建 PDF 文档时,它使用这两种配置来为 Racket 手册产生标准的布局; 也就是说,它选择了一个特定的页面布局,并替换了一些 racket/base 样式。

两种文件实现了这两种配置:

  • prefix file 决定了 HTML 输出的 DOCTYPE 行或 Latex 输出的 \documentclass 配置(也许还有一些额外的包使用或其他配置)。

    默认的前缀文件是 "scribble-prefix.html" 和 "scribble-prefix.tex" , 在 "scribble" 集合中。

  • A style file 完善了文档中使用的样式的实现–通常只是 scribble/base 使用的 “built-in” 样式。

    默认的样式文件, "scribble" 集合中的 "scribble-style.css" 和 "scribble-style.tex" ,没有改变样式的实现。

对于一个给定的输出配置,通常一个特定的前缀文件与一个特定的样式文件一起工作。一些前缀或样式文件可能更容易重复使用。 目前,阅读默认文件是了解它们如何互动的最好方法。 前缀和/或样式文件也可能需要额外的伴随文件;例如,Latex 模式的前缀文件可能需要一个相应的 Latex 类文件。 默认的前缀和样式文件不需要额外的文件。

当通过 scribble 命令行工具渲染文档时,使用标志来选择前缀文件、样式文件和额外的伴随文件:

  • 使用 --prefix 标志选择前缀文件。 (选择前缀文件也会取消默认的伴随文件列表,如果有的话)。

  • 使用 --style 标志替换样式文件。 使用 ++style 标志添加额外的样式定义和重新定义。

  • 用 ++extra 添加额外的伴随文件。

当使用 scribble 命令行工具时,文档可以通过 html-defaults 和/或l latex-defaults style property 声明其默认样式、前缀和额外文件。 特别是,当使用 scribble 命令行工具生成 Latex 或 PDF 文档时, 其主要部分是用 #lang scribble/manual 实现的, 结果具有标准的 Racket 手册配置, 因为 scribble/manual 将 latex-defaults style property 与导出的文档相关联。 scribble/sigplan 语言也同样将默认配置与导出的文档相关联。 然而,作为与 require 一起导入的库, scribble/manual 和 scribble/sigplan 只是以一种可组合的方式实现新的样式。

无论文档是否通过 style property 有一个默认的前缀和样式文件配置, 都可以使用 scribble 命令行标志来覆盖默认配置。 此外,像 scribble/manual 和 scribble/sigplan 这样的语言, 只有在主文档部分还没有通过 title 的 #:style 参数添加这样的属性时, 才会给它添加 html-defaults 和/或 latex-defaults style property 。
