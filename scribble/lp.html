5 文字化编程

使用 scribble/lp2 编写的程序同时是两件事:一个程序和一个描述程序的文件:

  • 当程序运行时,所有的 chunk 表达式都被收集起来并缝合成一个程序,其余的模块被丢弃。

  • 当程序被提供给 Scribble 时,或者通过 include-section 在另一个 Scribble 文档中使用 (submod ... doc) 模块路径时, 模块的整个内容被当作普通的 Scribble 文档,其中的 chunk 被以类似于 codeblock 的方式排版。

例如,考虑这个程序:

#lang scribble/lp2
@(require scribble/manual)
 
Literate programs have chunks of code, like this one:
 
@chunk[<f>
       (define (f x)
         <fs-body>)]
 
and this one:
 
@chunk[<fs-body>
       (* x x)]
 
that, when assembled, produce a complete program, in this case:
 
@racketblock[(define (f x)
               (* x x))]
 

当这个文件以正常方式被 require 时,它定义了一个函数 f ,它的参数是平方的,而文档被忽略了。 当它被渲染成 Scribble 文件时,输出看起来像这样:

Literate programs have chunks of code, like this one:

<f> ::=
(define (f x)
  <fs-body>)

and this one:

<fs-body> ::=

(* x x)

that, when assembled, produce a complete program, in this case:

(define (f x)
  (* x x))

5.1 scribble/lp2 语言

 #lang scribble/lp2 package: scribble-lib
scribble/lp 语言提供了对识字编程的核心支持。 它的阅读方式类似于 scribble/base 程序, 但是它的绑定方式用两种形式扩展了 racket/base : chunk 和 CHUNK 。

更确切地说, scribble/lp2 中的模块在 doc 子模块中具有类似于 racket/base 的内容,这可以被 raco scribble 等工具识别。 chunk 和 CHUNK 形式的内容被缝合起来作为模块的直接内容。

chunk 和 CHUNK 的内容是通过首先 expand 写好的模块发现的。 这些内容被收集到一个新的模块中, 然后原来的模块内容被放入一个被扩展的 doc 子模块中(这样内容就被有效地重新扩展了)。 doc 子模块是用 module* 声明的。

要把一个名为 "file.scrbl" 的 scribble/lp2 文档纳入另一个 Scribble 文档, 请导入 doc 子模块:

@include-section[(submod "file.scrbl" doc)]

Added in version 1.8 of package scribble-lib.
Changed in version 1.17: 用 module* 而不是 module 来声明 doc 子模块。

syntax

(chunk id form ...)

引入一个分块,绑定 id ,以便在其他分块中使用。 通常情况下, id 以 < 开始,以 > 结束。

当运行包围的程序时,只有块内的代码被运行,其余的被忽略。

如果 id 是 <*>, 那么这个 chunk 就被用作文件中的主 chunk。 如果 <*> 没有被使用,那么文件中的第一个块将被视为主块。 如果某些块没有被主块引用(可能是通过主块引用的其他块间接引用),那么它就不包括在程序中,因此不会被运行。

这些 form 是用 racketblock 排版的,所以 code:comment 等可以用来调整输出。 那些调整输出的表单在运行程序时将从每个 form 中剥离出来。

Changed in version 1.17 of package scribble-lib: 剥离 code:comment 等,用于运行。

syntax

(CHUNK id form ...)

像 chunk 一样,但是用 RACKETBLOCK 排版, 所以在每个 form 中可以正常使用 unsyntax 。要转义,请使用 UNSYNTAX 。

5.2 scribble/lp 语言

 #lang scribble/lp package: scribble-lib
使用较早的 scribble/lp 语言编写的程序与 scribble/lp2 程序相似,只是模块不能直接提供给Scribble。 相反,必须使用 lp-include 来提取文件内容。

scribble/lp 语言只有效地绑定了 chunk 和 CHUNK, 而所有其他的文档绑定都是从使用 lp-include 的环境中提取的。

5.3 scribble/lp-include 模块

 (require scribble/lp-include) package: scribble-lib
scribble/lp-include 库通常在 Scribble 文档中使用–也就是以 #lang scribble/base 或 #lang scribble/manual, 之类的开头的模块,而不是 #lang racket 。

syntax

(lp-include filename)

包括 filename 的来源,作为识字程序的排版版本。
