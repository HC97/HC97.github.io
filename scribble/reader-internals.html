6.2 @ 阅读器内部

6.2.1 使用 @ 阅读器

你可以通过 Racket 的 #reader 形式使用阅读器:

#reader scribble/reader @foo{This is free-form text!}

或使用 at-exp 元语言,如 在语言中添加 @ 表达式 中所述。

注意,Scribble 阅读器将 @-forms 读作 S 表达式。 这意味着要由你以通常的方式为这些表达式赋予意义:使用 Racket 函数,定义你的函数,或者要求函数。 例如,在 racket 中输入上述内容很可能会产生一个 "对未定义标识符的引用" 的错误,除非 foo 被定义。 你可以使用 string-append 来代替,或者你可以把 foo 定义为一个函数(有可变的 arity)。

@-reader 的一个常见用途是在使用 Scribble 作为制作手册的文档系统时。 在这种情况下,手册文本可能以

#lang scribble/doc

其中以 "文本模式" 开始安装 @ 阅读器,之后将文件内容包裹到 Racket 模块中, 其中有许多有用的 Racket 和文档相关的功能,并使用 scribble/decode 将正文解析成一个文档。 更多信息见 文档阅读器 。

另一种使用阅读器的方法是使用 use-at-readtable 函数, 将当前的 readtable 切换为解析 @-forms 的 readtable。 你可以在一个命令行中这样做:

  racket -ile scribble/reader "(use-at-readtable)"

6.2.2 语法属性

Scribble 阅读器将属性附加到语法对象上。 这些属性在一些罕见的情况下可能是有用的。

Scribble 读到的表单被标记为 'scribble 属性,其值是一个由三个元素组成的列表: 第一个是 'form, 第二个是被从基准部分读到的项目数,第三个是主体部分的项目数(字符串、子表单和转义)。 在这两种情况下, 0 表示一个空的基准/主体部分, #f 表示相应的部分被省略了。 如果表格既没有部分,该属性就不会附加到结果上。 这个属性可以用来给基准部分和主体部分的表达式赋予不同的含义,例如,隐含引号的关键词:

(define-syntax (foo stx)
  (let ([p (syntax-property stx 'scribble)])
    (printf ">>> ~s\n" (syntax->datum stx))
    (syntax-case stx ()
      [(_ x ...)
       (and (pair? p) (eq? (car p) 'form) (even? (cadr p)))
       (let loop ([n (/ (cadr p) 2)]
                  [as '()]
                  [xs (syntax->list #'(x ...))])
         (if (zero? n)
           (with-syntax ([attrs (reverse as)]
                         [(x ...) xs])
             #'(list 'foo `attrs x ...))
           (loop (sub1 n)
                 (cons (with-syntax ([key (car xs)]
                                     [val (cadr xs)])
                         #'(key ,val))
                       as)
                 (cddr xs))))])))

 

> @foo[x 1 y (* 2 3)]{blah}

>>> (foo x 1 y (* 2 3) "blah")

'(foo ((x 1) (y 6)) "blah")

此外,Scribble 分析器使用语法属性来标记原始源文件中没有的语法项目–缩进空格和换行。 这两者都有一个 'scribble 属性; 一个由空格组成的缩进字符串将有 'indentation 作为该属性的值, 而换行线将有一个 '(newline S) 值,其中 S 是原始的换行线字符串, 包括它前后的空格(其中包括下面项目的缩进)。 这可以用来实现逐字环境:放弃缩进字符串,使用原始的源字符串而不是单换行字符串。这里有一个这样的例子。

(define-syntax (verb stx)
  (syntax-case stx ()
    [(_ cmd item ...)
     #`(cmd
        #,@(let loop ([items (syntax->list #'(item ...))])
             (if (null? items)
               '()
               (let* ([fst  (car items)]
                      [prop (syntax-property fst 'scribble)]
                      [rst  (loop (cdr items))])
                 (cond [(eq? prop 'indentation) rst]
                       [(not (and (pair? prop)
                                  (eq? (car prop) 'newline)))
                        (cons fst rst)]
                       [else (cons (datum->syntax-object
                                    fst (cadr prop) fst)
                                   rst)])))))]))

 

> @verb[string-append]{
    foo
      bar
  }

"foo\n  bar"

6.2.3 在语言中添加 @ 表达式

 #lang at-exp package: at-exp-lib
at-exp 语言在用于读取模块的读表中安装 @-reader 支持, 然后链到紧随 at-exp 之后指定的另一种语言的阅读器。

例如, #lang at-exp racket/base 将 @-reader 支持添加到 racket/base, 因此

#lang at-exp racket/base
 
(define (greet who) @string-append{Hello, @|who|.})
(greet "friend")

报告 "Hello, friend." 。

除了为模块主体配置阅读器外, at-exp 还为模块附加了一个运行时配置注释,因此如果它作为主模块使用, current-read-interaction 参数会被调整为使用 @-reader 可读扩展。

Changed in version 1.2 of package at-exp-lib: 增加了 current-read-interaction 的运行时配置。

6.2.4 接口

 (require scribble/reader) package: at-exp-lib
scribble/reader 模块为高级需求提供直接的 Scribble 阅读器功能。

procedure

(read [in]) → any

  in : input-port? = (current-input-port)

procedure

(read-syntax [source-name in]) → (or/c syntax? eof-object?)

  source-name : any/c = (object-name in)
  in : input-port? = (current-input-port)
通过使用由以下文件产生的读表来实现 Scribble 阅读器的功能

(make-at-readtable #:command-readtable 'dynamic
                   #:datum-readtable 'dynamic)

Changed in version 1.1 of package at-exp-lib: 改为对命令和数据读表使用 'dynamic 。

procedure

(read-inside [in]) → any

  in : input-port? = (current-input-port)

procedure

(read-syntax-inside [source-name 
  in 
  #:command-char command-char]) 
 → (or/c syntax? eof-object?)
  source-name : any/c = (object-name in)
  in : input-port? = (current-input-port)
  command-char : char? = #\@
就像 read 和 read-syntax 一样, 但是从 @{...} 里面开始, 返回一个(句法)列表,这对于实现默认为文本的语言很有用。

给出的 command-char 用于定制阅读器使用的读表,有效地将其传递给 make-at-readtable 。

Changed in version 1.1 of package at-exp-lib: 改为在命令和数据读表中使用 'dynamic 。

procedure

(make-at-readtable [#:readtable readtable 
  #:command-char command-char 
  #:command-readtable command-readtable 
  #:datum-readtable datum-readtable 
  #:syntax-post-processor syntax-post-proc]) 
 → readtable?
  readtable : readtable? = (current-readtable)
  command-char : char? = #\@
  command-readtable : (or/c readtable? 'dynamic) = readtable
  datum-readtable : 
(or/c readtable?
      boolean?
      (readtable? . -> . readtable?)
      'dynamic)
 = #t
  syntax-post-proc : (syntax? . -> . syntax?) = values
构建一个 @-readtable 。 关键字参数可以以多种方式定制生成的读取器:

  • readtable — 一个读表,作为 @-readtable 的基础。

  • command-char — 用于 @-forms 的字符。

  • command-readtable — 决定为读取 @-form 的命令部分而扩展的读表:

    • readtable — 扩展为使 | 成为分隔符,而不是符号引号字符

    • 'dynamic — 在解析命令时扩展 (current-readtable)， 使之成为 | 分隔符

  • datum-readtable — 用于读取 @-form 的datum部分的读表:

    • #t — 使用构建的 @-readtable 本身

    • readtable — 使用给定的读表

    • readtable-to-readtable 函数 — 从生成的 @-readtable

    • 'dynamic — uses (current-readtable) 中调用以构建一个readtable

    我们的想法是,你可能希望对数据集部分有完全不同的用途, 例如,为属性引入一个方便的 key=val 语法。

  • syntax-post-proc — 在每个语法值被解析后(但在它被包裹为标点符号之前),对其应用的函数。 你可以用它来进一步控制 @-forms 的使用,例如,使命令成为一个列表的头部:

    (use-at-readtable
      #:syntax-post-processor
      (lambda (stx)
        (syntax-case stx ()
          [(cmd rest ...) #'(list 'cmd rest ...)]
          [else (error "@ forms must have a body")])))

Changed in version 1.1 of package at-exp-lib: 增加了 #:command-readtable 和 #:datum-readtable 的 'dynamic 选项。

procedure

(make-at-reader #:syntax? syntax?    
  #:inside? inside? ...) → procedure?
  syntax? : #t
  inside? : #f
构建一个 @-readtable 的变体。 参数与 make-at-readtable 中的相同,另外两个参数决定了将被创建的读取器函数的种类: syntax? 选择 read 或类似 read-syntax 语法的函数, 而 inside? 选择一个普通的读取器或一个 -inside 变体。

根据这些输入,所产生的函数有不同的契约和动作。 预期的输入与 read 或 read-syntax 一样,取决于 syntax?; 该函数将读取单个表达式,或者,如果 inside? 为真,则读取整个输入; 在这种情况下,它将返回一个表达式的语法列表而不是单个表达式。

注意, syntax? 默认为 #t, 因为当你处理具体语法的读取时,这是更期望的常见情况。

请注意,如果 syntax? 为真,类 read 函数的构造是将语法结果简单地转换为数据集。

procedure

(use-at-readtable ...) → void?

将所有参数传递给 make-at-readtable, 并使用 current-readtable 来安装生成的 readtable。 它还可以通过 port-count-lines! 来对当前输入端口进行行计数。

这对于在 REPL 上使用 Scribble 语法非常有用。
