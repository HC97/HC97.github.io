2 @ 语法

Scribble @ 符号旨在为 Racket 代码中的自由格式文本提供便利,其中 “@” 被选为现有 Racket 代码中最少使用的字符之一。 一个 @ 表达式就是一个变相的 S 表达式。

通常, @ 符号是通过 scribble/base 或类似的语言启用的, 但你也可以使用 at-exp 元语言将 @ 符号添加到基于 S 表达式的语言中。例如、

  #lang at-exp racket

  (define v '@op{str})

相当于

#lang racket
(define v '(op "str"))

使用 #lang at-exp racket 可能是尝试本章中的例子的最简单方法。

2.1 Scribble 语法一瞥

回顾一下 @ 语法基础知识, @-forms 的具体语法大致为

@ ‹cmd› [ ‹datum›* ] { ‹text-body›* }

其中 @ 后面的三个部分都是可选的,但至少应该有一个是存在的。(三部分之间不允许有空格。) 粗略地说,符合上述语法的表单被读作

(‹cmd› ‹datum›* ‹parsed-body›*)

其中 ‹parsed-body› 是输入中每个 ‹text-body› 的翻译。 因此,最初的 ‹cmd› 决定了输入被翻译成的 Racket 代码。 常见的情况是当 ‹cmd› 是一个 Racket 标识符,它被读作一个普通的 Racket 形式,带有数据参数和/或字符串参数。

下面是一个例子:

  

@foo{blah blah blah}

  reads as  

(foo "blah blah blah")

这个例子显示了一个输入语法是如何被读成 Racket 语法的,而不是它被评估为什么。 如果你想看一个例子翻译成 S 表达式的形式,在 #lang at-exp racket 模块中,在它前面加一个引号。例如,运行

  #lang at-exp racket

  '@foo{blah blah blah}

在 DrRacket 中打印出的输出是

(foo "blah blah blah")

而省略引号

  #lang at-exp racket

  @foo{blah blah blah}

会触发一个语法错误,因为 foo 没有被绑定,而

  #lang at-exp racket

  (define (foo str) (printf "He wrote ~s.\n" str))

  @foo{blah blah blah}

打印输出

He wrote "blah blah blah".

这里有更多的 @-forms 的例子:

  

@foo{blah "blah" (`blah'?)}

  reads as  

(foo "blah \"blah\" (`blah'?)")

  

  

@foo[1 2]{3 4}

  reads as  

(foo 1 2 "3 4")

  

  

@foo[1 2 3 4]

  reads as  

(foo 1 2 3 4)

  

  

@foo[#:width 2]{blah blah}

  reads as  

(foo #:width 2 "blah blah")

  

  

@foo{blah blah

     yada yada}

  reads as  

(foo "blah blah" "\n"
     "yada yada")

  

  

@foo{

  blah blah

  yada yada

}

  reads as  

(foo
  "blah blah" "\n"
  "yada yada")

正如在最后一个例子中所看到的,多行和分隔它们的换行线被解析为多个 Racket 字符串。 更广泛地说,一个 ‹text-body› 是由文本、换行线和嵌套的 @-forms 组成的, 其中无论在 ‹text-body› 上下文还是在 Racket 上下文中, @-forms 的语法都是一样的。 一个不是 @-form 的 ‹text-body› 被转换为其 ‹parsed-body› 的字符串表达式; 换行线和下面的缩进被转换为 "\n" 和全空的字符串表达式。

  

@foo{bar @baz{3}

     blah}

  reads as  

(foo "bar " (baz "3") "\n"
     "blah")

  

  

@foo{@b{@u[3] @u{4}}

     blah}

  reads as  

(foo (b (u 3) " " (u "4")) "\n"
      "blah")

  

  

@C{while (*(p++))

     *p = '\n';}

  reads as  

(C "while (*(p++))" "\n" "  "
   "*p = '\\n';")

@-form 的命令部分也是可选的。 在这种情况下, @-form 被当作一个列表来读,这通常算作一个函数应用, 但它在用 Racket 通常的 quote 引用时也很有用:

  

@{blah blah}

  reads as  

("blah blah")

  

  

@{blah @[3]}

  reads as  

("blah " (3))

  

  

'@{foo

   bar

   baz}

  reads as  

'("foo" "\n"
        "bar" "\n"
        "baz")

最后,我们还可以去掉数据和文本部分,这样我们就只剩下命令了–它被原封不动地读取,而不是在括号内。 这在阅读 Racket 代码时并不有用,但它可以在文本块中用于转义 Racket 标识符。 在需要的时候,可以用一个竖条 (|) 来限定转义的标识符。

  

@foo

  reads as  

foo

  

@{blah @foo blah}

  reads as  

("blah " foo " blah")

  

@{blah @foo: blah}

  reads as  

("blah " foo: " blah")

  

@{blah @|foo|: blah}

  reads as  

("blah " foo ": blah")

实际上,命令部分可以是任何 Racket 表达式 [, { 或 | 开头), 这种转义特别有用,因为它们可以用于任何表达式。

  

@foo{(+ 1 2) -> @(+ 1 2)!}

  reads as  

(foo "(+ 1 2) -> " (+ 1 2) "!")

  

@foo{A @"string" escape}

  reads as  

(foo "A string escape")

  

@"@"

  reads as  

"@"

请注意,作为一种特殊情况,转义的 Racket 字符串会与周围的文本合并。 如果你想在你的字符串中使用特殊字符,这很有用,但如果是平衡的,转义的大括号就没有必要了。

  

@foo{eli@"@"barzilay.org}

  reads as  

(foo "eli@barzilay.org")

  

  

@foo{A @"{" begins a block}

  reads as  

(foo "A { begins a block")

  

  

@C{while (*(p++)) {

     *p = '\n';

   }}

  reads as  

(C "while (*(p++)) {" "\n" "  "
   "*p = '\\n';" "\n"
   "}")

在某些情况下,一个文本包含许多字面的 @ ,单独引用会很麻烦。 对于这种情况,大括号有一种替代语法: 一个文本块可以以 “|{” 开始,并相应地以 “}|” 结束。 此外,任何嵌套的 @-forms 必须以 “|@” 开始。

  

@foo|{bar}@{baz}|

  reads as  

(foo "bar}@{baz")

  

@foo|{bar |@x{X} baz}|

  reads as  

(foo "bar " (x "X") " baz")

  

@foo|{bar |@x|{@}| baz}|

  reads as  

(foo "bar " (x "@") " baz")

如果这样还不够方便,可以在 | 和大括号以及嵌套形式的 @ 之间添加标点符号。 (标点符号与小括号和 <> 相呼应)。 有了这个扩展, @-form 的语法可以作为 "此处字符串 "的替代。

  

@foo|--{bar}@|{baz}--|

  reads as  

(foo "bar}@|{baz")

  

@foo|<<{bar}@|{baz}>>|

  reads as  

(foo "bar}@|{baz")

反过来说就是, @ 符号如何在 Racket 代码中使用? 这几乎不是问题,因为 Racket 字符串和字符的读取方式还是一样的,而且 @ 被设置为非结束性的读取宏, 所以它可以在 Racket 标识符中的任何地方使用,除了在标识符的第一个字符。 当 @ 必须作为标识符的第一个字符出现时,你必须像正常 S 表达式语法中的其他非标准字符一样对标识符进行引号: 用反斜杠或竖杠。

  

(define \@email "foo@bar.com")

  reads as  

(define @email "foo@bar.com")

  

(define |@atchar| #\@)

  reads as  

(define @atchar #\@)

注意, [ 或 { 前不允许有空格,否则它们将成为以下文本(或 Racket 代码)的一部分。 (下面有更多关于在正文中使用大括号的内容)。

  

@foo{bar @baz[2 3] {4 5}}

  reads as  

(foo "bar " (baz 2 3) " {4 5}")

最后,请记住, @-forms 只是 S 表达式的另一种形式。 像任何 Racket 代码一样,标识符仍然通过它们出现的词汇环境获得其意义。 具体来说,当上述 @-form 出现在 Racket 表达式的上下文中时,词法环境必须为 foo 提供绑定,作为一个过程或一个宏;它可以被定义,需要,或被本地绑定(例如,用 let )。

> (let* ([formatter (lambda (fmt)
          (lambda args (format fmt (apply string-append args))))]
         [bf (formatter "*~a*")]
         [it (formatter "/~a/")]
         [ul (formatter "_~a_")]
         [text string-append])
    @text{@it{Note}: @bf{This is @ul{not} a pipe}.})

"/Note/: *This is _not_ a pipe*."

2.2 命令部分

除了是一个 Racket 标识符外,@-form 的 ‹cmd› 部分可以有 Racket 标点符号的前缀, 这将最终包裹 whole 表达式。

  

@`',@foo{blah}

  reads as  

`',@(foo "blah")

  

@#`#'#,@foo{blah}

  reads as  

#`#'#,@(foo "blah")

在编写 Racket 代码时,这意味着 @`',@foo{blah} 与 `@',@foo{blah} 和 `',@@foo{blah} 完全相同,但与后两者不同的是,第一个构造可以出现在正文中,意义相同,而其他两个则不行(见下文)。

在可选的标点符号前缀之后, ‹cmd› 本身不限于标识符,它可以是 任何 Racket 表达式。

  

@(lambda (x) x){blah}

  reads as  

((lambda (x) x) "blah")

  

@`(unquote foo){blah}

  reads as  

`(,foo  "blah")

此外,命令可以完全省略,这将在翻译中省略它,结果是一个 S 表达式通常只包含,比如说,字符串:

  

@{foo bar

  baz}

  reads as  

("foo bar" "\n"
           "baz")

  

  

@'{foo bar

   baz}

  reads as  

'("foo bar" "\n"
            "baz")

如果命令部分以 ; 开头( @ 和 ; 之间没有换行),那么这个结构就是一个注释。 有两种注释形式,一种用于任意文本和可能的嵌套注释,另一种用于行注释:

@;{ ‹any›* }
 
@; ‹anything-else-without-newline›*

在第一种形式中,注释的主体必须仍然能正确解析;见下面对主体语法的描述。 在第二种形式中,从 @; 到行尾的所有文本 and 所有后面的空格(或制表符)都是注释的一部分 (类似于 TeX 中的 % 注释)。

  

@foo{bar @; comment

     baz@;

     blah}

  reads as  

(foo "bar bazblah")

提示:如果你在一些不支持 @-forms 的 Scheme 模式下使用编辑器,平衡的注释可能会引起混淆, 因为打开的括号看起来是注释了,而关闭的括号却没有。在这种情况下,将结尾的大括号也 "注释 "掉是很有用的:

  @;{

    ...

  ;}

这样,编辑器就不会把文件视为有不平衡的括号。

如果只指定了 @-form 的 ‹cmd› 部分,那么结果就是只有命令部分,没有额外的小括号。 这使得它适用于正文中的 Racket 转义。(在下面关于正文部分的描述中会有更多这方面的内容)。

  

@foo{x @y z}

  reads as  

(foo "x " y " z")

  

@foo{x @(* y 2) z}

  reads as  

(foo "x " (* y 2) " z")

  

@{@foo bar}

  reads as  

(foo " bar")

最后,请注意,目前没有关于在命令本身中使用 @ 的特殊规则,这可能会导致以下情况:

  

@@foo{bar}{baz}

  reads as  

((foo "bar") "baz")

2.3 基准部分

datum 部分可以包含任意的 Racket 表达式,这些表达式简单地堆叠在正文参数之前:

  

@foo[1 (* 2 3)]{bar}

  reads as  

(foo 1 (* 2 3) "bar")

  

@foo[@bar{...}]{blah}

  reads as  

(foo (bar "...") "blah")

主体部分仍然可以省略,这基本上是普通(非文本) S 表达式的替代语法:

  

@foo[bar]

  reads as  

(foo bar)

  

@foo{bar @f[x] baz}

  reads as  

(foo "bar " (f x) " baz")

datum 部分可以是空的,这没有什么区别,除非主体被省略了。 然而,更常见的是使用一个空的主体来达到同样的目的。

  

@foo[]{bar}

  reads as  

(foo "bar")

  

@foo[]

  reads as  

(foo)

  

@foo

  reads as  

foo

  

@foo{}

  reads as  

(foo)

datum 部分最常见的用途是用于 Racket 表单,该表单期望在文本参数的主体之前有关键字-值参数。

  

@foo[#:style 'big]{bar}

  reads as  

(foo #:style 'big  "bar")

2.4 主体部分

主体部分的语法是为了尽可能地方便自由文本。 它几乎可以包含任何文本–唯一有特殊意义的字符是 @ 代表子 @-forms, } 代表文本的结束。 此外, { 被允许作为文本的一部分,它使匹配的 } 也成为文本的一部分–所以平衡的大括号是有效的文本。

  

@foo{f{o}o}

  reads as  

(foo "f{o}o")

  

@foo{{{}}{}}

  reads as  

(foo "{{}}{}")

如上所述,文本变成了一连串的字符串参数,用于生成表格。 行首和行尾的空格被丢弃,换行线变成单独的 "\n" 字符串(也就是说,它们不会与其他主体部分合并); 也请看下面关于换行线和缩进的信息。 如果空格出现在打开的 { 之后(关闭的 } 之前),而后面(之前)也有文本, 则 不会 被丢弃;具体来说,在单行的正文中,空格会被保留。

  

@foo{bar}

  reads as  

(foo "bar")

  

@foo{ bar }

  reads as  

(foo " bar ")

  

@foo[1]{ bar }

  reads as  

(foo 1 " bar ")

如果 @ 出现在正文中,那么它被解释为 Racket 代码,这意味着 @-reader 被递归应用, 产生的语法作为 S 表达式的一部分,出现在其他字符串内容中。

  

@foo{a @bar{b} c}

  reads as  

(foo "a " (bar "b") " c")

如果嵌套的 @ 结构只有一个命令–没有主体或数据部分–它将不会出现在子表格中。 鉴于命令部分可以是任何 Racket 表达式,这使得 @ 成为任意 Racket 代码的一般转义。

  

@foo{a @bar c}

  reads as  

(foo "a " bar " c")

  

@foo{a @(bar 2) c}

  reads as  

(foo "a " (bar 2) " c")

这对字符串特别有用,它可以用来包含任意的文本。

  

@foo{A @"}" marks the end}

  reads as  

(foo "A } marks the end")

请注意,转义后的字符串会(有意地)与文本的其他部分合并。这也适用于 @ :

  

@foo{The prefix: @"@".}

  reads as  

(foo "The prefix: @.")

  

@foo{@"@x{y}" --> (x "y")}

  reads as  

(foo "@x{y} --> (x \"y\")")

2.4.1 替代的主体语法

除了上述方法外,还有一种用于正文的替代语法,它为正文的结尾指定了一个新的标记: 使用 |{ 作为开头的标记,让文本以 }| 结束。

  

@foo|{...}|

  reads as  

(foo "...")

  

@foo|{"}" follows "{"}|

  reads as  

(foo "\"}\" follows \"{\"")

  

@foo|{Nesting |{is}| ok}|

  reads as  

(foo "Nesting |{is}| ok")

这也适用于子 @-forms — @ 必须以 | 为前缀:

  

@foo|{Maze

      |@bar{is}

      Life!}|

  reads as  

(foo "Maze" "\n"
     (bar "is") "\n"
      "Life!")

  

  

@t|{In |@i|{sub|@"@"s}| too}|

  reads as  

(t "In " (i "sub@s") " too")

注意,子表格使用自己的分隔符, {...} 或 |{...}| 。 这意味着你可以自由地将 Scribble 文本与 @-forms 进行复制和粘贴, 只要在紧邻的文本有前缀的情况下将 @ 前缀化。

为了更好地控制,你可以在开头的分隔符中,在 | 和 { 之间添加字符。 放在那里的字符(只能是非字母数字的 ASCII 字符,不包括 { 和 @) 也应该用于子 @-forms, 正文结束标记应该有这些字符的反向顺序,并有类似于派生的字符((, [, <) 镜像。

  

@foo|<<<{@x{foo} |@{bar}|.}>>>|

  reads as  

(foo "@x{foo} |@{bar}|.")

  

@foo|!!{X |!!@b{Y}...}!!|

  reads as  

(foo "X " (b "Y") "...")

最后,请记住,你可以使用表达式转义与 Racket 字符串进行混淆的情况。 当你只需要引用短篇时,这个方法很有效,而当你有较大的多行正文时,上述方法也很有效。

2.4.2 Racket 表达式转义

在某些情况下,你可能想在触及以下文本的位置使用一个 Racket 标识符(或数字或布尔值等); 在这些情况下,你应该用一对 | 字符包围转义的 Racket 表达式。酒吧内的文本被解析为一个 Racket 表达式。

  

@foo{foo@bar.}

  reads as  

(foo "foo" bar.)

  

@foo{foo@|bar|.}

  reads as  

(foo "foo" bar ".")

  

@foo{foo@3.}

  reads as  

(foo "foo" 3.0)

  

@foo{foo@|3|.}

  reads as  

(foo "foo" 3 ".")

这种形式是一种通用的 Racket 表达式转义,当你使用这种形式时,没有正文或基准部分。

  

@foo{foo@|(f 1)|{bar}}

  reads as  

(foo "foo" (f 1) "{bar}")

  

@foo{foo@|bar|[1]{baz}}

  reads as  

(foo "foo" bar "[1]{baz}")

这也适用于字符串表达式,但请注意,与上面不同的是,字符串(故意的)不与其他文本合并:

  

@foo{x@"y"z}

  reads as  

(foo "xyz")

  

@foo{x@|"y"|z}

  reads as  

(foo "x" "y" "z")

表达式转义也适用于 任何 数量的表达式、

  

@foo{x@|1 (+ 2 3) 4|y}

  reads as  

(foo "x" 1 (+ 2 3) 4 "y")

  

  

@foo{x@|*

        *|y}

  reads as  

(foo "x" *
     * "y")

似乎 @|| 没有什么用处–但请记住,这些转义永远不会与周围的文本合并,当你想控制表单中的子表达式时,这可能很有用。

  

@foo{Alice@||Bob@|

     |Carol}

  reads as  

(foo "Alice" "Bob"
     "Carol")

请注意, @|{...}| 既可以被解析为转义表达式,也可以被解析为 @-form 的 Racket 命令部分。 本例中使用的是后者(因为 Racket 代码中使用大括号的意义不大。

  

@|{blah}|

  reads as  

("blah")

2.4.3 注释

如上所述,有两种 @-form 的注释: @;{...} 是对整个文本的(可嵌套的)注释(遵循与 @-forms 形式相同的规则), 而 @;... 是行注释。

  

@foo{First line@;{there is still a

                  newline here;}

     Second line}

  reads as  

(foo "First line"
     "\n"
     "Second line")

行注释的一个有用的属性是,它们会延续到行的末尾 and 所有后面的空格(或标签)。 利用这一点,你可以进一步控制子表格。

  

@foo{A long @;

     single-@;

     string arg.}

  reads as  

(foo "A long single-string arg.")

注意这与使用 @|| 有什么不同,因为它周围的字符串不会被合并。

2.4.4 空格、换行和缩进

@-form 语法以一种特殊的方式处理空格和换行符,是为了合理地处理文本。 如上所述,除了 { 和文本之间的空格,或者文本和 } 之间的空格,正文行的开头和结尾的空格都会被丢弃。

  

@foo{bar}

  reads as  

(foo "bar")

  

  

@foo{ bar }

  reads as  

(foo " bar ")

  

  

@foo{ bar

     baz }

  reads as  

(foo " bar" "\n"
     "baz ")

除非正文中只有换行符,否则在开括号之后或闭括号之前的单个换行符将被丢弃;其他换行符将作为 "\n" 字符串被读取。

  

@foo{bar

}

  reads as  

(foo "bar")

  

  

@foo{

  bar

}

  reads as  

(foo
  "bar")

  

  

@foo{

 

  bar

 

}

  reads as  

(foo
  "\n"
  "bar" "\n")

  

  

@foo{

  bar

 

  baz

}

  reads as  

(foo
  "bar" "\n"
  "\n"
  "baz")

  

  

@foo{

}

  reads as  

(foo "\n")

  

  

@foo{

 

}

  reads as  

(foo "\n"
     "\n")

  

  

@foo{ bar

     baz }

  reads as  

(foo " bar" "\n"
     "baz ")

主体行开头的空格不会出现在生成的 S 表达式中,但是每一行的列都会被注意到,所有空格的缩进字符串会被添加,所以结果具有相同的缩进。 根据每行与最左边的语法对象的距离,为每行添加缩进字符串(空行除外)。 (注意:如果你在 Racket REPL 上尝试这些例子,你应该知道读者不知道 “> ” 提示。)

  

@foo{

  bar

  baz

  blah

}

  reads as  

(foo
  "bar" "\n"
  "baz" "\n"
  "blah")

  

  

@foo{

  begin

    x++;

  end}

  reads as  

(foo
  "begin" "\n" "  "
  "x++;" "\n"
  "end")

  

  

@foo{

    a

   b

  c}

  reads as  

(foo "  "
     "a" "\n" " "
     "b" "\n"
     "c")

如果第一个字符串来自开头的 { 行,它不会被预置缩进(但它可以影响用于缩进的最左边的语法对象)。 这在格式化结构化代码以及文本时是有意义的(见下面区块中的最后一个例子)。

  

@foo{bar

       baz

     bbb}

  reads as  

(foo "bar" "\n" "  "
     "baz" "\n"
     "bbb")

  

  

@foo{ bar

        baz

      bbb}

  reads as  

(foo " bar" "\n" "   "
     "baz" "\n" " "
     "bbb")

  

  

@foo{bar

   baz

   bbb}

  reads as  

(foo "bar" "\n"
     "baz" "\n"
     "bbb")

  

  

@foo{ bar

   baz

   bbb}

  reads as  

(foo " bar" "\n"
     "baz" "\n"
     "bbb")

  

  

@foo{ bar

   baz

     bbb}

  reads as  

(foo " bar" "\n"
     "baz" "\n" "  "
     "bbb")

  

  

@text{Some @b{bold

  text}, and

  more text.}

  reads as  

(text "Some " (b "bold" "\n"
                 "text")", and" "\n"
                        "more text.")

注意,每个 @ 格式都被解析为一个 S 表达式,它有自己的缩进。 这意味着 Scribble 源码可以像代码一样缩进,但是如果缩进很重要,那么你可能需要将外部项目的缩进应用到内部项目的所有行中。 例如,在

@code{

  begin

    i = 1, r = 1

    @bold{while i < n do

            r *= i++

          done}

  end

}

中,格式化器需要将 2 个空格的缩进应用到 bold 主体的渲染中。

请注意,为了让第一行的文本被算作最左边的一行,输入端口的行和列核算应该打开 (use-at-readtable 为当前输入端口打开它们)。 如果没有这一点、

@foo{x1

       x2

       x3}

如果源核算没有打开,在解析的 S 表达式中就不会有 2 个空格的缩进,但是

@foo{x1

       x2

     x3}

会有(由于最后一行的原因)。请注意这一点,因为这可能是 Racket 代码的一个问题,比如说:

@code{(define (foo x)

        (+ x 1))}

对于行首(或行尾)的空格很重要的罕见情况,你可以用 @|| 来开始(或结束)一行。

  

@foo{

  @|| bar @||

  @|| baz}

  reads as  

(foo
  " bar " "\n"
  " baz")
