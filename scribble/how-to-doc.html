4.1 开始使用文档

尽管 scribble 命令行工具会从 Scribble 文档中生成输出,但 Racket 库的文档通常是由 raco setup 构建的。 本章强调 raco setup 的方法,它更自动地支持跨文档的链接。

关于使用 scribble 命令行工具的信息,请参见 开始使用 。

4.1.1 设置库文档

要记录一个集合,包括一个由 package 实现的集合:

  • 在你的集合中创建一个文件,文件扩展名为 ".scrbl" 。 注意,你选择的文件名将决定输出目录的名称,而且这个目录名在所有安装的文件中必须是唯一的。 这些说明的其余部分假设文件被称为 "manual.scrbl" (但在实践中要选一个更具体的名字)。

  • 像这样启动 "manual.scrbl" :
    #lang scribble/manual
     
    @title{My Library}
     
    Welcome to my documentation: @racket[(list 'testing 1 2 3)].

    第一行以 "文本 "模式启动文件,并选择 Racket 手册的输出格式。 它还引入了 title 和 racket 这样的绑定,用于编写 Racket 文档。

  • 在你的集合的 "info.rkt" 中添加以下条目:

    (define scribblings '(("manual.scrbl" ())))

    上面的 () 是一个选项列表。当你的文档大到你想把它分成多页时, 添加 'multi-page 选项(省略引号,因为整个定义的右侧已经被引号了)。

    如果你还没有一个 "info.rkt" 模块,这里有一个合适的完整模块:

    #lang info
    (define scribblings '(("manual.scrbl" ())))

  • 运行 raco setup 来构建你的文档。 对于一个集合,可以选择在集合名称后面提供 -l 来限制构建过程在该集合中进行。

  • 对于一个安装为用户专用的集合(例如,用户 package scope), 生成的文档是集合目录下的 "doc/manual/index.html" 。 然而,如果这个集合是安装范围的,那么生成的文档是 "manual/index.html" ,在安装的 "doc" 目录下。

4.1.2 Racket 排版和超链接

在本章开头的文档源 (设置库文档) 中, Racket 表达式 (list 'testing 1 2 3) 被正确排版, 但 list 标识符没有超链接到通常的定义。 要使 list 被超链接,可以添加一个 require 形式,像这样:

@(require (for-label racket))

这个带有 for-label 声明的 require 为 racket 模块的每个出口引入了一个文档时间绑定。 当文档被构建时, racket 表单检测到 list 的绑定,因此它产生了一个对 list 规范的引用。 设置过程检测到这个引用,它在现有的文档中找到匹配的规范,并最终将超链接指向该规范。

基于 for-label 和 racket 的超链接是链接到单个文档以外信息的首选机制。 这样的链接不需要关于约束力在哪里以及如何在其他地方被记录的信息:

#lang scribble/manual
@(require (for-label racket))
 
@title{My Library}
 
See also @racket[list].

racket 形式将一个 Racket 表达式排版为内联文本,所以它忽略了表达式的源格式。 与此相反,racketblock 形式是对内嵌的 Racket 代码进行排版,而且它保留了表达式在文档源中的格式化。

#lang scribble/manual
@(require (for-label racket))
 
@title{My Library}
 
Some example Racket code:
 
@racketblock[
(define (nobody-understands-me what)
  (list "When I think of all the"
        what
         "I've tried so hard to explain!"))
(nobody-understands-me "glorble snop")
]
4.1.3 章节超链接

文档中的 section 声明可以包括一个 #:tag 参数,声明一个超链接目标标签。 secref 函数会生成一个超链接,使用章节名称作为超链接的文本。 使用 seclink 来创建一个带有章节标题以外的文本的超链接。

下面的例子说明了章节超链接:

#lang scribble/manual
@(require (for-label racket))
 
 
@title{My Library}
 
Welcome to my documentation: @racket[(list 'testing 1 2 3)].
 
@table-of-contents[]
 
 
@section[#:tag "chickens"]{Philadelphia Chickens}
 
Dancing tonight!
 
 
@section{Reprise}
 
See @secref{chickens}.

由于页面太短,如果你修改 "info.rkt" 文件以添加 'multi-page 标志,上述例子中的超链接会更有效:

(define scribblings '(("manual.scrbl" (multi-page))))

一个部分可以有一个 tag prefix ,适用于从该部分以外看到的所有标签。 这样的前缀会在 raco setup 的过程中自动给予每个顶层文件。 因此,在不同的文档中引用一个章节标签需要使用一个前缀,这个前缀是基于目标文档的主源文件。 下面的例子链接到 Racket 参考手册中的一个章节:

#lang scribble/manual
@(require (for-label racket))
@(define ref-src
   '(lib "scribblings/reference/reference.scrbl"))
 
@title{My Library}
 
See also @italic{@secref[#:doc ref-src]{pairs}}.

然而,正如在 Racket 排版和超链接 中提到的,基于 (require (for-label ....)) 和 racket 的跨文档引用通常比使用 secref 的跨文档引用更好。

4.1.4 定义 Racket 的绑定方式

使用 defproc 来记录一个过程, defform 来记录一个句法形式, defstruct 来记录一个结构类型,等等。 这些形式提供了一致的定义格式,并且它们为基于 racket 的超链接声明了超链接目标。

要记录一个由包含 "manual.scrbl" 的 "my-lib" 集合中的 "helper.rkt" 导出的 my-helper 程序:

  • 使用 (require (for-label "helper.rkt")) 来导入关于 "helper.rkt" 的绑定信息, 以便在排版标识符时使用。 相对引用 "helper.rkt" 是有效的,因为它是相对于文档来源的。

  • 添加一个 @defmodule[my-lib/helper] 声明,它指定了在本节中被记录的库。 defmodule 形式需要一个绝对的模块名称 mylib/helper, 而不是一个相对的引用 "helper.rkt", 因为给 defmodule 的模块路径会逐字出现在生成的文档中。

  • 使用 defproc 来记录这个过程。

将这些片段添加到 "manual.scrbl" 中,我们可以得到以下内容:

#lang scribble/manual
@(require (for-label racket
                     "helper.rkt"))
 
@title{My Library}
 
@defmodule[my-lib/helper]
 
@defproc[(my-helper [lst list?])
         (listof
          (not/c (one-of/c 'cow)))]{
 
 Replaces each @racket['cow] in @racket[lst] with
 @racket['aardvark].}

在 defproc 中,程序的每个参数都有一个合同。 在这个例子中, lst 参数的契约是 list?, 这是一个列表的契约。 在结束参数序列的圆括号之后,必须给出结果的契约; 在这个例子中, my-helper 保证结果是一个列表,其中没有一个元素是 'cow 。

在这个例子和它产生的文档中,有一些事情需要注意:

  • 契约中的 list?, listof 等元素被超链接到它们的文档。

  • 结果合同在生成的文档中的格式与源文件中的格式相同。 也就是说,合同的源码布局被保留下来。 (在这种情况下,把合同全部放在一行会更好)。

  • 在记录 my-helper 的散文中, lst 被自动排成斜体,与蓝框中的排版一致。 racket 形式基本上知道它是在一个带有参数 lst 的过程的范围内使用。

  • 如果你把鼠标指针悬停在 my-helper 上, 会有一个弹出窗口报告说它是由 my-lib/helper 提供的。

  • 如果你现在在任何文档中使用 my-helper , 只要该文档源也有 "helper.rkt" 的 (require (for-label ....)) , 那么该引用就会超链接到上面的定义。

参见 defproc*、 defform 等,以了解更多关于记录 Racket 绑定的形式的信息。

4.1.5 展示Racket实例

来自 scribble/eval 的 examples 表格可以帮助你在文档中生成例子。 要使用 examples, 要记录的程序必须适合在文档时使用, 但 examples 表格不使用任何由 require 引入到文档源的绑定。 相反,使用 make-base-eval 创建一个新的评价器,有自己的命名空间, 并使用 interaction-eval 来要求该评价器中的 "helper.rkt" 。 最后,为 examples 提供相同的评估器:

#lang scribble/manual
@(require scribble/eval
          (for-label racket
                     "helper.rkt"))
 
@title{My Library}
 
@defmodule[my-lib/helper]
 
@defproc[(my-helper [lst list?])
         (listof
          (not/c (one-of/c 'cow)))]{
 Replaces each @racket['cow] in @racket[lst] with
 @racket['aardvark].
 
 @(define helper-eval (make-base-eval))
 @interaction-eval[#:eval helper-eval
                   (require "helper.rkt")]
 @examples[
     #:eval helper-eval
     (my-helper '())
     (my-helper '(cows such remarkable cows))
   ]}
4.1.6 多页章节

设置 'multi-page 选项(见 章节超链接) 会使文档的每个顶层章节都被渲染成一个单独的 HTML 页面。

要把子章节推到单独的页面上, 可以使用 'toc 样式来包围章节(如由 title,section, subsection 等开始), 并使用 local-table-of-contents 来生成子章节的超链接。

修改上一节中的 "cows.scrbl" :

#lang scribble/manual
 
@title[#:style '(toc)]{Cows}
 
@local-table-of-contents[]
 
@section[#:tag "singing"]{Singing}
Wherever they go, it's a quite a show.
 
@section{Dancing}
See @secref["singing"].

要运行这个例子,记得要修改 "info.rkt" 以添加 'multi-page 样式。 你可能还想在 "manual.scrbl" 中添加一个对 table-of-contents 的调用。

table-of-contents 和 local-table-of-contents 之间的区别是,后者在 Latex 输出时被忽略了。

当使用 local-table-of-contents 时, 在调用 local-table-of-contents 之前包含介绍性文本往往是有意义的。 当介绍性文本不那么重要,并且本地目录很短时,把介绍性文本放在调用 local-table-of-contents 之后可能是合适的。
