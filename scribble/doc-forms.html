4.2.3 记录表单、函数、结构类型和值

syntax

(defproc options prototype
         result-contract-expr-datum
         maybe-value
         pre-flow ...)
 
prototype = (id arg-spec ...)
  | (prototype arg-spec ...)
     
arg-spec = (arg-id contract-expr-datum)
  | (arg-id contract-expr-datum default-expr)
  | (keyword arg-id contract-expr-datum)
  | (keyword arg-id contract-expr-datum default-expr)
  | ellipses
  | ellipses+
     
options = maybe-kind maybe-link maybe-id
     
maybe-kind = 
  | #:kind kind-content-expr
     
maybe-link = 
  | #:link-target? link-target?-expr
     
maybe-id = 
  | #:id [src-id dest-id-expr]
     
maybe-value = 
  | #:value value-expr-datum
     
ellipses = ...
     
ellipses+ = ...+
产生一个流程元素的序列(封装在一个 splice 中)来记录一个名为 id 的过程。 嵌套的 prototype 对应于一个 curried 函数,就像 define 中一样。 除非指定 link-target?-expr 并产生 #f, 否则 id 是有索引的, 而且它也被注册了,所以该标识符的 racket-typeset 使用(具有相同的 for-label 绑定)会被超链接到这个文档。

例子:
@defproc[(make-sandwich [ingredients (listof ingredient?)])
         sandwich?]{
  Returns a sandwich given the right ingredients.
}
 
@defproc[#:kind "sandwich-maker"
         (make-reuben [ingredient sauerkraut?] ...
                      [#:veggie? veggie? any/c #f])
         sandwich?]{
  Produces a reuben given some number of @racket[ingredient]s.
 
  If @racket[veggie?] is @racket[#f], produces a standard
  reuben with corned beef. Otherwise, produces a vegetable
  reuben.
}

Renders like:

procedure

(make-sandwich ingredients) → sandwich?

  ingredients : (listof ingredient?)
Returns a sandwich given the right ingredients.

sandwich-maker

(make-reuben ingredient    
  ...    
  [#:veggie? veggie?]) → sandwich?
  ingredient : sauerkraut?
  veggie? : any/c = #f
Produces a reuben given some number of ingredients.

If veggie? is #f, produces a standard reuben with corned beef. Otherwise, produces a vegetable reuben.

当 id 被索引和注册时, 在一个封闭的部分中的 defmodule 或 declare-exporting 形式(或其中一个变体)决定了被定义的 id 绑定。 id 也应该有一个 for-label 绑定(如由 (require (for-label ....))) , 与定义绑定相匹配;否则,定义的 id 在定义中不能正确排版。

每个 arg-spec 必须有以下形式之一:

(arg-id contract-expr-datum)

一个参数,其合同 contract-expr-datum 指定,通过 racketblock0 进行排版。

(arg-id contract-expr-datum default-expr)

像前面的情况一样,但有一个默认值。 所有具有缺省值的参数必须被分组,但它们可以在必要参数的中间。

(keyword arg-id contract-expr-datum)

像第一种情况一样,但是对于一个基于关键字的参数。

(keyword arg-id contract-expr-datum default-expr)

和前面的情况一样,但有一个默认值。

...

前面的参数的任何数字。 这种形式通常用在最后,但基于关键字的参数可以合理地出现在后面。 关于最后一个参数前的 ... 的使用,也请参见 append 的文档。

...+

一个或多个前面的参数(通常在最后,像 ...) 。

The result-contract-expr-datum 是通过 racketblock0 来排版的,它代表了对过程结果的契约。

decode 后的 pre-flow 记录了该过程。 在这个描述中,使用 racket, racketblock 等对 arg-id 的引用被排版为过程参数。

在 pre-flow 之前的所有信息的排版都忽略了源的布局,除了对契约和默认值表达式保留了本地格式。 信息的格式化是为了适应(如果可能的话)由 current-display-width 参数指定的字符数。

一个可选的 #:kind 规范选择了装饰标签,默认为 "procedure" 。 kind-content-expr 的 #f 结果使用默认值,否则 kind-content-expr 应该产生 content? 意义上的内容。 替代的标签应该是全小写的。

如果提供了 #:id [src-id dest-id-expr] , 那么 src-id 就是出现在 prototype 中的标识符(将被一个定义实例所取代), 而 dest-id-expr 产生的标识符将代替 src-id 被记录。 src-id 和 dest-id-expr 角色之间的这种分割对于 defproc 的功能抽象很有用。

如果给出了 #:value value-expr-datum , value-expr-datum 将使用 racketblock0 进行排版并包含在文档中。 作为对读者的一种服务,请使用 #:value 来记录简单、简短的函数。

syntax

(defproc* options
          ([prototype
            result-contract-expr-datum
            maybe-value] ...+)
          pre-flow ...)
就像 defproc 一样,但是对于具有相同 id 的多个案例。 多个不同的 id 也可以用一个 defproc* 来定义, 因为最好是一次性记录一组相关的程序(但多个 defproc 由 deftogether 分组也适用于这种情况)。

当一个 id 有多个调用情况时,要么它们必须用一个 defproc* 来定义, 这样该 id 就有一个定义点,否则除了一个定义外,其他定义都应该使用 #:link-target? #f 。

例子:
@defproc*[([(make-pb&j) sandwich?]
           [(make-pb&j [jelly jelly?]) sandwich?])]{
  Returns a peanut butter and jelly sandwich. If @racket[jelly]
  is provided, then it is used instead of the standard (grape)
  jelly.
}

Renders like:

procedure

(make-pb&j) → sandwich?

(make-pb&j jelly) → sandwich?
  jelly : jelly?
Returns a peanut butter and jelly sandwich. If jelly is provided, then it is used instead of the standard (grape) jelly.

syntax

(defform options form-datum
  maybe-grammar maybe-contracts
  pre-flow ...)
 
options = maybe-kind maybe-link maybe-id maybe-literals
     
maybe-kind = 
  | #:kind kind-content-expr
     
maybe-link = 
  | #:link-target? link-target?-expr
     
maybe-id = 
  | #:id id
  | #:id [id id-expr]
     
maybe-literals = 
  | #:literals (literal-id ...)
     
maybe-grammar = 
  | #:grammar ([nonterm-id clause-datum ...+] ...)
     
maybe-contracts = 
  | 
#:contracts ([subform-datum contract-expr-datum]
             ...)
产生一个流元素的序列(封装在一个 splice 中)来记录一个由 id 命名的语法形式(或 id-expr 的结果),其语法由 form-datum 描述。 如果没有使用 #:id 来指定 id, 那么 form-datum 必须具有 (id . datum) 的形式。

如果提供了 #:kind kind-content-expr ,它的使用方式与 defproc 相同, 但默认的种类是 "syntax" 。

如果提供了 #:id [id id-expr] , 那么 id 就是出现在 form-datum 中的标识符(将被一个定义实例取代), 而 id-expr 产生要记录的标识符。 id 和 id-expr 角色之间的这种分割对于 defform 的功能抽象很有用。

除非 link-target?-expr 被指定并产生 #f, 否则 id (或 id-expr 的结果)是被索引的,它也被注册了, 所以该标识符的 racket-typeset 使用(具有相同的 for-label 绑定)被超链接到该文档。 defmodule 或 declare-exporting 的要求, 以及对 id (或 id-expr 的结果)的绑定要求,与 defproc 相同。

decode 后的 pre-flow 记录了该形式。 在这个描述中,通过 racket、 racketblock 等对 form-datum 中任何标识符的引用都被排版为一个子形式的非终端。 然而,如果提供了 #:literals 子句, literal-id 的实例被正常排版(即,由包围的上下文决定)。

如果提供了 #:grammar 子句,它包括一个以 id 形式显示的非术语的辅助语法。 每个 nonterm-id 都被指定为相应的 clause-datum 中的任何一个。

如果提供了 #:contracts 子句, 每个 subform-datum 通常是作为 form-datum 或 clause-datum 的元变量的标识符) 被显示为产生一个必须满足 contract-expr-datum 所描述的合约的值。 仅仅使用 #:contracts 来指定对表达式产生的 value 的约束; 对于 subform-datum 的 syntax 约束,使用语法符号来代替, 可能通过 #:grammar 指定的辅助语法。

form-datum, clause-datum, subform-datum 和 contract-expr-datum 的排版保留了源代码的布局,就像 racketblock 一样。

例子:
@defform[(sandwich-promise sandwich-expr)
         #:contracts ([sandwich-expr sandwich?])]{
  Returns a promise to construct a sandwich. When forced, the promise
  will produce the result of @racket[sandwich-expr].
}
 
@defform[#:literals (sandwich mixins)
         (sandwich-promise* [sandwich sandwich-expr]
                            [mixins ingredient-expr ...])
         #:contracts ([sandwich-expr sandwich?]
                      [ingredient-expr ingredient?])]{
  Returns a promise to construct a sandwich. When forced, the promise
  will produce the result of @racket[sandwich-expr]. Each result of
  the @racket[ingredient-expr]s will be mixed into the resulting
  sandwich.
}
 
@defform[(sandwich-factory maybe-name factory-component ...)
         #:grammar
         [(maybe-name (code:line)
                      name)
          (factory-component (code:line #:protein protein-expr)
                             [vegetable vegetable-expr])]]{
  Constructs a sandwich factory. If @racket[maybe-name] is provided,
  the factory will be named. Each of the @racket[factory-component]
  clauses adds an additional ingredient to the sandwich pipeline.
}

Renders like:

syntax

(specsubform maybe-literals datum maybe-grammar maybe-contracts
  pre-flow ...)
类似于带有 #:link-target? #f 的 defform ,但没有初始标识符作为隐式字面, 并且表格和流程被缩进排版。 这种形式是为了在完善 defform 或其他 specsubform 中使用的非终端的语法时使用。 例如,在 defproc 的文档中,它被用于 arg-spec 的可能形状的项目化。

pre-flow 列表被解析为记录该程序的流程。 在这个描述中,对 datum 中任何标识符的引用都被排版为子形式非终端。

syntax

(specspecsubform maybe-literals datum maybe-grammar maybe-contracts
  pre-flow ...)
就像 specsubform 一样,但是缩进了一个额外的层次。 由于在 specsubform 的正文中使用 specsubform 已经嵌套了缩进, 所以 specspecsubform 是用于额外的缩进而不嵌套描述。

syntax

(defthing options id contract-expr-datum maybe-value
  pre-flow ...)
 
options = maybe-kind maybe-link maybe-id
     
maybe-kind = 
  | #:kind kind-content-expr
     
maybe-link = 
  | #:link-target? link-target?-expr
     
maybe-id = 
  | #:id id-expr
     
maybe-value = 
  | #:value value-expr-datum
像 defproc 一样,但是对于一个非过程绑定。

如果提供了 #:kind kind-content-expr ,它的使用方式与 defproc 相同, 但默认的种类是 "value" 。

如果 #:id id-expr 被提供,那么 id-expr 的结果将被用来代替 id 。

如果给了 #:value value-expr-datum , value-expr-datum 将使用 racketblock0 进行排版,并包含在文档中。 广义的值会被放在一个单独的行中。

例子:
@defthing[moldy-sandwich sandwich?]{
  Don't eat this. Provided for backwards compatibility.
}
 
@defthing[empty-sandwich sandwich? #:value (make-sandwich empty)]{
  The empty sandwich.
}

Renders like:

value

value

empty-sandwich : sandwich? = (make-sandwich empty)

The empty sandwich.

syntax

(defthing* options ([id contract-expr-datum maybe-value] ...+)
  pre-flow ...)

例子:
@defthing*[([moldy-sandwich sandwich?]
            [empty-sandwich sandwich?])]{
  Predefined sandwiches.
}

Renders like:

value

moldy-sandwich : sandwich?

empty-sandwich : sandwich?
Predefined sandwiches.

syntax

(defstruct maybe-link struct-name ([field-name contract-expr-datum] ...)
  maybe-mutable maybe-non-opaque maybe-constructor
  pre-flow ...)
 
maybe-link = 
  | #:link-target? link-target?-expr
     
struct-name = id
  | (id super-id)
     
maybe-mutable = 
  | #:mutable

  | #:prefab
  | #:transparent
  | #:inspector #f

  | #:constructor-name constructor-id
  | #:extra-constructor-name constructor-id
  | #:omit-constructor
类似于 defform 或 defproc, 但用于结构定义。 defstruct* 形式对应于 struct, 而 defstruct 对应于 define-struct 。

例子:

An example using defstruct:
@defstruct[sandwich ([protein ingredient?] [sauce ingredient?])]{
  A structure type for sandwiches. Sandwiches are a pan-human foodstuff
  composed of a partially-enclosing bread material and various
  ingredients.
}

Renders like:

struct

(struct sandwich (protein sauce)
    #:extra-constructor-name make-sandwich)

syntax

(deftogether [def-expr ...+] pre-flow ...)

将由 def-expr 创建的定义合并到一个定义框中。 每个 def-expr 应该通过 defproc, defform 等产生一个定义点。 每个 def-expr 都应该有一个空的 pre-flow; deftogether 形式的 decode 的 pre-flow 序列记录了所收集的绑定。

  in writing test cases
}

syntax

(racketgrammar maybe-literals id clause-datum ...+)

 
maybe-literals = 
  | #:literals (literal-id ...)
创建一个表来定义 id 的语法。 在 clause-datum 中提到的每个标识符都被排版为非终端, 除了作为 literal-id 列出的标识符,它们被排版为 racket 。

syntax

(racketgrammar* maybe-literals [id clause-datum ...+] ...)

像 racketgrammar 一样,但用于一次排版多个产品,在 = 和 | 周围对齐。

procedure

(defidentifier id    
  [#:form? form?    
  #:index? index?    
  #:show-libs? show-libs?]) → element?
  id : identifier?
  form? : boolean? = #f
  index? : boolean? = #t
  show-libs? : boolean? = #t
将 id 作为 Racket 标识符进行排版,同时将标识符作为绑定的定义, 与 defproc, defform 等方式相同。 像往常一样,提供标识符的库必须通过 defmodule 或 declare-exporting 来声明一个封闭的部分。

如果 form? 是一个真值,那么标识符就被记录为一个语法形式, 所以标识符的使用(通常包括 id 本身)被排版为一个语法形式。

如果 index? 是一个真值,那么该标识符就被注册在索引中。

如果 show-libs? 是一个真值,那么标识符的定义模块可能会以排版的形式暴露出来 (例如,当查看 HTML 时,鼠标悬停在标识符上)。
