6.5 解码文本

 (require scribble/decode) package: scribble-lib
scribble/decode 库帮助你以自然的方式编写文档内容–更像纯文本,除了 @ 转义。 粗略地说,它处理字符串流以产生 scribble/struct 数据类型的实例(见 兼容性结构与处理) 。

在 flow 层面上,解码将空行识别为 paragraph 分隔符。 中间没有空行的区块和段落被收集成一个 compound paragraph 。

在 content 层面上,解码只做一些特殊的文本转换:

  • ---: 转换为 'mdash

  • --: 转换为 'ndash

  • ``: 转换为 'ldquo, 也就是花式的开放引号: “

  • '': 转换为 'rdquo, 也就是花哨的闭合引号: ”

  • ': 转换为 'rsquo, 也就是花式撇号: ’

  • `: 转换为 'lsquo, 也就花式引号: ‘

一些函数分别使用 pre-flow 或 pre-content 来 decode 一连串的 decode-flow 或 decode-content 参数。 例如, bold 函数接受任何数量的 pre-content 参数,因此在

  @bold{``apple''}

中， ``apple'' 参数被解码为使用花式引号,然后被加粗。

procedure

(pre-content? v) → boolean?

  v : any/c
如果 v 是一个 pre-content 值,则返回 #t : 一个字符串或其他非列表 content, 一个 pre-content 值的列表, 或一个包含 pre-content 值列表的 splice ;否则返回 #f 。

预内容通过 decode-content and decode-paragraph 等函数被解码成 content 。

procedure

(pre-flow? v) → boolean?

  v : any/c
如果 v 是一个 pre-flow 值:一个字符串或其他非列表 content, 一个 block, #<void>, 一个 pre-flow 值的列表, 或一个包含 pre-flow 值列表的 splice ,则返回 #t ;否则返回 #f 。

预流被解码成一个 flow (即一个 blocks 的列表),由类似 decode-flow 的函数来完成。

procedure

(pre-part? v) → boolean?

  v : any/c
如果 v 是一个 pre-part 值: 一个字符串或其他非列表 content, 一个 block, 一个 part, 一个 title-decl, 一个 part-start, 一个 part-index-decl, 一个 part-collect-decl, 一个 part-tag-decl, #<void>, 一个 pre-part 值的列表,或者一个包含 pre-part 值列表的 splice , 则返回 #t ; 否则返回 #f 。

通过 decode 和 decode-part 等函数将预部件序列解码成一个 part 。

procedure

(decode lst) → part?

  lst : (listof pre-part?)
解码一个文件,产生一个部分。 在 lst 中,列表和 splice 的实例被内联到列表中,并且 #<void> 被丢弃。 title-decl 的实例提供了该部分的标题,以及标签、样式和版本信息。 part-index-decl 的实例(在任何子部分之前)添加指向该部分的索引条目。 part-collect-decl 的实例为该部分添加元素,这些元素只在 collect pass 中使用。 part-tag-decl 的实例为部分标题添加超链接标签。 part-start 的实例在第 0 级触发子部分的处理。 section 触发器的实例被作为子部分使用, 而 paragraph 和其他流程元素数据类型的实例在包围的流程中被作为原样使用。

当一个部分被解码时,只要该部分的样式不包括 style property 'hidden 或 'no-index, 一个条目就会被添加到该部分的标题的文档索引中。

lst 在一个部分中的部分使用 decode-flow 流进行解码。

Changed in version 1.25 of package scribble-lib: 增加了 'no-index 支持。

procedure

(decode-part lst tags title depth) → part?

  lst : (listof pre-part?)
  tags : (listof string?)
  title : (or/c #f list?)
  depth : exact-nonnegative-integer?
与 decode 类似,但给定了该部分的标签字符串列表, 标题(如果 #f, 那么如果找到的话,就使用 title-decl 实例), 以及 part-start 的深度,以触发子部分的处理。

procedure

(decode-flow lst) → (listof block?)

  lst : (listof pre-flow?)
解码一个流程。 在 lst 中,列表和 splice 的实例被内联到列表中。 一个由两个或多个换行线组成的、仅由空白分隔的序列被解析为一个复合段落分隔符。

lst 中位于复合段落中的部分用 decode-compound-paragraph 进行解码。

procedure

(decode-compound-paragraph lst) → block?

  lst : (listof pre-flow?)
解码一个复合段落。在 lst 中,列表和 splice 的实例被内联到列表中。 paragraph 和其他 block 数据类型的实例在结果中按原样使用。 如果复合段落包含一个单独的块,那么该块将被返回,没有 compound-paragraph 的包装。

lst 中被 block 分隔的部分使用 decode-content 进行解码。

procedure

(decode-paragraph lst) → paragraph?

  lst : (listof pre-content?)
使用 decode-content 对一个段落进行解码,将 lst 解码为该段落的内容。

procedure

(decode-content lst) → list?

  lst : (listof pre-content?)
解码 content 。列表开头的元素如果是空白的(根据 whitespace?) 会被丢弃。 在嵌套的列表和拼接中丢弃空白是一个糟糕的实现选择,为了兼容而保留。 为了保护它,你可以利用同样不幸的事实,即一个空的列表不算是空白。 lst 中的列表和接片被平铺到列表中,同样地去掉前面的空白。 纯粹的字符串被 decoded; 非字符串、非列表的 content 被原封不动地包含在结果中。

procedure

(decode-elements lst) → list?

  lst : (listof pre-content?)
decode-content 的一个别名。

procedure

(decode-string s) → (listof content?)

  s : string?
对单个字符串进行 Decodes 以产生 content 。

procedure

(whitespace? v) → boolean?

  v : any/c
如果 v 是一个只包含空白的字符串,返回 #t ,否则返回 #f 。

struct

(struct title-decl (tag-prefix tags version style content)
    #:extra-constructor-name make-title-decl)
  tag-prefix : (or/c #f string?)
  tags : (listof string?)
  version : (or/c string? #f)
  style : style?
  content : content?
参见 decode 和 decode-part 。 tag-prefix 和 style 字段会被传播到产生的 part 。 如果 version 字段不是 #f, 它将被传播为该 part 的 document-version 样式属性。

struct

(struct part-start (depth tag-prefix tags style title)
    #:extra-constructor-name make-part-start)
  depth : integer?
  tag-prefix : (or/c #f string?)
  tags : (listof string?)
  style : style?
  title : content?
就像 title-decl 一样,但是对于一个子部分。参见 decode 和 decode-part 。

struct

(struct part-index-decl (plain-seq entry-seq)
    #:extra-constructor-name make-part-index-decl)
  plain-seq : (listof string?)
  entry-seq : list?
参见 decode 。 这两个字段和 index-element 一样。

struct

(struct part-collect-decl (element)
    #:extra-constructor-name make-part-collect-decl)
  element : (or/c element? part-relative-element?)
见 decode 。

struct

(struct part-tag-decl (tag)
    #:extra-constructor-name make-part-tag-decl)
  tag : tag?
见 decode 。

struct

(struct splice (run)
    #:extra-constructor-name make-splice)
  run : list?
见 decode, decode-part 和 decode-flow 。

procedure

(spliceof ctc) → flat-contract?

  ctc : flat-contract?
产生一个 splice 实例的合同,其 run 元素满足 ctc 。

procedure

(clean-up-index-string str) → string?

  str : string?
修剪前面和后面的空白,并将非空的空白序列转换为单个空格字符。
