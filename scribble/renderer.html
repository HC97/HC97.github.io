6.4 渲染器

渲染器是一个提供四个主要方法的对象: traverse, collect, resolve 和 render 。 每个方法都对应于 结构和处理 中描述的一个通道,它们被 render 函数串联起来,以渲染一个文档。

6.4.1 渲染驱动

 (require scribble/render) package: scribble-lib

procedure

(render 
  docs 
  names 
  [#:render-mixin render-mixin 
  #:dest-dir dest-dir 
  #:helper-file-prefix helper-file-prefix 
  #:keep-existing-helper-files? keep-existing-helper-files? 
  #:prefix-file prefix-file 
  #:style-file style-file 
  #:style-extra-files style-extra-files 
  #:extra-files extra-files 
  #:image-preferences image-preferences 
  #:xrefs xrefs 
  #:info-in-files info-in-files 
  #:info-out-file info-out-file 
  #:redirect redirect 
  #:redirect-main redirect-main 
  #:directory-depth directory-depth 
  #:quiet? quiet? 
  #:warn-undefined? warn-undefined?]) 
 → void?
  docs : (listof part?)
  names : (listof path-string?)
  render-mixin : (class? . -> . class?) = render-mixin
  dest-dir : (or/c #f path-string?) = #f
  helper-file-prefix : (or/c #f string?) = #f
  keep-existing-helper-files? : any/c = #f
  prefix-file : (or/c #f path-string?) = #f
  style-file : (or/c #f path-string?) = #f
  style-extra-files : (listof path-string?) = #f
  extra-files : (listof path-string?) = #f
  image-preferences : (listof (or/c 'ps 'pdf 'png 'svg 'gif))
   = null
  xrefs : (listof xref?) = null
  info-in-files : (listof path-string?) = null
  info-out-file : (or/c #f path-string?) = #f
  redirect : (or/c #f string?) = #f
  redirect-main : (or/c #f string?) = #f
  directory-depth : exact-nonnegative-integer? = 0
  quiet? : any/c = #t
  warn-undefined? : any/c = (not quiet?)
渲染给定的 docs ,每个文档都有一个从 names 的相应元素中得到的输出名称。 names 中一个名字的目录路径(如果有的话)被丢弃,文件的后缀被替换成(如果有的话)适合输出格式的后缀。

render-mixin 参数决定了输出格式。 默认情况下,它是来自 scribble/html-render 的 render-mixin 。

dest-dir 参数决定了输出目录,如果它是非 #f 且不存在, 则使用 make-directory* 创建。

helper-file-prefix, keep-existing-helper-files?, prefix-file, style-file, style-extra-files 和 extra-files 参数被传递给 render% 构造函数。

image-preferences 参数指定了图像文件和转换的首选格式,其中列表中较早列出的格式更受青睐。 render-mixin 所指定的渲染器可能不支持 image-preferences 中列出的所有格式。

xrefs 参数提供了额外的交叉引用信息,这些信息将在文档的 resolve pass 中使用。 info-in-files 参数以序列化的形式提供额外的交叉引用信息。 info-out-file 参数不是 #f 时,渲染后的文档的交叉引用信息会以序列化的形式写入指定的文件。

redirect 和 redirect-main 参数对应于 scribble/html-render 中 render-mixin 的 set-external-tag-path 和 set-external-root-url 方法, 所以它们应该只在 HTML 渲染中使用非 #f 。

directory-depth 参数对应于 render-multi-mixin 的 set-directory-depth 方法。

如果 quiet? 是一个假值,输出文件信息将被写入当前的输出端口。

如果 warn-undefined? 是一个真值,那么对缺失的交叉引用目标的引用会在当前错误端口上触发一个警告信息。

Changed in version 1.4 of package scribble-lib: 增加了 #:image-preferences 参数。
Changed in version 1.40: 增加了 --keep-existing-helper-files? 初始化参数,并修正了 --helper-file-prefix , 使之能正确用于 HTML 输出。

6.4.2 基础渲染器

 (require scribble/base-render) package: scribble-lib
scribble/base-render 模块提供了 render%, 它实现了一个渲染器的核心。 这个渲染类必须用 scribble/text-render, scribble/markdown-render 或 scribble/html-render 或 scribble/latex-render 中的一个混合素来完善。

mixin 结构是为了支持渲染器的特定文档扩展。 例如, scribble 命令行工具将来可能会从文档模块(除了文档本身之外)中提取渲染混合器。

请参阅 "base-render.rkt" 源文件,以了解更多关于渲染器方法的信息。 用更高层次构建的文档,例如 scribble/manual, 一般不会直接调用渲染对象的方法。

interface

render<%> : interface?

method

(send a-render traverse srcs dests) → (and/c hash? immutable?)

  srcs : (listof part?)
  dests : (listof path-string?)
执行 traverse pass, 产生一个哈希表, 其中包含 traverse-block 和 traverse-elements 的替换。 关于 dests 参数的信息请参见 render 。

method

(send a-render collect srcs dests fp [demand]) → collect-info?

  srcs : (listof part?)
  dests : (listof path-string?)
  fp : (and/c hash? immutable?)
  demand : (tag? collect-info? . -> . any/c)
   = (lambda (tag ci) #f)
执行 collect pass 。 关于 dests 参数的信息请参见 render 。 fp 参数是 traverse 方法的一个结果。

demand 参数根据需要提供外部标记映射。 当 collect-info 的结果后来被用来为一个标记寻找映射, 而没有映射可用时, demand 被调用,并带有标记和 collect-info 。 demand 函数返回 true,表示它向 collect-info 添加了信息,以便再次尝试查找; 如果不扩展 collect-info,则 demand 函数应返回 #f 。

method

(send a-render resolve srcs dests ci) → resolve-info?

  srcs : (listof part?)
  dests : (listof path-string?)
  ci : collect-info?
执行 resolve pass 。 关于 dests 参数的信息请参见 render 。 ci 参数是 collect 方法的结果。

method

(send a-render render srcs dests ri) → list?

  srcs : (listof part?)
  dests : (listof (or/c path-string? #f))
  ri : resolve-info?
生成最终输出。 ri 参数是 render 方法的一个结果。

dests 为 Latex 或单文件 HTML 输出提供文件名,或者为多文件 HTML 输出提供子目录名。 如果 dests 是相对的,它们是相对于当前目录的; 通常,它们应该表示在 render% 对象初始化时提供的 dest-dir 中的路径。

如果 dests 中的一个元素是 #f,那么结果列表中的相应位置就包含了一个渲染文档的字符串。 有些渲染器要求 dest 包含所有的路径字符串。

method

(send a-render serialize-info ri) → any/c

  ri : resolve-info?
将收集到的信息序列化到 ri 中。

method

(send a-render serialize-infos ri count doc) → list?

  ri : resolve-info?
  count : exact-positive-integer?
  doc : part?
和 serialize-info 类似, 但产生的 count 结果在一起的信息与 serialize-info 产生的信息相同。 doc 的结构被用来驱动分区(假设 ri 是由 doc 派生的)。

method

(send a-render deserialize-info v    
  ci    
  [#:root root-path]) → void?
  v : any/c
  ci : collect-info?
  root-path : (or/c path-string? false/c) = #f
将 v 的反序列化形式添加到 ci 中。

如果 root-path 不是 #f, 那么记录在 ci 中的相对于实例化提供的 root-path 的文件路径被反序列化为相对于给定的 root-path 。

method

(send a-render get-defined ci) → (listof tag?)

  ci : collect-info?
返回一个在 ci 所代表的文档中定义的标签列表。

method

(send a-render get-defineds ci count doc)

 → (listof (listof tag?))
  ci : collect-info?
  count : exact-positive-integer?
  doc : part?
类似于 serialize-infos: 为 get-defined 的结果的每个 count 分区返回一个标签列表, 使用 doc 的结构来驱动分区。

method

(send a-render get-external ri) → (listof tag?)

  ri : resolve-info?
返回在 ri 所代表的文档中被引用但未被定义的标签的列表 (尽管可能在通过 xref-transfer-info 转移到 ri 的交叉引用信息中发现)。

method

(send a-render get-undefined ri) → (listof tag?)

  ri : resolve-info?
返回一个被解析的文档所引用的标签的列表, 这些标签在 ri 所代表的解析的文档或通过 xref-transfer-info 转移到 ri 的交叉引用信息中都没有找到目标。

如果通过 resolve-search 引用了多个标签,并且使用相同的依赖键为其中任何一个标签找到了目标, 那么该集合中的任何标签都不会被包含在未定义标签的列表中。

(new render% 
    [dest-dir dest-dir]

    [root-path root-path] 
    [prefix-file prefix-file]

    [helper-file-prefix helper-file-prefix] 
    [keep-existing-helper-files? keep-existing-helper-files?] 
    [image-preferences image-preferences]]) 
 → (is-a?/c render%)
  dest-dir : path-string?
  refer-to-existing-files : any/c = #f
  root-path : (or/c path-string? #f) = #f
  prefix-file : (or/c path-string? #f) = #f
  style-file : (or/c path-string? #f) = #f
  style-extra-files : (listof path-string?) = null
  extra-files : (listof path-string?) = null
  helper-file-prefix : (or/c path-string? #f) = #f
  keep-existing-helper-files? : any/c = #f
  image-preferences : (listof (or/c 'ps 'pdf 'png 'svg 'gif))
   = null
创建一个渲染器,其输出将被送到 dest-dir 。 例如, dest-dir 可以命名为包含输出 Latex 文件的目录, 单文件输出的 HTML 文件,或者多文件 HTML 输出的输出子目录。

如果 refer-to-existing-files 为 true,那么当文档引用外部文件时, 例如图像或样式文件,那么该文件将从其源位置引用,而不是复制到文档目的地。

如果 root-path 不是 #f, 它通常与 dest-dir 或 dest-dir 的一个父级位置相同。 它导致交叉引用信息记录相对于 root-path 的目标文件; 当交叉引用信息被序列化时, 它可以通过 deserialize-info 以不同的根路径(表明目标文件已经移动)进行反序列化。

prefix-file, style-file 和 style-extra-files 参数设置了以特定形式控制输出样式的文件;更多信息见 配置输出 。

extra-files 参数命名了要被复制到输出位置的文件,比如图像文件或额外的配置文件。

helper-file-prefix 参数提供了一个前缀,用于主目标文件所使用的任何复制或生成的文件。 这个前缀不用于 extra-files 中列出的文件。 如果 keep-existing-helper-files? 为 true,那么任何现有的文件将被覆盖到一个辅助文件中, 而被保留下来,辅助文件被写入一个不同的名字,除非其内容与现有文件完全相同。

image-preferences 参数指定了图像文件和转换的首选格式,在列表中较早列出的格式更受青睐。 渲染器可能不支持 image-preferences 中列出的所有格式。

Changed in version 1.4 of package scribble-lib: 添加了 image-preferences 初始化参数。
Changed in version 1.40: 添加了 --keep-existing-helper-files? 初始化参数,并修正了 --helper-file-prefix , 以便在 HTML 输出中正确工作。

  • render-other 是由 render-content 的默认实现调用的, 适用于任何不满足 element? 或 convertible? 的内容。
