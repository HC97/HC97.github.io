3.5 SIGPLAN 论文格式

 #lang scribble/sigplan package: scribble-lib
scribble/sigplan 语言和 scribble/base 一样,但配置了 LaTeX 风格的默认值, 使用 Scribble 附带的 "sigplanconf.cls" 类文件。

syntax

preprint

启用 preprint 选项。只在 #lang 的同一行使用 preprint ， 在 scribble/sigplan 和 preprint 之间只有空白(或其他选项):

  #lang scribble/sigplan @preprint

syntax

10pt

启用 10pt 选项。只在 #lang 的同一行使用 10pt , 在 scribble/sigplan 和 10pt 之间只有空白(或其他选项):

  #lang scribble/sigplan @10pt

syntax

nocopyright

启用 nocopyright 选项。仅在与 #lang 相同的行中使用 nocopyright ， 在 scribble/sigplan 和 nocopyright 之间只有空白(或其他选项):

  #lang scribble/sigplan @nocopyright

syntax

onecolumn

启用 onecolumn 选项。仅在与 #lang 相同的行中使用, 在 scribble/sigplan 和 onecolumn 之间只有空白(或其他选项):

#lang scribble/sigplan @onecolumn

syntax

notimes

禁止在生成的 LaTeX 输出中使用 \usepackage{times} 。 只在 #lang 的同一行使用,在 scribble/sigplan 和 notimes 之间只有空白(或其他选项):

#lang scribble/sigplan @notimes

syntax

noqcourier

在生成的 LaTeX 输出中,禁止使用 \usepackage{qcourier} 。 仅在与 #lang 相同的行中使用, 在 scribble/sigplan 和 noqcourier 之间仅有空白(或其他选项):

#lang scribble/sigplan @noqcourier

10pt, preprint, nocopyright, onecolumn, notimes 和 noqcourier 选项可以一起使用,可以以任何顺序出现。

procedure

(abstract pre-content ...) → block?

  pre-content : pre-content?
为论文摘要生成一个 nested flow 。

syntax

(include-abstract module-path)

类似于 include-section, 但将文件纳入指定的模块作为一个摘要。 该文件必须没有标题或子部分。

procedure

(subtitle pre-content ...) → element?

  pre-content : pre-content?
作为 title 的最后一个参数,可以指定一个副标题。

procedure

(authorinfo name affiliation email) → block?

  affiliation : pre-content?
  email : pre-content?
替换 author ,将分支机构和电子邮件地址与作者姓名联系起来。

procedure

Added in version 1.13 of package scribble-lib.

procedure

(category CR-number    
  subcategory    
  third-level    
  [fourth-level]) → content?
  CR-number : pre-content?
  subcategory : pre-content?
  third-level : pre-content?
  fourth-level : (or/c #f pre-content?) = #f

procedure

(terms content ...) → content?

  content : pre-content?

procedure

(keywords content ...) → content?

  content : pre-content?
对论文的类别、术语和关键词信息进行排版,通常放在 abstract 表格之后。 另见 http://www.acm.org/about/class/how-to-use 。

对于 category, subcategory 参数应使用标题大写(即每个词的第一个字母大写), 并且是 "编程语言 "或 "软件工程 "级别的短语(而不是 "软件 "这样的类别或 "并发编程 "或 "处理器 "这样的第三级名称)。 third-level 短语应使用大写字母。 fourth-level 短语,如果有的话,不应该被大写。

对于 terms, 每个一般的术语都应使用大写字母。 术语通常是从一个固定的列表中抽取的,而且它们通常是可选的。

对于 keywords, 只需将第一个词的第一个字母大写,用逗号分隔短语,不要在最后一个词前加入 "和"。 关键词应该是名词短语,而不是形容词。
