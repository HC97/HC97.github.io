6.11.5 基本 Latex 宏

"scribble.tex" Latex 配置包括几个宏和环境,你可以重新定义以调整输出风格:

  • \preDoc — 在文档内容之前调用;默认情况下不做任何事情, 而 scribble/manual 配置启用了 \sloppy 。

  • \postDoc — 在文档内容之后调用;默认情况下不做任何事情。

  • 一组控制加载Latex包的基本集合的命令:

    • \packageGraphicx, 默认为 \usepackage{graphicx}

    • \packageHyperref, 默认为 \usepackage{hyperref}

    • \renewrmdefault, 默认为 \renewcommand{\rmdefault}{ptm}

    • \packageRelsize, 默认为 \usepackage{relsize}

    • \packageAmsmath, 默认为 \usepackage{amsmath}

    • \packageMathabx, 默认为 \usepackage{mathabx}

    • \packageWasysym, 默认为 \let\leftmoon\relax \let\rightmoon\relax \let\fullmoon\relax \let\newmoon\relax \let\diameter\relax \usepackage[nointegrals]{wasysym}

    • \packageTxfonts, 默认为 \let\widering\relax \usepackage{newtxmath}

    • \packageTextcomp, 默认为 \usepackage{textcomp}

    • \packageFramed, 默认为 \usepackage{framed}

    • \packageHyphenat, 默认为 \usepackage[htt]{hyphenat}

    • \packageColor, 默认为 \usepackage[usenames,dvipsnames]{color}

    • \doHypersetup, 默认为 \hypersetup{bookmarks=true,bookmarksopen=true,bookmarksnumbered=true}

    • \packageTocstyle, 默认为 \IfFileExists{tocstyle.sty}{\usepackage{tocstyle}\usetocstyle{standard}}{}

    • \packageCJK, 默认为 \IfFileExists{CJK.sty}{\usepackage{CJK}}{}

    Changed in version 1.36: 添加了 \packageTxfonts

    Changed in version 1.37: 添加了 \packageAmsmath; 改变了 \packageWasysym 以使用 nointegrals 选项; 改变了 \packageTxfonts 以加载 newtxmath 。 注意,由于新的 wasysym 选项和 newtxmath 的加入,文件可能看起来有所不同。 参见 racket/scribble#274 的例子。

  • \sectionNewpage — 在每个顶层章节开始之前调用;默认情况下不做任何事情, 而 scribble/manual 配置使用 \newpage 在新的一页上开始每一章。

  • \SecRefLocal{}{}{} — 第一个参数是 Latex 标签,第二个参数是章节编号,第三个参数是章节标题。 这个宏被 secref 用来引用一个有编号的章节(不是文档或文档中的顶级章节),而且是当前文档的本地章节。 默认扩展为 \SecRef ,只传递第二个和第三个参数(因此标签被忽略)。

  • \SecRef{}{} — 像 \SecRefLocal 一样, 但是当被引用的部分在不同的文档中时使用,这样就没有标签了。 默认显示 "section",后面是章节编号(忽略标题)。 这个宏的 scribble/manual 重新定义显示 "§"、节号和引号中的标题。

  • \ChapRefLocal{}{}{} 和 \ChapRef{}{} — 像 \SecRefLocal 和 \SecRef 类似,但用于文档中的顶级章节。 默认的实现方式是遵从 \SecRefLocal 或 \SecRef 。

  • \PartRefLocal{}{}{} 和 \PartRef{}{} — 像 \SecRefLocal 和 \SecRef 一样,但是对于文档中的顶级部分, 其部分具有 'grouper 风格属性。 默认的 \PartRef 显示 "part",后面是章节编号(忽略标题)。

  • \BookRefLocal{}{}{} 和 \BookRef{}{} — 像 \SecRefLocal and \SecRef 一样, 但是对于一个文档(而不是文档中的章节)。 默认的 \BookRef 实现以斜体显示标题。

  • \SecRefLocalUC{}{}{} 和 \SecRefUC{}{} — 像 \SecRefLocal 和 \SecRef 一样, 但适用于 Secref 。 默认的 \SecRefUC 显示 "Section",后面是章节编号。

  • \ChapRefLocalUC{}{}{} 和 \ChapRefUC{}{} — 像 \ChapRefLocal 和 \ChapRef 一样,但适用于 Secref 。 默认的 \ChapRefUCimplementation 会延迟到 \SecRefUC 。

  • \PartRefLocalUC{}{}{} 和 \PartRefUC{}{} — 像 \PartRefLocal 和 \PartRef 一样, 但适用于 Secref 。 默认的 \PartRefUC 显示 "Part",后面是章节编号。

  • \BookRefLocalUC{}{}{} 和 \BookRefUC{}{} — 像 \BookRefLocal 主 \BookRef 一样, 但适用于 Secref. 。 默认的 \BookRefUC 服从于 \BookRef 。

  • \SecRefLocalUN{}{}, \SecRefUCUN{}, \SecRefLocalUCUN{}{}, \SecRefUN{}, \PartRefLocalUN{}{}, \PartRefUN{}, \PartRefLocalUCUN{}{}, \PartRefUCUN{}, \BookRefLocalUN{}{}, \BookRefUN{}, \BookRefLocalUCUN{}{}, \BookRefUCUN{}, \ChapRefLocalUN{}{}, \ChapRefUN{}, \ChapRefLocalUCUN{}{} 和 \ChapRefUCUN{} — 像 \SecRefLocal, 等等。 , 但在节/部/章的编号不可用的情况下。 默认实现的 \BookRefUN 使用 \BookRef ,第一个参数为空。 默认的 \SecRefLocalUN 用引号扩展到它的第二个参数, 然后用第一个参数的 "on page" 作为一个 \pageref ,而默认的 \SecRefUN 用引号扩展到它唯一的参数。 默认的 \PartRef 和 \ChapRef 变体扩展到相应的 \SecRef 变体。

  • \Ssection{}{}, \Ssubsection{}{}, \Ssubsubsection{}{}, \Ssubsubsubsection{}{}, \Ssubsubsubsubsection{}{} — 用于顶级章节、二级章节等,其中最后一个变体用于所有深于四级的章节。 第一个参数对应的是 \section 的可选参数,用于目录。

  • \Ssectionstar{}, \Ssubsectionstar{}, \Ssubsubsectionstar{}, \Ssubsubsubsectionstar{}, \Ssubsubsubsubsectionstar{} — 像 \Ssection, 等,但用于目录中省略的未编号的章节。

  • \Ssectionstarx{}{}, \Ssubsectionstarx{}, \Ssubsubsectionstarx{}{}, \Ssubsubsubsectionstarx{}{}, \Ssubsubsubsubsectionstarx{}{} — 像 \Ssection, 等,但用于未编号的章节(但仍出现在目录中)。

  • \Sincsection, \Sincsubsection, \Sincsubsubsection, \Sincsubsubsubsection, \Sincsubsubsubsubsection — 增加章节计数器。

  • \Spart{}{}, \Spartstar{}, \Spartstarx{}{}, \Sincpart — 像 section , 但用于代替 \Ssection{}{}, \Ssectionstar{} 等, 用于具有 'grouper 风格属性的部分。

  • SInsetFlow 环境 — 用于具有 'inset 样式名称的 nested-flow 。

  • SCodeFlow 环境 — 用于具有 'code-inset 样式名称的 nested-flow 。

  • SVInsetFlow 环境 — 用于具有 'vertical-inset 样式名称的 nested-flow 。

  • SVerbatim 环境 — 用于通过 verbatim 创建的 table 。

  • \SCodeBox{}, \SVInsetBox{} — 分别用于具有 'code-inset 或 'vertical-inset 样式名称的 nested-flow ,并作为表格单元的内容。 使用 \setbox1 将内容安装到一个 TeX 框中。

此外, "racket.tex" Latex 配置包括几个宏,你可以重新定义,以调整 Racket 代码的输出样式:

  • \SColorize{}{} — 设置 Racket 代码的颜色方案。 可以重新定义以创建黑白代码。 第一个参数是要求的颜色,第二个参数是该颜色的文本。

  • \SHyphen{} — 启用或禁用 Racket 代码中标识符和关键词的连字符功能。 默认为启用(为了兼容)。重新定义以禁用或改变连字符的行为。 例如,为了使文本溢出而不是连字符,可以重新定义为: \renewcommand{\SHyphen}[1]{\mbox{#1}} 。第一个参数是一个标识符或代码块内的关键词。
