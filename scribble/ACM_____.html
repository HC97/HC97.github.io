3.6 ACM 论文格式

 #lang scribble/acmart package: scribble-lib
scribble/acmart 语言和 scribble/base 一样,但配置了 LaTeX 风格的默认值, 使用 acmart 类为计算机协会的出版物进行排版。

注意: 一个 scribble/acmart 文件必须包括 title 和 author 。

Example:

  #lang scribble/acmart

  @title{Surreal Numbers}

  @author{Ursula N. Owens}

  #lang scribble/acmart @acmsmall

启用给定的文档格式选项。只能在 #lang 的同一行使用该选项,在 scribble/acmart 和格式选项之间只有空格(或其他选项)。 可以使用任何数量的选项:

  #lang scribble/acmart @acmsmall @review @anonymous @natbib

如果使用了多个字体大小选项,除了最后一个,其他的都会被忽略。

ACM documentation (1.54版,2018-07-16,作者 Boris Veytsman)提供了这些默认值和描述:

为了禁用一个默认为真的选项(例如 natbib), 请将该选项作为一个函数调用,其值为 #false:

#lang scribble/acmart @natbib[#f] @sigplan

@author["David Van Horn"
        #:affiliation @affiliation[
                       #:institution
                       @institution[
                        #:departments (list @institution{Department of Computer Science}

                         University of Maryland}

                       #:state "Maryland"]
        #:email "dvanhorn@cs.umd.edu"]}
 
@abstract{This is an abstract.}

procedure

(authorsaddresses addresses ...) → block?

  addresses : pre-content?
设置某些样式中第一页的作者地址的文本。 默认情况下,这个字段被设置为作者和他们的分支机构信息。

addresses 参数采用地址文本。作为一种特殊情况,空列表可以完全删除地址栏。

#lang scribble/acmart @acmsmall
@title{A fancy paper}
@author["Ronon Dex"]
@authorsaddresses{}

procedure

(keywords content ...) → content?

  content : pre-content?
为论文的术语和关键词信息排版,通常放在摘要表格之后。 参见 http://www.acm.org/about/class/how-to-use 。

对于 terms, 每个一般的术语都应使用标题大写。 术语通常从一个固定的列表中抽取,而且通常是可选的。

对于 keywords, 只需将第一个词的第一个字母大写,用逗号分隔短语,不要在最后一个词前加入 "和"。 关键词应该是名词短语,而不是形容词。

procedure

(received [#:stage stage] date) → content?

  stage : string? = #f

设置出版物的历史。 如果省略了 stage ,第一次出现时默认为 "Received" ,以后使用时默认为 "revised" 。

@received{February 2007}
@received[#:stage "revised"]{March 2009}
@received[#:stage "accepted"]{June 2009}

(grantnum [#:url url] sponsorID num) → content?

  url : string? = #f

所有的财政支持都 must 使用 acks 中的 grantsponsor 和 grantnum 命令列出。

这里的 sponsorID 是用来匹配资助者的唯一ID, name 是资助者的名字。 grantnum 的 sponsorID 必须与 grantsponsor 命令的某个 sponsorID 相匹配。
