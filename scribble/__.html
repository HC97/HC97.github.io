4.2.9 书目

参见 scriblib/autobib 。

procedure

(cite key ...+) → element?

  key : string?
链接到一个书目条目,使用 key 来表示书目条目,并在方括号中作为链接文本。

procedure

(bibliography [#:tag tag] entry ...) → part?

  tag : string? = "doc-bibliography"
  entry : bib-entry?
创建一个包含给定条目的书目部分,每个条目都是用 bib-entry 创建的。条目按给定顺序排版。

procedure

(bib-entry #:key key    
  #:title title    
  [#:is-book? is-book?    
  #:author author    
  #:location location    
  #:date date    
  #:url url    
  #:note note]) → bib-entry?
  key : string?
  title : (or/c #f pre-content?)
  is-book? : boolean? = #f
  author : (or/c #f pre-content?) = #f
  location : (or/c #f pre-content?) = #f
  date : (or/c #f pre-content?) = #f
  url : (or/c #f pre-content?) = #f
  note : (or/c #f pre-content?) = #f
创建一个书目条目。 key 值用于通过 cite 来引用该条目。其他参数作为条目中的元素使用:

  • title 是被引用作品的标题。 如果 is-book? 是 #f 它将被引号包围,否则它将以 italic 形式排版。

  • author 列出了作者。按照通常的顺序使用名字(而不是 "最后一个,第一个"), 并在最后一个名字前用逗号分隔多个名字(如果有多个名字)。 author 在书目中按规定排版,如果是 #f ,则省略。

  • location 命名出版地点,如会议名称或带有卷号和页码的期刊。 location 在书目中按给定的方式排版,如果给定的是 #f ,则省略不计。

  • date 是一个日期,通常只是一个年份(作为一个字符串)。 它在书目中按给定的方式排版,或者如果给定的是 #f ,则省略不计。

  • url 是一个可选的 URL。 它在书目中用 tt 排版,并有超链接,如果给的是 #f ,则省略不计。

  • note 是一个关于作品的可选评论。 它在书目中被排版,并直接出现在日期(或 URL,如果有的话)之后,中间没有空格或标点符号。

Changed in version 1.29 of package scribble-lib: 增加了 #:note 选项。

procedure

(bib-entry? v) → boolean?

  v : any/c
如果 v 是由 bib-entry 创建的书目条目,返回 #t ,否则返回 #f 。
