4.2.8 索引

另请参阅 索引 了解 scribble/base 。

syntax

(indexed-racket datum ...)

racket 和 as-index 的组合,当提供单一的 datum 时,有以下的特殊情况:

  • 如果 datum 是一个 quote 形式, 那么引号将从键中移除(这样就可以用它的非引号形式进行排序)。

  • 如果 datum 是一个字符串,那么引号将从键中移除(这样它就会使用字符串内容进行排序)。

syntax

(indexed-scheme datum ...)

indexed-racket 的兼容别名。

procedure

(idefterm pre-content ...) → element?

  pre-content : pre-content?
结合了 as-index 和 defterm 。内容通常应该是复数,而不是单数。 可以考虑使用 deftech 来代替,它总是索引。

procedure

(pidefterm pre-content ...) → element?

  pre-content : pre-content?
像 idefterm 一样,但是是复数:在内容的末尾添加一个 "s" 作为索引条目。 可以考虑使用 deftech 来代替。

procedure

(indexed-file pre-content ...) → element?

  pre-content : pre-content?
file 和 as-index 的组合,但 iterm 索引的排序键不包括引号。

procedure

(indexed-envvar pre-content ...) → element?

  pre-content : pre-content?
envvar 和 as-index 的组合。
