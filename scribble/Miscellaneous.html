4.2.11 Miscellaneous

procedure

(t pre-content ...) → paragraph?

  pre-content : pre-content?
将 decode 后的 pre-content 包装成一个段落。

value

etc : element?

像 "etc.", 但有一个缩写的句号,用于句子的中间。

value

PLaneT : element?

"PLaneT" (以帮助确保你的字母大小写正确)。

value

manual-doc-style : style?

一个用于文档的主要 part 的样式, 以获得 #lang scribble/manual 的样式配置。 参见 手动渲染风格 。

procedure

(hash-lang) → element?

返回 #lang 的一个元素,该元素被超链接到一个解释。

value

void-const : element?

为 #<void> 返回一个元素。

value

undefined-const : element?

为 #<undefined> 返回一个元素。

procedure

(commandline content ...) → paragraph?

  content : content?
产生一个嵌入的命令行例子(例如,用打字机字体)。

procedure

(inset-flow pre-flow ...) → nested-flow?

  pre-flow : pre-flow?
创建一个 nested-flow ,在左边和右边缩进。

使用 'inset 风格的 nested 是一个更好的选择。

procedure

(centerline pre-flow ...) → nested-flow?

  pre-flow : pre-flow?
centered 的一个别名,用于向后兼容。

procedure

(math pre-content ...) → element?

  pre-content : any/c
decode 后的 pre-content 被进一步转换:

  • 任何直接的 'rsquo 都被转换为 'prime 。

  • 括号和直接字符串中的小数点序列保持原样,但任何其他直接字符串都被斜体化。

  • 当 _ 出现在一个非空的数字和字母序列前时,该序列被排成下标。

  • 当 ^ 出现在一个非空的数字和字母序列前时,该序列被排版为上标。

procedure

(filebox filename pre-flow ...) → block?

  filename : (or/c string? element?)
  pre-flow : pre-flow?
将 pre-flow 排版为 filename 的内容。 例如,内容可以在页面上嵌入, filename 在其上方。 如果 filename 是一个字符串,它被传递到 filepath 以获得一个 element 。

procedure

(deprecated [#:what what]    
  replacement    
  additional-notes ...) → block?
  what : content? = "library"
  replacement : content?
  additional-notes : content?
对于已废弃的库、函数等(如 what 所描述的)产生一个嵌套警告, 其中 replacement 描述了一个合适的替换。 additional-notes 包含在最初的弃用信息之后。

procedure

(image/plain filename-relative-to-source    
  pre-element ...) → element?
  filename-relative-to-source : string?
  pre-element : any/c
image 的一个别名,用于向后兼容。
