4.2.4 文档类和接口

syntax

(defclass maybe-link id super (intf ...) pre-flow ...)

 
maybe-link = 
  | #:link-target? link-target?-expr
     
super = super-id
  | (mixin-id super)
     
intf = intf-id
  | [#:no-inherit intf-id]
为一个作为 super 的子类并实现每个接口 intf-id 的类 id 创建文档。 super 中的每个标识符(除了 object%) 和 intf-id 都必须通过 defclass 或 definterface 在某个地方进行记录。

pre-flow 的解码应该从关于类的一般文档开始,接着是构造函数定义(见 defconstructor), 然后是字段和方法定义(见 defmethod)。 在渲染的形式中,构造函数和方法规范被缩进,以便在视觉上将它们归入类定义之下。

当 intf-id 被指定为 #:no-inherit 时, 那么 id 的继承方法集就不包括 intf-id 的方法。 当不需要直接引用 intf-id 的方法时,以这种方式省略方法可以避免对文档的依赖。

Changed in version 1.42 of package scribble-lib: 为 intf 添加了 #:no-inherit 。

syntax

(defclass/title maybe-link id super (intf ...) pre-flow ...)

和 defclass 一样,也包含了一个样式为 'hidden 的 title 声明。 此外,构造函数和方法都没有左旋。

这种形式通常用于创建一个要在自己的 HTML 上渲染的部分。 使用 'hidden 样式是因为定义框作为一个标题。

Changed in version 1.42 of package scribble-lib: 为 intf 添加了 #:no-inherit 。

syntax

(definterface id (intf ...) pre-flow ...)

就像 defclass 一样,但对接口而言。 当然, pre-flow 不应生成构造函数声明。

Changed in version 1.42 of package scribble-lib: 为 intf 添加了 #:no-inherit 。

syntax

(definterface/title id (intf ...) pre-flow ...)

像 definterface 一样,但用于单页渲染,如 defclass/title 。

Changed in version 1.42 of package scribble-lib: 为 intf 添加了 #:no-inherit 。

syntax

(defmixin id (domain-id ...) (range-id ...) pre-flow ...)

和 defclass 一样,但适用于 mixin。 任何数量的 domain-id 类和接口都被指定为 mixin 的输入要求, 而任何数量的 result 类和(更可能的)接口都被指定为 range-id 。 domain-id 提供了继承的方法,它们可以包括用 #:no-inherit 注释的接口。

Changed in version 1.42 of package scribble-lib: 为 domain-id 添加了 #:no-inherit 支持。

syntax

(defmixin/title id (domain-id ...) (range-id ...) pre-flow ...)

像 defmixin 一样,但用于单页渲染,如 defclass/title 。

syntax

(defconstructor (arg-spec ...) pre-flow ...)

 
arg-spec = (arg-id contract-expr-datum)
  | (arg-id contract-expr-datum default-expr)
像 defproc 一样,但对于 defclass 主体中的构造函数声明,所以没有指定返回契约。 另外,每个 arg-spec 的 new-style 关键字是由 arg-id 隐含的。

syntax

(defconstructor/make (arg-spec ...) pre-flow ...)

像 defconstructor 一样,但指定了按位置的初始化参数 (用于 make-object )而不是按名字的参数(用于 new)。

syntax

(defconstructor*/make [(arg-spec ...) ...] pre-flow ...)

像 defconstructor/make,但有多个构造函数模式,类似 defproc* 。

syntax

(defconstructor/auto-super [(arg-spec ...) ...] pre-flow ...)

像 defconstructor 一样,但构造函数被注解以表明接受额外的初始化参数并传播到超类。

syntax

(defmethod maybe-mode maybe-link (id arg-spec ...)
           result-contract-expr-datum
           pre-flow ...)
 
maybe-link = 
  | #:link-target? link-target?-expr
     
maybe-mode = 
  | #:mode public
  | #:mode public-final
  | #:mode override
  | #:mode override-final
  | #:mode augment
  | #:mode augment-final
  | #:mode extend
  | #:mode extend-final
  | #:mode pubment
类似于 defproc, 但是对于 defclass 或 definterface 主体中的方法。

maybe-mode 指定了该方法是否覆盖了来自超类的方法,等等。 (为了这些目的,在完善一个实现的接口的方法时,使用 #:mode override )。 extend 模式与 override 类似,但方法的描述应该只描述对超类实现的扩展。 当 maybe-mode 没有被提供时,它默认为 public 。

Changed in version 1.35 of package scribble-lib: 增加了对无效 maybe-mode 的检查。

syntax

(defmethod* maybe-mode maybe-link
            ([(id arg-spec ...)
              result-contract-expr-datum] ...)
            pre-flow ...)
像 defproc* 一样,但对于 defclass 或 definterface 主体中的方法。 maybe-mode 规范与 defmethod 相同。

syntax

(method class/intf-id method-id)

在 class/intf-id 命名的类或接口中,创建一个以 method-id 命名的方法的超链接。 该超链接仅命名该方法,另见 xmethod 。

For-label 绑定信息与 class/intf-id 一起使用,但不包括 method-id 。

syntax

(xmethod class/intf-id method-id)

与 method 类似,但超链接同时显示方法名和包含的类/接口。

syntax

(this-obj)

在 defmethod 或类似的表单中,排版为一个元变量,代表方法调用的目标。 使用 (this-obj) 比 "这个方法的对象" 这样的散文更精确。
