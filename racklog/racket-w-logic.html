3 在 Racklog 中使用传统的 Racket 表达式

Racklog 谓词的参数可以是任何 Racket 对象。 特别是,可以使用复合结构,如列表、向量、字符串、哈希表等, 也可以使用 Racket 构造和分解操作符的全部阵列的 Racket 表达。例如,考虑下面这个目标:

(%member x '(1 2 3))

这里, %member 是一个谓词, x 是一个逻辑变量, '(1 2 3) 是一个结构。 给予 %member 一个适当的直观定义, 对于 x = 1, 2 和 3 ,上述目标就成功了。

现在来定义像 %member 这样的谓词:

(define %member
  (%rel (x y xs)
    [(x (cons x xs))]
    [(x (cons y xs))
      (%member x xs)]))

也就是说, %member 是用三个局部变量定义的: x, y, xs 。 它有两个子句,确定确定成员资格的两种方式。

%member 的第一个子句说明了一个事实: 对于任何一个 x, x 是一个列表的成员,这个列表的头也是 x 。

%member 的第二个子句是一条规则: 如果我们能证明 x 是一个列表的 tail 成员,那么它就是该列表的成员。 换句话说,原来的 %member 目标被转化为一个 子 目标,它也是一个 %member 目标。

请注意,%member 定义中的变量 y 在第二个子句中只出现一次。 因此,它不需要你努力去命名它。 (Racklog 允许你使用表达式 (_) 来表示一个匿名变量。 (也就是说, _ 是一个 thunk,每次调用都会产生一个新的匿名变量)。 谓词 %member 可以被改写为

(define %member
  (%rel (x xs)
    [(x (cons x (_)))]
    [(x (cons (_) xs))
      (%member x xs)]))

3.1 构造函数

我们可以使用构造函数–用于创建结构的 Racket 程序–来模拟 Racklog 中的数据类型。 例如,让我们定义一个自然数数据类型,其中 0 表示零, (succ x) 表示自然数的直接前身是 x 。 构造函数 succ 可以在 Racket 中定义为:

(define succ
  (lambda (x)
    (vector 'succ x)))

加法和乘法可以定义为:

(define %add
  (%rel (x y z)
    [(0 y y)]
    [((succ x) y (succ z))
      (%add x y z)]))
 
(define %times
  (%rel (x y z z1)
    [(0 y 0)]
    [((succ x) y z)
     (%times x y z1)
     (%add y z1 z)]))

我们可以用这个来做很多算术。 例如,阶乘谓词看起来像:

(define %factorial
  (%rel (x y y1)
    [(0 (succ 0))]
    [((succ x) y)
      (%factorial x y1)
      (%times (succ x) y1 y)]))

3.2 %is

以上是一种非常低效的算术方式,尤其是当底层语言 Racket 提供了优秀的算术设施(包括全面的数字 "塔" 和精确的有理算术)。 在 Racklog 子句中直接使用 Racket 计算的一个问题是,所使用的表达式可能包含需要被解除引用的逻辑变量。 Racklog 提供的谓词 %is 可以解决这个问题。该目标

(%is X E)

将 X 与作为 Racket 表达式的 E 的值统一起来。 E 可以有逻辑变量,但通常它们至少应该是绑定的, 因为对于 E 中使用的 Racket 操作符来说,未绑定的变量可能是不受欢迎的值。

现在我们可以直接使用 Racket 的数字来写一个更有效的 %factorial 谓词:

(define %factorial
  (%rel (x y x1 y1)
    [(0 1)]
    [(x y) (%is x1 (- x 1))
           (%factorial x1 y1)
           (%is y (* y1 x))]))

这种效率的代价是,我们只能在其第一个参数已经实例化的情况下使用 %factorial 。 在许多情况下,这并不是一个不合理的限制。 事实上,鉴于这种限制,没有什么可以阻止我们直接使用 Racket 的阶乘:

(define %factorial
  (%rel (x y)
    [(x y)
     (%is y (racket-factorial
              x))]))

或者更好的是,用调用 racket-factorial 的 %is-expressions "内联" 任何对 %factorial 的调用,其中后者是以通常的方式定义的:

(define racket-factorial
  (lambda (n)
    (if (= n 0) 1
        (* n (racket-factorial
               (- n 1))))))

3.3 词汇范围

我们可以使用 Racket 的词法范围来加强谓词定义。 下面是一个使用隐藏辅助谓词定义的列表反转谓词:

(define %reverse
  (letrec
    ([revaux
       (%rel (x y z w)
         [('() y y)]
         [((cons x y) z w)
           (revaux y
             (cons x z) w)])])
    (%rel (x y)
      [(x y) (revaux x '() y)])))

(revaux X Y Z) 使用 Y 作为累加器, 将 X 反转为 Z (Y 开始是 () 。 X 的每个头都被 cons 在 Y 上。 最后,当 X 缠绕到 () 时, Y 包含反转的列表,可以作为 Z 返回)。

revaux 纯粹是作为 %reverse 的辅助谓词使用的,因此它可以被隐藏在一个词组轮廓中。 我们使用 letrec 而不是 let ,因为 revaux 是一个递归过程。

3.4 类型谓词

Racklog 提供了几个谓词,让用户可以探测对象的类型

目标

(%constant X)

如果 X 是一个 atomic 的对象,则成功。

谓词 %compound, 即 %constant 的否定,检查其参数是否不是一个原子对象。

以上只是对应 Racket 谓词的逻辑编程等价物。 用户可以使用谓词 %is 和 Racket 谓词,在 Racklog 中编写更多的类型检查。 因此,为了测试 X 是否是一个字符串,可以使用以下目标:

(%is #t (string? X))

用户定义的 Racket 谓词,除了原始的 Racket 谓词外,还可以这样导入。
