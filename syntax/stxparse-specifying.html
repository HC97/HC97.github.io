1.4 用句法类指定句法

语法类为 syntax patterns 提供了一种抽象机制。 提供的内置语法类可以识别基本类,如 identifier 和 keyword 。 程序员可以对基本的语法类进行组合,以建立更复杂的语法规范,如不同标识符的列表和带有关键字的正式参数。 操作相同语法结构的宏可以共享语法类的定义。

syntax

(define-syntax-class name-id stxclass-option ...
  stxclass-variant ...+)
(define-syntax-class (name-id . kw-formals) stxclass-option ...
  stxclass-variant ...+)
 
stxclass-option = #:attributes (attr-arity-decl ...)
  | #:auto-nested-attributes
  | #:description description-expr
  | #:opaque
  | #:commit
  | #:no-delimit-cut
  | #:literals (literal-entry ...)
  | #:datum-literals (datum-literal-entry ...)
  | #:literal-sets (literal-set ...)
  | #:conventions (convention-id ...)
  | #:local-conventions (convention-rule ...)
  | #:disable-colon-notation
     
attr-arity-decl = attr-name-id
  | (attr-name-id depth)
     
stxclass-variant = (pattern syntax-pattern pattern-directive ...)
 
  description-expr : (or/c string? #f)
将 name-id 定义为一个 syntax class, 它封装了一个或多个 single-term patterns 。

语法类可以有形式参数,在这种情况下,它们在主体中被绑定为变量。 语法类支持可选的参数和关键字参数,使用与 lambda 相同的语法。 语法类定义的主体包含一个非空的 pattern 变体序列。

支持以下选项:

#:attributes (attr-arity-decl ...)
 
attr-arity-decl = attr-id
  | (attr-id depth)

声明语法类的属性。 一个属性 arity 声明由属性名称和可选的省略号深度组成(如果没有明确指定,则为零)。

如果属性没有明确列出,它们将被推断为在语法类的每个变体中出现的所有 pattern variables 的集合。 出现在不同省略号深度的模式变量不包括在内,也不包括 annotated pattern variables 的嵌套属性。

#:auto-nested-attributes

已被弃用。 这个选项不能与 #:attributes 结合使用。

将语法类的属性声明为所有 pattern variables 的集合, 以及在该语法类的每个变体中出现的 annotated pattern variables 的嵌套属性。 只有严格定义在包围式语法类之前的语法类才会被用来计算嵌套属性; 用尚未定义的语法类注释的模式变量不会为输出提供嵌套属性。 请注意,使用这个选项,重新排列语法类的定义可能会改变它们输出的属性。

#:description description-expr
 
  description-expr : (or/c string? #f)

description 参数在一个包含语法类参数的范围内被评估。 如果结果是一个字符串,它将用于涉及语法类的错误信息中。 例如,如果一个术语被语法类拒绝, 可以合成一个形式为 "expected description" 的错误。 如果结果是 #f, 那么在搜索要报告的描述时就会跳过该语法类。

如果没有给出该选项,则使用语法类的名称来代替。

#:opaque

表示不应报告与语法类内部结构有关的错误。

#:commit

指示语法类 "提交 "给第一个成功的匹配。 当一个变体成功后,语法类中的所有选择点都被丢弃。另请参见 ~commit 。

#:no-delimit-cut

默认情况下,在一个语法类中的剪切 (~!) 只丢弃该语法类中的选择点。 也就是说,语法类的主体就像被包裹在一个 ~delimit-cut 形式中一样。 如果指定了 #:no-delimit-cut , 那么切割可能会影响到该语法类的调用环境(另一个语法类的模式或 syntax-parse 形式)的选择点。

同时使用 #:commit 和 #:no-delimit-cut 是一个错误。

#:literals (literal-entry ...)
#:datum-literals (datum-literal-entry ...)
#:literal-sets (literal-set ...)
#:conventions (convention-id ...)

声明适用于语法类的变体模式和它们的直接 #:with 子句的字面意义和约定。 语法类的子表达式中出现的模式(例如,在 #:fail-when 子句的右侧)不受影响。

#:local-conventions (convention-rule ...)
#:disable-colon-notation

这些选项的含义与 syntax-parse 中相同。

语法类的每个变体都被指定为一个单独的 pattern-form , 其语法模式是一个 single-term pattern 。

syntax

(define-splicing-syntax-class name-id stxclass-option ...
  stxclass-variant ...+)
(define-splicing-syntax-class (name-id . kw-formals) stxclass-option ...
  stxclass-variant ...+)
将 name-id 定义为一个 splicing syntax class, 类似于 syntax class ,但封装了 head patterns 而不是 single-term patterns 。

选项与 define-syntax-class 相同。

拼接语法类的每个变体都被指定为一个单独的 pattern-form ,其语法模式是一个 head pattern 。

syntax

(pattern syntax-pattern pattern-directive ...)

用来表示一个语法类或拼接语法类的变体。 该变体接受与给定的语法模式相匹配的语法,并附带 pattern directives 。

当在 define-syntax-class 中使用时, syntax-pattern 应该是一个 single-term pattern; 在 define-splicing-syntax-class 中,它应该是一个 head pattern 。

变体的属性是模式的属性以及 #:with 子句绑定的所有属性,包括由与模式变量相关的语法类产生的嵌套属性。

syntax

this-syntax

当作为语法类定义或 syntax-parse 表达式中的一个表达式使用时, 评估为被匹配的语法对象或 syntax pair 。

Examples:
> (define-syntax-class one (pattern _ #:attr s this-syntax))
> (syntax-parse #'(1 2 3) [(1 o:one _) (attribute o.s)])

#<syntax:eval:3:0 2>

> (syntax-parse #'(1 2 3) [(1 . o:one) (attribute o.s)])

'(#<syntax:eval:4:0 2> #<syntax:eval:4:0 3>)

> (define-splicing-syntax-class two (pattern (~seq _ _) #:attr s this-syntax))
> (syntax-parse #'(1 2 3) [(t:two 3) (attribute t.s)])

#<syntax:eval:6:0 (1 2 3)>

> (syntax-parse #'(1 2 3) [(1 t:two) (attribute t.s)])

'(#<syntax:eval:7:0 2> #<syntax:eval:7:0 3>)

当作为语法类定义或 syntax-parse 表达式之外的表达式使用时,会引发错误。

value

prop:syntax-class

 : 
(struct-type-property/c (or/c identifier?
                              (-> any/c identifier?)))
一个 structure type property , 用于识别作为 syntax class 或 splicing syntax class 别名的结构类型。 该属性值必须是一个标识符或一个参数的过程。

当 transformer 被绑定到具有此属性的结构实例时, 那么它可以被用作 syntax class 或 splicing syntax class , 其方式与 define-syntax-class 或 define-splicing-syntax-class 创建的绑定相同。 如果属性的值是一个标识符,那么它应该被绑定到一个 syntax class 或 splicing syntax class, 并且该绑定将被视为被引用的语法类的别名。 如果该属性的值是一个过程,那么将用 prop:syntax-class 属性对该值进行应用, 以获得一个标识符,然后像前一种情况一样使用该标识符。

Examples:
> (begin-for-syntax
    (struct expr-and-stxclass (expr-id stxclass-id)
      #:property prop:procedure
      (lambda (this stx) ((set!-transformer-procedure
                           (make-variable-like-transformer
                            (expr-and-stxclass-expr-id this)))
                          stx))
      #:property prop:syntax-class
      (lambda (this) (expr-and-stxclass-stxclass-id this))))
> (define-syntax is-id? (expr-and-stxclass #'identifier? #'id))
> (is-id? #'x)

#t

> (syntax-parse #'x
    [x:is-id? #t]
    [_ #f])

#t

1.4.1 模式指令

解析形式和语法类定义形式都支持 pattern directives ,用于注释语法模式和指定侧面条件。 模式指令的语法如下:

  pattern-directive = #:declare pvar-id stxclass maybe-role
  | #:post action-pattern
  | #:and action-pattern
  | #:with syntax-pattern stx-expr
  | #:attr attr-arity-decl expr
  | #:fail-when condition-expr message-expr
  | #:fail-unless condition-expr message-expr
  | #:when condition-expr
  | #:do [def-or-expr ...]
  | #:undo [def-or-expr ...]
  | #:cut

#:declare pvar-id stxclass maybe-role
 
stxclass = syntax-class-id
  | (syntax-class-id arg ...)
     
maybe-role = 
  | #:role role-expr

将 pvar-id 与一个语法类和可能的角色联系起来, 相当于用 (~var pvar-id stxclass maybe-role) 替换模式中出现的每个 pvar-id 。 stxclass 的第二种形式允许使用参数化的语法类,这些语法类不能用 "冒号" 符号表达。 arg 在 pvar-id 出现在模式中的范围内被评估。 支持关键字参数,使用与 #%app 中相同的语法。

如果 #:with 指令出现在主模式 (例如,在 syntax-parse 或 define-syntax-class 子句中)和 #:declare 之间, 那么只有 #:with 模式的变量可以被声明。

Examples:
> (syntax-parse #'P
    [x
     #:declare x id
     #'x])

#<syntax:eval:12:0 P>

> (syntax-parse #'L
    [x
     #:with y #'x
     #:declare x id
     #'x])

syntax-parse: identifier in #:declare clause does not appear

in pattern;

 this #:declare clause affects only the preceding #:with

pattern

  at: x

  in: (syntax-parse (syntax L) (x #:with y (syntax x)

#:declare x id (syntax x)))

> (syntax-parse #'T
    [x
     #:with y #'x
     #:declare y id
     #'x])

#<syntax:eval:14:0 T>

#:post action-pattern

在匹配主模式之后,执行给定的 action pattern 作为 “post-traversal check”。 也就是说,以下内容是等价的:
main-pattern #:post action-pattern
main-pattern #:and (~post action-pattern)
(~and main-pattern (~post action-pattern))

#:and action-pattern

与 #:post 类似,只是没有添加 ~post 包装器。也就是说,下面这些是等价的:
main-pattern #:and action-pattern
(~and main-pattern action-pattern)

#:with syntax-pattern stx-expr

在之前所有属性绑定的背景下评估 stx-expr ,并与模式匹配。 如果匹配成功,模式的属性将被添加到环境中,用于评估后续的侧条件。 如果 #:with 匹配失败,则匹配过程会回溯。 由于一个语法对象可能以多种方式匹配一个模式,回溯可能导致同一子句在到达下一个子句之前被多次尝试。

如果 stx-expr 的值不是一个语法对象,它将被隐含地转换为一个语法对象。 如果转换会产生 3D syntax—即包含不可写的值的语法,如过程、非预置结构等–那么就会引发一个异常。

相当于 #:post (~parse syntax-pattern stx-expr) 。

Examples:
> (syntax-parse #'(1 2 3)
    [(a b c)
     #:with rev #'(c b a)
     #'rev])

#<syntax:eval:15:0 (3 2 1)>

> (syntax-parse #'(['x "Ex."] ['y "Why?"] ['z "Zee!"])
    [([stuff ...] ...)
     #:with h #'(hash stuff ... ...)
     #'h])

#<syntax:eval:16:0 (hash (quote x) "Ex." (quote y) "Why?" (quote z) "Zee!")>

#:attr attr-arity-decl expr

在所有先前的属性绑定的背景下评估 expr ,并将其绑定到给定的属性。 expr 的值不需要是,甚至不需要包含语法–详情见 attribute 。

相当于 #:and (~bind attr-arity-decl expr) 。

Examples:
> (syntax-parse #'("do" "mi")
    [(a b)
     #:attr rev #'(b a)
     #'rev])

#<syntax:eval:17:0 ("mi" "do")>

> (syntax-parse #'(1 2)
    [(a:number b:number)
     #:attr sum (+ (syntax-e #'a) (syntax-e #'b))
     (attribute sum)])

3

#:attr 指令经常在语法类中使用:

Examples:
> (define-syntax-class ab-sum
    (pattern (a:number b:number)
      #:attr sum (+ (syntax-e #'a) (syntax-e #'b))))
> (syntax-parse #'(1 2)
    [x:ab-sum
     (attribute x.sum)])

3

#:fail-when condition-expr message-expr
 
  message-expr : (or/c string? #f)

在之前所有属性绑定的背景下评估 condition-expr 。 如果值是任何真值(不是 #f),匹配过程会回溯(给定的信息);否则,继续。 如果条件表达式的值是一个语法对象,那么它将被表示为错误的原因。

如果 message-expr 产生一个字符串,它将被用作故障信息;否则,故障将以包围的描述方式报告。

相当于 #:post (~fail #:when condition-expr message-expr) 。

Examples:
> (syntax-parse #'(m 4)
    [(m x:number)
     #:fail-when (even? (syntax-e #'x))
     "expected an odd number"
     #'x])

m: expected an odd number

  at: (m 4)

  in: (m 4)

> (syntax-parse #'(m 4)
    [(m x:number)
     #:fail-when (and (even? (syntax-e #'x)) #'x)
     "expected an odd number"
     #'x])

m: expected an odd number

  at: 4

  in: (m 4)

#:fail-unless condition-expr message-expr
 
  message-expr : (or/c string? #f)

类似于 #:fail-when ,条件被否定。

相当于 #:post (~fail #:unless condition-expr message-expr) 。

Example:
> (syntax-parse #'(m 5)
    [(m x:number)
     #:fail-unless (even? (syntax-e #'x))
     "expected an even number"
     #'x])

m: expected an even number

  at: (m 5)

  in: (m 5)

#:when condition-expr

在之前所有属性绑定的背景下评估 condition-expr 。如果其值是 #f, 则匹配过程会回溯。 换句话说, #:when 就像 #:fail-unless ,没有消息参数。

相当于 #:post (~fail #:unless condition-expr #f) 。

Example:
> (syntax-parse #'(m 5)
    [(m x:number)
     #:when (even? (syntax-e #'x))
     #'x])

m: bad syntax

  in: (m 5)

#:do [defn-or-expr ...]

接受一连串的定义和表达式,这些定义和表达式可能是混合的,并在所有先前的属性绑定的范围内对它们进行评估。 被定义绑定的名字在随后的模式和子句的表达式中都有范围。

目前还没有办法使用 #:do 块来绑定属性。在 #:do 块中用定义影射属性绑定是一个错误。

相当于 #:and (~do defn-or-expr ...) 。

#:undo [defn-or-expr ...]

在最初匹配时没有影响,但如果回溯到 #:undo 指令 之前 的某一点, defn-or-expr 就会被执行。 例子见 ~undo 。

相当于 #:and (~undo defn-or-expr ...) 。

#:cut

消除回溯选择点,在当前点提交解析到当前分支。

相当于 #:and ~! 。

1.4.2 模式变量和属性

一个 attribute 是一个由语法模式约束的名称。 一个属性可以是一个 pattern variable 本身, 也可以是一个 annotated pattern variable 绑定的 nested attribute 。 嵌套属性的名称是通过连接模式变量的名称和语法类的输出属性的名称来计算的,用一个点来分隔(见下面的例子)。

属性可以通过三种方式使用:使用 attribute 形式; 通过 syntax, quasisyntax 等在语法模板内使用; 以及在 datum 模板内使用。 属性名不能直接作为表达式使用,也就是说,属性不是变量。

syntax-valued attribute 是一个属性, 其值是一个语法对象或适当 ellipsis depth 的列表。 也就是说,一个省略号深度为 0 的属性,如果它的值是 syntax? ,就是语法值; 一个省略号深度为 1 的属性,如果它的值是 (listof syntax?) ,就是语法值; 一个省略号深度为 2 的属性,如果它的值是 (listof (listof syntax?)) ,就是语法值;等等。 如果该值包含 promises , 当完全强制产生一个合适的语法对象或列表时,该值被认为是语法值。 语法值属性可以作为语法模板的一部分在 syntax,quasisyntax 等内部使用。 如果一个属性被用在语法模板中,但它不是语法值的,那么就会发出错误信号。

非语法值属性是有用途的。 非语法值属性可以用来返回子项的解析表示或子项的分析结果。 非语法值属性必须使用 #:attr 指令或 ~bind 模式进行绑定; #:with 和 ~parse 将把右侧转换为一个(可能是 3D) 语法对象。

Example:
> (define-syntax-class table
    (pattern ((key value) ...)
             #:attr hashtable
                    (for/hash ([k (syntax->datum #'(key ...))]
                               [v (syntax->datum #'(value ...))])
                      (values k v))
             #:attr [sorted-kv 1]
                    (delay
                     (printf "sorting!\n")
                     (sort (syntax->list #'((key value) ...))
                           <
                           #:key (lambda (kv) (cadr (syntax->datum kv)))))))

table 语法类提供了四个属性: key, value, hashtable 和 sorted-kv 。 hashtable 属性的 ellipsis depth 为 0 ,其余的深度为 1; key,value 和 sorted-kv 是语法值, 但 hashtable 不是。 sorted-kv 属性的值是一个承诺;如果在模板中使用,它将被自动强制。

语法值属性可以在语法模板中使用:

> (syntax-parse #'((a 3) (b 2) (c 1))
    [t:table
     #'(t.key ...)])

#<syntax:eval:26:0 (a b c)>

> (syntax-parse #'((a 3) (b 2) (c 1))
    [t:table
     #'(t.sorted-kv ...)])

sorting!

#<syntax:eval:27:0 ((c 1) (b 2) (a 3))>

但是非语法值的属性不能:

> (syntax-parse #'((a 3) (b 2) (c 1))
    [t:table
     #'t.hashtable])

t.hashtable: attribute contains non-syntax value

  value: '#hash((a . 3) (b . 2) (c . 1))

  in: t.hashtable

attribute 表单会得到一个属性的值,无论它是否是语法值的。

> (syntax-parse #'((a 1) (b 2) (c 3))
    [t:table
     (attribute t.hashtable)])

'#hash((a . 1) (b . 2) (c . 3))

> (syntax-parse #'((a 3) (b 2) (c 1))
    [t:table
     (attribute t.sorted-kv)])

#<promise:sorted-kv496>

每个属性都有一个相关的 ellipsis depth , 决定了它在语法模板中的使用方式(见 syntax 中省略号的讨论)。 对于一个模式变量来说,省略号深度是该模式变量在模式中 "出现在" 省略号的数量。 一个由 #:attr 绑定的属性的深度为 0,除非另有声明。 对于一个嵌套的属性,深度是注释的模式变量的深度和由语法类导出的属性的深度之和。

考虑一下下面的代码:

(define-syntax-class quark
  (pattern (a b ...)))
(syntax-parse some-term
  [(x (y:quark ...) ... z:quark)
   some-code])

语法类 quark 导出了两个属性:深度为 0 的 a 和深度为 1 的 b 。 syntax-parse 模式有三个模式变量: 深度为 0 的 x 、深度为 2 的 y 和深度为 0 的 z 。 由于 y 和 z 被注释为 quark 语法类,该模式还绑定了以下嵌套属性: y.a 在深度 2, y.b 在深度 3, z.a 在深度 0,而 z.b 在深度 1。

一个属性的省略号嵌套深度并 不 能保证它是语法值或有任何列表结构。 特别是 ~or* 和 ~optional 模式可能会导致属性的列表嵌套级别低于预期, 而且 #:attr 和 ~bind 可以用来将属性绑定到任意的值。

Example:
> (syntax-parse #'(a b 3)
    [(~or* (x:id ...) _)
     (attribute x)])

#f

syntax

(attribute attr-id)

返回与名为 attr-id 的 attribute 相关的值。 如果 attr-id 没有被绑定为一个属性,会产生一个语法错误。

1.4.2.1 属性和 datum

datum 形式是除了 syntax 和 attribute 之外, 使用语法模式变量和属性的另一种方式。 与 syntax 不同, datum 不要求属性是语法值。 凡是 syntax 形式会根据其模板创建语法对象的地方(而不是重复使用由模式变量绑定的语法对象), datum 形式都会创建普通的 S 表达式。

继续上面的 table 例子,我们可以使用带有 key 属性的 datum ,如下所示:

> (syntax-parse #'((a 1) (b 2) (c 3))
    [t:table (datum (t.key ...))])

'(#<syntax:eval:32:0 a> #<syntax:eval:32:0 b> #<syntax:eval:32:0 c>)

一个 datum 模板可以包含在一些 S 表达式结构中组合的多个模式变量:

> (syntax-parse #'((a 1) (b 2) (c 3))
    [t:table (datum ([t.key t.value] ...))])

'((#<syntax:eval:33:0 a> #<syntax:eval:33:0 1>)

  (#<syntax:eval:33:0 b> #<syntax:eval:33:0 2>)

  (#<syntax:eval:33:0 c> #<syntax:eval:33:0 3>))

一个 datum 模板可以使用 ~@ 和 ~? 模板形式:

> (syntax-parse #'((a 1) (b 2) (c 3))
    [t:table (datum ((~@ t.key t.value) ...))])

'(#<syntax:eval:34:0 a>

  #<syntax:eval:34:0 1>

  #<syntax:eval:34:0 b>

  #<syntax:eval:34:0 2>

  #<syntax:eval:34:0 c>

  #<syntax:eval:34:0 3>)

> (syntax-parse #'((a 56) (b 71) (c 13))
    [t:table (datum ((~@ . t.sorted-kv) ...))])

sorting!

'(#<syntax:eval:35:0 c>

  #<syntax:eval:35:0 13>

  #<syntax:eval:35:0 a>

  #<syntax:eval:35:0 56>

  #<syntax:eval:35:0 b>

  #<syntax:eval:35:0 71>)

> (syntax-parse #'( ((a 1) (b 2) (c 3)) ((d 4) (e 5)))
    [(t1:table (~or* t2:table #:nothing))
     (datum (t1.key ... (~? (~@ t2.key ...))))])

'(#<syntax:eval:36:0 a>

  #<syntax:eval:36:0 b>

  #<syntax:eval:36:0 c>

  #<syntax:eval:36:0 d>

  #<syntax:eval:36:0 e>)

> (syntax-parse #'( ((a 1) (b 2) (c 3)) #:nothing)
    [(t1:table (~or* t2:table #:nothing))
     (datum (t1.key ... (~? (~@ t2.key ...))))])

'(#<syntax:eval:37:0 a> #<syntax:eval:37:0 b> #<syntax:eval:37:0 c>)

然而,与 syntax 不同的是, #f 的值只在省略号迭代需要一个列表时向 ~? 发出模板失败的信号, 就像前面的例子一样;当它作为叶子出现时,不会导致失败。对比一下下面的情况:

> (syntax-parse #'( ((a 1) (b 2) (c 3)) #:nothing)
    [(t1:table (~or* t2:table #:nothing))
     #'(~? t2 skipped)])

#<syntax:eval:38:0 skipped>

> (syntax-parse #'( ((a 1) (b 2) (c 3)) #:nothing)
    [(t1:table (~or* t2:table #:nothing))
     (datum (~? t2 skipped))])

#f

datum 形式对于访问非语法值的属性也很有用。 与 attribute 相比, datum 有以下优势: 在 datum 模板中使用省略号提供了对其结果的列表结构的视觉提醒。 例如,如果模式是 (t:table ...), 那么 (attribute t.hashtable) 和 (datum (t.hashtable ...)) 都会产生一个 (listof hash?) ,但是省略号让它更加明显。
