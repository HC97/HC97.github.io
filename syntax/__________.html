syntax class

(static predicate description) → syntax class

  predicate : (-> any/c any/c)
  description : (or/c string? #f)
static 语法类匹配一个标识符, 该标识符在语法环境中与满足给定谓词的静态信息(见 syntax-local-value) 绑定, 满足给定的 predicate 。 如果该术语不匹配, description 参数用于描述预期的语法。

当在宏转换器的动态范围之外使用时(见 syntax-transforming?), ,匹配失败。

属性 value 包含名称所绑定的值。

如果匹配成功, static 会将匹配的标识符添加到当前的 syntax-parse 状态中, 并将其放在使用 syntax-parse-state-cons! 的 'literals 键下, 这与使用 #:literals 或 ~literal 匹配的标识符的方式相同。

Changed in version 6.90.0.29 of package base: 改为在 'literals 键下将匹配的标识符添加到 syntax-parse 状态中。

syntax class

(expr/c contract-expr    
  [#:arg? arg?    
  #:positive pos-blame    
  #:negative neg-blame    
  #:name expr-name    
  #:macro macro-name    
  #:context context    
  #:phase phase]) → syntax class
  contract-expr : syntax?
  arg? : any/c = #t
  pos-blame : (or/c syntax? string? module-path-index? 'from-macro 'use-site 'unknown)
   = 'from-macro
  neg-blame : (or/c syntax? string? module-path-index? 'from-macro 'use-site 'unknown)
   = 'use-site
  expr-name : (or/c identifier? string? symbol?) = #f
  macro-name : (or/c identifier? string? symbol?) = #f
  context : (or/c syntax? #f) = determined automatically
  phase : exact-integer? = (syntax-local-phase-level)
接受一个表达式 (expr) 并计算一个属性 c , 该属性代表被 contract-expr 代表的契约所包裹的表达式。 请注意,每次运行宏生成的代码时, contract-expr 都有可能被评估; 为了获得最佳性能,contract-expr 应该是一个变量引用。

契约的正责代表了施加契约的宏的义务– expr/c 的最终使用者。 契约的负责代表被包装的表达式的义务。 默认情况下,正责是由宏的定义位置推断出来的(本身是由 context 参数推断出来的), 负责是作为当前被展开的模块,但是这两个责的位置都可以被重写。 当 arg? 为 #t 时,被匹配的词被解释为一个参数(即来自否定方); 当 arg? 为 #f 时,被匹配的词被解释为宏的结果(即来自肯定方)。

pos-blame 和 neg-blame 参数被转化为指责位置,具体如下:

  • 如果参数是一个字符串,它被直接作为责备标签使用。

  • 如果参数是语法,它的源位置被用来产生责备标签。

  • 如果参数是一个模块路径索引,它的解析模块路径被使用。

  • 如果参数是 'from-macro, 则从 macro-name 参数(如果 macro-name 是一个标识符)或 context 参数中推断出宏,并将其 defined 的模块作为责备位置。 如果既没有给出标识符 macro-name ,也没有给出 context 参数, 则位置为 "unknown" 。

  • 如果参数是 'use-site, 则使用被扩展的模块。

  • 如果参数是 'unknown, 责备标签是 "unknown" 。

如果 macro-name 是一个标识符,那么宏名参数用于确定宏的绑定。 如果给出了 expr-name , macro-name 也会包含在合约错误信息中。 如果 macro-name 被省略或 #f, 但是 context 是一个语法对象,那么 macro-name 将由 context 决定。

如果 expr-name 不是 #f, 它将被用于合约的错误信息中,以描述合约所应用的表达式。

必要时, context 参数被用来推断负责方的宏名称和合同错误信息。 context 应该是一个标识符,或者是一个在运算符位置上有标识符的语法对; 在任何一种情况下,该标识符都被当作最终请求契约包装的宏。

phase 参数必须指出将对契约表达式进行评估的 phase level 。 在不同的阶段水平上使用契约表达式将导致语法错误,因为它将包含在错误的阶段中绑定的引入的引用。 特别是:

  • 当收缩的表达式将在与当前被展开的形式相同的阶段被评估时, 使用默认值 (syntax-local-phase-level) 。通常是这种情况。

  • 在以下情况下使用 (add1 (syntax-local-phase-level)) : 收缩后的表达式将被放在 begin-for-syntax 表单中, 用于 define-syntax 或 let-syntax 表单的右侧, 或者传递给 syntax-local-bind-syntaxes 或 syntax-local-eval 。

除了 #f (label phase level) 之外的任何相位都是允许的, 但是除了 (syntax-local-phase-level) 和 (add1 (syntax-local-phase-level)) 之外的相位只能在 syntax transformer 的动态范围内或者在 visited 模块的时候使用(参见 syntax-transforming?), 否则会引发 exn:fail:contract? 。

参见 关于宏子表达式的契约 的例子。

重要提示: 使用 expr/c 时,请确保使用 c 属性。 expr/c 语法类并不改变模式变量的绑定方式;它只是计算了一个代表检查表达式的属性。

Changed in version 7.2.0.3 of package base: 增加了 #:arg? 关键字参数, 并改变了 #:positive 和 #:negative 参数的默认值和解释。
Changed in version 7.3.0.3: 增加了 #:phase 关键字参数。

1.8.2 字母集

syntax

kernel-literals

包含完全展开的程序 (完全展开的程序)。 这个集合包含了所有由 kernel-form-identifier-list 列出的形式,再加上 module, #%plain-module-begin, #%require 和 #%provide 。

注意,字面意义集使用的是 #%plain-lambda 和 #%plain-app, 而不是 lambda 和 #%app 。

> (syntax-parse #'(define ((foo x) y) 1)
    [(_ header:function-header body ...+) #'(header header.name header.params)])

#<syntax:eval:2:0 (((foo x) y) foo (x y))>

> (syntax-parse #'(lambda xs xs)
    [(_ fmls:formals body ...+) #'(fmls fmls.params)])

#<syntax:eval:3:0 (xs (xs))>

> (syntax-parse #'(lambda (x y #:kw [kw 42] . xs) xs)
    [(_ fmls:formals body ...+) #'(fmls fmls.params)])

#<syntax:eval:4:0 ((x y #:kw (kw 42) . xs) (x y kw xs))>

> (syntax-parse #'(lambda (x) x)
    [(_ (fml:formal) body ...+) #'(fml
                                   fml.name
                                   (~? fml.kw #f)
                                   (~? fml.default #f))])

#<syntax:eval:5:0 ((x) x #f #f)>

> (syntax-parse #'(lambda (#:kw [kw 42]) kw)
    [(_ (fml:formal) body ...+) #'(fml fml.name fml.kw fml.default)])

#<syntax:eval:6:0 ((#:kw (kw 42)) kw #:kw 42)>
