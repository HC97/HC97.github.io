1.9 不可逆转的状态

procedure

(syntax-parse-state-ref key [default]) → any/c

  key : any/c
  default : default/c = (lambda () (error ....))

procedure

(syntax-parse-state-set! key value) → void?

  key : any/c
  value : any/c

procedure

(syntax-parse-state-update! key    
  update    
  [default]) → void?
  key : any/c
  update : (-> any/c any/c)
  default : default/c = (lambda () (error ....))

procedure

(syntax-parse-state-cons! key value [default]) → void?

  key : any/c
  value : any/c
  default : default/c = null
获取或更新当前 syntax-parse 的状态。 当 syntax-parse 回溯时,对状态的更新将被取消。使用 eq? 对键进行比较。

状态只能在 ~do 模式(或 #:do 块)中被更新。 此外, syntax-parse 会自动在键 'literals 下添加与字词相匹配的标识符(来自 ~literal 模式和用 #:literals 声明的字词, 但不是来自 ~datum 或 #:datum-literals) 。

Examples:
> (define-syntax-class cond-clause
    #:literals (=> else)
    (pattern [test:expr => ~! answer:expr ...])
    (pattern [else answer:expr ...])
    (pattern [test:expr answer:expr ...]))
> (syntax-parse #'(cond [A => B] [else C])
    [(_ c:cond-clause ...) (syntax-parse-state-ref 'literals null)])

'(#<syntax:eval:2:0 else> #<syntax:eval:2:0 =>>)

Added in version 6.11.0.4 of package base.

procedure

(syntax-parse-track-literals stx    
  [#:introduce? introduce?]) → syntax?
  stx : syntax?
  introduce? : any/c = #t
为 stx 添加一个 'disappeared-use syntax property , 其中包含当前 syntax-parse 状态下存储在键 'literals 中的信息。 如果 stx 已经有一个 'disappeared-use 的属性, 添加的信息将被 cons 到该属性的当前值上。

由于 syntax-parse 自动将符合字面意义的标识符添加到键 'literals 下的状态中, 正如在 syntax-parse-state-ref 的文档中所描述的那样, syntax-parse-track-literals 可以用来自动将任何作为字面意义的标识符添加到 'disappeared-use 属性中。

如果 syntax-parse-track-literals 在 syntax transformer 的动态范围内被调用(见 syntax-transforming?), introduce? 不是 #f, 并且当前 syntax-parse 状态下 'literals 键下的值是一个列表, 那么在列表中的任何标识符被添加到 stx 的 'disappeared-use 属性之前, syntax-local-introduce 将被应用于这些标识符。

大多数情况下,没有必要直接调用这个函数。 相反,应该向 syntax-parse 提供 #:track-literals 选项, 它将自动调用 syntax-parse-track-literals 对语法值的结果。

Examples:
> (define-syntax-class cond-clause
    #:literals (=> else)
    (pattern [test:expr => ~! answer:expr ...])
    (pattern [else answer:expr ...])
    (pattern [test:expr answer:expr ...]))
> (syntax-property
   (syntax-parse #'(cond [A => B] [else C])
     [(_ c:cond-clause ...) (syntax-parse-track-literals #'#f)])
   'disappeared-use)

'(#<syntax:eval:4:0 else> #<syntax:eval:4:0 =>>)

Added in version 6.90.0.29 of package base.
