1.2.3 具有统一含义的变体

语法类不仅可以验证语法,还可以从中提取某种程度的意义。 从意义的角度来看,基本上有两种类型的语法类。 在第一种情况下,所有语法类的变体都具有相同的意义。 在第二种情况下,变体可能具有不同种类的意义。 换句话说,一些语法类的意义是产品,而另一些语法类的意义是总和。 本节讨论的是第一种,即具有统一含义的语法类。下一节将讨论 具有不同含义的变体 。

如果一个语法类的所有变体都表达相同的信息,那么这些信息就可以通过属性干净地表示出来, 并且可以用省略号进行简洁的处理。

具有统一含义的语法类的一个例子: class 宏的 init-decl 语法。 下面是 init-decl 的规范:

  init-decl = id
  | (maybe-renamed)
  | (maybe-renamed default-expr)
     
  maybe-renamed = id
  | (internal-id external-id)

init-decl 语法类有三个变体,加上一个辅助语法类,它本身有两个变体。 但是 init-decl 的所有形式最终都只携带三条信息:一个内部名称,一个外部名称,以及某种默认配置。 更简单的语法变体只是完整信息的缩写。

这三条信息决定了该语法类的属性。 使用 #:attributes 关键字明确地声明这些属性是很有用的;声明既是代码中的文档,也是对变体的检查。

(define-syntax-class init-decl
  #:attributes (internal external default)
  __)

接下来我们填入语法变体,推迟属性的计算:

(define-syntax-class init-decl
  #:attributes (internal external default)
  (pattern ???:id
           __)
  (pattern (???:maybe-renamed)
           __)
  (pattern (???:maybe-renamed ???:expr)
           __))

我们对 maybe-renamed 进行类似的分析:
(define-syntax-class maybe-renamed
  #:attributes (internal external)
  (pattern ???:id
           __)
  (pattern (???:id ???:id)
           __))

下面是一种将句法结构与 maybe-renamed 的属性相匹配的直接方法:

(define-syntax-class maybe-renamed
  #:attributes (internal external)
  (pattern internal:id
           #:with external #'internal)
  (pattern (internal:id external:id)))

鉴于 maybe-renamed 的定义,我们可以填入 init-decl 的大部分定义:

(define-syntax-class init-decl
  #:attributes (internal external default)
  (pattern internal:id
           #:with external #:internal
           #:with default ???)
  (pattern (mr:maybe-renamed)
           #:with internal #'mr.internal
           #:with external #'mr.external
           #:with default ???)
  (pattern (mr:maybe-renamed default0:expr)
           #:with internal #'mr.internal
           #:with external #'mr.external
           #:with default ???))

在这一点上,我们意识到我们还没有决定默认配置的表示方法。 事实上,这是一个 syntax with varied meanings (又称总和或不相干的联合)的例子。 下一节将更详细地讨论表示法选项;为了完整起见,我们在这里介绍其中的一种。

有两种默认配置。一种表示初始化参数是可选的,默认值是由给定的表达式计算的。 另一种表示初始化参数是强制性的。 我们将这些变体分别表示为包含默认表达式的(语法)列表和空(语法)列表。更确切地说:

(define-syntax-class init-decl
  #:attributes (internal external default)
  (pattern internal:id
           #:with external #:internal
           #:with default #'())
  (pattern (mr:maybe-renamed)
           #:with internal #'mr.internal
           #:with external #'mr.external
           #:with default #'())
  (pattern (mr:maybe-renamed default0:expr)
           #:with internal #'mr.internal
           #:with external #'mr.external
           #:with default #'(default0)))

另一种看待语法类设计的方式是将积之和(具体的语法变体)代数化为积之和(属性和抽象的语法变体)。 后一种形式的优点是在提取数据时使用 "点" 符号,避免或减少额外的案例分析,以及能够使用省略号简洁地操作序列。
