2 语法对象帮助器

2.1 解构句法对象

 (require syntax/stx) package: base

procedure

(stx-null? v) → boolean?

  v : any/c
如果 v 是空列表或代表空列表的 syntax object , 则返回 #t (即 syntax object 上的 syntax-e 返回空列表)。

Examples:
> (stx-null? null)

#t

> (stx-null? #'())

#t

> (stx-null? #'(a))

#f

procedure

(stx-pair? v) → boolean?

  v : any/c
如果 v 是一对,或者是代表一对的语法对象, 则返回 #t (参见 syntax pair)。

Examples:
> (stx-pair? (cons #'a #'b))

#t

> (stx-pair? #'(a . b))

#t

> (stx-pair? #'())

#f

> (stx-pair? #'a)

#f

procedure

(stx-list? v) → boolean?

  v : any/c
如果 v 是一个列表,或者如果它是一个指向语法对象的对的序列, 从而使 syntax->list 产生一个列表,则返回 #t 。

Examples:
> (stx-list? #'(a b c d))

#t

> (stx-list? #'((a b) (c d)))

#t

> (stx-list? #'(a b (c d)))

#t

> (stx-list? (list #'a #'b))

#t

> (stx-list? #'a)

#f

procedure

(stx->list stx-list) → (or/c list? #f)

  stx-list : stx-list?
通过使用 syntax->list 来平移尾部的语法对象,产生一个列表。

Examples:
> (stx->list #'(a b c d))

'(#<syntax:eval:14:0 a>

  #<syntax:eval:14:0 b>

  #<syntax:eval:14:0 c>

  #<syntax:eval:14:0 d>)

> (stx->list #'((a b) (c d)))

'(#<syntax:eval:15:0 (a b)> #<syntax:eval:15:0 (c d)>)

> (stx->list #'(a b (c d)))

'(#<syntax:eval:16:0 a> #<syntax:eval:16:0 b> #<syntax:eval:16:0 (c d)>)

> (stx->list (list #'a #'b))

'(#<syntax:eval:17:0 a> #<syntax:eval:17:0 b>)

> (stx->list #'a)

#f

procedure

(stx-car v) → any

  v : stx-pair?
取一个 syntax pair 的 car。

Examples:
> (stx-car #'(a b))

#<syntax:eval:19:0 a>

> (stx-car (list #'a #'b))

#<syntax:eval:20:0 a>

procedure

(stx-cdr v) → any

  v : stx-pair?
取一个 syntax pair 的 cdr。

Examples:
> (stx-cdr #'(a b))

'(#<syntax:eval:21:0 b>)

> (stx-cdr (list #'a #'b))

'(#<syntax:eval:22:0 b>)

procedure

(stx-map proc stxl ...) → list?

  proc : procedure?
  stxl : stx-list?
相当于 (map proc (stx->list stxl) ...) 。

Example:
> (stx-map (λ (id) (free-identifier=? id #'a)) #'(a b c d))

'(#t #f #f #f)

procedure

(module-or-top-identifier=? a-id b-id) → boolean?

  a-id : identifier?
  b-id : identifier?
如果 a-id 和 b-id 是 free-identifier=?, 或者如果 a-id 和 b-id 有相同的名字(由 syntax-e 提取), 并且 a-id 除了在顶层之外没有其他约束力,则返回 #t 。

这个程序与 syntax-case* 结合使用,可以匹配通常被 Racket 绑定的过程名称。 例如,i include 宏使用这个过程来识别 build-path; 在 module 之外使用 free-identifier=? 效果不好, 因为顶层的 build-path 是一个独立于 racket/base export 的变量 (尽管它最初被绑定到同一个过程)。

2.2 匹配完全扩展的表达式

 (require syntax/kerncase) package: base

syntax

(kernel-syntax-case stx-expr trans?-expr clause ...)

一个类似于 syntax-case* 的句法形式, 除了字面意义被内置为由 racket/base 导出的原始 Racket 形式的名称, 包括 letrec-syntaxes+values; 见 完全展开的程序 。

trans?-expr 布尔表达式取代了比较程序,而是简单地在正常阶段的比较或转化器阶段的比较之间进行选择。 这些 clause 与 syntax-case* 中的相同。

原始句法形式必须在 kernel-syntax-case 表达式的上下文中具有正常的绑定关系。 注意 kernel-syntax-case 不能在语言为这些原始语法形式提供不同绑定的模块中工作, 比如 mzscheme 没有提供原始的 if ， typed/racket 没有提供原始的 let-values 等等。

syntax

(kernel-syntax-case* stx-expr trans?-expr (extra-id ...) clause ...)

一个类似于 kernel-syntax-case 的语法形式, 只是它需要一个额外的字词列表,这些字词是 Racket 原始形式之外的。

syntax

(kernel-syntax-case/phase stx-expr phase-expr clause ...)

将 kernel-syntax-case 泛化为在任意阶段的工作,如 phase-expr 所示。

syntax

(kernel-syntax-case*/phase stx-expr phase-expr (extra-id ..)
  clause ...)
将 kernel-syntax-case* 泛化为任意阶段的工作,如 phase-expr 所示。

procedure

(kernel-form-identifier-list) → (listof identifier?)

返回一个标识符的列表,这些标识符被正常、for-syntax 和 for-template 绑定到表达式、内部定义位置、模块级和顶层位置的原始 Racket 形式。 这个函数对于生成提供给 local-expand 的停止点列表非常有用。

除了 完全展开的程序 中列出的标识符外,该列表还包括 letrec-syntaxes+values, 它是本地展开时间绑定的核心形式,可以出现在 local-expand 的结果中。

Changed in version 6.90.0.27 of package base: 在列表中增加了 quote-syntax 和 #%plain-module-begin , 它们之前被无意中遗漏。

2.3 带标识符键的字典

 (require syntax/id-table) package: base

这个模块提供了两种 identifier tables 的实现: 带有标识符键的字典,使用标识符特定的比较而不是 eq? 或 equal? 。 识别符表实现了 racket/dict 接口,它们有可变和不可变的两种变体。

2.3.1 free-identifier=? 的字典

一个自由标识符表是一个字典,其键值使用 free-identifier=? 进行比较。 自由标识符表实现了 racket/dict 的字典接口, 所以所有适当的通用函数 (dict-ref, dict-map, 等等) 都可以在自由标识符表中使用。

使用这些表的一个注意事项是,如果标识符的绑定在键值插入和查找之间发生了变化,查找可能会失败,并产生意想不到的结果。

例如,考虑下面的使用:

> (define-syntax-rule (m)
    (begin
      (begin-for-syntax
        (define table (make-free-id-table))
        ; set table entry to #t
        (free-id-table-set! table #'x #t)
        ; sanity check, it's set to #t
        (displayln (free-id-table-ref table #'x #f)))
  
      (define x 'defined-now)
  
      (begin-for-syntax
        ; might expect to get #t, but prints #f
        (displayln (free-id-table-ref table #'x #f)))))
> (m)

#t

#f

宏 m 扩展为代码,在编译时初始化一个标识符表,并为 #'x 和 #t 插入一个键值对。 然而, #'x 标识符没有约束力,直到定义 (define x 'defined-now) 被评估。

因此,在 m 结尾的查找将返回 #f 而不是 #t , 因为 #'x 的绑定符号在初始键值对被放入表中后发生了变化。 如果定义在初始插入 之前 被评估,两个表达式都会打印 #t 。

procedure

(make-free-id-table [init-dict    
  #:phase phase]) → mutable-free-id-table?
  init-dict : dict? = null
  phase : (or/c exact-integer? #f) = (syntax-local-phase-level)

procedure

(make-immutable-free-id-table [init-dict 
  #:phase phase]) 
 → immutable-free-id-table?
  init-dict : dict? = null
  phase : (or/c exact-integer? #f) = (syntax-local-phase-level)
分别产生一个可变的自由标识符表或不可变的自由标识符表。 这个字典使用 free-identifier=? 来比较键,但也使用基于符号平等的哈希表来使字典在普通情况下有效。

标识符是在阶段级别 phase 进行比较的。 默认的阶段, (syntax-local-phase-level), 一般适合于宏使用的标识符表,但分析完全扩展的程序的代码可能需要为模块的每个阶段创建单独的标识符表。

可选的 init-dict 参数提供初始映射。它必须是一个字典,而且它的键必须都是标识符。 如果 init-dict 字典有多个不同的条目,其键是 free-identifier=?, 那么只有一个条目出现在新的 id-table 中,并且不指定哪个条目被选中。

2.4 带有标识符键的集合

 (require syntax/id-set) package: base

这个模块提供了 identifier sets: 带有标识符键的集合,使用标识符特定的比较,而不是通常的平等运算符,如 eq? 或 equal? 。

本模块实现了两种标识符集: 一种是通过 free-identifier=? ,一种是通过 bound-identifier=? 。 每一种都有可变和不可变的变体,并实现了 gen:set,gen:stream, prop:sequence 和 gen:equal+hash 通用接口。

标识符集是用 identifier tables 实现的, 就像用 hash tables 实现的 hash sets 一样。

2.4.1 free-identifier=? 的集合

一个自由标识符集是一个集合,它的键是用 free-identifier=? 来比较的。 自由标识符集实现了 gen:set 接口, 所以所有适当的通用函数(例如, set-add, set-map,等等)都可以在自由标识符集上使用。

2.9 处理关键词语法的帮助工具

syntax/keyword 模块包含解析宏中关键字选项的程序。

 (require syntax/keyword) package: base

  keyword-table = (dict-of keyword (listof check-procedure))

关键词表是一个字典 (dict?) ,将关键词映射到 check-procedures 的列表中。 (注意,一个关联列表就是一个合适的字典)。 关键字的 arity 是程序列表的长度。

Example:
> (define my-keyword-table
    (list (list '#:a check-identifier)
          (list '#:b check-expression check-expression)))

  check-procedure = (syntax syntax -> any)

一个检查程序消耗要检查的语法和一个用于报告错误的上下文语法对象, 并且要么提出一个错误来拒绝该语法,要么返回一个值作为其解析后的表示。

Example:
> (define (check-stx-string stx context-stx)
    (unless (string? (syntax-e stx))
      (raise-syntax-error #f "expected string" context-stx stx))
    stx)

  options = (listof (list keyword syntax-keyword any ...))

被解析的选项被表示为一个选项条目的列表。 每个条目都包含关键字,关键字的语法(用于报错),以及由关键字的检查程序列表返回的解析值的列表。 列表中包含了按照输入中出现的顺序解析的选项,一个在输入中多次出现的关键词在选项列表中出现多次。

procedure

(parse-keyword-options stx 
  table 
  [#:context ctx 
  #:no-duplicates? no-duplicates? 
  #:incompatible incompatible 
  #:on-incompatible incompatible-handler 
  #:on-too-short too-short-handler 
  #:on-not-in-table not-in-table-handler]) 
 → 
options any/c
  stx : syntax?
  table : keyword-table
  ctx : (or/c false/c syntax?) = #f
  no-duplicates? : boolean? = #f
  incompatible : (listof (listof keyword?)) = '()
  incompatible-handler : 
(-> keyword? keyword?
    options syntax? syntax?
    (values options syntax?))
   = (lambda (....) (error ....))
  too-short-handler : 
(-> keyword? options syntax? syntax?
    (values options syntax?))
   = (lambda (....) (error ....))
  not-in-table-handler : 
(-> keyword? options syntax? syntax?
    (values options syntax?))
   = (lambda (....) (error ....))
解析语法 stx 中的关键词选项(stx 可能是一个不恰当的语法列表)。 关键字选项在 table 关联列表中描述。 table 中的每个条目都应该是一个列表,其第一个元素是一个关键字, 其后面的元素是检查关键字后面的参数的程序。 关键字的 arity(参数的数量)由条目中的程序数量决定。 只支持固定性质的关键字。

当语法表的头部没有关键字时,解析就会正常停止(它可能是空的,以非关键字的术语开始,或者它可能是一个非列表语法对象)。 返回两个值:被解析的 options 和语法的其余部分(通常是一个语法对象或一个语法对象的列表)。

在解析过程中可能会出现各种错误和特殊情况。下面的关键字参数决定了这些情况下的行为。

#:context ctx 参数用于报告解析语法的所有错误。 此外, ctx 被作为最后的参数传递给所有提供的处理程序。 使用 parse-keyword-options 的宏一般应将整个宏使用的语法对象作为 ctx 传递。

如果 no-duplicates? 是一个非假值,那么不允许重复的关键词选项。 如果看到重复的,关键词的相关检查程序就不会被调用,并且会报告一个 incompatibility 。

incompatible 参数是一个不兼容条目的列表,每个条目是一个 至少两个 关键词的列表。 如果条目中的任何关键字出现在条目中任何其他关键字之后,就会报告 incompatibility 。

请注意,在不兼容条目中包括一个关键词并不能防止它多次出现。 要禁止某些关键词的重复出现(相对于所有的关键词而言), 在 incompatible 列表中包括那些与自己不兼容的关键词。 也就是说,把它们列入两次:

; Disallow duplicates of only the #:foo keyword
(parse-keyword-options .... #:incompatible '((#:foo #:foo)))

当 incompatibility 发生时, incompatible-handler 会被调用, 包括导致不兼容的两个关键词(按照它们在语法列表中出现的顺序,所以触发不兼容的关键词出现在第二位), 从第二个关键词出现开始的语法列表,以及上下文 (ctx)。 如果不兼容是由于重复的原因,那么这两个关键词是相同的。

当一个关键词后面没有足够的参数时,根据其在 table, too-short-handler 会被尾随调用, 包括该关键词、到目前为止被解析的 options 、从该关键词出现开始的语法列表和 ctx 。

当在语法列表中出现不在 table 中的关键词时, not-in-table-handler 就会被尾随调用,包括该关键词、到目前为止所解析的 options 、 从该关键词出现开始的语法列表以及 ctx.

处理程序通常会逃脱–所有的默认处理程序都会引发错误–但是如果它们返回,它们应该返回两个值: 被解析的 options 和一个语法对象;这些值会作为 parse-keyword-options 的结果返回。

Examples:
> (parse-keyword-options
   #'(#:transparent #:property p (lambda (x) (f x)))
   (list (list '#:transparent)
         (list '#:inspector check-expression)
         (list '#:property check-expression check-expression)))

'((#:transparent #<syntax:eval:4:0 #:transparent>)

  (#:property

   #<syntax:eval:4:0 #:property>

   #<syntax:eval:4:0 p>

   #<syntax:eval:4:0 (lambda (x) (f x))>))

'()

> (parse-keyword-options
   #'(#:transparent #:inspector (make-inspector))
   (list (list '#:transparent)
         (list '#:inspector check-expression)
         (list '#:property check-expression check-expression))
   #:context #'define-struct
   #:incompatible '((#:transparent #:inspector)
                    (#:inspector #:inspector)
                    (#:inspector #:inspector)))
