7.2 包管理功能

 (require pkg/lib) package: base
pkg/lib 库提供了构建模块, pkg 库和 raco pkg 命令是在此基础上建立的。 它重新输出了 pkg/path 的绑定。

syntax

(with-pkg-lock body ...+)

syntax

(with-pkg-lock/read-only body ...+)

在持有锁的情况下评估 body ,以防止同时修改当前 package scope 的软件包数据库。 使用 with-pkg-lock/read-only 形式来进行只读访问。 该锁是可重入的,但不能从只读升级。

使用这些形式来包装来自 pkg/lib 的函数的使用,这些函数被记录为需要锁。 来自 pkg/lib 的其他函数会根据需要使用锁。

procedure

(pkg-stage desc    
  [#:checksum checksum    
  #:in-place? in-place?    
  #:namespace namespace    
  #:strip strip    
  #:force-strip? force-strip?    
  #:use-cache? use-cache?    
  #:quiet? quiet?]) → 
string?
path?
(or/c #f string?)
boolean?
(set/c module-path?)
  desc : pkg-desc?
  checksum : (or/c #f string?) = #f
  in-place? : boolean? = #f
  namespace : namespace? = (make-base-namespace)
  strip : (or/c #f 'source 'binary 'binary-lib) = #f
  force-strip? : boolean? = #f
  use-cache? : boolean? = #f
  quiet? : boolean? = #t
找到由 desc 指定的软件包的实现,并下载和解压到一个临时目录(根据需要)。

如果 desc 指的是一个现有的目录,并且 in-place? 是 true,那么该目录将被就地使用。

当软件包的 "info.rkt" 被加载时, namespace 参数被传递给 get-info/full 。

如果 strip 不是 #f, 那么文件和目录会从准备好的目录中删除,与创建相应种类的包时相同。 一个被就地分期的目录不能被剥离。 如果 force-strip? 为真,那么一致性检查(旨在避免将一个源码包剥离为二进制,例如)将被跳过。

如果 use-cache? 为 true, 那么在下载具有特定校验和的特定软件包之前,将查询本地缓存。 注意, use-cache? 的默认值是 #f, 而其他接受 #:use-cache? 的函数的默认值是 #t 。

结果是软件包名称,包含解压后的软件包内容的目录,解压后的软件包的校验和(如果有的话), 在不再需要软件包内容后是否应该删除该目录,以及该软件包提供的模块路径列表。
