7.6 包目录

 (require pkg/dirs-catalog) package: base
pkg/dirs-catalog 模块提供了 create-dirs-catalog, 它生成了一个 package catalog (作为一个目录),指的是在给定目录中发现的一组包。 包在给定的目录中被发现为子目录,这些子目录有一个 "info.rkt" 文件。

例如,Racket 的主开发库包括一个 "pkgs" 目录,其中存放着 base 等软件包, create-dirs-catalog 构建了一个目录,用于安装这些软件包。

当作为一个程序直接运行时, pkg/dirs-catalog 希望有一个目标目录,后面是任何数量的存放软件包的目录路径:

  racket -l- pkg/dirs-catalog ‹dest-catalog› ‹dir› ...

--immediate, --link, --merge, --check-metadata 和 --quiet 标志对应于 create-dirs-catalog 的可选关键字参数。

Added in version 6.1.1.6 of package base.
Changed in version 6.90.0.4: 增加了 --immediate 。

procedure

(create-dirs-catalog catalog-path    
  dirs    
  [#:immediate? immediate?    
  #:link? link?    
  #:merge? merge?    
  #:check-metadata? check-metadata?    
  #:status-printf status-printf]) → void?
  catalog-path : path-string?
  dirs : (listof path-string?)
  immediate? : any/c = #f
  link? : any/c = #f
  merge? : any/c = #f
  check-metadata? : any/c = #f
  status-printf : (string? any/c ... -> void?) = void
创建或修改 catalog-path 作为目录(见 包目录协议), 以列出 dirs 指定的每个目录中包含的软件包。 软件包在 dirs 中被发现为有 "info.rkt" 文件的子目录; 如果 immediate? 为真,那么在检查子目录之前, dirs 中的每个目录都会被检查是否有 "info.rkt" 文件。

如果 link? 为真,那么目录指定软件包应该作为目录链接来安装,而不是拷贝。

如果 merge? 为真,那么 catalog-path 中现有的目录条目将被保留,否则旧的目录条目将被删除。

为了给目录中的每个软件包创建作者和描述信息, create-dirs-catalog 在每个软件包的 "info.rkt" 文件中寻找 pkg-authors 和 pkg-desc 定义。 如果这两个定义缺失,并且 check-metadata? 为true,就会报告一个错误。

如果软件包的 "info.rkt" 文件定义了 pkg-name, 那么这个定义的名字将被用作软件包的名字, 而不是软件包目录的名字。

Changed in version 6.90.0.4 of package base: 增加了 #:immediate 参数。
Changed in version 7.3.0.2: 增加了对 pkg-name 命名软件包的支持。
