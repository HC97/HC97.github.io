2.13 服务于文件

 (require web-server/dispatchers/dispatch-files)
  package: web-server-lib
web-server/dispatchers/dispatch-files 模型 允许文件被送达。它定义了一个调度器构建程序。

procedure

(make 
  #:url->path url->path 
  [#:path->mime-type path->mime-type 
  #:path->headers path->headers 
  #:indices indices 
  #:cache-max-age cache-max-age 
  #:cache-smaxage cache-smaxage 
  #:cache-stale-while-revalidate cache-stale-while-revalidate 
  #:cache-stale-if-error cache-stale-if-error 
  #:cache-no-cache cache-no-cache 
  #:cache-no-store cache-no-store 
  #:cache-no-transform cache-no-transform 
  #:cache-must-revalidate cache-must-revalidate 
  #:cache-proxy-revalidate cache-proxy-revalidate 
  #:cache-must-understand cache-must-understand 
  #:cache-private cache-private 
  #:cache-public cache-public 
  #:cache-immutable cache-immutable]) 
 → dispatcher/c
  url->path : url->path/c
  path->mime-type : (path? . -> . (or/c false/c bytes?))
   = (lambda (path) #f)
  path->headers : (path? . -> . (listof header?))
   = (lambda (path) '())
  indices : (listof string?) = (list "index.html" "index.htm")
  cache-max-age : (or/c false/c (and/c exact-integer? positive?))
   = #f
  cache-smaxage : (or/c false/c (and/c exact-integer? positive?))
   = #f
  cache-stale-while-revalidate : (or/c false/c (and/c exact-integer? positive?))
   = #f
  cache-stale-if-error : (or/c false/c (and/c exact-integer? positive?))
   = #f
  cache-no-cache : boolean? = #f
  cache-no-store : boolean? = #f
  cache-no-transform : boolean? = #f
  cache-must-revalidate : boolean? = #f
  cache-proxy-revalidate : boolean? = #f
  cache-must-understand : boolean? = #f
  cache-private : boolean? = #f
  cache-public : boolean? = #f
  cache-immutable : boolean? = #f
使用 url->path 从请求对象中的 URL 中提取一个路径。 如果这个路径不存在,那么该调度器就不适用,并调用 next-dispatcher 。 如果路径是一个目录,那么就会检查 indices ,以寻找一个可以服务的索引文件。 在这种情况下,或者在路径已经是一个文件的情况下, path->mime-type 会被查询到路径的 MIME 类型。 同样地, path->headers 被用来查询路径的附加头文件。 然后,该文件被流向连接对象。

这个调度器支持 HTTP 范围的 GET 请求和 HEAD 请求。 如果请求的方法既不是 HEAD 也不是 GET,那么将调用 next-dispatcher 。

如果路径指向磁盘上的东西(要么是一个文件,要么,如果路径指向目录, 则是该目录中指定的 indices 文件之一),它需要被运行 Web 服务器的进程所读取。 已经存在但不可读的文件被当作不存在的文件处理。

以 cache- 开头的各种关键字参数 (cache-public, cache-max-age 等等) 都可以直接映射到标准 Cache-Control 响应头中的合法值。 默认情况下,所有的值都是 #f, 这样做的结果是这个调度器发出的响应没有 Cache-Control 头。 如果任何与缓存相关的关键字有一个非 #f 的值,响应中就会有一个 Cache-Control 头。 因此,如果 cache-immutable 是 #t ， cache-max-age 是 12345, 那么所有响应中都会有一个 Cache-Control ,其值是 max-age=12345, immutable 。 更多信息请参见 RFC 2616 section 14.9 “Cache Control” 和 Mozilla Developer Network documentation on Cache-Control 。 注意,某些缓存头的组合可能会导致非预期的行为。 例如:在 #:cache-public 和 #:cache-private 中同时使用 #t (这两个实际上是反义词)。 除了每个关键字参数的合同外, make 不会做额外的检查, 以确保提供的缓存相关参数是一个有意义的组合或适合您的网络应用。

Changed in version 1.7 of package web-server-lib: 支持非 {GET,HEAD} 请求。
Changed in version 1.7: 将不可读的文件视为不存在的文件。
Changed in version 1.9: 支持一些设置 Cache-Control 响应头的选项
Changed in version 1.10: 支持 #:path->headers 。
