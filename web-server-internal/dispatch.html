2.1  概述

 (require web-server/dispatchers/dispatch)
  package: web-server-lib
这个模块为一般的调度器提供了一些功能。

value

dispatcher/c : contract?

相当于 (-> connection? request? any) 。

Changed in version 1.3 of package web-server-lib: 削弱了范围契约,允许 any

procedure

(dispatcher-interface-version/c any) → boolean?

  any : any/c
相当于 (symbols 'v1)

struct

(struct exn:dispatcher ()
    #:extra-constructor-name make-exn:dispatcher)
抛出一个异常,表示一个调度器不适用于某个特定的请求。

procedure

(next-dispatcher) → any

引发 exn:dispatcher

正如 dispatcher/c 契约所建议的,dispatcher 是一个函数,它接收一个连接和请求对象并对它们做一些事情。 最有可能的是,它将生成一些响应并在连接上输出,但它也可能做一些不同的事情。 例如,它可能会对请求对象进行一些测试,也许是检查一个有效的源IP地址, 如果测试没有通过就会出错,否则就调用 next-dispatcher 。

考虑一下下面这个例子,它抓住了 URL 重写的本质:
; (url? -> url?) dispatcher/c -> dispatcher/c
(lambda (rule inner)
  (lambda (conn req)
    ; Call the inner dispatcher...
    (inner conn
           ; with a new request object...
           (struct-copy request req
                        ; with a new URL!
                        [request-uri (rule (request-uri req))]))))
