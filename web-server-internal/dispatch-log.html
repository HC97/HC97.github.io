2.10 基本日志

 (require web-server/dispatchers/dispatch-log)
  package: web-server-lib
web-server/dispatchers/dispatch-log 模型 定义了一个用于透明记录请求的调度器构造函数。 考虑使用 记录 中的设施来代替,因为它提供了更多的灵活性。

value

format-req/c : contract?

相当于 (-> request? string?) 。

value

paren-format : format-req/c

形成一个类似于 记录 中对应的请求,但没有响应代码信息。

value

extended-format : format-req/c

形成一个类似于 记录 中对应的请求,但没有响应代码信息。

value

apache-default-format : format-req/c

像 Apache 的默认格式一样来处理请求。 然而,Apache 的默认值包括请求的响应信息,这个函数不能访问这些信息, 所以它把最后两个字段默认为 - 和 - 。

value

log-format/c : contract?

相当于 (symbols 'parenthesized-default 'extended 'apache-default) 。

procedure

(log-format->format id) → format-req/c

  id : log-format/c
将 'parenthesized-default 映射到 paren-format, 'extended 映射到 extended-format, 'apache-default 映射到 apache-default-format 。

procedure

(make [#:format format #:log-path log-path]) → dispatcher/c

  format : (or/c log-format/c format-req/c) = paren-format
  log-path : (or/c path-string? output-port?) = "log"
将请求记录到 log-path, log-path 可以是文件路径,也可以是 output-port?, 使用 format 来格式化这些请求。 如果 format 是一个符号,那么将使用 log-format->format 来默许一个日志格式化。 然后调用 next-dispatcher 。

Changed in version 1.3 of package web-server-lib: 允许 log-path 是一个 output-port?
Changed in version 1.8: 允许 format 是一个符号(更确切地说,是一个 log-format/c)。
