4 网络服务器

网络服务器是一个具有 web-server^ 签名的单元。 最常见的构造方式是为 web-server@ 单元提供一个 web-config^ 单元。 构建 web-config^ 单元最常见的方法是使用 configuration-table->web-config@ , 从配置表文件中产生一个,比如 Racket 在 default-configuration-table-path 中提供的文件。

4.1 服务器单元

 (require web-server/web-server-sig)
  package: web-server-lib

signature

web-server^ : signature

procedure

(serve [#:confirmation-channel confirmation-ach]) → (-> any)

  confirmation-ach : 
(or/c #f (async-channel/c
          (or/c exn? port-number?)))
 = #f
与来自 dispatch-server^ 的 serve 相同。 dispatch-server^ 签名是 web-server^ 的别名。

procedure

(serve-ports ip op) → any

  ip : input-port?
  op : output-port?
异步地服务于由端口 ip 和 op 代表的单个连接。

 (require web-server/web-server-unit)
  package: web-server-lib

value

web-server-with-connect@

 : 
(unit/c (web-config*^ tcp^ dispatch-server-connect^)
        (web-server^))
使用 web-config*^ 构建一个 dispatcher/c 函数, 为 web-config*^ 中的每个虚拟主机设置一个虚拟主机调度器,对以下操作进行排序:

  • 将传入的请求以给定的格式记录到给定的文件中。

  • 用给定的密码文件执行 HTTP 基本认证

  • 允许 "/conf/refresh-passwords" URL 来刷新密码文件。

  • 允许 "/conf/collect-garbage" URL 调用垃圾收集器。

  • 允许 "/conf/refresh-servlets" URL 刷新 servlets 缓存。

  • 执行 servlets 映射 URL 到 htdocs 下给定的 servlet 根目录。

  • 服务于给定 htdocs 目录下的 "/" URL 下的文件。

使用这个 dispatcher/c, 它加载了一个提供 serve 和 serve-ports 功能的 dispatching 服务器,并按预期操作。

Added in version 1.1 of package web-server-lib.
Changed in version 1.6: 使用 web-config*^ 而不是 web-config^ 。 见 compatability note 。

value

web-server@ : 
(unit/c (web-config*^ tcp^)
        (web-server^))
与 web-server-with-connect@ 类似,但使用 raw:dispatch-server-connect@ 。

Changed in version 1.6 of package web-server-lib: 使用 web-config*^ 而不是 web-config^ 。 见 compatability note 。

Changed in version 1.6 of package web-server-lib: 为了与 make-safety-limits 保持一致,放宽了合同。

4.3 配置表

 (require web-server/configuration/configuration-table)
  package: web-server-lib
本模块提供读取、写入、解析和打印 configuration-table 结构的功能。

value

default-configuration-table-path : path?

默认的配置表 S 表达式文件。

value

configuration-table-sexpr? : (any . -> . boolean?)

相当于 list? 。

procedure

(sexpr->configuration-table sexpr) → configuration-table?

  sexpr : configuration-table-sexpr?
此函数将一个 configuration-table 从一个 S 表达式中转换出来。

procedure

(configuration-table->sexpr ctable)

 → configuration-table-sexpr?
  ctable : configuration-table?
该函数将 configuration-table 转换为 S 表达式。

配置表的格式是:
`((port ,integer?)
  (max-waiting ,exact-integer?)
  (initial-connection-timeout ,integer?)
  (default-host-table
    ,host-table-sexpr?)
  (virtual-host-table
   (list ,symbol? ,host-table-sexpr?)
   ...))
其中 host-table-sexpr 是:
`(host-table
  (default-indices ,string? ...)
  (log-format ,symbol?)
  (messages
   (servlet-message ,path-string?)
   (authentication-message ,path-string?)
   (servlets-refreshed ,path-string?)
   (passwords-refreshed ,path-string?)
   (file-not-found-message ,path-string?)
   (protocol-message ,path-string?)
   (collect-garbage ,path-string?))
  (timeouts
   (default-servlet-timeout ,integer?)
   (password-connection-timeout ,integer?)
   (servlet-connection-timeout ,integer?)
   (file-per-byte-connection-timeout ,integer?)
   (file-base-connection-timeout ,integer))
  (paths
   (configuration-root ,path-string?)
   (host-root ,path-string?)
   (log-file-path ,path-string?)
   (file-root ,path-string?)
   (servlet-root ,path-string?)
   (mime-types ,path-string?)
   (password-authentication ,path-string?)))
在这种语法中, 'messages 路径是相对于 'configuration-root 目录。 除了 'servlet-root 之外, 'paths 中的所有路径都是相对于 'host-root 的(显然不包括 'host-root 。) 'servlet-root 的路径是相对于 'file-root 。
允许的 'log-format 是那些由 log-format->format 接受的。
注意:你几乎总是想把 'paths 部分的所有内容都保留为默认值,除了 'host-root 。

procedure

(read-configuration-table path) → configuration-table?

  path : path-string?
这个函数从 path 中读取一个 configuration-table 。

procedure

(write-configuration-table ctable path) → void

  ctable : configuration-table?
  path : path-string?
此函数将 configuration-table 写入 path 。

4.5 标准应答器

 (require web-server/configuration/responders)
  package: web-server-lib
本模块提供了一些有助于构建 HTTP 响应器的函数。 这些函数被默认的调度器构造器使用(见 服务器单元) , 将 configuration-table 中给出的路径转化为相关情况的响应器。

procedure

(file-response http-code    
  short-version    
  text-file    
  header ...) → response?
  http-code : natural-number/c
  short-version : string?
  text-file : string?
  header : header?
生成一个 response? ,将给定的 http-code 和 short-version 作为相应的字段; 将 text-file 的内容作为正文;以及将 header 作为,你猜对了,头文件。

这不会导致重定向到一个著名的 URL,如 "conf/not-found.html" , 而是使用 "not-found.html" (例如)的内容作为其内容。 因此, text-file 中的任何相对 URL 都是相对于用来响应的任何 URL file-response 。 因此,你可能应该在这些文件中使用绝对的 URL。

procedure

(servlet-loading-responder url exn) → response?

  url : url?
  exn : exn?
将 exn 交给 current-error-handler ,并以堆栈跟踪和 "Servlet 没有加载" 的消息进行响应。

procedure

(gen-servlet-not-found file) → ((url url?) . -> . response?)

  file : path-string?
返回一个函数,生成一个标准的 "Servlet not found"。 错误,内容来自 file 。

procedure

(servlet-error-responder url exn) → response?

  url : url?
  exn : exn?
将 exn 交给 current-error-handler ,并以堆栈跟踪和 "Servlet 错误" 消息作为回应。

procedure

(gen-servlet-responder file)

 → ((url url?) (exn any/c) . -> . response?)
  file : path-string?
将 exn 打印到标准输出,并以 "Servlet error" 。 消息和 file 中的内容进行回应。

procedure

(gen-servlets-refreshed file) → (-> response?)

  file : path-string?
返回一个函数,生成一个标准的 "Servlet cache refreshed"。 消息,内容来自 file 。

procedure

(gen-passwords-refreshed file) → (-> response?)

  file : path-string?
返回一个函数,生成一个标准的 "Passwords refreshed" 。 消息,并包含 file 中的内容。

procedure

(gen-authentication-responder file)

 → ((url url?) (header header?) . -> . response?)
  file : path-string?
返回一个函数,该函数使用 file 和 header 中的内容作为 HTTP 标头生成身份验证失败错误。

procedure

(gen-protocol-responder file) → ((url url?) . -> . response?)

  file : path-string?
返回一个函数,该函数使用 file 中的内容生成 "Malformed request" 错误。

procedure

(gen-file-not-found-responder file)

 → ((req request?) . -> . response?)
  file : path-string?
返回一个函数,该函数使用 file 中的内容生成标准的 "File not found" 错误。

procedure

(gen-collect-garbage-responder file) → (-> response?)

  file : path-string?
返回一个函数,该函数使用 file 中的内容生成标准的 "Garbage collection run" 。
