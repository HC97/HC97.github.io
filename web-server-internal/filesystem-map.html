2.2 将 URL 映射到路径上

 (require web-server/dispatchers/filesystem-map)
  package: web-server-lib
这个模块提供了一种将 URL 映射到文件系统上的路径的方法。

value

url->path/c : contract?

这个契约等同于 (->* (url?) (path? (listof path-piece?))) 。 返回的 path? 是磁盘上的路径。 列表是对应于 URL 的路径的路径元素的列表。

procedure

(make-url->path base) → url->path/c

  base : path-string?
这个过程返回的 url->path/c 认为根 URL 是 base 。 它确保 URL 中的 ".." 不会转义为 base ,否则会默默地将其删除。

procedure

(make-url->valid-path url->path) → url->path/c

  url->path : url->path/c
运行底层的 url->path, 但只在路径指向一个实际存在的文件时返回。 如果不存在,那么 URL 的后缀元素将被删除,直到找到一个文件。 如果这从未发生,那么就会抛出一个错误。

这主要适用于允许在服务名称后面的路径信息用于数据的调度器,但服务是由一个文件代表的。 最突出的例子显然是servlets。

procedure

(filter-url->path regex url->path) → url->path/c

  regex : regexp?
  url->path : url->path/c
运行底层的 url->path ,但只有当路径被认为是一个字符串时,才会返回与 regex 匹配。 在使用 servlet 调度器时,这对禁止奇怪的文件(如 GIF)被认为是 servlets 很有用。 如果路径不匹配,它将返回 exn:fail:filesystem:exists? 异常。
