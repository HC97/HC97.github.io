5.2 连接管理器

 (require web-server/private/connection-manager)
  package: web-server-lib
这个模块提供了管理成对的输入和输出端口的功能。 我们有计划允许一些不同的策略来做这件事。

struct

(struct connection (timer i-port o-port custodian close?)
    #:extra-constructor-name make-connection)
  timer : timer?
  i-port : input-port?
  o-port : output-port?
  custodian : custodian?
  close? : boolean?
一个连接是一对端口 (i-port 和 o-port) , 如果 close? 是 #t 那么在当前工作结束后就可以关闭。 与该连接相关的资源应该在 custodian 下分配。 该连接将持续到 timer 触发。

使用 new-connection 构建 connection 实例,它与 connection managers 合作。 通过其他方式(如 make-connection 或 struct-copy) 构建连接可能会产生不良后果, 例如规避 safety limits 。

Changed in version 1.6 of package web-server-lib: 除了通过 new-connection 之外,取消其他的构造。

procedure

(connection-manager? v) → boolean?

  v : any/c

procedure

(start-connection-manager [#:safety-limits safety-limits])

 → connection-manager?
  safety-limits : safety-limits?
   = (make-unlimited-safety-limits)
connection manager 是一个不透明的值, 由谓词 connection-manager? 识别, 它与 new-connection 合作,控制 connection 实例的创建和行为。 它封装了一个定时器管理器(见 计时器), safety limits 策略和其他内部数据。 使用 start-connection-manager 来创建一个 connection manager 。

请注意,如果没有给出 safety-limits 参数, 默认的 safety limits 值对恶意或行为不当的客户端和 servlets 提供了最小的保护: 参见 make-unlimited-safety-limits 。 大多数程序不应该直接使用 start-connection-manager 或 new-connection : 更高级别的接口,如 dispatch-server-with-connect@ 和 serve, 包含了连接管理,默认提供了更多的保护。 start-connection-manager 的允许性默认 safety limits 使直接使用这些函数的低级程序的向后兼容能力最大化。 更多信息请参见 safety limits compatability note 。

Changed in version 1.6 of package web-server-lib: 增加了 safety-limits 参数。

procedure

(new-connection cm i-port o-port cust close?) → connection?

  cm : connection-manager?
  i-port : input-port?
  o-port : output-port?
  cust : custodian?
  close? : boolean?
(new-connection cm    
  timeout    
  i-port    
  o-port    
  cust    
  close?) → connection?
  cm : connection-manager?
  timeout : number?
  i-port : input-port?
  o-port : output-port?
  cust : custodian?
  close? : boolean?
与 connection manager cm 合作,构建一个新的 connection 实例。 该连接是通过一个有效调用 kill-connection! 的定时器创建的。 初始超时是由 cm 中封装的 safety limits 决定的。

为了向后兼容,我们提供了带有 timeout 参数的六参数形式。 在这种情况下, timeout 被用于连接的定时器的初始值, 与 cm 的 safety limits 无关。 safety limits 的其他方面仍然适用于产生的连接。

Changed in version 1.6 of package web-server-lib: 增加了使用 cm 的 safety limits 而不是 timeout 参数的五参数形式。

procedure

(kill-connection! c) → any

  c : connection?
关闭与 c 相关的端口,杀死定时器,并关闭监管者。

procedure

(adjust-connection-timeout! c t) → any

  c : connection?
  t : number?
用 c 后面的定时器和 t 调用 increment-timer! 。

procedure

(reset-connection-timeout! c t) → any

  c : connection?
  t : number?
用 c 后面的定时器和 t 调用 reset-timer! 。

Added in version 1.6 of package web-server-lib.
