13 Windows 注册表

 (require file/resource) package: base

procedure

(get-resource section 
  entry 
  [value-box 
  file 
  #:type type]) 
 → (or/c #f string? bytes? exact-integer? #t)
  section : string?
  entry : string?
  value-box : (or/c #f (box/c (or/c string? bytes? exact-integer?)))
   = #f
  file : (or/c #f path-string?) = #f
  type : (or/c 'string 'string/utf-16 'bytes 'bytes* 'integer)
   = derived-from-value-box
从 Windows 注册表或 ".ini" 文件中获取一个值。 为了向后兼容,对于 Windows 以外的平台,其结果是 #f 。 当 file 为 #f 且 section 为 "HKEY_CLASSES_ROOT", "HKEY_CURRENT_CONFIG", "HKEY_CURRENT_USER", "HKEY_LOCAL_MACHINE" 或 "HKEY_USERS" 时,注册表被 read 。 当 file 为 #f 且 section 不是特殊注册表字符串之一时, 那么 (build-path (find-system-path 'home-dir) "mred.ini") 就是 read 。

资源值是以 section 和 entry 的组合为关键的。 如果没有找到指定 section 和 entry 的值,则结果为 #f 。 如果 value-box 是一个盒子,那么如果找到了一个值,结果是 #t ,盒子里就会填上这个值; 当 value-box 是 #f, 时,结果就是找到的值。

任何格式的注册表值都可以被提取出来。 type 参数和资源类型的组合决定了资源最初如何被转换为 Racket 值:

  • REG_SZ 注册表值的字节首先通过无尾的 UTF-16 解释转换为字符串(不包括字符串中的终止符)– 除非 type 是 'bytes*, 在这种情况下,字节被保持为字节字符串。

  • REG_DWORD 注册表值的字节首先被解释为 32 位有符号的整数, 然后用 number->string 将整数转换为一个字符串。

  • 任何其他类型的寄存器值的字节都被保留为一个字节字符串。

这种最初的转换产生一个字符串或一个字节字符串。 然后请求的 type 可能会触发一个额外的转换:

  • 'string: 字符串被保持原样,但是字节串被转换为字符串,使用 bytes->string/utf-8 。 注意,UTF-8 转换 不 适合某些资源类型,例如 REG_EXPAND_SZ; 使用 'string/utf-16 代替。

  • 'string/utf-16: 字符串保持原样, 但字节串通过无尾的 UTF-16 解释转换为字符串(省略字符串中的无尾结尾)。

  • 'bytes: 字节串保持原样,但使用 string->bytes/utf-8 进行转换。 注意,这种转换不会产生 REG_SZ 资源的原始字节; 使用 'bytes* 代替,因为这避免了向字符串的初始转换。

  • 'bytes*: 与 'bytes 相同, 但 'bytes* 会影响 REG_SZ 资源的初始转换。

  • 'integer: 使用 string->number 将字符串转换为数字, 用 bytes->string/utf-8 与 string->number 合成一个字节字符串,进行转换。

如果 value-box 是一个盒子,那么默认 type 是由最初的盒子内容派生的: 'string, 'bytes 或 'integer 。 否则,默认 type 是 'string 。

来自 ".ini" 文件的资源总是字符串,并像 REG_SZ 注册表值一样被转换。

要获得一个注册表项的 "默认" 值,使用尾部反斜杠。 例如,下面的表达式得到了一个用于启动浏览器的命令行:

(get-resource "HKEY_CLASSES_ROOT"
              "htmlfile\\shell\\open\\command\\")

Changed in version 8.0.0.10 of package base: 为 type 添加了 'sting/utf-16 和 'bytes* 选项。

向Windows注册表或 ".ini" 文件写入一个值。 为了向后兼容,对于 Windows 以外的平台,其结果是 #f 。 当 file 为 #f 且 section 为 "HKEY_CLASSES_ROOT", "HKEY_CURRENT_CONFIG", "HKEY_CURRENT_USER", "HKEY_LOCAL_MACHINE" 或 "HKEY_USERS" 时,注册表被 written 。 当 file 为 #f 且 section 不是特殊注册表字符串之一时, 那么 (build-path (find-system-path 'home-dir) "mred.ini") 就是 written 。

资源值是以 section 和 entry 的组合为关键的。 如果写入注册表时 create-key? 为 false,资源条目必须已经存在,否则写入失败。 如果写入失败,结果是 #f ,如果成功是 #t 。

type 参数决定了写到注册表的值的格式以及它对字节的转换:

  • 'string: 写为 REG_SZ, 其中字符串 value 被转换为 UTF-16 字节, 加上一个 nul 终结符。 一个字节的字符串 value 首先用 bytes->string/utf-8 转换, 一个整数 value 首先用 number->string 转换,然后每种情况下的结果都被当作字符串处理。 请注意, 'string 不太可能成为一个字节字符串 value 的有用转换; 请使用 'bytes/string 。

  • 'expand-string: 像 'string, 但写成 REG_EXPAND_SZ 。 请注意, 'expand-string 不太可能成为字节字符串 value 的有效转换; 请使用 'bytes/expand-string 。

  • 'bytes: REG_BINARY, 其中字节字符串 value 按原样写入, 字符串 value 通过 string->bytes/utf-8转换为字节, 整数 value 通过 number->string 与 string->bytes/utf-8 组合转换为字节。

  • 'bytes/string: 以 REG_SZ 的方式写入, 其中字节字符串 value 按原样写入 (与 'string 不同,所以字节字符串必须是 UTF-16 编码,并加上一个空结尾), 字符串 value 被转换为 UTF-16 字节,并加上一个空结尾, 整数 value 通过 number->string 转换为字符串, 然后转换为 UTF-16 字节,并加上一个空结尾。

  • 'bytes/expand-string: 与 'bytes/string 类似, 但以 REG_EXPAND_SZ 的方式写入。

  • 'dword: 以 REG_DWORD 的方式写入, 其中整数 value 被转换为 32 位有符号的整数字节, 字符串 value 通过 string->number 转换,然后与整数相同, 字节字符串 value 通过 bytes->string/utf-8 与 string->number 合成转换, 然后与整数相同。

当写入 ".ini" 文件时,格式总是一个字符串,与 type 无关。

Changed in version 8.0.0.10 of package base: 为 type 添加了 'expand-string, 'bytes/string 和 'bytes/expand-string 选项。
