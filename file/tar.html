6 tar 文件的创建

 (require file/tar) package: base
file/tar 库提供了创建 USTA 格式的归档文件的工具, 就像 Unix 工具 pax 生成的归档文件。 使用 POSIX.1-2001/pax 或 GNU 格式的长路径都支持长路径。 产生的归档文件只包含目录、文件和符号链接,所有者信息不被保留;存储在归档文件中的所有者始终是 "root"。
符号链接(在 Unix 和 Mac OS 上)默认不被遵循。

procedure

(tar tar-file 
  path-or-entry ... 
  [#:follow-links? follow-links? 
  #:exists-ok? exists-ok? 
  #:format format 
  #:path-prefix path-prefix 
  #:path-filter path-filter 
  #:timestamp timestamp 
  #:get-timestamp get-timestamp]) 
 → exact-nonnegative-integer?
  tar-file : path-string?
  path-or-entry : (or/c path-string? tar-entry?)
  follow-links? : any/c = #f
  exists-ok? : any/c = #f
  format : (or/c 'pax 'gnu 'ustar) = 'pax
  path-prefix : (or/c #f path-string?) = #f
  path-filter : (or/c #f (path? . -> . any/c)) = #f
  timestamp : (or/c #f exact-integer?) = #f
  get-timestamp : (path? . -> . exact-integer?)
   = 
(if timestamp
    (lambda (p) timestamp)
    file-or-directory-modify-seconds)
创建 tar-file,它保存了所有 path-or-entry 的完整内容。 每个 path-or-entry 要么是指文件系统上的一个文件、目录或链接的路径, 要么是描述这种实体的 tar-entry ,而不要求它在文件系统上存在。

path-or-entry 之间的给定路径都被期望为现有目录和文件的相对路径 (即,相对于当前目录的 path-or-entry 是一个路径)。 如果在 path-or-entry 中提供了一个嵌套路径,它的祖先目录也会被添加到生成的 tar 文件中, 直到当前目录(使用 pathlist-closure) 。 如果 follow-links? 是 false,那么符号链接就会作为链接包含在结果的 tar 文件中。

如果 exists-ok? 是 #f, 那么如果 tar-file 已经存在,就会引发一个异常。 如果 exists-ok? 是真的,那么如果 tar-file 已经存在,就会被截断或替换。

format 参数决定了对长路径和长符号链接目标的处理。 如果 format 是 'pax, 那么就使用 POSIX.1-2001/pax 扩展。 如果 format 是 'gnu, 则使用 GNU 扩展。 如果 format 是 'ustar, 那么 tar 会对过长的路径或符号链接目标引发错误。

如果 path-prefix 不是 #f, 那么它就会在归档文件中的每个路径前缀上。

get-timestamp 函数用来获取每个文件或目录的修改日期,并记录在存档中。

像 tar 一样,但把给定的 paths-and-entries 中的每个元素打包成 tar 格式的档案, 直接写到 out 。 指定的 paths-and-entries 按原样包含(除了添加 path-prefix, 如果有的话); 如果指定了一个目录,其内容不会被自动添加,嵌套的目录被添加,没有父目录。

struct

(struct tar-entry (kind path content size attribs)
    #:extra-constructor-name make-tar-entry)
  kind : (or/c 'file 'directory 'link)
  path : (and/c path-string? relative-path?)
  content : (or/c input-port? (-> input-port?) #f path-string?)
  size : exact-nonnegative-integer?
  attribs : (hash/c symbol? any/c)
代表一个文件、目录或链接,将被包含在 USTAR 文件或流中。

如果 kind 是 'file, 那么 content 必须是一个输入端口或产生一个输入端口的 thunk, 并且它必须提供精确的 size 字节。 如果 kind 是 'directory, 那么 content 和 size 都应该是 #f 和 0 。 如果 kind 是 'link, 那么 content 必须是一个路径, 并且 size 应该是 0 。

attribs 字段包含一个哈希表,提供该条目的额外属性。 目前在写一个 USTAR 文件或流时,使用以下键:

  • 'permissions — 一个整数,代表读取、写入和执行权限, 其形式由 file-or-directory-permissions 接受。

  • 'modify-seconds — 一个代表修改时间的整数, 与 file-or-directory-modify-seconds 一致。

  • 'owner — 一个精确的整数,表示文件所有者的 ID。

  • 'owner-bytes — 一个代表文件所有者名称的字节字符串。

  • 'group — 一个确切的整数,表示文件组的 ID。

  • 'group-bytes — 代表文件组名称的一个字节字符串。

Added in version 8.1.0.5 of package base.
