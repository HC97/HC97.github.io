11 GIF 文件的写入

 (require file/gif) package: draw-lib

file/gif 库提供了向流中写入 GIF 文件的功能,包括带有多个图像和控件的 GIF 文件(如动画 GIF)。

一个 GIF 流由 gif-start 创建,然后用 gif-add-image 写入单个图像。 另外, gif-add-control 插入了渲染图像的指令。 gif-end 函数结束 GIF 流。

一个 GIF 流可以处于以下任何一种状态:

  • 'init : 没有图像或控件被添加到流中

  • 'image-or-control : 可以写入另一个图像或控件

  • 'image : 可以写入另一个图像(但不是控件,因为已经写入了一个控件)

  • 'done : 不能再添加任何东西

procedure

(gif-stream? v) → boolean?

  v : any/c
如果 v 是由 gif-write 创建的 GIF 流,返回 #t ,否则返回 #f 。

procedure

(image-ready-gif-stream? v) → boolean?

  v : any/c
如果 v 是一个没有处于 'done 模式的 GIF 流,则返回 #t ,否则返回 #f 。

procedure

(image-or-control-ready-gif-stream? v) → boolean?

  v : any/c
如果 v 是一个处于 'init 或 'image-or-control 模式的 GIF 流, 则返回 #t ,否则返回 #f 。

procedure

(empty-gif-stream? v) → boolean?

  v : any/c
如果 v 是一个处于 'init 模式的 GIF 流,则返回 #t ,否则返回 #f 。

procedure

(gif-colormap? v) → boolean?

  v : any/c
如果 v 重置了一个颜色映射,则返回 #t ,否则返回 #f 。 颜色映射是一个列表,其大小是 21 和 28 之间的 2 次方, 其元素是大小为 3 的向量,包含颜色(即 0 和 255 之间的精确整数)。

procedure

(color? v) → boolean?

  v : any/c
与 byte? 相同。

procedure

(dimension? v) → boolean?

  v : any/c
如果 v 是 0 到 65535 之间的精确整数, 返回 #t ,否则返回 #f 。

procedure

(gif-state stream) → symbol?

  stream : gif-stream?
返回 stream 的状态。

procedure

(gif-start out w h bg-color cmap) → gif-stream?

  out : output-port?
  w : dimension?
  h : dimension?
  bg-color : color?
  cmap : (or/c gif-colormap? #f)
将一个 GIF 文件的开始写入给定的输出端口,并返回一个加到输出端口的 GIF 流。

宽度和高度决定了整个 GIF 图像的一个虚拟空间。 添加到 GIF 流中的单个图像必须适合这个虚拟空间。 该空间由给定的背景颜色初始化。

最后,颜色数字的默认含义(如背景色)由给定的颜色映射决定,但 GIF 文件中的单个图像可以有自己的颜色映射。

不需要提供全局颜色映射,在这种情况下,必须为每个图像提供一个颜色映射。 请注意,如果不提供全局颜色映射, bg-color 的定义就不明确。

procedure

(gif-add-image stream    
  left    
  top    
  width    
  height    
  interlaced?    
  cmap    
  bstr) → void?
  stream : image-ready-gif-stream?
  left : dimension?
  top : dimension?
  width : dimension?
  height : dimension?
  interlaced? : any/c
  cmap : (or/c gif-colormap? #f)
  bstr : bytes?
在给定的 GIF 流中写入一个图像。 left, top, width 和 height 的值指定了图像在整个 GIF 图像的虚拟空间中的位置和大小。

如果 interlaced? 为 true,那么 bstr 应该提供字节的交错顺序, 而不是从上到下的顺序。隔行扫描的顺序是:

  • 每8行,从0开始

  • 每8行,从4开始

  • 每4行,从2开始

  • 每2行,从1开始

如果用 gif-start 提供了一个全局颜色,可以为 cmap 提供一个 #f 。

bstr 参数指定了图像的像素内容。每个字节指定一种颜色(即颜色映射中的一个索引)。 每一行都是从左到右提供的,各行可以从上到下或以交错的顺序提供(见上文)。 如果图像的前缀是一个指定了透明索引的控件(见 gif-add-control), 那么相应的 "颜色" 就不会绘制到整个 GIF 图像中。

如果 bstr 中的任何字节值大于 colormap 的长度, 如果 bstr 的长度不是 width 乘以 height, 或者如果 top, left, width 和 height 的尺寸指定了一个超出整个 GIF 图像虚拟空间的区域,就会产生一个异常。

procedure

(gif-add-control stream    
  disposal    
  wait-for-input?    
  delay    
  transparent) → void?
  stream : image-or-control-ready-gif-stream?
  disposal : (or/c 'any 'keep 'restore-bg 'restore-prev)
  wait-for-input? : any/c
  delay : dimension?
  transparent : (or/c color? #f)
将图像控制命令写入 GIF 流中。这样的控制必须出现在一个图像之前,并且适用于下面的图像。

GIF 图像模型包括逐个处理图像,将每个图像放置在整个图像的虚拟空间中的指定位置。 图像控制命令可以指定在添加图像之前的延迟时间(用于创建 GIF 动画), 它还可以指定在进入下一个图像之前,应如何从整个图像中保留或移除该图像(也用于 GIF 动画)。

disposal 参数指定如何进行:

  • 'any : 无所谓(也许是因为下一个图像会完全覆盖当前图像)

  • 'keep : 将图像留在原处

  • 'restore-bg : 用背景颜色替换图像

  • 'restore-prev : 将整个图像内容恢复到添加图像之前的内容。

如果 wait-for-input? 为真, 那么显示程序可能会在添加图像之前等待来自用户的一些提示(也许是鼠标点击)。

delay 参数指定了一个以 1/100 秒为单位的延迟。

如果 transparent 参数是一种颜色,那么它决定了一个索引, 用来表示后续图像中的透明像素(与索引的颜色映射所指定的颜色相反)。

如果一个控件已经被添加到 stream 中而没有相应的图像,则会产生一个异常。

procedure

(gif-add-loop-control stream iteration) → void?

  stream : empty-gif-stream?
  iteration : dimension?
向没有图像或其他命令的 GIF 流写入一个控制命令。 该命令导致 GIF 中图像的动画序列被重复’迭代-维度’次数,其中 0 可以用来表示 "无限"。

如果某些控件或图像已经被添加到流中,则会产生一个异常。

procedure

(gif-add-comment stream bstr) → void?

  stream : image-or-control-ready-gif-stream?
  bstr : bytes?
在 GIF 流中添加一个通用注释。

如果一个图像控制命令刚刚被写入流中(所以接下来需要一个图像),就会出现异常。

procedure

(gif-end stream) → void?

  stream : image-or-control-ready-gif-stream?
完成写一个 GIF 文件。GIF 流的输出端口不会被自动关闭。

如果一个图像控制命令刚刚被写入流中(因此接下来需要一个图像),则会出现一个异常。

procedure

(quantize bstr) → 
bytes? gif-colormap? (or/c color? #f)
  bstr : 
(and/c bytes?
       (lambda (bstr)
         (zero? (remainder (bytes-length bstr) 4))))
GIF流中的每个图像都被限制在 256 种颜色,包括透明的 "颜色",如果有的话。 quantize 功能将 24 位图像(加上阿尔法通道)转换为索引色图像,必要时减少颜色的数量。

给出一组以 ARGB 格式表示的像素(即每四个字节是一个像素的一组值:阿尔法、红、蓝、绿), quantize 产生的是

  • 图像的字节(即一个颜色数组,以字节字符串的形式表示)

  • 一个颜色映射

  • 一个 #f 或者一个透明 "颜色" 的颜色索引

该转换将小于 128 的 alpha 值视为透明像素,而将其他 alpha 值视为实体。

量化过程使用 Octrees [Gervautz1990] 为图像中的所有(非透明)颜色构建一个自适应调色板。 这个实现是基于 Dean Clark [Clark1996] 的一篇文章。

要转换所有具有相同量化程度的图像集合, 只需将它们追加到 quantize 的一次调用的输入中,然后将结果字节分解。
