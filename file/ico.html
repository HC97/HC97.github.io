12 ICO 文件的读取和写入

 (require file/ico) package: base

file/ico 库提供了读写 ".ico" 文件的功能,这些文件包含一个或多个图标。 每个图标最大为 256×256 像素,有一个特定的深度(即用于表示颜色的每个像素的位数)和掩码 (即是否显示一个像素,但对于每个像素有一个阿尔法值的 32 位图标,掩码可以被忽略)。 该库还提供了对 Windows 可执行文件中图标的读写支持。

procedure

(ico? v) → boolean?

  v : any/c
如果 v 代表一个图标,返回 #t ,否则返回 #f 。

procedure

(ico-width ico) → exact-positive-integer?

  ico : ico?

procedure

(ico-height ico) → exact-positive-integer?

  ico : ico?

procedure

(ico-depth ico) → (one-of/c 1 2 4 8 16 24 32)

  ico : ico?
返回图标的宽度或高度,单位是像素,或深度,单位是每像素的比特。

Changed in version 6.3 of package base: 一个 PNG 格式的图标可以有一个大于 256 的宽度或高度。

procedure

(ico-format ico) → (or/c 'bmp 'png)

  ico : ico?
报告图标的格式。

Added in version 6.3 of package base.

procedure

(read-icos src) → (listof ico?)

  src : (or/c path-string? input-port?)
将 src 解析为 ".ico" ,以提取一个图标列表。

procedure

(read-icos-from-exe src) → (listof ico?)

  src : (or/c path-string? input-port?)
将 src 解析为 ".exe" ,以提取代表 Windows 可执行文件的图标列表。

procedure

(write-icos icos dest [#:exists exists]) → void?

  icos : (listof ico?)
  dest : (or/c path-string? output-port?)
  exists : 
(or/c 'error 'append 'update 'can-update
      'replace 'truncate
      'must-truncate 'truncate/replace)
 = 'error
将 icos 中的每个图标作为一个 ".ico" 文件写入 dest 。 如果 dest 不是一个输出端口, 则会将 exists 传递给 open-output-file ,以打开 dest 进行写入。

procedure

(replace-icos icos dest) → void?

  icos : (listof ico?)
  dest : path-string?
将 icos 中的图标写成 Windows 可执行文件,以取代 dest 中的图标。 只有在可执行文件中现有的图标尺寸和深度被替换,而且只有在编码尺寸匹配的情况下。 现有尺寸和深度的最佳匹配从 icos 中提取(根据需要调整最佳匹配的比例和深度)。

使用 replace-all-icos, 而不是全盘替换一组图标,特别是当这组图标包括 PNG 格式的图标时。

procedure

(replace-all-icos icos dest) → void?

  icos : (listof ico?)
  dest : (or/c path-string? output-port?)
用给定的图标集替换可执行文件 dest 中的图标集。

procedure

(ico->argb ico) → bytes?

  ico : ico?
将 BMP 格式的图标(见 ico-format)转换为 ARGB 字节字符串, 其中图标的像素按从左到右、从上到下的顺序排列,每个像素有四个字节(alpha、红、绿、蓝通道)。

procedure

(ico->png-bytes ico) → bytes?

  ico : ico?
返回 PNG 格式的图标的 PNG 编码的字节数(见 ico-format)。

Added in version 6.3 of package base.

procedure

(argb->ico width height bstr [#:depth depth]) → ico?

  width : (integer-in 1 256)
  height : (integer-in 1 256)
  bstr : bytes?
  depth : (one-of/c 1 2 4 8 24 32) = 32
将一个 ARGB 字节字符串(与 ico->argb 的格式相同)转换为 BMP 格式中给定宽度、高度和深度的图标。

bstr 参数的长度必须是 (* 4 width height), 而 (* width depth) 必须是 8 的倍数。

procedure

(png-bytes->ico bstr) → ico?

  bstr : bytes?
将给定的 PNG 编码包装成一个 PNG 编码的图标。

Added in version 6.3 of package base.
