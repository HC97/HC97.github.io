7 tar 文件提取

 (require file/untar) package: base
file/untar 库提供了一个从 TAR/USTAR 档案中提取项目的功能, 使用 GNU 和/或 pax 扩展来支持长路径名。

procedure

(untar in    
  [#:dest dest-path    
  #:strip-count strip-count    
  #:permissive? permissive?    
  #:filter filter-proc    
  #:handle-entry handle-entry]) → void?

  dest-path : (or/c path-string? #f) = #f
  strip-count : exact-nonnegative-integer? = 0
  permissive? : any/c = #f
  filter-proc : 
(path? (or/c path? #f)
 symbol? exact-integer? (or/c path? #f)
 exact-nonnegative-integer?
 exact-nonnegative-integer?
 . -> . any/c)
   = (lambda args #t)
  handle-entry : 
((or/c 'file 'directory 'link)
 (and path? relative-path?)
 (or/c input-port? #f path?)
 exact-nonnegative-integer?
 (hash/c symbol? any/c)
 . -> . (listof (-> any)))

从 in 中提取 TAR/USTAR 内容,识别 POSIX.1-2001/pax 和 GNU 扩展的长路径和长符号链接目标。

如果 dest-path 不是 #f,存档中的每条路径都会有前缀,以确定被提取项目的目标路径。

如果 strip-count 是正数, 那么 strip-count 路径元素将从归档的项目路径中移除(在用 dest-path 对路径进行前缀之前); 如果项目的路径包含 strip-count 元素,那么它就不会被提取。

除非 permissive? 为真,否则包含上层目录指示器的归档项目被禁止, 而目标为绝对路径或包含上层目录指示器的链接项目也被禁止。 绝对路径总是被禁止的。一个不允许的路径会触发一个异常。

对于档案中的每个项目, filter-proc 被应用于

  • 该项目在档案中出现的路径;

  • 一个目标路径,基于存档中的路径、 strip-count 和 dest-path– 如果项目的路径没有 strip-count 或更多元素,可以是 #f ;

  • 代表项目类型的符号—'file, 'dir, 'link, 'hard-link, 'character-special, 'block-special, 'fifo, 'contiguous-file, 'extended-header, 'extended-header-for-next 或 'unknown—其中只有 'file, 'dir 或 'link 可以通过 untar 解包;

  • 一个精确的整数,代表项目的大小;

  • 一个 'link 类型的目标路径或任何其他类型的 #f ;

  • 一个整数,代表项目的修改日期;以及

  • 一个代表该项目的权限的整数。

如果 filter-proc 的结果是 #f, 那么该项目就不会被解包。

handle-entry 函数被调用来解压一个条目, handle-entry 的默认 handle-tar-entry 函数在文件系统上创建一个目录、文件或链接。 handle-entry 函数必须接受五个参数:

  • kind — 'file, 'directory 或 'link 中的一个。

  • path — 记录在 TAR 文件中的相对路径。

  • content — 一个输入端口,为 'file 条目提供内容, 在 handle-entry 返回之前,必须从该端口读取精确的 size 字节。 对于一个 'directory条目, content 是 #f 。 对于一个 'link 条目, content 是链接目标的一个路径。

  • size — 一个 'file 条目的字节数,其他条目为 0 。

  • attribs — 一个不可变的哈希表,将符号映射到属性值。 可用的键可能会改变,但目前包含的键与 tar-entry 中识别的键是一样的。

handle-entry 的结果是一个 thunks 列表,这些 thunks 在 TAR 输入被完全解包后被依次调用。 handle-entry 的结果 thunk 是有用的,例如,在所有文件都被写入目录后,设置目录的修改时间。

Changed in version 6.3 of package base: 增加了 #:permissive? 参数。
Changed in version 6.7.0.4: 支持使用 POSIX.1-2001/pax 和 GNU 扩展的长路径和长符号链接目标。
Changed in version 8.1.0.5: 增加了 #:handle-entry 参数。

procedure

(handle-tar-entry kind    
  path    
  content    
  size    
  attribs) → (listof (-> any))
  kind : (or/c 'file 'directory 'link)
  path : (and path? relative-path?)
  content : (or/c input-port? #f path?)
  size : exact-nonnegative-integer?
  attribs : (hash/c symbol? any/c)
作为 untar 的默认入口处理程序, handle-tar-entry 创建目录和文件,并返回一个 thunks 列表,通过设置目录权限和修改时间完成解包。

Added in version 8.1.0.5 of package base.
