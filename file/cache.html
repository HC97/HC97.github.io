14 缓存

 (require file/cache) package: base
file/cache 库提供了管理本地文件缓存的实用程序,例如下载的文件。 缓存对于跨进程的并发使用是安全的,因为它使用文件系统锁,并且它将客户与文件系统故障隔离开来。

procedure

(cache-file dest-file    
  [#:exists-ok? exists-ok?]    
  key    
  cache-dir    
  fetch    
  [#:notify-cache-use notify-cache-use    
  #:max-cache-files max-files    
  #:max-cache-size max-size    
  #:evict-before? evict-before?    
  #:log-error-string log-error-string    
  #:log-debug-string log-debug-string]) → void?
  dest-file : path-string?
  exists-ok? : any/c = #f
  key : (not/c #f)
  cache-dir : path-string?
  fetch : (-> any)
  notify-cache-use : (string? . -> . any) = void
  max-files : real? = 1024
  max-size : real? = (* 64 1024 1024)
  evict-before? : (hash? hash? . -> . boolean?)
   = 
(lambda (a b)
  (< (hash-ref a 'modify-seconds)
     (hash-ref b 'modify-seconds)))
  log-error-string : (string? . -> . any)
   = (lambda (s) (log-error s))
  log-debug-string : (string? . -> . any)
   = (lambda (s) (log-debug s))
在 cache-dir 中寻找以前用 key 缓存的文件,如果找到了缓存的文件, 就把它复制到 dest-file (必须不存在,除非 exists-ok? 为真)。 否则,就调用 fetch ;如果调用 fetch 后 dest-file 存在, 它就被复制到 cache-dir ,并与 key 一起记录。 当一个缓存条目被使用时, notify-cache-use 被调用,并给出缓存文件的名称。

当一个新的文件被缓存时, max-files (作为一个文件计数)和 max-size 以字节为单位)决定是否有任何先前缓存的文件应该被从缓存中驱逐。 如果是这样, evict-before? 确定现有缓存条目的驱逐顺序; evict-before? 的每个参数是一个哈希表,至少有以下几个键:

  • 'modify-seconds — 文件的修改日期

  • 'size — 文件的大小,以字节为单位

  • 'key — 缓存条目的密钥

  • 'name — 缓存文件的名称

log-error-string 和 log-debug-string 函数用于记录错误和调试信息。

procedure

(cache-remove key    
  cache-dir    
  [#:log-error-string log-error-string    
  #:log-debug-string log-debug-string]) → void?
  key : any/c
  cache-dir : path-string?
  log-error-string : (string? . -> . any)
   = (lambda (s) (log-error s))
  log-debug-string : (string? . -> . any)
   = (lambda (s) (log-debug s))
从 cache-dir 的缓存中删除与 key 匹配的缓存条目(如果有的话), 如果 key 是 #f ,则删除所有缓存文件。

log-error-string 和 log-debug-string 函数用来记录错误和调试信息。
