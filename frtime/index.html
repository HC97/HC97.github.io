FrTime: 一种反应式程序的语言

Greg Cooper


吴华均 译

 #lang frtime package: frtime

frtime 语言支持声明式地构建反应式系统,其语法与 Racket 的语法非常相似。 它扩展了 racket 。

在 DrRacket 中,作为使用 #lang frtime 的替代方案, 你可以从 Choose Language 菜单中选择 FrTime 。

本参考文件描述了语言 ; 所提供的功能,关于设计和语义模型的细节,请参考所附的研究论文:

  • Brown 2008 - Cooper. Integrating dataflow evaluation into a practical higher-order call-by-value language

  • ESOP 2006 — Cooper and Krishnamurthi. Embedding Dynamic Dataflow in a Call-by-Value Language

  • Brown 2004 - Cooper and Krishnamurthi. FrTime: Functional Reactive Programming in PLT Scheme

1 基元

value

undefined : any/c

代表一个未定义的值。

procedure

(undefined? val) → boolean?

  val : any/c
如果 val 是 undefined,则返回 #t 。

procedure

(behavior? val) → boolean?

  val : any/c
如果 val 是一个行为(一个时间变化的值,其当前值可以在任何时候被预测), 则返回 #t 。

procedure

(event? val) → boolean?

  val : any/c
如果 val 是一个事件(一个可以在任意时间发生的时间变化的值流), 则返回 #t 。

procedure

(signal? val) → boolean?

  val : any/c
如果 val 是一个信号,则返回 #t 。 (signal? v) 等同于 (or (behavior? v) (event? v)) 。

value

seconds : behavior?

大约每秒钟更新一次 (current-seconds) 的值。

value

milliseconds : behavior?

频繁更新,值为 (current-inexact-milliseconds) 。

value

never-e : event?

是一个从未发生的事件。

2 定义自定义输入信号

procedure

(new-cell [init-expr]) → signal?

  init-expr : signal? = undefined
返回一个信号, 其值最初跟踪 init-expr 的值,但可以通过 set-cell! 重新连接到一个不同的信号。

procedure

(set-cell! cell val) → void?

  cell : signal?
  val : signal?
重组 cell (必须由 new-cell 创建),以获得 val 的值。

procedure

(event-receiver) → event?

返回一个事件流,可以通过 send-event 强制触发。

procedure

(send-event rcvr val) → void?

  rcvr : event?
  val : any/c
在 rcvr (必须由event-receiver创建)上发出 val 。

3 信号处理程序

procedure

(value-now val) → any/c

  val : any/c
投射一个行为或常量的当前值。

procedure

(delay-by val duration) → behavior?

  val : behavior?
  duration : number?
通过 duration 毫秒延迟 val 。

procedure

(integral val) → behavior?

  val : (or/c number? behavior?)
计算 val 相对于时间(以毫秒为单位)的积分的数字近似值。

procedure

(derivative val) → behavior?

  val : behavior?
计算 val 相对于时间的导数的数字近似值。

procedure

(map-e proc ev) → event?

  proc : (-> any/c any)
  ev : event?

procedure

(==> ev proc) → event?

  ev : event?
  proc : (-> any/c any)
返回一个事件流,每当 ev 触发时就会触发,其值通过应用 proc 进行转换。

procedure

(filter-e pred ev) → event?

  pred : (-> any/c boolean?)
  ev : event?

procedure

(=#> ev pred) → event?

  ev : event?
  pred : (-> any/c boolean?)
返回一个事件流,只通过 pred 返回 #t 的 ev 。

procedure

(merge-e ev ...) → event?

  ev : event?
将所有输入的事件源合并为一个事件源。

procedure

(once-e ev) → event?

  ev : event?
返回一个事件源,它只携带 ev 的第一次出现(其余的被过滤掉)。

procedure

(changes val) → event?

  val : behavior?
返回一个事件源,该事件源在每次参数行为改变时发生。 出现的值是该行为的新值。

procedure

(hold ev [init]) → behavior?

  ev : event?
  init : any/c = undefined
构造一个从 init 开始的行为,然后接受由 ev 产生的最后一个值

procedure

(switch ev [init]) → behavior?

  ev : event?
  init : behavior? = undefined
返回一个以 init 开始的行为。 每当 ev 产生一个(可能是时间变化的)值时,该行为就切换到该值。

procedure

(accum-e ev init) → event?

  ev : event?
  init : any/c
通过累积初始值上的变化 (由给定的事件源携带)来构建一个事件源。

procedure

(accum-b ev init) → behavior?

  ev : event?
  init : any/c
结合 accum-e 和 hold 的功能来构造一个行为。 (accum-b ev init) 等同于 (hold init (accum-e ev init)) 。

procedure

(collect-e ev init proc) → event?

  ev : event?
  init : any/c
  proc : 
(-> any/c any/c
any)
与 accum-e 相似,只是转换函数是固定的,并应用于事件发生和当前的累积器(按顺序)。

procedure

(collect-b ev init proc) → behavior?

  ev : event?
  init : any/c
  proc : (-> any/c any/c any)
与 collect-e 相似,就像 accum-b 与 accum-e 相似一样。

procedure

(when-e val) → event?

  val : behavior?
返回一个事件流,每次 val 从 #f 变成其他东西时,都会携带一个事件发生。

procedure

(lift-strict proc val ...) → any

  proc : (-> [arg any/c] ... any)
  val : any/c
提供一种机制,将普通的 Racket 原语应用到行为中。 如果任何一个 val 是行为,则返回一个行为,其当前值总是等于 (proc (value-now arg) ...) 。 在 FrTime 中,许多 Racket 原语都是隐含地被提升的。

下面的形式允许导入基本 FrTime 语言中没有包含的被解除的程序。

(require (lifted module-spec proc-name ...) ...)
(require (lifted:nonstrict module-spec proc-name ...) ...)

5 图形化演示程序

要运行下列动画/GUI 演示程序,只需将语言级别设置为 FrTime,打开相应的文件,然后执行。更多信息请看演示的源代码。

"orbit-mouse.rkt" : 一个围绕鼠标指针转圈移动的球的集合。

"piston.rkt" : 模拟一个活塞/汽缸。

"rotation.rkt" : 以圆圈移动的球。

"delay-mouse.rkt" : 一个跟随鼠标移动的球的轨迹。

"ball-on-string.rkt" : 一个追赶鼠标的球。

"pong.rkt" : 一个简单的乒乓/曲棍球游戏。左边的球拍用数字键盘移动;右边的球拍用鼠标移动。 ’r’键可以重置分数。

"pizza.rkt" : 一个基于 HtDP 练习的简单 "比萨饼订购" 用户界面。

"calculator.rkt" : 一个简单的计算器界面,也是基于 HtDP 练习,只是结果随着参数和运算符的变化而连续更新。

接下来的三个动画例子是由 Robb Cutler 提供的:

"analog-clock.rkt" : 一个实时的动画时钟。一个滑块可以调整钟面的半径。点击并拖动来移动钟面。

"growing-points.rkt" : 一个随着鼠标接近而增长的点的区域。

"needles.rkt" : 一个指向鼠标的针的区域。
