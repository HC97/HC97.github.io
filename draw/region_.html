class

region% : class?

  superclass: object%

region% 对象指定了一个绘图区域的一部分(可能是不连续的)。 它通常用于剪裁绘图操作。

当区域被创建时,一个 region% 对象可以与一个特定的 dc<%> 对象关联。 在这种情况下,该区域使用绘图上下文的当前变换矩阵、平移、缩放和旋转,与安装该区域时的变换无关。 否则,当区域被安装到 dc<%> 中时,会像往常一样进行转换。 对于一个自动滚动的画布,画布的当前滚动总是在使用区域时适用(并且不影响区域的边界框)。

区域与 union 等操作的组合是近似的,它们是通过组合路径实现的。 某些组合只有在路径具有合适的填充模式时才能工作, 这种模式可以是 'winding, 'even-odd 或 flexible fill 模式。 当一个区域被安装为设备上下文的剪切区域时,如果任何其他路径使用 'even-odd 模式, 那么任何具有 flexible fill 模式的子路径都会使用 'even-odd 模式。

参见 set-clipping-region in dc<%> 和 get-clipping-region in dc<%> 。

constructor

(new region% [dc dc]) → (is-a?/c region%)

  dc : (or/c (is-a?/c dc<%>) #f)
创建一个空区域。 如果 dc 是 dc<%> 对象, dc<%> 的当前变换矩阵基本上被记录在该区域中。

method

(send a-region get-bounding-box) → 
real? real? real? real?
返回一个近似包围该区域的矩形。 返回值是该矩形的左、上、宽和高。 如果该区域有一个相关的绘图上下文,那么边界框就在该绘图上下文的当前逻辑坐标中。

method

(send a-region get-dc) → (or/c (is-a?/c dc<%>) #f)

如果创建了区域的绘图上下文,则返回该区域的绘图上下文。

method

(send a-region in-region? x y) → boolean?

  x : real?
  y : real?
如果给定的点大约在该区域内,返回 #t ,否则返回 #f 。 如果该区域有一个相关的绘图上下文,给定的点将根据该区域的 dc<%> 的当前变换矩阵进行有效的变换。

method

(send a-region intersect rgn) → void?

  rgn : (is-a?/c region%)
将区域设置为自身与给定区域的交点。

rgn 和 a-region 的绘图上下文必须是相同的,或者它们必须都没有关联到任何绘图上下文。

一个相交点对应于与这个区域的路径相交,然后与给定区域的路径相交。 进一步向该区域发送的组合对应于在初始剪辑前与原始路径的组合,而以该区域为参数的进一步组合对应于在初始剪辑后与给定路径的组合。 因此,一个相交的区域对于 union, subtract 或 xor 来说是一个糟糕的输入,但是在进一步调用 intersect 时,它可以正常相交。

method

(send a-region is-empty?) → boolean?

如果该区域近似为空,则返回 #t ,否则返回 #f ,但只有当该区域与一个绘图上下文相关联时才会返回。 如果该区域没有与任何绘图上下文相关联,将引发 exn:fail:contract 。

method

(send a-region subtract rgn) → void?

  rgn : (is-a?/c region%)
将区域设置为自身减去给定区域的减法。 换句话说,如果一个点被包含在给定的区域中,那么它就会从区域中被移除。 (给定区域可能包含不在当前区域中的点,这些点会被忽略)。

这个区域的绘图上下文和给定区域的绘图上下文必须是相同的,或者它们必须都不与任何绘图上下文相关联。

这个结果在不同的平台和设备上是一致的,但它绝不是一个真正的减法。 减法对应于将该区域的子路径与给定区域的反向子路径相结合,然后将结果与该区域相交。 这不能作为一个真正的减法,因为循环的边界(有 'odd-even 或 'winding 填充)是不明确的。

method

(send a-region union rgn) → void?

  rgn : (is-a?/c region%)
将区域设置为自身与给定区域的联合。

这个区域的绘图上下文和给定区域的绘图上下文必须是相同的,或者它们必须都不与任何绘图上下文相关联。

联合对应于将每个区域的子路径组合成一条路径, 如果其中一个区域使用 'odd-even 填充(否则使用 'winding 填充), 则使用 'odd-even 填充, 任何一个区域中的 'winding 填充使用 winding 填充, 或者如果两条路径都有 flexible fill ,则填充仍然是 flexible fill . 因此,虽然结果在不同的平台和设备上是一致的,但它只对某些输入区域是一个真正的联合。 例如,对于不重叠的 atomic 和联合区域,它是一个真正的联合。 对于所有顺时针且使用 'winding 填充的 atomic 和联合区域(可能是重叠的), 或者如果填充都是 flexible fills ,它也是一个真正的联合。

method

(send a-region xor rgn) → void?

  rgn : (is-a?/c region%)
将区域设置为自身与给定区域的 xoring (即包含的点正好被两个区域中的一个所包围)。

这个区域的绘图上下文和给定区域的绘图上下文必须是相同的,或者它们必须都没有关联到任何绘图上下文。

这个结果在不同的平台和设备上是一致的,但它不一定是一个真正的 xoring。 xoring 对应于将该区域的子路径与给定区域的反转子路径相结合。 如果任何一个区域使用 'odd-even 填充,则结果使用 'odd-even 填充; 如果任何一个区域使用 'winding 填充,则结果使用 winding 填充; 如果两个路径都有 flexible fill ,则结果仍然是 flexible fill 。 因此,该结果只对某些输入区域是可靠的 xoring。 例如,对于 atomic 和全部使用 'even-odd 填充的 xoring 区域,它是可靠的。
