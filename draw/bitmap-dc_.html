class

bitmap-dc% : class?

  superclass: object%

  extends: dc<%>
一个 bitmap-dc% 对象允许直接在一个位图上绘图。 必须在初始化时提供一个 bitmap% 对象,或者在调用 bitmap DC 的任何其他方法之前, 使用 set-bitmap 安装到该 DC 中, 除了 get-text-extent, get-char-height 或 get-char-width 。 如果在选择一个位图之前调用任何其他 bitmap-dc% 方法,该方法调用将被忽略。

用彩色位图在 bitmap-dc% 上绘图,保证产生与在 canvas% 实例上绘图相同的结果(有适当的剪裁和偏移)。 因此, bitmap-dc% 可以用于画布内容的屏幕外分期。

constructor

(new bitmap-dc% [bitmap bitmap]) → (is-a?/c bitmap-dc%)

  bitmap : (or/c (is-a?/c bitmap%) #f)
创建一个新的位图 DC。 如果 bitmap 不是 #f, 它将被安装到DC中,这样 DC 上的绘图命令就会绘制到 bitmap 上。 否则,没有位图被安装到 DC 中,在调用 DC 的任何其他方法之前必须调用 set-bitmap 。

method

(send a-bitmap-dc get-argb-pixels x    
  y    
  width    
  height    
  pixels    
  [just-alpha?    
  pre-multiplied?]) → void?
  x : exact-nonnegative-integer?
  y : exact-nonnegative-integer?
  width : exact-nonnegative-integer?
  height : exact-nonnegative-integer?
  pixels : (and/c bytes? (not/c immutable?))
  just-alpha? : any/c = #f
  pre-multiplied? : any/c = #f
获取位图中的一个矩形像素,受制于 get-pixel 的相同规则和性能特征,只是块状获取可能比单个获取序列更快。 另外, bitmap% 类也直接提供了同样的方法,所以没有必要将一个位图选入 DC 来提取其像素值。

像素的 RGB 值和阿尔法值被复制到 pixels 中(如果 just-alpha? 为真,则只复制阿尔法值)。 第一个字节代表 (x, y)处的像素的 alpha 值, 第二个字节代表 (x, y) 处的像素的红色值,第三个字节是绿色值,等等。 这样一来,第一个 width * height * 4 个字节的 pixels 被设置为反映 DC 中当前的像素值。 像素是以行为主的顺序,从左到右然后从上到下。

如果位图有一个 alpha 通道,那么每个像素的 alpha 值总是以 pixels 为单位设置。 如果 just-alpha? 是假的,并且位图没有 alpha 通道,那么每个像素的 alpha 值被设置为 255。 如果 just-alpha? 为真,那么每个像素的 alpha 值 只 被设置; 如果位图没有 alpha 通道,那么 alpha 值是基于每个像素的反转RGB平均值。 因此,当一个位图有一个单独的遮罩位图时,相同的 pixels 字节串一般是由两个位图填充的: 一个(主图)用于像素值,一个(遮罩)用于 alpha 值。

如果 pre-multiplied? 是 true, just-alpha? 是 false, 并且位图有一个 alpha 通道,那么结果中的 RGB 值会被相应的 alpha 值缩放(即,乘以 alpha 值,然后除以 255)。

如果位图的 backing scale 不是 1.0, get-argb-pixels 的结果就像位图被绘制到一个背景比例为 1.0 的位图上一样, 并返回目标位图的像素。

method

(send a-bitmap-dc get-bitmap) → (or/c (is-a?/c bitmap%) #f)

获取当前安装在DC中的位图,如果没有安装位图,则获取 #f 。参见 set-bitmap 获取更多信息。

method

(send a-bitmap-dc set-argb-pixels x    
  y    
  width    
  height    
  pixels    
  [just-alpha?    
  pre-multiplied?]) → void?
  x : real?
  y : real?
  width : exact-nonnegative-integer?
  height : exact-nonnegative-integer?
  pixels : bytes?
  just-alpha? : any/c = #f
  pre-multiplied? : any/c = #f
在位图中设置一个像素的矩形,除非 DC 的当前位图是由 make-screen-bitmap 或 make-bitmap in canvas% 产生的(在这种情况下, an exn:fail:contract exception is raised)。

pixels 的 RGB 值取自像素。 第一个字节代表一个阿尔法值,第二个字节代表一个红色值,用于 (x, y) 的像素,第三个字节是一个蓝色值,等等。 这样,第一个 width * height * 4 字节的 pixels 决定了 DC 中新的像素值。 像素是以行为主的顺序,从左到右然后从上到下。

如果 just-alpha? 是假的,那么只有当 DC 的当前位图有一个 alpha 通道时,才会使用每个像素的 alpha 值。 如果 just-alpha? 为真,并且位图有一个 alpha 通道,那么位图不会被修改。 如果 just-alpha? 为真,且位图没有 alpha 通道,那么每个像素 只 根据 alpha 值进行设置,但要反过来作为一个掩码。 因此,当处理有相关掩码位图而不是阿尔法通道的位图时,相同的 pixels 字节串被用于两个位图: 一个(主图像)用于像素值,一个(掩码)用于阿尔法值。

如果 pre-multiplied? 是 true, just-alpha? 是 false,并且位图有一个 alpha 通道, 那么 pixels 中的 RGB 值被解释为按相应的 alpha 值缩放(即乘以 alpha 值,然后除以 255)。 如果一个 R、G 或 B 值大于其相应的阿尔法值(如果该值被适当地缩放,这是不可能的),那么它实际上被减少到阿尔法值。

如果位图有一个 1.0 以外的 backing scale , 那么 pixels 会被衬底比例有效地缩放,以获得安装到位图的像素值。

method

(send a-bitmap-dc set-bitmap bitmap) → void?

  bitmap : (or/c (is-a?/c bitmap%) #f)
将一个位图安装到 DC 中,这样在位图 DC 上的绘图操作就会绘制到位图上。 通过将位图设置为 #f 来从 DC 中移除位图。

一个位图最多可以被选入一个位图 DC,而且只有在它不被控件使用(作为标签)或在 pen% 或 brush% 中使用(作为点阵)的情况下。 如果 set-bitmap 的参数已经被另一个 DC、一个控件、一个 pen% 或一个 brush% 使用, an exn:fail:contract exception is raised 。

method

(send a-bitmap-dc set-pixel x y color) → void?

  x : real?
  y : real?
  color : (is-a?/c color%)
设置位图中的一个像素。

当前的剪裁区域可能不会影响像素的变化。 在 X 下,将绘图命令与 set-pixel 调用交错在一起 (对于同一个 bitmap-dc% 对象)会产生很大的性能损失, 除了对 get-pixel, get-argb-pixels 和 set-argb-pixels.
