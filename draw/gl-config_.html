class

gl-config% : class?

  superclass: object%

gl-config% 对象封装了一个 OpenGL 绘图环境的配置信息。 使用 gl-config% 对象作为 canvas% 的初始化参数,或将其提供给 make-gl-bitmap 。

constructor

(new gl-config%) → (is-a?/c gl-config%)

创建一个 GL 配置,表示传统的 OpenGL、双缓冲、一个大小为 1 的深度缓冲、没有模板缓冲、没有累积缓冲、没有多采样和没有立体声。

method

(send a-gl-config get-accum-size) → (integer-in 0 256)

报告配置要求的累积缓冲区大小(红、绿、蓝、阿尔法中的每一个),其中0表示不要求累积缓冲区。

method

(send a-gl-config get-depth-size) → (integer-in 0 256)

报告配置要求的深度缓冲区大小,零意味着不要求深度缓冲区。

method

(send a-gl-config get-double-buffered) → boolean?

报告配置是否要求双重缓冲。

method

(send a-gl-config get-hires-mode) → boolean?

确定是否使用 hires 模式。在 Mac OS 上,租用模式意味着创建的 OpenGL 上下文可以访问完整的 Retina 分辨率,不会被绘图系统缩放 。在其他平台上,租用模式没有影响。

Added in version 1.5 of package draw-lib.

method

(send a-gl-config get-legacy?) → boolean?

决定是使用传统的 "Compatibility" OpenGL 还是 "Core" OpenGL。 Core OpenGL 配置文件目前在 Mac OS(10.7及以上版本)和 Linux (如果图形驱动支持的话)上被支持。

Added in version 1.2 of package draw-lib.

method

(send a-gl-config get-multisample-size) → (integer-in 0 256)

报告配置要求的多采样大小,其中 0 表示不要求多采样。

method

(send a-gl-config get-share-context)

 → (or/c #f (is-a?/c gl-context<%>))
返回一个 gl-context<%> 对象,该对象与新创建的 OpenGL 绘图上下文共享某些对象(纹理、显示列表等), 或者 #f 是没有设置的。

另请参见 set-share-context 。

method

(send a-gl-config get-stencil-size) → (integer-in 0 256)

报告配置要求的钢网缓冲区大小,其中 0 表示不要求钢网缓冲区。

method

(send a-gl-config get-stereo) → boolean?

报告配置是否要求立体声。

method

(send a-gl-config get-sync-swap) → boolean?

报告配置是否请求与屏幕刷新同步的缓冲区交换。

Added in version 1.10 of package draw-lib.

method

(send a-gl-config set-accum-size on?) → void?

  on? : (integer-in 0 256)
调整配置以请求每个通道(红、绿、蓝和阿尔法)的特定累积缓冲区大小,其中零意味着不请求累积缓冲区。

method

(send a-gl-config set-depth-size on?) → void?

  on? : (integer-in 0 256)
调整配置以请求一个特定的深度缓冲区大小,其中零意味着不请求深度缓冲区。

method

(send a-gl-config set-double-buffered on?) → void?

  on? : any/c
调整配置以请求双重缓冲或不缓冲。

method

(send a-gl-config set-hires-mode hires-mode) → void?

  hires-mode : any/c
调整配置以请求雇佣模式或不雇佣模式;见 get-hires-mode 。

Added in version 1.5 of package draw-lib.

method

(send a-gl-config set-legacy? legacy?) → void?

  legacy? : any/c
调整配置是否请求传统模式;见 get-legacy? 。

Added in version 1.2 of package draw-lib.

method

(send a-gl-config set-multisample-size on?) → void?

  on? : (integer-in 0 256)
调整配置以请求一个特定的多采样尺寸,其中零意味着不请求多采样。 如果多采样上下文不可用,这个请求将被忽略。

method

(send a-gl-config set-share-context context) → void?

  context : (or/c #f (is-a?/c gl-context<%>))
确定一个 gl-context<%> 对象,与新创建的 OpenGL 绘图上下文共享某些对象(纹理、显示列表等), 其中 #f 表示不应发生共享。

当一个上下文 B 与上下文 A 共享对象时,它也会与其他与 A 共享的上下文共享对象,反之亦然。

如果一个 OpenGL 实现不支持共享,那么在创建一个新的上下文时, context 实际上是被忽略的。 所有版本的 Mac OS 都应该支持共享。 在 Windows 和 Linux 上,共享是由 WGL_ARB_create_context 和 GLX_ARB_create_context 扩展的存在分别提供的(而 OpenGL 3.2要求这两个扩展)。

method

(send a-gl-config set-stencil-size on?) → void?

  on? : (integer-in 0 256)
调整配置以请求一个特定的网板缓冲区大小,其中 0 意味着不请求网板缓冲区。

method

(send a-gl-config set-stereo on?) → void?

  on? : any/c
调整配置以要求立体声或不要求立体声。

method

(send a-gl-config set-sync-swap on?) → void?

  on? : any/c
调整配置以请求与屏幕刷新同步的缓冲区交换或不交换。

Added in version 1.10 of package draw-lib.
