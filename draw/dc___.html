interface

dc<%> : interface?

dc<%> 对象是一个用于绘制图形和文本的绘图环境。 它以一种通用的方式表示输出设备;例如,画布有一个绘图上下文,打印机也有。

method

(send a-dc cache-font-metrics-key) → exact-integer?

返回一个整数,如果不是 0,则对应于一种特定的设备和缩放系数, 这样文本范围信息(来自 get-text-extent, get-char-height 等)是相同的。 这个键在所有 dc<%> 实例中都是有效的,甚至在不同的类中也是如此。

0 的结果表示 a-dc 的当前配置不符合一个共同的类别,因此没有键可以用来缓存文本范围信息。

method

(send a-dc clear) → void?

清除绘图区域(用当前的背景色填充,由 get-background 决定)。也请参见 erase 。

method

(send a-dc copy x y width height x2 y2) → void?

  x : real?
  y : real?
  width : (and/c real? (not/c negative?))
  height : (and/c real? (not/c negative?))
  x2 : real?
  y2 : real?
将由绘图环境的 x, y,width 和 height 定义的矩形复制到同一绘图环境的 x2 和 y2 指定的位置。 源区域和目的区域可以重叠。

Changed in version 1.12 of package draw-lib: 允许源区和目的区重叠。

method

(send a-dc draw-arc x    
  y    
  width    
  height    
  start-radians    
  end-radians) → void?
  x : real?
  y : real?
  width : (and/c real? (not/c negative?))
  height : (and/c real? (not/c negative?))
  start-radians : real?
  end-radians : real?
绘制一个逆时针方向的圆弧,即在 x (左), y (上), width 和 height 指定的矩形中刻画的椭圆的一部分。 弧线从 start-radians 指定的角度开始( 0 是 3 点钟方向, half-pi 是 12 点钟方向), 逆时针继续到 end-radians 。 如果 start-radians 和 end-radians 相同,就会画一个完整的椭圆。

当前画笔用于画弧。 如果当前画笔不是透明的,它将被用来填充由弧线和延伸到内嵌椭圆中心的线条(未画出)所围成的楔形。 如果画笔和笔刷都是不透明的,那么在用画笔画出弧线之前,楔形会被画笔填充。

楔形和弧形相遇时,它们之间不留任何空间,但楔形和弧形之间的精确重叠是根据平台和大小而定的。 通常情况下,画笔和钢笔绘制的区域是重叠的。在非平滑或对齐模式下,轮廓的路径是通过缩小边界椭圆的宽度和高度来调整的, 在缩放后,一个绘图单位除以对齐比例。

method

(send a-dc draw-bitmap source    
  dest-x    
  dest-y    
  [style    
  color    
  mask]) → boolean?
  source : (is-a?/c bitmap%)
  dest-x : real?
  dest-y : real?
  style : (or/c 'solid 'opaque 'xor) = 'solid
  color : (is-a?/c color%)
   = (send the-color-database find-color "black")
  mask : (or/c (is-a?/c bitmap%) #f) = #f
显示 source 位图。 dest-x 和 dest-y 参数是 DC 坐标。

对于彩色位图,绘图样式和颜色参数被忽略。 对于单色位图, draw-bitmap 使用样式和颜色参数的方式与画笔使用其样式和颜色设置来绘制单色点阵一样 (更多信息见 brush% )。

如果提供了一个 mask 位图,它必须具有与 source 位图相同的宽度和高度, 并且它的 ok? 必须返回 true,否则 an exn:fail:contract exception is raised 。 source 位图和 mask 位图可以是同一个对象,但如果绘图上下文是一个 bitmap-dc% 对象,这两个位图必须与目标位图不同,否则 an exn:fail:contract exception is raised 。

mask 对绘图的影响取决于 mask 位图的类型:

  • 如果 mask 位图是单色的,在目标 dc<%> 中只在遮罩位图包含黑色像素的地方发生绘制 (与 style 无关,它控制如何处理单色 source 的白色像素)。

  • 如果 mask 位图是带有阿尔法通道的彩色位图,它的阿尔法通道被用作绘制 source 的遮罩, 而它的颜色通道被忽略掉。

  • 如果 mask 位图是没有 alpha 通道的彩色位图,给定像素的颜色分量将被平均化以得出该像素的反 alpha 值。 特别是,如果 mask 位图是灰度的,那么每个遮罩像素的黑度就控制了所画像素的不透明度(即,遮罩作为一个倒置的 alpha 通道)。

DC 的当前笔刷、当前笔和当前文本对位图的绘制方式没有影响,但如果 DC 有刻度,位图就会被缩放, DC 的 alpha 设置决定了绘制的像素的不透明度(与 source 的 alpha 通道、任何给定的 mask 以及 source 为单色时 color 的 alpha 分量结合)。

对于 post-script-dc% 和 pdf-dc% 的输出,来自 source 的 alpha 通道、 来自 mask 或来自 color 的不透明度被舍入为全透明或不透明。

如果位图被成功绘制,结果是 #t ,否则是 #f (可能是因为位图的 ok? 方法返回 #f)。

参见 draw-bitmap-section 。

method

(send a-dc draw-ellipse x y width height) → void?

  x : real?
  y : real?
  width : (and/c real? (not/c negative?))
  height : (and/c real? (not/c negative?))
绘制一个包含在矩形中的椭圆,该矩形具有给定的左上角和尺寸。 当前的笔用于画出轮廓,当前的笔刷用于填充形状。 如果笔和刷子都是不透明的,在用笔画出轮廓之前,椭圆是用刷子填充的。

画笔填充和钢笔轮廓相遇,使它们之间不留空间,但填充和轮廓之间的精确重叠是由平台和大小决定的。 因此,画笔和钢笔绘制的区域可能部分重叠。 在非平滑或对齐模式下,轮廓的路径是通过缩放后,将椭圆的宽度和高度缩小一个绘图单位除以 alignment scale 来调整的。

method

(send a-dc draw-line x1 y1 x2 y2) → void?

  x1 : real?
  y1 : real?
  x2 : real?
  y2 : real?
画一条从一个点到另一个点的线。 当前的笔被用来画线。

在非平滑模式下,点对应于像素,线覆盖起点和终点。 对于比例宽度大于 1 的笔,线会被画在起点和终点的中心。

关于 'aligned 平滑模式的信息,也请参见 set-smoothing 。

method

(send a-dc draw-lines points    
  [xoffset    
  yoffset]) → void?
  points : 
(or/c (listof (is-a?/c point%))
      (listof (cons/c real? real?)))
  xoffset : real? = 0
  yoffset : real? = 0
使用一个 points 的列表点来画线,给每个点添加 xoffset 和 yoffset , 并使用笔的连接样式来连接一条线和共享点的下一条线。 一对点被视为一个点,其中一对点的 car 是 x 值, cdr 是 y 值。当前的笔被用来画线。

关于 'aligned 平滑模式的信息,也请参见 set-smoothing 。

method

(send a-dc draw-path path    
  [xoffset    
  yoffset    
  fill-style]) → void?
  path : (is-a?/c dc-path%)
  xoffset : real? = 0
  yoffset : real? = 0
  fill-style : (or/c 'odd-even 'winding) = 'odd-even
绘制给定的 dc-path% 对象的子路径,为每个点添加 xoffset 和 yoffset 。 (关于路径和子路径的一般信息见 dc-path% )。 当前的笔用于将路径画成线,而当前的笔刷用于填充路径所包围的区域。

如果钢笔和画笔都是不透明的,那么在用钢笔画出轮廓之前,先用画笔填充路径。 填充和轮廓相遇时,它们之间不留任何空间,但填充和轮廓之间的精确重叠是根据平台和大小而定的。 因此,由画笔和钢笔绘制的区域可能会重叠。 一般来说,笔在路径上居中,在非平滑模式下向左和向下取舍。

fill-style 参数指定了填充规则: 'odd-even 或 'winding 。 在 'odd-even 模式下,如果一个点被奇数个子路径循环所包围,那么这个点就被认为是被包围在路径中的。 在 'winding 模式下,如果一个点被顺时针方向的子路径环路包围的次数多于或少于逆时针方向的子路径环路, 那么这个点就被认为被包围在路径中。

关于 'aligned 平滑模式的信息,也请参见 set-smoothing 。

(send a-dc draw-point x y) → void?

  y : real?

method

(send a-dc draw-polygon points    
  [xoffset    
  yoffset    
  fill-style]) → void?
  points : 
(or/c (listof (is-a?/c point%))
      (listof (cons/c real? real?)))
  xoffset : real? = 0
  yoffset : real? = 0
  fill-style : (or/c 'odd-even 'winding) = 'odd-even
使用一个点的列表 points 绘制一个填充的多边形,给每个点添加 xoffset 和 yoffset 。 一对点被视为一个点,其中一对点的 car 是 x 值, cdr 是 y 值。 多边形是自动封闭的,所以第一个点和最后一个点可以是不同的。 当前的笔用于绘制轮廓,而当前的笔刷用于填充形状。

如果笔和画笔都是不透明的,那么在用笔画出轮廓之前,就用画笔填充多边形。 填充和轮廓相遇,使它们之间不留空间,但填充和轮廓之间的精确重叠是平台和形状特定的。 因此,由画笔和钢笔绘制的区域可能会重叠。 更普遍的情况是,笔在多边形线条上居中,在非平滑模式下向左和向下取舍。

fill-style 参数指定了填充规则: 在 'odd-even 或 'winding 。 在 'odd-even 模式下,如果一个点被奇数个环所包围,那么它就被认为是被包围在多边形内了。 在 'winding 模式下,如果一个点被顺时针方向的环路包围, 而不是被逆时针方向的环路包围,那么这个点就被认为被包围在多边形内。

关于 'aligned 平滑模式的信息,也请参见 set-smoothing 。

method

(send a-dc draw-rectangle x y width height) → void?

  x : real?
  y : real?
  width : (and/c real? (not/c negative?))
  height : (and/c real? (not/c negative?))
以给定的左上角和尺寸绘制一个矩形。当前的笔用于画出轮廓,当前的笔刷用于填充形状。 如果钢笔和画笔都是不透明的,那么在用钢笔画出轮廓之前,用画笔填充矩形。

在非平滑或对齐模式下,当笔的尺寸为 0 或 1 时,填充正好与整个轮廓重合。 更一般地说,在非平滑或对齐模式下,轮廓的路径是通过缩减矩形的宽度和高度来调整的, 缩减的幅度是一个绘图单位除以 alignment scale 。

关于 'aligned 平滑模式的信息,也请参见 set-smoothing 。

method

(send a-dc draw-rounded-rectangle x    
  y    
  width    
  height    
  [radius]) → void?
  x : real?
  y : real?
  width : (and/c real? (not/c negative?))
  height : (and/c real? (not/c negative?))
  radius : real? = -0.25
以给定的左上角和给定的尺寸绘制一个矩形。 边角是使用给定半径的四分之一圆。当前的笔用于画出轮廓,当前的笔刷用于填充形状。 如果钢笔和画笔都是非透明的,在用钢笔画出轮廓之前,用画笔填充矩形。

如果 radius 是正数,该值被用作圆角的半径。 如果 radius 是负的,绝对值被用作矩形的最小尺寸的 proportion 。

如果 radius 小于 -0.5 或超过 width 或 height 的一半, an exn:fail:contract exception is raised 。

画笔填充和钢笔轮廓相遇,使它们之间不留空间,但填充和轮廓之间的精确重叠是针对平台和尺寸的。 因此,画笔和钢笔绘制的区域可能部分重叠。 在非平滑或对齐模式下,轮廓的路径是通过在缩放后,将矩形的宽度和高度缩小一个绘图单位除以 alignment scale 来调整的。

关于 'aligned 平滑模式的信息,也请参见 set-smoothing 。

method

(send a-dc draw-text text    
  x    
  y    
  [combine-mode    
  offset    
  angle]) → void?
  text : string?
  x : real?
  y : real?
  combine-mode : any/c = #f
  offset : exact-nonnegative-integer? = 0
  angle : real? = 0
在指定的点上绘制一个文本字符串,使用当前的文本字体,以及当前的文本前景和背景颜色。 对于未旋转的文本,指定的点被用作绘制字符的左上角起始点(例如,如果绘制 "W",该点大致是 "W" 的左上角像素的位置)。 旋转的文本是围绕这个点旋转的。

text 字符串从 offset 字符开始绘制,一直到 text 的结尾或第一个空字符。

如果 combine-mode 是 #f, 那么结果与每个字符单独绘制相同,Unicode 控制字符被忽略。 如果 combine-mode 是 'grapheme, 那么结果与每个 Unicode 字母簇单独绘制相同,但 text 中形成字母簇的多个字符可能被绘制成一个字形。 否则,对于 combine-mode 来说,除了 'grapheme 之外, text 可以用相邻的字符组合成连字符字形来绘制,Unicode 将字符组合成一个字形, 用角标,用从右到左的方式呈现字符,等等。

字符串被逆时针旋转了 angle 弧度。 如果 angle 不为零,那么文本总是以透明模式绘制(见 set-text-mode)。

DC的当前笔刷和当前笔的设置对文本的绘制方式没有影响。

参见 get-text-extent 以了解绘制文本的大小。

另请参见 set-text-foreground, set-text-background 和 set-text-mode 。

Changed in version 1.20 of package draw-lib: 改为将 combine-mode 作为 'graheme 处理,与其他真值不同。

对于相关的设备,如果在页面没有被 start-page 启动的情况下调用 end-page ,会产生一个异常。

method

(send a-dc erase) → void?

对于一个有阿尔法通道的绘图环境, erase 将所有的阿尔法通道设置为零。 同样地,对于一个透明的画布, erase 会擦除所有的绘图,以允许背景窗口显示出来。 对于其他没有阿尔法通道或透明度的绘图环境, erase 将绘图环境填充成白色。

(send a-dc set-smoothing mode) → void?

平滑模式是 'unsmoothed, 'smoothed 或 'aligned 。 'aligned 或 'smoothed 都是可以抗锯齿的平滑模式, 而 'unsmoothed 和 'aligned 都会调整绘图坐标以匹配像素边界。 对于大多数向屏幕或位图绘图的应用程序来说, 'aligned 模式是最佳选择。

对于 canvas<%> 输出,调用 suspend-flush in canvas<%> 方法,对于其他类型的绘图上下文没有影响。

method

(send a-dc transform m) → void?

  m : (vector/c real? real? real? real? real? real?)
将一个由 m 组成的变换添加到绘图上下文的当前变换中。

参见 get-initial-matrix ,了解由向量 m 表示的矩阵的信息。

之后,绘图上下文的变换被表示在初始变换矩阵中,单独的原点、比例和旋转设置有它们的相同值。

method

(send a-dc translate dx dy) → void?

  dx : real?
  dy : real?
将 X 方向的 dx 和 Y 方向的 dy 的平移添加到绘图环境的当前变换中。

之后,绘图环境的变换在初始变换矩阵中被表示出来,单独的原点、比例和旋转设置具有它们的相同值。

method

(send a-dc try-color try result) → void?

  try : (is-a?/c color%)
  result : (is-a?/c color%)
确定用于绘制具有给定颜色的请求的实际颜色。 result 颜色被设置为该绘图上下文实际产生的 RGB 值,以绘制颜色 try 。
