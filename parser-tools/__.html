1.1 创建一个词典

syntax

(lexer maybe-suppress-warnings [trigger action-expr] ...)

 
maybe-suppress-warnings = 
  | #:suppress-warnings
     
trigger = re
  | (eof)
  | (special)
  | (special-comment)
     
re = id
  | string
  | character
  | (repetition lo hi re)
  | (union re ...)
  | (intersection re ...)
  | (complement re)
  | (concatenation re ...)
  | (char-range char char)
  | (char-complement re)
  | (id datum ...)
产生一个函数,它接受一个输入端口,与缓冲区中的 re 模式相匹配, 并返回执行相应的 action-expr 的结果。 当多个模式匹配时,词法发生器将选择最长的匹配,打破平局,有利于首先出现的规则。

syntax-color/racket-lexer 的实现包含一个用于 racket 语言的词法分析器。 此外,在 "parser-tools" 集合的 "examples" 子目录下的文件包含了更简单的实例解析器。

一个 re 的匹配方式如下:

  • id — 扩展到命名的 lexer abbreviation; 缩写是通过 define-lex-abbrev 定义的, 或者由 parser-tools/lex-sre 等模块提供。

  • string — 匹配 string 中的字符序列。

  • character — 匹配一个字面 character 。

  • (repetition lo hi re) — 匹配 re 在 lo 和 hi 之间重复的次数,包括在内; hi 可以是 +inf.0 ,用于无限制的重复。

  • (union re ...) — 匹配任何一个子表达式的匹配。

  • (intersection re ...) — 如果所有的 re 匹配,则匹配。

  • (complement re) — 匹配任何与 re 不匹配的内容。

  • (concatenation re ...) — 依次匹配每个 re 。

  • (char-range char char) — 匹配两个(包括)之间的任何字符; 单个字符串可以作为 char 使用。

  • (char-complement re) — 匹配任何未被 re 匹配的字符。 子表达式必须是一组字符 re 。

  • (id datum ...) — 展开名为 id 的 lexer macro ; 宏是通过 define-lex-trans 定义的。

注意 (concatenation) 和 "" 都匹配空字符串, (union) 不匹配任何东西, (intersection) 匹配任何字符串, (char-complement (union)) 匹配任何单个字符。

正则表达式语言不是用来直接使用的,而是作为用正则表达式宏编写的用户友好记号的基础。 例如, parser-tools/lex-sre 提供了来自 Olin Shivers 的 SREs 的操作符, parser-tools/lex-plt-v200 提供了来自本库上一版本的(已废弃)操作符。 由于这些库提供的运算符的名字与其他 Racket 绑定的运算符相匹配, 比如 * 和 +, 所以通常必须使用前缀来导入它们:

(require (prefix-in : parser-tools/lex-sre))

建议的前缀是 :, 这样 :* 和 :+ 就会被导入。 当然,除 : 以外的前缀(如 re-) 也可以使用。

由于否定不是正则表达式中常见的操作符,这里有几个例子,使用 : 前缀的 SRE 语法:

  • (complement "1")

    匹配除 "1" 以外的所有字符串,包括 "11", "111", "0", "01", "", 等等。

  • (complement (:* "1"))

    匹配所有不是 "1" 序列的字符串,包括 "0", "00", "11110", "0111", "11001010" 等。

  • (:& (:: any-string "111" any-string)
        (complement (:or (:: any-string "01") (:+ "1"))))

    匹配所有有 3 个连续的 1 的字符串,但不是以 "01" 结尾的字符串,也不是只有 1 的字符串。 这包括 "1110", "0001000111" 和 "0111" , 但不包括 "", "11", "11101", "111" 和 "11111" 。

  • (:: "/*" (complement (:: any-string "*/" any-string)) "*/")

    匹配 Java/C 块注释。 "/**/", "/******/", "/*////*/", "/*asg4*/" 等等。 它不匹配 "/**/*/", "/* */ */" 等。 (:: any-string "*/" any-string) 匹配任何有 "*/" 的字符串, 所以 (complement (:: any-string "*/" any-string)) 匹配任何没有 "*/" 的字符串。

  • (:: "/*" (:* (complement "*/")) "*/")

    匹配任何以 "/*" 开头、以 "*/" 结尾的字符串, 包括 "/* */ */ */" 。 (complement "*/") 匹配除 "*/" 以外的任何字符串。 这分别包括 "*" 和 "/" 。 因此, (:* (complement "*/")) 通过首先匹配 "*" , 然后匹配 "/" 来匹配 "*/" 。 任何其他的字符串则直接由 (complement "*/") 匹配。 换句话说, (:* (complement "xx")) = any-string 。 在 complement 周围加上 :* 通常是不正确的。

以下绑定在词法动作中具有特殊的意义:

  • start-pos — 匹配的第一个字符的 position 结构。

  • end-pos — 匹配的最后一个字符之后的 position 结构。

  • lexeme — 匹配的字符串。

  • input-port — 正在处理的输入端口(这对于用多个词库匹配输入很有用)。

  • (return-without-pos x) 是一个函数(continuation), 它立即从词典中返回 x 的值。 这在 src-pos 词法分析器中很有用,可以防止词法分析器添加源信息。比如说

    (define get-token
      (lexer-src-pos
      ...
      ((comment) (get-token input-port))
      ...))

    会将注释的源位置信息包裹在递归调用的值周围。 使用 ((comment) (return-without-pos (get-token input-port))) 将导致递归调用的值被返回, 而不把位置包在它周围。

如果没有一个正则表达式与输入相匹配,词法学家会引发一个异常 (exn:read) 。 提示:如果 (any-char custom-error-behavior) 是最后一条规则, 那么总是会有一个匹配,并且 custom-error-behavior 会被执行, 以按需要处理错误情况,只消耗输入缓冲区的第一个字符。

除了返回字符之外,输入端口还可以返回 eof-object 。 自定义输入端口还可以返回一个 special-comment 值来表示非文本注释,或者返回另一个任意值(一个特殊值)。 非 re trigger 形式处理这些情况:

  • 当输入端口返回一个 eof-object 值时, (eof) 规则会被匹配。 如果没有 (eof) 规则,那么当端口返回一个 eof-object 值时, 词典会返回符号 'eof 。

  • 当输入端口返回一个 special-comment 结构时, 将匹配 (special-comment) 规则。 如果没有 special-comment 规则, 词典会自动尝试从输入端口返回下一个标记。

  • 当输入端口返回一个除字符、 eof-object 或 special-comment 结构之外的值时, (special) 规则将被匹配。 如果没有 (special) 规则,词法学家会返回 (void) 。

文件结尾、特殊、特殊注释和特殊错误不能通过使用普通正则表达式的规则来解析 (但在某些情况下,下降并操纵端口来处理它们是可能的)。

由于词法分析器从端口获得其源信息,因此使用 port-count-lines! 来启用行和列信息的跟踪。 否则,行和列的信息将返回 #f 。

当从输入端口偷看时,会引发一个异常(比如由一个语法不规范的嵌入式 XML 编辑器进行偷看), 这个异常可以在异常前面的所有标记被返回之前引发。

每次编译词典的 racket 代码(例如,当一个包含 lexer 形式的 ".rkt" 文件被加载时), 都会运行词典生成器。 为了避免这种开销,将词法分析器放入一个模块,并将该模块编译为 ".zo" 字节码文件。

如果词法分析器可以接受空字符串,就会向 current-logger 发送一条消息。 这些警告可以通过给出 #:suppress-warnings 标记来禁用。

Examples:
> (define sample-input "( lambda (a ) (add_number a 42  ))")
; A function that partially tokenizes the sample input data
> (define (get-tokens a-lexer)
    (define p (open-input-string sample-input))
    (list (a-lexer p)
          (a-lexer p)
          (a-lexer p)
          (a-lexer p)
          (a-lexer p)))
; A lexer that uses primitive operations directly
> (define the-lexer/primitive
    (lexer
      [(eof) eof]
      ["(" 'left-paren]
      [")" 'right-paren]
      [(repetition 1 +inf.0 numeric) (string->number lexeme)]
      [(concatenation (union alphabetic #\_)
                      (repetition 0 +inf.0 (union alphabetic numeric #\_)))
       lexeme]
      ; invoke the lexer again to skip the current token
      [whitespace (the-lexer/primitive input-port)]))
> (get-tokens the-lexer/primitive)

'(left-paren "lambda" left-paren "a" right-paren)

; Another lexer that uses SRE operators but has the same functionality
> (require (prefix-in : parser-tools/lex-sre))
> (define the-lexer/SRE
    (lexer
      [(eof) eof]
      ["(" 'left-paren]
      [")" 'right-paren]
      [(:+ numeric) (string->number lexeme)]
      [(:: (:or alphabetic #\_) (:* (:or alphabetic numeric #\_)))
       lexeme]
      [whitespace (the-lexer/SRE input-port)]))
> (get-tokens the-lexer/SRE)

'(left-paren "lambda" left-paren "a" right-paren)

1.3 Lexer SRE 操作符

 (require parser-tools/lex-sre) package: parser-tools-lib

syntax

(* re ...)

重复 re 序列 0 次或更多次。

syntax

(+ re ...)

重复 re 序列 1 次或更多次。

syntax

(? re ...)

re 序列的零次或一次出现。

syntax

(= n re ...)

精确地出现了 n 个 re 序列,其中 n 必须是一个精确的非负数。

syntax

(>= n re ...)

至少出现了 n 个 re 序列的出现,其中 n 必须是一个确切的、非负的字面数字。

syntax

(** n m re ...)

在 n 和 m 之间(包括)出现 re sequence, 其中 n 必须是一个确切的、非负的字面数字, m 必须是 #f, +inf.0 或者一个确切的、非负的数字; m 的 #f 值与 +inf.0 相同。

syntax

(or re ...)

与 (union re ...) 相同。

syntax

(: re ...)

syntax

(seq re ...)

这两种形式都是将 re 连接起来。

syntax

(& re ...)

与 re 相交。

syntax

(- re ...)

re 的集差。

syntax

(~ re ...)

字符集互补,每个 re 必须精确匹配一个字符。

syntax

(/ char-or-string ...)

字符范围,在连续的字符对之间匹配字符。

1.5 Tokens

lexer 形式中的每个 action-expr 可以产生任何类型的值, 但是对于许多目的来说,产生一个 token 值是有用的。 在与由 parser-tools/yacc 或 parser-tools/cfg-parser 生成的解析器进行交互操作时,通常需要标记,但在其他情况下使用 lexer 时,标记可能不是正确的选择。

Examples:
> (define-tokens basic-tokens (NUM ID))
> (define-empty-tokens punct-tokens (LPAREN RPAREN EOF))
> (define the-lexer/tokens
    (lexer
      [(eof) (token-EOF)]
      ["(" (token-LPAREN)]
      [")" (token-RPAREN)]
      [(:+ numeric) (token-NUM (string->number lexeme))]
      [(:: (:or alphabetic #\_)
           (:* (:or alphabetic numeric #\_)))
       (token-ID (string->symbol lexeme))]
      [whitespace (the-lexer/tokens input-port)]))
; Use get-tokens defined in (part "Creating_a_Lexer")
> (get-tokens the-lexer/tokens)

(list 'LPAREN (token 'ID 'lambda) 'LPAREN (token 'ID 'a) 'RPAREN)

syntax

(define-tokens group-id (token-id ...))

将 group-id 与正在定义的标记组绑定。 对于每个 token-id, 创建一个函数 token-token-id , 该函数接受任何值并将其放入特定于 token-id 的 token 记录。 可以使用 token-name 和 token-value 来检查一个令牌。

一个令牌不能被命名为 error, 因为 error 在解析器中有特殊的用途。

syntax

(define-empty-tokens group-id (token-id ...))

像 define-tokens 一样, 但是每个 token 构造函数 token-token-id 不需要参数, 并返回 (quote token-id) 。

procedure

(token-name t) → symbol?

  t : (or/c token? symbol?)
返回由符号或令牌结构表示的令牌的名称。

procedure

(token-value t) → any/c

  t : (or/c token? symbol?)
返回由符号或令牌结构代表的令牌的值,如果是符号令牌,返回 #f 。

procedure

(token? v) → boolean?

  v : any/c
如果 val 是一个 token 结构,返回 #t ,否则返回 #f 。
