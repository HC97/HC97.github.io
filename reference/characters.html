4.6 Characters

字符（Character） in Racket 指南 introduces characters.

Characters 的范围是 scalar values, 其中包括从 #x0 到 #x10FFFF 的字符,但不包括 #xD800 到 #xDFFF 。 标量值是 Unicode code points 的一个子集。

如果两个字符对应相同的标量值,它们就是 eqv? 。 对于每个小于256的标量值,属于 eqv? 的字符值也是 eq? 。 由默认阅读器产生的字符在 read-syntax 模式下被 interned 。

See 阅读字符 for information on reading characters and 打印字符 for information on printing characters.

Changed in version 6.1.1.8 of package base: Updated from Unicode 5.0.1 to Unicode 7.0.0.

4.6.1 字符和标量值

procedure

(char? v) → boolean?

  v : any/c
如果 v 是一个字符,返回 #t ,否则返回 #f 。

procedure

(char->integer char) → exact-integer?

  char : char?
返回一个字符的码位数。

Example:
> (char->integer #\A)

65

procedure

(integer->char k) → char?

  k : 
(and/c exact-integer?
       (or/c (integer-in 0 #xD7FF)
             (integer-in #xE000 #x10FFFF)))
返回代码点编号为 k 的字符。 对于小于 256 的 k ,其结果是相同的 k 的对象。

Example:
> (integer->char 65)

#\A

procedure

(char-utf-8-length char) → (integer-in 1 6)

  char : char?
产生的结果与 (bytes-length (string->bytes/utf-8 (string char))) 相同。

4.6.2 字符比较

procedure

(char=? char1 char2 ...) → boolean?

  char1 : char?
  char2 : char?
如果所有的参数都是 eqv?,则返回 #t 。

Examples:
> (char=? #\a #\a)

#t

> (char=? #\a #\A #\a)

#f

  char1 : char?

Examples:

> (char<? #\a #\A)

#f

> (char<? #\a #\b #\c)

4.6.3 分类法

procedure

(char-alphabetic? char) → boolean?

  char : char?
如果 char 具有 Unicode “Alphabetic” 属性,返回 #t 。

procedure

(char-lower-case? char) → boolean?

  char : char?
如果 char 具有 Unicode “Lowercase” 属性,则返回 #t 。

procedure

(char-upper-case? char) → boolean?

  char : char?
如果 char 具有 Unicode “Uppercase” 属性,则返回 #t 。

procedure

(char-title-case? char) → boolean?

  char : char?
如果 char 的 Unicode 一般类别为 Lt,则返回 #t ,否则返回 #f 。

procedure

(char-numeric? char) → boolean?

  char : char?
如果 char 的 Unicode “Numeric_Type” 属性值不是 None ,则返回 #t 。

procedure

(char-symbolic? char) → boolean?

  char : char?
如果 char 的 Unicode 一般类别是 Sm, Sc, Sk 或 So ,则返回 #t ,否则返回 #f 。

procedure

(char-punctuation? char) → boolean?

  char : char?
如果 char 的 Unicode 大类是 Pc, Pd, Ps, Pe, Pi, Pf 或 Po ,则返回 #t ,否则返回 #f 。

procedure

(char-graphic? char) → boolean?

  char : char?
如果 char 的 Unicode 一般类别是 Ll, Lm, Lo, Lt, Lu, Nd, Nl, No, Mn, Mc 或 Me ,则返回 #t ， 或者如果以下情况在应用于 char 时产生 #t: char-alphabetic?, char-numeric?, char-symbolic? 或 char-punctuation? 。

procedure

(char-whitespace? char) → boolean?

  char : char?
如果 char 具有 Unicode 的 “White_Space” 属性,返回 #t。

procedure

(char-blank? char) → boolean?

  char : char?
如果 char 的 Unicode 一般类别是 Zs ,或者如果 char 是 #\tab , 则返回 #t 。(这些对应于水平留白。)

procedure

(char-iso-control? char) → boolean?

  char : char?
如果 char 在 #\nul 和 #\u001F 之间, 或者 #\rubout 和 #\u009F 之间,返回 #t 。

procedure

(char-extended-pictographic? char) → boolean?

  char : char?
如果 char 具有 Unicode 的 “Extended_Pictographic” "属性,则返回 #t 。

Added in version 8.6.0.1 of package base.

procedure

(char-general-category char) → symbol?

  char : char?
返回一个符号,代表该字符的 Unicode 大类,即 'lu, 'll, 'lt, 'lm, 'lo, 'mn, 'mc, 'me, 'nd, 'nl, 'no, 'ps, 'pe, 'pi, 'pf, 'pd, 'pc, 'po, 'sc, 'sm, 'sk, 'so, 'zs, 'zp, 'zl, 'cc, 'cf, 'cs, 'co 或 'cn 。

procedure

(char-grapheme-break-property char) → ?

  char : char?
返回 char 的 Unicode graheme-break 属性,即 'Other, 'CR, 'LF, 'Control, 'Extend, 'ZWJ, 'Regional_Indicator, 'Prepend, 'SpacingMark, 'L, 'V, 'T, 'LV 或 'LVT 。

Added in version 8.6.0.1 of package base.

procedure

(make-known-char-range-list)

 → 
(listof (list/c exact-nonnegative-integer?
                exact-nonnegative-integer?
                boolean?))
产生一个三元素列表,每个三元素列表代表一组连续的码点,Unicode 标准为其规定了字符属性。 每个三元素列表包含两个整数和一个布尔值; 第一个整数是起始码点值(包括),第二个整数是终止码点值(包括),布尔值是 #t , 当码点范围内的所有字符对于上述所有字符谓词具有相同的结果,对于 char-downcase, char-upcase 或 char-titlecase 具有类似的转换(在码点空间中移动相同的量,如果有的话), 并且具有相同的分解-规范化行为。 三元素列表在整个结果列表中的排序是这样的:后面的列表代表较大的码位值, 所有三元素列表之间至少有一个码位值没有被 Unicode 指定。

4.6.4 字符转换

procedure

(char-upcase char) → char?

  char : char?
产生一个与Unicode定义的 1-1 码位映射相一致的字符。 如果 char 没有大小写的映射, char-upcase 产生 char 。

字符串程序,例如 string-upcase , 处理 Unicode 定义的从码点到码点序列的独立于本地的映射(除了标量值上的 1-1 映射)的情况。

Examples:
> (char-upcase #\a)

#\A

> (char-upcase #\λ)

#\Λ

> (char-upcase #\space)

#\space

procedure

(char-downcase char) → char?

  char : char?
与 char-upcase 类似,但为 Unicode 下划线映射。

Examples:
> (char-downcase #\A)

#\a

> (char-downcase #\Λ)

#\λ

> (char-downcase #\space)

#\space

procedure

(char-titlecase char) → char?

  char : char?
与 char-upcase 类似,但用于 Unicode 标题大写的映射。

Examples:
> (char-upcase #\a)

#\A

> (char-upcase #\λ)

#\Λ

> (char-upcase #\space)

#\space

procedure

(char-foldcase char) → char?

  char : char?
与 char-upcase 类似,但用于 Unicode 大小写折合映射。

Examples:
> (char-foldcase #\A)

#\a

> (char-foldcase #\Σ)

#\σ

> (char-foldcase #\ς)

#\σ

> (char-foldcase #\space)

#\space

4.6.5 字符串-群组流

procedure

(char-grapheme-step char state) → 
boolean? fixnum?
  char : char?
  state : fixnum?
编码Unicode的字母群规范在一串码位上的状态机。 它接受一个字符作为序列中的下一个码点,并返回两个值:自最近报告的终止(或流的开始)以来, 一个(单一的)字母簇是否已经终止,以及一个新的状态, 用于 char-grapheme-step 和下一个字符。

state 的值为 0 ,代表初始状态或没有字符向新的边界待发的状态。 因此,如果一个字符序列用完了,并且累积的 state 不是 0 ,那么流的末端就会创建一个最后的字母簇边界。 当 char-grapheme-step 产生一个真值作为它的第一个结果和一个非 0 值作为它的第二个结果时, 那么给定的char 必须是唯一等待进入下一个字素簇的字符(根据Unicode字素簇的规则)。

char-grapheme-step 程序将为任何 fixnum state 产生一个结果, 但是非 0 state 的含义只规定在另一次调用 char-grapheme-step 时提供 char-grapheme-step 产生的这种状态,继续检测序列中的字母簇边界。

参见 string-grapheme-span 和 string-grapheme-count 。

Examples:
> (char-grapheme-step #\a 0)

#f

1

> (let*-values ([(consumed? state) (char-grapheme-step #\a 0)]
                [(consumed? state) (char-grapheme-step #\b state)])
    (values consumed? state))

#t

1

> (let*-values ([(consumed? state) (char-grapheme-step #\return 0)]
                [(consumed? state) (char-grapheme-step #\newline state)])
    (values consumed? state))

#t

0

> (let*-values ([(consumed? state) (char-grapheme-step #\a 0)]
                [(consumed? state) (char-grapheme-step #\u300 state)])
    (values consumed? state))

#f

5
