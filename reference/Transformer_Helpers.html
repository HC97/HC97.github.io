7.12 Transformer Helpers

 (require racket/unit-exptime) package: base

racket/unit-exptime 库提供了供宏转换器使用的程序。 特别是,该库通常使用 for-syntax 导入到一个用 define-syntax 定义宏的模块中。

procedure

(unit-static-signatures unit-identifier 
  err-syntax) 
 → 
(list/c (cons/c (or/c symbol? #f)
                identifier?))
(list/c (cons/c (or/c symbol? #f)
                identifier?))
  unit-identifier : identifier?
  err-syntax : syntax?
如果 unit-identifier 通过 define-unit (或其他类似形式)被绑定到静态单元信息,结果是两个值。 第一个值是该单元的进口,第二个值是该单元的出口。 每个结果值都是一个列表,其中每个列表元素都将一个符号或 #f 与一个标识符配对。 符号或 #f 表示进口的或出口的标签(其中 #f 表示没有标签),而标识符表示相应签名的绑定。

如果 unit-identifier 没有绑定到静态单元信息,那么会引发 exn:fail:syntax exception is raised 。 在这种情况下,给定的 err-syntax 参数被用作错误的来源, 其中 unit-identifier 被用作详细的来源位置。

procedure

(signature-members sig-identifier    
  err-syntax) → 
(or/c identifier? #f)
(listof identifier?)
(listof identifier?)
(listof identifier?)
  sig-identifier : identifier?
  err-syntax : syntax?
如果 sig-identifier 通过 define-signature (或其他类似形式)与静态单元信息绑定, 其结果是四个值:

  • 一个标识符或 #f ,表示由 sig-identifier 绑定扩展的签名(任何);

  • 代表签名所提供/要求的变量的标识符列表;

  • 签名中变量定义的标识符列表(即导入时提供的变量绑定,但不是由实现该签名的单元定义的);以及

  • 签名中具有语法定义的标识符的列表。

每个结果标识符都被赋予基于 sig-identifier 的词法信息, 所以这些名字适合于在 sig-identifier 的上下文中引用或绑定。 更多信息请见 define-signature 。

如果 sig-identifier 没有被绑定到一个签名,那么就会引发 exn:fail:syntax exception is raised 。 在这种情况下,给定的 err-syntax 参数被用作错误的来源, 其中 sig-identifier 被用作详细的来源位置。

procedure

(unit-static-init-dependencies unit-identifier 
  err-syntax) 
 → 
(list/c (cons/c (or/c symbol? #f)
                identifier?))
  unit-identifier : identifier?
  err-syntax : syntax?
如果 unit-identifier 通过 define-unit (或其他类似形式)被绑定到静态单元信息,结果是一个对的列表。 每个对结合了一个标签(或 #f 表示没有标签)和一个签名名, 表示单元对指定导入的初始化依赖性(即相同的标签和签名被包含在 unit-static-signatures 的第一个结果中)。

如果 unit-identifier 没有绑定到静态单元信息,那么会引发 exn:fail:syntax exception is raised 。 在这种情况下,给定的 err-syntax 参数被用作错误的来源, 其中 unit-identifier 被用作详细的来源位置。

Added in version 6.1.1.8 of package base.
