6.10 对象打印

要自定义类实例被 print, write 和 display 的方式, 实现 printable<%> 接口。

value

printable<%> : interface?

printable<%> 接口只包括 custom-print, custom-write 和 custom-display 方法。 custom-print 方法接受两个参数: 目的端口和当前 quasiquote 深度,是一个精确的非负整数。 custom-write 和 custom-display 方法各自接受一个参数, 即 write 或 display 对象的目标端口。

对 custom-print, custom-write 或 custom-display 方法的调用就像对通过 prop:custom-write 属性附加到结构类型的过程的调用。 特别是,递归打印可以触发对调用的逃避。

更多信息请参见 prop:custom-write 。 printable<%> 接口是通过 interface* 和 prop:custom-write 。

value

writable<%> : interface?

与 printable<%> 类似,但只包括 custom-write 和 custom-display 方法。 一个 print 请求被引导到 custom-write 。
