3.17 指派: set! 和 set!-values

赋值：set! in Racket 指南 introduces set!.

syntax

(set! id expr)

如果 id 有一个与 assignment transformer 绑定的 transformer , 如 make-set!-transformer 产生的,或者作为一个具有 prop:set!-transformer 属性的结构类型的实例,那么这个表单将通过调用具有完整表达式的赋值变换器来展开。 如果 id 有一个 transformer 绑定到由 make-rename-transformer 生成的 rename transformer , 或者作为一个具有 prop:rename-transformer 属性的结构类型的实例, 那么这种形式将通过用目标标识符替换 id 来扩展 (例如,提供给 make-rename-transformer 的标识符)。 如果一个变换器绑定同时具有 prop:set!-transformer 和 prop:rename-transformer 属性,则后者优先考虑。

否则,评估 expr 并将结果安装到 id 的位置, id 必须被绑定为本地变量或定义为 top-level variable 或 module-level variable 。 如果 id 指的是一个导入的绑定,会报告一个语法错误。 如果 id 指的是一个没有被定义的 top-level variable , 会引发 exn:fail:contract exception is raised 异常。

参见 compile-allow-set!-undefined。

Examples:
> (define x 12)
> (set! x (add1 x))
> x

13

> (let ([x 5])
    (set! x (add1 x))
    x)

6

> (set! i-am-not-defined 10)

set!: assignment disallowed;

 cannot set variable before its definition

  variable: i-am-not-defined

  in module: top-level

syntax

(set!-values (id ...) expr)

假设所有的 id 都指的是变量,这个表单评估 expr, 它必须产生与提供的 id 一样多的值。 每个 id 的位置都用 expr 的相应值来填充, 方法与 set! 相同。

Example:
> (let ([a 1]
        [b 2])
    (set!-values (a b) (values b a))
    (list a b))

'(2 1)

更一般地说, set!-values 形式被扩展为

(let-values ([(tmp-id ...) expr])
  (set! id tmp-id) ...)

如果任何 id 有一个与 assignment transformer 绑定的变换器, 就会触发进一步的扩展。
