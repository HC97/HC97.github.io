8.8 契约实用程序

procedure

(contract? v) → boolean?

  v : any/c
如果它的参数是一个 contract (即用本节中描述的组合器之一构造的,或者一个可以用作契约的值), 则返回 #t ,否则返回 #f 。

procedure

(chaperone-contract? v) → boolean?

  v : any/c
如果它的参数是一个 chaperone contract , 即保证它返回的值与原始的、未收缩的值相比通过 chaperone-of? , 则返回 #t 。

procedure

(impersonator-contract? v) → boolean?

  v : any/c
如果它的参数是一个 impersonator contract , 即一个既不是 chaperone contract 也不是 flat contract 的 contract , 则返回 #t 。

procedure

(flat-contract? v) → boolean?

  v : any/c
当它的参数是一个可以立即被检查的契约时,返回 #t (不像,例如,一个函数契约)。

例如, flat-contract 从谓词中构造 flat contracts , 而符号、布尔、数字和其他普通 Racket 值(被定义为 contracts) 也是 flat contracts 。

procedure

(list-contract? v) → boolean?

  v : any/c
识别某些接受 list? 的 contract? 值。

列表合约是一个坚持其参数是 list? 的合约,意味着该值不能是循环的, 必须是空的列表或用 cons 和另一个列表构造的一对。

Added in version 6.0.1.13 of package base.

procedure

(contract-name c) → any/c

  c : contract?
产生用于在错误信息中描述合同的名称。

procedure

(value-contract v) → (or/c contract? #f)

  v : has-contract?
如果有记录,则返回附在 v 上的合同。否则,它返回 #f 。

要在你自己的契约组合器中支持 value-contract 和 value-contract , 请使用 prop:contracted 或 impersonator-prop:contracted 。

procedure

(has-contract? v) → boolean?

  v : any/c
如果 v 是一个有记录契约的值,则返回 #t 。

procedure

(value-blame v) → (or/c blame? #f)

  v : has-blame?
如果有记录,则返回附在 v 上的合同的 blame object 。否则,它返回 #f 。

要在你自己的契约组合器中支持 value-contract 和 value-blame , 请使用 prop:blame 或 impersonator-prop:blame 。

Added in version 6.0.1.12 of package base.

procedure

(has-blame? v) → boolean?

  v : any/c
如果 v 是一个带有责备信息的合同的值,则返回 #t 。

Added in version 6.0.1.12 of package base.

procedure

(contract-late-neg-projection c)

 → (-> blame? (-> any/c (or/c #f any/c) any/c))
  c : contract?
生成定义合同行为的投影。

第一个参数, blame? 对象封装了关于合同检查的信息,主要用于在检测到违反合同时创建一个有意义的错误信息。 由此产生的函数的第一个参数是应该有合同的值,它的第二个参数是 blame object 的缺失方, 要传递给 raise-contract-error 。

如果可能的话,使用这个函数代替 contract-val-first-projection 或 contract-projection 。

procedure

(contract-projection c) → (-> blame? (-> any/c any/c))

  c : contract?
产生一个定义了合约行为的投影。 这个投影是一个由两个参数组成的咖喱函数:第一个应用接受一个责备对象,第二个接受一个值来保护合同。

如果可能的话,使用 contract-late-neg-projection 代替。

procedure

(contract-val-first-projection c)

 → (-> blame? (-> any/c (-> any/c any/c)))
  c : contract?
产生一个定义合约行为的投影。 这个投影类似于 contract-late-neg-projection 的结果,只是多了一层夤缘。

如果可能的话,请使用 contract-late-neg-projection 来代替。

procedure

(make-none/c sexp-name) → contract?

  sexp-name : any/c
制作一个不接受任何值的契约,并在发出违反契约的信号时报告名称 sexp-name

syntax

(recursive-contract contract-expr recursive-contract-option ...)

(recursive-contract contract-expr type recursive-contract-option ...)
 
recursive-contract-option = #:list-contract?
  | #:extra-delay
     
type = #:impersonator
  | #:chaperone
  | #:flat
延迟对其参数的评估,直到合同被检查,使递归合同成为可能。 如果没有给出 type ,就会创建一个冒名顶替的合同。

如果给出了 recursive-contract-option #:list-contract? , 那么结果就是一个 list-contract? ,并且 contract-expr 必须评估为一个 list-contract? 。

如果给出了 recursive-contract-option #:extra-delay , 那么 contract-expr 表达式只有在第一个要被检查的值被提供给合同时才会被评估。 如果没有这个选项, contract-expr 将提前被评估。 只有当 type 是 #:flat 时才支持这个选项。

Examples:
> (define even-length-list/c
    (or/c null?
          (cons/c any/c
                  (cons/c any/c
                          (recursive-contract even-length-list/c #:flat)))))
> (even-length-list/c '(A B))

#t

> (even-length-list/c '(1 2 3))

#f

Changed in version 6.0.1.13 of package base: 增加了 #:list-contract? 选项。
Changed in version 6.7.0.3: 增加了 #:extra-delay 选项。

syntax

(opt/c contract-expr maybe-name)

 
maybe-name = 
  | #:error-name id
这通过遍历其语法来优化其参数合同表达式,对于已知的合同组合器,将其融合到一个合同组合器中,尽可能地避免分配开销。 其结果是一个与它的参数行为相同的契约,只是速度更快。

如果 #:error-name 参数存在,并且 contract-expr 求值为一个非契约表达式, 那么 opt/c 会使用 id 作为基元的名称引发一个错误,而不是使用 opt/c 的名称。

Examples:
> (define/contract (f x)
    (opt/c '(not-a-contract))
    x)

opt/c: contract violation

  expected: contract?

  given: '(not-a-contract)

> (define/contract (f x)
    (opt/c '(not-a-contract) #:error-name define/contract)
    x)

define/contract: contract violation

  expected: contract?

  given: '(not-a-contract)

syntax

(define-opt/c (id id ...) expr)

这就定义了一个递归合约并同时优化了它。 只要定义的函数终止, define-opt/c 的行为就像不存在 -opt/c 一样,定义了一个关于合约的函数(除了主体表达式必须返回一个合约)。 但是,它也优化了该合约的定义,避免了额外的分配,就像 opt/c 做的那样。

例如,

(define-contract-struct bt (val left right))
 
(define-opt/c (bst-between/c lo hi)
  (or/c null?
        (bt/c [val (real-in lo hi)]
              [left (val) (bst-between/c lo val)]
              [right (val) (bst-between/c val hi)])))
 
(define bst/c (bst-between/c -inf.0 +inf.0))

定义了检查二进制搜索树不变量的 bst/c 契约。 去掉 -opt/c 也会产生一个二进制搜索树契约,但它的速度(大约)要慢 20 倍。

请注意,在某些情况下,对由 define-opt/c 定义的函数的调用可能会终止, 即使相应的基于 define 的函数也不会终止。 这是 define-opt/c 的一个缺陷,我们希望在某个时候能够理解并解决这个问题,但目前还没有具体计划。

value

contract-continuation-mark-key : continuation-mark-key?

在合同检查过程中出现的延续标记所使用的键。 这些标记的值是对应于当前正在检查的合同的 blame objects 。

procedure

(rename-contract contract name) → contract?

  contract : contract?
  name : any/c
产生一个行为类似于 contract 的合同,但名称是 name 。

如果合同是一个 flat contract ,那么产生的 contract 就是一个 flat contract 。

Added in version 6.3 of package base.

syntax

(contract-first-order-okay-to-give-up?)

这个表单返回一个布尔值,控制一阶联系检查的结果。 更具体地说,如果它返回 #t ,那么一阶检查可能会返回 #t ,即使整个一阶检查都没有发生。 如果它返回 #f ,那么一阶检查必须继续,直到返回一个确定的答案。

这只会在 or/c 或 first-or/c 的检查的动态范围内返回 #t ,以确定使用哪个分支。

Added in version 6.3.0.9 of package base.

syntax

(contract-first-order-try-less-hard e)

鼓励发生在 e 的动态范围内的一阶检查更有可能放弃。 也就是说,让 contract-first-order-okay-to-give-up? 更有可能返回 #t 。

如果不在 or/c 的动态范围内或 first-or/c 的检查中确定分支,那么这种形式就没有影响。

Added in version 6.3.0.9 of package base.

procedure

(if/c predicate then-contract else-contract) → contract?

  predicate : (-> any/c any/c)
  then-contract : contract?
  else-contract : contract?
产生一个契约,当应用于一个值时,首先用 predicate 测试该值;如果 predicate 返回真, 则应用 then-contract ;否则,应用 else-contract 。 如果 then-contract 和 else-contract 都是 flat contracts, 那么产生的合约就是 flat contract 。

例如,下面的契约强制规定,如果一个值是一个过程,它就是一个 thunk;否则它可以是任何(非过程)值:

(if/c procedure? (-> any) any/c)

请注意,下面的契约是 和 等价的:

(or/c (-> any) any/c) ; wrong!

最后一个契约与 any/c 相同,因为 or/c 会在高阶契约之前尝试 flat contracts 。

Added in version 6.3 of package base.

value

failure-result/c : contract?

一个描述诸如 hash-ref 等过程的失败结果参数的契约。

相当于 (if/c procedure? (-> any) any/c) 。

Added in version 6.3 of package base.

更多细节参见 make-contract 。

procedure

(get/build-late-neg-projection c)

 → (-> blame? (-> any/c any/c any/c))
  c : contract?
返回 c 的 late-neg 。

如果 c 没有一个 late-neg 合约,那么这个函数将使用它的原始投影, 并向 'racket/contract 记录器记录一个警告。
