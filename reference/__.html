10.7 退出

procedure

(exit [v]) → any

  v : any/c = #t
将 v 传给当前的 exit handler 。 如果退出处理程序没有逃脱或终止线程,则返回 #<void> 。

parameter

(exit-handler) → (any/c . -> . any)

(exit-handler proc) → void?
  proc : (any/c . -> . any)
一个 parameter ,决定了当前的 exit handler 。 该 exit handler 由 exit 调用。

Racket 可执行文件中的默认 exit handler 接受任何参数, 在原始 plumber 上调用 plumber-flush-all ,并关闭操作系统级 Racket 进程。 如果参数是 1 和 255 之间的精确整数(通常意味着 "失败"), 则被用作操作系统级的退出代码;否则,退出代码为 0, (通常意味着 "成功")。

parameter

(executable-yield-handler) → (byte? . -> . any)

(executable-yield-handler proc) → void?
  proc : (byte? . -> . any)
一个 parameter ,决定了在Racket进程即将正常退出时要调用的程序。 当 exit (或者更确切地说,默认 exit handler) 被用来提前退出时,与该参数相关的程序不会被调用。 该处理程序的参数是退出时返回给系统的状态代码。 默认的可执行-产出处理程序只是返回 #<void>。

racket/gui/base 库将此参数设置为等待,直到所有的帧被关闭,计时器停止,以及在主事件空间中处理排队的事件。 更多信息请参见 racket/gui/base 。
