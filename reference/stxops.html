12.2 语法对象内容

procedure

(syntax? v) → boolean?

  v : any/c
如果 v 是一个 syntax object,返回 #t ,否则返回 #f 。 参见 语法对象 。

Examples:
> (syntax? #'quinoa)

#t

> (syntax? #'(spelt triticale buckwheat))

#t

> (syntax? (datum->syntax #f 'millet))

#t

> (syntax? "barley")

#f

procedure

(identifier? v) → boolean?

  v : any/c
如果 v 是一个 syntax object,并且 (syntax-e stx) 产生一个符号,则返回 #t 。

Examples:
> (identifier? #'linguine)

#t

> (identifier? #'(if wheat? udon soba))

#f

> (identifier? 'ramen)

#f

> (identifier? 15)

#f

procedure

(syntax-source stx) → any/c

  stx : syntax?
返回 syntax object stx 的 source location 的源成分,如果不知道,则返回 #f 。 源是由一个任意的值表示的(例如,传递给 read-syntax 的值),但它通常是一个文件路径字符串。

参见 racket/syntax-srcloc 中的 syntax-srcloc 。

procedure

(syntax-line stx) → (or/c exact-positive-integer? #f)

  stx : syntax?
返回 syntax object 在其源码中开始的 source location 的行号(正整数), 如果行号或源码不详,则返回 #f 参见 计算位置、行和列。

Changed in version 7.0 of package base: 放弃了保证 syntax-line 和 syntax-column 都产生 #f 或都产生整数。

procedure

(syntax-column stx) → (or/c exact-nonnegative-integer? #f)

  stx : syntax?
返回 syntax object 在其源码中开始的 source location 的列号(非负的精确整数), 如果源码列不详,则返回 #f 。 参见 计算位置、行和列 。

Changed in version 7.0 of package base: 放弃了保证 syntax-line 和 syntax-column 都产生 #f 或都产生整数。

procedure

(syntax-position stx) → (or/c exact-positive-integer? #f)

  stx : syntax?
返回 syntax object 在其源码中开始的 source location 的位置(正的精确整数), 如果源码位置未知,则返回 #f 。 这个位置是指一个字符的位置、 但从没有启用行计数的端口读出的位置会产生 一个字节偏移的位置。 参见 计算位置、行和列 。

procedure

(syntax-span stx) → (or/c exact-nonnegative-integer? #f)

  stx : syntax?
返回源中 syntax object 的 source location 的跨度(非负精确整数), 如果跨度未知,则返回 #f 。 该跨度的目的是以字符为单位进行计数、 但从没有启用行计数的端口读取时, 会产生以字节为单位的跨度。 参见 (part "lineecol")。

procedure

(syntax-original? stx) → boolean?

  stx : syntax?
如果 stx 具有 read-syntax 附加到其生成的 syntax object 上的属性 (参见 语法对象属性), 并且 stx 的 lexical information 不包括任何宏引入范围(表明该对象是由语法转换器引入的; 见 语法对象),则返回 #t 。 否则,其结果是 #f 。

这个谓词可以用来区分扩展表达式中直接存在于原始表达式中的 syntax object , 而不是由宏插入的 syntax object 。

代表原始语法的(隐藏)属性对于作为编译代码的一部分被调集的语法对象来说是被放弃的;另见 current-compile 。

procedure

(syntax-source-module stx [source?])

 → (or/c module-path-index? symbol? path? resolved-module-path? #f)
  stx : syntax?
  source? : any/c = #f
返回一个包含 stx 的源模块的指示, 如果从词法上下文中无法推断出 stx 的源模块,则返回 #f 。 如果 source? 是 #f, 那么结果是一个模块路径索引或符号(见 编译的模块和引用) 或一个 resolved module path; 如果 source? 是true,那么结果是一个路径或符号, 对应于 current-module-declare-source 意义上的加载模块的源。

请注意, syntax-source-module 并 不 查询 stx 的源位置。 其结果是基于 stx 的 lexical information 。

procedure

(syntax-e stx) → any/c

  stx : syntax?
将直接数据结构从一个 syntax object 中解开,留下嵌套的语法结构(如果有的话)。 (syntax-e stx) 的结果是以下之一:

  • 一个符号

  • 一个 syntax pair (如下所述)

  • 空列表

  • 一个包含 syntax object 的不可变向量

  • 一个包含 syntax object 的不可变的盒子

  • 一个不可变的 hash table , 包含 syntax object 的值(但不一定是 syntax object 的键)

  • 一个包含 syntax object 的不可变的 prefab 结构

  • 一些其他类型的数据–通常是数字、布尔值或字符串– 当 datum-intern-literal 会转换值时,这些数据会被 interned

Examples:
> (syntax-e #'a)

'a

> (syntax-e #'(x . y))

'(#<syntax:eval:11:0 x> . #<syntax:eval:11:0 y>)

> (syntax-e #'#(1 2 (+ 3 4)))

'#(#<syntax:eval:12:0 1> #<syntax:eval:12:0 2> #<syntax:eval:12:0 (+ 3 4)>)

> (syntax-e #'#&"hello world")

'#&#<syntax:eval:13:0 "hello world">

> (syntax-e #'#hash((imperial . "yellow") (festival . "green")))

'#hash((festival . #<syntax:eval:14:0 "green">)

       (imperial . #<syntax:eval:14:0 "yellow">))

> (syntax-e #'#(point 3 4))

'#(#<syntax:eval:15:0 point> #<syntax:eval:15:0 3> #<syntax:eval:15:0 4>)

> (syntax-e #'3)

3

> (syntax-e #'"three")

"three"

> (syntax-e #'#t)

#t

syntax pair 是包含一个 syntax object 作为其第一个元素, 以及空列表、一个语法对或一个语法对象作为其第二个元素的一对。

syntax object 是 read-syntax 的结果,它通过为源文件中的每一对小括号创建一个语法对象, 以及 只 为源文件中的小括号创建一个对值的 syntax object,来反映在输入中使用划定的 . 。 更多信息请参见 读取成对和列表 。

如果 stx 是 tainted , 那么 (syntax-e stx) 的结果中的任何语法对象都是 tainted 。 多次调用 stx 的 syntax-e 的结果是 eq? 。

procedure

(syntax->list stx) → (or/c list? #f)

  stx : syntax?
返回一个 syntax object 的列表或 #f 。 当 (syntax->datum stx) 会产生一个列表时,其结果是一个 syntax object 的列表。 换句话说, (syntax-e stx) 中的 syntax pairs 对被平移了。

如果 stx 是 tainted , 那么 (syntax->list stx) 的结果中的任何语法对象都是 tainted 。

Examples:
> (syntax->list #'())

'()

> (syntax->list #'(1 (+ 3 4) 5 6))

'(#<syntax:eval:20:0 1>

  #<syntax:eval:20:0 (+ 3 4)>

  #<syntax:eval:20:0 5>

  #<syntax:eval:20:0 6>)

> (syntax->list #'a)

#f

procedure

(syntax->datum stx) → any/c

  stx : syntax?
通过剥离 stx 的词法信息、源位置信息、属性和篡改状态,返回一个数据集。 在成对的、(不可变的)向量、(不可变的)盒子、不可变的 hash table 值(不是键)和不可变的 prefab 结构中, syntax object 被递归剥离。

剥离操作并不改变 stx; 它根据需要创建新的对、向量、盒、哈希表和 prefab 结构, 以递归地剥离词法和源位置信息。

Examples:
> (syntax->datum #'a)

'a

> (syntax->datum #'(x . y))

'(x . y)

> (syntax->datum #'#(1 2 (+ 3 4)))

'#(1 2 (+ 3 4))

> (syntax->datum #'#&"hello world")

'#&"hello world"

> (syntax->datum #'#hash((imperial . "yellow") (festival . "green")))

'#hash((festival . "green") (imperial . "yellow"))

> (syntax->datum #'#(point 3 4))

'#(point 3 4)

> (syntax->datum #'3)

3

> (syntax->datum #'"three")

"three"

> (syntax->datum #'#t)

#t

procedure

(datum->syntax ctxt v [srcloc prop ignored]) → syntax?

  ctxt : (or/c syntax? #f)
  v : any/c
  srcloc : 
(or/c #f
      syntax?
      srcloc?
      (list/c any/c
              (or/c exact-positive-integer? #f)
              (or/c exact-nonnegative-integer? #f)
              (or/c exact-positive-integer? #f)
              (or/c exact-nonnegative-integer? #f))
      (vector/c any/c
               (or/c exact-positive-integer? #f)
               (or/c exact-nonnegative-integer? #f)
               (or/c exact-positive-integer? #f)
               (or/c exact-nonnegative-integer? #f)))
   = #f
  prop : (or/c syntax? #f) = #f
  ignored : (or/c syntax? #f) = #f
将 datum v 转换为一个 syntax object 。 如果 v 已经是一个 syntax object ,那么就不需要转换, v 会被原封不动的返回。 对、向量和盒子的内容, prefab 结构的字段,以及不可变的哈希表的值都会被递归转换。 prefab 结构的键和不可变的哈希表的键不被转换。 可变的向量和盒子被不可变的向量和盒子取代。 对于除对、向量、盒、不可变 hash table, 不可变 prefab 结构或 syntax object 之外的任何类型的值, 转换意味着在通过 datum-intern-literal 将值 interned 之后,用词法信息、源位置信息和属性来包裹该值。

v 中的转换对象被赋予 ctxt 的词法上下文信息和 srcloc 的源位置信息。 转换后的直接 syntax object 被赋予 prop 的属性(见 语法对象属性) (甚至是通过 syntax-property-symbol-keys 不可见的隐藏属性); 如果 v 是一个对、向量、盒、不可变 hash table 或不可变 prefab 结构, 递归转换的值不被赋予属性。 如果 ctxt 有 tainted ,那么从 datum->syntax 产生的语法对象也 tainted 。 ctxt 的 code inspector (如果有的话)与当前被转换的宏的模块的代码检查器(如果有的话)进行比较; 如果两个检查器都可用,并且其中一个与另一个相同或较差,那么结果语法具有相同/较差的检查器,否则它没有代码检查器。

ctxt, srcloc 或 prop 中的任何一个都可以是 #f, 在这种情况下,结果语法没有词法上下文、源信息和/或新属性。

如果 srcloc 不是 #f, srcloc 实例或 syntax object , 它必须是一个列表或向量,由五个元素组成,对应于 srcloc 字段。

在将 v 转换为 syntax object 时,图的结构并没有被保留下来。 相反, v 本质上被展开成一棵树。 如果 v 有一个通过对、向量、盒子、不可变 hash tables 和不可变 prefab 结构的循环, 那么就会引发 exn:fail:contract exception is raised 。

ignored 的参数是为了向后兼容而允许的,对返回的语法对象没有影响。

Changed in version 8.2.0.5 of package base: 允许一个 srcloc 值作为 srcloc 参数。

syntax-binding-set-extend 的前三个参数在 phase 上建立了 symbol 与 mpi 所引用的模块中定义的标识符的绑定。 提供 source-symbol 以使 symbol 的绑定指向 mpi 提供的不同变量,以此类推; 可选的参数对应于 identifier-binding 的结果。

Added in version 7.0.0.12 of package base.

procedure

(datum-intern-literal v) → any/c

  v : any/c
将一些值转换为与默认阅读器在 read-syntax 模式下产生的 interned 结果一致。

如果 v 是一个 number, character, string, byte string 或 regular expression, 那么结果是一个与 v equal? 的值, 并且与默认阅读器的潜在结果 eq? 。 (注意,可变的字符串和字节字符串被 interned 为不可变的字符串和字节字符串)。

如果 v 是一个 uninterned 或 unreadable symbol, 结果仍然是 v, 因为一个 interned 的符号不会与 v equal? 。

转换过程不会遍历复合值。 例如,如果 v 是一个包含字符串的 pair ,那么 v 内的字符串就不是 interned 。

如果 v1 和 v2 是 equal? ,但不是 eq?, 那么有可能 (datum-intern-literal v1) 会返回 v1 , 并且 — 在 v1 由垃圾收集器(见 垃圾回收)决定变得不可达之后 —(datum-intern-literal v2) 仍然可以返回 v2 。 换句话说, datum-intern-literal 可以采用一个给定的值作为 interned 代表, 但是如果以前的代表在其他方面变得不可达,那么 datum-intern-literal 可以采用一个新的代表。

procedure

(syntax-shift-phase-level stx shift) → syntax?

  stx : syntax?
  shift : (or/c exact-integer? #f)
返回一个类似于 stx 的语法对象,但是它的所有顶层和模块的绑定都被 shift phase levels 。 如果 shift 是 #f, 那么只有 phase level 为 0 的绑定被转移到 label phase level 。 如果 shift 是 0, 那么结果就是 stx 。

procedure

(generate-temporaries stx-pair) → (listof identifier?)

  stx-pair : (or syntax? list?)
返回一个与所有其他标识符不同的标识符列表。 列表包含的标识符数量与 stx-pair 包含的元素数量相同。 stx-pair 参数必须是一个语法对,可以被平铺成一个列表。 stx-pair 的元素可以是任何东西, 但是字符串、符号、关键字(可能被包装成语法)和标识符元素将被嵌入到相应的生成名称中,这对调试很有用。

生成的标识符是用内部符号(而不是 gensym )构建的;另见 打印编译后的代码 。

Examples:
> (generate-temporaries '(a b c d))

'(#<syntax a1> #<syntax b2> #<syntax c3> #<syntax d4>)

> (generate-temporaries #'(1 2 3 4))

'(#<syntax temp5> #<syntax temp6> #<syntax temp7> #<syntax temp8>)

> (define-syntax (set!-values stx)
    (syntax-case stx ()
      [(_ (id ...) expr)
       (with-syntax ([(temp ...) (generate-temporaries #'(id ...))])
         #'(let-values ([(temp ...) expr])
             (set! id temp) ... (void)))]))

procedure

(identifier-prune-lexical-context id-stx    
  [syms]) → identifier?
  id-stx : identifier?
  syms : (listof symbol?) = (list (syntax-e id-stx))
返回一个与 id-stx 相同的标识符,但可能没有来自 id-stx 的词法信息, 该信息不适用于 syms 中的符号,甚至进一步扩展词法信息会丢掉其他符号的信息。 特别是,通过 datum->syntax 将这个函数的结果中的词法上下文转移到 syms 中的一个符号之外, 可能会产生一个没有约束力的标识符。

目前,其结果总是 id-stx 的精确。 修剪的目的主要是在Racket的前一个版本中作为一种优化,但在目前的宏扩展器中,它的作用不大,而且很难有效地实现。

参见 quote-syntax/prune 。

Changed in version 6.5 of package base: 总是返回 id-stx 。

procedure

(identifier-prune-to-source-module id-stx) → identifier?

返回一个标识符,其词法上下文被最小化为 syntax-source-module 所需的词法。 最小化的词法上下文不包括任何绑定。

procedure

(syntax-debug-info stx [phase all-bindings?]) → hash?

  stx : syntax?
  phase : (or/c exact-integer? #f) = (syntax-local-phase-level)
  all-bindings? : any/c = #f
产生一个哈希表,描述 stx 的 lexical information (当 (syntax-e stx) 会返回一个复合值时,不计算组件)。 其结果可以包括–但不限于–以下键:

  • 'name — (syntax-e stx) 的结果,如果它是一个符号。

  • 'context — 一个向量的列表,每个向量代表连接到 stx 的一个作用域。

    每个向量以一个数字开始,这个数字对每个作用域都是不同的。 后面的符号提供了作用域的来源提示: 'module 表示 module 作用域, 'macro 表示宏介绍作用域, 'use-site 表示宏使用区作用域, 或者 'local 表示局部绑定形式。 在 'module 对应于内侧边缘的情况下,将显示模块的名称和一个阶段(因为每个阶段都会生成一个内侧边缘作用域)。

  • 'bindings — 绑定的列表,每个绑定都由一个哈希表表示。 绑定表可以包括–但不限于–以下键:

    • 'name — 绑定的符号名称。

    • 'context — 绑定的作用域,作为一个向量的列表。

    • 'local — 一个代表 local binding 的符号; 当这个键存在时, 'module 不存在。

    • 'module — 一个从另一个模块导入的编码;当这个键存在时, 'local 就不存在。

    • 'free-identifier=? — 是来自标识符的调试信息的哈希表,该标识符的绑定是一个别名。

  • 'fallbacks — 一个类似于 syntax-debug-info 产生的跨命名空间绑定回退的哈希表的列表。

Added in version 6.3 of package base.

返回 syntax object stx 的 source location ,如果不知道,则返回 #f 。
