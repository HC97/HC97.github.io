4.18 集合

一个 set 表示一个不同元素的集合。以下的数据类型都是集合:

  • hash sets;

  • 使用 equal? 来比较元素的 lists ;以及

  • structures ,其类型实现了 gen:set generic interface 。

 (require racket/set) package: base
The bindings documented in this section are provided by the racket/set and racket libraries, but not racket/base.

4.18.1 哈希集

hash set 是一个集合,其元素通过 equal?, equal-always?, eqv? 或 eq? 进行比较, 并通过 equal-hash-code, equal-always-hash-code, eqv-hash-code 或 eq-hash-code 进行分割。 一个哈希集要么是不可变的,要么是可变的;可变的哈希集可以强势或弱势地保留其元素。

像对不可变的哈希表的操作一样,"恒定时间" 的哈希集操作对于大小为 N 的集合来说实际上需要 O(log N) 的时间。

一个哈希集可以作为一个 stream (参见 Streams) , 从而作为一个单值 sequence (参见 Sequences) 使用。 集的元素作为流或序列的元素。 如果在迭代过程中,一个元素被添加到哈希集或从哈希集中移除, 那么迭代步骤可能以 exn:fail:contract 的方式失败,或者迭代可能跳过或重复元素。 参见 in-set 。

当两个哈希集使用相同的元素比较程序 (equal?, equal-always?, eqv? 或 eq?),都强或弱地持有元素,具有相同的可变性,并且具有相等的元素时,它们是 equal? 。 不可变哈希集支持有效的恒定时间访问和更新,就像可变哈希集一样;不可变操作的常数通常更大, 但不可变哈希集的功能性质在某些算法中可以得到回报。

所有的哈希集都 实现 set->stream, set-empty?, set-member?, set-count, subset?, proper-subset?, set-map, set-for-each, set-copy, set-copy-clear, set->list 和 set-first 。 不可变的哈希集还 实现 set-add, set-remove, set-clear, set-union, set-intersect, set-subtract 和 set-symmetric-difference 。 可变散列集还 实现 set-add!, set-remove!, set-clear!, set-union!, set-intersect!, set-subtract! 和 set-symmetric-difference! 。

对包含变异元素的集合的操作是不可预测的,就像 hash table 的操作在键变异时是不可预测的一样。

4.18.2 设置谓词和契约

procedure

(generic-set? v) → boolean?

  v : any/c
如果 v 是一个 set,则返回 #t ;否则返回 #f 。

Examples:
> (generic-set? (list 1 2 3))

#t

> (generic-set? (set 1 2 3))

#t

> (generic-set? (mutable-seteq 1 2 3))

#t

> (generic-set? (vector 1 2 3))

#f

procedure

(set-implements? st sym ...) → boolean?

  st : generic-set?
  sym : symbol?
如果 st 实现了 gen:set 中以 sym 命名的所有方法,则返回 #t ; 否则返回 #f 。 回退实现不会影响结果; st 可以通过回退实现来支持给定的方法,但会生成 #f 。

Examples:
> (set-implements? (list 1 2 3) 'set-add)

#t

> (set-implements? (list 1 2 3) 'set-add!)

#f

> (set-implements? (set 1 2 3) 'set-add)

#t

> (set-implements? (set 1 2 3) 'set-add!)

#t

> (set-implements? (mutable-seteq 1 2 3) 'set-add)

#t

> (set-implements? (mutable-seteq 1 2 3) 'set-add!)

#t

> (set-implements? (weak-seteqv 1 2 3) 'set-remove 'set-remove!)

#t

procedure

(set-implements/c sym ...) → flat-contract?

  sym : symbol?
识别支持 gen:set 中以 sym 命名的所有方法的集。

procedure

(set/c elem/c    
  [#:cmp cmp    
  #:kind kind    
  #:lazy? lazy?    
  #:equal-key/c equal-key/c]) → contract?
  elem/c : chaperone-contract?
  cmp : (or/c 'dont-care 'equal 'equal-always 'eqv 'eq)
   = 'dont-care
  kind : (or/c 'dont-care 'immutable 'mutable 'weak 'mutable-or-weak)
   = 'immutable
  lazy? : any/c = 
(not (and (equal? kind 'immutable)
          (flat-contract? elem/c)))
  equal-key/c : contract? = any/c
构造一个契约来识别元素与 elem/c 匹配的集合。

如果 kind 是 'immutable, 'mutable 或 'weak ,则生成的合约只接受分别为不可变、可变强持有键或可变弱持有键的 hash sets 。 如果 kind 是 'mutable-or-weak, 则生成的合约接受任何可变 hash sets, 而不管键持有强度如何。

如果 cmp 是 'equal, 'equal-always, 'eqv 或 'eq, 则生成的合约只接受使用 hash sets , 比较元素分别使用 equal?, equal-always?, eqv? 或 eq? 。

如果 cmp 是 'eqv 或 'eq, 那么 elem/c 必须是一个 flat contract 。

如果 cmp 和 kind 都是 'dont-care, 那么生成的合约将接受任何类型的集合,而不仅仅是 hash sets 。

如果 lazy? 不是 #f ,那么契约不会立即检查 set 的元素,只检查 set 本身(根据 cmp 和 kind 参数)。 如果 lazy? 是 #f ,则契约将立即检查元素。 当 set 契约接受泛型集时, lazy? 参数将被忽略 (即当 cmp 和 kind 都是 'dont-care 时); 在这种情况下,正在检查的值在这种情况下是一个 list? ,则契约不是懒惰的,否则契约是懒惰的。

如果 kind 允许可变集(即,是 'dont-care, 'mutable, 'weak 或 'mutable-or-weak) 和 lazy? 是 #f, 则立即检查元素,并在从集合中访问它们时检查元素。

当值被传递给内部使用的比较和散列函数时,使用 equal-key/c 契约。

当 elem/c 和 equal-key/c 都是 flat contracts 时, 结果合约将是 flat contract ， lazy? 是 #f, 而 kind 是 'immutable 。 当 elem/c 是一个 chaperone contract 时, 其结果将是一个 chaperone contract 。

Changed in version 8.3.0.9 of package base: 增加了随机生成的支持。
Changed in version 8.5.0.3: 增加了 'equal-always 以支持 cmp 。

4.18.3 通用集接口

syntax

gen:set

一个 generic interface (参见 通用接口) , 通过 struct 定义的 #:methods 选项为结构类型提供 set 方法实现。 此接口可用于实现记录为 Set 方法 。

Examples:
> (struct binary-set [integer]
    #:transparent
    #:methods gen:set
    [(define (set-member? st i)
       (bitwise-bit-set? (binary-set-integer st) i))
     (define (set-add st i)
       (binary-set (bitwise-ior (binary-set-integer st)
                                (arithmetic-shift 1 i))))
     (define (set-remove st i)
       (binary-set (bitwise-and (binary-set-integer st)
                                (bitwise-not (arithmetic-shift 1 i)))))])
> (define bset (binary-set 5))
> bset

(binary-set 5)

> (generic-set? bset)

#t

> (set-member? bset 0)

#t

> (set-member? bset 1)

#f

> (set-member? bset 2)

#t

> (set-add bset 4)

(binary-set 21)

> (set-remove bset 2)

(binary-set 1)

4.18.3.1 Set 方法

gen:set 的方法可以被分为三类,这是由它们的回退实现决定的:

  1. 没有回退的方法,

  2. 回退依赖其他非回退方法的方法,

  3. 以及其回退可以依赖于回退或非回退方法的方法。

作为一个例子,实现以下方法将保证 gen:set 中的所有方法至少有一个回退方法:

  • set-member?

  • set-add

  • set-add!

  • set-remove

  • set-remove!

  • set-first

  • set-empty?

  • set-copy-clear

可能还有其他这样的方法子集,可以保证每个方法至少有一个回退。

procedure

(set-member? st v) → boolean?

  st : generic-set?
  v : any/c
如果 v 在 st 中,返回 #t ,否则返回 #f 。没有退路。

procedure

(set-add st v) → generic-set?

  st : generic-set?
  v : any/c
产生一个包括 v 和 st 所有元素的集合。 对于 hash sets 来说,这个操作的运行时间是恒定的。没有后退。

procedure

(set-add! st v) → void?

  st : generic-set?
  v : any/c
将元素 v 添加到 st 中。 这个操作对 hash sets 来说是以恒定时间运行的。没有后退。

For hash sets, see also the caveats concerning concurrent modification for hash tables, which applies to hash sets.

procedure

(set-remove st v) → generic-set?

  st : generic-set?
  v : any/c
产生一个集合,其中包括 st 中除 v 以外的所有元素。 对于 hash sets 来说,这个操作的运行时间是恒定的。没有退路。

procedure

(set-remove! st v) → void?

  st : generic-set?
  v : any/c
从 st 中移除元素 v 。 这个操作对 hash sets 来说是以恒定时间运行的。 没有回退。

For hash sets, see also the caveats concerning concurrent modification for hash tables, which applies to hash sets.

procedure

(set-empty? st) → boolean?

  st : generic-set?
如果 st 没有成员,返回 #t ;否则返回 #f。

支持任何 implements set->stream 或 set-count 的 st 。

procedure

(set-count st) → exact-nonnegative-integer?

  st : generic-set?
返回 st 中元素的数量。

支持任何 supports set->stream 的 st 。

procedure

(set-first st) → any/c

  st : (and/c generic-set? (not/c set-empty?))
产生 st 的一个未指定的元素。 在 st 上多次使用 set-first 会产生相同的结果。

支持任何 implements set->stream 的 st 。

procedure

(set-rest st) → generic-set?

  st : (and/c generic-set? (not/c set-empty?))
生成一个集合,其中包含 st 的所有元素,除了 (set-first st) 。

支持任何 implements set-remove 与 set-first 或 set->stream 的 st 。

procedure

(set->stream st) → stream?

  st : generic-set?
生成包含 st 元素的流。

支持任何 implements

  • set->list

  • in-set

  • set-empty?, set-first, set-rest

  • set-empty?, set-first, set-remove

  • set-count, set-first, set-rest

  • set-count, set-first, set-remove

的 st

procedure

(set-copy st) → generic-set?

  st : generic-set?
生成一个与 st 具有相同类型和相同元素的新的可变集合。

支持任何 supports set->stream 并 implements set-copy-clear 和 set-add! 的 st 。

procedure

(set-copy-clear st) → (and/c generic-set? set-empty?)
