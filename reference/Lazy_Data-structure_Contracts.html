8.4 Lazy Data-structure Contracts

syntax

(contract-struct id (field-id ...))

NOTE: This library is deprecated; use struct, instead. 懒惰结构合约不再需要单独的结构声明;相反, struct/dc 和 struct/c 直接与 struct 和 define-struct 一起工作。

与 struct 类似,但有两点不同:它们不定义字段突变器,并且定义了两个契约构造函数: id/c 和 id/dc 。 第一个是一个过程,它接受与字段数量相同的参数,并返回字段与参数匹配的结构值的契约。 第二个是一种语法形式,它也产生结构的契约,但是后面的字段的契约可能取决于前面字段的值。

生成的契约组合器是 lazy: 它们只验证契约对某些数据结构中实际被检查的部分是否成立。 更确切地说,在选择器提取结构的一个字段之前,懒惰的数据结构契约不会被检查。

(id/dc field-spec ...)
 
field-spec = [field-id contract-expr]
  | [field-id (field-id ...) contract-expr]

在每个 field-spec 的情况下,第一个 field-id 指定了合同适用于哪个字段; 字段必须按照原始 contract-struct 的顺序来指定。 第一种情况是,当字段上的契约不依赖于任何其他字段的值时。 第二种情况是当字段上的契约确实依赖于其他一些字段时, 括号中的 field-id 指出它依赖于哪些字段;这些依赖关系只能是早期的字段。

syntax

(define-contract-struct id (field-id ...))

NOTE: This library is deprecated; use struct, instead. 懒惰的结构契约不再需要单独的结构声明;相反, struct/dc 和 struct/c 直接与 struct 和 define-struct 一起工作。

与 contract-struct 类似,但构造函数的名称是 make-id, 与 define-struct 非常相似。
