8.1 Data-structure Contracts

procedure

(flat-contract-with-explanation get-explanation 
  [#:name name]) 
 → flat-contract?
  get-explanation : (-> any/c (or/c boolean? (-> blame? any)))
  name : any/c = (object-name get-explanation)
提供了一种使用平面合约的方法,当合约失败时,提供更多关于失败的信息。

如果 get-explanation 返回一个布尔值,那么这个布尔值就被当作 flat contract 中的谓词。 如果它返回一个存储过程,那么它的处理方式与返回 #f 类似,只是调用结果存储过程来实际发出违反契约的信号。

name 参数被用作合同的名称;它默认为 get-explanation 函数的名称。

(flat-contract-with-explanation
 (λ (val)
   (cond
     [(even? val) #t]
     [else
      (λ (blame)
        (define more-information ...do-some-complex-computation-here...)
        (raise-blame-error blame val
                           '(expected: "an even number" given: "~e"
                                       "and, here is more help: ~s")
                           val more-information))])))

procedure

(flat-named-contract name    
  flat-contract    
  [generator]) → flat-contract?
  name : any/c
  flat-contract : flat-contract?
  generator : (or/c #f (-> exact-nonnegative-integer? (-> any/c)))
   = #f
产生一个像 flat-contract 一样的 flat contract ,但名称为 name 。

例如,
(define/contract i
  (flat-named-contract
   'odd-integer
   (lambda (x) (and (integer? x) (odd? x))))
  2)

generator 参数为 flat-named-contract 添加了一个生成器。 更多信息参见 contract-random-generate 。

value

any/c : flat-contract?

一个接受任何值的 flat contract 。

当使用这个契约作为函数契约的结果部分时,考虑使用 any 代替; 使用 any 会导致更好的内存性能,但它也允许多个结果。

value

none/c : flat-contract?

一个不接受任何值的 flat contract 。

procedure

(or/c contract ...) → contract?

  contract : contract?
接受任意数量的 contracts ,并返回一个 contract , 这个 contract 接受任意一个契约单独接受的任何值。

or/c 的结果是通过从左到右依次应用合约来测试任何值, 但例外的是它总是将非 flat contracts (如果有的话)移到最后,最后检查它们。 因此,像 (or/c (not/c real?) positive?) 这样的契约保证只在实数上调用 positive? 谓词。

如果所有的参数都是程序或 flat contracts ,结果就是一个 flat contract 。 如果只有一个参数是高阶合约,那么结果就是一个只检查平坦合约的合约,如果它们没有通过,就应用高阶合约。

如果有多个高阶合约, or/c 使用 contract-first-order-passes? 来区分它们。 更确切地说,当一个 or/c 被检查时,它首先检查所有的 flat contracts 。 如果它们都没有通过,它就用每个高阶合约调用 contract-first-order-passes? 。 如果只有一个返回真, or/c 就使用该合约。 如果它们都没有返回真,它就发出一个违反合约的信号。 如果多于一个返回真,它也会发出违反合同的信号。例如,这个契约
(or/c (-> number? number?)
      (-> string? string? string?))
不接受像这样的函数: (lambda args ...) ,因为它不能告诉两个箭头合约中的哪一个应该与该函数一起使用。

如果它的所有参数都是 list-contract?, 那么 or/c 会返回一个 list-contract? 。

procedure

(first-or/c contract ...) → contract?

  contract : contract?
接受任意数量的 contracts ,并返回一个 contract ,该合约接受任意一个合约单独接受的任何值。

first-or/c 的结果是通过从左到右的顺序应用合同来测试任何价值。 因此,像 (first-or/c (not/c real?) positive?) 这样的契约保证只在实数上调用 positive? 谓词。

如果所有的参数都是程序或 flat contracts,结果就是 flat contract , 同样,如果所有的参数都是 chaperone contracts ,结果也是如此。 否则,结果是一个 impersonator contract 。

如果有多个高阶合约, first-or/c 使用 contract-first-order-passes? 来区分它们。 更确切地说,当一个 first-or/c 被检查时,它检查第一个合约的第一阶传递与值。 如果它成功了,那么它就只使用那个合约。 如果它失败了,那么它就转到第二个合约,继续下去,直到它找到一个一阶检查成功的合约。 如果没有一个成功,就会发出一个违反合同的信号。

例如,这个合同
(first-or/c (-> number? number?)
       (-> string? string? string?))
接受了函数 (λ args 0), 将 (-> number? number?) 契约应用于该函数,因为它排在第一位, 尽管 (-> string? string? string?) 也适用。

如果它的所有参数都是 list-contract?, 那么 first-or/c 返回一个 list-contract? 。

procedure

(and/c contract ...) → contract?

  contract : contract?
接收任意数量的 contracts ,并返回一个接受同时满足所有合约的任何值的 contract 。

如果所有的参数都是程序或 flat contracts ,那么结果就是一个 flat contract 。

and/c 产生的合约通过从左到右依次应用这些合约来测试任何值。

这意味着 and/c 可以用来保护那些在合约中不完全的谓词。 例如,这个契约是乖巧的,正确地责备了 whoops-not-a-number 的定义不是一个数字:

Example:
> (define/contract whoops-not-a-number
    (and/c real? even?)
    "four")

  produced: "four"

  in: an and/c case of

      (and/c real? even?)

  contract from:

      (definition whoops-not-a-number)

  blaming: (definition whoops-not-a-number)

   (assuming the contract is correct)

  at: eval:2:0

但如果 and/c 的参数是相反的,那么合同本身就会产生一个错误:

Example:
> (define/contract whoops-not-a-number
    (and/c even? real?)
    "four")

even?: contract violation

  expected: integer?

  given: "four"

如果超过一个合同不是 flat contracts ,那么合同的高阶部分被测试的顺序可能是反直觉的。 作为一个例子,考虑一下这个函数,它以高阶方式使用 and/c , 其合约总是成功的,但当它们被调用时,会打印出来,以使我们看到它们被调用的顺序。

Examples:
> (define ((show-me n) x)
    (printf "show-me ~a\n" n)
    #t)
> (define/contract identity-with-complex-printing-contract
    (and/c (-> (show-me 4) (show-me 5))
           (-> (show-me 3) (show-me 6))
           (-> (show-me 2) (show-me 7))
           (-> (show-me 1) (show-me 8)))
    (λ (x) x))
> (identity-with-complex-printing-contract 101)

show-me 5

show-me 6

101

检查的顺序就像合同被双层包裹时的通常顺序一样。 首先被套上的合约,其域的检查在第二位,但其范围的检查在第一位,我们在这个例子中看到了类似的模式, 因为 and/c 简单地按顺序应用了这些合约。

procedure

(one-of/c v ...+) → flat-contract?

  v : any/c
接受任意数量的原子值,并返回一个能识别这些值的 flat contract ,使用 eqv? 作为比较谓词。 为了 one-of/c 的目的,原子值被定义为: characters, symbols, booleans, null, keywords, numbers, #<void> 和 #<undefined> 。

procedure

(vectorof c    
  [#:immutable immutable    
  #:flat? flat?    
  #:eager eager]) → contract?
  c : contract?
  immutable : (or/c #t #f 'dont-care) = 'dont-care
  flat? : boolean? = #f
  eager : (or/c #t #f exact-nonnegative-integer?) = #t
返回一个能够识别向量的 contract 。 矢量的元素必须与 c 匹配。

如果 flat? 参数是 #t, 那么得到的合约就是一个 flat contract, 而且 c 参数也必须是一个 flat contract 。 如果应用于可变向量,这样的 flat contracts 将是不健全的,因为它们不会检查对向量的未来操作。

如果 immutable 参数是 #t ， c 是一个 flat contract , 并且 eager 参数是 #t, 结果将是一个 flat contract 。 如果 c 参数是一个 chaperone contract ,那么结果将是一个 chaperone contract 。

如果 eager 参数是 #t , 那么当 c 是一个 flat contract 时,不可变的向量会被急切地检查。 如果 eager 参数是一个数字 n , 那么当 c 是一个 flat contract 并且向量的长度小于或等于 n 时,不可变的向量会被紧急检查。

当一个高阶 vectorof 合约应用于一个向量时,其结果不会与输入值 eq? 。 对于不可变的向量,结果将是一个拷贝,对于可变的向量,结果将是输入的 chaperone 或 impersonator , 除非 c 参数是一个 flat contract ,并且向量是不可变的,在这种情况下,结果是原始向量。

Changed in version 6.3.0.5 of package base: 修改了扁平向量合约,使其不复制不可变的向量。
Changed in version 6.7.0.3: 增加了 #:eager 选项。

请注意,大多数 flat contracts 可以直接作为谓词使用,但不是所有的。 这个函数可以用来为普通的 Racket 值建立谓词,这些值可以兼作合约,比如数字和符号。 当构建一个需要显式转换普通 Racket 值为平面合约的 contract combinator 时,考虑使用 coerce-flat-contract 而不是 flat-contract-predicate , 这样组合器可以引发错误,在错误信息中使用组合器的名字。

procedure

(property/c accessor ctc [#:name name]) → flat-contract?

  accessor : (-> any/c any/c)
  ctc : flat-contract?
  name : any/c = (object-name accessor)
构造一个 flat contract ,检查由 accessor 访问的一阶属性是否满足 ctc 。 得到的契约等同于

(lambda (v) (ctc (accessor v)))

除了在违反契约时产生的错误信息中包含更多的信息。 name 参数用来描述错误信息中被检查的属性。

Examples:
> (define/contract (sum-triple lst)
    (-> (and/c (listof number?)
               (property/c length (=/c 3)))
        number?)
    (+ (first lst) (second lst) (third lst)))
> (sum-triple '(1 2 3))

6

> (sum-triple '(1 2))

procedure

(suggest/c c field message) → contract?

  c : contract?
  field : string?
  message : string?
返回一个行为类似于 c 的契约,除了它在违反契约的错误信息中增加一行。

field 和 message 是按照 错误信息约定 中的指南添加的。

> (define allow-calls? #f)
> (define/contract (f)
    (suggest/c (->* () #:pre allow-calls? any)
               "suggestion" "maybe you should set! allow-calls? to #t")
    5)
