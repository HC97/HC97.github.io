6.6 Traits

 (require racket/trait) package: base
The bindings documented in this section are provided by the racket/trait library, not racket/base or racket.

一个 trait 是一个方法的集合,它可以被转换为一个 mixin ,然后应用于一个 class 。 在一个 trait 被转换 为mixin 之前,trait 的方法可以被单独重命名,并且多个 trait 可以被合并成一个新的 trait。

syntax

(trait trait-clause ...)

 
trait-clause = (public maybe-renamed ...)
  | (pubment maybe-renamed ...)
  | (public-final maybe-renamed ...)
  | (override maybe-renamed ...)
  | (overment maybe-renamed ...)
  | (override-final maybe-renamed ...)
  | (augment maybe-renamed ...)
  | (augride maybe-renamed ...)
  | (augment-final maybe-renamed ...)
  | (inherit maybe-renamed ...)
  | (inherit/super maybe-renamed ...)
  | (inherit/inner maybe-renamed ...)
  | method-definition
  | (field field-declaration ...)
  | (inherit-field maybe-renamed ...)
创建一个 trait 。 trait 形式的主体与 class* 形式的主体类似,但仅限于非私有的方法定义。 特别是, maybe-renamed, method-definition 和 field-declaration 的语法与 class* 相同, 每个 method-definition 都必须有一个相应的声明( public, override 等之一)。 和 class 一样,在直接调用、 super 调用和 inner 调用中使用方法名取决于通过 inherit, inherit/super, inherit/inner 和同一特质中的其他方法声明将方法名带入范围; 与 class 相比,一个例外是 overment 只在相应方法中绑定方法名,而不是在同一特质的其他方法中。 最后,诸如 public* 和 define/public 这样的宏在 trait 中的作用和在 class 中一样。

trait, trait-exclude, trait-exclude-field, trait-alias, trait-rename 和 trait-rename-field 形式的外部标识符要通过 define-member-name 和 define-local-member-name 来绑定。 尽管 trait 形式中不允许有 private 方法或字段, 但可以通过使用 public 或 field 声明和一个范围仅限于 trait 形式的名称来模拟它们。

procedure

(trait? v) → boolean?

  v : any/c
如果 v 是一个trait,返回 #t ,否则返回 #f 。

procedure

(trait->mixin tr) → (class? . -> . class?)

  tr : trait?
将 trait 转换为 mixin,它可以应用于一个 class 来产生一个新的 class 。 一个表达式的形式是

(trait->mixin
 (trait
   trait-clause ...))

等同于

(lambda (%)
  (class %
    trait-clause ...
    (super-new)))

然而,通常情况下,一个 trait 的方法在转换为 mixin 之前会被改变并与其他 trait 结合。

procedure

(trait-sum tr ...+) → trait?

  tr : trait?
产生一个 trait ,结合给定的 tr 的所有方法。例如,

(define t1
  (trait
    (define/public (m1) 1)))
(define t2
  (trait
    (define/public (m2) 2)))
(define t3 (trait-sum t1 t2))

创建一个 trait t3 ,它等同于

(trait
  (define/public (m1) 1)
  (define/public (m2) 2))

但 t1 和 t2 仍然可以单独使用或与其他 trait 组合使用。

当 trait 与 trait-sum 组合时,当另一个 trait 为相同的方法或字段名提供定义时, 该组合将放弃 inherit, inherit/super, inherit/inner 和 inherit-field 的声明。 如果任何要合并的特征定义了相同名称的方法或字段, 或者要放弃的 inherit/super 或 inherit/inner 声明与提供的定义不一致, 那么 trait-sum 操作就会失败(引发 exn:fail:contract exception is raised) 。 换句话说,用 inherit, inherit/super 或 inherit/inner, 声明一个方法不算是定义该方法; 同时,例如,一个包含方法 m 的 inherit/super 声明的trait不能与定义 m 为 augment 的 trait 结合, 因为当这个 trait 后来被转换为 mixin 并应用于一个类时, 没有一个类可以同时满足 augment 和 inherit/super 的要求。

syntax

(trait-exclude trait-expr id)

产生一个新的 trait ,与 trait-expr 的 trait 结果相似, 但删除了 id 命名的方法的定义;随着方法定义的删除, 会添加一个 inherit, inherit/super 或 inherit/inner 声明:

  • 用 public, pubment 或 public-final 声明的方法被替换为 inherit 声明。

  • 用 override 或 override-final 声明的方法被替换为 inherit/super 声明。

  • 用 augment, augride 或 augment-final 声明的方法被替换为 inherit/inner 声明。

  • 用 overment 声明的方法不会被任何 inherit 声明取代。

如果由 trait-expr 产生的 trait 中没有 id 的方法定义, 则会引发 exn:fail:contract exception is raised 。

syntax

(trait-exclude-field trait-expr id)

产生一个新的 trait ,它与 trait-expr 的 trait 结果相似, 但删除了以 id 命名的字段的定义; 随着字段定义的删除,一个 inherit-field 声明被添加。

syntax

(trait-alias trait-expr id new-id)

产生一个新的 trait ,与 trait-expr 的 trait 结果一样, 但是 id 命名的方法的定义和声明被重复,名称为 new-id 。 对产生的特征的一致性要求与 trait-sum 相同,否则会引发 exn:fail:contract exception is raised 。 这个操作不会重命名任何其他 id 的使用,比如在方法调用中 (甚至在克隆的 new-id 定义中对 identifier 的方法调用)。

syntax

(trait-rename trait-expr id new-id)

产生一个新的 trait ,它和 trait-expr 的 trait 结果一样, 但是所有定义和对名为 id 的方法的引用都被定义和对名为 new-id 的方法的引用所取代。 对结果trait的一致性要求与 trait-sum 相同,否则会引发 exn:fail:contract exception is raised 。

syntax

(trait-rename-field trait-expr id new-id)

产生一个新的 trait ,与 trait-expr 的 trait 结果一样, 但所有定义和对 id 字段的引用都被定义和对 new-id 字段的引用所取代。 对结果 trait 的一致性要求与 trait-sum 相同,否则会引发 exn:fail:contract exception is raised 。
