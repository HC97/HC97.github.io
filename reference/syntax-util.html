12.12 语法实用工具

 (require racket/syntax) package: base
The bindings documented in this section are provided by the racket/syntax library, not racket/base or racket.

12.12.1 创建格式化的标识符

procedure

(format-id lctx    
  fmt    
  v ...    
  [#:source src    
  #:props props    
  #:cert ignored    
  #:subs? subs?    
  #:subs-intro subs-introducer]) → identifier?
  lctx : (or/c syntax? #f)
  fmt : string?
  v : 
(or/c string? symbol? keyword? char? number?
      (syntax/c (or/c string? symbol? keyword? char? number?)))
  src : (or/c syntax? #f) = #f
  props : (or/c syntax? #f) = #f
  ignored : (or/c syntax? #f) = #f
  subs? : boolean? = #f
  subs-introducer : (-> syntax? syntax?)
   = (if (syntax-transforming?) syntax-local-introduce values)
像 format 一样,但是产生一个标识符,使用 lctx 表示词法上下文, src 表示源位置, props 表示属性。 用 #:cert 提供的参数会被忽略。(参见 datum->syntax 。)

格式字符串必须只使用 ~a 占位符。 参数列表中的语法对象会被自动解包(例如,标识符会被自动转换为符号)。

Examples:
> (define-syntax (make-pred stx)
    (syntax-case stx ()
      [(make-pred name)
       (format-id #'name "~a?" (syntax-e #'name))]))
> (make-pred pair)

#<procedure:pair?>

> (make-pred none-such)

none-such?: undefined;

 cannot reference an identifier before its definition

  in module: top-level

> (define-syntax (better-make-pred stx)
    (syntax-case stx ()
      [(better-make-pred name)
       (format-id #'name #:source #'name
                  "~a?" (syntax-e #'name))]))
> (better-make-pred none-such)

none-such?: undefined;

 cannot reference an identifier before its definition

  in module: top-level

(Scribble 没有显示,但是 DrRacket 可以准确地指出第二个错误的位置,但是没有指出第一个。)

如果 subs? 是 #t, 那么就会在结果中添加一个 'sub-range-binders 语法属性, 记录每个标识符在 v 中的位置。 subs-intro 程序被应用于每个标识符,其结果被包含在子范围绑定器记录中。 这个属性值覆盖了从 props 复制的 'sub-range-binders 属性。

Example:
> (syntax-property (format-id #'here "~a/~a-~a" #'point 2 #'y #:subs? #t)
                   'sub-range-binders)

'(#(#<syntax point/2-y> 8 1 0.5 0.5 #<syntax:eval:8:0 y> 0 1 0.5 0.5)

  #(#<syntax point/2-y> 0 5 0.5 0.5 #<syntax:eval:8:0 point> 0 5 0.5 0.5))

Changed in version 7.4.0.5 of package base: 增加了 #:subs? 和 #:subs-intro 参数。
Changed in version 8.7.0.7: 允许 v 是一个语法对象, 包裹一个字符串、一个关键字、一个字符或一个数字。

procedure

(format-symbol fmt v ...) → symbol?

  fmt : string?
  v : 
(or/c string? symbol? keyword? char? number?
      (syntax/c (or/c string? symbol? keyword? char? number?)))
像 format 一样,但产生一个符号。 格式字符串必须只使用 ~a 占位符。 参数列表中的语法对象会被自动解包(例如,标识符会被自动转换为符号)。

Example:
> (format-symbol "make-~a" 'triple)

'make-triple

Changed in version 8.7.0.7 of package base: 允许 v 是一个语法对象, 包裹着一个字符串、一个关键字、一个字符或一个数字。

12.12.2 模式变量

syntax

(define/with-syntax pattern stx-expr)

 
  stx-expr : syntax?
with-syntax 的定义形式。 也就是说,它将 stx-expr 的语法对象结果与 pattern 进行匹配, 并为 pattern 模式变量创建模式变量定义。

Examples:
> (define/with-syntax (px ...) #'(a b c))
> (define/with-syntax (tmp ...) (generate-temporaries #'(px ...)))
> #'([tmp px] ...)

#<syntax:eval:12:0 ((a9 a) (b10 b) (c11 c))>

> (define/with-syntax name #'Alice)
> #'(hello name)

#<syntax:eval:14:0 (hello Alice)>

12.12.3 错误报告

parameter

(current-syntax-context) → (or/c syntax? false/c)

(current-syntax-context stx) → void?
  stx : (or/c syntax? false/c)
当前的上下文语法对象,默认为 #f 。 它决定了由 wrong-syntax 创建的语法错误的特殊形式名称的前缀。

procedure

(wrong-syntax stx format-string v ...) → any

  stx : syntax?
  format-string : string?
  v : any/c
使用 (current-syntax-context) 的结果作为 "主要" 语法对象, 并使用提供的 stx 作为特定的语法对象,引发语法错误。 (后者, stx, 通常是被 DrRacket 高亮显示的对象)。 错误信息是用格式字符串和参数构建的,它的前缀是特殊的形式名称,如 current-syntax-context 中所述。

Examples:
> (wrong-syntax #'here "expected ~s" 'there)

eval:15:0: ?: expected there

  at: here

> (parameterize ([current-syntax-context #'(look over here)])
    (wrong-syntax #'here "expected ~s" 'there))

eval:16:0: look: expected there

  at: here

  in: (look over here)

一个使用 wrong-syntax 的宏可以在其转换的一开始设置语法上下文,如下所示::
(define-syntax (my-macro stx)
  (parameterize ([current-syntax-context stx])
    (syntax-case stx ()
      __)))
那么在宏的转换过程中,对 wrong-syntax 的任何调用都将引用 my-macro (更准确地说,是指在使用宏的地方引用 my-macro 的名字,由于重命名、前缀等原因,可能有所不同)。

12.12.4 记录消失的使用

parameter

(current-recorded-disappeared-uses)

 → (or/c (listof identifier?) false/c)
(current-recorded-disappeared-uses ids) → void?
  ids : (or/c (listof identifier?) false/c)
用于跟踪消失的使用的参数。 当该参数有一个非假值时,跟踪被 "启用"。 这是由 with-disappeared-uses 这样的表单自动完成的。

syntax

(with-disappeared-uses body-expr ... stx-expr)

 
  stx-expr : syntax?
评估 body-expr 和 stx-expr, 捕捉使用 syntax-local-value/record 查询的标识符。 将捕获的标识符添加到由 stx-expr 产生的语法对象的 'disappeared-use 属性中。

Changed in version 6.5.0.7 of package base: 增加了包括 body-expr 的选项。

procedure

(syntax-local-value/record id predicate) → any/c

  id : identifier?
  predicate : (-> any/c boolean?)
在句法环境中查找 id (作为 syntax-local-value)。 如果查找成功并返回一个满足谓词的值,则返回该值, 并通过调用 record-disappeared-uses 将 id 记录为一个消失的使用。 如果查找失败或者值不满足谓词,则返回 #f ,并且标识符不被记录为消失的用途。

procedure

(record-disappeared-uses id [intro?]) → void?

  id : (or/c identifier? (listof identifier?))
  intro? : boolean? = (syntax-transforming?)
将 id 添加到 (current-recorded-disappeared-uses) 。 如果 id 是一个列表,对所有的标识符执行同样的操作。 如果 intro? 为真,那么首先对标识符调用 syntax-local-introduce 。

如果不是在 with-disappeared-uses 表单或类似表单的范围内使用,则没有影响。

Changed in version 6.5.0.7 of package base: 增加了传递单个标识符的选项,而不是要求一个列表。
Changed in version 7.2.0.11: 增加了 intro? 参数。

12.12.5 其他实用工具

procedure

(generate-temporary [name-base]) → identifier?

  name-base : any/c = 'g
生成一个新的标识符。 generate-temporaries 的单数形式。 如果提供 name-base ,它将被用作标识符名称的基础。

procedure

(internal-definition-context-apply intdef-ctx    
  stx) → syntax?
  intdef-ctx : internal-definition-context?
  stx : syntax?
相当于 (internal-definition-context-introduce intdef-ctx stx 'add)。 提供 internal-definition-context-apply 函数是为了向后兼容; 首选 internal-definition-context-add-scopes 函数。

procedure

(syntax-local-eval stx [intdef-ctx]) → any

  stx : any/c
  intdef-ctx : 
(or/c internal-definition-context?
      #f
      (listof internal-definition-context?))
   = '()
将 stx 作为当前 transformer environment 中的一个表达式进行评估 (也就是在 phase level 1)。 如果 intdef-ctx 不是 #f, 为 intdef-ctx 提供的值将用于丰富 stx 的 lexical information , 并以与 local-expand 的第四个参数相同的方式扩展 local binding context 。

Examples:
> (define-syntax (show-me stx)
    (syntax-case stx ()
      [(show-me expr)
       (begin
         (printf "at compile time produces ~s\n"
                 (syntax-local-eval #'expr))
         #'(printf "at run time produces ~s\n"
                   expr))]))
> (show-me (+ 2 5))

at compile time produces 7

at run time produces 7

> (define-for-syntax fruit 'apple)
> (define fruit 'pear)
> (show-me fruit)

at compile time produces apple

at run time produces pear

Changed in version 6.90.0.27 of package base: 修改了 intdef-ctx ,除了接受单个内部定义上下文或 #f 外, 还接受内部定义上下文的列表。

syntax

(with-syntax* ([pattern stx-expr] ...)
  body ...+)
 
  stx-expr : syntax?
与 with-syntax 类似, 但每个 pattern 的模式变量都被绑定在后续子句的 stx-expr 以及 body 中, 而且这些 pattern 不需要绑定不同的模式变量;后面的绑定会影射前面的绑定。

Example:
> (with-syntax* ([(x y) (list #'val1 #'val2)]
                 [nest #'((x) (y))])
    #'nest)

#<syntax:eval:22:0 ((val1) (val2))>
