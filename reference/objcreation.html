6.3 创建对象

make-object 过程用 by-position 初始化参数创建一个新对象, new 形式用 by-name 初始化参数创建一个新对象, instantiate 形式用 by-position 和 by-name 初始化参数创建一个新对象。

新创建的对象中的所有字段最初都被绑定到特殊的 #<undefined> 值(见 Void)。 具有默认值表达式的初始化变量(和没有提供的值)也被初始化为 #<undefined> 。 在参数值被分配给初始化变量之后, field 子句、没有提供参数的 init-field 子句、 没有提供参数的 init 子句、私有字段定义和其他表达式被评估。 这些表达式是按照它们在类表达式中出现的顺序,从左到右进行评估的。

在评估表达式的过程中,超类声明的初始化必须通过使用 super-make-object 过程、 super-new 形式或 super-instantiate 形式评估一次。

没有匹配的初始化变量的类的副名初始化参数被隐式地作为副名参数添加到 super-make-object, super-new 或 super-instantiate 调用中,在显式参数之后。 如果为同一个名字提供了多个初始化参数,那么将使用第一个参数(如果有的话),而未使用的参数将被传播到超类中。 (注意,转换后的 by-position 参数总是放在显式 by-name 参数之前。) object% 类的初始化过程接受零个初始化参数;如果它收到任何副名初始化参数,那么就会引发 exn:fail:object exception is raised 。

如果在层次结构中的任何一个类的初始化结束时没有调用超类的初始化,那么就会引发 exn:fail:object exception is raised 。 另外,如果超类的初始化被调用了一次以上,就会引发 exn:fail:object exception is raised 。

从超类继承的字段在超类的初始化过程被调用之前不会被初始化。 相比之下,一个对象一旦被创建,所有的方法都是可用的;方法的重写不受初始化的影响(与 C++ 中的对象不同)。

procedure

(make-object class init-v ...) → object?

  class : class?
  init-v : any/c
创建一个 class 的实例。 init-v 作为初始化参数被传递,与新创建的对象的 class 的初始化变量绑定, 如 初始化变量 中所述。 如果 class 不是一个类,将引发 exn:fail:contract exception is raised 。

syntax

(new class-expr (id by-name-expr) ...)

创建 class-expr 的值(必须是一个类)的实例, 每个 by-name-expr 的值都作为相应 id 的副名参数提供。

syntax

(instantiate class-expr (by-pos-expr ...) (id by-name-expr) ...)

创建 class-expr 的值的实例(必须是一个类), by-pos-expr 的值作为 by-position 初始化参数提供。 此外,每个 by-name-expr 的值被提供为相应 id 的 by-name 参数。

syntax

super-make-object

产生一个接受副名参数并调用超类初始化的过程。 更多信息请参见 创建对象 。

syntax

(super-instantiate (by-pos-expr ...) (id by-expr ...) ...)

用指定的 by-position 和 by-name 参数调用超类初始化。 更多信息请参见 创建对象 。

syntax

(super-new (id by-name-expr ...) ...)

用指定的副名参数调用超类初始化。 更多信息请参见 创建对象 。
