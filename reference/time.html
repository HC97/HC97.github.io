15.6 时间

procedure

(current-seconds) → exact-integer?

返回以秒为单位的当前时间,自 epoch: 1970 年 1 月 1 日的 UTC 午夜。

procedure

(current-inexact-milliseconds) → real?

返回从 the epoch 开始的当前时间,单位是毫秒。结果可能包含毫秒的零头。

Example:
> (current-inexact-milliseconds)

1289513737015.418

在这个例子中, 1289513737015 的单位是毫秒, 418 的单位是微秒。

procedure

(current-inexact-monotonic-milliseconds) → real?

返回从一个未指定的起始时间开始的毫秒数。 current-inexact-milliseconds 对系统时钟很敏感,因此如果调整系统时钟,可能会比实际时间退得更快, 而 current-inexact-monotonic-milliseconds 的结果在 Racket 进程中总是与实际时间一起前进, 但不同进程的结果是不能比较的。

Example:
> (current-inexact-monotonic-milliseconds)

12772.418

Added in version 8.1.0.4 of package base.

procedure

(seconds->date secs-n [local-time?]) → date*?

  secs-n : real?
  local-time? : any/c = #t
接收 secs-n,自 the epoch 以来以秒为单位的时间 (如 (current-seconds)、(file-or-directory-modify-seconds path) 或 (/ (current-inexact-milliseconds) 1000) 的值,并返回一个 date* 结构类型的实例。 请注意, secs-n 可以包括一秒钟的零头。 如果 secs-n 过小或过大,将引发 exn:fail exception is raised 。

如果 local-time? 是 #t , 那么产生的 date* 反映了本地时区的时间,否则它反映的是 UTC 的日期。

struct

(struct date (second
    minute
    hour
    day
    month
    year
    week-day
    year-day
    dst?
    time-zone-offset)
    #:extra-constructor-name make-date
    #:transparent)
  second : (integer-in 0 60)
  minute : (integer-in 0 59)
  hour : (integer-in 0 23)
  day : (integer-in 1 31)
  month : (integer-in 1 12)
  year : exact-integer?
  week-day : (integer-in 0 6)
  year-day : (integer-in 0 365)
  dst? : boolean?
  time-zone-offset : exact-integer?
代表一个日期。 second 字段仅在闰秒时达到 60 。 week-day 字段是 0 代表星期天, 1 代表星期一,等等。 year-day 字段在1月1日为 0 ,1月2日为 1 ,等等; year-day 字段仅在闰年达到 365 。

如果日期反映了夏令时的调整,则 dst? 字段为 #t 。 time-zone-offset 字段报告了当前时区的 UTC (格林尼治标准时间)以东的秒数 (例如,太平洋标准时间是 -28800),包括任何日光节约调整(例如,太平洋夏令时间是 -25200)。 当一个 date 记录是由 seconds->date 生成的,第二个参数是 #f , 那么 dst? 和 time-zone-offset 字段分别是 #f 和 0 。

date 构造函数接受任何 dst? 值,并将任何非 #f 值转换为 #t 。

为 time-zone-offset 字段产生的值往往对 TZ 环境变量的值很敏感,特别是在 Unix 平台上; 咨询系统文档(通常在 tzset 下)以了解细节。

也可以参见 racket/date 库。

struct

(struct date* date (nanosecond time-zone-name)
    #:extra-constructor-name make-date*)
  nanosecond : (integer-in 0 999999999)
  time-zone-name : (and/c string? immutable?)
用纳秒和时区名称扩展 date , 如 "MDT", "Mountain Daylight Time" 或 "UTC" 。

当一个 date* 记录是由 seconds->date 生成的, #f 是第二个参数, 那么 time-zone-name 字段是 "UTC" 。

date* 构造函数接受一个可变的 time-zone-name 字符串,并将其转换为一个不可变的字符串。

procedure

(current-milliseconds) → exact-integer?

就像 current-inexact-milliseconds 一样,但被强制为一个 fixnum (可能是负数)。 由于结果是一个 fixnum, 在 32 位平台上,该值只在有限的时间内(尽管是合理的长时间)增加。

procedure

(current-process-milliseconds [scope]) → exact-integer?

  scope : (or/c #f thread? 'subprocesses) = #f
返回在底层操作系统上消耗的处理器时间,单位是 fixnum 毫秒,包括用户和系统时间。

  • 如果 scope 是 #f, 报告的时间是针对所有 Racket 线程和 places 。

  • 如果 scope 是一个线程,结果是针对该线程运行时的时间,但可能包括其他 places 的时间。 一个线程与其他线程的同步越多,每个线程的处理器时间记录就越不精确。

  • 如果 scope 是 'subprocesses, 结果是已知完成的子进程(见 进程) —和已知完成的子进程的子进程等的进程时间的总和,在 Unix and Mac OS 上—是所有 places 。

结果的精度与平台有关,由于结果是一个 fixnum, 在 32 位平台上,数值只在有限的时间内(尽管是合理的长时间)增加。

Changed in version 6.1.1.4 of package base: 增加了 'subprocesses 模式。

procedure

(current-gc-milliseconds) → exact-integer?

返回到目前为止 Racket 的垃圾收集所消耗的处理器时间,单位是 fixnum 毫秒。 这个时间是 (current-process-milliseconds) 报告的时间的一部分,也是类似的限制。

procedure

(time-apply proc lst) → 
list?
exact-integer?
exact-integer?
exact-integer?
  proc : procedure?
  lst : list?
收集一个程序应用的时间信息。

返回四个值:一个包含对 lst 中的参数应用 proc 的结果的列表, 获得该结果所需的 CPU 时间的毫秒数,该结果所需的 "真实" 毫秒数,以及用于垃圾回收的 CPU 时间的毫秒数(包括在第一个结果中)。

时间数字的可靠性取决于平台。 如果有多个 Racket 线程在运行,那么报告的时间可能包括其他线程执行的工作。

syntax

(time body ...+)

直接向当前的输出端口报告 expr 评估的 time-apply 式时间信息。 其结果是最后一个 body 的结果。

15.6.1 日期实用程序

关于更多的日期和时间操作,请参见 the Gregor: Date and Time documentation 或 srfi/19

 (require racket/date) package: base
The bindings documented in this section are provided by the racket/date library, not racket/base or racket.

procedure

(current-date) → date*?

(seconds->date (* 0.001 (current-inexact-milliseconds))) 的缩写。

procedure

(date->string date [time?]) → string?

  date : date?
  time? : any/c = #f
将一个日期转换为一个字符串。 只有在 time? 的情况下,返回的字符串才包含一天中的时间。 另见 date-display-format 。

参数,决定日期字符串的格式。最初的格式是 'american 。

procedure

(date->seconds date [local-time?]) → exact-integer?

  date : date?
  local-time? : any/c = #t
找到一个日期在平台特定的秒数中的表示。 如果平台不能表示指定的日期,将引发 exn:fail exception is raised 。

date 的 week-day, year-day 字段被忽略。 date 的 dst? 和 time-zone-offset 字段也被忽略; 默认情况下,日期被假定为本地时间,如果 local-time? 为 #f ,则为 UTC。

procedure

(date*->seconds date [local-time?]) → real?

  date : date?
  local-time? : any/c = #t
类似于 date->seconds ,但是如果 date 是一个 date* 实例, 则返回一个精确的数字,其中可以包括基于 (date*-nanosecond date) 的零头。

找到一个日期在特定平台上的秒数表示。 参数对应于 date 结构的字段,默认为本地时间,如果 local-time? 为 #f ,则为 UTC。 如果平台不能表示指定的日期,会发出错误信号,否则会返回一个整数。

procedure

  date : date?
