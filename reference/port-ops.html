13.1.2 管理端口

procedure

(input-port? v) → boolean?

  v : any/c
如果 v 是一个 input port,返回 #t ,否则返回 #f 。

procedure

(output-port? v) → boolean?

  v : any/c
如果 v 是一个 output port ,返回 #t ,否则返回 #f 。

procedure

(port? v) → boolean?

  v : any/c
如果 (input-port? v) 或 (output-port? v) 是 #t , 则返回 #t ,否则返回 #f 。

procedure

(close-input-port in) → void?

  in : input-port?
关闭输入端口 in 。对于某些类型的端口,关闭端口会释放低级别的资源,例如文件句柄。 如果该端口已经被关闭, close-input-port 就没有任何作用。

procedure

(close-output-port out) → void?

  out : output-port?
关闭输出端口 out 。 对于某些类型的端口,关闭端口会释放低级别的资源,比如文件句柄。 另外,如果端口是缓冲的,在关闭之前,关闭可能会先冲刷端口,而这个冲刷过程可能会阻塞。 如果端口已经被关闭, close-output-port 则没有任何作用。

procedure

(port-closed? port) → boolean?

  port : port?
如果输入或输出端口的 port 被关闭,返回 #t ,否则返回 #f 。

procedure

(port-closed-evt port) → evt?

  port : port?
返回一个 synchronizable event ,当 port 关闭时, 该事件将成为 ready for synchronization 。 The synchronization result of a port-closed event is the port-closed event itself 。

parameter

(current-input-port) → input-port?

(current-input-port in) → void?
  in : input-port?
一个 parameter , 决定了许多操作的默认输入端口,如 read 。

parameter

(current-output-port) → output-port?

(current-output-port out) → void?
  out : output-port?
一个 parameter ， 决定许多操作的默认输出端口，如 write 。

parameter

(current-error-port) → output-port?

(current-error-port out) → void?
  out : output-port?
一个决定输出端口的 parameter ,通常用于错误和日志记录。 例如,默认的错误显示处理程序会写到这个端口。

procedure

(file-stream-port? v) → boolean?

  v : any/c
如果 v 是一个 file-stream port (见 文件端口) , 返回 #t ,否则返回 #f 。

Changed in version 7.2.0.5 of package base: 将 file-stream-port? 扩展为任何值,而不是将域限制为端口

procedure

(terminal-port? v) → boolean?

  v : any/c
如果 v 是一个连接到交互式终端的端口,则返回 #t ,否则返回 #f 。

Changed in version 7.2.0.5 of package base: 将 terminal-port? 扩展为任何值,而不是将域限制在端口上

procedure

(port-waiting-peer? port) → boolean?

  port : port?
如果 port 没有准备好进行读写,因为它正在等待一个对等进程完成流的构建, 则返回 #t ,否则返回 #f 。

在 Unix 和 Mac OS 上,如果没有为同一个 fifo 的阅读器已经打开,那么为输出而打开的 fifo 会创建一个同行等待的端口。 在这种情况下,输出端口不准备写,直到有一个阅读器被打开;也就是说,写操作将被阻止。 如果有必要的话,可以使用 sync 来等待,直到写操作不会阻塞–也就是说,直到 fifo 的读端被打开。

Added in version 7.4.0.5 of package base.

value

eof : eof-object?

一个代表文件结束的值(与所有其他值不同)。

procedure

(eof-object? v) → boolean?

  v : any/c
如果 v 是 eof, 返回 #t ,否则返回 #f 。
