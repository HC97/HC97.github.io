7.11 Single-Signature Modules

 #lang racket/signature package: base
racket/signature 语言将模块主体视为一个单元签名。

模块主体必须符合以下 module-body 语法:

  module-body = (require require-spec ...) ... sig-spec ...

关于 sig-spec 的语法,请参见 创建单位 。 与 racket/unit 模块的主体不同, racket/signature 模块中的 require 必须是 require 的字面使用。

由此产生的签名被导出为 base^, 其中 base 来自包围模块的名称(即它的符号名称,或者它的路径,不包括目录和文件后缀)。 如果模块名以 -sig 结尾,那么 base 对应于 -sig 之前的模块名。 否则,模块名将作为 base 。

作为 sig-spec 的 struct 形式与 define-struct 引入的定义一致, 而不是 struct 引入的定义。 (这一行为最初是一个错误,但为了兼容,我们保留了这一行为)。
