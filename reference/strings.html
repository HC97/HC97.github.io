4.4 字符串

字符串（Unicode） in Racket 指南 introduces strings.

一个 string 是一个固定长度的 characters 数组。

字符串可以是 mutable 或 immutable 。 当一个不可变的字符串被提供给像 string-set! 这样的程序时, 会引发 exn:fail:contract exception is raised 。 由默认阅读器生成的字符串常量(参见 读取字符串) 是不可变的, 它们在 read-syntax 模式下被 interned 。 使用 immutable? 来检查一个字符串是否是不可变的。

当两个字符串具有相同的长度并包含相同的字符序列时,它们是 equal? 。

一个字符串可以作为一个单值序列使用(参见 Sequences)。 字符串的字符作为序列的元素。 也见 in-string 。

See 读取字符串 for information on reading strings and 打印字符串 for information on printing strings.

参见: immutable?, symbol->string, bytes->string/utf-8 。

4.4.1 字符串构造器、选择器和突变器

procedure

(string? v) → boolean?

  v : any/c
如果 v 是一个字符串,返回 #t , 否则返回 #f 。

参见 immutable-string? 和 mutable-string? 。

Examples:
> (string? "Apple")

#t

> (string? 'apple)

#f

procedure

(make-string k [char]) → string?

  k : exact-nonnegative-integer?
  char : char? = #\nul
返回一个新的长度为 k 的可变字符串, 字符串中的每个位置都被初始化为字符 char 。

Example:
> (make-string 5 #\z)

"zzzzz"

procedure

(string char ...) → string?

  char : char?
返回一个新的可变字符串,其长度为所提供的 char 的数量, 并且其位置被初始化为所给的 char 。

Example:
> (string #\A #\p #\p #\l #\e)

"Apple"

procedure

(string->immutable-string str) → (and/c string? immutable?)

  str : string?
返回一个与 str 相同内容的不可变的字符串,如果 str 是不可变的,则返回 str 本身。

procedure

(string-length str) → exact-nonnegative-integer?

  str : string?
返回 str 的长度。

Example:
> (string-length "Apple")

5

procedure

(string-ref str k) → char?

  str : string?
  k : exact-nonnegative-integer?
返回 str 中 k 位置的字符。 字符串中的第一个位置对应于 0 ,所以位置 k 必须小于字符串的长度, 否则会引发 exn:fail:contract exception is raised 。

Example:
> (string-ref "Apple" 0)

#\A

procedure

(string-set! str k char) → void?

  str : (and/c string? (not/c immutable?))
  k : exact-nonnegative-integer?
  char : char?
将 str 中的字符位置 k 改为 char 。 字符串中的第一个位置对应于 0 , 所以位置 k 必须小于字符串的长度,否则会引发 exn:fail:contract exception is raised 。

Examples:
> (define s (string #\A #\p #\p #\l #\e))
> (string-set! s 4 #\y)
> s

"Apply"

procedure

(substring str start [end]) → string?

  str : string?
  start : exact-nonnegative-integer?
  end : exact-nonnegative-integer? = (string-length str)
返回一个新的可变字符串,该字符串的长度为 (- end start) , 并且从 start (包括)到 end (包括)包含与 str 相同的字符。 字符串的第一个位置对应于 0 , 所以 start 和 end 参数必须小于或等于 str 的长度, 而且 end 必须大于或等于 start ,否则会引发 exn:fail:contract exception is raised。

Examples:
> (substring "Apple" 1 3)

"pp"

> (substring "Apple" 1)

"pple"

procedure

(string-copy str) → string?

  str : string?
返回 (substring str 0) 。

Examples:
> (define s1 "Yui")
> (define pilot (string-copy s1))
> (list s1 pilot)

'("Yui" "Yui")

> (for ([i (in-naturals)] [ch '(#\R #\e #\i)])
    (string-set! pilot i ch))
> (list s1 pilot)

'("Yui" "Rei")

procedure

(string-copy! dest    
  dest-start    
  src    
  [src-start    
  src-end]) → void?
  dest : (and/c string? (not/c immutable?))
  dest-start : exact-nonnegative-integer?
  src : string?
  src-start : exact-nonnegative-integer? = 0
  src-end : exact-nonnegative-integer? = (string-length src)
将 dest 中从位置 dest-start 开始的字符改为与 src 中从 src-start (包括)到 src-end (不包括)的字符相匹配,其中字符串的第一个位置对应于 0 。 字符串 dest 和 src 可以是同一个字符串,在这种情况下, 目标区域可以与源区域重叠;复制后的目标字符与复制前的源字符匹配。 如果 dest-start, src-start 或 src-end 中的任何一个超出了范围(考虑到字符串的大小以及源和目的区域),就会引发 exn:fail:contract exception is raised 。

Examples:
> (define s (string #\A #\p #\p #\l #\e))
> (string-copy! s 4 "y")
> (string-copy! s 0 s 3 4)
> s

"lpply"

procedure

(string-fill! dest char) → void?

  dest : (and/c string? (not/c immutable?))
  char : char?
改变 dest ,使字符串中的每个位置都被 char 填满。

Examples:
> (define s (string #\A #\p #\p #\l #\e))
> (string-fill! s #\q)
> s

"qqqqq"

procedure

(string-append str ...) → string?

  str : string?
返回 一个新的可变字符串, 其长度与给定的 str 的长度之和相同,并且包含给定的 str 的连接字符。 如果没有提供 str ,结果是一个零长度的字符串。

Example:
> (string-append "Apple" "Banana")

"AppleBanana"

procedure

(string-append-immutable str ...) → (and/c string? immutable?)

  str : string?
与 string-append 相同,但其结果是一个不可变的字符串。

Examples:
> (string-append-immutable "Apple" "Banana")

"AppleBanana"

> (immutable? (string-append-immutable "A" "B"))

#t

Added in version 7.5.0.14 of package base.

procedure

(string->list str) → (listof char?)

  str : string?
返回一个与 str 的内容相对应的新的字符列表。 也就是说,列表的长度是 (string-length str), str 中的字符序列是结果列表中的相同序列。

Example:
> (string->list "Apple")

'(#\A #\p #\p #\l #\e)

procedure

(list->string lst) → string?

  lst : (listof char?)
返回一个新的可变字符串, 其内容是 lst 中的字符列表。 也就是说,字符串的长度是 (length lst), lst 中的字符序列与结果字符串中的序列相同。

Example:
> (list->string (list #\A #\p #\p #\l #\e))

"Apple"

procedure

(build-string n proc) → string?

  n : exact-nonnegative-integer?
  proc : (exact-nonnegative-integer? . -> . char?)
将 proc 依次应用于从 0 到 (sub1 n) 的整数, 创建一个 n 个字符的字符串。 如果 str 是生成的字符串, 那么 (string-ref str i) 就是 (proc i) 生成的字符。

Example:
> (build-string 5 (lambda (i) (integer->char (+ i 97))))

"abcde"

4.4.3 字符串转换

procedure

(string-upcase str) → string?

  str : string?
返回 一个字符串,其字符是 str 中字符的大写转换。 转换使用 Unicode 的独立于区域设置的转换规则, 将代码点序列映射到代码点序列(而不是简单地在字符串上的代码点上映射 1 对 1 函数), 因此转换生成的字符串可以比输入字符串长。

Examples:
> (string-upcase "abc!")

"ABC!"

> (string-upcase "Straße")

"STRASSE"

procedure

(string-downcase string) → string?

  string : string?
像 string-upcase 一样,但要进行 downncase 转换。

Examples:
> (string-downcase "aBC!")

"abc!"

> (string-downcase "Straße")

"straße"

> (string-downcase "ΚΑΟΣ")

"καος"

> (string-downcase "Σ")

"σ"

procedure

(string-titlecase string) → string?

  string : string?
与 string-upcase 类似,但标题大写的转换只针对 str 中每个有大小写的字符序列中的第一个字符(忽略不分大小写的字符)。

Examples:
> (string-titlecase "aBC  twO")

"Abc  Two"

> (string-titlecase "y2k")

"Y2k"

> (string-titlecase "main straße")

"Main Straße"

> (string-titlecase "stra ße")

"Stra Sse"

procedure

(string-foldcase string) → string?

  string : string?
类似于 string-upcase ,但是是折叠式转换。

Examples:
> (string-foldcase "aBC!")

"abc!"

> (string-foldcase "Straße")

"strasse"

> (string-foldcase "ΚΑΟΣ")

"καοσ"

procedure

(string-normalize-nfd string) → string?

  string : string?
返回一个字符串, 该字符串是 string 的 Unicode 规范化形式 D。 如果给定的字符串已经是对应的 Unicode 标准格式,则字符串可能直接作为结果返回(而不是新分配的字符串)。

procedure

(string-normalize-nfkd string) → string?

  string : string?
与 string-normalize-nfd 类似,但用于规范化形式的 KD。

procedure

(string-normalize-nfc string) → string?

  string : string?
与 string-normalize-nfd 类似, 但适用于规范化的 C 形式。

procedure

(string-normalize-nfkc string) → string?

  string : string?
与 string-normalize-nfd 类似,但适用于规范化的 KC 形式。

4.4.5 字母簇

procedure

(string-grapheme-span str start [end]) → exact-nonnegative-integer?

  str : string?
  start : exact-nonnegative-integer?
  end : exact-nonnegative-integer? = (string-length str)
返回字符串中从 start 开始形成 Unicode 字母簇的字符(即码点)的数量, 假设 start 是字母簇的开始,并且不超过 end 之前的字符。 如果 start 等于 end ,则结果为 0 。

start 和 end 参数必须是 substring 的有效索引, 否则会引发 exn:fail:contract exception is raised 。

参见 char-grapheme-cluster-step 。

Examples:
> (string-grapheme-span "" 0)

0

> (string-grapheme-span "a" 0)

1

> (string-grapheme-span "ab" 0)

1

> (string-grapheme-span "\r\n" 0)

2

> (string-grapheme-span "\r\nx" 0)

2

> (string-grapheme-span "\r\nx" 2)

1

> (string-grapheme-span "\r\nx" 0 1)

1

Added in version 8.6.0.2 of package base.

procedure

(string-grapheme-count str start [end])

 → exact-nonnegative-integer?
  str : string?
  start : exact-nonnegative-integer?
  end : exact-nonnegative-integer? = (string-length str)

4.4.7 将数值转换为字符串

 (require racket/format) package: base
The bindings documented in this section are provided by the racket/format and racket libraries, but not racket/base.

racket/format 库提供了将 Racket 值转换为字符串的函数。 除了像填充和数字格式化这样的功能外,这些函数的优点是比 format (用 format string), number->string 或 string-append 要短。

procedure

(~a v    
  ...

  #:min-width min-width

  max-width : (or/c exact-nonnegative-integer? +inf.0)
   = (or width +inf.0)
  min-width : exact-nonnegative-integer? = (or width 0)
  limit-marker : string? = ""
  limit-prefix? : boolean? = #f
  align : (or/c 'left 'center 'right) = 'left
  pad-string : non-empty-string? = " "
  left-pad-string : non-empty-string? = pad-string
  right-pad-string : non-empty-string? = pad-string
在 display 模式下,将每个 v 转换为一个字符串–即像 (format "~a" v) 那样–然后将结果连接起来,在连续的项目之间使用 separator , 然后将字符串填充或截断,使其至少为 min-width 字符,最多为 max-width 字符。

Examples:
> (~a "north")

"north"

> (~a 'south)

"south"

> (~a #"east")

"east"

> (~a #\w "e" 'st)

"west"

> (~a (list "red" 'green #"blue"))

"(red green blue)"

> (~a 17)

"17"

> (~a #e1e20)

"100000000000000000000"

> (~a pi)

"3.141592653589793"

> (~a (expt 6.1 87))

"2.1071509386211452e+68"

~a 函数主要对字符串、数字和其他原子数据有用。 ~v 和 ~s 函数更适合于复合数据。

让 s 成为 v 加分隔符的串联字符串形式。 如果 s 长于 max-width 字符,它将被截断到正好是 max-width 字符。 如果 s 短于 min-width 字符,它将被填充到正好是 min-width 字符。 否则, s 被原封不动地返回。 如果 min-width 大于 max-width ,将产生一个异常。

如果 s 长于 max-width 字符,它将被截断, 并且用 limit-marker 替换字符串的末端。 如果 limit-marker 长于 max-width ,就会产生一个异常。 如果 limit-prefix? 是 #t,那么字符串的开头会被截断,而不是结尾。

Examples:
> (~a "abcde" #:max-width 5)

"abcde"

> (~a "abcde" #:max-width 4)

"abcd"

> (~a "abcde" #:max-width 4 #:limit-marker "*")

"abc*"

> (~a "abcde" #:max-width 4 #:limit-marker "...")

"a..."

> (~a "The quick brown fox" #:max-width 15 #:limit-marker "")

"The quick brown"

> (~a "The quick brown fox" #:max-width 15 #:limit-marker "...")

"The quick br..."

> (~a "The quick brown fox" #:max-width 15 #:limit-marker "..." #:limit-prefix? #f)

"The quick br..."

如果 s 比 min-width 短,它至少要被填充到 min-width 的字符。 如果 align 方式是 'left ,则只添加右填充; 如果 align 方式是 'right ,则只添加左填充; 如果 align 方式是 'center ,则添加大致相同数量的左填充和右填充。

padding 被指定为一个非空的字符串。 左边的填充由 left-pad-string 组成,尽可能多地重复其整体, 然后用 left-pad-string 的 prefix 来填充剩余的空间。 相比之下,右侧填充是由 right-pad-string 的 suffix 和 right-pad-string 的完整副本组成的。 因此,左边的填充从 left-pad-string 的开头开始, 右边的填充从 right-pad-string 的结尾结束。

Examples:
> (~a "apple" #:min-width 20 #:align 'left)

"apple               "

> (~a "pear" #:min-width 20 #:align 'left #:right-pad-string " .")

"pear . . . . . . . ."

> (~a "plum" #:min-width 20 #:align 'right #:left-pad-string ". ")

". . . . . . . . plum"

> (~a "orange" #:min-width 20 #:align 'center
                #:left-pad-string "- " #:right-pad-string " -")

"- - - -orange- - - -"

使用 width 同时设置 max-width 和 min-width ,确保产生的字符串正好是 width 字符的长度:

> (~a "terse" #:width 6)

"terse "

> (~a "loquacious" #:width 6)

"loquac"

    Examples:
    > (~r 12345)

    "12345"

    > (~r 12345 #:notation 'exponential)

    "1.2345e+04"

    > (let ([pick-notation
             (lambda (x)
               (if (or (< (abs x) 0.001) (> (abs x) 1000))

    如果 precision 是一个自然数,那么将显示 precision 以内的数字, 但是后面的零会被删除,如果小数点后的所有数字被删除,小数点也会被删除。 如果 precision 是 (list '= digits) , 那么小数点后的数字正好是 digits ,小数点永远不会被删除。

    "3.141593"

    "1.5"

    "1.5000"

    "50"

    > (~r 50 #:precision '(= 2))

    "50.00"

    > (~r 50 #:precision '(= 0))

    "50."

    Examples:
    > (~r 123.456)

    "123.456"

    "123,456"

  • groups 控制数字的整数部分的数字如何被分开成组。 groups 的最右边的数字用于分组积分部分的最右边的数字。 groups 的最左边的数字被重复使用,用于分组最左边的数字。 group-sep 参数指定了数字组之间使用的分隔符。

    Examples:

    "1,234,567,890"

    > (~r 1234567890 #:groups '(3 2) #:group-sep ",")

    "12,345,678,90"

    > (~r 1234567890 #:groups '(1 3 2) #:group-sep "_")

    "1_2_3_4_5_678_90"

    Examples:
    > (~r 17)

    "17"

    > (~r 17 #:min-width 4)

    "  17"

    > (~r -42 #:min-width 4)

    "-  42"

    > (~r 1.5 #:min-width 4)

    " 1.5"

    > (~r 1.5 #:precision 4 #:min-width 10)

    "       1.5"

    > (~r 1.5 #:precision '(= 4) #:min-width 10)

    "    1.5000"

    > (~r #e1e10 #:min-width 6)

    "10000000000"

  • pad-string — 指定用于填充数字的字符串, 至少要达到 min-width 的字符(不包括符号指示器)。 填充被放置在符号和 x.

    Examples:
    > (~r 17 #:min-width 4 #:pad-string "0")

    "0017"

    > (~r -42 #:min-width 4 #:pad-string "0")

    "-0042"

  • sign — 控制如何显示数字的符号:

    • 如果 sign 是 #f (默认值),如果 x 是正数或零, 则不产生符号输出,如果 x 是负数,则在前面加一个减号。

      Example:
      > (for/list ([x '(17 0 -42)]) (~r x))

      '("17" "0" "-42")

    • 如果 sign 是 '+ ,如果 x 是零,则不产生符号输出, 如果 x 是正数,则预加一个正号,如果 x 是负数,则预加一个负号。

      Example:
      > (for/list ([x '(17 0 -42)]) (~r x #:sign '+))

      '("+17" "0" "-42")

    • 如果 sign 是 '++ ,如果 x 是零或正数,就会加一个正号, 如果 x 是负数,就会加一个负号。

      Example:
      > (for/list ([x '(17 0 -42)]) (~r x #:sign '++))

      '("+17" "+0" "-42")

    • 如果 sign 是 'parens, 如果 x 是零或正数, 则不产生符号输出,如果 x 是负数,则数字被包围在小括号中。

      Example:
      > (for/list ([x '(17 0 -42)]) (~r x #:sign 'parens))

      '("17" "0" "(42)")

    • 如果 sign 是 (list pos-ind zero-ind neg-ind), 那么 pos-ind, zero-ind 和 neg-ind 分别用来表示正数、零和负数。 每个指标要么是一个用作前缀的字符串,要么是一个包含两个字符串的列表:一个前缀和一个后缀。

      Example:
      > (let ([sign-table '(("" " up") "an even " ("" " down"))])
          (for/list ([x '(17 0 -42)]) (~r x #:sign sign-table)))

      '("17 up" "an even 0" "42 down")

      默认行为相当于 '("" "" "-"); 'parens 模式相当于 '("" "" ("(" ")"))。

  • base — 控制 x 被格式化的基数。 如果 base 是一个大于 10 的数字,那么就使用小写字母。 如果 base 是 (list 'up base*) 并且 base* 大于 10, 则使用大写字母。

    Examples:
    > (~r 100 #:base 7)

    "202"

    > (~r 4.5 #:base 2)

    "100.1"

    > (~r 3735928559 #:base 16)

    "deadbeef"

    > (~r 3735928559 #:base '(up 16))

    "DEADBEEF"

    > (~r 3735928559 #:base '(up 16) #:notation 'exponential)

    "D.EADBEF*16^+07"

  • format-exponent — 决定了指数的显示方式。

    如果 format-exponent 是一个字符串,那么指数的显示就会有一个明确的符号 (如 '++ 的 sign )和至少两个数字, 用 "指数标记“ format-exponent 与显数分开:

    > (~r 1234 #:notation 'exponential #:format-exponent "E")

    "1.234E+03"

    如果 format-exponent 是 #f,如果 base 是 10 ,"指数标记"是 "e" ,否则就是涉及 base 的字符串:

    > (~r 1234 #:notation 'exponential)

    "1.234e+03"

    > (~r 1234 #:notation 'exponential #:base 8)

    "2.322*8^+03"

    如果 format-exponent 是一个过程,它将被应用到指数上,并且产生的字符串将被附加到显式上:

    > (~r 1234 #:notation 'exponential
               #:format-exponent (lambda (e) (format "E~a" e)))

    "1.234E3"
