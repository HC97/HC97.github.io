2 文档的符号

本章介绍了整个 Racket 文档中使用的基本术语和记号。

2.1 模块文档的注释

由于 Racket 程序被组织成 module ,文档反映了这种组织, 在描述特定模块提供的绑定的章节或小节的开头有一个注释。

例如,描述 racket/list 所提供的功能的部分开始于

 (require racket/list) package: base

一些模块用 #lang 来代替 require:

 #lang racket/base package: base

使用 #lang 意味着该模块通常被用作整个模块的语言–也就是说, 被一个以 #lang 为开头的模块所使用,而不是用 require 来导入。 然而,除非另有规定, 一个用 #lang 记录的模块名称也可以用 require 来获得语言的绑定。

模块注解还在右侧显示了该模块所属的 package。 关于包的更多细节,请参见 Racket 中的包管理。

因此,除非在章节或小节中另有规定,否则在 Racket 参考 中记录的绑定可以从 racket 和 racket/base 中获得。

2.2 句法形式文件的记号

标记法 in Racket 指南 introduces this notation for syntactic forms.

句法形式是用一个语法来指定的。 通常情况下,语法以一个开放的括号开始,后面是句法形式的名称,如 if 的语法:

syntax

(if test-expr then-expr else-expr)

由于每个 form 都是用 syntax objects 来表达的, 语法规范中的小括号表示一个包裹着列表的 syntax object , 前面的 if 是一个标识符, 它开始的列表的 binding 是被记录的模块的 if –本例是 racket/base 。 语法中的方括号以与小括号相同的方式表示 syntax-object 列表, 但在程序源代码中,方括号通常是按惯例使用。

语法中的斜体 identifiers 是对应于其他语法产品的 metavariables。 某些元变量名称具有隐含的语法产品:

  • 一个以 id 结尾的元变量代表一个 identifier。

  • 以 keyword 结尾的元变量代表一个 syntax-object 的 keyword。

  • 以 expr 结尾的元变量代表任何形式,并且该形式将被解析为一个表达式。

  • 以 body 结尾的元变量代表任何 form; 该形式将被解析为本地定义或表达式。 一个 body 只有在前面没有任何表达式的情况下才能被解析为定义, 而且最后一个 body 必须是表达式;另见 内部定义。

  • 以 datum 结尾的元变量代表任何 form, 并且该形式通常是不被解释的(例如, quote)。

  • 以 number 或 boolean 结尾的元变量分别代表任何 syntax-object (即字面意思) number 或 boolean。

在一个语法中, form ... 代表与形式相匹配的任何数量的形式(可能是零), 而 form, form ...+ 则代表与 form 相匹配的一个或多个形式。

没有隐含语法的元变量由句法形式的整体语法旁边的产品来定义。例如,在

syntax

(lambda formals body ...+)

 
formals = id
  | (id ...)
  | (id ...+ . rest-id)

formals 元变量代表一个 identifier, 一个 syntax-object 列表中的零个或多个 identifiers , 或者一个 syntax object ,对应于一个或多个配对链, 该链以一个 identifier 而不是一个空列表结束。

有些句法形式有多个顶层语法,在这种情况下,句法形式的文档会显示多个语法。例如

syntax

(init-rest id)

(init-rest)

表示 init-rest 既可以在其 syntax-object 列表中单独出现, 也可以在后面跟一个 identifier。

最后,包括 expr 元变量的语法规范可以用一些元变量的运行时 contract 来增加, 这些契约表示表达式的结果在运行时必须满足的谓词。例如

syntax

(parameterize ([parameter-expr value-expr] ...)
  body ...+)
 
  parameter-expr : parameter?

表示每个 parameter-expr 的结果必须是一个 (parameter? v) 返回真值的值 v 。

2.3 函数文档的符号

程序和其他值是用基于 contract 的符号来描述的。 从本质上讲,这些契约使用 Racket 谓词和表达式来描述文档库的接口。

例如,下面是一个典型程序定义的标题:

procedure

(char->integer char) → exact-integer?

  char : char?

被定义的函数, char->integer, 就像被应用一样被排版。 在函数名后面的元变量代表了参数。角落里的白色文字标识了被记录的值的种类。

每个元变量都有一个契约来描述。 在前面的例子中, metavariable char 有一个契约 char?。 这个契约规定,任何回答 char? 谓词的参数 char 都是有效的。 被记录的函数可能会也可能不会实际检查这个属性,但是这个契约表明了实现者的意图。

箭头右边的契约,即本例中的 exact-integer? ,指定了由函数产生的预期结果。

契约规范可以比仅仅是谓词的名称更有表现力。考虑以下 argmax 的标题:

procedure

(argmax proc lst) → any

  proc : (-> any/c real?)
  lst : (and/c pair? list?)

契约 (-> any/c real?) 表示一个函数契约, 指定 proc 的参数可以是任何单一的值,结果应该是一个实数。 lst 的契约 (and/c pair? list?) 指定 lst 应该同时传递 pair? 和 list? (即,它是一个非空的列表)。

-> 和 and/c 都是 contract combinator 的例子。 契约组合器,如 or/c, cons/c, listof 和其他组合器在整个文档中都有使用。 点击超链接的组合器名称将提供更多关于其含义的信息。

一个Racket函数可以被记录为有一个或多个可选参数。 read 函数就是这样一个函数的例子:

procedure

(read [in]) → any

  in : input-port? = (current-input-port)

应用语法中 in 参数周围的括号表示它是一个可选的参数。

read 的标头像往常一样为参数 in 指定了一个契约。 在契约的右边,它还指定了一个默认值 (current-input-port) , 如果 read 被调用时没有参数,就会使用这个值。

函数也可以被记录为接受强制性或可选的基于关键字的参数。 例如, sort 函数有两个可选的、基于关键字的参数:

procedure

(sort lst    
  less-than?    
  [#:key extract-key    
  #:cache-keys? cache-keys?]) → list?
  lst : list?
  less-than? : (any/c any/c . -> . any/c)
  extract-key : (any/c . -> . any/c) = (lambda (x) x)
  cache-keys? : boolean? = #f

extract-key 和 cache-keys? 参数周围的括号表示它们和以前一样是可选的。 头部的合同部分显示了为这些关键字参数提供的默认值。

2.4 结构类型文档的符号

structure type 也是用契约符号来记录的:

struct

(struct color (red green blue alpha))

  red : (and/c natural-number/c (<=/c 255))
  green : (and/c natural-number/c (<=/c 255))
  blue : (and/c natural-number/c (<=/c 255))
  alpha : (and/c natural-number/c (<=/c 255))

结构类型是在使用 struct 形式的程序源代码中声明的类型。 结构的每个字段都使用相应的契约进行记录,该契约指定该字段接受的值。

在上面的例子中,结构类型 color 有四个字段: red, green, blue 和 alpha。 结构类型的构造函数接受满足 (and/c natural-number/c (<=/c 255))的字段值, 即不超过 255 的非负精确整数。

在结构类型的文档中,其他关键字可能出现在字段名之后:

struct

(struct data-source (connector args extensions)
    #:mutable)
  connector : (or/c 'postgresql 'mysql 'sqlite3 'odbc)
  args : list?
  extensions : (listof (list/c symbol? any/c))

在这里, #:mutable 关键字表示 data-source 结构类型的实例的字段可以用它们各自的 setter 函数进行改变。

2.5 参数文档表示法

parameter 的记录方式与函数相同:

parameter

(current-command-line-arguments) → (vectorof string?)

(current-command-line-arguments argv) → void?
  argv : (vectorof (and/c string? immutable?))

由于 parameters 可以被引用或设置,所以在上面的标头中有两个条目。 调用 current-command-line-arguments 将访问参数的值, 该值必须是一个 vector,其元素同时传递 string? 和 immutable? 。 使用单个参数调用 current-command-line-arguments 设置参数的值, 其中值必须是其元素传递 string? (如果需要, parameter 上的保护将字符串强制为不可变的形式)。

2.6 其他文档符号

一些库提供到常量值的绑定。这些值有一个单独的头文件:

value

object% : class?

racket/class 库提供了 object% 值, 这是 Racket 中的类层次结构的根。它的文档头仅仅表明它是一个满足谓词 class? 的值。
