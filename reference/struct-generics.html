5.4 通用接口

 (require racket/generic) package: base

一个 generic interface 允许每个类型的方法与通用函数相关联。 泛型函数是用 define-generics 的形式来定义的。 结构类型的方法实现是用 #:methods 关键字定义的(见 定义结构类型: struct)。

syntax

(define-generics id
  generics-opt ...
  [method-id . kw-formals*] ...
  generics-opt ...)
 
generics-opt = #:defaults ([default-pred? default-impl ...] ...)
  | #:fast-defaults ([fast-pred? fast-impl ...] ...)
  | #:fallbacks [fallback-impl ...]
  | #:defined-predicate defined-pred-id
  | #:defined-table defined-table-id
  | #:derive-property prop-expr prop-value-expr
  | #:requires [required-method-id ...]
     
kw-formals* = (arg* ...)
  | (arg* ...+ . rest-id)
  | rest-id
     
arg* = arg-id
  | [arg-id]
  | keyword arg-id
  | keyword [arg-id]
定义了以下名称,加上任何由关键字选项指定的名称。

  • gen:id 作为一个转化器绑定,用于新的通用接口的静态信息;

  • id? 作为一个谓词,用于识别实现这个泛型组的结构类型的实例;以及

  • 每个 method-id 都是一个 generic method , 在 id? 为真时调用相应的方法。 每个 method-id 的 kw-formals* 必须包括一个必要的 by-position 参数, 该参数是 id 的 free-identifier=? 。 这个参数在泛型定义中被用来定位特殊化。

  • id/c 是一个合约组合器, 它可以识别实现了 gen:id 泛型接口的结构类型的实例。 该组合器接受成对的 method-id 和契约。 契约将被应用于每个相应的方法实现。 id/c 组合器的目的是用来约定实现了泛型接口的结构类型的构造过程的范围。

#:defaults 选项最多可以提供一次。 当它被提供时,每个泛型函数使用 default-pred? 来调度给定的 default method 实现, default-impl ,如果调度到泛型方法表失败。 default-impl 的语法与为 struct 的 #:methods 关键字提供的方法相同。

#:fast-defaults 选项最多可以提供一次。 它的作用与 #:defaults 相同,只是在派发到通用方法表之前会检查 fast-pred? 。 这个选项的目的是为调度内置数据类型提供一个快速路径,比如列表和向量, 这些数据类型不会与实现 gen:id 的结构重叠。

#:fallbacks 选项最多可以提供一次。 当它被提供时, fallback-impl 定义了 fallback method 的实现, 这些方法被用于没有提供特定实现的泛型接口的任何实例。 fallback-impl 的语法与为 struct 的 #:methods 关键字提供的方法相同。

#:defined-predicate 选项最多可以提供一次。 当它被提供时, defined-pred-id 被定义为一个程序,用来报告通用接口的特定实例是否实现了一组给定的方法。 具体来说，如果 v 有每个方法 name 的实现,不算 #:fallbacks 的实现, (defined-pred-id v 'name ...) 则产生 #t ,否则产生 #f 。 这个过程是为了让更高级别的 API 根据方法的可用性来调整它们的行为。

#:defined-table 选项最多可以提供一次。 当它被提供时, defined-table-id 被定义为一个过程,它接收一个通用接口的实例, 并返回一个不可改变的 hash table ,该表将对应于方法名称的符号映射为布尔值,代表该方法是否被该实例实现。 这个选项已被废弃,请使用 #:defined-predicate 来代替。

#:derive-property 选项可以被提供任意次数。 每次提供时,它都会通过 structure type property 指定一个结构类型的属性, 并通过 prop-value-expr 指定该属性的值。 所有通过 #:methods 实现通用接口的结构都会使用所提供的值自动实现该结构类型属性。 当 prop-value-expr 被执行时, 每个 method-id 都被绑定到它对 structure type 的具体实现。

#:requires 选项最多只能提供一次。 当它被提供时,通用接口的任何实例 必须 提供指定的 required-method-id 的实现。 否则,会产生一个编译时错误。

如果一个值 v 满足 id?, 那么 v 就是 gen:id 的一个 generic instance 。

如果一个泛型实例 v 通过 struct 中的 #:methods 或者 #:defaults 或 define-generics 中的 #:fast-defaults 为某个 method-id 提供了相应的实现, 那么 method-id 就是 v 的一个 implemented generic method 。

如果 method-id 不是一个泛型实例 v 的已实现泛型方法, 并且 method-id 有一个回退实现,在给定 v 时不会 exn:fail:support , 那么 method-id 就是 v 的 supported generic method 。

Changed in version 8.7.0.5 of package base: 增加了 #:requires 选项。

procedure

(raise-support-error name v) → none/c

  name : symbol?
  v : any/c
当名为 name 的 generic method 不支持 generic instance v 时, 会引发 exn:fail:support 异常。

Example:
> (raise-support-error 'some-method-name '("arbitrary" "instance" "value"))

some-method-name: not implemented for '("arbitrary"

"instance" "value")

struct

(struct exn:fail:support exn:fail ()
    #:transparent)
针对不支持给定的 generic instance 的 generic methods 引发。

syntax

(define/generic local-id method-id)

当在与 #:methods, #:fallbacks, #:defaults 或 #:fast-defaults 关键字相关的方法定义中使用时, 将 local-id 绑定到 method-id 的泛型。 这种形式对于在其他值上使用通用方法(相对于本地专用)的方法专用化很有用。

define/generic 形式只允许在以下情况下使用:

  • 在 struct (或 define-struct)中的 #:methods 规范

  • define-generics 中的 #:fallbacks, #:defaults 或 #:fast-defaults 规范。

在其他地方使用 define/generic 是一个语法错误。

Examples:
> (define-generics printable
    (gen-print printable [port])
    (gen-port-print port printable)
    (gen-print* printable [port] #:width width #:height [height])
    #:defaults ([string?
                 (define/generic super-print gen-print)
                 (define (gen-print s [port (current-output-port)])
                   (fprintf port "String: ~a" s))
                 (define (gen-port-print port s)
                   ; we can call gen-print alternatively
                   (super-print s port))
                 (define (gen-print* s [port (current-output-port)]
                                     #:width w #:height [h 0])
                   (fprintf port "String (~ax~a): ~a" w h s))]))
> (struct num (v)
    #:methods gen:printable
    [(define (gen-print n [port (current-output-port)])
       (fprintf port "Num: ~a" (num-v n)))
     (define (gen-port-print port n)
       (gen-print n port))
     (define (gen-print* n [port (current-output-port)]
                         #:width w #:height [h 0])
       (fprintf port "Num (~ax~a): ~a" w h (num-v n)))])
> (struct string+num (v n)
    #:methods gen:printable
    [(define/generic super-print gen-print)
     (define/generic super-print* gen-print*)
     (define (gen-print b [port (current-output-port)])
       (super-print (string+num-v b) port)
       (fprintf port " ")
       (super-print (string+num-n b) port))
     (define (gen-port-print port b)
       (gen-print b port))
     (define (gen-print* b [port (current-output-port)]
                         #:width w #:height [h 0])
       (super-print* (string+num-v b) #:width w #:height h)
       (fprintf port " ")
       (super-print* (string+num-n b) #:width w #:height h))])
> (define x (num 10))
> (gen-print x)

Num: 10

> (gen-port-print (current-output-port) x)

Num: 10

> (gen-print* x #:width 100 #:height 90)

Num (100x90): 10

> (define str "Strings are printable too!")
> (gen-print str)

String: Strings are printable too!

> (define y (string+num str x))
> (gen-print y)

String: Strings are printable too! Num: 10

> (gen-port-print (current-output-port) y)

String: Strings are printable too! Num: 10

> (gen-print* y #:width 100 #:height 90)

String (100x90): Strings are printable too! Num (100x90): 10

syntax

(generic-instance/c gen-id [method-id method-ctc] ...)

 
  method-ctc : contract?
创建一个契约,识别实现 generic interface gen-id 的结构, 并以相应的 method-ctc 约束它们对指定 method-id 的实现。

syntax

(impersonate-generics gen-id val-expr
  [method-id method-proc-expr] ...
  maybe-properties)
 
maybe-properties = 
  | #:properties props-expr
 
  method-proc-expr : (any/c . -> . any/c)
  props-expr : (list/c impersonator-property? any/c ... ...)
创建一个 val-expr 的 impersonator , 它必须是一个实现了 generic interface gen-id 的结构。 impersonator 将 method-proc-expr 的结果应用于该结构对相应 method-id 的实现, 并将方法实现替换为结果。

props-expr 可以提供附加到 impersonator 的属性。 props-expr 的结果必须是一个具有偶数元素的列表,其中列表的第一个元素是一个冒充者属性, 第二个元素是它的值,以此类推。

Changed in version 6.1.1.8 of package base: 增加 #:properties 。

syntax

(chaperone-generics gen-id val-expr
  [method-id method-proc-expr] ...
  maybe-properties)
像 impersonate-generics 一样,但是创建一个 val-expr 的 chaperone , 它必须是一个实现了 generic interface gen-id 的结构。 伴侣将指定的 method-proc 应用于该结构对相应 method-id 的实现, 并将方法实现替换为结果,该结果必须是原始方法的伴侣。

syntax

(redirect-generics mode gen-id val-expr
   [method-id method-proc-expr] ...
   maybe-properties)
与 impersonate-generics 类似,但如果 mode 评估为 #f, 则创建一个 val-expr 的 impersonator , 否则创建一个 val-expr 的 chaperone 。

syntax

(make-struct-type-property/generic
  name-expr
  maybe-guard-expr
  maybe-supers-expr
  maybe-can-impersonate?-expr
  property-option
  ...)
 
maybe-guard-expr = 
  | guard-expr
     
maybe-supers-expr = 
  | supers-expr
     
maybe-can-impersonate?-expr = 
  | can-impersonate?-expr
     
property-option = #:property prop-expr val-expr
  | #:methods gen:name-id method-defs
     
method-defs = (definition ...)
 
  name-expr : symbol?
  guard-expr : (or/c procedure? #f 'can-impersonate)
  supers-expr : (listof (cons/c struct-type-property? (-> any/c any/c)))
  can-impersonate?-expr : any/c
  prop-expr : struct-type-property?
  val-expr : any/c
创建一个新的结构类型属性并返回三个值,就像 make-struct-type-property 一样:

  • 一个 structure type property descriptor

  • 一个 property predicate 程序

  • 一个 property accessor 程序

任何实现该属性的结构也将实现 #:property 和 #:methods 声明中给出的属性和 generic interfaces 。 属性的 val-expr 和 method-def 在属性被创建时被急切地评估,而不是当它被附加到结构类型时。

syntax

(make-generic-struct-type-property
   gen:name-id
   method-def
   ...)
创建一个新的结构类型属性并返回 structure type property descriptor 。

任何实现该属性的结构也将实现由 gen:name-id 给出的 generic interface , 并给出 method-def 。 当属性被创建时, method-def 被急切地评估,而不是当它被附加到结构类型时。
