4.14 Boxes

Boxes in Racket 指南 introduces boxes.

一个 box 就像一个单元素的向量,通常被用作最小的可变存储。

一个 box 可以是 mutable ,也可以是 immutable 。 当一个不可变的盒子被提供给 set-box! 这样的过程时,会引发 exn:fail:contract exception is raised 。 由默认阅读器(见 读取字符串) 生成的盒子常量是不可变的。 使用 immutable? 来检查一个盒子是否是不可变的。

一个字面的或打印的盒子以 #& 开头。 See 阅览 Boxes for information on reading boxes and 打印 Boxes for information on printing boxes.

procedure

(box? v) → boolean?

  v : any/c
如果 v 是一个 box,返回 #t ,否则返回 #f 。

参见 immutable-box? 和 mutable-box? 。

procedure

(box v) → box?

  v : any/c
返回一个包含 v 的新的可变盒子。

procedure

(box-immutable v) → (and/c box? immutable?)

  v : any/c
返回一个包含 v 的新的不可变的盒子。

procedure

(unbox box) → any/c

  box : box?
返回 box 的内容。

对于任何 v, (unbox (box v)) 和 (unbox (box-immutable v)) 返回 v。

procedure

(set-box! box v) → void?

  box : (and/c box? (not/c immutable?))
  v : any/c
将 box 的内容设置为 v 。

procedure

(unbox* box) → any/c

  box : (and box? (not/c impersonator?))

procedure

(set-box*! box v) → void?

  box : (and/c box? (not/c immutable?) (not/c impersonator?))
  v : any/c
就像 unbox 和 set-box! ,但仅限于对非 impersonators 的盒子工作。

Added in version 6.90.0.15 of package base.

procedure

(box-cas! box old new) → boolean?

  box : (and/c box? (not/c immutable?) (not/c impersonator?))
  old : any/c
  new : any/c
原子化地将 box 的内容更新为 new , 前提是 box 当前包含的值与 old 是 eq? , 并在这种情况下返回 #t 。 如果 box 不包含 old ,那么结果是 #f 。

如果没有其他 threads 或 futures 试图访问 box ,该操作等同于

(and (eq? old (unbox box)) (set-box! box new) #t)

除了 box-cas! 在某些平台上可能会假性失败。 也就是说,在低概率情况下,结果可能是 #f ,而 box 中的值保持不变, 即使 box 中含有 old 。

当Racket被编译为支持 futures 时, box-cas! 被保证使用硬件 比较 和 设置 操作。 box-cas! 的使用在 future 中安全进行(即允许 future thunk 继续并行)。 另见 机器内存顺序 。
