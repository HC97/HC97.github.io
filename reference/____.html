8.12 随机生成

procedure

(contract-random-generate ctc [fuel fail]) → any/c

  ctc : contract?
  fuel : 5 = exact-nonnegative-integer?
  fail : (or/c #f (-> any) (-> boolean? any)) = #f
试图随机生成一个与合同相匹配的值。 fuel 参数限制了生成器试图生成一个与合同相匹配的值的努力程度,是对生成值大小的一个粗略限制。

生成器可能无法生成一个值,这是因为有些合同没有相应的生成器 (例如,不是所有的谓词都有生成器),或者是因为没有足够的燃料。 在这两种情况下,函数 fail 被调用。 如果 fail 接受一个参数,当 ctc 没有生成器时,它会被 #t 调用, 当有生成器时,会被 #f 调用,但生成器最后返回 contract-random-generate-fail 。

Example:
> (for/list ([i (in-range 10)])
    (contract-random-generate (or/c integer? #f)))

'(#f #f #f -177.0 0 33 2147483647.0 #f #f 145)

Changed in version 6.1.1.5 of package base: 允许 fail 接受一个布尔值。

procedure

(contract-exercise [#:fuel fuel    
  #:shuffle? shuffle?]    
  val ...+) → void?
  fuel : exact-nonnegative-integer? = 10
  shuffle? : any/c = #f
  val : any/c
试图让 val 破坏它们的合约(如果有的话)。

使用 value-contract 来确定任何 val 是否有契约, 对于那些有契约的,使用关于契约形状的信息来探查和探测值。 例如,如果值是函数,它将使用契约来告诉它要向值提供什么参数。

参数 fuel 决定了 contract-exercise 试图打破这些值的力度。 它既控制练习的迭代次数,也控制练习中产生的中间值的大小。

参数 shuffle? 控制 contract-exercise 是否随机化练习的顺序。 如果 shuffle? 不是 #f, contract-exercise 将在每次练习迭代中洗牌合同的顺序。

Examples:
> (define/contract (returns-false x)
    (-> integer? integer?)
    ; does not obey its contract
    #f)
> (contract-exercise returns-false)

returns-false: broke its own contract

  promised: integer?

  produced: #f

  in: the range of

      (-> integer? integer?)

  contract from: (function returns-false)

  blaming: (function returns-false)

   (assuming the contract is correct)

  at: eval:2:0

> (define/contract (calls-its-argument-with-eleven f)
    (-> (-> integer? integer?) boolean?)
    ; f returns an integer, but
    ; we're supposed to return a boolean
    (f 11))
> (contract-exercise calls-its-argument-with-eleven)

calls-its-argument-with-eleven: broke its own contract

  promised: boolean?

  produced: 11

  in: the range of

      (-> (-> integer? integer?) boolean?)

  contract from:

      (function calls-its-argument-with-eleven)

  blaming: (function calls-its-argument-with-eleven)

   (assuming the contract is correct)

  at: eval:4:0

Changed in version 7.0.0.18 of package base: 增加了 shuffle? 的可选参数。

procedure

(contract-random-generate/choose c fuel) → (or/c #f (-> c))

  c : contract?
  fuel : exact-nonnegative-integer?
这个函数与 contract-random-generate 相似,但它旨在与组合器一起使用, 这些组合器根据它们所拥有的子合同来生成数值。它必须在 contract-random-generate (和 contract-exercise) 创建生成器时被调用。 更准确地说, contract-random-generate/choose 只适用于 build-contract-property, build-chaperone-contract-property 或 build-flat-contract-property 中的 generate 和 exercise 参数, 并且只在调用 generate (和 exercise)的动态范围内可用。 也就是说,在它收到 c 和 fuel 参数之后,在它返回 thunk(或 exerciser)之前。

contract-random-generate/choose 永远不会失败, 但它可能会逃回一个封闭的调用或对 contract-random-generate 的原始调用。

它在几种可能的生成策略中选择一种,因此它可能不会实际使用与 c 相关的生成器, 而是使用它通过 contract-random-generate-stash 知道的与 c 匹配的存储值。

Added in version 6.1.1.5 of package base.

value

contract-random-generate-fail : contract-random-generate-fail?

一个原子值,用来表示生成器未能生成一个值。

Added in version 6.1.1.5 of package base.

procedure

(contract-random-generate-env? v) → boolean?

  v : any/c
识别合同生成环境。

Added in version 6.1.1.5 of package base.

procedure

(contract-random-generate-stash env c v) → void?

  env : contract-random-generate-env?
  c : contract?
  v : c
应该用被测程序在合约生成过程中提供的值来调用这个程序。 例如,当 (-> (-> integer? integer?) integer?) 被生成时,它可以调用其参数函数。 该参数函数可能会返回一个整数,如果是这样, 该整数应该通过调用 contract-random-generate-stash 来保存,这样它就可以被其他整数生成器使用。

Added in version 6.1.1.5 of package base.

procedure

(contract-random-generate-get-current-environment)

 → contract-random-generate-env?
返回当前正在生成的环境。 这个函数只能在合约生成的动态范围内被调用。 它的目的是在构建合约生成器的过程中抓取, 然后在生成过程中与 contract-random-generate-stash 一起使用。

Added in version 6.1.1.5 of package base.
