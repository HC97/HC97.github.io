12.7 语法对象属性

每个语法对象都有一个相关的 syntax property 列表,可以用 syntax-property 来查询或扩展。 一个属性被设置为 preserved 或不保留; 一个保留的属性对于一个被编入字节串或 ".zo" 文件的语法对象来说是被保留的,其他属性在编入时被丢弃。

在 read-syntax 中,阅读器将保留的 'paren-shape 属性附加到任何对或矢量语法对象上, 这些对象是通过解析对 [ 和 ] 或 { 和 } 生成的; 在前一种情况下,属性值为 #\[ ,在后一种情况下,属性值为 #\{ 。 syntax 形式将模板源中的任何 'paren-shape 属性复制到相应的生成语法中。

转化器的语法输入和转化器的语法结果都可以有相关的属性。 语法扩展器对这两组属性进行合并:原始属性和结果中不存在的每一个属性都被复制到结果中, 并将两者中存在的属性值用 cons 组合起来(结果值在前,原始值在后), 如果其中一个值被preserved ,则保留 cons 的值。

然而,在执行合并之前,语法扩展器会使用 'origin 键自动向原始语法对象添加一个属性。 如果源语法没有 'origin 属性,它就被设置为空列表。 然后,还是在合并之前,触发宏扩展的标识符(作为语法)被 cons 到目前为止的 'origin 属性。 这样, 'origin 属性就记录了(以相反的顺序)产生一个扩展表达式的宏扩展序列。 通常, 'origin 值是一个标识符的列表,但是转化器可能会返回已经被展开的语法,在这种情况下, 'origin 列表可以包含合并后的其他列表。 syntax-track-origin 程序实现了这种追踪。 'origin 属性被添加为 preserved 。

除了用于一般宏扩展的 'origin 跟踪, Racket 还为扩展后的语法添加属性(通常使用 syntax-track-origin) 以记录额外的扩展细节:

  • 当一个 begin 形式被拼接到一个有内部定义的序列中时(见 内部定义), syntax-track-origin 被应用到 begin 主体的每个拼接元素中。 syntax-track-origin 的第二个参数是 begin 形式, 第三个参数是 begin 关键字(从拼接的形式中提取)。

  • 当内部 define-values 或 define-syntaxes 形式被转换为 letrec-syntaxes+values 形式时(参见 内部定义), syntax-track-origin 被应用于每个生成的绑定子句。 syntax-track-origin 的第二个参数是转换后的形式, 第三个参数是转换后形式的 define-values 或 define-syntaxes 关键字形式。

  • 当一个 letrec-syntaxes+values 表达式被完全展开时,语法绑定消失, 结果是 letrec-values 形式(如果未展开的形式包含非语法绑定), 或者只有 letrec-syntaxes+values 形式的主体(如果主体包含多个表达式,则用 begin 包裹)。 为了记录消失的语法绑定,在扩展结果中添加了一个属性:一个来自消失的绑定的不可变的标识符列表, 作为 'disappeared-binding 属性。

  • 当子类型的 struct 形式被展开时,用于引用基类型的标识符不会出现在展开中。 因此, struct 转换器将标识符作为 'disappeared-use 属性添加到扩展结果中。

  • 当 rename transformer 被用来替换 set! 目标时, syntax-track-origin 被用于目标标识符(与标识符被用作表达式时相同)。

  • 当发现对一个模块中未导出或受保护的标识符的引用时, 'protected 属性会被添加到标识符中, 并带有 #t 值。

  • 当 read-syntax 生成一个语法对象时,它会给该对象附加一个属性(使用一个私钥),以标记该对象来自于读取。 syntax-original? 谓词会查找该属性以识别此类语法对象。 (更多信息请参见 语法对象内容 。 该属性不会被扩展器从宏转换器的输入端转移到其输出端,也不会被 syntax-track-origin 所转移)。

关于 'disappeared-use 和 'disappeared-binding 属性的一个客户端, 也请参见 Check Syntax 。

参见 扩展模块的信息 ,了解由模块声明的扩展产生的属性信息。 参见 lambda 和 推断值的名称 ,了解编译程序时识别的属性信息。 参见 current-compile ,了解关于属性和字节码的信息。

procedure

(syntax-property stx key v [preserved?]) → syntax?

  stx : syntax?
  key : (if preserved? (and/c symbol? symbol-interned?) any/c)
  v : any/c
  preserved? : any/c = (eq? key 'paren-shape)
(syntax-property stx key) → any
  stx : syntax?
  key : any/c
三参数或四参数形式通过将一个任意的属性值 v 与键 key 关联来扩展 stx ; 结果是一个具有关联的新语法对象(而 stx 本身没有变化)。 如果 preserved? 为真,该属性将被添加为 preserved , 在这种情况下, key 必须是一个 interned 符号, 而 v 应该是一个如下所述的值,可以被保存在sharshaled字节码中。

双参数形式返回一个任意的属性值,该值与 key 的 stx 相关, 如果没有值与 key 的 stx 相关,则返回 #f 。 如果 stx 是 tainted, 那么具有结果值的语法对象也是有污点的。

为了支持对字节码的处理,保留的语法属性的值必须是一个非循环的值,它要么是

  • 一 pair 包含允许的保留的属性值;

  • 一个包含允许的保留的属性值的 vector (未被掩盖为不可变的);

  • 一个包含允许的保留属性值的 box (未被遮盖为不可变);

  • 一个不可变的 prefab 结构,包含允许的保留属性值;

  • 一个不可变的 hash table ,其键和值都是允许的保全属性值;

  • 一个 syntax object; 或

  • 一个空的列表、 symbol, number, character, string, byte string 或 regexp value 。

任何其他的保全属性值都会在试图将拥有的语法对象编入字节码形式时触发异常。

Changed in version 6.4.0.14 of package base: 增加了 preserved? 参数。

procedure

(syntax-property-remove stx key) → syntax?

  stx : syntax?
  key : any/c
返回一个像 stx 一样的语法对象,但没有 key 的属性(如果有的话)。

Added in version 6.90.0.20 of package base.

procedure

(syntax-property-preserved? stx key) → boolean?

  stx : syntax?
  key : (and/c symbol? symbol-interned?)
如果 stx 为 key preserved 了属性值, 返回 #t ,否则返回 #f 。

Added in version 6.4.0.14 of package base.

procedure

(syntax-property-symbol-keys stx) → list?

  stx : syntax?
返回所有作为键的符号的列表,这些符号在 stx 中有相关的属性。 Uninterned 符号(见 符号)不包括在结果列表中。

procedure

(syntax-track-origin new-stx    
  orig-stx    
  id-stx) → any
  new-stx : syntax?
  orig-stx : syntax?
  id-stx : identifier?
将属性添加到 new-stx 中,其方式与宏扩展将属性添加到转化器结果中一样。 特别是,它将 orig-stx 的属性合并到 new-stx 中, 首先添加 id-stx 作为 'origin 属性, 然后删除由 syntax-original? 识别的属性,并返回属性扩展后的语法对象。 在一个宏转化器中使用 syntax-track-origin 过程, 该转化器丢弃语法(对应于带有关键字 id-stx 的 orig-stx ), 在其位置上留下一些其他的语法(对应于 new-stx)。

例如,表达式

(or x y)

展开为

(let ([or-part x]) (if or-part or-part (or y)))

反过来,它又扩展为

(let-values ([(or-part) x]) (if or-part or-part y))

最终表达式的语法对象将有一个 'origin 属性, 其值是 (list (quote-syntax let) (quote-syntax or)) 。

Changed in version 7.0 of package base: 在转移到 new-stx 的属性中包括了 syntax-original? 属性。
Changed in version 8.2.0.7: 更正为从转移到 new-stx 的集合中删除了 syntax-original? 属性。
