14.7 Custodians

关于 Racket 监护人模型的基本信息,请参见 保管人 。

procedure

(custodian? v) → boolean?

  v : any/c
如果 v 是一个 custodian 的值,返回 #t ,否则返回 #f 。

procedure

(make-custodian [cust]) → custodian?

  cust : (and/c custodian? (not/c custodian-shut-down?))
   = (current-custodian)
创建一个新的 custodian,它从属于 cust 。 当 cust 被指示(通过 custodian-shutdown-all) 关闭其所有的管理值时, 新的下级托管人也会自动被指示关闭其管理值。

procedure

(custodian-shutdown-all cust) → void?

  cust : custodian?

在 racket/gui/base 中, 由 cust 管理的 eventspaces 也被关闭了。

关闭所有由 cust (及其下级)管理的 file-stream ports, TCP ports, TCP listeners 和 UDP sockets , 并清空所有与 cust (及其下级)相关的 custodian box 。 它还删除了 cust (及其下属)作为所有线程的管理者;当一个线程没有管理者时, 它就会被杀死(或暂停;见 thread/suspend-to-kill) 如果当前线程要被杀死, 所有其他关闭动作都会在杀死线程之前发生。

如果 cust 已经被关闭了,那么 custodian-shutdown-all 就没有作用。当一个托管人被关闭并且它有下级托管人时, 下级不仅被关闭,而且不再算作下级。

procedure

(custodian-shut-down? cust) → boolean?

  cust : custodian?
如果 cust 已经被 custodian-shutdown-all 关闭,或者它是一个被关闭的保管人的下属, 则返回 #t ,否则返回 #f 。

Added in version 6.11.0.5 of package base.

parameter

(current-custodian) → custodian?

(current-custodian cust) → void?
  cust : custodian?

托管人也管理来自 racket/gui/base 的 eventspaces 。

一个 parameter ,决定了一个托管人承担新创建线程、 file-stream ports, TCP 端口, TCP listeners, UDP sockets 和 byte converters 。

procedure

(custodian-managed-list cust super) → list?

  cust : custodian?
  super : custodian?
返回一个立即管理的对象(不包括 custodian box)和 cust 的下级保管人的列表, 其中 cust 本身从属于 super (直接或间接)。 如果 cust 不是严格意义上的从属于 super ,会引发 exn:fail:contract exception is raised 。

如果 cust 已经被关闭,结果是 '() 。 如果 cust 是被关闭的托管人的下级,那么它就不能成为 super 的下级。

procedure

(custodian-memory-accounting-available?) → boolean?

内存核算通常是可用的,但在 CGC 实现中没有。

如果 Racket 在编译时支持每个托管人的内存计算,则返回 #t ,否则返回 #f 。

procedure

(custodian-require-memory limit-cust    
  need-amt    
  stop-cust) → void?
  limit-cust : custodian?
  need-amt : exact-nonnegative-integer?
  stop-cust : custodian?
如果 Racket 在编译时支持每个托管人的内存核算,则注册一个所需的内存检查,否则会引发 exn:fail:unsupported exception is raised 。

如果检查被注册了,并且如果 Racket 后来在垃圾收集后达到一个状态(见 垃圾回收), 分配给 limit-cust 的 need-amt 字节会失败或触发一些关闭, 那么 stop-cust 会被关闭。

stop-cust 必须是 limit-cust 的一个从属监护人。

procedure

(custodian-limit-memory limit-cust    
  limit-amt    
  [stop-cust]) → void?
  limit-cust : custodian?
  limit-amt : exact-nonnegative-integer?
  stop-cust : custodian? = limit-cust
如果Racket在编译时支持每个托管人的内存核算,则注册一个有限内存检查,否则会引发 exn:fail:unsupported exception is raised 。

如果检查被注册了,并且如果 Racket 后来在垃圾回收后达到 了limit-cust 拥有超过 limit-amt 字节的状态(见 垃圾回收) ,那么 stop-cust 将被关闭。

一个托管人的限制只在垃圾回收后被检查,除了在某些单独大于托管人限制的大的分配中也可能被检查。 一次垃圾回收可能会关闭多个托管人,即使只关闭其中一个托管人会减少其他托管人的内存使用。

为了可靠地关闭,custodian-limit-memory 的 limit-amt 必须远远低于可用的内存总量 (减去可能被使用但未被计入 limit-cust 的内存大小)。 此外,如果最初计入 limit-cust 的单个分配可以是任意大的, 那么 stop-cust 必须与 limit-cust 相同, 这样,过大的即时分配可以用 exn:fail:out-of-memory 异常来拒绝。

新的内存分配将被计入运行 thread 的管理托管人。 换句话说,一个托管人的限制只适用于它所管理的线程的分配。 参见 call-in-nested-thread 以了解更简单的设置。

Examples:
> (require racket/async-channel)
> (define ch (make-async-channel))
> (parameterize ([current-custodian (make-custodian)])
    (thread-wait
     (thread
      (λ ()
        (with-handlers ([exn:fail:out-of-memory?
                         (λ (e) (async-channel-put ch e))])
          (custodian-limit-memory (current-custodian) (* 1024 1024))
          (make-bytes (* 4 1024 1024))
          (async-channel-put ch "Not OK")))))
    (async-channel-get ch))

(exn:fail:out-of-memory "out of memory" #<continuation-mark-set>)

> (define cust (make-custodian))
> (with-handlers ([exn:fail:out-of-memory?
                   (λ (e) (error "Caught OOM exn"))])
    (call-in-nested-thread
     (λ ()
       (custodian-limit-memory cust (* 1024 1024))
       (make-bytes (* 4 1024 1024))
       "Not OK")
     cust))

Caught OOM exn

Non-examples:
> (parameterize ([current-custodian (make-custodian)])
    (custodian-limit-memory (current-custodian) (* 1024 1024))
    ; Allocation of make-bytes is charged to the current thread’s
    ; managing custodian, not the new custodian.
    (make-bytes (* 4 1024 1024))
    "Not OK")

"Not OK"

procedure

(make-custodian-box cust v) → custodian-box?

  cust : custodian?
  v : any/c
只要 cust 没有被关闭,就返回一个包含 v 的 custodian box 。 如果 cust 已经被关闭,保管箱的值会被立即删除。

一个 custodian box 是一个 synchronizable event (见 事件)。 当它的托管盒被关闭时, custodian box 就会准备好; the synchronization result of a custodian box is the custodian box itself 。

procedure

(custodian-box? v) → boolean?

  v : any/c
如果 v 是由 make-custodian-box 产生的 custodian box , 返回 #t ,否则返回 #f 。

procedure

(custodian-box-value cb) → any

  cb : custodian-box?
返回给定 custodian box 中的值,如果值已经被移除,则返回 #f 。
