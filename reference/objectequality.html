6.8 对象的平等性和哈希

默认情况下,作为不同类的实例或作为非透明类的实例的对象只有在 eq? 时才是 equal? 。 像透明结构一样,作为同一透明类的实例的两个对象(即该类的每个超类都有 #f 作为其检查器), 当它们的字段值 equal? 时,就是 equal? 。

要自定义一个类实例与其他实例进行 equal? 比较的方式,请实现 equal<%> 接口。

interface

equal<%> : interface?

equal<%> 接口包括三个方法,它们类似于为结构类型提供的 prop:equal+hash 的函数:

  • equal-to? — 接受两个参数。 第一个参数是一个对象,它是同一个类的实例(或一个没有重新声明其实现 equal<%> 的子类),并且正在与目标对象比较。 第二个参数是一个类似于 equal? 的两个参数的过程,应该用于递归平等测试。 如果对象和方法的第一个参数相等,其结果应该是一个真值,否则是 #f 。

  • equal-hash-code-of — 接收一个参数,这是一个由一个参数组成的过程,应该用于递归哈希代码计算。 其结果应该是一个精确的整数,代表目标对象的哈希代码。

  • equal-secondary-hash-code-of — 取一个参数, 这是一个由一个参数组成的过程,应该用于递归哈希代码计算。 其结果应该是一个精确的整数,代表目标对象的二级哈希代码。

equal<%> 接口的不同之处在于,声明接口的实现与继承接口是不同的。 只有当两个对象是明确实现 equal<%> 的最具体祖先的类的实例时,它们才能相等。
参见 prop:equal+hash ,了解更多关于平等比较和哈希码的信息。 equal<%> 接口是用 interface* 和 prop:equal+hash 实现的。

例如:
#lang racket
 
;; Case insensitive words:
(define ci-word%
  (class* object% (equal<%>)
 
    ;; Initialization
    (init-field word)
    (super-new)
 
    ;; We define equality to ignore case:
    (define/public (equal-to? other recur)
      (string-ci=? word (get-field word other)))
 
    ;; The hash codes need to be insensitive to casing as well.
    ;; We'll just downcase the word and get its hash code.
    (define/public (equal-hash-code-of hash-code)
      (hash-code (string-downcase word)))
 
    (define/public (equal-secondary-hash-code-of hash-code)
      (hash-code (string-downcase word)))))
 
;; We can create a hash with a single word:
(define h (make-hash))
(hash-set! h (new ci-word% [word "inconceivable!"]) 'value)
 
;; Lookup into the hash should be case-insensitive, so that
;; both of these should return 'value.
(hash-ref h (new ci-word% [word "inconceivable!"]))
(hash-ref h (new ci-word% [word "INCONCEIVABLE!"]))
 
;; Comparison fails if we use a non-ci-word%:
(hash-ref h "inconceivable!" 'i-dont-think-it-means-what-you-think-it-means)
