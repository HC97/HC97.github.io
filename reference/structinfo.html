5.7 结构类型转化器绑定

struct 形式将结构类型的名称作为 transformer 绑定, 记录与结构类型绑定的其它标识符、构造函数过程、谓词过程以及字段访问器和突变器过程。 这些信息可以在其他表达式的扩展过程中通过 syntax-local-value 来使用。

例如,子类型的 struct 变体使用基类型名称 t 来寻找包含基类型描述符的变量 struct:t ;它还将基类型的字段访问器和突变器信息折叠到子类型的信息中。 再比如,match 形式使用类型名称来查找结构类型的谓词和字段访问器。 在 unit 的导入签名中的 struct 形式会使 unit 转化器产生关于导入结构类型的信息,因此 match 和子类型的 struct 形式在单元内工作。

结构类型的扩展时间信息可以直接表示为六个元素的列表(与封装的过程必须返回的元素相同):

  • 与结构类型描述符绑定的标识符,如果不知道,则为 #f ;

  • 与结构类型的构造函数绑定的标识符,如果不知道,则为 #f ;

  • 与结构类型的谓词绑定的标识符,如果不知道,则为 #f ;

  • 绑定到结构类型的字段访问器的标识符列表,可以选择将 #f 作为列表的最后一个元素。 最后一个元素是 #f ,表示该结构类型可能有额外的字段,否则该列表就是该结构类型中字段数量的一个可靠指标。 此外,访问器是按照相应构造函数参数的相反顺序列出的。(反向的顺序使得子类型和它的基本类型的列表可以共享)。

  • 绑定到结构类型的字段突变器的标识符列表, 或者对于每个没有已知突变器的字段使用 #f , 也可以选择使用额外的 #f 作为列表的最后一个元素(如果访问器列表有这样一个 #f)。 列表的顺序和最后一个 #f 的含义与访问器标识符相同,并且变体列表的长度与访问器列表的长度相同。

  • 一个确定结构类型的超级类型的标识符,如果超级类型(如果有的话)未知,则为 #f , 如果没有超级类型,则为 #t 。 如果指定了一个超级类型,该标识符也被绑定到结构类型的扩展时间信息。

代替这种直接表示法,表示法可以是由 make-struct-info (或 struct:struct-info 的一个子类型的实例)创建的结构,它封装了一个没有参数并返回六个元素列表的过程。 另外,表示法可以是一个结构,其类型具有 prop:struct-info structure type property。 最后,表示法可以是一个从 struct:struct-info 派生的结构类型的实例, 或者是具有 prop:struct-info 属性的结构类型的实例, 它也实现了 prop:procedure ,并且该实例被 make-set!-transformer 进一步包装了。 此外,该表示法可以实现 prop:struct-auto-info 和 prop:struct-field-info 属性。

使用 struct-info? 来识别所有允许的信息形式, 使用 extract-struct-info 来从任何表示法中获得一个列表。

语法形式的实现者可以期望该形式的用户知道关于结构类型的哪种信息。 例如, match 实现可以与包含不完整的访问器绑定的结构信息一起工作, 因为用户被认为知道在 match 表达式的上下文中有哪些信息可用。 特别是, match 表达式可以以 unit 的形式出现在导入的结构类型中, 在这种情况下,用户被期望知道结构类型的签名中列出的字段集。

 (require racket/struct-info) package: base
The bindings documented in this section are provided by the racket/struct-info library, not racket/base or racket.

procedure

(struct-info? v) → boolean?

  v : any/c
如果 v 是一个具有正确形状的六元素列表,用于表示结构类型信息, 是由 make-struct-info 封装的过程,是具有 prop:struct-info 属性的结构, 或者是由 struct:struct-info 或 prop:struct-info 派生的结构类型, 并由 make-set!-transformer 包装,则返回 #t 。

procedure

(checked-struct-info? v) → boolean?

  v : any/c
如果 v 是由 make-struct-info 封装并由 struct 产生的过程, 则返回 #t ,但只有在没有指定父类型或者父类型也是通过与这种值绑定的转换器指定的情况下才会返回。

procedure

(make-struct-info thunk) → struct-info?

  thunk : (-> (and/c struct-info? list?))
封装了一个以列表形式返回结构类型信息的 thunk。 注意访问器是以相反的顺序列出的,正如 结构类型转化器绑定 中提到的。 请注意,对于用该方法创建的结构类型信息,字段名并没有很好地定义, 所以它很可能不能很好地与 struct-copy 和 struct* 这样的形式配合。

Examples:
> (define (new-pair? x) (displayln "new pair?") (pair? x))
> (define (new-car x) (displayln "new car") (car x))
> (define (new-cdr x) (displayln "new cdr") (cdr x))
> (define-syntax new-list
    (make-struct-info
     (λ () (list #f
                 #'cons
                 #'new-pair?
                 (list #'new-cdr #'new-car)
                 (list #f #f)
                 #t))))
> (match (list 1 2 3)
    [(new-list hd tl) (append tl (list hd))])

new pair?

new car

new cdr

'(2 3 1)

Examples:
> (struct A (x y))
> (define (new-A-x a) (displayln "A-x") (A-x a))
> (define (new-A-y a) (displayln "A-y") (A-y a))
> (define (new-A? a) (displayln "A?") (A? a))
> (define-syntax A-info
    (make-struct-info
     (λ () (list #'A
                 #'A
                 #'new-A?
                 (list #'new-A-y #'new-A-x)
                 (list #f #f)
                 #t))))
> (define-match-expander B
    (syntax-rules () [(_ x ...) (A-info x ...)]))
> (match (A 10 20)
    [(B x y) (list y x)])

A?

A-x

A-y

'(20 10)

procedure

(extract-struct-info v) → (and/c struct-info? list?)

  v : struct-info?
提取 v 所代表的结构类型信息的列表形式。

value

struct:struct-info : struct-type?

由 make-struct-info 返回的结构类型的 structure type descriptor 。 这个 structure type descriptor 主要用于创建结构子类型。 该结构类型包括一个卫兵,它以与 make-struct-info 相同的方式检查一个实例的第一个字段。

value

prop:struct-info : struct-type-property?

structure type property ,用于创建新的结构类型,如 struct:struct-info 。 该属性值必须是一个只有一个参数的过程,它接收一个实例结构并以列表形式返回结构类型信息。

value

prop:struct-auto-info : struct-type-property?

procedure

(struct-auto-info? v) → boolean?

  v : any/c

procedure

(struct-auto-info-lists sai)

 → (list/c (listof identifier?) (listof identifier?))
  sai : struct-auto-info?
prop:struct-auto-info 属性的实现是为了提供静态信息, 即结构类型的哪些访问器和突变器标识符对应于 #:auto 字段(因此它们在构造函数中没有相应的参数)。 该属性的值必须是一个接受该属性的实例结构的过程, 其结果必须是两个适合作为 struct-auto-info-lists 结果的标识符列表。

struct-auto-info? 谓词可以识别实现 prop:struct-auto-info 属性的值。

struct-auto-info-lists 函数从实现 prop:struct-auto-info 属性的值中提取两个标识符列表。 第一个列表应该是 sai 描述的结构类型的访问器标识符的子集,第二个列表应该是突变器标识符的子集。 这两个子集对应于 #:auto 字段。

value

prop:struct-field-info : struct-type-property?

procedure

(struct-field-info? v) → boolean?

  v : any/c

procedure

(struct-field-info-list sfi) → (listof symbol?)

  sfi : struct-field-info?
prop:struct-field-info 属性的实现是为了提供关于结构类型中字段名的静态信息。 该属性的值必须是一个接受该属性的实例结构的过程, 其结果必须是一个适合作为 struct-field-info-list 结果的符号列表。

struct-field-info? 谓词可以识别实现 prop:struct-field-info 属性的值。。

struct-field-info-list 函数从实现了 prop:struct-field-info 属性的值中提取一个符号列表。 该列表应包含每一个直接的字段名(即不包括来自其超级结构类型的字段),顺序相反。

Examples:
> (struct foo (x))
> (struct bar foo (y z))
> (define-syntax (get-bar-field-names stx)
    #`'#,(struct-field-info-list (syntax-local-value #'bar)))
> (get-bar-field-names)

'(z y)

Added in version 7.7.0.9 of package base.
