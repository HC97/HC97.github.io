13.5 Writing

procedure

(write datum [out]) → void?

  datum : any/c
  out : output-port? = (current-output-port)
向 out 写 datum ,通常是以核心数据类型的实例可以被读回的方式。 如果 out 有一个通过 port-write-handler 关联的处理程序,那么这个处理程序就被调用。 否则,将使用 default printer (在写 write 模式下),由各种参数配置。

参见 打印机 以了解关于默认打印机的更多信息。 特别要注意的是,由于初始循环检查, write 可能需要与被打印值的深度成比例的内存。

Examples:
> (write 'hi)

hi

> (write (lambda (n) n))

#<procedure>

> (define o (open-output-string))
> (write "hello" o)
> (get-output-string o)

"\"hello\""

procedure

(display datum [out]) → void?

  datum : any/c
  out : output-port? = (current-output-port)
向 out 显示 datum ,与 write 类似, 但通常是以基于字节和字符的数据类型被写成原始字节或字符的方式。 如果 out 通过 port-display-handler 有一个与之相关的处理程序,那么该处理程序将被调用。 否则,将使用 default printer (在 display 模式下),由各种参数配置。

参见 打印机 以了解关于默认打印机的更多信息。 特别要注意的是,由于初始循环检查, display 可能需要与被打印值的深度成比例的内存。

procedure

(print datum [out quote-depth]) → void?

  datum : any/c
  out : output-port? = (current-output-port)
  quote-depth : (or/c 0 1) = 0
打印 datum 到 out 。 如果 out 有一个通过 port-print-handler 关联的处理程序,那么这个处理程序会被调用。 否则,将调用 global-port-print-handler 指定的处理程序; 默认处理程序在 print 模式下使用 default printer 。

当 print-as-expression 参数被设置为 #t 时, 可选的 quote-depth 参数可以调整打印。 在这种情况下, quote-depth 指定了打印 datum 的起始引号深度。

提供 print 的理由是, display 和 write 都有特定的输出约定, 而这些约定限制了环境改变 display 和 write 程序行为的方式。 不应该为 print 假定任何输出约定, 这样环境就可以自由地以任何方式修改 print 产生的实际输出。

procedure

(writeln datum [out]) → void?

  datum : any/c
  out : output-port? = (current-output-port)
与 (write datum out) 相同,后跟 (newline out) 。

Added in version 6.1.1.8 of package base.

procedure

(displayln datum [out]) → void?

  datum : any/c
  out : output-port? = (current-output-port)
与 (display datum out) 相同,后跟 (newline out), 与Pascal或Java中的 println 类似。

procedure

(println datum [out quote-depth]) → void?

  datum : any/c
  out : output-port? = (current-output-port)
  quote-depth : (or/c 0 1) = 0
与 (print datum out quote-depth) 相同,后跟 (newline out) 。

println 函数不等同于其他语言中的 println , 因为 println 使用的打印约定更接近于 write 而不是 display 。 如果要与其他语言中的 println 更接近,请使用 displayln 。

Added in version 6.1.1.8 of package base.

procedure

(fprintf out form v ...) → void?

  out : output-port?
  form : string?
  v : any/c
打印格式化的输出到 out, 其中 form 是一个直接打印的字符串,除了特殊的格式化转义:

  • ~n 或 ~% 打印一个换行符(这相当于字面格式字符串中的 \n)

  • ~a 或 ~A display v 中的下一个参数

  • ~s 或 ~S write v 中的下一个参数

  • ~v 或 ~V print v 中的下一个参数

  • ~.‹c› 其中 ‹c› 是 a, A, s, S, v 或 V: 将默认处理程序的 display, write 或 print 输出截断为 (error-print-width) 字符,如果未截断的输出更长,则使用 ... 作为最后三个字符

  • ~e 或 ~E 使用当前的错误值转换处理程序 (见 error-value->string-handler)和当前的错误打印宽度输出 v 中的下一个参数

  • ~c 或 ~C 在 v 中 write-char 下一个参数; 如果下一个参数不是字符,就会引发 exn:fail:contract exception is raised

  • ~b 或 ~B 以二进制打印 v 中的下一个参数; 如果下一个参数不是一个精确的数字,就会引发 exn:fail:contract exception is raised

  • ~o 或 ~O 以八进制打印 v 中的下一个参数; 如果下一个参数不是一个精确的数字,会引发 exn:fail:contract exception is raised

  • ~x 或 ~X 以十六进制打印 v 中的下一个参数; 如果下一个参数不是一个精确的数字,就会产生 exn:fail:contract exception is raised

  • ~~ 打印一个斜体字。

  • ~‹w›, 其中 ‹w› 是一个空白字符(见 char-whitespace?), 跳过 form 中的字符,直到遇到一个非空白字符或遇到第二个行末(以先发生者为准)。 在所有平台上,行末可以是 #\return, #\newline 或 #\return 后紧跟 #\newline 。

form 字符串不能包含任何不是上述转义之一的 ~ , 否则会引发 exn:fail:contract exception is raised 异常。 当格式字符串需要的 v 多于提供的数量时,就会引发 exn:fail:contract exception is raised 。 同样地,当提供的 v 多于格式化字符串使用的数量时,会引发 exn:fail:contract exception is raised 。

Example:
> (fprintf (current-output-port)
           "~a as a string is ~s.\n"
           '(3 4)
           "(3 4)")

(3 4) as a string is "(3 4)".

procedure

(format form v ...) → string?

  form : string?
  v : any/c
格式化为一个字符串。 其结果与

(let ([o (open-output-string)])
  (fprintf o form v ...)
  (get-output-string o))

Example:
> (format "~a as a string is ~s.\n" '(3 4) "(3 4)")

"(3 4) as a string is \"(3 4)\".\n"

parameter

(print-syntax-width)

 → (or/c +inf.0 0 (and/c exact-integer? (>/c 3)))
(print-syntax-width width) → void?
  width : (or/c +inf.0 0 (and/c exact-integer? (>/c 3)))
一个 parameter ，控制 syntax objects 打印。 在 #<syntax...> 内(如果有的话,在 syntax object 的源位置之后), 最多可使用 width 字符来显示语法对象的基准形式,其中 ... 被用作最后三个字符, 否则打印出来的形式会比 width 字符长。 width 的值为 0 意味着完全不显示基准点。

procedure

(port-print-handler out)

 → ((any/c output-port?) ((or/c 0 1)) . ->* . any)
  out : output-port?
(port-print-handler out proc) → void?
  out : output-port?
  proc : (any/c output-port? . -> . any)
获取或设置 out 的 port write handler, port display handler 或 port print handler 。 当 write, display 或 print (分别)应用于端口时, 这个处理程序被调用以输出到端口。 每个处理程序必须接受两个参数:要打印的值和目标端口。处理程序的返回值被忽略。

一个 port print handler 可以选择接受第三个参数,它与要 print 的可选的第三个参数相对应; 如果给 port-print-handler 的程序不接受第三个参数,它将被包裹在一个抛弃可选的第三个参数的程序中。

默认的端口显示和写入处理程序用 Racket 的内置打印机打印 Racket 表达式(见 打印机 。 默认打印处理程序调用全局端口打印处理程序( global-port-print-handler 参数的值); 默认全局端口打印处理程序与默认写入处理程序相同。

procedure

(global-port-print-handler)

 → (->* (any/c output-port?) ((or/c 0 1)) any)
(global-port-print-handler proc) → void?
  proc : 
(or/c (->* (any/c output-port?) ((or/c 0 1)) any)
      (any/c output-port? . -> . any))
一个决定 global port print handler 的 parameter , 由默认端口打印处理程序(见 port-print-handler) 调用,将数值 print 到端口。 默认值等同于 default-global-port-print-handler 。

global port print handler 可以选择接受第三个参数, 这个参数与要 print 的可选第三个参数相对应。 如果给 global-port-print-handler 的过程不接受第三个参数, 它就会被包裹在一个过程中,这个过程会丢弃可选的第三个参数。

  [print-depth]) → void?

使用内置打印机(见 打印机) 在 print 模式下将 v 打印到 out 。
