7.10 单一单元模块

当 racket/unit 被用作带有 #lang 的语言名称时,模块主体被视为一个单元主体。 该主体必须符合以下 module-body 语法:

  module-body = 
require-decl ...
(import tagged-sig-expr ...)
(export tagged-sig-expr ...)
init-depends-decl
unit-body-expr-or-defn
...
     
  require-decl = (require require-spec ...)
  | (begin require-decl ...)
  | derived-require-form

在任何数量的 require-decl 之后,模块的内容与 unit 主体相同。

由此产生的单元被导出为 base@, 其中 base 来自包围模块的名称(即,其符号名称,或其路径,不含目录和文件后缀)。 如果模块名称以 -unit 结尾,那么 base 对应于 -unit 之前的模块名称。 否则,模块名将作为 base 。
