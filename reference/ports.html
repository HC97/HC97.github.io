13.1 端口

Ports 产生和/或消耗字节。 一个 input port 产生字节,而一个 output port 消耗字节(有些端口既是输入端口又是输出端口)。 当一个输入端口被提供给一个基于字符的操作时,字节被解码为字符,而基于字符的输出操作也同样将字符编码为字节; 参见 编码和地域 。 除了字节和编码为字节的字符之外,一些端口可以产生和/或消耗任意的值作为 special 的结果。

当一个端口对应于一个文件、网络连接或其他一些系统资源时, 必须通过 close-input-port 或 close-output-port 明确地关闭它 (或通过 custodian-shutdown-all 间接地关闭),以释放与该端口相关的低级资源。 对于任何类型的端口,在它被关闭后,试图从该端口读取或写入该端口会引发 exn:fail 。

由 input port 产生的数据可以被读取或 peeked 。 当数据被读取时,它被认为是被消耗并从端口的数据流中移除。 当数据被 peek 时,它将保留在端口的数据流中,并由下一次读取或 peek 再次返回。 之前偷看的数据可以被 committed, 这导致数据从端口中被移除,就像读一样, 可以通过一个 synchronizable event 与其他 peek 或读的尝试同步。 读取和 peek 操作通常都是阻塞的,也就是说,读取或 peek 操作直到数据从端口可用时才会完成; 读取和 peek 操作的非阻塞变体也是可用的。

全局变量 eof 被绑定到文件结束值上,并且 eof-object? 只在应用于此值时返回 #t 。 从一个端口读取数据会在该端口没有更多数据时产生文件结束的结果,但一些端口也可能在数据流中返回文件结束。 例如,当用户输入 control-D 时,连接到 Unix 终端的端口会返回 end-of-file; 如果用户提供了更多的输入,端口会在 end-of-file 之后返回额外的字节。

每个端口都有一个名字,由 object-name 报告。 这个名字可以是任何值, 它主要用于报告错误。 read-syntax 程序使用输入端口的名称作为它所产生的 syntax objects 的默认源位置。

一个端口可以作为一个 synchronizable event 使用。 当 read-byte 不会阻塞时,一个输入端口就 ready for synchronization , 而当write-bytes-avail 不会阻塞时, 或者当端口包含缓冲字符且 write-bytes-avail* 可以刷新部分缓冲区时 (尽管 write-bytes-avail 可能阻塞),一个输出端口就 ready for synchronization 。 一个既可以作为输入端口又可以作为输出端口的值,可以作为一个 synchronizable event 的输入端口。 The synchronization result of a port is the port itself.
