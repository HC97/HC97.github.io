14.8 线程组

thread group 是一个线程和其他线程组的集合,它们对 CPU 有同等的要求。 通过嵌套线程组和在某些组内创建某些线程,程序员可以控制分配给一组线程的 CPU 数量。 每个线程都属于一个线程组,它由线程创建时的 current-thread-group 参数决定。 线程组和托管人(见 Custodians)是独立的。

根线程组接收操作系统给予 Racket 的所有 CPU。 一个特定的线程组中的每个线程或嵌套组都会收到同等的 CPU 分配(该组的一部分访问量), 尽管一个线程可以通过睡眠或与其他进程同步而放弃部分分配。

procedure

(make-thread-group [group]) → thread-group?

  group : thread-group? = (current-thread-group)
创建一个属于 group 的新线程组。

procedure

(thread-group? v) → boolean?

  v : any/c
如果 v 是一个线程组的值,返回 #t ,否则返回 #f 。

parameter

(current-thread-group) → thread-group?

(current-thread-group group) → void?
  group : thread-group?
一个 parameter ,决定了新创建的线程组。
