5.1 定义结构类型: struct

程序员定义的数据类型 in Racket 指南 introduces struct.

syntax

(struct id maybe-super (field ...)
        struct-option ...)
 
maybe-super = 
  | super-id
     
field = field-id
  | [field-id field-option ...]
     
struct-option = #:mutable
  | #:super super-expr
  | #:inspector inspector-expr
  | #:auto-value auto-expr
  | #:guard guard-expr
  | #:property prop-expr val-expr
  | #:transparent
  | #:prefab
  | #:sealed
  | #:authentic
  | #:name name-id
  | #:extra-name name-id
  | #:constructor-name constructor-id
  | #:extra-constructor-name constructor-id
  | #:reflection-name symbol-expr
  | #:methods gen:name-id method-defs
  | #:omit-define-syntaxes
  | #:omit-define-values
     
field-option = #:mutable
  | #:auto
     
method-defs = (definition ...)
创建一个新的 structure type (或者如果指定了 #:prefab , 则使用预先存在的结构类型),并绑定与 structure type 相关的转换器和变量。

一个有 n 个 field 的 struct 形式最多可以定义 4+2n个名字:

  • struct:id, 代表 structure type 的 structure type descriptor 值。

  • constructor-id (默认为 id), 一个 constructor 过程, 接收 m 个参数并返回 structure type 的新实例, 其中 m 是不包含 #:auto 选项的 field 数量。

  • name-id (默认为 id), 一个封装了结构类型声明信息的 transformer 绑定。 这个绑定被用来定义子类型,它也适用于 shared 和 match 形式。 有关 name-id 绑定的详细信息,请参见 结构类型转化器绑定 。

    constructor-id 和 name-id 可以是相同的,在这种情况下, name-id 同时执行两个角色。 在这种情况下,将 name-id 扩展为表达式会产生一个无法访问的标识符,该标识符被绑定到构造函数上; 扩展后的标识符有一个 'constructor-for 属性, 其值是一个与 name-id 属于 free-identifier=? 的标识符, 以及一个可通过 syntax-procedure-alias-property 访问的语法属性, 其标识符是与 name-id 构成 free-identifier=? 。

  • id?, 一个 predicate 过程, 对 structure type 的实例(由 constructor-id 或子类型的 constructor 构造)返回 #t ,对于任何其他值返回 #f 。

  • id-field-id, 对于每个 field; 一个 accessor 过程,接收 structure type 的实例并提取相应字段的值。

  • set-id-field-id!, 对于包含 #:mutable 选项的每个 field , 或者当 #:mutable 选项被指定为 struct-option 时; 一个 mutator 过程,接收 structure type 的一个实例和一个新的字段值。 该结构被破坏性地更新为新的值,并返回 #<void> 。

如果提供了 super-id , 它必须有一个与 name-id 绑定的同类型的转化器(参见 结构类型转化器绑定), 并且它为结构类型指定了一个超级类型。 另外, #:super 选项可以用来指定一个必须产生 structure type descriptor 的表达式。 关于结构子类型和超级类型的更多信息,请参见 结构 。 如果同时提供了 super-id 和 #:super ,将会报告一个语法错误。

Examples:
> (struct document (author title content))
> (struct book document (publisher))
> (struct paper (journal) #:super struct:document)

如果为单个字段指定了 #:mutable 选项,那么该字段可以在结构类型的实例中被突变, 并且绑定了一个 mutator 过程。 将 #:mutable 作为一个 struct-option 与为所有 field 提供该选项是一样的。 如果 #:mutable 同时被指定为 field-option 和 struct-option, 将报告一个语法错误。

Examples:
> (struct cell ([content #:mutable]) #:transparent)
> (define a-cell (cell 0))
> (set-cell-content! a-cell 1)

#:inspector, #:auto-value 和 #:guard 选项分别指定了一个检查器、自动字段的值和保护程序。 关于结构类型的这些属性的更多信息,请参见 make-struct-type 。 #:property 选项,可以多次提供,为结构类型附加一个属性值; 关于属性的更多信息,参见 结构类型属性 。 #:transparent 选项是 #:inspector #f 的简写。

Examples:
> (struct point (x y) #:inspector #f)
> (point 3 5)

(point 3 5)

> (struct celsius (temp)
    #:guard (λ (temp name)
              (unless (and (real? temp) (>= temp -273.15))
                (error "not a valid temperature"))
              temp))
> (celsius -275)

not a valid temperature

使用 prop:procedure 属性来实现一个 applicable structure, 使用 prop:evt 来创建一个结构类型,其实例是 synchronizable events, 依此类推。 按照惯例,属性名以 prop: 开头。

#:prefab 选项可以获得一个 prefab (预先定义的、全局共享的)结构类型, 而不是创建一个新的结构类型。 这样的结构类型本质上是透明和非密封的,而且它不能有防护或属性,所以将 #:prefab 与 #:transparent, #:inspector, #:guard, #:property, #:sealed, #:authentic 或 #:methods 一起使用是一个语法错误。 如果指定了一个超类型,它也必须是一个 prefab 结构类型。

Examples:
> (struct prefab-point (x y) #:prefab)
> (prefab-point 1 2)

'#s(prefab-point 1 2)

> (prefab-point? #s(prefab-point 1 2))

#t

#:sealed 选项是 #:property prop:sealed #t 的简写, 它可以防止结构类型被用作另一个结构类型的超类型。 更多信息请参见 prop:sealed 。

#:authentic 选项是 #:property prop:authentic #t 的缩写, 它可以防止结构类型的实例被冒充(参见 impersonate-struct), 陪衬(参见 chaperone-struct), 或获得非 flat contract (参见 struct/c)。 更多信息请参见 prop:authentic 。 如果指定了一个超级类型,它也必须有 prop:authentic 属性。

如果 name-id 是通过 #:extra-name 提供的,并且它不是 id, 那么 name-id 和 id 都被绑定到结构类型的信息上。 一个 struct 形式中只能提供 #:extra-name 和 #:name 中的一个, 而且 #:extra-name 不能与 #:omit-define-syntaxes 相结合。

Examples:
> (struct ghost (color name) #:prefab #:extra-name GHOST)
> (match (ghost 'red 'blinky)
    [(GHOST c n) c])

'red

如果提供了 constructor-id ,那么 name-id 的 transformer 绑定就会将 constructor-id 记录为构造函数绑定; 因此,例如 struct-out 将 constructor-id 作为一个出口。 如果 constructor-id 是通过 #:extra-constructor-name 提供的, 并且它不是 id, 在构造函数上应用 object-name 会产生 id 的符号形式, 而不是 constructor-id 。 如果 constructor-id 是通过 #:constructor-name 提供的, 并且它与 name-id 不一样,那么 name-id 就不能作为一个构造函数, 在构造函数上应用 object-name 会产生 constructor-id 的符号形式。 在一个 struct 形式中只能提供 #:extra-constructor-name 和 #:constructor-name 中的一个。

Examples:
> (struct color (r g b) #:constructor-name -color)
> (struct rectangle (w h color) #:extra-constructor-name rect)
> (rectangle 13 50 (-color 192 157 235))

#<rectangle>

> (rect 50 37 (-color 35 183 252))

#<rectangle>

如果提供了 #:reflection-name symbol-expr , 那么 symbol-expr 必须产生一个符号,用于在反射操作(如 struct-type-info )中识别结构类型。 它对应于make-struct-type 的第一个参数。 结构打印使用反射名称,由 struct 绑定的各种程序也是如此。

Examples:
> (struct circle (radius) #:reflection-name '<circle>)
> (circle 15)

#<<circle>>

> (circle-radius "bad")

<circle>-radius: contract violation

  expected: <circle>?

  given: "bad"

如果 #:methods gen:name-id method-defs 被提供(可能是多次), 那么 gen:name-id 必须是由 define-generics 产生的泛型接口的静态信息的转化器绑定。 method-defs 定义了 gen:name-id 接口的方法。 一个 define/generic 形式或辅助定义和表达式也可以出现在 method-defs 中。

Examples:
> (struct constant-stream (val)
    #:methods gen:stream
    [(define (stream-empty? stream) #f)
     (define (stream-first stream)
       (constant-stream-val stream))
     (define (stream-rest stream) stream)])
> (stream-ref (constant-stream 'forever) 0)

'forever

> (stream-ref (constant-stream 'forever) 50)

'forever

如果提供了 #:omit-define-syntaxes 选项, 那么 name-id (以及 id, 如果指定了 #:extra-name) 就不会被绑定为一个转化器。 如果提供了 #:omit-define-values 选项, 那么通常的变量都不会被绑定,但 id 被绑定。 如果两个选项都提供了,那么 struct 形式就等同于 (begin) 。

Examples:
> (struct square (side) #:omit-define-syntaxes)
> (match (square 5)
    ; fails to match because syntax is omitted
    [(struct square x) x])

eval:28:0: match: square does not refer to a structure

definition

  at: square

  in: (struct square x)

> (struct ellipse (width height) #:omit-define-values)
> ellipse-width

ellipse-width: undefined;

 cannot reference an identifier before its definition

  in module: top-level

提供给 #:auto-value 的表达式被评估一次,并在结构类型的每个实例之间共享。 特别是,对一个可变的 #:auto-value 的更新会影响所有当前和未来的实例。

如果 #:auto 是作为 field-option 提供的, 那么结构类型的 constructor 过程就不接受与字段对应的参数。 相反,结构类型的自动值被用于该字段,由 #:auto-value 选项指定, 或者当 #:auto-value 没有被提供时,默认为 #f 。 该字段是可变的(例如,通过反射操作),但只有在指定了 #:mutable 的情况下才会绑定一个 mutator 过程。

如果一个 field 包括 #:auto 选项,那么它之后的所有字段也必须包括 #:auto, 否则会报告语法错误。 如果任何 field-option 或 struct-option 关键字被重复, 除了 #:property 之外,将报告一个语法错误。

Examples:
(struct posn (x y [z #:auto #:mutable])
  #:auto-value 0
  #:transparent)

 

> (posn 1 2)

(posn 1 2 0)

> (posn? (posn 1 2))

#t

> (posn-y (posn 1 2))

2

> (posn-z (posn 1 2))

0

 

(struct color-posn posn (hue) #:mutable)
(define cp (color-posn 1 2 "blue"))

 

> (color-posn-hue cp)

"blue"

> cp

(color-posn 1 2 0 ...)

> (set-posn-z! cp 3)

关于序列化,请参见 define-serializable-struct 。

Changed in version 6.9.0.4 of package base: 增加 #:authentic 。
Changed in version 8.0.0.7: 增加 #:sealed 。

syntax

(struct-field-index field-id)

这种形式只能作为表达式出现在 struct 形式中; 通常情况下,它与 #:property 一起使用,特别是对于像 prop:procedure 这样的属性。 struct-field-index 表达式的结果是一个精确的非负整数, 对应于 field-id 命名的字段在结构声明中的位置。

Examples:
(struct mood-procedure (base rating)
  #:property prop:procedure (struct-field-index base))
(define happy+ (mood-procedure add1 10))

 

> (happy+ 2)

3

> (mood-procedure-rating happy+)

10

syntax

(define-struct id-maybe-super (field ...)
               struct-option ...)
 
id-maybe-super = id
  | (id super-id)
与 struct 类似,只是提供 super-id 的语法不同, 如果既没有提供 #:extra-constructor-name 也没有提供 #:constructor-name , 那么在 id 上有 make- 前缀的 constructor-id 将通过 #:extra-constructor-name 隐式提供。

提供这种形式是为了向后兼容; struct 是首选。

Examples:
(define-struct posn (x y [z #:auto])
   #:auto-value 0
   #:transparent)

 

> (make-posn 1 2)

(posn 1 2 0)

> (posn? (make-posn 1 2))

#t

> (posn-y (make-posn 1 2))

2

syntax

(struct/derived (id . rest-form)
id (field ...) struct-option ...)
(struct/derived (id . rest-form)
id super-id (field ...) struct-option ...)
与 struct 相同,但有一个额外的 (id . rest-form) 子表单, 被视为语法错误报告的整体表单,其他情况下被忽略。 对错误报告的子形式的唯一限制是它以 id 开头。 struct/derived 形式是为扩展到 struct 的宏所使用的。

Examples:
(define-syntax (fruit-struct stx)
  (syntax-case stx ()
   [(ds name . rest)
    (with-syntax ([orig stx])
      #'(struct/derived orig name (seeds color) . rest))]))

 

> (fruit-struct apple)
> (apple-seeds (apple 12 "red"))

12

> (fruit-struct apple #:mutable)
> (set-apple-seeds! (apple 12 "red") 8)
; this next line will cause an error due to a bad keyword
> (fruit-struct apple #:bad-option)

eval:54:0: fruit-struct: unrecognized struct-specification

keyword

  at: #:bad-option

  in: (fruit-struct apple #:bad-option)

Added in version 7.5.0.16 of package base.

syntax

(define-struct/derived (id . rest-form)
  id-maybe-super (field ...) struct-option ...)
与 struct/derived 类似,除了提供 super-id 的语法不同, 如果既没有提供 #:extra-constructor-name ,也没有提供 #:constructor-name , 那么在 id 上有 make- 前缀的 constructor-id 将通过 #:extra-constructor-name 隐式提供。 define-struct/derived 形式是为扩展到 define-struct 的宏所使用的。

Examples:
(define-syntax (define-xy-struct stx)
  (syntax-case stx ()
   [(ds name . rest)
    (with-syntax ([orig stx])
      #'(define-struct/derived orig name (x y) . rest))]))

 

> (define-xy-struct posn)
> (posn-x (make-posn 1 2))

1

> (define-xy-struct posn #:mutable)
> (set-posn-x! (make-posn 1 2) 0)
; this next line will cause an error due to a bad keyword
> (define-xy-struct posn #:bad-option)

eval:60:0: define-xy-struct: unrecognized

struct-specification keyword

  at: #:bad-option

  in: (define-xy-struct posn #:bad-option)

Changed in version 7.5.0.16 of package base: 将主要描述移至 struct/derived ,并以差异代替。
