15.1.2 More Path Utilities

 (require racket/path) package: base
The bindings documented in this section are provided by the racket/path and racket libraries, but not racket/base.

procedure

(file-name-from-path path) → (or/c path-for-some-system? #f)

  path : (or/c path-string? path-for-some-system?)
返回 path 的最后一个元素。 如 path 在语法上是一个目录路径(见 split-path),那么结果是 #f 。

procedure

(path-get-extension path) → (or/c bytes? #f)

  path : (or/c path-string? path-for-some-system?)
返回一个字节的字符串,即 path 中文件名的扩展部分,包括 . 分隔符。 如果路径没有扩展名,则返回 #f 。

关于文件名扩展名的定义,见 path-replace-extension 。

Examples:
> (path-get-extension "x/y.rkt")

#".rkt"

> (path-get-extension "x/y")

#f

> (path-get-extension "x/y.tar.gz")

#".gz"

> (path-get-extension "x/.racketrc")

#f

Added in version 6.5.0.3 of package base.

procedure

(path-has-extension? path ext) → boolean?

  path : (or/c path-string? path-for-some-system?)
  ext : (or/c bytes? string?)
判断 path 的最后一个元素是否以 ext 结尾,但与 ext 不完全相同。

如果 ext 是一个具有扩展名形状的 byte string (即以 . 开始,不包括另一个 .), 这个检查等同于检查 (path-get-extension path) 是否产生 ext 。

Examples:
> (path-has-extension? "x/y.rkt" #".rkt")

#t

> (path-has-extension? "x/y.ss" #".rkt")

#f

> (path-has-extension? "x/y" #".rkt")

#f

> (path-has-extension? "x/.racketrc" #".racketrc")

#f

> (path-has-extension? "x/compiled/y_rkt.zo" #"_rkt.zo")

#t

Added in version 6.5.0.3 of package base.

procedure

(filename-extension path) → (or/c bytes? #f)

  path : (or/c path-string? path-for-some-system?)

NOTE: This function is deprecated; use path-get-extension, instead.

返回一个字节字符串,它是 path 中文件名的扩展部分,没有 . 分隔符。 如果 path 在语法上是一个目录(见 split-path) 或者 path 没有扩展名,则返回 #f 。

procedure

(find-relative-path base 
  path 
  [#:more-than-root? more-than-root? 
  #:more-than-same? more-than-same? 
  #:normalize-case? normalize-case?]) 
 → (or/c path-for-some-system? path-string?)
  base : (or/c path-string? path-for-some-system?)
  path : (or/c path-string?  path-for-some-system?)
  more-than-root? : any/c = #f
  more-than-same? : any/c = #t
  normalize-case? : any/c = #t
找到一个相对于 base 的相对路径名,该路径名与 path 相同的文件或目录。 base 和 path 都必须是 simple-form-path 的意义上的简化。 如果 path 与 base 没有共同的子路径,则返回 path 。

如果 more-than-root? 为真,如果 base 和 path 只共享一个 Unix 根, 并且如果 base 和 path 都不只是一个根路径,那么 path 被返回。

如果 path 与 base 相同,那么只有当 more-than-same? 为 #f 时, 才会返回 (build-path 'same) 。 否则,默认情况下,当 path 与 base 相同时,会返回 path 。

如果 normalize-case? 为 true (默认),那么要比较的路径元素对首先通过 normal-case-path 进行转换,这意味着路径元素在 Windows 上被不分大小写地比较。 如果 normalize-case? 是 #f, 那么路径元素和路径根只有在大小写相同时才匹配。

结果通常是 path 意义上的 path? 。 只有当 path 被提供为字符串并且也作为结果返回时,结果才是一个字符串。

Changed in version 6.8.0.3 of package base: 默认情况下使路径元素的大小写正常化,并增加了 #:normalize-case? 参数。
Changed in version 6.90.0.21: 增加了 #:more-than-same? 参数。

procedure

(normalize-path path [wrt]) → path?

  path : path-string?
  wrt : (and/c path-string? complete-path?)
   = (current-directory)

在大多数情况下, simple-form-path 是规范化路径的首选机制, 因为它适用于包括不存在的目录组件的路径,而且可以避免不必要地扩展软链接。

通过使路径完整,扩展完整的路径,并解决所有的软链接(这需要咨询文件系统), 返回一个完整版本的 path 。 如果 path 是相对的,那么 wrt 被用作基本路径。

字母大小写 不 被 normalize-path 所规范。 由于这个原因和其他原因,比如路径在语法上是否是一个目录, normalize-path 的结果不适合用于确定两个路径是否指向同一个文件或目录的比较 (也就是说,比较可能产生错误的否定结果)。

如果输入的路径包含一个不存在的目录的嵌入路径,或者检测到一个无限循环的软链接, normalize-path 会发出错误信号。

Example:
> (equal? (current-directory) (normalize-path "."))

#t

procedure

(path-element? path) → boolean?

  path : any/c
如果 path 是一个 path element,则返回 #t : 某个平台的路径值(见 path-for-some-system?) , 如果 split-path 应用于 path , 其第一个结果是 'relative ,第二个结果是 path。 否则,结果是 #f 。

procedure

(path-only path) → (or/c #f path-for-some-system?)

  path : (or/c path-string? path-for-some-system?)
如果 path 在语法上不是一个目录,则返回没有最后路径元素的 path ; 如果 path 只有一个非目录路径元素,则返回 #f 。 如 path 在语法上是一个目录, 那么 path 的返回结果是不变的(但如果它是一个字符串,则作为一个路径)。

Examples:
> (path-only (build-path "a" "b"))

#<path:a/>

> (path-only (build-path "a"))

#f

> (path-only (path->directory-path (build-path "a")))

#<path:a/>

> (path-only (build-path 'up 'up))

#<path:../..>

procedure

(simple-form-path path) → path?

  path : path-string?
返回 (simplify-path (path->complete-path path)), 确保结果是一个完整的路径,不包含向上或同目录的指示符。

procedure

(some-system-path->string path) → string?

  path : path-for-some-system?
使用 UTF-8 编码的路径字节,将 path 转换为一个字符串。

在处理不同系统的路径(其路径名的编码可能与当前 locale 的编码无关)以及用字符串开始和结束时,请使用该函数。

procedure

(string->some-system-path str kind) → path-for-some-system?

  str : string?
  kind : (or/c 'unix 'windows)
使用 UTF-8 编码的路径字节,将 str 转换为 kind 路径。

在处理不同系统的路径(其路径名的编码可能与当前 locale 的编码无关)以及以字符串开始和结束时,请使用此函数。

procedure

(shrink-path-wrt pth other-pths) → (or/c #f path?)

  pth : path?
  other-pths : (listof path?)
返回一个与 other-pths 的后缀没有任何共同之处的 pth 的后缀, 如果不可能,则返回 pth (例如,当 other-pths 为空或者只包含与 pth 相同元素的路径)。

Examples:
> (shrink-path-wrt (build-path "racket" "list.rkt")
                   (list (build-path "racket" "list.rkt")
                         (build-path "racket" "base.rkt")))

#<path:list.rkt>

> (shrink-path-wrt (build-path "racket" "list.rkt")
                   (list (build-path "racket" "list.rkt")
                         (build-path "racket" "private" "list.rkt")
                         (build-path "racket" "base.rkt")))

#<path:racket/list.rkt>
