15.5 记录

一个 logger 接受包含信息的事件,为相关方记录。 一个 log receiver 代表一个感兴趣的一方,它异步地接收记录的事件。 每个事件都有一个主题和详细程度, log receiver 订阅特定主题或所有主题的某一详细程度(或更低)的日志事件。 这些级别,按细节递增的顺序,是 'none, 'fatal, 'error, 'warning, 'info 和 'debug 。

为了帮助组织记录的事件,一个 logger 可以有一个默认的主题和/或一个父记录器。 每一个报告给记录仪的事件都会被传播到它的父记录仪(如果有的话),如果消息还没有主题的话, 事件消息会以记录仪的主题(如果有的话)为前缀。 此外,从记录仪传播到其父级的事件可以通过级别和主题进行过滤。

启动时,Racket 会创建一个初始记录器,用来记录来自核心运行时系统的事件。 例如,每次垃圾回收都会报告一个 'debug 事件(见 垃圾回收) 。 对于这个初始记录器,还创建了两个日志接收器:一个将事件写到进程的原始错误输出端口,一个将事件写到系统日志。 每种情况下写入事件的级别都是针对系统的,默认值可以通过命令行标志(见 命令行)或环境变量来改变:

  • 如果 PLTSTDERR 环境变量被定义,并且没有被命令行标志覆盖, 它决定了将事件传播到原始错误端口的 log receiver 的级别。

    环境变量的值可以是一个 ‹level›: none, fatal, error, warning, info 或 debug (从低细节到高细节); 所有在相应的细节级别或更低的事件都被打印出来。 在最初的 ‹level› 之后, 该值可以包含以空格分隔的 ‹level›@‹topic› 形式的规范, 它只打印在给定的 ‹level› 或更高的级别中主题与 ‹topic› 匹配的事件 (其中 ‹topic› 包含除空格或 @ 之外的任何字符)。 前面和后面的空白将被忽略。 例如,值 "error debug@GC" 打印 'error 级别及以上的所有事件, 但打印 'debug 级别及以上(包括所有级别)的 'GC 主题的事件。

    默认是 "error" 。

  • 如果定义了 PLTSTDOUT 环境变量,并且没有被命令行标志所覆盖, 它决定了将事件传播到原始输出端口的 log receiver 的级别。 可能的值与 PLTSTDERR 相同。

    默认值是 "none" 。

  • 如果 PLTSYSLOG 环境变量被定义,并且没有被命令行标志覆盖, 它决定了将事件传播到系统日志的 log receiver 的级别。 可能的值与 PLTSTDERR 相同。

    Unix 的默认值是 "none" ,Windows 和 Mac OS 的默认值是 "error" 。

current-logger parameter 决定了 current logger , 它被 log-warning 等形式所使用。 在启动时,该参数的初始值是初始记录器。 运行时系统有时会使用当前的记录器来报告事件。 例如,字节码编译器有时会报告 'warning 事件,当它检测到一个表达式如果被评估会产生一个运行时错误。

Changed in version 6.6.0.2 of package base: 在6.6.0.2版本之前, PLTSTDERR 和 PLTSYSLOG 的解析是非常严格的。 禁止使用前导和尾部的空白,并且拒绝使用任何在两个规范之间不完全是一个空格的字符。
Changed in version 6.90.0.17: 增加了 PLTSTDOUT 。

15.5.1 创建记录器

procedure

(logger? v) → boolean?

  v : any/c
如果 v 是一个 logger,返回 #t ,否则返回 #f 。

procedure

(make-logger [topic    
  parent    
  propagate-level    
  propagate-topic ...]    
  ...) → logger?
  topic : (or/c symbol? #f) = #f
  parent : (or/c logger? #f) = #f
  propagate-level : log-level/c = 'debug
  propagate-topic : (or/c #f symbol?) = #f
创建一个新的 logger ,有一个可选的主题和父级。

可选的 propagate-level 和 propagate-topic 参数限制了从新的记录仪传播到 parent 的事件(当 parent 不是 #f 时), 其方式与 make-log-receiver 中描述日志接收器的事件一样。 默认情况下,所有事件都会被传播到 parent 。

Changed in version 6.1.1.3 of package base: 删除了一个指定通知回调的可选参数,并为要传播的事件添加了 propagate-level 和 propagate-topic 的限制。

procedure

(logger-name logger) → (or/c symbol? #f)

  logger : logger?
报告 logger 的默认主题,如果有的话。

parameter

(current-logger) → logger?

(current-logger logger) → void?
  logger : logger?
一个 parameter ,决定了 current logger 。

syntax

(define-logger id maybe-parent)

 
maybe-parent = 
  | #:parent parent-expr
 
  parent-expr : (or/c logger? #f)
定义 log-id-fatal, log-id-error, log-id-warning, log-id-info 和 log-id-debug 等形式, 如 log-fatal, log-error,log-warning, log-info 和 log-debug 。 define-logger 表单还定义了 id-logger, 这是一个默认主题为 'id 的记录器,是 parent-expr (如果 parent-expr 不产生 #f)的结果的子代, 或者是 (current-logger) (如果 parent-expr 没有提供)的子代; log-id-fatal, etc. 等表单使用这个新记录器。 新的日志记录器是在评估 define-logger 时创建的。

Changed in version 7.1.0.9 of package base: 增加了 #:parent 选项。

15.5.2 记录事件

procedure

(log-message logger    
  level    
  [topic]    
  message    
  [data    
  prefix-message?]) → void?
  logger : logger?
  level : log-level/c
  topic : (or/c symbol? #f) = (logger-name logger)
  message : string?
  data : any/c = #f
  prefix-message? : any/c = #t
将事件报告给 logger ,而记录仪又将信息分发给附属于 logger 或其祖先的、 对 level 或更高的事件感兴趣的任何 log receivers 。

Log receivers 可以根据 topic 来过滤事件。 此外,如果 topic 和 prefix-message? 不是 #f, 那么 message 在发送给接收者之前,会在 topic 后面加上 ": " 。

Changed in version 6.0.1.10 of package base: 增加了 prefix-message? 参数。
Changed in version 7.2.0.7: data 参数是可选的。

procedure

(log-level? logger level [topic]) → boolean?

  logger : logger?
  level : log-level/c
  topic : (or/c symbol? #f) = #f
报告任何附属于 logger 或其祖先之一的 log receiver 是否对 topic 的 level 事件(或可能更低)感兴趣。 如果 topic 是 #f, 结果表明是否有 log receiver 对任何 topic 的 level 事件感兴趣。

使用这个函数可以避免在没有接收者对信息感兴趣的情况下为 log-message 生成事件; 然而,这个快捷方式被内置于 log-fatal, log-error, log-warning, log-info, log-debug 以及被 define-logger 绑定的表单中,所以它不应该被用于这些表单。

如果垃圾回收确定一个日志接收器不再被访问(因此它收到的任何事件信息将永远不会被访问),这个函数的结果可能会改变。

Changed in version 6.1.1.3 of package base: 增加了 topic 参数。

procedure

(log-max-level logger [topic]) → (or/c log-level/c #f)

  logger : logger?
  topic : (or/c symbol? #f) = #f
与 log-level? 类似, 但报告了 log-level? 和 logger 和 topic 上返回 #t 的最大细节级别的日志。 如果 log-level? 与 logger 和 topic 当前对所有级别都返回 #f , 则结果是 #f 。

Changed in version 6.1.1.3 of package base: 增加了 topic 参数。

procedure

(log-all-levels logger) → 
(list/c (or/c #f log-level/c)
        (or/c #f symbol?)
        ... ...)
  logger : logger?
总结了 log-max-level 对所有可能的 interned 符号的可能结果。 结果列表包含一个符号和 #f 的序列,其中第一个、第三个等列表元素对应一个级别, 第二个、第四个等列表元素表示一个相应的主题。 级别是 log-max-level 对主题产生的结果, 其中 #f 主题的级别(总是出现在结果列表中)表示任何没有出现在列表中的 interned 符号的主题的结果。

该结果适合作为 make-log-receiver 的参数序列(在一个 logger 参数之后), 为当前在 logger 中有接收器的事件创建一个新的接收器。

Added in version 6.1.1.4 of package base.

procedure

(log-level-evt logger) → evt?

  logger : logger?
创建一个 synchronizable event , 当 log-level?,log-max-level 或 log-all-levels 的结果可能与调用 log-level-evt 之前不同时,该事件就 ready for synchronization 。 该事件的 synchronization result 就是事件本身。

事件报告的条件是一个保守的近似值: 即使 log-level?, log-max-level 和 log-all-levels 的结果没有变化, 事件也可以成为 ready for synchronization 。 尽管如此,期望由 log-level-evt 产生的事件成为就绪的情况并不频繁,因为它们是由创建一个日志接收器而触发的。

Added in version 6.1.1.4 of package base.

syntax

(log-fatal string-expr)

(log-fatal format-string-expr v ...)

syntax

(log-error string-expr)

(log-error format-string-expr v ...)

syntax

(log-warning string-expr)

(log-warning format-string-expr v ...)

syntax

(log-info string-expr)

(log-info format-string-expr v ...)

syntax

(log-debug string-expr)

(log-debug format-string-expr v ...)
用 current logger 记录一个事件,只有当记录器有对该事件感兴趣的接收器时, 才会评估 string-expr 或 (format format-string-expr v ...) 。 此外,当前的 continuation 的 continuation marks 与消息字符串一起被发送到记录器。

这些形式对于使用当前的日志记录器是很方便的, 但是图书馆一般应该为特定的主题使用一个日志记录器–通常是通过 define-logger 生成的类似的方便形式。

对于每个 log-level,

(log-level string-expr)

相当于

(let ([l (current-logger)])
  (when (log-level? l 'level)
    (log-message l 'level string-expr
                 (current-continuation-marks))))

而

(log-level format-string-expr v ...)

相当于

(log-level (format format-string-expr v ...))

15.5.3 接收记录的事件

procedure

(log-receiver? v) → boolean?

  v : any/c
如果 v 是一个 log receiver,返回 #t ,否则返回 #f 。

procedure

(make-log-receiver logger level [topic ...] ...) → log-receiver?

  logger : logger?
  level : log-level/c
  topic : (or/c #f symbol?) = #f
创建一个 log receiver 来接收报告给 logger 及其后代的细节 level 及以下的事件, 只要 topic 是 #f 或者事件的topic与 topic 匹配。

一个 log receiver 是一个 synchronizable event 。 当收到一个日志事件时,它就成为 ready for synchronization ,所以使用 sync 来接收一个日志事件。 log receiver 的 synchronization result 是一个不可变的向量,包含四个值: 作为符号的事件的级别,事件消息的不可变的字符串, 事件被记录时作为 log-message 的最后一个参数提供的任意值, 以及事件主题的符号或 #f 。

可以提供多对 level 和 topic , 以表示不同 topic 的不同具体 level (其中 topic 默认为 #f ,仅用于最后给出的 level)。 一个 #f topic 的 level 只适用于其主题与任何其他提供的 topic 不匹配的事件。 如果同一个 topic 被多次提供,那么参数列表中最后一个实例所提供的 level 将被优先考虑。

15.5.4 额外的日志功能

 (require racket/logging) package: base
The bindings documented in this section are provided by the racket/logging library, not racket/base or racket.

procedure

(log-level/c v) → boolean?

  v : any/c
如果 v 是一个有效的日志级别 ('none, 'fatal, 'error, 'warning, 'info 或 'debug),则返回 #t ,否则返回 #f 。

Added in version 6.3 of package base.

procedure

(with-intercepted-logging interceptor    
  proc    
  [#:logger logger]    
  level    
  [topic ...]    
  ...) → any
  interceptor : 
(-> (vector/c
      log-level/c
      string?
      any/c
      (or/c symbol? #f))
     any)
  proc : (-> any)
  logger : logger? = #f
  level : log-level/c
  topic : (or/c #f symbol?) = #f
运行 proc ,对 proc 执行过程中向 current-logger 发出的任何日志事件按照指定的级别和主题调用 interceptor 。 如果指定了 #:logger ,则拦截发送到该记录器的事件,否则使用当前记录器的一个新的子记录器。 返回任何 proc 返回的结果。

Example:
> (let ([warning-counter 0])
    (with-intercepted-logging
      (lambda (l)
        (when (eq? (vector-ref l 0)
                   'warning)
          (set! warning-counter (add1 warning-counter))))
      (lambda ()
        (log-warning "Warning!")
        (log-warning "Warning again!")
        (+ 2 2))
      'warning)
    warning-counter)

procedure

(with-logging-to-port port    
  proc    
  [#:logger logger]    
  level    
  [topic ...]    
  ...) → any
  port : output-port?
  proc : (-> any)
  logger : logger? = #f
  level : log-level/c
  topic : (or/c #f symbol?) = #f
运行 proc, 在指定的级别和主题下, 将 proc 的执行所发出的任何日志输出到 current-logger 。 如果指定了 #:logger ,则拦截发送到该记录仪的事件,否则使用当前记录仪的一个新的子记录仪。 返回任何 proc 返回的结果。

Example:
> (let ([my-log (open-output-string)])
    (with-logging-to-port my-log
      (lambda ()
        (log-warning "Warning World!")
        (+ 2 2))
      'warning)
    (get-output-string my-log))
