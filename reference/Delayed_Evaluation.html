10.3 Delayed Evaluation

 (require racket/promise) package: base
The bindings documented in this section are provided by the racket/promise and racket libraries, but not racket/base.

一个 promise 封装了一个表达式,通过 force 来进行评估。 在一个承诺被 force 后,以后每一次 force 该承诺都会产生相同的结果。

procedure

(promise? v) → boolean?

  v : any/c
如果 v 是一个 promise,返回 #t ,否则返回 #f 。

syntax

(delay body ...+)

创建一个承诺,当 force 时,评估 body 以产生其值。 这个结果会被缓存起来,所以继续使用 force 会立即产生缓存的值。 这包括多个值和异常情况。

syntax

(lazy body ...+)

像 delay 一样,如果最后一个 body 在 force 时产生一个承诺, 那么这个承诺也被 force ,以获得一个值。 换句话说,这个表单创建了一个可组合的承诺,它的主体的计算被 "附加" 到下面的承诺的计算上, 并且一个 force 迭代了整个链条,尾随调用每一步。

注意,这种形式的最后一个 body 必须产生一个单一的值, 但这个值本身可以是一个返回多个值的 delay 承诺。

lazy 形式对于实现懒惰库和语言很有用,在这种情况下,尾部调用可以被包裹在一个承诺中。

procedure

(force v) → any

  v : any/c
如果 v 是一个 promise,那么该 promise 被强制获得一个值。 如果该承诺之前没有被强制过,那么结果将被记录在该承诺中,这样未来对该承诺的 force 将产生相同的值(或值)。 如果强制承诺引发了一个异常,那么这个异常也同样被记录下来,这样强制承诺每次都会引发同样的异常。

如果 v 在最初调用 force 返回之前再次被 force ,那么会引发 exn:fail exception is raised 。

如果 v 不是一个承诺,那么它将作为结果返回。

procedure

(promise-forced? promise) → boolean?

  promise : promise?
如果 promise 已经被强制执行,则返回 #t 。

procedure

(promise-running? promise) → boolean?

  promise : promise?
如果 promise 目前正在被强制执行,则返回 #t 。 (注意,一个 promise 既可以是运行中的,也可以是强制的,但不能同时是。)

10.3.1 Additional Promise Kinds

syntax

(delay/name body ...+)

创建一个 "call-by-name" 的承诺,与 delay-promises 类似,只是产生的值不会被缓存。 这种承诺本质上是一个以 force 方式包装的 thunk。

如果一个 delay/name 强迫自己,不会引发任何异常,这个承诺永远不会被认为是 "运行" 或 "强迫" 的, 就像 promise-running? 和 promise-forced? 。

procedure

(promise/name? promise) → boolean?

  promise : any/c
如果 promise 是一个用 delay/name 创建的 promise,返回 #t 。

Added in version 6.3 of package base.

syntax

(delay/strict body ...+)

创建一个 "严格" 的 promise:它被立即评估,结果被包裹在一个 promise 值中。 注意,主体可以评估为多个值,强制产生的承诺将返回这些值。

syntax

(delay/sync body ...+)

产生一个承诺,如果当前运行该承诺的线程以外的线程试图 force 该承诺, 会导致 force 阻塞,直到有一个结果可用。 这种承诺也是一个 synchronizable event ,可与 sync 一起使用; 在承诺上 sync 并不 force 它,而只是等待,直到另一个线程强制得到一个值。 synchronization result 是 #<void> 。

如果一个由 delay/sync 创建的承诺被强迫在一个已经在运行该承诺的线程上, 那么就会引发一个异常,其方式与用 delay 创建的承诺相同。

syntax

(delay/thread body/option ...+)

 
body/option = body
  | #:group thread-group-expr
与 delay/sync 类似,但在一个新创建的线程上立即开始计算。 线程是在 thread-group-expr 指定的 thread group 下创建的, 其默认值为 (make-thread-group) 。 一个 #:group 规范最多可以出现一次。

由 body 引发的异常会像往常一样被捕获,并且只在承诺被 force 时被引发。 与 delay/sync 不同的是,如果运行 body 的线程在没有产生结果或异常的情况下终止, 那么承诺的 force 会引发一个异常(而不是阻塞)。

syntax

(delay/idle body/option ...+)

 
body/option = body
  | #:wait-for wait-evt-expr
  | #:work-while while-evt-expr
  | #:tick tick-secs-expr
  | #:use use-ratio-expr
与 delay/thread 类似,但有以下区别:

  • 在 wait-evt-expr 产生的事件准备好之前,计算不会开始,默认是 (system-idle-evt);

  • 计算线程只有在由 while-evt-expr 决定的进程空闲时才开始工作, 其默认值也是 (system-idle-evt);

  • 该线程只允许定期运行:在每一个 tick-secs-expr (默认为 0.2) 秒中, 该线程被允许按比例运行 use-ratio-expr (默认为 0.12) 的时间; 即,该线程运行 (* tick-secs-expr use-ratio-expr) 秒。

如果承诺在计算完成之前被 forced ,它就会立即运行其余的计算,而不需要等待事件或周期性地限制评估。

一个 #:wait-for, #:work-while, #:tick 或 #:use 规范最多只能出现一次。

syntax

(for/list/concurrent maybe-group (for-clause ...)
  body-or-break ... body)
 
maybe-group = 
  | #:group thread-group-expr
 
  thread-group-expr : thread-group?
像 for/list 一样进行迭代, 但主体(在任何 #:break 或 #:final 子句之后)被包裹在 delay/thread 中。 每个 promise 都在返回结果列表之前被强制执行。

线程是在 thread-group-expr 下创建的,其默认值为 (make-thread-group) 。 可以提供一个可选的 #:group 子句,在这种情况下,线程将在该线程组下创建。

这种形式不支持返回多个值。

Example:
> (time
   (for/list/concurrent ([i (in-range 5)])
     (define duration (/ 1.0 (random 50 100)))
     (sleep duration)
     (printf "thread ~a slept for ~a milliseconds~n" i (truncate (* duration 1000)))
     i))

thread 2 slept for 11.0 milliseconds

thread 0 slept for 11.0 milliseconds

thread 4 slept for 14.0 milliseconds

thread 3 slept for 15.0 milliseconds

thread 1 slept for 16.0 milliseconds

cpu time: 47 real time: 16 gc time: 0

'(0 1 2 3 4)

Added in version 8.6.0.4 of package base.

syntax

(for*/list/concurrent maybe-group (for-clause ...)
  body-or-break ... body)
与 for/list/concurrent 类似,但具有 for*/list 的隐式嵌套功能

Added in version 8.6.0.4 of package base.
