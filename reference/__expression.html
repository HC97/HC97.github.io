3.4 表达式包装器: #%expression

syntax

(#%expression expr)

产生的结果与 expr 相同。 使用 #%expression 可以强制将一个表单解析为一个表达式。

Examples:
> (#%expression (+ 1 2))

3

> (#%expression (define x 10))

eval:8:0: define: not allowed in an expression context

  in: (define x 10)

#%expression 形式在递归定义的背景下很有帮助, 在这种情况下,扩展一个后续定义可以为当前表达式提供编译时信息。 例如,考虑一个 define-sym-case 宏, 它在编译时简单地记录了给定标识符中的一些符号。
(define-syntax (define-sym-case stx)
  (syntax-case stx ()
    [(_ id sym ...)
     (andmap identifier? (syntax->list #'(sym ...)))
     #'(define-syntax id
         '(sym ...))]))
然后是 case 的一个变体,它检查确保表达式中使用的符号与先前定义中给出的符号相匹配:
(define-syntax (sym-case stx)
  (syntax-case stx ()
    [(_ id val-expr [(sym) expr] ...)
     (let ()
       (define expected-ids
         (syntax-local-value
          #'id
          (λ ()
            (raise-syntax-error
             'sym-case
             "expected an identifier bound via define-sym-case"
             stx
             #'id))))
       (define actual-ids (syntax->datum #'(sym ...)))
       (unless (equal? expected-ids actual-ids)
         (raise-syntax-error
          'sym-case
          (format "expected the symbols ~s"
                  expected-ids)
          stx))
       #'(case val-expr [(sym) expr] ...))]))

如果定义像这样跟随使用,那么 define-sym-case 宏就没有机会绑定 id ， sym-case 宏就会发出错误信号:
> (let ()
    (sym-case land-creatures 'bear
              [(bear) 1]
              [(fox) 2])
    (define-sym-case land-creatures bear fox))

eval:11:0: sym-case: expected an identifier bound via

define-sym-case

  at: land-creatures

  in: (sym-case land-creatures (quote bear) ((bear) 1)

((fox) 2))

但是如果 sym-case 被包裹在一个 #%expression 中, 那么扩展器不需要扩展它就知道它是一个表达式, 它就会转到 define-sym-case 表达式。
> (let ()
    (#%expression (sym-case sea-creatures 'whale
                            [(whale) 1]
                            [(squid) 2]))
    (define-sym-case sea-creatures whale squid)
    'more...)

'more...

当然,像 sym-case 这样的宏不应该要求它的客户添加 #%expression; 相反,它应该检查其参数的基本形状,然后扩展到 #%expression , 并包裹在一个调用 syntax-local-value 并完成扩展的辅助宏中。
