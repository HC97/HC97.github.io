7.7 扩展签名的语法

syntax

(define-signature-form sig-form-id expr)

(define-signature-form (sig-form-id id) body ...+)
(define-signature-form (sig-form-id id intro-id) body ...+)
绑定 sig-form-id ,在 define-signature 表单中使用。

在第一种形式中, expr 的结果必须是一个接受一个参数的转换过程。 在第二种形式中, sig-form-id 被绑定到一个转换过程,其参数是 id ,其主体是 body 。 第三种形式与第二种形式类似,但是 intro-id 被绑定到一个过程中, 这个过程与签名形式扩展的 syntax-local-introduce 类似。

转换过程的结果必须是一个语法对象的列表, 在 define-signature 扩展中被替换为 sig-form-id 的使用。 (结果是一个列表,这样转化器可以产生多个声明; define-signature 没有拼接的 begin 形式)。

Changed in version 8.1.0.7 of package base: 增加了对带有转化器 expr 的形式的支持。

syntax

(struct/ctc id ([field contract-expr] ...) struct-option ...)

 
field = id
  | [id #:mutable]
     
struct-option = #:mutable
  | #:omit-constructor
  | #:omit-define-syntaxes
  | #:omit-define-values
与 define-signature 一起使用。 struct/ctc 形式的工作原理与 struct 类似,但构造器、谓词、字段访问器和字段突变器被适当地收缩。
