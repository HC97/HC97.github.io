14.5 Impersonators and Chaperones

impersonator 是一个值的包装器,包装器重定向了该值的一些操作。 冒充者只适用于过程、有访问器或突变器的 structures 、 structure types, hash tables, vectors, box, channels 和 prompt tag 。 冒充者与原值 equal? ,但不是与原值 eq? 。

chaperone 是一种冒名顶替者, 它对一个值的操作的细化只限于副作用(特别是包括引发异常)或陪护提供给操作或由操作产生的值。 例如,如果被访问的向量槽包含一个字符串,向量陪护者可以重定向 vector-ref 以引发一个异常, 或者它可以使 vector-ref 的结果成为被访问的向量槽中的值的陪护变量, 但是它不能重定向 vector-ref 以产生一个与向量槽中的值任意不同的值。

相比之下,一个非 chaperone impersonator 可以完善一个操作,将一个值换成任何其他的值。 冒充者不能应用于不可变的值,也不能细化对 structure type 实例中不可变字段的访问, 因为任意重定向的操作相当于对冒充的值进行突变。

请注意,以下每个操作都可以通过操作参数上的冒充者被重定向到一个任意的过程–假设该操作对冒充者的创建者是可用的:

  • 结构域访问器

  • 结构字段突变器

  • 结构类型属性访问器

  • 程序的应用

  • unbox

  • set-box!

  • vector-ref

  • vector-set!

  • hash-ref

  • hash-set

  • hash-set!

  • hash-remove

  • hash-remove!

  • channel-get

  • channel-put

  • call-with-continuation-prompt

  • abort-current-continuation

衍生操作,例如打印一个值,由于使用了访问器函数,可以通过冒名顶替者重定向。 相反,equal?, equal-hash-code 和 equal-secondary-hash-code 操作可以绕过冒充者(但它们不是必须的)。

除了重定向对一个值的操作外,一个冒名顶替者可以包括 impersonator properties 的属性。 impersonator property 类似于 structure type property,但它适用于冒充者而不是结构类型及其实例。

procedure

(impersonator? v) → boolean?

  v : any/c
如果 v 是一个由 impersonate-procedure 或 impersonate-struct 等程序创建的 impersonator , 返回 #t ,否则返回 #f 。

程序和库通常应该避免使用 impersonator? ,并将 impersonator 与非 impersonator 值同等对待。 在极少数情况下, impersonator? 可能是需要的,以防止一个冒名顶替者将操作重定向到一个任意的过程。

impersonator? 的一个限制是, 它 不 能识别通过实例化一个带有 prop:impersonator-of 属性的结构类型而创建的 impersonator 。 这个限制反映了这些 impersonator 不能将结构访问和变异操作重定向到任意过程。

procedure

(chaperone? v) → boolean?

  v : any/c
如果 v 是一个 chaperone,返回 #t ,否则返回 #f 。

程序和库通常应该避免 chaperone? ,原因与他们应该避免 impersonator? 。 chaperone? 的真值意味着 impersonator? 的真值。

procedure

(impersonator-of? v1 v2) → boolean?

  v1 : any/c
  v2 : any/c
表示 v1 是否可以被认为是与 v2 等价的冒名顶替者。

任何两个相互 eq? 的值也是 impersonator-of? 。 对于不包括冒充者的值,如果 v1 和 v2 是 equal? ,则被视为彼此的冒充者。

如果 v1 或 v2 中至少有一个是冒充者:

  • 如果 v1 冒充 v1* , 那么当且仅当 (impersonator-of? v1* v2) 为 #t 时， (impersonator-of? v1 v2) 为 #t 。

  • 如果 v2 是一个冒充 v2* 的非冒充者,即它所有的冒充程序都是 #f, 那么当且仅当 (impersonator-of? v1 v2*) 为 #t 时, (impersonator-of? v1 v2) 是 #t 。

  • 当 v2 是一个用至少一个非 #f 的互置程序构造的冒名顶替者, 但 v1 不是冒名顶替者时,那么 (impersonator-of? v1 v2) 就是 #f 。

否则,如果 v1 或 v2 都不是一个冒充者, 但其中任何一个包含一个冒充者作为子部分(例如, v1 是一个以冒充者为其元素之一的列表), 那么 (impersonator-of? v1 v2) 就会递归地比较 v1 和 v2 (如同 equal?),如果所有子部分都是 impersonator-of? ,则返回 true。

Examples:
> (impersonator-of? (impersonate-procedure add1 (λ (x) x))
                    add1)

#t

> (impersonator-of? (impersonate-procedure add1 (λ (x) x))
                    sub1)

#f

> (impersonator-of? (impersonate-procedure
                      (impersonate-procedure add1 (λ (x) x)) (λ (x) x))
                    add1)

#t

> (impersonator-of? (impersonate-procedure add1 (λ (x) x))
                    (impersonate-procedure add1 #f))

#t

> (impersonator-of? (impersonate-procedure add1 (λ (x) x))
                    (impersonate-procedure add1 (λ (x) x)))

#f

> (impersonator-of? (list 1 2)
                    (list 1 2))

#t

> (impersonator-of? (list (impersonate-procedure add1 (λ (x) x)) sub1)
                    (list add1 sub1))

#t

procedure

(chaperone-of? v1 v2) → boolean?

  v1 : any/c
  v2 : any/c
表示 v1 是否可以被认为是与 v2 等价的陪衬。

对于不包括陪护者或其他冒充者的值,如果 v1 和 v2 是 equal-always? , 就可以认为是彼此的陪护者,这要求它们是 equal? , 除了 v1 和 v2 中对应的可变向量、盒子、哈希表、字符串、字节串、 mutable pairs 和可变结构必须是 eq? 。

否则, v2 中的陪护者和其他冒名顶替者必须在 v1 中完好无损,类似于 impersonator-of? 要求冒名顶替者被保留的方式。 此外, v1 不能有任何非陪护者的冒充者,其在 v2 中的对应值不是同一个冒充者。 请注意, chaperone-of? 意味着 impersonator-of?,但反之亦然。

procedure

(impersonator-ephemeron v) → ephemeron?

  v : any/c
产生一个 ephemeron ,可以用来连接 v 的可达性 (在垃圾收集的意义上;见 垃圾回收)和 v 是一个 impersonator 的任何值的可达性。 也就是说,只要结果 ephemeron 是可及的,除了 v 所冒充的任何值(包括它自己),值 v 将被认为是可及的。

procedure

(procedure-impersonator*? v) → boolean?

  v : any/c
对于任何由 impersonate-procedure* 或 chaperone-procedure* 产生的程序冒充者, 或者是由 impersonate-procedure* 或 chaperone-procedure* 创建的值的冒充者/陪衬者 (可能是过渡性的),返回 #t 。

14.5.1 冒充者构造器

procedure

(impersonate-procedure proc 
  wrapper-proc 
  prop 
  prop-val ... 
  ...) 
 → (and/c procedure? impersonator?)
  proc : procedure?
  wrapper-proc : (or/c procedure? #f)
  prop : impersonator-property?
  prop-val : any/c
返回一个与 proc 具有相同算数、名称和其他属性的冒名顶替过程。 当应用冒名顶替过程时,参数首先被传递给 wrapper-proc (当它不是 #f时), 然后 wrapper-proc 的结果被传递给 proc 。 wrapper-proc 也可以提供一个处理 proc 结果的过程。

wrapper-proc 的arity必须包括 proc 的arity。 wrapper-proc 允许的关键字参数必须是 proc允许的关键字的超集。 wrapper-proc 的必要关键字参数必须是 proc 的必要关键字的一个子集。

对于没有关键字的应用程序, wrapper-proc 的结果必须至少与提供给它的值数量相同。 额外的结果可以被提供–在与提供的值相对应的值之前–以下列模式提供:

  • 一个可选的过程, result-wrapper-proc, 它将被应用于 proc 的结果;然后是

  • 任何数量的 'mark key val 的重复(即三个值), 其中调用 proc 被包起来以安装一个 continuation mark key 和 val 。

如果产生了 result-wrapper-proc ,它必须是一个过程,接受与 proc 产生的结果相同的数量; 它必须返回相同数量的结果。 如果没有提供 result-wrapper-proc ,则相对于对冒名顶替者的调用而言, 将在 tail position 调用 proc 。

对于包含关键字参数的应用程序, wrapper-proc 必须在任何其他值之前, 但在 result-wrapper-proc 和 'mark key val 序列(如果有的话)之后返回一个附加值。 该附加值必须是提供给冒名顶替者的关键字参数的替换列表(也就是说,不包括没有提供的可选参数)。 这些参数必须按照所提供的参数的关键字的排序顺序排列。

如果 wrapper-proc 为 #f, 那么应用产生的冒充者与应用 proc 相同。 如果 wrapper-proc 是 #f ,并且没有提供 prop , 那么将返回 proc ,并且不会被冒充。

prop 和 prop-val (impersonate-procedure 的参数数必须是偶数) 对添加 impersonator 属性或覆盖 proc 的 impersonator 属性值。

如果任何一个 prop 是 impersonator-prop:application-mark , 并且如果相关的 prop-val 是一对, 那么对 proc 的调用将用 with-continuation-mark 进行包装, 使用 (car prop-val) 作为标记键, (cdr prop-val) 作为标记值。 此外,如果对被冒充过程的调用的直接延续帧包括 (car prop-val) 的值–也就是说,如果 call-with-immediate-continuation-mark 会在调用的延续帧中产生 (car prop-val) 的值–那么该值也会作为 (car prop-val) 的即时值被安装, 作为调用 wrapper-proc 时的标记 (这允许在 wrapper-proc 中检测到与包裹冒充者有关的冒充者的尾巴调用)。

Changed in version 6.3.0.5 of package base: 增加了对来自 wrapper-proc 的 'mark key val 结果的支持。

Examples:
> (define (add15 x) (+ x 15))
> (define add15+print
    (impersonate-procedure add15
                           (λ (x)
                             (printf "called with ~s\n" x)
                             (values (λ (res)
                                       (printf "returned ~s\n" res)
                                       res)
                                     x))))
> (add15 27)

42

> (add15+print 27)

called with 27

returned 42

42

> (define-values (imp-prop:p1 imp-prop:p1? imp-prop:p1-get)
    (make-impersonator-property 'imp-prop:p1))
> (define-values (imp-prop:p2 imp-prop:p2? imp-prop:p2-get)
    (make-impersonator-property 'imp-prop:p2))
> (define add15.2 (impersonate-procedure add15 #f imp-prop:p1 11))
> (add15.2 2)

17

> (imp-prop:p1? add15.2)

#t

> (imp-prop:p1-get add15.2)

11

> (imp-prop:p2? add15.2)

#f

> (define add15.3 (impersonate-procedure add15.2 #f imp-prop:p2 13))
> (add15.3 3)

18

> (imp-prop:p1? add15.3)

#t

> (imp-prop:p1-get add15.3)

> (imp-prop:p2? add15.3)

#t

> (imp-prop:p2-get add15.3)

> (define add15.4 (impersonate-procedure add15.3 #f imp-prop:p1 101))
> (add15.4 4)

> (imp-prop:p1? add15.4)

#t

> (imp-prop:p1-get add15.4)

> (imp-prop:p2? add15.4)

> (imp-prop:p2-get add15.4)

procedure

(impersonate-procedure* proc 
  wrapper-proc 
  prop 
  prop-val ... 
  ...) 
 → (and/c procedure? impersonator?)
  proc : procedure?
  wrapper-proc : (or/c procedure? #f)
  prop : impersonator-property?
  prop-val : any/c
与 impersonate-procedure 类似,只是 wrapper-proc 在所有其他参数之前收到一个额外的参数。 这个附加参数是最初应用的程序 orig-proc 。

如果直接应用 impersonate-procedure* 的结果,那么 orig-proc 就是该结果。 然而,如果结果在被应用之前被进一步冒充, orig-proc 就是进一步冒充者。

orig-proc 参数可能很有用, 这样 wrapper-proc 就可以提取被其他冒充者覆盖的 impersonator properties ,比如说。

Added in version 6.1.1.5 of package base.

procedure

(impersonate-struct v    
  [struct-type]    
  orig-proc    
  redirect-proc ...    
  ...    
  prop    
  prop-val ...    
  ...) → any/c
  v : any/c
  struct-type : struct-type? = unspecified
  orig-proc : 
(or/c struct-accessor-procedure?
      struct-mutator-procedure?
      struct-type-property-accessor-procedure?)
  redirect-proc : (or/c procedure? #f)
  prop : impersonator-property?
  prop-val : any/c
返回一个 v 的冒充者,该冒充者对被冒充的值重定向某些操作。 orig-proc 指出要重定向的操作,而相应的 redirect-proc 提供重定向的操作。 可选的 struct-type 参数,当提供时,作为 v 的表示的见证, 它必须是一个 struct-type 的实例。

redirect-proc 的协议取决于相应的 orig-proc , 其中 self 指的是 orig-proc 最初应用到的值:

  • 一个结构域访问器: redirect-proc 必须接受两个参数, self 和 orig-proc 为 v 产生的值 field-v ； 它必须返回 field-v 的替换。 相应的字段必须不是不可变的,并且该字段的结构类型必须可以通过当前 inspector 访问, 或者其他 orig-proc 必须是同一字段的结构-字段突变器之一。

  • 结构字段突变器: redirect-proc 必须接受两个参数, 即 self 和提供给突变器的值 field-v ; 它必须返回 field-v 的替换,并传播给 orig-proc 和 v 。

  • 属性访问器: redirect-proc 使用与结构域访问器相同的协议。 访问器的属性必须以 'can-impersonate 作为 make-struct-type-property 的第二个参数来创建。

当 redirect-proc 为 #f 时,相应的 orig-proc 不受影响。 为 redirect-proc 提供 #f 是有用的, 它允许其 orig-proc 作为 v 的表示的 "证人",并允许增加 prop 。

prop 和 prop-val (如果提供了 struct-type, impersonate-struct 的参数数必须是偶数,否则是奇数) 的配对添加了 impersonator 属性或覆盖了 v 的impersonator属性值。

每个 orig-proc 必须表示一个不同的操作。 如果没有提供 struct-type 和 orig-proc ,那么必须不提供 prop 。 如果 orig-proc 只提供了 #f redirect-proc , 并且没有提供 prop ,那么 v 将被返回,并且不会被冒充。

如果任何 orig-proc 本身是一个冒充者,那么对 orig-proc 所冒充的访问器或变体的使用将被重定向, 以便在 redirect-proc 之前(如果是访问器)或 redirect-proc 之后(如果是变体)对v 使用 orig-proc 。

Changed in version 6.1.1.2 of package base: 将访问器或突变器的 redirect-proc 的第一个参数从 v 改为 self 。
Changed in version 6.1.1.8: 增加了可选的 struct-type 参数。

procedure

(impersonate-vector vec 
  ref-proc 
  set-proc 
  prop 
  prop-val ... 
  ...) 
 → (and/c vector? impersonator?)
  vec : (and/c vector? (not/c immutable?))
  ref-proc : (or/c (vector? exact-nonnegative-integer? any/c . -> . any/c) #f)
  set-proc : (or/c (vector? exact-nonnegative-integer? any/c . -> . any/c) #f)
  prop : impersonator-property?
  prop-val : any/c
返回一个 vec 的冒充者,该冒充者重定向 vector-ref 和 vector-set! 操作。

ref-proc 和 set-proc 的参数必须都是程序,或者都是 #f 。 如果它们都是 #f ，那么 impersonate-vector 就不会介入 vec, 但仍然允许附加impersonator属性。

如果 ref-proc 是一个过程,它必须接受 vec, 一个传递给 vector-ref 的索引,以及 vector-ref 在 vec 上为给定索引产生的值; 它必须产生一个值的替换,这个值是 vector-ref 在 impersonator 上的结果。

如果 set-proc 是一个过程,它必须接受 vec, 传递给 vector-set! 的索引和传递给 vector-set! 的值; 它必须产生一个值的替换,该替换与 vector-set! 一起在原始 vec 上使用,以安装该值。

prop 和 prop-val (impersonate-vector 的参数数必须是奇数) 的一对增加了 impersonator 属性或覆盖 vec 的 impersonator-property 值。

Changed in version 6.9.0.2 of package base: 增加了非交互式向量的冒充者。

procedure

(impersonate-vector* vec 
  ref-proc 
  set-proc 
  prop 
  prop-val ... 
  ...) 
 → (and/c vector? impersonator?)
  vec : (and/c vector? (not/c immutable?))
  ref-proc : (or/c (vector? vector? exact-nonnegative-integer? any/c . -> . any/c) #f)
  set-proc : (or/c (vector? vector? exact-nonnegative-integer? any/c . -> . any/c) #f)
  prop : impersonator-property?
  prop-val : any/c
像 impersonate-vector 一样, 除了 ref-proc 和 set-proc 各自在其他参数之前接收一个额外的向量作为参数。 额外的参数是原始的被冒充的向量,对它的访问首先触发了插值。

额外的向量参数可能很有用,例如, ref-proc 或 set-proc 可以提取被其他冒充者覆盖的冒充者属性。

Added in version 6.9.0.2 of package base.

procedure

(impersonate-box box    
  unbox-proc    
  set-proc    
  prop    
  prop-val ...    
  ...) → (and/c box? impersonator?)
  box : (and/c box? (not/c immutable?))
  unbox-proc : (box? any/c . -> . any/c)
  set-proc : (box? any/c . -> . any/c)
  prop : impersonator-property?
  prop-val : any/c
返回 box 的一个冒充者,它重定向 unbox 和 set-box! 操作。

unbox-proc 必须接受 box 和 unbox 在 box 上产生的值; 它必须产生一个替换值,这是 unbox 对冒充者的结果。

prop 和 prop-val (impersonate-box 的参数数量必须是奇数) 的对子增加了 impersonator 属性或覆盖了 box 的 impersonator 属性值。

  equal-key-proc]

返回一个 hash 的冒充者,它重定向 hash-ref, hash-set! 或 hash-set (如适用), hash-remove 或 hash-remove! (如适用), hash-clear 或 hash-clear! (视情况而定,如果 clear-proc 不是 #f) 操作。 当 hash-set, hash-remove 或 hash-clear 被用于哈希表的冒名顶替者时, 其结果是一个具有相同重定向程序的冒名顶替者。 此外,像 hash-iterate-key 或 hash-map 这样从表中提取键的操作, 使用 key-proc 来替换从表中提取的键。 像 hash-iterate-value 或 hash-values 这样的操作隐含地使用 hash-ref , 因此通过 ref-proc 重定向。 hash-ref-key 操作同时使用 ref-proc 和 key-proc, 前者用于查找请求的键,后者用于提取。

ref-proc 必须接受 hash 和传递给 hash-ref 的一个key。 它必须返回一个替换键和一个过程。 只有当通过 hash-ref 在 hash 中找到返回的键时,才会调用返回的过程, 在这种情况下,该过程会被调用,并带有 hash, 先前返回的键和找到的值。 返回的过程本身必须返回一个替换找到的值。 返回的过程会被 hash-ref-key 所忽略。

set-proc 必须接受 hash, 传递给 hash-set! 或 hash-set 的键, 以及传递给 hash-set! 或 hash-set 的值;它必须产生两个值:一个键的替换值和一个值的替换。 返回的键和值与原始 hash 上的 hash-set! 或 hash-set 一起使用,以安装该值。

remove-proc 必须接受 hash 和传递给 hash-remove! 或 hash-remove 的键; 它必须产生一个键的替换值, 该替换值与原始 hash 上的 hash-remove! 或 hash-remove 一起使用, 以删除任何使用(冒名顶替者替换的)键的映射。

key-proc 必须接受 hash 和一个从 hash 中提取的密钥(通过 hash-ref-key, hash-iterate-key 或其他内部使用 hash-iterate-key 的操作); 它必须产生一个密钥的替换,然后将其报告为从表中提取的一个密钥。

如果 clear-proc 不是 #f, 它必须接受 hash 作为参数,并且其结果被忽略。 clear-proc 返回的事实(而不是引发一个异常或以其他方式逃避)授予了从 hash 中删除所有键的能力。 如果 clear-proc 是 #f, 那么冒名顶替者上的 hash-clear 或 hash-clear! 将使用 hash-iterate-key 和 hash-remove 或 hash-remove! 实现。

如果 equal-key-proc 不是 #f, 它将有效地介入对 equal?, equal-hash-code 和 equal-secondary-hash-code 的调用,用于 hash 的键。 equal-key-proc 必须接受 hash 和一个由 hash 映射的键或者传递给 hash-ref 等的键作为参数,后者可能已经被相应的 ref-proc 等调整过。 其结果是一个值,根据需要传递给 equal?, equal-hash-code 和 equal-secondary-hash-code ,以便对键进行散列和比较。 在 hash-set! 或 hash-set 的情况下, 传递给 equal-key-proc 的键是存储在哈希表中的键,以便将来查找。

hash-iterate-value, hash-map 或 hash-for-each 函数使用 hash-iterate-key 和 hash-ref 的组合。 如果一个由 key-proc 产生的键没有通过 hash-ref 产生一个值, 那么就会引发 exn:fail:contract exception is raised 。

prop 和 prop-val 对添加冒充者属性或覆盖 hash 的冒充者属性值。

在不可变的哈希表的情况下,如果两个冒充的哈希表最初是通过调用 impersonate-hash 或 chaperone-hash (并可能通过 hash-set, hash-remove 或 hash-clear 传播)附加到哈希表上的, 只要第一个哈希表的内容是第二个哈希表的 impersonator-of? , 就算作 "同一个值"(为了 impersonator-of? 目的)。

prop:impersonator-of : struct-type-property?

一个 structure type property (见 结构类型属性) , 它提供了一个从代表冒充者的结构中提取冒充值的程序。该属性用于 impersonator-of? 以及 equal? 。

value

prop:authentic : struct-type-property?

一个 structure type property ,宣布一个结构类型为 authentic 。 与该属性相关的值被忽略;该属性本身的存在使得该结构类型是真实的。

authentic 结构类型的实例不能通过 impersonate-struct 冒充, 也不能通过 chaperone-struct 陪同。 因此,只有当一个 authentic 的结构类型的实例是一个 flat contract 时, 它才能被赋予契约(见 struct/c) 。

将结构类型声明为 authentic 的可以防止不必要的结构冒充,但是暴露的结构类型通常应该支持冒充者或陪护者以促进契约。 将结构类型声明为 authentic 的也可以稍微提高结构谓词、选择器和突变器的性能, 这对于在库中频繁使用的私有数据结构来说是合适的。

同步时调用 proc 生成器,很像传递给 guard-evt 的过程, 只是 proc 被赋予了 evt 。 proc 必须返回两个值:一个 synchronizable event ,它是 evt 的陪衬, 以及一个用于检查事件结果的过程,如果它在选择中被选中。 后面的过程接受 evt 的结果,它必须返回该值的陪衬物。

prop 和 prop-val (chaperone-evt 的参数数必须是偶数) 的一对增加了 impersonator 属性或覆盖了 evt 的 impersonator-perty 值。

其结果是 chaperone-of? 参数 evt。 然而,如果 evt 是一个 thread, semaphore, input port, output port 或 will executor ,那么结果就不会被识别为这样。 例如, thread? 应用于 chaperone-evt 的结果将总是产生 #f 。

14.5.3 冒名顶替者属性

procedure

(make-impersonator-property name)

 → 
impersonator-property?
(-> any/c boolean?)
(->* (impersonator?) (any/c) any)
  name : symbol?
创建一个新的 impersonator property 并返回三个值:

  • impersonator property descriptor, 用于 impersonate-procedure, chaperone-procedure, 和其他impersonator构造函数;

  • impersonator property predicate 过程,它接受一个任意的值, 如果该值是一个具有该属性值的冒名顶替者,则返回 #t ,否则返回 #f ;

  • impersonator property accessor 过程,它返回与该属性的 impersonator 相关的值; 如果给访问器的值不是一个 impersonator 或者没有该属性的值(即如果相应的 impersonator 属性谓词返回 #f), 那么选择器的第二个可选参数决定了它的响应:如果没有提供第二个参数,就会引发 exn:fail:contract exception is raised, 如果第二个参数是一个过程,就用零参数进行尾随呼叫,否则就返回第二个参数。

procedure

(impersonator-property? v) → boolean?

  v : any/c
如果 v 是一个 impersonator property descriptor 的值, 返回 #t ,否则返回 #f 。

(impersonator-property-accessor-procedure? v) → boolean?

  v : any/c
如果 v 是一个由 make-impersonator-property 产生的访问过程, 则返回 #t ,否则返回 #f 。
