13.1.7 管道

Racket pipe 是 Racket 内部的,与用于不同进程之间通信的操作系统级管道无关。 操作系统级的管道可以由 subprocess 创建,在 Unix 文件系统上打开一个现有的命名文件, 或者用管道启动 Racket 的原始输入、输出或错误端口。 这种管道是 file-stream ports ,与 make-pipe 产生的管道不同。

procedure

(make-pipe [limit input-name output-name])

 → 
input-port? output-port?
  limit : exact-positive-integer? = #f
  input-name : any/c = 'pipe
  output-name : any/c = 'pipe
返回两个端口值:第一个端口是一个输入端口,第二个是一个输出端口。 写到输出端口的数据从输入端口读取,没有中间缓冲。 与其他一些类型的端口不同,管道端口不需要明确关闭就可以被 garbage collection.

如果 limit 是 #f, 那么新的管道将持有无限数量的未读字节(即,仅受可用内存的限制)。 如果 limit 是一个正数,那么管道将最多容纳 limit 的未读/未读字节; 此后,向管道的输出端口的写入将被阻止,直到从输入端口的读取或偷看带来更多可用空间。 (窥视可以有效地延长端口的容量,直到被窥视的字节被读完)。

可选的 input-name 和 output-name 分别作为返回的输入和输出端口的名称。

procedure

(pipe-content-length pipe-port) → exact-nonnegative-integer?

  pipe-port : port?
返回一个管道中包含的字节数,其中 pipe-port 是由 make-pipe 产生的管道的任一端口。 管道的内容长度计算所有已经写入管道但尚未读取(尽管可能被偷看)的字节。
