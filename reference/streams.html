4.16.2 Streams

stream 是一种 sequence , 通过 stream-first 和 stream-rest 支持功能迭代。 stream-cons 形式构造了一个懒惰的流,但是普通的列表也可以作为流使用, 而且 in-range 和 in-naturals 等函数也可以创建流。

 (require racket/stream) package: base
The bindings documented in this section are provided by the racket/stream and racket libraries, but not racket/base.

procedure

(stream? v) → boolean?

  v : any/c
如果 v 可以作为一个 stream 使用,返回 #t ,否则返回 #f 。

procedure

(stream-empty? s) → boolean?

  s : stream?
如果 s 没有元素,返回 #t ,否则返回 #f 。

procedure

(stream-first s) → any

  s : (and/c stream? (not/c stream-empty?))
返回 s 中第一个元素的值。

procedure

(stream-rest s) → stream?

  s : (and/c stream? (not/c stream-empty?))
返回一个与 s 等价的没有第一个元素的流。

syntax

(stream-cons first-expr rest-expr)

(stream-cons #:eager first-expr rest-expr)
(stream-cons first-expr #:eager rest-expr)
(stream-cons #:eager first-expr #:eager rest-expr)
产生一个流,其第一个元素由 first-expr 决定,其余元素由 rest-expr 决定。

如果 first-expr 前面没有 #:eager ,那么 first-expr 不会被立即评估。 相反,结果流上的 stream-first 强制评估 first-expr (一次)以产生流的第一个元素。 如果评估 first-expr 引发异常或试图强迫自己,那么就会引发 exn:fail:contract exception is raised , 而未来强迫评估的尝试将引发另一个异常。

如果 rest-expr 前面没有 #:eager ,那么 rest-expr 就不会立即被评估。 相反,在结果流上的 stream-rest 会产生另一个流, 与 (stream-lazy rest-expr) 产生的流一样。

由 first-expr 产生的流的第一个元素可以是多值。 rest-expr 在被评估时必须产生一个流,否则会引发 exn:fail:contract? exception is raised 。

Changed in version 8.0.0.12 of package base: 增加 #:eager 选项。
Changed in version 8.8.0.7: 改为允许多个值。

syntax

(stream-lazy stream-expr)

(stream-lazy #:who who-expr stream-expr)
类似于 (delay stream-expr), 但结果是一个流而不是一个 promise , 而且 stream-expr 在最终强制执行时必须产生一个流。 stream-lazy 产生的流与 stream-expr 产生的流具有相同的内容;也就是说, 在结果流上进行像 stream-first 这样的操作会强制 stream-expr 并对其结果进行重试。

如果评估 stream-expr 引起异常或试图强迫自己,那么就会产生 exn:fail:contract exception is raised , 而未来强迫评估的尝试将触发另一个异常。

如果提供了 who-expr ,它将在构造延迟的流时被评估。 如果 stream-expr 后来产生了一个不是流的值,并且如果 who-expr 产生了一个符号值, 那么这个符号将被用于错误信息。

Added in version 8.0.0.12 of package base.

procedure

(stream-force s) → stream?

  s : stream?
从 stream-lazy, stream-cons 的 stream-rest 等处强制评估一个延迟流,返回强制流。 如果 s 不是一个延迟流,那么 s 被返回。

通常情况下, stream-force 是不需要的,因为像 stream-first, stream-rest 和 stream-empty? 这样的操作会根据需要强制产生一个延迟流。 在极少数情况下, stream-force 对于揭示一个流的底层实现是很有用的 (例如,一个流是一个具有 prop:stream 属性的结构类型的实例)。

Added in version 8.0.0.12 of package base.

syntax

(stream elem-expr ...)

 
elem-expr = (values single-expr ...)
  | single-expr
嵌套的 stream-cons 的缩写,以 empty-stream 结束。 作为一个匹配模式, stream 匹配一个有 elem-expr 一样多的元素的流, 并且每个元素必须匹配相应的 elem-expr 模式。 模式 elem-expr 可以是 (values single-expr ...), 它与流中多个有价值的元素匹配。

Changed in version 8.8.0.7 of package base: 改为允许多个值。

syntax

(stream* elem-expr ... tail-expr)

嵌套的 stream-cons 的简写,但是 tail-expr 必须在强制时产生一个流, 并且该流被用作流的其余部分而不是 empty-stream 。 与 list* 类似,但用于流。 作为一种匹配模式, stream* 类似于 stream 模式, 但是 tail-expr 模式匹配最后一个 elem-expr 之后的流的 "其余部分"。

Added in version 6.3 of package base.
Changed in version 8.0.0.12: 改为延迟 rest-expr ,即使提供的 expr 为零。
Changed in version 8.8.0.7: 改为允许多个值。

procedure

(in-stream s) → sequence?

  s : stream?
返回一个等同于 s 的序列。
An in-stream application can provide better performance for streams iteration when it appears directly in a for clause.
See for for information on the reachability of stream elements during an iteration.

Changed in version 6.7.0.4 of package base: 改进了 for 中流的元素可及性保证。

value

empty-stream : stream?

一个没有元素的流。

procedure

(stream->list s) → list?

  s : stream?
返回一个列表,其元素是 s 的元素,每个元素必须是一个单值。如果 s 是无限的,这个函数不会终止。

procedure

(stream-length s) → exact-nonnegative-integer?

  s : stream?
返回 s 的元素数。 如果 s 是无限的,这个函数不会终止。

在懒惰流的情况下,这个函数只强制评估子流,而不是流的元素。

procedure

(stream-ref s i) → any

  s : stream?
  i : exact-nonnegative-integer?
返回 s 的第 i 个元素(可能是多个值)。

procedure

(stream-tail s i) → stream?

  s : stream?
  i : exact-nonnegative-integer?
返回一个等同于 s 的流,只是省略了前 i 个元素。

如果从 s 中提取元素涉及到副作用,那么在从结果流中提取第一个元素之前,这些元素将不会被提取出来。

procedure

(stream-take s i) → stream?

  s : stream?
  i : exact-nonnegative-integer?
返回 s 的前 i 个元素的流。

procedure

(stream-append s ...) → stream?

  s : stream?
返回一个流,该流包含每个流中的所有元素,并按照它们在原始流中出现的顺序排列。 新的流被懒散地构造,而最后给出的流被用于结果的尾部。

procedure

(stream-map f s) → stream?

  f : procedure?
  s : stream?
返回一个包含应用于 s 中每个元素的 f 的流,新的流被懒散地构造。

procedure

(stream-andmap f s) → boolean?

  f : (-> any/c ... boolean?)
  s : stream?
如果 f 对 s 的每个元素都返回一个真结果,则返回 #t 。 如果 s 是无限的,并且 f 从未返回一个错误的结果,这个函数不会终止。

procedure

(stream-ormap f s) → boolean?

  f : (-> any/c ... boolean?)
  s : stream?
如果 f 在 s 的某个元素上返回一个真结果,则返回 #t 。 如果 s 是无限的,并且 f 从未返回一个真实的结果,这个函数不会终止。

procedure

(stream-for-each f s) → void?

  f : (-> any/c ... any)
  s : stream?
将 f 应用于 s 的每个元素。 如果 s 是无限的,这个函数不会终止。

procedure

(stream-fold f i s) → any/c

  f : (-> any/c any/c ... any/c)
  i : any/c
  s : stream?
将 f 折叠在 s 的每个元素上,以 i 作为初始累加器。 如果 s 是无限的,这个函数不会终止。 f 函数将累加器作为其第一个参数,将下一个流元素作为其第二个参数。

procedure

(stream-count f s) → exact-nonnegative-integer?

  f : procedure?
  s : stream?
返回 f 返回真结果的 s 中的元素数。 如果 s 是无限的,这个函数不会终止。

procedure

(stream-filter f s) → stream?

  f : (-> any/c ... boolean?)
  s : stream?
返回一个流,其元素是 f 返回真结果的 s 的元素。 虽然新的流是懒散地构建的,但如果 s 有无限多的元素,其中 f 返回一个错误的结果, 那么在无限的子流中,对这个流的操作将不会终止。

procedure

(stream-add-between s e) → stream?

  s : stream?
  e : any/c
返回一个流,其元素是 s 的元素,但在 s 的每一对元素之间有 e 。 新流是懒惰地构建的。

syntax

(for/stream (for-clause ...) body-or-break ... body)

syntax

(for*/stream (for-clause ...) body-or-break ... body)

分别像 for/list 和 for*/list 那样进行迭代, 但结果被懒散地收集到一个 stream 中,而不是一个列表。

与大多数 for 形式不同,这些形式是懒惰地评估的, 所以每个 body 都不会被评估,直到产生的流被强制。 这允许 for/stream 和 for*/stream 在无限的序列上进行迭代, 而不像它们的有限对应形式。

Examples:
> (for/stream ([i '(1 2 3)]) (* i i))

#<stream>

> (stream->list (for/stream ([i '(1 2 3)]) (* i i)))

'(1 4 9)

> (stream-ref (for/stream ([i '(1 2 3)]) (displayln i) (* i i)) 1)

2

4

> (stream-ref (for/stream ([i (in-naturals)]) (* i i)) 25)

625

> (stream-ref (for/stream ([i (in-naturals)]) (values i (add1 i))) 10)

10

11

Added in version 6.3.0.9 of package base.
Changed in version 8.8.0.7: 改为允许多个值。

value

gen:stream : any/c

将三个方法关联到一个结构类型,以实现流的 generic interface (参见 通用接口) 。

为了提供方法的实现,应该在结构类型定义中使用 #:methods 关键字。 以下三个方法必须被实现:

  • stream-empty? : 接受一个参数

  • stream-first : 接受一个参数t

  • stream-rest : 接受一个参数

Examples:
> (struct list-stream (v)
    #:methods gen:stream
    [(define (stream-empty? stream)
       (empty? (list-stream-v stream)))
     (define (stream-first stream)
       (first (list-stream-v stream)))
     (define (stream-rest stream)
       (list-stream (rest (list-stream-v stream))))])
> (define l1 (list-stream '(1 2)))
> (stream? l1)

#t

> (stream-first l1)

1

Changed in version 8.7.0.5 of package base: 增加了一个检查,使省略 stream-empty?, stream-first 和 stream-rest 中的任何一个现在都是一个语法错误。

value

prop:stream : struct-type-property?

一个结构类型属性,用于定义流 API 的自定义扩展。 不鼓励使用 prop:stream 属性, 而是使用 gen:stream generic interface 。 接受三个程序的向量,其参数与 gen:stream 中的方法相同。

procedure

(stream/c c) → contract?

  c : contract?
返回一个能识别流的契约。 流的所有元素必须与 c 。

如果 c 参数是一个平面合同或一个陪衬合同,那么结果将是一个陪衬合同。 否则,结果将是一个冒名顶替者契约。

当一个 stream/c 合约被应用于一个流时,其结果不是与输入的 eq? 。 结果将是输入的 chaperone 或 impersonator ,这取决于合约的类型。

流上的契约在必要时被懒散地评估(因为流可能是无限的)。 违反合约的情况不会被提出,直到违反的值被从流中检索出来。 作为这一规则的例外,作为列表的流会被立即检查,就像 c 被用在 listof 上一样。

如果一个契约被应用于一个流,而这个流随后被用作另一个流的尾部(作为 stream-cons 的第二个参数), 新的元素将不会被检查到契约,但是尾部的元素仍将被执行。

Added in version 6.1.1.8 of package base.
