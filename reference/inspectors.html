14.9 结构检查器

inspector 提供对结构字段和结构类型信息的访问,而不需要正常的字段访问器和突变器。 (检查器也被用来控制对模块绑定的访问;见 Code Inspectors)。 检查器主要用于调试器的使用。

当一个结构类型被创建时,可以提供一个检查器。 给定的检查器不是控制新结构类型的检查器;相反,给定检查器的父级将控制该类型。 通过使用给定检查器的父检查器,该结构类型对无法访问父检查器的 "同行 "代码保持不透明。

current-inspector parameter 决定了新结构类型的默认检查器参数。 可以通过 struct 形式的 #:inspector 选项(参见 定义结构类型: struct), 或者通过 make-struct-type 的可选 inspector 参数,提供一个替代的检查器。

procedure

(inspector? v) → boolean?

  v : any/c
如果 v 是一个检查器, 返回 #t ,否则返回 #f 。

procedure

(make-inspector [inspector]) → inspector?

  inspector : inspector? = (current-inspector)
返回一个新的检查器,该检查器是 inspector 的子检查器。 任何由新检查器控制的结构类型也被其祖先检查器所控制,但没有其他检查器。

procedure

(make-sibling-inspector [inspector]) → inspector?

  inspector : inspector? = (current-inspector)
返回一个新的检查器,它是与 inspector 相同的检查器的一个子检查器。 也就是说, inspector 和结果检查器控制的结构类型集是相互不相干的。

procedure

(inspector-superior? inspector    
  maybe-subinspector) → boolean?
  inspector : inspector?
  maybe-subinspector : inspector?
如果 inspector 是 maybe-subinspector 的祖先(并且不等于 maybe-subinspector), 则返回 #t ,否则返回 #f 。

Added in version 6.5.0.6 of package base.

parameter

(current-inspector) → inspector?

(current-inspector insp) → void?
  insp : inspector?
一个 parameter ,决定了新创建的结构类型的默认检查器。

procedure

(struct-info v) → 
(or/c struct-type? #f) boolean?
  v : any/c
返回两个值:

  • struct-type: 结构类型描述符或 #f; 结果是一个结构类型描述符,即 v 是一个实例的最具体类型,并且当前检查器对其有控制权, 如果当前检查器不控制任何 struct 是实例的结构类型,则结果是 #f 。

  • skipped?: 如果第一个结果对应的是 v 的最具体的结构类型, 则为 #f ,否则为 #t 。

procedure

(struct-type-info struct-type)

 → 
symbol?
exact-nonnegative-integer?
exact-nonnegative-integer?
struct-accessor-procedure?
struct-mutator-procedure?
(listof exact-nonnegative-integer?)
(or/c struct-type? #f)
boolean?
  struct-type : struct-type?
返回八个数值,提供结构类型描述符 struct-type 的信息,假设该类型由当前检查器控制:

  • name: 结构类型的名称,作为一个符号;

  • init-field-cnt: 结构类型提供给构造程序的字段数量(不包括由其祖先类型创建的字段);

  • auto-field-cnt: 结构类型定义的字段的数量, 在构造函数中没有对应的字段(不包括由其祖先类型创建的字段);

  • accessor-proc: 结构类型的访问程序,就像 make-struct-type 所返回的那样;

  • mutator-proc: 结构类型的一个突变过程,就像 make-struct-type 返回的那个;

  • immutable-k-list: 一个精确的非负整数的不可变列表,对应于结构类型的不可变区域;

  • super-type: 当前检查器控制的该类型的最具体的祖先的结构类型描述符, 如果当前检查器没有控制任何祖先,则为 #f ;

  • skipped?: 如果第七个结果是最具体的祖先类型或该类型没有超级类型, 则为 #f ,否则为 #t 。

如果 struct-type 的类型不受当前检查器的控制,会引发 exn:fail:contract exception is raised 。

procedure

(struct-type-sealed? struct-type) → boolean?

  struct-type : struct-type?
报告 struct-type 是否具有 prop:sealed 结构类型属性。

Added in version 8.0.0.7 of package base.

procedure

(struct-type-authentic? struct-type) → boolean?

  struct-type : struct-type?
报告 struct-type 是否具有 prop:authentic 结构类型属性。

Added in version 8.0.0.7 of package base.

procedure

(struct-type-make-constructor struct-type 
  [constructor-name]) 
 → struct-constructor-procedure?
  struct-type : struct-type?
  constructor-name : (or/c symbol? #f) = #f
返回用于创建 struct-type 实例的 constructor 过程。 如果 constructor-name 不是 #f, 它将作为生成的 constructor 过程的名称。 如果 struct-type 的类型不受当前检查器的控制,将引发 exn:fail:contract exception is raised 。

procedure

(struct-type-make-predicate struct-type) → any

  struct-type : any/c
返回一个用于识别 struct-type 实例的 predicate过程。 如果 struct-type 的类型不受当前检查器的控制,则会引发 exn:fail:contract exception is raised 。

procedure

(object-name v) → any

  v : any/c
如果 v 有名字,返回 v 的名字的值,否则返回 #f 。 参数 v 可以是任何值,但只有(部分)过程 structures, structure types, structure type properties, regexp values, ports, loggers 和 prompt tags 有名字。参见 推断值的名称 。

如果一个 structure 的类型实现了 prop:object-name 属性, 并且 prop:object-name 属性的值是一个整数,那么该结构的相应字段就是该结构的名称。 否则,该属性值必须是一个过程,该过程以结构为参数被调用,其结果是结构的名称。 如果一个 structure 是由它的一个字段实现的过程 (也就是说,结构类型的 prop:procedure 属性值是一个整数),那么它的名字就是实现过程的名称。 否则,它的名称与它所实例化的 structure type 的名称相匹配。

过程的名称(如果有的话)是一个符号,除非该过程也是一个结构,其类型具有 prop:object-name属性, 在这种情况下, prop:object-name 优先。 procedure-rename 函数用一个特定的名字创建一个过程。

regexp value 的名称是一个字符串或字节字符串。 将字符串或字节字符串传递给 regexp, byte-regexp, pregexp 或 byte-pregexp (取决于其名称被提取的 regexp 的种类) 会产生一个与相同输入相匹配的值。

端口的名称可以是任何值,但许多工具使用路径或字符串名称作为端口的为(例如,报告源位置)。

logger 的名字是一个符号或 #f 。

prompt tag 的名字是给 make-continuation-prompt-tag 的可选符号或者 #f 。

Changed in version 7.9.0.13 of package base: 识别延续性提示标签的名称。

value

prop:object-name : struct-type-property?

structure type property ,允许结构类型自定义应用于其实例的 object-name 的结果。 该属性值可以是以下任何一种:

  • 一个参数的过程 proc : 在这种情况下,过程 proc 接收结构作为一个参数, proc 的结果是结构的 object-name。

  • 一个精确的非负整数,介于 0 (包括)和结构类型中的非自动字段数量之间(不包括,不计算超类型字段): 这个整数标识了结构中的一个字段,并且该字段必须被指定为不可变的。 该字段的值被用作该结构的 object-name 。

Added in version 6.2 of package base.
