14.11 Plumbers

plumber 支持 flush callbacks ,这通常是在 Racket 进程或 place 退出之前触发的。 例如, flush callback 可以刷新一个输出端口的缓冲区。

Flush callbacks 大致上类似于标准 C 库的 atexit, 但冲洗回调也可以用于其他类似的情况。

不能保证在进程终止前调用 flush 回调–要么是因为管道工不是被默认 exit handler 刷新的原始管道工, 要么是因为进程被强行终止(例如,通过监护人关闭)。

procedure

(plumber? v) → boolean?

  v : any/c
如果 v 是一个 plumber 的值,返回 #t ,否则返回 #f 。

Added in version 6.0.1.8 of package base.

procedure

(make-plumber) → plumber?

创建一个新的 plumber 。

管道工没有层次结构(不像 custodians 或 inspectors), 但可以在一个管道工中注册一个 flush callback , 以便与另一个管道工一起调用 plumber-flush-all 。

Added in version 6.0.1.8 of package base.

parameter

(current-plumber) → plumber?

(current-plumber plumber) → void?
  plumber : plumber?
一个 parameter ,它决定了flush callbacks 的 current plumber 。 例如,创建一个输出 file stream port , 在 current plumber 中注册一个 flush callback ,只要该端口被打开,就可以冲洗该端口。

Added in version 6.0.1.8 of package base.

procedure

(plumber-flush-all plumber) → void?

  plumber : plumber?
调用所有在 plumber 那里注册的 flush callbacks 。

要调用的 flush callbacks 是在调用第一个回调之前从 plumber 那里收集的。 如果一个 flush callback 注册了一个新的 flush callback ,新的回调就 不 会被调用。 如果一个 flush callback 引发了一个异常或者其他情况下的逃逸, 那么其余的 flush callbacks 就不会被调用。

Added in version 6.0.1.8 of package base.

procedure

(plumber-flush-handle? v) → boolean?

  v : any/c
如果 v 是一个 flush handle,代表注册了一个 flush callback, 返回 #t ,否则返回 #f 。

Added in version 6.0.1.8 of package base.

procedure

(plumber-add-flush! plumber proc [weak?]) → plumber-flush-handle?

  plumber : plumber?
  proc : (plumber-flush-handle? . -> . any)
  weak? : any/c = #f
将 proc 注册为 plumber 的 flush callback , 因此当 plumber-flush-all 应用于 plumber 时,会调用 proc 。

flush handle 结果代表回调的注册,可与 plumber-flush-handle-remove! 一起使用,以取消回调的注册。

给定的 proc 可以从 flush handle 中获得,但如果 weak? 为真, 则 plumber 只保留对结果 flush handle (以及 proc )的 weak reference 。

当 proc 作为 flush callback 被调用时,会将 plumber-add-flush! 返回的值传递给它, 以便 proc 能够方便地取消注册。 对 proc 的调用是在一个 continuation barrier 内进行的。

Added in version 6.0.1.8 of package base.

procedure

(plumber-flush-handle-remove! handle) → void?

  handle : plumber-flush-handle?
取消对产生 handle 的 plumber-add-flush! 调用所注册的 flush callback 的注册。

如果 handle 所代表的注册已经被删除,那么 plumber-flush-handle-remove! 就没有任何作用。

Added in version 6.0.1.8 of package base.
