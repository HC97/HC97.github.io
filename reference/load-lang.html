14.3 racket/load 语言

 #lang racket/load package: base

racket/load 语言支持评估, 模块主体中的每个顶层形式都以与 load 相同的方式分别传递给 eval 。

评价的命名空间与 racket/load 模块实例共享 module registry , 但它有一个单独的顶层环境,并且它是用 racket 的绑定初始化的。 一个命名空间是为每个 racket/load 模块的实例创建的 (也就是说,使用 racket/load 语言的多个模块共享一个命名空间)。 racket/load 库只输出 #%module-begin 和 #%top-interaction 形式,有效地交换了评价命名空间并调用 eval 。

例如,使用 racket/load 的模块主体可以包括 module 形式, 所以运行下面的模块会打印出 5:

#lang racket/load
 
(module m racket/base
  (provide x)
  (define x 5))
 
(module n racket/base
  (require 'm)
  (display x))
 
(require 'n)

使用 racket/load 的模块中的定义在当前命名空间中被评估, 这意味着 load 和 eval 可以看到这些定义。 例如,运行下面的模块会打印出 6:

#lang racket/load
 
(define x 6)
(display (eval 'x))

由于 racket/load 模块中的所有形式都是在顶层评估的,所以不能用 provide 从模块中导出绑定。 同样,由于模块主体形式的评估是固有的动态的,模块的编译基本上没有任何好处。 由于这些原因,使用 racket/load 只能对顶层形式进行交互式探索,而不能用于构建更大的程序。
