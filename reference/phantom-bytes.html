16.5 幻影字节串

phantom byte string 是一个小的 Racket 值,它被 Racket 内存管理器视为具有任意大小, 在创建 phantom byte string 时或通过 set-phantom-bytes! 改变它时指定。

phantom byte string 是对 Racket 内存管理器的一个提示, 即内存是在进程中分配的,但通过一个单独的分配器, 例如通过 ffi/unsafe 访问的外国库。 这个提示被用来触发 garbage collections 或计算 current-memory-use 的结果。

procedure

(phantom-bytes? v) → boolean?

  v : any/c
如果 v 是一个 phantom byte string,返回 #t ,否则返回 #f 。

procedure

(make-phantom-bytes k) → phantom-bytes?

  k : exact-nonnegative-integer?
创建一个 phantom byte string ,Racket 内存管理器将其视为 k 字节大小。 对于足够大的 k, exn:fail:out-of-memory exception is raised —会被引发–要么是因为大小大得难以置信, 要么是因为用 custodian-limit-memory 安装了一个内存限制。

procedure

(set-phantom-bytes! phantom-bstr k) → phantom-bytes?

  phantom-bstr : phantom-bytes?
  k : exact-nonnegative-integer?
调整 phantom byte string 的大小,因为它被 Racket 内存管理器所处理。

例如,如果 phantom-bstr 所代表的内存通过国外的库被释放, 那么 (set-phantom-bytes! phantom-bstr 0) 可以反映内存使用的变化。

当 k 大于 phantom-bstr 的当前大小时, 那么这个函数可以引发 exn:fail:out-of-memory ,就像 make-phantom-bytes 。
