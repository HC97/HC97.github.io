6.4 字段和方法访问

在类定义的表达式中,类的初始化变量、字段和方法都是环境的一部分。 在一个方法体中,只有类的字段和其他方法可以被引用;对任何其他类引入的标识符的引用是一个语法错误。 在类的其他地方,所有类引入的标识符都是可用的,并且字段和初始化变量可以用 set! 。

6.4.1 方法

在一个类中使用的方法名只能在应用表达式的过程位置使用,任何其他的使用都是语法错误。

为了允许方法被应用到参数列表中,方法应用可以有以下形式:

(method-id arg ... . arg-list-expr)

这种形式以类似于 (apply method-id arg ... arg-list-expr) 的方式调用该方法。 arg-list-expr 不能是一个括号内的表达式。

方法从类外以 send, send/apply 和 send/keyword-apply 形式被调用。

syntax

(send obj-expr method-id arg ...)

(send obj-expr method-id arg ... . arg-list-expr)
评估 obj-expr 以获得一个对象,并在该对象上调用具有(外部)名称 method-id 的方法, 提供 arg 结果作为参数。 每个 arg 和 #%app 一样: 要么是 arg-expr ,要么是 keyword arg-expr 。 在第二种形式中, arg-list-expr 不能是一个括号内的表达式。

如果 obj-expr 没有产生一个对象,就会引发 exn:fail:contract exception is raised 。 如果该对象没有名为 method-id 的公共方法,则会引发 exn:fail:object exception is raised 。

syntax

(send/apply obj-expr method-id arg ... arg-list-expr)

与 send 的加点形式一样,但 arg-list-expr 可以是任何表达式。

syntax

(send/keyword-apply obj-expr method-id
                    keyword-list-expr value-list-expr
                    arg ... arg-list-expr)
像 send/apply 一样,但像 keyword-apply 一样用表达式来表示关键词和参数列表。

procedure

(dynamic-send obj    
  method-name    
  v ...    
  #:<kw> kw-arg ...) → any
  obj : object?
  method-name : symbol?
  v : any/c
  kw-arg : any/c
在 obj 上调用名称与 method-name 匹配的方法, 并传递所有给定的 v 和 kw-arg 。

syntax

(send* obj-expr msg ...+)

 
msg = (method-id arg ...)
  | (method-id arg ... . arg-list-expr)
调用同一对象的多个方法(按顺序)。每个 msg 对应于 send 的一个用法。

例如,

(send* edit (begin-edit-sequence)
            (insert "Hello")
            (insert #\newline)
            (end-edit-sequence))

与以下情况相同

(let ([o edit])
  (send o begin-edit-sequence)
  (send o insert "Hello")
  (send o insert #\newline)
  (send o end-edit-sequence))

syntax

(send+ obj-expr msg ...)

 
msg = (method-id arg ...)
  | (method-id arg ... . arg-list-expr)
从 obj-expr 产生的对象开始调用方法(按顺序)。 每个方法调用将在最后一个方法调用的结果上被调用,这个结果预计是一个对象。 每个 msg 对应于 send 的使用。

这就是 send* 的功能类似物。

Examples:
(define point%
  (class object%
    (super-new)
    (init-field [x 0] [y 0])
    (define/public (move-x dx)
      (new this% [x (+ x dx)] [y y]))
    (define/public (move-y dy)
      (new this% [y (+ y dy)] [x x]))
    (define/public (get-pair)
      (cons x y))))

 

> (send+ (new point%)
         (move-x 5)
         (move-y 7)
         (move-x 12)
         (get-pair))

'(17 . 7)

syntax

(with-method ([id (obj-expr method-id)] ...)
  body ...+)
从一个对象中提取方法,并为每个方法绑定一个可以直接应用的本地名称(与类中声明的方法的方式相同)。 每个 obj-expr 必须产生一个对象,该对象必须有一个由相应的 method-id 命名的公共方法。 相应的 id 被绑定,这样它就可以被直接应用(见 方法) 。

例子:

(let ([s (new stack%)])
  (with-method ([push (s push!)]
                [pop (s pop!)])
    (push 10)
    (push 9)
    (pop)))

等于

(let ([s (new stack%)])
  (send s push! 10)
  (send s push! 9)
  (send s pop!))
6.4.2 字段

syntax

(get-field id obj-expr)

从 obj-expr 的值中提取具有(外部)名称 id 的字段。

如果 obj-expr 没有产生一个对象,就会引发 exn:fail:contract exception is raised 。 如果该对象没有 id 字段,则引发 exn:fail:object exception is raised 。

procedure

(dynamic-get-field field-name obj) → any/c

  field-name : symbol?
  obj : object?
从 obj 中提取与 field-name 匹配的(外部)字段。 如果该对象没有与 field-name 匹配的字段,则引发 exn:fail:object exception is raised 。

syntax

(set-field! id obj-expr expr)

将具有(外部)名称 id 的字段从 obj-expr 的值设置为 expr 的值。

如果 obj-expr 没有产生一个对象,就会引发 exn:fail:contract exception is raised 。 如果该对象没有 id 字段,则引发 exn:fail:object exception is raised 。

procedure

(dynamic-set-field! field-name obj v) → void?

  field-name : symbol?
  obj : object?
  v : any/c
将 obj 中与 field-name 匹配的(外部)字段设置为 v 。 如果对象没有与 field-name 匹配的字段,就会引发 exn:fail:object exception is raised 。

syntax

(field-bound? id obj-expr)

如果 obj-expr 的对象结果有一个(外部)名称为 id 的字段,产生 #t , 否则产生 #f 。

如果 obj-expr 没有产生一个对象,就会引发 exn:fail:contract exception is raised 。

syntax

(class-field-accessor class-expr field-id)

返回一个访问过程,该访问过程接收由 class-expr 产生的类的实例, 并返回对象的字段(外部)名称 field-id 的值。

如果 class-expr 没有产生一个类,就会引发 exn:fail:contract exception is raised 。 如果该类没有 field-id 字段,则会引发 exn:fail:object exception is raised 。

syntax

(class-field-mutator class-expr field-id)

返回一个突变过程,该过程接收一个由 class-expr 产生的类的实例和一个值, 并将该对象的字段(外部)名称 field-id 的值设置为给定的值。其结果是 #<void> 。

如果 class-expr 没有产生一个类,就会引发 exn:fail:contract exception is raised 。 如果该类没有 field-id 字段,就会引发 exn:fail:object exception is raised 。

6.4.3 泛型

可以用一个 generic 来代替方法名,以避免在类中按名称重新定位方法的成本。

syntax

(generic class-or-interface-expr id)

产生一个泛型,该泛型对由 class-or-interface-expr 产生的类或接口的实例 (或由 class-or-interface 派生的类/接口的实例)起作用,调用具有(外部) id 名称的方法。

如果 class-or-interface-expr 没有产生一个类或接口, 则引发 exn:fail:contract exception is raised。 如果产生的类或接口不包含名为 id 的方法,就会引发 exn:fail:object exception is raised 。

syntax

(send-generic obj-expr generic-expr arg ...)

(send-generic obj-expr generic-expr arg ... . arg-list-expr)
调用由 obj-expr 产生的对象的一个方法,该方法由 generic-expr 产生的泛型指示。 每个 arg 和 #%app 一样:要么是 arg-expr ,要么是 keyword arg-expr 。 第二种形式类似于用 apply 调用过程,其中 arg-list-expr 不是一个括号内的表达式。

如果 obj-expr 没有产生一个对象,或者如果 generic-expr 没有产生一个泛型, 就会引发 exn:fail:contract exception is raised 。 如果 obj-expr 的结果不是由 generic-expr 的结果所封装的类或接口的实例, 就会引发 exn:fail:object exception is raised 。

procedure

(make-generic type method-name) → generic?

  type : (or/c class? interface?)
  method-name : symbol?
与 generic 形式类似,但作为一个过程,接受一个符号化的方法名。
