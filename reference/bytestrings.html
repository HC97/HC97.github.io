4.5 字节串

字节（Byte）和字节字符串（Byte String） in Racket 指南 introduces byte strings.

byte string 是一个固定长度的字节数组。 一个 byte 是 0 到 255 之间的一个精确的整数。

字节串可以是 mutable 或 immutable 。 当一个不可变的字节串被提供给像 bytes-set! 这样的过程时, 会引发 exn:fail:contract exception is raised 。 由默认阅读器(参见 读取字符串)生成的字节串常量是不可变的, 它们在 read-syntax 模式下被 interned 。 使用 immutable? 来检查一个字节字符串是否是不可变的。

当两个字节串具有相同的长度并包含相同的字节序列时,它们是 equal? 。

字节串可以作为一个单值序列使用(参见 Sequences) 。 字符串的字节作为序列的元素。 也请参见 in-bytes 。

See 读取字符串 for information on reading byte strings and 打印字符串 for information on printing byte strings.

参见: immutable? 。

4.5.1 字节串构造器、选择器和突变器

procedure

(bytes? v) → boolean?

  v : any/c
如果 v 是一个字节字符串, 返回 #t ,否则返回 #f 。

参见 immutable-bytes? 和 mutable-bytes? 。

Examples:
> (bytes? #"Apple")

#t

> (bytes? "Apple")

#f

procedure

(make-bytes k [b]) → bytes?

  k : exact-nonnegative-integer?
  b : byte? = 0
返回一个新的长度为 k 的可变字节串,其中字节串的每个位置都被初始化为字节 b

Example:
> (make-bytes 5 65)

#"AAAAA"

procedure

(bytes b ...) → bytes?

  b : byte?
返回一个新的可改变的字节字符串, 其长度为所提供的 b 的数量,其位置以给定的 b 初始化。

Example:
> (bytes 65 112 112 108 101)

#"Apple"

procedure

(bytes->immutable-bytes bstr) → (and/c bytes? immutable?)

  bstr : bytes?
返回一个与 bstr 内容相同的不可变的字节字符串, 如果 bstr 是不可变的,则返回 bstr 本身。

Examples:
> (bytes->immutable-bytes (bytes 65 65 65))

#"AAA"

> (define b (bytes->immutable-bytes (make-bytes 5 65)))
> (bytes->immutable-bytes b)

#"AAAAA"

> (eq? (bytes->immutable-bytes b) b)

#t

procedure

(byte? v) → boolean?

  v : any/c
如果 v 是一个字节 (即 0 到 255 之间的精确整数), 返回 #t ,否则返回 #f 。

Examples:
> (byte? 65)

#t

> (byte? 0)

#t

> (byte? 256)

#f

> (byte? -1)

#f

procedure

(bytes-length bstr) → exact-nonnegative-integer?

  bstr : bytes?
返回 bstr 的长度。

Example:
> (bytes-length #"Apple")

5

procedure

(bytes-ref bstr k) → byte?

  bstr : bytes?
  k : exact-nonnegative-integer?
返回 bstr 中位置为 k 的字节。 字节中的第一个位置对应于 0 ,所以位置 k 必须小于字节的长度, 否则会引发 exn:fail:contract exception is raised 。

Example:
> (bytes-ref #"Apple" 0)

65

procedure

(bytes-set! bstr k b) → void?

  bstr : (and/c bytes? (not/c immutable?))
  k : exact-nonnegative-integer?
  b : byte?
将 bstr 中位置 k 的字节改为 b 。 字节串中的第一个位置对应于 0 ,所以位置 k 必须小于字节的长度,否则将引发 exn:fail:contract exception is raised 。

Examples:
> (define s (bytes 65 112 112 108 101))
> (bytes-set! s 4 121)
> s

#"Apply"

procedure

(subbytes bstr start [end]) → bytes?

  bstr : bytes?
  start : exact-nonnegative-integer?
  end : exact-nonnegative-integer? = (bytes-length str)
返回一个新的可变字节字符串, 该字符串的长度为 (- end start) 字节,并且包含与 bstr 相同的字节, 从 start (包括)到 end (包括)。 start 和 end 参数必须小于或等于 bstr 的长度, 而且 end 必须大于或等于 start ,否则会引发 exn:fail:contract exception is raised 。

Examples:
> (subbytes #"Apple" 1 3)

#"pp"

> (subbytes #"Apple" 1)

#"pple"

procedure

(bytes-copy bstr) → bytes?

  bstr : bytes?
返回 (subbytes str 0) 。

procedure

(bytes-copy! dest    
  dest-start    
  src    
  [src-start    
  src-end]) → void?
  dest : (and/c bytes? (not/c immutable?))
  dest-start : exact-nonnegative-integer?
  src : bytes?
  src-start : exact-nonnegative-integer? = 0
  src-end : exact-nonnegative-integer? = (bytes-length src)
改变从 dest-start 位置开始的 dest 的字节,使其与 src 中从 src-start (包括)到 src-end (不包括)的字节相匹配。 dest 和 src 的字节串可以是同一个字节串,在这种情况下,目标区域可以与源区域重叠; 复制后的目标字节与复制前的源字节相匹配。 如果 dest-start, src-start 或 src-end 中的任何一个超出了范围(考虑到字节串以及源和目的区域的大小),就会引发 exn:fail:contract exception is raised 。

Examples:
> (define s (bytes 65 112 112 108 101))
> (bytes-copy! s 4 #"y")
> (bytes-copy! s 0 s 3 4)
> s

#"lpply"

procedure

(bytes-fill! dest b) → void?

  dest : (and/c bytes? (not/c immutable?))
  b : byte?
改变 dest ,使字节中的每个位置都填上 b 。

Examples:
> (define s (bytes 65 112 112 108 101))
> (bytes-fill! s 113)
> s

#"qqqqq"

procedure

(bytes-append bstr ...) → bytes?

  bstr : bytes?
返回 一个新的可变字节字符串, 其长度与给定的 bstr 的长度之和相同,并且包含给定的 bstr 的连接字节。 如果没有提供 bstr ,结果是一个零长度的字节串。

Example:
> (bytes-append #"Apple" #"Banana")

#"AppleBanana"

procedure

(bytes->list bstr) → (listof byte?)

  bstr : bytes?
返回一个新的字节列表, 与 bstr 的内容相对应。 也就是说,列表的长度是 (bytes-length bstr) , 而 bstr 中的字节序列与结果列表中的序列相同。

Example:
> (bytes->list #"Apple")

'(65 112 112 108 101)

procedure

(list->bytes lst) → bytes?

  lst : (listof byte?)
返回一个新的可变字节字符串, 其内容是 lst 中的字节列表。 也就是说,字节串的长度是 (length lst) , 并且 lst 中的字节序列与结果字节串中的序列相同。

Example:
> (list->bytes (list 65 112 112 108 101))

#"Apple"

procedure

(make-shared-bytes k [b]) → bytes?

  k : exact-nonnegative-integer?
  b : byte? = 0
返回一个新的长度为 k 的可变字节串,其中字节串中的每个位置都被初始化为字节 b , 对于 places 之间的通信,新的字节串被分配在 shared memory space 。

Example:
> (make-shared-bytes 5 65)

#"AAAAA"

procedure

(shared-bytes b ...) → bytes?

  b : byte?
返回一个新的可变的字节串, 其长度是所提供的 b 的数量,其位置是用所给的 b 初始化的。 对于 places 之间的通信,新的字节串被分配到 shared memory space 。

Example:
> (shared-bytes 65 112 112 108 101)

#"Apple"

4.5.2 字节串比较

procedure

(bytes=? bstr1 bstr2 ...) → boolean?

  bstr1 : bytes?
  bstr2 : bytes?
如果所有的参数都是 eqv?, 返回 #t 。

Examples:
> (bytes=? #"Apple" #"apple")

#f

> (bytes=? #"a" #"as" #"a")

#f

Changed in version 7.0.0.13 of package base: Allow one argument, in addition to allowing two or more.

procedure

(bytes<? bstr1 bstr2 ...) → boolean?

  bstr1 : bytes?
  bstr2 : bytes?
如果参数是按词典排序增加的,则返回 #t ,其中单个字节按 < 排序,否则返回 #f 。

Examples:
> (bytes<? #"Apple" #"apple")

#t

> (bytes<? #"apple" #"Apple")

#f

> (bytes<? #"a" #"b" #"c")

#t

Changed in version 7.0.0.13 of package base: Allow one argument, in addition to allowing two or more.

procedure

(bytes>? bstr1 bstr2 ...) → boolean?

  bstr1 : bytes?
  bstr2 : bytes?
Like bytes<?, but checks whether the arguments are decreasing.

Examples:
> (bytes>? #"Apple" #"apple")

#f

> (bytes>? #"apple" #"Apple")

#t

> (bytes>? #"c" #"b" #"a")

#t

Changed in version 7.0.0.13 of package base: Allow one argument, in addition to allowing two or more.

4.5.3 字节到/从字符,解码和编码

procedure

(bytes->string/utf-8 bstr [err-char start end]) → string?

  bstr : bytes?
  err-char : (or/c #f char?) = #f
  start : exact-nonnegative-integer? = 0
  end : exact-nonnegative-integer? = (bytes-length bstr)
通过将 bstr 的 start 到 end 的子串解码为 Unicode 码位的 UTF-8 编码,产生一个字符串。 如果 err-char 不是 #f , 那么它将用于落在 128 到 255 范围内但不属于有效编码序列的字节。 (这个规则与从端口读取字符是一致的;更多细节见 编码和地域 。 如果 err-char 是 #f , 并且如果 bstr 的 start 到 end 的子串整体上不是有效的 UTF-8 编码, 那么会引发 exn:fail:contract exception is raised 。

Example:
> (bytes->string/utf-8 (bytes 195 167 195 176 195 182 194 163))

"çðö£"

procedure

(bytes->string/locale bstr    
  [err-char    
  start    
  end]) → string?
  bstr : bytes?
  err-char : (or/c #f char?) = #f
  start : exact-nonnegative-integer? = 0
  end : exact-nonnegative-integer? = (bytes-length bstr)
通过对 bstr 的从 start 到 end 的子串进行解码, 产生一个字符串,使用的是当前 locale 的编码(另见 编码和地域)。 如果 err-char 不是 #f ,它将用于 bstr 中不属于有效编码的每个字节; 如果 err-char 是 #f , 并且如果 bstr 的从 start 到 end 的子串总体上不是有效的编码, 那么将引发 exn:fail:contract exception is raised 。

procedure

(bytes->string/latin-1 bstr    
  [err-char    
  start    
  end]) → string?
  bstr : bytes?
  err-char : (or/c #f char?) = #f
  start : exact-nonnegative-integer? = 0
  end : exact-nonnegative-integer? = (bytes-length bstr)
通过将 bstr 的 start 到 end 的子串解码为 Unicode 码点的 Latin-1 编码, 产生一个字符串;也就是说,每个字节都直接使用 integer->char 翻译为一个字符,所以解码总是成功的。 err-char 参数被忽略,但为了与其他操作保持一致而出现。

Example:
> (bytes->string/latin-1 (bytes 254 211 209 165))

"þÓÑ¥"

procedure

(string->bytes/utf-8 str [err-byte start end]) → bytes?

  str : string?
  err-byte : (or/c #f byte?) = #f
  start : exact-nonnegative-integer? = 0
  end : exact-nonnegative-integer? = (string-length str)
通过 UTF-8 编码 str 的 start 到 end 的子串,产生一个字节字符串(总是成功的)。 err-byte 参数被忽略,但为了与其他操作保持一致,它被包括在内。

Examples:
> (define b
    (bytes->string/utf-8
     (bytes 195 167 195 176 195 182 194 163)))
> (string->bytes/utf-8 b)

#"\303\247\303\260\303\266\302\243"

> (bytes->string/utf-8 (string->bytes/utf-8 b))

"çðö£"

procedure

(string->bytes/locale str [err-byte start end]) → bytes?

  str : string?
  err-byte : (or/c #f byte?) = #f
  start : exact-nonnegative-integer? = 0
  end : exact-nonnegative-integer? = (string-length str)
通过使用当前 locale 的编码对 str 的 start 到 end 的子串进行编码, 产生一个字符串(参见 编码和地域)。 如果 err-byte 不是 #f ,则对 str 中不能被当前 locale 编码的每个字符进行编码; 如果 err-byte 是 #f , 并且 str 的 start 到 end 的子串不能被编码, 则会引发 exn:fail:contract exception is raised 。

procedure

(string->bytes/latin-1 str    
  [err-byte    
  start    
  end]) → bytes?
  str : string?
  err-byte : (or/c #f byte?) = #f
  start : exact-nonnegative-integer? = 0
  end : exact-nonnegative-integer? = (string-length str)
通过使用 Latin-1 对 str 的 start 到 end 的子串进行编码,产生一个字符串; 即使用 char->integer 将每个字符直接转换为一个字节。 如果 err-byte 不是 #f ,它将用于 str 中每个数值大于 255 的字符。 如果 err-byte 是 #f, 并且如果 str 的从 start 到 end 的子串有一个值大于 255 的字符, 那么就会引发 exn:fail:contract exception is raised 。

Examples:
> (define b
    (bytes->string/latin-1 (bytes 254 211 209 165)))
> (string->bytes/latin-1 b)

#"\376\323\321\245"

> (bytes->string/latin-1 (string->bytes/latin-1 b))

"þÓÑ¥"

procedure

(string-utf-8-length str [start end]) → exact-nonnegative-integer?

  str : string?
  start : exact-nonnegative-integer? = 0
  end : exact-nonnegative-integer? = (string-length str)
返回 str 的子串从 start 到 end 的 UTF-8 编码的字节长度,但不需要实际生成编码的字节。

Examples:
> (string-utf-8-length
    (bytes->string/utf-8 (bytes 195 167 195 176 195 182 194 163)))

8

> (string-utf-8-length "hello")

5

procedure

(bytes-utf-8-length bstr [err-char start end])

 → (or/c exact-nonnegative-integer? #f)
  bstr : bytes?
  err-char : (or/c #f char?) = #f
  start : exact-nonnegative-integer? = 0
  end : exact-nonnegative-integer? = (bytes-length bstr)
返回 bstr 的子串从 start 到 end 的 UTF-8 解码的字符长度,但没有实际生成解码的字符。 如果 err-char 是 #f ,并且子串总体上不是 UTF-8 编码,结果就是 #f 。 否则, err-char 用于解决解码错误,就像在 bytes->string/utf-8 中一样。

Examples:
> (bytes-utf-8-length (bytes 195 167 195 176 195 182 194 163))

4

> (bytes-utf-8-length (make-bytes 5 65))

5

procedure

(bytes-utf-8-ref bstr [skip err-char start end]) → (or/c char? #f)

  bstr : bytes?
  skip : exact-nonnegative-integer? = 0
  err-char : (or/c #f char?) = #f
  start : exact-nonnegative-integer? = 0
  end : exact-nonnegative-integer? = (bytes-length bstr)
返回 bstr 的子串从 start 到 end 的 UTF-8 解码中的 skip 字符, 但是不需要实际生成其他的解码字符。 如果子串不是 UTF-8 编码,直到 skip 字符为止(当 err-char 为 #f 时), 或者如果子串解码产生的字符少于 skip 字符,结果为 #f 。 如果 err-char 不是 #f , 那么它就会像 bytes->string/utf-8 那样用来解决解码错误。

Examples:
> (bytes-utf-8-ref (bytes 195 167 195 176 195 182 194 163) 0)

#\ç

> (bytes-utf-8-ref (bytes 195 167 195 176 195 182 194 163) 1)

#\ð

> (bytes-utf-8-ref (bytes 195 167 195 176 195 182 194 163) 2)

#\ö

> (bytes-utf-8-ref (bytes 65 66 67 68) 0)

#\A

> (bytes-utf-8-ref (bytes 65 66 67 68) 1)

#\B

> (bytes-utf-8-ref (bytes 65 66 67 68) 2)

#\C

procedure

(bytes-utf-8-index bstr 
  skip 
  [err-char 
  start 
  end]) 
 → (or/c exact-nonnegative-integer? #f)
  bstr : bytes?
  skip : exact-nonnegative-integer?
  err-char : (or/c #f char?) = #f
  start : exact-nonnegative-integer? = 0
  end : exact-nonnegative-integer? = (bytes-length bstr)
返回 bstr 中的偏移量(以字节为单位),在对 bstr 的子串进行 UTF-8 解码时, skip 字符的编码从 start 到 end 开始(但没有实际生成其他解码的字符)。 其结果是相对于 bstr 的开始,而不是相对于 start 。 如果子串不是 UTF-8 编码,直到 skip 字符为止(当 err-char 为 #f 时), 或者如果子串解码产生的字符少于 skip 字符,结果为 #f 。 如果 err-char 不是 #f , 那么它就会像 bytes->string/utf-8 那样用来解决解码错误。

Examples:
> (bytes-utf-8-index (bytes 195 167 195 176 195 182 194 163) 0)

0

> (bytes-utf-8-index (bytes 195 167 195 176 195 182 194 163) 1)

2

> (bytes-utf-8-index (bytes 195 167 195 176 195 182 194 163) 2)

4

> (bytes-utf-8-index (bytes 65 66 67 68) 0)

0

> (bytes-utf-8-index (bytes 65 66 67 68) 1)

1

> (bytes-utf-8-index (bytes 65 66 67 68) 2)

2

4.5.4 字节到字节的编码转换

procedure

(bytes-open-converter from-name to-name)

 → (or/c bytes-converter? #f)
  from-name : string?
  to-name : string?
生成一个 byte converter , 从 from-name 命名的编码到 to-name 命名的编码。 如果请求的转换对不可用,则返回 #f 而不是转换器。

某些编码组合总是可用的:

  • (bytes-open-converter "UTF-8" "UTF-8") — 身份转换, 除了输入中的编码错误会导致解码失败。

  • (bytes-open-converter "UTF-8-permissive" "UTF-8") — 身份转换,除了任何不属于有效编码序列的输入字节会被 #\uFFFD 的 UTF-8 编码序列有效取代。 (这种对无效序列的处理与将端口字节流解释为字符是一致的;参见 端口 。)

  • (bytes-open-converter "" "UTF-8") — 从当前 locale 的默认编码 (参见 编码和地域) 转换为UTF-8。

  • (bytes-open-converter "UTF-8" "") — 从 UTF-8 转换为当前 locale 的默认编码(参见 编码和地域)。

  • (bytes-open-converter "platform-UTF-8" "platform-UTF-16") — 在 Unix and Mac OS 上将 UTF-8 转换为 UTF-16, 其中每个 UTF-16 代码单元是由两个字节组成的序列,按当前平台的字节数排序。 在 Windows 上,转换方式与 (bytes-open-converter "WTF-8" "WTF-16") 相同, 以支持未配对的代用代码单元。

  • (bytes-open-converter "platform-UTF-8-permissive" "platform-UTF-16") — 与 (bytes-open-converter "platform-UTF-8" "platform-UTF-16") 一样, 但不属于有效 UTF-8 编码序列的输入字节(或对 Windows 上的未配对代用扩展有效) 会被有效替换成 #\uFFFD 。

  • (bytes-open-converter "platform-UTF-16" "platform-UTF-8") — 在 Unix and Mac OS 上将 UTF-16 (按当前平台的字节数排序)转换为UTF-8。 在 Windows 上,转换方式与 (bytes-open-converter "WTF-16" "WTF-8") 相同,以支持未配对的代用词。 在 Unix and Mac OS 上,代用符被假定为成对的:一对带有 #xD800 位的字节开始一个代用符对, #x03FF 位从该对和后面的对中使用(与 #xDC00 位的值无关)。 在所有平台上,当从输入字节串内的奇数偏移量进行解码时,性能可能很差。

  • (bytes-open-converter "WTF-8" "WTF-16") — 将 WTF-8 [Sapin18] UTF-8 的超集转换为 UTF-16 的超集,以支持非成对的代用代码单元, 其中每个 UTF-16 代码单元是由两个字节组成的序列,按当前平台的字节数排序。

  • (bytes-open-converter "WTF-8-permissive" "WTF-16") — 类似于 (bytes-open-converter "WTF-8" "WTF-16"), 但不属于有效的 WTF-8 编码序列的输入字节实际上被替换成 #\uFFFD 。

  • (bytes-open-converter "WTF-16" "WTF-8") — 将 UTF-16 的 WTF-16 [Sapin18] 超集转换为 UTF-8 的 WTF-8 超集。 输入可以包括 UTF-16 代码单元,这些代码单元是未配对的代理, 相应的输出包括每个代理在 UTF-8 的自然扩展中的编码。

一个新打开的字节转换器被注册在当前的托管人那里(参见 Custodians), 这样当托管人关闭时,转换器就被关闭了。如果一个转换器是 Unix 上不涉及 "" 的保证组合之一, 或者是 Windows 和 Mac OS 上的任何保证组合(包括 "") ,那么它就不会在托管人那里注册(也不需要被关闭)。

在 Windows 的 Racket 软件发行版中,一个合适的 "iconv.dll" 被包含在 "libmzschVERS.dll" 中。

可用的编码和组合因平台而异,取决于所安装的 iconv 库; from-name 和 to-name 参数会传递给 iconv_open。 在 Windows 上, "iconv.dll" 或 "libiconv.dll" 必须与 "libmzschVERS.dll" (其中 VERS 是一个版本号)在同一目录中, 在用户的路径中,在系统目录中,或在运行时在当前可执行文件的目录中, 而且 DLL 必须提供 _errno 或链接到 "msvcrt.dll" 以获得 _errno; 否则,只有保证的组合可用。

使用 bytes-convert 与结果来转换字节字符串。

Changed in version 7.9.0.17 of package base: 增加了 "WTF-8", "WTF-8-permissive" 和 "WTF-16" 的内置转换器。

procedure

(bytes-close-converter converter) → void

关闭给定的转换器,这样它就不能再与 bytes-convert 或 bytes-convert-end 一起使用。

procedure

(bytes-convert converter 
  src-bstr 
  [src-start-pos 
  src-end-pos 
  dest-bstr 
  dest-start-pos 
  dest-end-pos]) 
 → 
(or/c bytes? exact-nonnegative-integer?)
exact-nonnegative-integer?
(or/c 'complete 'continues 'aborts 'error)
  converter : bytes-converter?
  src-bstr : bytes?
  src-start-pos : exact-nonnegative-integer? = 0
  src-end-pos : exact-nonnegative-integer?
   = (bytes-length src-bstr)
  dest-bstr : (or/c bytes? #f) = #f
  dest-start-pos : exact-nonnegative-integer? = 0
  dest-end-pos : (or/c exact-nonnegative-integer? #f)
   = 
(and dest-bstr
     (bytes-length dest-bstr))
在 src-bstr 中转换从 src-start-pos 到 src-end-pos 的字节。

如果 dest-bstr 不是 #f ,则将转换后的字节从 dest-start-pos 到 dest-end-pos 写进 dest-bstr 。 如果 dest-bstr 是 #f,那么一个新分配的字节串将保存转换结果, 如果 dest-end-pos 不是 #f, 那么结果字节串的大小将不超过 (- dest-end-pos dest-start-pos)。

bytes-convert 的结果是三个值:

  • result-bstr 或 dest-wrote-amt — 如果 dest-bstr 是 #f 或者没有提供,则是一个字节串,否则就是写进 dest-bstr 的字节数。

  • src-read-amt — 从 src-bstr 成功转换的字节数。

  • 'complete, 'continues, 'aborts 或 'error — 表示转换的终止情况:

    • 'complete: 整个输入被处理, src-read-amt 将等于 (- src-end-pos src-start-pos) 。

    • 'continues: 由于结果大小或 dest-bstr 中的空间限制,转换停止; 在这种情况下,如果需要更多的空间来处理 src-bstr 中的下一个完整的编码序列, 那么可能会返回少于 (- dest-end-pos dest-start-pos) 字节的数据。

    • 'aborts: 输入在编码序列的中途停止,需要更多的输入字节来继续。 例如,如果 "UTF-8-permissive" 解码的最后一个输入字节是 195 , 结果是 'aborts ,因为需要另一个字节来确定如何使用 195 字节。

    • 'error: src-bstr 中从 (+ src-start-pos src-read-amt) 字节开始的字节没有形成合法的编码序列。 对于一些编码,即所有的字节序列都是有效的编码,绝不会产生这种结果。 例如,由于 "UTF-8-permissive" 通过删除字符或生成 "?" 来处理无效的 UTF-8 序列, 所以每个字节序列都是有效的。

应用一个转换器会在转换器中积累状态(即使 bytes-convert 的第三个结果是 'complete) 。 这种状态可以影响输入的进一步处理和输出的进一步生成,但只适用于涉及 "移位序列" 的转换,以改变一个流中的模式。 要终止一个输入序列并重置转换器,请使用 bytes-convert-end 。

Examples:
> (define convert (bytes-open-converter "UTF-8" "UTF-16"))
> (bytes-convert convert (bytes 65 66 67 68))

#"\376\377\0A\0B\0C\0D"

4

'complete

> (bytes 195 167 195 176 195 182 194 163)

#"\303\247\303\260\303\266\302\243"

> (bytes-convert convert (bytes 195 167 195 176 195 182 194 163))

#"\0\347\0\360\0\366\0\243"

8

'complete

> (bytes-close-converter convert)

(bytes-convert-end converter 
  [dest-bstr 
  dest-start-pos

(or/c 'complete 'continues)

  dest-bstr : (or/c bytes? #f) = #f

与 bytes-convert 类似,但这个程序不是转换字节, 而是为转换生成一个结束序列(有时称为 "移位序列"),如果有的话。 很少有编码使用移位序列,所以对于大多数编码来说,这个函数会成功,但没有输出。 在任何情况下,成功输出一个(可能是空的)移位序列会将转换器重置为初始状态。

bytes-convert-end 的结果是两个值:

  • result-bstr 或 dest-wrote-amt — 如果 dest-bstr 是 #f 或没有提供,则是一个字节字符串, 否则就是写进 dest-bstr 的字节数。

  • 'complete 或 'continues — 表示转换是否完成。 如果 'complete ,那么就产生了一个完整的结束序列。 如果 'continues, 那么由于结果大小的限制或 dest-bstr 中的空间, 转换无法完成,第一个结果是一个空字节串或 0 。

(bytes-converter? v) → boolean?

4.5.5 额外的字节字符串函数

 (require racket/bytes) package: base
The bindings documented in this section are provided by the racket/bytes and racket libraries, but not racket/base.

procedure

(bytes-append* str ... strs) → bytes?

  str : bytes?
  strs : (listof bytes?)
和 bytes-append 一样,但最后一个参数被用作 bytes-append 的参数列表, 所以 (bytes-append* str ... strs) 和 (apply bytes-append str ... strs) 一样。 换句话说, bytes-append 和 bytes-append* 的关系类似于 list 和 list* 。

Examples:
> (bytes-append* #"a" #"b" '(#"c" #"d"))

#"abcd"

> (bytes-append* (cdr (append* (map (lambda (x) (list #", " x))
                                     '(#"Alpha" #"Beta" #"Gamma")))))

#"Alpha, Beta, Gamma"

procedure

(bytes-join strs sep) → bytes?

  strs : (listof bytes?)
  sep : bytes?
添加 strs 中的字节字符串,在 strs 中每一对字节之间插入 sep 。

Example:
> (bytes-join '(#"one" #"two" #"three" #"four") #" potato ")

#"one potato two potato three potato four"
