13.7.1 读取表

中的调度表 定界符和调度 与默认的 readtable 相对应。 通过创建一个新的可读表并通过 current-readtable 参数来安装它,阅读器的行为可以被扩展。

读取器会在特定的时间内查询一个可读表:

  • 当寻找一个数据集的开始时;

  • 当决定如何解析一个以 # 开头的数据集时;

  • 当寻找一个分隔符来结束一个符号或数字的时候;

  • 当寻找一个开端(如 (), 接近 )), 或 . 后的第一个字符被解析为一对, 列表,向量,或哈希表的序列;或

  • 当在一个指定长度为 ‹n› 的向量中寻找 #‹n› 之后的开启器时。

在其他时候,读表会被忽略。特别是,在解析了一个被映射为默认行为的字符 ; 后, 可读表被忽略,直到发现注释的结束换行。 同样地,在发现结尾的双引号之前,readtable 也不会影响字符串的解析。 同时,如果一个字符被映射到 ( 的默认行为,那么它开始一个序列, 该序列被任何被映射到结尾小括号的字符关闭 ) 。 一个明显的例外是, | 的默认解析对一个符号进行引号, 直到找到一个匹配的字符,但解析器只是使用开始引号的字符;它并没有查阅读表。

在许多情况下, #f 标识的是默认的可读表。 特别是, #f 是 current-readtable 参数的初始值, 这将导致读取器的行为如 阅读器 所述。

procedure

(readtable? v) → boolean?

  v : any/c
如果 v 是一个可读表,返回 #t ,否则返回 #f 。

procedure

(make-readtable readtable key mode action ...) → readtable?

  readtable : (or/c readtable? #f)
  key : (or/c char? #f)
  mode : 
(or/c 'terminating-macro
      'non-terminating-macro
      'dispatch-macro
      char?)
  action : 
(or/c procedure?
      readtable?
      #f)
创建一个新的可读表,它和 readtable 一样(可以是 #f ,表示默认的可读表), 只是根据给定的 mode 和 action,对每个 key 修改阅读器的行为。 make-readtable 的 ... 适用于 key, mode 和 action 中的所有三个;换句话说, make-readtable 的参数总数必须是 1 模 3 。

key, mode 和 action 的可能组合如下:

  • char 'terminating-macro proc — 导致 char 被解析为分隔符,输入字符串中的一个未加引号/未加注释的 char 会触发对 reader macro proc 的调用; 下面将进一步描述 proc 的活动。 从概念上讲,像 ;, ( 和 ) 这样的字符在默认的读表中被映射为终止的读者宏。

  • char 'non-terminating-macro proc — 与 'terminating-macro 类似,但 char 不被视为分隔符,因此它可以用在标识符或数字的中间。 从概念上讲, # 在默认的读表中被映射为一个非终止的宏。

  • char 'dispatch-macro proc — 与 'non-terminating-macro 类似, 但只适用于 char 后面的 # (或者更准确地说,当字符后面有一个已经被映射到默认读表中的 # 行为时)。

  • char like-char readtable — 导致 char 的解析方式与 like-char 在 readtable 中的解析方式相同, 其中 readtable 可以是 #f ,表示默认的 readtable。 ( char 的映射在 # 之后并不适用,它是通过 'dispatch-macro 单独配置的)。 在默认的阅读器中,将一个字符映射到与 | 相同的动作, 意味着该字符开始为符号加引号,并且相同的字符终止加引号; 相反,将一个字符映射到与 " 相同的动作,意味着该字符开始一个字符串, 但是该字符串仍然以结束符 " 终止。 最后,将一个字符映射到默认读取表中的一个动作,意味着该字符的行为对影响原始字符的参数很敏感; 例如,将一个字符映射到默认读取表中的大括号 { 的同一个动作, 意味着当 read-curly-brace-as-paren 参数设置为 #f 时,该字符是不允许的。

  • #f 'non-terminating-macro proc — 替换用于解析没有特定映射的字符的宏: 即可以用默认读表开始一个符号或数字的字符(除 # 或 | 外)。

如果为一个 char 提供了多个 'dispatch-macro 映射,那么除了最后一个,其他的都会被忽略。 同样,如果为一个 char 提供了多个非 'dispatch-macro , 那么除了最后一个之外,其他的都被忽略。

读者宏 proc 必须接受六个参数,它可以选择接受两个参数。 前两个参数始终是触发阅读器宏的字符和用于读取的输入端口。 当读取器宏由 read-syntax(或 read-syntax/recursive)触发时, 该过程会被传递四个额外的参数,这些参数代表已经消耗的字符的源位置:源名称,行号或 #f, 列号或 #f, 以及位置或 #f 。 当读取器宏由 read (或 read/recursive) 触发时, 如果程序接受两个参数,则只传递两个参数,否则就传递六个参数,其中第三个参数总是 #f 。 关于该过程的结果,请参见 读者扩展程序 。

读取器宏通常从给定的输入端口读取字符,产生一个值,作为所消耗的字符的 "读取器宏扩展"。 读取器宏可能会产生一个特殊注释值(见 特殊注释) ,以使所消耗的字符被视为空白, 它可能使用 read/recursive 或 read-syntax/recursive 。

procedure

(readtable-mapping readtable char)

 → 
(or/c char?
      'terminating-macro
      'non-terminating-macro)
(or/c #f procedure?)
(or/c #f procedure?)
  readtable : readtable?
  char : char?
产生关于 char 的 readtable 中的映射的信息。其结果是三个值:

  • 要么是一个字符(映射到与默认读表中的字符相同的行为), 'terminating-macro, 或 'non-terminating-macro; 这个结果报告 char 的主要(即非 'dispatch-macro) 映射。 当结果是一个字符时,那么 char 被映射到与默认读表中返回的字符相同的行为。

  • 要么是 #f ,要么是一个读者-宏程序; 当第一个结果是 'terminating-macro 或 'non-terminating-macro 时,结果是一个程序。

  • 无论是 #f 是读者-宏程序;当字符在 readtable 中具有 'dispatch-macro 时, 其结果是一个程序,以覆盖默认的调度行为。

注意,默认读表的读者-宏程序是不能直接访问的。 要调用默认行为,请使用 read/recursive 或 read-syntax/recursive 与一个字符和 #f readtable 。

Examples:
; Provides raise-read-error and raise-read-eof-error
> (require syntax/readerr)
> (define (skip-whitespace port)
    ; Skips whitespace characters, sensitive to the current
    ; readtable's definition of whitespace
    (let ([ch (peek-char port)])
      (unless (eof-object? ch)
        ; Consult current readtable:
        (let-values ([(like-ch/sym proc dispatch-proc)
                      (readtable-mapping (current-readtable) ch)])
          ; If like-ch/sym is whitespace, then ch is whitespace
          (when (and (char? like-ch/sym)
                     (char-whitespace? like-ch/sym))
            (read-char port)
            (skip-whitespace port))))))
> (define (skip-comments read-one port src)
    ; Recursive read, but skip comments and detect EOF
    (let loop ()
      (let ([v (read-one)])
        (cond
         [(special-comment? v) (loop)]
         [(eof-object? v)
          (let-values ([(l c p) (port-next-location port)])
            (raise-read-eof-error
             "unexpected EOF in tuple" src l c p 1))]
         [else v]))))
> (define (parse port read-one src)
    ; First, check for empty tuple
    (skip-whitespace port)
    (if (eq? #\> (peek-char port))
        null
        (let ([elem (read-one)])
          (if (special-comment? elem)
              ; Found a comment, so look for > again
              (parse port read-one src)
              ; Non-empty tuple:
              (cons elem
                    (parse-nonempty port read-one src))))))
> (define (parse-nonempty port read-one src)
    ; Need a comma or closer
    (skip-whitespace port)
    (case (peek-char port)
      [(#\>) (read-char port)
       ; Done
       null]
      [(#\,) (read-char port)
       ; Read next element and recur
       (cons (skip-comments read-one port src)
             (parse-nonempty port read-one src))]
      [else
       ; Either a comment or an error; grab location (in case
       ; of error) and read recursively to detect comments
       (let-values ([(l c p) (port-next-location port)]
                    [(v) (read-one)])
         (cond
          [(special-comment? v)
           ; It was a comment, so try again
           (parse-nonempty port read-one src)]
          [else
           ; Wasn't a comment, comma, or closer; error
           ((if (eof-object? v)
                raise-read-eof-error
                raise-read-error)
            "expected `,` or `>`" src l c p 1)]))]))
> (define (make-delims-table)
    ; Table to use for recursive reads to disallow delimiters
    ;  (except those in sub-expressions)
    (letrec ([misplaced-delimiter
              (case-lambda
               [(ch port) (misplaced-delimiter ch port #f #f #f #f)]
               [(ch port src line col pos)
                (raise-read-error
                 (format "misplaced `~a` in tuple" ch)
                 src line col pos 1)])])
      (make-readtable (current-readtable)
                      #\, 'terminating-macro misplaced-delimiter
                      #\> 'terminating-macro misplaced-delimiter)))
> (define (wrap l)
    `(make-tuple (list ,@l)))
> (define parse-open-tuple
    (case-lambda
     [(ch port)
      ; ‘read‘ mode
      (wrap (parse port
                   (lambda ()
                     (read/recursive port #f
                                     (make-delims-table)))
                   (object-name port)))]
     [(ch port src line col pos)
      ; ‘read-syntax‘ mode
      (datum->syntax
       #f
       (wrap (parse port
                    (lambda ()
                      (read-syntax/recursive src port #f
                                             (make-delims-table)))
                    src))
       (let-values ([(l c p) (port-next-location port)])
         (list src line col pos (and pos (- p pos)))))]))
> (define tuple-readtable
    (make-readtable #f #\< 'terminating-macro parse-open-tuple))
> (parameterize ([current-readtable tuple-readtable])
    (read (open-input-string "<1 , 2 , \"a\">")))

'(make-tuple (list 1 2 "a"))

> (parameterize ([current-readtable tuple-readtable])
    (read (open-input-string
           "< #||# 1 #||# , #||# 2 #||# , #||# \"a\" #||# >")))

'(make-tuple (list 1 2 "a"))

> (define tuple-readtable+
    (make-readtable tuple-readtable
                    #\* 'terminating-macro (lambda a
                                             (make-special-comment #f))
                    #\_ #\space #f))
> (parameterize ([current-readtable tuple-readtable+])
    (read (open-input-string "< * 1 __,__  2 __,__ * \"a\" * >")))

'(make-tuple (list 1 2 "a"))
