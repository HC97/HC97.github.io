3.14 定义: define, define-syntax, ...

定义：define in Racket 指南 introduces definitions.

syntax

(define id expr)

(define (head args) body ...+)
 
head = id
  | (head args)
     
args = arg ...
  | arg ... . rest-id
     
arg = arg-id
  | [arg-id default-expr]
  | keyword arg-id
  | keyword [arg-id default-expr]
第一种形式将 id 与 expr 的结果 bind , 第二种形式将 id 与一个过程 bind 。 在第二种情况下,生成的过程是 (CVT (head args) body ...+), 使用 CVT 元函数,定义如下:

(CVT (id . kw-formals) . datum)   = (lambda kw-formals . datum)
(CVT (head . kw-formals) . datum) = (lambda kw-formals expr)
                                     if (CVT head . datum) = expr

在 internal-definition context 中, define 形式引入了一个局部绑定;参见 内部定义。 在顶层, id 的顶层绑定在评估 expr 之后被创建, 如果它不存在的话, id 的顶层映射 (在与编译后的定义相连的 namespace 中)也同时被设置为绑定。

在允许 define 的 liberal expansion 的上下文中, 如果 expr 是一个带有关键字参数的直接 lambda 形式, 或者 args 包括关键字参数,那么 id 就被绑定为语法。

Examples:
(define x 10)

 

> x

10

 

(define (f x)
  (+ x 1))

 

> (f 10)

11

 

(define ((f x) [y 20])
  (+ x y))

 

> ((f 10) 30)

40

> ((f 10))

30

syntax

(define-values (id ...) expr)

评估 expr, 如果结果的数量与 id 的数量相符, 则按顺序将结果与 id bind ; 如果 expr 产生不同数量的结果,则引发 exn:fail:contract exception is raised 异常。

在 internal-definition context 中(参见 内部定义), define-values 形式引入了局部绑定。 在顶层,每个 id 的顶层绑定在评估 expr 之后被创建,如果它还不存在的话, 并且每个 id 的顶层映射 (在与编译后的定义相连的 namespace 中)同时被设置为绑定。

Examples:
> (define-values () (values))
> (define-values (x y z) (values 1 2 3))
> z

3

如果模块主体中的函数定义的 define-values 表单中的 'compiler-hint:cross-module-inline syntax property 为真值,那么 Racket 会将该属性视为性能提示。 更多信息请参见 Racket 指南 中的 函数调用优化 , 也请参见 begin-encourage-inline。

syntax

(define-syntax id expr)

(define-syntax (head args) body ...+)
第一种形式是用 expr 的值创建 id 的 transformer 绑定 (参见 变换器绑定) ， expr 是相对于周围环境的 phase level 1 的表达。 (关于 phase levels 的信息请参见 标识符、绑定和范围 。 对 expr 一方的评估是 parameterize , 以便像 let-syntax 中那样设置 current-namespace 。

第二种形式是与 define 相同的速记;它扩展为第一种形式的定义, 其中 expr 是一个 lambda 形式。

在 internal-definition context 中(参见 内部定义), 一个 define-syntax 形式引入了一个局部绑定。

Examples:
> (define-syntax foo
    (syntax-rules ()
      ((_ a ...)
       (printf "~a\n" (list a ...)))))
> (foo 1 2 3 4)

(1 2 3 4)

> (define-syntax (bar syntax-object)
    (syntax-case syntax-object ()
      ((_ a ...)
       #'(printf "~a\n" (list a ...)))))
> (bar 1 2 3 4)

(1 2 3 4)

syntax

(define-syntaxes (id ...) expr)

像 define-syntax 一样,但是为每个 id 创建一个 transformer 绑定。 expr 应该产生和 id 一样多的值,并且每个值都被绑定到相应的 id 上。

当 expr 为顶层 define-syntaxes 产生零值时(即不在模块或内部定义的位置), 那么 id 实际上是在没有绑定的情况下声明的;见 宏引入的绑定。

在 internal-definition context 中(参见 内部定义), define-syntaxes 形式引入了局部绑定。

Examples:
> (define-syntaxes (foo1 foo2 foo3)
    (let ([transformer1 (lambda (syntax-object)
                          (syntax-case syntax-object ()
                            [(_) #'1]))]
          [transformer2 (lambda (syntax-object)
                          (syntax-case syntax-object ()
                            [(_) #'2]))]
          [transformer3 (lambda (syntax-object)
                          (syntax-case syntax-object ()
                            [(_) #'3]))])
      (values transformer1
              transformer2
              transformer3)))
> (foo1)

1

> (foo2)

2

> (foo3)

3

syntax

(define-for-syntax id expr)

(define-for-syntax (head args) body ...+)
和 define 一样,只是相对于它的上下文来说,绑定是在 phase level 1 而不是 phase level 0 。 绑定的表达式也是在 phase level 1 。 (关于 phase levels 的信息请参见 标识符、绑定和范围 )。 这种形式是 (begin-for-syntax (define id expr)) 或 (begin-for-syntax (define (head args) body ...+)) 的简写。

在一个模块中,由 define-for-syntax 引入的绑定必须出现在它们的使用之前, 或者以相同的 define-for-syntax 形式出现 (也就是说, define-for-syntax 形式必须在使用被展开之前被展开)。 特别是,由 define-for-syntax 约束的相互递归函数必须由相同的 define-for-syntax 形式来定义。

Examples:
> (define-for-syntax helper 2)
> (define-syntax (make-two syntax-object)
    (printf "helper is ~a\n" helper)
    #'2)
> (make-two)

helper is 2

2

; ‘helper' is not bound in the runtime phase
> helper

helper: undefined;

 cannot reference an identifier before its definition

  in module: top-level

> (define-for-syntax (filter-ids ids)
    (filter identifier? ids))
> (define-syntax (show-variables syntax-object)
    (syntax-case syntax-object ()
      [(_ expr ...)
       (with-syntax ([(only-ids ...)
                      (filter-ids (syntax->list #'(expr ...)))])
         #'(list only-ids ...))]))
> (let ([a 1] [b 2] [c 3])
    (show-variables a 5 2 b c))

'(1 2 3)

syntax

(define-values-for-syntax (id ...) expr)

像 define-for-syntax 一样, 但是 expr 必须产生与所提供的 id 一样多的值, 并且所有的 id 都被绑定(在 phase level 1)。

Examples:
> (define-values-for-syntax (foo1 foo2) (values 1 2))
> (define-syntax (bar syntax-object)
    (printf "foo1 is ~a foo2 is ~a\n" foo1 foo2)
    #'2)
> (bar)

foo1 is 1 foo2 is 2

2

3.14.1 require Macros

 (require racket/require-syntax) package: base
The bindings documented in this section are provided by the racket/require-syntax library, not racket/base or racket.

syntax

(define-require-syntax id proc-expr)

(define-require-syntax (id args ...) body ...+)
第一种形式与 define-syntax 类似, 但适用于 require 子形式。 proc-expr 必须产生一个接受并返回代表 require 子形式的语法对象的过程。

这种形式通过使用 make-require-transformer 扩展到 define-syntax (更多信息请参见 require 变形器 )。

第二种形式是与 define-syntax 相同的速记; 它扩展为第一种形式的定义,其中 proc-expr 是一个 lambda 形式。

procedure

(syntax-local-require-introduce stx) → syntax?

  stx : syntax?
仅用于向后兼容;等同于 syntax-local-introduce。

Changed in version 6.90.0.29 of package base: 使之等同于 syntax-local-introduce。
