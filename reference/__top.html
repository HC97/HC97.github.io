3.5 变量引用和 #%top

syntax

id

指向顶层、模块层或本地的绑定,当 id 没有被绑定为一个转化器时(参见 扩展 ) 。 在运行时,该引用被评估为与绑定相关的 location 中的值。

当扩展器遇到一个没有被模块级或局部绑定的 id 时,它将表达式转换为 (#%top . id) 给 #%top 提供该 id 的词法上下文; 通常,该上下文是指 #%top 。 另见 扩展步骤。

Examples:
> (define x 10)
> x

10

> (let ([x 5]) x)

5

> ((lambda (x) x) 2)

2

syntax

(#%top . id)

当 id 被绑定到模块级或顶层变量时,等同于 id 。 在顶层上下文中, (#%top . id) 总是指代一个顶层变量, 即使 id unbound 或绑定到语法中,只要 id 没有本地绑定。 在所有上下文中,如果 id 有本地绑定, (#%top . id) 就是一个语法错误。

在一个 module 中, (#%top . id) 只扩展为 id , 只要 id 是在模块中定义的,并且在其上下文中没有本地绑定。 在 phase level 0, 如果 id 没有被绑定, (#%top . id) 是一个即时语法错误。 phase level 1 和更高级别的阶段, 如果 id 在 module-body partial expansion 结束时没有在相应的阶段定义, 则报告语法错误。

关于扩展器如何引 #%top 标识符的信息,请参见 扩展步骤 。

Examples:
> (define x 12)
> (#%top . x)

12

Changed in version 6.3 of package base: 改变了顶层上下文中 #%top 的引入, 仅适用于 unbound 标识符。
Changed in version 8.2.0.7: 改变了对本地绑定的 id 的处理方式, 即使在模块之外,也总是报告语法错误。
