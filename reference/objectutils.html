6.11 对象、类和接口实用程序

procedure

(object? v) → boolean?

  v : any/c
如果 v 是一个对象,返回 #t ,否则返回 #f 。

Examples:
> (object? (new object%))

#t

> (object? object%)

#f

> (object? "clam chowder")

#f

procedure

(class? v) → boolean?

  v : any/c
如果 v 是一个类,返回 #t ,否则返回 #f 。

Examples:
> (class? object%)

#t

> (class? (class object% (super-new)))

#t

> (class? (new object%))

#f

> (class? "corn chowder")

#f

procedure

(interface? v) → boolean?

  v : any/c
如果 v 是一个接口,返回 #t ,否则返回 #f 。

Examples:
> (interface? (interface () empty cons first rest))

#t

> (interface? object%)

#f

> (interface? "gazpacho")

#f

procedure

(generic? v) → boolean?

  v : any/c
如果 v 是一个 generic,返回 #t ,否则返回 #f 。

Examples:
> (define c%
    (class object%
      (super-new)
      (define/public (m x)
        (+ 3.14 x))))
> (generic? (generic c% m))

#t

> (generic? c%)

#f

> (generic? "borscht")

#f

procedure

(object=? a b) → boolean?

  a : object?
  b : object?
确定 a 和 b 是否是由同一个调用 new 返回的。 如果两个对象都有字段,这个过程将确定改变一个对象的字段是否会改变另一个对象的字段。

这个过程在精神上与 eq? 相似,但也能与契约正常工作(并且有更强的保证)。

Examples:
> (define obj-1 (new object%))
> (define obj-2 (new object%))
> (define/contract obj-3 (object/c) obj-1)
> (object=? obj-1 obj-1)

#t

> (object=? obj-1 obj-2)

#f

> (object=? obj-1 obj-3)

#t

> (eq? obj-1 obj-1)

#t

> (eq? obj-1 obj-2)

#f

> (eq? obj-1 obj-3)

#f

procedure

(object-or-false=? a b) → boolean?

  a : (or/c object? #f)
  b : (or/c object? #f)
和 object=? 一样,但接受 #f 作为任一参数,如果两个参数都是 #t ,则返回 #f 。

Examples:
> (object-or-false=? #f (new object%))

#f

> (object-or-false=? (new object%) #f)

#f

> (object-or-false=? #f #f)

#t

Added in version 6.1.1.8 of package base.

procedure

(object=-hash-code o) → fixnum?

  o : object?
返回对应于平等关系 object=? 的 o 的哈希码。

Added in version 7.1.0.6 of package base.

procedure

(object->vector object [opaque-v]) → vector?

  object : object?
  opaque-v : any/c = #f
返回一个代表 object 的向量,显示其可检查的字段,类似于 struct->vector 。

Examples:
> (object->vector (new object%))

'#(object:object% ...)

> (object->vector (new (class object%
                         (super-new)
                         (field [x 5] [y 10]))))

'#(object:eval:107:0 ...)

procedure

(class->interface class) → interface?

  class : class?
返回由 class 隐含定义的接口。

Example:
> (class->interface object%)

#<interface:object%>

procedure

(object-interface object) → interface?

  object : object?
返回 object 的类所隐含定义的接口。

Example:
> (object-interface (new object%))

#<interface:object%>

procedure

(is-a? v type) → boolean?

  v : any/c
  type : (or/c interface? class?)
如果 v 是一个类 type 的实例或一个实现了接口 type 的类, 返回 #t ,否则返回 #f 。

Examples:
> (define point<%> (interface () get-x get-y))
> (define 2d-point%
    (class* object% (point<%>)
      (super-new)
      (field [x 0] [y 0])
      (define/public (get-x) x)
      (define/public (get-y) y)))
> (is-a? (new 2d-point%) 2d-point%)

#t

> (is-a? (new 2d-point%) point<%>)

#t

> (is-a? (new object%) 2d-point%)

#f

> (is-a? (new object%) point<%>)

#f

procedure

(subclass? v cls) → boolean?

  v : any/c
  cls : class?
如果 v 是一个派生于(或等于) cls 的类,返回 #t ,否则返回 #f 。

Examples:
> (subclass? (class object% (super-new)) object%)

#t

> (subclass? object% (class object% (super-new)))

#f

> (subclass? object% object%)

#t

procedure

(implementation? v intf) → boolean?

  v : any/c
  intf : interface?
如果 v 是一个实现了 intf 的类,返回 #t ,否则返回 #f 。

Examples:
> (define i<%> (interface () go))
> (define c%
    (class* object% (i<%>)
      (super-new)
      (define/public (go) 'go)))
> (implementation? c% i<%>)

#t

> (implementation? object% i<%>)

#f

procedure

(interface-extension? v intf) → boolean?

  v : any/c
  intf : interface?
如果 v 是一个扩展了 intf 的接口,返回 #t ,否则返回 #f 。

Examples:
> (define point<%> (interface () get-x get-y))
> (define colored-point<%> (interface (point<%>) color))
> (interface-extension? colored-point<%> point<%>)

#t

> (interface-extension? point<%> colored-point<%>)

#f

> (interface-extension? (interface () get-x get-y get-z) point<%>)

#f

procedure

(method-in-interface? sym intf) → boolean?

  sym : symbol?
  intf : interface?
如果 intf 或其任何一个祖先接口)包括一个名称为 sym 的成员, 返回 #t ,否则返回 #f 。

Examples:
> (define i<%> (interface () get-x get-y))
> (method-in-interface? 'get-x i<%>)

#t

> (method-in-interface? 'get-z i<%>)

#f

procedure

(interface->method-names intf) → (listof symbol?)

  intf : interface?
返回 intf 中方法名称的符号列表,包括从超接口继承的方法, 但不包括名称为本地的方法(即用 define-local-member-name 声明的)。

Examples:
> (define i<%> (interface () get-x get-y))
> (interface->method-names i<%>)

'(get-x get-y)

procedure

(object-method-arity-includes? object    
  sym    
  cnt) → boolean?
  object : object?
  sym : symbol?
  cnt : exact-nonnegative-integer?
如果 object 有一个接受 cnt 参数的名为 sym 的方法, 返回 #t ,否则返回 #f 。

Examples:
> (define c%
    (class object%
      (super-new)
      (define/public (m x [y 0])
        (+ x y))))
> (object-method-arity-includes? (new c%) 'm 1)

#t

> (object-method-arity-includes? (new c%) 'm 2)

#t

> (object-method-arity-includes? (new c%) 'm 3)

#f

> (object-method-arity-includes? (new c%) 'n 1)

#f

procedure

(field-names object) → (listof symbol?)

  object : object?
返回所有绑定在 object 中的字段名的列表,包括从超接口继承的字段, 但不包括名字是本地的字段(即用 define-local-member-name 声明的)。

Examples:
> (field-names (new object%))

'()

> (field-names (new (class object% (super-new) (field [x 0] [y 0]))))

'(y x)

procedure

(object-info object) → 
(or/c class? #f) boolean?
  object : object?
返回两个值,类似于 struct-info 的返回值:

  • class: 一个类或 #f; 如果当前检查器不控制任何 object 是实例的类, 则结果为 #f 。

  • skipped?:如果第一个结果对应于最具体的 object 类, 则为 #f ,否则为 #t 。

procedure

(class-info class)

 → 
symbol?
exact-nonnegative-integer?
(listof symbol?)
(any/c exact-nonnegative-integer? . -> . any/c)
(any/c exact-nonnegative-integer? any/c . -> . any/c)
(or/c class? #f)
boolean?
  class : class?
返回七个值,类似于 struct-type-info 的返回值:

  • name: 作为符号的类的名称;

  • field-cnt: 该类定义的字段(公共和私有)的数量;

  • field-name-list: 对应于类的公共字段的符号列表; 这个列表可以大于 field-cnt ,因为它包括继承的字段;

  • field-accessor: 一个访问程序,用于获取类的实例中的字段值; 该访问程序接收一个实例和一个介于 0 (包括)和 field-cnt (排他)之间的字段索引;

  • field-mutator:用于修改类的实例中的字段值的突变程序; 突变程序接收一个实例, 一个介于 0(包括)和 field-cnt (排他)之间的字段索引,以及一个新的字段值;

  • super-class: 当前检查器控制的给定类的最具体的祖先的类, 如果当前检查器不控制任何祖先,则为 #f ;

  • skipped?:如果第六个结果是最具体的祖先类,则 #f ,否则 #t 。

在与 class 相关的失败中引发,例如试图调用一个不是由对象提供的方法。

procedure

(class-seal class    
  key    
  unsealed-inits    
  unsealed-fields    
  unsealed-methods    
  inst-proc    
  member-proc) → class?
  class : class?
  key : symbol?
  unsealed-inits : (listof symbol?)
  unsealed-fields : (listof symbol?)
  unsealed-methods : (listof symbol?)
  inst-proc : (-> class? any)
  member-proc : (-> class? (listof symbol?) any)
在给定的类中添加一个封印,其关键是符号 key 。 给定的 unsealed-inits, unsealed-fields 和 unsealed-methods 列出了不受封印影响的相应类成员。

当一个类有任何封印时, inst-proc 过程在实例化时被调用(通常,这被用来在实例化时引发错误), 当子类试图添加未被列在未封印列表中的类成员时, member-proc 函数被调用(同样,这通常被用来引发错误)。

inst-proc 函数被调用,其上有试图进行实例化的类值。 member-proc 是用类的值和初始化参数、字段或方法名称的列表来调用的。

procedure

(class-unseal class key wrong-key-proc) → class?

  class : class?
  key : symbol?
  wrong-key-proc : (-> class? any)
移除对一个类的封印,该类之前已经用 class-seal 函数和给定的 key 进行了封印。

如果解封删除了该类的所有封印,那么该类值就可以自由实例化或子类化。 如果给定的类值不包含或任何封印,或者不包含任何给定键的封印, 那么就会用该类值调用 wrong-key-proc 函数。
