8 Contracts

合约 in Racket 指南 introduces contracts.

契约系统保护程序的一部分不受另一部分的影响。 程序员通过 (provide (contract-out ....)) 指定一个模块的出口行为,而合约系统则强制执行这些约束。

 (require racket/contract) package: base
The bindings documented in this section are provided by the racket/contract and racket libraries, but not racket/base.

Contracts 有两种形式:一种是由本节手册中列出的各种操作构建的契约, 另一种是各种普通的 Racket 值,它们可以作为契约,包括

  • symbols, booleans, keywords 和 null, 它们被视为可以识别自己的契约,使用 eq?,

  • strings, byte strings, characters, +nan.0 和 +nan.f, 它们被视为使用 equal? 识别自己的合约,

  • numbers (除了 +nan.0 和 +nan.f), 它们被视为使用 = 识别自己的合同,

  • regular expressions, 它们被视为能够识别 byte strings 和与正则表达式相匹配的 strings 的契约,以及

  • predicates: 任何算数为1的程序都被视为谓词。 在合约检查过程中,它被应用于出现的值,应该返回 #f 表示合约失败,而其他的则表示通过。

Contract combinators 是一些函数,如 -> 和 listof ,它们接受契约并产生其他契约。

Racket中的合约被细分为三个不同的类别:

  • 扁平 contracts 可以立即对一个给定的值进行全面检查。 这些种类的 contracts 本质上是谓词函数。 使用 flat-contract-predicate, 你可以从一个任意的扁平合约中提取谓词; 一些扁平合约可以像函数一样被应用,在这种情况下,它们接受一个参数,并返回 #t 或 #f 来表示给定值是否会被合约接受。 本库中由函数返回的所有平面契约都可以直接作为谓词使用,但作为平面契约的普通Racket值(如数字或符号)不能。

    函数 flat-contract? 识别了一个扁平合约。

  • 陪伴 contracts 可以以这样的方式包装一个值, 即在以后使用该值时发出违反契约的信号,但保证不改变其他行为。 例如,一个函数合约包装了一个函数值,随后检查输入和输出;该函数值在被合约包装之前的任何属性都被合约包装者保留下来。

    所有的 flat contracts 都可以用在期望有 chaperone contracts 的地方(但反之则不然)。 函数 chaperone-contract? 可以识别一个陪护者合约。

  • 冒名顶替者的 contracts 可以包裹价值,并且不提供任何保证。 冒名顶替者合约可以隐藏值的属性,甚至使它们完全不透明(例如, new-∀/c)。

    所有的 contracts 都可以在期望使用冒名顶替者合约的地方使用。 函数 impersonator-contract? 可以识别一个 impersonator 契约。

关于这个层次结构的更多信息,请参见 “Impersonators and Chaperones” 一节, 以及一篇关于陪护者、冒充者以及如何使用它们来实现合约的研究论文 [Strickland12] 。

Changed in version 6.1.1.8 of package base: 修改了 +nan.0 和 +nan.f 改为基于 equal? 的契约。
