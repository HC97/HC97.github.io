3.9 本地绑定: let, let*, letrec, ...

局部绑定 in Racket 指南 introduces local binding.

syntax

(let ([id val-expr] ...) body ...+)

(let proc-id ([id init-expr] ...) body ...+)
第一种形式从左到右评估 val-expr , 为每个 id 创建一个新的 location ,并将值放入这些位置。 然后,它评估 body ,在这个 body 中, id 被绑定。 最后一个 body 表达式相对于 let 形式处于尾部位置。 根据 bound-identifier=? ,这些 id 必须是不同的。

Examples:
> (let ([x 5]) x)

5

> (let ([x 5])
    (let ([x 2]
          [y x])
      (list y x)))

'(5 2)

第二种形式,通常被称为 named let, 对 init-expr 进行评估; 得到的值成为过程应用中的参数 (lambda (id ...) body ...+) , 其中 proc-id 在 body 中被绑定到过程本身。

Example:
> (let fac ([n 10])
    (if (zero? n)
        1
        (* n (fac (sub1 n)))))

3628800

syntax

(let* ([id val-expr] ...) body ...+)

和 let 一样,但是逐个评估 val-expr ,一旦有值, 就为每个 id 创建一个 location 。 这些 id 和 body 一样被绑定在其余的 val-expr 中, 并且这些 id 不需要是不同的;后来的绑定会影射先前的绑定。

Example:
> (let* ([x 1]
         [y (+ x 1)])
    (list y x))

'(2 1)

syntax

(letrec ([id val-expr] ...) body ...+)

和 let 一样,包括从左到右的 val-expr 评估, 但是所有 id 的 locations 首先被创建, 所有 id 都被绑定在所有 val-expr 以及 body 中, 并且每个 id 在相应的 val-expr 被评估后立即被初始化。 根据 bound-identifier=? ,这些 id 必须是不同的。

在初始化前引用或赋值给一个 id 会引发 exn:fail:contract:variable 。 如果一个 id 即绑定实例或 id) 有一个 'undefined-error-name syntax property , 其值是一个符号,该符号被用作错误报告的变量名称,而不是 id 的符号形式。

Example:
> (letrec ([is-even? (lambda (n)
                       (or (zero? n)
                           (is-odd? (sub1 n))))]
           [is-odd? (lambda (n)
                      (and (not (zero? n))
                           (is-even? (sub1 n))))])
    (is-odd? 11))

#t

Changed in version 6.0.1.2 of package base: 改变了对一个未初始化的 id 的引用或赋值,导致错误。

syntax

(let-values ([(id ...) val-expr] ...) body ...+)

和 let 一样,除了每个 val-expr 必须产生和对应 id 一样多的值, 否则会引发 exn:fail:contract exception is raised 异常。 为每个 id 创建一个单独的 location ,所有的 id 都被绑定在 body 中。

Example:
> (let-values ([(x y) (quotient/remainder 10 3)])
    (list y x))

'(1 3)

syntax

(let*-values ([(id ...) val-expr] ...) body ...+)

和 let* 一样,除了每个 val-expr 必须产生和对应 id 一样多的值。 为每个 id 创建一个单独的 location , 所有的 id 都被绑定在后面的 val-expr 和 body 中。

Example:
> (let*-values ([(x y) (quotient/remainder 10 3)]
                [(z) (list y x)])
    z)

'(1 3)

syntax

(letrec-values ([(id ...) val-expr] ...) body ...+)

和 letrec 一样,除了每个 val-expr 必须产生和对应 id 一样多的值。 为每个 id 创建一个单独的 location , 所有的 id 都被绑定在所有的 val-expr 和 body 中。

Example:
> (letrec-values ([(is-even? is-odd?)
                   (values
                     (lambda (n)
                       (or (zero? n)
                           (is-odd? (sub1 n))))
                     (lambda (n)
                       (or (= n 1)
                           (is-even? (sub1 n)))))])
    (is-odd? 11))

#t

syntax

(let-syntax ([id trans-expr] ...) body ...+)

See also splicing-let-syntax.

用 trans-expr 的值为每个 id 创建一个 transformer 绑定 (参见 变换器绑定),这是一个相对于周围环境的 phase level 1 的表达式。(关于 phase levels 的信息,参见 标识符、绑定和范围 。)

每个 trans-expr 的评估都是 parameterize , 将 current-namespace 设置为一个 namespace , 该命名空间与被用来扩展 let-syntax 形式的命名空间共享 bindings 和 variables ,只是其 base phase 要大一号。

每个 id 都被绑定在 body 中,而不是在其他 trans-expr 中。

syntax

(letrec-syntax ([id trans-expr] ...) body ...+)

参见 splicing-letrec-syntax。

和 let-syntax 一样,除了每个 id 也被绑定在所有 trans-expr 中。

syntax

(let-syntaxes ([(id ...) trans-expr] ...) body ...+)

参见 splicing-let-syntaxes。

和 let-syntax 一样, 但是每个 trans-expr 必须产生和对应 id 一样多的值, 每个值都被绑定到对应的值。

syntax

(letrec-syntaxes ([(id ...) trans-expr] ...) body ...+)

参见 splicing-letrec-syntaxes。

和 let-syntax 一样,除了每个 id 也被绑定在所有 trans-expr 中。

syntax

(letrec-syntaxes+values ([(trans-id ...) trans-expr] ...)
                        ([(val-id ...) val-expr] ...)
   body ...+)
将 letrec-syntaxes 与 letrec-values 的一个变体结合起来: 每个 trans-id 和 val-id 都被绑定在所有 trans-expr 和 val-expr 中。

letrec-syntaxes+values 形式是本地编译时绑定的核心形式, 因为像 letrec-syntax 和 internal-definition contexts 这样的形式会扩展到它。 在一个完全扩展的表达式中(参见 完全展开的程序), 跨 trans-id 绑定被丢弃,该形式减少为 letrec-values 或 let-values 的组合。

对于由 letrec-syntaxes+values 绑定的变量, 其 location 创建规则与 letrec-values 略有不同。 [(val-id ...) val-expr] 绑定的子句被划分为满足以下规则的最小子句集: 如果一个子句的 val-id 绑定被先前子句的 val-expr 所引用(在完全展开中), 这两个子句和所有在这之间的子句都在同一个集里。 如果一个集合由一个单独的子句组成, 其 val-expr 不引用该子句的任何 val-id , 那么 val-id 的 locations 将在 val-expr 被评估 后 创建。 否则,在一个集合中的所有 val-id 的 locations 将在该集合中的第一个 val-expr 被评估之前被创建。 为了形成集合,一个 (quote-syntax datum #:local) 表单算作对 letrec-syntaxes+values 表单中所有绑定的引用。

location 创建规则的最终结果是,范围和评估顺序与 letrec-values 相同, 但编译器有更多的自由来优化 location 创建。 这些规则也对应于 let-values 和 letrec-values 的嵌套, 这就是 letrec-syntaxes+values 的完全扩展表达式的方式。

也请参见 local,它支持与 define, define-syntax 等的本地绑定。
