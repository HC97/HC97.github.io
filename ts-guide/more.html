3 指定类型

上一节介绍了 Typed Racket 类型系统的基本情况。 在这一节中,我们将看到该语言的几个新特性,允许类型被指定和使用。

3.1 类型注解和绑定形式

一般来说,Typed Racket 中的变量必须用其类型进行注释。 后面的小节 (什么时候需要类型注解?) 介绍了一个启发式方法,它更精确地说明了什么时候需要类型注释。

3.1.1 注解定义

我们已经看到了 : 类型注释的形式。 这对定义是很有用的,无论是在模块的顶层

(: x Number)
(define x 7)

和内部定义中都很有用。

(let ()
  (: x Number)
  (define x 7)
  (add1 x))

除了 : 形式,几乎所有来自 racket 的绑定形式都被替换为允许指定类型的对应形式。 Typed Racket 的 define 形式允许在顶层和内部背景下定义变量。

(define x : Number 7)
(define (id [z : Number]) : Number z)

在这里, x 的类型是 Number, 而 id 的类型是 (-> Number Number) 。 在 id 的主体中, z 的类型为 Number 。

3.1.2 局部绑定注释
(let ([x : Number 7])
  (add1 x))

let 的形式与 racket 的 let 完全一样,但是可以为每个绑定的变量提供类型注释。 这里, x 被赋予 Number 类型。 let* 和 letrec 是类似的。注释在 let 和变体中是可选的。

(let-values ([([x : Number] [y : String]) (values 7 "hello")])
  (+ x (string-length y)))

let*-values 和 letrec-values 形式类似。

3.1.3 注解函数

函数表达式也可以绑定变量,这些变量可以用类型来注释。 这个函数期望有两个参数,一个 Number 和一个 String:

(lambda ([x : Number] [y : String]) (+ x 5))

这个函数至少接受一个 String, 然后是任意多的 Number 。 在正文中, y 是一个 Number 的列表。

(lambda ([x : String] . [y : Number *]) (apply + y))

这个函数的类型是 (-> String Number * Number) 。 由 case 定义的函数也可以被注解:

(case-lambda [() 0]
             [([x : Number]) x])

这个函数的类型是 (case-> (-> Number) (-> Number Number)) 。

3.1.4 单变量注解

当单个变量绑定需要注释时,可以使用阅读器扩展将注释应用于单个变量:

(let ([#{x : Number} 7]) (add1 x))

这等同于先前使用的 let 。 这对那些没有由 Typed Racket 提供的对应形式的绑定来说非常有用,比如 match:

(: assert-symbols! ((Listof Any) -> (Listof Symbol)))
(define (assert-symbols! lst)
  (match lst
    [(list (? symbol? #{s : (Listof Symbol)}) ...) s]
    [_ (error "expected only symbols, given" lst)]))
3.1.5 注解表达式

我们也可以为一个特定的表达式提供一个预期的类型。

(ann (+ 7 1) Number)

这可以确保表达式,这里是 (+ 7 1) ,具有期望的类型,这里是 Number 。 否则,类型检查器会发出一个错误信号。比如说:

> (ann "not a number" Number)

eval:2:0: Type Checker: type mismatch

  expected: Number

  given: String

  in: Number

3.2 类型推断

在很多情况下,如果 Typed Racket 可以推断出类型,那么类型注释是可以避免的。 例如,所有使用 let 和 let* 的局部绑定的类型都可以被推断出来。

(let ([x 7]) (add1 x))

在这个例子中, x 的类型是 Exact-Positive-Integer 。

同样地,顶级常量定义也不需要注释:

(define y "foo")

在这个例子中, y 的类型是 String 。

最后,循环的参数类型是由它们的初始值推断出来的。

(let loop ([x 0] [y (list 1 2 3)])
  (if (null? y) x (loop (+ x (car y)) (cdr y))))

这里 x 的推断类型为 Integer, y 的推断类型为 (Listof Integer) 。 loop 变量的类型为 (-> Integer (Listof Integer) Integer) 。

3.2.1 什么时候需要类型注解?

过去的几个小节解释了几种添加类型注解的方法,并解释了类型推理允许一些注解被省略。 由于注释经常可以被省略,所以了解在哪些情况下实际需要注释是有帮助的。

以下四条经验法则通常足以确定类型注解是否是必要的。

一个表达式或定义需要类型注释,如果它是

  • 是一个函数的 define 形式,

  • 是一个立即与一个变量绑定的 lambda ,

  • 是一个作为多态函数参数的 lambda ,或

  • 定义了一个可变的变量。

下面是对应于上述每种情况的例子:

例 1:
(: fn (-> String Symbol))
(define (fn str) ...)
例 2:
(: fn (-> String Symbol))
(define fn (lambda (str) ...))
例 3:

(map (lambda ([n : Integer]) (add1 n)) '(1 2 3))

例 4:
(: maybe-animal (Option String))
(define maybe-animal #f)
(set! maybe-animal "Odontodactylus scyllarus")

在这四种情况下,如果省略了类型注释,那么推断出的类型往往会过于保守(例如 Any) ,代码可能不会进行类型检查。

3.3 新类型名称

任何类型都可以用 define-type 的方式被赋予一个名字。

(define-type NN (-> Number Number))

在任何使用 NN 这个名字的地方,它都被扩展为 (-> Number Number) 。 类型名可以是递归的,甚至是相互递归的。
