5 出现类型

5.1 基本出现类型

Typed Racket 的一个显著的类型系统特征是 occurrence typing, 它允许类型系统根据谓词检查的成功或失败来赋予更精确的类型。

为了说明这一点,请看下面的代码:

(: flexible-length (-> (U String (Listof Any)) Integer))
(define (flexible-length str-or-lst)
  (if (string? str-or-lst)
      (string-length str-or-lst)
      (length str-or-lst)))

上面的 flexible-length 函数计算了一个字符串或一个列表的长度。 函数体使用了典型的 Racket 成语,即使用一个谓词进行调度(例如, string?) 。

类型化的 Racket 成功地对这个函数进行了类型检查,因为类型系统知道在 if 表达式的 "then" 分支中, 谓词 string? 必须返回一个真值。 类型系统还知道,如果 string? 返回真值, 那么 str-or-lst 变量的类型必须是 String , 并且可以从 String 和 (Listof Any) 的原始联合中缩小类型。 这使得 "then" 分支中对 string-length 的调用能够成功地进行类型检查。

此外,类型系统还知道,在 if 表达式的 "else" 分支中,谓词必须返回 #f 。 这意味着通过排除法,变量 str-or-lst 的类型必须是 (Listof Any) , 因此调用 (length str-or-lst) 类型检查。

总而言之,如果 Typed Racket 能够根据条件表达式中的谓词检查来确定一个变量必须具有的类型,那么它就可以在条件的适当分支中缩小变量的类型。

5.2 命题和谓词

在上一节中,我们展示了 Typed Racket 程序员可以利用发生类型的优势,对具有联合类型和条件的函数进行类型检查。 这可能会引起一个问题:Typed Racket 如何知道如何根据谓词缩小类型?

答案是,Typed Racket中的谓词类型是用逻辑 propositions 来注释的, 这些逻辑命题告诉类型检查器,当谓词检查成功或失败时,会获得哪些额外的信息。

例如,考虑 REPL 对 string? 的类型打印输出:

> string?

- : (-> Any Boolean : String)

#<procedure:string?>

的类型 (-> Any Boolean : String) 有三个部分。 前两部分与其他函数类型相同,表示该谓词接受任何值并返回布尔值。 第三部分,在 : 之后,表示类型检查器从应用该函数的结果中学习到的逻辑 propositions :

  1. 如果谓词检查成功(即产生一个非 #f 值),则参数变量的类型为 String

  2. 如果谓词检查失败(即产生 #f), 参数变量 不 具有 String 类型

所有内置类型的谓词都有类似的命题注释,允许类型系统对谓词检查进行逻辑推理。

5.2.1 单侧命题

有时,一个谓词在成功时可能会提供信息,但在失败时则不会。 例如,考虑这个函数:

(define (legal-id? s)
  (and (symbol? s)
       (not (member s '(cond else if)))))

这个函数只在给定一个符号时返回 #t ,所以满足这个谓词的东西的类型可以被细化为 Symbol。

然而,未能通过这个谓词的值不能被细化为非符号;像 'else 这样的符号也未能满足这个谓词。

在诸如此类的情况下,有可能提供一个只适用于 "正" 断言的命题。具体来说,这种类型

(: legal-id? (Any -> Boolean : #:+ Symbol))

... 捕捉到这样的想法:如果这个谓词返回 #t, 那么已知参数是一个符号,而不对这个谓词返回 #f 的值做任何主张。

还有一种否定的形式,它允许类型只指定关于导致谓词返回 #f 的值的命题。

5.3 其他条件和断言

毕竟,这些控制流结构最后都宏观地扩展到了 if 。

到目前为止,我们已经看到发生类型允许对 if 表达式进行精确推理。 发生类型法适用于 Racket 中的大多数控制流构造,如 cond, when 和其他。

例如,先前的 flexible-length 函数可以被重新编写为使用 cond ,而不需要额外的努力:

(: flexible-length/cond (-> (U String (Listof Any)) Integer))
(define (flexible-length/cond str-or-lst)
  (cond [(string? str-or-lst) (string-length str-or-lst)]
        [else (length str-or-lst)]))

在某些情况下,类型系统没有足够的信息或者过于保守,无法对表达式进行类型检查。 例如,考虑下面的交互:

> (: a Positive-Integer)
> (define a 15)
> (: b Positive-Integer)
> (define b 20)
> (: c Positive-Integer)
> (define c (- b a))

eval:13:0: Type Checker: type mismatch

  expected: Positive-Integer

  given: Integer

  in: a

在这种情况下,类型系统只知道 a 和 b 是正整数, 不能得出结论说它们的差值在定义 c 时总是正的。 在这样的情况下,发生类型化可以用来使代码使用 assertion 进行类型检查。 比如说、

(: d Positive-Integer)
(define d (assert (- b a) positive?))

使用关于 positive? 的逻辑命题, Typed Racket 可以给整个 assert 表达式分配 Positive-Integer 的类型。 这样就可以进行类型检查,但要注意,如果谓词返回 #f ,断言可能会在运行时引发异常。

请注意, assert 是 Typed Racket 中的一个派生概念,是发生类型化的自然结果。 上面的断言本质上等同于下面的内容:

(: e Positive-Integer)
(define e (let ([diff (- b a)])
            (if (positive? diff)
                diff
                (error "Assertion failed"))))

5.4 关于 set! 的注意事项

如果一个变量在它被定义的作用域中被 set! 变更,Typed Racket 就不能对该变量使用发生类型。 这个预防措施是必要的,以确保对一个变量的并发修改不会使 Typed Racket 对该变量类型的了解失效。 参见 使用赋值的指导原则 。

此外,这意味着 REPL 中 top-level variable 的类型也不能被 Typed Racket 完善。 这是因为顶层变量的范围包括未来的顶层交互,其中可能包括变异。 我们可以通过将变量移到模块内或像 let 这样的本地绑定形式来解决这个问题。

5.5 访问结构域

发生类型可以与不可变的结构字段的访问器一起工作。

(struct apple ([a : Any]))
(struct (A) fruit ([a : A]))
 
(define (f [obj : Any]) : Number
  (cond
    [(and (apple? obj) (number? (apple-a obj))) (apple-a obj)]
    [(and (fruit? obj) (number? (fruit-a obj))) (fruit-a obj)]
    [else 42]))

5.6 let 别名

类型化的 Racket 能够推理出一些由 let 表达式引入的变量别名其他值的情况(例如,当它们别名非变异的标识符, car/cdr/struct 访问到不可变的值时,等等...)。 这使得那些明确依赖于发生类型和别名的程序可以进行类型检查:

(: f (Any -> Number))
(define (f x)
  (let ([y x])
    (cond
      [(number? y) x]
      [(and (pair? y)
            (number? (car y)))
       (car x)]
      [else 42])))

它还允许类型检查器检查那些使用内部严重依赖let绑定的宏的程序(比如 match):

(: g (Any -> Number))
(define (g x)
  (match x
    [(? number?) x]
    [`(_    _ . ,(? number?))  (cddr x)]
    [`(_    _ . ,(? pair? p))
     (if (number? (caddr x))
         (car p)
         41)]
    [_ 42]))
