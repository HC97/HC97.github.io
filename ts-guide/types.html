4 Typed Racket 中的类型

Typed Racket 提供了丰富的类型来描述数据。 本节将对它们进行介绍。

4.1 基本类型

Typed Racket 中最基本的类型是用于原始数据的类型,比如 True 和 False 用于布尔运算, String 用于字符串, Char 用于字符。

> '"hello, world"

- : String

"hello, world"

> #\f

- : Char

#\f

> #t

- : True

#t

> #f

- : False

#f

每个符号都被赋予一个只包含该符号的唯一类型。 Symbol 类型包括所有的符号。

> 'foo

- : 'foo

'foo

> 'bar

- : 'bar

'bar

Typed Racket 还提供了一个丰富的层次结构来描述特定种类的数字。

> 0

- : Integer [more precisely: Zero]

0

> -7

- : Integer [more precisely: Negative-Fixnum]

-7

> 14

- : Integer [more precisely: Positive-Byte]

14

> 3.2

- : Flonum [more precisely: Positive-Float-No-NaN]

3.2

> 7.0+2.8i

- : Float-Complex

7.0+2.8i

最后,任何值本身就是一个类型:

> (ann 23 23)

- : Integer [more precisely: 23]

23

4.2 函数类型

我们已经看到了一些函数类型的例子。 函数类型是用 -> 构建的,其中最后一个类型是结果类型,其他的是参数类型。 下面是一些函数类型的例子:

(-> Number Number)
(-> String String Boolean)
(-> Char (Values String Natural))

第一个类型需要一个 Number 作为输入,并产生一个 Number 。 第二种需要两个参数。 第三种需要一个参数,并产生 multiple values, 类型为 String 和 Natural 。 下面是这些类型中每一种的函数示例。

> (lambda ([x : Number]) x)

- : (-> Number Number)

#<procedure>

> (lambda ([a : String] [b : String]) (equal? a b))

- : (-> String String Boolean)

#<procedure>

> (lambda ([c : Char]) (values (string c) (char->integer c)))

- : (-> Char (values (String : (Top | Bot)) (Index : (Top | Bot))))

#<procedure>

4.3 带有可选参数或关键字参数的函数类型

Racket 函数除了标准的强制性参数外,还经常接受可选参数或关键字参数。 这些函数的类型可以用 ->* 类型构造函数简洁地写出来。下面是一些例子:

(->* () (Number) Number)
(->* (String String) Boolean)
(->* (#:x Number) (#:y Number) (values Number Number))

第一个类型描述了一个没有强制性参数的函数,一个 Number 类型的可选参数,并返回一个 Number 。

第二个需要两个必选参数,没有可选参数,并产生一个 Boolean 。 这个函数类型可以用 -> 写成 (-> String String Boolean) 。

第三个需要一个带有关键字 #:x 的强制性参数,并接受一个带有关键字 #:y 的可选参数。 其结果是两个 Number 类型的值。

4.4 联合类型

有时一个值可以是几种类型中的一种。 为了指定这一点,我们可以使用一个联合类型,用类型构造函数 U 来写。

> (let ([a-number 37])
    (if (even? a-number)
        'yes
        'no))

- : (U 'no 'yes)

'no

任何数量的类型都可以在一个联合中组合在一起,嵌套的联合是平铺的。

(U Number String Boolean Char)

4.5 递归类型

Recursive types 是指其定义指向自身的类型。 这允许一个类型描述一个无限的数据家族。 例如,这就是数字的二进制树的类型。

递归类型也可以在不使用 define-type 的情况下使用 Rec 类型构造函数匿名创建。

(define-type BinaryTree (U Number (Pair BinaryTree BinaryTree)))

类型也可以是 mutually recursive 。 例如,上面的类型定义也可以这样写。

(define-type BinaryTree (U BinaryTreeLeaf BinaryTreeNode))
(define-type BinaryTreeLeaf Number)
(define-type BinaryTreeNode (Pair BinaryTree BinaryTree))

当然,所有的递归类型必须通过契约性检查。 换句话说,直接引用自己的类型是不允许的。 它们必须被用作生产性类型构造器的参数,比如 Listof 和 Pairof 。 例如,在下面的定义中,只有最后一个定义是合法的。

> (define-type BinaryTree BinaryTree)

eval:18:0: Type Checker: Error in macro expansion -- parse

error in type;

 not in a productive position

  variable: BinaryTree

  in: BinaryTree

> (define-type BinaryTree (U Number BinaryTree))

eval:19:0: Type Checker: Error in macro expansion -- parse

error in type;

 not in a productive position

  variable: BinaryTree

  in: BinaryTree

> (define-type BinaryTree (U Number (Listof BinaryTree)))

4.6 结构类型

使用 struct 引入了新的类型,与之前的任何类型不同。

(struct point ([x : Real] [y : Real]))

这种结构的实例,如 (point 7 12), 具有类型 point 。

如果提供了一个结构的超类型,那么新定义的类型就是父类型的一个 subtype 。

4.7 结构类型属性类型

为了注释由 make-struct-type-property 创建的新结构类型属性, 必须在顶层或模块层通过 define-values 来定义它:

> (: prop:foo (Struct-Property (-> Self Number)))
> (: foo-pred (-> Any Boolean : (Has-Struct-Property prop:foo)))
> (: foo-accessor (-> (Has-Struct-Property prop:foo)
                      (Some (X) (-> X Number) : #:+ X)))
> (define-values (prop:foo foo-pred foo-accessor)
                 (make-struct-type-property 'foo))

Struct-Property 为结构类型属性描述符创建了一个类型,其参数是属性值的预期类型。 特别是,当一个结构类型的属性期望与接收器一起应用一个函数时,属性值被提取的结构实例, Self 被用来表示接收器的类型。 对于在 struct 定义中为这种属性提供的值,我们使用结构类型作为 Self 的副定位参数:

(struct apple ([a : Number])
   #:property prop:foo
   (lambda ([me : apple]) : Number
       (apple-a me)))

一个属性 predicate 告诉参数变量是一个 Has-Struct-Property ,如果该谓词检查成功。 Has-Struct-Property 描述了结构类型和附加在它们身上的属性之间的 subtyping 关系。 在上面的例子中,, apple 是 (Has-Struct-Property prop:foo) 的一个子类型。

对于一个属性访问程序,参数必须有一个 Has-Struct-Property 类型。 如果一个属性期望的值是一个与接收器一起调用的函数,即 Self 出现在相应的属性描述符的类型中, 则需要一个 existential type result 类型的结果。 它的量词需要对应于 Self ,并且也出现在 proposition 中。 这样的返回类型可以确保提取的函数不能被结构类型的另一个实例或接收器以外的子结构类型所调用:

> (let ([a1 : apple (apple 42)])
    ((foo-accessor a1) a1))

- : Number

42

> (let ([a1 : apple (apple 42)])
    ((foo-accessor a1) (apple 10)))

eval:27:0: Type Checker: type mismatch

  expected: X

  given: apple

  in: 10

否则,返回类型应该与描述符的 Struct-Property 的类型参数相同。

4.8 子类型

在 Typed Racket 中,所有的类型都被放在一个层次结构中,基于类型中包含的值。 当一个较大类型的元素被期待时,可以提供一个较小类型的元素。 较小的类型被称为大类型的 subtype 。 大的类型被称为 supertype 。 例如, Integer 是 Real 的一个子类型,因为每个整数都是一个实数。 因此,下面的代码对类型检查器来说是可以接受的:

(: f (-> Real Real))
(define (f x) (* x 0.75))
 
(: x Integer)
(define x -125)
 
(f x)

所有类型都是 Any 类型的子类型。

一个联合类型的元素单独是整个联合的子类型,所以 String 是 (U String Number) 的一个子类型。 一个函数类型是另一个函数类型的子类型,如果它们有相同数量的参数, 子类型的参数更宽松(是一个超级类型),子类型的结果类型更宽松(是一个子类型)。 例如, (-> Any String) 是 (-> Number (U String #f)) 的一个子类型。

4.9 多态性

类型化的 Racket 提供了对类型和值的抽象化。 这允许定义使用 parametric polymorphism 的函数。

4.9.1 类型构造函数

内置集合的类型是由 built-in type constructors 创建的。 用户也可以通过 define-type 来定义他们自己的类型构造函数。

请注意,类型和类型构造函数是不同的。 如果一个类型构造器被用在一个类型的位置上,类型检查器将报告一个类型错误:
> (ann 10 (Listof Listof))

eval:28:0: Type Checker: Error in macro expansion -- parse

error in type;

 expected a valid type not a type constructor

  given: Listof

  in: Listof

反之,类型不能被用作类型构造函数:
> (ann 10 (Number Number))

eval:29:0: Type Checker: Error in macro expansion -- parse

error in type;

 bad syntax in type application: expected a type constructor

  given a type: Number

  in: (Number Number)

4.9.2 多态数据结构

几乎每个 Racket 程序都会使用列表和其他集合。 幸运的是,Typed Racket 也可以处理这些。 一个简单的列表处理程序可以这样写:

#lang typed/racket
(: sum-list (-> (Listof Number) Number))
(define (sum-list l)
  (cond [(null? l) 0]
        [else (+ (car l) (sum-list (cdr l)))]))

这看起来与我们前面的程序相似–除了 l 的类型,它看起来像一个函数应用。 事实上,这是对 type constructor Listof 的使用, 它将另一个类型作为其输入,这里是 Number 。 我们可以使用 Listof 来构造我们可能想要的任何种类的列表的类型。

我们也可以定义我们自己的类型构造函数。 例如,这里是哈斯克尔的 Maybe 类型构造函数的一个类似例子:

#lang typed/racket
(struct None ())
(struct (a) Some ([v : a]))
 
(define-type (Opt a) (U None (Some a)))
 
(: find (-> Number (Listof Number) (Opt Number)))
(define (find v l)
  (cond [(null? l) (None)]
        [(= v (car l)) (Some v)]
        [else (find v (cdr l))]))

第一个 struct 定义 None 是一个没有内容的结构。

第二个定义

(struct (a) Some ([v : a]))

创建了一个类型构造函数 Some, 并定义了一个有一个元素的同名结构,其类型是 Some 的类型参数。 这里的类型参数(本例中只有一个,即 a) 被写在类型名之前,并可以在字段的类型中引用。

该类型定义

(define-type (Opt a) (U None (Some a)))

创建了一个类型构造器 — Opt 是一个潜在的容器,用于存放所提供的任何类型。

find 函数接收一个数字 v 和列表, 当在列表中找到该数字时产生 (Some v) ,否则产生 (None) 。 因此,它产生一个 (Opt Number) ,就像注解中指定的那样。

4.9.3 多态函数

有时在多态数据结构上的函数只关心结构的形式。 例如,我们可以写一个函数来获取一个数字列表的长度:

#lang typed/racket
(: list-number-length (-> (Listof Number) Integer))
(define (list-number-length l)
  (if (null? l)
      0
      (add1 (list-number-length (cdr l)))))

和一个获取字符串列表长度的函数:

#lang typed/racket
(: list-string-length (-> (Listof String) Integer))
(define (list-string-length l)
  (if (null? l)
      0
      (add1 (list-string-length (cdr l)))))

注意,这两个函数的定义几乎完全相同;唯一的区别是函数的名称。 这是因为两个函数在定义中都没有使用元素的类型。

我们可以对元素的类型进行抽象,如下所示:

#lang typed/racket
(: list-length (All (A) (-> (Listof A) Integer)))
(define (list-length l)
  (if (null? l)
      0
      (add1 (list-length (cdr l)))))

新的类型构造函数 All 接收一个类型变量的列表和一个主体类型。 类型变量被允许在 All 形式的主体中自由出现。

4.9.4 类型变量的词法作用域

当 : 注释形式包括用于 parametric polymorphism 的类型变量时, 类型变量是 lexically scoped 。 换句话说,类型变量被绑定在你注释的定义的主体中。

例如,下面这个 my-id 的定义使用类型变量 a 来注释参数 x:

(: my-id (All (a) (-> a a)))
(define my-id (lambda ([x : a]) x))

词法范围也意味着类型变量可以被影射,比如在下面的例子中:

(: my-id (All (a) (-> a a)))
(define my-id
  (lambda ([x : a])
    (: helper (All (a) (-> a a)))
    (define helper
      (lambda ([y : a]) y))
    (helper x)))

内层 lambda 里面对 a 的引用指的是 helper 注释中的类型变量。 这个 a 与外层 lambda 表达式注释中的 a 不 一样。

4.10 变量有效性函数: 使用其余参数编程

类型化的 Racket 可以处理一些休息参数的使用。

4.10.1 统一变址函数

在 Racket 中,人们可以编写一个接受任意数量参数的函数,如下所示:

#lang racket
(define (sum . xs)
  (if (null? xs)
      0
      (+ (car xs) (apply sum (cdr xs)))))
 
(sum)
(sum 1 2 3 4)
(sum 1 3)

函数中超出非休止参数的参数被转换为一个列表,并被分配给休止参数。 所以上面的例子评估为 0, 10 和 4 。

我们也可以在 Typed Racket 中定义这样的函数:

#lang typed/racket
(: sum (-> Number * Number))
(define (sum . xs)
  (if (null? xs)
      0
      (+ (car xs) (apply sum (cdr xs)))))

当休息参数中的每个元素都以相同的类型使用时,这种类型可以被分配给函数。

4.10.2 非统一变量rity函数

然而,rest 参数可以作为一个异质列表使用。 以这个 R6RS 函数 fold-left 的(简化)定义为例:

#lang racket
(define (fold-left f i as . bss)
  (if (or (null? as)
          (ormap null? bss))
      i
      (apply fold-left
             f
             (apply f i (car as) (map car bss))
             (cdr as)
             (map cdr bss))))
 
(fold-left + 0 (list 1 2 3 4) (list 5 6 7 8))
(fold-left + 0 (list 1 2 3) (list 2 3 4) (list 3 4 5) (list 4 5 6))
(fold-left (λ (i v n s) (string-append i (vector-ref v n) s))
           ""
           (list (vector "A cat" "A dog" "A mouse")
                 (vector "tuna" "steak" "cheese"))
           (list 0 2)
           (list " does not eat " "."))

在这里,组成其余参数 bss 的不同列表可以有不同的类型, 但是 bss 中每个列表的类型都对应于 f 的相应参数的类型。 我们还知道,为了避免 arity 错误, bss 的长度必须比 f 的 arity 少两个。 f 的第一个参数是累加器,而 as 对应于 f 的第二个参数。

fold-left 的例子是: 36, 42 和 "A cat does not eat cheese." 。

在 Typed Racket 中,我们可以定义 fold-left 如下:

#lang typed/racket
(: fold-left
   (All (C A B ...)
        (-> (-> C A B ... B C) C (Listof A) (Listof B) ... B
            C)))
(define (fold-left f i as . bss)
  (if (or (null? as)
          (ormap null? bss))
      i
      (apply fold-left
             f
             (apply f i (car as) (map car bss))
             (cdr as)
             (map cdr bss))))

注意,类型变量 B 后面有一个省略号。 这表示 B 是一个带点的类型变量,它对应于一个类型的列表,就像其余参数对应于一个值的列表。 当 fold-left 的类型被实例化为一个类型列表时, 那么被 B 约束的每个类型 t (由带点的前类型 t ... B 表示)被扩展为 t 的若干个副本,其数量等于分配给 B 的序列的长度。 然后,每个副本中的 B 被替换成序列中的相应类型。

所以 (inst fold-left Integer Boolean String Number) 的类型是

(-> (-> Integer Boolean String Number Integer) Integer (Listof Boolean) (Listof String) (Listof Number) Integer) 。
