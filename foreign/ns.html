5.16 Cocoa 基础

ffi/unsafe/nsalloc 和 ffi/unsafe/nsstring 库提供了与 Cocoa 和/或 Mac OS Foundation 库一起工作的基本设施(通常与 ffi/objc 一起)。

5.16.1 字符串

 (require ffi/unsafe/nsstring) package: base

value

_NSString : ctype?

一个在 Racket 字符串和 NSString* (又名 CFStringRef) 值之间转换的类型。 也就是说,使用 _NSString 作为一个外来函数 NSString* 参数或结果的类型。

_NSString 转换保持了从 Racket 字符串到转换后的字符串的弱映射, 因此多次转换同一个字符串(在 equal? 意义上)可以避免分配多个 NSString 对象。

5.16.2 分配池

 (require ffi/unsafe/nsalloc) package: base
调用任何分配的基金会 API 都需要安装一个 NSAutoreleasePool 。 ffi/unsafe/nsalloc 库提供了一个函数和速记语法形式来设置这样一个上下文。 (然而, _NSString 类型在从 Racket 字符串转换时隐含地创建了一个自动释放池)。

procedure

(call-with-autorelease thunk) → any

  thunk : (-> any)
以原子模式调用 thunk ,并使用一个新的 NSAutoreleasePool , 在 thunk 返回后 releas 。

syntax

(with-autorelease expr)

这是 (call-with-autorelease (lambda () expr)) 的缩写。
