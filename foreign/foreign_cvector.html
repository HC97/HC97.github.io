5.2 Safe C Vectors

 (require ffi/cvector) package: base
 (require ffi/unsafe/cvector)
ffi/unsafe/cvector 库导出本节的绑定。 ffi/cvector 库输出相同的绑定,除了不安全的 make-cvector* 操作。

cvector 形式可以作为C类型的向量(即一个内存块的指针)。

syntax

(_cvector mode type maybe-len)

_cvector
和 _bytes 一样, _cvector 可以作为一个简单的类型, 对应于一个指针,在 Racket 端作为一个安全的 C 向量管理。 较长的形式与 _list 和 _vector 自定义类型的行为类似, 只是 _cvector 的效率更高;不需要 Racket 列表或向量。

procedure

(make-cvector type length) → cvector?

  type : ctype?
  length : exact-nonnegative-integer?
使用给定的 type 和 length 分配一个 C 向量。 产生的向量不保证包含任何特定的值。

procedure

(cvector type val ...) → cvector?

  type : ctype?
  val : any/c
创建一个给定 type 的 C 向量,初始化为给定的 val 列表。

procedure

(cvector? v) → boolean?

  v : any/c
如果 v 是一个 C 向量,返回 #t ,否则返回 #f 。

procedure

(cvector-length cvec) → exact-nonnegative-integer?

  cvec : cvector?
返回一个 C 向量的长度。

procedure

(cvector-type cvec) → ctype?

  cvec : cvector?
返回 C 向量的 C 类型对象。

procedure

(cvector-ptr cvec) → cpointer?

  cvec : cvector?
返回指向给定 C 向量的起始块的指针。

procedure

(cvector-ref cvec k) → any

  cvec : cvector?
  k : exact-nonnegative-integer?
引用 cvec C 向量的第 k 个元素。 结果具有该 C 向量的类型。

procedure

(cvector-set! cvec k val) → void?

  cvec : cvector?
  k : exact-nonnegative-integer?
  val : any
将 cvec C 向量的第 k 个元素设置为 val 。 val 参数应该是一个可以与 C 向量使用的类型一起使用的值。

procedure

(cvector->list cvec) → list?

  cvec : cvector?
将 cvec C 向量对象转换为一个值的列表。

procedure

(list->cvector lst type) → cvector?

  lst : list?
  type : ctype?
将列表 lst 转换为一个给定 type 的 C 向量。

procedure

(make-cvector* cptr type length) → cvector?

  cptr : any/c
  type : ctype?
  length : exact-nonnegative-integer?

使用现有的指针对象构造一个 C 向量。 这个操作是不安全的,所以它是在 type 和 length 已知的特定情况下使用。
