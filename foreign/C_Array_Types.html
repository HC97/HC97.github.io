3.8 C Array Types

procedure

(make-array-type type count) → ctype?

  type : ctype?
  count : exact-nonnegative-integer?
用于创建新的 C 数组类型的原始类型构造函数。 像 C 结构类型一样,数组类型是新的原始类型,没有相关的转换函数。 当用作函数参数或返回类型时,数组类型的行为像指针类型; 否则,数组类型的行为像结构类型(即,一个结构有多少个字段就有多少个元素),特别是当用于一个结构类型中的字段时。

因为一个数组被当作一个结构,所以将一个指针类型 cast 到一个数组类型并不可行。 相反,使用 ptr-ref 与一个指针,一个用 _array 构建的数组类型, 以及索引 0 来转换一个指针为 Racket 表示法,与 array-ref 和 array-set! 一起使用。

procedure

(_array type count ...+) → ctype?

  type : ctype?
  count : exact-nonnegative-integer?
创建一个数组类型,其 Racket 表示法是适用于 array-ref 和 array-set! 的数组。 该数组不被复制;Racket 表示法是由底层的 C 表示法支持的。

为一个多维数组提供多个 count 。 由于 C 语言对数组使用行主序, (_array t n m) 等同于 (_array (_array t m) n), 这与数组的指针数组不同。

当一个值被用作一个数组类型的实例时(例如,作为传递给一个外国函数的实例), 检查确保给定的值是一个至少具有预期长度的数组,并且其元素根据 ctype->layout 具有相同的表示方法; 该数组可以有额外的元素,并且它可以有不同的元素类型,只要该类型与预期类型的布局相匹配。

procedure

(array? v) → boolean?

  v : any/c
如果 v 是Racket通过 _array 表示的C值,则返回 #t ,否则返回 #f 。

procedure

(array-ref a i ...+) → any/c

  a : array?
  i : exact-nonnegative-integer?
从一个数组中提取一个元素。 在多维数组访问中使用多个 i 索引;使用少于数组维度的索引产生一个子数组。

procedure

(array-set! a i ...+ v) → void?

  a : array?
  i : exact-nonnegative-integer?
  v : any/c
设置一个数组中的元素。 使用多个 i 索引进行多维数组更新;使用少于数组维数的索引设置一个子数组 (即 v 必须是一个与子数组相同大小的数组,并且 v 被复制到子数组中)。

procedure

(array-ptr a) → cpointer?

  a : array?
提取一个数组的存储指针。

procedure

(array-length a) → exact-nonnegative-integer?

  a : array?
提取一个数组的长度。 对于一个多维数组,其结果仍然是一个单一的数字;提取一个元素得到一个子数组,得到下一维的长度,以此类推。

procedure

(array-type a) → ctype?

  a : array?
提取数组的类型。 对于一个多维数组,结果是嵌套数组的 ctype。

procedure

(in-array a [start stop step]) → sequence?

  a : array?
  start : exact-nonnegative-integer? = 0
  stop : (or/c exact-integer? #f) = #f
  step : (and/c exact-integer? (not/c zero?)) = 1
当没有提供可选参数时,返回一个相当于 a 的序列。

可选参数 start, stop 和 step 与 in-vector 相同。

procedure

(_array/list type count ...+) → ctype?

  type : ctype?
  count : exact-nonnegative-integer?
和 _array 一样,但是 Racket 的表示方式是一个列表(或者多维数组的列表),是复制到底层 C 数组中的元素。

procedure

(_array/vector type count ...+) → ctype?

  type : ctype?
  count : exact-nonnegative-integer?
和 _array 一样,但 Racket 的表示方法是一个向量(或多维数组的向量),是复制到底层 C 数组的元素。
