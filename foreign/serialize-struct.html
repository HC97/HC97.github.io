5.4 标记的 C 语言指针类型

 (require ffi/serialize-cstruct)
  package: serialize-cstruct-lib

syntax

(define-serializable-cstruct _id ([field-id type-expr] ...)
                             property ...)
 
property = #:alignment alignment-expr
  | #:malloc-mode malloc-mode-expr
  | #:serialize-inplace
  | #:deserialize-inplace
  | #:version vers
  | 
#:other-versions ([other-vers deserialize-chain-expr
                              convert-proc-expr
                              unconvert-proc-expr
                              cycle-convert-proc-expr]
                   ...)
  | #:property prop-expr val-expr
像 define-cstruct 一样,但定义了一个可序列化的类型,有几个改变了的附加绑定:

  • make-id — 始终使用 'atomic 分配,即使 #:malloc-mode 被指定(出于历史原因)。

  • make-id/mode — 与 make-id 的行为类似, 但使用通过 malloc-mode-expr 指定的模式或分配器(出于历史原因)。

  • deserialize:cstruct:id (对于 vers 为 0) 或 deserialize:cstruct:id-vvers (对于 vers 为 1 或更多) - 从 deserialize-info 子模块自动导出的反序列化信息。

  • deserialize-chain:cstruct:id (对于 vers 为 0) 或 deserialize-chain:cstruct:id-vvers (对于 vers 为 1 或更多) - 通过 #:other-versions 在其他 define-serializable-cstruct 形式中使用的反序列化信息。

  • deserialize:cstruct:id (对于 other-vers 为 0) 或 deserialize:cstruct:id-vother-vers (对于 other-vers 为 1 或更多) — 从 deserialize-info 子模块自动导出的反序列化信息。

新类型的实例满足 serializable? 的谓词,并且可以被用于 serialize 和 deserialize 。 如果其中一个字段包含一个任意的指针、一个嵌入的不可序列化的 C 结构,或者一个指向不可序列化的 C 结构的指针,序列化可能会失败。 只要数组类型不包含这些类型之一,就支持数组类型。

默认 vers 为 0, vers 必须是一个字面的、精确的、非负的整数。 一个 #:other-versions 子句为名称为 id 的结构的先前版本提供了反序列化器, 因此在改变 id 的声明之后,先前序列化的数据可以被反序列化。 对于每个 other-vers, deserialize-chain-expr 应该是 deserialize:cstruct:other-id 绑定的值, 用于一些用 define-serializable-cstruct 声明的其他 "other-id" , 其形状与先前版本的 id 相同; 由 convert-proc-expr 产生的函数应该将 other-id 的实例转换为 id 的实例。 unconvert-proc-expr 和 cycle-convert-proc-expr 产生的函数在一条记录被卷入一个循环时使用; unconvert-proc-expr 产生的函数将 convert-proc-expr 的函数产生的 id 实例带回一个 other-id, 而 cycle-convert-proc-expr 返回两个值:一个 id 的外壳实例和接受一个填充的 other-id 的函数, 其内容应该被移到 id 的外壳实例中。

malloc-mode-expr 参数在反序列化和 make-id/mode 过程中控制该类型的内存分配。 它可以是 malloc 的一个模式参数,或者是一个程序

(-> exact-positive-integer? cpointer?)

来分配给定大小的内存。默认情况下是使用 'atomic 的 malloc 。

当 #:serialize-inplace 被指定时,序列化的表示与 C 结构对象共享内存。 虽然效率更高,特别是对于大型对象,但在序列化后对对象的改变可能会导致序列化表示的改变。

如果可能的话, #:deserialize-inplace 选项可以重用序列化表示的内存。 这个选项对于大型对象来说更加有效,但是对于循环结构来说,它可能会退回到通过 malloc-mode-expr 分配。 由于序列化表示的分配模式默认为 'atomic , 或者在指定 #:serialize-inplace 的情况下可能是任意的,因此只要对象包含指针,就应该谨慎地使用原地反序列化。

当 C 结构包含指针时,建议使用一个自定义的分配器。 它应该基于像 'raw 这样的非移动内存分配,可能需要手动释放以避免垃圾回收后的内存泄漏。

Changed in version 1.1 of package serialize-cstruct-lib: 增加了 #:version 和 #:other-versions 。

Examples:
> (define-serializable-cstruct fish ([color _int]))
> (define f0/s (serialize (make-fish 1)))
> (fish-color (deserialize f0/s))

1

> (define-serializable-cstruct aq ([a (_gcable fish-pointer)]
                                   [d (_gcable aq-pointer/null)])
    #:malloc-mode 'nonatomic)
> (define aq1 (make-aq/mode (make-fish 6) #f))
> (set-aq-d! aq1 aq1) ; create a cycle
> (define aq0/s (serialize aq1))
> (aq-a (aq-d (aq-d (deserialize aq0/s))))

#<cpointer:fish>

; Same shape as original aq:
> (define-serializable-cstruct old-aq ([a (_gcable fish-pointer)]
                                       [d (_gcable _pointer)])
          #:malloc-mode 'nonatomic)
; Replace the original aq:
> (define-serializable-cstruct aq ([a (_gcable fish-pointer)]
                                   [b (_gcable fish-pointer)]
                                   [d (_gcable aq-pointer/null)])
    #:malloc-mode 'nonatomic
    #:version 1
    #:other-versions ([0 deserialize-chain:cstruct:old-aq
                         (lambda (oa)
                           (make-aq/mode (old-aq-a oa)
                                         (old-aq-a oa)
                                         (cast (old-aq-d oa) _pointer aq-pointer)))
                         (lambda (a)
                           (make-old-aq/mode (aq-a a)
                                             (aq-d a)))
                         (lambda ()
                           (define tmp-fish (make-fish 0))
                           (define a (make-aq/mode tmp-fish tmp-fish #f))
                           (values a
                                   (lambda (oa)
                                     (set-aq-a! a (old-aq-a oa))
                                     (set-aq-b! a (old-aq-a oa))
                                     (set-aq-d! a (cast (old-aq-d oa) _pointer aq-pointer)))))]))
; Deserialize old instance to new cstruct:
> (fish-color (aq-a (aq-d (aq-d (deserialize aq0/s)))))

6

> (define aq1/s (serialize (make-aq/mode (make-fish 1) (make-fish 2) #f)))
; New version of fish:
> (define-serializable-cstruct old-fish ([color _int]))
> (define-serializable-cstruct fish ([weight _float]
                                     [color _int])
    #:version 1
    #:other-versions ([0 deserialize-chain:cstruct:old-fish
                         (lambda (of)
                           (make-fish 10.0 (old-fish-color of)))
                         (lambda (a) (error "cycles not possible!"))
                         (lambda () (error "cycles not possible!"))]))
; Deserialized content upgraded to new fish:
> (fish-color (aq-b (deserialize aq1/s)))

2

> (fish-weight (aq-b (deserialize aq1/s)))

10.0
