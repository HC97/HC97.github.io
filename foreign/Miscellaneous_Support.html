6 Miscellaneous Support

procedure

(list->cblock lst    
  type    
  [expect-length    
  #:malloc-mode malloc-mode]) → cpointer?
  lst : list?
  type : ctype?
  expect-length : (or/c exact-nonnegative-integer? #f) = #f
  malloc-mode : (or/c #f symbol?) = #f
分配一个适当大小的内存块–使用带 type 和 (length lst) 的 malloc –并使用 lst 中的值初始化它。 lst 必须持有所有可以根据给定 type 转换为 C 值的值。

如果 expect-length 不是 #f ,并且与 (length lst) 不一样, 那么就会引发一个异常,而不是分配内存。

如果 malloc-mode 不是 #f, 它将作为 malloc 的一个额外参数提供。

Changed in version 7.7.0.2 of package base: 增加了 #:malloc-mode 参数。

procedure

(saved-errno) → exact-integer?

(saved-errno new-value) → void?
  new-value : exact-integer?
返回或设置为当前 Racket 线程保存的错误代码。 保存的错误代码是在使用非 #f #:save-errno 选项的外来调用后设置的 (见 _fun 和 _cprocedure), 但也可以明确设置(例如,为测试而创建模拟外来函数)。

Changed in version 6.4.0.9 of package base: 增加了单参数变体。

procedure

(lookup-errno sym) → (or/c exact-integer? #f)

  sym : symbol?
返回一个平台特定的正整数,对应于 POSIX errno 代码,如果代码未知,则返回 #f 。 如果一个代码是下面描述的公认的符号之一, 并且 该代码是由用于编译 Racket 的 "errno.h" 头定义的,那么该代码的值就是已知的。 请注意, "errno.h" 的内容根据平台和编译器的不同而不同。

目前认可的符号包括 IEEE Std 1003.1, 2013 Edition (也称为 POSIX.1)定义的 81 个代码,包括 'EINTR, 'EEXIST 和 'EAGAIN 。

Changed in version 6.6.0.5 of package base: 放宽了契约,增加了对更多符号的支持。

procedure

(cast v from-type to-type) → any/c

  v : any/c
  from-type : ctype?
  to-type : ctype?
将 v 从一个匹配 from-type 的值转换为一个匹配 to-type 的值, 其中 (ctype-sizeof from-type) 匹配 (ctype-sizeof to-type) 。

这个转换大致上相当于

(let ([p (malloc from-type)])
  (ptr-set! p from-type v)
  (ptr-ref p to-type))

如果 v 是一个 cpointer, (cpointer-gcable? v) 为真, 并且 from-type 和 to-type 都是基于 _pointer 或 _gcpointer 的, 那么 from-type 会隐含地与 _gcable 转换, 以确保结果 cpointer 被视为指向由垃圾回收器管理的内存。

如果 v 是一个带有偏移量的指针(例如,来自 ptr-add), (cpointer-gcable? v) 为真,并且结果是一个 cpointer,那么结果指针具有和 v 一样的偏移量。 如果 (cpointer-gcable? v) 是假的,那么任何偏移量都被折叠到结果的指针基中。
