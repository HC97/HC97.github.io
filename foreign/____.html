3.5 指针类型

value

_pointer : ctype?

对应于 Racket C pointer 值。 这些指针可以有一个任意的 Racket 对象作为类型标签。 该标签被内置功能忽略;它的目的是被接口使用。 参见 安全的 C 向量 ,以创建使用这些标签的指针类型,以确保安全。 一个 #f 值被转换为　NULL ,反之亦然。

作为一个结果类型, _pointer 值所引用的地址不得引用由垃圾收集器管理的内存 (除非该地址对应于支持内部指针的值,并且以其他方式引用以保留该值不被垃圾收集)。 该引用不会被垃圾收集器追踪或更新。 作为一个参数类型, _pointer 适用于对 GC 管理的内存的引用,或者不适用。

当 C 指针(包括 _gcpointer 的指针和可能包含偏移量的指针)指向相同的地址时, equal? 谓词将它们等同起来–但 C 指针是具有 prop:cpointer 属性的结构类型的实例除外, 在这种情况下,相关结构类型的平等规则适用。

value

_gcpointer : ctype?

与作为参数类型的 _pointer 相同,但作为结果类型, _gcpointer 对应于一个指向由垃圾收集器管理的内存的 C 指针值。

在 Racket 的 BC 实现中, _gcpointer 结果指针可以引用不由垃圾收集器管理的内存, 但要注意使用一个可能最终由垃圾收集器管理的地址。 例如,如果一个引用是由 malloc 用 'raw 创建的, 然后由 free　释放,那么 free 可能会让引用以前占用的内存以后被垃圾收集器使用。

cpointer-gcable? 函数对于通过 _gcpointer 结果类型产生的 cpointer 返回 #t 。 参见　cpointer-gcable? 获取更多信息。

value

_racket : ctype?

value

_scheme : ctype?

一个可以用于任何 Racket 对象的类型;它对应于 Racket 的 C API 的 Scheme_Object* 类型(见 Inside: Racket C API)。 _racket 或 _scheme 类型只对那些了解 Racket 的 C API 的库有用。

作为一个具有函数类型的结果类型, _racket 或　_scheme 允许多个值, 但多个值不允许与　_cprocedure 或 _fun　中的 　#:in-original-place? 或 #:async-apply 的真值相结合。

value

_fpointer : ctype?

与 _pointer　类似, 除了当 _fpointer 被用作 get-ffi-obj 或 ffi-obj-ref　的类型时,会跳过一层指示。 此外,对于来自使用　_fpointer　的 get-ffi-obj 或 ffi-obj-ref 的 C 指针值, 作为 _fpointer 的指针上的 ptr-ref 只是返回指针而不是解除引用它。 像 _pointer　一样, _fpointer 将 #f 视为　NULL ,反之亦然。

由 _cprocedure 或 _fun 生成的类型建立在　_fpointer　上, 通常应该使用 _cprocedure 或 _fun　来代替 _fpointer　。

procedure

(_or-null ctype) → ctype?

  ctype : ctype?
创建一个与 ctype　相似的类型,但是 #f 被转换为 NULL ,反之亦然。 给定的 ctype 必须与 _pointer,　_gcpointer 或 _fpointer　有相同的 C 表示。

procedure

(_gcable ctype) → ctype?

  ctype : ctype?
创建一个与 ctype　相似的类型,但其基本表示法与 _gcpointer 相似,而不是　_pointer　。 给定的 ctype 必须有一个像 _pointer 或 _gcpointer 的基数表示 (在后一种情况下,其结果是 ctype)。
