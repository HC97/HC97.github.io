5.13 Operating System Threads

 (require ffi/unsafe/os-thread) package: base
ffi/unsafe/os-thread 库提供了在操作系统层面上以独立的线程运行受限 Racket 代码的功能。 除了 os-thread-enabled?, ffi/unsafe/os-thread 的函数目前只在 (system-type 'vm) 返回 'chez-scheme 时才被支持,即使如此也只在某些构建模式下被支持。 这些函数在不被支持时引发 exn:fail:unsupported 。

Added in version 6.90.0.9 of package base.

procedure

(os-thread-enabled?) → boolean?

如果 ffi/unsafe/os-thread 的其他函数都能正常工作而不引发 exn:fail:unsupported, 则返回 #t ,否则返回 #f 。

procedure

(call-in-os-thread thunk) → void?

  thunk : (-> any)
在一个独立的操作系统线程中运行 thunk ,该线程与所有 Racket 线程同时运行。

thunk 是以 atomic mode 运行的,它不能检查其延续, 也不能使用任何 Racket 线程函数(如 thread 或 current-thread), 任何 Racket 同步函数(如 semaphore-post 或 sync), 或任何参数(如 current-output-port) 。 变量可以用 set! 安全地进行变异, 向量、可变对、盒子、可变结构域以及基于 eq? 和 eqv? 的哈希表都可以被变异, 但是变异对其他线程的可见性是没有规定的,除非是通过 os-semaphore-wait 和 os-semaphore-post 进行同步。

procedure

(make-os-semaphore) → any

创建一个 semaphore,可以与 os-semaphore-wait 和 os-semaphore-post 一起使用, 使操作系统线程与 Racket 线程和其他操作系统线程同步。

procedure

(os-semaphore-post sema) → void?

  sema : any/c
类似于 semaphore-post, 它是通过 make-os-semaphore 创建的一个semaphore。

procedure

(os-semaphore-wait sema) → void?

  sema : any/c
类似于 semaphore-wait ,但是在由 make-os-semaphore 创建的 semaphore 上等待。 等待会阻塞当前线程;如果当前线程是 Racket 线程,那么等待也会阻塞所有 Racket 线程。

5.13.1 操作系统异步通道

 (require ffi/unsafe/os-async-channel) package: base
ffi/unsafe/os-async-channel 库提供了一个与操作系统线程一起工作的异步通道, 其中不允许使用普通的 racket 通道或 place 通道。 这些通道通常与 ffi/unsafe/os-thread.

一个异步的操作系统通道是一个 synchronizable event, 所以它可以和 sync 一起使用,在 Racket 线程中接收一个值。 其他线程必须使用 os-async-channel-try-get 或 os-async-channel-get 。

当一个线程在由 make-os-async-channel 产生的 不 可访问的异步通道上被阻塞时,该线程就不能用于垃圾回收。 这与线程在普通的 Racket 通道或地方通道上被阻塞是不同的。

Added in version 8.0.0.4 of package base.

procedure

(make-os-async-channel) → os-async-channel?

创建一个新的、空的异步通道,供操作系统线程使用。

procedure

(os-async-channel? v) → boolean?

  v : any/c
如果 v 是由 make-os-async-channel 产生的异步通道, 返回 #t ,否则返回 #f 。

procedure

(os-async-channel-put ch v) → void?

  ch : os-async-channel?
  v : any/c
将 v 排在异步通道 ch 中。 这个函数可以从 Racket 线程或任何操作系统线程中调用。

procedure

(os-async-channel-try-get ch [default-v]) → any/c

  ch : os-async-channel?
  default-v : any/c = #f
从异步通道 ch 中取出一个值,如果有可用的值,则返回该值。 如果通道中没有立即可用的值,则返回 default-v 。 这个函数可以从 Racket 线程或任何操作系统线程中调用。

procedure

(os-async-channel-get ch) → any/c

  ch : os-async-channel?
从异步通道 ch 中取出一个值并返回,阻塞直到有可用的值。 这个函数可以从任何非 Racket 操作系统线程中调用。 这个函数 不 应该从 Racket 线程中调用,因为它的阻塞方式会阻塞一个地方的所有 Racket 线程; 在 Racket 线程中,使用 sync 代替。
