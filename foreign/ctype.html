3.1 类型构造器

procedure

(make-ctype type racket-to-c c-to-racket) → ctype?

  type : ctype?
  racket-to-c : (or/c #f (any/c . -> . any))
  c-to-racket : (or/c #f (any/c . -> . any))
创建一个新的 C type 值,其对外来代码的表示与 type　的表示相同。

给出的转换函数可以转换为新类型的 Racket 表示。 任何一个转换函数都可以是　#f, 这意味着相应方向的转换是身份函数。 如果两个函数都是　#f,则返回 type 。

racket-to-c 函数接收任何值,如果它是新类型的有效表示,则将其转换为 type　的表示。 c-to-racket 函数接收一个 type 的表示,并产生一个新类型的表示。

procedure

(ctype? v) → boolean?

  v : any/c
如果 v 是一个　C type,返回 #t ,否则返回 #f　。

Examples:
> (ctype? _int)

#t

> (ctype? (_fun _int -> _int))

#t

> (ctype? #f)

#f

> (ctype? "foo")

#f

procedure

(ctype-sizeof type) → exact-nonnegative-integer?

  type : ctype?
(ctype-alignof type) → exact-nonnegative-integer?
  type : ctype?
返回当前平台上给定 type 的大小或对齐方式。

Examples:
> (ctype-sizeof _int)

4

> (ctype-sizeof (_fun _int -> _int))

8

> (ctype-alignof _int)

4

> (ctype-alignof (_fun _int -> _int))

8

procedure

(ctype->layout type)

 → (flat-rec-contract rep symbol? (listof rep))
  type : ctype?
返回一个值来描述该类型的最终 C 表示法。 它可以是以下任何一个符号:

'int8 'uint8 'int16 'uint16 'int32 'uint32 'int64 'uint64
'float 'double 'bool 'void 'pointer 'fpointer
'bytes 'string/ucs-4 'string/utf-16

结果也可以是一个列表,它描述了一个 C 结构,其元素的表示在列表中按顺序提供。 最后,结果可以是一个大小为 2 的向量,包含一个元素表示,后面是一个精确的整数。

Examples:
> (ctype->layout _int)

'int32

> (ctype->layout _void)

'void

> (ctype->layout (_fun _int -> _int))

'fpointer

procedure

(compiler-sizeof sym) → exact-nonnegative-integer?

  sym : (or/c symbol? (listof symbol?))
sym 的可能值是 'int, 'char, 'wchar, 'short, 'long, '*, 'void, 'float, 'double, 或符号列表,如　'(long long)　。 其结果是根据当前平台的 C 语言 sizeof 操作符来确定相应类型的大小。 compiler-sizeof 操作应该用来收集关于当前平台的信息, 比如将 _int 这样的别名类型定义为　_int32　这样的已知类型。

Examples:
> (compiler-sizeof 'int)

4

> (compiler-sizeof '(long long))

8
