5.17.2 COM 类和接口

 (require ffi/unsafe/com) package: base
ffi/unsafe/com 库导出了 ffi/com 的全部内容, 而且它还支持直接、基于 FFI 的对 COM 对象方法的调用。

5.17.2.1 描述 COM 接口

syntax

(define-com-interface (_id _super-id)
  ([method-id ctype-expr maybe-alloc-spec] ...))
 
maybe-alloc-spec = 
  | #:release-with-function function-id
  | #:release-with-method method-id
  | #:releases
将 _id 定义为扩展 _super-id 的接口, 其中 _super-id 通常是 _IUnknown, 并且包括由 method-id 命名的方法。 _id 和 _super-id 的标识符必须以下划线开始。 一个 _super-id _vt 也必须被定义,用于派生一个虚拟方法表类型。

method-id 的顺序必须符合 COM interface 的规范,不包括从 _super-id 继承的方法。 由 ctype-expr 产生的每一个不是 _fpointer 的方法类型必须是一个函数类型, 其第一个参数是 "self" 指针,通常用 _mfun 或 _hmfun 构建。

The define-com-interface 形式为每个 ctype-expr 不是 _fpointer 的方法绑定了 _id,id ?, _id -pointer, _id _ vt (用于虚拟方法表), _id _ vt-pointer 和 method-id 。 (换句话说,使用 _fpointer 作为你不需要调用的接口方法的占位符。) 接口的一个实例将具有 _id -pointer 类型。 每个定义的 method-id 都被绑定到一个类似函数的宏上, 该宏希望将 _id -pointer 作为其第一个参数,并将方法参数作为其余参数。

maybe-alloc-spec 描述了一个方法的分配和终结信息,与 ffi/unsafe/alloc 相似。 如果 maybe-alloc-spec 是 #:release-with-function function-id, 那么 function-id 将被用来解除方法产生的结果的分配,除非该结果在变得不可及之前被明确地解除分配; 例如, #:release-with-function Release 适合于返回一个最终必须被释放的 COM 接口引用的方法。 #:release-with-method method-id 的形式类似, 只是去分配者是与分配方法相同的对象上的方法(即其他 method-id 之一或继承的方法)。 一个 #:releases 注解表示一个方法是一个去分配器 (因此,如果使用该方法明确地去分配一个值,它就不应该被自动去分配)。

参见 COM 接口实例 ,了解使用 define-com-interface 的例子。

5.17.2.2 获取 COM 接口引用

procedure

(QueryInterface iunknown    
  iid    
  intf-pointer-type) → (or/c cpointer? #f)
  iunknown : com-iunknown?
  iid : iid?
  intf-pointer-type : ctype?
试图为给定的 COM object 提取一个 COM interface 指针。 如果该对象不支持所请求的接口,结果是 #f ,否则将被转换为 intf-pointer-type 的类型。

特定的 IIDs 和 intf-pointer-type 类型是一起的。 例如, IID_IUnknown 与 _IUnknown-pointer 在一起。

对于一个非 #f 的结果, Release 函数是产生的指针的自动去分配器。 指针在被转换为 intf-pointer-type 之后被注册了一个去分配器, 这就是为什么 QueryInterface 接受 intf-pointer-type 参数的原因(因为转换产生了一个新的引用)。

procedure

(AddRef iunknown) → exact-positive-integer?

  iunknown : com-iunknown?

procedure

(Release iunknown) → exact-nonnegative-integer?

  iunknown : com-iunknown?
增加或减少 iunknown 上的引用计数,如果计数为零,返回新的引用计数并释放接口引用。

procedure

(make-com-object iunknown    
  clsid    
  [#:manage? manage?]) → com-object?
  iunknown : com-iunknown?
  clsid : (or/c clsid? #f)
  manage? : any/c = #t
将一个 COM object 转换为可以使用COM自动化函数的对象,例如 com-invoke 。

如果 manage? 为真,产生的对象将被注册到当前的托管人和一个终结器, 以便在托管人关闭或对象无法访问时调用 com-release 。

5.17.2.3 COM FFI 帮助器

syntax

(_wfun fun-option ... maybe-args type-spec ... -> type-spec
   maybe-wrapper)
像 _fun ,但增加了 #:abi winapi 。

syntax

(_mfun fun-option ... maybe-args type-spec ... -> type-spec
   maybe-wrapper)
像 _wfun 一样,但增加了一个 _pointer 类型(用于方法的 "self" 参数) 作为第一个参数的 type-spec 。

syntax

(_hfun fun-option ... type-spec ... -> id maybe-allow output-expr)

 
maybe-allow = 
  | #:allow [result-id allow?-expr]
像 _wfun 一样,但是对于一个返回 _HRESULT 的函数。 如果 #:allow 被指定,结果将被绑定到 result-id ,否则结果不能直接访问。

_hfun 形式处理外来调用的 _HRESULT 值,如下所示:

  • 如果结果是 0,或者指定了 #:allow ,并且 allow?-expr 产生了 #t, 那么 output-expr (如 _fun 的 maybe-wrapper )决定了结果。

  • 如果结果是 RPC_E_CALL_REJECTED 或 RPC_E_SERVERCALL_RETRYLATER, 则该调用会自动重试,最多重试 (current-hfun-retry-count) 次, 每次尝试之间有 (current-hfun-retry-delay) 秒延迟。

  • 否则,会使用 windows-error 并使用 id 作为失败函数的名称来引发错误。

Changed in version 6.2 of package base: 增加了 #:allow 和自动重试。

syntax

(_hmfun fun-option ... type-spec ... -> id output-expr)

和 _hfun 一样,但和 _mfun 一样, _pointer 被添加到第一个参数中。

parameter

(current-hfun-retry-count) → count

(current-hfun-retry-count exact-nonnegative-integer?) → void?
  exact-nonnegative-integer? : count

parameter

(current-hfun-retry-delay) → (>=/c 0.0)

(current-hfun-retry-delay secs) → void?
  secs : (>=/c 0.0)
确定 _hfun 的自动重试行为的参数。

Added in version 6.2 of package base.

procedure

(HRESULT-retry? r) → boolean?

  r : exact-nonnegative-integer?
如果 r 是 RPC_E_CALL_REJECTED 或 RPC_E_SERVERCALL_RETRYLATER,返回 #t ,否则返回 #f 。

Added in version 6.2 of package base.

5.17.2.4 COM 接口实例

下面是一个使用标准组件类别管理器列举已安装的属于不同系统定义类别的 COM 类的例子。 这个例子说明了通过 CLSID 实例化一个 COM 类,用 define-com-interface 描述 COM 接口, 并使用分配规范来确保即使遇到错误或程序被中断也能回收资源。

#lang racket/base
(require ffi/unsafe
         ffi/unsafe/com)
 
(provide show-all-classes)
 
; The function that uses COM interfaces defined further below:
 
(define (show-all-classes)
  (define ccm
    (com-create-instance CLSID_StdComponentCategoriesMgr))
  (define icat (QueryInterface (com-object-get-iunknown ccm)
                               IID_ICatInformation
                               _ICatInformation-pointer))
  (define eci (EnumCategories icat LOCALE_SYSTEM_DEFAULT))
  (for ([catinfo (in-producer (lambda () (Next/ci eci)) #f)])
    (printf "~a:\n"
            (cast (array-ptr (CATEGORYINFO-szDescription catinfo))
                  _pointer
                  _string/utf-16))
    (define eg
      (EnumClassesOfCategories icat (CATEGORYINFO-catid catinfo)))
    (for ([guid (in-producer (lambda () (Next/g eg)) #f)])
      (printf " ~a\n" (or (clsid->progid guid)
                          (guid->string guid))))
    (Release eg))
  (Release eci)
  (Release icat))
 
; The class to instantiate:
 
(define CLSID_StdComponentCategoriesMgr
  (string->clsid "{0002E005-0000-0000-C000-000000000046}"))
 
; Some types and variants to match the specification:
 
(define _ULONG _ulong)
(define _CATID _GUID)
(define _REFCATID _GUID-pointer)
(define-cstruct _CATEGORYINFO ([catid _CATID]
                               [lcid _LCID]
                               [szDescription (_array _short 128)]))
 
; —— IEnumGUID ——-
 
(define IID_IEnumGUID
  (string->iid "{0002E000-0000-0000-C000-000000000046}"))
 
(define-com-interface (_IEnumGUID _IUnknown)
  ([Next/g (_mfun (_ULONG = 1) ; simplifed to just one
                  (guid : (_ptr o _GUID))
                  (got : (_ptr o _ULONG))
                  -> (r : _HRESULT)
                  -> (cond
                       [(zero? r) guid]
                       [(= r 1) #f]
                       [else (windows-error "Next/g failed" r)]))]
   [Skip _fpointer]
   [Reset _fpointer]
   [Clone _fpointer]))
 
; —— IEnumCATEGORYINFO ——-
 
(define IID_IEnumCATEGORYINFO
  (string->iid "{0002E011-0000-0000-C000-000000000046}"))
 
(define-com-interface (_IEnumCATEGORYINFO _IUnknown)
  ([Next/ci (_mfun (_ULONG = 1) ; simplifed to just one
                   (catinfo : (_ptr o _CATEGORYINFO))
                   (got : (_ptr o _ULONG))
                   -> (r : _HRESULT)
                   -> (cond
                       [(zero? r) catinfo]
                       [(= r 1) #f]
                       [else (windows-error "Next/ci failed" r)]))]
   [Skip _fpointer]
   [Reset _fpointer]
   [Clone _fpointer]))
 
; —— ICatInformation ——-
 
(define IID_ICatInformation
  (string->iid "{0002E013-0000-0000-C000-000000000046}"))
 
(define-com-interface (_ICatInformation _IUnknown)
  ([EnumCategories (_hmfun _LCID
                           (p : (_ptr o _IEnumCATEGORYINFO-pointer))
                           -> EnumCategories p)]
   [GetCategoryDesc (_hmfun _REFCATID _LCID
                            (p : (_ptr o _pointer))
                            -> GetCategoryDesc
                            (begin0
                             (cast p _pointer _string/utf-16)
                             (SysFreeString p)))]
   [EnumClassesOfCategories (_hmfun (_ULONG = 1) ; simplifed
                                    _REFCATID
                                    (_ULONG = 0) ; simplifed
                                    (_pointer = #f)
                                    (p : (_ptr o
                                               _IEnumGUID-pointer))
                                    -> EnumClassesOfCategories p)
                            #:release-with-function Release]
   [IsClassOfCategories _fpointer]
   [EnumImplCategoriesOfClass _fpointer]
   [EnumReqCategoriesOfClass _fpointer]))
