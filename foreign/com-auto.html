5.17.1 COM 自动化

 (require ffi/com) package: base
ffi/com 库建立在 COM 自动化的基础上, 提供对支持 IDispatch 接口的 COM 对象的安全使用。

ffi/com 库是基于 Paul Steckler 的 MysterX 库。 MysterX 包含在 Racket 中,但已经废弃,它将在下一个版本中被一个部分兼容的库所取代,该库将重定向到这个库。

5.17.1.1 GUIDs, CLSIDs, IIDs 和 ProgIDs

procedure

(guid? v) → boolean?

  v : any/c

procedure

(clsid? v) → boolean?

  v : any/c

procedure

(iid? v) → boolean?

  v : any/c
如果 v 是一个代表 GUID 的结构,返回 #t ,否则返回 #f 。 clsid? 和 iid? 函数与 guid? 相同。

一个 GUID 对应于不安全层的一个 _GUID 结构。

procedure

(string->guid str) → guid?

  str : string?

procedure

(string->clsid str) → clsid?

  str : string?

procedure

(string->iid str) → iid?

  str : string?
将形式为 "{00000000-0000-0000-0000-0000000000}" 的字符串转换为 GUID, 其中每个 0 可以是一个十六进制数字。如果 str 没有预期的形式,就会引发 exn:fail 异常。

string->clsid 和 string->iid 函数与 string->guid 相同。

procedure

(guid->string g) → string?

  g : guid?
将一个 GUID 转换为其字符串形式。

procedure

(guid=? g1 g2) → boolean?

  g1 : guid?
  g2 : guid?
判断 g1 和 g2 是否代表同一个 GUID 。

procedure

(progid->clsid progid) → clsid?

  progid : string?

procedure

(clsid->progid clsid) → (or/c string? #f)

  clsid : clsid?
将一个 ProgID 转换为 CLSID ,反之亦然。 不是每个 COM class 都有一个 ProgID, 所以 clsid->progid 的结果可能是 #f 。

progid->clsid 函数接受一个无版本的 ProgID, 在这种情况下,它产生最新的可用版本的 CLSID 。 clsid->progid 函数总是产生一个带有其版本的 ProgID 。

5.17.1.2 COM 对象

procedure

(com-object? obj) → boolean?

  obj : com-object?
如果参数代表一个 COM object,返回 #t ,否则返回 #f 。

procedure

(com-create-instance clsid-or-progid [where]) → com-object?

  clsid-or-progid : (or/c clsid? string?)
  where : (or/c (one-of/c 'local 'remote) string?) = 'local
返回由 clsid-or-progid 指定的 COM class 的一个实例,它是一个 CLSID 或 ProgID 。

可选的 where 参数表示运行该实例的位置,可以是 'local, 'remote 或表示机器名称的字符串。更多信息见 远程 COM 服务器 (DCOM) 。

一个对象可以通过这种方式为任何 COM 类创建,但是像 com-invoke 这样的功能只有在该对象支持 IDispatch COM 自动化接口的情况下才能工作。

创建的对象被注册到当前的托管人那里,托管人保留对该对象的引用,直到它被 com-release 释放或托管人被关闭。

procedure

(com-release obj) → void?

  obj : com-object?
释放给定的 COM object 。 给定的 obj 随后不能使用,并且底层的 COM 对象被销毁,除非它的引用计数被增加(通过 COM 方法或不安全操作)。

如果 obj 已经被释放, com-release 没有任何作用。

procedure

(com-get-active-object clsid-or-progid) → com-object?

  clsid-or-progid : (or/c clsid? string?)
像 com-create-instance 一样,但得到一个现有的活动对象(总是本地的),而不是创建一个新的。

procedure

(com-object-clsid obj) → clsid?

  obj : com-object?
返回 obj 所实例化的 COM 类的 "CLSID" ,如果不知道 COM 类,则引发错误。

procedure

(com-object-set-clsid! obj clsid) → void?

  obj : com-object?
  clsid : clsid?
将 obj 的 COM CLSID 设置为 clsid 。 当 COM 事件处理程序只能获得关于对象的 COM 类的模糊信息时,这很有用。

procedure

(com-object-eq? obj1 obj2) → boolean?

  obj1 : com-object?
  obj2 : com-object?
如果 obj1 和 obj2 引用的是同一个 COM object, 返回 #t ,否则返回 #f 。

如果对一个 COM 对象的两个引用根据 com-object-eq? 是相同的,那么根据 equal? ,它们也是相同的。 然而,两个 com-object-eq? 引用不一定是 eq? 。

procedure

(com-type? v) → boolean?

  v : any/c
如果 v 代表关于 COM 对象类型的反射性信息,则返回 #t ,否则返回 #f 。

procedure

(com-object-type obj) → com-type?

  obj : com-object?
返回一个独立于对象本身的 COM 对象的类型表示。

procedure

(com-type=? t1 t2) → boolean?

  t1 : com-type?
  t2 : com-type?
如果 t1 和 t2 代表相同的类型信息,返回 #t ,否则返回 #f 。

 → 
(list/c '-> (listof type-description?)
            type-description?)

如果可能的话,参数的类型通过 com-method-type 确定的, v 中的 type-describe 包装器被简单地替换成它们所包装的值。 如果 com-method-type 不能提供类型,那么将为每个 v 推断类型, 并注意 v 中任何 type-describe 包装器的描述。

procedure

(com-get-properties obj/type) → (listof string?)

  obj/type : (or/c com-object? com-type?)
返回一个字符串列表,表示 obj/type 中可读属性的名称。

procedure

(com-get-property-type obj/type

(com-get-property obj property ...+) → any/c

每个 property 都是一个属性名字符串,或者是一个以属性名字符串开始并以参数化属性的参数继续的一个列表。

(com-set-properties obj/type) → (listof string?)

  obj/type : (or/c com-object? com-type?)
返回一个表示 obj/type 中可写属性名称的字符串列表。

procedure

(com-set-property-type obj/type 
  property-name) 
 → (list/c '-> (list/c type-description?) 'void)
  obj/type : (or/c com-object? com-type?)
  property-name : string?
像 com-method 的结果一样为 property-name 返回一个类型,其中唯一的参数类型与属性值类型相对应。 关于符号的信息请参见 COM 类型 。

procedure

(com-set-property! obj property ...+ v) → void?

  obj : com-object?
  property : 
(or/c string?

通过跟随 property 将 obj 中最终属性的值设置为 v , 其中每个中间属性的值必须是一个 COM 对象。 一个 property 可以是一个列表而不是一个字符串,以表示一个参数化的属性及其参数。

procedure

(com-events obj/type) → (listof string?)

(com-event-type obj/type event-name)

 → (list/c '-> (listof type-description?) 'void)
  obj/type : (or/c com-object? com-type?)
  event-name : string?
返回一个列表,表示 obj/type 中指定事件的类型。 在 '-> 后面的列表代表参数类型。更多信息请参见 COM 类型 。

procedure

(com-event-executor? v) → boolean?

  v : any/c
如果 v 是一个 COM event executor,返回 #t ,该执行器排队等待事件回调。 一个 COM event executor com-ev-ex 在 sync 的意义上是一个可同步的事件, (sync com-ev-ex) 返回一个就绪回调的thunk。

procedure

(com-make-event-executor) → com-event-executor?

创建一个新的 COM event executor ,与 com-register-event-callback 一起使用。

procedure

(com-register-event-callback obj    
  name    
  proc    
  com-ev-ex) → void?
  obj : com-object?
  name : string?
  proc : procedure?
  com-ev-ex : com-event-executor?
为 obj 中 name 命名的事件注册一个回调。 当事件发生时,对事件参数(取决于 obj 和 name)的 proc 调用将在 com-ev-ex 中排队。 在 com-ev-ex 上同步产生一个thunk,将 proc 应用于事件参数并返回结果。

每个 obj 和 name 组合只能注册一个回调。

事件回调的注册依赖于事先注册由 Racket 分发的 "myssink.dll" 实现的 COM 类。 (该 DLL 对所有 Racket 版本都是一样的)。

procedure

(com-unregister-event-callback obj name) → void?

  obj : com-object?
  name : string?
移除 obj 中 name 的任何现有回调。

5.17.1.8 远程 COM 服务器 (DCOM)

com-create-instance 的可选 where 参数可以是 'remote 。 在这种情况下,服务器实例会在注册表键值所给的位置运行

HKEY_CLASSES_ROOT\AppID\‹CLSID›\RemoteServerName

其中 ‹CLSID› 是应用程序的 CLSID。 这个键可以用 dcomcnfg 工具来设置。 在 dcomcnfg 中,在 Applications 标签上挑选要运行的应用程序, 然后点击 Properties 按钮。 在 Location 选项卡上,选择 Run application on the following computer, 并输入机器名称。

要运行 COM 远程服务器,客户机上的注册表必须包含一个条目

HKEY_CLASSES_ROOT\CLSID\‹CLSID›

其中 ‹CLSID› 是服务器的 CLSID。 服务器应用程序本身不需要安装在客户机上。

有一些与 DCOM 有关的配置问题。 关于如何为 DCOM 设置客户机和服务器的更多信息,请参见

http://www.distribucon.com/dcom95.html

5.17.1.9 COM 类型

在 com-method-type 这样的函数的结果中,符号被用来表示各种原子类型:

  • 'int — 一个 32 位有符号的整数

  • 'unsigned-int — 一个 32 位无符号的整数

  • 'short-int — 一个 16 位有符号的整数

  • 'unsigned-short — 一个 16 位无符号的整数

  • 'signed-char — 一个 8 位有符号的整数

  • 'char — 一个 8 位无符号整数

  • 'long-long — 一个 64 位有符号整数

  • 'unsigned-long-long — 一个 64 位无符号整数

  • 'float — 一个 32 位的浮点数字

  • 'double — 一个 64 位的浮点数字

  • 'currency — 一个精确的数字,当乘以 10,000 时,是一个 64 位有符号的整数

  • 'boolean — 一个布尔值

  • 'string — 一个字符串

  • 'date — 一个 date 或 date*; 当转换为 date* 时,时区被报告为 "UTC" ， year-day 字段为 0

  • 'com-object — 一个 COM object ,如 com-object?

  • 'iunknown — 和 'com-object 一样, 但也接受一个 IUnknown 的指针,如 com-iunknown?

  • 'com-enumeration — 一个 32 位有符号的整数

  • 'any — 以上任何一种,或者当不嵌套在一个数组类型中时,是一个数组

  • '... — 和 'any 一样,但是当它出现在参数类型序列的末尾时,允许前面的类型出现 0 次或多次

  • 'void — 没有值

一个用 'box 包裹的列表中的类型符号,比如 '(box int) ,是一个逐次调用的参数。 当方法返回时,为该参数提供的 box 会被更新为一个新的值。

一个用 'opt 包裹的类型,例如 '(opt (box int)), 是一个可选的参数。 该参数可以被省略或者用 com-omit 替换。

一个用 'array 和一个正整数包裹的类型,例如 '(array 7 int), 代表一个要作为 COM 数组使用的值的向量。 可以用 '? 代替长度整数来支持任何长度的向量。 具有非 '? 长度的数组类型可以被嵌套,以指定一个由嵌套向量表示的多维数组。

用 'variant 包裹的类型,例如 '(variant (array 7 int)), 与被包裹的类型相同,但是 'variant 包裹在 'array 类型中,可以防止构建另一个数组维度。 例如, '(array 2 (array 3 int)) 是一个二维的整数数组, 但是 '(array 2 (variant (array 3 int))) 是一个一维数组,其元素是一维的整数数组。

当类型信息不可用时,像 com-invoke 这样的函数从参数中推断出类型描述。 推断选择 'boolean 用于布尔运算; 'int, 'unsigned-int, 'long-long, 'unsigned-long-long 中第一个适合精确整数的类型; 'double 用于非精确实数; 'string 用于字符串; 'com-object 和 'iunknown 用于相应的 COM 对象引用; 以及 'array 类型用于向量,其中元素类型由向量值推断, 如果任何两个元素的推断类型不同,则诉诸于 'any 。

procedure

  v : any/c

procedure

  v : any/c

type-described? 谓词识别用 type-describe 产生的包装器, type-described-value 和 type-described-description 提取 type-describe 值的值和描述部分。

一个 type-describe 包装器结合了一个基本值和一个类型描述。 当 COM 自动化中的方法 com-invoke 或属性 com-set-property! 没有可用的类型时, 描述被用来代替自动推断的 COM 参数类型。 封装器可以放在一个即时值上,也可以放在一个盒子或矢量中的值上。

5.17.1.10 类显示名称

 (require ffi/com-registry) package: base
ffi/com-registry 库提供了一个从 coclass 名称到 CLSIDs 的映射,以便与旧的 MysterX 接口兼容。

一个 coclass 名称对应于一个 COM 类的显示名称;显示名称并不唯一地映射到一个 COM 类,而且一些 COM 类没有显示名称。
