XML: 解析和编写

Paul Graunke and Jay McCarthy


吴华均 译

 (require xml) package: base

xml 库提供了解析和生成 XML 的功能。 XML 可以被表示为 document 结构类型的实例,或者被称为 X-expression 的一种 S 表达式。

xml 库不提供文档类型声明(DTD)处理,包括在读取的文档中保存 DTD,或验证。 它也不扩展用户定义的实体或在属性中读取用户定义的实体。 它也不解释命名空间。

procedure

(valid-char? x) → boolean?

  x : any/c
如果 x 是一个精确的非负整数, 其在 UTF-8 下的字符解释是来自集合 ([#x1-#xD7FF] | [#xE000-#xFFFD] | [#x10000-#x10FFFF]), 根据 XML 1.1 规范的第 2.2 节,返回 true。

The string 字段被认为是 <![CDATA[‹content›]]> 的形式, 并对 ‹content› 进行适当的引号。 否则, write-xml 会产生不正确的输出。

procedure

(xexpr? v) → boolean?

  v : any/c
如果 v 是一个 X-expression,返回 #t ,否则返回 #f 。

下面的语法描述了创建 X-expressions 的表达式:

  xexpr = string
  | (list symbol (list (list symbol string) ...) xexpr ...)
  | (cons symbol (list xexpr ...))
  | symbol
  | valid-char?
  | cdata
  | misc

一个 string 是字面数据。 当转换为 XML 流时,数据的字符将根据需要被转义。

一个对代表一个元素,可以选择带有属性。 每个属性的名称由一个符号表示,其值由一个字符串表示。

一个 symbol 代表一个符号实体。 例如, 'nbsp 代表 &nbsp; 。

一个 valid-char? 代表一个数字实体。 例如, #x20 代表 &#x20; 。

一个 cdata 是 cdata 结构类型的一个实例, 一个 misc 是 comment 或 p-i 结构类型的一个实例。

value

xexpr/c : contract?

一个像 xexpr? 的契约,除了在值不是 X-expression 时产生一个更好的错误信息。

3 读取和写入 XML

procedure

(read-xml [in]) → document?

  in : input-port? = (current-input-port)
从给定或当前的输入端口读入一个 XML 文档。 XML 文档正好包含一个元素,如果输入流有零个元素或多个元素,则引发 xml-read:error 。

畸形的 xml 会以 ‹l›.‹c›/‹o› 的形式报告源位置, 其中 ‹l›, ‹c› 和 ‹o› 分别是行号、列号和下一个端口位置, 由 port-next-location 返回。

除了从输入端口读出的 eof 以外的任何非字符都会出现在文档内容中。 这样的特殊值只可能出现在 XML 内容可能出现的地方。 关于创建返回非字符值的端口的信息,请参见 make-input-port 。

Example:
> (xml->xexpr (document-element
               (read-xml (open-input-string
                          "<doc><bold>hi</bold> there!</doc>"))))

'(doc () (bold () "hi") " there!")

procedure

(read-xml/document [in]) → document?

  in : input-port? = (current-input-port)
像 read-xml 一样,除了阅读器在单个元素之后停止,而不是试图在元素之后读取 "杂七杂八" 的 XML 内容。 由 read-xml/document 返回的文档总是有一个空的 document-misc 。

procedure

(read-xml/element [in]) → element?

  in : input-port? = (current-input-port)
从端口读取一个单一的 XML 元素。 读取的下一个非空白字符必须开始一个 XML 元素,但输入端口可以在元素之后包含其他数据。

procedure

(syntax:read-xml [in #:src source-name]) → syntax?

  in : input-port? = (current-input-port)
  source-name : any/c = (object-name in)
读入一个 XML 文档,并产生一个语法对象版本(像 read-syntax) 的 X-expression 。

procedure

(syntax:read-xml/element [in    
  #:src source-name]) → syntax?
  in : input-port? = (current-input-port)
  source-name : any/c = (object-name in)
像 syntax:real-xml 一样,但它读一个 XML 元素,像 read-xml/element 一样。

关于 indentation 的解释 display-xml/content 。

procedure

(display-xml/content content    
  [out    
  #:indentation indentation]) → void?
  content : content/c
  out : output-port? = (current-output-port)
  indentation : (or/c 'none 'classic 'peek 'scan) = 'classic
将文档内容写到给定的输出端口。

缩进可以使输出更加可读,尽管当空白显著时技术上不太正确。 四种 indentation 模式如下:

  • 'none — 不添加任何空白。 这是唯一能保证在所有情况下 100% 准确的模式。

  • 'classic — 几乎每一个节点周围都添加了空白。 这种模式主要是为了兼容。

  • 'scan — 如果一个 element? 的任何一个子节点是 pcdata? 或 entity?, 那么该元素内部将不添加空白。 这种模式对于不包含混合内容的 XML 来说效果很好,但是 'peek 应该同样好,而且更快。

  • 'peek — 和 'scan 一样,只是只检查第一个子元素。 这种模式对不包含混合内容的 XML 效果很好。

Examples:
> (define example-data
    '(root (a "nobody")
           (b "some" "body")
           (c "any" (i "body"))
           (d (i "every") "body")))
> (define (show indentation [data example-data])
    (display-xml/content (xexpr->xml data)
                         #:indentation indentation))
; 
; `none` is guaranteed to be accurate:
> (show 'none)

<root><a>nobody</a><b>somebody</b><c>any<i>body</i></c><d><i>every</i>body</d></root>

; 
; `classic` adds the most whitespace.
; Even the 'nobody' pcdata has whitespace added:
> (show 'classic)

<root>

  <a>

    nobody

  </a>

  <b>

    some

    body

  </b>

  <c>

    any

    <i>

      body

    </i>

  </c>

  <d>

    <i>

      every

    </i>

    body

  </d>

</root>

; 
; `peek` cannot see that <d> contains a pcdata child:
> (show 'peek)

<root>

  <a>nobody</a>

  <b>somebody</b>

  <c>any<i>body</i></c>

  <d>

    <i>every</i>

    body

  </d>

</root>

; 
; `scan` sees that <d> contains a pcdata child:
> (show 'scan)

<root>

  <a>nobody</a>

  <b>somebody</b>

  <c>any<i>body</i></c>

  <d><i>every</i>body</d>

</root>

Examples:
> (define html-data '(span (i "no") (b "body")))
> (show 'scan html-data)

<span>

  <i>no</i>

  <b>body</b>

</span>

4 XML 和 X 表达式的转换

parameter

(permissive-xexprs) → boolean?

(permissive-xexprs v) → void?

如果设置为非假,那么 xml->xexpr 将允许非 XML 对象, 如其他结构,出现在转换后的 XML 内容中,并在产生的 “X-expression” 。

(xml->xexpr content) → xexpr/c

procedure

(xexpr->xml xexpr) → content/c

  xexpr : xexpr/c
将一个 X-expression 转换为 XML 内容。

procedure

(xexpr->string xexpr) → string?

  xexpr : xexpr/c
将一个 X-expression 转换为一个包含 XML 的字符串。

procedure

(string->xexpr str) → xexpr/c

  str : string?
将用字符串表示的 XML 转换为 X-expression 。

procedure

(xml-attribute-encode str) → string?

  str : string?
按照XML属性的要求对字符串进行转义。

对属性字符串进行的转义与对主体字符串进行的转义略有不同,因为双引号必须被转义,否则它们将终止包围的字符串。

请注意,这种转换是由 xexpr->string 在属性位置上自动执行的, 因此你不太可能需要这个函数,除非你使用 include-template 将字符串直接插入到 HTML 的属性位置。

Added in version 6.6.0.7 of package base.

procedure

((eliminate-whitespace [tags choose]) elem) → element?

  tags : (listof symbol?) = empty
  choose : (boolean? . -> . boolean?) = (λ (x) x)
  elem : element?
有些元素不应该包含任何文本,只应该包含其他标签,只是它们经常包含空白,用于格式化。 给定一个标签名称的列表作为 tag ,给定身份函数作为 choose, eliminate-whitespace 产生一个函数,从这些元素中过滤出完全由空白组成的 PCDATA, 如果出现任何非空白的文本,它就会引发一个错误。 传入 not 作为 choose 可以过滤所有未在 tags 列表中命名的元素。 使用 (lambda (x) #t) 作为 choose 作为选择过滤所有元素, 而不考虑 tags 列表。

procedure

(validate-xexpr v) → #t

  v : any/c
如果 v 是一个 X-expression, 其结果是 #t 。 否则, exn:invalid-xexpr 将被提出,并有一个 “Expected ‹something›, 给出 ‹something-else›” 的消息。 异常的 code 域是 v 中引起异常的部分。

Examples:
> (validate-xexpr '(doc () "over " (em () "9000") "!"))

#t

> (validate-xexpr #\newline)

Expected a string, symbol, valid numeric entity, comment,

processing instruction, or list, given #\newline

procedure

(correct-xexpr? v success-k fail-k) → any/c

  v : any/c
  success-k : (-> any/c)
  fail-k : (exn:invalid-xexpr? . -> . any/c)
和 validate-xexpr 一样,除了在每个有效的叶子上调用 success-k , 在无效的叶子上调用 fail-k ； fail-k 可以返回一个值,而不是引发一个异常或其他转义。 来自叶子的结果与 and 结合,得出最终结果。

parameter

(empty-tag-shorthand)

 → (or/c (one-of/c 'always 'never) (listof symbol?))
(empty-tag-shorthand shorthand) → void?
  shorthand : (or/c (one-of/c 'always 'never) (listof symbol?))
一个参数,决定输出函数是否应该对没有内容的元素使用 <‹tag›/> 标签符号而不是 <‹tag›></‹tag›> 。

当该参数设置为 'always 时,总是使用缩写的符号。当设置为 'never 时,从不生成缩写符号。 当设置为提供一个符号列表时,名称在列表中的标签被缩写。

缩写形式是首选的 XML 记号。 然而,大多数为 HTML 设计的浏览器只有在文档使用两种格式的混合时才会正确地呈现 XHTML。 html-empty-tags 常量包含 W3 联盟推荐的应使用缩写的 XHTML 标签列表。 这个列表是 empty-tag-shorthand 的默认值。

value

html-empty-tags : (listof symbol?)

参见 empty-tag-shorthand 。

Example:
> (parameterize ([empty-tag-shorthand html-empty-tags])
    (write-xml/content (xexpr->xml `(html
                                      (body ((bgcolor "red"))
                                        "Hi!" (br) "Bye!")))))

<html><body bgcolor="red">Hi!<br />Bye!</body></html>

parameter

(read-comments) → boolean?

parameter

(xml-count-bytes) → boolean?

(xml-count-bytes count-bytes?) → void?

你可能想使用 #t 例如,你将把这些偏移量传达给一个 C 程序, 它可以更容易地处理字符流中的字节偏移量,而不是 UTF-8 字符偏移量。

parameter

(xexpr-drop-empty-attributes drop?) → void?

6 PList 库

 (require xml/plist) package: base

xml/plist 库提供了读写符合 plist DTD 的 XML 文档的能力, plist DTD 是用来存储字符串-值关联的字典的。 这种格式被 Mac OS(包括操作系统和其应用程序)用来存储各种数据。

一个 plist value 值是一个可以由匹配以下 pl-expr 语法的表达式创建的值, 其中由 dict-expr 创建的值是一个 plist dictionary:

  pl-expr = string
  | (list 'true)
  | (list 'false)
  | (list 'integer integer)
  | (list 'real real)
  | (list 'data string)
  | (list 'date string)
  | dict-expr
  | (list 'array pl-expr ...)
     
  dict-expr = (list 'dict assoc-pair ...)
     
  assoc-pair = (list 'assoc-pair string pl-expr)

procedure

(plist-value? any/c) → boolean?

  any/c : v
如果 v 是一个 plist value,返回 #t ,否则返回 #f 。

procedure

(plist-dict? any/c) → boolean?

  any/c : v
如果 v 是一个 plist dictionary,则返回 #t ,否则返回 #f 。

procedure

(read-plist in) → plist-value?

  in : input-port?
从一个端口读取一个 plist,并产生一个 plist value 。

procedure

(write-plist dict out) → void?

  dict : plist-value?
  out : output-port?
写一个 plist value 到给定的端口。

Examples:
> (define my-dict
    `(dict (assoc-pair "first-key"
                       "just a string with some  whitespace")
           (assoc-pair "second-key"
                       (false))
           (assoc-pair "third-key"
                       (dict))
           (assoc-pair "fourth-key"
                       (dict (assoc-pair "inner-key"
                                         (real 3.432))))
           (assoc-pair "fifth-key"
                       (array (integer 14)
                              "another string"
                              (true)))
           (assoc-pair "sixth-key"
                       (array))
           (assoc-pair "seventh-key"
                       (data "some data"))
           (assoc-pair "eighth-key"
                       (date "2013-05-10T20:29:55Z"))))
> (define-values (in out) (make-pipe))
> (write-plist my-dict out)
> (close-output-port out)
> (define new-dict (read-plist in))
> (equal? my-dict new-dict)

#t

7 简单的 X 表达式路径查询

 (require xml/path) package: base

这个库为 X 表达式提供了一个简单的路径查询库。

value

se-path? : contract?

一个符号序列后面有一个可选的关键字。

符号的前缀指定了一条从叶子到根的隐含任意序列的标签路径。 最后的可选关键字指定了一个属性。

procedure

(se-path*/list p xe) → (listof any/c)

  p : se-path?
  xe : xexpr?
返回一个由 X 表达式 xe 中的路径 p 指定的所有值的列表。

procedure

(se-path* p xe) → any/c

  p : se-path?
  xe : xexpr?
返回 (se-path*/list p xe) 的第一个答案。

Examples:
> (define some-page
    '(html (body (p ([class "awesome"]) "Hey") (p "Bar"))))
> (se-path*/list '(p) some-page)

'("Hey" "Bar")

> (se-path* '(p) some-page)

"Hey"

> (se-path* '(p #:class) some-page)

"awesome"

> (se-path*/list '(body) some-page)

'((p ((class "awesome")) "Hey") (p "Bar"))

> (se-path*/list '() some-page)

'((html (body (p ((class "awesome")) "Hey") (p "Bar")))

  (body (p ((class "awesome")) "Hey") (p "Bar"))

  (p ((class "awesome")) "Hey")

  "Hey"

  (p "Bar")

  "Bar")
