6.16 用于具体化用户特定文档的 API

 (require setup/materialize-user-docs)
  package: racket-index

Added in version 1.1 of package racket-index.

procedure

(materialize-user-docs 
  [on-setup 
  #:skip-user-doc-check? skip-user-doc-check?]) 
 → void?
  on-setup : ((-> boolean?) -> any) = (lambda (setup) (setup))
  skip-user-doc-check? : any/c = #f
检查在 (find-user-doc-dir) 是否已经存在一个用户特定的文档入口点, 如果没有, 则以创建入口点的模式运行 raco setup (内容与安装的文档入口点相同)。 如果 skip-user-doc-check? 不是 #f, 那么就跳过对用户特定文档入口点的检查。

raco setup 的运行被打包成一个 thunk,提供给 on-setup, 它可以适当地调整当前的输出和错误端口,并检查 thunk 的结果是否成功。

如果文档入口点已经存在于 (find-user-doc-dir) 中,则不会调用 on-setup 参数。

Changed in version 1.1 of package racket-index: 增加了 skip-user-doc-check? 参数。
