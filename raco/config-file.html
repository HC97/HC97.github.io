19 安装配置和搜索路径

在安装时,Racket 可执行文件中内置了一个 configuration directory 路径, 它的位置可以通过 PLTCONFIGDIR 目录或 --config/-G 命令行标志改变。 使用 find-config-dir 来定位配置目录。

修改 configuration directory 下的 "config.rktd" 文件以配置其他目录,如下所述。 使用 setup/dirs 库(它结合了来自配置文件和其他来源的信息)来定位配置的目录, 而不是直接读取 "config.rktd" 。 一个 "config.rktd" 文件也可以出现在目录中 (build-path (find-system-path 'addon-dir) "etc"), 但它只控制 find-addon-tethered-console-bin-dir 和 find-addon-tethered-gui-bin-dir 的结果。

main collection directory 的路径是内置在 Racket 可执行文件中的, 它可以通过 --collects/-X 标志改变,所以它在 "config.rktd" 中没有条目。 大多数在 "config.rktd" 中指定的路径都有默认值,是相对于主集合目录的。 因此,configuration directory 和 main collection directory 的路径共同决定了 Racket 的配置。

配置目录下的 "config.rktd" 文件应该包含一个可 read 的哈希表, 其中有以下任何一个符号键,相对路径是相对于 main collection directory:

  • 'installation-name — 安装名称的字符串,用于确定用户和版本的特定路径, 如 find-library-collection-paths 产生的初始路径和在 user package scope 内安装的包的位置。 默认是 (version) 。

  • 'collects-search-dirs — 一个代表集合搜索路径的路径、字符串、字节串或 #f 的列表。 列表中的每一个 #f ,如果有的话,会被替换成 main collection directory 。

  • 'lib-dir — main library directory 的路径、字符串或字节字符串。 它默认为 main collection directory 的 "lib" 同级目录。

  • 'lib-search-dirs — 一个路径、字符串、字节字符串或 #f 的列表, 代表包含外国库的目录的搜索路径。 列表中的每一个 #f ,如果有的话,将被替换为默认的搜索路径, 即用户和版本特定的 "lib" 目录,然后是 main library directory 。

  • 'dll-dir — 包含主可执行文件共享库的目录的路径、字符串或字节字符串。 它的默认值是 main library directory 。

  • 'share-dir — 主共享文件目录的路径、字符串或字节字符串,它通常包括已安装的软件包。 它的默认值是主集合目录的 "share" 同级目录。

  • 'share-search-dirs — 类似于 'lib-search-dirs, 其中 #f 被默认的搜索路径所取代,它是一个用户和版本特定的目录, 后面是通过 'share-dir 可能配置的目录。

    Added in version 8.1.0.6.

  • 'links-file — collection links file 的路径、 字符串或字节字符串。 它默认为 main shared-file directory 下的 "links.rktd" 文件。

  • 'links-search-files — 与 'lib-search-dirs 类似, 但用于 collection links file 。 一个 #f 被默认的搜索路径所取代,它具有通过 'links-file 可能配置的链接文件。 一个用户和版本特定的链接文件总是被添加到搜索的开始。

  • 'pkgs-dir — 具有 installation package scope 的路径、字符串或字节字符串。 它默认为主共享文件目录下的 "pkgs" 。

  • 'pkgs-search-dirs — 与 'lib-search-dirs 类似, 但适用于大致属于 installation package scope 的软件包。 更确切地说,列表中的 #f 值被替换为 'pkgs-dir 指定的目录, 搜索列表中的那个点对应的是 installation 范围。 列表中 #f 值之前或之后的路径可以被选为作用域,在列表中该路径的点开始搜索。 在 'pkgs-search-dirs 中列出的目录通常有义务在 'links-search-files 中找到相应的条目, 其中相应的条目是目录中的 "links.rktd" 。

    Changed in version 7.0.0.19: 以一般的方式调整目录范围内的包搜索路径。

  • 'compiled-file-roots — 用于初始化 current-compiled-file-roots 的路径和 'same 的列表。 路径是相对的或绝对的,可以指定为字符串或字节字符串,分别用 string->path 或 bytes->path 转换为路径。

    Added in version 8.0.0.9.

  • 'bin-dir — 安装的目录中包含可执行文件的路径、字符串或字节串。 它默认为 main collection directory 的 "bin" 同级目录。

  • 'gui-bin-dir — 一个包含 GUI 可执行文件的安装目录的路径、字符串或字节字符串。 它默认为 'bin-dir ,如果配置了的话,或者以特定平台的方式默认: 在 Unix 上默认为 main collection directory 的 "bin" 同级目录, 在 Windows 和 Mac OS 上默认为 main collection directory 的父目录。

    Added in version 6.8.0.2.

  • 'bin-search-dirs — 类似于 'lib-search-dirs, 但用于寻找可执行文件,如 racket 。 一个 #f 被默认的搜索路径所取代,它是一个用户和版本特定的目录, 后面是可能通过 'bin-dir 配置的主控制台可执行目录。

    Added in version 8.1.0.6.

  • 'gui-bin-search-dirs — 类似于 'bin-search-dirs, 但是针对 GUI 可执行文件,默认为 'bin-search-dirs 的值。

    Added in version 8.1.0.6.

  • 'apps-dir — 安装 ".desktop" 文件的目录的路径、字符串或字节字符串。 它默认为 main shared-file directory 下的 "applications" 子目录。

  • 'man-dir — 一个路径、字符串或字节字符串,表示安装的手册目录。 它默认为 main shared-file directory 下的 "man" 兄弟目录。

  • 'man-search-dirs — 类似于 'lib-search-dirs, 其中 #f 被默认的搜索路径所取代, 它是一个用户和版本特定的目录,后面是一个可能通过 'man-dir 配置的目录。

    Added in version 8.1.0.6.

  • 'doc-dir — 主要文档目录的路径、字符串或字节字符串。 该值默认为 main collection directory 的 "doc" 同级目录。

  • 'doc-search-dirs — 类似于 'lib-search-dirs, 其中 #f 被默认的搜索路径所取代, 它是一个用户和版本特定的目录,后面是一个可能通过 'doc-dir 配置的目录。

  • 'doc-search-url — 一个用版本和搜索标签查询增强的 URL 字符串,以形成一个远程文档参考。

  • 'doc-open-url — 一个 URL 字符串或 #f; 一个字符串提供了一个 URL,用来代替本地路径搜索和可能打开的文档页面(通常只有在打开本地 HTML 文件不工作的环境下才有意义)。

  • 'include-dir — 包含C头文件的主目录的路径、字符串或字节串。 它默认为 main collection directory 的一个 "include" 同级目录。

  • 'include-search-dirs — 与 doc-search-dirs 类似,但用于包含C头文件的目录。

  • 'catalogs — 一个 URL 字符串的列表,用作解析软件包名称的搜索路径。 列表中的 #f 会被替换成默认的搜索路径。 一个不以字母开头的字符串,后面跟着 :// ,被视为路径,其中相对路径是相对于配置目录的。

  • 'default-scope — "user" 或 "installation", 决定软件包管理操作的默认 package scope 。

  • 'download-cache-dir — 一个路径字符串,用作存储下载的软件包档案的位置。 如果没有指定,软件包会被缓存在用户的缓存目录下的 "download-cache" 目录中, 如 (find-system-path 'cache-dir) 所报告的那样。

  • 'download-cache-max-files 和 'download-cache-max-bytes — 实数,决定了对下载缓存的限制。 如果不指定,缓存允许容纳 1024 个文件,总容量不超过 64 MB 。

  • 'build-stamp — 一个标识构建的字符串,它可以用来增加 Racket 的版本号,以更具体地标识构建。 一个空的字符串通常适用于发布版的构建。

  • 'absolute-installation? — 一个布尔值,如果安装使用绝对路径名, 则为 #t ,否则为 #f 。

  • 'cgc-suffix — 个字符串, 用作 "CGC" 可执行文件的后缀(在实际后缀之前,例如 ".exe") 。 使用 Windows 风格的大小写,并且该字符串将根据情况被降级(例如,对于 Unix 二进制名称)。 一个 #f 值意味着如果 racket 二进制文件将自己识别为 CGC, 那么后缀就是 "", 否则就是 "CGC" 。

  • '3m-suffix — 类似于 'cgc-suffix, 但用于3m。 一个 #f 值意味着如果 racket 的二进制识别为 3m,那么后缀为 "" ,否则为 "BC" 。

  • 'cs-suffix — 类似于 'cgc-suffix,但适用于 CS。 一个 #f 值意味着如果 racket 二进制文件将自己识别为 CS, 那么后缀就是 "", 否则就是 "CS" 。

  • 'config-tethered-console-bin-dir 和 'config-tethered-gui-bin-dir — 一个存放可执行文件额外副本的目录路径, 该目录与配置目录(由 find-config-dir 报告)相联系,在创建可执行文件时处于活动状态。 参见 拴住的安装, find-config-tethered-console-bin-dir 和 find-config-tethered-gui-bin-dir 。

  • 'interactive-file 和 'gui-interactive-file — 在 REPL 启动时运行的交互式模块的模块路径, 除非提供 -q/--no-init-file 。 默认为 'racket/interactive 和 'racket/gui/interactive 。
