1.8 字节码路径的 API

 (require compiler/compilation-path) package: base

Added in version 6.0.1.10 of package base.

procedure

(get-compilation-dir+name path 
  [#:modes modes 
  #:roots roots 
  #:default-root default-root]) 
 → 
path? path?
  path : path-string?
  modes : (non-empty-listof (and/c path-string? relative-path?))
   = (use-compiled-file-paths)
  roots : (non-empty-listof (or/c path-string? 'same))
   = (current-compiled-file-roots)
  default-root : (or/c path-string? 'same) = (car roots)
确定存放 path 的字节码形式和 path 的基本名称的目录。

该目录是通过依次检查 roots ,并对 roots 的每个元素依次检查 modes 来确定的。 第一个包含文件的目录,其名称与 path 加了 ".zo" (path-add-suffix 的意思)相匹配,被报告为返回目录路径。 如果没有找到这样的文件,则结果对应于与 default-root 组合的 modes 的第一个元素。

Changed in version 7.1.0.9 of package base: 增加了 #:default-root 参数。

procedure

(get-compilation-dir path    
  [#:modes modes    
  #:roots roots    
  #:default-root default-root]) → path?
  path : path-string?
  modes : (non-empty-listof (and/c path-string? relative-path?))
   = (use-compiled-file-paths)
  roots : (non-empty-listof (or/c path-string? 'same))
   = (current-compiled-file-roots)
  default-root : (or/c path-string? 'same) = (car roots)
与 get-compilation-dir+name 相同,但只返回第一个结果。

Changed in version 7.1.0.9 of package base: 增加了 #:default-root 参数。

procedure

(get-compilation-bytecode-file path 
  [#:modes modes 
  #:roots roots 
  #:default-root default-root]) 
 → path?
  path : path-string?
  modes : (non-empty-listof (and/c path-string? relative-path?))
   = (use-compiled-file-paths)
  roots : (non-empty-listof (or/c path-string? 'same))
   = (current-compiled-file-roots)
  default-root : (or/c path-string? 'same) = (car roots)
与 get-compilation-dir+name 相同,但将结果合并并加上 ".zo" 后缀,得出字节码文件路径。

Changed in version 7.1.0.9 of package base: 增加了 #:default-root 参数。
