12.2 通过 C 语言嵌入模块

raco ctool 是由 "cext-lib" 包提供的。

raco ctool 的 --c-mods 模式接收一组 Racket 模块并生成一个 C 源文件, 可以作为嵌入 Racket 运行系统的程序的一部分。 关于嵌入程序的解释,请看 Inside: Racket C API 中的 Embedding into a Program (BC) 。 --mods 模式与此类似,但它为编译后的模块生成原始字节,而不对C语言声明中的字节进行编码。

生成的源文件或编译文件会嵌入指定的模块。生成的 C 源码定义了一个 declare_modules 函数,将模块声明放入命名空间。 因此,使用 raco ctool --c-mods 的输出,一个程序可以用一组模块嵌入 Racket, 这样它就不需要一个 "collects" 目录来在运行时加载模块。

如果嵌入的模块指的是运行时文件,这些文件可以通过向 raco ctool --cmods 提供 --runtime 参数来收集, 并指定一个目录 ‹dir› 来存放这些文件。 通常, ‹dir› 是一个相对路径,文件在运行时可以在相对于可执行文件的 ‹dir› 中找到, 但是可以用 --runtime-access 指定一个单独的运行时路径(通常是相对的)。

通常情况下, raco ctool --c-mods 与 ++lib 一起使用,指定一个基于集合的模块路径。例如、

  raco ctool --c-mods base.c ++lib racket/base

生成一个 "base.c" ，其 declare_modules 函数使 racket/base 可通过 scheme_namespace_require 或 scheme_dynamic_require 函数在嵌入应用程序中使用。

当一个模块文件被提供给 raco ctool --c-mods, 那么 declare_modules 就会用模块文件的符号名称声明一个模块。例如、

  raco ctool --c-mods base.c hello.rkt

创建一个 declare_modules 来定义模块 'hello, 它可以用 (namespace-require ''hello) 或类似的方式在 C 级别上被要求进入当前命名空间:

  p = scheme_make_pair(scheme_intern_symbol("quote"),

                       scheme_make_pair(scheme_intern_symbol("hello"),

                                        scheme_make_null()));

  scheme_namespace_require(p);
