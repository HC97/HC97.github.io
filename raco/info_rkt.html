6.4 "info.rkt" File Format

 #lang info package: base
 #lang setup/infotab

在每个集合中,一个特殊的模块文件 "info.rkt" 提供了关于一个集合的一般信息,供各种工具使用。 例如, "info.rkt" 文件指定了如何建立一个集合的文档, 并且它列出了该集合提供的 DrRacket 的插件工具或 raco 的命令。

在 "info.rkt" 文件中指定的字段对于包来说在 包的元数据 中记录, 对于集合来说在 用 "info.rkt" 文件控制 raco setup 中记录。

尽管 "info.rkt" 文件包含一个模块声明,但该声明有一个高度约束的形式。 它必须符合 info-module 的以下语法:

  info-module = 
(module info info-mod-path
  decl
  ...)
     
  info-mod-path = info
  | setup/infotab
  | (lib "info/main.rkt")
  | (lib "setup/infotab.ss")
  | (lib "setup/infotab.rkt")
  | (lib "main.rkt" "info")
  | (lib "infotab.rkt" "setup")
  | (lib "infotab.ss" "setup")
     
  decl = (define id info-expr)
     
  info-expr = (quote datum)
  | (quasiquote datum)
  | (if info-expr info-expr info-expr)
  | (info-primitive info-expr ...)
  | id
  | string
  | number
  | boolean
     
  info-primitive = cons
  | car
  | cdr
  | list
  | list*
  | reverse
  | append
  | equal?
  | string-append
  | make-immutable-hash
  | hash
  | hash-set
  | hash-set*
  | hash-remove
  | hash-clear
  | hash-update
  | path->string
  | build-path
  | collection-path
  | system-library-subpath
  | getenv

例如,下面的声明可能是 "games" 集合的 "info.rkt" 库。 它包含了三个 info 标签的定义, name, gracket-launcher-libraries 和 gracket-launcher-names 。

#lang info
(define name "Games")
(define gracket-launcher-libraries '("main.rkt"))
(define gracket-launcher-names     '("PLT Games"))

如本例所示, "info.rkt" 文件可以使用 #lang 符号, 但只能使用 info (或 setup/infotab) 语言。

尽管 getenv 在 info 模块中是允许的,但 get-info 函数用一个环境来加载模块, 该环境会删去任何未在 PLT_INFO_ALLOW_VARS 环境变量中列出的变量, 该变量持有一个 ; 分隔的变量名称列表。 默认情况下,允许的环境变量集是空的。

另请参见 setup/getinfo 中的 get-info 。

Changed in version 6.5.0.2 of package base: 添加了 if, equal? 和 getenv 。
