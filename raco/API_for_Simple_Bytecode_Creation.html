1.7 API for Simple Bytecode Creation

 (require compiler/compile-file) package: base

procedure

(compile-file src [dest filter]) → path?

  src : path-string?
  dest : path-string?
   = 
(let-values ([(base name dir?) (split-path src)])
  (build-path base "compiled"
              (path-add-suffix name #".zo")))
  filter : (any/c . -> . any/c) = values
编译Racket文件 src ,并将编译后的代码保存到 dest 。 如果没有提供 dest ,并且 "compiled" 子目录还不存在,则创建该子目录。 compile-file 的结果是目标文件的路径。

如果提供了 filter 程序,它将被应用于每个源表达式,并对结果进行编译。

请注意, compile-file 使用当前的阅读器参数化来读取 src 。 通常情况下,应该从传递给 with-module-reading-parameterization 的 thunk 中调用 compile-file ,以便以一致的方式解析源程序并允许 #lang 。

src 中的每个表达式都是独立编译的。 如果 src 不包含单一的 module 表达式,那么当 src 被直接加载时, 早期的表达式会影响后期表达式的编译。 适当的 filter 可以使编译表现得像评估,但这个问题也可以通过 compile-zos 程序解决(尽可能地)。

另请参见 managed-compile-zo 。
