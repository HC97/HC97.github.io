6.12 集合搜索的 API

 (require setup/collection-search) package: base

Added in version 6.3 of package base.

procedure

(collection-search mod-path 
  [#:init result 
  #:combine combine 
  #:break? break? 
  #:all-possible-roots? all-possible-roots?]) 
 → any/c
  mod-path : normalized-lib-module-path?
  result : any/c = #f
  combine : (any/c (and/c path? complete-path?) . -> . any/c)
   = (lambda (r v) v)
  break? : (any/c . -> . any/c) = (lambda (r) #f)
  all-possible-roots? : any/c = #f
泛化 collection-file-path ,以支持对当前配置中基于 collection 的文件的所有可能位置进行折叠。 与 collection-file-path 不同, collection-search 的形式获取文件的位置, 但总是作为一个 'lib 路径。

文件的每个可能路径(不包括从 ".ss" 到/从 ".rkt" 的转换)都作为第二个参数提供给 combine 函数,其中第一个参数是当前结果,而 combine 产生的值成为新的结果。 #:init 参数提供初始结果。

break? 函数在当前值的基础上缩短了搜索时间。例如,它可以用来在找到合适的路径后缩短搜索时间。

如果 all-possible-roots? 是 #f, 那么 combine 只在类似 "collects" 的目录(对于当前配置)内的路径上调用,其中至少存在一个匹配的集合目录。

procedure

(normalized-lib-module-path? v) → boolean?

  v : any/c
如果 v 是一个 '(lib str) 形式的模块路径(在 module-path? 的意义上), 其中 str 至少包含一个斜线,则返回 #t 。 collapse-module-path 函数为基于集合的模块引用产生这样的模块路径。
