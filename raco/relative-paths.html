6.10 相对路径的 API

Racket 安装树通常可以在文件系统中移动。 为了支持这一点,必须注意避免绝对路径。 下面两个 API 涵盖了这方面的两个方面:一种将路径转换为相对于 "collects" 树的值的方法, 以及一种显示这种路径的方法(例如,在错误信息中)。

6.10.1 表示基于集合的路径

 (require setup/collects) package: base

procedure

(path->collects-relative path 
  #:cache cache) 
 → 
(or/c path-string?
      (cons/c 'collects
              (cons/c bytes? (non-empty-listof bytes?))))
  path : path-string?
  cache : (or/c #f (and/c hash? (not/c immutable?)))
检查 path (通过 path->complete-path 和 simplify-path 进行规范化处理, #f 是其第二个参数)是否与 collection-file-path 的结果匹配。 如果是,结果是一个以 'collects 开头的列表,包含相关的路径元素的字节字符串。如果不是,路径就会被原样返回。

如果需要的话, cache 参数与 path->pkg 一起使用。

procedure

(collects-relative->path rel) → path-string?

  rel : 
(or/c path-string?
      (cons/c 'collects
              (cons/c bytes? (non-empty-listof bytes?))))
path->collects-relative 的反义词:如果 rel 是以 'collects 开头的一对, 那么就用 collection-file-path 把它转换回一个路径。

procedure

(path->module-path path #:cache cache)

 → (or/c path-string? module-path?)
  path : path-string?
  cache : (or/c #f (and/c hash? (not/c immutable?)))
就像 path->collects-relative 一样, 但结果是 path 或一个规范化的(在 collapse-module-path 的意义上)模块路径。

6.10.2 表示与 "collects" 相关的路径

 (require setup/main-collects) package: base

procedure

(path->main-collects-relative path)

 → (or/c path? (cons/c 'collects (non-empty-listof bytes?)))
  path : (or/c bytes? path-string?)
检查 path 的前缀是否与 (find-collects-dir) 确定的 "collects" 主目录的前缀一致。 如果是,结果是一个以 'collects 开头的列表,包含其余的路径元素的字节字符串。如果不是,路径将被原样返回。

path 参数应该是一个完整的路径。 在 path->main-collects-relative 之前应用 simplify-path 通常是一个好主意。

由于历史原因, path 可以是一个字节字符串,用 bytes->path 将其转换为路径。

也请参见 collects-relative->path 。

procedure

(main-collects-relative->path rel) → path?

  rel : 
(or/c bytes?
      path-string?
      (cons/c 'collects (non-empty-listof bytes?)))
path->main-collects-relative 的反义词:如果 rel 是以 'collects 开头的一对, 那么它被转换回相对于 (find-collects-dir) 的路径。

6.10.3 表示相对于文档的路径

 (require setup/main-doc) package: base

procedure

(path->main-doc-relative path)

 → (or/c path? (cons/c 'doc (non-empty-listof bytes?)))
  path : (or/c bytes? path-string?)
和 path->main-collects-relative 一样,除了它检查相对于 (find-doc-dir) 的前缀, 如果是,则返回以 'doc 开头的列表。

procedure

(main-doc-relative->path rel) → path>

  rel : 
(or/c bytes?
      path-string?
      (cons/c 'doc (non-empty-listof bytes?)))
和 path->main-collects-relative 一样,除了它是 path->main-doc-relative 的逆向。

6.10.4 显示相对于共同根的路径

 (require setup/path-to-relative) package: base

procedure

(path->relative-string/library path    
  [default    
  #:cache cache]) → any/c
  path : path-string?
  default : (or/c (-> path-string? any/c) any/c)
   = (lambda (x) (if (path? x) (path->string x) x))
  cache : (or/c #f (and/c hash? (not/c immutable?))) = #f
产生一个适合在错误信息中显示的字符串。 如果路径是一个绝对值,并且在一个包内,结果是一个以 "<pkgs>/" 开头的字符串。 如果路径是在 "collects" 树内的绝对路径,结果是以 "<collects>/" 开头的字符串。 同样,在用户专用的 collects 中的路径结果是前缀 "<user-collects>/", PLaneT 的路径结果是 "<planet>/", 而进入文档的路径结果是 "<doc>/" 或 "<user-doc>/" 。

如果 cache 不是 #f, 它将被用作 pkg->path 的缓存参数, 以加快检测和转换软件包路径的速度。

如果路径不是绝对值,或者不在其中任何一项中,它将被原样返回(如果需要,将被转换为字符串)。 如果给出 default ,则指定返回值:它可以是一个应用于路径以获得结果的过程,或者是结果本身。

请注意,只有在给出 default 的情况下,这个函数才能返回非字符串,而且它不返回字符串。

procedure

(path->relative-string/setup path    
  [default    
  #:cache cache]) → any/c
  path : path-string?
  default : (or/c (-> path-string? any/c) any/c)
   = (lambda (x) (if (path? x) (path->string x) x))
  cache : (or/c #f (and/c hash? (not/c immutable?))) = #f
与 path->relative-string/library 相同,以便向后兼容。

procedure

(make-path->relative-string dirs [default])

 → (path-string? any/c . -> . any)
  dirs : (listof (cons (-> path?) string?))
  default : (or/c (-> path-string? any/c) any/c)
   = (lambda (x) (if (path? x) (path->string x) x))
这个函数产生类似 path->relative-string/library 和 path->relative-string/setup 的函数。

dirs 参数决定了前缀的替换。 它必须是一个关联列表,将路径产生的 thunk 映射到指定路径中的前缀字符串。

default 决定了所产生的函数的默认值(总是可以被这个函数的一个额外参数所覆盖)。
