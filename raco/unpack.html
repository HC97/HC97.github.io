11 raco unpack: 解压图书馆收藏

raco unpack 命令将 ".plt" 归档文件(见 raco pack: 打包图书馆集合) 解压到当前目录,而不尝试安装任何集合。 使用 raco pkg (见 Racket 中的包管理) 将 ".plt" 归档文件作为一个包来安装,或者使用 raco setup -A (见 raco setup: 安装管理) 从 ".plt" 归档文件中解压并安装集合。

命令行标志:

  • -l 或 --list — 列出归档文件的内容,而不对其进行解包。

  • -c 或 --config — 在解包或列出存档内容之前显示存档配置。

  • -f 或 --force — 替换已经存在的文件;如果没有这个标志,档案中说应该被替换的文件将被替换。

11.1 解压API

 (require setup/unpack) package: base
setup/unpack 库为解压 ".plt" 文件提供原始支持。

解压缩 archive 。

main-collects-parent-dir 参数被传递给 get-target-plt-directory 。

print-status 参数用于报告解压进度。

get-target-directory 参数用于获取解压存档的目标目录,其内容是相对于一个任意的目录。

如果 force? 为真,那么版本和所需集合的不匹配(将存档中的信息与当前的安装进行比较)会被忽略。

get-target-plt-directory 函数被调用来为归档文件选择一个安装目标,该归档文件的内容是相对于安装的。 该函数通常应在其前两个参数中返回一个;第三个参数仅包含前两个参数,但如果前两个参数相同则只有一个元素。 如果存档不要求所有用途的安装,那么前两个参数将是不同的,前者是用户特定的位置,而后者是指主要的安装。

遍历 archive 的内容,归档必须是一个 ".plt" 归档,它是用默认的解包单元和配置表达式创建的。 配置表达式不被评估,解包单元不被调用,文件也不被解包到文件系统中。 相反,存档中的信息通过 on-config, on-setup-unit, on-directory 和 on-file 报告出来, 每一个都可以建立在一个以 initial-value 开始的累积值上,其最终值被返回。

on-config-fn 函数被调用一次,其 S 表达式代表一个实现配置信息的函数。 on-config 的第二个参数是 initial-value, 该函数的结果作为最后一个参数传递给 on-setup-unit 。

on-setup-unit 函数被调用时带有安装单元的 S 表达式表示,一个指向文件其余部分的输入端口,以及累积值。 这个输入端口与其他处理过程中使用的端口相同,所以如果 on-setup-unit 从该端口消耗了任何数据, 那么该数据将不会被其余的函数消耗。 (这意味着 on-setup-unit 可能会使处理过程处于不一致的状态,而这种状态不会被任何东西检查,因此可能会导致错误)。 on-setup-unit 的结果成为新的累积值。

对于存档正常解压时创建的每个目录, on-directory 都会被调用, 其中包括目录路径(下文会详细描述)和到那时为止的累积值,其结果就是新的累积值。

对于每一个正常解包时由档案馆创建的文件, on-file 被调用, 包含文件路径(下文有更多描述),一个包含文件内容的输入端口,一个可选的模式符号, 表明文件是否应该被替换,以及到该点为止的累积值;其结果是新的累积值。 输入端口可以被使用或被忽略,对文件其余部分的解析无论如何都会继续。 然而,在 on-file 返回控制后,输入端口的内容就被抽走了。

目录或文件路径可以是一个普通的路径, 也可以是一个包含 'collects, 'doc, 'lib 或 'include 和一个相对路径的列表。 后者对应于相对于目标安装的收集目录(在 find-collects-dir 的意义上)、 文档目录(在 find-doc-dir 的意义上)、 库目录(在 find-lib-dir 的意义上)或 "include" 目录(在 find-include-dir 的意义上)的目录或文件。
