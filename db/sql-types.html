4 SQL 类型转换

连接会自动将查询结果转换为适当的 Racket 类型。 同样,查询参数也被接受为 Racket 值,并转换为适当的 SQL 类型。

Examples:
> (query-value pgc "select count(*) from the_numbers")

4

> (query-value pgc "select false")

#f

> (query-value pgc "select 1 + $1" 2)

3

如果一个查询结果包含一个本库不支持的 SQL 类型的列,会产生一个异常。作为一种变通方法,将该列转换为支持的类型:

Examples:
> (query-value pgc "select inet '127.0.0.1'")

query-value: unsupported type

  type: inet

  typeid: 869

> (query-value pgc "select cast(inet '127.0.0.1' as varchar)")

"127.0.0.1/32"

结果列中不支持的类型的异常是在查询执行时产生的,而不是在准备时产生的;对于参数,它在提供参数值时产生。 因此,即使是无法执行的准备好的语句也可以使用 prepared-statement-parameter-types 和 prepared-statement-result-types 进行检查。

'char1 类型,在 PostgreSQL 的 SQL 语法中写为 "char" (引号很重要),是一个字节, 本质上是一个写成字符的小整数。

一个 decimal 类型的 SQL 值被转换为精确有理数或 +nan.0 。 当把 Racket 值转换为 SQL decimal 时,可由有限小数字符串表示的精确有理数值的转换不会损失精度。 (当然,如果该值被存储在一个精度较低的数据库字段中,精度可能会损失)。 其他的实数值被转换为小数,但会有精度损失。 在 PostgreSQL 中, numeric 和 decimal 指的是同一类型。

Examples:
> (query-value pgc "select real '+Infinity'")

+inf.0

> (query-value pgc "select numeric '12345678901234567890'")

12345678901234567890

一个 SQL timestamp with time zone 被转换成 Racket sql-timestamp 的 UTC–也就是 tz 字段为 0 。 如果一个没有时区的 Racket sql-timestamp (tz 是 #f) 被用于 timestamp with time zone 类型的参数,它将被视为 UTC 的时间戳。 参见 PostgreSQL 的时间戳和时区 。

像 'point 这样的几何类型由 db/util/geometry 和 db/util/postgresql 模块中定义的结构表示。

如果基础类型被支持,PostgreSQL 用户定义的 domains 在查询结果中被支持。 记录集头和 prepared-statement-result-types 以底层类型的方式报告它们。 目前不支持具有用户定义域类型的参数。 作为一种变通方法,将参数转换为底层类型。 例如,如果 $1 的类型是一个基础类型为 integer 的域, 那么用 ($1::integer) 替换 $1 。

对于上表中的每个类型,也支持相应的 array type , 使用 pg-array 结构。 对一个数组参数使用 = ANY 语法,而不是动态地构造一个 SQL IN 表达式:

Examples:
> (query-value pgc "select 1 in (1, 2, 3)")

#t

> (query-value pgc "select 1 = any ($1::integer[])"
               (list->pg-array (list 1 2 3)))

#t

可以为数组参数提供一个列表,在这种情况下,会使用 list->pg-array 自动转换。 当数组类型可以从左侧推断出来时,类型注释可以被放弃。

Examples:
> (query-value pgc "select 1 = any ($1)" (list 1 2 3))

#t

> (query-value pgc "select $1::integer = any ($2)"
               1 (list 1 2 3))

#t

> (query-value pgc "select $1 = any ($2)" ; what type are we using?
               1 (list 1 2 3))

query-value: cannot convert given value to SQL type

  given: 1

  type: string

  expected: string?

  dialect: PostgreSQL

PostgreSQL 定义了许多其他类型,例如网络地址和行类型。 目前不支持这些类型,但在这个库的未来版本中可能会增加支持。

Changed in version 1.1 of package db-lib: 增加了对 'uuid 类型的支持。

MySQL 没有为准备好的查询报告特定的参数类型,所以它们被分配为伪类型 'any 。 Racket 值到参数的转换接受字符串、数字 (rational?—没有无穷大或 NaN)、字节、SQL 日期/时间结构 (sql-date?, sql-time?,sql-timestamp? 和 sql-day-time-interval?), 比特 (bit-vector?) 和几何值 (geometry2d?) 。 如果可能的话,数字将作为 64 位有符号整数发送,否则将作为双精度浮点数字发送。

CHAR 或 VARCHAR 类型的字段通常被报告为 'var-string, 而 BINARY 或 VARBINARY 类型的字段通常被报告为 'var-binary 。

MySQL time 类型表示时间间隔,它可能不对应于一天中的时间(例如,间隔可能是负的或大于 24 小时)。 在从 MySQL 结果转换为 Racket 值时, 那些代表一天中的时间的 time 值被转换为 sql-time 值; 其余的则用 sql-interval 值表示。

MySQL 的 enum 和 set 类型不被支持。 作为一种变通方法,可以将它们转换为整数或字符串。

4.2 SQL 数据

本节描述了用于表示各种 SQL 类型的数据类型,这些类型在 Racket 中没有合适的对应物。

4.2.1 SQL NULL

SQL NULL 被翻译成唯一的 sql-null 值。

value

sql-null : sql-null?

一个特殊的值,用来表示查询结果中的 NULL 值。 sql-null 值可以用 eq? 来识别。

Example:
> (query-value pgc "select NULL")

#<sql-null>

procedure

(sql-null? x) → boolean?

  x : any/c
如果 x 是 sql-null,返回 #t ;否则返回 #f 。

procedure

(sql-null->false x) → any/c

  x : any/c
如果 x 是 sql-null,返回 #f ;否则返回 x 。

Examples:
> (sql-null->false "apple")

"apple"

> (sql-null->false sql-null)

#f

> (sql-null->false #f)

#f

procedure

(false->sql-null x) → any/c

  x : any/c
如果 x 是 sql-null ,则返回 #f ;否则返回 x 。

Examples:
> (false->sql-null "apple")

"apple"

> (false->sql-null #f)

#<sql-null>

4.2.2 日期和时间

DATE, TIME (WITH TIME ZONE 和没有), TIMESTAMP (WITH TIME ZONE 和没有) 和 INTERVAL SQL 类型由以下结构表示。

更多关于日期时间值的功能,请参见 日期类型实用程序 。

struct

(struct sql-date (year month day))

  year : exact-integer?
  month : (integer-in 0 12)
  day : (integer-in 0 31)
代表一个 SQL 日期。

具有零值的 month 或 day 成分的日期是 MySQL 的扩展。

struct

(struct sql-time (hour minute second nanosecond tz))

  hour : exact-nonnegative-integer?
  minute : exact-nonnegative-integer?
  second : exact-nonnegative-integer?
  nanosecond : exact-nonnegative-integer?
  tz : (or/c exact-integer? #f)

struct

(struct sql-timestamp (year
    month
    day
    hour
    minute
    second
    nanosecond
    tz))
  year : exact-nonnegative-integer?
  month : exact-nonnegative-integer?
  day : exact-nonnegative-integer?
  hour : exact-nonnegative-integer?
  minute : exact-nonnegative-integer?
  second : exact-nonnegative-integer?
  nanosecond : exact-nonnegative-integer?
  tz : (or/c exact-integer? #f)
代表 SQL 时间和时间戳。

tz 字段表示时区偏移, 是格林尼治标准时间以东的秒数(如 srfi/19)。 如果 tz 是 #f, 时间或时间戳不携带时区信息。

sql-time 和 sql-timestamp 结构以纳秒级精度存储小数秒, 以便与 srfi/19 兼容。 但是请注意,数据库系统一般不支持纳秒精度;例如,PostgreSQL 只支持微秒精度。

Examples:
> (query-value pgc "select date '25-dec-1980'")

(sql-date 1980 12 25)

> (query-value pgc "select time '7:30'")

(sql-time 7 30 0 0 #f)

> (query-value pgc "select timestamp 'epoch'")

(sql-timestamp 1970 1 1 0 0 0 0 #f)

> (query-value pgc "select timestamp with time zone 'epoch'")

(sql-timestamp 1970 1 1 0 0 0 0 0)

struct

(struct sql-interval (years
    months
    days
    hours
    minutes
    seconds
    nanoseconds))
  years : exact-integer?
  months : exact-integer?
  days : exact-integer?
  hours : exact-integer?
  minutes : exact-integer?
  seconds : exact-integer?
  nanoseconds : exact-integer?
代表时间的长度。 间隔被规范化以满足以下约束:

  • years 和 months 有相同的符号

  • months 的范围从 -11 到 11

  • days, hours, minutes, seconds 和 nanoseconds 都有相同的符号

  • hours 的范围从 -23 到 23

  • minutes 和 seconds 的范围是 -59 到 59

  • nanoseconds 的范围是从 (- (sub1 #e1e9)) 到 (sub1 #e1e9)

也就是说,一个区间由两组成分组成:年-月和日-时,而规范化只在组内进行。 事实上,SQL 标准分别识别这两种类型的间隔(见下面的 sql-year-month-interval? 和 sql-day-time-interval? ),并且不允许将它们结合起来。 像 1 month 3 days 这样的时间间隔是PostgreSQL的扩展。

procedure

(sql-year-month-interval? x) → boolean?

  x : any/c
如果 x 是一个 sql-interval 值, 其中 days, hours, minutes, seconds 和 nanoseconds 的字段为零,则返回 #t 。

procedure

(sql-day-time-interval? x) → boolean?

  x : any/c
如果 x 是一个 sql-interval 值, 并且 years 和 months 的字段为零,则返回 #t 。

procedure

(sql-interval->sql-time interval [failure]) → any

  interval : sql-interval?
  failure : any/c = (lambda () (error ....))
如果 interval 是一个代表一天中某个时间的日-时间隔,返回相应的 sql-time 值。 特别是,以下内容必须为真:

  • hours, minutes, seconds 和 nanoseconds 都必须是非负值

  • hours 必须在 0 到 23 之间

对 minutes 等的相应约束由构造函数强制执行。

如果 interval 超出范围, failure 值会被调用(如果它是一个过程),或者返回(否则)。

procedure

(sql-time->sql-interval time) → sql-day-time-interval?

  time : sql-time?
将 time 转换为区间。 如果 time 有时区信息,会被忽略。
