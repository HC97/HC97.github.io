5 实用工具

本节中描述的绑定是由下面的特定模块提供的, 而不是由 db 或 db/base 提供的。

5.1 日期类型实用程序

 (require db/util/datetime) package: db-lib

procedure

(sql-datetime->srfi-date t) → srfi:date?

  t : (or/c sql-date? sql-time? sql-timestamp?)

procedure

(srfi-date->sql-date d) → sql-date?

  d : srfi:date?

procedure

(srfi-date->sql-time d) → sql-time?

  d : srfi:date?

procedure

(srfi-date->sql-time-tz d) → sql-time?

  d : srfi:date?

procedure

(srfi-date->sql-timestamp d) → sql-timestamp?

  d : srfi:date?

procedure

(srfi-date->sql-timestamp-tz d) → sql-timestamp?

  d : srfi:date?
在本库的日期和时间值与 SRFI 19 的日期值之间进行转换(参见 srfi/19))。 SRFI 日期比 SQL 日期和时间存储了更多的信息,因此将 SQL 时间转换为 SRFI 日期,例如,在年、月、日字段中置零。

Examples:
> (sql-datetime->srfi-date
   (query-value pgc "select time '7:30'"))

(date* 0 30 7 1 1 0 0 0 #f 0 0 "")

> (sql-datetime->srfi-date
   (query-value pgc "select date '25-dec-1980'"))

(date* 0 0 0 25 12 1980 4 359 #f 0 0 "")

> (sql-datetime->srfi-date
   (query-value pgc "select timestamp 'epoch'"))

(date* 0 0 0 1 1 1970 4 0 #f 0 0 "")

procedure

(sql-day-time-interval->seconds interval) → rational?

  interval : sql-day-time-interval?
返回 interval 的长度,单位是秒。

5.2 几何类型

 (require db/util/geometry) package: db-lib

下面的结构和函数处理了基于 OpenGIS(ISO 19125)模型的几何值。

注意:使用 PostGIS 扩展到 PostgreSQL 的几何列不被直接支持。 相反,数据应该以 Well-Known Binary 格式进行交换; wkb->geometry 和 geometry->wkb 函数支持以下结构与 WKB 格式的转换。

struct

(struct point (x y))

  x : real?
  y : real?
代表一个 OpenGIS Point 。

struct

(struct line-string (points))

  points : (listof point?)
代表一个 OpenGIS LineString 。

struct

(struct polygon (exterior interiors))

  exterior : linear-ring?
  interiors : (listof linear-ring?)
代表一个 OpenGIS Polygon 。

struct

(struct multi-point (elements))

  elements : (listof point?)
代表一个 OpenGIS MultiPoint, 一个点的集合。

struct

(struct multi-line-string (elements))

  elements : (listof line-string?)
代表一个 OpenGIS MultiLineString, 一个线字符串的集合。

struct

(struct multi-polygon (elements))

  elements : (listof polygon?)
代表一个 OpenGIS MultiPolygon, 一个多边形的集合。

struct

(struct geometry-collection (elements))

  elements : (listof geometry2d?)
代表一个 OpenGIS GeometryCollection, 一个任意几何值的集合。

procedure

(geometry2d? x) → boolean?

  x : any/c
如果 x 是一个 point, line-string, polygon, multi-point, multi-line-string, multi-polygon 或 geometry-collection , 返回 #t ;否则返回 #f 。

procedure

(line? x) → boolean?

  x : any/c
如果 x 是一个由两个点 (参照 OpenGIS Line) 组成的 line-string , 则返回 #t ;否则返回 #f 。

procedure

(linear-ring? x) → boolean?

  x : any/c
如果 x 是一个 line-string , 其第一个点和最后一个点相等 (参照 OpenGIS LinearRing)，则返回 #t ; 否则,返回 #f 。

procedure

(geometry->wkb g #:big-endian? big-endian?) → bytes?

  g : geometry2d?
  big-endian? : (system-big-endian?)
返回几何值 g 的 Well-Known Binary (WKB) 编码。 big-endian? 参数决定了所使用的字节顺序(WKB 格式包括字节顺序标记,所以一个强大的客户端应该接受两种编码)。

procedure

(wkb->geometry b) → geometry2d?

  b : bytes?
解码几何值的 Well-Known Binary (WKB) 表示。

5.3 PostgreSQL 特定类型

 (require db/util/postgresql) package: db-lib

struct

(struct pg-array (dimensions
    dimension-lengths
    dimension-lower-bounds
    contents))
  dimensions : exact-nonnegative-integer?
  dimension-lengths : (listof exact-positive-integer?)
  dimension-lower-bounds : (listof exact-integer?)
  contents : vector?
代表一个 PostrgreSQL 数组。 dimension-lengths 和 dimension-lower-bounds 字段都是 dimensions 元素的列表。 默认情况下,PostgreSQL 数组的索引以 1 开始(而不是 0), 所以 dimension-lower-bounds 通常是一个 1 的列表。

procedure

(pg-array-ref arr index ...+) → any/c

  arr : pg-array?
  index : exact-integer?
返回 arr 中位于给定位置的元素。 index 参数的数量必须与 arr 的维度相同。 记得 PostgreSQL 的数组索引通常以1开始,而不是0。

procedure

(pg-array->list arr) → list?

  arr : pg-array?
返回 arr 的内容的列表。 arr 的维度必须是 1;否则会产生错误。

procedure

(list->pg-array lst) → pg-array?

  lst : list?
返回一个维数为 1 的 pg-array ,其中包含 lst 的内容。

struct

(struct pg-empty-range ())

代表一个空的范围。

struct

(struct pg-range (lb includes-lb? ub includes-ub?))

  lb : range-type
  includes-lb? : boolean?
  ub : range-type
  includes-ub? : boolean?
代表一个从 lb (下限)到 ub (上限)的数值范围。 includes-lb? 和 includes-ub? 字段表示范围的每一端是开放还是封闭的。

lb 和 ub 字段必须具有相同的类型; 允许的类型是精确整数、实数和 sql-timestamp 。 任意一个或两个边界也可以是 #f, 这表明该范围在该端是无界的。

procedure

(pg-range-or-empty? v) → boolean?

  v : any/c
如果 v 是一个 pg-range 或 pg-empty-range 实例, 返回 #t ;否则,返回 #f 。

procedure

(uuid? v) → boolean?

  v : any/c
如果 v 是一个 string, 符合 UUID 的十六进制表示格式,则返回 #t 。 具体来说,它必须是一系列由破折号隔开的十六进制数字,符合以下模式:

 

‹uuid›

 ::= 

‹digit16›{8,8} - ‹digit16›{4,4} - ‹digit16›{4,4} - ‹digit16›{4,4} - ‹digit16›{12,12}

这些数字本身是不区分大小写的,大写和小写的字符都可以接受。 否则,如果 v 不是一个符合上述模式的字符串,该函数将返回 #f 。

Added in version 1.1 of package db-lib.
Changed in version 1.8: UUID 前后不允许有任何字符。

struct

(struct pg-box (ne sw))

  ne : point?
  sw : point?

struct

(struct pg-path (closed? points))

  closed? : boolean?
  points : (listof point?)

struct

(struct pg-circle (center radius))

  center : point?
  radius : real?
这些结构代表了 PostgreSQL 内置的某些几何类型,这些类型在 OpenGIS 模型中没有适当的类似物: box, path 和 circle 。 point, lseg 和 polygon PostgreSQL 的内置类型是用 point, line-string (line?) 和 polygon 结构表示。

注意:PostgreSQL 的内置几何类型与 PostGIS 扩展库所提供的类型不同(见 几何类型)。

procedure

(pg-custom-type typeid    
  typename    
  [basetype    
  #:recv recv-convert    
  #:send send-convert]) → pg-custom-type?
  typeid : exact-nonnegative-integer?
  typename : symbol?
  basetype : (or/c #f symbol? exact-nonnegative-integer?) = #f
  recv-convert : (or/c #f (-> any/c any/c)) = values
  send-convert : (or/c #f (-> any/c any/c)) = values
创建一个可以与 PostgreSQL 连接使用的自定义类型描述符, 见 add-custom-types 。

typeid 指的是服务器 pg_type 系统表中的一行的 OID 。 typename 符号是本库在参数描述、错误信息等方面使用的类型名称; 它不一定与服务器的 typeid 的类型名称相同。 服务器的类型名称必须在 SQL 语句中使用。

当 typeid 标识的类型出现在查询结果中时,首先根据 basetype 接收结果值, 然后使用 recv-convert 值。 如果 basetype 是 #f (默认值),它将被当作 bytea 类型处理; 也就是说, recv-convert 得到服务器发送的字节字符串。 如果 recv-convert 是 #f, 该类型不允许作为结果类型。

当 typeid 标识的类型被用作查询参数时, 参数值首先使用 send-convert 进行转换,转换后的值根据 basetype 发送。 如果 send-convert 是 #f,该类型不允许作为参数类型。

Examples:
> (define cidr-typeid
    (query-value pgc "select oid from pg_type where typname = $1" "cidr"))
> cidr-typeid

650

> (send pgc add-custom-types
        (list (pg-custom-type cidr-typeid 'cidr
                              #:recv bytes->list
                              #:send list->bytes)))
> (query-value pgc "select cidr '127.0.0.0/24'")

'(2 24 1 4 127 0 0 0)

Added in version 1.8 of package db-lib.

procedure

(pg-custom-type? v) → boolean?

  v : any/c
如果 v 是用 pg-custom-type 创建的,返回 #t ,否则返回 #f 。

Added in version 1.8 of package db-lib.

interface

postgresql-connection<%> : interface?

用 postgresql-connect 创建的连接实现额外操作的接口。

method

(send a-postgresql-connection add-custom-types types) → void?

  types : (listof pg-custom-type?)
在 a-postgresql-connection 中注册给定的 types ,作为查询参数和结果类型使用。 详见 pg-custom-type 。

method

(send a-postgresql-connection async-message-evt) → evt?

返回一个可同步的事件,当后端输入可用时,该事件就准备就绪。 当事件被选中时,它试图处理任何异步通知和通知消息; 如果有消息被事件的同步处理,其同步结果为 #t ,否则为 #f 。

请注意,该事件非常容易出现 "误报",当它准备好了却产生 #f 。

Added in version 1.8 of package db-lib.

5.5 测试数据库程序

 (require db/util/testing) package: db-lib

本模块提供了测试使用数据库连接的程序的实用工具。

procedure

(high-latency-connection connection 
  latency 
  [#:sleep-atomic? sleep-atomic?]) 
 → connection?
  connection : connection?
  latency : (>=/c 0)
  sleep-atomic? : any/c = #f
返回 connection 的代理连接, 在需要与数据库后端 prepare,query, start-transaction 等通信的操作前, 引入 latency 的额外秒数的延迟。

在性能测试中使用这个函数来大致模拟与数据库后端进行高延迟通信的环境。

如果 sleep-atomic? 为真,那么代理在睡觉前会进入原子模式, 以更好地模拟长期运行的FFI调用的效果(见 基于 FFI 的连接和并发性)。 即便如此,它也可能无法准确地模拟内部使用游标来获取数据的 ODBC 连接,因为每次获取都会带来额外的延迟。

5.6 不安全的 SQLite3 扩展

本节中记录的程序是 unsafe 。

在下面的函数中,连接参数必须是一个 SQLite 连接,否则会产生一个异常。

 (require db/unsafe/sqlite3) package: db-lib

Added in version 1.4 of package db-lib.

procedure

(sqlite3-load-extension c extension-path) → void?

  c : connection?
  extension-path : path-string?
在 extension-path 加载 extension library , 供连接 c 使用。 如果当前的安全防护没有授予 extension-path 上的读取和执行权限,就会引发异常。

procedure

(sqlite3-create-function c name arity func) → void?

  c : connection?
  name : (or/c string? symbol?)
  arity : (or/c exact-nonnegative-integer? #f)
  func : procedure?
创建一个名为 name 的普通函数,供连接 c 使用。 arity 参数决定了参数的合法数量; 如果 arity 是 #f ,那么允许任何数量的参数(最多到系统确定的最大值)。 可以为同一名称的不同 arity 提供不同的实现。

procedure

(sqlite3-create-aggregate c    
  name    
  arity    
  init-acc    
  step-func    
  final-func) → void?
  c : connection?
  name : (or/c string? symbol?)
  arity : (or/c exact-nonnegative-integer? #f)
  init-acc : any/c
  step-func : procedure?
  final-func : procedure?
与 sqlite3-create-aggregate 类似,但是创建一个聚合函数。 聚合函数的实现就像 fold 的参数:

  • init-acc 是初始累加器的值。

  • step-func 接收 arity+1 的参数: 当前的累加器值,然后是当前 “step” 的参数;函数的结果成为下一个步骤的累加器值。 步骤参数是 SQLite 值;累积器参数和结果可以是任意的 Racket 值。

  • final-func 接收一个参数:最后的累加器值; 该函数产生聚合函数的结果,它必须是一个 SQLite 值。

下面的关系大致上是成立的:

(begin (sqlite3-create-aggregate c "agg" 1 init-acc step-func final-func)
       (query-value c "select agg(expr) from table"))
= (final-func
    (for/fold ([accum init-acc])
              ([v (in-query c "select expr from table")])
      (step-func accum v)))
