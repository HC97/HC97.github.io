3 值和类型 (CS)

一个 Racket 值由一个指针大小的值来表示。 该值的低位表示它所使用的编码。 例如,两个(在 32 位平台上)或三个(在 64 位平台上)低位表示一个 fixnum 编码, 而一个低位和零的第二低位表示一个对,其在内存中的地址由其他位指定。

Racket 值的 C 类型是 ptr 。 对于大多数 Racket 类型,提供了一个构造函数来创建该类型的值。 例如,Scons 接收两个 ptr 值, 并返回两个值的 cons 作为一个新的 ptr 值。 除了提供构造函数外,Racket 还定义了几个全局常量 Racket 值,比如 #t 的 Strue 。

3.1 全局常量

有六个全局常量:

  • Strue — #t

  • Sfalse — #f

  • Snil — null

  • Seof_object — eof-object

  • Svoid — (void)

Srecord_uniform_ref

(

ptr rec,

访问记录信息,其中 Racket 结构被实现为记录。 Srecord_type 返回一个代表记录类型的值(所以是结构类型)。 给定一个记录类型, Srecord_type_parent 返回其上级类型或 Sfalse, Srecord_type_size 返回一个记录的分配大小,单位为字节, Srecord_type_uniformp 表示所有的记录字段是否都是 Scheme 值–对于 Racket 结构来说这总是真的。 当一个记录有所有 Scheme 值的字段时,分配大小是字段数加上指针大小的 1 倍,单位是字节。

当一个记录有所有 Scheme 字段时(所有 Racket 结构都是这种情况), Srecord_uniform_ref 访问字段值的方式与 unsafe-struct*-ref 相同。

从 cpointer? 意义上的 C-pointer 对象中提取地址和偏移量, 但只适用于使用预定义表示法的值,而不是字节串, #f, 或用 prop:cpointer 实现的新结构类型。

(

ptr cptr)

void

 

Sunlock_object

(

ptr cptr)

"锁定" 或 "解锁" n 个对象,这可以防止它被垃圾回收或移动到一个不同的地址。

尽量少锁对象,因为垃圾收集器不是被设计来处理大量的被锁对象的。 为了从 C 语言中保留多个值的使用,一个好的方法可能是分配和锁定一个向量, 这个向量有一个槽给其他每个(解锁的)对象保留。
