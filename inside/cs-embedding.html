2 嵌入到程序中 (CS)

要在程序中嵌入 Racket CS,请遵循以下步骤:

  • 找到或 build Racket CS 库。

    在 Unix 下,这个库是 "libracketcs.a" 。 从源码构建和安装将库放到安装的 "lib" 目录中。

    在 Windows 上,链接到 "libracketcsx.dll" (其中 x 代表版本号)。 在运行时, "libracketcsx.dll" 必须被移到标准 DLL 搜索路径中的某个位置, 或者你的嵌入程序必须 “delayload” 链接 DLL ,并在使用前明确加载它们。 ("Racket.exe" 使用的是后一种策略)。 请参阅 Linking to DLLs on Windows .

    在 Mac OS 上,除了 "libracketcs.a" 用于静态链接外, "Racket" 框架还提供了一个动态库,通常安装在 "lib" 子目录中。 在链接时向 gcc 提供 -framework Racket , 同时提供 -F 和 "lib" 目录的路径。 在运行时, "Racket.framework" 必须被移到标准框架搜索路径中的某个位置, 或者你的嵌入可执行文件必须提供框架的特定路径 (可能是使用 Mach-O @executable_path 前缀的可执行相关的路径)。 当针对 Hardened Runtime 时,你必须启用 “Allow Unsigned Executable Memory” 权限, 否则你在调用 racket_boot 时将会遇到 “out of memory” 的错误。

  • 对于每个使用 Racket 库函数的 C 文件, #include 文件 "chezscheme.h" 和 "racketcs.h" 。

    "chezscheme.h" 和 "racketcs.h" 文件与 Racket 软件一起发布在安装的 "include" 目录下。 从源码构建和安装也会将这个文件放在安装的 "include" 目录下。

  • 在你的程序中,调用 racket_boot 。 racket_boot 函数需要一个指向 racket_boot_arguments_t 的指针,用于配置 Racket 实例。 在将 racket_boot_arguments_t 的值清零后(通常是用 memset), 只有以下字段需要设置:

    • exec_file — 一个由 (find-system-path 'exec-file) 报告的路径, 通常是 argv[0] ,用于程序 main 接收的 argv 。

    • boot1_path — "petite.boot" 的路径。 使用一个至少包括一个目录分隔符的路径。

    • boot2_path — "scheme.boot" 的路径(包含一个分隔符)。

    • boot3_path — 到 "racket.boot" 的路径(有分隔符)。

    "petite.boot", "scheme.boot" 和 "racket.boot" 文件随 Racket 软件一起发布在 Windows 的 "lib" 目录下, 在 Mac OS X 上它们发布在 "Racket" 框架内; 在 Unix 上它们必须从源代码 built 。 这些文件可以组合成一个文件,甚至可以嵌入到可执行文件中–只要 racket_boot_arguments_t 的 boot1_offset, boot2_offset 和 boot3_offset 字段被设置为识别文件中每个引导图像的起始偏移量。

    关于在可执行文件中嵌入像 "petite.boot" 这样的文件的建议, 请参阅 Embedding Files in Executable Sections , 或者考虑使用 racket_get_self_exe_path 和 racket_path_replace_filename 来建立相对于可执行文件的路径。

  • 通过添加模块声明来配置主线程的命名空间。 初始命名空间只包含一些原始模块的声明,如 '#%kernel, 并且没有绑定被导入到顶层环境。

    要嵌入像 racket/base 这样的模块(连同其所有的依赖关系), 使用 raco ctool --c-mods ‹dest›, 它生成一个 C 文件 ‹dest› ,其中包含以静态数组封装的编译形式的模块。 生成的 C 文件定义了一个 declare_modules 函数, 该函数不需要参数,将模块安装到环境中,它调整了模块名称解析器以访问嵌入的声明。 如果嵌入的模块指的是需要随身携带的运行时文件,给 raco ctool --c-mods 提供 --runtime , 以收集运行时文件到一个目录中;更多信息见 通过 C 语言嵌入模块 。

    或者,在传递给 racket_boot 的 racket_boot_arguments_t 中设置 collects_dir, config_dir, 和/或 argv 等字段, 以定位集合/包,并以运行 racket 可执行文件时的相同方式初始化命名空间。

    在 Windows 上, raco ctool --c-mods ‹dest› --runtime ‹dest-dir› 包括在 ‹dest-dir› 中的可选 DLL , 这些 DLL 被 Racket 库引用以支持 bytes-open-converter 。 在 racket_boot_arguments_t 中设置 dll_dir 来注册 ‹dest-dir› , 这样就可以在运行时找到这些 DLL 。

    在 raco ctool 中不使用 --c-mods ,而是使用 --mods, 嵌入文件内容(见 Embedding Files in Executable Sections), 然后用 racket_embedded_load_file_region 加载内容。

  • 通过 racket_dynamic_require, racket_eval 和/或本手册中描述的其他函数访问 Racket。

    如果嵌入程序配置内置参数的方式应该被视为默认配置的一部分, 那么在之后调用原始 #%boot 模块提供的 seal 函数。 seal 获取的参数值快照被用于某些特权操作,比如安装 PLaneT 软件包。

  • 编译程序并将其与 Racket 库链接。

在任何时候, Racket 的值都可能被移动或被垃圾收集, 因为 racket_... 函数被用来运行 Racket 代码。 不要在这种调用中保留对任何 Racket 值的引用。

例如,下面是一个简单的嵌入程序,运行模块 "run.rkt", 假设 "run.c" 被创建为

  raco ctool --c-mods run.c "run.rkt"

来生成 "run.c", 它封装了 "run.rkt" 的编译形式和它所有的传递性导入(因此它们不需要在运行时被单独发现)。 "petite.boot", "scheme.boot" 和 "racket.boot" 的副本在启动时必须在当前目录中。

"main.c"

  #include <string.h>

  #include "chezscheme.h"

  #include "racketcs.h"

  

  #include "run.c"

  

  int main(int argc, char *argv[])

  {

    racket_boot_arguments_t ba;

  

    memset(&ba, 0, sizeof(ba));

  

    ba.boot1_path = "./petite.boot";

    ba.boot2_path = "./scheme.boot";

    ba.boot3_path = "./racket.boot";

  

    ba.exec_file = argv[0];

  

    racket_boot(&ba);

  

    declare_modules();

  

    ptr mod = Scons(Sstring_to_symbol("quote"),

                    Scons(Sstring_to_symbol("run"),

                          Snil));

  

    racket_dynamic_require(mod, Sfalse);

  

    return 0;

  }

作为另一个例子,下面是一个简单的嵌入程序,它评估命令行上提供的所有表达式并显示结果, 然后运行 read-eval-print 循环,全部使用 racket/base 。运行

  raco ctool --c-mods base.c ++lib racket/base

来生成 "base.c", 它封装了 racket/base 和它所有的传递性导入。

"main.c"

  #include <string.h>

  #include "chezscheme.h"

  #include "racketcs.h"

  

  #include "base.c"

  

  static ptr to_bytevector(char *s);

  

  int main(int argc, char *argv[])

  {

    racket_boot_arguments_t ba;

  

    memset(&ba, 0, sizeof(ba));

  

    ba.boot1_path = "./petite.boot";

    ba.boot2_path = "./scheme.boot";

    ba.boot3_path = "./racket.boot";

  

    ba.exec_file = argv[0];

  

    racket_boot(&ba);

  

    declare_modules();

  

    racket_namespace_require(Sstring_to_symbol("racket/base"));

  

    {

      int i;

      for (i = 1; i < argc; i++) {

        ptr e = to_bytevector(argv[i]);

        e = Scons(Sstring_to_symbol("open-input-bytes"),

                  Scons(e, Snil));

        e = Scons(Sstring_to_symbol("read"), Scons(e, Snil));

        e = Scons(Sstring_to_symbol("eval"), Scons(e, Snil));

        e = Scons(Sstring_to_symbol("println"), Scons(e, Snil));

  

        racket_eval(e);

      }

    }

  

    {

      ptr rbase_sym = Sstring_to_symbol("racket/base");

      ptr repl_sym = Sstring_to_symbol("read-eval-print-loop");

  

      racket_apply(Scar(racket_dynamic_require(rbase_sym,

                                               repl_sym)),

                   Snil);

    }

  

    return 0;

  }

  

  static ptr to_bytevector(char *s)

  {

    iptr len = strlen(s);

    ptr bv = Smake_bytevector(len, 0);

    memcpy(Sbytevector_data(bv), s, len);

    return bv;

  }

如果嵌入可执行文件的模块需要访问运行时文件(通过 racket/runtime-path 形式), 请向 raco ctool 提供 --runtime 标志, 指定一个收集运行时文件的目录。 然后,生成的 ".c" 文件中的模块将引用该目录中的文件。
