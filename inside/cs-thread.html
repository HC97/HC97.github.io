7 管理操作系统级别的线程 (CS)

Chez Scheme 的功能只能由 Chez Scheme 运行系统已知的操作系统级线程访问。 否则,这种访问和由其他线程触发的垃圾回收之间会有一个竞赛条件。

一个不是由 Chez Scheme 创建的线程可以通过用 Sactivate_thread 激活它而被运行时系统知道。 只要一个线程不运行 Chez Scheme 代码而处于激活状态,该线程就会阻止所有其他运行中的线程的垃圾收集。 使用 Sdeactivate_thread 停用一个线程。

int

 

Sactivate_thread

(

)

激活当前的操作系统级线程。 一个已经激活的线程可以被再次激活,但每次激活都必须通过去激活来平衡。 如果该线程之前被激活,结果为 0 ,否则为 1 。

void

 

Sdeactivate_thread

(

)

停用当前的操作系统级线程,或者至少在激活时进行平衡,如果没有剩余的激活与停用进行平衡,则使该线程停用。

int

 

Sdestroy_thread

(

)

释放与当前操作系统线程相关的任何 Chez Scheme 资源,这些资源必须是先前被激活的,但不能仍然被激活。
