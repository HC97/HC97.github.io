2.1 幻灯片的主要功能

procedure

(slide [#:title title    
  #:name name    
  #:aspect aspect    
  #:layout layout    
  #:gap-size sep-gap-size    
  #:inset inset    
  #:timeout secs    
  #:condense? condense?]    
  element ...) → void?
  title : (or/c #f string? pict?) = #f
  name : (or/c #f string?) = title
  aspect : aspect? = #f
  layout : (or/c 'auto 'center 'top 'tall) = 'auto
  sep-gap-size : real? = (current-gap-size)
  inset : slide-inset? = (make-slide-inset 0 0 0 0)
  secs : (or/c #f real?) = #f
  condense? : any/c = (and secs #t)
  element : 
(flat-rec-contract elem/c
  (or/c pict-convertible?
       'next 'next! 'alts 'alts~ 'nothing
       comment?
       (listof (listof elem/c))))
创建并注册一个幻灯片。 有关 element 的信息,请参见 分段式幻灯片 。 多个 element 图片用 sep-gap-size 的垂直空间分开。

当这个函数在非打印模式下第一次被调用时,那么查看器窗口就会被打开。 此外,对该函数的每次调用都会 yield ,这样就可以刷新查看器窗口,这样用户就可以在幻灯片中步步深入。 如果用户关闭了幻灯片窗口,那么 slide 就会触发一个错误,除非在窗口关闭之前, set-allow-new-slides-after-close! 被调用为真值。

如果 title 不是 #f, 那么就会为幻灯片显示一个标题。 这个 name 会在幻灯片导航对话框中使用,并且默认为 title 。

如果 layout 是 'top, 那么内容是顶层对齐的, 在标题和内容之间有 (* 2 sep-gap-size) 的空间。 'tall 布局与此类似,但只有 sep-gap-size 大小的空间。 'center 模式使内容居中(忽略了标题所消耗的空间)。 'auto 模式与 'center 相似,除了当 title 为非 #f 和标题与内容之间的空间小于 (* 2 sep-gap-size) 时,在这种情况下它的行为与 'top 相似。

inset 参数提供了一个 inset,当显示幻灯片时,它使幻灯片浏览窗口变小。 参见 make-slide-inset 获取更多信息。

如果 #:timeout 的 secs 参数不是 #f, 那么在 secs 秒之后,查看器会自动从这张幻灯片推进到下一张,而手动推进会跳过这张幻灯片。

如果 condense? 为 true,那么在 condense 模式下(如 -c 命令行标志所指定),幻灯片不会被创建和注册。

Changed in version 1.5 of package slideshow-lib: 增加了 #:aspect 参数。

procedure

(para [#:aspect aspect    
  #:width width    
  #:align align    
  #:fill? fill?    
  #:decode? decode?]    
  element ...) → pict?
  aspect : aspect? = #f
  width : real? = ((get-current-para-width #:aspect aspect))
  align : (or/c 'left 'center 'right) = 'left
  fill? : any/c = #t
  decode? : any/c = #t
  element : 
(flat-rec-contract elem/c
  (or/c string? pict? (listof elem/c)))
生成一个不宽于 width 单位的段落图片,如果 fill? 为真,则正好是 width 单位。 如果 fill? 是 #f, 那么生成的 pict 与最宽的行一样宽。

每个 element 内的列表被拼接成字符串和 pict 元素的序列。 如果 decode? 为真,那么 element 中的字符串将通过执行以下替换进行解码: --- → —, -- → –, `` → “, '' → ”, ' → ’ 。 此外,为了更好地使用 at-exp 符号,如果一个 element 是 "\n", 那么它将和下一个元素开始的任何空格一起被删除。

字符串在空格处被分割,以适应页面,并且在元素之间添加一个空格。 如果一个字符串元素以下列标点符号之一开始(解码后),则在该字符串前不加空格:

   - ' , .   : ; ? ! ) ” ’

align 参数指定如何对齐段落内的行。

参见行与行之间的间距是由 current-line-sep 参数决定的。

Changed in version 1.5 of package slideshow-lib: 增加了 #:aspect 参数。

procedure

(item [#:aspect aspect    
  #:width width    
  #:gap-size sep-gap-size    
  #:bullet blt    
  #:align align    
  #:fill? fill?    
  #:decode? decode?]    
  element ...) → pict?
  aspect : aspect? = #f
  width : real? = ((get-current-para-width #:aspect aspect))
  sep-gap-size : real? = (current-gap-size)
  blt : pict? = (scale bullet (/ sep-gap-size gap-size))
  align : (or/c 'left 'center 'right) = 'left
  fill? : any/c = #t
  decode? : any/c = #t
  element : 
(flat-rec-contract elem/c
  (or/c string? pict? (listof elem/c)))
与 para 类似,但在 blt 后面加上 (/ sep-gap-size 2) 的空间, 在产生的段落中水平附加,与顶行对齐。 blt 加上 (/ sep-gap-size 2) 的段落宽度将从该段落的最大宽度中减去。

procedure

(clickback pict thunk) → pict?

  pict : pict?
  thunk : (-> any)
创建一个嵌入给定图片的图片,其大小与给定图片相同,但在演示过程中点击时会调用 thunk 。

procedure

(interactive pict proc) → pict?

  pict : pict?
  proc : (frame% . -> . (-> any))
创建一个嵌入给定图片的图片,但在演示时在图片在屏幕上的位置创建一个浮动框架。 在浮动框架创建后(在显示前), proc 被应用于该框架。 proc 的结果必须是一个过程,当窗口被移除时(例如因为幻灯片改变),该过程将被调用。

procedure

(size-in-pixels pict [#:aspect aspect]) → pict?

  pict : pict?
  aspect : aspect? = #f
缩放 pict , 使其在屏幕上显示为 (pict-width pict) 像素宽和 (pict-height pict) 像素高。 当使用 pict 的全屏显示器时,或使用 1360 x 766 的宽屏显示器时,结果是 pict。

Added in version 1.1 of package slideshow-lib.

procedure

(make-outline name    
  title    
  subitems ...    
  [#:aspect aspect]) → (symbol? . -> . void?)
  name : (or/c symbol? (listof symbol?))
  title : (or/c string? pict?)
  subitems : 
(or/c #f null?
      (symbol? . -> . pict?))
  aspect : aspect? = #f
返回一个函数,该函数接收一个符号并生成一个幻灯片的轮廓。

上面的 ... 适用于所有三个参数一起。 每个三个参数都定义了大纲的一个部分:

  • 节的 name 是一个符号或一个符号列表。 当以后调用大纲函数来制作大纲时,给定的符号将与该部分的符号进行比较,如果符号匹配,该部分将被标记为当前。

  • title 被用来作为该节的显示名称。

  • 当该部分处于活动状态时,将显示 subitems 。 如果不显示子项目,它可以是 #f 或 null (出于历史原因)。 否则,它应该是一个函数,接收一个符号(与传递给大纲制作者的符号相同)并产生一个图片。

Changed in version 1.5 of package slideshow-lib: 增加了 #:aspect 参数。
