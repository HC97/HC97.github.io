5 动画

 (require slideshow/play) package: slideshow-lib
slideshow/play 模块提供了生成动画的工具, 作为多个自动推进的幻灯片。

许多工具都是基于一个函数,该函数接收 0.0 和 1.0 之间的数字并产生一个图片。 对输入 0.0 产生的 pict 是动画的起始图像,对 1.0 产生的 pict 是结束图像,而中间值则产生中间图像。 例如、

(lambda (n)
  (cellophane (t "Hello") n))

对应于一个淡化 "Hello" 这个词的动画。

5.1 生成动画的幻灯片

procedure

(play gen    
  [#:steps steps    
  #:delay delay-secs    
  #:skip-first? skip-first?    
  #:title title    
  #:name name    
  #:aspect aspect    
  #:comment comment    
  #:layout layout]) → void?
  gen : ((real-in 0.0 1.0) . -> . pict?)
  steps : exact-positive-integer? = (current-play-steps)
  delay-secs : real? = 0.05
  skip-first? : any/c = #f
  title : 
(or/c string? pict? #f
      ((real-in 0.0 1.0) . -> . (or/c string? pict? #f)))
   = #f
  name : 
(or/c string? #f
      ((real-in 0.0 1.0) . -> . (or/c string? #f)))
   = title
  aspect : aspect? = #f
  comment : (or/c comment? #f) = #f
  layout : (or/c 'auto 'center 'top 'tall) = 'auto
通过对从 0.0(包括)到 1.0(不包括)的等间隔值调用 gen , 生成 steps+1 的幻灯片。 除了第一张幻灯片,每张幻灯片都有一个 delay-secs 的超时时间,以便下一张幻灯片自动出现。

通常情况下, play 是通过 play-n 调用的, 这实际上是在 1.0 上调用 gen ,没有超时,以完成动画并停止幻灯片的自动前进。 play-n 函数也可以管理多步动画。

如果 skip-first? 是 #f, 那么就会少生成一张幻灯片, 因为在 0.0时不调用 gen 。

title, name, aspect 和 layout 参数被传递给 slide, 至少当 title 和/或 name 不是函数时是这样。 当 title 或 name 是一个函数时,该函数被应用到用于生成幻灯片内容的值上, 而产生的title或name被传递给 slide 。

comment 参数就像 slide 的评论参数一样被使用。

在浓缩模式下(即,当 condense? 为 #t 时),任何会被登记为超时的幻灯片反而会被跳过。

Changed in version 1.7 of package slideshow-lib: 增加了 aspect 参数。

procedure

(play-n gen*    
  [#:steps steps    
  #:delay delay-secs    
  #:skip-first? skip-first?    
  #:skip-last? skip-last?    
  #:title title    
  #:name name    
  #:aspect aspect    
  #:comments comment    
  #:layout layout]) → void?
  gen* : 
(and/c (unconstrained-domain-> pict?)
       (λ (x) (number? (procedure-arity x))))
  steps : 
(list*of exact-positive-integer?
         (or/c exact-positive-integer? '()))
   = (current-play-steps)
  delay-secs : real? = 0.05
  skip-first? : any/c = #f
  skip-last? : any/c = #f
  title : 
(or/c string? pict? #f
      ((real-in 0.0 1.0) . -> . (or/c string? pict? #f)))
   = #f
  name : 
(or/c string? #f
      ((real-in 0.0 1.0) . -> . (or/c string? #f)))
   = title
  aspect : aspect? = #f
  comment : (list*of comment? (or/c comment? #f '())) = #f
  layout : (or/c 'auto 'center 'top 'tall) = 'auto
通过调用 gen* 生成一连串的幻灯片,每个参数的数值从 0.0 到 1.0 。 如果 gen* 接受 n 个参数,那么结果是一连串的动画, 每个 n 段之间有一个停顿(即不是自动前进)。

如果 gen* 接受一个参数,那么 play-n 就像 play 一样, 只是 gen* 也是用 1.0 调用的,以生成一个没有超时的幻灯片。 如果 gen* 接受多个参数,那么通过调用 gen* , 第一个参数从 0.0 到 1.0 不等,而其他参数都是 0.0 ,就可以生成幻灯片。 然后,第一个参数保持在 1.0 ,而第二个参数从 0.0 到 1.0 变化,以此类推。

比如说

(play-n
 (lambda (n1 n2)
   (cellophane (t "Hello")
               (* n1 (- 1.0 n2)))))

生成一个动画来淡化 "Hello" 这个词,然后暂停手动前进,然后再淡化 "Hello"。

如果 skip-first? 是 #t, 那么序列中的第一个幻灯片就被跳过。 同样地,如果 skip-last? 是 #t, 那么序列的最后一张幻灯片就被跳过。

steps 参数控制在动画的每个阶段发生多少个步骤。如果它是一个数字,那么每个阶段就使用这个数字。 如果它是一对两个数字,那么第一个数字用于第一阶段,第二个数字用于其余的阶段。 同样,如果是 (cons num_1 (cons num_2 num_3)), num_1 和 num_2 用于前两个阶段, num_3 用于其余阶段。

comment 参数中的元素与 steps 参数一样使用,只是作为注释传递,而不是作为步骤计数使用。

delay-secs, title,name, aspect 和 layout 参数被传递给 n 段动画的 play 中。

Changed in version 1.7 of package slideshow-lib: 增加了 aspect 参数。

procedure

(animate-slide element ...)

 → (() (listof (real-in 0.0 1.0)) . ->* . pict?)
  element : 
(flat-rec-contract elem/c
  (or/c pict? 'next 'alts
       (listof (listof elem/c))))
接受与 slide 类似的带有 'next 和 'alts 的幻灯片内容, 并产生一个适合用于 play-n 的程序。 其结果类似于使用 slide , 但在 'next 和 'alts 的转换中使用了渐变(也许是为了更好地适应周围动画的风格)。

parameter

(current-play-steps) → exact-positive-integer?

(current-play-steps n) → void?
  n : exact-positive-integer?
一个参数,决定了滑动动画的默认步数。 该参数的初始值是 10 。

Added in version 1.6 of package slideshow-lib.
