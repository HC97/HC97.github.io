1.11 创建可执行文件

DrRacket 的 Create Executable... 菜单项可以为你的程序创建一个可执行文件, 你可以在不启动 DrRacket 的情况下启动它。 要创建可执行文件,首先要将程序保存到文件中,并设置语言和教学包。 点击 Run,只是为了确保程序按照你的期望运行。 你创建的可执行程序不会有一个读-评价-冲刺-循环,所以在创建可执行程序之前,一定要在定义窗口中有一个启动你的程序运行的表达式。

一旦你对你的程序满意,从 Racket 菜单中选择 Create Executable... 菜单项。 你会被要求选择一个可执行文件名或一个存档文件名。 在后一种情况下,解压生成的归档文件(在这台机器上或另一台机器上)以访问可执行文件。 无论哪种情况,你都能以启动计算机上任何其他程序的方式启动该可执行文件。

Create Executable... 的结果是 launcher executable, stand-alone executable 或 a distribution archive, 并且它使用 Racket (文本) 或 GRacket (图形)引擎。 对于用某些语言实现的程序, Create Executable... 会提示你选择可执行文件的类型和引擎,而其他语言只支持一种类型或引擎。

每种类型都有优点和缺点:

  • 一个 launcher executable 在启动时使用你的程序源文件的最新版本。 它在运行时还会访问你的 DrRacket 安装中的库文件。 由于启动器可执行文件包含访问这些文件的特定路径,因此启动器通常不能从一台机器移到另一台机器。

  • 一个 stand-alone executable 嵌入了你的程序和你的程序使用的任何 Racket 库的编译副本。 当可执行文件被启动时,它使用嵌入的副本,不需要你的原始源文件。 然而,它可能会访问你的 DrRacket 安装的 DLLs、框架、共享库或辅助执行程序。 因此,独立的可执行文件通常不能从一台机器移到另一台机器。

  • 一个 distribution archive 将独立的可执行文件与任何需要的 DLLs、框架、共享库和辅助执行程序打包在一起。 发行档案可以在任何与你的操作系统相同的机器上解压和运行。

当 DrRacket 从程序的源代码中推断出一种语言时,它给了你最多的选择。 其他大多数语言只允许一种类型的可执行文件。 例如,教学语言会在发行版中创建独立的可执行文件。 传统的语言只创建启动器。

提示: 在创建可执行文件之前,请在语言对话框中禁用调试功能。 启用调试后,你会看到带有错误信息的堆栈跟踪,但你的程序会运行得更慢。 要禁用调试,请打开语言对话框,点击 Show Details 按钮, 并选择 No debugging or profiling ,如果它可用的话。

,如果它可用的话。

  • 在 Windows 上,提供一个 ".ico" 文件作为图标。 只使用 ".ico" 文件中的 16x16、32x32 或 48x48 图像。

  • 在 Mac OS 上,提供一个 ".icns" 文件作为图标。 你可以用一个 ".creator" 文件来设置应用程序的创建者(它的前四个字节被使用), 你可以通过一个 ".utiexports" 文件来设置应用程序的文件 (更多信息请参见 create-embedding-executable 中的 'uti-exports )。

  • 在 Unix 上,提供一个 ".png" 或 ".ico" 文件作为图标。
