3.2 首选项

首选项对话框由几个面板组成。

3.2.1 字体

该面板控制 DrRacket 使用的主要字体。

3.2.2 颜色

Colors面板有几个子面板,可以配置 DrRacket 用于编辑器背景的颜色, 用于突出显示匹配的括号,用于 Racket 和 Java 模式的语法着色, 用于 Check Syntax, 以及用于 interactions window 中文本的颜色。

它还有两个按钮, White on Black 和 Black on White, 为颜色偏好设置一些默认值,并改变 DrRacket 行为的一些其他方面,使 DrRacket 的颜色在这两种模式下看起来更漂亮。

3.2.3 编辑

Editing 面板由几个子面板组成:

  • Indenting

    这个面板控制 DrRacket 识别哪些关键词用于缩进,以及如何处理每个关键词。

    为了决定某一行的缩进方式,DrRacket 从该行的开头开始,寻找一个封闭的 S 表达式。 它找到该开放小括号后的第一个符号,并以此来决定如何缩进该行。

    如果该符号没有列在首选项对话框的任何列表中,那么它就像函数调用一样缩进,例如

    (f a

       b

       c)

    (f

     a

     b

     c)

    如果该符号被列在 Begin-like Keywords 部分,那么当第一个子表达式在不同的行上时, 它会得到一个额外的空格,否则它看起来就像一个应用程序。

    (begin a

           b

           c)

    (begin

      a

      b

      c)

    如果该符号被列在 Define-like Keywords 部分,那么所有的子表达式都会多缩进一个空格,无论第一行发生什么:

    (define x

      1)

    (struct s t

      (a b c))

    (define

      (f x)

      1)

    如果该符号被列在 Lambda-like Keywords 部分,那么它的缩进方式与 define 相同, 只是当第一个子表达式与该关键词不在同一行时,会有三个额外的缩进空间:

    (λ (x y)

      1)

    (lambda

        (x y)

      1)

    如果该符号被列在 For/fold-like Keywords 部分, 那么它的缩进就像 define 部分一样,只是前两个子表达式被特别处理:

    (for/fold ([acc 0])

              ([x (in-range 10)]

               [y (in-list l)])

      b

      c)

    如果该符号没有被列在各部分的任何一个列表框中,就会查阅 Extra Regexp 面板的内容。 它们是 regular expressions ,与符号的打印表示法相匹配,如果它们相匹配,就会使用相应的缩进样式。

  • Square bracket

    该面板控制 DrRacket 使用哪些关键词来决定何时将 [ 改写为 ( 。 对于类似 cond 的关键词,括号中的数字表示在开始使用方括号之前跳过多少个子表达式。

    关于列中条目的行为细节,请参见 用括号进行编辑 。

  • General Editing

    • Map delete to backspace — 如果选中,编辑器会将删除键视为退格键。

    • Wrap words in editor buffers — 如果选中,DrRacket 编辑器默认会自动包住文本行。 改变此偏好仅影响新窗口。

    • Enable keybindings in menus — 如果选中,一些 DrRacket 菜单项会有按键绑定。 否则,任何菜单项目都没有按键绑定。 这个选项是为那些习惯于用 Emacs 编辑的人设计的,他们发现标准菜单的按键绑定会干扰 Emacs 的按键绑定。

    • Treat command key as meta — 如果选中,DrRacket 将使用命令键进行一些类似 Emacs 的按键绑定, 而不是将其用于菜单快捷键。该选项仅适用于Mac OS。

    • Color syntax interactively — 如果选中,DrRacket 会在你输入时为你的语法着色。

    • Search using anchors — 如果选中,DrRacket 的搜索模式将直接跳到第一个搜索结果, 如果搜索字符串改变,则使用 "锚点" 来确定搜索位置。

    • Normalize pasted strings — 如果选中,DrRacket 会调整粘贴到编辑器中的字符串以避免混淆。 例如,非破折号看起来就像空格,但不像普通空格那样被视为分隔符。 如果勾选这个选项,DrRacket 会自动将这些非断裂空格变成普通空格。其他(不太常见)的字符也是如此。

    • Enable overwrite mode keybindings — 如果勾选, DrRacket 将启用插入键盘绑定,以切换到覆盖模式。

    • Enable automatic parentheses — 如果选中, 输入开放小括号、大括号、方括号、双引号或竖条字符会自动插入一个匹配的字符。 此外,在许多情况下,当光标位于此类字符之间时,退格将自动删除匹配的空对。

    • Add one pixel of extra space between lines — 如果选中,那么在编辑器中的行与行之间会添加一个额外像素的空白。 默认值是特定于平台的;一些字体(特别是那些带有 ╔══╗ 字符的字体)只有在不勾选的情况下才会看起来正确。

    • Always use the platform-specific linefeed convention — 如果选中,DrRacket 总是用 CRLF 行结束符保存文件。 如果不勾选,DrRacket 在打开每个文件时,如果每一行都以 CRLF 结尾(而且至少有一行),则以 CRLF 结尾保存文件, 否则就以 LF 结尾保存(遵循 Mac OS 和 Linux 的惯例)。 当文件要用 CRLF 结尾保存时,DrRacket 窗口底部的状态行会显示 "CRLF"。

      该选项仅在 Windows 下可用。 在其他操作系统中,所有文件总是以 LF 行结束符保存。

    • Maximum character width guide — 如果选中,DrRacket 会在超过给定最大宽度的文件中画一条竖线。 垂直线显示给定最大宽度的位置。

    • Show line numbers — 如果选中,DrRacket 会在左侧栏中显示正在编辑的文件的行号。

    • Show definitions/interactions labels — 如果选中,那么教学语言会显示大的友好标签, 表明哪个窗口是定义窗口,哪个是交互窗口。]

  • Racket

    • Highlight between matching parens — 如果选中,当闪烁的圆点在小括号旁边时, 编辑器会用灰色背景(彩色)或波纹图案(单色)标记匹配小括号之间的区域。

    • Automatically adjust closing parens — 如果选中,编辑器会自动将输入的 ) 转换为 ] 以匹配 [ , 或者将输入的 ] 转换为 ) 以匹配 ( 。

    • Automatically adjust opening square brackets 如果选中, 编辑器会改变输入的 [ 以匹配上下文(如 用括号进行编辑 中的解释)。

    • Flash paren match — 如果选中,键入的闭合小括号、方括号或引号会闪烁匹配的开放小括号/括号/引号。

3.2.4 警告

  • Ask before changing save format — 如果勾选, DrRacket 在保存非文本格式的文件前会询问用户(见 程序文件)。

  • Verify exit — 如果选中,DrRacket 在退出前会咨询用户。

  • Ask about normalizing strings — 如果选中,DrRacket 会在规范化粘贴到编辑器中的字符串之前咨询用户。

  • Only warn once when executions and interactions are not synchronized — 如果选中,DrRacket 会在改变定义窗口、语言或教学包后的第一次交互时警告用户, 而不相应点击 Run 。否则,警告会出现在每次交互中。

  • Ask about clearing test coverage — 如果选中,当显示测试覆盖率注释时,DrRacket 会提示删除这些注释。 该设置仅适用于PLT语言。DrRacket 在教学语言中从不询问。

  • Check for newer Racket versions — 如果选中, DrRacket 会定期轮询服务器以确定是否有更新的 DrRacket 版本。

3.2.5 常规

  • Number of recent items — 控制 Open Recent 菜单(在 File 菜单中)的长度。

  • Make backups for unsaved files — 如果选中, 编辑器会对三十秒后未保存的文件生成打开文件的副本(见 备份和首次更改文件) 。

  • Create first-change files — 如果选中,在每个编辑会话中首次保存文件时, 文件的原始副本将被复制到同一目录中的一个新文件。 新文件的名称与原文件相同,只是以 ".bak" 或 "~" 结尾。

  • Show status-line — 如果选中,DrRacket 会在每个窗口的底部显示状态线。

  • Count column numbers from one — 如果选中,状态行的列数计数器从一开始计算。 否则,从零开始计算。

  • Display line numbers in buffer; not character offsets — 如果选中,状态行会显示当前选择的 ‹line›:‹column› ,而不是文本中的字符偏移量。

  • Automatically print to PostScript file — 如果选中,打印将自动保存 PostScript 文件。 如果没有,打印将使用你电脑的标准打印机制。

  • Open files in separate tabs (not separate windows) — 如果选中,DrRacket 将使用最前面的窗口中的标签来打开新文件,而不是为新文件创建新窗口。

  • Automatically open interactions window when running a program — 如果选中,DrRacket 会在运行程序时显示交互窗口(如果它被隐藏)。

  • Automatically switch to the module language when opening a module — 如果选中,DrRacket 将识别有 #lang 行的文件并自动调整语言设置。

  • Put the interactions window beside the definitions window — 如果选中,DrRacket 会把交互窗口放在定义窗口的右边。 默认情况下,交互窗口在定义窗口的下面。

  • Always show the #lang line in the Module language — 如果选中,模块语言始终显示 #lang 行(即使它通常会被滚出页面),假设 #lang 行是文件的第一行。

  • Show test coverage results summary — 如果选中,DrRacket 会显示一个小提示, 表明除了不改变颜色外,所有测试都被覆盖了。 如果不勾选,当测试覆盖率被启用但所有代码都被覆盖时,DrRacket 保持沉默(不改变)。

  • Save files whenever switching tabs or windows — 如果选中,DrRacket 在窗口或标签之间切换时将自动保存任何未保存的文件。

  • Don’t ask and don’t save files when clicking Run with unsaved tabs or windows — 如果选中, DrRacket 在点击 Run 按钮时不会显示关于未保存标签的警告对话框,也不会自动保存文件。 如果勾选此选项,之前的复选框的值将被忽略。

3.2.6 剖析

这个偏好面板配置了剖析报告。 颜色带显示了剖析函数所采取的颜色范围。 靠近右边的颜色用于不经常被调用的代码,右边的颜色用于经常被调用的代码。

如果你对低端的更多细节感兴趣,选择 Square root 复选框。 如果你对高端的更多细节感兴趣,请选择 Square 复选框。
