2.3 How to Design Programs 教学程序

DrRacket 的五种语言是专门为教学设计的:

  • Beginning Student language 是 Racket 的一个小版本,是为计算机科学初学者量身定做的。

  • Beginning Student with List Abbreviations languages 是初级学生语言的扩展, 它用 list 代替 cons 来打印列表,并接受 quasiquote 输入。

  • Intermediate Student language 增加了局部绑定和高阶函数。

  • Intermediate Student with Lambda language 增加了匿名函数。

  • Advanced Student language 增加了易变的状态。

这些教学语言在很多方面都与传统的 Racket 不同:

  • Case-sensitive identifiers and symbols — 在大小写敏感的语言中, 变量名 x 和 X 是不同的,符号 'x 和 'X 也是不同的。 在不区分大小写的语言中, x 和 X是等同的, 'x 和 'X 代表相同的值。 教学语言默认是大小写敏感的。 大小写敏感度可以通过语言选择对话框的细节部分进行调整。

  • All numbers are exact unless #i is specified — 在 Beginning Student 到 Intermediate Student with Lambda languages 中,包含小数点的数字被解释为精确数字。 这种解释允许学生使用熟悉的十进制符号,而不会无意中触发不精确的算术。 带有小数点的精确数字也以小数点形式打印。不精确的输入和结果明确地用 #i 标记。

  • Procedures must take at least one argument — 在 Beginning Student 到 Intermediate Student 语言中,定义的程序必须至少有一个参数。 由于这些语言没有副作用,零参数的函数是没有用的,拒绝这样的函数定义有助于发现混乱的语法错误。

  • Identifier required at function call position — 在 Beginning Student 到 Intermediate Student 语言中, 过程调用必须是 (identifier ...) 的形式。 这一限制有助于检测括号的混乱误用,例如 (1) 或 ((+ 3 4)), 这在习惯了代数的可选括号的初学者中是一个常见的错误。

  • Top-level required at function call position — 在 Beginning Student 语言中,过程调用必须是 (top-level-identifier ...) 的形式, 如果 define 了 top-level-identifier ,实际参数的数量必须与形式参数的数量一致。 这一限制有助于检测括号的混乱误用,如 (x) ,其中 x 是一个函数参数。 DrRacket 可以从语法上检测出这种错误,因为 Beginning Student 不支持高阶程序。

  • Primitive and defined functions allowed only in function call position — 在 Beginning Student 语言中, 原始运算符或定义函数的名称只能在函数调用的开括号之后使用 (除非教学包扩展允许其他情况,如 convert-gui 教学包中)。对基元的不正确使用会触发一个语法错误。 对定义名称的不正确使用会引发运行时错误。 DrRacket 可以检测到这种错误,因为 Beginning Student 不支持高阶程序。

  • lambda allowed only in definitions — 在 Beginning Student 到 Intermediate Student 语言中, lambda 只能出现在定义中,而且只能作为定义变量的值。

  • Free variables are not allowed — 在 Beginning Student 到 Advanced Student 语言中, 定义窗口中引用的每个变量都必须是定义的、预先定义的,或者是局部函数参数的名称。

  • quote works only on symbols, quasiquote disallowed — 在 Beginning Student 语言中, quote 和 ’ 只能指定符号。 这一限制避免了向初学者解释为什么 1 和 '1 在标准 Racket 中是等同的。 此外, quasiquote, `, unquote, ,, unquote-splicing 和 ,@ 也是不允许的。

  • Unmatched cond/case is an error — 在 Beginning Student 到 Advanced Student 语言中, 在没有匹配子句的情况下通过 cond 或 case 表达式是一个运行时错误信号。 这一惯例有助于检测程序中的语法和逻辑错误。

  • Conditional values must be true or false — 在 Beginning Student 到 Advanced Student 语言中, 一个表达式的值被当作布尔值,必须返回一个实际的布尔值,即 true 或 false 。 这一限制适用于 if, cond, and 和 or 表达式, 有助于检测省略布尔函数应用的错误。

  • +, *, and / take at least two arguments in Beginning languages. — 在 Beginning Student 和 Beginning Student with List Abbreviations 语言中,代数符号中的数学运算符在 DrRacket 中需要至少两个参数。 这一限制有助于发现运算符的参数缺失。

  • and, or, nand, and nor require at least 2 expressions — 在 Beginning Student 到 Advanced Student 语言中,布尔组合形式要求至少有两个子表达式。 这一限制有助于检测布尔表达式中缺失或不符合格式的子表达式。

  • set! disallowed on arguments — 在 Advanced Student 语言中, set! 不能被用来替换被 lambda 绑定的变量。 这一限制确保了函数应用的替换模型与 DrRacket 的评估一致。

  • Improper lists disallowed — proper list 是指空列表或通过 cons 到恰当的列表上创建的列表。 在 Beginning Student 到 Advanced Student 语言中, cons 只构造 proper lists, 如果第二个参数不是适当的列表,就会提示错误。 由于初级学生不需要不恰当的列表,这一限制有助于检测递归函数中的逻辑错误。

  • Dot is disallowed — 在 Beginning Student 到 Advanced Student 语言中,不允许使用划线的句号 . , (例如,作为引号形式中的不正当列表构造函数,或者用于定义多极程序)。

  • Syntactic form names disallowed as variable names — 在 Beginning Student 到 Advanced Student 语言中, 所有的语法形式名称都是关键词,不能作为变量名称使用。

  • Re-definitions are disallowed — 在 Beginning Student 到 Advanced Student 语言中,顶级名称永远不能被重新定义。

  • Function definitions are allowed only in the definitions window — 在 Beginning Student 语言中,不允许在交互窗口中进行函数定义。

教学语言在打印数值方面也偏离了传统的 Racket。 通过语言选择对话框的细节部分,可以为任何语言选择不同的打印格式。

  • Constructor-style output — 参见 输出打印样式 。

  • Quasiquote-style output — 参见 输出打印样式 。

  • Rational number printing – 在教学语言中,所有具有有限小数扩展的数字都以小数形式打印。 对于那些没有有限小数扩展的数字(如 4/3) ,DrRacket 提供了一个选择。 它要么以混合分数形式打印,要么以重复小数形式打印,其中小数扩展的重复部分以大括号显示。 此外,DrRacket 只显示数字的小数扩展的前 25 位。 如果有更多的数字,数字的末尾会出现一个省略号。点击省略号可以看到后面 25 位的扩展。

    这个设置只控制一个数字的初始显示。 在数字上点击右键或点击控制键(Mac OS)可以让你从分数表示法变为小数表示法。

  • write output — 用 write 的方式打印数值。

  • Show sharing in values — 使用 shared 语法打印交互结果,它暴露了一个值中的共享结构。 例如,由 (let ([lt (list 0)]) (list lt lt)) 创建的列表打印为

    (shared ((-1- (list 0))) (list -1- -1-))

    而不是

    (list (list 0) (list 0))

A program in the teaching languages should be tested using the check forms — (check-expect value value), (check-within value value value) 或 (check-error value string) 进行测试。 测试在运行程序时进行评估:当没有测试时,在交互窗口中出现警告; 当所有测试成功时,在交互窗口中出现确认;否则,出现一个测试窗口报告结果。 关于配置报告行为的细节,见 查看 。

如果有必要,测试可以被禁用,详情见 Racket 。

这些教学语言与其他语言的最后一个区别是它们保存文件的方式。 也就是说,当 DrRacket 保存文件且当前语言是这五种教学语言之一时,它会插入三行元数据,记录精确的语言(包括任何选项设置)和教学包。 这有两个好处:稍后打开文件可以恢复设置,而且元数据是以可执行代码的方式制定的, 因此在 shell 中对该文件运行 racket 或 gracket 会以适当的语言运行程序。

这个元数据总是由三行组成,因此可以通过三次对 read-line 的调用来剥离出来。
