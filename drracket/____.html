1.7 图形语法

除了正常的文本程序外,DrRacket 还支持某些图形元素作为程序中的表达方式。 插件工具可以扩展可用的图形语法,但本节介绍一些更常用的元素。

1.7.1 图像

DrRacket 的 Insert|Insert Image... 菜单项可以让你从磁盘上选择一个图像文件(有 GIF、PNG 和 BMP 等多种格式), 该图像会被插入到当前的编辑光标处。

作为一个表达式,图像的行为类似于数字或字符串常数:它对自己进行评估。 DrRacket 的 interactions window 知道如何绘制图像值结果或通过 print 显示的图像。

程序可以通过各种方式处理图像值,如使用 htdp/image 库或作为 image-snip% 值。

1.7.2 XML 框和 Racket 框

DrRacket 对 XML 具体语法有特殊支持。 Special|Insert XML Box 菜单项会在你的程序中插入一个嵌入式编辑器。 在该嵌入式编辑器中,你可以输入 XML 的具体语法。 当一个包含 XML 框的程序被评估时,XML 框被翻译成一个 x-expression (或 xexpr), 这是一个 XML 表达式的 S 表达式表示。 每个 xexpr 是一个列表,其第一个元素是命名标签的符号,第二个元素是代表属性的关联列表,其余元素是嵌套的 XML 表达式。

XML 盒子有两种处理空白的模式。 在一种模式下,所有的空白在产生的 xexpr 中被完整地保留。 在另一种模式下,任何只包含嵌套的 XML 表达式和空白的标签都会被删除空白。 你可以在这些模式之间切换,方法是在 XML 框的顶部部分点击右键或点击控制键(Mac OS)。

除了包含 XML 文本外,XML 框还可以包含 Racket 框。Racket 框包含 Racket 表达式。 这些表达式被评估,其内容被放入包含 XML 框的 xexpr 中。 有两种类型的 Racket 框:标准 Racket 框和拼接 Racket 框。 标准 Racket 框将其值插入到包含 xexpr 中。 拼接框的内容必须评估为一个列表,列表中的元素被 "压扁" 到包含 xexpr 中。 在 Racket 框的顶部点击右键或点击控制键(Mac OS)可以打开一个菜单,在 Racket 框和 Racket 拼接框之间进行切换。
