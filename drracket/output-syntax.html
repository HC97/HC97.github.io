2.5 输出打印样式

许多 Racket 语言都支持 Output Syntax 的选择, 它决定了评估结果如何在 interactions window 中打印。 这个设置也适用于通过明确调用 print 产生的输出。

Constructor output 模式类似于 Rackets 的正常打印模式, 只是即使是可引用的也是用构造函数打印的,构造函数和表格被用来近似一些本来不可打印的值。 例如, Constructor 输出以 lambda 形式打印一个程序。 对于向图形环境的输出,有理数是使用一个特殊的 snip% 对象打印的, 该对象允许用户在不完全分数、混合分数和重复小数之间进行选择。

Quasiquote output 模式与 Constructor 输出类似, 但它使用 quasiquote (用 ` 缩写)来打印列表, 并使用 unquote (用 , 缩写)在需要时转回到 Constructor 打印。

write output 模式对应于通过 write 程序的传统 Scheme 打印。 例如,列表的打印方式是将列表元素的打印形式放在括号内,没有前导引号或构造函数名称。

print output 模式对应于 Racket 默认的通过 print 程序进行的打印。 通过 print 输出是可以通过运行时设置进一步配置的,比如 print-as-expression 参数, 它可以由 #lang 指定的语言来调整。 例如, scheme 语言将 print-as-expression 参数设置为 #f, 这实质上使 print 模式与 write 模式一样。

Constant Style 选项存在时,控制 true, false 和 empty 的打印方式; 对于布尔运算,它决定是否有 # 前缀,对于空列表,决定它是打印成 empty 还是 '() 。

对于任何一种输出方式,DrRacket 都会设置 global-port-print-handler ,从而使 print 过程产生所选输出。
