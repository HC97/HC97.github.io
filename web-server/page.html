8 页: 常用模式的简称

 (require web-server/page) package: web-server-lib

Web Server 提供了一个简单的工具库,用于构建主要由 send/suspend/dispatch 创建的页面和请求处理组成的 Web 应用程序。

大多数 Web 应用程序在很大程度上依赖于 send/suspend/dispatch ,并且通常使用该模式:
(send/suspend/dispatch
 (λ (my-embed/url)
   .... (my-embed/url other-page) ....))

syntax

(page e ...)

page 宏通过将 (page e ...) 扩展为 send/suspend/dispatch 的用法, 其中语法参数 embed/url 被绑定到 send/suspend/dispatch 的参数上,实现了自动化。

syntax

embed/url

当在 page 里面使用时,在语法上是一个程序嵌入函数的重名转换器;否则,是一个语法错误。

一个简单的例子:
(page
 (response/xexpr
  `(html
    (body
     (a ([href
          ,(embed/url
            (λ (req)
              "You clicked!"))])
        "Click me")))))

同样地,许多 Web 应用程序几乎完全使用了作为 embed/url 的参数的函数, 并立即调用 send/suspend/dispatch 。

syntax

(lambda/page formals e ...)

syntax

(define/page (id . formals) e ...)

lambda/page 和 define/page 通过扩展到接受请求作为第一个参数的函数 (后面是 formals 中指定的任何参数)并立即将其主体包裹在 page 中,从而实现自动化。 这个函数也与 get-binding 合作,将请求与 current-request 参数绑定。

web-server/http 的绑定接口很强大,但由于其对敌对客户端的保护,使用起来很微妙,很方便。

parameter

(current-request) → request?

(current-request req) → void?
  req : request?

value

binding-id/c : contract?

value

binding-format/c : contract?

procedure

(get-binding id [req #:format format])

 → (or/c false/c string? bytes? binding?)
  id : binding-id/c
  req : request? = (current-request)
  format : binding-format/c = 'string

procedure

(get-bindings id [req #:format format])

 → (listof (or/c string? bytes? binding?))
  id : binding-id/c
  req : request? = (current-request)
  format : binding-format/c = 'string
get-binding 接口试图通过提供一个强大的接口和方便的默认值来解决这个问题。

get-binding 从一个请求中提取表单输入的第一个绑定,而 get-bindings 则提取所有绑定。

它们接受一个表单标识符 (id) ,可以是一个字节串,一个字符串,或一个符号。 在每一种情况下,用户输入的内容都会以区分大小写的方式与表单输入进行比较。

它们接受一个可选的请求参数 (req) , 默认为 lambda/page 和 define/page 使用的 current-request 参数的值。

最后,它们接受一个可选的关键字参数 (format),指定所需的返回格式。 默认的 'string 产生一个 UTF-8 字符串(如果字节字符串不能转换为 UTF-8,则为 #f )。 'bytes 格式总是产生原始字节字符串。 'file 格式产生上传的文件内容(如果表单输入不是上传的文件,则产生 #f )。 'binding 格式产生绑定对象。
