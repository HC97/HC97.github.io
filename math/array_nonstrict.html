6.5 非限制性数组

除了少数例外,默认情况下,由 math/array 导出的函数会返回 strict 数组, 即数组的 procedures 通过在矢量中查找元素来计算。

这种保守的默认值往往会浪费时间和空间。 在对数组进行操作的函数式代码中,大多数中间数组中的元素正好被引用一次, 所以为它们分配和填充存储空间应该是不必要的。 例如,考虑下面这个数组:
> (define (make-hellos)
    (array-map string-append
               (array-map string-append
                          (array #["Hello " "Hallo " "Jó napot "])
                          (array #["Ada" "Edsger" "John"]))
               (make-array #(3) "!")))
> (define arr (make-hellos))
> (array-strict? arr)

- : Boolean

#t

> arr

(array #["Hello Ada!" "Hallo Edsger!" "Jó napot John!"])

默认情况下,内部 array-map 的结果为其分配了存储空间, 并填充了诸如 "Hello Ada" 这样的字符串,尽管其存储空间将在下一个垃圾收集周期被扔掉。

随着 Racket 对并行计算支持的提高,另一个问题变得更加重要。 为中间数组分配存储空间是长计算中的一个同步点,它将长计算分成许多短计算,使其难以并行化。

* Haskell 中的规则、形状多态的并行数组, Gabriele Keller, Manuel Chakravarty, Roman Leshchinskiy, Simon Peyton Jones, and Ben Lippmeier. ICFP 2010. (PDF)

一个解决方案是构建 nonstrict arrays*, 即数组的程序可以做更多的事情而不是简单地查找元素。 将参数 array-strictness 设置为 #f 会使几乎所有的 math/array 函数都返回非严格的数组:
> (define arr (parameterize ([array-strictness #f])
                (make-hellos)))
> (array-strict? arr)

- : Boolean

#f

> arr

(array #["Hello Ada!" "Hallo Edsger!" "Jó napot John!"])

在 arr 中,第一个元素是计算 computation (string-append (string-append "Hello " "Ada") "!"), 而不是值 "Hello Ada!" 。 每次提到第一个元素时, "Hello Ada!" 的值都会被重新计算。

为了有效地使用非严格的数组,把每个数组都看成是数组的 procedure 本身。换句话说、

一个 array 只是一个具有有限矩形域的函数。

有些数组是可变的,有些是懒惰的,有些是严格的,有些是稀疏的,大多数甚至不分配连续的空间来存储其元素。 所有这些都是可以应用于索引来检索元素的函数。

两种最常见的操作,在数组上映射和转换数组,是一种组合。 将 f 映射到数组 arr 上,只不过是将 f 与 arr 的程序进行组合。 使用 g(一个从新索引到旧索引的函数)来转换 arr , 只不过是将 arr 的过程与 g 进行组合。

6.5.1 缓存非限制性元素

非严格数组不是懒惰的。 很少有非严格数组会缓存计算过的元素,但是和函数一样,每次被引用时都会重新计算。 与函数不同的是,通过使它们成为 strict ,它们可以一次性地计算和缓存每个元素。

要计算和存储一个数组的元素,请使用 array-strict! 或 array-strict:
> (array-strict? arr)

- : Boolean

#f

> (array-strict! arr)
> (array-strict? arr)

- : Boolean

#t

> (array-strict arr)

- : #(struct:Array

      (Indexes Index (Boxof Boolean) (-> Void) (-> Indexes String))

      #<syntax:/home/ice/racket-8.9-2023-07-14/share/pkgs/math-lib/math/private/array/typed-array-struct.rkt:55:13 prop:custom-write>

      #<syntax:/home/ice/racket-8.9-2023-07-14/share/pkgs/math-lib/math/private/array/typed-array-struct.rkt:54:13 prop:custom-print-quotable>)

(array #["Hello Ada!" "Hallo Edsger!" "Jó napot John!"])

如果数组已经是严格的,如上面的最后一个例子, array-strict! 和 array-strict 不会做任何事情。

要对一个数组进行严格的 copy 而不使原来的数组变得严格,可以使用 array->mutable-array 。

6.5.2 性能方面的考虑

非严格数组的一个缺点是对其操作的性能进行推理比较困难。 另一个缺点是,用户必须决定哪些数组是严格的。 幸运的是,有一个简单的经验法则:

当你必须引用大多数元素超过一次或两次时,就要使数组严格。

必须为一个数组命名是一个很好的指标,表明它应该是严格的。 在下面的例子中,计算 (+ (expt x x) (expt x x)) 的 x 从 0 到 2499, xrr 中的每个元素在参考 res 中的相应元素时被计算两次:
(define xrr (array-map expt
                       (index-array #(50 50))
                       (index-array #(50 50))))
(define res (array+ xrr xrr))
必须命名 xrr 意味着我们应该让它变得严格:
(define xrr (array-strict
             (array-map expt
                        (index-array #(50 50))
                        (index-array #(50 50)))))
(define res (array+ xrr xrr))
这样做可以使计算 res 的元素的时间减半。

当从一个函数中返回一个数组时,应按原样返回非严格的数组,以便让调用者决定其结果是否应该是严格的。

当编写可能被调用的 (array-strictness #t) 或 (array-strictness #f) 的库函数时, 对非严格数组进行操作,并用 array-default-strict 来包装结果,以返回用户所期望的。 例如,如果 make-hellos 是一个库函数,它应该被写成
(define (make-hellos)
  (array-default-strict
   (parameterize ([array-strictness #f])
     (array-map string-append
                (array-map string-append
                           (array #["Hello " "Hallo " "Jó napot "])
                           (array #["Ada" "Edsger" "John"]))
                (make-array #(3) "!")))))

如果你不能确定是否使数组严格,或者使用数组进行所谓的 "动态编程", 你可以使用 array-lazy 使它们变得懒惰。
