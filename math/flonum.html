2.1 额外的 Flonum 函数

procedure

(fl x) → Flonum

  x : Real
相当于 (real->double-flonum x), 但更容易读和写。

Examples:
> (fl 1/2)

0.5

> (fl 0.5)

0.5

> (fl #i0.5)

0.5

注意 exact->inexact 并不总是将 Real 转换为 Flonum:
> (exact->inexact #i0.5)

0.5

> (flabs (exact->inexact #i0.5))

0.5

你应该选择 fl 而不是 exact->inexact, 特别是在类型化的 Racket 代码中。

procedure

(flhypot x y) → Flonum

  x : Flonum
  y : Flonum
计算 (flsqrt (+ (* x x) (* y y))) 的方式,只有当答案太大,才会溢出。

Examples:
> (flsqrt (+ (* 1e+200 1e+200) (* 1e+199 1e+199)))

+inf.0

> (flhypot 1e+200 1e+199)

1.0049875621120889e+200

procedure

(flsum xs) → Flonum

  xs : (Listof Flonum)
就像 (apply + xs), 但只产生一次舍入错误。

Examples:
> (+ 1.0 1e-16)

1.0

> (+ (+ 1.0 1e-16) 1e-16)

1.0

> (flsum '(1.0 1e-16 1e-16))

1.0000000000000002

sum 函数对异质的整数列表也有同样的作用。

最坏情况下的时间复杂度是 O(n2), 尽管观察四次方时间所需的病态输入是指数级的不可能,而且很难有目的地产生。 预期的时间复杂度是 O(n log(n)) 。

参见 flvector-sums ,其变体是计算 xs 中所有的部分和。

procedure

(fllog1p x) → Flonum

  x : Flonum

procedure

(flexpm1 x) → Flonum

  x : Flonum
像 (fllog (+ 1.0 x)) 和 (- (flexp x) 1.0), 但是当 x 很小的时候(在 1 个 ulp 内)很精确。

例如, (fllog (+ 1.0 x)) 和 (- (flexp x) 1.0) 的一个困难输入是 x = 1e-14, fllog1p 和 flexpm1 计算正确:
> (fllog (+ 1.0 1e-14))

9.992007221626358e-15

> (fllog1p 1e-14)

9.99999999999995e-15

> (- (flexp 1e-14) 1.0)

9.992007221626409e-15

> (flexpm1 1e-14)

1.0000000000000049e-14

这些函数是互逆的:
> (plot (list
         (function (λ (x) x) #:color 0 #:style 'long-dash)
         (function (compose fllog1p fl) #:label "fllog1p x")
         (function (compose flexpm1 fl) #:label "flexpm1 x" #:color 2))
        #:x-min -4 #:x-max 4 #:y-min -4 #:y-max 4)

请注意,两个图形都通过原点。 因此,接近 0.0 的输入,即 flonums 特别密集的周围,导致输出也接近 0.0 。 此外,两个函数在 0.0 附近都是近似的同一函数,所以输出密度也是近似的。

许多以 fllog 和 flexp 定义的 flonum 函数, 当它们的定义表达式被放在 fllog1p 和 flexpm1 中时,会变得更加精确。 本模块和 math/special-functions 导出的函数广泛地使用了它们。

一个臭名昭著的罪魁祸首是 (flexpt (- 1.0 x) y), 当 x 接近 0.0 时。 直接计算它往往会导致错误的答案:
> (flexpt (- 1.0 1e-20) 1e+20)

1.0

我们应该预料到,将一个刚好小于 1.0 的数字乘以它自己那么多倍,会得到小于 1.0 的结果。 问题来自于首先从 1.0 中减去这样一个小数字:
> (- 1.0 1e-20)

1.0

幸运的是,我们可以通过将表达式放在 fllog1p 中来正确计算,这就避免了容易出错的减法:
> (flexp (* 1e+20 (fllog1p (- 1e-20))))

0.36787944117144233

但请看 flexpt1p,它仍然更准确。

procedure

(fllog2 x) → Flonum

  x : Flonum
计算 x 对数比 (/ (fllog x) (fllog 2.0)) 更准确。 特别是, (fllog2 x) 对任何 2 的幂的 x 都是正确的。

Examples:
> (fllog2 4.5)

2.169925001442312

> (/ (fllog (flexp2 -1066.0)) (fllog 2.0))

-1066.0000000000002

> (fllog2 (flexp2 -1066.0))

-1066.0

最大的观察误差是 0.5006 ulps, 但几乎总是不超过 0.5(也就是说,它几乎总是 correct)。

procedure

(fllogb b x) → Flonum

  b : Flonum
  x : Flonum
计算 x 的基 b 对数比 (/ (fllog x) (fllog b)) 更准确,并正确处理极限值。

Example:
> (plot3d (contour-intervals3d (λ (b x) (fllogb (fl b) (fl x))) 0 4 0 4)
          #:x-label "b" #:y-label "x")

最大的观察误差是 2.1 ulps, 但通常小于0.7(即接近四舍五入误差)。

除了可能在极限值(如 0.0 和 +inf.0, 以及 b = 1.0)和内部表达式欠流或溢出时, 对于 b > 0.0, fllogb 大约符合这些特性:

  • 左逆: (fllogb b (flexpt b y)) = y

  • 右逆: (flexpt b (fllogb b x)) = x 当 x > 0.0 时

与 flexpt 不同, fllogb 在极限值时的行为没有标准。 幸运的是,推导出以下规则(按顺序应用)并不是非常困难的。

这些规则中的大多数都是通过对数学上的基数 b 的对数函数进行限制而得出的。 除了 (fllogb 1.0 x), 当这样做会引起歧义时, 它们会用 flexpt 的行为来解决,它遵循 IEEE 754 和 C99 的 pow 。

例如,考虑 (fllogb 0.0 0.0) 。 取一个相互关联的极限,如果外部极限是关于 x 的,我们得到 ∞,如果外部极限是关于 b 的,得到 0。 这通常意味着 (fllogb 0.0 0.0) = +nan.0 。

然而,选择 +inf.0 可以确保这些额外的左逆和右逆特性成立:
(fllogb 0.0 (flexpt 0.0 +inf.0)) = +inf.0
(flexpt 0.0 (fllogb 0.0 0.0)) = 0.0
此外,选择 0.0 并不能确保任何额外的相同点成立。

procedure

(flbracketed-root f a b) → Flonum

  f : (Flonum -> Flonum)
  a : Flonum
  b : Flonum
使用 Brent-Dekker method 方法在 a 和 b 之间找到 f 的浮点根 (一个 x : Flonum ,对它来说, (f x) 非常接近于一个过零点)。 (f a) 和 (f b) 的值必须有相反的符号,但 a 和 b 可以有任何顺序。

Examples:
> (define (f x) (+ 1.0 (* (+ x 3.0) (sqr (- x 1.0)))))
> (define x0 (flbracketed-root f -4.0 2.0))
> (plot (list (x-axis)
              (function f -4 2)
              (function-label f x0))
        #:y-min -10)

> (f (flprev x0))

-7.105427357601002e-15

> (f x0)

6.661338147750939e-16

> (flbracketed-root f -1.0 2.0)

+nan.0

注意事项:

  • 我们不能保证 flbracketed-root 能找到任何 特定的 根。 此外,未来对其实现的更新可能会使它找到不同的根。

  • 目前还不能保证它能找到 最接近 精确根的 x 。

  • 目前它最多可以运行 5000 次迭代。

通常它需要的迭代次数要少得多,特别是在初始界线 a 和 b 很紧的情况下。

procedure

(make-flexpt x) → (Flonum -> Flonum)

  x : Real
当 x 是 flonum 时,相当于 (λ (y) (flexpt x y)) , 但是当 x 不能被 flonum 精确表示时,对于大的 y 来说要精确得多。

假设我们想计算 π y, 其中 y 是一个 flonum。 如果我们使用带有无理基 π 近似值 的 flexpt , 误差在零点附近很低,但会随着离原点的距离增加:
> (bf-precision 128)

> (define y 150.0)

> (define pi^y (bigfloat->rational (bfexpt pi.bf (bf y))))

> (flulp-error (flexpt pi y) pi^y)

43.12619934359266

如果使用 make-flexpt, 误差在任何地方都接近舍入误差:
> (define flexppi (make-flexpt (bigfloat->rational pi.bf)))

> (flulp-error (flexppi y) pi^y)

0.8738006564073412

这个例子在 zeta 和 psi 的实现中使用。

2.2 对数空间算术

特别是在处理概率和概率密度时,用 log space 或其真实值的自然对数来表示非负数通常很有用。 一般来说,原因是 最小 的正数是 太大 。

例如,假设我们想要标准正态分布(钟形曲线)的概率密度在离零的 50 个标准差处:
> (require math/distributions)
> (pdf (normal-dist) 50.0)

0.0

从数学上讲,密度在任何地方都是非零的,但 50 处的密度小于 +min.0 。 然而,它在对数空间的密度,或者说它的对数密度,是可以表示的:
> (pdf (normal-dist) 50.0 #t)

-1250.9189385332047

虽然这个例子可能看起来很牵强,但在计算数据 vector 的密度时, 密度的乘积太小而无法直接表示,这是非常常见的。

在对数空间中,指数化变成了乘法,乘法变成了加法,而加法则变得很棘手。 解决方案见 lg+ 和 lgsum 。

分别等同于 (fl+ logx logy), (fl- logx logy) 和 (flsum logxs) 。

procedure

(lg+ logx logy) → Flonum

  logx : Flonum
  logy : Flonum

procedure

(lg- logx logy) → Flonum

  logx : Flonum
  logy : Flonum
分别类似于 (fllog (+ (flexp logx) (flexp logy))) 和 (fllog (- (flexp logx) (flexp logy))),但更准确,不容易发生溢出和下溢。

当 logy > logx 时, lg- 返回 +nan.0 。 两个函数都正确地将 -inf.0 视为对数空间 0.0 。

要将两个以上的对数空间的数字以相同的保证相加,请使用 lgsum 。

Examples:
> (lg+ (fllog 0.5) (fllog 0.2))

-0.35667494393873234

> (flexp (lg+ (fllog 0.5) (fllog 0.2)))

0.7

> (lg- (fllog 0.5) (fllog 0.2))

-1.203972804325936

> (flexp (lg- (fllog 0.5) (fllog 0.2)))

0.30000000000000004

> (lg- (fllog 0.2) (fllog 0.5))

+nan.0

虽然比天真的实现更准确, 但这两个函数在输出接近 0.0 (或对数空间 1.0) 的值的区域都容易出现 catastrophic cancellation 。 虽然这些输出有很高的相对误差,但它们的绝对误差却很低,当被指数化时,几乎只有舍入误差。 此外,当 logx 和 logy 本身有误差时,灾难性的取消是不可避免的,而这是迄今为止最常见的情况。

当然,这些都是浮点研究中的借口。 目前还没有合理快速的算法来计算低相对误差的 lg+ 和 lg- 。 目前,如果你需要这样的精度,请使用 math/bigfloat 。

2.3 调试 Flonum 函数

以下函数和常数在编写和调试必须在最大可能的域上准确的 flonum 函数时非常有用。

假设我们使用以 1.0 为中心的泰勒级数对 flexp 进行近似,并在三期后截断(一个二阶多项式):
(define (exp-taylor-1 x)
  (let ([x  (- x 1.0)])
    (* (flexp 1.0) (+ 1.0 x (* 0.5 x x)))))

我们可以用 plot 和 flstep (下面有记录)来比较它和 flexp 在非常小的区间内的输出:
> (plot (list (function exp-taylor-1 #:label "exp-taylor-1 x")
              (function exp #:color 2 #:label "exp x"))
        #:x-min (flstep 1.00002 -40)
        #:x-max (flstep 1.00002 40)
        #:width 480)

当以两种不同近似方法的边界为中心时,这种图特别有用。

对于较大的区间,假设被逼近的函数是相当平滑的, 我们可以用 flulp-error 得到一个更好的概念,即逼近的程度:
> (plot (function (λ (x) (flulp-error (exp-taylor-1 x) (exp x))))
        #:x-min 0.99998 #:x-max 1.00002 #:y-label "Error (ulps)")

我们可以从这个图中推断出, 我们的泰勒级数近似方法在 1.0 附近有接近四舍五入的误差(不超过一个 ulp) , 但在更远的地方很快就变得更糟。

为了得到一个真实的函数,比如 exp 来进行测试, 尽可能准确地使用精确的有理数或高精度的 bigfloats 来计算输出。

2.3.1 测量浮点误差

procedure

(flulp x) → Flonum

  x : Flonum
返回 x 的 ulp, 即 最后一位: x 中最小有效位的大小。

Examples:
> (flulp 1.0)

2.220446049250313e-16

> (flulp 1e-100)

1.2689709186578246e-116

> (flulp 1e+200)

1.6996415770136547e+184

procedure

(flulp-error x r) → Flonum

  x : Flonum
  r : Real
返回 x 和 r 之间的绝对数 ulps 之差。

对于非有理参数,如 +nan.0, flulp-error 返回 0.0 , 如果 (eqv? x r); 否则返回 +inf.0 。

一个最大误差为 0.5 ulps 的 flonum 函数只表现出舍入误差;它是 correct 。 一个最大误差不超过几个 ulps 的 flonum 函数是 accurate 。 大多数中等复杂的 flonum 函数,在直接实现时,似乎有超过十万个 ulps 的最大误差。

Examples:
> (flulp-error 0.5 1/2)

0.0

> (flulp-error 0.14285714285714285 1/7)

0.2857142857142857

> (flulp-error +inf.0 +inf.0)

0.0

> (flulp-error +inf.0 +nan.0)

+inf.0

> (flulp-error 1e-20 0.0)

+inf.0

> (flulp-error (- 1.0 (fl 4999999/5000000)) 1/5000000)

217271.6580864

* 当结果要被指数化时,你可以做一个例外。 如果 x 有小的 absolute-error, 那么 (exp x) 就有小的 relative-error 和小的 flulp-error 。 最后一个例子减去了两个附近的楞次,其中第二个已经被四舍五入了,导致了可怕的错误。 这是一个 catastrophic cancellation 的例子。 尽可能避免减去附近的 flonum。*

当近似值不一定由 flonum 来表示时,请看 relative-error 来测量近似值的类似方法。

可与 1.0 相加产生更大数字的最小楞次,或 1.0 中最小有效位的量级。

> epsilon.0

> (flulp 1.0)

Epsilon 经常被用于迭代或加法近似方法的停止条件中。 例如,下面的函数用它来停止牛顿方法计算平方根。 (请不要认为这个例子是稳健的。)
(define (newton-sqrt x)
  (let loop ([y  (* 0.5 x)])
    (define dy (/ (- x (sqr y)) (* 2.0 y)))
    (if ((abs dy) . <= . (abs (* 0.5 epsilon.0 y)))
        (+ y dy)
        (loop (+ y dy)))))
当 (<= (abs dy) (abs (* 0.5 epsilon.0 y))) 时, 将 dy 加到 y 上很少会产生不同的 flonum。 0.5 这个值可以被改变以允许更宽松的近似。 当近似值不需要尽可能接近时(例如,它只是另一种近似方法的起点), 或者已知 dy 的计算不准确时,这是一个好主意。

近似误差通常用 epsilons 的相对误差来理解。 epsilons 数的相对误差与 ulps 的误差大致对应,除非近似是次正态的。

2.4 双浮点操作

为了获得额外的精度,浮点计算可以使用两个不重叠的楞次来表示一个数字。 这种数对通常被称为 double-double 数。 这对数字的精确之和就是它所代表的数字。(因为它们是不重叠的,所以浮点数的和等于最大的那个)。

为了提高速度,特别是在算术运算中,没有双倍数的数据类型。 它们总是非盒式的:作为两个参数给出,并作为两个值接收。 在这两种情况下,量级较高的数字在前。

从来没有检查过输入以确保它们是分类的和不重叠的,但是如果输入是分类的和不重叠的,那么输出就保证是分类的和不重叠的。

procedure

(fl2 x) → (Values Flonum Flonum)

  x : Real
(fl2 x y) → (Values Flonum Flonum)
  x : Flonum
  y : Flonum
将一个实数或两个flonums的总和转换为一个 double-double 。

Examples:
> (fl 1/7)

0.14285714285714285

> (relative-error (fl 1/7) 1/7)

5.551115123125783e-17

> (define-values (x2 x1) (fl2 1/7))
> (list x2 x1)

'(0.14285714285714285 7.93016446160826e-18)

> (fl (relative-error (+ (inexact->exact x2)
                         (inexact->exact x1))
                      1/7))

3.0814879110195774e-33

注意,在前面的例子中, x2 和 x1 的精确之和的相对误差非常小。

2.4.2 调试双浮点函数

下面的语法形式是 fl+/error 等函数的快速版本。 它们之所以快速,是因为它们对参数的大小和参数之间的关系做了假设,并且没有正确地处理非有理数的双倍数楞次。

syntax

(fast-mono-fl+/error x y)

syntax

(fast-mono-fl-/error x y)

返回两个值: (fl+ x y) 或 (fl- x y), 以及它的舍入误差。 两者都假定 (flabs x) > (flabs y) 。 当 x 或 y 不是有理数时,这些值是未指定的。

syntax

(fast-fl+/error x y)

syntax

(fast-fl-/error x y)

像 fast-mono-fl+/error 和 fast-mono-fl-/error, 但不假设 (flabs x) > (flabs y) 。

syntax

(fast-fl*/error x y)

syntax

(fast-fl//error x y)

syntax

(fast-flsqr/error x)

像 fl*/error, fl//error 和 flsqr/error, 但速度更快,当一个参数是亚正常或接近无限时,可能返回垃圾。

syntax

(flsplit x)

返回非重叠的 (values y2 y1), 每个都有 26 位精度, (flabs y2) > (flabs y1), 这样 (fl+ y2 y1) = x 。 对于 (flabs x) > 1.3393857490036326e+300, 返回 (values +nan.0 +nan.0) 。

用于实现双-双乘法。
