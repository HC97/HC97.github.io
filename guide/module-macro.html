6.7 模块和宏

Racket 的模块系统与 Racket 的 macro 系统紧密合作,为Racket添加新的语法形式。 例如,就像导入 racket/base 引入 require 和 lambda 的语法一样, 导入其他模块也可以引入新的语法形式(除了更传统的导入类型,如函数或常量)。

我们将在后面的 宏 中详细介绍宏,但这里有一个定义了基于模式的宏的模块的简单例子:

(module noisy racket
  (provide define-noisy)
 
  (define-syntax-rule (define-noisy (id arg ...) body)
    (define (id arg ...)
      (show-arguments 'id  (list arg ...))
      body))
 
  (define (show-arguments name args)
    (printf "calling ~s with arguments ~e" name args)))

这个模块提供的 define-noisy 绑定是一个 macro , 它的作用类似于函数的 define ,但它使每次对函数的调用都会打印提供给函数的参数:

> (require 'noisy)
> (define-noisy (f x y)
    (+ x y))
> (f 1 2)

calling f with arguments '(1 2)

3

粗略地说, define-noisy 形式的工作原理是将

(define-noisy (f x y)
  (+ x y))

with

(define (f x y)
  (show-arguments 'f (list x y))
  (+ x y))

由于 show-arguments 不是由 noisy 模块提供的,然而,这种文字上的替换并不完全正确。 实际的替换正确地跟踪了像 show-arguments 这样的标识符的来源, 所以它们可以指代定义宏的地方的其他定义–即使这些标识符在使用宏的地方不可用。

宏和模块之间的交互不仅仅是标识符的绑定。 define-syntax-rule 形式本身就是一个宏,它扩展为编译时代码, 实现了从 define-noisy 到 define 的转变。 模块系统跟踪哪些代码需要在编译时运行,哪些需要正常运行,在 编译和运行时阶段 和 模块实例化和访问 中会有更多解释。
