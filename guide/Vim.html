24.3 Vim

许多 Vim 的发行版都支持 Scheme,这对 Racket 来说是非常有用的。 从 version 7.3.518 开始,Vim 检测到扩展名为 .rkt 的文件具有 scheme 文件类型。 Version 8.2.3368 增加了对 .rktd 和 .rktl 的支持。

在旧版本中,你可以通过以下方式启用 Racket 文件的文件类型检测为 scheme:

  if has("autocmd")

  autocmd filetypedetect BufReadPost *.rkt,*.rktl,*.rktd set filetype=scheme

  endif

如果你的 Vim 支持 ftdetect 系统,在这种情况下,它很可能已经足够支持 Racket 了,尽管如此, 你还是可以把下面的内容放在 "~/.vim/ftdetect/racket.vim" 中 (在 MS-Windows 中是 "$HOME/vimfiles/ftdetect/racket.vim" ;见 :help runtimepath)。

  " :help ftdetect

  " If you want to change the filetype only if one has not been set

  autocmd BufRead,BufNewFile *.rkt,*.rktl,*.rktd setfiletype scheme

  " If you always want to set this filetype

  autocmd BufRead,BufNewFile *.rkt,*.rktl,*.rktd set filetype=scheme

24.3.1 插件

另外,你也可以使用一个插件,如

  • wlangstroth/vim-racket

  • benknoble/vim-racket

两者之间的主要区别是 benknoble/vim-racket 分叉支持更多的功能,而且更新更频繁。

来实现 Racket 文件的自动检测、缩进和语法高亮功能。

这些插件通过设置基于 #lang 行的 filetype 选项来工作。 比如说:

  • 以 #lang racket 或 #lang racket/base 开头的文件, filetype 等于 racket 。

  • 以 #lang scribble/base 或 #lang scribble/manual 开头的文件, 其 filetype 等于 scribble 。

根据你的插件,像 at-exp 这样的修饰语也可能被忽略, 所以 #lang at-exp racket 仍然是 racket 的一个 filetype 。

这种方法更灵活,但可能会导致更多的工作。 由于每个 #lang 都有自己的 filetype, 选项、语法高亮和其他功能需要为每个文件类型进行配置。 这可以通过标准的 ftplugin 机制来完成。 例如参见 :help ftplugin-overrule 和 :help ftplugin: 将 <lang> 的选项放在 "~/.vim/after/ftplugin/<lang>.vim" 中 (在 MS-Windows 中为 "$HOME/vimfiles/after/ftplugin/<lang>.vim" ）。 同样,语法文件也遵循 :help syntax 中记载的标准机制。

这两个插件都作为 ftplugin 为Racket(可能还有其他 #lang) 提供了配置。 要启用它们,请使用 :filetype 命令,如 :help :filetype 中所记录的。 你可能想打开文件类型插件 (:help :filetype-plugin-on) 和文件类型缩进插件 (:help :filetype-indent-on)。

24.3.2 缩进

你可以通过设置 Vim 中的 lisp 和 autoindent 选项来启用 Racket 的缩进功能。 你需要自定义缓冲区本地的 lispwords 选项,以控制特殊形式的缩进方式。 参见 :help ’lispwords’ 。 插件 中提到的两个插件都为你设置了这个选项。

然而,缩进可能是有限的,可能不如你在 Emacs 中得到的那样完整。 你也可以使用 Dorai Sitaram 的 scmindent 来实现 Racket 代码的更好缩进。 关于如何使用缩进器的说明可以在网站上找到。

24.3.3 高亮显示

Vim 的 Rainbow Parenthesis 脚本对于更明显的括号匹配很有用。 许多平台上的 Vim 都提供了 Scheme 的语法高亮,这在很大程度上也适用于 Racket。 vim-racket 脚本为你提供了良好的默认高亮设置。

24.3.4 结构化的编辑

Slimv 插件有一 paredit 模式， 就像 Emacs 里的 paredit 工作方式。然而，插件不知道 Racket。 你可以设置 Vim 去把 Racket 作为 Scheme 文件，也可以修改 paredit 脚本以加载 ".rkt"文件。

要想获得一套更像 Vim 的键值映射,可以将

  • guns/vim-sexp

  • benknoble/vim-sexp

个键值中的任何一个配对起来。

benknoble/vim-sexp 的叉子是稍微现代的 vimscript。

与 tpope/vim-sexp-mappings-for-regular-people。 其体验与 paredit 相当,但对手指来说更舒适。

24.3.5 REPL

有许多通用的 Vim + REPL 插件。这里有几个支持 Racket 的开箱即用:

  • rhysd/reply.vim

  • kovisoft/slimv, 如果你正在使用 scheme 文件类型的话。

  • benknoble/vim-simpl

24.3.6 Scribble

Vim 对书写 Scribble 文档的支持是由以下机构提供的

  • wilbowma/scribble.vim

  • benknoble/scribble.vim

同样, benknoble/scribble.vim 的更新更频繁,也更现代一些。

24.3.7 杂项

如果你要安装许多 Vim 插件(不一定是 Racket 特有的),我们建议使用一个插件,这样可以使加载其他插件更容易。有很多插件管理器。

Pathogen 就是一个这样的插件; 使用它,你可以将新的插件解压到个人 Vim 文件的 "bundle" 文件夹下的子目录中 (Unix 下为 "~/.vim" ,MS-Windows 下为 "$HOME/vimfiles" ）,从而安装这些插件。

对于较新的 Vim 版本,你可以使用软件包系统 (:help packages)。

关于各种管理器的一个相对最新的参考资料是: What are the differences between the vim plugin managers? 。 同一个网站, Vi & Vim 是一个获得 Vimers 帮助的好地方。
