Trace: 显示函数调用的工具

Calltrace 是一个显示对用户过程的所有调用的工具。 它显示调用的参数,并缩进以显示延续的深度。

1 快速指导

  • 扔掉源代码的 ".zo" 版本

  • 在你的程序前加上

    (require trace)

    也许通过在参数前以
      racket -l trace ...
    开始 racket 来加载程序。

  • 运行程序

trace 模块是奇数;不要将它导入到另一个模块中。 相反, trace 模块应该从顶层调用,这样它就可以安装求值处理程序、异常处理程序等。

要重用 trace 的部分代码,请导入 trace/calltrace-lib 。 它包含这里描述的所有绑定,但是它不设置 current-eval 参数。

2 安装 Calltrace

 (require trace) package: trace
调用 trace 模块将计算处理程序 (通过 current-eval) 设置为检测Racket源代码。

注意: trace 对作为编译字节码(即从 ".zo" 文件)或本机代码 (即从 ".dll", ".so" 或 ".dylib" 文件)加载的代码没有影响。

可以通过 instrumenting-enabled 参数显式禁用 Calltrace 的插装功能。 默认情况下,检测处于开启状态。 instrumenting-enabled 参数只影响源代码的编译方式,而不影响异常信息的报告方式。

在写入 ".zo" 文件之前不要加载 trace 。 调用 trace 工具的 s 表达式,其值不可打印;如果将检测的 s 表达式传递给默认的 eval 处理程序, 则可以正常工作,但是s表达式及其字节码形式都不能封送为字符串。

3 Calltrace 库

 (require trace/calltrace-lib) package: trace
trace/calltrace-lib 模块提供了实现 trace 的函数。

parameter

(instrumenting-enabled) → boolean?

(instrumenting-enabled on?) → void?
  on? : any/c
确定是否启用跟踪检测的参数。

procedure

(calltrace-eval-handler e) → any

  e : any/c
适合与 current-eval 一起使用的过程, 它为 Calltrace 输出检测表达式(当没有通过 instrumenting-enabled 禁用检测时)。

要求 trace 将此过程安装为 current-eval 的值。

procedure

(annotate e) → syntax?

  e : any/c
用 Calltrace 支持检测表达式 e 。 此函数通常由 calltrace-eval-handler 使用。
