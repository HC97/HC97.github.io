13 Legacy Forms

为了向后兼容,Typed Racket 提供了以下形式。

syntax

(lambda: formals maybe-ret . body)

 
maybe-ret = 
  | : type
     
formals = ([v : t] ...)
  | ([v : t] ... v : t *)
  | ([v : t] ... v : t ooo bound)
一个形式参数 v 的函数,其中每个形式参数都有相关的类型。 如果有一个休止符参数存在,那么它的类型是 (Listof t) 。

syntax

(λ: formals maybe-ret . body)

一个使用 lambda: 的相同形式的别名。

syntax

(plambda: (a ...) formals maybe-ret . body)

(plambda: (a ... b ooo) formals maybe-ret . body)
一个多态的函数,在类型变量 a 上抽象出来。 类型变量 a 既被绑定在形式的类型中,也被绑定在 body 的任何类型表达中。

syntax

(opt-lambda: formals maybe-ret . body)

 
formals = ([v : t] ... [v : t default] ...)
  | ([v : t] ... [v : t default] ... v : t *)
  | ([v : t] ... [v : t default] ... v : t ooo bound)
一个有可选参数的函数。

syntax

(popt-lambda: (a ...) formals maybe-ret . body)

(popt-lambda: (a ... a ooo) formals maybe-ret . body)
一个有可选参数的多态函数。

syntax

case-lambda:

An alias for case-lambda.

syntax

(pcase-lambda: (a ...) [formals body] ...)

(pcase-lambda: (a ... b ooo) [formals body] ...)
一个有多个参数的多态函数。

syntax

(let: ([v : t e] ...) . body)

(let: loop : t0 ([v : t e] ...) . body)
本地绑定,如 let, 每个都有相关的类型。 在第二种形式中, t0 是 loop 结果的类型 (因此也是整个表达式的结果,以及 body 中的最终表达式)。 类型注释是可选的。

Examples:
> (: filter-even : (Listof Natural) (Listof Natural) -> (Listof Natural))
> (define (filter-even lst accum)
    (if (null? lst)
        accum
        (let: ([first : Natural (car lst)]
               [rest  : (Listof Natural) (cdr lst)])
              (if (even? first)
                  (filter-even rest (cons first accum))
                  (filter-even rest accum)))))
> (filter-even (list 1 2 3 4 5 6) null)

- : (Listof Nonnegative-Integer)

'(6 4 2)

Examples:
> (: filter-even-loop : (Listof Natural) -> (Listof Natural))
> (define (filter-even-loop lst)
    (let: loop : (Listof Natural)
          ([accum : (Listof Natural) null]
           [lst   : (Listof Natural) lst])
          (cond
            [(null? lst)       accum]
            [(even? (car lst)) (loop (cons (car lst) accum) (cdr lst))]
            [else              (loop accum (cdr lst))])))
> (filter-even-loop (list 1 2 3 4))

- : (Listof Nonnegative-Integer)

'(4 2)

syntax

(plet: (a ...) ([v : t e] ...) : t0 . body)

let: 的多态版本,对类型变量 a 进行抽象。 类型变量 a 在形式的类型和 body 中的任何类型表达式中都被绑定。 不支持let的循环形式。

syntax

(letrec: ([v : t e] ...) . body)

syntax

(let*: ([v : t e] ...) . body)

syntax

(let-values: ([([v : t] ...) e] ...) . body)

syntax

(letrec-values: ([([v : t] ...) e] ...) . body)

syntax

(let*-values: ([([v : t] ...) e] ...) . body)

letrec, let*, let-values, letrec-values 和 let*-values 的类型注解版本。 与 let: 一样,类型注解是可选的。

syntax

(let/cc: v : t . body)

syntax

(let/ec: v : t . body)

let/cc 和 let/ec 的类型注解版本。 与 let: 一样,类型注解是可选的。

syntax

(define: v : t e)

(define: (a ...) v : t e)
(define: (a ... a ooo) v : t e)
(define: (f . formals) : t . body)
(define: (a ...) (f . formals) : t . body)
(define: (a ... a ooo) (f . formals) : t . body)
这些形式定义了带有类型注解的变量。 第一种形式用类型 t 和值 e 定义 v 。 第二种形式做了同样的事情,但允许指定类型变量。 第三种形式允许多点变量。 第四、第五和第六种形式用适当的类型定义函数 f 。 在大多数情况下,使用 : 比使用 define: 更好。

Examples:
> (define: foo : Integer 10)
> (define: (A) mt-seq : (Sequenceof A) empty-sequence)
> (define: (add [first : Integer]
                [rest  : Integer]) : Integer
    (+ first rest))
> (define: (A) (poly-app [func : (A A -> A)]
                         [first : A]
                         [rest  : A]) : A
    (func first rest))

syntax

(define-struct/exec name-spec ([f : t] ...) [e : proc-t] maybe-type-name)

 
name-spec = name-id
  | (name-id parent)
     
maybe-type-name = 
  | #:type-name type-id
等同于使用 define-struct 定义一个带有 prop:procedure 属性的结构体, 该属性与 proc-t 类型的存储过程 e 一起提供。

Changed in version 1.13 of package typed-racket-lib: 已废弃

Changed in version 1.4 of package typed-racket-lib: 添加了 #:type-name 选项。
