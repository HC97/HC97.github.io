4 类型化类

警告: 本节描述的功能是实验性的,可能无法正常工作。一些功能将在下一个版本中改变。 特别是,类的类型化-非类型化交互将不会向后兼容,所以不要依赖当前的语义。

类型化的 Racket 提供了对面向对象编程的支持,由 racket/class 库提供的类和对象。

4.1 特殊形式

 (require typed/racket/class) package: typed-racket-lib

下面的特殊形式由 typed/racket/class 和 typed/racket 模块提供, 但不是由 typed/racket/base 提供。 typed/racket/class 模块额外提供了所有其他来自 racket/class 的绑定。

产生一个带有类型注释的类,允许 Typed Racket 对类中的方法、字段和其他条款进行类型检查。

类子句的含义与来自 racket/class 库的 class 形式相同,除了额外的可选类型注释。 上面的语法中没有列出的来自 class 的其他类子句形式,目前在类型化 Racket 中不支持。

Examples:
> (define fish%
    (class object%
      (init [size : Real])
  
      (: current-size Real)
      (define current-size size)
  
      (super-new)
  
      (: get-size (-> Real))
      (define/public (get-size)
        current-size)
  
      (: grow (Real -> Void))
      (define/public (grow amt)
        (set! current-size (+ amt current-size)))
  
      (: eat ((Object [get-size (-> Real)]) -> Void))
      (define/public (eat other-fish)
        (grow (send other-fish get-size)))))
> (define dory (new fish% [size 5.5]))

在一个类型化类的形式中,其中一个类子句必须是对 super-new 的调用。 不调用 super-new 将导致一个类型错误。 此外, super-new 的动态使用(例如,在类形式的动态范围内调用一个单独的函数)也受到限制。

Example:

    ; Note the missing `super-new`
    (init-field [x : Real 0] [y : Real 0]))

如果任何带有可选的类型注解的标识符没有注解,类型检查器将假定该标识符的类型为 Any (或方法为 Procedure) 。

Examples:
> (define point%
    (class object%
      (super-new)
      (init-field x y)))
> point%

- : (Class (init (x Any) (y Any)) (field (x Any) (y Any)))

#<class:point%>

当 type-variable 被提供时,该类在给定的类型变量上被参数化。 这些类型变量在类的主体中是有范围的。 产生的类可以使用 inst 在特定的类型上被实例化。

Examples:
> (define cons%
    (class object%
      #:forall (X Y)
      (super-new)
      (init-field [car : X] [cdr : Y])))
> cons%

- : (All (X Y) (Class (init (car X) (cdr Y)) (field (car X) (cdr Y))))

#<class:cons%>

> (new (inst cons% Integer String) [car 5] [cdr "foo"])

- : (Object (field (car Integer) (cdr String)))

(object:cons% ...)

初始化参数可以用 new 形式按名称提供,用 make-object 形式按位置提供, 或者用 instantiate 形式同时提供。

就像在普通的 Racket 类中,初始化参数的声明顺序决定了类类型中初始化类型的顺序。

此外,一个类也可以有一个类型化的 init-rest 子句,在这种情况下,类的构造函数按位置接受一个不受限制的参数。 init-rest 子句的类型必须是 List 类型, Listof 类型或其他任何列表类型。

Examples:
> (define point-copy%
    ; a point% with a copy constructor
    (class object%
      (super-new)
      (init-rest [rst : (U (List Integer Integer)
                           (List (Object (field [x Integer]
                                                [y Integer]))))])
      (field [x : Integer 0] [y : Integer 0])
      (match rst
        [(list (? integer? *x) *y)
         (set! x *x) (set! y *y)]
        [(list (? (negate integer?) obj))
         (set! x (get-field x obj))
         (set! y (get-field y obj))])))
> (define p1 (make-object point-copy% 1 2))
> (make-object point-copy% p1)

- : (Object (field (x Integer) (y Integer)))

(object:point-copy% ...)

这些形式大多等同于 racket/class 库中相同名称的形式,并且会扩展到它们。 然而,它们也允许在初始化参数、字段或方法名称中使用类型注释,就像上面描述的 class 的形式一样。

4.2 类型

syntax

(Class class-type-clause ...)

 
class-type-clause = name+type
  | (init init-type ...)
  | (init-field init-type ...)
  | (init-rest name+type)
  | (field name+type ...)
  | (augment name+type ...)
  | #:implements type-alias-id
  | #:implements/inits inits-id
  | #:row-var row-var-id
     
init-type = name+type
  | [id type #:optional]
     
name+type = [id type]
一个具有给定初始化参数、方法和字段类型的类的类型。

Example:
> (: food% (Class (init [liquid? Boolean])
                  (field [nutrition Integer])
                  [get-nutrition (-> Integer)]))

方法的类型要么不提供关键字,在这种情况下它们对应于公共方法, 要么提供 augment 关键字,在这种情况下它们对应于可以增强的方法。

一个初始化参数类型指定了一个名称和类型,还可以选择 #:optional 关键字。 一个带有 #:optional 的初始化参数类型对应于一个不需要在对象实例化时提供的参数。

Example:
> (: drink% (Class (init [color String]
                         [carbonated? Boolean]
                         [viscosity Positive-Real #:optional])))

初始化参数在类型中的顺序很重要,因为它决定了用于 make-object 和 instantiate 的 by-position 参数的类型。 一个给定的 Class 类型也可能只包含一个 init-rest 子句。

Examples:
> (define drink%
    (class object%
      (super-new)
      ; The order of `color' and `carbonated?' cannot be swapped
      (init color carbonated? [viscosity 1.002])))
; The order of initialization matches the order in the type
> (make-object drink% "purple" #t)

- : (Object)

(object:drink% ...)

当提供 type-alias-id 时,产生的类类型包括所有来自指定类型别名(必须是一个类类型的别名)的方法和字段类型。 这样做的目的是允许子类的类型包括其父类类型的一部分。 然而,父类的初始化参数类型 不 包括在内,因为子类不一定与它的父类共享相同的初始化参数。

初始化参数类型可以通过提供 inits-id 与 #:implements/inits 关键字从父类中包含。 除了初始化参数的行为之外,这与 #:implements 子句是相同的。 对于一个单一的 Class 类型,只能提供一个 #:implements/inits 子句。 从父类型复制的初始化参数被附加到通过 init 和 init-field 子句指定的初始化参数上。

可以为一个单一的类类型提供多个 #:implements 子句。 用于 #:implements 子句的类型将被依次合并, 并且使用给定方法名或字段的最后一个类型( Class 类型本身的类型优先)

Examples:
> (define-type Point<%> (Class (field [x Real] [y Real])))
> (: colored-point% (Class #:implements Point<%>
                           (field [color String])))

当提供 row-var-id 时,类的类型是一个抽象的类型,它是行多态的。 一个行多态的类类型可以通过使用 inst 在特定的行被实例化。 一个类类型中只能出现一个 #:row-var 子句。

syntax

ClassTop

所有类类型的超级类型。 这种类型的值不能用于子类化、对象创建或大多数其他类的功能。 它的主要用途是用于反射性操作,如 is-a? 。

syntax

(Object object-type-clause ...)

 
object-type-clause = name+type
  | (field name+type ...)
一个具有给定字段和方法类型的对象的类型。

Examples:
> (new object%)

- : (Object)

(object)

> (new (class object% (super-new) (field [x : Real 0])))

- : (Object (field (x Real)))

(object:eval:20:0 ...)

syntax

(Instance class-type-expr)

对应于 class-type-expr 的对象的类型。

这与拥有 class-type-expr 的所有方法和字段类型的 Object 类型相同。 类类型中 augment 和 init 子句的类型被忽略。

Examples:
> (define-type Point% (Class (init-field [x Integer] [y Integer])))
> (: a-point (Instance Point%))
> (define a-point
    (new (class object%
           (super-new)
           (init-field [x : Integer 0] [y : Integer 0]))))

syntax

(Row class-type-clause ...)

代表一个行,用于实例化行多态的函数类型。 接受 Class 形式接受的所有子句,除了关键字参数。

行不是类型,因此除了在 row-inst 形式中,不能在任何情况下使用。 参见 row-inst 的例子。
